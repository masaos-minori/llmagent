# 履歴圧縮の固定ターン要約から重要度ベース圧縮への変更

- 改善案:
  - 先頭 compress_turns * 2 件を固定で圧縮する方式をやめ、重要度・pin・圧縮禁止区分を持つ履歴圧縮へ変更する
    現行は古いが重要な制約や方針も機械的に圧縮対象になり得る
- 効果:
  - 長時間セッションでも、重要なユーザー指示、レビュー指摘、作業制約、失敗理由を保持したまま履歴を縮約できること。セッション再開時の品質低下も抑制できる
- リスク:
  - 重要度判定ロジックが不適切な場合、残すべき履歴を削るか、逆に圧縮効果が弱くなる
- 実装方式:
  - LLMMessage に importance または pinned 相当のメタを追加し、system、note、tool error、方針修正発話などを圧縮対象外へ分類し、残余を重み付き選別して圧縮する方式。圧縮結果とともに「失われた重要項目一覧」を別管理する方式
- 実装対象:
  - history_manager.py
  - rag_types.py 相当の LLMMessage
  - agent_session.py
  - agent_commands.py
