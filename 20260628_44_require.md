# Update `/db` help description in CommandRegistry

## Priority
High

## Labels
agent, commands, help, db, cleanup

## Dependencies
- Update `/db` command docstring to scoped-only forms

## Background
The command registry help text may still describe flat DB aliases. `/help` should advertise only the canonical scoped forms.

## Task
Update the `/db` `CommandDef` description in `registry.py`.

## Implementation Requirements
- Remove flat alias wording from the `/db` command description.
- Do not advertise the following as direct `/db <subcmd>` forms:
  - `/db urls`
  - `/db clean`
  - `/db rebuild-fts`
  - `/db recover`
- Use scoped wording such as:
  - `rag stats|urls|clean|rebuild-fts|vec-rebuild|reconcile-url|recover|consistency; session stats|health|checkpoint|vacuum|purge|recover`

## Acceptance Criteria
- `/help` documents `/db rag ...` and `/db session ...`.
- `/help` does not document flat DB aliases.
- `/db` remains registered as a prefix async command.
