# [High] Fix mdq-mcp syntax errors and incomplete server entry point

## Priority
High

## Labels
`mcp`, `mdq`, `bug`, `server`, `startup`

## Dependencies
None

## Background
Several MDQ files contain invalid Python syntax or incomplete server code. The server must be importable and startable before any production hardening can be validated.

## Task
Fix Python syntax errors and complete the `mdq-mcp` server entry point.

## Implementation Requirements
- Fix invalid future imports:
  - Replace invalid `from **future** import annotations` with `from __future__ import annotations`.
- Fix invalid dunder references:
  - Replace invalid `**name**` / `**main**` patterns with `__name__` / `"__main__"`.
- Complete the `MdqMCPServer` class.
- Complete the `if __name__ == "__main__"` entry point.
- Ensure `python -m mcp.mdq.server` starts the server.
- Ensure import of all MDQ modules succeeds.

## Acceptance Criteria
- `python -m mcp.mdq.server` starts without syntax errors.
- `pytest` can import all MDQ modules.
- The server exposes `/health`, `/v1/tools`, and `/v1/call_tool`.
- No syntax errors remain in:
  - `server.py`
  - `models.py`
  - `service.py`
  - `tools.py`
  - `parser.py`
  - `indexer.py`
  - `search.py`
