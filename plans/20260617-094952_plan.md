# Plan: Enable watchdog by default in production profiles (Req 26)

## Goal

Enable MCP watchdog-based auto-recovery by default when security_profile is PRODUCTION, and log watchdog status at startup so operators can verify it immediately.

## Scope

**In scope:**
- `scripts/agent/config_builders.py` — profile-aware default for `mcp_watchdog_interval`
- `scripts/agent/repl.py` — log watchdog enabled/disabled state at startup
- `config/agent.toml` — remove explicit `mcp_watchdog_interval = 0` (let profile default apply)
- `docs/05_agent_08_configuration.md` — document production watchdog defaults
- `docs/04_mcp_03_routing_lifecycle_and_execution.md` — document watchdog lifecycle

**Out of scope:**
- Adding a separate "production profile" config key (reuse existing `security_profile`)
- Changing watchdog behavior (interval, restart logic) — only change the default value
- Modifying `config/tools.toml` (separate tool config file)

## Assumptions

1. Production deployments set `security_profile = "production"` in their config.
2. The dataclass default of 30.0 in `MCPConfig` is a documentation artifact; the real default comes from `config_builders.py` which uses 0.0.
3. Operators who explicitly set `mcp_watchdog_interval = 0` in config should still get watchdog disabled (config overrides profile default).

## Implementation Steps

### Step 1: Profile-aware watchdog default in `config_builders.py`

In the MCPConfig construction, use `security_profile` to determine the default:

```python
# Determine watchdog default based on security profile
security_profile_val = SecurityProfile(cfg.get("security_profile", "local"))
watchdog_interval_default = 30.0 if security_profile_val == SecurityProfile.PRODUCTION else 0.0

mcp=MCPConfig(
    mcp_servers=_build_mcp_servers(cfg),
    mcp_watchdog_interval=float(cfg.get("mcp_watchdog_interval", watchdog_interval_default)),
    mcp_watchdog_max_restarts=int(cfg.get("mcp_watchdog_max_restarts", 3)),
    github_url=cfg.get("github_server_url", "http://127.0.0.1:8006"),
    security_profile=security_profile_val,
),
```

This ensures config values always override profile defaults, but the default itself is profile-aware.

### Step 2: Log watchdog status at startup in `repl.py`

Add logging after `_start_mcp_servers()` and before `_check_services()`:

```python
watchdog_interval = self._ctx.cfg.mcp.mcp_watchdog_interval
if watchdog_interval > 0:
    logger.info("Watchdog enabled: interval=%ss, max_restarts=%s",
                watchdog_interval, self._ctx.cfg.mcp.mcp_watchdog_max_restarts)
else:
    logger.info("Watchdog disabled (mcp_watchdog_interval=0)")
```

### Step 3: Update `config/agent.toml`

Remove the explicit `mcp_watchdog_interval = 0` line so the profile-aware default applies. Keep `mcp_watchdog_max_restarts = 3`.

For LOCAL profile deployments, they can explicitly set `mcp_watchdog_interval = 0` if they want watchdog disabled.

### Step 4: Update documentation

In `docs/05_agent_08_configuration.md`:
- Update the `mcp_watchdog_interval` row to show profile-aware defaults: `30.0 (PRODUCTION) / 0.0 (LOCAL)`

In `docs/04_mcp_03_routing_lifecycle_and_execution.md`:
- Add a "Production Profile" subsection under watchdog documentation
- Document that PRODUCTION enables watchdog by default with 30s interval
