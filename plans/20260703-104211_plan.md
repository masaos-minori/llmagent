# Plan: Document `command_defs_list.py` as the Canonical Source for Built-in Commands

## Goal

Add module-level docstrings to the three command-registry source files and update two developer documentation pages to make `command_defs_list.py` the explicitly documented single source of truth for `_COMMANDS`.

---

## Scope

### In-Scope

- Update module docstring of `scripts/agent/commands/command_defs.py` to state it does **not** own `_COMMANDS`
- Update module docstring of `scripts/agent/commands/command_defs_list.py` to declare it as the single source of truth for `_COMMANDS`
- Update module docstring of `scripts/agent/commands/registry.py` to state it imports `_COMMANDS` and does not define it
- Update `docs/01_overview-files.md`: correct the comment for `command_defs.py` (currently mislabelled "コマンド定義の単一ソース") and add a missing entry for `command_defs_list.py`
- Update `docs/05_agent_07_cli-and-commands.md`: expand the CommandRegistry section to document the three-module ownership model

### Out-of-Scope

- Changing any command behavior, dispatch logic, or runtime code
- Adding or removing slash commands
- `/exit` handling
- Plugin command dispatch behavior

---

## Assumptions

1. The current codebase already satisfies the acceptance criteria at the code level: `_COMMANDS` is defined only in `command_defs_list.py` (verified by grep).
2. No tests import module docstrings; docstring-only changes cannot break existing tests.
3. `docs/01_overview-files.md` and `docs/05_agent_07_cli-and-commands.md` are the canonical developer-facing documentation files for the commands subsystem.
4. No additional developer documentation files (e.g., wiki pages, README files) need to be updated beyond the two identified `docs/` files.
5. `deploy/deploy.sh` does not reference any of the affected source or documentation files (verified by grep; no matches found).

## Implementation Steps

### Step 1 — Update `command_defs.py` module docstring

**What:** Replace the existing two-line module docstring with an expanded version that explicitly states:
- Owns: `CommandDef`, `SubcommandSpec`
- Does NOT own `_COMMANDS`
- Directs readers to `command_defs_list.py` for the command list

**Completion criteria:** The module docstring contains a clear statement that `_COMMANDS` is not defined here and points to `command_defs_list.py`.

---

### Step 2 — Update `command_defs_list.py` module docstring

**What:** Expand the existing one-line docstring to add:
- Explicit declaration: this module is the single source of truth for built-in slash commands
- Instruction: all future built-in command additions must be made here
- Note: `CommandDef` class is defined in `command_defs.py`; this module only owns the list

**Completion criteria:** The module docstring contains "single source of truth" or equivalent phrasing and an ownership statement for `_COMMANDS`.

---

### Step 3 — Update `registry.py` module docstring

**What:** Expand the existing docstring to add a section:
- States: imports `_COMMANDS` from `command_defs_list`
- States: does NOT define `_COMMANDS`
- States: owns dispatch behavior

**Completion criteria:** The module docstring contains an explicit statement that `_COMMANDS` is imported (not defined here) and names the source module.

---

### Step 4 — Fix `docs/01_overview-files.md`

**What:**
1. Correct line 95: change comment for `command_defs.py` from `# CommandDef / SubcommandSpec データクラス (コマンド定義の単一ソース)` to `# CommandDef / SubcommandSpec データクラス (データクラス定義のみ; _COMMANDS は持たない)`
2. Add a new entry after `command_defs.py`:
   `├─ command_defs_list.py  # _COMMANDS: 全組み込みスラッシュコマンドの単一ソース (コマンド追加はここへ)`

**Completion criteria:** `command_defs_list.py` appears in the file tree with an ownership comment; `command_defs.py` comment no longer claims to be the single source for command definitions.

---

### Step 5 — Expand `docs/05_agent_07_cli-and-commands.md`

**What:** In the `## CommandRegistry` section (currently lines 56–64), add a sub-section immediately after the existing content titled "Module Ownership" containing a table:

| Module | Owns | Does NOT Own |
|--------|------|--------------|
| `command_defs.py` | `CommandDef`, `SubcommandSpec` dataclasses | `_COMMANDS` list |
| `command_defs_list.py` | `_COMMANDS` — single source of truth for built-in commands | Dispatch logic |
| `registry.py` | Dispatch behavior; imports `_COMMANDS` | `_COMMANDS` definition |

Also add a note: "Future command additions: add a new `CommandDef(...)` entry to `command_defs_list.py` only."

**Completion criteria:** The section contains the ownership table and the "future additions" note.
