## 8. Risks & Mitigations
- **Risk**: `test_diagnostics_fusion_mode` in `test_rag_quality_regression.py` calls `pipeline_rrf.get_diagnostics()` on a pipeline built inside a `with patch(...)` block — if the pipeline's `_cfg` attribute is not a SimpleNamespace with `use_rrf`, `get_diagnostics()` may fail → **Mitigation**: Verify `_make_pipeline()` passes `_cfg` correctly; `_make_rag_cfg()` returns a `SimpleNamespace`, which `get_diagnostics()` accesses via `self._cfg.use_rrf` — confirmed compatible.
- **Risk**: New files `test_rag_http_mode.py` and `test_rag_refiner.py` may duplicate assertions from existing files, creating maintenance burden → **Mitigation**: Explicitly document in each new file which scenarios are NOT covered by the existing files; restrict new tests to `SearchDiagnostics` field assertions and `get_diagnostics()` counter assertions only.
- **Risk**: Mocking `rag.pipeline.refine_context` in `test_rag_refiner.py` may break if the import path changes → **Mitigation**: Use the exact patch target `"rag.pipeline.refine_context"` (where it is used, not where it is defined), consistent with existing test patterns.
- **Risk**: `SemanticCache` threshold of `0.0` in `test_diagnostics_semantic_cache_hits` may cause cosine_sim to accept any non-zero vector as a hit — intentional for testing, but must use a non-zero vector → **Mitigation**: Use `[1.0] * 384` as the test vector (confirmed non-zero); document the threshold as intentionally permissive for the cache hit test.
