## Implementation Steps

1. **Add `plugin_strict` field to `ToolConfig` in `config_dataclasses.py`**
   - Add `plugin_strict: bool = False`
   - Add validation in `__post_init__` (must be boolean)

2. **Modify `load_plugins()` in `plugin_registry.py`**
   - Add `strict_mode: bool = False` parameter
   - In strict mode, re-raise the exception instead of logging and continuing
   - Improve warning message format: `"Plugin load failed (<filename>): <error_type>: <message>"`
   - Return value unchanged (count of loaded plugins)

3. **Update `_init_plugin_registry()` in `factory.py`**
   - Read `plugin_strict` from `ctx.cfg.tool.plugin_strict`
   - Pass to `plugin_registry.load_plugins(plugin_dir, strict_mode=plugin_strict)`
   - Log the loading mode: `"Plugin loading mode: {strict|fail-open}"`

4. **Add config entry in `agent.toml`**
   - Add `plugin_strict = false` under `[tool]` section

5. **Add tests for strict mode** (`test_plugin_registry.py`)
   - Test: strict mode with all plugins loading â†’ returns count, no exception
   - Test: strict mode with one broken plugin â†’ raises the original exception
   - Test: non-strict mode (default) with broken plugin â†’ logs warning, continues
   - Test: missing directory in strict mode â†’ still returns 0 (dir not found is not an error)

6. **Update documentation** (`docs/05_agent_11_extension-points.md`)
   - Document the strict mode behavior and configuration
   - Add example config snippet
