# Plan: Clarify and Expose Refiner Empty-Output Fallback Path (req_35)

## 1. Goal

Document the specific conditions under which `_augment_refiner()` returns empty output
and falls back to raw chunks; make that fallback visible to operators without requiring
`--debug` or log inspection.

## 2. Scope

**In-Scope:**
- Document the empty-output conditions in `pipeline_refiner.py:refine_context()` docstring
- Document empty-output vs exception fallback in `docs/03_rag_03_query_pipeline.md`
- Emit a `logger.info` in `pipeline.py:augment()` after refiner fallback (both paths),
  making the event visible at INFO log level without `--debug`
- Add a user-visible hint in `/rag search` output when refiner fallback occurred:
  write a warning line after context is printed when `stage_results` shows
  `Refiner: fallback`

**Out-of-Scope:**
- Removing refiner fallback behavior
- Redesigning the refiner model or prompt
- Changing `_extract_chat_content()` behavior
- Adding retry logic

## 3. Confirmed Facts

**Empty-output path (confirmed):**
- `RagLLM.refine_context()` calls `_extract_chat_content(response)` which returns
  `content.strip()` — can return `""` when LLM returns empty or whitespace-only content
- `pipeline_refiner.refine_context()` checks `if refined:` — empty string is falsy → falls through
- Produces `RefineResult(text=None, reason="refiner_returned_empty")`
- `logger.warning("Refiner returned empty output; falling back to chunks")` is emitted (line 101)

**Exception path (confirmed):**
- `httpx.HTTPStatusError`, `httpx.RequestError`, `ValueError` caught → `RefineResult(text=None, reason="refiner_exception: {e}")`
- `logger.warning("Refiner failed, falling back to original chunks: %s", e)` is emitted (line 104)

**Stage tracking (confirmed):**
- `pipeline.py:augment()` adds `StageResult(status="fallback", fallback_reason=refined.reason)`
  to `last_stage_results`
- `/rag search --debug` shows `~ Refiner: fallback — refiner_returned_empty` via `_print_rag_debug()`
- `get_diagnostics()` includes `fallback_reasons` list

**Gap confirmed:**
- Without `--debug`, refiner fallback produces no user-visible signal
- The distinction between `"refiner_returned_empty"` and `"refiner_exception: ..."` exists
  in code but is NOT documented in `docs/03_rag_03_query_pipeline.md`
- Specific trigger conditions for empty output are not documented

## 4. Assumptions

1. `RagLLM.refine_context()` can return `""` only when:
   (a) LLM response `content` field is `""` or whitespace-only
   (b) This can happen due to: content policy enforcement, LLM hallucinating empty,
       prompt format producing no extractable key points, or token limit edge cases
2. `_extract_chat_content()` raises `ValueError` on malformed response → always goes
   to exception path, never empty-output path
3. Adding `logger.info` in `augment()` (in addition to the `logger.warning` already in
   `refine_context()`) is appropriate — the `augment()`-level log captures which refine
   path was chosen by the pipeline (useful for tracing without `--debug`)
4. Writing a hint in `/rag search` output (not just `--debug`) requires checking
   `pipeline.last_stage_results` after `augment()` returns

## 7. Implementation Steps

1. **Update `refine_context()` docstring in `pipeline_refiner.py`**
   - Add to "LLM returned empty/falsy output" row:
     `Trigger: LLM response content is "" or whitespace-only after strip(). Can occur due to
     content policy, empty generation, or prompt producing no key points.`

2. **Add `logger.info` in `pipeline.py:augment()` after Refiner block**
   - After `self.last_stage_results.append(StageResult(stage_name="Refiner", ...))`:
     ```python
     if refined.text is None:
         logger.info(
             "augment: refiner fallback (reason=%s); using raw chunks", refined.reason
         )
     ```

3. **Add refiner fallback hint in `_cmd_rag()` in `cmd_ingest.py`**
   - After `self._out.write(context)`, before the `if debug:` block:
     ```python
     refiner_fb = next(
         (sr for sr in pipeline.last_stage_results
          if sr["stage_name"] == "Refiner" and sr["status"] == "fallback"),
         None,
     )
     if refiner_fb:
         self._out.write(
             f"[warn] refiner fallback: {refiner_fb['fallback_reason']}"
             " — raw chunks used (run with --debug for stage details)"
         )
     ```

4. **Update `docs/03_rag_03_query_pipeline.md`**
   - Add a "Refiner fallback reasons" table:
     ```
     | reason                 | Condition |
     |------------------------|-----------|
     | refiner_returned_empty | LLM returned "" or whitespace-only content |
     | refiner_exception: {e} | HTTP, transport, or ValueError raised during LLM call |
     ```
   - Add note: "Both reasons visible via `--debug`, `get_diagnostics()`, and logger.info"

5. **Tests**
   - `test_pipeline_refiner_fallback.py`:
     - Mock `RagLLM.refine_context()` returning `""` → `reason="refiner_returned_empty"`
     - Mock raising `httpx.HTTPStatusError` → `reason="refiner_exception: ..."`
   - `test_cmd_rag_refiner_hint.py`:
     - Verify `_cmd_rag()` writes hint when `last_stage_results` has Refiner:fallback

## 9. Risks

- **Risk:** Refiner hint in `/rag search` output appears even when user does not want it
  → **Mitigation:** Use `[warn]` prefix consistent with other warning lines; non-breaking for scripts
    since it's a new non-result line
- **Risk:** `logger.info` in `augment()` duplicates `logger.warning` in `refine_context()` —
  two log lines for same event
  → **Mitigation:** Different level (info vs warning) and different message focus:
    `refine_context` warns about the empty condition; `augment` logs the pipeline decision
    to use raw chunks — not duplicate, but complementary context
