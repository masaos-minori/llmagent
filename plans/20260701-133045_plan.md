# MCP stdio transport removal — Work Plan

## Goal

MCP runtime supports HTTP transport only; all stdio transport code paths are removed from the active implementation.

## Scope

**In-Scope:**
- Remove `TransportType.STDIO` from `shared/mcp_config.py`
- Remove stdio-only enums: `StartupMode.ONDEMAND`, `HealthcheckMode.PING_TOOL`, `HealthcheckMode.PROCESS`
- Remove stdio-only config fields: `cmd`, `working_dir`, `env`, `idle_timeout_sec`
- Remove `StdioTransport` class from `shared/tool_executor.py`
- Delete `agent/stdio_lifecycle.py`
- Remove `StdioLifecycleProtocol` and `restart_stdio()` from `agent/lifecycle_protocol.py`
- Remove stdio branches from `agent/factory.py` (`_ServerLifecycleRouter`)
- Remove stdio watchdog and tool-list probing from `agent/repl_health.py`
- Remove `__list_tools__` support from `mcp/server.py`
- Remove `run_stdio()` and `--stdio` entrypoint handling from `mcp/server.py`
- Update all MCP server scripts to remove `run_stdio()` calls
- Update `scripts/checks/check_no_compat.py` to remove ALLOWED_STDIO_PATTERNS
- Update tests: remove stdio-specific test cases, update remaining tests

**Out-of-Scope:**
- `agent/config.py` re-export stub cleanup
- `McpServerConfig._cast_enums()` compatibility shim removal
- MCP documentation cleanup
- `_MCP_TOOLS` to `TOOL_LIST` migration
- Audit format migration

## Assumptions

1. All current MCP servers are configured with `transport: http` in mcp_servers.toml
2. No new stdio-based MCP servers will be added in the near future
3. The `_cast_enums()` compatibility shim can be removed independently (tracked separately)
4. HTTP-only healthcheck mode is sufficient for all MCP servers
