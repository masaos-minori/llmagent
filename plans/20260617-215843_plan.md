# Plan: Return structured plugin load results instead of only an integer (req 65)

## Goal

Change `load_plugins()` to return a structured result containing loaded count, failed plugin list with paths and error messages, so callers can check plugin loading programmatically instead of only parsing logs.

## Scope

**In scope:**
- `shared/plugin_registry.py` — change `load_plugins()` return type from `int` to a structured result object; track failures during the import loop
- `tests/test_plugin_registry.py` — update existing tests that assert on integer return value; add new tests for failure detail fields

**Out of scope:**
- Changes to `agent/factory.py` caller (only needs one line adjustment to extract count from struct)
- Adding new config files or changing plugin loading behavior (fail-open semantics stay the same)
- Changes to `test_agent_factory.py` (mocks `load_plugins`, unaffected by return type change)
- Changes to other layers' handling of plugin results

## Assumptions

1. The fail-open behavior stays unchanged — failed plugins are still logged and skipped, not re-raised (unless `strict_mode=True`).
2. The structured result should be a simple dataclass or TypedDict, not an enum-based sum type.
3. Backward compatibility: the existing caller in `agent/factory.py` uses the return value only for logging (`n_plugins`), so extracting `.loaded_count` is a minimal change.
4. No new dependencies are needed — the result object will use only stdlib types.

## Unknowns

| Unknown | Evidence missing | Resolution | Blocking |
|---|---|---|---|
| Result object type: dataclass vs TypedDict vs namedtuple | Both are valid; dataclass supports mutability (not needed here), TypedDict is minimal, namedtuple is backward-compatible with tuple unpacking | Use a **frozen dataclass** (`PluginLoadResult`): it is explicit, serializable for logging/debugging, and supports attribute access (`.loaded_count`, `.failed`). TypedDict would work but offers less IDE support for field names. namedtuple could allow `n, failed = result` but is less clear semantically. | no |
| Whether to include `loaded_count` as a separate field or derive it from `len(loaded_names)` | Both are equivalent; a separate field makes the API more explicit and avoids requiring callers to compute length | Include both: `loaded_count: int` (for quick access) and `failed: list[PluginFailure]` (structured failure details). The loaded names are already registered in `_tools`, `_commands`, etc., so listing them separately would be redundant. | no |
| Whether the result object should include a `success` boolean field | `loaded_count > 0 or failed` is implicit; a separate field could be misleading if "success" means "all loaded successfully" vs "some loaded" | Omit a top-level `success` field. Callers check `len(result.failed) == 0` for full success, or `result.loaded_count > 0` for partial success. | no |

## Design

### New types

```python
@dataclass(frozen=True)
class PluginFailure:
    path: str            # Plugin file name (e.g., "bad_plugin.py")
    error: str           # Full error string: "PluginLoadError(<ExceptionType>): <message>"


@dataclass(frozen=True)
class PluginLoadResult:
    loaded_count: int                    # Number of plugins successfully imported
    failed: tuple[PluginFailure, ...]    # Ordered list of failures (import order)
```

### Updated `load_plugins()` signature and behavior

```python
def load_plugins(
    plugin_dir: str | Path,
    *,
    known_tools: frozenset[str] = frozenset(),
    override_policy: str = "reject",
    strict_mode: bool = False,
) -> PluginLoadResult:
    """Import all *.py files from plugin_dir; returns PluginLoadResult with success/failure details."""
    ...
    failures: list[PluginFailure] = []
    for py_file in sorted(plugin_path.glob("*.py")):
        try:
            # ... import logic (unchanged) ...
            loaded += 1
        except (ImportError, SyntaxError, AttributeError, RuntimeError) as e:
            failures.append(PluginFailure(
                path=py_file.name,
                error=f"Plugin load failed ({py_file.name}): {type(e).__name__}: {e}",
            ))
            if strict_mode:
                raise
            logger.warning(failures[-1].error)

    _validate_tool_conflicts(known_tools, override_policy)
    return PluginLoadResult(loaded_count=loaded, failed=tuple(failures))
```
