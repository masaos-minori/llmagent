## Goal

`docs/04_mcp_*.md` のソースオブトゥルース記述を cleanup ポリシーと一致させ、削除済み機能の履歴・互換注記・旧フォーマット記述を active docs から除去する。

## Scope

**In**:
- `docs/04_mcp_*.md` の文言修正。
- 削除済みエンドポイント履歴の除去（`POST /v1/search`）。
- 解決済み互換注記の除去（`mdq.db` → `mdq.sqlite`）。
- stdio transport セクションの除去（上流実装完了後）。
- `_MCP_TOOLS` → `TOOL_LIST` 移行完了後の 04_mcp_07 更新。
- audit log フォーマット記述の統一（コード変更完了後）。

**Out**:
- `scripts/` 配下のコード変更（それは 20260701_38/39 の実装が担う）。
- `mcp/audit.py` の JSON-lines 対応（別途新規計画が必要、UNK-03 参照）。
- `04_mcp_90_inconsistencies_and_known_issues.md` の全面廃止（スコープ外）。

## Assumptions

1. 計画 20260701_38 実装後に stdio transport が全ファイルから削除される。
2. 計画 20260701_39 実装後に `_MCP_TOOLS` が `TOOL_LIST` に統一される。
3. `AUDIT session=` key=value フォーマットは現行の実装フォーマット（`mcp/audit.py`）と一致しており、ドキュメントだけ書き換えると実装と乖離する。
4. `POST /v1/search` 削除履歴と `mdq.db` 互換注記は他の前提計画を待たず即時削除できる。
