# Plan: テストヘルパー関数のリファクタリング — 4ファイル

## Goal

テストヘルパー関数の複雑さを軽減し、`dataclasses.replace()` による部分更新パターンを導入する。
11 引数を持つ `_write_chunk()` を最優先にリファクタリングし、他のヘルパーはキーワード引数・デフォルト値の整理に留める。

5. `dataclasses.replace()` を使う場合は `@dataclass(frozen=True)` が適切。

## Design

### _ChunkSpec dataclass の導入 (test_ingester.py)

```python
import dataclasses

@dataclasses.dataclass(frozen=True)
class _ChunkSpec:
    url: str = "https://example.com/doc"
    title: str = "Test Doc"
    lang: str = "ja"
    content: str = "本文"
    normalized_content: str | None = "normalized"
    chunk_index: int = 0
    chunk_type: str = ""
    source_file: str = ""
    chunking_strategy: str = "heading"
    etag: str | None = None
    last_modified: str | None = None

_DEFAULT_CHUNK = _ChunkSpec()

def _write_chunk(dest_dir: Path, filename: str, spec: _ChunkSpec = _DEFAULT_CHUNK) -> Path:
    dest_dir.mkdir(parents=True, exist_ok=True)
    path = dest_dir / filename
    path.write_bytes(orjson.dumps(dataclasses.asdict(spec)))
    return path
```

既存の呼び出しは `_write_chunk(tmp_path / "chunk", "c.txt", chunk_index=7)` 形式から
`_write_chunk(tmp_path / "chunk", "c.txt", dataclasses.replace(_DEFAULT_CHUNK, chunk_index=7))` 形式に変更する。

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `dataclasses.asdict()` が `orjson.dumps()` と非互換の型を含む | low | `_ChunkSpec` の全フィールドは `str / int / None` のみ |
| `chunk_type` / `source_file` が既存のテストケースで期待値と不一致 | low | `_ChunkSpec` のデフォルト値を `""` とし、既存テストのアサーションを確認 |
| `_write_chunk` の呼び出し側変更漏れ | medium | `grep -n "_write_chunk" tests/test_ingester.py` で全箇所を確認してから変更 |
