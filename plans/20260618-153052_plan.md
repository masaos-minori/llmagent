# Plan: Separate Diagnostic Data from Normal Conversation History

Derived from `01_plan.md` using requirement `requires/20260618_28_require.md`.

---

## Goal

Normal conversation history is clean; diagnostic data lives in a separate queryable channel. Debugging and replay still work.

---

## Scope

**In scope:**
- `scripts/agent/session.py` тАФ refactor `save_diagnostic()` to write to a dedicated diagnostic store
- `scripts/agent/session_message_repo.py` тАФ remove `_diagnostic` param; `fetch_messages()` no longer needs `WHERE role != 'diagnostic'`
- `scripts/agent/orchestrator.py` тАФ update `_handle_llm_transport_error()` call to use new diagnostic API
- `scripts/agent/repl.py` тАФ unify `_persist_session_diagnostics()` with the new diagnostic store
- `scripts/db/helper.py` тАФ optional: add diagnostic table schema / migration
- `docs/05_agent_04_state-and-persistence.md` тАФ update diagnostic storage description

**Out of scope:**
- Mid-turn synthetic tool message pattern (Pattern B тАФ `ErrorInjectionService`) тАФ design decision to change later
- `diagnostics.jsonl` format changes

---

## Assumptions

1. A dedicated `session_diagnostics` table (or equivalent) is acceptable from a schema migration cost perspective.
2. Existing diagnostic data in the `messages` table can be migrated or left in place (read-only path for old data).
3. `save_diagnostic()` is called only once per session (orchestrator.py:341) тАФ low call volume.

## Implementation Steps

1. **Confirm all callers of `save_diagnostic()`**
   - `grep -r save_diagnostic scripts/` тАФ confirm single caller
   - `grep -r _persist_session_diagnostics scripts/` тАФ confirm single caller
   - Criterion: full inventory of current usage

2. **Create `DiagnosticStore` module**
   - New file: `scripts/agent/diagnostic_store.py`
   - Methods: `save(session_id, kind, content)`, `fetch(session_id)`, `fetch_all(limit)`
   - Underlying: `session_diagnostics` table (id, session_id, kind, content, created_at)
   - Criterion: independent of `SessionMessageRepository`

3. **Migrate `save_diagnostic()` to delegate to `DiagnosticStore`**
   - Orchestrator writes to `DiagnosticStore` instead of `session.save_diagnostic()`
   - Optionally keep old `save_diagnostic()` as a pass-through for backward compat
   - Criterion: `fetch_messages()` no longer needs `WHERE role != 'diagnostic'`

4. **Unify `repl.py` diagnostics.jsonl path**
   - `_persist_session_diagnostics()` writes to `DiagnosticStore` as well
   - Criterion: single path for all diagnostics

5. **Add DB migration**
   - Add `session_diagnostics` table to schema
   - Criterion: migration script exists and is idempotent

6. **Document**
   - Update `docs/05_agent_04_state-and-persistence.md` with new diagnostic storage design
   - Criterion: clear separation model

7. **Lint, type, arch, test validation**
   - `ruff check scripts/` тАФ 0 errors
   - `mypy scripts/` тАФ no new errors
   - `lint-imports` тАФ 0 violations
