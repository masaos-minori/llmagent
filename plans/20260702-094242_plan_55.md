# Plan: Fix deploy/init_db.sh to use the unified schema initialization entrypoint

Source: `requires/20260701_55_require.md`

---

## Goal

`deploy/init_db.sh` を修正し、全 DB（rag/session/workflow/eventbus）を `python -m db.create_schema` 経由で初期化するようにする。

---

## Scope

**In-Scope**
- `deploy/init_db.sh` から `EVENTBUS_SCHEMA` 変数と `eventbus/schema.sql` チェック・ロードを削除
- `python -m db.workflow_schema` の呼び出しが存在する場合は削除
- `python -m db.create_schema` による一括初期化に統一
- `eventbus.sqlite` のテーブル確認コメント追加
- shellcheck で検出されるシェル構文エラーを修正（`$/...` などの壊れた変数参照）
- `uv run` の維持（現在のスクリプトでも使用済み）

**Out-of-Scope**
- `scripts/eventbus/schema.sql` の削除（`create_schema.py` 統合完了後に別途）
- Event Bus ランタイム統合

---

## Assumptions

1. 現在の `init_db.sh` には `EVENTBUS_SCHEMA` 変数はあるが、変数参照の壊れたパターン（`$/...`）はない（要件説明より）。ただしスクリプトの `sqlite3 ${EVENTBUS_DB} ".read ${EVENTBUS_SCHEMA}"` ブロックが削除対象。
2. `python -m db.create_schema` が 53番（`create_eventbus_schema` 追加）完了後に全 4 DB を初期化することを前提とする。
3. `uv run python` の使用は既存スクリプトと同様に維持する。
