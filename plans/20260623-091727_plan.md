# Plan: CI/CD・GitHub 書き込みデフォルトの安全化

**Source:** `requires/20260622_54_require.md`
**Created:** 2026-06-23 09:17:27

---

## Goal

CI/CD の `workflow_allowlist` 空時の fail-open を本番モードで遮断し、GitHub の `allow_force_push` / `require_pr_review` デフォルトを安全側へ反転させる。起動時 audit でオペレーターが設定を確認できるようにする。

---

## Scope

**In-Scope:**
- `scripts/mcp/github/models_config.py` — `GitHubConfig` の Python デフォルト変更 (`allow_force_push: True→False`, `require_pr_review: False→True`) および `from_dict()` フォールバック変更
- `config/github_mcp_server.toml` — デプロイ済み TOML の明示的デフォルト変更
- `scripts/agent/repl_health.py` — `audit_security_defaults()` に cicd `workflow_allowlist` 空チェック (production_mode=True で RuntimeError) と GitHub 書き込み設定の可視化警告を追加
- `docs/04_mcp_04_server_catalog.md` — github-mcp・cicd-mcp セクションのデフォルト値記述を更新

**Out-of-Scope:**
- `scripts/mcp/cicd/service_guards.py` の `_assert_allowed_workflow()` 動作変更（エージェント側 audit で代替）
- firejail 以外の sandbox 技術変更
- GitHub / CI/CD サーバーアーキテクチャ変更

---

## Assumptions

1. `CiCdService._assert_allowed_workflow()` の fail-open 動作自体は変更しない。本番強制は agent の `audit_security_defaults()` で行う（sandbox 計画と同パターン）。
2. GitHub の `allow_force_push` / `require_pr_review` は warn のみ（RuntimeError は raise しない）。要件は「startup audit で明示」のみ要求しており、GitHub 設定違反を起動失敗扱いにするとは読めない。
3. TOML に明示的な値がある場合は Python デフォルトを上書きするため、既存の明示 opt-in は影響を受けない（`allow_force_push = true` と書かれていれば引き続き有効）。

## Design

### 1. `GitHubConfig` デフォルト変更 (`models_config.py`)

**現状:**
```python
allow_force_push: bool = True
require_pr_review: bool = False
# from_dict():
allow_force_push=bool(d.get("allow_force_push", True)),
require_pr_review=bool(d.get("require_pr_review", False)),
```

**修正後:**
```python
allow_force_push: bool = False
require_pr_review: bool = True
# from_dict():
allow_force_push=bool(d.get("allow_force_push", False)),
require_pr_review=bool(d.get("require_pr_review", True)),
```

### 2. `github_mcp_server.toml` 変更

**現状:**
```toml
allow_force_push = true
require_pr_review = false
```

**修正後:**
```toml
allow_force_push = false
require_pr_review = true
```

### 3. `audit_security_defaults()` への追加 (`repl_health.py`)

**cicd workflow_allowlist チェック (新規):**
```python
# Check cicd workflow_allowlist (fail-open — empty = allow all workflows)
try:
    from mcp.cicd.models import CicdConfig
    cicd_cfg = CicdConfig.load()
    if not cicd_cfg.workflow_allowlist:
        msg = "cicd.workflow_allowlist is empty (fail-open: all workflows allowed)"
        if production_mode:
            raise RuntimeError(
                f"Production mode requires explicit workflow_allowlist. {msg}"
            )
        logger.warning("Security: %s", msg)
        warnings.append(f"Security: {msg}")
except RuntimeError:
    raise
except Exception:
    pass
```

**GitHub 書き込み設定の可視化 (新規):**
```python
# Surface GitHub write settings
try:
    from mcp.github.models_config import GitHubConfig
    gh_cfg = GitHubConfig.load()
    if gh_cfg.allow_force_push:
        msg = "github.allow_force_push=true (force push and rebase merge permitted)"
        logger.warning("Security: %s", msg)
        warnings.append(f"Security: {msg}")
    if not gh_cfg.require_pr_review:
        msg = "github.require_pr_review=false (PR merge without review permitted)"
        logger.warning("Security: %s", msg)
        warnings.append(f"Security: {msg}")
except Exception:
    pass
```
