# Plan: [High][eventbus][dlq] リトライ上限到達時のインライン DLQ 昇格

Source: `requires/20260625_26_require.md`

## Goal

`delivery_failure_count` がリトライ上限に達した瞬間に DLQ 昇格をインラインで実行し、60秒待機を不要にする。バックグラウンドループはスイープ兼リカバリー用途として残す。

## Scope

**In-Scope**
- `scripts/eventbus/app.py` の nack ハンドラーにインライン昇格チェックを追加
- `scripts/eventbus/dlq.py` の `promote_to_dlq()` を単一イベント昇格にも使えるよう汎用化
- バックグラウンドループはリカバリー目的で残す (昇格ロジック自体の重複は避ける)
- インライン昇格のテスト追加

**Out-of-Scope**
- バックグラウンドループの廃止
- スキーマ変更 (req #28 の対象)
- コンシューマー outcome API (req #25 の対象)

## Assumptions

1. req #24 / req #25 が実装済みで nack ハンドラーと `delivery_failure_count` が存在する
2. インライン昇格は nack エンドポイント内の同一トランザクション (またはスレッドロック保持中) で行う
3. バックグラウンドループは `delivery_failure_count >= max_retry AND dlq_at IS NULL` の残留を拾うスイープとして継続稼働

## Design

**nack ハンドラーのインライン昇格ロジック:**

```python
@app.post("/events/{event_id}/nack")
async def nack_event_endpoint(event_id: str) -> dict:
    def _nack_and_maybe_promote() -> dict:
        with get_db_lock():
            count = nack_event(_db, event_id)  # delivery_failure_count += 1
            if count < 0:
                return {"found": False}
            assert _cfg is not None
            if count >= _cfg.max_retry:
                promoted = promote_single(_db, _cfg.deadletter_dir, event_id)
            else:
                promoted = False
            return {"found": True, "failure_count": count, "promoted": promoted}
    result = await asyncio.to_thread(_nack_and_maybe_promote)
    if not result["found"]:
        raise HTTPException(status_code=404, detail="event not found")
    return {"event_id": event_id, "failure_count": result["failure_count"], "promoted": result["promoted"]}
```

**dlq.py に `promote_single()` を追加:**

```python
def promote_single(db: sqlite3.Connection, deadletter_dir: str, event_id: str) -> bool:
    """Promote one event immediately. Returns True if promoted (False if already in DLQ or not found)."""
    row = db.execute(
```

## Implementation Steps

1. `dlq.py` に `promote_single()` を追加
2. `app.py` の nack ハンドラーに `if count >= max_retry: promote_single(...)` を追加
3. `db.py` の `nack_event()` が failure_count を返すことを確認
4. テスト: nack × max_retry 回で即時 DLQ 昇格されること、バックグラウンドループが後続スイープで重複昇格しないことを検証
5. テスト通過確認

| Risk | Likelihood | Mitigation |
|---|---|---|
