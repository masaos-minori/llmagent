# Plan: H-4 — Make ensure_ready actually start missing subprocess MCP servers

## Goal

Update `_ServerLifecycleRouter.ensure_ready()` so that subprocess-mode HTTP MCP servers are started (via `_http_mgr.start()`) when the process is missing or not running, while persistent-mode servers and the shutdown guard remain unaffected.

## Scope

**In**: `scripts/agent/factory.py` — `_ServerLifecycleRouter.ensure_ready()`.

**Out**: No changes to `http_lifecycle.py` (existing `start()` is reused), `context.py`, `LifecycleManagerProtocol`, config, or DB schema.

## Assumptions

1. `ensure_ready()` is always called from an async context (it is already `async def`).
2. `HttpServerLifecycleManager.start()` is idempotent for already-running processes (confirmed by existing code: line 97–103).
3. Enum comparisons for `TransportType.HTTP` and `StartupMode.SUBPROCESS` are preferred over string literals (per requirement).
4. `_shutting_down` guard (H-3) must be respected; this plan assumes H-3 is implemented first, or the guard check is included here.

## Implementation Steps

1. **Inspect `shared/mcp_config.py` for `TransportType` and `StartupMode`**
   - Confirm whether they are enums or string literals.
   - Update comparisons in `ensure_ready()` accordingly.
   - Criterion: comparison uses the canonical type (enum or string, whichever is authoritative).

2. **Update `ensure_ready()` to call `start()` when needed**
   - After `verify_running()` returns `False`: call `await self._http_mgr.start(server_key, cfg)`.
   - Keep the `_shutting_down` guard check as the first statement (from H-3, or add it here if H-3 not yet merged).
   - Keep no-op for persistent-mode and unknown server keys.
   - Criterion: `ensure_ready()` calls `start()` iff `verify_running()` returns False for a subprocess-mode server.

3. **Add tests**
   - Test: `verify_running()` returns False → `start()` is called.
   - Test: `verify_running()` returns True → `start()` is NOT called.
   - Test: persistent-mode server → `start()` is NOT called.
   - Test: unknown server key → no error, no start.
   - Test: `_shutting_down=True` → `start()` is NOT called.
   - Criterion: `uv run pytest tests/test_agent_factory.py -v` passes.
