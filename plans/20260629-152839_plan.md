## 1. Goal
Add regression tests that lock down five cross-cutting design inconsistencies: compressed history reload, diagnostic persistence, `/undo` + `/tool show` artifact consistency, branch-aware memory retrieval, and JSONL config resolution.

## 2. Scope
- **In-Scope**:
  - New test file: `tests/test_regression_history_compress_reload.py` (compression → session reload consistency)
  - New test file: `tests/test_regression_diagnostic_persist.py` (diagnostic persistence and restore-path)
  - New test file: `tests/test_regression_undo_artifact.py` (`/undo` + `/tool show` artifact consistency)
  - New test file: `tests/test_regression_memory_branch.py` (branch-aware memory retrieval)
  - New test file: `tests/test_regression_jsonl_config.py` (JSONL config resolution and naming)
- **Out-of-Scope**:
  - General performance / load testing
  - Broad unrelated CLI test expansion
  - Modifying any production source file

## 3. Assumptions
- All existing tests pass before this work begins (`uv run pytest` is green).
- `SQLiteHelper` is patchable via `unittest.mock.patch` as used in existing tests (e.g., `test_diagnostic_store.py`, `test_memory_retriever.py`).
- `HistoryManager.compress()` is the primary API for triggering compression; `restore_session()` in `agent/services/session_restore.py` is the primary API for reload.
- `session.undo_last_turn()` in `agent/session.py` and `undo_last_turn()` in `agent/services/undo_service.py` are the paired APIs under test.
- `HybridRetriever.search()` and `top_semantic()` in `agent/memory/retriever.py` are the primary APIs for branch-aware retrieval.
- `JsonlMemoryStore` in `agent/memory/jsonl_store.py` and `AgentConfig.memory.memory_jsonl_dir` in `agent/config_dataclasses.py` are the JSONL config APIs.
- Tests use in-memory SQLite (as in `test_diagnostic_store.py`) or `tmp_path` fixtures (as in `test_jsonl_store.py`) — no real DB or filesystem side effects.

## 8. Risks & Mitigations
- **Risk**: UNK-04 branch filter behavior is by design but undocumented → test documents wrong expectation → **Mitigation**: Mark Scenario B and C with `# Needs confirmation` comment; use `pytest.mark.xfail(strict=False)` if behavior is ambiguous; update after design decision.
- **Risk**: `HistoryManager.compress()` requires an async LLM call → test complexity → **Mitigation**: Use `AsyncMock` as in `test_history_manager.py`; reuse `_make_manager()` helper pattern.
- **Risk**: `AgentSession.undo_last_turn()` uses real SQLiteHelper → test needs full DB setup → **Mitigation**: Use `_FakeSQLiteHelper` pattern from `test_diagnostic_store.py`; patch `SQLiteHelper` at `agent.session.SQLiteHelper`.
- **Risk**: JSONL filename wiring (UNK-05) is in `factory.py` which has complexity C (11) and is frequently modified → **Mitigation**: Read factory before writing test; do not hard-code internal path logic; test only the public contract (config field → file path segment).
