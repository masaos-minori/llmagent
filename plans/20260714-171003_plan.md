# Work Plan: Clarify MDQ Production-Ready Scope and Hybrid Search Status

## Goal

Make it explicit that MDQ is production-ready only for FTS5-based Markdown structural search, while hybrid embedding/vector search must not be presented as production-ready since it is stubbed and returns empty results.

## Scope

**In-Scope:**
- Inspect MDQ implementation to confirm hybrid/vector search is not functional
- Document production-ready scope limited to FTS5 indexing, Markdown structural search, outline, get_chunk, grep_docs, refresh/index operations, stats/admin consistency operations
- Document non-production-ready scope: hybrid semantic search, embedding-backed vector search
- Update guidance language across all affected documents

**Out-of-Scope:**
- Implementing MDQ hybrid search (Non-Goal)
- Changing MDQ database schema (Non-Goal)
- Changing RAG behavior (Non-Goal)
- Changing routing (Non-Goal)

## Assumptions

- This is primarily a documentation task; no source code changes are required unless inspection reveals that the `use_embedding=true` path has been partially implemented beyond what's documented.

## Unknowns

- Does `use_embedding=true` produce any meaningful vector search results at all? Need to verify the `_search_vector` stub behavior.
- Is there any test coverage for `use_embedding=true`? Check `tests/test_mdq*.py`.

## Affected areas

- MDQ document guide
- MDQ server catalog section
- Security and safety model (fail-open/fail-closed for MDQ)
- Known issues/inconsistencies documentation

## Design

### Step 1: Confirm hybrid search is stubbed

From grep analysis, the following confirms hybrid search is not functional:

1. **`scripts/mcp_servers/mdq/search.py:151-157`**: `_search_vector()` function contains:
   ```python
   # Placeholder: actual embedding generation and search would require an LLM API call
   # For now, return empty results — hybrid search requires embedding model integration
   logger.info("MDQ hybrid search mode requested but embeddings not yet implemented")
   return []
   ```
   This returns an empty list regardless of input.

2. **`scripts/mcp_servers/mdq/tools.py:35`**: Tool description explicitly states `"bm25/grep (hybrid is not yet supported)"`.

3. **`scripts/mcp_servers/mdq/service.py:88-92`**: Config fields exist (`use_embedding`, `vector_table`, `embedding_model`, `embedding_dims`) but they're only used for DB schema creation, not for actual search execution.

4. **DB schema** (`db_schema.py:135-156`): Vector table is created conditionally when `use_embedding=True`, but the table is never populated or queried meaningfully.

Conclusion: Hybrid/vector search is confirmed as stubbed/not implemented.

### Step 2: Define production-ready vs non-production-ready scopes

**Production-ready scope:**
- FTS5 indexing and full-text search
- Markdown structural search (headings, sections, outlines)
- `outline` tool
- `get_chunk` tool
- `grep_docs` tool
- `refresh_index`, `index_paths` tools
- `stats` tool
- Admin consistency operations (`fts_consistency_check`, `fts_rebuild`) — if implemented

**Non-production-ready scope:**
- Hybrid semantic search (FTS5 + vector fusion via RRF)
- Embedding-backed vector search
- Any feature that depends on `use_embedding=true`
