5. `repl.py` の `ctx.services.memory.on_session_start()` / `on_session_stop()` は現在 AttributeError を引き起こす状態であり、修正対象とする。

## Design

### D-1. `repl.py` と `MemoryServices` のインタフェース不整合の解消

**現状**: `repl.py` line 274 が `ctx.services.memory.on_session_start(session_id)` を呼び出すが、`MemoryServices` はこのメソッドを持たない。

**方針**: `MemoryServices` に facade メソッドを追加する。
- `on_session_start(session_id) → list[str]` : `self.injection.on_session_start(session_id)` に委譲
- `async on_session_stop(session_id, history, turn_id) → None` : `self.ingestion.on_session_stop(...)` に委譲
- `async on_user_prompt(query, session_id) → list[str]` : `self.injection.on_user_prompt(...)` に委譲

`services.py` を dataclass から通常クラスへ変更し、外部 API を安定させる。`repl.py` は変更しない（既存の呼び出しがそのまま通るため）。

### D-2. `MemoryEntry` 復元の統一

**現状**: `mapper.py:row_to_entry()` と `jsonl_store.py:_entry_from_dict()` に類似した復元ロジックが存在する。

**方針**: `mapper.py:row_to_entry()` を唯一の復元関数として維持し、`jsonl_store.py:_entry_from_dict()` は `row_to_entry(d)` に委譲する形に縮退させる。
- `_entry_from_dict` は JSONL 固有の `memory_type` 事前検証を行った後、`row_to_entry(d)` を呼ぶ。

### D-3. スコア契約の明文化

**現状**: `VectorRetriever.knn_search()` は `score=-distance` を返し、`ingestion.py` が `-h.score` で距離に逆変換している。

**方針**: `MemoryHit.score` のコメントを "higher is better" と明記し、型ドキュメントに距離変換の事実を記録する。さらに `ingestion.py` の `_has_near_duplicate` / `_link_duplicates` に `distance = -hit.score` の変換と意図をコメントで明示する。型変更（新フィールド追加）は今回スコープ外とし、コメントと命名による明文化に留める。

### D-4. `injection.py` の prefix 不整合解消

**現状**:
- `on_session_start`: `f"[Memory] {e.summary or e.content[:100]}"` を使用
- `on_user_prompt`: `self._policy.format_prefix_semantic` / `format_prefix_episodic` を使用

**方針**: `on_session_start` も `InjectionPolicy.format_prefix_semantic` を使用するよう統一する。`session_id` 引数は `InjectionPolicy` 検索時に将来使用される余地があるが、現状未使用のため削除する。

### D-5. `ingestion.py` 保存経路統一

**現状**: `on_session_stop` はdedup + link + 保存を行うが、`_persist_entry` は dedup / link なしで直接保存する。

**方針**: dedup / link は自動抽出 (`on_session_stop`) 専用の責務として意図的に分離する設計に整理する。`write_semantic` / `write_episodic` は手動書き込みであり dedup 不要のため、現在の `_persist_entry` の動作は正しい。ただし、この設計判断をコメントで明示する。

### D-6. `extract.py` の設定化

**現状**: `MIN_CONTENT_CHARS`, `MAX_ENTRIES`, `_SEMANTIC_KEYWORDS` などがハードコード定数。

**方針**: モジュールレベル定数を `ExtractionPolicy` dataclass にまとめ、`extract_memories()` の引数として受け取れるようにする。デフォルト値は現在値を維持。`AgentConfig.memory` からの設定注入は今回スコープ外とし、dataclass 化のみ行う。
