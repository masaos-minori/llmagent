# Plan: Align DocumentStore.chunk_insert() with chunk metadata schema

Source requirement: `requires/20260702_43_require.md`

---

## Goal

Extend `DocumentStore.chunk_insert()` and its SQLite implementation to accept `chunk_type` and `source_file` parameters, eliminating the inconsistency between the protocol API and the canonical `chunks` schema.

---

## Scope

**In-Scope**:
- `scripts/db/store_protocols.py` — add `chunk_type` and `source_file` parameters to `DocumentStore.chunk_insert()`
- `scripts/db/store_impl.py` — add the same parameters to `SQLiteDocumentStore.chunk_insert()` and pass them through to the INSERT
- `docs/90_shared_05_db_api_and_operations.md` — update `chunk_insert` signature documentation
- `tests/test_db_store_impl.py` — update existing `chunk_insert` tests and add coverage for new parameters

**Out-of-Scope**:
- Backward-compatible API shims (callers will pass keyword args; defaults handle backward compat)
- Existing DB migration (schema already includes `chunk_type` and `source_file`)
- Refactoring `RagIngester._insert_chunk()` — it already bypasses the protocol; no change to its call path

---

## Assumptions

1. The `chunks` table already has `chunk_type TEXT` and `source_file TEXT` columns — confirmed in `RagIngester._insert_chunk()` SQL at line 456.
2. Option A (extend protocol and implementation) is the selected approach. Option B (document bypass) would leave the inconsistency unresolved and is excluded.
3. Default values of `chunk_type=""` and `source_file=""` preserve backward compatibility for existing callers that do not pass these arguments.
4. `RagIngester._insert_chunk()` is not changed: it operates on `SQLiteHelper` directly and already handles both fields.
5. `chunk_insert` in the protocol uses keyword-argument defaults; `runtime_checkable` Protocol does not enforce default values at runtime, so adding optional params does not break structural subtyping for existing implementors that lack the params — but `SQLiteDocumentStore` is the only implementation, so this is a non-issue.

## Implementation Steps

### Phase 1: Protocol and implementation update

1. **Update `DocumentStore.chunk_insert()` signature in `store_protocols.py`**
   - Add `chunk_type: str = ""` and `source_file: str = ""` as keyword parameters after `normalized`
   - Update docstring to describe the new parameters
   - Completion criterion: `store_protocols.py` compiles with `mypy` without errors

2. **Update `SQLiteDocumentStore.chunk_insert()` in `store_impl.py`**
   - Match the new signature from the protocol
   - Extend the INSERT SQL to include `chunk_type` and `source_file` columns
   - Completion criterion: `store_impl.py` compiles with `mypy`; the SQL insert matches the schema

### Phase 2: Documentation update

3. **Update `docs/90_shared_05_db_api_and_operations.md`**
   - Update line 152 (`def chunk_insert(...)` signature) to include `chunk_type` and `source_file`

   - Completion criterion: all store tests pass

5. **Run full validation suite**
