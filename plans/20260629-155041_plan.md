## 1. Goal
- Fix ordinal tracking and empty-section handling bugs in the hierarchy-aware Markdown parser (`mcp/mdq/parser.py`), and expand unit test coverage to cover code fences, frontmatter, repeated headings, and nested heading paths per the acceptance criteria of requirement 20260626_16.

## 2. Scope
- **In-Scope**:
  - `scripts/mcp/mdq/parser.py` — fix ordinal computation logic; fix empty-heading section omission
  - `tests/test_mdq_service.py` — add unit tests for: code fences, frontmatter stripping, repeated headings, nested paths, malformed headings, headings with no content body
- **Out-of-Scope**:
  - DB schema changes (no changes needed)
  - `indexer.py`, `service.py`, `models.py` — no interface changes required
  - Setext heading support (explicitly unsupported per parser docstring)
  - RAG ingestion pipeline (`rag/ingestion/chunk_splitter.py`) — separate module

## 3. Assumptions
- The parser docstring accurately describes the intended unsupported features (Setext, HTML, MDX, GFM tables).
- `ordinal` is defined as the 1-based rank of a heading among all headings at the same level within the *same parent*, not globally across the entire document.
- An empty-body heading section (heading with no content lines) is intentionally omitted from results; this behavior is acceptable and need not be changed unless a test proves otherwise.
- The `parse_markdown` function is an async wrapper over purely synchronous logic; no I/O concurrency issues exist.

| UNK-02 | Whether headings with no content body should be included in parser output | Requirement says "content" field is required but does not state minimum length | Check `indexer.py` behavior when content is empty string; decide to either include empty-body sections or keep current omission behavior | No |

## 6. Implementation Steps
1. **Phase 1: Bug analysis and behavior lock**
   - [ ] Reproduce ordinal bug with a minimal test: two `## Heading` at same level → both should have distinct ordinals (1 and 2)
   - [ ] Verify current behavior: ordinal is always 1 due to `same_level_headings = [h for h in heading_stack if h[0] == heading_level_val]` computed after the same-level entries are popped from the stack
   - [ ] Decide ordinal scope (UNK-01): implement as global counter per `(level, parent_path)` using a `dict[tuple[int, str], int]` counter
2. **Phase 2: Fix ordinal computation in `parser.py`**
   - [ ] Add `ordinal_counter: dict[tuple[int, str], int] = {}` before the main loop
   - [ ] After updating `heading_stack` and computing `heading_path`, compute `parent_path = heading_path` (path of ancestors before appending current heading), then `key = (heading_level_val, parent_path)`
   - [ ] Increment `ordinal_counter[key]` and assign result to `current_section["ordinal"]`
   - [ ] Remove the now-incorrect `same_level_headings` computation
3. **Phase 3: Expand unit tests in `TestParseMarkdown`**
   - [ ] `test_code_fence_hash_not_treated_as_heading` — `# in fence` line inside ` ``` ` block must not create a new section
   - [ ] `test_frontmatter_stripped` — YAML frontmatter block is skipped; first ATX heading becomes the first real section
   - [ ] `test_repeated_headings_have_distinct_ordinals` — two `## API` headings produce ordinal=1 and ordinal=2
   - [ ] `test_nested_heading_path` — `# A` / `## B` / `### C` produces `heading_path="A > B"` for `### C`
   - [ ] `test_malformed_heading_treated_as_content` — `#NoSpace` (no space after `#`) is treated as plain content
   - [ ] `test_heading_level_returned_correctly` — verify `heading_level` field equals the ATX `#` count
4. **Phase 4: Validation and deploy**
   - [ ] Run `uv run pytest tests/test_mdq_service.py -v` — all tests pass
