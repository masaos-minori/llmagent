## Pre-Analysis: Circular Import Fix

`scripts/shared/tool_executor_helpers.py:32` currently uses a lazy import:
```python
from shared.tool_executor import TransportErrorInfo  # noqa: F821
```
This creates a circular dependency: `tool_executor` â†’ `tool_executor_helpers` â†’ `tool_executor`.

Fix: change to `from shared.transport_dto import TransportErrorInfo` (direct, no cycle).

## Design

### Lines to remove from `scripts/shared/tool_executor.py`

```python

    format_transport_error,
    is_side_effect,
    tool_hash_key,

    ToolCallResult,  # noqa: F401 â€” re-export for backward compat
```

- Keep `from shared.transport_dto import ToolCallResult, TransportErrorInfo` (used internally, not re-exports)
- Remove the `# noqa: F401 â€” re-export for backward compat` comment
