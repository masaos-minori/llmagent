## Goal

Remove all `_migrate_*` functions and `db.conn` direct-access patterns from `scripts/db/create_schema.py` after the migration helpers are no longer needed, keeping the file focused solely on DDL execution.

## Scope

**In-Scope**:
- Remove `_migrate_rag_schema()` from `scripts/db/create_schema.py`
- Remove `_migrate_add_undone_column()` from `scripts/db/create_schema.py`
- Remove `_migrate_session_schema()` from `scripts/db/create_schema.py`
- Remove `_migrate_workflow_schema()` from `scripts/db/create_schema.py`
- Remove all `db.conn` direct access and their `# type: ignore[arg-type]` annotations
- Remove stale comments referencing migration behavior from module docstring
- Update module docstring to reflect DDL-only behavior
- Verify `ruff check` passes with no unused imports after removal

**Out-of-Scope**:
- Behavioral changes to DDL execution logic
- Changes to `db/helper.py`, `db/schema_sql.py`, or `db/store_protocols.py`
- Modifications to tests other than those that assert migration behavior
- Any changes to other modules that import `create_schema`

## Assumptions

1. The schema DDL in `db/schema_sql.py` already includes all columns that the `_migrate_*` helpers add (`chunk_type`, `source_file`, `undone`, FK on `tool_results.session_id`, `workflow_id`), so removing the helpers does not break new installations.
2. All existing production databases have already been migrated; the migration helpers are dead paths for any db created after the schema DDL was updated.
3. `sqlite3` and `sys` imports remain necessary after migration removal (`sqlite3` for exception types, `sys` for `sys.exit`).
4. `db.store_protocols.get_embedding_dims` and `db.helper.SQLiteHelper` remain necessary; only migration helper calls are removed.
