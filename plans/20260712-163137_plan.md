## Design

1. **`_TOOL_COUNT_RE`の修正:**
   `re.compile(r"\bTools\((\d+)\)")` → `re.compile(r"ツール（(\d+)個）")`
   (全角括弧・日本語「個」に対応)。
2. **`_find_server_section_for_line()`の修正:**
   見出し行の抽出を `m = re.match(r"^## (\S+)", line)` → `m = re.match(r"^## ([\w-]+)", line)`
   に変更する。`[\w-]+`は全角括弧`（`の手前で自然に止まるため、`section_name`は
   `"web-search-mcp"`のように正しく単独抽出される。以降の`.split("(")[0].strip()`は
   実質的に不要になるが、副作用はないため残置してよい(将来ASCII括弧表記に戻っても安全)。
3. **`check_tool_counts()`の書き換え:**
   - 対象ファイル名を明示的な5件のリストとして定義する(要望書が提案した「サーバー名→ファイル名」
     対応表は不要と判明したため作らない。対象ファイル名の列挙のみで足りる):
     ```python
     _TOOL_COUNT_CATALOG_FILES = (
         "04_mcp_04_01_web-search-file-read-github.md",
         "04_mcp_04_02_file-write-file-delete-shell.md",
         "04_mcp_04_03_rag-pipeline-and-cicd.md",
         "04_mcp_04_04_mdq.md",
         "04_mcp_04_05_git.md",
     )
     ```
   - `files`の中から`doc.rel_path in _TOOL_COUNT_CATALOG_FILES`に該当する`DocFile`を全て収集する。
   - 1件も見つからない場合のみ、既存と同様の「not found」WARNINGを返す(メッセージは
     5ファイル分をまとめて言及する形に更新)。
   - 見つかったファイルごとに、既存の行走査ロジック(`_TOOL_COUNT_RE`探索 →
     `_find_server_section_for_line()` → `_SERVER_TOOLS_MAP`突合)を適用する
     (ロジック自体は現行のまま、ファイル単位のループでラップする)。
   - `_TOOL_COUNT_CATALOG_FILES`のうち実際には見つからなかったファイル(部分的欠落)があれば、
     個別に「<filename> not found」のWARNINGを追加する(全件見つかった場合は追加しない)。
4. **web-search-mcpの扱い:** `docs/04_mcp_04_01_web-search-file-read-github.md`の
   web-search-mcpセクションは`**ツール:**`(個数注記なし、ツールが1個のため)であり、
   `_TOOL_COUNT_RE`は一致しない。この場合、既存の`if m:`ゲートにより単にスキップされ、
   エラーにもWARNINGにもならない(既存の優雅な劣化動作と同じ)。ドキュメント本文の修正は
   スコープ外のため、この1サーバーはカウント検証の対象外のままとする。
5. **テスト更新:** `tests/test_check_mcp_docs_consistency.py`の`TestCheckToolCounts`
   (233〜306行)の全6テストについて、フィクスチャを次のように更新する:
   - `rel_path`を実際の5ファイル名のいずれかに変更する
   - 見出し行を`"## web-search-mcp（ポート 8015）"`のような全角括弧表記に変更する
   - ツール数行を`"**ツール（1個）:** search_web"`のような`ツール（N個）`表記に変更する
   - `test_missing_catalog_file_returns_warning`は、5ファイルすべてが`files`に存在しない
     ケースに更新する
   - 新規: 5ファイルのうち一部のみ欠落しているケースのテストを追加する

## Implementation steps

1. `tools/check_mcp_docs_consistency.py`の`_TOOL_COUNT_RE`を`ツール（N個）`表記に対応させる。
2. `_find_server_section_for_line()`の見出し抽出正規表現を`[\w-]+`に修正する。
3. `check_tool_counts()`を5ファイル走査(`_TOOL_COUNT_CATALOG_FILES`定数使用)に書き換える。
4. `tests/test_check_mcp_docs_consistency.py`の`TestCheckToolCounts`全テストのフィクスチャを
   実表記に更新し、部分欠落ケースのテストを追加する。
5. `uv run pytest tests/test_check_mcp_docs_consistency.py -v` で全テストが通ることを確認する。
6. `uv run python tools/check_mcp_docs_consistency.py` を実行し、`04_mcp_04_server_catalog.md
   not found`の警告が消え、実ドキュメントに対してツール数の突合が実際に走ることを確認する
   (現状の`_SERVER_TOOLS_MAP`と実ドキュメントの記載が一致していれば新規WARNINGは出ないはず)。
7. 復旧確認のため、一時的にスクラッチ領域([[scratchpad]]配下)に対象ファイルのコピーを作り
   ツール数を意図的に1つずらして`check_tool_counts()`を直接呼び出し、WARNINGが発火することを
   手動確認する(実ドキュメントは変更しない)。
