## Goal

Restore automatic startup and watchdog recovery for the RAG Pipeline and CICD MCP servers by changing `startup_mode` from `"none"` to `"subprocess"` and configuring the missing `GITHUB_TOKEN`.

## Scope

**In-Scope:**

- Change `startup_mode` from `"none"` to `"subprocess"` for `rag_pipeline` in `config/agent.toml`

**Out-of-Scope:**

## Assumptions

1. A valid GitHub PAT exists and can be used for CICD operations
2. The `startup_timeout_sec` parameter handles startup failures gracefully if the server doesn't become healthy within the timeout period
3. Port conflicts will not occur because the lifecycle manager checks port availability before starting
4. Both servers' dependencies are installed (`uv sync --dev` has been run)
5. The fix applies to both `/opt/llm` (production) and `/home/sugimoto/llmagent` (development) environments

## Risks & Mitigations

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| GitHub PAT unavailable (UNK-01 blocking) | High | Block on resolution before proceeding; use environment variable approach if config file is insecure |
| Token exposure if committed to repository | Medium | Use environment variables where possible; never commit actual token values to version control |
| Startup timeout if server fails to start | Low | `startup_timeout_sec` already handles this ŌĆö agent logs error and continues |
| Port conflict if server is already running | Low | Lifecycle manager checks port availability before starting |
| Configuration drift between deploy and runtime | Medium | Apply changes through standard deployment workflow; verify both environments after deploy |
| Token expiration causing silent degradation | Low-Medium | Monitor token expiration dates; add watchdog alert for expiring tokens (long-term) |
| Server crashes due to missing dependencies | Low | Ensure `uv sync --dev` has been run; verify server starts independently before agent integration |
