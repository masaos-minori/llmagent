# Plan: Add `/skill` slash command for loading Skills into ephemeral system prompt

Source requirement: `requires/20260712_10_require.md`

## Goal

Add a `/skill <name> [args]` slash command to AgentREPL that injects the content of
`skills/<name>/SKILL.md` as an ephemeral system message (reusing the existing
`_ephemeral: True` pattern), so skill procedures can be loaded into the LLM context
without pasting them manually every time.

## Scope

**In-Scope**
- New `CommandDef` entry for `/skill` in `scripts/agent/commands/command_defs_list.py`
- New file `scripts/agent/commands/cmd_skill.py` implementing `_SkillMixin._cmd_skill(args)`
  - No-arg invocation: list directory names under `repo_root/skills/` (directories only,
    `DESIGN.md` and any other file entries excluded), no LLM call
  - `<name>` invocation: read `repo_root/skills/<name>/SKILL.md`, remove any previous
    `_skill_ephemeral: True` message from `ctx.conv.history`, append new
    `{"role": "system", "content": <SKILL.md text [+ args]>, "_ephemeral": True, "_skill_ephemeral": True}`
  - Error path: `<name>` not found or not a directory -> error message via `self._out`,
    no LLM call, no history mutation
- Register `_SkillMixin` in `CommandRegistry` base class list in
  `scripts/agent/commands/registry.py`
- New characterization tests in `tests/test_cmd_skill.py` (list output, replace-on-repeat
  behavior, error handling, `_ephemeral`/`_skill_ephemeral` flag correctness)
- Update `routing.md` line 7 area to mention `/skill <name>` as the runtime-invocable form
- Update `docs/05_agent_07_11_cli-and-commands-slash-commands-memory-other.md` with a new
  "Skillカテゴリ" section (see Design §5)

**Out-of-Scope**
- Any change to `config/agent.toml` or `AgentConfig` (no new config key, no multi-path
  search)
- A separate personal-scope skills directory
- Any change to `docs/05_agent_07_01_cli-and-commands-cli-reference.md` beyond what is
  already an index (see Unknowns — this file itself is a table-of-contents and is not the
  place for command details; corrected target is 05_agent_07_11)

## Assumptions

1. Every directory under `repo_root/skills/` currently contains a `SKILL.md` — verified
   directly (`skills/deploy`, `git-commit-and-sync`, `mcp-server-add`,
   `python-debug-root-cause`, `python-design`, `python-documentation`,
   `python-implementation`, `python-issue-to-plan`, `python-lint-typecheck`,
   `python-refactoring`, `python-test-and-fix` — all 11 have `SKILL.md`; `DESIGN.md` is a
   loose file, not a directory, and is correctly excluded by an `is_dir()` check).
2. `repo_root` from `scripts/agent/commands/cmd_skill.py` resolves via
   `Path(__file__).resolve().parent.parent.parent.parent` (commands -> agent -> scripts ->
   repo_root), mirroring the pattern in `scripts/shared/config_loader.py:49`
   (`parent.parent.parent` from `scripts/shared/`).
3. `/skill` itself never triggers an LLM call — confirmed by the ephemeral-injection
   precedents (`mode_classification.classify_and_inject_mode`,
   `Orchestrator._handle_memory_injection`), which only mutate `ctx.conv.history` and are
   picked up on the next turn via `Orchestrator._sync_system_prompt()`.
4. `_sync_system_prompt()` (`scripts/agent/orchestrator.py:500-519`) strips only messages
   with `_ephemeral` or `_memory_injected` truthy; adding `_skill_ephemeral` alongside
   `_ephemeral: True` is sufficient for the message to be auto-removed next turn without
   any change to `_sync_system_prompt()` itself.
5. `CommandRegistry.__init__` fail-fast handler check (`registry.py:84-88`) requires the
   new mixin's method name to exactly match the `handler` string in the `CommandDef`.

### 2. `scripts/agent/commands/cmd_skill.py` (new file) — full content

Mirrors `_PluginsMixin` (`cmd_plugins.py`) structurally; adds a `Path`-based helper since
this handler reads from the filesystem instead of a registry singleton.

```python
"""agent/commands/cmd_skill.py
/skill slash command: list skills or inject a skill's SKILL.md as ephemeral system context.
"""

from __future__ import annotations

from pathlib import Path
from typing import Any

from agent.commands.mixin_base import MixinBase


class _SkillMixin(MixinBase):
    def __init__(self, *args: Any, **kwargs: Any) -> None:
        super().__init__(*args, **kwargs)

    def _skills_dir(self) -> Path:
        """repo_root/skills — repo_root is 4 levels above this file."""
        return Path(__file__).resolve().parent.parent.parent.parent / "skills"

    def _cmd_skill(self, args: str = "") -> None:
        """Handle /skill [name] [args]."""
        args = args.strip()
        if not args:
            for name in sorted(
                p.name for p in self._skills_dir().iterdir() if p.is_dir()
            ):
                self._out.write(name)
            return

        name, _, rest = args.partition(" ")
        skill_dir = self._skills_dir() / name
        if not skill_dir.is_dir():
            self._out.write(f"Unknown skill: {name}")
            return

        content = (skill_dir / "SKILL.md").read_text(encoding="utf-8")
        rest = rest.strip()
        if rest:
            content = f"{content}\n\nInvocation args: {rest}"

        ctx = self._ctx
        ctx.conv.history = [
            m for m in ctx.conv.history if not m.get("_skill_ephemeral")
        ]
        ctx.conv.history.append(
            {
                "role": "system",
                "content": content,
                "_ephemeral": True,
                "_skill_ephemeral": True,
            }
        )
```

Notes:
- `_skills_dir()` resolves `repo_root` the same way as `config_loader.py:49`
  (`Path(__file__).resolve().parent.parent.parent`), scaled by one extra `.parent` because
  `cmd_skill.py` lives one directory deeper (`scripts/agent/commands/` vs.
  `scripts/shared/`).
- File entries under `skills/` (e.g. `DESIGN.md`) are excluded from both the listing and
  name resolution by the same `is_dir()` check, per requirement §提案設計-1/2.
- No `write_table` — plain `self._out.write(name)` per line, since the requirement asks
  for names only, not descriptions.
- Content label uses English (`"Invocation args: ..."`) rather than the requirement's
  Japanese example (`"呼び出し引数: ..."`) — confirmed with the user: standardize on
  English to stay consistent with `rules/coding.md`'s English-only convention, even though
  this text is LLM-facing content rather than a code comment or log line.

### 4. `tests/test_cmd_skill.py` (new file) — full content

Uses the `SimpleNamespace`-based lightweight `_ctx` convention from
`test_agent_cmd_memory.py:43-52` (not a real `AgentContext`), and the
`MagicMock(spec=CliOutputPort)` convention from `test_cmd_plugins.py`.

```python
"""tests/test_cmd_skill.py
Tests for /skill command handler in cmd_skill.py.
"""

from __future__ import annotations

from pathlib import Path
from types import SimpleNamespace
from unittest.mock import MagicMock

import pytest

from agent.commands.cmd_skill import _SkillMixin
from agent.commands.output_port import CliOutputPort


@pytest.fixture
def skills_root(tmp_path: Path) -> Path:
    (tmp_path / "alpha").mkdir()
    (tmp_path / "alpha" / "SKILL.md").write_text("Alpha content", encoding="utf-8")
    (tmp_path / "beta").mkdir()
    (tmp_path / "beta" / "SKILL.md").write_text("Beta content", encoding="utf-8")
    (tmp_path / "DESIGN.md").write_text("not a skill dir", encoding="utf-8")
    return tmp_path


def _make_mixin(skills_root: Path) -> _SkillMixin:
    mixin = object.__new__(_SkillMixin)
    mixin._out = MagicMock(spec=CliOutputPort)  # type: ignore[attr-defined]
    mixin._ctx = SimpleNamespace(  # type: ignore[attr-defined]
        conv=SimpleNamespace(history=[])
    )
    mixin._skills_dir = MagicMock(return_value=skills_root)  # type: ignore[attr-defined]
    return mixin


class TestCmdSkillList:
    def test_no_args_lists_directories_only(self, skills_root: Path) -> None:
        mixin = _make_mixin(skills_root)
        mixin._cmd_skill("")
        written = [c.args[0] for c in mixin._out.write.call_args_list]
        assert written == ["alpha", "beta"]  # DESIGN.md excluded, sorted

    def test_list_does_not_touch_history(self, skills_root: Path) -> None:
        mixin = _make_mixin(skills_root)
        mixin._cmd_skill("")
        assert mixin._ctx.conv.history == []


class TestCmdSkillLoad:
    def test_unknown_skill_writes_error_no_history_mutation(
        self, skills_root: Path
    ) -> None:
        mixin = _make_mixin(skills_root)
        mixin._cmd_skill("nonexistent")
        mixin._out.write.assert_called_once_with("Unknown skill: nonexistent")
        assert mixin._ctx.conv.history == []

    def test_file_entry_treated_as_unknown(self, skills_root: Path) -> None:
        mixin = _make_mixin(skills_root)
        mixin._cmd_skill("DESIGN")
        mixin._out.write.assert_called_once_with("Unknown skill: DESIGN")
        assert mixin._ctx.conv.history == []

    def test_known_skill_appends_ephemeral_message(self, skills_root: Path) -> None:
        mixin = _make_mixin(skills_root)
        mixin._cmd_skill("alpha")
        assert len(mixin._ctx.conv.history) == 1
        msg = mixin._ctx.conv.history[0]
        assert msg["role"] == "system"
        assert msg["content"] == "Alpha content"
        assert msg["_ephemeral"] is True
        assert msg["_skill_ephemeral"] is True

    def test_args_appended_to_content(self, skills_root: Path) -> None:
        mixin = _make_mixin(skills_root)
        mixin._cmd_skill("alpha extra context")
        msg = mixin._ctx.conv.history[0]
        assert "Invocation args: extra context" in msg["content"]

    def test_repeated_call_replaces_previous_skill_message(
        self, skills_root: Path
    ) -> None:
        mixin = _make_mixin(skills_root)
        mixin._cmd_skill("alpha")
        mixin._cmd_skill("beta")
        assert len(mixin._ctx.conv.history) == 1
        assert mixin._ctx.conv.history[0]["content"] == "Beta content"

    def test_unrelated_ephemeral_messages_are_preserved(
        self, skills_root: Path
    ) -> None:
        mixin = _make_mixin(skills_root)
        mixin._ctx.conv.history.append(
            {"role": "system", "content": "mode hint", "_ephemeral": True}
        )
        mixin._ctx.conv.history.append(
            {"role": "system", "content": "memory", "_memory_injected": True}
        )
        mixin._cmd_skill("alpha")
        assert len(mixin._ctx.conv.history) == 3
        contents = [m["content"] for m in mixin._ctx.conv.history]
        assert "mode hint" in contents
        assert "memory" in contents
        assert "Alpha content" in contents
```

## Risks

- **Risk**: Multiple `_ephemeral: True` messages coexisting (mode hint + memory injection +
  skill injection) could interact unexpectedly in `_sync_system_prompt()`'s filter/rebuild
  order. → **Mitigation** (likelihood: medium): add a test asserting all three flag
  combinations survive/are-removed independently; no change needed to
  `_sync_system_prompt()` itself since it already filters on both flags generically.
- **Risk**: `_skill_ephemeral` replace-filter accidentally scoped too broadly (e.g. matching
  on `_ephemeral` alone) would delete unrelated ephemeral messages. → **Mitigation**
  (likelihood: medium): filter strictly on `_skill_ephemeral` truthy only, covered by a
  dedicated test per Design step 4.
- **Risk**: Large `SKILL.md` files inflate token usage every time `/skill <name>` is
  invoked. → **Mitigation** (likelihood: low): explicitly out of scope per requirement;
  no action this task, flagged for future operational monitoring.
- **Risk**: Requirement's stated doc target (`05_agent_07_01`) diverges from actual
  detail-holding file (`05_agent_07_11`), risking the change being written to the wrong
  file if not caught. → **Mitigation** (likelihood: low, already resolved): corrected in
  Scope/Design above; implementer must target `05_agent_07_11`, not `05_agent_07_01`.
