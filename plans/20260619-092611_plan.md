# Plan: Add DB integration tests for chunk metadata preservation

## Goal

Add comprehensive DB integration tests that verify chunk metadata is correctly written to SQLite, not just parsed from JSON. Fill identified gaps in test coverage.

## Scope

**In scope:**
- Add real DB integration tests (using `_FakeSQLiteHelper` with in-memory SQLite) for:
  - `documents.chunking_strategy` preservation
  - `chunks.chunk_index` preservation
  - `chunks.normalized_content` preservation
  - `--force` reinsert path verification
- Fill test gaps identified in exploration:
  - `chunk_type` field passthrough (future-proofing)
  - `source_file` field passthrough (future-proofing)
  - `etag`/`last_modified` passthrough from first chunk to documents table
  - `_read_chunk_json()` validation paths (missing url, missing content)
  - Invalid `chunk_index` fallback to 0
  - Mixed URL grouping
- Update existing tests if assertions need adjustment

**Out of scope:**
- Testing embedding vector storage (covered by existing `test_embedding_failure_returns_false`)
- Testing file movement to registered/ (covered by `test_moves_processed_files_to_registered`)
- Testing parallel chunk ingestion ordering (non-deterministic, low value)
- Testing OSError resilience paths (edge case, low value)

## Assumptions

1. `_FakeSQLiteHelper` pattern is the established approach for real DB tests in this codebase.
2. Existing tests in `test_rag_ingester.py` use `MagicMock` — new tests should use real DB assertions.
3. Chunk JSON file format matches what ChunkSplitter produces (10 fields).
4. Tests should be in `tests/test_rag_ingester.py` (canonical location for ingester tests).
