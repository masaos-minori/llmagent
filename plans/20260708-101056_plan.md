# Plan: H-9 — Remove or fully orphan ToolResultStore and tool_results DB access

## Goal

`ToolResultStore` および `tool_results` テーブルへの全アクセスをランタイムコードから除去する。
`/tool` コマンド (`/tool list`, `/tool show <id>`) も合わせて削除する。

### U-1: `_cmd_tool` の CommandRegistry 登録箇所
- **状態**: 未確認。`cmd_tooling.py` に `_ToolingMixin._cmd_tool` は定義済みだが、`CommandRegistry` への登録 (`COMMANDS` dict や decorator など) の場所を要調査。

- **状態**: 分析済み。`agent/services/models.py` は agent 層の呼び出し元のための re-export stub。`ToolResultRow` を削除すればこの re-export も削除する。

### U-4: `test_undo_artifact_consistency.py` の削除可否
- **状態**: 要確認。undo 機能そのもののテストが含まれている可能性。`ToolResultStore` 以外の undo ロジックテストが混在する場合はファイル分割が必要。

## Design

### 削除方針

- `ToolResultStore` と `ToolResultRow` は完全削除 (orphan ではなく delete)。
- `ToolResultView` も `/tool` コマンドとともに削除。
- `ctx.tool_result_store` フィールドは `AgentContext.__init__` から除去し、型アノテーションも削除。
- `tool_runner.py` の `result_id = ctx.tool_result_store.store(...)` 削除後、`result_id` を参照している `_apply_turn_char_limit` の呼び出しシグネチャも変更が必要 (result_id 引数の除去または None 渡し)。

### `_apply_turn_char_limit` の影響

`tool_runner.py` で `result_id` を `_apply_turn_char_limit` に渡している。この関数が result_id を使ってユーザーへのメッセージに ID を埋め込んでいる可能性がある。実装を確認して適切に対処する。

### `/plan` コマンドの保持

`_ToolingMixin._cmd_plan` は `/plan` コマンドを実装している。`cmd_tooling.py` からは `_cmd_plan` のみ残し、ファイル名変更または別 mixin への移動を検討する。あるいは `_ToolingMixin` を `_PlanMixin` にリネームして `_cmd_tool` 関連メソッドを削除する。

## Risks

### R-1: `_apply_turn_char_limit` への `result_id` 依存
- **内容**: `result_id` を使って LLM に渡すメッセージに ID を埋め込んでいる場合、削除するとユーザーが `/tool show <id>` を参照できないメッセージが残る可能性がある。
- **緩和**: `tool_runner.py` の `_apply_turn_char_limit` 実装を Step 1 で確認し、ID 埋め込みなら代替テキストに差し替える。

### R-2: `_cmd_plan` の依存関係破壊
- **内容**: `_ToolingMixin` から `/tool` 関連を削除する際に `_cmd_plan` も誤って削除するリスク。
- **緩和**: Step 2 で `_cmd_plan` の存在を明示確認してから削除を実施する。

### R-3: undo 機能への副作用
- **内容**: `mark_turn_undone()` を削除するため、`/undo` の動作が変わる。undo 時に tool_results に `undone=1` がセットされなくなる。
- **緩和**: テーブル自体を削除するため影響なし。undo 機能 (`undo_service.py`) の残存コードが正常動作するか `test_orchestrator.py` で確認する。

### R-4: `test_undo_artifact_consistency.py` に undo ロジックテストが混在
- **内容**: undo 機能全般のテストが同ファイルに含まれる場合、ファイルごと削除すると undo テストが消える。
- **緩和**: Step 16 でファイル内容を精読し、`ToolResultStore` 以外のテストが存在する場合は別ファイルに移動してから削除する。

### R-5: CommandRegistry 登録箇所が複数存在する可能性
- **内容**: `/tool` コマンドの登録が複数箇所に散在していると、1箇所削除しても参照が残る。
- **緩和**: Step 17 で `grep -rn "_cmd_tool\|\"tool\"" scripts/agent/commands/` を実施して全登録を確認する。
