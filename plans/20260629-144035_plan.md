## 8. Risks & Mitigations
- **Risk**: Extending `_process_turn` return tuple breaks `_handle_workflow_engine` which unpacks `(answer, error_kind)` â†’ **Mitigation**: Update both call sites (`handle_turn` and `_handle_workflow_engine`) atomically in the same commit; mypy will catch missed sites.
- **Risk**: `stat_partial_completions` is incremented after `_process_turn` returns, so reading it inside `_process_turn` would give stale value â†’ **Mitigation**: Derive `is_partial` from `result.exception is not None and bool(result.exception.partial_text)` directly in `_process_turn`, matching the existing `_handle_partial_completion` condition.
- **Risk**: Tests that mock `_diagnostic_store.save` call count may become fragile if a new save call is added â†’ **Mitigation**: Use `assert_called()` rather than `assert_called_once()` where the exact call count is not semantically meaningful.
