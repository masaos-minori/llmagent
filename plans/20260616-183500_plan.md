# Plan: Add readiness/status metadata to /v1/tools

## Goal

Extend `/v1/tools` responses with optional per-tool and server-level readiness metadata so clients can distinguish production-ready tools from stub, experimental, or config-dependent ones.

## Scope

**In scope**:
- `scripts/mcp/server.py` — add shared helper for enriching tool lists with server-level metadata
- All MCP server `/v1/tools` endpoints (11 servers) — use the shared helper
- `scripts/mcp/git/tools.py` — add `requires_config` to write tools
- `scripts/mcp/cicd/tools.py` — add `requires_config` to all 4 CICD tools
- `scripts/mcp/installer_templates.py` — include metadata fields in generated tool definitions
- `tests/test_mcp_tools_validation.py` — add tests for metadata enrichment

**Out of scope**:
- Changing the `/v1/call_tool` endpoint or dispatch logic
- Adding server-level health metadata to `/v1/tools` (already in `/health`)
- Modifying consumers that already read `/v1/tools` (backward compatible — extra fields ignored)

## Assumptions

1. Extra keys on tool dicts are backward compatible — `build_discovery_map()` only reads `name` and `server_key`.
2. Metadata is additive: optional fields have no default; tools without metadata remain unchanged.
3. The `status` field uses a small enum: `"production"`, `"beta"`, `"experimental"`, `"stub"`.
4. `requires_config` is a list of config key names (e.g., `["github_token", "repo_allowlist"]`).

## Implementation Steps

1. **Add `_enrich_tools_with_server_key()` helper to `mcp/server.py`**
   - New function: takes `(tools: list[dict], server_key: str)` → returns enriched tool list
   - Replaces the inline `{**t, "server_key": key} for t in tools` pattern
   - Preserves all existing fields; adds `server_key` if not already present
   - Optionally accepts a `server_metadata` dict for server-level fields

2. **Update all 11 `/v1/tools` endpoints to use the helper**
   - git, sqlite, cicd, shell, web_search, rag_pipeline, mdq, github, file_read, file_write, file_delete
   - Replace `{**t, "server_key": "..."} for t in _MCP_TOOLS` with `_enrich_tools_with_server_key(_MCP_TOOLS, "...")`

3. **Add `requires_config` metadata to git write tools** (`mcp/git/tools.py`)
   - Add `"requires_config": ["read_only=false"]` to: `git_add`, `git_commit`, `git_checkout`, `git_pull`, `git_push`
   - Read-only tools unchanged

4. **Add `requires_config` metadata to all CICD tools** (`mcp/cicd/tools.py`)
   - Add `"requires_config": ["github_token", "repo_allowlist"]` to all 4 tools

5. **Update installer templates** (`mcp/installer_templates.py`)
   - Include optional `status: "experimental"` and `requires_config` fields in generated tool definitions

6. **Add tests** (`tests/test_mcp_tools_validation.py`)
   - Test `_enrich_tools_with_server_key()` preserves existing fields and adds server_key
   - Test metadata enrichment with server_metadata dict
   - Test that extra fields are ignored by `build_discovery_map()`

7. **Update documentation** (`docs/04_mcp_05_security_and_safety_model.md`)
   - Document the new tool metadata fields
