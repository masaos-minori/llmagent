| Unknown | Evidence missing | Resolution | Blocking |
|---|---|---|---|
| `context_detection.py` が tests/ で参照されていないか | tests/ 配下の rg 実施済み（結果: ゼロ件） | 確認済み — 参照なし | No |

| `rag/stages/search.py` が `db=None` を必要とする理由（`PipelineContext` に `db` がない） | `search.py` の `run()` 実装を確認 | `db` が `PipelineContext` に含まれるべき設計上の問題。テスト追加と同時に `db` を `PipelineContext` フィールドに移すか検討が必要 | Yes — 設計判断が必要 |
| コマンドミックスインのテストを `pytest.mark.asyncio` なしで書けるか | 既存 `test_cmd_mcp.py` パターンを確認 | `@pytest.mark.asyncio` で非同期ハンドラ、sync ハンドラは通常 pytest で対応可能 | No |

**Blocking Unknown の解決:**

`rag/stages/search.py` の `db=None` 問題は以下の設計判断が必要:

**Option A**: `PipelineContext` に `db: SQLiteHelper | None = None` フィールドを追加し、`search.py` が `ctx.db` を使う形に変更する。  
**Option B**: `search.py` の `__init__` に `db` を注入し、`run()` では参照しない形にする。  
**Option C**: 現状維持 — `**kwargs` 経由で `db` を渡す形を維持し、型注釈のみ追加する。

本計画では **Option C** を採用（動作変更なし、実装リスク最小）。`run()` シグネチャを `async def run(self, ctx: PipelineContext, db: SQLiteHelper | None = None, **kwargs: Any) -> None:` に変更して型安全性を確保する。
