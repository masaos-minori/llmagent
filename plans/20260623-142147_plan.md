## Assumptions

1. The `ConfigReloadOutcome` dataclass is not frozen — adding a field with default `field(default_factory=list)` is safe.
2. The startup-only fields are exactly: `use_memory_layer` (in `cfg.memory`) and `plugin_strict` (in `cfg.tool`). DB paths (`rag_db_path`, `session_db_path`) are also startup-only but are owned by `common.toml` and consumed by `SQLiteHelper`, not by `ConfigReloadService`. They are excluded because:
   - They are not fields of `AgentConfig` (they go to `SQLiteHelper` directly)
   - `apply_config_dict()` never sees them in the output of `ConfigLoader().load_all()`
   - Adding detection would require access to `SQLiteHelper` state outside config scope
3. The doc claim "startup-only: `use_memory_layer`, `plugin_strict`" is correct and complete for fields visible to `apply_config_dict()`.
4. `_cmd_reload()` current `WARNING:` line is redundant with the structured `[RESTART]` output — keep it for now (the requirement asks for clarity, not removal of existing warnings).
5. Detection approach: compare `new_cfg.get("use_memory_layer")` to `ctx.cfg.memory.use_memory_layer` and `new_cfg.get("plugin_strict")` to `ctx.cfg.tool.plugin_strict`. If they differ, add the field name to `startup_only`.
6. `05_agent_07_cli-and-commands.md` line 185 is the only wrong description of reload scope; `05_agent_08_configuration.md` line 52 is the authoritative correct version.

## Design

### Change 1: `config_reload.py` — `startup_only` field and detection

**Add field to `ConfigReloadOutcome`:**
```python
@dataclass
class ConfigReloadOutcome:
    applied: list[str] = field(default_factory=list)
    needs_restart: list[str] = field(default_factory=list)
    skipped: list[str] = field(default_factory=list)
    source_files: list[str] = field(default_factory=list)
    deferred: list[str] = field(default_factory=list)
    startup_only: list[str] = field(default_factory=list)    # add this line
```

**Add `_detect_startup_only()` private method to `ConfigReloadService`:**
```python
def _detect_startup_only(
    self,
    new_cfg: dict[str, Any],
) -> list[str]:
    """Return names of startup-only fields that differ between new_cfg and running cfg."""
    changed: list[str] = []
    ctx = self._ctx
    v = _get_bool(new_cfg, "use_memory_layer")
    if v is not None and v != ctx.cfg.memory.use_memory_layer:
        changed.append("use_memory_layer")
    v = _get_bool(new_cfg, "plugin_strict")
    if v is not None and v != ctx.cfg.tool.plugin_strict:
        changed.append("plugin_strict")
    return changed
```

**Call in `apply_config_dict()` (at the end, before return):**
```python
result.startup_only = self._detect_startup_only(new_cfg)
return result
```

### Change 2: `cmd_config.py` — display startup-only items

Add after the `[SKIP]` block in `_cmd_reload()`:

```python
if result.startup_only:

    for item in result.startup_only:
        self._out.write(f"  [STARTUP-ONLY] - {item}")
```

Update the summary line logic to include startup-only case:

```python
        self._out.write("No changes detected.")
```
