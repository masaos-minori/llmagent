## Implementation Steps
1. **Phase 1: ドキュメント更新**
   - [ ] `docs/90_shared_90_inconsistencies_and_known_issues.md` を再読してプラグインロード関連 known issue エントリを確認し、存在すれば "resolved" マークを追加、存在しなければスキップ
   - [ ] `docs/90_shared_03_runtime_and_execution.md` §4 の `load_plugins()` シグネチャを `-> PluginLoadResult` に修正
   - [ ] 同 §4 に以下の型定義ドキュメントを追加:
     - `PluginFailure(path: str, error: str)` — frozen dataclass
     - `PluginLoadResult(loaded_count, failed, tool_conflicts_shadowed, tool_conflicts_allowed, command_shadows)` — frozen dataclass
     - `PluginLoadError` — `RuntimeError` サブクラス; strict_mode 時にすべての失敗を集約して raise
     - `get_last_load_result() -> PluginLoadResult | None` — 最後のロード結果を返す

2. **Phase 2: テスト追加**
   - [ ] `tests/test_plugin_registry.py` に `TestStructuredLoadResultRegression` クラスを追加し、以下を実装:
     - `test_missing_dir_returns_empty_result`: 欠損ディレクトリで `loaded_count==0, failed==()` を確認
     - `test_syntax_error_plugin_in_failed`: `SyntaxError` のあるプラグインが `result.failed` に含まれることを確認
     - `test_strict_mode_raises_plugin_load_error`: `strict_mode=True` で `PluginLoadError` が raise されることを確認
     - `test_reject_policy_increments_conflict_counters`: reject ポリシーで `tool_conflicts_shadowed > 0, tool_conflicts_allowed == 0` を確認
     - `test_command_shadow_count_reported`: builtin コマンドシャドウ時に `command_shadows > 0` を確認
     - `test_get_last_load_result_returns_most_recent`: `load_plugins()` 後に `get_last_load_result()` が同一オブジェクトを返すことを確認

3. **Phase 3: Validation**
   - [ ] `uv run pytest tests/test_plugin_registry.py -v`
   - [ ] `pre-commit run --all-files`
