# Plan: Make github-mcp allowed_workflows fail-closed or remove if not implemented

Source: `requires/20260625_08_require.md`

---

## Goal

`github-mcp` の `allowed_workflows` が実装されているか確認し、実装済みなら fail-closed 化、未実装なら docs・config から削除する。

---

## Scope

**In-Scope**
- `scripts/mcp/github/service_security.py`: `allowed_workflows` の実装有無確認
- `scripts/mcp/github/service_*.py`: 全ファイルで `allowed_workflows` の使用箇所確認
- `config/github_mcp_server.toml`: `allowed_workflows` キーの確認・削除または fail-closed 化
- 関連 docs の更新
- テスト追加

**Out-of-Scope**
- `allowed_repos` の変更（既に fail_closed モードをサポート）

---

## Assumptions

1. `service_security.py` の `grep` 結果に `allowed_workflows` が見当たらなかった → 未実装の可能性が高い。
2. `allowed_repos` は `fail_closed` / `fail_open` モードをサポートしている（確認済み）。
3. `allowed_workflows` はドキュメントにのみ存在し、runtime では何も検査していない可能性が高い。
