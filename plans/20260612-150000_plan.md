# Work Plan: `scripts/rag/` リファクタリング — except Exception撲滅・assert除去・str()変換除去

## Goal

`scripts/rag/` 配下のファイルから `except Exception`（4箇所）・`assert`（1箇所）・無条件 `str()` 変換（1箇所）・境界外 `dict[str, Any]` パラメータを排除し、具体的例外型・明示的ガード・typed DTO アクセスに統一する。

## Scope

**In scope:**
- `scripts/rag/pipeline.py` — `assert isinstance(result, list)` → `TypeError` raise; `str(body.get("context", ""))` → schema validation
- `scripts/rag/ingestion/ingester.py` — 3箇所の `except Exception` → 具体型
- `scripts/rag/ingestion/chunk_splitter.py` — 1箇所の `except Exception` → 具体型
- `scripts/rag/llm.py` — `_mqe_prompt(cfg: dict[str, Any])` → `RagLLM._cfg` (config は既に `self._cfg` として保持) に寄せる; `score_map: dict[str, Any]` → schema validation
- `scripts/rag/ingestion/ingester.py` — `first_data.get("url", "")` 等の silent fallback → 型チェック後変換

**Out of scope:**
- `scripts/rag/cache.py` / `scripts/rag/enums.py` / `scripts/rag/exceptions.py` / `scripts/rag/models.py` / `scripts/rag/types.py` — クリーン済み
- `scripts/rag/repository.py` / `scripts/rag/stage.py` / `scripts/rag/utils.py` — クリーン済み
- `scripts/rag/stages/*.py` — クリーン済み
- TOML 境界での `dict[str, Any]` (`_get_cfg()`/`ConfigLoader().load()` の戻り値) — 外部境界として正当
- `scripts/rag/ingestion/crawler.py` の config `dict[str, Any]` — TOML 境界として正当

## Assumptions

1. `assert isinstance(result, list)` (pipeline.py:133) は `get_embedding()` の戻り値をチェックしている。`get_embedding()` は `list[float] | None` を返す。`None` は前行の `isinstance(result, Exception)` で除外済み。→ `if not isinstance(result, list): raise TypeError(...)` に変換。
2. `str(body.get("context", ""))` (pipeline.py:208) は HTTP RAG サービスのレスポンス body から context を取得。`body["context"]` が `str` かチェックし、そうでなければ `ValueError` raise。
3. `ingester.py` の `except Exception:` (line 59) は `__del__` での `self._client.close()` → `except OSError`。
4. `ingester.py` の `except Exception as e:` (line 280) は ThreadPoolExecutor の future 結果取得 → `except (OSError, RuntimeError, ValueError) as e:`。
5. `ingester.py` の `except Exception:` (line 312) は `ingest_url_group` 呼び出し → `except (OSError, RuntimeError, ValueError, sqlite3.OperationalError)`。
6. `chunk_splitter.py` の `except Exception:` (line 72) は `process_file` 呼び出し → `except (OSError, RuntimeError, ValueError)`。
7. `llm.py` の `_mqe_prompt(cfg: dict[str, Any])` — `cfg` を `RagLLM._cfg` から取得するので引数から除去できる。または `cfg.get("mqe_prompt_template", "")` など specific アクセスは許容範囲。
8. `ingester.py` の `first_data.get("url", "")` 等 — `first_data` は `read_json_file()` が返す `ChunkDocument` (前回リファクタ済み) なので属性アクセスに変更可能。ただし `read_json_file` は以前 `ChunkDocument` を返すように変更済みかを確認する必要がある。

## Unknowns

| Unknown | Resolution | Blocking |
|---|---|---|
| `ingester.py` の `first_data` の型 | **解消**: `_read_chunk_json` は `dataclasses.asdict(read_json_file(path))` を返すため `dict[str, Any]` のまま。`.get()` アクセスは正当 (dict の外部境界)。変更不要。 | no |

| `pipeline.py` の `body.get("selected_hits", [])` | **解消**: fallback `[]` は正当 (HTTP サービス optional field)。変更不要。 | no |

### Step 1: `rag/pipeline.py` — assert → TypeError + str() → validated

1. Line 133: `assert isinstance(result, list)` →
   ```python
   if not isinstance(result, list):
       logger.warning(f"Unexpected embedding result type for '{q}': {type(result).__name__}")
       continue
   ```
   (Warning + continue が適切。TypeError raise だと上位の asyncio.gather が catch できない)

2. Line 208: `str(body.get("context", ""))` →
   ```python
   context_raw = body.get("context")
   if context_raw is None:
       return ""
   if not isinstance(context_raw, str):
       raise ValueError(
           f"RAG service 'context' field must be str, got {type(context_raw).__name__}"
       )
   return context_raw
   ```
