## 1. Goal
- Enforce mandatory tool approval checks inside `execute_all_tool_calls()` so that write, delete, shell, and GitHub write tools cannot execute without passing `run_approval_checks()`.

## 2. Scope
- **In-Scope**:
  - `scripts/agent/tool_runner.py` ‚Äî approval gate integration (`_run_approval_gate`, `_build_denied_messages`)
  - `scripts/agent/tool_approval.py` ‚Äî `run_approval_checks()`, plan-mode block, gitops block
  - `scripts/agent/llm_turn_runner.py` ‚Äî calls `execute_all_tool_calls()` with no bypass
  - `tests/test_tool_runner.py` ‚Äî four new approval-enforcement tests
- **Out-of-Scope**:
  - DB schema changes
  - New module/package creation
  - Workflow-level approval logic (kept separate per requirement)
  - `agent/tool_policy.py`, `agent/tool_audit.py` (no changes needed)

## 3. Assumptions
- The implementation has been verified as **already complete** at the time of planning (see investigation below).
- All 21 tests in `tests/test_tool_runner.py` and 88 tests in approval test files pass without modification.

- The `_GITHUB_WRITE_TOOLS` frozenset in `tool_approval.py` is the authoritative list of GitHub write operations subject to `gitops_push_blocked`.

## 8. Risks & Mitigations
- **Risk**: `skip_in_workflow_mode=True` when `workflow_require_approval=True` and no pending approval could silently bypass per-tool approval in edge cases ‚Üí **Mitigation**: The condition also checks `not getattr(ctx.workflow, "approval_pending", False)` so approval is only skipped when workflow approval is active and not pending; add integration test covering this branch if gaps found.
- **Risk**: High cyclomatic complexity in `_execute_with_dag` (D=26) could hide edge cases where approved_calls bypass is missed ‚Üí **Mitigation**: Approval gate runs before `_execute_with_dag` is entered; no path inside the DAG executor can re-introduce unapproved calls.
- **Risk**: Future callers of `execute_all_tool_calls()` could set `ctx.cfg.workflow_require_approval=True` to bypass per-tool approval unintentionally ‚Üí **Mitigation**: Document the `skip_in_workflow_mode` contract in the docstring; consider adding an assertion or warning when both flags are set unexpectedly.
