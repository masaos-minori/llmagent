# Work Plan: `scripts/db/` リファクタリング — except Exception撲滅・assert除去・dict戻り値DTO化

## Goal

`scripts/db/` 配下の全 Python ファイルから `except Exception`・`assert` ビジネスロジック・`dict[str, Any]` 戻り値を排除し、具体的例外型・`RuntimeError` によるガード・typed DTO に統一する。

## Scope

**In scope:**
- `scripts/db/helper.py` — 4箇所の `except Exception` → 具体型; `health_check()` 戻り値 → `DbHealthMetrics` DTO; `checkpoint()` 戻り値は `db/models.py` の `WalCheckpointCounts` に変更（現状は `dict[str, int]` を返しているが `WalCheckpointCounts` は既存）; `params` 型 `dict[str, Any] | tuple[Any, ...]` の見直し
- `scripts/db/store.py` — 3箇所の `assert cur.lastrowid is not None` → `RuntimeError` raise; `doc_get()`/`doc_list()`/`session_list()`/`message_list()` の `dict[str, Any]` 戻り値 → typed DTO (`DocumentRow`, `SessionRow`, `MessageRow`)
- `scripts/db/create_schema.py` — 3箇所の `except Exception` → `sqlite3.OperationalError`; `if __name__` ブロックの `except Exception` → `(RuntimeError, sqlite3.DatabaseError, OSError)`
- `scripts/db/models.py` — `DbHealthMetrics`, `DocumentRow`, `SessionRow`, `MessageRow` DTO 追加
- 新規 DTO は `db/models.py` に追加（既存 `WalCheckpointCounts`/`PurgeCounts`/`ToolResultRow` と同居）

**Out of scope:**
- `scripts/db/config.py` — 変更不要（純粋な config DTO 定義、クリーン）
- `scripts/db/maintenance.py` — 既に適切な例外処理済み（前回リファクタで対応済み）
- `scripts/db/tool_results.py` — 既に `ToolResultRow` DTO 返却済み（前回対応済み）
- `scripts/db/__init__.py` — 空ファイル

## Assumptions

1. `helper.py` の `begin_immediate()` / `begin_exclusive()` 内の `except Exception: pass` は `except sqlite3.OperationalError: pass` に変更する（ROLLBACK失敗は OperationalError か接続切れのみ）。
2. `helper.py` の `close()` 内 `except Exception` は `except OSError` に変更する（接続クローズ失敗は OSError）。
3. `helper.py.__init__` の `except Exception` は `build_db_config()` が raise する例外型を確認して変更する（`ValueError` / `OSError` / `FileNotFoundError`）。
4. `store.py` の `assert cur.lastrowid is not None` は `RuntimeError("INSERT did not produce a lastrowid")` に変更する。
5. `store.py` の `dict[str, Any]` 戻り値は `sqlite3.Row` を DTO に変換する。`DocumentRow`, `SessionRow`, `MessageRow` は `db/models.py` に定義する。
6. `create_schema.py` の `except Exception` は `sqlite3.OperationalError` / `sqlite3.DatabaseError` に変更する。DDL の失敗はこれらのみ。
7. `helper.py.checkpoint()` は `dict[str, int]` を返しているが、`WalCheckpointCounts` に変更する（`db_maintenance_service.py` 側の `raw.pages_checkpointed` アクセスは既に対応済み）。
8. `helper.py.health_check()` の新 DTO `DbHealthMetrics` は `db/models.py` に定義する。

## Unknowns

| Unknown | Resolution | Blocking |
|---|---|---|
| `build_db_config()` が raise する例外型 | **解消**: `ValueError` のみ raise（全 raise が ValueError）。`except Exception` → `except ValueError` に変更。 | no |
| `health_check()` 戻り値の caller | **解消**: `db_maintenance_service.py:50` が `raw.get("integrity")` / `raw.get("db_size_bytes", 0)` でアクセス。DTO 化後は `raw.integrity` / `raw.db_size_bytes` に変更必要。 | no |
| `checkpoint()` 戻り値の caller | **解消**: `maintenance.py:74` が `raw["busy"]`/`raw["pages_in_wal"]`/`raw["pages_checkpointed"]` でアクセス。`WalCheckpointCounts` 返却後は dict アクセス → 属性アクセスに変更必要。 | no |
| `store.py` dict戻り値の callers | **解消**: `grep -rn "doc_get\|doc_list\|session_list\|message_list" scripts/` → agent 層の AgentSession 経由でアクセスするため直接 import なし。DTO 変更は store.py のみで完結。 | no |

## Design

### `DbHealthMetrics` DTO (追加先: `db/models.py`)

```python
@dataclass(frozen=True)
class DbHealthMetrics:
    journal_mode: str
    integrity: str
    page_count: int
    page_size: int
    freelist_count: int
    db_size_bytes: int
```

### `DocumentRow` DTO (追加先: `db/models.py`)

```python
@dataclass(frozen=True)
class DocumentRow:
    url: str
    lang: str | None
    chunk_count: int
    fetched_at: str
```

### `SessionRow` DTO (追加先: `db/models.py`)

```python
@dataclass(frozen=True)
class SessionRow:
    session_id: int
    title: str | None
    created_at: str
    is_current: bool
```

### `MessageRow` DTO (追加先: `db/models.py`)

```python
@dataclass(frozen=True)
class MessageRow:
    message_id: int
    session_id: int
    role: str
    content: str
    created_at: str
```

### `checkpoint()` 戻り値変更

`helper.py` の `checkpoint()` メソッドは現在 `dict[str, int]` を返す。`WalCheckpointCounts` に変更する（`db_maintenance_service.py` の `.pages_checkpointed` アクセスは既に対応済み）。

## Implementation Steps

### Step 1: `db/models.py` — DTO 追加

- `DbHealthMetrics`, `DocumentRow`, `SessionRow`, `MessageRow` を追加
- 完了基準: mypy, ruff

### Step 2: `db/helper.py` — except Exception → 具体型 + DTO 戻り値

- `__init__` の `except Exception` → `except ValueError`
  (build_db_config は ValueError のみ raise、確認済み)
- `close()` の `except Exception` → `except OSError`
- `begin_immediate()` / `begin_exclusive()` の inner `except Exception: pass` → `except sqlite3.OperationalError: pass`
- `health_check() -> dict[str, Any]` → `health_check() -> DbHealthMetrics`
- `checkpoint() -> dict[str, int]` → `checkpoint() -> WalCheckpointCounts`
- `params: dict[str, Any] | tuple[Any, ...]` → `params: dict[str, object] | tuple[object, ...]`
  (Any を局所化; sqlite3 は Any パラメータを受け入れるが dict/tuple 型は保持)
- 完了基準: `grep -n "except Exception" scripts/db/helper.py` → 0 hits; mypy; pytest

### Step 3: `db/store.py` — assert → RuntimeError + dict → DTO

- `assert cur.lastrowid is not None` (3箇所) → `if cur.lastrowid is None: raise RuntimeError("INSERT did not produce a lastrowid")`
- `doc_get()` → `DocumentRow | None`
- `doc_list()` → `list[DocumentRow]`
- `session_list()` → `list[SessionRow]`
- `message_list()` → `list[MessageRow]`
- 完了基準: `grep -n "assert cur" scripts/db/store.py` → 0 hits; `grep -n "dict\[str, Any\]" scripts/db/store.py` → 0 hits; mypy

### Step 4: `db/create_schema.py` — except Exception → 具体型

- `create_rag_schema()` 内 `except Exception` → `except (sqlite3.OperationalError, sqlite3.DatabaseError)`
- `create_session_schema()` 内 `except Exception` → `except (sqlite3.OperationalError, sqlite3.DatabaseError)`
- `if __name__ == "__main__"` ブロック `except Exception` → `except (RuntimeError, sqlite3.DatabaseError, OSError)`
- 完了基準: `grep -n "except Exception" scripts/db/create_schema.py` → 0 hits; mypy
