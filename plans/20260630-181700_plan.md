# Plan: Add registry-level tests for removed `/note` command

## Goal
`tests/test_cmd_registry_note_removal.py` を拡張し、`CommandRegistry.dispatch()` を実際に呼び出して `/note` 系コマンドがすべて `False` を返すことを確認するregistry-levelテストを追加する。

## Scope
**In-Scope**:
- `tests/test_cmd_registry_note_removal.py` に `CommandRegistry.dispatch()` を呼び出すテストを追加
- `/note add hello`, `/note list`, `/note delete 1`, `/note pin 1`, `/note unpin 1`, `/note search hello` の6入力それぞれが `dispatch()` で `False` を返すことの検証
- `/memory` コマンドが dispatch される guard test の追加（`dispatch()` で `True` を返すこと）
- `_NotesMixin` に依存しないこと（依存が存在しないことを明示的に保証）

**Out-of-Scope**:
- `registry.py` の変更
- 他のコマンドのdispatchテスト
- `_NotesMixin` の削除（既に存在しない）
- plugin_registry 経由のdispatchテスト

## Assumptions
- `CommandRegistry.__init__` は `_COMMANDS` に登録されたすべてのハンドラが `self` に存在することを検証する（fail-fast）。スタブは全ミックスインが要求するメソッド名を満たすため、`AgentContext` の完全な構造ではなく `SimpleNamespace` ベースの軽量スタブで代替可能
- `dispatch()` は `asyncio` を使うため `@pytest.mark.asyncio` が必要
- 既存の `_Ctx` スタブ（`test_cmd_registry_note_removal.py` 内）は `CommandRegistry` のインスタンス化に不十分な可能性がある。実際に `CommandRegistry(ctx)` を呼んで確認し、必要に応じてスタブを拡充する
- `plugin_registry.iter_commands()` は空を返すため `/note` がプラグイン経由でも登録されていないことを前提とする
