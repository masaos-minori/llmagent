# Redesign memory source-of-truth model

**Requirement:** `requires/20260624_62_require.md`

## Goal

Eliminate rebuild inconsistencies and deletion resurrection by making SQLite the authoritative source of truth for memory state, and treating JSONL as an append-only archive/export log rather than a rebuild source.

## Decision: SQLite as source of truth

**Chosen direction: Option 2 — SQLite as source of truth; JSONL as append-only archive.**

Rationale:
1. SQLite already correctly tracks all mutations: `delete()`, `clear_by_session()`, `pin()`, `unpin()` all operate on SQLite atomically via `begin_immediate()`.
2. JSONL is `append-only` — expressing mutations (delete, pin) in an append-only store requires event sourcing (tombstones, update events, ordered replay) which is significantly more complex.
3. `rebuild_from_jsonl()` is the root cause: it re-inserts all JSONL entries including deleted ones, because JSONL has no tombstones.
4. Disaster recovery is better served by SQLite backup (periodic `.bak` files or file copy) rather than JSONL replay.
5. Flipping write order (SQLite first, JSONL second) ensures SQLite is the authoritative write; JSONL write failure does not corrupt state.

6. The existing `/memory rebuild` command becomes `/memory import` — imports from JSONL archive but warns that deletions are not replayed.

## Design

### Root Cause

`MemoryStore.rebuild_from_jsonl()` (called by `/memory rebuild`) deletes all SQLite rows and re-inserts all JSONL entries. Because `MemoryStore.delete()` and `clear_by_session()` remove entries from SQLite but JSONL is append-only (no tombstones), deleted entries reappear after rebuild. Pin state (from `pin()` / `unpin()`) is also lost because JSONL entries retain the original `pinned` value.

### Fix

#### 1. `jsonl_store.py` — update module docstring

```python
"""agent/memory/jsonl_store.py
Append-only JSONL archive for MemoryEntry.

Each line is one JSON object representing a MemoryEntry at the time of write.
The JSONL file is an append-only archive; it does NOT record mutations (delete,
pin, unpin). SQLite (via MemoryStore) is the authoritative source of truth.

Use read_all() for audit, export, or one-time import. Do not use it to rebuild
authoritative state — use MemoryStore directly or restore from a SQLite backup.
"""
```

#### 2. `store.py` — rename method and update docstring

```python
def import_from_jsonl(
    self,
    jsonl_store: object,
    *,
    dry_run: bool = False,
) -> tuple[int, int]:
    """Import entries from a JSONL archive into SQLite memories/FTS/vec tables.

    WARNING: This does NOT replay deletions, pin state, or dedup history.
    Entries deleted from SQLite will be re-inserted. This is intended for
    initial import from an external archive only — NOT for disaster recovery
    or routine consistency repair.

    For consistency repair (memories vs FTS vs vec out of sync), use
    repair_index() instead.

    Returns (jsonl_count, inserted_count).
    When dry_run=True, returns (jsonl_count, 0) without modifying SQLite.
    """
```

#### 3. `ingestion.py` — flip write order

Change `_persist_entry_with_dedup()`:
```python
# Before (JSONL first, then SQLite)
await self._jsonl.write(entry)
self._store.upsert(entry, embedding=embedding)

# After (SQLite first = authoritative, JSONL second = archive)
self._store.upsert(entry, embedding=embedding)
try:
    await self._jsonl.write(entry)
except Exception:
    logger.warning(
        "memory.jsonl_write_failed memory_id=%r — entry saved in SQLite only",
        entry.memory_id,
    )
```

Same change in `_persist_entry()`.

#### 4. `cmd_memory.py` — update call site and consistency check

Line 313: `rebuild_from_jsonl` → `import_from_jsonl`

Line 301 (consistency check message): Change from comparing JSONL vs SQLite count to SQLite vs FTS vs vec:
```python
# Before: consistency = (jsonl_count == report.memories) and (memories == fts) and (vec ok)
# After:  consistency = (memories == fts) and (vec ok)
# Remove JSONL count from the consistency equation entirely
```

Update consistency row display:
```python
rows = [
    # Remove: ["JSONL records", str(jsonl_count)],
    ["SQLite memories (authoritative)", str(report.memories)],
    ["JSONL archive records", str(jsonl_count)],  # info only; not part of consistency check
    ["FTS5 rows", str(report.fts)],
    ["Vec rows", str(report.vec)],
    ...
    ["Consistent", "Yes" if ok else "NO - use /memory rebuild to repair FTS/vec index"],
]
```

Update `/memory rebuild` command docstring/description to clarify it now calls `import_from_jsonl()` with the deletion-not-replayed warning.
