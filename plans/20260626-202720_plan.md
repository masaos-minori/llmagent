## Goal
Implement path access control for mdq-mcp using configured allowed roots to prevent unauthorized file access.

## Scope
**In-Scope**:
- Add `allowed_dirs` config to `config/mdq_mcp_server.toml`
- Create path authorization helper function
- Apply path authorization to `index_paths`, `refresh_index`, `outline`
- Prevent `..` traversal and symlink escape

**Out-of-Scope**:
- Adding new tools or features
- Changing existing tool behavior beyond access control

## Assumptions
1. Empty allowlist = fail-closed (deny all)
2. Use `Path.resolve()` for path resolution before authorization
3. Symlink escape attempts are denied by checking if the resolved path starts with any allowed root

## Affected Areas & Tool Evidence
- **Affected Files**:
  - `config/mdq_mcp_server.toml` — add `allowed_dirs` config
  - `scripts/mcp/mdq/service.py` — add path auth to outline, refresh_index
  - `scripts/mcp/mdq/indexer.py` — add path auth to index_paths

- **Blast Radius**: Medium — changes path authorization for all path-based tools. Existing valid paths will still work; invalid paths will be denied.

## Implementation Steps
1. **Phase 1: Add config**
   - Add `allowed_dirs = []` to `config/mdq_mcp_server.toml`
2. **Phase 2: Create path authorization helper**
   - Create `_authorize_path(path: Path, allowed_dirs: list[str]) -> bool` function
   - Resolve path with `Path.resolve()`
   - Check if resolved path starts with any allowed root (using `os.path.commonpath`)
   - Reject if allowlist is empty
3. **Phase 3: Apply to index_paths**
   - Load allowed_dirs from config
   - Authorize each path before indexing
4. **Phase 4: Apply to refresh_index**
   - Same as index_paths
5. **Phase 5: Apply to outline**
   - Same authorization pattern
