# Plan: Expose use_rrf and rrf_k more clearly at runtime

## Goal

Improve runtime visibility of RRF (Reciprocal Rank Fusion) configuration by plugging `rrf_k` from config to the pipeline, adding logging at fusion stage initialization, and augmenting debug output to show fusion mode and parameters.

## Scope

**In scope:**
- Pass `rrf_k` from config to `FusionStage` (currently hardcoded to 60)
- Add `rrf_k` field to `RagConfigProtocol`
- Add `rrf_k` field to `RagPipelineConfig` and wire it through config adapter
- Log fusion config at pipeline initialization
- Augment CLI debug output to show `use_rrf` mode and `rrf_k` value
- Add optional logging in `RagScorer.rrf_merge()` for per-query metrics

**Out of scope:**
- Changing the RRF algorithm itself
- Adding RRF metrics to tracing/telemetry (beyond basic logging)
- Modifying the memory retriever's RRF implementation (separate code path)

## Assumptions

1. The current `rrf_k=60` default is correct; plan just makes it configurable.
2. Adding config fields should be backward compatible (use defaults if not specified).
3. Logging at pipeline init is sufficient; per-query RRF logging is optional/low priority.

## Unknowns

## Implementation Steps

1. **Add `rrf_k` to `RagConfigProtocol`** in `shared/types.py:54`
   - Add `rrf_k: int` field
   - Default value: 60 (consistent with current hardcoded default)

2. **Wire `rrf_k` from MCP config** in `mcp/rag_pipeline/models.py`
   - Add `rrf_k: int = 60` to `RagPipelineConfig` dataclass
   - Update `build_rag_cfg_adapter()` to pass `rrf_k` from config

3. **Pass `rrf_k` to FusionStage** in `pipeline.py:208`
   - Change `FusionStage(use_rrf=self._cfg.use_rrf)` to include `rrf_k=...`
   - Read from `self._cfg.fusion.rrf_k` or `self._cfg.rrf_k` (whichever is available)

4. **Log fusion config at pipeline initialization** in `pipeline.py:134-137`
   - Add `logger.info("Pipeline fusion: use_rrf=%s, rrf_k=%d", ...)`
   - Update status reporting to show actual `rrf_k` value used

5. **Augment CLI debug output** in `cli_view.py:127-159`
   - Show fusion mode: `[RRF]` or `[DEDUP]` based on `use_rrf`
   - Show `rrf_k` value: `rrf_k=60`
   - Example: `  [debug] RRF merge (mode=RRF, k=60): 15 unique candidates (top 5):`

6. **Optional: Add per-query RRF logging** in `repository.py:187-217`
   - Log number of input lists, total unique chunk_ids
   - Use DEBUG level to avoid noise in normal operation
