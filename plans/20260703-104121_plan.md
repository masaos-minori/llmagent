# Plan: Normalize prefix command arguments in `CommandRegistry.dispatch()`

## Goal

Strip leading and trailing whitespace from argument strings passed to built-in and plugin prefix command handlers in `CommandRegistry`.

## Scope

### In-Scope

- Apply `.strip()` to `args` in the built-in prefix dispatch branch (`dispatch()`)
- Apply `.strip()` to `args` in the plugin prefix dispatch branch (`_dispatch_plugin()`)
- Add or extend a unit test that asserts handlers receive stripped args

### Out-of-Scope

- Changing individual command handler parsers (`cmd_db.py`, `cmd_mdq.py`, etc.)
- `/exit` handling
- Plugin command conflict policy
- Removing defensive `strip()` calls already present in handlers (separate cleanup task)

## Assumptions

1. The only callers of `dispatch()` are the REPL loop and existing tests; no callers expect un-stripped args.
2. Stripping args does not break multi-word arguments because `.strip()` only removes leading/trailing whitespace, not internal whitespace.
3. The plugin registry's `iter_commands()` contract is unchanged; only the `args` value passed to handlers changes.
4. `test_command_registry_consistency.py` covers handler signature validation but does not assert on args content; a new test is needed.

## Implementation Steps

### Step 1 — Normalize args in `dispatch()` (built-in prefix branch)

File: `scripts/agent/commands/registry.py`, line 123.

Change:
```python
# before
args = line[len(cmd.name) :]
# after
args = line[len(cmd.name) :].strip()
```

Completion criteria: `args` value for `/db help` is `"help"` (no leading space).

### Step 2 — Normalize args in `_dispatch_plugin()` (plugin prefix branch)

File: `scripts/agent/commands/registry.py`, line 145.

Change:
```python
# before
args = line[len(cmd_name) :]
# after
args = line[len(cmd_name) :].strip()
```

Completion criteria: Plugin handlers receive stripped args for inputs like `/myplugin  arg`.

### Step 3 — Add dispatch args-normalization test

File: `tests/test_command_registry_consistency.py` (or a new file `tests/test_command_registry_dispatch.py`).

Add a parametrized async test that:
- Calls `registry.dispatch("/db help")` with a mock `_cmd_db` handler and asserts it received `"help"`.
- Calls `registry.dispatch("/mcp")` and asserts the handler received `""`.

Completion criteria: `uv run pytest tests/test_command_registry_consistency.py` (or new file) passes with the new test cases.
