## Design

### Consolidated Fail-Open Settings Table

| Setting | Fail-Open Condition | Startup Warning | Risk Level |
|---|---|---|---|
| `workflow_allowlist` | empty list | WARNING | High — allows all workflow patterns |
| `allowed_repos_mode` | `"fail_open"` | WARNING | High — allows all repos |
| `path_denylist` | empty list | INFO | Medium — no path restrictions |
| `protected_branches` | empty list | INFO | Medium — no branch protection |

### Pre-Production Checklist

- [ ] `workflow_allowlist` is non-empty or explicitly set to restricted patterns
- [ ] `allowed_repos_mode` is `"fail_closed"` (default for production)
- [ ] `path_denylist` contains at minimum `/etc`, `/root`, production credential paths
- [ ] `protected_branches` contains at minimum `main`, `master`
