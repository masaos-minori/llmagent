## Design

### 問題の詳細

既存テストの構造:

```python
with ThreadPoolExecutor(max_workers=4) as executor:
    results = list(executor.map(lambda _: AgentSession(), range(8)))
# executor がここで閉じる
for s in results:  # 逐次実行 — 並列ではない
    s.start()
    sessions.append(s)
```

`start()` の呼び出しが executor の外側にあるため、逐次実行になっている。これでは concurrent session ID 生成の競合を検知できない。

### 修正方針

`AgentSession()` の生成と `start()` の呼び出しを1つの関数にまとめて executor 内で実行する:

```python
def _make_and_start() -> AgentSession:
    s = AgentSession()
    s.start()
    return s

with ThreadPoolExecutor(max_workers=4) as executor:
    sessions = list(executor.map(lambda _: _make_and_start(), range(8)))
```

両テストで同じ修正パターンを適用する。
