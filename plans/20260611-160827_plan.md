# Work Plan: リファクタリング — DTO強化・print除去・except Exception撲滅

## Goal

7つの優先ファイルから `except Exception`・直接 `print()`・ルーズな dict/tuple 戻り値を排除し、typed DTO・injected output interface・fail-fast バリデーションに置き換える。

## Scope

- `scripts/db/maintenance.py` — `except Exception` 2箇所、`dict[str, int]` 戻り値 2関数
- `scripts/agent/services/session_title.py` — LLM JSON レスポンスのスキーマバリデーション
- `scripts/agent/services/context_view.py` — `str(content)` 無条件変換・`m.get("content") or ""` (None/空文字列の等価視)
- `scripts/agent/commands/cmd_db.py` — `str(r["url"])` 等の無条件変換・`formatter` 直接 print
- `scripts/agent/services/mcp_install.py` — 調査済み: 既に厳格バリデーション済み → 軽微修正のみ
- `scripts/db/tool_results.py` — `get` が `dict | None`、`list_recent` が `list[dict]` → typed DTO化
- `scripts/agent/commands/cmd_session.py` — raw dict アクセス (`r['session_id']` 等)、`r["title"] or ""`
- `scripts/agent/commands/formatter.py` — `print()` を Callable output interface に変換
- `scripts/agent/services/models.py` — 新 DTO 追加: `WalCheckpointCounts`、`PurgeCounts`、`ToolResultRow`、`DocumentRow`、`SessionRow`

## Assumptions

1. `formatter.py` の `print()` を Callable[..., None] output interface に変換する際、既存テストは `capsys` で output をキャプチャしているため、interface 化後も同じ出力が得られる。
2. `LLMMessage` は TypedDict なので `m["role"]` でアクセス可能 (KeyError は設計上起こらない前提)。
3. `ToolResultRow` dataclass に変換後、`cmd_tooling.py` が `result["tool_name"]` 等でアクセスしているため `cmd_tooling.py` も変更対象になる。ただし priority 外なので adapter 対応で最小変更に留める。
4. `list_sessions()` の戻り値は `AgentSession` 側に `SessionRow` DTO を追加する必要があるが、今回は `cmd_session.py` 内で dict → DTO への変換層を追加するアプローチをとる。

### ステップ 8: formatter.py — OutputWriter Protocol 化

- `OutputWriter` Protocol を `agent/commands/output_writer.py` (新規) に定義:
  ```python
  class OutputWriter(Protocol):
      def success(self, msg: str) -> None: ...
      def error(self, msg: str) -> None: ...
      def no_data(self, msg: str) -> None: ...
      def validation_error(self, msg: str) -> None: ...
      def table(self, headers: list[str], rows: list[list[str]]) -> None: ...
      def kv_list(self, pairs: list[tuple[str, str]], key_width: int = 22) -> None: ...
  ```
- `PrintOutputWriter` (concrete) を同ファイルに実装 (既存 print ロジックをラップ)
- `formatter.py` の既存関数は `PrintOutputWriter` インスタンスに委譲する shim として残す (既存テストへの後方互換)
- 完了基準: mypy, 既存テスト通過

| `OutputWriter` DI 化により全 cmd_*.py のコンストラクタ変更が波及 | high | `PrintOutputWriter` をデフォルト引数にして既存コードに変更なし |
