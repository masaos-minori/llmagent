# Plan: Make memory-layer activation and startup behavior production-friendly

Source: `requires/20260625_94_require.md`

---

## Goal

memory layer の activation 状態（disabled / FTS-only / hybrid）を startup 出力と `/memory status` で明示的に表示する。半設定状態を検出しやすくする。

---

## Scope

**In-Scope**
- `scripts/agent/memory/services.py`: `MemoryServices` に `get_status()` メソッド追加
- `scripts/agent/memory/embedding_client.py`: `EmbeddingClientStatus` の活用（すでに実装済み）
- startup 出力: memory mode を banner に追加
- `/memory status` コマンド: disabled / degraded / FTS-only / hybrid モードを表示
- docs: `05_agent_08_configuration.md`, `05_agent_10_operations-and-observability.md`

**Out-of-Scope**
- 自動 memory setup
- startup-only policy 自体の変更

---

## Assumptions

1. `EmbeddingClientStatus.enabled` はすでに存在（`embedding_client.py` line 33-38）。
2. memory mode は embedding 状態と組み合わせで決まる:
   - `use_memory_layer=False` → disabled
   - `use_memory_layer=True` + embedding disabled → FTS-only
   - `use_memory_layer=True` + embedding enabled + circuit open → degraded (FTS-only)
   - `use_memory_layer=True` + embedding enabled + circuit closed → hybrid
3. `/memory status` は現在の mode を1行で表示する。
