# Plan: Ensure watchdog never observes pending MCP reload config

Source requirement: `requires/20260708_30_require.md` (H-8)

## Goal

Guarantee that `watchdog_loop()` / `_watchdog_check_http()` in
`agent/repl_health.py` always see the startup-time `ctx.cfg.mcp.mcp_servers`
values, never a pending-restart config, and add a watchdog-focused test plus
a documenting comment confirming this.

## Scope

In scope:
- `scripts/agent/services/config_reload.py` — no new change beyond the H-1
  design (`plans/20260709-094640_plan.md`); this requirement is a
  watchdog-focused confirmation of that same no-mutation guarantee.
- `scripts/agent/repl_health.py` — add a short docstring/comment on
  `watchdog_loop()` documenting that it reads the startup-time MCP snapshot
  and is unaffected by `/reload`.
- `tests/test_watchdog.py` — add a test proving watchdog input is unchanged
  across a reload classification call.

Out of scope:
- Any change to `_watchdog_check_http()`'s decision logic — it already reads
  `srv_cfg.url` and `srv_cfg.startup_mode` from the live
  `ctx.cfg.mcp.mcp_servers` dict passed in by `watchdog_loop()`; no pending-
  config overlay exists anywhere in this path.

## Assumptions

1. `watchdog_loop()` re-reads `ctx.cfg.mcp.mcp_servers.items()` fresh on every
   iteration of its `while True` loop (`scripts/agent/repl_health.py:642`),
   and `_watchdog_check_http()` receives `srv_cfg` by reference from that same
   dict, using `srv_cfg.url` (line 559) and `srv_cfg.startup_mode` (line 601)
   directly — verified by reading `scripts/agent/repl_health.py:536-652`.
2. There is no intermediate cache, overlay, or "pending config" structure
   between `ctx.cfg.mcp.mcp_servers` and the watchdog — verified by the same
   reading; the loop takes no snapshot copy, it iterates the live dict.
3. Given assumptions 1-2, the H-1 design's no-mutation guarantee
   (`_classify_mcp_server_changes()` never writes to
   `ctx.cfg.mcp.mcp_servers`) is sufficient by construction to satisfy every
   acceptance criterion in this requirement — no watchdog-side code change is
   needed, only a confirming test and a documentation comment.
4. `tests/test_watchdog.py` has no existing reload-interaction test —
   verified by `grep -n "reload" tests/test_watchdog.py` (no matches).

## Design

No new production logic. Two additions:

1. Documentation comment on `watchdog_loop()`:

```python
async def watchdog_loop(ctx: AgentContext) -> None:
    """Periodically probe MCP server health and restart via lifecycle manager on failure.

    Runs until cancelled (e.g. when the REPL exits).
    Restart attempts per server are capped at mcp_watchdog_max_restarts to
    prevent infinite restart loops.

    Reads `ctx.cfg.mcp.mcp_servers` directly on every iteration. MCP server
    definitions are restart-time snapshots (see ConfigReloadService); `/reload`
    never mutates this dict, so the watchdog always observes the startup-time
    URL and startup_mode, never a pending-restart value.
    """
```

2. Test in `tests/test_watchdog.py`:

```python
async def test_watchdog_sees_unchanged_config_after_reload_classification() -> None:
    """Reload classification must not alter the dict watchdog_loop reads from."""
    from agent.services.config_reload import ConfigReloadService
    from shared.mcp_config import McpServerConfig, StartupMode, TransportType

    old_srv = McpServerConfig(
        transport=TransportType.HTTP,
        url="http://localhost:8080",
        cmd=[],
        startup_mode=StartupMode.SUBPROCESS,
    )
    ctx = MagicMock()
    ctx.cfg.mcp.mcp_servers = {"svc": old_srv}

    new_srv = McpServerConfig(
        transport=TransportType.HTTP,
        url="http://127.0.0.1:9999",
        cmd=[],
        startup_mode=StartupMode.PERSISTENT,
    )
    svc = ConfigReloadService(ctx)
    with patch("agent.config_builders._build_mcp_servers", return_value={"svc": new_srv}):
        outcome = svc._classify_mcp_server_changes(ctx, {})

    # What watchdog_loop() would read is exactly this object, unchanged:
    watchdog_visible = ctx.cfg.mcp.mcp_servers["svc"]
    assert watchdog_visible is old_srv
    assert watchdog_visible.url == "http://localhost:8080"
    assert watchdog_visible.startup_mode == StartupMode.SUBPROCESS
    assert "mcp/svc.url" in outcome.needs_restart
    assert "mcp/svc.startup_mode" in outcome.needs_restart
```

## Risks

- **Risk**: This requirement is satisfied entirely by H-1's guarantee; if a
  future change to `config_reload.py` reintroduces mutation without updating
  this plan's test, the regression could go unnoticed if this test is skipped
  or deleted.
  → **Mitigation**: The new watchdog-specific test asserts
  `watchdog_visible is old_srv` (identity, not just equality) — a stronger
  guard than a value-equality check, catching both mutation-in-place and
  replacement-with-a-new-object regressions.
- **Risk**: The documentation comment could drift from the code if
  `_classify_mcp_server_changes()` is renamed again in a future change.
  → **Mitigation**: Comment references the concept ("MCP server definitions
  are restart-time snapshots") rather than the literal function name, so it
  remains accurate even if the function is renamed.
