## Design

### Canonical Correlation Keys Table

| Key | Type | Generated by | Appears in |
|---|---|---|---|
| `X-Session-Id` | HTTP header | Agent (per session) | Transport logs, server audit |
| `X-Request-Id` | HTTP header | Agent (per tool call) | Transport logs, server audit |
| `server_key` | Log field | Agent routing | Agent dispatch logs |
| `tool_name` | Log field | Agent routing | Agent logs, server audit |

### Operator Tracing Flow

```
Agent dispatch log → find X-Request-Id
→ Search transport logs for X-Request-Id
→ Search server audit log for X-Request-Id
→ Check health state for server_key at that timestamp
→ Check watchdog actions if health changed
```
