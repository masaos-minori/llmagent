| `ingester.py` vs `rag_ingester.py` の命名非一貫性 | `03_spec_rag.md` / `03_rag-ingestion-run.md` は `scripts/rag/ingestion/ingester.py` と記載。`03_rag-ref-ingester.md` のタイトルのみ `rag_ingester.py` と記載。実ファイルは `ingester.py`（クラス名は `RagIngester`）。 → 90 ファイルに "Document inconsistency" として記載 | 解消済み |

| 外部 RAG サービス委譲の未定義領域 | `03_spec_rag.md` に明示: `rag_service_url` 設定時の認証・エラー処理の仕様が未定義。`05_ref-rag.md` では `_augment_http()` の実装は記載されているが認証仕様なし | 解消済み（Needs confirmation として記録）|

### Unknowns 解消後の追加判明事項

- **インジェスト BUG-1/2/3 は同一根本原因**: `ingester._read_chunk_json()` が `dataclasses.asdict(read_json_file(path))` を経由することで `ChunkDocument` に存在しないフィールド（`chunking_strategy`, `normalized_content`, `chunk_index`）が消える。3 件のバグを 1 つの根本原因 + 3 派生 Issue として `90` ファイルに記載する
- **`use_rrf=False` 時の実装**: `FusionStage` は `use_rrf` フラグをチェックしないため、設定に関わらず常に RRF を実行する。`_dedup_hits` フォールバックは dead code 状態
- **インジェストパイプラインのステージ数表記の不一致**: `03_spec_rag.md` では「4段階」（クロール・チャンク分割・埋め込み生成・SQLite格納）と記載されている箇所がある一方、`03_rag-ingestion-run.md` では「3ステップ」（crawler → chunk_splitter → ingester）として記載。実際は 3 スクリプトで 4 処理（埋め込み生成と格納は `ingester.py` が行う）

## Design

### ドキュメント再構成の基本方針

- 入力 7 ファイルを全件読み込み、コンテンツを責務ごとに分類する
- 要件の "Keep Different Information Types Separate" (Rule 3) に従い、仕様・実装挙動・運用手順・既知の問題を明確に分離する
- 要件の "Optimize for AI Readability" (Rule 4) に従い、箇条書き・コードブロック・テーブルを積極的に活用する
- 要件の Critical Issues 14 項目はすべて `03_rag_90_inconsistencies_and_known_issues.md` に独立アイテムとして記載する
- ソースマッピング (`03_rag_99_source_mapping.md`) は、すべての重要な情報が漏れなく再構成先に移動したことを確認するための監査テーブルを含む

### 情報分類の基準

| 情報種別 | 出力先ファイル |
|---|---|
| システム全体像・アーキテクチャ | `03_rag_01_system_overview.md` |
| 取り込みパイプライン (Crawler/Splitter/Ingester) の詳細 | `03_rag_02_ingestion_pipeline.md` |
| クエリパイプライン (MQE/Search/Fusion/Rerank/Augment) の詳細 | `03_rag_03_query_pipeline.md` |
| DB スキーマ・型定義・クラス・メソッド | `03_rag_04_data_model_and_interfaces.md` |
| 設定ファイル・運用手順・トラブルシューティング | `03_rag_05_configuration_and_operations.md` |
| ドキュメント間の非一貫性・実装バグ・未定義領域 | `03_rag_90_inconsistencies_and_known_issues.md` |
| ソース→再構成先のマッピング | `03_rag_99_source_mapping.md` |
| 全ファイルへのナビゲーション | `03_rag_00_document-guide.md` |
