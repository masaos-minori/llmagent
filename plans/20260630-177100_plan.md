# Plan: Normalize `None` message content at the session persistence boundary

## Goal
`SessionMessageRepository.save()` および `save_many()` で `content=None` を `""` に正規化してDBに保存する — この変更はすでに実装済みであり、実装作業は不要である。

## Scope
**In-Scope**:
- `scripts/agent/session_message_repo.py` の `content: str | None` 対応 (実装済み)
- `tests/test_session_message_repo.py` の `TestNoneContentNormalization` クラス (実装済み)

**Out-of-Scope**:
- `db/schema_sql.py` のスキーマ変更
- `messages.content` の NULL 許容化
- DB マイグレーション

## Assumptions
- `content=None` の正規化は repository boundary (SessionMessageRepository) のみで行い、SQLiteSessionStore 側では行わない。
- `fetch_messages()` は常に `str` を返す (`""` を `None` に変換しない)。
- DB スキーマ (`content TEXT NOT NULL`) は変更しない。
