# Plan: Redesign MCP Health Semantics and Watchdog Restart Policy

## Goal

Extend the MCP `/health` response schema with `restart_recommended` and `operator_action_required` fields, and update the watchdog to restart subprocess servers only when `restart_recommended=true`.

## Scope

### In-Scope

- Add `restart_recommended` and `operator_action_required` fields to the `/health` response body in `mcp/server.py` (base `MCPServer.health()`)
- Update each concrete MCP server health implementation to classify dependency failures into restartable vs. operator-action categories
- Update `probe_mcp_health()` to return a structured result (body fields + HTTP status) instead of a bare `bool`
- Update `_watchdog_check_http()` to inspect body fields and skip restart when `restart_recommended=false`
- Fix `watchdog_loop()` to return immediately when `mcp_watchdog_interval <= 0` (currently enters `while True` with `asyncio.sleep(0)`)
- Add `tests/test_watchdog.py` with the four required test cases
- Update `tests/test_mcp_health.py` (rename from `test_mcp_server_health_status.py` per the requirement) to validate new schema fields
- Update four MCP docs to reflect the new liveness/readiness/restart separation

### Out-of-Scope

- Changes to `StartupMode` enum values or `McpServerConfig` schema
- Changes to the HTTP lifecycle manager (`lifecycle.restart()`) or subprocess spawn logic
- Changes to the eventbus health endpoint (`test_eventbus_health.py`)
- Changes to the `mdq` health check classification logic (mdq's dependencies are DB/schema — classify as `operator_action_required`)

## Assumptions

1. The existing `probe_mcp_health()` signature (`async def probe_mcp_health(http, base_url) -> bool`) is called only from `_watchdog_check_http()` and `tests/test_repl_health.py`; it can be replaced with a richer return type without breaking other callers.
2. The four concrete server health functions that currently return `HTTP 503` for missing credentials (cicd `GITHUB_TOKEN`, github `_GITHUB_TOKEN`) should classify as `operator_action_required=true, restart_recommended=false`.
3. Servers with missing filesystem dependencies (file-read, file-write, file-delete, mdq DB) should also classify as `operator_action_required=true, restart_recommended=false` (a missing workspace path cannot be fixed by restart).
4. Missing binary dependencies (git-mcp `git` not in PATH, shell-mcp `sh` not in PATH) should classify as `restart_recommended=false, operator_action_required=true` because binaries do not appear by themselves on restart.
5. Missing config dependencies (rag-pipeline `embed_url` not configured) are `operator_action_required=true, restart_recommended=false`.
6. The base `MCPServer.health()` method in `mcp/server.py` defaults to `restart_recommended=false, operator_action_required=false` when `deps` is empty (healthy), and `restart_recommended=false, operator_action_required=true` when degraded — subclasses override as needed.
7. The `test_mcp_health.py` filename the requirement refers to is the existing `tests/test_mcp_server_health_status.py`; the plan treats them as the same file.

## Implementation Steps

### Step 1 — Extend health response schema in `mcp/server.py`

Add `liveness`, `restart_recommended`, `operator_action_required` to the `MCPServer.health()` base method return dict. Default: `restart_recommended=False`, `operator_action_required=False` when healthy; `restart_recommended=False`, `operator_action_required=True` when degraded. Update the docstring to describe the new fields.

**Completion criteria:** `MCPServer().health()` returns a dict with the five keys (`status`, `ready`, `liveness`, `restart_recommended`, `operator_action_required`, `dependencies`, `details`); `test_mcp_server_base.py` passes.

---

### Step 2 — Update each concrete MCP server health endpoint

For each server listed below, add `restart_recommended=False` and `operator_action_required=True` (or `False` for healthy) to the `JSONResponse` body dict returned by the `health()` route function.

Servers to update:
- `mcp/cicd/server.py` — `github_token` missing → `operator_action_required=True`
- `mcp/git/server.py` — `git` not in PATH → `operator_action_required=True`
- `mcp/github/server.py` — `github_token` not set → `operator_action_required=True`
- `mcp/shell/server.py` — `sh` not in PATH → `operator_action_required=True`
- `mcp/rag_pipeline/server.py` — `embed_url` not configured → `operator_action_required=True`
- `mcp/file/read_server.py`, `write_server.py`, `delete_server.py` — filesystem missing → `operator_action_required=True`
- `mcp/mdq/health_check.py` — db_file/db_schema/config failures → `operator_action_required=True`

**Completion criteria:** Each server's `/health` endpoint returns all new fields; no existing `test_mcp_server_health_status.py` test regresses.

---

### Step 3 — Introduce `McpHealthProbeResult` in `agent/shared/health_models.py`

Add a frozen dataclass:

```python
@dataclasses.dataclass(frozen=True)
class McpHealthProbeResult:
    reachable: bool
    status_code: int | None
    restart_recommended: bool
    operator_action_required: bool
    body: dict[str, object]
```

This carries the full structured probe result from a `/health` GET.

**Completion criteria:** Dataclass importable; no existing tests broken.

---

### Step 4 — Update `probe_mcp_health()` in `agent/repl_health.py`

Keep the existing `probe_mcp_health(http, base_url) -> bool` signature as a thin wrapper for backward compatibility (used by startup readiness check and existing tests). Add a new internal helper `_probe_mcp_health_detail(http, base_url) -> McpHealthProbeResult` that parses the response body and extracts `restart_recommended` and `operator_action_required`.

The helper:
- Returns `McpHealthProbeResult(reachable=False, ...)` on `httpx.HTTPError` / `OSError` / `TimeoutError`
- On HTTP response, parses JSON body; falls back to `restart_recommended=False, operator_action_required=False` if JSON parse fails
- Does not raise

**Completion criteria:** `_probe_mcp_health_detail` parses fields correctly; `probe_mcp_health` still returns `bool` correctly for existing callers.

---

### Step 5 — Update `_watchdog_check_http()` in `agent/repl_health.py`

Replace the call to `probe_mcp_health()` with `_probe_mcp_health_detail()`. Change restart decision logic:

- Server unreachable (no HTTP response): existing behavior — attempt restart if `startup_mode=subprocess` and under restart limit
- Server reachable + `restart_recommended=True`: attempt restart as above
- Server reachable + `restart_recommended=False`: do **not** restart
- Server reachable + `operator_action_required=True`: log `WARNING "Watchdog: {key} requires operator action — not restarting"` (or `ERROR` level)
- Externally-managed (non-subprocess) servers: existing warning-only behavior unchanged

**Completion criteria:** Restart only occurs when `reachable=False` or `restart_recommended=True`; `operator_action_required=True` produces a log message without restart.

---

### Step 6 — Fix `watchdog_loop()` early-exit for `interval <= 0`

Currently `watchdog_loop()` logs the disabled message and then enters `while True` with `asyncio.sleep(0)`, which will spin indefinitely.

Change to:
```python
if interval <= 0:
    logger.warning("Watchdog: disabled (interval=%.0f) ...", interval)
    return
while True:
    await asyncio.sleep(interval)
    ...
```

Note: `repl.py:_start_watchdog()` already guards `watchdog_interval > 0` before creating the task, so this fix is defense-in-depth and satisfies the acceptance criterion.

**Completion criteria:** `watchdog_loop()` with `interval=0` returns immediately without sleeping; `repl.py` behavior unchanged (guard already prevents task creation).

---

### Step 7 — Add `tests/test_watchdog.py`

Create the new test file with the four required test cases (all using `AsyncMock` and mocked `AgentContext`):

1. `HTTP 503` + `restart_recommended=True` → `lifecycle.restart()` called
2. `HTTP 503` + `restart_recommended=False` → `lifecycle.restart()` not called
3. `HTTP 200` + `ready=False` → `lifecycle.restart()` not called (should be `restart_recommended=False` in that body)
4. `mcp_watchdog_interval=0` → `watchdog_loop()` returns immediately without sleeping

**Completion criteria:** All four tests pass with `uv run pytest tests/test_watchdog.py -v`.

**Completion criteria:** All existing + new assertions in this file pass with `uv run pytest tests/test_mcp_server_health_status.py -v`.

Update `TestProbeMcpHealth` to cover `_probe_mcp_health_detail()`:
- `reachable=True, restart_recommended=False` for HTTP 200 response with no `restart_recommended` field
- `reachable=True, restart_recommended=True` for HTTP 503 + `restart_recommended=true` in body
- `reachable=False` on exception

Keep existing `probe_mcp_health()` bool tests intact.

**Completion criteria:** `uv run pytest tests/test_repl_health.py -v` passes.

---

### Step 10 — Update MCP documentation

- `docs/04_mcp_02_protocol_and_transport.md`: Update canonical `/health` response schema table to include `liveness`, `restart_recommended`, `operator_action_required` fields and their semantics.
- `docs/04_mcp_03_routing_lifecycle_and_execution.md`: Update the Watchdog section to document that restart is gated on `restart_recommended=true`, and that `operator_action_required=true` triggers log-only.
- `docs/04_mcp_06_configuration_and_operations.md`: Update the health semantics description and curl examples to show new fields; update the statement "The watchdog checks only the HTTP status code" to reflect the new body-inspection behavior.
- `docs/04_mcp_90_inconsistencies_and_known_issues.md`: Mark the existing watchdog inconsistency entry as resolved, noting the `restart_recommended` field fix.

**Completion criteria:** All four doc files updated; no references to "watchdog checks only HTTP status code" remain without qualification.
