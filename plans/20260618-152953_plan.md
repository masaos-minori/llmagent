# Plan: Add Detailed API Documentation for the Memory Layer

Derived from `01_plan.md` using requirement `requires/20260618_27_require.md`.

---

## Goal

Developers can understand the memory layer's internal modules (responsibility, API surface, disabled behavior) without reading source code.

---

## Scope

**In scope:**
- `docs/05_agent_12_memory.md` — new file, API reference for all 14 memory modules
- `scripts/agent/memory/__init__.py` — document as public API barrel
- `scripts/agent/memory/store.py` — CRUD API, schema
- `scripts/agent/memory/retriever.py` — FtsRetriever / VectorRetriever / HybridRetriever
- `scripts/agent/memory/injection.py` — MemoryInjectionService lifecycle hooks
- `scripts/agent/memory/ingestion.py` — MemoryIngestionService dedup/persist
- `scripts/agent/memory/embedding_client.py` — EmbeddingClient API
- `scripts/agent/memory/extract.py` — extraction policy and rules
- `scripts/agent/memory/jsonl_store.py` — JSONL source of truth
- `scripts/agent/memory/services.py` — MemoryServices facade
- `scripts/agent/memory/types.py` — core data types
- `scripts/agent/memory/models.py` — DTOs
- `scripts/agent/memory/enums.py` — domain enums
- `scripts/agent/memory/exceptions.py` — exception hierarchy
- `scripts/agent/memory/mapper.py` — row conversion utils
- `routing.md` — update outdated `layer` → `services.py` reference

**Out of scope:**
- Code changes (name changes, refactoring)
- Dashboard/UI documentation
- Test documentation

## Assumptions

1. Each module's public symbols (exported in `__all__` or `__init__.py`) represent the intended API contract.
2. Disabled behavior is already handled via `_disabled` pattern or config-gated checks within each module.

## Implementation Steps

1. **Read disabled behavior in each service module**
   - Search for `_disabled`, `enabled`, `cfg.memory.enabled` in `services.py`, `injection.py`, `ingestion.py`
   - Document what happens when memory is disabled (no-ops, fallbacks)
   - Criterion: disabled behavior described accurately

2. **Create `docs/05_agent_12_memory.md`**
   - Section per module: responsibility, public classes/functions, key parameters, failure modes
   - Include a module responsibility map diagram (ASCII or table)
   - Criterion: a developer can use the doc without opening source files

3. **Update `routing.md`**
   - Change `layer` to `services.py` in the memory routing entry
   - Add `docs/05_agent_12_memory.md` to the memory task mapping
   - Criterion: routing points to the new doc

4. **Lint check (doc only)**
   - No code to lint; markdown-only
   - Criterion: no broken internal links
