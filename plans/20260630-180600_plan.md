# Plan: Review and remove MCP installer module chain

## Goal
`McpInstallRequest` / `McpInstallRenderModel` の2クラスを `agent/commands/models.py` から削除することで、`/mcp install` 削除後に取り残されたデッドコードをリポジトリから完全に除去する。

## Scope
**In-Scope**:
- `scripts/agent/commands/models.py` から `McpInstallRequest` / `McpInstallRenderModel` クラスを削除
- 削除前にリポジトリ全体で参照がないことを grep で確認
- lint / type check / テストの通過確認

**Out-of-Scope**:
- `__pycache__` 内 `.pyc` ファイルの手動削除 (通常の `git clean` / CI で対処)
- `.mypy_cache` 内 installer 関連 `.json` ファイルの削除
- `scripts/mcp/installer_*.py` / `scripts/mcp_installer.py` / `scripts/agent/services/mcp_install.py` / `tests/test_installer_*.py` の削除 (これらはソースファイルとして既に削除済み)

## Assumptions
- `McpInstallRequest` / `McpInstallRenderModel` はソースコード上の参照が `agent/commands/models.py` 内の定義のみであり、runtime で動的にインポートされていない
- `test_cmd_mcp.py` の `test_invalid_subcommand_shows_usage` は `install` サブコマンドが `UnknownSubcommandError` を返すことを確認するテストであり、installer DTO に依存しないため変更不要
- `__pycache__` に残る `.pyc` は git 管理対象外であり、追跡・削除は不要
