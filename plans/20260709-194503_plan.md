## Goal

Align configuration documentation with the current canonical layout (unified `agent.toml`) and clarify config ownership.

## Scope

- **In-Scope**: Updating `05_agent_08_configuration.md` and `04_mcp_06_configuration_and_operations.md` to accurately describe the current config layout; clarifying that `agent.toml` is the sole base config file loaded by `load_all()`; clarifying per-server MCP config file ownership.
- **Out-of-Scope**: Changing config layout; removing split-file support; changing runtime semantics.

## Assumptions

1. `agent.toml` is the only base config file; all previously split files (common.toml, llm.toml, http.toml, rag.toml, context.toml, tools.toml, memory.toml, otel.toml, security.toml, system_prompts.toml, tools_definitions.toml) no longer exist.
2. Per-server MCP config files (e.g., `github_mcp_server.toml`) are loaded individually by each server, not via `load_all()`.
3. No runtime behavior change is required.
