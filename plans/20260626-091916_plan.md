# Plan: Enforce firejail sandbox as production-safe default for shell-mcp

Source: `requires/20260625_09_require.md`

---

## Goal

production mode で `shell_sandbox_backend="none"` を指定した場合の startup hard error が `repl_health.py` に既に実装されていることを確認・補強し、production config テンプレートのデフォルト値を `firejail` に設定する。

---

## Scope

**In-Scope**
- `scripts/agent/repl_health.py`: line 484-486 の hard error 実装の確認・補強
- `config/shell_mcp_server.toml`: `sandbox_backend` のデフォルト値確認・production template 更新
- ドキュメントの local dev vs production の明確な分離
- startup rejection テスト追加

**Out-of-Scope**
- firejail の インストール・設定
- `none` 以外の sandbox backend の追加

---

## Assumptions

1. `repl_health.py` line 484-486 に `if shell_cfg.shell_sandbox_backend == "none" / production_mode` の hard error が既に存在する（確認済み）。
2. `shell/service.py` の `init_sandbox()` は firejail 不在時に `none` にフォールバックする。
3. production template と local dev template が分かれているかは未確認。
