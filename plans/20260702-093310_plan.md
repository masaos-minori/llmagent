# Plan: Add eventbus_db_path to DbConfig, common.toml, and SQLiteHelper

Source: `requires/20260701_51_require.md`

---

## Goal

`DbConfig` に `eventbus_db_path` を追加し、`common.toml` にデフォルトパス `/opt/llm/db/eventbus.sqlite` を定義、`SQLiteHelper("eventbus")` が `eventbus.sqlite` を開けるようにする。

---

## Scope

**In-Scope**
- `DbConfig` dataclass に `eventbus_db_path` フィールドを追加
- `build_db_config()` で `eventbus_db_path` を読み込む
- `common.toml` に `eventbus_db_path` を追加
- `DbTarget.EVENTBUS = "eventbus"` を追加
- `SQLiteHelper.__init__` が `"eventbus"` / `DbTarget.EVENTBUS` を受け付けるように修正
- `eventbus` target は sqlite-vec をロードしない（`_default_load_vec = False`）
- `DbConfig.__post_init__` の検証に `eventbus_db_path` を追加
- DB config テスト (`test_agent_cmd_config.py` / `test_sqlite_helper.py`) を更新
- ドキュメント `docs/90_shared_04_db_architecture_and_schema.md` を更新（3→4 DB files）
- ドキュメント `docs/90_shared_05_db_api_and_operations.md` を更新（DbTarget, DbHelper 記述）

**Out-of-Scope**
- Event Bus ランタイム統合（イベントキュー・パブリッシュ・サブスクライブ処理）
- `eventbus.sqlite` のスキーマ定義・`create_schema.py` の変更
- `deploy.sh` 以外のデプロイスクリプト変更

---

## Assumptions

1. `/opt/llm/db/` ディレクトリはすでに存在しており、テスト環境では `tmp_path` フィクスチャで代替できる。
2. `eventbus.sqlite` は sqlite-vec 拡張を必要としない（全文検索・ベクター検索対象外）。
3. `DbConfig.__post_init__` の親ディレクトリ検証は `eventbus_db_path` にも適用する（他 3 パスと同一ポリシー）。
4. `common.toml` の `workflow_db_path` はデフォルト値がコードに埋め込まれており同じパターンを踏襲する。

## Implementation Steps

1. **Phase 1: DbConfig 更新**
   - [ ] `scripts/db/config.py`: `DbConfig` に `eventbus_db_path: str = "/opt/llm/db/eventbus.sqlite"` を追加
   - [ ] `__post_init__` の検証ループに `("eventbus_db_path", self.eventbus_db_path)` を追加
   - [ ] `build_db_config()` に `eventbus_db_path=cfg.get("eventbus_db_path", "/opt/llm/db/eventbus.sqlite")` を追加

2. **Phase 2: SQLiteHelper 更新**
   - [ ] `scripts/db/helper.py`: `DbTarget.EVENTBUS = "eventbus"` を追加
   - [ ] `SQLiteHelper.__init__` の有効 target 集合に `"eventbus"` を追加
   - [ ] パス解決の条件分岐に `eventbus` ケースを追加（`db_cfg.eventbus_db_path` を使用）
   - [ ] `_default_load_vec` は `resolved == "rag"` のままで eventbus は False になることを確認

3. **Phase 3: common.toml 更新**
   - [ ] `config/common.toml` に `eventbus_db_path = "/opt/llm/db/eventbus.sqlite"` を追記

4. **Phase 4: テスト更新**
   - [ ] `tests/test_sqlite_helper.py`: `DbTarget.EVENTBUS` と文字列 `"eventbus"` で `DB_PATH` が正しく解決されることを確認するテストを追加
   - [ ] `"eventbus"` で sqlite-vec がロードされないことを確認するテストを追加
   - [ ] 無効 target が `ValueError` を上げることを確認するテストを追加（既存テストがあれば更新）
   - [ ] DB config テストに `eventbus_db_path` のカバレッジを追加

5. **Phase 5: ドキュメント更新**
   - [ ] `docs/90_shared_04_db_architecture_and_schema.md`: 「Three DB files」→「Four DB files」、`eventbus.sqlite` 行を表に追加
   - [ ] `docs/90_shared_05_db_api_and_operations.md`: `DbTarget` enum 記述と `SQLiteHelper` 使用例に `eventbus` を追加
