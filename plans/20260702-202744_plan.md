# Plan: Remove package-level GitHub service re-exports from `mcp/github/__init__.py`

Source requirement: `requires/20260702_55_require.md`

## Goal

Remove service symbol re-exports from `mcp/github/__init__.py` so that no code imports `GitHubService`, `build_service`, or `_GITHUB_TOKEN` via the `mcp.github` package namespace.

## Scope

**In**:
- Remove `from mcp.github.service import _GITHUB_TOKEN, GitHubService, build_service` from `mcp/github/__init__.py`
- Remove `__all__` entries for those three symbols
- Update the package docstring in `mcp/github/__init__.py` to remove the `service â€” re-export stub for backward compatibility` line
- Update `tests/test_github_mcp_service.py` line 28: replace `from mcp.github.service import GitHubService` with `from mcp.github.service_dispatch import GitHubService`

**Out**:
- Removing or modifying `mcp/github/service.py` itself (that is a separate ticket)
- Changing `GitHubService` business logic or dispatch behaviour
- Removing the `mcp.github` package itself
- Any changes to `mcp/github/service_dispatch.py`, `mcp/github/service_init.py`, or other sub-modules

## Assumptions

1. `mcp/github/service.py` continues to exist during this change; only the `__init__.py` re-export is removed.
2. No code outside `scripts/`, `tests/`, and `docs/` imports from `mcp.github` (verified by grep â€” no matches found in docs, no matches found in scripts outside the package).
3. The correct replacement import for `GitHubService` in `tests/test_github_mcp_service.py` is `from mcp.github.service_dispatch import GitHubService` (it is the combined dispatch+business class per the package docstring and `service.py` re-export chain).
4. Leaving `mcp/github/__init__.py` with only a docstring and no `__all__` is the intended end-state (confirmed by the requirement's "Undecided items" question â€” this plan treats it as acceptable; see Unknowns).
