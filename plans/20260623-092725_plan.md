## Assumptions

1. `check_tool_definitions_runtime(ctx)` は「実行時チェック (always non-strict)」として残す。startup でのみ strict が適用される。
2. `tool_definitions_strict` は `ctx.cfg.tool.tool_definitions_strict` 経由で参照できる (config_dataclasses.py L198 で確認済み)。
3. `startup.py` が `check_tool_definitions_runtime` を呼んでいることで strict モードが事実上無効化されていた (設計漏れ)。

## Design

### 問題の二重構造

1. **`_check_tool_definitions()` のバグ (L210–217):**

現状:
```python
if not server_names:
    if unreachable:
        msg = f"All MCP servers unreachable during strict validation: {sorted(set(unreachable))}; skipping tool definition check"
        logger.info(msg)   # ← strict=True でも INFO + 空の HealthCheckResult を返す
    ...
    return HealthCheckResult()   # ← エラーなし
```

修正後:
```python
if not server_names:
    if unreachable:
        if strict:
            msg = (
                f"Strict mode: all MCP servers unreachable — cannot validate tool definitions. "
                f"Unreachable servers: {sorted(set(unreachable))}."
            )
            logger.error(msg)
            raise RuntimeError(msg)
        msg = f"All MCP servers unreachable; skipping tool definition check. Unreachable: {sorted(set(unreachable))}"
        logger.info(msg)
    else:
        msg = "No tool definitions in config and no servers reachable; skipping validation"
        logger.info(msg)
    return HealthCheckResult()
```

2. **`startup.py` の設計漏れ:**

現状 (L166):
```python
tool_result = await check_tool_definitions_runtime(ctx)  # 常に strict=False
```

修正後:
```python
tool_result = await check_tool_definitions_startup(ctx)  # config フラグに従う
```

`check_tool_definitions_startup()` (repl_health.py に追加):
```python
async def check_tool_definitions_startup(ctx: AgentContext) -> HealthCheckResult:
    """Startup validation: respects tool_definitions_strict config flag."""
    strict = getattr(ctx.cfg.tool, "tool_definitions_strict", False)
    return await _check_tool_definitions(ctx, strict=strict)
```

### エラーメッセージの区別

| ケース | メッセージパターン | ログレベル |
|---|---|---|
| 全到達不能 (strict=False) | `All MCP servers unreachable; skipping ... Unreachable: [...]` | INFO |
| 全到達不能 (strict=True) | `Strict mode: all MCP servers unreachable — cannot validate tool definitions. Unreachable servers: [...]` | ERROR + RuntimeError |
| 一部到達不能 + mismatch (strict=False) | `Tools in agent.toml but not on any server: [...]` | WARNING |
| mismatch (strict=True) | `Strict mode: tool definition mismatch detected. Mismatches: .... Unreachable servers: [...]` | ERROR + RuntimeError (既存) |
