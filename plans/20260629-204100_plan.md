## Goal
MDQ がサポートする Markdown 構文の互換性マトリクスを `docs/04_mcp_04_server_catalog.md` の mdq-mcp セクションに追記し、未サポート構文のフォールバック動作を明示する。

## Scope
**In-Scope**:
- `docs/04_mcp_04_server_catalog.md` の mdq-mcp セクションに Markdown 互換性表を追加
- サポート済み: ATX 見出し（全レベル）、フェンスコードブロック、YAML frontmatter、ルートセクション（最初の見出し前コンテンツ）、重複見出し
- 未サポート: Setext 見出し（=== / ---）、HTML ブロック、MDX、GFM テーブル、インラインタグ
- 未サポート構文のフォールバック動作（プレーンテキストとして扱われる）の明記

**Out-of-Scope**:
- `parser.py` の実装変更
- 新しい Markdown 機能のサポート追加
- テストの変更（既存テストは parser.py モジュールと整合済み）

## Assumptions
1. `scripts/mcp/mdq/parser.py` のモジュールドキュメント（L10-L30）が互換性の正式な情報源
2. `docs/04_mcp_04_server_catalog.md` の mdq-mcp セクション（L283-294）が更新対象
3. Setext 見出し（`===`, `---`）はプレーンテキストとして扱われる（parser.py:L25-L28 記載）
4. GFM テーブルはパースされないが、セクション分割には影響しない（コンテンツとして保持）
