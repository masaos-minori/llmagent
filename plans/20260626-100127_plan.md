# Plan: Extend memory observability to show memory quality, not just status

Source: `requires/20260625_96_require.md`

---

## Goal

`/memory status` を拡張し、enabled/disabled 状態だけでなく、semantic/episodic カウント、source type 分布、embed-skip カウント、last retrieval mode など品質シグナルを表示する。

---

## Scope

**In-Scope**
- `/memory status` コマンドハンドラ: カウントと quality シグナルの追加
- `scripts/agent/memory/services.py` / `ingestion.py`: `stat_embed_skip`（すでに存在）の活用
- `scripts/agent/memory/retriever.py`: `last_retrieval_mode`, `fts_fallback_count`（すでに存在）の活用
- `docs/05_agent_10_operations-and-observability.md`: status 出力の説明更新

**Out-of-Scope**
- metrics platform / dashboard
- extraction アルゴリズム変更

---

## Assumptions

1. `HybridRetriever.last_retrieval_mode` と `fts_fallback_count` はすでに存在（`retriever.py` line 261-262）。
2. `MemoryIngestionService.stat_embed_skip` はすでに存在（`ingestion.py` line 56）。
3. memory type / source type 別カウントは SQLite `memories` テーブルから集計する。

## Implementation Steps

- [ ] 1. `MemoryServices.get_stats()` を追加:
  ```python
  def get_stats(self) -> dict:
      # SQLite から集計
      return {
          "total": total_count,
          "semantic": semantic_count,
          "episodic": episodic_count,
          "by_source": {"RULE": .., "DECISION": .., "FAILURE": .., "CONVERSATION": ..},
          "embed_skip": self.ingestion.stat_embed_skip,
          "last_retrieval_mode": self.retriever.last_retrieval_mode,
          "fts_fallback_count": self.retriever.fts_fallback_count,
      }
  ```
- [ ] 2. `/memory status` にこれらを追加表示:
  ```
  Memory Status:
    mode: hybrid
    entries: 142 (semantic=89, episodic=53)
    source types: RULE=34, DECISION=22, FAILURE=15, CONVERSATION=71
    embed_skip: 8
    last_retrieval: hybrid (fts_fallback=2)
  ```
- [ ] 3. docs に status 出力の各フィールドの意味を説明
