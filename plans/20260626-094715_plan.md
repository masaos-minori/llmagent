# Plan: Add pagination to /replay and /dlq instead of unbounded fetchall()

Source: `requires/20260625_47_require.md`

---

## Goal

`/replay` と `/dlq` の unbounded `fetchall()` を chunked/page-limited な取得に置き換え、大量イベント時のメモリ安全性を確保する。

---

## Scope

**In-Scope**
- `scripts/eventbus/app.py`: `/replay`（SSE streaming）と `/dlq`（GET list）のクエリを chunked 化
- `/dlq`: `limit` / `cursor` パラメータ追加
- `/replay`: `LIMIT` または `OFFSET`-based ページング追加
- API docs / テスト更新

**Out-of-Scope**
- DB スキーマ変更（index 追加は 48 番で対応）
- SSE 以外の streaming 方式への変更

---

## Assumptions

1. `/dlq`（line 205-206）は `fetch_dlq()` → `fetchall()` で全件取得している（確認済み）。
2. `/replay`（line 135）は SSE だが内部で一括ロードしている可能性がある。
3. FastAPI での cursor pagination または limit/offset pagination を導入する。
