# Plan: Expose shell sandbox state in health and operations views

## Goal

Make the actual shell sandbox backend state visible in `/health`, `/mcp` status display, and operational docs so operators cannot assume sandboxing is active when it is not.

## Scope

**In scope:**
- `scripts/mcp/shell/service.py` — expose `_sandbox_backend` state
- Shell-mcp `/health` endpoint
- `/mcp` status display
- Shell operational docs

**Out of scope:**
- Changing the sandbox backend selection logic (firejail vs none)
- Installing or configuring firejail
- Modifying other MCP servers' health endpoints
- Changes to shell command allowlist or CWD validation logic

## Assumptions

1. `ShellService.__init__()` already calls `init_sandbox(policy.sandbox_backend)` which resolves to "firejail" or "none" — this resolved value is stored in `self._sandbox_backend` but never exposed.
2. The `/health` endpoint for shell-mcp currently returns basic health info but does not include sandbox backend state.
3. Operators may assume shell execution is always sandboxed based on docs describing shell-mcp as "sandboxed" — this assumption is dangerous when `sandbox_backend=none`.

## Implementation Steps

1. **Add sandbox backend to shell-mcp `/health` response**
   - Check `scripts/mcp/shell/server.py` for current health endpoint
   - Add `"sandbox_backend": self._service._sandbox_backend` to health response dict
   - Example: `{"status": "ok", "sandbox_backend": "none"}` or `{"status": "ok", "sandbox_backend": "firejail"}`
   - Completion criterion: `/health` endpoint includes sandbox_backend field

2. **Expose `_sandbox_backend` as a public property (if needed)**
   - If server.py accesses service via `self._service`, add a public property:
     ```python
     @property
     def sandbox_backend(self) -> str:
         return self._service._sandbox_backend
     ```
   - Completion criterion: `_sandbox_backend` is accessible from server layer

3. **Update shell-mcp documentation**
   - In `04_mcp_04_server_catalog.md`, update shell-mcp section to clearly state:
     - Default sandbox backend is "none" (no sandboxing)
     - Sandbox can be enabled by setting `sandbox_backend = "firejail"` in config
     - Firejail must be installed for sandbox mode to work
   - Completion criterion: Docs accurately describe actual sandbox behavior

4. **Include sandbox state in `/mcp` status display (if applicable)**
   - If `/mcp` endpoint aggregates server health, include shell sandbox state
   - Completion criterion: /mcp output shows sandbox backend for shell-mcp
