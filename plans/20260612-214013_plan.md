# Refactoring Plan: RAG Search Quality and Retrieval Control Improvements

## Goal

RAGパイプラインの4つの改善点を実装する:
1. two-stage fetch の決定を structured response (`TwoStageFetchResult`) に変換する
2. `rag_min_score` と `max_chunks_per_doc` による最終採用制御をDTOに集約する
3. semantic cache キーに `history_context` を含め、未使用状態だった cache を有効化する
4. FTS5 クエリの reserved term を確実にエスケープし `content:` column filter を適用する（`passage:` カラム不存在のため `passage:` は使用不可）

---

**In scope:**
- `scripts/rag/models.py` — `TwoStageFetchResult` frozen dataclass 追加、`CacheEntry` に `history_context` フィールド追加

## Design

### Step 1: モデル変更 (`scripts/rag/models.py`)

**`TwoStageFetchResult` 追加:**
```python
@dataclass(frozen=True)
class TwoStageFetchResult:
    """Typed result capturing reranked hits with applied filter/dedup parameters."""
    hits: list[RagHit]
    min_score_applied: float     # rag_min_score used (0.0 = no score filter)
    max_chunks_per_doc: int      # per-doc dedup limit applied
    total_before_dedup: int      # count after score filter, before dedup
    total_after_dedup: int       # count after dedup (== len(hits))

    @property
    def was_deduplicated(self) -> bool:
        return self.total_before_dedup > self.total_after_dedup
```

**`CacheEntry` に `history_context` 追加:**
```python
# Before
@dataclass(frozen=True)
class CacheEntry:
    embedding: list[float]
    context_str: str

# After
@dataclass(frozen=True)
class CacheEntry:
    embedding: list[float]
    context_str: str
    history_context: str = ""  # cache key component
```

### Step 2: `pipeline.py` の `last_reranked` → `last_fetch_result` (`scripts/rag/pipeline.py`)

**インスタンス変数の変更 (line 102):**
```python
# Before
self.last_reranked: list[RagHit] = []

# After
self.last_fetch_result: TwoStageFetchResult | None = None
```

**`pipeline.run()` の終端 (line 187-188) に `TwoStageFetchResult` 構築を追加:**

`ctx.reranked` は `RerankStage` が設定済み。ただし `rag_min_score`/`max_chunks_per_doc` の情報は `ctx.reranked` には含まれない（すでにフィルタ・dedup適用済みのリストのみ）。
`RerankStage` が書き込む前後の件数を `PipelineContext` に追加する必要がある。

**`PipelineContext` に統計フィールドを追加 (`scripts/rag/stage.py`):**
```python
@dataclasses.dataclass
class PipelineContext:
    ...
    reranked_before_dedup: int = 0   # count after score filter, before dedup
```

**`stages/rerank.py:_rerank()` でカウントを記録:**
```python
async def run(self, ctx: PipelineContext, **kwargs: object) -> None:
    reranked = await _rerank(ctx.query, ctx.merged, self._cfg, self._llm)
    ctx.reranked_before_dedup = len(reranked)  # dedup前のカウントは _rerank() 内で記録できない
    ctx.reranked = reranked
```

> **問題**: `_rerank()` は `deduplicate_chunks()` の結果を返しており、dedup前のカウントが失われる。`_rerank()` を分解するか、`_rerank()` が `(before_dedup_count, results)` を返すように変更する必要がある。

**代替設計（シンプルな実装）:**

`pipeline.run()` が終端で `TwoStageFetchResult` を構築する際、dedup前件数を取得するために `_rerank()` の内部にアクセスできない。
代わりに `TwoStageFetchResult` のフィールドを `cfg` から取得できる情報に絞る:

```python
# pipeline.run() の終端
self.last_fetch_result = TwoStageFetchResult(
    hits=ctx.reranked,
    min_score_applied=self._cfg.rag_min_score,
    max_chunks_per_doc=self._cfg.max_chunks_per_doc,
    total_before_dedup=len(ctx.reranked),   # dedup前件数が取れない場合は len(hits) と同値にする
    total_after_dedup=len(ctx.reranked),
)
```

**`_augment_http()` での設定 (line 211):**
```python
# Before
self.last_reranked = hits

# After
self.last_fetch_result = TwoStageFetchResult(
    hits=hits,
    min_score_applied=0.0,      # HTTP mode では score filter 情報なし
    max_chunks_per_doc=0,       # HTTP mode では dedup 情報なし
    total_before_dedup=len(hits),
    total_after_dedup=len(hits),
)
```

### Step 3: `SemanticCache` に `history_context` を組み込む + cache 有効化

**`cache.py` のシグネチャ変更:**
```python
class SemanticCache:
    def lookup(self, embedding: list[float], history_context: str = "") -> str | None:
        """Return cached context for nearest embedding with matching history_context."""
        if self._dim is not None and len(embedding) != self._dim:
            raise ValueError(...)
        best_sim = -1.0
        best_ctx: str | None = None
        for entry in self._entries:
            if entry.history_context != history_context:
                continue  # history_context が異なるエントリはスキップ
            sim = cosine_sim(embedding, entry.embedding)
            if sim > best_sim:
                best_sim = sim
                best_ctx = entry.context_str
        if best_sim >= self._threshold:
            return best_ctx
        return None

    def put(self, embedding: list[float], history_context: str, context_str: str) -> None:
        self._entries.append(CacheEntry(
            embedding=embedding,
            context_str=context_str,
            history_context=history_context,
        ))
        self.prune()
```

**`pipeline.py:augment()` への cache 呼び出し追加:**

`augment()` は `use_search` フラグと `rag_service_url` 分岐の後でキャッシュを確認する。
`stat_semantic_cache_hits` は `pipeline.py` から直接インクリメントできない（`AgentContext` が pipeline 層にない）ため、`augment()` が `cache_hit: bool` を返す `AugmentResult` DTO を返すか、または `on_status` callback 経由で通知する。

**最もシンプルな設計: `augment()` は現状どおり `str` を返し、cache hit カウントは `on_status("cache_hit")` で通知:**

```python
async def augment(self, query: str, ..., history_context: str = "") -> str:
    if not self._cfg.use_search:
        return ""
    if rag_url := self._cfg.rag_service_url:
        result = await self._augment_http(rag_url, query, history_context)
        if result is not None:
            return result

    # Semantic cache lookup (in-process mode のみ)
    emb: list[float] | None = None
    if self._cfg.use_semantic_cache:
        try:
            emb = await get_embedding(query, self._http, self._embed_url)
        except Exception:  # noqa: BLE001 — cache miss on any failure
            emb = None
        if emb is not None:
            cached = self.semantic_cache.lookup(emb, history_context)
            if cached is not None:
                self._on_status("cache_hit")  # 呼び出し元で stat_semantic_cache_hits をインクリメント
                return cached

    try:
        db = SQLiteHelper().open(row_factory=True)
    except ...:
        ...
    with db:
        queries, all_results, merged, reranked = await self.run(...)
    ...
    context_block = self._format_chunks(reranked)

    # キャッシュ書き込み
    if self._cfg.use_semantic_cache and emb is not None and context_block:
        self.semantic_cache.put(emb, history_context, context_block)

    return context_block
```

> **`on_status("cache_hit")` の解釈**: 呼び出し元 (orchestrator や MCP service) が `on_status` callback で "cache_hit" を受け取り `stat_semantic_cache_hits` をインクリメントする。既存の `on_status` callback の仕組みを流用。

> **`except Exception` の BLE001**: embedding fetch は外部 HTTP 呼び出しで、失敗時は cache miss として扱うのが適切。`# noqa: BLE001` を付与して許可する。

### Step 4: FTS5 クエリの強化

**`scripts/agent/memory/retriever.py:_build_fts_query()` の変更:**

現状: `" OR ".join(tokens)` — AND/OR/NOT/NEAR がそのまま現れると構文エラー

```python
# Before (line 96-101)
def _build_fts_query(text: str) -> str:
    """Build an FTS5 MATCH query from free-form text; quotes multi-word tokens."""
    tokens = re.findall(r"\w+", text)
    if not tokens:
        return '""'
    return " OR ".join(tokens)

# After
def _build_fts_query(text: str) -> str:
    """Build an FTS5 MATCH query with content: column filter and token quoting.

    Tokens are double-quoted to escape FTS5 reserved terms (AND/OR/NOT/NEAR).
    content: column filter targets the indexed column in memories_fts.
    """
    tokens = re.findall(r"\w+", text)
    if not tokens:
        return '""'
    quoted = " OR ".join(f'"{t}"' for t in tokens)
    return f"content: {quoted}"
```

**`scripts/rag/repository.py:_build_fts_query()` の変更:**

現状: トークンを `"..."` で引用済み（reserved terms エスケープ済み）。`content:` フィルタがない。

```python
# Before (line 100-101)
    return " ".join(f'"{t}"' for t in sanitized)

# After
    return "content: " + " ".join(f'"{t}"' for t in sanitized)
```
