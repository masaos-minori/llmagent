# Plan: Replace silent persistence skips with visible warnings or strict mode (Req 28)

## Goal

Make persistence skips visible through logging and counters, and add an optional strict mode that treats them as errors instead of silently ignoring them.

## Scope

**In scope:**
- `scripts/agent/session_message_repo.py` — add counters for skipped saves, log warnings for missing session_id, count invalid roles in `save_many()`
- `scripts/agent/session.py` — pass strict mode flag to repository, expose counters on facade
- `tests/test_session_message_repo.py` — tests for counters and strict mode behavior
- `docs/05_agent_04_state-and-persistence.md` — update message save rules documentation

**Out of scope:**
- Adding a new DB table or column for persistence metrics
- Modifying the history compression skip logic (separate concern)
- Changing the `_VALID_ROLES` set itself

## Assumptions

1. Counters should be on `SessionMessageRepository` instance — they track per-session skip events.
2. Strict mode is a per-repository setting, configurable via `AgentSession`.
3. Existing warning for invalid role in `save()` (line 35) is sufficient — only missing session_id and save_many() silent filters need new logging.

### Step 1: Add counters and strict mode to `SessionMessageRepository`

Add instance attributes to track skips:

```python
def __init__(self, session_id: int | None = None, *, strict_mode: bool = False) -> None:
    self.session_id = session_id
    self.strict_mode = strict_mode
    # Counters for skipped persistence operations
    self.stat_skipped_no_session: int = 0
    self.stat_skipped_invalid_role: int = 0
```

Update `save()`:
- When `session_id is None`, increment counter and log warning (or raise in strict mode)
- Keep existing invalid role warning (already present)

```python
def save(self, role: str, content: str, ..., _diagnostic: bool = False) -> None:
    if self.session_id is None:
        self.stat_skipped_no_session += 1
        logger.warning("Persistence skipped: no session_id (role=%r)", role)
        if self.strict_mode:
            raise RuntimeError(f"Cannot save message: no session_id (strict mode)")
        return
    if not _diagnostic and role not in _VALID_ROLES:
        self.stat_skipped_invalid_role += 1
        logger.warning("Invalid role %r; message not saved", role)
        if self.strict_mode:
            raise RuntimeError(f"Cannot save message with invalid role {role!r} (strict mode)")
        return
    # ... rest of save logic
```

Update `save_many()`:
- Count and log invalid roles filtered out in the list comprehension
- Log warning when session_id is None (with counter)

```python
def save_many(self, messages: list[...]) -> None:
    if self.session_id is None or not messages:
        if self.session_id is None:
            self.stat_skipped_no_session += 1
            logger.warning("Persistence skipped: no session_id (save_many, %d messages)", len(messages))
            if self.strict_mode:
                raise RuntimeError(f"Cannot save messages: no session_id (strict mode)")
        return
    # Count invalid roles before filtering
    invalid_roles = [role for role, _, _, _ in messages if role not in _VALID_ROLES]
    if invalid_roles and len(invalid_roles) == len(messages):
        self.stat_skipped_invalid_role += len(invalid_roles)
        logger.warning(
            "Persistence skipped: all %d messages have invalid roles", len(messages)
        )
        return
    rows = [
        ... for role, content, tc, tc_id in messages if role in _VALID_ROLES
    ]
    if not rows:
        self.stat_skipped_invalid_role += len(invalid_roles)
        logger.warning("Persistence skipped: %d messages had invalid roles", len(invalid_roles))
        return
    # ... rest of save_many logic
```
