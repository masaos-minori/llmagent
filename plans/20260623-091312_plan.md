# Plan: 本番環境シェル sandbox 強制

**Source:** `requires/20260622_53_require.md`
**Created:** 2026-06-23 09:13:12

## Goal

本番モード (`SecurityProfile.PRODUCTION`) でシェル実行が `sandbox_backend="none"` のまま許可される問題を修正する。エージェント起動時に失敗 (RuntimeError) させ、かつ `init_sandbox()` が firejail 要求時に firejail 未インストールなら警告ではなく即時失敗するよう変更する。

---

## Scope

**In-Scope:**
- `scripts/agent/repl_health.py` — `audit_security_defaults()` で `shell_sandbox_backend="none"` かつ `production_mode=True` の場合 RuntimeError を raise
- `scripts/mcp/shell/service_static_helpers.py` — `init_sandbox()` でfirejail 要求時に未インストールなら警告+フォールバックではなく RuntimeError を raise
- `tests/test_shell_mcp_service.py` — `init_sandbox()` の変更に合わせてテスト更新
- `docs/04_mcp_04_server_catalog.md` — セキュリティノートを更新

**Out-of-Scope:**
- firejail 以外の sandbox 技術への移行
- シェルコマンド allowlist の変更
- `shell_mcp_server.toml` へ新フィールド追加（後述の設計選択による）

---

## Assumptions

1. 「production mode の startup failure」 = agent 起動時の `audit_security_defaults()` が RuntimeError を raise することと解釈する。既存の auth_token 違反と同じパターン。
2. shell-mcp サーバー自体が SecurityProfile を直接読めないため、サーバー側の単独起動失敗は実装しない（agent-level enforcement で要件を満たす）。
3. `init_sandbox("firejail")` で firejail 未インストール時のサイレントフォールバックは「設計バグ」であり、要件とは独立してハードエラー化する。
4. 既存の tests が firejail 未インストール時のフォールバックを期待しているため、テスト修正が必要。

## Design

### 1. `audit_security_defaults()` の変更 (`repl_health.py`)

**現状 (L453–456):**
```python
if shell_cfg.shell_sandbox_backend == "none":
    msg = "Security: shell_sandbox_backend=none (no sandbox for shell commands)"
    logger.warning(msg)
    warnings.append(msg)
```

**修正後:** `production_mode=True` の場合 RuntimeError を raise (auth_token 違反と同パターン)
```python
if shell_cfg.shell_sandbox_backend == "none":
    msg = "shell_sandbox_backend=none is not permitted in production mode"
    if production_mode:
        raise RuntimeError(
            f"Production mode requires shell sandbox. {msg}"
        )
    logger.warning("Security: %s", msg)
    warnings.append(f"Security: {msg}")
```

### 2. `init_sandbox()` の変更 (`service_static_helpers.py`)

**現状 (L20–27):**
```python
def init_sandbox(backend: str) -> str:
    if backend == "firejail" and shutil.which("firejail") is None:
        logger.warning("firejail not found in PATH; shell_sandbox_backend falling back to 'none'")
        return "none"
    return backend
```

**修正後:** 警告+フォールバックではなく即時 RuntimeError
```python
def init_sandbox(backend: str) -> str:
    if backend == "firejail" and shutil.which("firejail") is None:
        raise RuntimeError(
            "shell_sandbox_backend=firejail is configured but firejail is not found in PATH"
        )
    return backend
```

### 3. テスト更新 (`test_shell_mcp_service.py`)

`test_firejail_fallback_to_none_when_not_found` (L203) を `pytest.raises(RuntimeError)` に変更。
