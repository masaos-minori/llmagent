# Plan: Optimize dedup, retention, and rebuild policies for long-term memory use

Source: `requires/20260625_95_require.md`

---

## Goal

memory type / source type ごとの dedup・retention policy を見直し、長期蓄積での retrieval 品質を維持する。JSONL canonical model と `/memory prune` / `/memory rebuild` の操作性を強化する。

---

## Scope

**In-Scope**
- `scripts/agent/memory/ingestion.py`: source type ごとの dedup 閾値差異化
- `scripts/agent/memory/jsonl_store.py`: retention days の source type 別設定
- `scripts/agent/memory/enums.py`: `DedupPolicy` に source type 別設定フィールドを追加（必要なら）
- `/memory prune` / `/memory rebuild` の docs 更新

**Out-of-Scope**
- JSONL canonical model の置換
- storage backend の変更

---

## Assumptions

1. `DedupPolicy` と retention days はすでに `enums.py` または `ingestion.py` に存在。
2. source type 別 dedup 閾値: RULE/DECISION は dedup 閾値を高く（重複感知しにくい = 保存しやすい）、CONVERSATION は低く（同内容は積極的に dedup）。
3. source type 別 retention: RULE/DECISION は長期保持（無期限）、CONVERSATION は 90 日程度。

## Implementation Steps

- [ ] 1. `DedupPolicy` に source type 別 threshold を追加（または type-keyed dict）
  - RULE/DECISION: `dedup_threshold=0.98`（同一でなければ保持）
  - CONVERSATION: `dedup_threshold=0.85`（近ければ dedup）
  - FAILURE: `dedup_threshold=0.90`
- [ ] 2. `ingestion.py` の dedup ロジックで `source_type` に応じた threshold を使用
- [ ] 3. `jsonl_store.py` の retention filter に source type 別 retention_days を追加
  - RULE/DECISION: `retention_days=None`（無期限）
  - FAILURE: `retention_days=180`
  - CONVERSATION: `retention_days=90`
- [ ] 4. `/memory prune` の docs に source type 別 retention policy を記述
- [ ] 5. tests: source type 別 dedup 閾値の動作テスト追加
