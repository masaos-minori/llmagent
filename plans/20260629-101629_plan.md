## 3. Assumptions
- The string `"external"` is NOT used as a `startup_mode` value anywhere in existing docs (verified: zero occurrences). The requirement title overstates the problem; the actual issue is (a) missing cross-field validation for `ondemand`+`http` and (b) inconsistent prose describing `persistent` (http) across docs.
- `persistent` (http) canonically means "agent connects to a pre-existing HTTP server; no lifecycle action taken" â€” this is already the correct meaning per `04_mcp_01_system_overview.md` and `_validate_cross_fields`.
- `ondemand` is exclusively valid for `stdio` transport (per lifecycle code in `stdio_lifecycle.py`). Using `ondemand` with `http` silently has no effect (factory.py line 77 skips it), so a `ValueError` should be raised at config construction time.
- Option A (use `persistent` for externally managed HTTP, remove `external` from docs) is effectively already implemented; the plan just needs to enforce it with validation and tests.
