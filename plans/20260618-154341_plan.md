## Goal

---

- `scripts/rag/pipeline.py` тАФ add `"failure"` status to `StageResult`, wire fallback reasons for refiner and HTTP

## Implementation Steps

1. **Enrich `StageResult` with failure tracking**
   - When a stage raises, catch and record `{"status": "failure", "fallback_reason": str(e)}`
   - Criterion: exceptions are visible in `last_stage_results`

2. **Expose refiner/HTTP fallback reasons**
   - Refiner: record `"refiner_returned_none"` or `"refiner_exception: {e}"`
   - HTTP: record `"http_fallback: {reason}"` in the augment path
   - Criterion: fallback reasons propagated through `StageResult`

3. **Wire REPL debug path**
   - Pass `debug_fn` callback and `on_status` to `pipeline.augment()` in `/rag search --debug`
   - Show fallback reasons alongside stage timings
   - Criterion: `/rag search --debug` shows full pipeline state

4. **Add ingestion skipped-chunk counter**
   - Return `(doc_id, n_success, n_failed)` or similar from `ingest_url_group()`
   - Log aggregate at end of `ingest_all()`
   - Criterion: skipped chunks are visible in ingestion log

5. **Document diagnostic fields**
   - Update `docs/05_agent_10_operations-and-observability.md`
   - Criterion: operator knows what diagnostics are available

6. **Lint, type, arch, test validation**
   - `ruff check scripts/` тАФ 0 errors
   - `mypy scripts/` тАФ no new errors
   - `lint-imports` тАФ 0 violations
