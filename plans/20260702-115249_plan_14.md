## Assumptions

1. `chunks_fts` is an FTS5 content table (`content='chunks'`, `content_rowid='chunk_id'`). In this mode, standard `DELETE FROM chunks_fts WHERE ...` is NOT valid SQL on the FTS5 virtual table itself; deletion must use the FTS5 `'delete'` command syntax.
2. `chunks.embedding` column does NOT exist in the canonical `schema_sql.py` definition; `rebuild_vec()` reads `embedding` from `chunks` — this column is added via `create_schema.py` ALTER TABLE migrations, not the base DDL. Tests must either migrate or use a fixture DB that includes the column.
3. `chunks_vec_ad` trigger (line 61–63 of `schema_sql.py`) auto-deletes `chunks_vec` rows when a `chunks` row is deleted. Therefore `delete_document_chain()` deletes `chunks_vec` explicitly before deleting `chunks`, which is harmless redundancy but not incorrect.
4. The `_FakeSQLiteHelper` pattern from `test_fts_fallback.py` is the standard test harness for in-memory SQLite tests; new tests should follow the same pattern.
5. `tests/test_fts_fallback.py` schema includes `chunks_vec` as a plain table (`CREATE TABLE IF NOT EXISTS chunks_vec (chunk_id INTEGER PRIMARY KEY)`), not a `vec0` virtual table. New tests must use the same simplified schema since `vec0` extension is unavailable in the test environment.
6. `reconcile_url()` line 103 (`DELETE FROM chunks_fts WHERE chunk_id IN (...)`) will silently fail or produce incorrect results on an FTS5 virtual table; this is a latent bug that must be fixed before the test can pass.

**UNK-01 resolution (confirmed by code inspection):**
`reconcile_url()` at line 103 of `rag_maintenance_service.py` uses:
```python
db.execute(f"DELETE FROM chunks_fts WHERE chunk_id IN ({placeholders})", tuple(chunk_ids))
```
FTS5 virtual tables expose a `content=` external-content table configuration. In this mode, `chunks_fts` does NOT have a `chunk_id` column visible for WHERE clauses — only FTS-internal columns (`rowid`, the indexed text column). A direct `DELETE FROM chunks_fts WHERE chunk_id = ?` will raise `OperationalError: no such column: chunk_id`. The fix is to use FTS5 delete command per row:
```python
for cid in chunk_ids:
    row = db.execute("SELECT content, normalized_content FROM chunks WHERE chunk_id = ?", (cid,)).fetchone()
    if row:
        fts_text = row["normalized_content"] or row["content"]
        db.execute("INSERT INTO chunks_fts(chunks_fts, rowid, content) VALUES('delete', ?, ?)", (cid, fts_text))
```
Or alternatively use `INSERT INTO chunks_fts(chunks_fts) VALUES('delete-all')` then re-insert all rows for the document. The current per-URL reconcile uses the per-row approach as it is already inside a loop.
