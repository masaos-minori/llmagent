# Plan: _embed_and_store のフォールバックテストの追加 — test_ingester.py

## Goal

`tests/test_ingester.py` の `TestEmbedAndStore` クラスに、
ファイルパース失敗・空コンテンツ・無効 chunk_index の各フォールバック経路をテストするケースを追加し、
`_embed_and_store()` のエラーパスカバレッジを強化する。

## Scope

**In-Scope:**
- `tests/test_ingester.py` — `TestEmbedAndStore` にテストケース追加

**Out-of-Scope:**
- `scripts/rag/ingestion/ingester.py` の実装変更
- `TestIngestUrlGroup` 等の他のテストクラスの変更

## Assumptions

1. `_embed_and_store` の制御フロー:
   - `_read_chunk_json(path)` が `None` → `return (False, False)` (ファイルパース失敗)
   - `_get_embedding(content)` が `None` → `return (False, False)` (埋め込み失敗)
     - `_get_embedding` は `not text or not text.strip()` のとき即 `None` を返す (空コンテンツ)
   - 成功 → `return (True, True)`
2. 無効 `chunk_index` (`"abc"` 等) は `ValueError`/`TypeError` をキャッチして `idx=0` にフォールバックし、処理を継続する。
3. `_read_chunk_json` を `patch.object` でパッチすると、ファイル非存在エラーなしにパース失敗を再現できる。
4. 空コンテンツテストはチャンクファイルの `content` フィールドを `""` に設定することで再現できる。

## Design

### _embed_and_store の制御フロー (ingester.py:379-420)

```
_read_chunk_json(path) → None?  →  return (False, False)  ← [1] パース失敗
content = data.get("content", "")
_get_embedding(content)
  └─ not text.strip()?  →  return None  ← [2] 空コンテンツ
  └─ HTTP 全リトライ失敗  →  return None  ← [3] 既存テスト
embedding is None?  →  return (False, False)
                                           ← [4] chunk_index 無効 → idx=0 で成功
```

### 追加するテストケース

**1. `test_read_chunk_json_returns_none_fails_immediately`**
- `patch.object(ingester, "_read_chunk_json", return_value=None)` でパース失敗を注入
- 期待: `result == (False, False)` かつ `chunks` テーブルが空

**2. `test_empty_content_causes_embedding_failure`**
- `_write_chunk(..., content="")` で空コンテンツのチャンクを作成
- `_get_embedding("")` → `not text` → `None` → 埋め込み失敗
- 期待: `result == (False, False)` かつ `chunks` テーブルが空

**3. `test_invalid_chunk_index_falls_back_to_zero`**
- `_write_chunk(...)` で `chunk_index="invalid_str"` を含む JSON を作成
  (既存 `_write_chunk` は `chunk_index` に任意値を受け付ける)
- 期待: `result == (True, True)` かつ `SELECT chunk_index FROM chunks` == 0
