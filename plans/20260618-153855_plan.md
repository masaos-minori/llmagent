# Plan: Define the Boundary Between RAG, MDQ, and the Agent Layer

Derived from `01_plan.md` using requirement `requires/20260618_46_require.md`.

---

## Goal

Clear ownership boundaries for document retrieval and indexing: when to use RAG, when to use MDQ, and how the agent accesses each.

---

## Scope

**In scope:**
- `docs/04_mcp-mdq.md` — **create** new doc with RAG vs MDQ boundary definition
- `docs/03_rag_01_system_overview.md` — reference MDQ boundary doc
- `docs/05_agent_09_data-layer.md` — clarify agent-side document access paths
- `docs/03_rag_90_inconsistencies_and_known_issues.md` — resolve OQ-2

**Out of scope:**
- Code changes (MDQ stub status, routing, tool implementations)
- Promoting MDQ from experimental to production

---

## Assumptions

1. MDQ remains an experimental/complementary service — not replacing RAG for general-purpose retrieval.
2. RAG is the primary document retrieval system for the agent; MDQ is for Markdown-specific queries where FTS5 is sufficient.
3. The agent always accesses both through MCP tool calls (not direct DB access).
