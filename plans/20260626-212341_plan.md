## Goal

Implement stable chunk IDs for MDQ chunks to replace integer row IDs with deterministic identifiers based on normalized source path, heading path, ordinal, and content hash.

## Scope

**In-Scope**:
- Generate stable `chunk_id` from deterministic inputs (normalized source path, heading path, ordinal, content hash)
- Update `search_docs` to return `chunk_id`
- Update `get_chunk` to accept stable `chunk_id`
- Update `grep_docs` to return `chunk_id`
- Update models.py type annotations for chunk_id fields

**Out-of-Scope**:
- Token counting implementation (placeholder for now)
- Adding new tools or features

## Assumptions

1. chunk_id = SHA-256 hash of (normalized source path + heading_path + ordinal + content_hash)
2. normalized_content = content with extra whitespace collapsed
3. backward compatibility is not required — prefer stable IDs over legacy integer IDs

- **Blast Radius**: High — changes the database schema and all indexing/querying code. Existing consumers must be updated.

## Implementation Steps

1. **Phase 1: Define chunk_id generation algorithm**
   - Generate stable chunk_id from SHA-256 of (normalized source path + heading_path + ordinal + content_hash)
   - Compute content_hash = SHA-256 of content text
   - Compute normalized_content = content with extra whitespace collapsed

2. **Phase 2: Update models.py**
   - ChunkRecord: already has chunk_id: str field
   - GetChunkRequest.chunk_id: int → str
   - GrepDocMatch.chunk_id: int → str
   - SearchResultItem: add chunk_id field

3. **Phase 3: Update indexer.py**
   - Generate chunk_id for each section during indexing
   - Write to documents/chunks tables instead of sections
   - Compute and store content_hash, normalized_content, char_count

4. **Phase 4: Update search.py**
   - Return chunk_id in search results
   - Query chunks_fts instead of sections_fts

5. **Phase 5: Update service.py**
   - get_chunk(): query chunks table by chunk_id
   - grep_docs(): return chunk_id in matches
   - stats(): query documents/chunks tables
