# Work Plan: Update tests for HTTP-only MCP transport

## Goal

Remove stdio-specific tests and rewrite MCP config/tool executor tests for HTTP-only transport.

## Scope

**In-Scope:**
- Remove stdio-specific test classes/functions from 7 test files
- Replace stdio config validation tests with HTTP-only equivalents
- Update StdioTransport references in remaining tests (mocks, spec)
- Ensure no test imports or references StdioTransport, TransportType.STDIO, HealthcheckMode.PING_TOOL

**Out-of-Scope:**
- Runtime removal of StdioTransport (handled in separate issue)
- Changes to shared/tool_executor.py source code
- Changes to shared/mcp_config.py source code (only test updates)

## Assumptions

1. StdioTransport will be fully removed from shared/tool_executor.py in a future PR, so no new stdio test code should be written
2. McpServerConfig can still accept TransportType.STDIO for backward compatibility during transition, but tests should not exercise it
3. HealthcheckMode.PING_TOOL is stdio-only and will be removed alongside StdioTransport
4. The integration tests (test_agent_mcp_integration.py) that use real subprocesses for stdio will need to be rewritten or disabled
