## Design

**`broker.py` への追加メソッド:**

```python
_SLOW_CONSUMER_THRESHOLD = 100  # queue depth at which a subscriber is considered slow

def subscriber_count(self) -> int:
    return len(self._subscribers)

def max_queue_depth(self) -> int:
    if not self._subscribers:
        return 0
    return max(sub.queue.qsize() for sub in self._subscribers)

def slow_consumer_count(self) -> int:
    return sum(1 for sub in self._subscribers if sub.queue.qsize() >= _SLOW_CONSUMER_THRESHOLD)
```

**`health()` の拡張 (broker メトリクス部分):**

```python
# broker metrics (sync; runs on event loop)
active_subscribers = _broker.subscriber_count() if _broker else 0
max_queue_depth = _broker.max_queue_depth() if _broker else 0
slow_consumers = _broker.slow_consumer_count() if _broker else 0

# degraded conditions
if max_queue_depth >= 500:
    degraded_reasons.append("broker_queue_backlog_high")
if slow_consumers > 0:
    degraded_reasons.append("slow_consumers_detected")

# response
return {
    ...,
    "active_subscribers": active_subscribers,
    "max_queue_depth": max_queue_depth,
    "slow_consumers": slow_consumers,
}
```

Note: broker メトリクス取得は asyncio.Queue.qsize() (O(1)) であり、to_thread は不要。health() がすでに asyncio.to_thread でDB クエリを行う場合、broker メトリクスはその前後に同期的に取得する。
