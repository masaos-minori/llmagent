### 2.1 agent/config.py — re-export 削除

```python

    DbConfig,
    build_db_config,

```

外部 caller が 0件のため、単純削除。

### 2.2 agent/lifecycle.py — facade 削除 + 型分離

**Blocking Unknown 解決後に確定。** 設計オプション:

**Option A — Protocol:**
```python
# agent/lifecycle_protocol.py (新規)
from typing import Protocol
class LifecycleManagerProtocol(Protocol):
    async def start_all(self) -> None: ...
    async def shutdown_all(self) -> None: ...
    async def restart(self, name: str) -> None: ...
    # ...共通 public methods
```
- `AgentContext.services.lifecycle: LifecycleManagerProtocol`
- `factory.py` は startup_mode に応じて `HttpServerLifecycleManager` または `StdioServerLifecycleManager` を直接渡す

**Option B — Union 型:**
```python
from agent.http_lifecycle import HttpServerLifecycleManager
from agent.stdio_lifecycle import StdioServerLifecycleManager
LifecycleManager = HttpServerLifecycleManager | StdioServerLifecycleManager
```

**推奨: Option A (Protocol)** — 具体型への直接依存を避け、テスト時の mock が容易。

### 2.4 agent/llm_turn_runner.py — interface 整理

現状: `on_turn_start` / `on_turn_end` は Orchestrator から素通りで渡される（"same ctx/callbacks" 前提）。

変更方針: Orchestrator が callbacks を自身の `_handle_llm_turn` 内で直接呼び、`LLMTurnRunner.run()` は callback を受け取らない設計へ。`tracer` は LLMTurnRunner の内部 span 計装に必要なため残す（Non-blocking unknown の調査結果次第）。

---

### 2.5 agent/orchestrator.py — 重複エラー処理削減

現状: `_handle_llm_turn` と `_process_turn` の両方で `LLMTransportError` を捕捉。

変更方針: `_process_turn` での再捕捉を削除し、`_handle_llm_turn` を単一の LLMTransportError 処理入口とする。

---

### 2.6 agent/history.py — alias / wrapper 削除

削除対象:
