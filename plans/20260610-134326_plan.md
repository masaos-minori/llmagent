## Assumptions

1. `MemoryQuery.session_id` は FTS 検索・KNN 検索の SQL に現在反映されていない。今回は削除方針（不使用フィールドを型から除去）を採用する。
2. `store.py` の `INSERT OR REPLACE` は `INSERT INTO memories (...) ON CONFLICT(memory_id) DO UPDATE SET ...` に変更する。`memories_fts` の FTS 同期は手動 delete→insert のまま維持し、同期失敗は全体失敗にする（trigger 化は別 issue）。
3. JSONL は audit log（append-only イベントログ）として位置付け、malformed 行は隔離ファイルに書き出して継続せず例外を投げる。
4. `EmbeddingResult.error_kind` は `StrEnum` に変更し、既存の文字列リテラル比較が壊れないようにする。
5. `"query: {text}"` のハードコードを `EmbeddingClientConfig.query_prefix: str = "query: "` に切り出す。
6. `_score(0.0, ...)` after RRF の問題（FTS の BM25 信号を捨てて importance/pin/recency ブーストだけ再適用）は、RRF スコアをそのまま採用する方式に変更する（`_score` 呼び出しを削除）。
7. `MemoryQuery` の `limit` / `query` バリデーションは `__post_init__` に追加する。

## Design

### D-1. `EmbeddingErrorKind` StrEnum

```python
class EmbeddingErrorKind(StrEnum):
    DISABLED      = "disabled"
    CIRCUIT_OPEN  = "circuit_open"
    TIMEOUT       = "timeout"
    HTTP_ERROR    = "http_error"
    INVALID_RESPONSE = "invalid_response"
    UNKNOWN_ERROR = "unknown_error"
```

`EmbeddingResult.error_kind` の型を `str | None` → `EmbeddingErrorKind | None` に変更。`StrEnum` なので既存の文字列比較 `r.error_kind == "disabled"` は壊れない。

### D-2. `MemoryQuery` バリデーション + session_id 削除

```python
@dataclass
class MemoryQuery:
    query: str
    memory_type: str | None = None
    limit: int = 10

    def __post_init__(self) -> None:
        if not self.query.strip():
            raise ValueError("MemoryQuery.query must not be empty")
        if self.memory_type is not None and self.memory_type not in ("semantic", "episodic"):
            raise ValueError(f"MemoryQuery.memory_type must be 'semantic', 'episodic', or None")
        if self.limit < 1:
            raise ValueError(f"MemoryQuery.limit must be >= 1, got {self.limit}")
```

### D-3. `store.py` — `ON CONFLICT DO UPDATE`

```python
_UPSERT_SQL = (
    "INSERT INTO memories"
    " (memory_id, memory_type, source_type, session_id, turn_id,"
    "  project, repo, branch, content, summary, tags,"
    "  importance, pinned, created_at, updated_at)"
    " VALUES (?,?,?,?,?,?,?,?,?,?,?,?,?,?,?)"
    " ON CONFLICT(memory_id) DO UPDATE SET"
    "   memory_type=excluded.memory_type,"
    "   source_type=excluded.source_type,"
    "   content=excluded.content,"
    "   summary=excluded.summary,"
    "   tags=excluded.tags,"
    "   importance=excluded.importance,"
    "   pinned=excluded.pinned,"
    "   updated_at=excluded.updated_at"
)
```

### D-4. `retriever.py` — RRF 後再スコア削除

現状 (line 278-280):
```python
# After RRF merge, re-apply _score so FTS boosts are reflected.
for hit in merged:
    hit.score = _score(0.0, hit.entry, project, repo, self._recency_days)
merged.sort(key=lambda h: h.score, reverse=True)
```

変更後: RRF スコアをそのまま採用し、余計な再スコアは行わない。

```python
# RRF score captures rank fusion; no re-scoring needed.
merged.sort(key=lambda h: rrf_scores[h.entry.memory_id], reverse=True)
```

### D-5. `jsonl_store.py` — malformed 行隔離

malformed 行を例外にするか隔離するかは呼び出し元が選択できるようにする。`read_all()` は `strict: bool = False` パラメータを追加し、`strict=True` の場合は malformed で即 `ValueError`、デフォルトは隔離ファイルへ書き出して継続する。

### D-6. `embedding_client.py` — query_prefix policy 化

```python
@dataclass
class EmbeddingClientConfig:
    embed_url: str = ""
    timeout: float = 5.0
    max_retries: int = 2
    circuit_open_after: int = 3
    circuit_reset_sec: float = 60.0
    query_prefix: str = "query: "  # new
```

`_fetch_embedding()` 内の `f"query: {text}"` を `f"{config.query_prefix}{text}"` に変更。
