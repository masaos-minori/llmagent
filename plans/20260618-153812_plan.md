# Plan: Document `_augment_http()` Fallback Conditions Clearly

Derived from `01_plan.md` using requirement `requires/20260618_44_require.md`.

---

## Goal

The exact conditions under which `_augment_http()` returns `None`, `""`, or a normal context string are documented clearly at the method and caller level.

---

## Scope

**In scope:**
- `scripts/rag/pipeline.py` — update docstrings on `_augment_http()` and `augment()`
- `scripts/rag/pipeline_service.py` — update docstring on `call_rag_service()`
- `docs/03_rag_01_architecture.md` — optionally mention the HTTP fallback chain

---

## Assumptions

1. The current return contract is correct and stable — only the documentation is insufficient.
2. The `call_rag_service` docstring is mostly complete but could use explicit list of error types.

---

## Return contract (current behavior)

| Return value | Condition |
|---|---|
| `str` (context) | HTTP 200, `body.context` is a non-empty string |
| `""` (empty string) | HTTP 200, `body.context` is `None` or missing |
| `None` | 4xx client error, transport error, JSON parse error, or all retries exhausted on 5xx |

In `augment()`: `None` triggers in-process fallback; `""` and normal strings pass through to the caller.
