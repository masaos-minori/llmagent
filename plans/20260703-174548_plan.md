## Goal

Rename `scripts/shared/tool_routing.py` → `scripts/shared/tool_routing_validation.py` to reflect its actual responsibility (drift validation, not runtime routing). Update all import sites to use the new module name. No backward-compatible re-export stub.

## Scope

**In-Scope:**
- Rename `scripts/shared/tool_routing.py` → `scripts/shared/tool_routing_validation.py`
- Update module docstring in the renamed file
- Fix `scripts/agent/repl_health.py`: import `validate_routing_against_config` from `shared.tool_routing_validation` (currently broken — imports from `shared.tool_registry` where the function is not defined)
- Fix `tests/test_tool_registry.py`: import `validate_all_routing`, `validate_routing_against_config`, `validate_routing_against_live` from `shared.tool_routing_validation` (currently broken — imports from `shared.tool_registry`)

**Out-of-Scope:**
- Leaving any backward-compatible re-export in `shared.tool_registry` or `shared.tool_routing`
- Changing validation semantics
- Changing `ToolRouteResolver`

Current state of imports referencing validation functions:

| File | Current import | Status |
|---|---|---|
| `scripts/agent/repl_health.py:272` | `from shared.tool_registry import validate_routing_against_config` | Broken — function not in `tool_registry.py` |

Both callers reference a nonexistent location. The rename to `tool_routing_validation.py` and updating these imports to the correct module fixes both issues simultaneously.

### No current callers import `shared.tool_routing` directly

Grep confirms zero imports of `shared.tool_routing` (the module itself). The callers use the wrong module (`shared.tool_registry`) where the functions were presumably moved from. The rename plan implicitly cleans up this confusion.

### Overlap with require-28 plan

The require-28 plan (`20260703-174440_plan.md`) also covers this rename but adds re-export stubs. Require-29 explicitly forbids re-export stubs ("Do not leave a backward-compatible re-export stub"). If both plans are implemented, require-28's re-export stubs should be removed as part of this plan.
