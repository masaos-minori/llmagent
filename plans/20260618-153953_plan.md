# Plan: Add FTS5 Fallback Integration Tests When `normalized_content` IS NULL

Derived from `01_plan.md` using requirement `requires/20260618_47_require.md`.

---

## Goal

FTS5 fallback to raw `content` when `normalized_content` is NULL is verified for English and code chunks through the application-level search path.

---

## Scope

**In scope:**
- `tests/test_rag_repository.py` — add real-DB integration tests for `RagRepository.fts_search()` with NULL normalized_content
- English content with NULL normalized → matches via FTS5
- Code content with NULL normalized → matches via FTS5
- Non-NULL normalized → takes precedence (COALESCE verification)

**Out of scope:**
- Japanese content (already covered in `test_fts_japanese.py`)
- Trigger-level tests (already covered)
- Performance/BM25 tests

---

## Assumptions

1. Real in-memory SQLite can run the full `schema_sql.py` schema without the vec0 extension (vector columns are separate and not needed for FTS-only tests).
2. The `RagRepository` constructor accepts a DB connection that was created externally.

## Implementation Steps

1. **Check `RagRepository.__init__()` signature**
   - Determine if it accepts a plain DB connection or requires `SQLiteHelper`
   - Criterion: know the test setup API

2. **Add test fixtures**
   - `_make_db()` → in-memory SQLite with full rag schema (excluding vec0 tables if they cause issues)
   - `_make_repository(db)` → `RagRepository` wrapping the in-memory DB
   - Criterion: reusable fixture pattern

3. **Add `test_english_fts_fallback`**
   - Insert a chunk with `content="SELECT * FROM users"`, `normalized_content=NULL`
   - Call `fts_search("SELECT")` — confirm at least 1 hit
   - Criterion: English code content matched via fallback

4. **Add `test_code_fts_fallback`**
   - Insert a chunk with `content="def foo:\n  return 42"`, `normalized_content=NULL`
   - Call `fts_search("return 42")` — confirm at least 1 hit
   - Criterion: code content matched via fallback

5. **Add `test_normalized_content_takes_precedence`**
   - Insert a chunk with `content="raw text"`, `normalized_content="better text"`
   - Verify FTS matches "better" but not "raw" (or verify BM25 favors normalized terms)
   - Criterion: COALESCE precedence confirmed via application path

6. **Lint, type, test validation**
   - `ruff check tests/` — 0 errors
   - `mypy tests/` — no new errors
   - `pytest tests/test_rag_repository.py -x -q` — all pass
