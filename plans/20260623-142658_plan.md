# Plan: Memory Layer Consistency Check and Rebuild Hardening

**Requirement:** `requires/20260622_78_require.md`

---

## Goal

Fix three gaps in the existing `/memory check-consistency` and `/memory rebuild` operator flows:
1. The consistency check doesn't include `vec` in the `ok` decision even when embedding is enabled.
2. The rebuild command clears `memories_vec` without warning operators that vec is now empty.
3. Consistency mismatch has no per-session stat counter ŌĆö operators can't see historical failure count without reading audit logs.

---

## Scope

**In:**
- `scripts/agent/commands/cmd_memory.py` ŌĆö fix `ok` logic in `_memory_check_consistency()` to include vec when `memory_embed_enabled=True`; add vec cleared warning in `_memory_rebuild()`
- `scripts/agent/context.py` ŌĆö add `stat_memory_consistency_failures: int = 0` to `RuntimeStats`
- `scripts/agent/commands/cmd_config_stats.py` ŌĆö expose `stat_memory_consistency_failures` in `/stats`
- `scripts/agent/commands/models.py` ŌĆö add `memory_consistency_failures: int = 0` to `StatsViewModel`
- `scripts/agent/repl.py` ŌĆö persist `memory_consistency_failures` in `_persist_session_diagnostics()`

**Out:**
- Implementing async re-embedding after rebuild (marked as "Needs confirmation" per requirement)
- Changing `check_consistency()` in `store.py` (already correct ŌĆö it returns all three counts)
- Changing `rebuild_from_jsonl()` in `store.py` (semantics are correct ŌĆö JSONL is source of truth for structured data; vec re-indexing is a separate operation)
- Adding new `ConsistencyReport` fields (existing `memories`, `fts`, `vec` counts are sufficient)

---

## Assumptions

1. `check_consistency()` in `store.py` already returns `ConsistencyReport(memories, fts, vec)` ŌĆö all three counts. The gap is only in the CLI's `ok` decision.
2. `memory_embed_enabled=True` means `memories_vec` should have the same row count as `memories`. When `memory_embed_enabled=False`, `memories_vec` is expected to be 0 ŌĆö vec mismatch is not meaningful.
3. After `rebuild_from_jsonl()`, `memories_vec` is always empty (0 rows) because rebuild only calls `_write_fts()`, not `_write_vec()`. This is correct behavior (re-embedding requires async calls to the embedding service), but it must be communicated clearly.
4. `ctx.cfg.memory.memory_embed_enabled` is accessible in `_memory_check_consistency()` via `self._ctx.cfg.memory.memory_embed_enabled`.
5. `RuntimeStats` is a `@dataclass` (not frozen) ŌĆö adding a new field with a default value is safe.

## Design

### Change 1: `cmd_memory.py` ŌĆö fix `ok` logic

**Before (line 272):**
```python
ok = (jsonl_count == report.memories) and (report.memories == report.fts)
```

**After:**
```python
embed_enabled = self._ctx.cfg.memory.memory_embed_enabled
vec_expected = embed_enabled  # vec must match when embedding is on
ok = (
    (jsonl_count == report.memories)
    and (report.memories == report.fts)
    and (not vec_expected or report.vec == report.memories)
)
```

Add a `Vec expected` row to the table:
```python
rows = [
    ["JSONL records", str(jsonl_count)],
    ["SQLite memories", str(report.memories)],
    ["FTS5 rows", str(report.fts)],
    ["Vec rows", str(report.vec)],
    ["Vec check required", "Yes" if vec_expected else "No (embedding disabled)"],
    ["Consistent", "Yes" if ok else "NO - use /memory rebuild to repair"],
]
```

After the table, increment the stat:
```python
if not ok:
    self._ctx.stats.stat_memory_consistency_failures += 1
```

### Change 2: `cmd_memory.py` ŌĆö rebuild vec warning

After the success write in `_memory_rebuild()`, add:
```python
embed_enabled = self._ctx.cfg.memory.memory_embed_enabled
if not dry_run and embed_enabled:
    self._out.write(
        "  Note: memories_vec cleared ŌĆö embeddings not re-indexed."
        " Run /memory rebuild again after re-embedding or disable use to silence."
    )
```

**Why:** `rebuild_from_jsonl()` calls `DELETE FROM memories_vec` but does not call `_write_vec()`. Until re-embedding runs, semantic search returns no results. This is correct behavior (JSONL is source of truth; vec is a derived index), but operators must know.

## Vec Re-embedding: Needs Confirmation

The requirement states "Add an operator-facing rebuild or reindex flow from JSONL to SQLite." The current `rebuild_from_jsonl()` rebuilds `memories` and `memories_fts` but not `memories_vec`. Full re-indexing would require:
1. An async embedding call to the embedding service for each entry
2. A progress indicator (could take minutes for large collections)
3. Error handling for partial embedding failures

This is out of scope for this plan because:
- It requires async I/O not available in the sync `store.py` methods
- Partial re-embedding failure semantics are unclear
- The embedding service may be unavailable during maintenance

**Recommendation:** A future `/memory reindex-vec` subcommand (or `--reindex` flag on rebuild) should be added as a separate task once re-embedding semantics are confirmed.
