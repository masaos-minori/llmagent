## Goal

Remove all active stdio transport references from four MCP documentation files (04_mcp_01, 04_mcp_02, 04_mcp_03, 04_mcp_90) and rewrite transport sections as HTTP-only (persistent + subprocess), reflecting the current codebase where `TransportType.STDIO` no longer exists in `scripts/shared/mcp_config.py`.

## Scope

**In**:
- `docs/04_mcp_01_system_overview.md` — remove stdio transport section, stdio rows in startup-mode table, `StdioTransport` component row, stdio call timeout row, cross-reference to `04_mcp_02 §When to use stdio`
- `docs/04_mcp_02_protocol_and_transport.md` — remove "stdio Transport" section, `__list_tools__` RPC section, "When to use / When NOT to use stdio" guidance, stdio request/response format blocks, stdio scalability design note, `run_stdio()` in entry-point pattern, `StdioTransport error handling` note, stdio relevance note in dispatch_tool; rewrite "HTTP vs stdio Mode" comparison table as "HTTP Startup Modes" (persistent vs subprocess)
- `docs/04_mcp_03_routing_lifecycle_and_execution.md` — remove `StdioTransport` API reference block, `ensure_ready (ondemand stdio)` section, stdio rows in startup_mode behavior table, `healthcheck_mode="ping_tool"` stdio reference in Watchdog section, stdio lines in Lifecycle Flow diagram
- `docs/04_mcp_90_inconsistencies_and_known_issues.md` — remove MCP-01 entry ("Startup mode terminology mismatch"), which described a StdioTransport conflation that this change set resolves

**Out**:
- `docs/04_mcp_06_configuration_and_operations.md` — owned by require_26 (McpServerConfig dataclass alignment); do not touch in this plan
- `scripts/shared/mcp_config.py` — already HTTP-only; no code changes required
- `scripts/shared/tool_executor.py` — StdioTransport already removed from source; no code changes required
- `mcp/server.py` `run_stdio()` method — code change out of scope; UNK-01 determines whether the doc reference is removed or annotated as deprecated
- `docs/04_mcp_04_server_catalog.md`, `docs/04_mcp_05_security_and_safety_model.md` — no stdio references found in pre-edit grep

## Assumptions

1. `TransportType.STDIO` and `StdioTransport` are already removed from production code — confirmed: `scripts/shared/mcp_config.py` defines only `TransportType.HTTP`; `tool_executor.py` has no stdio references in grep.
2. The `mcp/server.py` `run_stdio()` method and `--stdio` flag may still exist in server base class code; this plan removes only documentation references unless UNK-01 confirms otherwise.
3. Historical context preservation is not required — requirement specifies active references must be removed.
4. require_26 will handle `docs/04_mcp_06_configuration_and_operations.md` independently; no ordering dependency.
5. MCP-01 in `04_mcp_90` is fully resolved by this change set; removing it is safe.
6. Issue IDs in `04_mcp_90` will not be renumbered after MCP-01 removal; existing IDs (MCP-02 through MCP-09) remain stable.
