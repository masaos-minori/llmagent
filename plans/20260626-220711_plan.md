## Goal

Define MDQ tool error taxonomy and error handling by creating explicit MDQ domain exceptions and mapping them to appropriate MCP tool responses.

## Scope

**In-Scope**:
- Create MdqValidationError, MdqAuthorizationError, MdqNotFoundError, MdqIndexNotReadyError, MdqDatabaseError, MdqConsistencyError
- Map validation errors to tool error
- Map not found to tool error
- Map unauthorized path to tool error
- Map DB unavailable to health impact
- Handle unexpected exceptions consistently

**Out-of-Scope**:
- Adding new tools or features
- Changes to other MCP servers' error handling

## Assumptions

1. Tool-level errors return is_error=true in MCP response
2. Transport-level failures remain distinguishable from tool errors
3. Logs include error kind for operational debugging

## Affected Areas & Tool Evidence

- **Affected Files**:
  - `scripts/mcp/mdq/models.py` — add error class hierarchy (MdqServiceError base, subclasses for each error type)
  - `scripts/mcp/mdq/service.py` — raise appropriate exceptions per error condition
  - `scripts/mcp/mdq/server.py` — update FastAPI exception handlers for each error type

- **Blast Radius**: Medium — changes error response format. Existing consumers expecting HTTP 500 will need updates; tool-level errors will now return appropriate status codes.

## Implementation Steps

1. **Phase 1: Create error class hierarchy in models.py**
   - MdqServiceError(RuntimeError) — base class
   - MdqValidationError(MdqServiceError) — for validation errors
   - MdqAuthorizationError(MdqServiceError) — for authorization errors
   - MdqNotFoundError(MdqServiceError) — for not found errors
   - MdqIndexNotReadyError(MdqServiceError) — for index not ready errors
   - MdqDatabaseError(MdqServiceError) — for database errors
   - MdqConsistencyError(MdqServiceError) — for consistency errors

2. **Phase 2: Update service.py to raise appropriate exceptions**
   - Validation errors: raise MdqValidationError
   - Not found: raise MdqNotFoundError
   - Unauthorized path: raise MdqAuthorizationError
   - DB unavailable: raise MdqDatabaseError
   - Index not ready: raise MdqIndexNotReadyError

3. **Phase 3: Update FastAPI exception handlers in server.py**
   - Replace single handler with per-error-type handlers
   - Map tool errors to HTTP 400
   - Map DB errors to HTTP 503
   - Map others to HTTP 500

4. **Phase 4: Add error kind logging**
   - Log error kind in FastAPI exception handlers
   - Include error kind in audit log detail
