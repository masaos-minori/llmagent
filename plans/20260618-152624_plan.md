# Plan: Remove RAG-Specific DB Operations from AgentSession

Derived from `01_plan.md` using requirement `requires/20260618_24_require.md`.

---

## Goal

`AgentSession` has zero RAG-layer database dependencies; all RAG document operations go through a RAG-facing service or MCP path.

---

## Scope

**In scope:**
- `scripts/agent/session.py` — verify zero RAG imports and methods (already the case)
- `scripts/agent/services/db_maintenance_service.py` — move RAG subcommands to RAG-side service if they live here
- `scripts/agent/commands/cmd_db.py` — verify `/db` RAG subcommands route through RAG service, not session
- `scripts/db/maintenance.py` — identify if RAG functions need relocation to `rag/`
- `scripts/rag/` — optionally create `rag/maintenance.py`
- `docs/05_agent_04_state-and-persistence.md` — document session vs RAG responsibility boundary

**Out of scope:**
- `AgentSession` SQLiteHelper usage for session/message/note tables (legitimate session concerns)
- RAG pipeline query logic
- MCP server changes

## Assumptions

1. `AgentSession` currently has zero RAG methods (confirmed by grep) — the requirement may target latent or transitive coupling.
2. The `/db` command dispatches RAG subcommands through `DbMaintenanceService` — needs verification that this service does not use `AgentSession`.
3. If coupling exists, it is in the service/command layer, not in `AgentSession` itself.
