## Goal

Reorganize MCP test files so that each file has exactly one clearly-scoped responsibility: routing behavior, config model validation, registry drift validation, and count regression testing.

## Scope

**In**: Reorganize test responsibilities across 4 files without changing production code.
- Move `TestStartupModeValidation` and `TestValidateRoutingDrift` from `tests/test_route_resolver.py` to the correct destination files.
- Create `tests/test_tool_registry.py` to own registry drift validation tests.
- Rename classes and test methods to match their narrowed scope.

**Out**: No changes to production code (`scripts/shared/`). No changes to `conftest.py`, `pyproject.toml`, or CI workflow files. No changes to `test_github_tool_registry.py` or `test_tool_registry_counts.py` content (counts-only file stays as-is).

## Assumptions

1. `tests/test_tool_registry.py` does not yet exist; it must be created.
2. `conftest.py` appends `scripts/` to `sys.path`; the new file inherits this automatically.
3. No CI job references test files by explicit name (pattern-based discovery via `uv run pytest tests/`).
4. `TestStartupModeValidation` belongs in `test_mcp_config.py` because it tests `McpServerConfig.__post_init__` validation, not routing logic.
5. `TestValidateRoutingDrift` belongs in `test_tool_registry.py` because it tests `validate_routing_against_config` from `shared.tool_registry`.
6. `TestRegistryRouting`, `TestConfigDrivenRouting`, `TestRegistryWithoutConfig` are correctly placed in `test_route_resolver.py` and stay.
