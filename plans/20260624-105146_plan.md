## Design

### MCP Failure Diagnosis Flow (to add in `04_mcp_06`)

```
1. Check transport: was the request delivered?
   → Transport failure: see §Transport Errors
   → Delivered: continue

2. Check tool response: did the tool return an error?
   → Tool error: see §Tool-Level Errors
   → Success: continue

3. Check health state: did the server health change?
   → Health degraded/failed: see §Health States and Watchdog
   → No change: continue

4. Check repeated-failure threshold: watchdog action taken?
   → Watchdog restart: see §Watchdog Configuration
   → No restart: check serialization (see §Serialization Behavior)
```

**Restart-worthy:** health transition to failed + repeated tool errors
**Not restart-worthy:** single tool error, timeout within threshold, serialization delay
