## Design

### Task 1: args masking

`tool_runner.py` の `_collect_tool_result_msgs()` (L97) で既に `masked = mask_args(args, ...)` が計算済み。
L112 を `orjson.dumps(masked).decode()` に変更するだけ。

スキーマ変更:
```sql
-- _SESSION_MIGRATE_SQL に追加
ALTER TABLE tool_results RENAME COLUMN args_json TO args_masked;
```

カラム名変更に伴うコード更新:
- `db/tool_results.py`: `args_json` 参照 → `args_masked`
- `cmd_tooling.py:50`: `result.get("args_json")` → `result.get("args_masked")`
- テスト: `_SCHEMA_SQL` のカラム名、`result["args_json"]` → `result["args_masked"]`

### Task 2: latency + /rag search

`rag/pipeline.py` の `run()` — 各ステージ呼び出しを `time.perf_counter()` で挟み、`self.last_timings[stage.__class__.__name__] = elapsed` に記録。

`/rag search <query>`:
- `cmd_ingest.py` の `_IngestMixin` に `_cmd_rag()` を追加 (または新規 `cmd_rag.py`)
- `await self._ctx.rag_pipeline.augment(query)` を呼び出し
- `last_timings` も表示する dry-run/debug 出力を `--debug` フラグ付きで提供
