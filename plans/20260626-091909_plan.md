# Plan: Restore pending approvals from durable workflow state after restart

Source: `requires/20260625_02_require.md`

---

## Goal

REPL 再起動後も workflow の pending approval を解決できるよう、`startup.py` の `_recover_pending_approvals()` と `cmd_workflow.py` の `/approve` / `/reject` が durable StateStore を主に参照し、transient state に依存しないことを保証する。

---

## Scope

**In-Scope**
- `agent/startup.py`: `_recover_pending_approvals()` の実装内容確認・補強
- `agent/commands/cmd_workflow.py`: `ctx.turn.pending_approval_id` への依存削除、StateStore 直参照の確認
- `agent/workflow/state_store.py`: `find_latest_pending_approval()` の動作確認
- `/approve` / `/reject` が再起動後も機能する CLI パスの確認・追加
- pending approval 一覧表示 CLI コマンドの追加
- restart/recovery 動作のドキュメント化

**Out-of-Scope**
- DB スキーマ変更（`workflow_approvals` テーブルはそのまま）
- workflow engine の実行フロー変更

---

## Assumptions

1. `startup.py:_recover_pending_approvals()` は既に存在するが内容が不完全またはコンテキスト未復元の可能性がある。
2. `cmd_workflow.py` の `/approve` / `/reject` は `find_latest_pending_approval()` を呼ぶため StateStore 経由で動作する可能性が高い。
3. `ctx.turn.pending_approval_id` と `ctx.workflow.approval_pending` は transient（再起動でクリア）。
4. StateStore は再起動後もデータを保持する（SQLite 永続）。
