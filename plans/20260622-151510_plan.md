# Plan: start_http_subprocess タイムアウト境界条件テスト追加 — test_lifecycle.py

## Goal

`tests/test_lifecycle.py` の `TestStartHttpSubprocess` に `startup_timeout_sec` の境界条件テストを追加し、
ゼロタイムアウト・リトライ成功・時刻制御でのタイムアウト発火の各ケースを検証する。

## Scope

**In-Scope:**
- `tests/test_lifecycle.py` — `TestStartHttpSubprocess` への 3 テスト追加
- `agent.http_lifecycle.time.monotonic` の patch による時刻制御

**Out-of-Scope:**
- 実装コードの変更
- `asyncio.sleep` の実際の待機時間テスト

## Assumptions

1. `start_http_subprocess` の while ループ:
   ```python
   deadline = time.monotonic() + cfg.startup_timeout_sec
   while time.monotonic() < deadline:
       ...
       await asyncio.sleep(0.5)
   ```
2. `timeout=0` → `deadline = time.monotonic() + 0` → while 条件が即 False → 0 イテレーション → `client.get` 未呼び出し → `HttpStartupError` 送出。
   既存テスト `test_raises_on_timeout` は `timeout=0` だが `client.get.assert_not_called()` を検証していない。
3. `time.monotonic` を patch して時刻を制御できる: `patch("agent.http_lifecycle.time.monotonic")`
4. `time.monotonic` は deadline 計算 (1 回) + while 条件チェック (N+1 回) の計 N+2 回呼ばれる。
   side_effect のリストでイテレーションを制御可能。

## Design

### 追加テスト一覧

**Test 1: `test_zero_timeout_skips_health_polls`**
- `timeout=0`, mock proc の poll は None (running)
- `client.get` が呼ばれないことを検証 (while が 0 イテレーション)
- `HttpStartupError` 送出を検証

```python
@pytest.mark.asyncio
async def test_zero_timeout_skips_health_polls(self) -> None:
    cfg = _http_subprocess_cfg(url="http://127.0.0.1:9999", cmd=["python", "s.py"], timeout=0)
    ex = _mock_tool_executor()
    mgr = _ServerLifecycleRouter({"s": cfg}, ex, {})

    mock_proc = MagicMock()
    mock_proc.poll.return_value = None
    mock_proc.stderr = None

    with (
        patch("agent.http_lifecycle.subprocess.Popen", return_value=mock_proc),
        patch("agent.http_lifecycle.httpx.AsyncClient") as MockClient,
        pytest.raises(HttpStartupError),
    ):
        client_instance = AsyncMock()
        MockClient.return_value.__aenter__ = AsyncMock(return_value=client_instance)
        MockClient.return_value.__aexit__ = AsyncMock(return_value=False)
        await mgr.start_http_subprocess("s", cfg)

    client_instance.get.assert_not_called()
```

**Test 2: `test_health_poll_retries_before_success`**
- `timeout=5`, health 1 回失敗 → 2 回目に status_code=200 で成功
- `client.get` が 2 回呼ばれたことを検証 (retry あり)
- 例外が送出されないことを検証

```python
@pytest.mark.asyncio
async def test_health_poll_retries_before_success(self) -> None:
    cfg = _http_subprocess_cfg(url="http://127.0.0.1:9999", cmd=["python", "s.py"], timeout=5)
    ex = _mock_tool_executor()
    mgr = _ServerLifecycleRouter({"s": cfg}, ex, {})

    mock_proc = MagicMock()
    mock_proc.poll.return_value = None
    mock_proc.stderr = None

    fail_resp = MagicMock()
    fail_resp.status_code = 503
    ok_resp = MagicMock()
    ok_resp.status_code = 200

    with (
        patch("agent.http_lifecycle.subprocess.Popen", return_value=mock_proc),
        patch("agent.http_lifecycle.asyncio.sleep", new=AsyncMock()),
        patch("agent.http_lifecycle.httpx.AsyncClient") as MockClient,
    ):
        client_instance = AsyncMock()
        client_instance.get = AsyncMock(side_effect=[fail_resp, ok_resp])
        MockClient.return_value.__aenter__ = AsyncMock(return_value=client_instance)
        MockClient.return_value.__aexit__ = AsyncMock(return_value=False)
        await mgr.start_http_subprocess("s", cfg)

    assert client_instance.get.call_count == 2
```

**Test 3: `test_timeout_boundary_fires_after_controlled_time`**
- `timeout=1`, `time.monotonic` を patch
- side_effect = [T, T] (deadline 計算) → T+0.1 (1 回目 while: < T+1 → 進む) → T+1.1 (2 回目 while: ≥ T+1 → exit)
- health 1 回失敗 → timeout → `HttpStartupError` 送出

```python
@pytest.mark.asyncio
async def test_timeout_boundary_fires_after_controlled_time(self) -> None:
    cfg = _http_subprocess_cfg(url="http://127.0.0.1:9999", cmd=["python", "s.py"], timeout=1)
    ex = _mock_tool_executor()
    mgr = _ServerLifecycleRouter({"s": cfg}, ex, {})

    mock_proc = MagicMock()
    mock_proc.poll.return_value = None
    mock_proc.stderr = None

    T = 1000.0
    # time.monotonic() calls: deadline=T+1 (call 1), while T+0.1<T+1 enter (call 2),
    # asyncio.sleep, then while T+1.1>=T+1 exit (call 3)
    monotonic_values = [T, T + 0.1, T + 1.1]

    with (
        patch("agent.http_lifecycle.subprocess.Popen", return_value=mock_proc),
        patch("agent.http_lifecycle.asyncio.sleep", new=AsyncMock()),
        patch("agent.http_lifecycle.time.monotonic", side_effect=monotonic_values),
        patch("agent.http_lifecycle.httpx.AsyncClient") as MockClient,
        pytest.raises(HttpStartupError),
    ):
        client_instance = AsyncMock()
        client_instance.get = AsyncMock(side_effect=Exception("connection refused"))
        MockClient.return_value.__aenter__ = AsyncMock(return_value=client_instance)
        MockClient.return_value.__aexit__ = AsyncMock(return_value=False)
        await mgr.start_http_subprocess("s", cfg)

    assert client_instance.get.call_count == 1  # 1 iteration before timeout
```

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `time.monotonic` の呼び出し回数が予想と異なり side_effect が StopIteration する | medium | `monotonic_values` を十分に長くするか、`itertools.cycle` + カウンタで制御する |
| `asyncio.sleep` のパスが `agent.http_lifecycle.asyncio.sleep` と異なる | low | `import asyncio; await asyncio.sleep(...)` のため `agent.http_lifecycle.asyncio.sleep` が正しい |

| `fail_resp.status_code = 503` → health check が 200 以外を受け取った際の挙動が実装と一致するか | low | 実装は `if resp.status_code == HTTPStatus.OK` のみチェック → 503 は continue して sleep に進む |
