# Plan: Roll back SQLite transactions on all normal exceptions

## Goal
`SQLiteHelper.begin_immediate()` と `begin_exclusive()` が `sqlite3.Error` 以外の通常の Python 例外でもロールバックすること、およびそれを検証するテストが存在することを確認・記録する。

## Scope
**In-Scope**:
- `scripts/db/helper.py` の `begin_immediate()` / `begin_exclusive()` のトランザクション例外処理

**Out-of-Scope**:
- `db/store_impl.py` やその他の DB 層クラスへの変更
- `BaseException` (KeyboardInterrupt, SystemExit) のキャッチ

## Assumptions

現在の実装の確認事項:

- `begin_immediate()`: `except Exception:` でロールバック、`except sqlite3.OperationalError:` でロールバック失敗を抑制、`raise` で元例外を再 raise
- `begin_exclusive()`: 同上
- `BaseException` はキャッチしない (KeyboardInterrupt/SystemExit が伝播する)
- ロールバック失敗時は `sqlite3.OperationalError` のみ抑制
