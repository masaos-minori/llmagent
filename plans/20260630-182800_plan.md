# Plan: Add registry-level tests for removed `/ingest` command

## Goal
`dispatch()`レベルで`/ingest`がFalseを返すことと、`/rag`/`/export`/`/compact`が引き続き正常にdispatchされることをテストで保証する。

## Scope
**In-Scope**:
- `tests/test_cmd_registry_ingest_removal.py` への dispatch レベルテスト追加
  - `/ingest https://example.com` → `dispatch()` が `False` を返す
  - `/ingest /tmp/file.md` → `dispatch()` が `False` を返す
  - `/ingest https://example.com --snippets-only` → `dispatch()` が `False` を返す
  - `/rag search query` → `dispatch()` が `True` を返す
  - `/export` → `dispatch()` が `True` を返す
  - `/compact` → `dispatch()` が `True` を返す
- テストは `_cmd_ingest()` に依存しない
- テストは `CommandRegistry` の `dispatch()` を直接呼び出す

**Out-of-Scope**:
- `registry.py` や `command_defs.py` 本体の変更
- `/ingest` ハンドラー実装の変更
- plugin_registry 経由の動作検証

## Assumptions
- `CommandRegistry.__init__` は `hasattr(self, _cmd.handler)` で全ハンドラーの存在を検証する。`/ingest` の dispatch テストではハンドラーが呼ばれないため、`AgentContext` スタブは `__init__` 通過に必要な最小限で足りる
- `dispatch()` が `False` を返す = `_COMMANDS` にもプラグインにも一致しない = unknown-command 動作
- `plugin_registry.iter_commands()` はテスト環境で空を返す（スタブ不要またはmockで対応）
- `test_cmd_registry_ingest_removal.py` の既存クラス `TestIngestCommandRemoved` に dispatch テストを追加するか、別クラスとして同ファイルに追加する
