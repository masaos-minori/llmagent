# Plan: Add integration tests for FTS5 fallback when normalized_content is NULL

## Goal

Strengthen integration test coverage for FTS5 fallback behavior when `normalized_content` is NULL (English and code chunks). Verify that the COALESCE trigger correctly falls back to `content` and that search results are accurate.

## Scope

**In scope:**
- Add end-to-end integration tests for English chunk FTS fallback
- Add integration tests for code chunk FTS fallback with edge cases
- Verify NULL vs empty string behavior in triggers
- Test mixed Japanese/English chunks in the same document
- Test BM25 score ordering for English queries

**Out of scope:**
- Modifying the FTS5 schema or trigger logic
- Adding new search algorithms or tokenization strategies
- Testing the full ingestion pipeline (covered by separate requirements)

## Assumptions

1. Existing tests in `test_rag_repository.py` and `test_fts_japanese.py` provide a good foundation.
2. The COALESCE trigger logic is correct; tests should verify behavior, not fix bugs.
3. Tests should use in-memory SQLite with real FTS5 (not mocks).

## Implementation Steps

1. **Create new test file `tests/test_fts_fallback.py`**
   - Use in-memory SQLite with the same schema as production
   - Set up triggers via `schema_sql.py` DDL
   - Test class: `TestEnglishFtsFallback`
   - Test class: `TestCodeFtsFallback`
   - Test class: `TestNormalizedContentEdgeCases`

2. **Add English chunk FTS fallback tests**
   - `test_english_chunk_indexed_on_content_when_normalized_null`: Insert chunk with NULL normalized_content, search for keyword, verify match
   - `test_english_multi_word_query`: Verify multi-word English queries (e.g., "machine learning") correctly match via FTS5
   - `test_english_bm25_score_ordering`: Verify more relevant matches have better BM25 scores

3. **Add code chunk FTS fallback tests**
   - `test_code_chunk_with_symbols`: Insert Python code with symbols (`->`, `===`, backticks), verify alphanumeric tokens are extracted correctly
   - `test_code_search_returns_original_content`: Verify returned content is original text (not normalized)
   - `test_code_chunk_indexed_on_content_when_normalized_null`: Insert code chunk with NULL normalized_content, search for keyword, verify match

4. **Add edge case tests**
   - `test_empty_string_vs_null_normalized`: Verify behavior when `normalized_content=""` vs `NULL` (empty string returns empty in FTS due to COALESCE semantics)
   - `test_mixed_japanese_english_document`: Insert both JA and EN chunks in same document, verify each is indexed correctly

5. **Add full pipeline integration test**
   - `test_end_to_end_english_ingestion_search`: Write chunk JSON with NULL normalized_content → insert via DB → search via `fts_search()` → verify results
