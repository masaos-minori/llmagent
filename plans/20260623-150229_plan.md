# Plan: Workflow Pending Approval Interrupt Guard and State Recovery

**Requirement:** `requires/20260623_01_require.md`

---

## Goal

Two gaps in workflow pending-approval handling:
1. **Interrupt Guard:** `orchestrator.handle_turn()` does not block regular user prompts when `ctx.workflow.approval_pending=True`, allowing accidental new workflow tasks to be created mid-approval
2. **State Recovery:** REPL restart loses the in-memory `approval_pending` flag; stale `pending` rows remain in `workflow.sqlite` with no re-hydration

---

## Current State

**Gap 1 (No guard):**
`orchestrator.handle_turn()` starts immediately without checking `ctx.workflow.approval_pending`. A user typing a regular prompt while approval is pending creates a new workflow task.

**Gap 2 (No recovery):**
`startup.py._check_services()` runs RAG consistency check, tool validation, etc. — but no scan of `workflow.sqlite` for tasks with `status='pending_approval'`. `StateStore` has `get_pending_approval(task_id)` but no method to scan by session.

**Flow:**
```
REPL._dispatch_line()
  ├── line.startswith("/")  → _cmds.dispatch()   (handles /approve, /reject)
  └── else                  → orchestrator.handle_turn()  ← gap: no approval_pending guard
```

---

## Scope

**In:**
- `scripts/agent/orchestrator.py` — add `approval_pending` guard at the top of `handle_turn()`
- `scripts/agent/workflow/state_store.py` — add `list_pending_approvals_for_session(session_id) -> list[tuple[str, ApprovalRecord]]` (task_id + approval)
- `scripts/agent/startup.py` — add `_recover_pending_approvals()` called from `run()`; set `ctx.workflow.approval_pending` and `ctx.turn.pending_approval_id` if found; emit terminal warning

**Out:**
- Blocking slash commands other than `/approve`/`/reject` (those are informational; only `handle_turn()` LLM path needs blocking)
- Persisting `approval_pending` flag to disk beyond what's already in `workflow.sqlite`
- Timeout-based auto-reject of stale approvals
- Using `time.sleep()` (explicitly excluded by requirement)

---

## Assumptions

1. The guard in `handle_turn()` covers the requirement: regular user prompts are the only path that creates new workflow tasks
2. `workflow_mode != "disabled"` is the only context where `approval_pending` can be True — the guard should be skipped if `_workflow_mode == "disabled"`
3. `StateStore.__init__` opens `workflow.sqlite` with `write_mode=True` — for the read-only scan during startup, we can open read-only or use the same constructor
4. Session ID at startup: `ctx.session.session_id` is available after `build_agent_context(ctx, self._view)` in `_initialize()`
5. `view.write_warning()` is available in `startup.py` as `self._view.write_warning()`
6. `cmd_workflow.py._cmd_approve()` already resets `ctx.turn.pending_approval_id = None` (line 43) — we need to also reset `ctx.workflow.approval_pending = False` there (already tracked as gap in req 81 plan)

## Design

### Change 1: `orchestrator.py` — guard at `handle_turn()` top

Add immediately after `ctx = self._ctx`:
```python
async def handle_turn(self, line: str) -> None:
    """Call LLM with the user message and persist to DB."""
    ctx = self._ctx
    # Guard: block LLM processing while a workflow approval is pending
    if ctx.workflow.approval_pending:
        logger.warning(
            "Turn blocked: workflow pending approval. Use /approve or /reject."
        )
        self._on_error(
            RuntimeError(
                "[workflow] Approval is pending — use /approve [reason] or /reject [reason]."
            )
        )
        return
    turn_started_at = time.perf_counter()
    ...
```

Note: `self._on_error` is `self._view.write_llm_error` — it writes a terminal-visible error message. This is the correct surface for user-facing errors in the orchestrator.

### Change 2: `state_store.py` — `find_pending_approval_by_session()`

Add a new read-only query method:
```python
def find_pending_approval_by_session(
    self, session_id: str
) -> tuple[str, ApprovalRecord] | None:
    """Return (task_id, approval) for the most recent pending-approval task in this session, or None."""
    rows = self._db.fetchall(
        """
        SELECT t.task_id, a.approval_id, a.stage_id, a.reason, a.created_at, a.resolved_at
        FROM tasks t
        JOIN approvals a ON t.task_id = a.task_id
        WHERE t.session_id = ?
          AND t.status = 'pending_approval'
          AND a.status = 'pending'
        ORDER BY a.created_at DESC
        LIMIT 1
        """,
        (session_id,),
    )
    if not rows:
        return None
    r = rows[0]
    return r["task_id"], ApprovalRecord(
        approval_id=r["approval_id"],
        task_id=r["task_id"],
        stage_id=r["stage_id"],
        status="pending",
        reason=r["reason"],
        created_at=r["created_at"],
        resolved_at=r["resolved_at"],
    )
```

### Change 3: `startup.py` — `_recover_pending_approvals()`

Add to `run()` after `_check_services()`:
```python
async def run(self) -> tuple[CommandRegistry, Orchestrator]:
    self._initialize()
    await self._start_servers()
    await self._check_services()
    await self._recover_pending_approvals()  # ← new
    await self._setup_prompt()
    ...
```

Add the method:
```python
async def _recover_pending_approvals(self) -> None:
    """Re-hydrate approval_pending state from workflow.sqlite if a pending approval exists."""
    ctx = self._ctx
    if self._orchestrator is None or self._orchestrator._workflow_mode == "disabled":
        return
    session_id = str(ctx.session.session_id) if ctx.session.session_id else None
    if not session_id:
        return
    try:
        store = StateStore()
        result = store.find_pending_approval_by_session(session_id)
        store.close()
        if result is not None:
            task_id, approval = result
            ctx.workflow.approval_pending = True
            ctx.turn.pending_approval_id = approval.approval_id
            self._view.write_warning(
                f"[workflow] Pending approval restored: task={task_id},"
                f" approval={approval.approval_id}."
                " Use /approve [reason] or /reject [reason] to continue."
            )
            logger.info(
                "Pending approval restored at startup: task=%s, approval=%s",
                task_id,
                approval.approval_id,
            )
    except Exception as e:  # noqa: BLE001 — non-critical; must not crash startup
        logger.debug("Pending approval recovery skipped: %s", e)
```

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `on_error` in orchestrator is not the right channel for "blocked turn" — it may show an error-styled message | Low | The rejection message is an error condition: the user's turn is blocked. `write_llm_error` is appropriate. Alternatively, use `self._view.write_warning()` directly — but it's not in scope from `orchestrator.py` (no `_view` reference). | 
| `StateStore` opened write mode during `_recover_pending_approvals()` — write mode needed even for read? | Low | `SQLiteHelper("workflow").open(write_mode=False)` can be used for read-only. Restructure `find_pending_approval_by_session()` to accept a read-only `db` parameter. |
| Session ID is `None` at startup before session is loaded | Low | Guard with `if not session_id: return` |
| `workflow.sqlite` absent (workflow not configured) | Low | `except Exception` handler in `_recover_pending_approvals()` swallows the exception — logs at DEBUG |
