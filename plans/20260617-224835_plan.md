# Plan: Remove module-level global state from token_counter (req 69)

## Goal

Remove the `_warned_unavailable` module-level global variable from `shared/token_counter.py`, moving warning suppression to caller-controlled state so tests no longer depend on global state.

## Scope

**In scope:**
- `shared/token_counter.py` ‚Äî remove `_warned_unavailable` global; add optional `warn_once` parameter to `get_token_count()`
- `agent/history.py` ‚Äî create own `_WarnOnce` instance as attribute; pass it to `get_token_count()`
- `tests/test_token_counter.py` ‚Äî remove `_reset_warned()` helper and all `.reset()` calls; update warn-once test to not depend on global state
- `docs/06_shared_90_inconsistencies_and_known_issues.md` ‚Äî update GLOBAL-01 entry to RESOLVED

**Out of scope:**
- No changes to `_estimate_tokens()`, `_estimate_chars()`, `_serialise_for_tokenize()` ‚Äî these are pure functions with no state
- No changes to constants `_RATIO_TEXT`, `_RATIO_TOOL_CALL`, `_RATIO_SYSTEM` ‚Äî immutable module-level constants are not a concern
- No changes to other layers (rag/, mcp/) ‚Äî they don't directly call `get_token_count()`

## Assumptions

1. The `_WarnOnce` class (lines 35-49) is kept as-is and reused by the caller; no changes needed to its logic.
2. Only one production caller exists: `agent/history.py:184` (`HistoryManager.count_tokens()`).
3. Tests currently call `tc_module._warned_unavailable.reset()` before ~8 test functions to ensure isolation between test runs.
4. The warn-once behavior should be preserved in production (one warning per session lifetime) but eliminated as a test concern.

### Proposed design

**`token_counter.py` ‚Äî `get_token_count()` signature:**

```python
async def get_token_count(
    history: list[LLMMessage],
    tokenize_url: str,
    http: httpx.AsyncClient,
    timeout: float = 3.0,
    warn_once: _WarnOnce | None = None,
) -> tuple[int, bool]:
```

**Warning logic:**

```python
except (...) as exc:
    if warn_once is not None:
        warn_once.log(
            "token_counter: /tokenize unavailable (%s), using category-aware estimate",
            exc,
        )
    else:
        logger.warning(
            "token_counter: /tokenize unavailable (%s), using category-aware estimate",
            exc,
        )

# On success path (line 170-172):
if n_tokens > 0:
    if warn_once is not None:
        warn_once.reset()
    return n_tokens, True
```

**`agent/history.py` ‚Äî `HistoryManager.__init__`:**

Add after existing instance attributes:


Update line 184:

**`tests/test_token_counter.py` ‚Äî changes:**

```python
async def test_warn_only_once_on_repeated_failure(
    caplog: pytest.LogCaptureFixture,

    http.post.side_effect = httpx.ConnectError("refused")
```
