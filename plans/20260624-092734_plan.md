# Plan: Add Periodic Offset Checkpointing for Durability

**Requirement:** `requires/20260623_25_require.md`
**Task size:** Path B (new checkpoint mechanism in subscribe loop + doc)

---

## Goal

Reduce offset-loss risk on abnormal termination by adding threshold-based `write_offset()` calls inside the `/subscribe` poll loop.

---

## Scope

**In:**
- `scripts/eventbus/app.py` — add checkpoint: call `write_offset()` every N events delivered (e.g., every 10 events or every 30s), not just on disconnect
- `scripts/eventbus/offsets.py` — verify `write_offset()` is idempotent (safe to call frequently)
- `tests/test_eventbus_offsets.py` — add test: verify offset is written after N events even without disconnect
- Event Bus delivery-semantics docs — document new durability level

**Out:**
- Replacing file-based offsets
- Changing `consumer_id` sanitization behavior

---

## Assumptions

1. `write_offset()` is already imported in subscribe path (confirmed: `app.py:173-176`).
2. Per-event checkpoint is too expensive (file I/O per event); every 10 events or 30s interval is a reasonable balance.
3. Threshold of 10 events is configurable via `EventBusConfig`.
