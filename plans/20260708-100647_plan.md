# Plan: H-1 Remove Tool Result Summarization from tool execution path

## Goal

`agent/tool_runner.py` からツール結果要約機能を完全に削除する。
長いツール出力は従来通り `tool_result_max_llm_chars` でトランケートするのみとする。
`rag.llm_client.summarize_tool_result` の import を除去し、
`use_tool_summarize` / `tool_summarize_threshold` への依存をこのパスから排除する。

## Scope

### 変更対象ファイル (主)
- `scripts/agent/tool_runner.py` — import 削除、分岐削除、ロギング削除
- `scripts/agent/tool_result_formatter.py` — `is_summarized()` 関数の扱い変更
- `tests/test_tool_runner.py` — 関連テストの追加・更新

### 変更対象ファイル (副)
- `scripts/agent/config_dataclasses.py` — `use_tool_summarize`, `tool_summarize_threshold` フィールド (要件外だが潜在的クリーンアップ対象; 今回は**変更しない**)
- `scripts/agent/config_builders.py` — 同上 (今回は変更しない)
- `scripts/agent/services/config_reload.py` — 同上 (今回は変更しない)
- `tests/test_tool_result_formatter.py` — `is_summarized()` テストの更新

### スコープ外
- `rag/llm_client.py` 内の `summarize_tool_result` 実装本体の削除 (別タスク)
- `cmd_config_display.py` の表示フィールド削除 (別タスク)
- config フィールド自体の削除 (後続タスク)

## Assumptions

- `scripts/` 以下が本番デプロイ対象のソースツリーである (`agent/` は `scripts/agent/` を指す)
- `rag/llm_client.py` の `summarize_tool_result` は他の呼び出し元がないため、今回の変更後は dead code となるが、削除は本タスクの範囲外
- `_collect_tool_result_msgs()` 内の `is_summarized()` 呼び出しは `summary` カラムへの永続化判定にのみ使われる。要約が常に行われなくなるため、`summary=None` を固定渡しにすることで `is_summarized()` 呼び出し自体が不要になる
- テストの `_cfg()` に残る `use_tool_summarize`/`tool_summarize_threshold` キーは、config_builders が要求するため削除しない

## Unknowns

### U-1: `is_summarized()` は `_collect_tool_result_msgs()` からのみ呼ばれているか
**分析結果 (解決済み):**
`grep` で確認した結果、`is_summarized` の呼び出し箇所は
`scripts/agent/tool_runner.py:195` の1箇所のみ。
`tool_result_formatter.py` の `is_summarized` import は `tool_runner.py` からのみ参照される。
要約を廃止すれば `summary=None` 固定渡しが可能となり、`is_summarized()` の呼び出しを削除できる。
ただし `is_summarized()` 関数定義と import は `tool_result_formatter.py` に残ったままでよい (他テストが参照)。
tool_runner.py 側の import から `is_summarized` を除去するのみ。

### U-2: `ctx.services_required.http` は summarize ブランチ以外で使われているか
**分析結果 (解決済み):**
`tool_runner.py` 内で `ctx.services_required.http` を参照しているのは summarize ブランチ (行153-166) のみ。
削除後は `tool_runner.py` 内での http 参照がなくなる。他モジュールへの影響なし。

## Design

### execute_one_tool_call() の変更後ロジック

```
# Before (lines 152-172):
if (
    ctx.cfg.tool.use_tool_summarize
    and not is_error
    and len(text) > ctx.cfg.tool.tool_summarize_threshold
    and ctx.services_required.http is not None
):
    llm_text = await summarize_tool_result(...)
    logger.info("Tool result %s summarized: ...", ...)
else:
    llm_text = (
        text[:tool_result_max_llm_chars] + "\n... (truncated)"
        if len(text) > tool_result_max_llm_chars
        else text
    )

# After:
llm_text = (
    text[:ctx.cfg.tool.tool_result_max_llm_chars] + "\n... (truncated)"
    if len(text) > ctx.cfg.tool.tool_result_max_llm_chars
    else text
)
```

### _collect_tool_result_msgs() の変更後ロジック

`is_summarized()` 呼び出しを削除し `summary=None` を直接渡す:

```python
# Before:
summarized = is_summarized(ctx.cfg, text, llm_text, is_error)
result_id = ctx.tool_result_store.store(
    ...
    summary=llm_text if summarized else None,
    ...
)

# After:
result_id = ctx.tool_result_store.store(
    ...
    summary=None,
    ...
)
```

### import の変更

```python
# Before:
from rag.llm_client import summarize_tool_result

    TURN_LIMIT_HINT,
    is_summarized,
    mask_args,

# After:

    TURN_LIMIT_HINT,
    mask_args,

```

## Risks

### R-1: `_collect_tool_result_msgs` の `summary=None` 固定化がDBスキーマと不整合になるか
**分析:** `ctx.tool_result_store.store()` の `summary` 引数は `Optional[str]` 型。
`None` を渡すのは既存の非要約パスと同じ動作 (従来も `use_tool_summarize=False` 時は `None` が渡されていた)。
**軽減策:** 変更は `summary=llm_text if summarized else None` → `summary=None` に絞る。DBスキーマ変更なし。リスクは低い。

### R-2: 他テストが `use_tool_summarize=True` を設定して summarize パスを検証している場合、テストが失敗する
**分析:** `grep` の結果、`test_agent_repl_tool_exec.py:336` に `ctx.cfg.tool.use_tool_summarize = False` の設定はあるが、`True` にして summarize パスを通すテストは `test_tool_result_formatter.py` の `is_summarized()` 単体テストのみ。
`is_summarized()` 定義は今回削除しないため、これらのテストは引き続きパスする。
`tool_runner.py` 側で summarize パスを通すテストは存在しないことを確認済み。**リスクなし。**

### R-3: `test_tool_runner.py` の `_cfg()` に `use_tool_summarize`/`tool_summarize_threshold` キーが残ることによるノイズ
**分析:** これらのキーは `config_builders.py` が要求するため `_cfg()` では必要。フィールド自体は `config_dataclasses.py` に残る。
`tool_runner.py` がこれらのフィールドを参照しなくなるだけであり、設定が残ることによる実害はない。
**軽減策:** 今回は削除しない。フィールド削除は後続タスクで実施。
