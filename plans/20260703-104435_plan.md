## Implementation Steps

### Step 1 — Rewrite the `Mixins:` block in the module docstring

**What to do:**

Replace lines 9-19 of the module docstring with a consistently aligned table. Use the following column layout (confirmed from actual file + class imports):

```
Mixins:
  cmd_session.py    — _SessionMixin:    /session commands
  cmd_mcp.py        — _McpMixin:        /mcp commands
  cmd_config.py     — _ConfigMixin:     /config, /stats, /set, /reload
  cmd_context.py    — _ContextMixin:    /context, /clear, /undo, /history, /system
  cmd_db.py         — _DbMixin:         /db
  cmd_tooling.py    — _ToolingMixin:    /tool, /plan
  cmd_debug.py      — _DebugMixin:      /debug
  cmd_audit.py      — _AuditMixin:      /audit
  cmd_rag_export.py — _RagExportMixin:  /export, /compact, /rag
  cmd_memory.py     — _MemoryMixin:     /memory
  cmd_workflow.py   — _WorkflowMixin:   /approve, /reject
  cmd_plugins.py    — _PluginsMixin:    /plugin
  cmd_mdq.py        — _MdqMixin:        /mdq commands
```

Column layout rationale:
- Column 1 (filename): left-aligned, pad to width of longest entry `cmd_rag_export.py` (17 chars) + 1 space before `—`
- Column 2 (class name): left-aligned, pad to width of longest entry `_RagExportMixin:` (16 chars) + 1 space before commands
- Order: follows the MRO order in `CommandRegistry` class definition
