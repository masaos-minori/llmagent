## Design

### Phase A — 横断的品質改善（影響小・自動化可能）

**A1 — logging f-string 変換（要望18）**:
- ruff の `G004` ルールで **174 エラー** を検出済み
- `--fix` / `--unsafe-fixes` では自動修正不可（ruff G004 は修正候補を提示しない）
- 変換方法: `sed -i` スクリプトまたは手動変換
  - 例: `logger.info(f"foo {x}")` → `logger.info("foo %s", x)`
  - 単純な 1 変数の場合は sed で自動化可能。複数変数・複雑な式は手動確認が必要

**A2 — logging 戦略整理（要望0）**:
- `shared/logger.py` は既に `_JsonFormatter`（JSON-lines）+ `_ContextFilter`（kvlog フィールド注入）を実装済み
- 残作業: `_ContextFilter._fields` が **スレッド共有 dict** のため非同期/並行実行で値が混在する。`contextvars.ContextVar` を使った task-local 付与への移行が必要
- 対象: `shared/logger.py` の `_ContextFilter` クラスのみ

**A3 — テストカバレッジ追加（要望16）**:
- `cmd_session.py`, `cmd_memory.py`, `cmd_notes.py`, `cmd_db.py`, `cmd_context.py` の
  characterization tests を `tests/test_cmd_*.py` に追加
- `mcp/dispatch.py`, `tool_scheduler.py`, `shared/formatters.py` の未テスト経路を補強
- 目標: 各ファイル ≥ 80%

### Phase B — 設定・CLI 層改善（中規模）

**B1 — config_reload.py 差分適用（要望5）**:
- `apply_config_dict()` を唯一の公開入口にする
- domain 別 handler (llm/mcp/approval/tool_rag/prompt) に分割
- before/after 差分レポートを `ConfigReloadOutcome` に追加

**B2 — CLI command 責務縮小（要望2）**:
- `ctx.conv.history` 書き換えは **`cmd_ingest.py:152` の 1 箇所のみ**（force_compress 結果代入）。他は read-only 参照。
- `ctx.stats` は `mixin_base.py` の `reset_session_stats()` が setter（既に helper に集約済み）。`cmd_config.py` は read-only 参照のみ。
- 主な残作業: undo / system prompt / session restore の service 委譲（多くは既に委譲済みの可能性あり → 実装前に `rg "ctx\.\(conv\|session\|stats\)\." agent/commands/` で全件確認が必要）

**B3 — dispatch/引数解析共通化（要望3）**:
- `utils.py` の `parse_command_args` を拡張して全コマンドに適用
- `/session`, `/db`, `/memory`, `/note`, `/tool`, `/mcp` の手書き split を置換
- formatter/presenter/safe JSON decode helper を共通化

### Phase C — コアアーキテクチャ改善（大規模）

**C1 — Orchestrator 責務再分離（要望4）**:
- `orchestrator.py` を turn lifecycle 管理に限定
- prompt assembly, audit, error policy を専用 service へ

**C2 — ライフサイクル整理（要望6）**:
- `repl.py` から bootstrap/health/watchdog を切り出す
- `repl_health.py` を probe/validator/watchdog に分割

**C3 — DB 層責務再分離（要望7）**:
- `helper.py` の設定キャッシュ廃止
- `store.py`/`tool_results.py` の戻り値を typed DTO に統一
- silent fallback → 明示的エラー

### Phase D — Memory 層改善（大規模）

**D1〜D4**: Memory 永続化パイプライン統一、型バリデーション強化、ポリシー設定化
