## Goal
Replace the line-based Markdown parser with a hierarchy-aware parser that correctly handles fenced code blocks, heading hierarchy, and produces rich section metadata.

## Scope
**In-Scope**:
- Rewrite `scripts/mcp/mdq/parser.py` with hierarchy-aware parsing
- Add new fields to ParsedSection TypedDict (heading_level, heading_path, start_line, end_line, ordinal, parent_heading)
- Update ParsedSectionRequest if needed

**Out-of-Scope**:
- Setext heading support (documented as unsupported)
- Adding new tools or features

## Assumptions
1. ATX-style headings (e.g., `# Heading`) are the only heading format supported
2. Fenced code blocks use triple backticks (```) or tildes (~~~)
3. Frontmatter uses YAML delimiters (`---`)

## Implementation Steps
1. **Phase 1: Update ParsedSection TypedDict**
   - Add heading_level, heading_path, start_line, end_line, ordinal, parent_heading fields
2. **Phase 2: Rewrite parser.py**
   - Track fenced code block state to skip `#` inside code blocks
   - Build heading path from ancestor headings
   - Compute ordinal among same-level headings
   - Track line ranges (start_line, end_line)
   - Handle content before first heading as `<root>` section
3. **Phase 3: Update consumers**
   - Verify indexer.py still works with new ParsedSection structure
