# Plan: Strengthen ToolExecutor Error Classification Tests

## Goal

Add regression tests to `tests/test_tool_executor.py` that lock down the boundary between tool-level errors and transport-level errors in `ToolExecutor`, including `error_type` classification, error counters, and `HealthRegistry` recording behavior.

## Scope

**In-Scope**
- Add new test class(es) to `tests/test_tool_executor.py` covering all scenarios listed in the requirement
- All test scenarios exercise `ToolExecutor._raw_execute()` or `ToolExecutor.execute()` (via cache path) to test the full classification pipeline, not just `HttpTransport.call` in isolation
- Tests use the same `_make_executor` / `_http_cfg` pattern established in `tests/test_tool_executor_routing.py`

**Out-of-Scope**
- Changes to `scripts/shared/tool_executor.py` (unless tests reveal implementation defects)
- Changes to `tests/test_tool_executor_routing.py`
- New test files
- Any changes to `McpServerHealthRegistry` behavior itself (already covered in `test_tool_executor_routing.py`)

## Assumptions

1. `tests/test_tool_executor_routing.py` is authoritative for `McpServerHealthRegistry` unit tests; the new tests here exercise the _integration_ between `ToolExecutor._raw_execute` and `HealthRegistry` around `error_type` and counter semantics.
2. The `_make_executor` / `_http_cfg` helpers are duplicated into `test_tool_executor.py` (they are file-private in `test_tool_executor_routing.py`); a shared conftest is not required.
3. `ToolCallResult.error_type` field correctly reflects `"tool"` vs `"transport"` vs `""` based on the current implementation — tests are regression locks, not new feature validation.
4. HTTP 500 from `HttpTransport` raises `TransportError` (non-retryable branch confirmed in `HttpTransport.call`).
5. Cache-hit path returns `ToolCallResult` with `request_id == ""` and does not call `HealthRegistry` — this is the existing contract documented in `_execute_with_cache`.
6. `uv run pytest` is the only required test runner.

### Step 2: Add `TestToolExecutorErrorClassification` class

Add a new test class immediately after `TestHttpTransportRetry`. The class contains the following test methods:

#### 2a. `test_http_200_success_error_type_empty`
- Arrange: `ToolExecutor` with `McpServerHealthRegistry`; `_transports["file_read"]` replaced with `AsyncMock` returning `ToolCallResult(output="ok", is_error=False, request_id="req-1", server_key="file_read", error_type="")`.
- Act: `await ex._raw_execute("read_text_file", {})`.
- Assert: `result.is_error == False`, `result.error_type == ""`, `registry.get_state("file_read") == McpServerHealthState.HEALTHY`.

#### 2b. `test_http_200_tool_error_increments_stat_tool_errors`
- Arrange: transport mock returns `is_error=True, error_type="tool"`.
- Act: `await ex._raw_execute("read_text_file", {})`.
- Assert: `result.is_error == True`, `result.error_type == "tool"`, `ex.stat_tool_errors.get("file_read", 0) == 1`, `registry.get_state("file_read") == McpServerHealthState.HEALTHY` (tool error does not degrade health).

#### 2c. `test_http_500_transport_error_classification`
- Arrange: transport mock raises `TransportError("HTTP 500")`.
- Act: `await ex._raw_execute("read_text_file", {})`.
- Assert: `result.is_error == True`, `result.error_type == "transport"`, `ex.stat_transport_errors.get("file_read", 0) == 1`, `registry.get_state("file_read") == McpServerHealthState.DEGRADED`.

#### 2d. `test_http_503_retry_exhaustion_is_transport_error`
- Arrange: Use real `HttpTransport` with a `_FakeClient` that always returns HTTP 503; inject into `ToolExecutor._transports`; `McpServerHealthRegistry` attached.
- Act: `await ex._raw_execute("read_text_file", {})` with `asyncio.sleep` patched.
- Assert: `result.error_type == "transport"`, `ex.stat_transport_errors.get("file_read", 0) == 1`, `"Retry exhausted" in result.output`.

#### 2e. `test_timeout_is_transport_error`
- Arrange: Use real `HttpTransport` with a `_FakeClient` that raises `httpx.TimeoutException`; inject into `ToolExecutor._transports`; `McpServerHealthRegistry` attached.
- Act: `await ex._raw_execute("read_text_file", {})`.
- Assert: `result.error_type == "transport"`, `ex.stat_transport_errors.get("file_read", 0) == 1`, registry records failure.

#### 2f. `test_malformed_response_is_transport_error`
Parametrize over three malformed cases:
- non-dict JSON (e.g., `[1, 2]`)
- missing `result` key (e.g., `{"is_error": False}`)
- non-bool `is_error` (e.g., `{"result": "x", "is_error": 1}`)

For each: `HttpTransport._parse_http_response` raises `ValueError`, which is caught and re-raised as `TransportError`. Inject via real `HttpTransport` with `_FakeClient` returning HTTP 200 with malformed body. Assert `result.error_type == "transport"` and `stat_transport_errors` incremented.

#### 2g. `test_cache_hit_no_health_registry_update`
- Arrange: Pre-populate `executor._cache` with a valid `CacheEntry`; attach `McpServerHealthRegistry`.
- Act: `await ex._execute_with_cache("read_text_file", {})`.
- Assert: `result.request_id == ""`, `ex.stat_cache_hits == 1`, `registry.get_state(...)` unchanged (no `record_success` / `record_failure` called) — confirm by checking `McpServerHealthState.HEALTHY` with a `MagicMock` registry or by asserting the state stays at its initial value without any mock transport call.

**Completion criteria:** `uv run pytest tests/test_tool_executor.py -v` reports all existing 10 tests plus the 7 new tests passing.
