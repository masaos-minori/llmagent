## Goal
Event Bus の SQLite 共有接続モデルにおいて `asyncio.to_thread()` + `threading.Lock` によるスレッド安全性を確認し、`docs/06_eventbus_03` の誤った説明（「シングルスレッドで安全」）を実装実態に合わせて修正する。

## Scope
**In-Scope**:
- `scripts/eventbus/db.py` の `_db_lock = threading.Lock()` と `check_same_thread=False` モデルの確認
- `scripts/eventbus/app.py` の全 DB 操作が `asyncio.to_thread()` + `with get_db_lock():` で保護されているかの確認
- `docs/06_eventbus_03_persistence_schema_and_replay.md:L7` の誤記修正: 「FastAPI がシングルスレッドで実行するため安全」→「`asyncio.to_thread()` + `threading.Lock` でシリアライズする」
- `docs/06_eventbus_90` の FastAPI thread pool 問題を Resolved に更新（実装確認後）
- `tests/test_eventbus_concurrent.py` の並行テストカバレッジ確認

**Out-of-Scope**:
- SQLite を他の DB に置き換える
- per-request 接続モデルへの移行
- Event Bus のクラスタリング

## Assumptions
1. `db.py:L11`: `_db_lock = threading.Lock()` が実装済み（確認済み）
2. `db.py:L27-28`: "Concurrent access from asyncio.to_thread() is serialized by the module-level _db_lock. WAL mode additionally serializes concurrent writers at the SQLite level." と docstring に記述済み
3. `app.py`: 全 DB 操作が `asyncio.to_thread()` で実行されることを確認済み（L105, L127, L202, L248, L309, L354-355, L380, L427 など）
4. `app.py` の DB 操作が `with get_db_lock():` でラップされていることを確認済み（L102, L124, L192, L245, L253, L295, L347, L351, L370, L414, L453, L493 など）
5. `docs/06_eventbus_03:L7` が「FastAPI runs on a single async event loop thread. All DB operations execute on that thread.」と記述しており、実際の実装（`asyncio.to_thread()` = スレッドプール実行）と乖離している
