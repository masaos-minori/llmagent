# Plan: Remove diagnostics.jsonl from implementation and active documentation

## 1. Goal

Remove the legacy `diagnostics.jsonl` runtime diagnostics path and make `session_diagnostics` the only supported diagnostics persistence mechanism.

## 2. Scope

- **In-Scope**:
  - Remove JSONL write path in `repl.py` (`_persist_session_diagnostics`)
  - Remove or update test for JSONL write in `test_repl.py`
  - Update documentation: `05_agent_04_state-and-persistence.md`, `05_agent_10_operations-and-observability.md`, `05_agent_90_inconsistencies_and_known_issues.md`
  - Add SQL examples for `session_diagnostics` queries

- **Out-of-Scope**:
  - Changes to DiagnosticStore itself (already correct)
  - Changes to session_diagnostics table schema
  - Changes to other diagnostic-related features

## 3. Assumptions

- `diagnostics.jsonl` is no longer needed since `DiagnosticStore.save()` handles persistence to `session_diagnostics` table
- No external tools or monitoring depend on reading `diagnostics.jsonl`
- The JSONL file was only written at session end for post-mortem analysis
