## Goal

Simplify lifecycle protocols and factory wiring after stdio transport removal so that lifecycle management is HTTP-only with no stdio-specific methods.

## Scope

**In-Scope**:
- `scripts/agent/lifecycle_protocol.py` — remove `StdioLifecycleProtocol` class, `restart_stdio()`, and `shutdown_idle()` from `LifecycleManagerProtocol`
- `scripts/agent/factory.py` — remove `StdioServerLifecycleManager` import, `restart_stdio()` method, and `shutdown_idle()` delegation to stdio manager; simplify `_ServerLifecycleRouter`
- `scripts/agent/repl_health.py` — remove `restart_stdio()` call (line 423) and `shutdown_idle()` call (line 472)
- `docs/05_agent_13_reference-api.md` — update lifecycle API reference (line 169: remove restart_stdio note)
- `docs/01_overview-files.md` — update file tree (line 73: remove lifecycle.py restart_stdio note; line 75: remove stdio_lifecycle.py)
- `docs/05_agent_02_runtime-architecture.md` — update architecture description (line 32, 180: remove stdio lifecycle references)
- `docs/04_mcp_03_routing_lifecycle_and_execution.md` — remove `shutdown_idle()` from API reference (line 283) and idle timeout watchdog description (line 433)
- `docs/04_mcp_01_system_overview.md` — update `_ServerLifecycleRouter` description (line 107)

**Out-of-Scope**:
- `agent/http_lifecycle.py` — does not exist; do not create
- Removing `StdioTransport` from tool_executor.py (covered by 20260701-172032)
- Removing `stdio_lifecycle.py` file (covered by 20260701-172032)

**Dependency**: Requires 20260701-172032 (stdio runtime removal) to be completed first.

## Assumptions

1. `shutdown_idle()` is stdio-ondemand-specific; HTTP servers (persistent or subprocess) have no idle timeout. Removing it from the protocol is safe once stdio is gone.
2. `restart_stdio()` in `repl_health.py:423` is inside the stdio watchdog branch (`_watchdog_check_stdio()`), which is also being removed. The call will disappear with the stdio watchdog removal.
3. `shutdown_idle()` in `repl_health.py:472` is inside a watchdog cycle; removing it is safe when there are no ondemand servers.
4. No test files reference `StdioLifecycleProtocol`, `restart_stdio()`, or `StdioServerLifecycleManager` (confirmed by grep).
5. `HttpServerLifecycleManager` does not exist as a separate module; lifecycle is handled by `_ServerLifecycleRouter` in factory.py.
