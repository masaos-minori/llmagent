## Design

### `04_mcp_01` change — add transport selection callout

After the transport type table (near line 67), add:

```markdown
### Transport Selection Guide

> **Production default: always use HTTP (`transport = "http"`, `startup_mode = "external"`).**
> HTTP supports watchdog, health checks, concurrent requests, and remote monitoring.
>
> **Use stdio only for:** local testing, CI pipelines, embedded single-tool subprocesses
> with low concurrency and no external clients.
>
> See [04_mcp_02 §When to use stdio](04_mcp_02_protocol_and_transport.md#when-to-use-stdio) for the full decision guide.
```

### `04_mcp_03` change — promote production note

Change the inline `> **Production note:**` inside the startup_mode table description (line 255) from:
```
> **Production note:** For production, prefer `transport = "http"` with `startup_mode = "external"`. HTTP transports support watchdog, health checks, and remote monitoring. stdio is for local/embedded use only.
```

To a standalone callout block BEFORE the startup_mode table:

```markdown
> **Production transport guidance:**
> For production deployments, use `transport = "http"` with `startup_mode = "external"`.
> HTTP transports support watchdog, health checks, and remote monitoring.
> `stdio` transport serializes all requests through a single `asyncio.Lock` and provides
> no health-check endpoint — use it only for local/embedded single-tool subprocesses.
>
> See [04_mcp_02 §When to use stdio](04_mcp_02_protocol_and_transport.md#when-to-use-stdio).
```
