## Goal

Improve audit and observability for plugin tool execution by ensuring plugin tool executions emit distinguishable audit events, enhancing `/plugin status` output, and documenting differences from MCP execution.

## Scope

- **In-Scope**: Audit event source field standardization; plugin result distinguishability; `/plugin status` enhancement; documentation updates; tests.
- **Out-of-Scope**: Banning plugin tools; removing `plugin_tool_override`; routing plugin tools through MCP.

## Assumptions

1. Plugin tool audit events are currently suppressed because `audit_tool_exec()` in `tool_audit.py:169` returns early when `mcp_request_id` is empty (plugin tools always have `request_id=""`).
2. `ToolExecEvent.model.source` field exists (default `"agent"`) and can carry `"plugin"` or `"mcp"`.
3. `PluginLoadResult` already contains `tool_conflicts_shadowed`, `tool_conflicts_allowed`, `command_shadows_rejected`, `failed` details.
