## Goal
`scripts/agent/commands/cmd_mdq.py` の全 7 コマンドに対するユニットテストを `tests/test_cmd_mdq.py` として追加し、MDQ スラッシュコマンドが MCP ツール呼び出しを通じて正しく動作することを検証する。

## Scope
**In-Scope**:
- `tests/test_cmd_mdq.py` の新規作成
- 各コマンド（status, index, refresh, search, outline, get, grep）のテスト
- MCP tool executor (`ctx.services.tools.execute`) の mock によるユニットテスト
- エラーレスポンス（`is_error=True`）時の出力確認
- MCP ツール不在時のエラー出力確認

**Out-of-Scope**:
- `cmd_mdq.py` の実装変更（すべてのコマンドは実装済み）
- 実際の MDQ MCP サーバーを起動した統合テスト
- コマンドライン引数パーサーの大幅な改修

## Assumptions
1. `cmd_mdq.py` の全 7 コマンドはすでに実装済みかつ CommandRegistry に登録済み
2. 他のコマンドテスト（例: `test_agent_rag.py`）のパターンを参考に mock を構築できる
3. `ctx.services.tools.execute(tool_name, args)` が `DispatchResult(is_error, output)` 相当のオブジェクトを返す
4. 各コマンドが MCP ツール呼び出しのみを行い、`mdq.sqlite` を直接読まないことが確認済み（cmd_mdq.py を精査）
