## Implementation Steps

### Step 1 — Fix `_check_tool_definitions` docstring in `scripts/agent/repl_health.py`

**File:** `scripts/agent/repl_health.py`, lines 155-164 (the "All servers unreachable" scenario block in the docstring).

Replace the Option B description:
```
      - All servers unreachable:
          INFO "All MCP servers unreachable during strict validation: [...]; skipping tool definition check"
          returns HealthCheckResult() — no warnings, no error even in strict mode
```

With Option A description:
```
      - All servers unreachable, strict=True:
          ERROR "Strict mode: all MCP servers unreachable — cannot validate tool definitions. Unreachable servers: [...]."
          raises RuntimeError
      - All servers unreachable, strict=False:
          INFO "All MCP servers unreachable; skipping tool definition check. Unreachable: [...]"
          returns HealthCheckResult() — no warnings
```

**Completion criteria:** Docstring text no longer says "no error even in strict mode"; it distinguishes strict vs non-strict for the all-unreachable case.

### Step 3 — Add missing tests in `tests/test_repl_health.py`

The acceptance criteria requires 5 scenarios. Current coverage:

| Scenario | Covered by existing test? |
|----------|--------------------------|
| All reachable | Yes — `test_no_mismatch_returns_empty` (mock returns `({"read_file","write_file"}, [])`) |
| Partial unreachable | No — no test has `(some_names, ["srv-a"])` |
| All unreachable, strict=False | Partially — `test_returns_empty_when_all_servers_unreachable` uses `(set(), [])` (no unreachable list; hits a different code branch). Need a test with `(set(), ["srv-a"])` |
| All unreachable, strict=True | Yes — `test_raises_in_strict_mode_when_all_unreachable` |
| Mismatch strict=True | Yes — `test_raises_in_strict_mode` |
| Mismatch strict=False | Yes — `test_returns_warning_on_missing_in_server` |

Add to `TestCheckToolDefinitions`:

**Test A: `test_partial_unreachable_non_strict_returns_warning_if_mismatch`**
- `_collect_server_tool_names` returns `({"read_file"}, ["srv-b"])` (one tool name, one unreachable server).
- `tool_definitions = [read_file, write_file]`.
- `strict=False`.
- Assert: result has issues; warning contains "write_file".

**Test B: `test_partial_unreachable_strict_raises`**
- `_collect_server_tool_names` returns `({"read_file"}, ["srv-b"])`.
- `tool_definitions = [read_file, write_file]`.
- `strict=True`.
- Assert: raises `RuntimeError` matching "Strict mode".

**Test C: `test_all_unreachable_non_strict_skips_validation`**
- `_collect_server_tool_names` returns `(set(), ["srv-a"])`.
- `tool_definitions = [read_file]`.
- `strict=False`.
- Assert: result has no issues (validation skipped).

Note: Test C is logically equivalent to what `test_returns_empty_when_all_servers_unreachable` tests, but the existing test uses an empty unreachable list `(set(), [])` which hits a different code branch (line 184-186 in repl_health.py, the `else` branch). Test C exercises the `if unreachable and not strict` branch (lines 182-183). Both should be kept.

**Completion criteria:** `uv run pytest tests/test_repl_health.py -v` passes with the three new tests; no existing tests removed or modified.
