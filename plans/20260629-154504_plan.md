## 1. Goal

Align `mcp/mdq/server.py` with the `MCPServer` base class conventions by adding the missing `server_key` class attribute, attaching auth middleware, and ensuring error responses follow the MCP tool-error contract rather than breaking the transport.

## 2. Scope

- **In-Scope**:
  - Add `server_key = "mdq"` to `MdqMCPServer`
  - Add explicit `http_host = "127.0.0.1"` to `MdqMCPServer`
  - Call `attach_auth_middleware(app, ...)` on the FastAPI app (matching git-mcp, shell-mcp pattern)
  - Load auth token from config (via `MdqConfig` or `ConfigLoader`) and pass to `attach_auth_middleware`
  - Refactor `/v1/tools` to use `list_tools_with_server_key()` (or keep inline but ensure `server_key` uses class attribute)
  - Review the 7 FastAPI exception handlers: decide whether to keep transport-level HTTP errors for infrastructure errors (db unavailable, auth) vs convert dispatch-level errors to `is_error=True` MCP tool responses
  - Add/update tests for `MdqMCPServer` conformance to base class contract
- **Out-of-Scope**:
  - Adding new MDQ tools or changing tool input schemas
  - DB schema changes
  - Hybrid search mode (MDQ-02)
  - Admin tool (`fts_consistency_check`, `fts_rebuild`) status changes

## 3. Assumptions

- The requirement's "7 MDQ tools" refers to the 7 production-status tools in `_MCP_TOOLS`; the 2 admin tools (`fts_consistency_check`, `fts_rebuild`) are intentionally not counted in the primary tool set
- Other servers (git-mcp, shell-mcp) also use FastAPI exception handlers for infrastructure-level errors (auth, not found) and this is acceptable "common MCP behavior"
- The `mdq_mcp_server.toml` config file may or may not have an `auth_token` field; if absent, auth middleware should be a no-op (empty token)
- `attach_auth_middleware` is already idempotent with empty token (auth skipped, only X-Request-Id injected)

## 8. Risks & Mitigations

- **Risk**: Adding `attach_auth_middleware` with non-empty token may break existing integrations that don't send Bearer token → **Mitigation**: Default `auth_token` to empty string; only enable auth if explicitly configured in `mdq_mcp_server.toml`
- **Risk**: Converting FastAPI exception handlers to dispatch-level `is_error=True` may change HTTP status codes that callers depend on → **Mitigation**: Only convert domain-level errors (validation/not found/auth); keep infrastructure errors (DB, index not ready) as HTTP 503; add regression tests before changing
- **Risk**: `list_tools_with_server_key()` using `server_key` attribute vs inline comprehension divergence may cause future inconsistency → **Mitigation**: Always use `list_tools_with_server_key()` in `/v1/tools` endpoint after adding `server_key` class attribute
