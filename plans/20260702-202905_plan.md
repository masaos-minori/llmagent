## Goal

Remove `agent/config.py` re-export stub and update all callers to import directly from `agent.config_builders` or `agent.config_dataclasses`, so that no code imports from `agent.config`.

Source: `requires/20260702_52_require.md`

---

## Scope

**In**:
- Replace all `from agent.config import ...` statements in `scripts/`, `tests/`, `docs/`
- Delete `scripts/agent/config.py`
- Remove the re-export comment in `scripts/agent/config_builders.py` (line 20: `# noqa: F401 — re-exported via agent.config`)
- Update three documentation files that reference `agent/config.py` as the source location

**Out**:
- Changing config dataclass behavior
- Changing config loading behavior
- Keeping deprecated import aliases or deprecation shims
- Modifying packaging metadata or entrypoints (confirmed: no references found in non-venv config/package files)

---

## Assumptions

1. All symbols re-exported by `agent/config.py` come exclusively from `agent.config_builders` or `agent.config_dataclasses`; no other modules are involved.
2. The lazy/deferred imports inside function bodies (in `config_reload.py` and `cmd_config_display.py`) exist to avoid circular imports; redirecting them to the sub-modules preserves the same avoidance.
3. The `TYPE_CHECKING`-gated imports in `tool_policy.py` and `tool_result_formatter.py` have no runtime effect and can be redirected freely.
4. No packaging entrypoints (console_scripts, etc.) reference `agent.config` — confirmed by grep over all non-venv config/package files (no matches).
5. `uv run pytest` and `PYTHONPATH=scripts uv run lint-imports` are the definitive validation gates.
