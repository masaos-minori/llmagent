## 1. Goal

Call `validate_routing_against_live()` during agent startup, emit per-server warnings
when live `/v1/tools` ownership differs from the `ToolRegistry`, and keep the check
warning-only (agent always starts).

## 2. Scope

**In-Scope:**
- Add `check_routing_drift_live(ctx)` async function to `agent/repl_health.py`

## 3. Assumptions

1. `validate_routing_against_live()` in `shared/tool_registry.py` accepts
   `live_tool_lists: dict[str, list[str]] | None` (tool names as strings, not dicts).
   **Verified:** signature at line 217â€“237 of `tool_registry.py` confirms this.
2. HTTP server `/v1/tools` responses are JSON `{"tools": [{"name": "...", ...}]}`.
   **Verified:** `_collect_server_tool_names()` already uses `t["name"]` on that response.
3. `check_routing_drift_live()` should follow the same error-swallowing pattern as
   `check_routing_drift()` (bare `except Exception` returns `[]`), keeping startup non-fatal.
4. `_check_services()` in `startup.py` is the single call site for all startup health functions.
   **Verified:** lines 149â€“172 of `startup.py`.
5. The per-server tool list needed for `validate_routing_against_live()` differs from
   `_collect_server_tool_names()` which returns a merged `set[str]`. A new private helper
   `_collect_server_tool_lists(ctx)` returning `dict[str, list[str]]` is required.

## 6. Implementation Steps

### Phase 1: Add helper and new health check function

- [ ] In `scripts/agent/repl_health.py`, add `_collect_server_tool_lists()`:
  - Signature: `async def _collect_server_tool_lists(ctx: AgentContext) -> dict[str, list[str]]`
  - Iterates `ctx.cfg.mcp.mcp_servers.items()` for `transport == "http"` servers with a non-empty URL
  - GETs `{srv_cfg.url}/v1/tools` with `timeout=5.0`
  - On HTTP 200: adds `server_key â†’ [t["name"] for t in resp.json().get("tools", [])]`
  - On HTTP non-200 or exception: logs `WARNING "{key} unreachable at {url}/v1/tools: ..."` and skips (same pattern as `_collect_server_tool_names`)
  - Returns the collected dict (may be empty if all unreachable)
- [ ] In `scripts/agent/repl_health.py`, add `check_routing_drift_live()`:
  - Signature: `async def check_routing_drift_live(ctx: AgentContext) -> list[str]`
  - Calls `_collect_server_tool_lists(ctx)` to get `live_tool_lists: dict[str, list[str]]`
  - If `live_tool_lists` is empty: log `INFO "Live routing drift check skipped: no reachable HTTP servers"` and return `[]`
  - Calls `validate_routing_against_live(live_tool_lists=live_tool_lists)` from `shared.tool_registry`
  - Iterates drift dict: emits `logger.warning(f"Live routing drift [{server_key}]: {msg}")` for each mismatch
  - Wraps in `try/except Exception` â†’ `logger.warning("Live routing drift check failed: %s", exc)` and return `[]`
  - Returns list of warning strings

### Phase 2: Wire into startup

- [ ] In `scripts/agent/startup.py`:
