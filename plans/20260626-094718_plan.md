# Plan: Add reconcile and repair support for SQLite vs JSONL archive divergence

Source: `requires/20260625_50_require.md`

---

## Goal

EventBus の JSONL アーカイブ書き込み失敗が silent warning で終わる現状を改善し、divergence の検出・修復パスを追加する。

---

## Scope

**In-Scope**
- `scripts/eventbus/app.py` or `db.py`: `_append_jsonl()` の失敗ハンドリング確認・改善
- divergence 検出メトリクス（スキップ数カウンタ）の追加
- admin エンドポイントまたは CLI での divergence 確認パス
- operational docs 更新

**Out-of-Scope**
- JSONL を primary storage に昇格させる変更
- SQLite の変更

---

## Assumptions

1. publish パス: SQLite write → JSONL append の順で、JSONL 失敗時は warning ログのみで success を返す。
2. JSONL は補足アーカイブであり SQLite が authoritative。
3. `_append_jsonl()` は `app.py` または `db.py` に実装されている。
