## Goal

## Scope

**In scope:**
- 入力: docs/ 内の 05_*.md ファイル (12 ファイル確認済み、詳細は後述)
- 出力: 13 の新規 Markdown ファイル (`05_agent_00_` ～ `05_agent_12_`)

**Out of scope:**

## Assumptions

1. 出力ファイル (`05_agent_NN_*.md`) は既存ファイルとは別名のため、旧ファイルとの命名衝突は発生しない。
2. 旧ファイル削除の可否は実装フェーズで判断する (本計画ではインデックス化を推奨)。
3. 要件で定義された 9 セクション構成テンプレートを各章に適用する (不適用セクションは省略)。
4. `05_ref-*` ドキュメントが canonical source であり、`05_agent-impl-*` は補足・説明資料として扱う。
5. 既存の `05_spec_agent.md` (routing.md の参照先) は本タスクで変更しない。
6. `routing.md` の "Docs → task mapping" に新ファイルへのエントリを追加する。

## Implementation Steps

1. **全入力ファイルの読み込みと責務マッピング**
   - 12 ファイルを読み込み、セクション単位で "どの出力章に属するか" のマッピングを作成
   - Spec Conflict / Open Question を特定してリストアップ
   - 完了基準: マッピング表が確定している

2. **重複トピックの canonical ソース決定**
   - 各重複トピックについて ref-* を優先して canonical を 1 つ選択
   - 完了基準: 重複トピック全件に canonical 章が割り当てられている

3. **05_agent_00 〜 05_agent_06 の作成 (概要・フロー・状態・通信・ツール)**
   - 各章に要件の必須テンプレート (Purpose / Scope / ...) を適用
   - 完了基準: 各ファイルが Markdown として valid であり、要件テンプレートのセクションを含む

4. **05_agent_07 〜 05_agent_12 の作成 (CLI・設定・データ・運用・拡張・API)**
   - 同上
   - 完了基準: 同上

5. **Spec Conflict / Open Question セクションの挿入**
   - 旧仕様と現行仕様が混在する箇所に `Spec Conflict` ブロックを挿入
   - 完了基準: 矛盾箇所が全て明示されている

6. **旧ファイルのインデックス化**
   - `rg` で各旧ファイルへの cross-reference を確認
   - 旧ファイルの内容を「新章へ移行済み」インデックスに置換
   - 完了基準: 旧ファイルが新ファイル群へのリンクを含むインデックスになっている

7. **routing.md の更新**
   - "Docs → task mapping" に 13 新ファイルのエントリを追加
   - 旧ファイルへの参照を新ファイルへ差し替え
   - 完了基準: `routing.md` に新章 13 件が記載されている

8. **Git commit**
   - コミットメッセージに "why" (責務境界の明確化・AI 読み取り性向上) を記載
