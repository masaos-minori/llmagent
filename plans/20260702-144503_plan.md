## Goal

Implement `scripts/checks/check_mcp_docs_consistency.py` — a standalone CI checker that detects active stdio references, stale 4-layer routing language, tool_names described as a routing input, references to deleted docs, and the absence of `04_mcp_07_tool_schema_export_policy.md` from the document guide.

## Scope

**In-scope:**
- New file: `scripts/checks/check_mcp_docs_consistency.py`
- Fix `.github/workflows/mcp-docs-consistency.yml` — wrong script path (`scripts/check_mcp_docs_consistency.py` → `scripts/checks/check_mcp_docs_consistency.py`)
- Fix active reference to deleted `04_mcp_07_mdq_rag_boundary.md` in `docs/04_mcp_04_server_catalog.md` (line 354), `docs/03_rag_01_system_overview.md` (line 22), and `docs/05_agent_09_data-layer.md` (line 178, 180) — these must pass the checker
- Add `04_mcp_07_tool_schema_export_policy.md` entry to `docs/04_mcp_00_document-guide.md`

**Out-of-scope:**
- Changes to `check_docs_consistency.py` (RAG checker — separate module, separate workflow)
- Refactoring of `docs/04_mcp_02_protocol_and_transport.md` stdio content (all stdio references there are legitimate active documentation, not stale)
- Changes to `docs/04_mcp_06_configuration_and_operations.md` stdio content (active configuration spec)
- Updating any Python source files outside `scripts/checks/`

## Assumptions

1. `scripts/checks/` is the canonical location for all doc-check scripts, consistent with `check_docs_consistency.py` and `check_no_compat.py`.
2. The CI workflow file (`mcp-docs-consistency.yml`) already exists but points to the wrong script path — updating path is sufficient without creating a new workflow.
3. stdio references in `docs/04_mcp_02_protocol_and_transport.md` and `docs/04_mcp_06_configuration_and_operations.md` are active/legitimate and must be allowlisted by filename, not flagged.
4. Historical/archive sections within docs (headed by `## Legacy`, `## Historical`, `## Removed`, `## Archive`, `## Deleted`) are allowlisted for deleted-doc references.
5. `04_mcp_07_mdq_rag_boundary.md` is confirmed deleted (not present in `docs/`); references in active sections of non-MCP docs (`03_rag_01_system_overview.md`, `05_agent_09_data-layer.md`) are stale and must be fixed before the checker runs green.
6. The checker is a standalone script callable with `python scripts/checks/check_mcp_docs_consistency.py [files...]` from the repo root, matching the interface of the existing checker.
