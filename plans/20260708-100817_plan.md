# Plan: H-4 — Remove ToolResultStore persistence from partial completion handling

## Goal

`handle_partial_completion()` から `ctx.tool_result_store.store(...)` の呼び出しを削除し、
partial completion の永続化を `session_diagnostics` のみに限定する。
合わせて `tool_result_store` への参照を含むユーザー向けメッセージ・ログも修正する。

## Scope

変更対象は以下の 4 ファイル:

- `scripts/agent/llm_transport_errors.py` (primary: store 呼び出しと try/except 削除)
- `scripts/agent/repl.py` (user-facing warning メッセージ修正)
- `scripts/agent/commands/cmd_config_stats.py` (stats 表示メッセージ修正)
- `scripts/agent/commands/cmd_context.py` (context 表示メッセージ修正)

インターフェース変更・スキーマ変更なし。Path A (small, no interface/schema changes)。

## Assumptions

- `DiagnosticStore.save()` および `DiagnosticStore.save_partial_completion()` は既存の実装を維持する
- `ctx.tool_result_store` 自体は他の用途で引き続き使用されるため削除しない
- `stat_partial_completions` カウンタは維持する
- docs (`05_agent_03_turn-processing-flow.md`) も更新が必要だが、本 plan のスコープ外とする

なし (全て解析済み)。

## Design

### 変更方針

- `handle_partial_completion()` のシグネチャは変更しない
- `ctx: AgentContext` パラメータは削除しない (session_id と stat_turns は引き続き使用)
- `tool_result_store` への依存は `llm_transport_errors.py` 内では完全に除去される
- `orjson` import は `handle_non_partial_error()` でも使用されているため保持

### コード変更後のイメージ (`llm_transport_errors.py`)

```python
def handle_partial_completion(
    e: LLMTransportError,
    ctx: AgentContext,
    diagnostic_store: DiagnosticStore,
) -> None:
    """Save partial text to diagnostic channel only."""
    incomplete_msg = f"{e.partial_text}\n[INCOMPLETE: {e.kind}]"
    diagnostic_store.save(ctx.session.session_id, "llm_transport_error", incomplete_msg)
    diagnostic_store.save_partial_completion(
        session_id=ctx.session.session_id,
        turn=ctx.stats.stat_turns,
        reason=e.kind,
        content_length=len(e.partial_text),
    )
    ctx.services_required.llm.stat_partial_completions += 1
    logger.warning("Partial LLM completion saved: %s", e.kind)
```
