# Plan: Guard Against Stale ETag/Last-Modified Overwrite in Skip Paths

## 1. Goal

Prevent `_update_etag()` from overwriting newer `documents.etag` and `documents.last_modified`
values with stale metadata from older chunk files during skip-path ingestion.

## 2. Scope

**In-Scope:**
- Add a `fetched_at` column read from the DB in `_update_etag()` to compare recency
- Guard: only update `etag`/`last_modified` if the incoming `fetched_at` (from chunk payload) is
  >= the stored `fetched_at` — newer fetch wins; stale chunk data is ignored
- Add `fetched_at` to the UPDATE in `_update_etag()` when a newer value is accepted
- Log the guard decision: "skip-path etag update: incoming newer, updating" vs
  "skip-path etag update: incoming stale, keeping existing"
- Update documentation of skip-path semantics

**Out-of-Scope:**
- Removing `etag`/`last_modified` update on skip paths entirely
- Replacing Conditional GET design

## 3. Confirmed Facts

- `fetched_at` column EXISTS in `documents` table (`scripts/db/schema_sql.py:20`)
- `_update_etag()` does `UPDATE documents SET etag=?, last_modified=? WHERE doc_id=?` — no `fetched_at` guard
- The crawl payload includes `fetched_at: datetime.now().isoformat(...)` (crawler.py:94)
- The chunk first_data (`first_data.get("etag")`) flows into `_register_url()` but `fetched_at`
  from the chunk payload is NOT currently passed to `_update_etag()`

## 4. Assumptions

1. `fetched_at` in chunk payload = crawl time of that chunk (from `crawler.py:94`)
2. `fetched_at` in DB = most recent accepted crawl time for that URL
3. If incoming `fetched_at >= stored`: incoming data is at least as fresh → update is safe
4. If incoming `fetched_at < stored`: incoming data is older → guard; keep existing DB values
5. ISO datetime strings sort lexicographically in a way that is chronologically correct
   (format `YYYY-MM-DDTHH:MM:SS` — true for ISO 8601 without timezone, as used here)

## 6. Affected Areas

**Affected Files:**
- `scripts/rag/ingestion/ingester.py`: extract `fetched_at` from `first_data`, pass to `_register_url()`;
  update `_update_etag()` to accept `new_fetched_at`, query stored `fetched_at`, guard accordingly

**Blast Radius:** Low — guard adds one SELECT query per skipped URL; logic is additive

**Risk Metrics:** `ingester.py` is high-churn; change is confined to `_update_etag()`

**Deploy Impact:** None (no schema change)

## 7. Implementation Steps

1. **Extract `fetched_at` from `first_data` in `_process_url_group()`**
   - `ingester.py:153`: add `fetched_at: str | None = first_data.get("fetched_at")`
   - Pass `fetched_at` to `_register_url()` and then to `_update_etag()`

2. **Update `_update_etag()` signature and guard**
   ```python
   def _update_etag(
       self, db, doc_id: int,
       etag: str | None, last_modified: str | None,
       new_fetched_at: str | None,
   ) -> None:
       """Refresh ETag/Last-Modified only if incoming data is not stale."""
       if etag is None and last_modified is None:
           return
       if new_fetched_at is not None:
           row = db.fetchone(
               "SELECT fetched_at FROM documents WHERE doc_id=?", (doc_id,)
           )
           stored_fetched_at = row["fetched_at"] if row else None
           if stored_fetched_at and new_fetched_at < stored_fetched_at:
               logger.info(
                   "skip-path etag update skipped: incoming stale (%s < %s) for doc_id=%d",
                   new_fetched_at, stored_fetched_at, doc_id,
               )
               return
       db.execute(
           "UPDATE documents SET etag=?, last_modified=?, fetched_at=COALESCE(?, fetched_at)"
           " WHERE doc_id=?",
           (etag, last_modified, new_fetched_at, doc_id),
       )
       db.commit()
       logger.info("skip-path etag updated for doc_id=%d", doc_id)
   ```

3. **Update `_register_url()` call site**
   - Line 274: `self._update_etag(db, existing_doc_id, etag, last_modified)` →
     `self._update_etag(db, existing_doc_id, etag, last_modified, fetched_at)`

4. **Tests**
   - Test: incoming `fetched_at` newer than stored → etag updated
   - Test: incoming `fetched_at` older than stored → etag NOT updated, logger.info called
   - Test: missing `fetched_at` in chunk → always updates (legacy behavior)
   - Test: `etag is None and last_modified is None` → early return (no DB query)

5. **Update docs**
   - Document "skip-path guard" behavior in RAG ingestion docs

## 9. Risks

- **Risk:** `COALESCE(?, fetched_at)` in UPDATE: if `new_fetched_at` is None, keeps existing `fetched_at` — correct
  → **Mitigation:** confirmed SQLite behavior; COALESCE returns first non-NULL
- **Risk:** ISO timestamp comparison fails for TZ-aware vs naive strings
  → **Mitigation:** crawler uses `datetime.now().isoformat(timespec="seconds")` — always naive; consistent
