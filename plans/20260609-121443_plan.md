## Goal

`MemoryLayer` 互換 facade および private API 依存・二重書込 API・無言フォールバック・未使用設定を除去し、各責務サービス (`MemoryInjectionService` / `MemoryIngestionService` / `MemoryRetriever` / `MemoryStore`) が明示的な public API のみで連携するアーキテクチャへ移行する。

## Design

### D-1: `MemoryServices` dataclass を新設する

`layer.py` 削除後、`AppServices.memory` に格納するオブジェクトとして `MemoryServices` を `agent/memory/services.py` に定義する。

```python
@dataclass
class MemoryServices:
    injection: MemoryInjectionService
    ingestion: MemoryIngestionService
    store: MemoryStore
    retriever: MemoryRetriever
```

`cmd_memory.py` は `mem.store.get_by_id(mid)` 等のサービスメソッドを直接呼ぶ。

### D-2: `MemoryRetriever` を 3 クラスに分割する

- `FtsRetriever`: FTS5 BM25 検索 + rescoring
- `VectorRetriever`: KNN 検索 (`knn_search()` が public API)
- `HybridRetriever`: RRF merge + 外部への検索窓口

`ingestion.py` は `VectorRetriever.knn_search()` を経由して dedup 判定を行う。

### D-3: `EmbeddingResult` を `types.py` に定義する

```python
@dataclass
class EmbeddingResult:
    success: bool
    embedding: list[float] | None
    error_kind: str | None  # "disabled" | "circuit_open" | "timeout" | "http_error" | "invalid_response"
```

`EmbeddingClient.fetch()` の返却型を `EmbeddingResult` に変更。呼出側は `result.success` で分岐する。

### D-4: `mapper.py` / `jsonl_store.py` のフォールバック削除戦略

- `mapper.py` の `row_to_entry()` からフォールバックを削除し `ValueError` を伝播する
- `MemoryEntry.__post_init__` が既に同等の `ValueError` を発生させるため、二重保護を除去するだけ
- `jsonl_store.py._entry_from_dict()` のフォールバックを削除し、`MemoryEntry()` コンストラクタの `ValueError` を catch して `None` を返す（現行の「スキップ」動作は維持、ただし理由を明示）
- `tags` の `orjson.loads` 失敗時は `ValueError` を raise し、呼出側でスキップ
