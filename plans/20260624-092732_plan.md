# Plan: Upgrade /health to Runtime-Aware Health Reporting

**Requirement:** `requires/20260623_23_require.md`
**Task size:** Path B (code change to /health endpoint + test)

---

## Goal

Expand `/health` from `{"status": "ok"}` to include DB readiness, schema state, and DLQ background-task liveness.

---

## Scope

**In:**
- `scripts/eventbus/app.py` — update `health()` to check: (1) `_db` is not None and can execute a lightweight query, (2) `_dlq_task` is not done/cancelled
- `tests/test_eventbus_health.py` — new or existing test file; add tests for healthy and degraded states

**Out:**
- Full monitoring integration
- Readiness/liveness split (document as "Needs confirmation" if unclear)

---

## Assumptions

1. Lightweight DB check: `_db.execute("SELECT 1")` — fast, non-destructive.
2. DLQ task liveness: `_dlq_task is not None and not _dlq_task.done()`.
3. Response shape: `{"status": "ok"|"degraded", "db": "ok"|"unavailable", "dlq_task": "running"|"stopped"}`.
