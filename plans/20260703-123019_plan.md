# Plan: Unify Audit Log JSON-lines Schema and Preserve Transport-Error Audit Events

## Goal

Rename `session`/`request` to `session_id`/`request_id` in the MCP server audit helper, remove the `mcp_request_id == ""` guard in `audit_tool_exec`, convert `write_round_exec` to JSON-lines output, add `source` field to agent-side events, and update tests and docs to reflect the unified schema.

---

## Scope

**In-Scope**

- `scripts/mcp/audit.py` — rename field keys, ensure `error_type`/`server_key` always present
- `scripts/agent/tool_audit.py` — remove empty-request-id guard; convert `write_round_exec` to JSON-lines; add `source="agent"` via model or inline dict
- `scripts/agent/shared/models.py` — add `source: str = "agent"` field to `ToolExecEvent`
- `tests/test_audit_log_format.py` — fix broken `TestAgentAuditLogFormat` tests; update field-name assertions in `TestMcpAuditLogFormat`
- `tests/test_tool_audit.py` — update `test_skips_when_mcp_request_id_is_empty` to reflect new behavior
- `docs/04_mcp_02_protocol_and_transport.md` — replace key=value format example with JSON-lines; update field table
- `docs/04_mcp_06_configuration_and_operations.md` — replace key=value audit format description with JSON-lines; update log-viewing commands and per-server audit table
- `docs/04_mcp_90_inconsistencies_and_known_issues.md` — mark the audit format inconsistency as resolved; update cross-reference format to JSON-lines

**Out-of-Scope**

- Per-server audit logs in shell/file-delete/github/mdq (ISO8601 + op= format) — those use separate log files and are not the shared audit log
- Adding `source` field to `ToolApprovalEvent` or `ApprovalDecisionEvent` — not required by the spec
- Changing the SQLite or cicd audit pipeline configuration
- Altering `deploy.sh` — `rsync scripts/` and `rsync tests/` already cover all affected files

---

## Assumptions

1. All callers of `_audit_log()` in MCP servers already pass `session_id=` and `request_id=` as keyword arguments (confirmed: cicd, github, shell, mdq servers all use these kwarg names).
2. Existing tests in `test_tool_audit.py` that assert `ctx.services.audit_logger` are using a MagicMock, and the test mocks need to match `ctx.services_required.audit_logger` as used by the implementation — or the tests themselves are wrong. (Evidence: `TestAgentAuditLogFormat` fails because the mock sets `ctx.services.audit_logger` but implementation reads `ctx.services_required.audit_logger`.)
3. The required schema specifies `"error_type": ""` (always present) and `"server_key": "file_read"` (always present) — current code conditionally omits them when empty; this must change to match the required schema.
4. `write_round_exec` currently calls `logger.info("round_exec", extra={...})`, which does NOT produce a parseable JSON-lines record; it relies on a log formatter to render the `extra` dict. Converting to `logger.info(json.dumps({...}))` will break the existing `TestWriteRoundExec` tests that assert `call_args[1]["extra"]`.
5. The `deploy.sh` file uses `rsync scripts/` and `rsync tests/` — no deploy.sh change is needed.

## Implementation Steps

### Step 1 — Fix `scripts/mcp/audit.py`: rename fields and always emit `error_type`/`server_key`

**Changes:**
- Rename `"session"` → `"session_id"` and `"request"` → `"request_id"` in the `record` dict.
- Move `error_type` and `server_key` out of the conditional blocks; always include them in the record (empty string when not provided).

**Completion criteria:**
- `_audit_log(logger, session_id="s", request_id="r", action="t", target="/f", outcome="ok")` produces a record with `session_id`, `request_id`, `error_type=""`, `server_key=""`.
- No `"session"` or `"request"` key in the emitted dict.

**Completion criteria:**

### Step 3 — Fix `scripts/agent/tool_audit.py`: remove empty-request-id guard and convert `write_round_exec` to JSON-lines

**Changes in `audit_tool_exec`:**
- Remove `or not mcp_request_id` from the guard on line 162: change `if ctx.services_required.audit_logger is None or not mcp_request_id:` to `if ctx.services_required.audit_logger is None:`.

**Changes in `write_round_exec`:**
- Replace the `ctx.services_required.audit_logger.info("round_exec", extra={...})` call with a `ctx.services_required.audit_logger.info(_json_dumps({...}))` call, where the dict includes `"event": "round_exec"`, `"source": "agent"`, `"ts": time.time()`, and all existing fields as top-level keys.

**Completion criteria:**
- Calling `audit_tool_exec(ctx, "tool", {}, True, "")` with a non-None logger produces a logged event with `mcp_request_id=""`.
- Calling `write_round_exec(ctx, ...)` produces a `logger.info` call with a single positional JSON string argument (no `extra=` kwarg).
