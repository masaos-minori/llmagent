# Plan: Remove MCP deferred output from /reload

Source requirement: `requires/20260708_31_require.md` (H-9)

## Goal

Ensure `/reload` never prints `[DEFER]` for MCP server config changes, and
that auth token / startup mode changes render only under `[RESTART]`.

## Scope

In scope:
- `scripts/agent/services/config_reload.py` â€” no new change beyond the H-1
  design (`plans/20260709-094640_plan.md`), which already stops MCP fields
  from ever populating `deferred`.
- `tests/test_cmd_config_refactor.py` â€” the existing
  `TestCmdReloadDeferred::test_deferred_items_rendered` test uses the example
  string `"mcp/svc.auth_token"` (an MCP-specific example) to prove the
  generic `[DEFER]` rendering mechanism works. After H-4, this string can
  never be produced by real code, so the example is now misleading and
  contradicts this requirement's intent if left as-is; relabel it with a
  non-MCP example.

Out of scope:
- `scripts/agent/commands/cmd_config.py` â€” verified below that its rendering
  is already fully conditional (`if result.deferred:` / `elif
  result.needs_restart:` chains), so once `deferred` is never populated by
  MCP, the deferred section and status line are automatically skipped. No
  code change needed there.
- Deprecating the `deferred` field/rendering path entirely â€” the requirement
  explicitly frames this as optional future work ("consider... in a separate

## Design

No new production code. This requirement is satisfied structurally by the
H-1/H-4/H-5 changes; the only concrete action is a test relabel plus a
documented finding:

**Finding (not actioned in this plan)**: once H-1/H-4/H-5 land, no code path
writes to `ConfigReloadOutcome.deferred`. The field, its rendering branch in
`cmd_config.py`, and `test_deferred_items_rendered`'s generic-mechanism test
become dead weight kept alive only for a hypothetical future non-MCP
consumer. The requirement explicitly defers the decision to remove/deprecate
it to "a separate cleanup" â€” this plan does not remove it, only records the
finding for that future decision.

Test relabel:

```python
outcome = ConfigReloadOutcome(deferred=["example_field"])
...
assert "[DEFER]" in out
assert "example_field" in out
assert "Deferred (next connection)" in out
```

(replaces `deferred=["mcp/svc.auth_token"]` / `assert "mcp/svc.auth_token" in out`)

## Implementation steps

1. Confirm `_classify_mcp_server_changes()` (H-1 design, incorporating H-4/H-5)
   never appends to `deferred`.
2. In `tests/test_cmd_config_refactor.py`, replace the MCP-flavored example in
   `test_deferred_items_rendered` with a non-MCP placeholder string, per
   Design above, so the test no longer implies MCP changes can be deferred.
3. Add an end-to-end reload test exercising the real
   `_classify_mcp_server_changes()` (not just a hand-built
   `ConfigReloadOutcome`) with a changed `auth_token`, asserting the printed
   `/reload` output contains `[RESTART]` and does not contain `[DEFER]`:

```python
def test_reload_mcp_auth_token_change_prints_restart_not_defer(self, capsys) -> None:
    old_srv = McpServerConfig(transport=TransportType.HTTP, url="http://localhost:8080", cmd=[], auth_token="old")
    ctx = _make_ctx()
    ctx.cfg.mcp.mcp_servers = {"svc": old_srv}
    cmd = _FakeCmd(ctx)
    new_cfg = {"mcp_servers": {"svc": {"transport": "http", "url": "http://localhost:8080", "auth_token": "new"}}}
    with patch("shared.config_loader.ConfigLoader.load_all", return_value=new_cfg):
        cmd._cmd_reload()
    out = capsys.readouterr().out
    assert "[RESTART] - mcp/svc.auth_token" in out
    assert "[DEFER]" not in out
```
   (Adapt fixture construction to match `_make_ctx()`'s actual shape in
   `tests/test_agent_cmd_config.py`.)
4. Run the full validation sequence (`rules/toolchain.md`) and fix any findings.
