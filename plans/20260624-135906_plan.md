## Goal

`tool_results.session_id` に `REFERENCES sessions(session_id) ON DELETE CASCADE` FK 制約を追加し、セッション削除時に孤立行が残らないようスキーマと migration を整備する。

## Scope

**In**:
- `scripts/db/schema_sql.py` — `tool_results` DDL に FK 制約追加
- `scripts/db/create_schema.py` — `_migrate_session_schema()` によるテーブル再作成 migration 追加

**Out**:
- `ToolResultStore` API の変更
- `tool_results` を `messages` テーブルに統合する変更
- nullable `session_id` セマンティクスの変更

## Assumptions

1. `tool_results.session_id` は NULL を許容したまま FK を付与する。SQLite では NULL 値は FK 制約から除外されるため、"sessionless" な tool result は引き続き挿入可能。
2. SQLite では `ALTER TABLE ... ADD CONSTRAINT` が不可能なため、FK 追加には `CREATE TABLE ... AS` + `DROP` + `RENAME` によるテーブル再作成が必要。
3. `PRAGMA foreign_keys = ON` は migration 実行中のみ有効にすれば十分（WAL 環境では接続ごとに設定が必要）。
4. 既存の `tool_results` データは migration で保全する。
5. 既存 `idx_tool_results_session` インデックスは新テーブルでも再作成する。

## Design

### schema_sql.py の変更

```sql
-- 変更前
CREATE TABLE IF NOT EXISTS tool_results (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER,
    ...
);

-- 変更後
CREATE TABLE IF NOT EXISTS tool_results (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER REFERENCES sessions(session_id) ON DELETE CASCADE,
    ...
);
```

### create_schema.py の変更

`_migrate_session_schema(conn)` を追加:
1. `PRAGMA table_info(tool_results)` で既存カラムを確認
2. FK を持つ新テーブル `tool_results_new` を作成
3. `INSERT INTO tool_results_new SELECT ... FROM tool_results`
4. `DROP TABLE tool_results`
5. `ALTER TABLE tool_results_new RENAME TO tool_results`
6. `CREATE INDEX IF NOT EXISTS idx_tool_results_session ON tool_results(session_id)`

`create_session_schema()` 末尾で `_migrate_session_schema(db.conn)` を呼び出す。

### "sessionless" tool results の扱い

`session_id IS NULL` の行は FK 制約から除外される（SQLite 仕様）。既存の `store(session_id=None, ...)` と `list_recent(session_id=None)` は変更不要。

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| テーブル再作成 migration 実行中にクラッシュするとデータ損失 | Low | migration をトランザクション内で実行; `SAVEPOINT` か明示的 `BEGIN`/`ROLLBACK` で保護 |
| NULL session_id の既存行が FK 追加後に挙動変更される | Low | SQLite では NULL は FK 非対象; `INSERT` / `SELECT` 動作変わらず |
| `PRAGMA foreign_key_list` が空でも FK が既にある場合の誤 migration 実行 | Low | migration ガードを冪等に実装（新テーブル作成失敗なら即 rollback） |
