# Plan: Clarify Timestamp Policy for Newly Recreated DBs Only

**Source requirement:** `requires/20260702_48_require.md`
**Created:** 2026-07-02

---

## Goal

Document and enforce a single canonical timestamp format for newly recreated databases, while explicitly stating that existing DB records are not automatically converted.

---

## Scope

**In:**
- Define canonical timestamp format for new DB creation
- Update `docs/90_shared_04_db_architecture_and_schema.md` §7a to reflect the chosen format and no-migration policy
- If the canonical format is chosen to be `YYYY-MM-DDTHH:MM:SSZ` (Z suffix): update workflow table DDL in `scripts/db/schema_sql.py` to use `strftime('%Y-%m-%dT%H:%M:%SZ', 'now')` as DEFAULT, and update `_now()` in `scripts/agent/workflow/state_store.py` to emit `Z`-suffix strings
- Document that Python-side `datetime.now(UTC).isoformat()` (+00:00 suffix) is currently used for workflow tables and may continue for legacy DBs

**Out:**
- Automatic conversion of existing DB timestamps
- Migration of any existing DB files
- Changes to Event Bus runtime (publisher/subscriber/dispatcher/DLQ worker)
- Changes to RAG or Session table schemas (already use Z suffix DDL defaults)

---

## Assumptions

1. The preferred canonical format is `YYYY-MM-DDTHH:MM:SSZ` (Z suffix), consistent with RAG/Session tables and the Event Bus DDL already documented in `schema_sql.py`.
2. Existing `workflow.sqlite` files produced before this change will retain `+00:00` timestamps; no conversion is performed.
3. `_now()` in `state_store.py` currently produces `+00:00` suffix via `datetime.now(UTC).isoformat()`.
4. No other callers generate workflow timestamps outside of `state_store.py`.
5. The "no-migration" policy is intentional and permanent for existing DBs.

## Implementation Steps

### Phase 1: Confirm canonical format (prerequisite)

- [ ] 1a. Confirm with user: Z suffix (`YYYY-MM-DDTHH:MM:SSZ`) or Python ISO (`+00:00`)?
  - If Z suffix → proceed to Phase 2.
  - If `+00:00` → skip Phase 2; proceed to Phase 3 (docs only).

### Phase 2: Code changes (Z suffix path only)

- [ ] 2a. In `scripts/db/schema_sql.py`, update `_WORKFLOW_SCHEMA` — add `DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` to:
  - `tasks.created_at`, `tasks.updated_at`
  - `attempts.started_at`, `attempts.ended_at` (ended_at is nullable; add DEFAULT only to started_at)
  - `processed_events.recorded_at`
  - `artifacts.created_at`
  - `approvals.created_at`
  - Keep existing NOT NULL constraints unchanged.
- [ ] 2b. In `scripts/agent/workflow/state_store.py`, update `_now()`:
  ```python
  # Before
  return datetime.now(UTC).isoformat()
  # After
  return datetime.now(UTC).strftime('%Y-%m-%dT%H:%M:%SZ')
  ```
  This affects: `create_task`, `update_task_status`, `start_attempt`, `finish_attempt`, `begin_stage_if_new`, `record_artifact`, `request_approval`, `resolve_approval`.

### Phase 3: Documentation update

- [ ] 3a. In `docs/90_shared_04_db_architecture_and_schema.md`, update §7a:
  - State the chosen canonical format explicitly.
  - Add a sentence: "Existing DB rows retain the format they were written with; no automatic conversion is performed."
  - Add a note distinguishing: (a) new DB DDL defaults, (b) Python-side generation, (c) legacy +00:00 rows in workflow tables created before this change.
  - Cross-reference §8 (Schema Generation and Migration Approach) to confirm no migration is triggered.

### Phase 4: Validation

- [ ] 4a. Run `ruff check scripts/db/schema_sql.py scripts/agent/workflow/state_store.py`
- [ ] 4b. Run `mypy scripts/agent/workflow/state_store.py`
- [ ] 4c. Run `uv run pytest tests/ -x -q` — confirm no timestamp-related test failures
- [ ] 4d. Review doc diff to ensure no sentence implies existing records are rewritten

| Risk | Likelihood | Mitigation |
|---|---|---|
| Existing workflow.sqlite rows have +00:00; queries that compare timestamps across old and new records may sort inconsistently | Low | Document clearly; both formats are valid ISO-8601 and SQLite TEXT comparison is lexicographic — they sort consistently only if all values use the same offset notation. Add a note in the docs warning about mixed-format ordering. |
| `_now()` change in `state_store.py` may break tests that assert exact timestamp format | Low | Run pytest before and after; fix any format assertions. |
| DDL DEFAULT addition has no effect on existing tables (IF NOT EXISTS skips re-creation) | Known / by design | This is intentional. Document in §8 that the DEFAULT is for new DB creation only. |
