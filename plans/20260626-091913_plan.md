# Plan: Expose dry_run in /v1/tools inputSchema for all affected MCP tools

Source: `requires/20260625_06_require.md`

---

## Goal

`write_file`, `move_file`, `delete_file`, `delete_directory`, `trigger_workflow` の `/v1/tools` inputSchema に `dry_run` フィールドが正しく公開されていることを確認・修正し、LLM が dry_run を直接使用できる状態にする。

---

## Scope

**In-Scope**
- `mcp/file/write_tools.py`: `write_file`, `move_file` の inputSchema 確認（確認済み: dry_run あり）
- `mcp/file/delete_tools.py`: `delete_file`, `delete_directory` の inputSchema 確認（確認済み: dry_run あり）
- `mcp/cicd/`: `trigger_workflow` の inputSchema 確認・追加
- `/v1/tools` レスポンスのリグレッションテスト追加

**Out-of-Scope**
- dry_run の実装ロジック変更
- 他ツールへの dry_run 追加

---

## Assumptions

1. `file/write_tools.py` と `file/delete_tools.py` は既に `dry_run` を inputSchema に含んでいる（コード確認済み）。
2. `cicd/models.py` line 102 に `dry_run: bool = Field(...)` はあるが、cicd の tools 定義ファイルで inputSchema に含まれているかは未確認。
3. `cicd/tools.py` または `cicd/server.py` に `/v1/tools` の tool 定義がある。
