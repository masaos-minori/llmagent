# Plan: H-7 — Clean up _http_procs entries during shutdown_all

## Goal

Update `HttpServerLifecycleManager.shutdown_all()` to pop and remove all entries from `_http_procs` (including already-exited processes) after termination, so that repeated `shutdown_all()` calls are safe no-ops and stale records do not accumulate.

## Scope

**In**: `scripts/agent/http_lifecycle.py` — `shutdown_all()` method only.

**Out**: No changes to `factory.py`, `startup.py`, `restart()`, or any other method.

## Assumptions

1. After H-1 is implemented, `shutdown_all()` must also close associated stderr file handles in `_stderr_files`; this plan assumes H-1 may or may not be merged first — include the `_stderr_files` cleanup as a conditional block.
2. Iterating `list(self._http_procs.items())` then clearing is acceptable; alternatively, pop one at a time. Pop-while-iterating is safe if we snapshot first.
3. An already-exited process (`proc.poll() is not None`) does not need `terminate()`; just remove and log.
4. A failed termination for one server must not prevent cleanup of the others.

## Implementation Steps

1. **Refactor `shutdown_all()` to pop entries**
   - Replace the current `for key, proc in list(self._http_procs.items()):` loop with one that pops each entry:
     ```python
     async def shutdown_all(self) -> None:
         keys = list(self._http_procs.keys())
         for key in keys:
             proc = self._http_procs.pop(key, None)
             if proc is None:
                 continue
             if proc.poll() is not None:
                 logger.debug("Lifecycle: %r already exited; removing entry", key)
                 continue
             try:
                 await self._terminate_with_timeout(proc, key, timeout=5.0)
             except (OSError, TimeoutError) as e:
                 logger.warning("Lifecycle: error stopping HTTP subprocess %r: %s", key, e)
             # Close stderr file handle if H-1 is merged
             stderr_file = getattr(self, "_stderr_files", {}).pop(key, None)
             if stderr_file is not None:
                 try:
                     stderr_file.close()
                 except OSError as e:
                     logger.warning("Lifecycle: error closing stderr log for %r: %s", key, e)
     ```
   - Criterion: after `shutdown_all()`, `self._http_procs` is empty.

2. **Add tests**
   - Test: two fake processes → `shutdown_all()` → `_http_procs` is empty.
   - Test: one already-exited process → `shutdown_all()` → entry is removed, no error.
   - Test: termination raises `OSError` → cleanup continues for remaining servers.
   - Test: `shutdown_all()` twice → no error, `_http_procs` still empty.
   - Criterion: `uv run pytest tests/test_lifecycle.py -v` passes.
