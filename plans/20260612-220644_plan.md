```python
def pin_note(self, note_id: int) -> bool:
    """Set pinned=1 for a note. Returns False when not found."""
    ...

def unpin_note(self, note_id: int) -> bool:
    """Set pinned=0 for a note. Returns False when not found."""
    ...

def get_pinned_notes(self) -> list[dict]:
    """Return all notes where pinned=1, ordered by note_id."""
    SELECT note_id, content, pinned, created_at FROM notes WHERE pinned=1 ORDER BY note_id

def search_notes(self, query: str, limit: int = 5) -> list[dict]:
    """Return notes matching query via LIKE search, ordered by note_id."""
    SELECT note_id, content, pinned, created_at
    FROM notes
    WHERE content LIKE '%' || ? || '%'
    ORDER BY note_id
    LIMIT ?
```

### orchestrator.py への turn ごと注入追加 (`_handle_note_injection()`)

`_handle_memory_injection()` と並列に呼ばれる新メソッド:

```python
async def _handle_note_injection(self, line: str) -> None:
    """Search notes by current query and inject relevant ones into history."""
    ctx = self._ctx
    if not ctx.cfg.tool.auto_inject_notes:
        return
    notes = ctx.session.search_notes(line, limit=5)
    if not notes:
        return
    note_block = "[Relevant Notes]\n" + "\n".join(
        f"- {n['content']}" for n in notes
    )
    ctx.conv.history.append({
        "role": "system",
        "content": note_block,
        "_note_injected": True,  # type: ignore[typeddict-unknown-key]
    })
```

`_process_turn()` の `await self._handle_memory_injection(line)` の直後に `await self._handle_note_injection(line)` を追加。

### cmd_notes.py コマンド拡張

```python
def _note_pin(self, arg: str) -> None:
    """Pin a note by ID."""
    if not arg.isdigit():
        self._out.write_validation_error("/note pin <id>")
        return
    ok = self._ctx.session.pin_note(int(arg))
    if ok:
        self._out.write_success(f"Note {arg} pinned.")
    else:
        self._out.write_no_data(f"Note {arg} not found.")

def _note_unpin(self, arg: str) -> None:
    """Unpin a note by ID."""
    if not arg.isdigit():
        self._out.write_validation_error("/note unpin <id>")
        return
    ok = self._ctx.session.unpin_note(int(arg))
    if ok:
        self._out.write_success(f"Note {arg} unpinned.")
    else:
        self._out.write_no_data(f"Note {arg} not found.")

def _note_search(self, query: str) -> None:
    """Search notes by content."""
    if not query:
        self._out.write_validation_error("/note search <query>")
        return
    notes = self._ctx.session.search_notes(query)
    if not notes:
        self._out.write_no_data("No matching notes.")
        return
    rows = [
        [str(n["note_id"]), "✓" if n["pinned"] else "", n["created_at"][:19],
         n["content"][:41] + "..." if len(n["content"]) > 44 else n["content"]]
        for n in notes
    ]
    self._out.write_table(["ID", "Pin", "Created", "Content"], rows)
```

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| 既存 DB に `pinned` カラムがないため `ALTER TABLE` が必要 | high | `ALTER TABLE notes ADD COLUMN pinned INTEGER NOT NULL DEFAULT 0` で既存データに後方互換 |
| `LIKE` 検索が注入ミスの原因になる | medium | ピン留めノートが「保証ベースライン」として機能するため retrieval miss は限定的。将来的に FTS5 化を検討 |
| 全件注入から変更後、既存ユーザーが非ピン留めノートが届かないと感じる | medium | `auto_inject_notes` のドキュメント更新と、`/note pin` の積極的な利用を促す |
| `_handle_note_injection()` が orchestrator に追加されることで turn 処理が遅くなる | low | `search_notes()` は `LIKE` + `LIMIT 5` で高速。注入スキップ条件 (`auto_inject_notes=False`) あり |
