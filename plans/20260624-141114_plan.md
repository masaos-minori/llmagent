## Goal

`db/maintenance.py` の `ConfigLoader().load_all()` 3 箇所を `ConfigLoader().load("common.toml")` に変更し、メンテナンスポリシーのコンフィグ読み取りを `common.toml` に限定する。

## Scope

**In**:
- `scripts/db/maintenance.py` — `load_all()` → `load("common.toml")` (3 箇所)

**Out**:
- メンテナンス操作の全体的な再設計
- メンテナンスのスケジューリングポリシーの変更
- `config_loader.py` の変更

## Assumptions

1. 対象コンフィグキー (`sqlite_retention_max_sessions`, `sqlite_retention_max_age_days`, `sqlite_wal_checkpoint_mode`, `sqlite_archive_dir`) はすべて `common.toml` に存在する (grep 確認済み)。
2. `load("common.toml")` は `common.toml` が存在しない場合 `ValueError` を raise する。現行の `load_all()` は欠損時に static default で補完する挙動だったが、`common.toml` はデプロイ必須ファイルなので early raise が望ましい。
3. 変更後もデフォルト値 (`cfg.get("key", default)`) のフォールバックは維持する (キーが `common.toml` に存在しても `.get` が None 返した場合のみ使用)。
