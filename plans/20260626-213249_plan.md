## Goal

Treat index_paths and refresh_index as side-effect tools by adding safety, scheduling, and concurrency controls for MDQ indexing operations.

## Scope

**In-Scope**:
- Mark index_paths and refresh_index as write/admin tools in tool safety tiers
- Add requires_serial: true or resource_scope: "mdq_index" metadata
- Configure concurrency limit for mdq server
- Serialize concurrent indexing operations
- Define behavior for search while indexing is running
- Invalidate cache after successful index updates

**Out-of-Scope**:
- Adding new tools or features
- Changes to other MCP servers' safety controls

## Assumptions

1. SQLite FTS5 triggers provide row-level integrity but not concurrent write protection
2. Cache invalidation should occur after successful index refresh
3. Search during indexing should either wait or return partial results with a warning

## Affected Areas & Tool Evidence

- **Affected Files**:
  - `config/mdq_mcp_server.toml` — add concurrency limit configuration
  - `scripts/mcp/mdq/tools.py` — add write/admin metadata to index_paths and refresh_index
  - `scripts/mcp/mdq/service.py` — add concurrency control, cache invalidation
  - `agent.toml` — add mdq to tool_safety_tiers and tool_concurrency_limits

- **Blast Radius**: Medium — changes safety classification for MDQ indexing tools. Existing valid queries will still work; concurrent indexing operations will be serialized.

## Implementation Steps

1. **Phase 1: Add tool metadata**
   - Update index_paths in tools.py: add `"is_write": true`, `"requires_serial": true` or `"resource_scope": "mdq_index"`
   - Update refresh_index in tools.py: add same metadata

2. **Phase 2: Add concurrency config**
   - Add `concurrency_limit = 1` to config/mdq_mcp_server.toml for mdq server
   - Add `mdq` key to tool_concurrency_limits in agent.toml

3. **Phase 3: Add serialization control**
   - Add threading.Lock or asyncio.Lock in service.py for index_paths/refresh_index
   - Wrap indexing operations with lock context manager

4. **Phase 4: Define search behavior during indexing**
   - Add `_is_indexing` flag in service.py
   - When searching during indexing, return partial results with warning

5. **Phase 5: Add cache invalidation**
   - After successful index refresh, invalidate relevant tool caches
   - Clear any cached search results or outline data
