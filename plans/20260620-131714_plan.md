# Plan: github-mcp app_module consistency + regression test (requirement 20260619_21)

## Goal

Ensure all MCP server `app_module` values follow the standard dotted module path pattern and add a regression test to detect future inconsistencies.

## Scope

**In scope:**
- Verify all 11 MCP server `app_module` values use `mcp.<submodule>.server:app` pattern (already done in commit `85dcf25`)
- Add a regression test that verifies every MCP server's `app_module` is importable via the standard uvicorn startup path

**Out of scope:**
- Replacing the OpenRC startup mechanism
- Refactoring unrelated GitHub MCP features
- Changing any other MCP server behavior

2. All MCP servers subclass `MCPServer` from `mcp/server.py` and define `app_module` as a class attribute
3. A test can iterate over all server classes, extract `app_module`, and attempt import via `importlib.import_module()`

## Risks & Mitigations

- **Risk**: Test discovery mechanism may miss servers in nested directories (e.g., `mcp/file/`) → **Mitigation**: Use recursive glob pattern for server.py files
- **Risk**: Import side effects during test (e.g., FastAPI app startup) → **Mitigation**: Only import the module, do not instantiate the app; use `importlib.util.find_spec()` as a lighter alternative
- **Risk**: Test runs slowly if many servers → **Mitigation**: Use `pytest.mark.slow` marker or skip in CI if needed; module import is typically fast
