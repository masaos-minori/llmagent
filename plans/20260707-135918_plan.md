# Plan: M-6 — Add lifecycle snapshot API for MCP subprocess status

## Goal

Add `get_process_info(server_key)` and `list_processes()` to `HttpServerLifecycleManager`, expose them through `_ServerLifecycleRouter`, and surface process details (`managed`, `pid`, `pgid`, `running`, `last_exit_code`, `stderr_log`) in `/mcp status` via `McpStatusService`.

## Scope

**In**: `scripts/agent/http_lifecycle.py` — add `get_process_info()` and `list_processes()`; `scripts/agent/factory.py` — expose snapshot methods on `_ServerLifecycleRouter`; `scripts/agent/services/mcp_status.py` — call snapshot API; `scripts/agent/services/models.py` — add `ProcessInfoSnapshot` DTO.

**Out**: No changes to `LifecycleManagerProtocol` (optional method pattern); no changes to `cmd_mcp.py` renderer beyond what M-4 already adds; no changes to config or DB schema.

## Assumptions

1. H-8 adds `_http_pgids: dict[str, int]` to `HttpServerLifecycleManager`; this plan uses it with a `getattr` guard for pre-H-8 compatibility.
2. H-1 adds `_stderr_files` with associated log paths; `stderr_log` field is the file path as a string.
3. `last_exit_code` is available via `proc.poll()` for already-exited processes (None if still running).
4. Persistent (externally managed) servers have no entry in `_http_procs`; `managed=False` for these.
5. Adding snapshot methods to `_ServerLifecycleRouter` without extending `LifecycleManagerProtocol` is acceptable; callers cast or use duck-typing.

## Implementation Steps

1. **Add `ProcessInfoSnapshot` DTO to `models.py`**
   - `@dataclass(frozen=True)` with fields: `server_key: str`, `managed: bool`, `pid: int | None`, `pgid: int | None`, `running: bool`, `last_exit_code: int | None`, `stderr_log: str`.
   - Criterion: importable from `agent.services.models`.

2. **Implement `get_process_info()` in `HttpServerLifecycleManager`**
   - `def get_process_info(self, server_key: str) -> ProcessInfoSnapshot | None`
   - If `server_key` not in `_http_procs`: return `None` (caller treats as unmanaged).
   - Otherwise: `proc = self._http_procs[server_key]`; `running = proc.poll() is None`; `last_exit_code = proc.poll()`.
   - `pid = proc.pid if proc else None`.
   - `pgid = getattr(self, "_http_pgids", {}).get(server_key)`.
   - `stderr_log` from `getattr(self, "_stderr_log_paths", {}).get(server_key, "")` (H-1 stores path alongside handle).
   - Criterion: returns populated snapshot for known subprocess servers.

3. **Implement `list_processes()` in `HttpServerLifecycleManager`**
   - Returns `list[ProcessInfoSnapshot]` for all known `_http_procs` keys.
   - Criterion: empty list after `shutdown_all()`.

4. **Expose in `_ServerLifecycleRouter`**
   - Add `def get_process_info(self, server_key: str) -> ProcessInfoSnapshot | None` delegating to `self._http_mgr.get_process_info(server_key)`.
   - Add `def list_processes(self) -> list[ProcessInfoSnapshot]` delegating to `self._http_mgr.list_processes()`.
   - For unknown server key: return `None`.
   - Criterion: callable from `McpStatusService` via duck-typing on `ctx.services_required.lifecycle`.

5. **Populate `McpProbeResult` fields in `McpStatusService._probe_single_server()`**
   - Call `ctx.services_required.lifecycle.get_process_info(key)` if method exists (use `getattr`).
   - Set `managed=True`, `pid`, `pgid`, `running`, `last_exit_code`, `stderr_log` from snapshot.
   - Persistent servers: `managed=False`, others `None`.
   - Criterion: subprocess servers have `managed=True` and non-None `pid` in `McpProbeResult`.

6. **Add tests**
   - Test snapshot for running subprocess: `managed=True`, `pid` non-None, `running=True`.
   - Test snapshot for exited subprocess: `running=False`, `last_exit_code` set.
   - Test snapshot after `shutdown_all()`: `list_processes()` returns empty list.
   - Test router delegation calls manager.
   - Criterion: `uv run pytest tests/test_lifecycle.py tests/test_agent_factory.py -v` passes.
