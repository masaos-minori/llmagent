## Goal

`.md` / `.markdown` / `.mdx` ファイルを常に見出しベース chunking で処理し、長大セクションのみ通常 text chunking にフォールバックする方針に変更する。あわせて `documents` テーブルに `chunking_strategy` カラムを追加し、`/db urls` で表示できるようにする。

## Assumptions

1. `.md` / `.markdown` / `.mdx` のURL判定は拡張子チェックのみで十分（content-based heuristic は md_index_enable フラグ有効時のみ維持）
2. SQLite の `ALTER TABLE ADD COLUMN` は既存行の DEFAULT 値が自動適用されるため、既存レコードの backfill は不要
3. `chunking_strategy` は chunk JSON ファイルに書き込んでから `RagIngester` が読み取る経路で受け渡す
4. `_chunk_markdown_by_heading()` のフォールバック（`_chunk_english()`）は現状のままで十分
5. `document_repo.py` が `SQLiteHelper("session")` を使用しているが、`documents` テーブルは `rag.sqlite` に存在する — この不整合については下記 Unknowns を参照

## Design

### ChunkSplitter の変更

**変更前:**
```python
def _is_markdown_source(self, data):
    if not self._md_index_enable:
        return False
    url = data.get("url", "")
    if url.endswith((".md", ".markdown", ".mdx")):
        return True
    content = data.get("content", "")
    return len(re.findall(MARKDOWN_HEADING_RE + r" .+", content, re.MULTILINE)) >= 2
```

**変更後:**
```python
def _is_markdown_source(self, data):
    url = data.get("url", "")
    # .md は常に見出しベース chunking
    if url.endswith((".md", ".markdown", ".mdx")):
        return True
    # 非 .md は md_index_enable フラグが有効な場合のみヒューリスティック判定
    if not self._md_index_enable:
        return False
    content = data.get("content", "")
    return len(re.findall(MARKDOWN_HEADING_RE + r" .+", content, re.MULTILINE)) >= 2
```

`_build_chunk_list()` 内で `use_markdown` の値を `_write_chunk_files()` に渡すため、chunk JSON のペイロードに `chunking_strategy: "heading" | "text"` を追加する。

### DB スキーマ変更

```sql
ALTER TABLE documents ADD COLUMN chunking_strategy TEXT NOT NULL DEFAULT 'text';
```

`create_schema.py` の `_RAG_SCHEMA_TEMPLATE` にも追加する。

### RagIngester の変更

`ingest_url_group()` で `first_data.get("chunking_strategy", "text")` を読み取り、`_get_or_create_document()` に渡す。INSERT 文に `chunking_strategy` を追加する。

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `document_repo.py` が `session.sqlite` を使用しているため `chunking_strategy` クエリが失敗する | 高 | **Step 0** で `SQLiteHelper("session")` → `"rag"` に修正。本番環境では既存のバグも同時解消。 |
| 既存の `rag.sqlite` に `chunking_strategy` カラムがなく本番クラッシュ | 高 | `db/maintenance.py` のマイグレーションで ALTER TABLE を実行; カラムが既に存在する場合は `sqlite3.OperationalError` を無視する |
| `.md` ファイルの見出し分割により chunk 粒度が細かすぎて文脈不足になる | 中 | `md_snippet_max_chars` (デフォルト 600) の値を調整可能にしている; 初期値を変更せずリリース |
| 既存の ingest 済みドキュメントの `chunking_strategy` が全て `'text'` のままになる | 低 | DEFAULT 値 `'text'` は既存行に適用される; 正確な値を反映するには `--force` で再 ingest が必要（今回の scope 外） |
| `chunk_splitter.py` と `ingester.py` の並列処理で `chunking_strategy` が混入する | 低 | `chunking_strategy` は URL 単位で決定するため並列処理の影響を受けない |
