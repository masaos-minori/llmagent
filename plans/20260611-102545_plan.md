# agent/services/ 11 ファイル改修計画書

## Goal

`agent/services/` 配下 11 ファイルから `except Exception`・`dict[str,Any]` 内部モデル使用・
`str(msg.get(...))`・直接 `print`/`input`・`tuple[bool,str]`/`dict`/`object` 戻り値・
後方互換フォールバックを除去し、DTO / Enum / 専用例外による fail-fast 設計に置き換える。

## Assumptions

1. `asyncio.create_task(on_first_turn(line))` で呼ばれる `session_title.generate()` において、`SessionTitleGenerationError` を送出した場合の未処理例外警告は、`_generate_session_title()` 側で catch して `logger.warning()` に変換することで対応する。`generate()` 本体はフォールバックを一切持たず例外送出のみ担う。
2. `cmd_session.py:115` の `_, message = restore_session(...)` と `cmd_context.py:114` の `_, message = undo_last_turn(...)` は `result.message` への変更で対応可能。`_` を捨てている箇所のみの軽微な変更。
3. `rag.ingestion.WebCrawler`, `ChunkSplitter`, `RagIngester` は `OSError`・`ValueError`・`RuntimeError` を送出しうる（各モジュールの `raise` 箇所から確認済み）。`except Exception` の具体化は `except (OSError, ValueError, RuntimeError)` で行い、実装時に各モジュールの具体例外を追加確認する。
4. `collect_context_state()` の `total_chars // 4` フォールバックは `hist_mgr is None` のときのみ使用。`hist_mgr is None` を `ContextStateBuildError` として fail-fast にするか、または `None` を DTO に表現するかを選択。計画ではフォールバック削除・`ContextStateBuildError` を採用する。
5. `db.maintenance.*` 関数の戻り値 (`dict`) は現行のまま DB 層の変更はしない。`DbStats`, `DbHealth` 等の DTO は service 層で DB 戻り値から構築する。
6. `mcp_status.py` の `_get_http_status()` は HTTP 通信エラー時に結果文字列を返す現在の設計から、`McpProbeResult` DTO を返す設計に変更する。ただし probe 失敗は致命的ではないため、`McpProbeError` は送出せず DTO の `available=False` として表現する。

## Design

### 新設ファイル概要

**`agent/services/enums.py`**:
```python
class IngestStage(str, Enum):
    OK = "ok"; CRAWL = "crawl"; SPLIT = "split"; INGEST = "ingest"

class McpTier(str, Enum):
    READ_ONLY = "READ_ONLY"; WRITE_SAFE = "WRITE_SAFE"
    WRITE_DANGEROUS = "WRITE_DANGEROUS"; ADMIN = "ADMIN"

class McpAvailability(str, Enum):
    OK = "OK"; STOPPED = "STOPPED"; NOT_STARTED = "NOT_STARTED"
    DEAD = "DEAD"; NO_URL = "no-url"; HTTP_ERROR = "http_error"; FAIL = "fail"

class ConversationActionType(str, Enum):
    CLEAR = "clear"; SWITCH_PROMPT = "switch_prompt"

class ExportFormat(str, Enum):
    JSON = "json"; MARKDOWN = "markdown"
```

**`agent/services/models.py`** (主要な DTO):
```python
@dataclass(frozen=True)
class IngestOutcome:
    stage: IngestStage

    n_removed: int

@dataclass(frozen=True)
class ConversationActionResult:
    action: ConversationActionType; message: str
```

## Implementation Steps

各ステップは独立コミット可能。`pytest tests/test_agent_cmd_*.py tests/test_agent_mcp_status.py` が pass してから次へ。

1. **[新設] enums.py / exceptions.py / models.py / io_ports.py を作成**
   - 既存コードへの変更なし
   - コミット: `feat(services): add enums, exceptions, models, io_ports foundation`

2. **[tuple 戻り値 DTO 化] session_restore.py / undo_service.py / conversation_service.py**
   - `tuple[bool,str]` → `SessionRestoreResult` / `UndoResult` DTO
   - `str` 戻り値 → `ConversationActionResult` DTO
   - `ValueError` → `ConversationStateError`
   - `cmd_session.py` / `cmd_context.py` の call site 追従修正
   - コミット: `refactor(services): DTO return types for restore/undo/conversation`

3. **[ingest_workflow.py] except Exception 除去 + Enum化**
   - `except Exception` → `except (OSError, ValueError, RuntimeError)` (実装時に精査)
   - `IngestResult.stage: str` → `IngestStage` Enum
   - `error: str | None` → `IngestStageError` 送出に変更
   - 戻り値 `IngestResult` → `IngestOutcome` (frozen dataclass)
   - コミット: `refactor(services): IngestOutcome DTO, fail-fast except in ingest_workflow`

4. **[session_title.py] except Exception 除去 + fallback 削除**
   - `except Exception` → `except (httpx.HTTPStatusError, httpx.RequestError, orjson.JSONDecodeError, KeyError)` + `SessionTitleGenerationError` 送出
   - フォールバック (`ctx.session.set_title(first_input[:50])`) 削除
   - 戻り値 `None` → `SessionTitleResult` DTO
   - `cmd_session._generate_session_title()` で catch して logger.warning
   - コミット: `refactor(services): SessionTitleResult DTO, remove fallback in session_title`

5. **[mcp_status.py] except Exception 除去 + 未知 tier fail-fast**

   - コミット: `refactor(services): McpProbeResult DTO, fail-fast tier, specific exceptions`

6. **[export_formatter.py] print 除去 + ExportFormat Enum + LLMMessage strict 化**
   - `print(content)` / `print(f"Exported...")` / `print(f"Export failed...")` → `ExportOutputPort` 委譲

   - `str(msg.get("content") or "")` → 型付きアクセス
   - `OSError` → `ExportWriteError`
   - コミット: `refactor(services): ExportOutputPort, ExportFormat Enum, remove print`
