## Design

### 方針決定: `n_artifacts_marked` を削除

- `tool_results.undone` への書き込みは `undo_last_turn()` が唯一のパスだった
- フィールドを always-zero で残すとコードの意図が曖昧になる
- 外部参照なし (cmd_context.py では `n_removed` のみ使用) のため削除コストゼロ
- テストファイルはbehavior消滅に合わせて削除

### `undo_last_turn()` 変更後のロジック

```
def undo_last_turn(ctx: AgentContext) -> UndoResult:
    # 1. historyから最後のuserメッセージindexを探す
    # 2. メモリinjectionマーカーをさかのぼってcut_idxを決める
    # 3. ctx.conv.history をトリム
    # 4. ctx.stats.stat_turns をデクリメント
    # 5. ctx.session.undo_last_turn() を呼ぶ
    # 6. UndoResult(n_removed=removed) を返す
    # ToolResultStore呼び出しなし
```

### `UndoResult` 変更後

```python
@dataclass(frozen=True)
class UndoResult:
    n_removed: int
```
