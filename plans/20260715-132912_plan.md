# Plan: Remove `/set` and `/db`; move `/export` and `/db session` under `/session`

Source requirement: `requires/20260715_03_require.md` (moved to `requires/done/` at the end of this workflow).

## Goal

After this change, `/set` and `/db` no longer exist as commands; `/session export markdown|json [file]` and `/session stats|health|checkpoint|vacuum|purge|recover [...]` produce output identical to the removed `/export md|json [file]` and `/db session ...` commands, with no behavior change to `llm_temperature`/`llm_max_tokens` (still configurable via `config/agent.toml` + `/reload`) and no dead code left behind for the removed commands.

## Scope

### In scope
- Delete `CommandDef` entries for `/set` and `/db`; remove `/export` as a standalone `CommandDef` (folded into `/session`'s help text).
- Delete `scripts/agent/commands/cmd_config_set.py`, `scripts/agent/commands/cmd_db.py`.
- Delete `scripts/agent/commands/db_help_display.py` (see Design — additional finding not in the requirement's Target Files list).
- Remove `_cmd_export` from `_RagExportMixin` in `cmd_rag_export.py` (keep `_cmd_compact` and the file).
- Remove the dead `_db_stats()` method from `db_stats_display.py`; keep `_db_session_stats()`, moved to live alongside `cmd_session.py`.
- Add `export`, `stats`, `health`, `checkpoint`, `vacuum`, `purge`, `recover` branches to `_cmd_session` in `cmd_session.py`, reusing `render_export()`/`write_export()` and `DbSessionOps`/`_db_session_stats` unchanged.
- Remove the `_ConfigSetMixin` and `_DbMixin` compositions from `registry.py`.
- Remove `"/set"` and `"/db"` from `AgentREPL.SLASH_COMMANDS` in `repl.py`.
- Delete `tests/test_cmd_config_refactor.py` and `tests/test_agent_cmd_db.py` in full; add ~15 equivalent test functions (health, checkpoint, vacuum, purge, session-scope, backward-compat) to `tests/test_agent_cmd_session.py`; update `tests/test_cmd_registry_ingest_removal.py` and `tests/test_command_def_sync.py`.
- Update `docs/05_agent_07_08_...md` (fold `/export` and `/db session` tables into the `/session` row/section, remove `/set` rows), `docs/05_agent_07_09_...md` (remove only the DB category section, lines 37-65 — file survives, see Design), `docs/05_agent_07_07_...md` (remove `/db rag recover`/`/db rag stats` migration rows referencing the now-removed `/db`).

### Out of scope (per requirement Non-Goals)
- `llm_temperature`/`llm_max_tokens` field names, defaults, or `/reload` behavior.
- New export formats.
- The general `_COMMANDS`-as-single-source-of-truth-for-completion redesign tracked in `requires/done/20260715_10_require.md` (see Assumptions — confirmed NOT yet implemented).
- `/memory` and memory config keys.
- Auditing the rest of the test suite for unrelated pre-existing failures.
- `scripts/agent/commands/utils.py` (`parse_flag_int`/`parse_flag_str`) — see Risks; flagged but not touched.

## Design

1. **`command_defs_list.py`**: delete the `/set` (lines 119-125) and `/db` (lines 112-118) `CommandDef` entries outright. Delete the standalone `/export` entry (lines 91-97); extend `/session`'s existing entry's help string to mention `export markdown|json [file]` alongside `list [n] | load <id> | rename <title> | delete <id>` plus the 6 DB ops, e.g.:
   `"list [n] | load <id> | rename <title> | delete <id> | export markdown|json [file] | stats|health|checkpoint|vacuum|purge|recover"`.
2. **`cmd_session.py`**: add a dict-based dispatch table for the 7 new subcommands (per UNK-02), constructed once in `__init__` similar to the deleted `_cmd_db_session`:
   ```python
   self._db_session_ops = DbSessionOps(self._ctx, self._out)  # already exists as db_session_ops.DbSessionOps
   ```
   `_cmd_session` gains an `export` branch calling a new `_session_export(args)` helper that maps `markdown`→`"md"` (UNK-01) and calls `render_export`/`write_export` exactly as `_cmd_export` did. The 6 DB-op branches call `DbSessionOps` methods and `_db_session_stats` unchanged.
3. **`db_stats_display.py`**: delete `_db_stats()` (lines 18-27, dead — no caller even today); keep `_db_session_stats()` (lines 29-38). Since `DbStatsDisplay` was only mixed into `_DbMixin` (being deleted), move `_db_session_stats` to live as a plain helper/mixin consumed by `_SessionMixin` in `cmd_session.py` (e.g. rename the surviving function/class and import it from `cmd_session.py`, or inline it — implementer's choice, not behavior-affecting).
4. **`db_help_display.py`**: delete in full (UNK-04). `/session`'s existing "unknown subcommand" fallback message (`_cmd_session`'s final `write_validation_error` call) is extended to mention the new subcommands; no separate help-table method is required unless the implementer wants parity with the old `/db help` table — not required by acceptance criteria.
5. **`registry.py`**: remove `_ConfigSetMixin`... note `_ConfigSetMixin` is composed via `_ConfigMixin` in `cmd_config.py`, not directly in `registry.py` — so the direct edit in `registry.py` is only removing `_DbMixin` from the import list and the `CommandRegistry` base-class tuple, plus updating the module docstring's mixin list (drop the `cmd_db.py` line, since `cmd_rag_export.py`'s docstring entry already omits `/rag`... note the docstring at line 19 is already stale ("cmd_rag_export.py — _RagExportMixin: /export, /compact, /rag") — update it to drop `/export` too while touching this line, since no `_cmd_rag` handler exists).
