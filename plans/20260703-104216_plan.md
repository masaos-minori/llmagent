### Step 2 — Add parametrized per-command tests (if gap exists after Step 1)

**Action:** If Step 1 reveals any acceptance criterion without coverage, add `@pytest.mark.parametrize` tests to `tests/test_command_registry_consistency.py` using `inspect.signature()` and `inspect.iscoroutinefunction()` as specified in the requirement.

Example skeleton (add only if needed):

```python
@pytest.mark.parametrize("cmd", _COMMANDS, ids=lambda c: c.name)
def test_handler_signature_per_command(cmd, registry):
    assert hasattr(registry, cmd.handler)
    fn = getattr(registry, cmd.handler)
    if cmd.is_async:
        assert inspect.iscoroutinefunction(fn)
    else:
        assert not inspect.iscoroutinefunction(fn)
    sig = inspect.signature(fn)
    positional = [
        p for p in sig.parameters.values()
        if p.kind in (
            inspect.Parameter.POSITIONAL_OR_KEYWORD,
            inspect.Parameter.POSITIONAL_ONLY,
        )
    ]
    if cmd.prefix:
        assert len(positional) >= 1, f"{cmd.handler} must accept args param"
    else:
        required = [p for p in positional if p.default is inspect.Parameter.empty]
        assert required == [], f"{cmd.handler} must have no required params"
```

**Completion criterion:** Each `CommandDef` in `_COMMANDS` has an individual parametrized test entry; a deliberate mutation of one `CommandDef` causes exactly that test to fail.
