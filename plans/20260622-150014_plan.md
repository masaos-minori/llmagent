# Plan: SimpleNamespace から dataclass への移行 — test_rag_pipeline_stage.py

## Goal

`tests/test_rag_pipeline_stage.py` の `SimpleNamespace` 使用箇所を、
`RagConfig` Protocol を満たす `@dataclass` (`_RagCfg`) に置き換え、
mypy の型チェックが機能するよう修正する。

## Scope

**In-Scope:**
- `tests/test_rag_pipeline_stage.py` の全 `SimpleNamespace` 使用箇所の置き換え
- `_RagCfg` dataclass の新設
- `_make_rag_cfg()` の戻り型を `_RagCfg` に変更

**Out-of-Scope:**
- `scripts/shared/types.py` の `RagConfig` Protocol 変更
- 実装コードの変更
- 他のテストファイルの変更

## Assumptions

1. `RagConfig` は `scripts/shared/types.py` で `Protocol` として定義されており、21 フィールドを持つ。
   `@dataclass` が同じフィールド名・型を持てば Protocol を構造的に満たす。
2. 現在の `_make_rag_cfg()` は `use_semantic_cache` フィールドが欠落しており、
   mypy では `RagConfig` を満たさない (Protocol 違反)。
3. `MqeStage`・`SearchStage`・`RerankStage` の個別テストでは `SimpleNamespace` を
   `_make_rag_cfg()` + `dataclasses.replace()` に置き換えることで最小限の変更が可能。
4. `_RagCfg` は `frozen=False` (デフォルト) とし、`dataclasses.replace()` による部分更新を許可する。

## Design

### _RagCfg dataclass (RagConfig Protocol の全フィールドを実装)

```python
import dataclasses

@dataclasses.dataclass
class _RagCfg:
    semantic_cache_max_size: int = 0
    semantic_cache_threshold: float = 0.0
    use_semantic_cache: bool = False   # ← Protocol に必要; SimpleNamespace 時は欠落
    use_mqe: bool = False
    top_k_search: int = 5
    use_rerank: bool = False
    rag_top_k: int = 3
    max_chunks_per_doc: int = 5
    top_k_rerank: int = 10
    rag_min_score: float = 0.0
    use_rrf: bool = True
    rrf_k: int = 60
    use_search: bool = True
    rag_service_url: str = ""
    rag_auth_token: str = ""
    use_refiner: bool = False
    refiner_max_tokens: int = 512
    refiner_max_chars_per_chunk: int = 800
    refiner_timeout: float = 30.0
```

### _make_rag_cfg() の更新

```python
def _make_rag_cfg(**overrides) -> _RagCfg:
    return dataclasses.replace(_RagCfg(), **overrides)
```
