# Plan: Remove pinned notes injection from startup prompt setup

## Goal
`StartupOrchestrator._setup_prompt()` がピン留めノートをシステムプロンプトに注入しないことをテストで明示的に保証し、関連コードが将来混入しないようにリグレッション防止を確立する。

## Scope
**In-Scope**:
- `tests/test_startup.py` に `_setup_prompt()` の振る舞いをカバーするテストクラスを追加
  - ピン留めノート (`[Pinned Notes]` ブロック) が注入されないことをアサート
  - メモリレイヤー (`memory.on_session_start()`) のスニペットは注入されることをアサート
  - メモリ無効時はシステムプロンプトがそのまま設定されることをアサート

**Out-of-Scope**:
- `scripts/agent/startup.py` の本体コード変更 (対象コードはすでに存在しない)
- `scripts/agent/session.py` の変更 (`get_pinned_notes()` は存在しない)
- `scripts/agent/config_dataclasses.py` の変更 (`auto_inject_notes` フィールドは存在しない)
- 他モジュールへの変更

## Assumptions
- 調査の結果、`startup.py` の `_setup_prompt()` にピン留めノート注入コードは**現時点で存在しない**
- `AgentSession` に `get_pinned_notes()` メソッドは**現時点で存在しない**
- `ToolConfig` に `auto_inject_notes` フィールドは**現時点で存在しない**
- 要件は「削除後の状態をテストで保証する」フェーズのタスクと解釈する
- `_setup_prompt()` に対するテストが現時点でゼロであるため、リグレッション防止テストの追加が必要
