# Plan: Include common.toml in the main config loading flow

## Goal

Make `common.toml` part of `ConfigLoader.load_all()` so that DB paths, embedding settings, and sqlite-vec path are loaded from a single predictable path, eliminating scattered workaround calls.

## Scope

**In scope:**
- `scripts/shared/config_loader.py` — add `"common.toml"` to `_BASE_CONFIG_FILES`
- `scripts/db/config.py` — update `build_db_config()` docstring; keep current `load("common.toml")` call or replace with `load_all()` extraction
- `scripts/db/maintenance.py` — remove 3 workaround calls (`load("common.toml")`), use shared config
- `scripts/rag/pipeline.py` — remove workaround `load("common.toml", "agent.toml")`, use shared config
- `scripts/rag/llm_client.py` — remove workaround `load("common.toml", "agent.toml")`, use shared config
- `scripts/rag/ingestion/ingester.py` — remove workaround `load("common.toml", "rag_pipeline.toml")`, use shared config
- `scripts/agent/commands/cmd_ingest.py` — remove workaround `load("common.toml", "rag.toml")`, use shared config

**Out of scope:**
- Adding new config keys to `common.toml`
- Changing the structure or schema of `common.toml`
- Refactoring other config file loading patterns (e.g. MCP server TOML files)
- Deployment scripts or init.d configuration

## Assumptions

1. `common.toml` values are stable at startup time and do not change during execution.
2. No module relies on the current behavior where `common.toml` is intentionally excluded from `load_all()` results.
3. Adding `common.toml` to `_BASE_CONFIG_FILES` will not cause key conflicts with other config files loaded in the same call (other files use different key namespaces: `rag.*`, `agent.*`, `mcp.*`, etc.).
4. The embedding URL (`embed_url`) is used by both `db/config.py` (DbConfig) and RAG pipeline modules; consolidating loading will not break either consumer.
