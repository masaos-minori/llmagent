# Plan: Remove backward-compatibility POST /v1/search endpoint from rag-pipeline-mcp

Source: `requires/20260625_71_require.md`

---

## Goal

`rag-pipeline-mcp` の `POST /v1/search` 後方互換エンドポイントを削除し、`POST /v1/call_tool` 経由の canonical MCP API のみを公開する。

---

## Scope

**In-Scope**
- `scripts/mcp/rag_pipeline/server.py`: `@app.post("/v1/search")` の削除（line 115-）
- `scripts/mcp/rag_pipeline/service.py`: `/v1/search` ハンドラコードの削除
- `scripts/mcp/rag_pipeline/models.py`: `RagSearchRequest`, `RagSearchResponse` の削除
- テスト削除・negative テスト追加

**Out-of-Scope**
- 呼び出し元の migration（72 番で対応）
- canonical tool エンドポイントの変更

---

## Assumptions

1. 72 番（呼び出し元の migration）が完了した後に実施する。
2. `RagSearchRequest` / `RagSearchResponse` は `/v1/search` のみで使用（確認済み: models.py line 206-213 の記述から）。
3. 削除後 `/v1/search` が 404 を返すことが期待動作。
