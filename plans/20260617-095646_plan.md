# Plan: Separate incomplete LLM output from normal conversation history (Req 27)

## Goal

Store incomplete LLM outputs in a diagnostic channel separate from normal conversation history, preventing them from polluting future LLM turns while keeping them accessible via debug inspection.

## Scope

**In scope:**
- `scripts/agent/orchestrator.py` — modify `_handle_llm_transport_error()` to skip history injection for incomplete messages
- `scripts/agent/session.py` — add `save_diagnostic()` method for storing diagnostic-only messages
- `scripts/agent/session_message_repo.py` — add support for a `diagnostic` flag in message persistence
- `tests/test_orchestrator.py` — update existing partial completion tests + add new diagnostic channel tests
- `docs/05_agent_03_turn-processing-flow.md` — document the new persistence behavior for incomplete outputs

**Out of scope:**
- Adding a new DB column for diagnostic messages (use existing messages table with a role-based approach)
- Modifying `_handle_synthetic_tool_error()` if it exists with similar behavior (check first)
- UI/REPL changes to display diagnostic messages differently

## Assumptions

1. Incomplete LLM outputs should still be persisted to the DB (for debugging) but NOT added to `ctx.conv.history` (in-memory conversation context).
2. The existing `tool_result_store.store()` call for partial completions is the correct diagnostic channel — no new store needed.
3. Diagnostic messages can use a special role like `"diagnostic"` or be stored with an existing role but tracked separately.
4. `/tool show` and debug commands already access `tool_result_store`, so diagnostic data is already accessible there.

### Step 2: Add `save_diagnostic()` to `session.py`

```python
def save_diagnostic(self, content: str, summary: str = "") -> None:
    """Persist a diagnostic-only message; not included in normal history retrieval."""
    if self.session_id is None:
        return
    self._message_repo.save(
        role=_DIAGNOSTIC_ROLE,
        content=content,
        _diagnostic=True,
    )
```
