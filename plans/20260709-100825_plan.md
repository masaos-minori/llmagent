## Design

Docstring expansion:

```python
def apply_config(self, *, cache_ttl: float | None = None) -> None:
    """Update hot-reloadable configuration fields without recreating the instance.

    Only `cache_ttl` is hot-reloadable here. The following are restart-required
    and are never touched by this method: server configs (`_server_configs`),
    HTTP transports (`_transports`), the route resolver (`_resolver`), server
    add/remove, and concurrency limits/semaphores (`_concurrency_limits`,
    `_semaphores`). MCP server definitions are restart-time snapshots — see
    `ConfigReloadService._classify_mcp_server_changes()`.
    """
    if cache_ttl is not None:
        self._cache_ttl = cache_ttl
```

Test addition:

```python
def test_apply_config_does_not_recreate_transports_or_resolver(self) -> None:
    ex = self._make_executor()
    transports_before = ex._transports
    transport_before = ex._transports["file_read"]
    server_configs_before = ex._server_configs
    resolver_before = ex._resolver

    ex.apply_config(cache_ttl=600.0)

    assert ex._cache_ttl == 600.0
    assert ex._transports is transports_before
    assert ex._transports["file_read"] is transport_before
    assert ex._server_configs is server_configs_before
    assert ex._resolver is resolver_before
```

## Risks

- **Risk**: Since this plan makes no behavior change, there is a temptation
  to skip it as "not real work." Skipping it would leave the "prevent future
  accidental MCP server hot reload implementations" intent unmet — the
  documentation and regression test are the deliverable, not a byproduct.
  → **Mitigation**: Implementation steps 1-2 are both required; the test in
  particular acts as a guardrail against a future accidental change to
  `apply_config()`'s scope, which is the requirement's explicit motivation.
- **Risk**: If `ToolExecutor` gains a new hot-reloadable field later, the
  expanded docstring's explicit "only cache_ttl" claim would need updating
  alongside the code change.
  → **Mitigation**: Low risk — the new identity-preservation test would fail
  immediately if a future change altered `_transports`/`_resolver` without
  updating the test, forcing the docstring to be revisited too.
