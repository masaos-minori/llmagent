## Assumptions

1. `ctx.services.tools.execute()` は例外を送出しない（内部で捕捉して `(error_msg, True, "")` を返す）。`tools is None` ガードのみ必要。
2. `ctx.tool_result_store.store()` は DB エラー時に `sqlite3.Error` (または `sqlite3.OperationalError`) を送出する（docstring に "Raises on DB error" と明記されている）。
3. `StdioTransport.start()` は `ValueError`（working_dir 不存在）と `OSError`（cmd 不存在）を送出しうる。
4. `StdioTransport.stop()` は内部で全例外を吸収し、呼び出し元に伝播させない。`_stop_stdio` の `except Exception` は `OSError` に限定可能。
5. `_TIER_TO_RISK[tier]` は `cfg.approval.tool_safety_tiers.get(tool_name, "WRITE_DANGEROUS")` の結果を受け取るため、現状では `KeyError` は発生しない。ただし未知 tier を渡した場合は `KeyError` が潜在するため、明示的な `Enum` 変換で防御する。
6. `LLMMessage["tool_calls"]` は現在 `list[dict]` として渡されている。`ToolCallRequest` への変換境界は `run_approval_checks()` および `execute_all_tool_calls()` のエントリポイントに置く。

## Design

### 新設ファイル概要

**`agent/tool_enums.py`**:
```python
class RiskLevel(str, Enum):
    NONE = "none"
    MEDIUM = "medium"
    HIGH = "high"

class OperationType(str, Enum):
    WRITE = "write"
    DELETE = "delete"
    EXECUTE = "execute"
    API_WRITE = "api_write"
    READ = "read"

class ApprovalDecisionType(str, Enum):
    AUTO = "auto"
    APPROVED = "approved"
    DENIED = "denied"
    DRY_RUN = "dry_run"

class GuardDecisionType(str, Enum):
    PASS = "pass"
    CYCLE = "cycle"
    DEDUP = "dedup"
    RETRY = "retry"
    ERROR_LIMIT = "error_limit"
```

**`agent/tool_models.py`** (最低限の定義):
```python
@dataclass(frozen=True)
class ToolCallRequest:
    id: str
    name: str
    args: dict[str, Any]  # 境界での一時的な Any 使用; 内部では型付きアクセサ経由

@dataclass(frozen=True)
class ToolMeta:
    resource_scope: str
    requires_serial: bool
    is_write: bool

@dataclass(frozen=True)
class ToolExecutionResult:
    tc_id: str
    name: str
    args: dict[str, Any]
    full_text: str
    is_error: bool
    llm_text: str

@dataclass(frozen=True)
class ApprovalOutcome:
    tool_name: str
    risk_level: RiskLevel
    decision: ApprovalDecisionType
    escalation_reason: str = ""
    preview: str = ""

@dataclass(frozen=True)
class GuardDecision:
    type: GuardDecisionType
    message: str | None = None
```

**`agent/tool_exceptions.py`**:
```python
class ToolArgumentsDecodeError(ValueError): ...
class ToolExecutorUnavailableError(RuntimeError): ...
class PolicyViolationError(RuntimeError):
    def __init__(self, audit_decision: str, message: str) -> None: ...
class ApprovalPreviewError(RuntimeError): ...
class AuditUnavailableError(RuntimeError): ...
class LifecycleConfigurationError(RuntimeError): ...
```

**`agent/tool_output.py`**:
```python
def emit_tool_call(name: str, args_json: str) -> None: ...
def emit_tool_result(name: str, display: str) -> None: ...
def emit_approval_prompt(risk: str, tool_name: str, preview: str) -> None: ...
def emit_denied(tool_name: str, reason: str) -> None: ...
```

### tool_policy.py API 変更

`preflight_deny_reason()` を廃止し、`check_preflight()` に統一:
```python
def check_preflight(cfg: AgentConfig, tool_name: str, args: dict) -> None:
    """Raise PolicyViolationError when pre-flight check fails."""
```

### check_approval() API 変更

```python
async def check_approval(ctx, tool_name, args) -> ApprovalOutcome:
    """Raise on config error; return ApprovalOutcome (approved/denied/dry_run)."""
```

呼び出し側 `run_approval_checks()` はこの戻り値で分岐する。

## Implementation Steps

各ステップは独立コミット可能。前ステップの `pytest` が pass してから次へ進む。

1. **[新設] tool_enums.py / tool_exceptions.py / tool_models.py / tool_output.py を作成**
   - 既存コードへの変更なし; テスト: import できること
   - コミット: `feat(agent): add tool DTO, enum, exception, output foundation files`

2. **[tool_policy.py] Enum化 + preflight_deny_reason 廃止**
   - `classify_risk()` → `RiskLevel` 返却
   - `classify_operation_type()` → `OperationType` 返却
   - `check_preflight()` を新設、`preflight_deny_reason()` を廃止
   - `str(args.get(...))` → 型付きアクセサで代替
   - テスト: `test_tool_policy.py` 全 pass
   - コミット: `refactor(tool_policy): use Enum + check_preflight, remove str(args.get(...))`

3. **[tool_approval.py] DTO化 + dry-run 例外化 + print 除去**
   - `ApprovalDecision` TypedDict → `ApprovalOutcome` dataclass に変更
   - `_build_preview_with_dry_run()` の `except Exception: pass` → `except (OSError, TimeoutError, ValueError)` で `ApprovalPreviewError` を送出
   - `check_approval()` 戻り値 `bool` → `ApprovalOutcome`
   - JSON decode fallback 除去 → `ToolArgumentsDecodeError`
   - `print` → `tool_output.emit_*()`
   - テスト: `test_tool_approval.py` 全 pass
   - コミット: `refactor(tool_approval): ApprovalOutcome DTO, fail-fast dry-run, remove print`

4. **[tool_runner.py] assert除去 + JSON fail-fast + except Exception除去 + print除去**
   - `assert ctx.services.tools is not None` → `ToolExecutorUnavailableError`
   - `args = {}` fallback → `ToolArgumentsDecodeError` で即時失敗
   - `_collect_tool_result_msgs()` の `except Exception` → `except sqlite3.Error`
   - 戻り値 `tuple[str, str, dict, str, bool, str]` → `ToolExecutionResult`
   - `print` → `tool_output.emit_*()`
   - テスト: `test_tool_runner.py` 全 pass
   - コミット: `refactor(tool_runner): ToolExecutionResult DTO, fail-fast JSON, remove print`

5. **[tool_scheduler.py] ToolCallRequest + ToolMeta シグネチャ変更**

   - `tool_runner.py` の呼び出し側も同時に更新
   - テスト: `test_tool_scheduler.py` 全 pass
   - コミット: `refactor(tool_scheduler): typed ToolCallRequest/ToolMeta, fail on missing meta`

6. **[stdio_lifecycle.py] except Exception除去 + cfg.cmd 例外化**

   - `except Exception as e:` → `except (ValueError, OSError) as e:`
   - `_stop_stdio()` の `except Exception as e:` → `except OSError as e:`
   - コミット: `refactor(stdio_lifecycle): LifecycleConfigurationError, specific exception types`

7. **[tool_audit.py] DTO ベース + audit 未設定方針の確定**

8. **[tool_result_formatter.py] str(args.get(...)) 除去 + ToolCallRequest 入力統一**
