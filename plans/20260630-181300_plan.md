# Plan: Delete `NoteRepository`

## Goal
`NoteRepository` および `notes` テーブルに関するすべての残存参照をドキュメントから除去し、コードベースに note 関連の dead reference が存在しない状態にする。

## Scope
**In-Scope**:
- `docs/01_overview-files.md` から `note_repo.py` / `cmd_notes.py` のエントリ削除
- `docs/05_agent_04_state-and-persistence.md` から `notes` テーブル行・`note_repo` インポート記述を削除
- `docs/05_agent_08_configuration.md` から `auto_inject_notes` 設定項目を削除
- `docs/05_agent_09_data-layer.md` から `auto_inject_notes` 参照を削除
- `docs/90_shared_04_db_architecture_and_schema.md` から `notes` テーブルスキーマ定義を削除
- `tests/test_cmd_registry_note_removal.py` の継続保持(削除しない — `/note` コマンド削除を検証するテストとして有効)

**Out-of-Scope**:
- Python ソースコードの変更(`note_repo.py`、`cmd_notes.py`、テストソースはすでに存在しない)
- `scripts/db/schema_sql.py` の変更(`notes` テーブル DDL はすでに存在しない)
- `__pycache__` 内 `.pyc` ファイルの削除(ビルド成果物、git 管理外)
- `session.sqlite` の既存 `notes` テーブルの DROP(運用 DB マイグレーションはスコープ外)
- `test_cmd_registry_note_removal.py` の削除(有効な regression test として保持)

## Assumptions
- `note_repo.py` のソースファイルは既に削除済みであり、`.pyc` のみが `__pycache__` に残存している
- `cmd_notes.py` のソースファイルも同様に削除済み
- `scripts/agent/session.py` はすでに `note_repo` をインポートしていない(実測確認済み)
- `auto_inject_notes` 設定フィールドは `config_dataclasses.py` / `config.py` に存在しない(実測確認済み)
- `notes` テーブルの DDL は `schema_sql.py` に存在しない(実測確認済み)
- `test_cmd_registry_note_removal.py` は `/note` コマンドが登録されていないことを検証する有効なテストである
