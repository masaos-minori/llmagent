# Plan: Update compatibility-remnant checks for Shared/DB schema cleanup

Source: `requires/20260701_62_require.md`

---

## Goal

`scripts/checks/check_no_compat.py` に Shared/DB スキーマクリーンアップで廃止したパターンの検出ルールを追加し、再導入を防ぐ。

---

## Scope

**In-Scope**
- `COMPAT_PATTERNS` に以下の stale 参照パターンを追加:
  - `workflow_schema.py`（ファイル参照）
  - `db.workflow_schema`（モジュール import）
  - `python -m db.workflow_schema`（実行コマンド）
  - `07_ref-sqlite.md`（削除済みドキュメント参照）
  - `07_spec_db.md`（削除済みドキュメント参照）
  - `DOCMISS-01`、`UNDOC-03`、`TYPE-01`、`DESIGN-01`、`UNIMPL-01`（廃止 issue ID）
  - `.read eventbus/schema.sql`（Event Bus 直接 SQL ロード）
  - `eventbus/schema.sql` を schema initialization source として参照
- `DEFAULT_ALLOWLIST` に必要なファイルを追加（このチェッカー自身、plans/、requires/ 等）
- `COMPAT_PATTERNS` に追加しないもの:
  - `retry_count`（意図的な deprecated 保持）
  - `messages.tool_call_id`（意図的な保持・使用）

**Out-of-Scope**
- `check_no_compat.py` のリファクタリング
- 新たな CI フック設定（すでに `.pre-commit-config.yaml` または CI から呼び出されていれば不要）

---

## Assumptions

1. `check_no_compat.py` は `COMPAT_PATTERNS` dict に正規表現を追加するだけで新しいパターンを検出できる。
2. `plans/` ディレクトリはスキャン対象外（`scripts/`, `docs/`, `tests/` のみスキャン）であるため、allowlist 追加不要。
3. `requires/done/` ディレクトリもスキャン対象外であるため、allowlist 追加不要。
4. `workflow_schema.py` パターンは `db.workflow_schema` のインポートを検出する regex で十分カバーできる。
5. 本ファイル自身（`check_no_compat.py`）は `DEFAULT_ALLOWLIST` に含まれているため、追加パターンを記述しても誤検出しない。
