# Plan: Remove dead `mqe_prompt_template` / `rerank_prompt_template` keys from `config/agent.toml`

Source requirement: `requires/20260721_04_require.md` (moved to `requires/done/` after this cycle completes)

## Goal
Remove the two dead top-level keys `mqe_prompt_template` and `rerank_prompt_template` (lines 169-170 of
`config/agent.toml`, under the "System prompts" section) that are loaded by `ConfigLoader.load_all()` but
never consumed by any `AgentConfig` dataclass field or builder function, so that no reader mistakes them
for the effective control of MQE/rerank prompt behavior (which is actually driven by the identically-named
keys in `config/rag_pipeline_mcp_server.toml`).

## Scope

**In scope**:
- `config/agent.toml`: delete the `mqe_prompt_template` and `rerank_prompt_template` lines (and any
  comments referring to them) from the "System prompts" section.
- Any `docs/05_agent_08_*.md` file that documents these two keys as `agent.toml`/`AgentConfig` fields:
  correct or remove those entries.
- Confirm via grep that no other reference to either key name remains in `config/agent.toml` or in agent-
  side config code/docs (excluding `config/rag_pipeline_mcp_server.toml` and its consumers).

**Out of scope**:
- `config/rag_pipeline_mcp_server.toml` and its two identically-named keys — unchanged, remain the
  effective setting.
- `scripts/mcp_servers/rag_pipeline/rag_pipeline_models.py`, `scripts/mcp_servers/rag_pipeline/rag_pipeline_service.py`,
  `scripts/rag/llm_prompts.py` — no code changes, per Non-Goals in the requirement.
- Re-opening `requires/done/20260714_13_require.md`'s already-completed RAG-pipeline-section cleanup.
- Any implementation work: this cycle produces a plan document only.

## Assumptions
1. `config/agent.toml` lines 169-170 currently contain exactly `mqe_prompt_template` and
   `rerank_prompt_template` under a "System prompts" heading, as described in the requirement — to be
   confirmed by direct read in Step 3.
2. `scripts/agent/config_dataclasses.py` and `scripts/agent/config_builders.py` contain no field or
   reference named `mqe_prompt_template` or `rerank_prompt_template` — to be confirmed by grep in Step 3.
3. Pydantic dataclasses used for `AgentConfig` silently ignore unknown/extra keys from the merged config
   dict (per `docs/05_agent_08_01_configuration-loading-agent-config-part1.md`), so removing these two
   keys from `config/agent.toml` has zero effect on any currently-passing runtime behavior or test.
4. No test in `tests/` asserts the presence of these two keys in `config/agent.toml`'s parsed dict (to be
   confirmed by grep in Step 3).
5. TOML deletion is a simple text edit; no schema/migration tooling is involved.

## Design

- **Nature of change**: pure deletion of two dead top-level scalar keys from a TOML config file. No code
  change, no dataclass change, no builder change — this is intentionally a config-only, no-op-at-runtime
  cleanup (Non-Goals explicitly forbid touching the real consumers under `rag_pipeline_mcp_server.toml`/
  `rag_pipeline_models.py`/`llm_prompts.py`).
- **Mechanics**: In `config/agent.toml`, delete lines 169 (`mqe_prompt_template = "..."`) and 170
  (`rerank_prompt_template = "..."`) in full, leaving line 168 (`system_prompt_tool = "..."`) immediately
  followed by the existing blank line 171 and then the `[system_prompts]` table at line 172 — i.e. the
  net effect is exactly two fewer lines, no other lines renumbered in meaning, no change to the blank-line
  spacing convention around the `[system_prompts]` table.
- **Doc alignment**: No `docs/05_agent_08_*.md` edits are required (Step 3 confirmed no such doc lists
  these keys as `agent.toml` fields). The two doc files that do mention these key names
  (`docs/04_mcp_04_03_rag-pipeline-and-cicd.md`, `docs/03_rag_05_1-configuration-reference.md`) already
  correctly attribute them to the RAG pipeline MCP server's own config and require no correction.
- **Verification design**: Since there are zero consumers, verification is grep-based (confirm no
  remaining reference to either key name in `config/agent.toml` after the edit) plus the existing test
  suite (`uv run pytest`) passing unchanged, since no test exercises these two keys in an `agent.toml`
  context.
