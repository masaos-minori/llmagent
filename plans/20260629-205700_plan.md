## Goal
RAG インジェスチョン関連ドキュメント全体で「スクリプト」と「処理フェーズ」の術語を統一し、`stage` の曖昧な使用を排除する。

## Scope
**In-Scope**:
- `docs/03_rag_01_system_overview.md`: 3-script / 4-phase 定義の存在確認; 曖昧な `stage` 使用の検出
- `docs/03_rag_02_ingestion_pipeline.md`: `stage_name` フィールド説明の曖昧さ確認; スクリプト名と処理フェーズ名の一貫性確認
- `docs/03_rag_05_configuration_and_operations.md`: "per script" / "per phase" 記述の一貫性確認
- 上記 3 ファイルで `stage` をインジェスチョン処理フェーズの意味で使っている箇所を `phase` または `processing phase` に修正

**Out-of-Scope**:
- クエリパイプラインの Stage クラス名変更（`MqeStage`, `FusionStage` など）
- `stage_name` DB フィールド名の変更（後方互換性を壊す）
- ランタイム動作の変更

## Assumptions
1. `docs/03_rag_01_system_overview.md:L70-80` に "3 scripts / 4 processing phases" の明示的な定義が存在（確認済み）
2. `03_rag_01:L100` の "Stage" 表は query pipeline 用で正しい使用（変更不要）
3. `03_rag_02:L652` の `stage_name` はインジェスター DB フィールド名 — 説明文 "Stage name string" が曖昧な可能性があり確認が必要
4. `grep -n "\bstage\b"` の結果から、`03_rag_02` でのインジェスチョン文脈での `stage` 使用は L652 のみ
5. `03_rag_05` での `stage` 使用は query pipeline 文脈のみ（インジェスチョン文脈では未検出）
