# Plan: Remove deferred startup_mode reload for MCP servers

Source requirement: `requires/20260708_27_require.md` (H-5)

## Goal

Stop `/reload` from assigning `old_srv.startup_mode = new_srv.startup_mode`
or reporting it as deferred; report `startup_mode` changes (in either
direction) as `mcp/<server_key>.startup_mode` in
`ConfigReloadOutcome.needs_restart`.

## Scope

In scope:
- `scripts/agent/services/config_reload.py` — same shared change as
  H-1/H-2/H-3/H-4 (`plans/20260709-094640_plan.md` and follow-on plans);
  this requirement is the `startup_mode`-specific slice.
- `tests/test_config_reload.py` — add/confirm `startup_mode`-specific
  assertions for both transition directions.

Out of scope:
- `tests/test_lifecycle.py` — verified it constructs `McpServerConfig` with
  `startup_mode` only for subprocess-lifecycle behavior (health polling,
  process management), never for `/reload` classification; no reload logic
  is exercised there.

## Assumptions

1. No test outside `tests/test_config_reload.py`,
   `tests/test_config_reload_classification.py` exercises
   `startup_mode` through `ConfigReloadService` — verified by

## Design

No new production design beyond H-1's `_classify_mcp_server_changes()`
(`plans/20260709-094640_plan.md` § Design). `!=` on the `startup_mode` field
is inherently symmetric, so no direction-specific logic is required — both
`persistent → subprocess` and `subprocess → persistent` already produce
`mcp/<key>.startup_mode` in `needs_restart`.

Test addition (parametrized over both directions):

```python
@pytest.mark.parametrize(
    "old_mode,new_mode",
    [
        (StartupMode.PERSISTENT, StartupMode.SUBPROCESS),
        (StartupMode.SUBPROCESS, StartupMode.PERSISTENT),
    ],
)
def test_startup_mode_change_reports_restart_not_deferred(
    self, old_mode: StartupMode, new_mode: StartupMode
) -> None:
    svc, old_srv = self._make_svc(old_startup=old_mode.value)
    new_srv = McpServerConfig(
        transport=TransportType.HTTP,
        url="http://localhost:8080",
        cmd=[],
        startup_mode=new_mode,
    )
    result = self._run(svc, {"svc": new_srv})

    assert "mcp/svc.startup_mode" in result.needs_restart
    assert not any("startup_mode" in item for item in result.deferred)
    assert not any("startup_mode" in item for item in result.applied)
    assert old_srv.startup_mode == old_mode  # unchanged
```

This replaces `test_startup_mode_change_populates_deferred` in
`tests/test_config_reload.py::TestDeferredReload` (see H-1 plan step 3).

## Risks

- **Risk**: Symmetric `!=` comparison relies on `StartupMode` being a
  well-behaved enum with value equality; if it is ever redefined as a class
  without `__eq__`, detection could silently break.
  → **Mitigation**: `StartupMode` is a standard `Enum` (used elsewhere via
  `.value` comparisons in tests); no custom `__eq__` override exists, so
  default enum identity/equality applies correctly.
- **Risk**: Same disruptive-message risk as H-4 (deferred → restart-required
  is a stronger user-facing message).
  → **Mitigation**: Explicit requirement intent ("Lifecycle behavior cannot
  change mid-process due to reload").
