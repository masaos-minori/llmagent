# Plan: Production startup readiness checks for required services (Req 25)

## Goal

Add production-mode startup readiness checks that fail fast when required services (LLM, Embed) are unavailable, while keeping warning-only behavior for development mode.

## Scope

**In scope:**
- `scripts/agent/repl_health.py` — add `check_readiness()` function that wraps `check_service_health()` and raises in production mode
- `scripts/agent/repl.py` — update `_check_services()` to call `check_readiness()` when security_profile is PRODUCTION
- `scripts/agent/shared/health_models.py` — add `errors` field to `HealthCheckResult` to distinguish warnings from errors
- `tests/test_repl_health.py` — tests for readiness check in production vs development mode
- `docs/05_agent_10_operations-and-observability.md` — document startup readiness policy

**Out of scope:**
- Adding new config keys (use existing `security_profile` enum)
- Modifying watchdog behavior (separate concern)
- Adding per-service criticality configuration (out of scope for this requirement)

## Assumptions

1. "Required services" = LLM endpoint and Embedding service — these are the two probes in `check_service_health()`.
2. Production mode is determined by `security_profile == SecurityProfile.PRODUCTION` (already used in `_check_services()`).
3. When a required service fails, startup should raise `RuntimeError` with a clear message listing which services failed.
4. The existing `HealthCheckResult` structure can be extended with an `errors` field.

### Step 2: Update `check_service_health()` to distinguish warnings from errors

In `repl_health.py`, modify `check_service_health()` to categorize results:
- LLM service failure → error (critical)
- Embed service failure → error (critical)
- Both are critical in production; both generate warnings in development

Actually, keep `check_service_health()` unchanged for backward compatibility. Create a new function `check_readiness()` that calls it and converts warnings to errors based on mode.

### Step 3: Add `check_readiness()` to `repl_health.py`

```python
async def check_readiness(ctx: AgentContext, *, production_mode: bool = False) -> HealthCheckResult:
    """Probe required services at startup; raise in production mode on failure.

    In production mode, any failed health check raises RuntimeError with
    a clear message listing which services are unavailable.

    In development mode, behaves like check_service_health(): warnings only.
    """
    result = await check_service_health(ctx)
    
    if production_mode and result.has_issues:
        # Convert all warnings to errors and raise
        error_msgs = [f"{w.label}: {w.message}" for w in result.warnings]
        msg = f"Startup readiness check failed (required services unavailable): {'; '.join(error_msgs)}"
        logger.error(msg)
        raise RuntimeError(msg)
    
    return result
```

### Step 4: Update `_check_services()` in `repl.py`

Replace the current `_check_service_health()` call with `check_readiness()`:

```python
async def _check_services(self) -> None:
    """Probe LLM / Embed service health, validate tool definitions, and audit security defaults."""
    production_mode = (
        self._ctx.cfg.mcp.security_profile == SecurityProfile.PRODUCTION
    )
    audit_security_defaults(self._ctx, production_mode=production_mode)

    # Readiness check: raises in production mode if critical services are down
    await check_readiness(self._ctx, production_mode=production_mode)

    # Validate tool definitions against live MCP servers (warns or raises)
    await self._check_tool_definitions()
```

Remove the direct call to `self._check_service_health()` — it's now wrapped by `check_readiness()`.
