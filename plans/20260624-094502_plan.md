## Design

### Phase 1: Config key list generation
Read `config/rag_pipeline.toml` + dataclass annotations from `scripts/rag/models_config.py` → emit Markdown table of key / type / default / description.

### Phase 2: CLI argument list generation
Use `argparse` `--help` stdout capture from crawler, chunk_splitter, ingester → emit Markdown code blocks in ops doc.

### Generation boundary convention
```markdown
<!-- AUTO-GENERATED: gen_rag_reference.py config -->
| Key | Type | Default | Description |
...
<!-- END AUTO-GENERATED -->
```
