# Plan: Consolidate workflow schema DDL into a single canonical source

## Goal
`build_workflow_schema_sql()` in `db/schema_sql.py` is the sole workflow DDL source, and both `create_workflow_schema()` and `workflow_schema.init_schema()` apply identical DDL plus the same migration logic so they produce the same schema.

## Scope
**In-Scope**:
- Confirm `db/workflow_schema.py` no longer duplicates DDL (already done — verify and document)
- Extract `_migrate_workflow_schema()` from `db/workflow_schema.py` into a shared location so `create_workflow_schema()` can also call it
- Update `db/create_schema.py::create_workflow_schema()` to call `_migrate_workflow_schema()`
- Update `tests/test_create_schema.py` to verify both paths produce the same tables and apply the same migration

**Out-of-Scope**:
- Changes to `db/schema_sql.py` DDL content itself (DDL is already canonical and correct)
- Changes to rag or session schema paths
- Timestamp format normalization (separate concern per doc §7a policy)

## Assumptions
- `db/workflow_schema.py` already imports and uses `build_workflow_schema_sql()` with no local DDL copy — confirmed by code inspection
- `_migrate_workflow_schema()` (adds `workflow_id` column to `tasks` if missing) is the only migration that needs to be shared between paths
- `SQLiteHelper("workflow").open(write_mode=True)` provides `db.conn` reliably — confirmed by existing `init_schema()` code
- `_make_db_config` in `test_create_schema.py` does not support `workflow` target; tests use `DbConfig` constructor directly — confirmed by inspection

## Design

**Sharing `_migrate_workflow_schema`**

Two options:

- Option A: Move `_migrate_workflow_schema()` to `db/create_schema.py`, import it in `db/workflow_schema.py`. Keeps migration logic collocated with other `_migrate_*` functions.
- Option B: Duplicate the function in both modules. Simpler but defeats the consolidation goal.

Chosen: Option A. `db/create_schema.py` already owns `_migrate_rag_schema()` and `_migrate_session_schema()`. Adding `_migrate_workflow_schema()` there is consistent. `db/workflow_schema.py` imports it from `db.create_schema`.

**`create_workflow_schema()` change**

```python
def create_workflow_schema() -> None:
    with SQLiteHelper("workflow").open(write_mode=True) as db:
        try:
            db.executescript(build_workflow_schema_sql())
        except (sqlite3.OperationalError, sqlite3.DatabaseError) as e:
            logger.error("Failed to execute workflow schema DDL: %s", e)
            raise
        _migrate_workflow_schema(db.conn)  # type: ignore[arg-type]
    logger.info("Workflow schema created successfully.")
```

**`workflow_schema.init_schema()` change**

Remove the local `_migrate_workflow_schema` definition, import from `db.create_schema`:

```python
from db.create_schema import _migrate_workflow_schema
```
