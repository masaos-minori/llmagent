# Plan: Guard Stale ETag/Last-Modified Overwrite in Ingester Skip Path

**Requirement:** `requires/20260622_84_require.md`

---

## Goal

Prevent stale chunk-file metadata from silently overwriting newer ETag/Last-Modified values already stored in `documents`. The guard for timestamp-comparable cases already exists (`fetched_at` comparison); the gap is the no-timestamp path where `new_fetched_at is None` — currently it unconditionally writes `etag = ?` and `last_modified = ?`.

---

## Current State

`_update_etag()` in `scripts/rag/ingestion/ingester.py:260-294` has a stale guard for the timestamp-comparable case:

```python
if new_fetched_at is not None:
    rows = db.fetchall("SELECT fetched_at FROM documents WHERE doc_id = ?", (doc_id,))
    stored_fetched_at = rows[0][0] if rows else None
    if stored_fetched_at and new_fetched_at < stored_fetched_at:
        return  # stale — skip
db.execute(
    "UPDATE documents SET etag = ?, last_modified = ?,"
    " fetched_at = COALESCE(?, fetched_at) WHERE doc_id = ?",
    (etag, last_modified, new_fetched_at, doc_id),
)
```

The three bypass cases:

| Scenario | Guard check | Current outcome | Correct? |
|---|---|---|---|
| `new_fetched_at` ≥ `stored_fetched_at` | guard passes | overwrite | Yes |
| `new_fetched_at` < `stored_fetched_at` | guard fails | skip | Yes |
| `new_fetched_at is None`, stored exists | no guard | overwrite | **No — gap** |
| `stored_fetched_at is None`, incoming exists | `if stored_fetched_at` false | overwrite | Yes (filling missing data) |

**The gap (scenario 3):** When `new_fetched_at is None` — legacy chunk files without a fetch timestamp — the code writes `etag = ?` unconditionally, which can overwrite a fresher ETag already stored from a more recent crawl.

**Existing test:** `test_missing_fetched_at_always_updates` in `tests/test_ingester_etag_guard.py:59-68` explicitly tests and asserts the current (incorrect) bypass behavior. This test must be updated.

## Assumptions

1. `new_fetched_at is None` means the chunk JSON predates the `fetched_at` field — it carries no freshness signal. `COALESCE(etag, ?)` correctly expresses "fill-in if missing, never overwrite existing."
2. `stored_fetched_at is None` means the DB record predates the `fetched_at` field — we have no stored freshness baseline. Allowing the incoming write (whatever timestamp it carries) is correct: it promotes the record to the new schema semantics.
3. ISO-8601 strings (`2026-06-01T10:00:00`) sort correctly as plain strings — the existing `<` comparison is correct.
4. The unit tests use `MagicMock` for the DB layer — the SQL string passed to `db.execute()` is directly inspectable in assertions.

## Design

### Change 1: `ingester.py` — split SQL on freshness signal presence

**Before (lines 288-293):**
```python
db.execute(
    "UPDATE documents SET etag = ?, last_modified = ?,"
    " fetched_at = COALESCE(?, fetched_at) WHERE doc_id = ?",
    (etag, last_modified, new_fetched_at, doc_id),
)
```

**After:**
```python
if new_fetched_at is not None:
    # Freshness proven by guard above: overwrite is safe.
    db.execute(
        "UPDATE documents SET etag = ?, last_modified = ?,"
        " fetched_at = COALESCE(?, fetched_at) WHERE doc_id = ?",
        (etag, last_modified, new_fetched_at, doc_id),
    )
else:
    # No freshness signal — fill NULL only; never overwrite existing values.
    db.execute(
        "UPDATE documents SET etag = COALESCE(etag, ?), last_modified = COALESCE(last_modified, ?)"
        " WHERE doc_id = ?",
        (etag, last_modified, doc_id),
    )
```

Logic table after fix:

| `new_fetched_at` | `stored_fetched_at` | `new < stored` | Action |
|---|---|---|---|
| provided | provided | True | skip (stale) |
| provided | provided | False | overwrite (proven newer) |
| provided | None | — | overwrite (fills missing stored timestamp) |
| None | any | — | COALESCE fill-only (no proof of freshness) |

### Change 2: `tests/test_ingester_etag_guard.py`

**Update** `test_missing_fetched_at_always_updates` → verify COALESCE SQL is used:
```python
def test_missing_fetched_at_uses_coalesce_fill_only(self) -> None:
    ingester = _make_ingester()
    db = _make_db("2026-06-10T10:00:00")

    ingester._update_etag(db, 42, "etag-x", "Mon, 01 Jun 2026", None)

    db.execute.assert_called_once()
    db.commit.assert_called_once()
    db.fetchall.assert_not_called()
    sql = db.execute.call_args[0][0]
    assert "COALESCE(etag, ?)" in sql
    assert "COALESCE(last_modified, ?)" in sql
```

**Add** `test_missing_fetched_at_does_not_overwrite_existing_etag`:
```python
def test_missing_fetched_at_does_not_overwrite_existing_etag(self) -> None:
    # Confirm SQL semantics: even if "etag-x" is passed, existing DB etag wins via COALESCE.
    ingester = _make_ingester()
    db = _make_db("2026-06-10T10:00:00")  # stored has timestamp

    ingester._update_etag(db, 42, "etag-stale", "Mon, 01 Jun 2026", None)

    sql = db.execute.call_args[0][0]
    assert "COALESCE(etag, ?)" in sql, "Must use fill-only SQL when new_fetched_at is None"
```

**Add** `test_missing_fetched_at_fills_null_etag`:
```python
def test_missing_fetched_at_fills_null_etag(self) -> None:
    # When DB etag is NULL (first seen), COALESCE(etag, ?) writes the incoming value.
    ingester = _make_ingester()
    db = _make_db(None)  # stored has no timestamp

    ingester._update_etag(db, 42, "etag-first", "Mon, 01 Jun 2026", None)

    db.execute.assert_called_once()
    sql = db.execute.call_args[0][0]
    assert "COALESCE(etag, ?)" in sql
```

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| Callers that pass `new_fetched_at=None` expecting a full overwrite (e.g., to correct a wrong ETag) will now silently do nothing if DB already has a value | Low | Correct semantics: without freshness signal we cannot determine which value is right — COALESCE is the safe choice. Operators who need forced overwrite can use `--force` to delete and re-ingest. |
| `COALESCE(etag, ?)` in SQLite returns first non-NULL of `{etag, ?}` — if existing etag is empty string (not NULL), it wins. | Very low | ETag values are HTTP opaque strings or SHA-256 hex; empty string is not a valid value in practice. |
