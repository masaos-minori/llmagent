## Goal

Implement true incremental refresh_index behavior for MDQ by tracking file metadata (mtime_ns, size_bytes, content_hash) and only re-indexing changed files.

## Scope

**In-Scope**:
- Track file metadata: mtime_ns, size_bytes, content_hash
- Skip unchanged files based on metadata comparison
- Re-index changed files only
- Remove chunks for deleted files
- Add force option for full re-index
- Return structured refresh summary (indexed count, skipped count, deleted count, failed count, elapsed time)

**Out-of-Scope**:
- Adding new tools or features
- Changes to other MCP servers' indexing behavior

## Assumptions

1. index_state table already exists in service.py for tracking file metadata
2. Force option should bypass all skip logic and re-index everything
3. Deleted files should be detected by comparing current filesystem state with index_state records

## Affected Areas & Tool Evidence

- **Affected Files**:

  - `scripts/mcp/mdq/service.py` — update refresh_index to use incremental indexer, add force option
  - `scripts/mcp/mdq/models.py` — update RefreshIndexRequest and RefreshIndexResponse models

- **Blast Radius**: Medium — changes indexing behavior. Existing queries will still work; indexing operations will be faster for unchanged files.

## Implementation Steps

1. **Phase 1: Update models.py**
   - RefreshIndexRequest: add `force: bool = False` field
   - RefreshIndexResponse: replace message with structured summary (indexed_count, skipped_count, deleted_count, failed_count, elapsed_seconds)

2. **Phase 2: Add incremental indexing logic in indexer.py**
   - Load index_state records for current index_paths
   - Compare mtime_ns/content_hash for each file
   - Skip files where metadata matches (unchanged)
   - Re-index files where metadata differs (changed)
   - Detect deleted files by comparing index_state paths with current filesystem

3. **Phase 3: Update service.py refresh_index**
   - Pass force option to indexer
   - Collect and return structured summary from indexer

4. **Phase 4: Add force option support**
   - When force=True, skip all metadata comparison and re-index everything
   - Clear index_state before full re-index
