# Plan: Simplify plugin example score filter to avoid unreachable code

## Goal

Replace the unreachable-return pattern in `plugins/example.py` with an explicit `ENABLE_SCORE_FILTER` flag so the template is easier to read, copy, and modify safely.

---

## Scope

### In-Scope

- Modify `plugins/example.py` only
- Introduce `ENABLE_SCORE_FILTER = False` module-level flag
- Replace the dead-code `return hits` / unreachable second return with a conditional branch
- Remove the `# noqa: F821` suppression that exists solely to silence the unreachable-code lint error

### Out-of-Scope

- Plugin hook behavior (`register_pipeline_stage` logic)
- Plugin registry implementation (`shared/plugin_registry.py`)
- Production plugin loading or deploy process beyond what already exists
- Any other plugin files

---

## Assumptions

1. The `# noqa: F821` suppressor on line 47 of `plugins/example.py` was added only to suppress the lint error caused by the unreachable second `return` statement; removing it once the dead code is gone will not cause new lint failures.
2. No other file in the codebase imports from `plugins/example.py` directly or depends on the current `stage_score_filter` body shape.
3. `deploy/deploy.sh` copies `plugins/*.py` with `cp -n` (no-clobber), so an already-deployed `example.py` in `/opt/llm/plugins/` will not be overwritten automatically; manual copy or re-deploy is needed to propagate the change to production.
4. The development venv is managed by `uv` and `uv run python -m agent` is sufficient to verify plugin load.

## Implementation Steps

### Step 1 — Add explicit feature flag and rewrite `stage_score_filter`

**File:** `plugins/example.py`

**Change:**

1. After the `from shared.plugin_registry import ...` block, add:
   ```python
   # ── Feature flags ─────────────────────────────────────────────────────────────
   # Set to True to activate post-rerank score filtering.
   ENABLE_SCORE_FILTER = False
   ```

2. Replace the current `stage_score_filter` body:
   ```python
   # Before
   return hits
   return [h for h in hits if h.get("score", 1.0) >= 0.05]  # noqa: F821

   # After
   if not ENABLE_SCORE_FILTER:
       return hits
   return [h for h in hits if h.get("score", 1.0) >= 0.05]
   ```

3. Update the docstring to reflect the flag:
   ```
   Drop hits with score below 0.05.

   Disabled by default (ENABLE_SCORE_FILTER = False).
   Set ENABLE_SCORE_FILTER = True to activate.
   ```

4. Remove the inline comment `# Remove the next line to enable filtering`.

**Completion criteria:**
- `plugins/example.py` has no second `return` that is unreachable
- `ENABLE_SCORE_FILTER = False` is present at module level
- No `# noqa: F821` suppressor remains
- `uv run ruff check plugins/example.py` reports no errors
