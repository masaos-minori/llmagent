# Plan: Disallow GitHub MCP allowed_repos_mode="fail_open" in production

Source requirement: `requires/20260709_01_require.md`

## Goal

Make `security_profile="production"` startup fail fast (`RuntimeError`) when
GitHub MCP is configured with `allowed_repos_mode="fail_open"`, while
keeping `fail_open` fully functional in local/development mode (warning
only) and unchanged at the runtime-enforcement layer (`_assert_allowed_repo`).

## Scope

In scope:
- `scripts/agent/security_audit_config.py`: add `allowed_repos_mode: str` to
  `GitHubAuditConfig`; populate it in `load_github_audit_config()`.
- `scripts/agent/repl_health.py`: in `audit_security_defaults()`, add a
  production fail-fast check next to the existing `allowed_repos` empty-list
  check (both already load `github_cfg` via `load_github_audit_config()`).
- `config/github_mcp_server.toml`: replace the misleading
  `"backward compat default"` comment with the wording specified in the
  requirement.
- `docs/04_mcp_05_security_and_safety_model.md`: state that `fail_open` is
  prohibited in production.
- `docs/04_mcp_06_configuration_and_operations.md`: add a Pre-Production
  checklist item requiring `allowed_repos_mode="fail_closed"`.
- `tests/test_repl_health.py`: add the production/local audit tests (not
  `tests/test_github_mcp_service.py` — see Assumption 3).
- `tests/test_security_audit_config_isolation.py` and
  `tests/test_audit_security_failures.py`: update the 2 other direct
  `GitHubAuditConfig(...)` construction sites so they don't break once the
  new field is added (see Assumption 6 — found via the Risks section's own
  mitigation grep, not in the requirement's original Target Files list).

Out of scope (per requirement's Non-Goals):
- Removing `fail_open` entirely, or changing the default (`fail_closed`
  already is the default in both `GitHubConfig` and
  `config/github_mcp_server.toml`).
- `scripts/mcp/github/service_security.py` (`_assert_allowed_repo`'s
  runtime fail_open/fail_closed logic) — already correct and must stay
  unchanged.
- `scripts/mcp/github/models_config.py` (`GitHubConfig.allowed_repos_mode`)
  — already exists with the correct default; no change needed there.

## Design

**`security_audit_config.py`** — extend the DTO and loader:

```python
@dataclass(frozen=True)
class GitHubAuditConfig:
    allowed_repos: list[str]
    allowed_repos_mode: str
    allow_force_push: bool
    require_pr_review: bool
```

```python
def load_github_audit_config() -> GitHubAuditConfig | None:
    ...
    cfg = GitHubConfig.load()
    return GitHubAuditConfig(
        allowed_repos=list(cfg.allowed_repos),
        allowed_repos_mode=cfg.allowed_repos_mode,
        allow_force_push=cfg.allow_force_push,
        require_pr_review=cfg.require_pr_review,
    )
```

**`repl_health.py`** — insert immediately after the existing
`fail_closed_empty.append("github.allowed_repos")` block (after line 804),
reusing `github_cfg`:

```python
    if github_cfg is not None and github_cfg.allowed_repos_mode == "fail_open":
        msg = (
            "github.allowed_repos_mode='fail_open' is not permitted in production mode. "
            "Set allowed_repos_mode='fail_closed' in github_mcp_server.toml."
        )
        if production_mode:
            raise RuntimeError(msg)
        logger.warning("Security: %s", msg)
        warnings.append(f"Security: {msg}")
```

This mirrors the existing `shell_sandbox_backend == "none"` pattern
(raise in production, warn otherwise) already used earlier in the same
function (lines 732-737).

**`config/github_mcp_server.toml`** — replace lines 14-15's comment:

```toml
# allowed_repos_mode: behaviour when allowed_repos is empty
# "fail_open" = allow all when allowed_repos is empty; retained for backward compatibility.
# "fail_closed" = deny all when allowed_repos is empty; default and recommended.
allowed_repos_mode = "fail_closed"
```

**`docs/04_mcp_05_security_and_safety_model.md`** — add one sentence after
the existing mode table (after line 64):

```markdown
`allowed_repos_mode="fail_open"` is prohibited in production
(`security_profile="production"`) — startup raises `RuntimeError`. It
remains available in local/development mode for backward compatibility
(startup emits a warning instead).
```

**`docs/04_mcp_06_configuration_and_operations.md`** — add a checklist item

line, around line 839):

```markdown
- [ ] `allowed_repos_mode = "fail_closed"` in `github_mcp_server.toml` (`"fail_open"` is rejected at production startup)
```

7. Add 3 tests to `tests/test_repl_health.py::TestAuditSecurityDefaults`:

```python
def test_github_fail_open_raises_in_production(self) -> None:
    ctx = self._make_ctx(
        servers={"svc": {"auth_token": "tok"}}, security_profile="production"
    )
    gh_cfg = GitHubAuditConfig(
        allowed_repos=[], allowed_repos_mode="fail_open",
        allow_force_push=False, require_pr_review=True,
    )
    with (
        patch("agent.repl_health.load_shell_audit_config", return_value=None),
        patch("agent.repl_health.load_git_audit_config", return_value=None),
        patch("agent.repl_health.load_github_audit_config", return_value=gh_cfg),
        patch("agent.repl_health.load_cicd_audit_config", return_value=None),
    ):
        with pytest.raises(RuntimeError, match="fail_open"):
            audit_security_defaults(ctx, production_mode=True)

def test_github_fail_open_warns_in_local(self) -> None:
    ctx = self._make_ctx(servers={"svc": {"auth_token": "tok"}})
    gh_cfg = GitHubAuditConfig(
        allowed_repos=[], allowed_repos_mode="fail_open",
        allow_force_push=False, require_pr_review=True,
    )
    with (
        patch("agent.repl_health.load_shell_audit_config", return_value=None),
        patch("agent.repl_health.load_git_audit_config", return_value=None),
        patch("agent.repl_health.load_github_audit_config", return_value=gh_cfg),
        patch("agent.repl_health.load_cicd_audit_config", return_value=None),
    ):
        warnings = audit_security_defaults(ctx, production_mode=False)
    assert any("fail_open" in w for w in warnings)

def test_github_fail_closed_no_error_in_production(self) -> None:
    ctx = self._make_ctx(
        servers={"svc": {"auth_token": "tok"}}, security_profile="production"
    )
    gh_cfg = GitHubAuditConfig(
        allowed_repos=["owner/repo"], allowed_repos_mode="fail_closed",
        allow_force_push=False, require_pr_review=True,
    )
    with (
        patch("agent.repl_health.load_shell_audit_config", return_value=None),
        patch("agent.repl_health.load_git_audit_config", return_value=None),
        patch("agent.repl_health.load_github_audit_config", return_value=gh_cfg),
        patch("agent.repl_health.load_cicd_audit_config", return_value=None),
    ):
        audit_security_defaults(ctx, production_mode=True)  # must not raise
```
   Note: `_make_ctx(security_profile="production")` alone does not set
   `production_mode=True` for `audit_security_defaults()` — that is a
   separate function parameter; pass it explicitly as shown above (verify
   the exact relationship between `_make_ctx`'s `security_profile` param and
   `production_mode` against existing tests in the same class during
   implementation).
8. Run the full validation sequence (`rules/toolchain.md`) and fix any findings.

## Risks

- **Risk**: `audit_security_defaults()` already loads `load_github_audit_config()`
  twice for unrelated purposes (lines 786 and 840); a reviewer might expect
  the new check to live near the second call site instead.
  → **Mitigation**: The first call site (`github_cfg`) is topically about
  `allowed_repos`/`allowed_repos_mode` fail-open/closed behavior, matching
  this check's subject exactly; the second call site is explicitly
  documented as "warning only — not a production-mode hard error," which
  would contradict this check's fail-fast requirement if placed there.
- **Risk**: `GitHubAuditConfig` is a `frozen=True` dataclass; adding a new
  required field changes its constructor signature, breaking any caller
  that constructs it directly (not just via `load_github_audit_config()`).
  → **Mitigation**: Already resolved during planning —
  `grep -rn "GitHubAuditConfig(" scripts/ tests/` found 5 construction
  sites across 4 files (the loader itself, plus 3 test files —
  `test_repl_health.py` has two); all 3 test files are now in scope
  (Assumption 6, Implementation steps 6-7). `tests/test_github_mcp_service.py`
  was confirmed unaffected — it configures `GitHubConfig` via dicts, not
  `GitHubAuditConfig` directly.
