# Plan: Stabilize `SQLiteHelper.open(reuse_connection=True)` lifecycle behavior

## Goal
`SQLiteHelper.open()` の reuse/non-reuse ライフサイクル挙動を明確化し、テストが環境依存なく全通過する状態にする。

## Scope
**In-Scope**:
- `scripts/db/helper.py`: `__exit__` の `getattr` フォールバックを `self._reuse_connection` 直接参照に簡素化
- `tests/test_db_performance.py`: `SQLiteHelper` のインスタンス化を `patch` スコープ内に移動し、vec 拡張ロードエラーを解消する

**Out-of-Scope**:
- `__init__` / `open()` の `_reuse_connection` 初期化ロジック（既に要件通り実装済み）
- DB スキーマ変更
- 他のテストファイルへの変更

## Assumptions
- `SQLiteHelper.__init__` では既に `self._reuse_connection = False` が初期化されている (line 54)
- `open()` では既に `self._reuse_connection = reuse_connection` が設定されており、`False` の場合も明示的に代入される (line 135)
- テスト失敗の根因は patch スコープが `__init__` を含んでいないこと — `helper = SQLiteHelper("rag")` が patch 外で実行されるため `self._vec_so` に実際のパスが設定される
- `_make_cfg` が返す `DbConfig` は `sqlite_vec_so=""` (デフォルト) であるため、`__init__` が patch スコープ内で呼ばれれば vec ロードをスキップできる
- `__exit__` の `getattr(self, "_reuse_connection", False)` フォールバックは `__init__` で常に初期化される今となっては冗長だが、動作に影響はない
