## Goal

Extend `check_no_compat.py` with patterns for compatibility-only stubs and shims introduced in plan 52–55 targets (`agent/config.py`, `McpServerConfig._cast_enums()`, `mcp/github/service.py`, `mcp/github/__init__.py`), so that reintroducing these patterns is detected by CI and local validation.

## Scope

**In-Scope**:
- Add grep patterns to `COMPAT_PATTERNS` in `scripts/checks/check_no_compat.py` covering:
  - `backward-compatible` (keyword in docstrings/comments)
  - `re-export stub` (phrase in module docstrings)
  - `compatibility shim` (phrase in function docstrings)
  - `existing imports continue to work` (phrase in module docstrings)
  - `_cast_enums` (method name)
- Add allowlist entries for files that legitimately reference these phrases as historical notes.
- Wire `check_no_compat.py` into `.pre-commit-config.yaml` (currently missing).
- Add a pytest test to `tests/` that runs `check_no_compat.main()` and verifies no issues are found on the current codebase.

**Out-of-Scope**:
- Building a complex static analyzer or AST-based checker.
- Changing the pattern for other existing stubs not listed in the requirement.
- Modifying CI workflow `backward-compat-check.yml` (already wired correctly).

## Assumptions

1. `scripts/checks/check_no_compat.py` is the canonical location for backward compatibility guards; new patterns go there.
2. The four compatibility targets (`agent/config.py`, `McpServerConfig._cast_enums()`, `mcp/github/service.py`, `mcp/github/__init__.py`) will be removed before or alongside this guard being activated; until removal they must be allowlisted.
3. Historical documentation that mentions these patterns as past-tense notes (e.g., `docs/04_mcp_00_document-guide.md`) is acceptable and must be allowlisted.
4. `DEFAULT_ALLOWLIST` is the correct mechanism for per-file exemptions; no external allowlist file is required.
5. `.pre-commit-config.yaml` currently lacks a hook for `check_no_compat.py`; adding one is in scope.
