## Design

### Change Strategy

1. **Count** active MCP servers from `config/agent.toml` — enumerate each `mcp_servers.<key>` entry.
2. **Count** MCP tools from `shared/tool_constants.py` — enumerate all tool names.
3. **Update** all documentation to use the same counts consistently.
4. **Remove** stale server entries if a server no longer exists in config.
5. **Add** missing server catalog entries if an active server is omitted from docs.
6. **Clarify** that startup-mode values are deployment/configured values, not universal defaults.
7. **Document** mode definitions clearly.
8. **Update** catalog startup modes to match current `config/agent.toml`.

### Mode Definitions

```
subprocess: Agent-managed HTTP server. Agent starts it and may restart it.
persistent: Externally managed HTTP server. Agent connects and probes it.
none:       Disabled. Tool calls are rejected before network access.
```

### Consistency Rules

- After changes, every mention of MCP server count must use the same number.
- Every mention of MCP tool count must use the same number.
- Startup mode values must be labeled as "configured value" not "default".
- `startup_mode="none"` must be documented as disabled.
