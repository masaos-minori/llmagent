# Work Plan: Agent Documentation Restructuring (06)

## Goal

Reorganize 13 existing agent documentation files into a 13-file top-down reading structure that separates overview, execution flow, state, communication, tools, commands, configuration, operations, and API reference -- while preserving all information and explicitly surfacing conflicts between old and current specifications.

## Scope

**In scope:**
- Input files: `docs/05_agent.md`, `docs/05_agent-impl-class.md`, `docs/05_agent-impl-flow.md`, `docs/05_agent-ops.md`, `docs/05_ref-agent-config.md`, `docs/05_ref-agent-context.md`, `docs/05_ref-agent-history.md`, `docs/05_ref-agent-llm.md`, `docs/05_ref-agent-repl.md`, `docs/05_ref-agent-session.md`, `docs/05_ref-agent-view.md`, `docs/05_ref-agent-commands.md`, `docs/05_spec_agent.md`
- Output: 13 new files under `docs/05_agent_XX_*.md` naming convention
- Update `00_llm-implementation-guide.md` and `routing.md` with new file references

**Out of scope:**
- MCP documentation (`docs/04_*`) -- only cross-referenced, not reorganized
- RAG documentation (`docs/03_*`) -- only cross-referenced, not reorganized
- Database documentation (`docs/07_*`) -- only cross-referenced, not reorganized
- Shared infrastructure documentation (`docs/06_ref-infra.md`, `docs/06_ref-mcp.md`) -- only cross-referenced
- Source code changes -- this is a documentation-only task
- `docs/05_spec_agent.md` -- kept as-is (spec file, separate domain)

## Assumptions

1. `05_agent-impl.md` and `05_ref-agent.md` are intentionally deleted (merged in previous commit ee933d7); their content is already absorbed into `05_agent-impl-class.md`
2. `06_ref-*` files are the canonical source per requirement Section Critical Rules 2 -- but these do not exist for agent docs (only `05_ref-*` exist)
3. The requirement's output file names (`05_agent_00_*` through `05_agent_12_*`) should be used literally
4. Each output file should follow the chapter template (Section Required Chapter Template) where sections are applicable
5. Cross-references between agent docs and other doc domains (MCP, RAG, DB) should use existing link conventions

## Implementation Steps

1. **Create output directory structure**
   - Create `docs/05_agent_*.md` files (13 total) using the naming convention from the requirement
   - Each file starts with the chapter template header

2. **Create Section 00 document-guide**
   - Write purpose, audience, reading order instructions
   - Document canonical source rules (ref-* > impl-* > overview)
   - Define glossary of abbreviations (SSE, REPL, MCP, RAG, OTel, FTS, RRF, MQE, etc.)
   - Document how Spec Conflict and Open Question annotations work

3. **Create Section 01 system-overview**
   - Extract agent overview from `05_agent.md` Sections 1-2
   - Add entry point, interaction model, tool-calling model summary
   - Include high-level sessions/SSE/compression explanation
   - List out-of-scope areas

4. **Create Section 02 runtime-architecture**
   - Create text diagram of component relationships
   - Extract component responsibilities from `05_agent-impl-class.md` Sections 1.2/1.7 and ref files
   - Document dependency relationships

5. **Create Section 03 turn-processing-flow**
   - Convert `05_agent-impl-flow.md` Section 1 flow diagram into structured steps
   - Include memory injection, append, compress, LLM, tool loop
   - Add error handling, partial completion, state transitions per stage

6. **Create Section 04 state-and-persistence**
   - Extract AgentContext state model from `05_ref-agent-context.md`
   - Document session persistence from `05_ref-agent-session.md`
   - Document HistoryManager compression and protected turns
   - Add data classification (session-scoped/turn-scoped/persisted)

7. **Create Section 05 llm-and-streaming**
   - Convert `05_ref-agent-llm.md` into chapter template format
   - Include LLMTransportError, SSE parser, reconnect behavior

8. **Create Section 06 tool-execution-and-approval**
   - Extract from `05_agent-impl-flow.md` Section 3.4 and `05_ref-agent-repl.md`
   - Cover transport, routing, parallel/sequential, approval flow, plan mode
   - Include safety controls (allowed_tools, allowed_root, GitHub repos)

9. **Create Section 07 cli-and-commands**
   - Consolidate command lists from `05_agent.md` Section 3 and `05_agent-impl-class.md` Section 1.3
   - Categorize into 11 groups per requirement specification
   - Include CLIView role, multiline input

10. **Create Section 08 configuration**
    - Extract from `05_ref-agent-config.md` (full config spec) and `05_agent.md` Section 5
    - Document all config dataclasses (LLMConfig, RAGConfig, ToolConfig, etc.)
    - Add reload scope and restart-required distinction

11. **Create Section 09 data-layer**
    - Extract from `05_ref-agent-session.md`, `05_spec_agent.md`
    - Document SQLite structures and ownership boundaries

12. **Create Section 10 operations-and-observability**
    - Convert `05_agent-ops.md` into chapter template format
    - Include startup, verification, troubleshooting, OTel, /context, /stats

13. **Create Section 11 extension-points**
    - Extract from `05_spec_agent.md` plugin sections
    - Document register_command/register_tool/register_pipeline_stage

14. **Create Section 12 reference-api**
    - Condense all ref-* files into per-module summary tables
    - Include role, main classes/functions, public APIs, callers, callees, config, commands, failure behavior

15. **Update cross-references**
    - Update `00_llm-implementation-guide.md` with new file entries
    - Update `routing.md` agent task entries
    - Fix internal cross-refs between new files

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| Information loss during consolidation | Medium | Preserve all source content; use "see also" references instead of deleting; run line-count comparison after each file creation |
| Spec Conflict between old impl-* docs and ref-* docs | High (existing docs warn about mixed old/new specs) | Create explicit Spec Conflict sections; prefer ref-* as canonical per requirement rules; flag items requiring manual review |
| Over-fragmentation -- 13 files still too many | Low | Each file has a single responsibility per requirement Section Critical Rules 3; follow the exact output structure specified |
| Cross-reference breakage during restructuring | Medium | Fix all refs in a dedicated step (Step 15); validate with automated script before committing |
| `06_ref-agent-config.md` naming confusion | Low -- already resolved | Use `05_ref-agent-config.md` consistently; the `06_` prefix is a stale reference in existing docs that should be updated |
