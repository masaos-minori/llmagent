# Plan: 並行テストのスレッド数削減 — test_semantic_cache_concurrency.py

## Goal

`tests/test_semantic_cache_concurrency.py` の `_THREADS = 50` を `_THREADS = 5` に削減し、
テストの安定性と再現性を向上させる。

## Scope

**In-Scope:**
- `tests/test_semantic_cache_concurrency.py` — `_THREADS` 定数の変更 (1 行)

**Out-of-Scope:**
- テストロジックの変更
- 実装コードの変更

## Assumptions

1. `_THREADS = 50` は 3 テスト (`test_concurrent_lookups_no_crash`,
   `test_concurrent_puts_no_data_loss`, `test_concurrent_put_and_lookup_consistent`) で使用される。
2. `test_concurrent_put_and_lookup_consistent` は `_THREADS * 2 = 100` スレッドを生成する。
   `_THREADS = 5` に変更すると 10 スレッドになる。
3. GIL による Python スレッドの実質的シリアライズにより、スレッド数が少なくても
   スレッドセーフティのバグを検出できる。
4. 50 スレッドは CI 環境によっては OS スレッド数制限・メモリ制限に当たる可能性がある。
