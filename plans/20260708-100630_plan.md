# 作業計画書

## Goal

`TURN_LIMIT_HINT` から `/tool show <id>` 参照と `result_id` 埋め込みを除去する。
`ToolResultStore` および `/tool show` コマンドが削除される前提として、per-turn ツール結果制限の
オーバーフロー通知メッセージをリトリーバルフリーな文言に置き換える。

## Scope

- `scripts/agent/tool_result_formatter.py`: `TURN_LIMIT_HINT` 定数を新文言に変更
- `scripts/agent/tool_runner.py`:
  - `_apply_turn_char_limit()` から `result_id` 引数を削除
  - `id_hint` 生成ロジックを削除
  - `TURN_LIMIT_HINT.replace(...)` 呼び出しを削除
  - `_collect_tool_result_msgs()` 内の呼び出し箇所を更新
- `tests/test_tool_result_formatter.py` および `tests/test_tool_runner.py`: 影響箇所があれば修正

変更しないもの:
- `ToolResultStore` 自体の削除 (別タスクのスコープ)
- `cmd_tooling.py` の `/tool show` コマンド実装 (別タスクのスコープ)
- per-turn 文字数制限ロジックの動作

## Assumptions

- `ToolResultStore` / `/tool show` 削除は H-1/H-2 等の別タスクで行われる前提
- H-3 は「ヒントメッセージの文言変更」のみが対象
- `_collect_tool_result_msgs()` は引き続き `ctx.tool_result_store.store()` を呼ぶ (別タスクで削除予定)
- per-turn 制限が 0 のときは制限なし (既存動作を維持)

## Unknowns

なし。ソースコードを確認し、変更箇所・影響範囲ともに特定済み。

## Affected areas

- `scripts/agent/tool_result_formatter.py` (行 17-20): `TURN_LIMIT_HINT` 定数
- `scripts/agent/tool_runner.py` (行 250-267): `_apply_turn_char_limit()` 関数全体
- `scripts/agent/tool_runner.py` (行 205-210): `_collect_tool_result_msgs()` 内の呼び出し
- `tests/test_tool_runner.py`: `_apply_turn_char_limit` を直接テストするケースがあれば修正

## Design

### 変更後の TURN_LIMIT_HINT

```python
TURN_LIMIT_HINT = "[Result omitted: per-turn tool result limit reached.]"
```

### 変更後の `_apply_turn_char_limit()`

```python
def _apply_turn_char_limit(
    llm_text: str,
    turn_chars: int,
    limit: int,
) -> str:
    """Apply per-turn char limit; return hint if exceeded."""
    if limit > 0 and (turn_chars + len(llm_text)) > limit:
        logger.info(
            "Per-turn tool result limit reached: %s chars > %s; result replaced with hint",
            turn_chars + len(llm_text),
            limit,
        )
        return TURN_LIMIT_HINT
    return llm_text
```

### `_collect_tool_result_msgs()` 内の呼び出し変更

```python
llm_text = _apply_turn_char_limit(
    llm_text,
    turn_chars,
    limit=ctx.cfg.tool.tool_results_turn_max_chars,
)
```

`result_id` 引数の渡しを削除する。

## Implementation steps

1. `scripts/agent/tool_result_formatter.py` の `TURN_LIMIT_HINT` を新文言に変更
2. `scripts/agent/tool_runner.py`:
   a. `_apply_turn_char_limit()` のシグネチャから `result_id: int` を削除
   b. 関数本体の `id_hint` 生成と `TURN_LIMIT_HINT.replace(...)` を削除し、直接 `TURN_LIMIT_HINT` を返すよう変更
   c. `_collect_tool_result_msgs()` 内の呼び出しから `result_id=result_id` を削除
3. テストを実行して既存のテストが通ることを確認

## Validation plan

- `uv run pytest tests/test_tool_result_formatter.py tests/test_tool_runner.py -v` が全パスすること
- `TURN_LIMIT_HINT` の文字列に `/tool show` が含まれないこと (`grep` で確認)
- `TURN_LIMIT_HINT` の文字列に `id=` が含まれないこと (`grep` で確認)
- per-turn 制限付き (`tool_results_turn_max_chars > 0`) で複数ツール実行時、制限超過後のツールが
  `[Result omitted: per-turn tool result limit reached.]` に置き換えられること
- 制限を超えないケースでは `llm_text` がそのまま返ること

## Risks

- `_apply_turn_char_limit()` のシグネチャ変更により、テスト内で直接呼んでいる箇所があれば
  コンパイルエラーになるが、test_tool_runner.py を確認した限り直接呼び出しは存在しない。
  **軽微: テスト修正のみで対応可能**
- `result_id` をログ出力に使っているが、削除後はツール名と文字数で代替できる。
  **影響なし: デバッグ可用性は維持される**
