# Plan: Record and Analyze Round-Wide Serialization Caused by Side-Effect Tools

Derived from `01_plan.md` using requirement `requires/20260618_30_require.md`.

---

## Goal

Every round-wide serialization event caused by side-effect tools is recorded with the triggering tool name and timing impact, enabling data-driven optimization decisions.

---

## Scope

**In scope:**
- `scripts/agent/tool_runner.py` — instrument `_execute_standard()` and `execute_all_tool_calls()`
- `scripts/agent/tool_audit.py` — add `round_exec` audit event type
- `scripts/agent/llm_turn_runner.py` — optionally capture round boundaries
- `docs/05_agent_06_tool-execution-and-approval.md` — document serialization event schema

**Out of scope:**
- `scripts/agent/tool_scheduler.py` DAG path instrumentation (DAG mode is separate)
- Changing serialization rules
- Optimization or rule changes
- Dashboard/Grafana

---

## Assumptions

1. Audit log is the right channel (JSON-lines) — already used for `tool_exec`, `tool_approval` events.
2. Field `trigger_tool` (name of first side-effect tool found) is sufficient to identify the cause.
3. A single serialization event per round is recorded (not per-tool granularity — that already exists in `tool_exec` events).

## Implementation Steps

1. **Read `_execute_standard()` fully**
   - Identify the exact line where the parallel↔serial decision happens
   - Identify where per-tool timing could be captured
   - Criterion: exact instrumentation points known

2. **Add `round_exec` event to `tool_audit.py`**
   - Fields: `round_id`, `tool_count`, `mode` (parallel/serial/DAG), `has_side_effect`, `trigger_tool`, `elapsed_ms`
   - Criterion: event schema matches required data

3. **Instrument `_execute_standard()`**
   - Before parallel↔serial decision: record `has_side_effect` and `trigger_tool`
   - After round: record `elapsed_ms` and call `write_round_exec()`
   - Criterion: every serialized round produces an audit event

4. **Document event schema**
   - Update `docs/05_agent_06_tool-execution-and-approval.md`
   - Criterion: analyst can query audit log for serialization events

5. **Lint, type, arch validation**
   - `ruff check scripts/` — 0 errors
   - `mypy scripts/` — no new errors
   - `lint-imports` — 0 violations
