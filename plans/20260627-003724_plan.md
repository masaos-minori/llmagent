## Goal

Add a startup safety guard to reduce the risk of exposing the unauthenticated Event Bus API on public or LAN interfaces by detecting and warning/failing-fast on public bind addresses.

## Scope

**In-Scope**:
- Detect public bind addresses such as `0.0.0.0` or `::` at startup
- Emit a clear startup warning or fail-fast by default
- Consider explicit override such as `allow_public_bind=true`
- Log: no authentication enabled, bind address, whether public bind is allowed
- Update deployment docs with reverse proxy + authentication guidance
- Add tests for safe bind, unsafe bind, and explicit override

**Out-of-Scope**:
- Implementing API-key auth
- Implementing mTLS
- Adding ACLs

## Assumptions

1. The Event Bus has no authentication or ACL — confirmed in system overview doc
2. Host binding is controlled via uvicorn CLI arguments (not TOML config)
3. Fail-fast is preferred over warning-only for security-sensitive operations
4. An explicit override flag is needed for development/testing scenarios where public bind may be required

## Implementation Steps

1. **Phase 1: Add `allow_public_bind` config field**
   - Add `allow_public_bind: bool = False` to EventBusConfig in config.py
   - Add validation that rejects public bind unless explicitly allowed
   - Update TOML config example in deployment docs if needed

2. **Phase 2: Implement startup guard**
   - In app.py lifespan or FastAPI startup handler, detect the bind address
   - If bind address is `0.0.0.0` or `::` and `allow_public_bind=False`:
     - Option A (fail-fast): raise RuntimeError, prevent startup
     - Option B (warning-only): log critical warning, continue startup
   - Log: no authentication enabled, bind address, whether public bind is allowed

3. **Phase 3: Update deployment docs**
   - Add explicit guidance about reverse proxy + authentication for public-facing deployments
   - Document the `allow_public_bind` config field and its security implications
   - Add warning about binding to 0.0.0.0 in production

4. **Phase 4: Add startup guard tests**
   - Test: Safe bind (127.0.0.1) — startup succeeds without warning
   - Test: Unsafe bind (0.0.0.0) with allow_public_bind=False — fail-fast or critical warning
   - Test: Unsafe bind (0.0.0.0) with allow_public_bind=True — startup proceeds with warning
