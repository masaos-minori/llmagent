# Plan: Remove `auto_inject_notes` from configuration

## Goal
`auto_inject_notes` をTOML設定ファイルおよびドキュメント・ルールファイルから完全に除去し、設定モデル・ランタイム・テストのいずれにも参照が残らない状態にする。

## Scope
**In-Scope**:
- `config/tools.toml` から `auto_inject_notes = true` エントリとコメント行の削除
- `config/agent.toml` から `auto_inject_notes = true` エントリとコメント行の削除
- `docs/05_agent_08_configuration.md` の `ToolConfig` フィールド表から `auto_inject_notes` 行の削除
- `docs/05_agent_09_data-layer.md` の `notes` テーブル説明から `auto_inject_notes` 参照の削除
- `rules/env.md` の `notes` テーブル行から `auto_inject_notes=true` 参照の削除

**Out-of-Scope**:
- Pythonソース変更（`config_dataclasses.py`, `startup.py`, `config_reload.py`, `cmd_config_display.py` などはすでに `auto_inject_notes` を持たない）
- テスト変更（テストファイルに `auto_inject_notes` の参照は存在しない）
- `notes` テーブル自体の削除（本要件のスコープ外）
- `/note` コマンド関連の残留参照（別プランが担当: `20260630-181100_plan.md`）

## Assumptions
- `config_dataclasses.py` の `ToolConfig` にはすでに `auto_inject_notes` フィールドが存在しない（`grep` で確認済み）
- Pythonソース（`startup.py`, `config_reload.py`, `cmd_config_display.py`, `config_builders.py`, `config.py`）にも `auto_inject_notes` は存在しない（`grep` で確認済み）
- テスト (`tests/**/*.py`) に `auto_inject_notes` は存在しない（`grep` で確認済み）
- TOMLローダーは未知のキーを無視するため、`ToolConfig` に存在しないキーをTOMLに残しても起動エラーにはならないが、混乱を招くため削除が必要
