# Plan: shell-mcp `/health` レスポンス標準化 — `details.sandbox_backend` への統一

**Source:** `requires/20260622_60_require.md`
**Created:** 2026-06-23 11:01:42

---

## Goal

`shell-mcp` の `/health` レスポンスにおける `sandbox_backend` フィールドをルートレベルから `details.sandbox_backend` に移動し、全ドキュメントと読み取りコードをその形式に統一することで、オペレーターが一貫した方法でサンドボックス状態を確認できるようにする。

---

## Background: 現状分析

### 実装 (verified from source)

`scripts/mcp/shell/server.py` lines 97-103:
```python
return {
    "status": "ok",
    "ready": ready,
    "sandbox_backend": _service.sandbox_backend,   # ← ルートレベル
    "dependencies": deps,
    "details": {},
}
```

### ドキュメントの不整合

**問題1 — `docs/04_mcp_02_protocol_and_transport.md` line 202:**
```
| Others | Base response only: {"status":"ok","ready":bool,"dependencies":{},"details":{}} |
```
shell-mcp が "Others" に分類されているが、実際は `sandbox_backend` という server-specific フィールドを持つ。

**問題2 — `docs/04_mcp_04_server_catalog.md` lines 189-190:**
```
The active backend is visible in the `/health` response
(`"sandbox_backend": "none"` or `"sandbox_backend": "firejail"`).
```
ルートレベルの形式を文書化しているが、これは他のサーバー固有フィールド (`details.service`, `dependencies.github_token` 等) のパターンと不一致。

### 読み取りコード (blast radius 確認)

`scripts/agent/services/mcp_status.py` line 121:
```python
body.get("sandbox_backend", "")
```
ルートレベルから読み取る。`details` への移動で破損する。

## Assumptions

1. ベースヘルス形式は `{"status","ready","dependencies","details"}` の 4 フィールドが基本。サーバー固有データは `details.*` に入れるのが確立パターン。
2. `sandbox_backend` は "dependency" ではなく "operational detail" なので `details.sandbox_backend` が適切。
3. `scripts/agent/commands/cmd_mcp.py` line 35-36 は `McpStatusRow.sandbox_backend` を参照しており、`McpStatusRow` の `sandbox_backend` フィールド自体は変わらない — 影響なし。
4. `/health` を直接呼び出す外部ツール・スクリプトは存在しないか、変更後は `details.sandbox_backend` から読み取れる。
5. `tests/test_shell_mcp_service.py` にはヘルスエンドポイント専用テストが存在しない (grep 確認済み) — 新規追加。

## Design

### 1. `scripts/mcp/shell/server.py` — ヘルスレスポンス形式変更

**変更前:**
```python
return {
    "status": "ok",
    "ready": ready,
    "sandbox_backend": _service.sandbox_backend,
    "dependencies": deps,
    "details": {},
}
```

**変更後:**
```python
return {
    "status": "ok",
    "ready": ready,
    "dependencies": deps,
    "details": {"sandbox_backend": _service.sandbox_backend},
}
```

### 2. `scripts/agent/services/mcp_status.py` — 読み取り更新

**変更前 (line 120-121):**
```python
sandbox = (
    body.get("sandbox_backend", "")
    if isinstance(body, dict)
    else ""
)
```

**変更後:**
```python
sandbox = (
    body.get("details", {}).get("sandbox_backend", "")
    if isinstance(body, dict)
    else ""
)
```

### 3. `docs/04_mcp_02_protocol_and_transport.md` — Server-Specific テーブル更新

**変更前 (line 197-202):**
```
| Server | `/health` overrides |
|---|---|---|
| web-search-mcp | `dependencies.brave_api_key`, `dependencies.bing_api_key` (both `"set"`/`"not_set"`); `details.providers` (list) |
| github-mcp | `dependencies.github_token` (`"set"`/`"not_set"`) |
| mdq-mcp | root-level `"stub": true` (marks experimental status); `details.service: "mdq-mcp"` |
| Others | Base response only: `{"status":"ok","ready":bool,"dependencies":{},"details":{}}` |
```

**変更後:**
```
| Server | `/health` overrides |
|---|---|---|
| web-search-mcp | `dependencies.brave_api_key`, `dependencies.bing_api_key` (both `"set"`/`"not_set"`); `details.providers` (list) |
| github-mcp | `dependencies.github_token` (`"set"`/`"not_set"`) |
| mdq-mcp | root-level `"stub": true` (marks experimental status); `details.service: "mdq-mcp"` |
| shell-mcp | `details.sandbox_backend` (`"firejail"` or `"none"`) |
| Others | Base response only: `{"status":"ok","ready":bool,"dependencies":{},"details":{}}` |
```

### 4. `docs/04_mcp_04_server_catalog.md` — security note 更新 (lines 189-190)

**変更前:**
```
The active backend is visible in the `/health` response
(`"sandbox_backend": "none"` or `"sandbox_backend": "firejail"`).
```

**変更後:**
```
The active backend is visible in the `/health` response under
`details.sandbox_backend` (`"none"` or `"firejail"`).
```

### 5. テスト追加

**`tests/test_shell_mcp_service.py`**: FastAPI `TestClient` を使い `/health` レスポンス形式を検証

```python
def test_health_response_includes_sandbox_backend_in_details():
    from fastapi.testclient import TestClient
    from mcp.shell.server import app
    client = TestClient(app)
    resp = client.get("/health")
    assert resp.status_code == 200
    body = resp.json()
    assert "sandbox_backend" not in body              # ルートレベルにない
    assert "sandbox_backend" in body.get("details", {})  # details に存在
    assert body["details"]["sandbox_backend"] in ("none", "firejail")
```

**`tests/test_agent_mcp_status.py`**: `_get_http_status` が `details.sandbox_backend` を読み取ることを検証

```python
async def test_get_http_status_reads_sandbox_backend_from_details(respx_mock):
    # health レスポンスに details.sandbox_backend がある場合
    respx_mock.get("http://localhost:8009/health").mock(
        return_value=httpx.Response(200, json={
            "status": "ok", "ready": True,
            "dependencies": {},
            "details": {"sandbox_backend": "firejail"},
        })
    )
    service = McpStatusService(...)
    avail, sandbox = await service._get_http_status(client, "http://localhost:8009")
    assert sandbox == "firejail"
```
