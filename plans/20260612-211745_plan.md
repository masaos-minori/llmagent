# Structured Output First Migration Plan

## Goal

`scripts/` 全体で「LLM応答・ツール呼び出し結果・ワークフロー応答・MCPペイロード」を自然言語解釈からスキーマ駆動の構造化JSONへ移行し、全機械判断経路を型安全な action/result オブジェクトで表現する。

---

**In scope:**
- `scripts/shared/action_result.py` (新規) — 共通 ActionResult スキーマ定義
- `scripts/agent/orchestrator.py` — `TurnResult` を frozen dataclass に変換し `action`/`reason` 追加
- `scripts/agent/llm_turn_runner.py` — `run()` 戻り値 `str` → `TurnResult`

- 各 MCP サーバー (`cicd`, `file/×3`, `git`, `github`, `mdq`, `rag_pipeline`, `shell`, `sqlite`, `web_search`, `installer_templates`) — `dispatch()` 戻り値変更

## Assumptions

1. `LLMTurnRunner.run()` は `str` を返し、`orchestrator._handle_llm_turn()` が `TurnResult` に wrap している。`run()` 自体を `TurnResult` 返却に変更して wrap を削除できる（`orchestrator.py:147-152` の wrap ロジックが消える）。
2. `MCPServer.dispatch()` の `(str, bool)` タプルは新規 `DispatchResult(output: str, is_error: bool)` に置き換える。`ToolCallResult` はフィールドが 4 つある（`request_id`, `server_key` も含む）ので MCP 層向けには軽量な `DispatchResult` を `mcp/dispatch.py` に定義するのが適切。
3. `run_stdio()` の `raw_result, is_error = await self.dispatch(...)` (server.py:174) はタプルアンパックを使っている。`DispatchResult` に変更後は `result.output`, `result.is_error` でアクセスする。
4. `repl_health.py` の `check_service_health()` / `check_tool_definitions_runtime()` が返す `list[str]` は warning 文字列。呼び出し元 (`repl.py`) で `write_warning()` に渡している。`HealthCheckResult.warning_messages()` で後方互換を維持できる。
5. `memory.injection.on_user_prompt()` の `list[str]` は `orchestrator.py:117` で `"\n".join(f"- {s}" for s in memory_snippets)` として使われる。`MemorySnippet.text` に変えれば同等。`repl.py:280` でも同様のパターン。
6. `_parse_mqe_response()` はすでに JSON decode + 型チェックあり。戻り値 `list[str]` を `MqeParseResult(queries: list[str])` に変更することで呼び出し元が戻り値の意味を型で把握できる。

## Design

### 共通 ActionResult スキーマ (`scripts/shared/action_result.py`)

```python
"""shared/action_result.py
Universal action/result contract for all machine-interpreted outputs.
"""
from __future__ import annotations

from dataclasses import dataclass, field
from typing import Literal

ActionType = Literal[
    "continue",
    "call_tool",
    "retrieve_more_context",
    "ask_user",
    "fail",
    "retry",
]

@dataclass(frozen=True)
class ActionResult:
    """Schema-driven result for any machine decision path."""
    action: ActionType
    reason: str = ""
    required_context: list[str] = field(default_factory=list)
    payload: dict[str, object] = field(default_factory=dict)
    errors: list[str] = field(default_factory=list)
    confidence: float = 1.0
```

### TurnResult の frozen dataclass 化 (`scripts/agent/orchestrator.py`)

現状: 通常クラス (`__init__` で `success`/`answer`/`error_kind`)

```python
# Before
class TurnResult:
    def __init__(self, success: bool, answer: str = "", error_kind: str | None = None):
        self.success = success
        self.answer = answer
        self.error_kind = error_kind

# After
from dataclasses import dataclass
from typing import Literal

@dataclass(frozen=True)
class TurnResult:
    """Typed result of one LLM turn."""
    action: Literal["continue", "fail"]
    answer: str = ""
    error_kind: str | None = None
    reason: str = ""

    @property
    def success(self) -> bool:
        """Backward-compatible accessor."""
        return self.action == "continue"
```

呼び出し箇所の変更:
```python
# Before
return TurnResult(success=True, answer=answer)
return TurnResult(success=False, answer="", error_kind=str(e))

# After
return TurnResult(action="continue", answer=answer)
return TurnResult(action="fail", answer="", error_kind=str(e))
```

### LLMTurnRunner.run() の戻り値変更 (`scripts/agent/llm_turn_runner.py`)

```python
# Before
async def run(self, llm_url: str) -> str:
    ...
    return await self._finalize_answer(message)
    ...
    return "Maximum tool turns reached."

# After
async def run(self, llm_url: str) -> TurnResult:
    ...
    return TurnResult(action="continue", answer=await self._finalize_answer_text(message))
    ...
    return TurnResult(action="fail", answer="Maximum tool turns reached.", reason="max_tool_turns")
```

`orchestrator._handle_llm_turn()` の wrap を削除:
```python
# Before
answer = await self._llm_runner.run(llm_url)
...
return TurnResult(success=True, answer=answer)

# After
result = await self._llm_runner.run(llm_url)
...
return result
```

`_handle_llm_error` も `TurnResult` を返すように変更。

### DispatchResult と dispatch_tool() の変更 (`scripts/mcp/dispatch.py`)

```python
# 新規追加
@dataclass(frozen=True)
class DispatchResult:
    """Typed result from MCPServer.dispatch()."""
    output: str
    is_error: bool

# dispatch_tool() 変更
async def dispatch_tool(...) -> DispatchResult:
    ...
    return DispatchResult(output="...", is_error=True)  # error cases
    return DispatchResult(output=result, is_error=False)  # success
```

`mcp/server.py` の `run_stdio()` 変更:
```python
# Before
raw_result, is_error = await self.dispatch(name, dict(req.get("args", {})))
tr = _truncate_with_meta(raw_result)

# After
dispatch_result = await self.dispatch(name, dict(req.get("args", {})))
tr = _truncate_with_meta(dispatch_result.output)
is_error = dispatch_result.is_error
```

各 MCP サーバーの `dispatch()` 変更パターン (cicd を例に):
```python
# Before
async def dispatch(self, name: str, args: dict[str, Any]) -> tuple[str, bool]:
    return await _dispatch_cicd_tool(name, args)

# After
async def dispatch(self, name: str, args: dict[str, Any]) -> DispatchResult:
    return await _dispatch_cicd_tool(name, args)
```

### HealthCheckResult DTO (`scripts/agent/shared/health_models.py`)

```python
"""agent/shared/health_models.py
Typed DTOs for service health check results.
"""
from __future__ import annotations
from dataclasses import dataclass, field

@dataclass(frozen=True)
class ServiceWarning:
    label: str
    url: str
    message: str

@dataclass(frozen=True)
class HealthCheckResult:
    warnings: list[ServiceWarning] = field(default_factory=list)

    @property
    def has_issues(self) -> bool:
        return bool(self.warnings)

    def warning_messages(self) -> list[str]:
        """Return flat list of warning message strings (for write_warning calls)."""
        return [w.message for w in self.warnings]
```

`repl_health.check_service_health()` 変更:
```python
# Before
async def check_service_health(ctx: AgentContext) -> list[str]:
    warnings: list[str] = []
    ...
    warnings.append(msg)
    return warnings

# After
async def check_service_health(ctx: AgentContext) -> HealthCheckResult:
    warnings: list[ServiceWarning] = []
    ...
    warnings.append(ServiceWarning(label=label, url=health_url, message=msg))
    return HealthCheckResult(warnings=warnings)
```

呼び出し元 (`repl.py`) の変更:
```python
# Before
for msg in await check_service_health(self._ctx):
    self._view.write_warning(msg)

# After
result = await check_service_health(self._ctx)
for msg in result.warning_messages():
    self._view.write_warning(msg)
```

## Expected Benefits

- **安定した分岐ロジック**: `action` フィールドが明示的なため、文字列比較やフラグチェックが不要になる
- **決定論的なパースと再試行**: JSON decode + 型バリデーションにより、自然言語パーサーの曖昧さが排除される
- **監査可能性と再現性**: 全ての機械判断が構造化イベントとして記録可能になる
- **一貫性**: LLM / RAG / MCP / tool call / workflow 層が同一の action contract を共有する
- **保守性**: 型注釈によってコンパイル時に契約違反が検出可能になる
