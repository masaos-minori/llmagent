# Structured Output First Migration Plan

## Goal

`scripts/` 全体で「LLM応答・ツール呼び出し結果・ワークフロー応答・MCPペイロード」を自然言語解釈からスキーマ駆動の構造化JSONへ移行し、全機械判断経路を型安全な action/result オブジェクトで表現する。

---

## Scope

**In scope:**
- `scripts/agent/orchestrator.py` — `TurnResult` を action-oriented DTO に拡張
- `scripts/agent/llm_turn_runner.py` — `run()` 戻り値を `TurnResult` に変更
- `scripts/rag/llm.py` — `_parse_mqe_response` / `_parse_rerank_response` を型安全化
- `scripts/mcp/server.py` — `dispatch()` 戻り値を `(str, bool)` から typed DTO に変更
- `scripts/shared/tool_executor.py` — `ToolCallResult` を action contract に適合させる
- `scripts/agent/repl_health.py` — `list[str]` 戻り値を typed DTO に変更
- `scripts/agent/memory/injection.py` / `services.py` — `list[str]` 戻り値を typed DTO に変更
- `scripts/agent/services/export_formatter.py` — `print()` を OutputPort 経由に変更
- 新規: `scripts/shared/action_result.py` — 共通 ActionResult スキーマ定義
- 新規: `scripts/agent/shared/health_models.py` — HealthCheckResult DTO

**Out of scope:**
- プレゼンテーション層 (`cli_view.py`, `tool_output.py`) — 既に `print()` が OutputPort 経由で集中管理されており今回は変更しない
- `scripts/mcp/` 各サーバーの詳細 dispatch 実装 — Phase 3 で個別対応
- DB スキーマ変更 — 影響なし

## Assumptions

1. `LLMTurnRunner.run()` が `str` を返す現状は、呼び出し元 (`orchestrator.py`) が `TurnResult` に wrap している。`run()` 自体を `TurnResult` 返却に変更し、wrap を削除できる。
2. `MCPServer.dispatch()` の `(str, bool)` タプルは `ToolCallResult(output, is_error)` に置き換え可能。`ToolCallResult` はすでに `shared/tool_executor.py` に定義されている。
3. `repl_health.py` の `check_service_health()` / `check_tool_definitions_runtime()` が返す `list[str]` (警告文字列) は `HealthCheckResult` DTO (問題あり/なし + 詳細リスト) に変換できる。呼び出し元は文字列を `write_warning()` に渡しているだけなので変更コストは低い。
4. `memory.injection.on_session_start()` / `on_user_prompt()` が返す `list[str]` (スニペット文字列) は `MemorySnippet` DTO のリストに変換できる。注入先 (`repl.py`) は文字列として扱っているため、DTO に `text: str` フィールドを持たせれば後方互換を最小コストで維持できる。
5. `rag/llm.py` の `_parse_mqe_response` はすでに JSON decode + 型チェックを行っている。`MqeResult` DTO (queries: list[str], raw_response: str) を追加する形で型安全化できる。
6. 共通 `ActionResult` スキーマは Python `frozen dataclass` として定義し、すべての機械判断経路で使用する。JSON Schema の代わりに dataclass の型注釈を検証ゲートとして使う。

## Design

### 共通 ActionResult スキーマ (`scripts/shared/action_result.py`)

```python
from __future__ import annotations
from dataclasses import dataclass, field
from typing import Literal

ActionType = Literal[
    "continue",
    "call_tool",
    "retrieve_more_context",
    "ask_user",
    "fail",
    "retry",
]

@dataclass(frozen=True)
class ActionResult:
    """Universal action/result contract for all machine-interpreted outputs."""
    action: ActionType
    reason: str = ""
    required_context: list[str] = field(default_factory=list)
    payload: dict[str, object] = field(default_factory=dict)
    errors: list[str] = field(default_factory=list)
    confidence: float = 1.0
```

### TurnResult の拡張 (`scripts/agent/orchestrator.py`)

```python
@dataclass(frozen=True)
class TurnResult:
    action: Literal["continue", "fail"]
    answer: str = ""
    error_kind: str | None = None
    reason: str = ""

    @property
    def success(self) -> bool:
        return self.action == "continue"
```

### MCPServer.dispatch() 戻り値変更 (`scripts/mcp/server.py`)

```python
# Before
async def dispatch(self, name: str, args: ToolArgs) -> tuple[str, bool]:

# After
async def dispatch(self, name: str, args: ToolArgs) -> ToolCallResult:
```

`ToolCallResult` はすでに `shared/tool_executor.py` で定義済み。`mcp/server.py` がそれをインポートするとレイヤー違反の可能性があるため、`ToolCallResult` を `shared/` に移動するか、`mcp/` に別途 `DispatchResult` を定義する。

> **設計決定:** `ToolCallResult` を `shared/tool_executor.py` から `shared/tool_models.py` (新規) に移動し、`mcp/server.py` がインポートできるようにする。`.importlinter` の `mcp → shared` は許可されているため問題なし。

### HealthCheckResult DTO (`scripts/agent/shared/health_models.py`)

```python
@dataclass(frozen=True)
class ServiceWarning:
    label: str
    url: str
    message: str

@dataclass(frozen=True)
class HealthCheckResult:
    warnings: list[ServiceWarning] = field(default_factory=list)

    @property
    def has_issues(self) -> bool:
        return bool(self.warnings)

    def warning_messages(self) -> list[str]:
        return [w.message for w in self.warnings]
```

### MemorySnippet DTO (`scripts/agent/memory/models.py` に追加)

```python
@dataclass(frozen=True)
class MemorySnippet:
    text: str
    source: str = ""
    score: float = 0.0
```

## Expected Benefits

- **安定した分岐ロジック**: `action` フィールドが明示的なため、文字列比較やフラグチェックが不要になる
- **決定論的なパースと再試行**: JSON decode + 型バリデーションにより、自然言語パーサーの曖昧さが排除される
- **監査可能性と再現性**: 全ての機械判断が構造化イベントとして記録可能になる
- **一貫性**: LLM / RAG / MCP / tool call / workflow 層が同一の action contract を共有する
- **保守性**: 型注釈によってコンパイル時に契約違反が検出可能になる
