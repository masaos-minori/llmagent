## Goal

Make `tasks.workflow_id` a required, non-nullable invariant by removing the `or str(uuid.uuid4())` fallback in the task-resume path and by changing `create_task()` to require a non-None `workflow_id`, so that a task without a workflow ID cannot be created or resumed.

## Scope

**In**:
- Remove `workflow_id = task.workflow_id or str(uuid.uuid4())` from `Orchestrator._init_workflow_task()` (`orchestrator.py:305`); replace with a hard `RuntimeError` if `task.workflow_id` is None or empty
- Change `create_task()` in `scripts/agent/workflow/task_ops.py` so `workflow_id: str | None = None` becomes `workflow_id: str` (required, non-optional)
- Change `StateStore.create_task()` in `scripts/agent/workflow/state_store.py` similarly
- Update `TaskRecord.workflow_id` type in `scripts/agent/workflow/models.py` from `str | None` to `str`
- Replace `task.workflow_id or ""` fallbacks in `scripts/agent/workflow/workflow_engine.py` (lines 123, 146, 203, 222) with direct access (assert or raise if empty)
- Replace `ctx.workflow.workflow_id or ""` in `orchestrator.py:76` and other places where `workflow_id` is known to be set with an assertion or direct use
- Add tests: resume task without workflow_id fails; new task always has workflow_id; approval resume keeps workflow_id; OTel span uses stable ID

**Out**:
- Backfilling workflow IDs in existing task rows
- Supporting legacy nullable workflow IDs
- Migration scripts
- Changing the `tasks` table SQL schema (column is already present; only application-level enforcement changes)

## Assumptions

1. `orchestrator.py:280` (`workflow_id = str(uuid.uuid4())` for new tasks) is correct and stays unchanged.
2. `TaskRecord` is a dataclass or named tuple in `models.py`; changing `workflow_id: str | None` to `str` requires ensuring all construction sites pass a non-None value.
3. The only existing task resume path is `orchestrator.py:299–306` — `get_task_by_id()` followed by `task.workflow_id or str(uuid.uuid4())`.
4. `workflow_engine.py` spans use `task.workflow_id or ""` because `TaskRecord.workflow_id` is typed as `str | None`; after this change the type becomes `str` and `or ""` is unnecessary.
5. Old tasks in the database without `workflow_id` (NULL) will fail to resume after this change — this is intentional per requirement.
