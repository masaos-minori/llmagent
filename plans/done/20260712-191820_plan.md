## Design

Extend the existing same-line skip condition with a small set of literal Japanese negation
substrings, mirroring the existing `"never"` check exactly (same design, same risk profile,
per the requirement's explicit preference for option "a" over blanket-skipping `> **注記:**`
blocks):

```python
# tools/check_mcp_docs_consistency.py, inside check_transport_error_is_error()
_NEGATION_MARKERS = ("never", "ことはない", "しない", "返さない")  # module-level, near _TRANSPORT_IS_ERROR_RE

...
            if any(marker in line.lower() if marker == "never" else marker in line
                   for marker in _NEGATION_MARKERS) or "ToolCallResult" in line:
                continue
```

(Exact expression to be refined at implementation time — the key requirement is: `"never"`
stays case-insensitive as today, the Japanese markers are checked case-sensitively since
Japanese has no case-folding concern, and `"ToolCallResult"` remains its own independent
condition, unchanged.)

This keeps the fix scoped to literal substrings, consistent with the existing `"never"`
precedent, and does not attempt general negation parsing (explicitly rejected in the
requirement's Risks table as unnecessary scope).
