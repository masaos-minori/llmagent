## Goal

Add async-handler validation to `register_tool()` at registration time, and ensure both registration-time and runtime contract violations are handled per `plugin_strict` mode.

## Scope

**In**: Add `asyncio.iscoroutinefunction()` check in `register_tool()`. Ensure `plugin_auto_discover.py` catches the resulting error correctly. Add tests for all contract violation cases.

**Out**: Changing MCP tool return contracts, adding plugin sandboxing, documentation-only updates.

## Assumptions

1. `register_tool()` already validates return annotation (`tuple[str, bool]`); the gap is the missing async check.
2. `plugin_auto_discover.py` catches `ValueError` from `register_tool()` at import time, records it as a `PluginFailure`, and does NOT register the plugin.
3. Runtime validation in `plugin_tool_invoker.py` already covers: tuple length, `str` type, `bool` type.
4. When `strict_mode=True`, a `PluginLoadError` is raised if any plugin fails or has conflicts.
5. Non-async handlers would cause `TypeError` when awaited in `plugin_tool_invoker.py` — currently this would raise uncaught in production.

## Design

**Change to `register_tool()`**:
```python
import asyncio
import inspect

def register_tool(name: str) -> Callable[[_F], _F]:
    def decorator(fn: _F) -> _F:
        if not asyncio.iscoroutinefunction(fn) and not inspect.iscoroutinefunction(fn):
            raise ValueError(
                f"[plugin] tool contract violation: '{name}' handler must be async "
                f"(defined with 'async def')"
            )
        # ... existing annotation checks ...
```

This `ValueError` will be caught by `plugin_auto_discover.py`'s `except (..., ValueError)` block, recording the plugin as failed. In non-strict mode, loading continues. In strict mode, `PluginLoadError` is raised.

**Runtime validation gaps** (already covered in `plugin_tool_invoker.py`):
- tuple length != 2: `ValueError` (raises, not caught by outer except)
- first element not str: `TypeError` (raises, not caught)
- second element not bool: `TypeError` (raises, not caught)
- These will propagate up to the caller of `try_execute()` which is `tool_executor.py`.

The current `plugin_tool_invoker.py` raises `ValueError`/`TypeError` for runtime contract violations. These are not caught within `try_execute()`. The require asks these to produce "clear tool execution errors" — the current behavior raises rather than returning `ToolCallResult(is_error=True)`. This should be wrapped.

1. **Phase 1: Registration-time async check**
   - [ ] Add `asyncio.iscoroutinefunction()` check in `register_tool()`, raise `ValueError` with clear message.

2. **Phase 2: Runtime contract wrapping**
   - [ ] In `plugin_tool_invoker.try_execute()`, wrap the `ValueError`/`TypeError` runtime checks to return `ToolCallResult(is_error=True, ...)` instead of raising.

3. **Phase 3: Tests**
