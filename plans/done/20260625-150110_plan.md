# Plan: [High][eventbus][subscribe] ポーリングベース SSE 配信を in-memory push に置換

Source: `requires/20260625_35_require.md`

## Goal

`/subscribe` の live 配信を SQLite ポーリング (500ms 間隔) から in-memory asyncio.Queue ベースのプッシュ通知に切り替え、新規イベントを購読者に即座に届ける。

**Out-of-Scope**

- replay SQLite パスの変更 (req #37 で詳細設計)

## Assumptions

1. asyncio ベースのブローカーはすべてイベントループ上で動作するため threading.Lock は不要
2. 購読者ごとに `asyncio.Queue(maxsize=1000)` を割り当て、満杯時は最古エントリを drop (back-pressure は将来タスク)
3. ブローカーへの登録はリプレイ開始前に行い、遷移期間中のイベントを取りこぼさない
4. `_broker` は lifespan で初期化し、テストでは `monkeypatch` で差し替え可能にする

## Design

**`broker.py` の構成:**

```python
import asyncio
import logging
from dataclasses import dataclass, field
from typing import Any

logger = logging.getLogger(__name__)

@dataclass
class _Subscriber:
    queue: asyncio.Queue[dict[str, Any] | None]  # None = sentinel for unsubscribe
    topics: list[str]  # empty = all topics

class EventBroker:
    def __init__(self) -> None:
        self._subscribers: list[_Subscriber] = []

    def subscribe(self, topics: list[str]) -> _Subscriber:
        sub = _Subscriber(queue=asyncio.Queue(maxsize=1000), topics=topics)
        self._subscribers.append(sub)
        return sub

    def unsubscribe(self, sub: _Subscriber) -> None:
        try:
            self._subscribers.remove(sub)
        except ValueError:
            pass

    def publish(self, event: dict[str, Any]) -> int:
        """Fan out event to matching subscribers. Returns delivery count."""
        delivered = 0
        topic = event.get("topic", "")
        for sub in self._subscribers:
            if sub.topics and topic not in sub.topics:
                continue
            try:
                sub.queue.put_nowait(event)
                delivered += 1
            except asyncio.QueueFull:
                logger.warning("broker: subscriber queue full, dropping event seq=%s", event.get("seq"))
        return delivered
```

**`app.py` の変更 (概要):**

```python
_broker: EventBroker | None = None

# lifespan:
_broker = EventBroker()

# subscribe() SSE generator (live phase):
sub = _broker.subscribe(topic)
try:
    # Phase 1: replay from SQLite (req #37 で詳細設計)
    # Phase 2: live from broker queue
    while True:
        event = await sub.queue.get()
        if event is None:  # shutdown sentinel
            break
        yield f"data: {orjson.dumps(event).decode()}\n\n"
except asyncio.CancelledError:
    ...
finally:
    _broker.unsubscribe(sub)
```

## Implementation Steps

1. `scripts/eventbus/broker.py` を新規作成 (`EventBroker`, `_Subscriber`)
2. `app.py` に `_broker: EventBroker | None = None` グローバル追加
3. `lifespan` で `_broker = EventBroker()` 初期化
4. `subscribe()` SSE generator の live 配信パスを broker queue 読み取りに変更 (replay フェーズは現行の SQLite ループを暫定的に残す)
5. lifespan shutdown 時に全 subscriber に None sentinel を送信して graceful shutdown

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| broker.publish() が publisher より先にリターンするため、SSE client が receive する前にイベントが drop される | Low | `Queue(maxsize=1000)` で十分なバッファ。drop は WARNING ログ |
| subscribe → broker 登録 → replay の間にイベントが lost する | High | broker 登録後にリプレイを開始し、重複を seq で除去 (req #37 で実装) |
| lifespan shutdown 時に購読者が queue を待ち続ける | Medium | None sentinel を全 queue に put してブロックを解除 |
