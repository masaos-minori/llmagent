# Plan: Add /undo compressed-summary warning

## Goal

Warn the user when `/undo` targets a `[Conversation summary]` message (a compressed session summary), because rolling past it destroys the compressed context and prevents recovery of the original messages.

## Scope

- `scripts/agent/services/undo_service.py` — add detection and warning

## Assumptions

- The compressed summary message is inserted as a regular assistant-role message whose `content` begins with `[Conversation summary]`.
- The `/undo` command already raises `NothingToUndoError` when nothing can be undone; the warning is informational only (it does not block the undo).
- The warning should use `logger.warning()`, not a UI notification, because `undo_service.py` has no access to the REPL view layer. Callers (`cmd_context.py`) can surface the warning to the user if needed.
- Currently no caller actually surfaces the warning. The undo flow is: `cmd_context` → `undo_service.undo_last_turn()`. The warning will appear in logs but not on the user's terminal unless `cmd_context` is updated to check for it.

## Design

### Combined approach (log + REPL)

1. Extend `UndoResult` with `warning: str | None = None`.
2. In `undo_last_turn()`, detect compressed summary and set both `logger.warning()` and `result.warning`.
3. In `cmd_context.py`, after undo call, emit `view.write_warning()` if `result.warning` is set.

## Implementation steps

1. Read `scripts/agent/commands/cmd_context.py` to see how undo is invoked.
2. Add `warning` field to `UndoResult` in `models.py`.
3. Implement detection and warning in `undo_service.py`.
4. Surface warning in `cmd_context.py`.
5. Run `ruff check` + `mypy`.

## Validation plan

- Unit test: create a history with a `[Conversation summary]` message, call `undo_last_turn()`, assert the warning is emitted (log capture or `UndoResult.warning`).
- Manual: start session, trigger compression, run `/undo`, observe warning.

## Risks

- Low: false positives if a legitimate assistant response happens to start with `[Conversation summary]`.
- Mitigation: the prefix is unique to the compression mechanism; no ordinary assistant response uses it.
