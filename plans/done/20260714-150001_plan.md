# Work Plan: Align Configuration Documentation with Consolidated config/agent.toml Layout

## Goal

Update all documentation so it consistently states that the agent process configuration is consolidated in `config/agent.toml`. Remove obsolete references to the old 12-base-config-file model.

## Scope

**In-Scope:**
- Rewrite/remove references to "12 base config files" across all docs
- Describe `config/agent.toml` as the canonical agent process configuration file
- Keep historical split config references only as legacy notes
- Update `/reload` documentation to describe current behavior
- Ensure `ConfigLoader.load_all()` documentation matches implementation
- Update document guide and reference API if they still imply split-file ownership

**Out-of-Scope:**
- Reintroducing split config files
- Changing runtime config loading unless the implementation contradicts the confirmed specification
- Moving MCP server-local application settings into `agent.toml`

1. The confirmed specification is: agent configuration is consolidated in `config/agent.toml`; historical split config files are no longer canonical.
2. `ConfigLoader.load_all()` loads `config/agent.toml` as the primary config (confirmed via inspection of `shared/config_loader.py`).
3. `/reload` applies reloaded config to live services without restarting the agent (confirmed via `agent/services/config_reload.py`).
4. The existing plan format (see `plans/20260714-143629_plan.md`) is the canonical template.

## Design

### Change Strategy

1. **Remove** all text describing the old 12-base-config-file model as current behavior.
2. **Replace** with unified statement: "The agent process configuration is consolidated in `config/agent.toml`. Historical split configuration files are no longer canonical."
3. **Rewrite** `/reload` documentation to describe actual behavior: reloads `config/agent.toml` and applies changes to live services.
4. **Keep** historical notes where they clarify why something was changed (e.g., `_FORBIDDEN_KEYS` entries mentioning old keys).
5. **Ensure** consistency across all affected documents — every mention of config loading must use the same model.

### Consistency Rules

- After changes, every mention of agent config must state it is consolidated in `config/agent.toml`.
- No document should describe the old 12-base-config-file model as current behavior.
- If a document previously described both the old model and the new model, retain only the new model plus a brief historical note.
