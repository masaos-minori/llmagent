# Plan: Add MCP Documentation Consistency Checks to CI

## 1. Goal

Integrate the existing `scripts/check_mcp_docs_consistency.py` script into CI so that MCP documentation drift is automatically detected on every commit.

## 2. Scope

**In-Scope**:
- Create GitHub Actions workflow file (`.github/workflows/mcp-docs-consistency.yml`)
- Integrate existing `scripts/check_mcp_docs_consistency.py` into CI pipeline
- Add local run instructions to the script

**Out-of-Scope**:
- Adding new consistency checks (already implemented in the script)
- Fixing existing documentation drift warnings found by the script
- Modifying the consistency check script itself

## 3. Assumptions

1. GitHub Actions is available for CI (standard for this repo)
2. Python 3.10+ is available in the CI runner (the script has no external dependencies beyond stdlib)
3. The existing checks in `scripts/check_mcp_docs_consistency.py` are correct and do not need modification

## 4. Unknowns & Gaps

| ID | Unknown Description | Evidence Missing | Resolution Path | Blocking? (True/False) |
|---|---|---|---|---|
| UNK-01 | Whether other CI workflows exist that should be referenced for style consistency | No .github/workflows/ files found in the repo | Check if CI exists elsewhere (e.g., other CI providers) | False |

## 5. Affected Areas & Tool Evidence

- **Affected Files**:
  - `.github/workflows/mcp-docs-consistency.yml` — new file, creates the CI job
  - `scripts/check_mcp_docs_consistency.py` — existing script, no changes needed (already standalone with no external deps)

- **Blast Radius**: Low — only adds a CI check; does not modify any MCP server or agent code.

- **Risk Metrics**: The consistency check script currently reports 8 warnings (active inconsistencies not cross-referenced). These are pre-existing and should be addressed separately before enabling the CI check.

- **Deploy Impact**: No deploy impact. This is purely a CI workflow addition.

## 6. Implementation Steps

1. **Phase 1: Create CI workflow**
   - [ ] Create `.github/workflows/mcp-docs-consistency.yml`
   - [ ] Configure to run on push/PR to main branch
   - [ ] Use Ubuntu runner with Python 3.10+
   - [ ] Run `scripts/check_mcp_docs_consistency.py --all` as the CI step

2. **Phase 2: Add local run instructions**
   - [ ] Add a "Local execution" section to the script's docstring explaining how to run it locally
   - [ ] Optionally add a Makefile target or shell alias reference

3. **Phase 3: Address pre-existing warnings**
   - [ ] Before enabling CI, address the 8 WARNING-level issues found by the current check
   - [ ] Either fix the docs (cross-reference active inconsistencies) or add allowlists for known exceptions

## 7. Validation Plan

|---|---|---|---|

## 8. Risks & Mitigations

- **Risk**: CI fails due to pre-existing documentation drift warnings (currently 8 warnings) → **Likelihood: High** → **Mitigation**: Address the warnings before enabling CI, or add allowlists for known exceptions
- **Risk**: CI workflow conflicts with existing CI patterns if other workflows exist → **Likelihood: Low** → **Mitigation**: Check for existing workflows; follow their conventions (Python version, runner image)
- **Risk**: False positives from overly strict regex patterns in the check script → **Likelihood: Medium** → **Mitigation**: Add `--skip` flags for specific checks if needed; add allowlists for intentional exceptions
