## Design

`check_allowed_root()` のパス解決フロー:
```
allowed_root → Path(allowed_root).resolve()  # symlink 解決 + 絶対パス化
path_arg     → Path(val).resolve()           # 同上
              → is_relative_to(root)         # root 配下か判定
```

追加するテストケース:
1. **相対パス (root 内)**: `"."` 等の相対パスが `resolve()` で絶対パスに変換され allowed と判定される
2. **`../` path traversal**: `allowed/subdir/../../../outside.txt` 形式が denied される
3. **symlink → allowed_root 内**: allowed_root 内の実体を指す symlink → True
4. **symlink → allowed_root 外**: allowed_root 外の実体を指す symlink → False
