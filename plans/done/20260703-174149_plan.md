## Goal

Consolidate `ToolExecutor.__init__()` after extracting `PluginToolInvoker` (require-22), `ToolResultCache` (require-23), and `ToolTransportInvoker` (require-25). The constructor becomes pure dependency wiring with no implementation ownership.

## Scope

**In-Scope:**
- Update `scripts/shared/tool_executor.py` `__init__()` body:
  - Replace inline `OrderedDict[str, CacheEntry]` field with `ToolResultCache` instantiation
  - Replace inline `HttpTransport` loop with `ToolTransportInvoker` instantiation
  - Remove direct fields: `_http`, `_server_configs`, `_cache_max_size`, `_concurrency_limits`, `_semaphores`, `_transports`, `_lifecycle`, `_health_registry`, `stat_tool_errors`, `stat_transport_errors`
  - Add `_plugin_invoker`, `_cache`, `_transport_invoker` delegated component fields
  - Keep `_inflight`, `stat_cache_hits`, `_tool_error_threshold`, `_cache_ttl` (needed for `apply_config`)
- Verify existing `ToolExecutor` tests still pass without modification

**Out-of-Scope:**
- Changing `ToolExecutor.__init__()` public signature
- Dependency injection framework changes
- Any changes to `PluginToolInvoker`, `ToolResultCache`, or `ToolTransportInvoker` APIs

**Sequencing dependency:** This plan assumes require-22 (`PluginToolInvoker`), require-23 (`ToolResultCache`), and require-25 (`ToolTransportInvoker`) are already implemented. Do not start this plan until those three are complete.

## Assumptions

1. `PluginToolInvoker()` takes no constructor arguments.
2. `ToolResultCache(ttl, max_size)` accepts the same values currently stored in `_cache_ttl` and `_cache_max_size`.
3. `ToolTransportInvoker(http, server_configs, concurrency_limits, lifecycle)` accepts the corresponding `ToolExecutor.__init__` parameters.
4. `ToolTransportInvoker` owns the unknown-key concurrency limit warning (moved from `ToolExecutor.__init__`).
5. `_cache_ttl` field is kept on `ToolExecutor` only because `apply_config()` uses it; alternatively it can be removed if `apply_config` delegates to `self._cache.apply_config()` directly.
6. `stat_tool_errors` and `stat_transport_errors` live on `ToolTransportInvoker` after require-25; `ToolExecutor.get_error_counters()` delegates.
