# Plan: Consolidate built-in slash command definitions into `command_defs.py`

Source: `requires/20260702_63_require.md`

---

## Goal

Make `scripts/agent/commands/command_defs.py` the single source of truth for all built-in slash command definitions by removing the duplicate `_COMMANDS` list from `registry.py` and importing it instead.

---

## Scope

**In-Scope**:
- Add `/approve` and `/reject` entries to `_COMMANDS` in `command_defs.py`
- Add `subcommands` list to `/db` entry in `command_defs.py` (already present; verify parity)
- Fix `/db` `is_async` flag in `command_defs.py` (currently `False`; must be `True` to match `async def _cmd_db`)
- Remove the local `_COMMANDS` definition from `registry.py`
- Add `from agent.commands.command_defs import _COMMANDS` to `registry.py` (import already has `CommandDef`; extend it)
- Preserve command order in `/help` output

**Out-of-Scope**:
- `/exit` handling
- Plugin command registry redesign
- `sqlite-vec` handling
- Any change to `_cmd_*` handler implementations

---

## Assumptions

1. `command_defs.py` is already imported in `registry.py` (`from agent.commands.command_defs import CommandDef`); only `_COMMANDS` needs to be added to the import.
2. The `_cmd_db` handler is `async def`; therefore the `CommandDef` for `/db` must have `is_async=True`.
3. `/approve` and `/reject` handlers are defined in `cmd_workflow.py` (`_WorkflowMixin`) and are already present in `CommandRegistry` via mixin inheritance.
4. The `__init__` fail-fast check (`hasattr(self, _cmd.handler)`) is sufficient validation that no handler is missing after the consolidation.
5. No test currently directly asserts the content of `_COMMANDS` in `registry.py`; behavior tests (dispatch, `/help`) are the primary regression guard.
