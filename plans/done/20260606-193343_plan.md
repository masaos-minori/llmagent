# MCP ヘルスチェックの劣化制御付き運用強化 — Work Plan

Source instruction: `instructions/20260606_08_instruction.md`

---

## Goal

起動時警告表示のみの方式から、MCP サーバごとに `healthy / degraded / unavailable` 状態を持ち、劣化時に該当ツールを自動的に無効化する方式へ強化する。

---

## Scope

**In:**
- `McpServerHealthState` 型の定義（healthy / degraded / unavailable）
- 障害時のツール自動無効化（`ToolExecutor` のディスパッチ対象外）
- `/mcp` 出力への状態表示追加
- 復旧検出ロジック（定期再ヘルスチェック）

**Out:**
- 自動復旧の起動（Watchdog が担当）
- 異常を外部通知する仕組み（別タスク）

---

## Assumptions

1. 現在の `repl_health.py` は起動時 + Watchdog の定期ヘルスチェックを行うが、状態を構造化で管理していない。
2. `ToolExecutor` は現在 health 状態を保持していない。
3. `McpStatusService` は `/mcp` 表示に使われている（`mcp_status.py`）。

## Implementation Steps

1. `McpServerHealthState` 型を `shared/mcp_config.py` または新規モジュールに定義
2. `watchdog_loop()` で各サーバの状態を `healthy / degraded / unavailable` に更新
3. `ToolExecutor.execute()` で `server_key` が `unavailable` の場合は即座に `is_error=True` を返す
4. `McpStatusService` の status 表示に `healthy/degraded/unavailable` を追加
5. 復旧検出：Watchdog が次のチェックで `healthy` に戻った場合は状態をリセット
6. `python-test-and-fix` スキルでテスト追加
7. バリデーション実行 + instructions/done/ へ移動
