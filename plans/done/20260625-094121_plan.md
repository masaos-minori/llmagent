# Introduce centralized repository gateway for all write operations

**Requirement:** `requires/20260624_63_require.md`

## Goal

Create `agent/repository_gateway.py` as the single enforcement point for all write, delete, and API-write operations. Policy checks, approval enforcement, and audit logging flow through the gateway; read operations bypass it.

## Current State

```
execute_all_tool_calls()
  → run_approval_checks(ctx, tool_calls)   ← batch approval UI (tool_approval.py)
    → classify_risk() + check_preflight()  ← policy (tool_policy.py)
  → execute_one_tool_call()
    → ctx.services.tools.execute()         ← ToolExecutor (shared/)
      → plugin_registry.get_tool() [plugin path — no gateway]
      → HTTP/stdio transport [MCP path]
```

Policy and approval are enforced at the batch level in `execute_all_tool_calls()`. Plugin tools bypass MCP routing but still pass through `run_approval_checks()` because approval operates on tool names (not execution paths). Ad hoc writes (programmatic calls not via `execute_all_tool_calls`) bypass all checks.

## Scope

**In:**
- new `scripts/agent/repository_gateway.py` — `RepositoryGateway` class: wraps `ToolExecutor`, enforces policy + approval + audit for write operations; read operations delegated directly
- `scripts/agent/tool_runner.py` — route ALL tool calls through `RepositoryGateway.execute()` instead of `ctx.services.tools.execute()` directly; remove `run_approval_checks()` from `execute_all_tool_calls()` (approval moves into the gateway)
- `scripts/agent/context.py` — add `AppServices.gateway: RepositoryGateway | None` field; set alongside `ctx.services.tools`
- `scripts/agent/factory.py` — construct `RepositoryGateway(executor, ...)` and assign to `AppServices.gateway`
- `scripts/agent/tool_approval.py` — make `run_approval_checks()` callable from gateway (currently public; keep signature compatible)
- `tests/test_repository_gateway.py` — new file: policy enforcement, approval bypass for read tools, audit emission

**Out:**
- Full Publisher role introduction
- Complete workflow phase redesign
- Removing `tool_policy.py` or `tool_approval.py` (they become dependencies of the gateway)
- Changing plugin registration API
- Write-path enforcement for plugin tools that are called outside of `execute_all_tool_calls()`  (plugin calls via `execute_one_tool_call` already pass through approval)

## Assumptions

1. All LLM-initiated tool calls go through `execute_all_tool_calls()` → `execute_one_tool_call()`. There are no other programmatic write callers in the current codebase. Verified by grep: `execute_one_tool_call` only called from `_execute_standard`, `_execute_with_dag`.
2. Plugin tools that perform writes ARE covered by `run_approval_checks()` in `execute_all_tool_calls()` because approval checks operate on tool names from the tool call list before any execution happens.
3. `RepositoryGateway.execute()` has the same signature as `ToolExecutor.execute(tool_name, args)`. Callers (`execute_one_tool_call`) are unchanged except they receive the gateway via `ctx.services.gateway` instead of calling `ctx.services.tools` directly.
4. `run_approval_checks()` is moved OUT of `execute_all_tool_calls()` and into the gateway's `execute()` method (per-tool approval check). This changes approval timing: currently batch (all tool calls approved before any executes); after: per-call (approval checked at execution time). The behavioral difference is acceptable — it simplifies the gateway design.
5. Read tools (`classify_operation_type() == READ`) bypass approval and policy checks in the gateway (as today).
6. `AppServices.gateway` replaces `AppServices.tools` as the entry point for tool execution in `execute_one_tool_call()`. `AppServices.tools` (the `ToolExecutor`) remains accessible for lifecycle management (health registry, session ID propagation, transport injection).

## Design

### `RepositoryGateway`

```python
class RepositoryGateway:
    """Single write enforcement boundary for all repository mutation operations.
    
    Wraps ToolExecutor. Write/delete/API-write tool calls are gated through
    policy checks, approval prompts, and audit logging. Read-only tool calls
    are forwarded directly without additional checks.
    """
    
    def __init__(
        self,
        executor: ToolExecutor,
        cfg: AgentConfig,
        output: OutputPort,
        audit_logger: logging.Logger | None = None,
    ) -> None:
        self._executor = executor
        self._cfg = cfg
        self._output = output
        self._audit_logger = audit_logger
    
    async def execute(
        self,
        ctx: AgentContext,
        tool_name: str,
        args: dict[str, Any],
    ) -> ToolCallResult:
        """Execute one tool call, enforcing write boundary policy.
        
        Read-only tools: direct passthrough.
        Write/delete/API-write tools: policy check → approval → execution → audit.
        """
        op = classify_operation_type(tool_name)
        if op == OperationType.READ:
            return await self._executor.execute(tool_name, args)
        return await self._gate_write(ctx, tool_name, args, op)
    
    async def _gate_write(
        self,
        ctx: AgentContext,
        tool_name: str,
        args: dict[str, Any],
        op: OperationType,
    ) -> ToolCallResult:
        """Enforce policy, prompt for approval, execute, audit."""
        # 1. Policy preflight (raises PolicyViolationError on violation)
        try:
            check_preflight(self._cfg, tool_name, args)
        except PolicyViolationError as e:
            return _denied_result(f"Policy blocked: {e}")
        
        # 2. Approval check
        risk = classify_risk(self._cfg, tool_name, args)
        if risk != RiskLevel.NONE:
            approved = await _prompt_approval(ctx, tool_name, args, risk, self._output)
            if not approved:
                return _denied_result("Denied by user")
        
        # 3. Execute
        result = await self._executor.execute(tool_name, args)
        
        # 4. Audit
        _emit_write_audit(ctx, tool_name, args, result, op, self._audit_logger)
        return result
```

result = await ctx.services.tools.execute(name, args)
