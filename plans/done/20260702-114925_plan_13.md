## Assumptions

1. `RagHit` (`RawHit | MergedHit | RankedHit`) carries only `content: str`, not `normalized_content`. The `normalized_content` field lives in `chunks` DB table and `IngestionChunk` TypedDicts but is never loaded into `RagHit`. Tests simulate the invariant by constructing a `RagHit` with readable `content` and a separate forbidden string representing what `normalized_content` would look like.
2. `RagPipeline._format_chunks` (static method in `pipeline.py`) and `augment._format_chunks` (module-level in `stages/augment.py`) are functionally equivalent with one behavioral difference: `augment._format_chunks([])` returns `{START}\n\n{END}`, while `RagPipeline._format_chunks([])` returns `{START}\n\n{END}` (empty join). Both must be tested independently.
3. `AugmentStage.run()` delegates to `augment._format_chunks`, not to `RagPipeline._format_chunks`. The authoritative implementation for the Augment stage is `scripts/rag/stages/augment.py`.
4. TEST-DESIGN2-02 is already partially covered by `tests/test_fts_fallback.py`; if gap exists, add one targeted assertion there rather than in the pipeline test files.
