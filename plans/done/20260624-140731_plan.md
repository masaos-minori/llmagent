## Goal

`workflow_schema.py::init_schema()` が直接 `sqlite3.connect()` を呼ぶ代わりに `SQLiteHelper("workflow")` を使用するよう変更し、接続ポリシー (WAL/FK/busy_timeout) を一元管理する。

## Scope

**In**:
- `scripts/db/workflow_schema.py` — `init_schema()` を `SQLiteHelper("workflow")` 使用に変更、`db_path` パラメータを削除、`__main__` ブロックを更新

**Out**:
- フル migration フレームワーク
- workflow スキーマ設計の変更
- `helper.py` の変更 (既に `DbTarget.WORKFLOW` サポート済み)

## Assumptions

1. `SQLiteHelper("workflow")` は `build_db_config().workflow_db_path` を使用するため、`__main__` での `build_db_config()` 明示呼び出しが不要になる。
2. `deploy/init_db.sh` は `python -m db.workflow_schema` として呼ぶため、`__main__` の動作を保持することが必須。
3. `init_schema()` の外部呼び出しは `workflow_schema.py::__main__` のみ。テストでも `init_schema` は確認なし。
4. `SQLiteHelper("workflow").open(write_mode=True)` は WAL + FK + busy_timeout を適用する。これは現行 `executescript(_DDL)` 内の `PRAGMA journal_mode=WAL; PRAGMA foreign_keys=ON;` と同等以上。

## Design

### 変更前

```python
def init_schema(db_path: str) -> None:
    """Create workflow tables if they do not exist."""
    conn = sqlite3.connect(db_path)
    try:
        conn.executescript(_DDL)
        conn.commit()
        cursor = conn.execute("PRAGMA table_info(tasks)")
        columns = [row[1] for row in cursor.fetchall()]
        if "workflow_id" not in columns:
            conn.execute("ALTER TABLE tasks ADD COLUMN workflow_id TEXT")
            conn.commit()
    finally:
        conn.close()

if __name__ == "__main__":
    cfg = build_db_config()
    Path(cfg.workflow_db_path).parent.mkdir(parents=True, exist_ok=True)
    init_schema(cfg.workflow_db_path)
    logger.info("workflow schema initialised: %s", cfg.workflow_db_path)
```

### 変更後

```python
def _migrate_workflow_schema(conn: sqlite3.Connection) -> None:
    """Add missing workflow columns idempotently."""
    columns = {row[1] for row in conn.execute("PRAGMA table_info(tasks)").fetchall()}
    if "workflow_id" not in columns:
        conn.execute("ALTER TABLE tasks ADD COLUMN workflow_id TEXT")

def init_schema() -> None:
    """Create workflow tables using SQLiteHelper for connection-policy consistency."""
    from db.helper import SQLiteHelper
    with SQLiteHelper("workflow").open(write_mode=True) as db:
        db.executescript(_DDL)
        _migrate_workflow_schema(db.conn)

if __name__ == "__main__":
    cfg = build_db_config()
    Path(cfg.workflow_db_path).parent.mkdir(parents=True, exist_ok=True)
    init_schema()
    logger.info("workflow schema initialised: %s", cfg.workflow_db_path)
```

### _DDL の PRAGMA 重複について

`_DDL` 内に `PRAGMA journal_mode=WAL; PRAGMA foreign_keys=ON;` が含まれているが、`SQLiteHelper` の `open(write_mode=True)` が既に同じ PRAGMA を適用する。重複は無害だが、`_DDL` から PRAGMA 行を削除することで `schema_sql.py` パターンに近づけてもよい（この判断は Phase 1 で決定）。
