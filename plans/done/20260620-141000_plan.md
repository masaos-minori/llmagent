## Design

### Code change
- `scripts/mcp/mdq/service.py:38`: Change hardcoded default from `/opt/llm/db/mdq.db` to `/opt/llm/db/mdq.sqlite`.
- This is the fallback when no `db_path` is provided via config. TOML config already specifies `mdq.sqlite`, so this only affects direct instantiation or JSON config paths that were using `mdq.db`.

### Config change
- `config/mdq_mcp_server.json:9`: Change path from `/opt/llm/db/mdq.db` to `/opt/llm/db/mdq.sqlite`.

### Documentation change
- `docs/04_mcp-mdq.md:93`: Replace the "DB path mismatch" known issue with a resolution note that includes migration guidance for environments that may still have `mdq.db` files.

### Migration guidance (to be added to docs)
For environments that may still reference `mdq.db`:
1. Explicit migration note in documentation.
2. No code-level compatibility bridge needed — the config file is the single source of truth, and the code default now matches.
3. If an existing deployment has a `mdq.db` file on disk, operators should rename it to `mdq.sqlite` or update their JSON config.
