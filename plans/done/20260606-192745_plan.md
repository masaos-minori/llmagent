## Implementation Steps

### Step 1: `db/create_schema.py` — 旧テーブルの baseline DDL 除去

`_SESSION_SCHEMA_TEMPLATE` から以下を削除する：
- `CREATE TABLE IF NOT EXISTS memory_entries (...)` (旧)
- `CREATE VIRTUAL TABLE IF NOT EXISTS memory_vec USING vec0(...)` (旧)

対応する migration の DROP 文も整理する（旧テーブルがなければ不要）。

`python-refactoring` スキルを使用する。

### Step 2: `db/migrate.py` の確認
- `_copy_table()` が named column ではなく `SELECT *` を使っているかを確認する
- 問題があれば named column ベースの copy に修正する

### Step 3: `db/maintenance.py` の failure policy 確認
- silent fallback / データロスの可能性があるパスを特定する
- failure audit event がなければ追加する

### Step 4: バリデーション実行
- `uv run ruff check scripts/`
- `uv run mypy scripts/`
- `lint-imports`
- `uv run pytest`

### Step 5: instructions/done/ へ移動
- `instructions/20260603_03_instruction.md` を `instructions/done/` に移動する
