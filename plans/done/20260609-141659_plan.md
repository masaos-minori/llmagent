# 作業計画: MCP 基盤改修 Phase 1 (server / installer / echo_server)

instruction: `instructions/20260609_06_instruction.md`

---

## Goal

`mcp/` 配下の backward-compat alias・二重 API・init.d fallback・thin facade・docstring 不整合を除去し、MCP サーバ基盤の責務を明確化する。Phase 1 High 項目のみを対象とする。

---

## Scope

**In** (Phase 1 High のみ):
- `scripts/mcp/server.py` — `run()` backward-compat alias を削除；`_truncate()` を削除 (`_truncate_with_meta()` に一本化)
- `scripts/mcp/installer_port.py` — init.d fallback (`_ports_from_initd()`) を削除；`_REPO_ROOT` class-level 暗黙依存をコメント注記で明確化 (完全削除は外部 API 変更のため別 Phase)
- `scripts/mcp/installer_templates.py` — `{cls}().run()` を `{cls}().run_http()` に変更；生成 docstring の `stdlib only` 記述はこのファイルには存在しないため対象外
- `scripts/mcp/installer.py` — thin facade を削除し、呼び出し元を直接 import に変更
- `scripts/mcp/echo_server.py` — docstring の "stdlib only" 記述を削除 (`orjson` 実態に合わせる)
- `scripts/agent/services/mcp_install.py` — `installer.py` facade 削除後の import を更新
- `scripts/agent/commands/cmd_mcp.py` — `installer.py` facade 削除後の import を更新
- `tests/` — 上記変更に対応するテスト更新

**Out** (Phase 2/3 は別タスク):
- `dispatch.py` の結果型変更 (`tuple[str, bool]` → `DispatchResult`)
- `models.py` / `tool_validators.py` の検証統合
- `audit.py` の監査 API 再定義
- `installer_validation.py` の structured validation 化
- `installer_writer.py` の InstallPlan 分離
- `installer_port.py` の `_REPO_ROOT` 完全削除 (API 変更を伴う)
- `server.py` の transport 別分離 (大規模再設計)

---

## Assumptions

1. `server.py` の `run()` alias が呼ばれているのは `installer_templates.py` の生成テンプレート内のみ (grep 確認: `MCPServer.run()` への直接参照はゼロ、テンプレート文字列内に `{cls}().run()` が 1 か所)。
2. `_truncate()` は `server.py` 内部でのみ使われている (外部から `_truncate` をインポートしている箇所がないか確認が必要)。
3. `installer.py` の facade は `mcp_install.py` と `cmd_mcp.py` の 2 箇所から `from mcp.installer import ...` でインポートされている。
4. `installer_port.py` の `scan_used_ports()` は既に `|=` (和集合) 実装になっており、instruction で指摘された「上書き」バグは現行コードでは修正済み。Phase 1 では init.d fallback 削除と `_REPO_ROOT` コメント明確化のみ行う。
5. `echo_server.py` の docstring "stdlib only" 削除は 1 行変更で完了する。

## Design

### D-1: `server.py` の `run()` alias 削除

```python
# Delete lines 141-143:
def run(self) -> None:
    # backward-compat alias; callers should migrate to run_http()
    self.run_http()
```

### D-2: `server.py` の `_truncate()` 削除

`_truncate()` を削除し、呼び出し元を `_truncate_with_meta()` に置き換える。
`server.py` 内の `_truncate()` 呼出箇所を `_truncate_with_meta().text` に変更する。

### D-3: `installer_port.py` の init.d fallback 削除

`_ports_from_initd()` 関数を削除。
`scan_used_ports()` の `used |= _ports_from_initd(...)` 行を削除。
`_REPO_ROOT` に「Phase 2 で削除予定」コメントを追加。

### D-4: `installer_templates.py` の `run()` → `run_http()`

テンプレート文字列の `{cls}().run()` を `{cls}().run_http()` に変更。

### D-5: `installer.py` 削除

ファイルを `git rm` する。
2 つの呼び出し元を直接 import に変更:
- `mcp_install.py`: `from mcp.installer import X` → 各 sub-module から直接 import
- `cmd_mcp.py`: 同様

### D-6: `echo_server.py` docstring 修正

1 行目の `"""Minimal JSON-RPC echo server for integration tests. stdlib only."""` から `stdlib only` を削除。

## Implementation Steps

1. **[server] `run()` alias 削除と `_truncate()` 削除**

   - 完了条件: `grep "init.d\|initd" scripts/mcp/installer_port.py` → 0 matches

3. **[installer_templates] `run()` → `run_http()`**

   - 完了条件: `grep "\.run()" scripts/mcp/installer_templates.py` → 0 matches

5. **[echo_server] docstring 修正**
   - "stdlib only" 記述を削除
   - 完了条件: `grep "stdlib only" scripts/mcp/echo_server.py` → 0 matches

6. **テスト更新・全テスト確認**
   - `uv run pytest` 全件パス確認
