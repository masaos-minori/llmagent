# Plan: H-6 — Add startup rollback shutdown for already-started MCP subprocesses

## Goal

Wrap the post-`_initialize()` startup steps in `StartupOrchestrator.run()` with a try/except so that if any step after `_start_servers()` raises, `lifecycle.shutdown_all()` is called before re-raising the original exception.

## Scope

**In**: `scripts/agent/startup.py` — `run()` method: add rollback logic using a `_servers_started` flag and `try/except/finally` pattern.

**Out**: No changes to `factory.py`, `http_lifecycle.py`, `_initialize()` logic, or any other module.

## Assumptions

1. Services (including `lifecycle`) are guaranteed to exist after `_initialize()` completes; calling `ctx.services_required.lifecycle.shutdown_all()` after `_initialize()` is safe.
2. `_initialize()` failures are NOT rolled back — services may not exist yet and subprocess startup hasn't occurred.
3. Rollback shutdown failures must be logged as warnings, not re-raised, so the original exception is preserved.
4. The rollback only applies when `_start_servers()` was reached (tracked by a boolean flag).

## Implementation Steps

1. **Refactor `run()` to track server start and wrap with rollback**
   - Add `_servers_started = False` before `_start_servers()`.
   - After `await self._start_servers()`, set `_servers_started = True`.
   - Wrap `_start_servers()` through `_setup_prompt()` in `try/except Exception`:
     ```python
     _servers_started = False
     try:
         await self._start_servers()
         _servers_started = True
         await self._check_services()
         await self._recover_pending_approvals()
         await self._setup_prompt()
     except Exception:
         if _servers_started:
             try:
                 await self._ctx.services_required.lifecycle.shutdown_all()
             except Exception as shutdown_err:
                 logger.warning("Rollback shutdown failed: %s", shutdown_err)
         raise
     ```
   - Criterion: `_initialize()` failures are not wrapped; rollback only fires after `_start_servers()`.

2. **Add tests**
   - Mock `_start_servers()` success and `_check_services()` raising `RuntimeError`; assert `shutdown_all()` is called.
   - Mock `_start_servers()` success and `_recover_pending_approvals()` raising; assert `shutdown_all()` is called.
   - Mock rollback `shutdown_all()` raising; assert original `_check_services()` exception is re-raised (not `shutdown_all()` exception).
   - Mock `_initialize()` raising; assert `shutdown_all()` is NOT called.
   - Mock `_start_servers()` raising before setting `_servers_started`; assert `shutdown_all()` is NOT called.
   - Criterion: `uv run pytest tests/test_startup.py -v` passes.
