# Plan: Remove `McpServerConfig._cast_enums()` compatibility shim

## Goal

Remove `McpServerConfig._cast_enums()` and replace it with `_validate_enum_types()` so that `McpServerConfig` requires normalized enum values on direct construction, while the TOML-based builder path (`_build_single_server()`) remains unchanged.

## Scope

**In-Scope**:
- Delete `McpServerConfig._cast_enums()` from `scripts/shared/mcp_config.py`
- Remove the call to `_cast_enums()` in `McpServerConfig.__post_init__()`
- Add `_validate_enum_types()` that raises `ValueError` when any of `transport`, `startup_mode`, or `healthcheck_mode` is not the expected enum instance
- Update `__post_init__()` to call `_validate_enum_types()` then `_validate_cross_fields()`
- Fix test helper functions in `tests/test_route_resolver.py`, `tests/test_lifecycle.py`, and `tests/test_cmd_mcp.py` that pass raw string values to `McpServerConfig(...)` directly
- Add a new test case that explicitly confirms valid string values (e.g. `"http"`) are rejected

**Out-of-Scope**:
- Removing enum conversion from `_build_single_server()`
- Changing TOML config file format
- Supporting string enum values anywhere in the direct construction path

## Assumptions

1. `TransportType`, `StartupMode`, and `HealthcheckMode` are all `StrEnum`; their `isinstance` check against the enum class is sufficient to distinguish enum instances from plain strings.
2. All test helpers that currently pass string literals (`"http"`, `"subprocess"`, etc.) are doing so because `_cast_enums()` silently accepted them. They must be updated to pass enum instances.
3. The `transport="stdio"` string used in `test_lifecycle.py` and `test_cmd_mcp.py` cannot be converted to a valid `TransportType` enum (stdio was removed). Those helpers are dead code or already skipped; they must be updated to use `TransportType.HTTP` or removed.
4. No production script under `scripts/` constructs `McpServerConfig(...)` with raw strings — confirmed by grep: only `_build_single_server()` constructs it, which already passes enum values.
5. The `healthcheck_mode` empty-string auto-inference currently inside `_cast_enums()` is already handled in `_build_single_server()`; it does not need to be replicated in `_validate_enum_types()`.
