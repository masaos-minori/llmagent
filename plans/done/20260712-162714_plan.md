## Design

1. 13グループそれぞれについて、front matter の `related:`(該当ファイルは`source:`も)から
   存在しない参照行を削除する。`03_rag_02_02_ingestion_pipeline-crawler-part1/2.md`と同じ
   スタイル(YAMLリストから該当行を除去するのみ、他の実在参照はそのまま残す)。
2. `03_rag_91_design_notes-part1.md` / `-part2.md` は、重複した `03_rag_91_design_notes.md`
   参照(2箇所)をどちらも削除する。
3. `tools/validate_docs_structure.py` に `check_related_links()` 関数を新規追加する:
   - `check_front_matter()`と同様にfront matterをYAMLパースする(共通化のため、front matter
     パース処理を切り出すか、`check_front_matter()`内のパース結果を再利用できるよう
     `validate_file()`側で1回のパース結果を両関数に渡す形にリファクタリングするかは実装時に判断)。
   - `related`/`source`の各エントリについて、`(path.parent / entry).resolve().is_file()`を
     `check_links()`(86〜95行)と同じロジックで検証する。
   - 存在しない場合 `f"{path.name}: front matter references missing file '{entry}' (field: related|source)"`
     形式で issue を返す。
   - `validate_file()`(98〜108行)の呼び出し列に追加する。
