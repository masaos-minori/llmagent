# H-2: Remove ToolResultStore writes from tool result collection

**Timestamp:** 20260708-100636
**Require:** requires/20260708_02_require.md
**Priority:** High

---

## Goal

`_collect_tool_result_msgs()` から `ctx.tool_result_store.store(...)` の呼び出しを削除し、
tool output のフルテキストを `tool_results` テーブルへ書き込まなくする。
tool role message の conversation flow (messages テーブル) への永続化は維持する。

---

## Scope

- **変更対象:** `scripts/agent/tool_runner.py` のみ
- **変更なし:** `scripts/db/tool_results.py`, `scripts/agent/context.py`, `scripts/agent/commands/cmd_tooling.py`
- **Path:** Path A (変更 <= 3 ファイル, インターフェース変更なし)

---

## Assumptions

- `ctx.tool_result_store` フィールド自体は `context.py` に残す。
  `/tool list` と `/tool show <id>` は `cmd_tooling.py` で引き続き `ctx.tool_result_store` を参照しており、
  これらのコマンドは本タスクのスコープ外のため削除しない。
- `_apply_turn_char_limit()` の `result_id: int` パラメータは不要になるため削除し、
  `TURN_LIMIT_HINT` をそのまま返すよう変更する (`id_hint` 付与ロジック削除)。
- テストファイル `tests/test_tool_runner.py` の `ctx.tool_result_store.store.assert_called_once()` は削除が必要。

## Unknowns

### U-1: `_apply_turn_char_limit()` の `result_id` 依存
**分析結果:** `result_id` は `TURN_LIMIT_HINT` 末尾に `(id=N)` を付与するためだけに使用されている
(tool_runner.py L258-266)。`store()` を削除すると `result_id` が存在しないため、このロジックも合わせて削除が必要。
`TURN_LIMIT_HINT` の末尾修飾を削除しても、ユーザーへの `/tool show <id>` 誘導ヒントは残す。
ただし `<id>` は示せなくなるため、ヒントのメッセージ自体も要修正。
→ **解決:** `_apply_turn_char_limit()` の `result_id` パラメータを削除し、
`TURN_LIMIT_HINT` をそのまま返す。ヒント文言中の `/tool show <id>` 部分は、
`tool_results` テーブルへの書き込みをやめる以上、IDが存在しないため文言も変更する。
→ **判断:** `TURN_LIMIT_HINT` は `tool_result_formatter.py` で定義されており複数箇所参照の可能性がある。
本タスクでは `_apply_turn_char_limit()` の id_hint ロジックのみ削除し、`TURN_LIMIT_HINT` 定数は変更しない。
ヒントメッセージはそのまま残すが、`(id=N)` の動的付与をやめる。

### U-2: `cmd_tooling.py` の `/tool show` との整合性
**分析結果:** `/tool show <id>` は `ctx.tool_result_store.get(tool_id)` を呼び出す。
本タスク後は新しいツール実行結果が `tool_results` テーブルに書き込まれなくなるため、
既存セッションの古いレコードのみ参照可能になる。
→ **解決:** これはアーキテクチャ上の意図した変更であり、本タスクのスコープ内。
`cmd_tooling.py` は変更不要 (スコープ外)。

## Design

### 変更の方針

1. `_collect_tool_result_msgs()` から以下を削除:
   ```python
   result_id = ctx.tool_result_store.store(
       session_id=ctx.session.session_id,
       turn=turn,
       tool_name=name,
       args_masked=_json_dumps(masked),
       full_text=text,
       summary=llm_text if summarized else None,
       is_error=is_error,
   )
   ```

2. `_apply_turn_char_limit()` のシグネチャ変更:
   - 変更前: `def _apply_turn_char_limit(llm_text, turn_chars, limit, result_id)`
   - 変更後: `def _apply_turn_char_limit(llm_text, turn_chars, limit)`
   - 内部の `id_hint` と `TURN_LIMIT_HINT.replace(...)` ロジックを削除
   - `return TURN_LIMIT_HINT` をそのまま返す

3. 呼び出し側の `_apply_turn_char_limit` 呼び出しから `result_id=result_id` 削除

4. `sqlite3` import は他の箇所 (`_sqlite3_error = sqlite3.Error`) で使われているので削除不要

5. `is_summarized()` 呼び出しと `summarized` 変数: `store()` 呼び出しが不要になっても
   `summarized` 変数への計算は依存が消えるため合わせて削除

### 維持すること

- `ctx.conv.history.append({"role": "tool", ...})` — LLM コンテキストへの追加
- `ctx.session.save_many(tool_msgs)` — messages テーブルへの永続化
- `_update_stats_for_result()` — 統計更新

## Risks

### R-1: `_apply_turn_char_limit` の `result_id` 削除によるヒント品質低下
**影響:** per-turn 制限時のヒントに `(id=N)` が付かなくなり、ユーザーが `/tool show N` でアーカイブを参照できなくなる
**緩和:** そもそも `tool_results` テーブルへの書き込みをやめる変更であり、IDが存在しないため参照不可能。ヒントからID参照を削除するのは整合性が取れた対応。既存の古いレコードへの影響なし。

### R-2: `tests/test_tool_runner.py` の `tool_result_store.store.assert_called_once()` が失敗
**影響:** テストが変更なしだと失敗する
**緩和:** テスト修正をソース変更と同時に実施する。

### R-3: `is_summarized()` の削除による影響
**影響:** `summarized` 変数は `store()` の `summary` パラメータにのみ使われていた。`store()` 削除後は `summarized` 計算も不要。
**緩和:** `is_summarized()` は `tool_result_formatter.py` に定義され `test_tool_result_formatter.py` で独立テスト済み。import は維持 (他用途での参照可能性) か削除するかは lint で確認する。使われなければ ruff が unused import として検出するので対応する。
