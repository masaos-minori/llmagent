## Design

### 情報分類の基準

| 情報種別 | 出力先ファイル |
|---|---|
| システム全体像・サーバー分類・主要コンポーネント | `04_mcp_01_system_overview.md` |
| プロトコル・HTTP API・stdio JSON-RPC・共通型 | `04_mcp_02_protocol_and_transport.md` |
| ルーティング・ライフサイクル・ToolExecutor・起動モード | `04_mcp_03_routing_lifecycle_and_execution.md` |
| 各サーバー（11台）の仕様 | `04_mcp_04_server_catalog.md` |
| セキュリティ制約・安全モデル・リスク分類 | `04_mcp_05_security_and_safety_model.md` |
| 設定ファイル・運用手順・監視・起動確認 | `04_mcp_06_configuration_and_operations.md` |
| 非一貫性・実装バグ・未実装・未定義挙動 | `04_mcp_90_inconsistencies_and_known_issues.md` |
| ソース → 再構成先のマッピング | `04_mcp_99_source_mapping.md` |
| 全ファイルへのナビゲーション | `04_mcp_00_document-guide.md` |

### canonical source の判断基準

- `06_ref-mcp.md` — ToolExecutor / HttpTransport / StdioTransport の詳細 API
- `04_spec_mcp.md` — システム全体仕様・McpServerConfig・起動モード・エラーハンドリング
- `04_mcp-protocol.md` — プロトコル詳細・ウォッチドッグ・新規サーバー追加手順
- `04_mcp-*.md`（各サーバー） — 各サーバーの canonical source
