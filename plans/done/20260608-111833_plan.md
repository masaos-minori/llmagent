# Refactoring: commands/ 層の backward-compat 除去と責務分離 — Work Plan

Source instruction: `instructions/20260608_04_instruction.md`

## Goal

`registry.py` の `mask_args` 後方互換 re-export を削除し、`cmd_session.py` の `_cmd_session` (CC=16) を分割することで、commands 層の公開 API を整理し保守コストを下げる。

## Scope

**In:**
- `registry.py`: `mask_args` re-export 削除、`__all__` 更新、モジュール docstring 更新
- `cmd_session.py`: `_load_session()` / `_generate_session_title()` を明示的なメソッドに整理し、`_cmd_session` の分岐を読みやすくする
- モジュール docstring から "Extracted from agent_commands.py" 等の旧来コメントを削除

**Out:**
- Service 層の新規作成（Session Switch Service 等）— 大規模変更のため別フェーズ
- `cmd_context.py` / `cmd_db.py` / `cmd_memory.py` の Service 層抽出 — 別フェーズ
- `cmd_config.py` / `cmd_ingest.py` / `cmd_tooling.py` の Presenter 分離 — 別フェーズ

## Assumptions

1. `mask_args` は `registry.py` 経由でインポートしているサイトが **ゼロ** であることを確認済み（grep 結果: 全 caller が `agent.tool_result_formatter` から直接 import している）。
2. `_budget_breakdown` は `registry.py` の `__all__` に含まれているが、外部参照サイトを確認する。
3. `cmd_session.py` の `_cmd_session` (CC=16) は既存テスト (`test_agent_cmd_session.py`: 12 件) がカバーしている。
