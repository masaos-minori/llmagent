# Plan: Replace module-global mutable state with app state in EventBus

Source: `requires/20260625_53_require.md`

---

## Goal

`scripts/eventbus/app.py` のモジュールグローバル変数（`_cfg`, `_db`, `_envelope_schema`, `_dlq_task`）を FastAPI app state または依存注入コンテナに移動し、テスト可能性と分離性を向上させる。

---

## Scope

**In-Scope**
- `scripts/eventbus/app.py`: `global _cfg, _db, ...` を `app.state.*` に移行
- lifespan 関数での初期化パターンの維持
- handler の依存参照を `request.app.state.*` または `Annotated[..., Depends(...)]` に変更
- テスト追加

**Out-of-Scope**
- FastAPI 以外への移行
- 外部 API 変更

---

## Assumptions

1. `app.py` の `lifespan` 関数で global 変数が初期化されている（確認済み）。
2. handler は module-global `_cfg`, `_db` を直接参照している（確認済み）。
3. FastAPI の `app.state` を使う方法が最小変更で実現できる。

## Implementation Steps

- [ ] 1. `lifespan` 関数内で `app.state.cfg = _cfg` 等を設定するよう変更

- [ ] 3. テストで独立した app インスタンスを生成できることを確認
