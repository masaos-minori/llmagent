# Plan: Add tests for CommandDef and handler consistency

## 1. Goal

Add regression tests that validate all built-in `CommandDef` entries in `command_defs.py` against the actual handler methods on `CommandRegistry`, so that mismatches between command definitions and handler implementations are caught before runtime.

## 2. Scope

**In-Scope**:
- Add tests verifying that every `CommandDef.handler` string exists as a method on `CommandRegistry`
- Add tests verifying `prefix=False` handlers can be called with no positional argument (i.e., zero required params)
- Add tests verifying `prefix=True` handlers accept exactly one `str` argument
- Add tests verifying `is_async=True` handlers are coroutine functions (`inspect.iscoroutinefunction`)
- Add tests verifying `is_async=False` handlers are not coroutine functions
- Tests must run without starting external MCP servers or any LLM/embed service

**Out-of-Scope**:
- Testing actual MCP server behavior
- Testing `/exit` command
- Testing sqlite-vec behavior
- Testing plugin command dispatch (`_dispatch_plugin`)
- End-to-end dispatch integration tests

## 3. Assumptions

1. `CommandRegistry` can be instantiated with a mock `AgentContext` in tests (as done in existing tests).
2. The single source of truth for built-in commands is `command_defs._COMMANDS`; `registry._COMMANDS` is a duplicate that will be reconciled in a separate task.
3. `inspect.signature()` correctly reflects the calling convention for bound methods on `CommandRegistry`.
4. "Zero required parameters" for exact-match handlers means: after removing `self`, the remaining parameters either have defaults or the signature accepts no further positional args.
5. Dispatch passes `""` (empty string) to exact-match handlers and a `str` (may be empty) to prefix handlers.
6. Tests are added to `tests/test_command_defs.py` (already exists and covers `_COMMANDS` structure) or a new dedicated file ‚Äî see Unknowns.

| UNK-03 | Where to place new tests: extend `tests/test_command_defs.py` or create `tests/test_command_registry_consistency.py`? | No explicit rule | Existing `test_command_defs.py` covers only dataclass structure; consistency tests with `CommandRegistry` require importing `registry.py` which has heavier deps ‚Äî separate file is cleaner | False |

**Note on UNK-01**: `command_defs._COMMANDS` is declared the single source of truth per its module docstring. Tests will target this list. Divergences in `registry._COMMANDS` are a separate issue.

**Note on UNK-02**: Inspection of `registry.py` dispatch (lines 299-302) shows exact-match handlers are always called as `handler("")`. `_cmd_help(self)` has no `args` parameter, so `handler("")` would fail at runtime. The test should verify that exact-match handlers either have zero required params (and the dispatch path must be fixed to call `handler()` not `handler("")`) OR accept one optional `str`. This ambiguity does not block test writing ‚Äî write tests against the contract stated in the requirement (zero required params for exact, one param for prefix) and let test failures surface the bug.
