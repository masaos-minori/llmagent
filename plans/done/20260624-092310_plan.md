# Plan: Add Connection Reuse Option to SQLiteHelper #2

**Requirement:** `requires/20260623_15_require.md`
**Task size:** Path B (Large — public interface change on `open()`, thread-safety constraints)
**See also:** `plans/20260624-092027_plan.md` (identical scope from req 13)

---

## Goal

Add an optional `reuse_connection: bool = False` parameter to `SQLiteHelper.open()` that, when `True`, causes `__exit__` to skip `close()`, allowing connection reuse across multiple `with` blocks in the same execution scope.

---

## Scope

**In:**
- `scripts/db/helper.py` — add `_reuse_connection` instance var; add `reuse_connection` param to `open()`; modify `__exit__` to conditional close
- `tests/test_db_performance.py` — `test_connection_reuse_same_instance` and `test_backward_compatibility_per_query`

**Out:**
- Thread-local connection pool — not required; caller is responsible for instance isolation per thread
- Profiling infrastructure — AC mentions profiling; out of scope for this plan (code change only)

---

## Assumptions

1. `reuse_connection=True` transfers close responsibility to the caller (explicit `close()` required).
2. All 4 existing call sites use default `open()` — backward compatible (confirmed by grep).
3. Tests use in-memory SQLite + `patch("db.helper.build_db_config")` (pattern from `test_sqlite_helper.py`).

## Design

### `scripts/db/helper.py`

```python
# __init__: add
self._reuse_connection: bool = False

# open(): add param
def open(self, *, write_mode=False, row_factory=False, load_vec=None, reuse_connection=False):
    self._reuse_connection = reuse_connection
    if self.conn is not None and self._reuse_connection:
        return self  # skip reconnect
    # ... existing logic ...

# __exit__: conditional close
def __exit__(self, *_):
    if not self._reuse_connection:
        self.close()
```
