# Plan: Add explicit configuration validation for Event Bus value ranges

Source: `requires/20260625_51_require.md`

---

## Goal

EventBus の `config.py` に数値範囲チェック・必須パス検証を追加し、無効な設定を startup 時に明確なエラーとして拒否する。

---

## Scope

**In-Scope**
- `scripts/eventbus/config.py`: `poll_interval_ms`, `offset_checkpoint_interval`, `max_retry` の範囲検証
- path-like フィールドの非空チェック
- 無効設定時の明確なエラーメッセージ
- テスト追加

**Out-of-Scope**
- config 構造の変更

---

## Assumptions

1. `config.py` line 27-44 に `EventBusConfig` dataclass があり、minimal validation のみ（確認済み）。
2. 現行は `data["max_retry"]` を直接取得しており、欠落時は KeyError のみ。

- [ ] 1. `EventBusConfig` に `__post_init__` バリデーション追加:
