Source instruction: `instructions/20260604_01_instruction.md`

## Goal

`agent/` の後方互換レイヤーを除去し、構造化 API への完全移行・不要ファイルの削除・LLM エラー処理の統一を行う。

---

## Scope

**In:**
- `config.py` の `__getattr__`/`__setattr__` backward-compat 除去
- `context.py` のフラットアクセス backward-compat 除去
- `repl_debug.py`, `context_detection.py`, `rag_debug.py` の削除
- `factory.py` の `_audit_logger_instance` グローバル除去
- `session.py` のリポジトリ分割（クラスを別ファイルへ移動）

**Out:**
- `orchestrator.py` / `llm_turn_runner.py` の LLM エラー統一（別タスクとして分離推奨）
- テスト追加（影響範囲が広いため `python-test-and-fix` スキルと連携）

---

## Assumptions

1. `config.py` の backward-compat を除去した場合、呼び出しサイトで `cfg.llm_url` → `cfg.llm.llm_url` などの修正が必要。
2. `repl_debug.py` / `context_detection.py` / `rag_debug.py` は現在のコードから参照されていない（調査済み）。
3. 変更前に `uv run pytest` で動作確認ベースラインを確立する。

## Implementation Steps

### Step 1: 不要ファイルの削除（影響なし・優先度高）
- `scripts/agent/repl_debug.py` を削除
- `scripts/agent/context_detection.py` を削除  
- `scripts/agent/rag_debug.py` (`scripts/agent/` 内のもの) を削除
- `uv run pytest` で既存テストの通過を確認

### Step 2: `config.py` の backward-compat 除去
- `AgentConfig.__getattr__` / `__setattr__` を削除
- `grep -r "cfg\." scripts/` で flat access を使っている箇所を列挙
- 全箇所を `cfg.llm.llm_url` 等の nested access に変換
- `AgentConfig` と `DbConfig` を別ファイルに分割（`agent/agent_config.py` / `agent/db_config.py` 等）
- `python-refactoring` スキルを使用する

### Step 3: `context.py` の backward-compat 除去
- `AgentContext` の flat attribute compat を削除
- 全呼び出しサイトを `ctx.conv` / `ctx.turn` / `ctx.stats` / `ctx.services` に変換
- `python-refactoring` スキルを使用する

### Step 4: `factory.py` の `_audit_logger_instance` グローバル除去
- `_audit_logger_instance` をランタイム設定から構築する `_build_audit_logger(ctx)` 経由に統一
- グローバルインスタンスを除去

### Step 5: `session.py` リポジトリ分割
- `SessionMessageRepository` → `agent/session_message_repo.py`
- `DocumentRepository` → `agent/document_repo.py`  
- `NoteRepository` → `agent/note_repo.py`
- `AgentSession` を facade として維持し、各 repo を内部に持つ
- `python-refactoring` スキルを使用する

### Step 6: バリデーション実行
- `uv run ruff check scripts/`
- `uv run mypy scripts/`
- `lint-imports`
- `uv run pytest`

### Step 7: instructions/done/ へ移動
- `instructions/20260604_01_instruction.md` を `instructions/done/` に移動する
