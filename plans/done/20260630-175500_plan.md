# Plan: Synchronize Event Bus schema documentation with `schema.sql` and `event_envelope.json`

## Goal

4つのEvent Busドキュメントが `scripts/eventbus/schema.sql` および `schemas/event_envelope.json` の内容と完全に一致した状態にする。

## Scope

**In-Scope**:
- `docs/06_eventbus_03_persistence_schema_and_replay.md` — DBスキーマDDL・フィールドセマンティクス・インデックスの検証と補足
- `docs/06_eventbus_02_http_api_and_runtime.md` — publishエンベロープ制約テーブルへの `additionalProperties: false` 明示追加
- `docs/06_eventbus_06_reference_api.md` — スキーマフィールドおよびインデックスへの言及が存在しないため、`scripts/eventbus/app.py`/`db.py` に対応する参照注記を追加
- `docs/06_eventbus_90_inconsistencies_and_known_issues.md` — Schema vs Implementation Differences テーブルの検証と `dlq_requeue_count` 記述の確認

**Out-of-Scope**:
- `scripts/eventbus/schema.sql`、`schemas/event_envelope.json` のコード変更は行わない
- テストの追加・変更は行わない
- `06_eventbus_01_system-overview.md`、`06_eventbus_04_dlq_offsets_and_delivery_semantics.md`、`06_eventbus_05_configuration_deploy_and_operations.md` の変更は行わない

## Assumptions

- `scripts/eventbus/schema.sql` が唯一の権威ある DDL 定義であり、現行のデプロイ済みスキーマと一致している
- `schemas/event_envelope.json` が publish エンベロープの唯一の権威ある JSON Schema 定義である
- `06_eventbus_03_persistence_schema_and_replay.md` は既に `schema.sql` と実質一致しており、細部の確認のみで大規模な書き直しは不要

2. **Phase 2: Core Implementation**
   - [ ] `docs/06_eventbus_03_persistence_schema_and_replay.md`

     - [ ] POST /publish 制約テーブルに「追加プロパティ不可 (`additionalProperties: false`)」を記述する行または注記を追加

     - [ ] `published_at` の format が `date-time` であることの記述が正確であることを確認

     - [ ] `scripts/eventbus/db.py` の `open_db` 説明箇所に、スキーマ初期化 (`schema.sql`) を参照する注記を追加
