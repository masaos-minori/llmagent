## Goal

Extend `tools/check_no_compat.py` with a new pattern group that catches workflow compatibility and direct-execution patterns, and wire it into the `no-compat-stubs` pre-commit hook so CI fails if any forbidden pattern reappears.

## Scope

**In**:
- Add 10 new forbidden patterns to `COMPAT_PATTERNS` in `tools/check_no_compat.py`:
  - `workflow_mode` field reference: `r"workflow_mode\s*[=:]"` (config assignment or type annotation)
  - `allow_startup_fallback`: `r"allow_startup_fallback\b"`
  - `is_workflow_enabled`: `r"is_workflow_enabled\b"`
  - `mode="disabled"` literal: `r'mode\s*=\s*["\']disabled["\']'`
  - `mode="auto"` literal: `r'mode\s*=\s*["\']auto["\']'`
  - `workflow_mode="disabled"`: `r'workflow_mode\s*=\s*["\']disabled["\']'`
  - `workflow_mode="auto"`: `r'workflow_mode\s*=\s*["\']auto["\']'`
  - `direct LLM path` phrase: `r"direct\s+LLM\s+path"`
  - `direct-execution fallback` phrase: `r"direct.execution\s+fallback"`
  - `Workflow mode=disabled` log string: `r'Workflow\s+mode=disabled'`
- Add to `DEFAULT_ALLOWLIST`:
  - `tests/test_workflow_execution_policy.py` (contains patterns as test data — deleted after req13, so add now and remove entry when file is deleted)
  - `tests/test_check_no_compat.py` (contains patterns as test fixtures)
  - `plans/` directory entries (plan documents may mention forbidden patterns for documentation)
  - `requires/done/` entries (requirement documents mention patterns)
  - This plan file itself: `tools/check_no_compat.py` (already in allowlist via its own docstring)
- Add a test in `tests/test_check_no_compat.py` for each new pattern: assert it is detected and reported
- Add an allowlist test: assert that allowlisted files pass through without error

**Note on `LLMTurnRunner.run()` without workflow context**: The pattern `LLMTurnRunner.run(` without workflow context is a semantic check that static regex cannot reliably detect (it would require call-graph analysis). This is **explicitly excluded** from the regex-based scan and is instead enforced through the type system after plan 20260707-103631 makes workflow context required parameters.

**Out**:
- Semantic code analysis beyond regex
- Automatic code rewriting on pattern detection
- Scanning files outside `scripts/`, `docs/`, `tests/`
- Changing the CI runner (patterns added to existing `no-compat-stubs` hook)

## Assumptions

1. `tools/check_no_compat.py` already integrates with pre-commit via the `no-compat-stubs` hook in `.pre-commit-config.yaml`.
2. The hook runs on `types_or: [python, markdown]` — the new patterns apply to Python files; this is sufficient.
3. Plan documents (`plans/*.md`) are not scanned by the hook (it only targets `scripts/`, `docs/`, `tests/`).
4. `requires/done/*.md` files are not scanned either (outside the default scan dirs).
5. After req13 (plan 20260707-105309), `test_workflow_execution_policy.py` will be deleted — the allowlist entry for it can be removed at that point.
6. This plan is Path A (≤3 files, no DB/runtime interface changes).
