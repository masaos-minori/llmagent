# Shrink REPL into a pure UI adapter

**Requirement:** `requires/20260624_64_require.md`

## Goal

Remove all workflow execution state ownership from `AgentREPL` and `cmd_workflow.py`. The REPL reads state from durable storage (via Orchestrator API or StateStore) rather than from in-memory flags.

## Current State

Three concrete violations of the "thin UI adapter" contract:

1. **`repl.py:402-411` (`_get_workflow_status()`)**: accesses `self._orchestrator._workflow_mode` and `self._orchestrator._workflow_def` — private attributes. REPL should call a public API.

2. **`cmd_workflow.py:23` and `cmd_workflow.py:49`**: `_cmd_approve()` / `_cmd_reject()` read `self._ctx.turn.pending_approval_id` (display cache) for the approval ID. If this is `None` (after REPL restart), the command silently fails. This is the root cause addressed by req #60 plan (`20260625-092947_plan.md`).

3. **`cmd_workflow.py:44` and `cmd_workflow.py:70`**: `_cmd_approve()` / `_cmd_reject()` clear `ctx.workflow.approval_pending` and `ctx.turn.pending_approval_id` — display cache mutation that belongs in the display layer but is acceptable as a cache clear.

## Scope

**In:**
- `scripts/agent/orchestrator.py` — add `workflow_status() -> dict[str, str]` public method returning `mode`, `tracking_enabled` (removes REPL's access to private `_workflow_mode` / `_workflow_def`)
- `scripts/agent/repl.py` — change `_get_workflow_status()` to call `self._orchestrator.workflow_status()` instead of accessing private attrs
- `scripts/agent/commands/cmd_workflow.py` — change `_cmd_approve()` / `_cmd_reject()` to query `StateStore.find_latest_pending_approval()` (per req #60 plan) instead of reading `ctx.turn.pending_approval_id`; note: this is the req #60 fix, included here for completeness
- `scripts/agent/startup.py` — change `_recover_pending_approvals()` to use `find_latest_pending_approval()` (per req #60 plan)

**Out:**

## Assumptions

1. `Orchestrator._workflow_mode` and `_workflow_def` are private (single underscore). Adding a public `workflow_status()` method is the cleanest way to expose this without making private attrs public.
2. `CLIView.write_startup_banner()` already accepts `workflow_status: str` — no change needed there.
3. `cmd_workflow.py`'s cache-clearing (`ctx.workflow.approval_pending = False`, `ctx.turn.pending_approval_id = None`) is acceptable in the display layer — it's clearing the UI cache, not the authoritative state.
4. The `/workflow status` or `/approve` / `/reject` commands are the only commands that read workflow/approval state. Confirmed by grep: no other commands read `ctx.workflow.*`.

## Design

### `Orchestrator.workflow_status()`

```python
def workflow_status(self) -> dict[str, str]:
    """Return public workflow status for display purposes.
    
    Keys:
      mode: "auto" | "required" | "disabled"
      tracking: "enabled" | "not_loaded"
    """
    return {
        "mode": self._workflow_mode,
        "tracking": "enabled" if self._workflow_def is not None else "not_loaded",
    }
```

### `repl.py._get_workflow_status()` change

```python
def _get_workflow_status(self) -> str:
    if self._orchestrator is None:
        return "unknown"
    status = self._orchestrator.workflow_status()
    mode = status["mode"]
    if mode == "disabled":
        return "disabled"
    tracking = status["tracking"]
    if tracking == "enabled":
        return f"{mode} (tracking enabled)"
    return f"{mode} (definition not loaded)"
```
