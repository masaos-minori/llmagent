# Memory Layer Backward-Compat Removal — Work Plan

Source instruction: `instructions/20260605_02_instruction.md`

## Goal

`agent/memory/` の backward-compat コードを除去し、公開 API 境界を明確化する。

## Scope

**In:**
- `layer.py`: 旧コンストラクタ backward-compat 除去、`_fetch_embedding` 再エクスポート除去、SQLite ハウスキーピングをストア層へ移動
- `types.py`: `SOURCE_TYPES` 定数除去（`SourceType` に統一）
- `retriever.py`: モジュールレベル定数除去、`_vec_search()` を非公開維持しつつ公開 dedup API を追加、BLOB 変換ロジック統一
- `scoring.py`: `ScoringPolicy` をスコアリングの唯一の源泉に
- `ingestion.py`: `DedupAction.LINK_ONLY` 除去（compat のみなら）、`_vec_search()` 直接呼び出しを公開 API 経由に
- `store.py`: `memory_links` 書き込みをストア層に集約、BLOB ロジック統一
- `embedding_client.py`: モジュールレベル `_fetch_embedding()` パスを除去
- `injection.py`: `dedup_window` フィールド除去、検索とフォーマットを分離
- `extract.py`: 抽出ルールをポリシーオブジェクトに移動

**Out:** 動作仕様の変更

## Assumptions

1. すべての backward-compat 機能は `python-test-and-fix` スキルでテストを先行させてから除去する。
2. `ScoringPolicy` は `scoring.py` にすでに存在する。

## Unknowns

| Unknown | Evidence missing | Resolution | Blocking |
|---|---|---|---|
| `DedupAction.LINK_ONLY` が compat のみか | `ingestion.py` 未確認 | 実装前に確認する | no |
| `SOURCE_TYPES` の呼び出しサイト | `types.py` 外の参照先 | `grep -r "SOURCE_TYPES"` で確認 | no |

## Affected areas

`agent/memory/` の全 9 ファイル（layer/types/retriever/scoring/ingestion/store/embedding_client/injection/extract）

## Design

priority 順（指示書 §Priority order より）:
1. `layer.py` 旧 compat 除去
2. `types.py` `SOURCE_TYPES` 除去
3. retriever/scoring スコアリング統一
4. ingestion dedup API 公開
5. store 永続化集約
6. injection/extract 責任分離

## Implementation steps

1. `python-test-and-fix` スキルで各ファイルの behavior-lock テストを先行作成する。
2. priority 順に各ファイルの backward-compat を除去する。
3. `uv run ruff`/`mypy`/`lint-imports`/`pytest` でバリデーション。

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| compat 除去でテスト破壊 | high | behavior-lock テストを先行、incremental に変更 |
| BLOB 変換ロジック統一で既存 DB の値が読めなくなる | medium | 変換関数は同一ロジックを保つ、read/write で対称性を保証 |
