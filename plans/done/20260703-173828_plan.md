## Goal

Fix `ToolExecutor._execute_with_stampede_protection()` so that when `_raw_execute()` raises an exception, concurrent waiters awaiting the inflight future receive the exception instead of hanging indefinitely.

## Scope

**In-Scope:**
- Update `_execute_with_stampede_protection()` in `scripts/shared/tool_executor.py` with proper `except`/`else`/`finally` structure
- New test file `tests/shared/test_tool_executor_stampede.py`

**Out-of-Scope:**
- Moving `_inflight` into `ToolResultCache`
- Changing cache key format
- Changing the concurrent success path behavior
- `deploy/deploy.sh`: no change

## Assumptions

1. The current bug is that if `_raw_execute()` raises, the `inflight` future is never resolved (no `set_result` or `set_exception`), so any concurrent caller already `await`-ing the future hangs forever.
2. The `finally` block removing from `_inflight` runs correctly in both success and exception paths — this is not the bug; the bug is the unresolved future.
3. `inflight.done()` guard is necessary because a concurrent waiter could theoretically cancel the future between creation and setting it.
4. `tests/shared/__init__.py` will exist (created by require-22 or require-23 plan).

## Design

### Current buggy implementation (lines 306-327)

```python
async def _execute_with_stampede_protection(
    self,
    cache_key: str,
    tool_name: str,
    args: dict[str, Any],
) -> ToolCallResult:
    inflight = self._inflight.get(cache_key)
    if inflight is not None and not inflight.done():
        return await inflight
    if inflight is not None and inflight.done():
        return inflight.result()
    loop = asyncio.get_running_loop()
    inflight = loop.create_future()
    self._inflight[cache_key] = inflight
    try:
        result = await self._raw_execute(tool_name, args)
        if not inflight.done():
            inflight.set_result(result)
        return result
    finally:
        self._inflight.pop(cache_key, None)
```

**Bug:** On `_raw_execute()` exception, execution jumps directly to `finally` (which cleans up `_inflight`), but `inflight` is never resolved — any concurrent caller stuck on `await inflight` hangs forever.

### Fixed implementation

```python
async def _execute_with_stampede_protection(
    self,
    cache_key: str,
    tool_name: str,
    args: dict[str, Any],
) -> ToolCallResult:
    inflight = self._inflight.get(cache_key)
    if inflight is not None and not inflight.done():
        return await inflight
    if inflight is not None and inflight.done():
        return inflight.result()
    loop = asyncio.get_running_loop()
    inflight = loop.create_future()
    self._inflight[cache_key] = inflight
    try:
        result = await self._raw_execute(tool_name, args)
    except Exception as e:
        if not inflight.done():
            inflight.set_exception(e)
        raise
    else:
        if not inflight.done():
            inflight.set_result(result)
        return result
    finally:
        self._inflight.pop(cache_key, None)
```

**Fix rationale:**
- `except` branch: sets exception on inflight before re-raising, so concurrent waiters get the exception propagated
- `else` branch: only executed on success (no exception); sets result and returns
- `finally` branch: always cleans up `_inflight` entry regardless of outcome

### Test structure

`tests/shared/test_tool_executor_stampede.py` — class `TestStampedeProtection`:

Tests require a minimal `ToolExecutor` instance. Use a fixture that patches `_raw_execute` to avoid actual HTTP calls.

- `test_concurrent_success_calls_raw_execute_once` — two concurrent calls with same cache key → `_raw_execute` called once, both return same result
- `test_concurrent_exception_both_receive_exception` — `_raw_execute` raises `RuntimeError` → both concurrent callers raise the same exception

- `test_error_tool_result_not_cached` — `_raw_execute` returns `is_error=True` result → `_cache` is not populated
