# Plan: Remove all client/deployment/operational references to /v1/search

Source: `requires/20260625_72_require.md`

---

## Goal

`/v1/search` を呼ぶ全クライアント（`rag/pipeline_service.py`, `rag/pipeline.py`）を canonical MCP tool call（`POST /v1/call_tool` の `rag_run_pipeline`）に移行する。

---

## Scope

**In-Scope**
- `scripts/rag/pipeline_service.py`: `/v1/search` 呼び出しを `rag_run_pipeline` の MCP tool call に置き換え
- `scripts/rag/pipeline.py`: HTTP mode の `rag_service_url` 分岐を更新
- deployment スクリプト・verification 手順の更新

**Out-of-Scope**
- `rag_service_url` 設定の廃止（設定は残す、呼び出し先 URL のみ変更）

---

## Assumptions

1. `pipeline_service.py:105` の `{rag_url}/v1/search` を `{rag_url}/v1/call_tool` + `{"name": "rag_run_pipeline", ...}` に変更する。
2. MCP call の request/response 形式は `{"name": str, "args": dict}` / `{"result": str, "is_error": bool}`。
3. `pipeline.py` の HTTP mode は `rag_service_url` が設定されている場合のみ有効。
