# Plan: Relax `_get_handler()` typing to support no-argument exact handlers

**Source requirement**: `requires/20260702_62_require.md`
**Task classification**: Path A (Small Task — 1 file affected, no interface or schema changes)

---

## Goal

Change `_get_handler()` in `scripts/agent/commands/registry.py` to return `Callable[..., Any]` so that exact-match handlers with no arguments are correctly typed and `dispatch()` no longer needs to call them with a spurious `""` argument.

---

## Scope

**In**:
- Update `_get_handler()` return type annotation from `Callable[[str], None] | Callable[[str], Awaitable[None]]` to `Callable[..., Any]`
- Remove unused `is_async: bool` parameter from `_get_handler()` signature (it is accepted but never used inside the method)
- Keep the `AttributeError` for missing handlers

**Out**:
- Full command signature framework redesign
- `/exit` handling
- Plugin dispatch changes (`_dispatch_plugin`)
- `command_defs.py` changes

---

## Assumptions

1. `_get_handler()` is called only from `dispatch()` within `registry.py`; no external callers exist (confirmed by grep — no hits outside `registry.py`).
2. The `is_async` parameter passed to `_get_handler()` is redundant: `dispatch()` already uses `cmd.is_async` directly to decide whether to `await` the returned callable.
3. Removing `is_async` from `_get_handler()` will not affect `dispatch()` logic, because `dispatch()` reads `cmd.is_async` independently.
4. Existing `# type: ignore[misc]` comments in `dispatch()` can be removed or retained as-is after the return type change; removal is preferred if mypy passes cleanly.

## Implementation Steps

1. **Update `_get_handler()` return type and remove unused parameter**
   - [ ] Change return type from `Callable[[str], None] | Callable[[str], Awaitable[None]]` to `Callable[..., Any]`
   - [ ] Remove the `is_async: bool` parameter from the method signature (positional-only `/` can be dropped along with it)
   - [ ] Add `Any` to the import from `collections.abc` (or import from `typing` — follow existing import style)
   - [ ] Remove `# type: ignore[no-any-return]` comment on the `return handler` line if mypy no longer requires it

2. **Update `dispatch()` call site**
   - [ ] Change `self._get_handler(cmd, cmd.is_async)` to `self._get_handler(cmd)`
   - [ ] Verify that `await handler(args)` and `handler(args)` for prefix commands still work
   - [ ] For exact-match commands, change `handler("")` calls to `handler()` (no argument) — this is the primary behavioral fix described in the requirement
   - [ ] Remove `# type: ignore[misc]` on `await handler(args)` / `await handler("")` lines if mypy passes

3. **Validate**
   - [ ] Run `uv run pytest tests/test_cmd_registry_ingest_removal.py tests/test_cmd_registry_note_removal.py tests/test_cmd_plugins.py tests/test_repl.py -x -q`
   - [ ] Run `uv run pytest -x -q` (full suite)
   - [ ] Run `ruff check scripts/agent/commands/registry.py`
   - [ ] Run `mypy scripts/agent/commands/registry.py`
