# 作業計画: DB 層責務整理・ConfigLoader 分散排除・silent fallback 廃止

instruction: `instructions/20260609_05_instruction.md`

## Goal

`db/` 層 6 ファイルから分散した `ConfigLoader` 直接読込・class-level 状態キャッシュ・silent fallback・後方互換前提を除去し、`DbConfig` を唯一の設定入口とした明確な責務構造へ移行する。

---

## Scope

**In** (Phase 1 — 安全な変更に限定):
- `scripts/db/config.py` — `DbConfig.__post_init__()` の path 実在チェックを「ファイル存在必須」から「親ディレクトリ存在 + 書込可能性」へ緩和；`sqlite_busy_timeout_ms` を追加
- `scripts/db/helper.py` — `_ensure_config()` 内の `ConfigLoader` を `build_db_config()` に置換；class-level キャッシュはひとまず維持して移行コストを抑える
- `scripts/db/store.py` — `get_embedding_dims()` / `get_embedding_bytes()` の `ConfigLoader` 直読込を `build_db_config()` 経由に変更
- `scripts/db/maintenance.py` — `RetentionConfig.from_config()` と `_get_archive_dir()` などの `ConfigLoader` 直読込を `build_db_config()` 経由に変更；`SQLiteHelper._RAG_PATH` / `_SESSION_PATH` private 依存を `SQLiteHelper._ensure_config()` 後の属性アクセスではなく `build_db_config()` で取得
- `scripts/db/tool_results.py` — `store()` の silent fallback にログ追加；`get()` / `list_recent()` の None 返却と DB 異常の区別を明記するコメント追加 (完全な型変更は別 Phase)
- `scripts/db/create_schema.py` — 固定パス `/opt/llm/logs/create_schema.log` を設定から取得；`_SAFE_MIGRATION_ERRORS` にコメントで廃止予告を追加

**Out** (Phase 2/3 は別タスク):
- `SQLiteHelper` の DI 化 (83+ 外部参照のため一括変更は巨大すぎる)
- `create_schema.py` の versioned migration 再設計
- `store.py` の `dict[str, Any]` → TypedDict 化
- `tool_results.py` の完全な結果型変更
- `SQLiteHelper` の `health_check()` / `checkpoint()` / `vacuum()` 移管
- DB スキーマ変更なし

## Assumptions

1. `SQLiteHelper` の `_ensure_config()` / class-level 属性 (`_RAG_PATH` 等) は Phase 1 では削除しない。外部参照が 83+ あり、一括変更は非現実的。`_ensure_config()` 内の `ConfigLoader` 直読込を `build_db_config()` に置き換えるだけで、外向きの API 変更なし。
2. `DbConfig` の `__post_init__` が「ファイル存在チェック」を行っているため、新規 DB 作成フロー（ファイルがまだ存在しない）で `build_db_config()` が例外を出す。この問題は `__post_init__` 変更で解決する。
3. `build_db_config()` は `ConfigLoader().load_all()` を使っており、`common.toml` を確実に読む。`helper.py` の `_ensure_config()` は `ConfigLoader().load("common.toml")` を使っている。両者はほぼ等価。
4. `maintenance.py` の `SQLiteHelper._ensure_config()` 呼出後の `SQLiteHelper._RAG_PATH` 参照は、`build_db_config().rag_db_path` に置き換えてもよい（同一の値を返すため）。
5. `create_schema.py` の固定ログパスは `common.toml` の `log_dir` キーで代替できる。

## Design

### D-1: `DbConfig.__post_init__` 緩和

「ファイル存在必須」を「親ディレクトリ存在 + 書込可能性チェック」へ変更。DB ファイルが未存在の場合は自動作成されるため、実在チェックは不適切。

```python
# rag_db_path/session_db_path: 親ディレクトリが存在すれば OK
parent = Path(self.rag_db_path).parent
if not parent.exists():
    raise ValueError(f"rag_db_path parent directory does not exist: {parent}")
```

### D-2: `helper.py` ConfigLoader 置換

```python
# Before:
cfg = ConfigLoader().load("common.toml")
cls._RAG_PATH = cfg.get("rag_db_path", "")
...

# After:
from db.config import build_db_config
try:
    db_cfg = build_db_config()
except Exception as e:
    logger.warning(f"DbConfig load failed: {e}")
    return  # leave _config_loaded = False
cls._RAG_PATH = db_cfg.rag_db_path
cls._SESSION_PATH = db_cfg.session_db_path
cls.SQLITE_VEC_SO = db_cfg.sqlite_vec_so
cls._SQLITE_TIMEOUT = db_cfg.sqlite_timeout
cls._BUSY_TIMEOUT_MS = db_cfg.sqlite_busy_timeout_ms  # new field
```

### D-3: `store.py` embedding dims

```python
# Before:
cfg = ConfigLoader().load("common.toml")
return int(cfg.get("embedding_dims", 384))

# After:
from db.config import build_db_config
db_cfg = build_db_config()
return db_cfg.embedding_dims  # new field on DbConfig
```

### D-4: `maintenance.py` ConfigLoader → build_db_config

各 `ConfigLoader().load("common.toml")` を `build_db_config()` に統合。
`SQLiteHelper._RAG_PATH` / `_SESSION_PATH` 参照を `build_db_config()` 呼出に変更。

### D-5: `tool_results.py` ロギング追加

`store()` の `except Exception as e:` に `logger.exception(...)` を追加し、silent fallback を可視化。

## Implementation Steps

1. **[config] `DbConfig` に `sqlite_busy_timeout_ms` と `embedding_dims` を追加；`__post_init__` のパス検証を緩和**
   - `sqlite_busy_timeout_ms: int = 30000` フィールドを追加
   - `embedding_dims: int = 384` フィールドを追加
   - `build_db_config()` でこれらを読み込む
   - `__post_init__` のファイル存在チェックを親ディレクトリ存在チェックへ変更
   - 完了条件: `build_db_config()` が DB ファイル未存在時に例外を出さない

2. **[helper] `_ensure_config()` の `ConfigLoader` → `build_db_config()` 置換**
   - `from shared.config_loader import ConfigLoader` を `from db.config import build_db_config` に変更
   - `_ensure_config()` 内を `build_db_config()` 経由に変更
   - `_BUSY_TIMEOUT_MS` を `db_cfg.sqlite_busy_timeout_ms` から読む
   - 完了条件: `helper.py` に `ConfigLoader` の import が残らない

3. **[store] `get_embedding_dims()` の `ConfigLoader` → `build_db_config()` 置換**
   - `ConfigLoader().load(...)` を `build_db_config().embedding_dims` に変更
   - 完了条件: `store.py` に `ConfigLoader` の import が残らない

4. **[maintenance] `ConfigLoader` 直読込と private 属性依存を除去**
   - `RetentionConfig.from_config()` を `build_db_config()` 経由に変更
   - `_get_archive_dir()` 等の `ConfigLoader` 読込を `build_db_config()` に変更
   - `SQLiteHelper._ensure_config()` + `SQLiteHelper._RAG_PATH` 参照を `build_db_config()` に変更
   - 完了条件: `maintenance.py` に `ConfigLoader` import と `SQLiteHelper._RAG_PATH` 参照が残らない

5. **[tool_results] silent fallback にロギング追加**
   - `store()` の `except Exception as e:` に `logger.exception()` 追加
   - 完了条件: `logger` が実際のログ出力に使われている

6. **[create_schema] 固定ログパス設定化・`_SAFE_MIGRATION_ERRORS` 廃止予告**
   - `Logger(__name__, "/opt/llm/logs/create_schema.log")` を設定値から取得
   - `_SAFE_MIGRATION_ERRORS` に `# DEPRECATED: will be removed after versioned migration is adopted` コメント追加
   - 完了条件: 固定パスが消える；廃止予告が追記される

7. **テスト更新・全テスト確認**
   - `uv run pytest` 全件パス確認

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `DbConfig.__post_init__` 緩和で既存の検証が弱くなる | low | 親ディレクトリ存在チェックで代替；空文字チェックは維持 |
| `build_db_config()` が `ConfigLoader` を使っており、helper.py から呼ぶと循環 import が発生する可能性 | medium | `_ensure_config()` 内部の lazy import で回避；`db.config` は `db.helper` を import しないため循環なし |
| `maintenance.py` の `_RAG_PATH` 参照除去で `rotate_rag_db()` 等の動作が変わる | low | `build_db_config().rag_db_path` は `_RAG_PATH` と同一値を返すため動作変更なし |
| `DbConfig` に新フィールドを追加した際、既存の `DbConfig(...)` 直接構築コードが引数不足でエラーになる | low | `sqlite_busy_timeout_ms` と `embedding_dims` はデフォルト値付きで追加するため既存コードに影響なし |
