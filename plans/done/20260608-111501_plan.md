# Refactoring Phase 2: factory.py / history.py 分解 — Work Plan

Source instruction: `instructions/20260608_02_instruction.md`

## Goal

`factory.py` のサービスビルダーを薄い bootstrap モジュールへ再編成し、`history.py` の圧縮選択ロジックとサマリー生成を分離することで、単体テスト容易性を向上させる。

## Scope

**In:**
- `factory.py`: `build_agent_context()` → `agent/bootstrap.py` でラップし、`factory.py` はビルダー関数を維持（段階的移行）
- `history.py`: `_select_turns_to_compress()` / `_classify_importance()` を `agent/history_selection_policy.py` へ抽出し、`_call_compress_llm()` を `agent/history_summarizer.py` へ抽出
- `session.py`: 分割済み（`session_message_repo.py`/`document_repo.py`/`note_repo.py`）— 確認のみ

**Out:**
- `repl.py` の変更（bootstrap API は現状の `build_agent_context()` を維持）
- `AppServices` フィールドの変更
- DB スキーマ変更
- `session.py` の再構成（本セッションで分割済みのため対象外）

## Assumptions

1. `session.py` の 3 クラス分割は `session_message_repo.py`/`document_repo.py`/`note_repo.py` として本セッション完了済み。
2. `build_agent_context()` の外部呼び出しは `repl.py` のみ（grep 確認）。
3. `HistoryManager.compress()` の公開 API（`tuple[list, CompressResult]` を返す）は変更しない。
4. `_select_turns_to_compress()` と `_classify_importance()` は `HistoryManager` の internal として現在動作中。

## Design

### factory.py

`factory.py` は 205 行で既に十分モジュール化されている（各ビルダーが独立した関数）。`bootstrap.py` の新規作成は任意とし、まず `plugin_dir` パスを設定化することに集中する。

### history.py

`_classify_importance` (CC=14) と `_select_turns_to_compress` (CC=11) は複雑度が高い。これらを `HistorySelectionPolicy` クラスに抽出する。`HistoryManager` は引き続き公開 API（`compress()`）を保持し、内部で `HistorySelectionPolicy` に委譲する。

## Implementation steps

1. `scripts/agent/history_selection_policy.py` を新規作成し、`_classify_importance` / `_select_turns_to_compress` / `_sort_by_importance` / `_partition_by_class` を `HistorySelectionPolicy` クラスとして抽出する。
2. `scripts/agent/history.py` の `HistoryManager` を更新し、`HistorySelectionPolicy` に委譲する。
3. `tests/test_history_manager.py` の `_classify_importance` 参照を更新する。
4. `ruff`/`mypy`/`lint-imports`/`pytest` でバリデーション。
5. `factory.py` の `plugin_dir` を設定から読むよう変更する（`ToolConfig` or `ObservabilityConfig` への追加）。
