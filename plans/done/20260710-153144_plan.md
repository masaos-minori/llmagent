## Goal

Print workflow definition metadata (name, version, stage IDs) and SHA256 checksums of both the source and deployed `default.json` during `deploy/deploy.sh`, and abort deployment if the source and deployed checksums differ.

## Scope

**In-Scope:**
- `scripts/agent/workflow/validate.py`: add an optional `--print-metadata` flag that, on successful validation, additionally prints the workflow's `name`, `version`, and comma-joined stage IDs (reusing the already-parsed `WorkflowDef` object from `WorkflowLoader.load()` — no new parsing logic)
- `deploy/deploy.sh`: print the metadata block (Source/Deployed paths, Name, Version, Stages, both SHA256 checksums) and abort if the source/deployed checksums differ

**Out-of-Scope:**
- Introducing a `jq` dependency into `deploy.sh` for JSON field extraction — this plan reuses the canonical Python parser (`WorkflowLoader`) already relied on by `plans/done/20260710-152424_plan.md`'s validator, rather than adding a second, duplicate JSON-parsing path in shell
- Re-validating or re-parsing the deployed copy's `name`/`version`/`stages` separately from the source — if the SHA256 checksums of source and deployed files match, the deployed file is byte-identical to the already-validated source, so its metadata is provably identical without a second parse (see Design)
- Any change to `deploy/init_db.sh` or `deploy/setup_services.sh` — this requirement's Scope names only `deploy.sh` and the validation CLI

## Design

### `validate.py` extension

```python
parser.add_argument("--print-metadata", action="store_true",
                     help="On success, also print name/version/stage IDs")
...
try:
    wdef = WorkflowLoader(workflows_dir=args.path.parent).load(name=args.path.stem)
except WorkflowLoadError as exc:
    print(f"[FATAL] Invalid workflow definition {args.path}: {exc}", file=sys.stderr)
    return 1

print(f"OK: {args.path} is a valid workflow definition")
if args.print_metadata:
    print(f"Name     : {wdef.name}")
    print(f"Version  : {wdef.version}")
    print(f"Stages   : {', '.join(s.id for s in wdef.stages)}")
return 0
```

(Note: this requires `validate_path()`/`main()` from the prior plan's design to be refactored slightly so `main()` retains the `WorkflowDef` object returned by `.load()` rather than discarding it — a small, backward-compatible internal restructuring, not a change to the existing CLI's argument contract or exit-code behavior for callers that omit `--print-metadata`.)

### `deploy.sh` integration

Placed immediately after the existing pre-copy validation call (from `plans/done/20260710-152424_plan.md`), before any `mkdir`/`cp`:

```bash
echo "Workflow definition:"
echo "Source   : config/workflows/default.json"
echo "Deployed : /opt/llm/config/workflows/default.json"
PYTHONPATH="${REPO_ROOT}/scripts" uv run python -m agent.workflow.validate \
  --print-metadata "${REPO_ROOT}/config/workflows/default.json"
SOURCE_SHA256=$(sha256sum "${REPO_ROOT}/config/workflows/default.json" | awk '{print $1}')
echo "SHA256 (source)   : ${SOURCE_SHA256}"
```

Placed immediately after the existing workflow-files copy block (`cp -r "${REPO_ROOT}/config/workflows/." "${DEPLOY_CONFIG}/workflows/"`):

```bash
DEPLOYED_SHA256=$(sha256sum "${DEPLOY_CONFIG}/workflows/default.json" | awk '{print $1}')
echo "SHA256 (deployed) : ${DEPLOYED_SHA256}"
if [ "${SOURCE_SHA256}" != "${DEPLOYED_SHA256}" ]; then
  echo "[FATAL] Deployed workflow definition checksum does not match source; deployment corrupted." >&2
  exit 1
fi
```

`SOURCE_SHA256` is a plain shell variable set earlier in the same script process, so it remains in scope at the later checkpoint without needing a temp file or export — `deploy.sh` runs as a single non-subshelled script body throughout.
