## Goal

`db/store_protocols.py` と `db/store.py` のモジュールコメントと Protocol クラスの docstring を改善し、Protocol → SQLite 実装のマッピングと stable import surface を一目で把握できるようにする。

## Scope

**In**:
- `scripts/db/store_protocols.py` — モジュール docstring 更新、各 Protocol クラス docstring に実装クラス名を追記
- `scripts/db/store.py` — モジュール docstring を stable import entrypoint として明確化

**Out**:
- 新 storage backend の追加
- 全 DB モジュールの再編

## Assumptions

1. 変更はコメント・docstring のみ。実装コードは変更しない。
2. `store.py` は backward-compatibility 用 re-export モジュールでもあるが、新規コードのための stable import surface でもあることを明記する。

## Design

### store_protocols.py モジュール docstring (変更後)

```python
"""db/store_protocols.py
Public storage-layer contracts for the RAG pipeline and session/memory layers.

Protocol → SQLite implementation mapping (all in db/store_impl.py):
  VectorStore        → SQLiteVectorStore
  DocumentStore      → SQLiteDocumentStore
  SessionStore       → SQLiteSessionStore
  MemoryDeleteStore  → SQLiteMemoryDeleteStore

Embedding helpers (used by implementations and callers):
  get_embedding_dims()     — return configured dimension count (default 384)
  get_embedding_bytes()    — return expected float32 BLOB byte size
  validate_embedding_blob(blob) — raises TypeError/ValueError on wrong size

Stable import surface: use db/store.py for all imports.
"""
```

### Protocol クラスの docstring に実装クラス名を追記

各 Protocol に `Implementation: SQLiteXxxStore (db/store_impl.py)` を 1 行追加。

### store.py モジュール docstring (変更後)

```python
"""db/store.py
Stable import surface for the DB layer.

Import from this module (not store_protocols.py or store_impl.py directly):
    from db.store import VectorStore, SQLiteVectorStore
    from db.store import DocumentStore, SQLiteDocumentStore
    from db.store import SessionStore, SQLiteSessionStore
    from db.store import MemoryDeleteStore, SQLiteMemoryDeleteStore
    from db.store import get_embedding_dims, get_embedding_bytes, validate_embedding_blob

Protocol contracts are defined in db/store_protocols.py.
SQLite implementations are in db/store_impl.py.
"""
```
