## Implementation Steps

### Step 1: Remove `TransportType.STDIO` and `StartupMode.ONDEMAND` from `scripts/shared/mcp_config.py`

**Action:**
- Delete the line `STDIO = "stdio"` from `TransportType`.
- Delete the line `ONDEMAND = "ondemand"` from `StartupMode`.
- No other logic in `mcp_config.py` references these values explicitly (validation uses `isinstance` checks, not specific enum comparisons).

**Completion criteria:**
- `rg "STDIO\|ONDEMAND" scripts/shared/mcp_config.py` returns no results.
- The two existing tests `test_stdio_transport_rejected` and `test_ondemand_startup_mode_rejected` in `tests/test_mcp_config.py` now pass (they construct with a bare string, which will fail the isinstance check because the StrEnum no longer accepts those values).

---

### Step 2: Fix `_build_single_server()` to parse and pass `cmd` and `env`

**Action in `scripts/shared/mcp_config.py`:**
- In `_build_single_server()`, extract `cmd = list(v.get("cmd", []))` and `env = dict(v.get("env", {}))` from the raw dict.
- Pass both to the `McpServerConfig(...)` constructor call.

**Why:** `McpServerConfig` already defines and validates these fields. The omission in `_build_single_server()` means any TOML entry with `startup_mode = "subprocess"` and a `cmd` key would fail `_validate_cross_fields()` because `cmd` would default to `[]` even though the value is present in the raw config.

**Completion criteria:**
- `_build_single_server()` passes `cmd` and `env` to `McpServerConfig`.
- New tests in Step 3 pass.

---

### Step 3: Add missing tests to `tests/test_mcp_config.py`

**Add to `TestBuildMcpServers`:**

1. `test_subprocess_http_with_cmd_env_parsed` — builds a config with `startup_mode="subprocess"`, `cmd=["python3", "server.py"]`, `env={"PORT": "9000"}`; asserts `cmd` and `env` are set correctly on the result.
2. `test_subprocess_without_cmd_raises` — builds a config with `startup_mode="subprocess"` and no `cmd` key; asserts `ValueError` is raised (validation in `McpServerConfig._validate_cross_fields`).
3. `test_unsupported_transport_raises_from_toml` — passes `transport="stdio"` in the raw dict to `_build_mcp_servers`; asserts `ValueError` is raised.
4. `test_unsupported_startup_mode_raises_from_toml` — passes `startup_mode="ondemand"` in the raw dict to `_build_mcp_servers`; asserts `ValueError` is raised.

**Completion criteria:**
- `uv run pytest tests/test_mcp_config.py -v` passes with all new tests green.
