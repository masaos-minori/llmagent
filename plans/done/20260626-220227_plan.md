## Goal

Improve MDQ audit logging target and details per tool by setting appropriate targets (query for search_docs, chunk ID for get_chunk, path for outline, etc.) and adding detail fields (result count, elapsed time, truncated flag, indexed count, skipped count, deleted count, error kind).

## Scope

**In-Scope**:
- Set audit target per tool: query+path for search_docs, chunk_id for get_chunk, path for outline, first path for index_paths/refresh_index, pattern for grep_docs, service name for stats
- Add detail fields: result count, elapsed time, truncated flag, indexed count, skipped count, deleted count, error kind
- Preserve X-Session-Id and X-Request-Id correlation

**Out-of-Scope**:
- Adding new tools or features
- Changes to other MCP servers' audit logging

## Assumptions

1. Elapsed time measurement uses time.perf_counter() like cicd/server.py
2. Detail fields are only populated when available (some may be None)
3. Error kind is derived from exception type (ValueError, PermissionError, etc.)

## Affected Areas & Tool Evidence

- **Affected Files**:
  - `scripts/mcp/mdq/server.py` — update target extraction logic, add elapsed time measurement, add detail fields
  - `scripts/mcp/mdq/service.py` — return structured results with metadata (result count, truncated flag)
  - `mcp/audit.py` — verify detail parameter is supported

- **Blast Radius**: Low — changes audit logging behavior. Existing queries will still work; audit logs will have more detailed information.

## Implementation Steps

1. **Phase 1: Update target extraction per tool**
   - Replace flat `req.args.get("query", req.args.get("path", ""))` with tool-aware logic
   - search_docs: target = query + optional path filter
   - get_chunk: target = chunk_id
   - outline: target = path
   - index_paths: target = first path
   - refresh_index: target = first path
   - grep_docs: target = regex pattern
   - stats: target = service name

2. **Phase 2: Add elapsed time measurement**
   - Add time.perf_counter() before/after tool call in server.py
   - Calculate elapsed time and add to detail

3. **Phase 3: Add detail fields**
   - result count: from search/grep results length
   - truncated flag: from _truncate_with_meta result
   - indexed count: from index operation summary
   - skipped count: from index operation summary
   - deleted count: from index operation summary
   - error kind: from exception type

4. **Phase 4: Update audit log call**
   - Pass detail parameter to _audit_log with structured metadata
