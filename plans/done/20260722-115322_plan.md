## Goal
Standardize HTTP timeout parameter usage across the codebase by using `httpx.Timeout(timeout=X)` consistently, fixing the broken `connect`/`read` only pattern in `http_lifecycle.py`.

## Scope
**In**: `scripts/agent/services/mcp_tool_discovery.py`, `scripts/agent/http_lifecycle.py`, `scripts/agent/services/mcp_status.py` — unify timeout parameter format
**Out**: Any changes to timeout values themselves, or changes to other HTTP clients outside these three files

## Assumptions
1. `httpx.Timeout(timeout=X)` provides equivalent functionality to `timeout=X` but is more explicit
2. The `httpx.Timeout(connect=5.0, read=5.0)` pattern is indeed broken per httpx API requirements
3. A shared constant would improve maintainability

## Unknowns
1. ~~Is there a specific scenario where `httpx.Timeout(connect=5.0, read=5.0)` raises ValueError?~~ Resolved: httpx requires all four parameters (connect, read, write, idle) to be provided together; partial parameters raise ValueError.

## Affected areas
- MCP server startup health checks
- Tool discovery HTTP requests
- Server health probing

## Design
Create a module-level constant and use `httpx.Timeout(timeout=X)` consistently across all files. Define constant in `http_lifecycle.py` since it's the primary consumer of this timeout value.

## Implementation steps
1. Add `MCPSERVER_HEALTH_TIMEOUT = 5.0` constant to http_lifecycle.py
2. Replace bare float timeouts with `httpx.Timeout(timeout=MCPSERVER_HEALTH_TIMEOUT)` in mcp_tool_discovery.py
3. Replace `httpx.Timeout(connect=5.0, read=5.0)` with `httpx.Timeout(timeout=MCPSERVER_HEALTH_TIMEOUT)` in mcp_status.py
4. Run tests to verify no regressions
