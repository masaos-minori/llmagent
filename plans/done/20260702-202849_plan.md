# Plan: Align documentation with the no-backward-compatibility policy

## Goal

Remove all documentation references to deprecated import paths (`agent.config`, `mcp.github.service`, `from mcp.github import GitHubService`) and compatibility-only behavior (`McpServerConfig._cast_enums()`), and replace them with direct import guidance only.

## Scope

**In-Scope:**
- Remove or replace `agent.config` import examples in docs with `agent.config_builders` / `agent.config_dataclasses`
- Remove or replace `mcp.github.service` import examples in docs with `mcp.github.service_dispatch` / `mcp.github.service_init`
- Remove or replace `from mcp.github import GitHubService` examples in docs with `from mcp.github.service_dispatch import GitHubService`
- Remove documentation that says `McpServerConfig._cast_enums()` is a supported behavior
- State clearly that TOML string enum conversion is performed by the config builder (not inside `McpServerConfig`)
- Update `docs/05_agent_08_configuration.md` ŌĆö fix call site reference `agent/config.py:627` which points to the shim, not the builder
- Update known-issues docs if they describe deprecated import paths as valid
- Extend `check_no_compat.py` patterns to catch `agent.config` and `mcp.github.service` import references

**Out-of-Scope:**
- Changing any runtime behavior
- Removing the shim files (`agent/config.py`, `mcp/github/service.py`, `mcp/github/__init__.py`) ŌĆö those are covered by a separate refactoring task
- Updating test files that currently use deprecated imports ŌĆö separate cleanup task
- Updating scripts that import from the shims (`server_pull_requests.py`, etc.)

## Assumptions

1. `agent/config.py` and `mcp/github/service.py` are compatibility shims slated for removal, but are still present; docs must not recommend them.
2. The canonical import targets are:
   - `agent.config_builders` for `build_agent_config`, `load_config`, `ConfigLoadError`
   - `agent.config_dataclasses` for `AgentConfig`, `LLMConfig`, `RAGConfig`, etc.
   - `mcp.github.service_dispatch` for `GitHubService` (with dispatch formatters)
   - `mcp.github.service_init` for `build_service`, `_GITHUB_TOKEN`
3. `McpServerConfig._cast_enums()` remains in the source for runtime callers but should not be documented as a public API or recommended usage pattern.
4. `docs/05_agent_08_configuration.md` line 14 references `agent/config.py:627` ŌĆö this is an inaccurate pointer since `build_agent_config` lives in `agent/config_builders.py`.
