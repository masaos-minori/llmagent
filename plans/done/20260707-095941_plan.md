## Goal

Remove `workflow_mode="auto"` startup degradation so that any workflow initialization failure (missing definition, invalid JSON, missing schema, missing tables) aborts agent startup instead of allowing degraded operation.

## Scope

**In**:
- Remove `allow_startup_fallback()` from `WorkflowExecutionPolicy` (class deletion covered by plan 20260707-095938; this plan documents the specific behavior site)
- Remove the `if self._policy.requires_startup_definition()` guard from `Orchestrator.__init__()` (lines 136–141): make every `WorkflowLoadError` a hard startup failure
- Remove the `logger.warning("WorkflowLoader failed — workflow tracking disabled")` silent degradation path (orchestrator.py:142)
- Remove `_check_workflow_definition()` dependency on `WorkflowExecutionPolicy` in `startup.py` (line 130): always run preflight as hard check
- Remove error messages suggesting `workflow_mode=auto` or `workflow_mode=disabled`
- Update `check_workflow_definition()` signature/behavior to always-fail on error
- Remove and rewrite tests that expect degraded startup to succeed
- Ensure Orchestrator construction fails if workflow loader fails

**Out**:
- `WorkflowExecutionPolicy` class deletion (plan 20260707-095938)
- `workflow_mode` config field removal (plan 20260707-095938)
- Direct-execution path removal (plans 20260707-095939, 20260707-095940)
- Approval changes (plan 20260707-095942)

## Assumptions

1. `check_workflow_definition()` is defined in `startup.py` or an imported module; it currently accepts a `WorkflowExecutionPolicy` to decide whether to raise or warn.
2. `WorkflowLoader().load()` raises `WorkflowLoadError` on any failure (missing file, invalid JSON, schema error).
3. After this change, any `WorkflowLoadError` during `Orchestrator.__init__()` must propagate as a `RuntimeError` that aborts startup.
4. Local development environments must have `config/workflows/default.json` and the workflow schema tables present before startup.
5. No error message in startup output should suggest `workflow_mode=disabled` or `workflow_mode=auto` as a workaround.

## Implementation Steps

1. **Phase 1: Harden Orchestrator.__init__()**
   - [ ] In `orchestrator.py:__init__()`, remove the `requires_startup_definition()` guard:
     ```python
     # Before:
     if self._policy.is_workflow_enabled():
         try:
             self._workflow_def = WorkflowLoader().load()
         except (WorkflowLoadError, Exception) as exc:
             if self._policy.requires_startup_definition():
                 raise RuntimeError(...)
             logger.warning("WorkflowLoader failed — workflow tracking disabled")
     ```
   - [ ] Replace with always-fail:
     ```python
     try:
         self._workflow_def = WorkflowLoader().load()
     except (WorkflowLoadError, Exception) as exc:
         raise RuntimeError(
             f"[workflow] WorkflowLoader failed: {exc}. "
             f"Expected workflow definition at: {WORKFLOWS_DIR / 'default.json'}."
         ) from exc
     ```
   - [ ] Remove any reference to `allow_startup_fallback()`, `requires_startup_definition()`
