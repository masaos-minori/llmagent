## 3. Assumptions

1. The `--targets-file` TOML format mirrors `rag_pipeline.toml`'s `target_urls` syntax: a TOML array of inline 2-element arrays `[[url, lang], ...]`. Example: `target_urls = [["file:///path/to/file.py", "en"], ["https://example.com/", "en"]]`.
2. `file://` URL validation will use `urlparse(url).scheme == "file"` check, not `validate_url()` (which only accepts http/https). The new `_validate_target_url()` helper in `crawler_utils.py` will accept both `http`, `https`, and `file` schemes.
3. Path resolution for `--targets-file` is relative to CWD; no special project-root logic is needed.
4. Invalid targets (unrecognized scheme, invalid lang, unreadable targets file) should abort the entire run via `parser.error()` â€” matching the existing `--url` validation pattern in `main()`.
5. `crawl_file()` already exists and handles `file://` paths; `--targets-file` will route `file://` entries to `crawl_file()` in the existing `crawl()` loop (this is already done via `crawl_site` being called for each target in `crawl()`).

   **Verification needed**: `crawl_site()` calls `validate_url()` which rejects `file://`. The routing of `file://` URLs must be handled before `crawl_site()` is called â€” either in `crawl()` or in `main()`. Current `crawl()` iterates targets and calls `crawl_site()` for all of them. `file://` targets must be dispatched to `crawl_file()` instead.

6. The existing `crawl()` method already has a branching pattern for dispatch; no new public methods on `WebCrawler` are needed beyond what exists.
7. `radon`, `pydeps`, `grimp`, `pyan3` â€” not installed; manual static analysis used instead.
