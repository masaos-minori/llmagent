### Step 5: Add `TestToolExecutorErrorBoundary` to `tests/test_tool_executor.py`

**File:** `tests/test_tool_executor.py`

Add a new test class after the existing `TestHttpTransportRetry` class. The class must cover all six scenarios from the acceptance criteria.

Required test methods and their assertions:

| Test method | Scenario | Key assertions |
|-------------|----------|----------------|
| `test_http_200_is_error_true_increments_tool_error` | HTTP 200 + `is_error=true` | `result.error_type == "tool"`, `stat_tool_errors[server_key] == 1`, `record_success` called once, `record_failure` not called |
| `test_http_500_raises_transport_error_internally` | HTTP 500 | `result.error_type == "transport"`, `stat_transport_errors[server_key] == 1`, `record_failure` called once, `record_success` not called |

| `test_malformed_response_classified_as_transport_error` | Missing `result` key in body | `result.error_type == "transport"`, `stat_transport_errors[server_key] == 1`, `record_failure` called |
| `test_503_retry_exhausted_becomes_transport_error` | HTTP 503 repeated 3 times | `result.error_type == "transport"`, `stat_transport_errors[server_key] == 1`, `record_failure` called |
| `test_tool_error_calls_record_success` | HTTP 200 + `is_error=true` | `record_success` called (server responded), `record_failure` not called |

- Use `unittest.mock.MagicMock` or `unittest.mock.AsyncMock` to spy on `HealthRegistry.record_success` / `record_failure`.
- Use `patch("asyncio.sleep", return_value=None)` for retry tests.
