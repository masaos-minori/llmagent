# Plan: Session Title Generation Fallback and Failure Visibility

**Requirement:** `requires/20260622_73_require.md`

---

## Goal

Harden the existing title generation fallback path by: (1) catching DB errors in the fallback `set_title()` call so a session is never left with an empty title due to a secondary failure, (2) emitting an audit log warning on failure so operators can see it without reading Python logs, and (3) storing the `asyncio.create_task` reference to prevent unhandled task exceptions from being silently dropped.

---

## Scope

**In:**
- `cmd_session.py` — wrap fallback `set_title()` in try/except; add `audit_logger.warning()` call
- `orchestrator.py` — store `asyncio.create_task()` result in `_background_tasks` set
- `tests/test_agent_cmd_session.py` — add tests for audit log warning and fallback DB error path

**Out:**
- Redesigning session lifecycle logic
- Removing async title generation
- Advanced title re-ranking or editing
- Adding terminal `write_warning()` output (timing is non-deterministic relative to LLM streaming; audit log is the right operator-facing surface here)

---

## Assumptions

1. The deterministic fallback title already exists in `_generate_session_title()` (verified: empty input → `"(New Session)"`, long → truncated, short → as-is).
2. `logger.warning("Session title generation failed, using fallback: %s", e)` is already present but writes only to the Python log system, not to the operator-facing audit log.
3. `ctx.services.audit_logger` is a `logging.Logger` available in `cmd_session.py` via `self._ctx.services.audit_logger`.
4. `asyncio.create_task()` result is currently discarded in `orchestrator.py:413` — unhandled task exceptions produce asyncio's "Task exception was never retrieved" warnings instead of being surfaced clearly.
5. The `set_title_pending(False)` call in the `finally` block is correct and must be preserved.

## Design

### Change 1: `cmd_session.py` — `_generate_session_title()` hardening

**Before (lines 37-51):**
```python
self._ctx.session.set_title_pending(True)
try:
    await SessionTitleService().generate(self._ctx, first_input)
except SessionTitleGenerationError as e:
    logger.warning("Session title generation failed, using fallback: %s", e)
    clean_input = first_input.strip() if first_input else ""
    if not clean_input:
        fallback_title = "(New Session)"
    elif len(clean_input) > SESSION_TITLE_MAX_CHARS:
        fallback_title = clean_input[:SESSION_TITLE_TRUNCATE_AT] + "..."
    else:
        fallback_title = clean_input
    self._ctx.session.set_title(fallback_title)
finally:
    self._ctx.session.set_title_pending(False)
```

**After:**
```python
self._ctx.session.set_title_pending(True)
try:
    await SessionTitleService().generate(self._ctx, first_input)
except SessionTitleGenerationError as e:
    logger.warning("Session title generation failed, using fallback: %s", e)
    clean_input = first_input.strip() if first_input else ""
    if not clean_input:
        fallback_title = "(New Session)"
    elif len(clean_input) > SESSION_TITLE_MAX_CHARS:
        fallback_title = clean_input[:SESSION_TITLE_TRUNCATE_AT] + "..."
    else:
        fallback_title = clean_input
    try:
        self._ctx.session.set_title(fallback_title)
    except Exception as db_err:  # noqa: BLE001
        logger.error(
            "Session title fallback set_title failed: %s (session_id=%s)",
            db_err,
            self._ctx.session.session_id,
        )
    else:
        if self._ctx.services.audit_logger is not None:
            self._ctx.services.audit_logger.warning(
                "session_title_fallback session_id=%s fallback=%r reason=%s",
                self._ctx.session.session_id,
                fallback_title,
                e,
            )
finally:
    self._ctx.session.set_title_pending(False)
```

Key points:
- `set_title()` failure is caught and logged at `ERROR` level — this is the secondary failure path
- `audit_logger.warning()` is only called when the fallback succeeds (in `else:` branch)
- `set_title_pending(False)` remains in `finally` — always fires regardless

### Change 2: `orchestrator.py` — store `asyncio.create_task()` result

In `_append_user_message()` (line 413):

**Before:**
```python
asyncio.create_task(self._on_first_turn(line))
```

**After:**
```python
_task = asyncio.create_task(self._on_first_turn(line))
_task.add_done_callback(self._background_tasks.discard)
self._background_tasks.add(_task)
```

In `__init__()`, add:
```python
self._background_tasks: set[asyncio.Task] = set()
```

This keeps the task alive (preventing GC), and the done-callback removes it from the set when finished. Any exception in the task is still raised when `_task.result()` is called, but at minimum the asyncio "Task exception was never retrieved" warning is suppressed (asyncio marks the exception as retrieved when a done-callback is attached).

class TestGenerateSessionTitleVisibility:
    """audit_logger.warning is called on fallback; fallback set_title DB error is handled."""

    @pytest.mark.asyncio
    async def test_audit_logger_warning_called_on_failure(self) -> None:
        from unittest.mock import AsyncMock, MagicMock, patch
        from agent.services.exceptions import SessionTitleGenerationError

        cmd = _make_cmd()
        audit_logger = MagicMock()
        cmd._ctx.services = SimpleNamespace(
            hist_mgr=MagicMock(), llm=MagicMock(), http=MagicMock(),
            audit_logger=audit_logger,
        )

        with patch("agent.services.session_title.SessionTitleService") as MockSvc:
            MockSvc.return_value.generate = AsyncMock(
                side_effect=SessionTitleGenerationError("LLM error")
            )

        audit_logger.warning.assert_called_once()
        call_args = audit_logger.warning.call_args[0]
        assert "session_title_fallback" in call_args[0]

    @pytest.mark.asyncio
    async def test_fallback_set_title_db_error_does_not_propagate(self) -> None:
        """If fallback set_title() raises, the exception is logged but not re-raised."""
        import sqlite3
        from unittest.mock import AsyncMock, patch
        from agent.services.exceptions import SessionTitleGenerationError

        cmd = _make_cmd()
        cmd._ctx.session.set_title.side_effect = sqlite3.Error("disk full")

        with patch("agent.services.session_title.SessionTitleService") as MockSvc:
            MockSvc.return_value.generate = AsyncMock(
                side_effect=SessionTitleGenerationError("fail")
            )
            # Must not raise
            await cmd._generate_session_title("hello")

        cmd._ctx.session.set_title_pending.assert_called_with(False)

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `add_done_callback(self._background_tasks.discard)` causes a reference cycle | Low | The set holds `Task`, and the lambda discards it on completion — no persistent cycle |
| `audit_logger.warning()` line in `else:` branch never fires if `set_title()` succeeds but `audit_logger is None` | Low | Covered by null guard; test mocks `audit_logger` as `MagicMock()` |
| Concurrent background tasks sharing `_background_tasks` set cause data races | Low | asyncio is single-threaded; set operations are safe |
