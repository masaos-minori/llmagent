## Assumptions

1. `registry.py` の `mask_args` re-export はすでに削除済み (`rg` で 0件確認)。`tool_result_formatter.py` docstring のみが stale。
2. `_check_github_repo_allowed()` は `tool_approval.py` 内で定義されているが呼び出し元が存在しない (`rg` で 0件)。dead code として削除可能。
3. `_escalate_by_args()` の recursive delete / path escalation の判定は `tool_policy.py` の `_special_case_risk()` / `_escalate_for_path()` と重複している。force/overwrite/clobber フラグ判定のみが未移植。
4. `build_execution_groups()` はリソーススコープなしの write ツールをすべて `parallel` グループに入れる。`_execute_with_dag_legacy()` 削除後は、resource_scope 未設定の write ツールが並列実行になる（既存挙動との差異）。この変更は許容できる（tool_definitions に resource_scope を付与する方向で管理するため）。
5. `tool_loop_guard.py` の guard は `llm_turn_runner.py` と `orchestrator.py` で正しく使用されており、caller 側に重複ロジックなし。Phase 3 は確認のみで実装変更なし。

### 2.2 tool_runner.py — _execute_with_dag_legacy() 削除

`_execute_with_dag()` の fallback 分岐と `_execute_with_dag_legacy()` 関数を削除し、常に `build_execution_groups()` を使うよう変更。

```python
# 変更前
async def _execute_with_dag(ctx, approved_calls, turn):
    ...
    if not any(m.get("resource_scope") or m.get("requires_serial") for m in tool_meta.values()):
        return await _execute_with_dag_legacy(ctx, approved_calls, turn)  # ← 削除
    groups = build_execution_groups(approved_calls, tool_meta)
    ...

# 変更後
async def _execute_with_dag(ctx, approved_calls, turn):
    ...
    # metadata なしの場合も scheduler に委譲 (resource_scope なし write は parallel グループへ)
    groups = build_execution_groups(approved_calls, tool_meta)
    ...
```

### 2.5 tool_audit.py — audit event schema 統一

`audit_tool_exec()` に `operation_type` / `resource_scope` を追加し、`audit_approval()` との schema を揃える。
`log_approval_decision()` に `task_id` / `ts` を追加。

```python
def audit_tool_exec(ctx, tool_name, args, is_error, mcp_request_id):
    ...
    masked = mask_args(args, ctx.cfg.tool.masked_fields)
    # 追加
    path_keys = set(ctx.cfg.approval.approval_resource_keys.get("path_keys", []))
    branch_keys = set(ctx.cfg.approval.approval_resource_keys.get("branch_keys", []))
    resource_scope = {k: v for k, v in masked.items() if k in path_keys | branch_keys}
    ctx.services.audit_logger.info(orjson.dumps({
        "event": "tool_exec",
        "task_id": ctx.turn.current_turn_id,
        "tool": tool_name,
        "operation_type": classify_operation_type(tool_name),  # 追加
        "resource_scope": resource_scope,                       # 追加
        "mcp_request_id": mcp_request_id,
        "is_error": is_error,
        "args_preview": masked,
        "ts": time.time(),
    }).decode())
```
