## Goal

Add tests and optional compression metadata to make history compression, session persistence, and `/undo` behavior explicit, deterministic, and verifiable after compression.

## Scope

**In**: Tests for session reload after compression. Tests for `/undo` behavior after compression. Tests for `tool_results` retention after compressed undo. Optional: `compression_id`, `source_message_ids`, `compressed_range_start`, `compressed_range_end` metadata. Fallback truncation tests.

**Out**: Removing compression, changing summarization prompt behavior, documentation-only updates.

## Assumptions

1. `HistoryManager.compress()` replaces old messages with a compressed summary in `ctx.conv.history`.
2. `AgentSession.replace_messages()` persists the compressed history to the `messages` table.
3. `session.py:replace_messages()` at line 52 calls `self._message_repo.replace_messages(session_id, messages)`.
4. After compression, `/undo` operates on the compressed summary message, not original turn boundaries — this is the behavior to make explicit.
5. `tool_results` rows are separate from `messages` and retain `undone` marker after undo.
6. Compression metadata (source IDs, range) is OPTIONAL per the require — add only if no schema change conflict exists.

## Design

**Priority 1 (no schema change): Tests**

1. Compression → reload: after `compress()` + `replace_messages()`, create new session, load messages, verify compressed history matches.
2. `/undo` after compression: verify deterministic behavior (removes the summary message or is a no-op?).
3. `tool_results.undone` after undo on compressed history: verify rows are marked `undone=True`.
4. Fallback truncation → reload: verify same properties as LLM compression.

**Priority 2 (optional, with schema change): Compression metadata**

Add `compression_id TEXT` and `compressed_range_start INT`, `compressed_range_end INT` to `messages` table.
Return `CompressResult.compression_id` and range from `compress()`.
Store via `replace_messages()`.
This enables: "these messages replaced rows N–M" for audit purposes.
