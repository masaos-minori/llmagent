## Goal
Standardize HTTP timeout parameter usage across the codebase by using `httpx.Timeout(timeout=X)` consistently, fixing the broken `connect`/`read` only pattern in `http_lifecycle.py`.

## Scope
**In**: `scripts/agent/services/mcp_tool_discovery.py`, `scripts/agent/http_lifecycle.py` — unify timeout parameter format
**Out**: Any changes to timeout values themselves, or changes to other HTTP clients outside these two files

## Assumptions
1. `httpx.Timeout(timeout=X)` provides equivalent functionality to `timeout=X` but is more explicit
2. The `httpx.Timeout(connect=5.0, read=5.0)` pattern is indeed broken per httpx API requirements
3. A shared constant would improve maintainability

## Implementation Steps
1. Fix `http_lifecycle.py` line 249: replace `httpx.Timeout(connect=5.0, read=5.0)` with `httpx.Timeout(timeout=5.0)`
2. Fix `mcp_tool_discovery.py` line 123: replace `timeout=5.0` with `timeout=httpx.Timeout(timeout=5.0)`
3. Add shared constant `MCPSERVER_HEALTH_TIMEOUT = 5.0` in a central location (e.g., `http_lifecycle.py` module level)
4. Update both files to use the shared constant
