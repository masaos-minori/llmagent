## Goal

Change `LLMTurnRunner.run()` to require `workflow_id`, `task_id`, `stage_id`, and `attempt_id` parameters so that every LLM streaming and tool-call loop iteration is associated with a workflow stage context and cannot be called without one.

## Scope

**In**:
- Change `LLMTurnRunner.run(llm_url: str)` to `run(llm_url: str, *, workflow_id: str, task_id: str, stage_id: str, attempt_id: str)` in `scripts/agent/llm_turn_runner.py`
- Raise `RuntimeError` if any workflow context parameter is empty at entry to `run()`
- Pass workflow context down to `_stream_llm()`, `_handle_llm_error()`, and `_span_ctx()`
- Update `_handle_llm_error()` to include `workflow_id` and `task_id` in `DiagnosticStore.save()` payload
- Update `_span_ctx()` to accept and set `workflow_id`, `task_id`, `stage_id`, `attempt_id` as OTel attributes
- Update the call site in `Orchestrator._handle_llm_turn()` (`orchestrator.py:425`) to pass workflow context (sourced from `ctx.workflow`)
- Update `Orchestrator._process_turn()` to accept and propagate workflow context to `_handle_llm_turn()`
- Rewrite `tests/test_llm_turn_runner.py` to always supply workflow context; add test for missing context raises

**Out**:
- Changes to LLM streaming protocol (HTTP/SSE)
- Changes to `execute_all_tool_calls` signature (tool runner)
- Compatibility shim for old `run(llm_url)` signature
- `DiagnosticStore` schema changes (add columns) — only pass existing optional fields

## Assumptions

1. `ctx.workflow.workflow_id`, `ctx.workflow.current_task_id` are set before `_process_turn()` is called from `execute_fn()` inside `_handle_workflow_engine()`.
2. `stage_id` and `attempt_id` are available in the execute stage callback context; they can be sourced from the `TaskRecord` or the `WorkflowEngine` stage tracking.
3. All production call paths to `run()` go through `Orchestrator._handle_llm_turn()` → `_process_turn()` → `_handle_llm_turn()`.
4. `tests/test_llm_turn_runner.py` currently calls `runner.run(llm_url)` directly without workflow context.
5. After this change, tests must supply all four workflow context parameters.
