# Plan: H-1 — Redirect HTTP MCP subprocess stderr to per-server log files

## Goal

Replace `stderr=subprocess.PIPE` in `HttpServerLifecycleManager.start()` with per-server log file redirection so that long-running MCP subprocesses cannot block due to pipe buffer exhaustion, and startup failure diagnostics are read from the log file tail.

## Scope

**In**: `scripts/agent/http_lifecycle.py` — replace `subprocess.PIPE` with per-server stderr log file; close file handles on failure, restart, and shutdown; tail log file for `StartupFailure.stderr_full`.

**Out**: No changes to `factory.py`, `context.py`, config files, DB schema, or any other module.

## Assumptions

1. The log directory `/opt/llm/logs/mcp` is writable at agent startup.
2. `server_key` values are filesystem-safe (alphanumeric, hyphens, underscores); no path traversal sanitization needed beyond replacing `.` with `_`.
3. 64 KiB is a sufficient cap for the stderr tail in `StartupFailure.stderr_full`.
4. File handles opened for stderr must be closed on any code path that exits `start()` before normal return.

## Unknowns

## Implementation Steps

1. **Add stderr log file management to `HttpServerLifecycleManager`**
   - Add `self._stderr_files: dict[str, IO[bytes]]` to `__init__()`.
   - Add `_open_stderr_log(server_key: str) -> IO[bytes]` helper: `mkdir -p /opt/llm/logs/mcp`, open `{server_key}.stderr.log` in `ab` mode.
   - Sanitize `server_key` for filename safety (replace non-alphanumeric except `-_` with `_`).
   - Criterion: helper opens a file handle and returns it.

2. **Update `start()` to use the log file**
   - Replace `stderr=subprocess.PIPE` with `stderr=self._open_stderr_log(server_key)`.
   - Store handle in `self._stderr_files[server_key]`.
   - On early exit (`proc.poll() is not None`): read tail (64 KiB) from the log file; close the handle; remove from `_stderr_files`.
   - On timeout: read tail; call `_terminate_with_timeout()`; close handle; remove from `_stderr_files`.
   - On normal return (healthy): leave handle open (process still writing).
   - Criterion: `start()` uses no `proc.stderr`; all `StartupFailure.stderr_full` reads come from log file.

3. **Close handles in `restart()` and `shutdown_all()`**
   - `restart()`: close and remove `_stderr_files[server_key]` before calling `_terminate_with_timeout()`.
   - `shutdown_all()`: close all handles in `self._stderr_files` after terminating processes.
   - Criterion: no open file handles remain after shutdown.

4. **Update tests**
   - Add test: subprocess writes repeatedly to stderr → process does not block (no pipe buffer fill).
   - Add test: startup timeout → `HttpStartupError.failure.stderr_full` contains log file tail.
   - Add test: early exit → `stderr_full` contains log tail.
   - Add test: restart → old log file handle is closed.
   - Criterion: `uv run pytest tests/test_lifecycle.py -v` passes.
