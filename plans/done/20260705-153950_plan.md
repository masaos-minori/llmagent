## Goal

## Scope

**In**: Update `build_discovery_map()` to return duplicate ownership info. Update `check_routing_drift_vs_live()` to include duplicate warnings in `HealthCheckResult`. Raise in strict mode.

**Out**: Supporting multi-owner routing. Changing ToolRegistry ownership model. Documentation-only updates.

## Assumptions

1. `build_discovery_map()` at `route_resolver.py:23` currently logs `logger.warning(...)` for duplicates but returns only `route_map` — no structured duplicate list is returned.
2. `check_routing_drift_vs_live()` at `repl_health.py:339` calls `build_discovery_map()` but discards duplicate information.

def build_discovery_map(
    server_tool_lists: dict[str, list[dict[str, object]]],
) -> tuple[dict[str, str], dict[str, list[str]]]:
    """Build routing map and detect duplicate ownership.
    
    Returns:
        route_map: {tool_name: server_key}
        duplicates: {tool_name: [server_key_1, server_key_2, ...]} for tools appearing on multiple servers
    """
    route_map: dict[str, str] = {}
    all_claims: dict[str, list[str]] = {}  # tool_name → [server_keys claiming it]
    for server_key, tools in server_tool_lists.items():
        for tool in tools:
            name = ...
            all_claims.setdefault(name, []).append(server_key)
            if name not in route_map:
                route_map[name] = server_key
            elif route_map[name] != server_key:

    duplicates = {n: keys for n, keys in all_claims.items() if len(keys) > 1}
