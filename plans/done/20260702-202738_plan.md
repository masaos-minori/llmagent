# Plan: Clarify `rotate_db()` vs `rotate_all_dbs()` Documentation

Source requirement: `requires/20260702_51_require.md`

---

## Goal

Update `docs/90_shared_05_db_api_and_operations.md` and optionally `scripts/db/maintenance.py` docstrings to unambiguously document the scope of each DB rotation function and recommend `rotate_all_dbs()` for pre-recreation workflows.

---

## Scope

**In**:
- Add `rotate_workflow_db()` and `rotate_all_dbs()` rows to the maintenance function table in `docs/90_shared_05_db_api_and_operations.md`
- Add a clarifying note that `rotate_db()` is a rag/session-only helper (does **not** archive `workflow.sqlite`)
- Add a recommendation to call `rotate_all_dbs()` before DB schema recreation
- Update the module-level docstring in `scripts/db/maintenance.py` to replace the `rotate_db()` example with `rotate_all_dbs()` for the "Before schema change" use case

**Out**:
- Renaming `rotate_db()` — requires separate approval
- Removing `rotate_db()` — out of scope
- Changing function behavior or signatures

---

## Assumptions

1. `rotate_workflow_db()` and `rotate_all_dbs()` exist in `scripts/db/maintenance.py` and behave as their docstrings describe (verified by code inspection).
2. `docs/90_shared_05_db_api_and_operations.md` is the canonical operational reference for DB maintenance; operators read it before performing schema changes.

4. No API-consuming code calls these functions based on the doc table alone; the doc change carries zero runtime risk.

## Implementation Steps

### Phase 1: Update the maintenance function table in docs

- [ ] In `docs/90_shared_05_db_api_and_operations.md`, add two rows after the existing `rotate_db()` row:
  - `rotate_workflow_db(archive_dir=None)` | `-> Path` | Archive `workflow.sqlite` only; returns archive path
  - `rotate_all_dbs(archive_dir=None)` | `-> tuple[Path, Path, Path]` | Archive all three DBs (rag, session, workflow); returns `(rag_dest, session_dest, wf_dest)`
- [ ] Add a note below the table clarifying:
  - `rotate_db()` is a rag/session-only helper — it does **not** archive `workflow.sqlite`
  - Use `rotate_all_dbs()` when recreating the full schema

### Phase 2: Update the DB recreation workflow recommendation in docs

- [ ] Locate the section in `docs/90_shared_05_db_api_and_operations.md` that describes pre-recreation workflow (if it exists).
- [ ] Add or update a callout/note that recommends `rotate_all_dbs()` before any schema recreation that affects all three DBs.

### Phase 3: Fix module docstring in maintenance.py

- [ ] In `scripts/db/maintenance.py`, change line 14:
  - Before: `Before schema change  : rotate_db()`
  - After:  `Before schema change  : rotate_all_dbs()  # archives rag, session, and workflow`
