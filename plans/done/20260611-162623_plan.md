# Work Plan: 大規模リファクタリング — DTO統一・OutputAdapter・fail-fast・後方互換除去

## Goal

後方互換エントリポイント・再エクスポート・`except Exception`・`dict[str, Any]`・直接 `print()` を全 command/service/memory レイヤーから排除し、typed DTO・OutputWriter adapter・fail-fast validation に統一する。

## Scope

### Phase 1: 後方互換除去
- `scripts/agent.py` — レガシーエントリポイント削除
- `scripts/agent/commands/utils.py` — `render_export` / `render_history_md` / `write_export` 再エクスポート削除

### Phase 2: Output layer
- `scripts/agent/commands/formatter.py` — `OutputWriter` Protocol 化 + `PrintOutputWriter` 実装
- 全 `cmd_*.py` (cmd_context, cmd_db, cmd_memory, cmd_mcp, cmd_session, cmd_debug, cmd_tooling, cmd_config, cmd_ingest, cmd_notes) — `print()` → `OutputWriter` 経由に変換

### Phase 3: Command registry
- `scripts/agent/commands/registry.py` — string handler 参照 → 型付き handler reference、stringly-typed dispatcher 構造化

### Phase 4: Service layer DTOs
- `scripts/agent/services/config_reload.py` — `dict[str, Any]` → `ConfigReloadRequest` DTO、各カテゴリ別 adapter
- `scripts/agent/services/conversation_service.py` — dict-based history access → typed access
- `scripts/agent/services/ingest_workflow.py` — 確認済: 既に `IngestStageError` / `IngestStage` enum 使用 → 軽微確認
- `scripts/agent/services/mcp_install.py` — `McpInstallRequest` DTO 追加
- `scripts/agent/services/mcp_status.py` — 確認済: 既に `McpTier` enum + fail-fast → 軽微確認

### Phase 5: Memory layer
- `scripts/agent/memory/types.py` — `memory_type: str` → `MemoryType` enum、`EmbeddingResult` の `success=False` パターン見直し
- `scripts/agent/memory/embedding_client.py` — `except Exception` 削除、`resp.json().get("embedding")` → schema validation
- `scripts/agent/memory/jsonl_store.py` — `read_all(strict=False)` quarantine継続ロジック除去、`_entry_from_dict` の `.get("memory_type", "")` fallback 除去
- `scripts/agent/memory/mapper.py` — `sqlite3.Row | dict[str, Any]` → typed row DTO、`.get(..., default)` + 無条件 `float()`/`bool()` 除去
- `scripts/agent/memory/retriever.py` — `_recency_boost` の `(ValueError, OverflowError) → 0.0` → schema error
- `scripts/agent/memory/extract.py` — `from shared.types import LLMMessage` → memory-specific DTO、`msg.get("content") or ""` + `str(content_raw)` 除去
- `scripts/agent/memory/ingestion.py` — `except Exception` in `_link_duplicates` → specific exceptions、stage DTO 定義
- `scripts/agent/memory/injection.py` — `not query.strip() → []` → ValidationError、`embedding=None` 継続 → 明示的結果型、`summary or content[:100]` → DTO 経由
- `scripts/agent/memory/services.py` — `LLMMessage` 依存 → memory-specific history DTO

### Phase 6: 共通 DTO・enum・例外 (新規 `agent/shared/`)
- `scripts/agent/shared/__init__.py`
- `scripts/agent/shared/models.py` — `CommandResult`, `ValidationErrorDetail`, `McpInstallRequest`, `McpServerStatus` 等
- `scripts/agent/shared/enums.py` — `CommandName`, `ExtractionDecision`, `RetrievalMode`, `ToolSafetyTier`, `McpRole`, `WorkflowStage`, `MemoryType`
- `scripts/agent/shared/exceptions.py` — `ValidationError`, `ConfigurationSchemaError`, `WorkflowStageError`, `UnknownTierError`, `UnknownRoleError`

## Assumptions

1. `agent/shared/` は新規パッケージ。`.importlinter` の layer contract (`agent` → all) に準拠するため `agent/shared/` は `shared/` 外部のみ import 可。
2. `formatter.py` の `OutputWriter` DI 化は `MixinBase` に `_out: OutputWriter` を持たせることで全 mixin が共有する。
3. `registry.py` の handler 参照変更は文字列 key を維持しつつ `Callable` 型アノテーションを追加する最小変更で対応 (完全 enum 化は Phase 3 の個別ステップ)。
4. `config_reload.py` の `dict[str, Any]` パラメータは既存の `ConfigReloadRequest` dataclass を拡張して typed にする (既に `agent/services/models.py` に `ConfigReloadRequest` が存在)。
5. `conversation_service.py` の `ctx.conv.history[0]["role"]` は `LLMMessage` TypedDict アクセスであり、TypedDict として正しい型アクセス (`m["role"]`) に変更する。
6. memory layer の `LLMMessage` → memory-specific DTO への変換は `agent/shared/models.py` に `MemoryMessage` dataclass を定義し、呼び出し側でアダプタ変換する。

**Step 5-3: `jsonl_store.py` — strict=False 除去**

**Step 5-6: `ingestion.py` — except Exception 除去 + stage DTO**

- 完了基準: mypy, pytest

**Step 5-7: `retriever.py` — _recency_boost fail-fast**
- `except (ValueError, OverflowError): return 0.0` → `raise MemorySchemaError(...)` (または `ValueError`)
- 呼び出し元で適切に処理
- 完了基準: mypy

**Step 5-8: `injection.py` — fail-fast + InjectionPolicy immutable**

- 完了基準: mypy

**Step 5-9: `services.py` — LLMMessage → MemoryMessage**
- facade の public API を `MemoryMessage` ベースに統一
- 完了基準: mypy

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `OutputWriter` DI 化で 10 cmd ファイルへの波及が大きい | high | `PrintOutputWriter()` をデフォルト DI にし、段階的に変換。テストは `capsys` で引き続き動作 |
| `MemoryType` enum 移行で全 memory テストが連鎖更新 | medium | 移行後すぐに `pytest tests/test_memory*.py` を実行して確認 |
| `read_all(strict=False)` 除去で既存の quarantine 動作テストが壊れる | medium | テスト内の `strict=False` 呼び出しを修正し、quarantine テストは削除または `strict=True` に変更 |
| `_recency_boost` の fail-fast 変更で retriever が実環境データで raise する | medium | `created_at` が常に ISO-8601 で保存されているか DB を事前確認 |
| `agent/shared/` 追加で `.importlinter` 違反 | low | 追加直後に `lint-imports` を実行して確認 |
| `config_reload.py` の `dict[str, Any]` 除去でカバーが薄い箇所が runtime エラー | medium | 変更前に `pytest tests/test_config_reload*.py` を実行してベースライン確認 |
