## Goal

Add `tests/test_mcp_tool_schema_exports.py` — a cross-server unit test that asserts
every active MCP tool schema module exports `TOOL_LIST` as a non-empty list of dicts
each containing a `"name"` key, and that no module relies on the legacy `_MCP_TOOLS`
name — so the export policy in `docs/04_mcp_07_tool_schema_export_policy.md` is
enforced automatically by CI.

## Scope

**In**:
- New file `tests/test_mcp_tool_schema_exports.py`
- Minor fix: add explicit type annotation to `TOOL_LIST` in
  `scripts/mcp/file/read_tools.py` (currently untyped)
- Update `docs/04_mcp_07_tool_schema_export_policy.md` to reference the new test
  in the Validation section

**Out**:
- No changes to server logic, service layers, or the export values themselves
- No renaming of existing symbols; all modules already export `TOOL_LIST`
- No changes to `scripts/checks/check_no_compat.py` (already bans `_MCP_TOOLS`)
- Admin/audit tools (`scripts/mcp/audit.py`) — file has no tool list; excluded

## Assumptions

1. `mcp/` top-level directory does not exist; all server code lives under
   `scripts/mcp/` — confirmed by filesystem scan.
2. All 12 tool schema modules already export `TOOL_LIST` (confirmed by grep).
   No migration work is needed; this is a pure test-addition task.
3. `_MCP_TOOLS` is not present in any active module (only in
   `scripts/checks/check_no_compat.py` as a banned-pattern string) — confirmed.
4. `McpTool`, `MCPToolSchema`, and `dict[str, Any]` are all runtime-compatible
   with `dict`; the test can use `isinstance(tool, dict)` uniformly.
5. `scripts/mcp/file/read_tools.py` missing type annotation on `TOOL_LIST` is
   cosmetic but worth fixing for consistency; it does not affect the test.
6. sqlite server (`scripts/mcp/sqlite/`) has no `tools.py` and is excluded.
7. `scripts/mcp/github/tools.py` is the aggregator; sub-files
   (`tools_file.py`, `tools_issues.py`, `tools_pull_requests.py`,
   `tools_repository.py`) are internal and tested indirectly via the aggregator.

### Phase 2: Core Logic Implementation

- [ ] 2. Create `tests/test_mcp_tool_schema_exports.py`.

  The test module must:

  **2a. Define the canonical module list** (import paths relative to `scripts/`):
  ```python
  _TOOL_MODULES: list[tuple[str, str]] = [
      ("mcp.shell.tools",        "TOOL_LIST"),
      ("mcp.cicd.tools",         "TOOL_LIST"),
      ("mcp.git.tools",          "TOOL_LIST"),
      ("mcp.rag_pipeline.tools", "TOOL_LIST"),
      ("mcp.web_search.tools",   "TOOL_LIST"),
      ("mcp.mdq.tools",          "TOOL_LIST"),
      ("mcp.github.tools",       "TOOL_LIST"),
      ("mcp.file.read_tools",    "TOOL_LIST"),
      ("mcp.file.write_tools",   "TOOL_LIST"),
      ("mcp.file.delete_tools",  "TOOL_LIST"),
  ]
  ```

  **2b. Parametrize** over `_TOOL_MODULES` using `pytest.mark.parametrize`.

  **2c. For each module, assert**:
  - `importlib.import_module(module_path)` succeeds
  - `hasattr(mod, "TOOL_LIST")` is `True`
  - `isinstance(mod.TOOL_LIST, list)` is `True`
  - `len(mod.TOOL_LIST) > 0`
  - Every element is a `dict`
  - Every element has a non-empty `"name"` key

  **2d. Add a separate test** that asserts `_MCP_TOOLS` is not exported by any
  module in `_TOOL_MODULES`:
  ```python
  def test_no_legacy_mcp_tools_export():
      for module_path, _ in _TOOL_MODULES:
          mod = importlib.import_module(module_path)
          assert not hasattr(mod, "_MCP_TOOLS"), ...
  ```

  **2e. Use `sys.path` manipulation** in a `conftest`-style fixture or module-level
  setup to ensure `scripts/` is on the path (same pattern as
  `tests/test_mcp_tools_validation.py`).
