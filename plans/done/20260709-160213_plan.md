# Plan: Prevent process termination from test-suite and production killpg hazards

Source: `issues/20260709_00_issue.md` — Test execution causes process termination (LLM agent included).

## Scope note (read first)

This plan implements **Option C** (chosen via user decision): both test-level mocking (A) and production-code guard strengthening (B). The issue identified two affected test files (`test_lifecycle.py` HIGH, `test_mcp_tools_validation.py` MEDIUM) and one affected production module (`scripts/agent/http_lifecycle.py`).

## Classification (per Step 0 of python-issue-to-plan)

Large Task (Path B): affects > 3 files.

---

## 1. Goal

Prevent accidental process termination during test runs by eliminating real `os.killpg` / `subprocess.Popen` calls in integration tests, and by hardening `_terminate_with_timeout` against PID-reuse races in production.

## 2. Scope

**In-scope:**

- `tests/test_lifecycle.py`: add `subprocess.Popen` / `os.killpg` / `os.getpgid` / `httpx.AsyncClient` mocks to `TestHttpLifecycleStderrLog` (3 tests). The `test_start_timeout_stderr_from_log` test additionally needs synthetic stderr content in the log file so the assertion still passes.
- `tests/test_mcp_tools_validation.py`: patch `os.killpg` in `_terminate_process_group` and replace the `_process_group_gone` assertion (which uses `os.killpg(pgid, 0)`) with a `proc.poll()`-based check.
- `scripts/agent/http_lifecycle.py`: change `_terminate_with_timeout` to use `self._http_pgids.get(server_key)` (no fallback to `proc.pid`) and fall back to `proc.terminate()` when pgid is unavailable; change `start()` to not store `proc.pid` as pgid when `os.getpgid()` fails.

**Out-of-scope:**

- Not moving integration tests to `tests/integration/` (directory already exists but migration is a separate refactoring task).
- Not adding new test files or new feature code.
- Not changing `test_shell_mcp_service.py` / `test_agent_factory.py` / `test_startup.py` — already fully mocked.
- Not modifying `test_mcp_tools_validation.py` `subprocess.Popen` in `mcp_server` fixture — the fixture needs to spawn real servers for HTTP tool-validation tests.

## 3. Assumptions

1. `_terminate_with_timeout` is only called after `start()` has stored the pgid in `self._http_pgids[server_key]` — verified by `start()` (L217) and `restart()` (pgid popped after terminate). The fallback `proc.pid` path in `self._http_pgids.get(server_key, proc.pid)` is never reached in current code, but removing it eliminates the PID-reuse race.
2. `test_mcp_tools_validation.py`'s `_process_group_gone` assertion is a post-condition check only — it can be replaced with `proc.poll() is not None` without losing test value, because the preceding `proc.wait(timeout=timeout)` already confirms the process exited.
3. The `test_start_timeout_stderr_from_log` test's purpose is verifying that stderr content is captured in `StartupFailure.stderr_full`. This can be achieved without a real subprocess by pre-populating the stderr log file before the health poll reads it.

## 6. Implementation Steps

### Phase 1: Production-code hardening (`scripts/agent/http_lifecycle.py`)

1. **`_terminate_with_timeout` pgid guard (L96-98, L108):**
   - Change `pgid = self._http_pgids.get(server_key, proc.pid)` → `pgid = self._http_pgids.get(server_key)`
   - Guard `os.killpg(pgid, signal.SIGTERM)` and `os.killpg(pgid, signal.SIGKILL)` with `if pgid is not None: ... else: proc.terminate() / proc.kill()`
   - Applies to both the SIGTERM block (L96-100) and the SIGKILL block (L108-112).

2. **`start()` pgid storage (L217-219):**
   - Change `except OSError: self._http_pgids[server_key] = proc.pid` → `except OSError: pass` (leave pgid unset, so `_terminate_with_timeout` falls back to `proc.terminate()` instead of `os.killpg(proc.pid, ...)`).

   **Completion criterion for Phase 1:** `uv run pytest tests/test_lifecycle.py -k TestProcessGroupShutdown -v` passes (existing tests verify killpg behavior with monkeypatched os.killpg).

### Phase 2: Test-mocking in `tests/test_lifecycle.py`

3. **`test_start_large_stderr_does_not_block` (L479-494):**
   - Add `patch("agent.http_lifecycle.subprocess.Popen", return_value=_make_mock_proc(exit_code=1))` to mock the subprocess.
   - Add `patch("agent.http_lifecycle.httpx.AsyncClient")` to mock health checks.
   - The mock process exits immediately (exit_code=1), so the health loop detects early exit and raises `HttpStartupError("exited early")`.
   - Pre-populate stderr log file with `"x" * 200_000` before `mgr.start()`.

4. **`test_start_early_exit_stderr_from_log` (L498-513):**
   - Same pattern as step 3: mock `subprocess.Popen` returning a mock proc with `exit_code=1`.
   - Mock `httpx.AsyncClient`.
   - Pre-populate stderr log file with "BOOT_FAIL".

5. **`test_start_timeout_stderr_from_log` (L515-532):**
   - Mock `subprocess.Popen` returning a mock proc with `exit_code=None` (process stays alive).
   - Mock `httpx.AsyncClient` with `side_effect=Exception("connect refused")`.
   - Mock `os.killpg` / `os.getpgid` (via monkeypatch on `agent.http_lifecycle.os`).
   - Pre-populate stderr log file with "NEVER_READY".
   - The health poll loop times out after 1s → `_terminate_with_timeout` is called (with mocked killpg) → `HttpStartupError("did not become healthy")` is raised.
   - Assert `exc_info.value.failure.stderr_full` contains "NEVER_READY".

   **Completion criterion for Phase 2:** `uv run pytest tests/test_lifecycle.py -k TestHttpLifecycleStderrLog -v` passes without real subprocess or signal.

### Phase 3: Fix `tests/test_mcp_tools_validation.py`

6. **`_process_group_gone` replacement (L80-88):**
   - Change from `os.killpg(pgid, 0)` probe to `proc.poll() is not None` check.
   - Pass `proc` into the function or inline the check.

7. **`_terminate_process_group` killpg guard (L106-117):**
   - Patch `os.killpg` at the module level in the fixture, or wrap the `_terminate_process_group` calls in the fixture with `patch("os.killpg")`.
   - Since `_terminate_process_group` is a helper function (not a method), use `unittest.mock.patch` in the fixture's teardown path.

8. **`mcp_server` fixture teardown (L175):**
   - Wrap `_terminate_process_group(proc, timeout=5)` with `patch("tests.test_mcp_tools_validation.os.killpg")` context manager or `try/finally` block that patches the global `os.killpg`.

   **Completion criterion for Phase 3:** `uv run pytest tests/test_mcp_tools_validation.py -v` passes with `mock` applied (no real signals). Note: the integration tests (`@pytest.mark.integration`) will still spawn real servers via the `mcp_server` fixture; the patch only prevents signal delivery during teardown.

## 8. Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `test_start_timeout_stderr_from_log` loses coverage of real subprocess stderr capture path after mocking | Medium | Acceptable trade-off: the stderr tail reading logic (`_read_stderr_tail`) is a simple file-read method and doesn't need integration-level coverage. The test with pre-populated stderr still validates the `StartupFailure` construction path. |
| `test_mcp_tools_validation.py` fixture still spawns real subprocesses for the actual server | Low (by design) | The fixture's `subprocess.Popen` is NOT mocked because the tests need real MCP server HTTP endpoints. Only the kill-signal path is patched. The `_terminate_process_group` function still uses `proc.terminate()` (which sends real SIGTERM via Python's `Popen.terminate()`) — this is safe because it targets a specific PID, not a PGID. |
| `_terminate_with_timeout` change breaks existing behavior when `os.getpgid` fails in production | Low | The current code stores `proc.pid` as fallback when `os.getpgid` fails. The proposed change uses `proc.terminate()` instead. Both achieve the same goal (terminate the subprocess), but `proc.terminate()` is safer because it targets the exact PID without risking a group kill on a reused PID. |
