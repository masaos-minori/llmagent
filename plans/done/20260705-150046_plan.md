## Goal

Add automated tests that freeze the intended `/reload` configuration classification, covering hot-reloadable, deferred, restart-required, and startup-only settings.

## Scope

**In**: Unit tests for `ConfigReloadService.apply_config_dict()` covering all classification categories. Snapshot `ConfigReloadOutcome` structure for each key setting.

**Out**: Changing config file format, implementing live activation of startup-only subsystems, documentation-only updates.

## Assumptions

1. `ConfigReloadService.apply_config_dict()` is the central entry point for all reload logic.
2. Hot-reloadable fields apply immediately to live service instances via `_sync_services()`.
3. Deferred fields are stored in `ctx.cfg` but not applied to live connections until reconnect.
4. Restart-required fields are reported in `ConfigReloadOutcome.needs_restart` without being applied.
5. Startup-only fields are detected in `_detect_startup_only()` and reported in `ConfigReloadOutcome.startup_only`.
6. `_detect_startup_only()` currently only checks `use_memory_layer` and `plugin_strict`; missing `workflow_mode` and `workflow_require_approval`.

## Design

**`_detect_startup_only()` additions**:
```python
v = _get_str(new_cfg, "workflow_mode")
if v is not None and v != ctx.cfg.workflow.workflow_mode:
    changed.append("workflow_mode")
v = _get_bool(new_cfg, "workflow_require_approval")
if v is not None and v != ctx.cfg.workflow.workflow_require_approval:
    changed.append("workflow_require_approval")
```

**Test structure**: Use `AgentContext` with mocked services. Call `apply_config_dict({"key": new_value})`. Assert `ConfigReloadOutcome` field membership.
