## Goal
Ensure stderr resources are properly cleaned up when `os.getpgid()` fails after `Popen()` succeeds in `McpServerLifecycle.start_server()`.

## Scope
**In**: `scripts/agent/http_lifecycle.py` — refactor `getpgid()` error handling path
**Out**: Any changes to `Popen()` behavior, subprocess management logic, or other lifecycle methods

## Assumptions
1. A process whose pgid cannot be obtained should not be managed by the lifecycle manager
2. The `OSError` from `getpgid()` indicates the process has already exited or is in an unrecoverable state
3. Logging a warning is sufficient — no need to retry or alert on this transient condition

| Unknown | Evidence missing | Resolution | Blocking |

## Implementation Steps
1. Wrap `getpgid()` call in try/except with proper cleanup
2. Add warning log message when cleanup occurs
3. Remove process reference from `_http_procs` when cleanup happens
4. Return early from `start_server()` to prevent health check attempts on invalid process
