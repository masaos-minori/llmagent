# Plan: Add Connection Reuse Option to SQLiteHelper

**Requirement:** `requires/20260623_13_require.md`
**Task size:** Path B (Large — public interface change on `open()`, thread-safety constraints)

---

## Goal

Add an optional `reuse_connection: bool = False` parameter to `SQLiteHelper.open()` that, when `True`, causes `__exit__` to skip `close()`, allowing the same connection to be reused across multiple `with` blocks until explicit `close()` is called.

---

## Scope

**In:**
- `scripts/db/helper.py` — add `reuse_connection` param to `open()`; modify `__exit__` to skip `close()` when flag is set
- `tests/test_db_performance.py` — new file with `test_connection_reuse_same_instance` and `test_backward_compatibility_per_query`

**Out:**
- Thread-local storage — not required; caller is responsible for not sharing instances across threads (per AC: "thread-local boundaries must never be breached by different execution threads")
- Nested `open()` calls — out of scope; only multiple `with` blocks over same instance
- `tests/shared/test_db_performance.py` — `shared/` subdir doesn't exist; corrected to `tests/test_db_performance.py`

---

## Assumptions

1. `reuse_connection=True` transfers close responsibility to the caller, who must explicitly call `helper.close()`.
2. Multiple threads must not share a single `SQLiteHelper` instance with `reuse_connection=True`; this is enforced by convention, not by the library.
3. All existing callers use the default `reuse_connection=False`; backward compatibility is guaranteed (confirmed: only 4 call sites, all use default `open()`).
4. Tests use in-memory SQLite (`:memory:`) and `patch("db.helper.build_db_config")` to avoid real DB setup (confirmed pattern from `test_sqlite_helper.py`).

## Design

### `scripts/db/helper.py` changes

```python
def __init__(self, ...):
    ...
    self._reuse_connection: bool = False  # set by open()

def open(
    self,
    *,
    write_mode: bool = False,
    row_factory: bool = False,
    load_vec: bool | None = None,
    reuse_connection: bool = False,
) -> "SQLiteHelper":
    """...
    reuse_connection: when True, __exit__ skips close(); caller must call close() explicitly.
    """
    self._reuse_connection = reuse_connection
    # Skip reconnect if connection already open and reuse_connection=True
    if self.conn is not None and self._reuse_connection:
        return self
    use_vec = self._default_load_vec if load_vec is None else load_vec
    conn = self._connect()
    if use_vec and self._vec_so:
        self._load_vec_extension(conn)
    self._apply_connection_pragmas(conn, write_mode=write_mode)
    if row_factory:
        conn.row_factory = sqlite3.Row
    self.conn = conn
    return self

def __exit__(self, *_: object) -> None:
    if not self._reuse_connection:
        self.close()
```

### `tests/test_db_performance.py`

```python
def test_connection_reuse_same_instance(tmp_path, monkeypatch):
    # patch build_db_config to use tmp_path
    helper = SQLiteHelper.__new__(SQLiteHelper)
    # ... set up minimal instance with tmp_path db ...
    helper.open(reuse_connection=True)
    conn1_id = id(helper.conn)
    with helper:
        pass  # __exit__ should NOT close
    conn2_id = id(helper.conn)
    assert conn1_id == conn2_id
    assert helper.conn is not None
    helper.close()

def test_backward_compatibility_per_query(tmp_path, monkeypatch):
    # reuse_connection=False (default): __exit__ closes
    helper = SQLiteHelper.__new__(SQLiteHelper)
    helper.open()  # default reuse_connection=False
    with helper:
        pass
    assert helper.conn is None  # closed by __exit__
```

## Risks & Mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| Connection leak if caller forgets `close()` with `reuse_connection=True` | Medium | Document in docstring; no auto-cleanup (by design per AC) |
| Re-open with `reuse_connection=True` while connection already open creates ambiguity | Low | Skip reconnect if `conn` already open; document this behavior |
| `__exit__` skip-close breaks WAL checkpoint on crash | Low | WAL auto-checkpoints; no production impact since SQLite handles it |
| High churn file (8 recent commits) — merge conflicts | Medium | Minimal isolated change (2 lines in `__exit__`, 1 param in `open()`) |
