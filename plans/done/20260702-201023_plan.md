# Plan: Update `tests/test_create_schema.py` for DDL-only schema creation

## Goal

Remove all references to migration helpers from `tests/test_create_schema.py` and align the patched test schemas (`_RAG_SCHEMA_NO_VEC0`, `_SESSION_SCHEMA_NO_VEC0`) with the canonical DDL in `scripts/db/schema_sql.py` so that schema creation tests validate only latest DDL creation and idempotent `CREATE ... IF NOT EXISTS` behavior.

## Scope

**In**:
- `tests/test_create_schema.py` — remove migration-related comments; add missing columns to `_RAG_SCHEMA_NO_VEC0` and `_SESSION_SCHEMA_NO_VEC0`; add `session_diagnostics` table if confirmed; add column-presence assertions for `chunk_type`, `source_file`, `undone`
- `tests/test_create_schema.py` — add/update test methods that assert the new columns exist in the patched schema

**Out**:
- `scripts/db/create_schema.py` — no changes (migration helpers remain; out of scope)
- `scripts/db/schema_sql.py` — no changes
- Adding `chunk_type` / `source_file` to the canonical DDL `_RAG_SCHEMA_TEMPLATE` (separate task)
- Testing `ALTER TABLE` or old DB upgrade paths

## Assumptions

1. The canonical DDL in `scripts/db/schema_sql.py` is the ground truth; `_RAG_SCHEMA_NO_VEC0` and `_SESSION_SCHEMA_NO_VEC0` are intentional subsets (vec0 virtual tables removed for test environment).
2. `chunk_type TEXT NOT NULL DEFAULT 'text'` and `source_file TEXT NOT NULL DEFAULT ''` are required in `chunks` per the requirement; they are currently absent from both the canonical DDL (`_RAG_SCHEMA_TEMPLATE`) and the test patch (`_RAG_SCHEMA_NO_VEC0`).
3. `undone INTEGER NOT NULL DEFAULT 0` is already present in the canonical `_SESSION_SCHEMA_TEMPLATE` (`tool_results` table) but absent from `_SESSION_SCHEMA_NO_VEC0`.
4. `session_diagnostics` exists in the canonical `_SESSION_SCHEMA_TEMPLATE`; it is referenced in `TestTimestampDefaults.test_session_schema_timestamps` but absent from `_SESSION_SCHEMA_NO_VEC0`, which causes that test to fail against the patched schema — confirmation required.
5. All test commands are run with `uv run pytest`.
