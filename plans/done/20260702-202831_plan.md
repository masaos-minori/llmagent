# Plan: L-3 — Generate `/help` from canonical command definitions

Source: `requires/20260702_70_require.md`

---

## Goal

Make `_cmd_help()` in `registry.py` iterate over `_COMMANDS` imported from `command_defs.py` instead of the locally-defined duplicate, so that `/help` output is always in sync with the canonical command list.

---

## Scope

**In**:
- Remove the duplicate `_COMMANDS` list from `registry.py`
- Import `_COMMANDS` from `agent.commands.command_defs` in `registry.py`
- Ensure `_cmd_help()` and `dispatch()` both use the imported `_COMMANDS`
- Sync the two `_COMMANDS` lists: add `/approve` and `/reject` to `command_defs.py` (currently missing from it; present only in `registry.py`)

**Out**:
- Full redesign of the help UI
- `/exit` handling
- Plugin command help display redesign
- Changes to `AgentREPL.SLASH_COMMANDS` in `repl.py`

---

## Assumptions

1. `command_defs.py` is the declared single source of truth (per its module docstring: "single source of truth for all built-in slash commands").
2. The `/approve` and `/reject` entries present in `registry.py`'s `_COMMANDS` but absent from `command_defs.py` are an unintentional omission that must be resolved before the duplicate can be removed.
3. Tests for `CommandRegistry.__init__` (fail-fast handler validation) and `dispatch()` exist and cover the affected code paths.
4. No external code imports `_COMMANDS` directly from `registry.py` (confirmed: `factory.py` imports from `command_defs`, not `registry`).

## Implementation Steps

### Phase 1: Sync command_defs.py

1. **Add `/approve` and `/reject` to `command_defs.py`**
   - Insert both `CommandDef` entries at the correct position (after `/audit`, before `/plugin` — matching `registry.py` order)
   - Completion criterion: `command_defs._COMMANDS` contains exactly the same entries as `registry._COMMANDS`

### Phase 2: Update registry.py

2. **Replace the local `_COMMANDS` definition in `registry.py` with an import**
   - Remove the `_COMMANDS: list[CommandDef] = [...]` block from `registry.py`
   - Add `from agent.commands.command_defs import _COMMANDS, CommandDef, SubcommandSpec` to the import section
   - Keep `CommandDef` import (already present; just extend to include `_COMMANDS`)
   - Completion criterion: `registry.py` no longer defines `_COMMANDS` locally

3. **Verify `_cmd_help()` and `dispatch()` still reference `_COMMANDS` correctly**
   - No code change needed if the variable name is unchanged; verify by inspection
   - Completion criterion: both methods iterate `_COMMANDS` without modification

### Phase 3: Validation

4. **Run test suite**
   - `uv run pytest tests/ -x -q`
   - Confirm `CommandRegistry.__init__` fail-fast validation still passes

5. **Smoke-test the running agent**
   - `uv run python -m agent`
   - Type `/help` and confirm `/approve`, `/reject`, `/db`, `/mdq`, and `/plugin` are listed
