## 8. Risks & Mitigations
- **Risk**: `test_rag_http_mode.py` duplicates assertions already in `test_pipeline_http_result_kind.py` → **Mitigation**: Focus `test_rag_http_mode.py` on `SearchDiagnostics` field values (`result_source`, `http_result_kind`, `fallback_reason`) and `augment()` return value contract; leave `get_diagnostics()["http_result_kind"]` string mapping to the existing file
- **Risk**: `test_rag_refiner.py` duplicates `test_pipeline_refiner_fallback.py` → **Mitigation**: Keep `test_rag_refiner.py` focused on pipeline-level `get_diagnostics()` aggregation (refiner_fallback_count, refiner_returned_empty, refiner_exception_count); `test_pipeline_refiner_fallback.py` tests `refine_context()` unit-level
- **Risk**: Existing `test_rag_ingestion_pipeline.py` already covers most required invariants → **Mitigation**: Read file carefully before adding; only add `TestRagSchemaIndependence` if the schema-independence invariant is absent
- **Risk**: `RagDbMaintenanceService.rebuild_fts()` uses `SQLiteHelper("rag").open()` which expects a real DB file → **Mitigation**: Patch `SQLiteHelper` in tests; or test the `INSERT INTO chunks_fts(chunks_fts) VALUES('rebuild')` SQL directly on an in-memory FTS5 table
- **Risk**: FTS5 virtual table creation requires SQLite with FTS5 compiled in (standard Python `sqlite3` module includes FTS5) → **Mitigation**: No mitigation needed; Python's bundled `sqlite3` always includes FTS5; only `vec0` requires extension
