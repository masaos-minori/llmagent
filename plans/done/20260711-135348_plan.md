## Design

### `get_tool_names()` sorted-order guarantee (tool_registry.py)

```python
def get_tool_names(self, server_key: str) -> list[str]:
    """Return all tool names for a server_key, sorted alphabetically.

    Ordering is a guaranteed contract of this method (not just an artifact of
    default registration order) — callers and tests may rely on it directly
    without re-sorting.
    """
    return sorted(self._by_server.get(server_key, []))
```

### Snapshot tests (tests/test_tool_registry_counts.py)

One test class per remaining server group (mirroring the existing `TestRegistryRagPipelineCounts` pattern), each asserting the exact count from Assumption 1, with a failure message naming the server key and observed list:

```python
class TestRegistryServerGroupCounts:
    def test_file_read_tool_count(self) -> None:
        from shared.tool_registry import _reset_registry_for_testing, get_registry
        _reset_registry_for_testing()
        registry = get_registry()
        tools = registry.get_tool_names("file_read")
        assert len(tools) == 9, f"Expected 9 file_read tools, got {tools}"

    # ... repeated for file_write=4, file_delete=2, cicd=4, mdq=9, git=10, shell=1, web_search=1

    def test_get_servers_returns_all_ten_keys(self) -> None:
        from shared.tool_registry import _reset_registry_for_testing, get_registry
        _reset_registry_for_testing()
        registry = get_registry()
        expected = [
            "cicd", "file_delete", "file_read", "file_write", "git",
            "github", "mdq", "rag_pipeline", "shell", "web_search",
        ]  # sorted, matching get_servers()'s documented guarantee
        assert registry.get_servers() == expected, (
            f"Expected server keys {expected}, got {registry.get_servers()}"
        )
```

Each assertion's failure message states the server key and the actual list/count observed, per the Acceptance Criteria ("Test failure messages clearly indicate which count, server key, or ordering assumption drifted") — matching the existing file's own established style (`f"Expected N {server} tools, got {tools}"`).

### Ordering unit test (tests/test_tool_registry.py)

```python
def test_get_tool_names_returns_sorted_order() -> None:
    registry = ToolRegistry()
    registry.register(ToolDefinition(name="zebra", server_key="s1"))
    registry.register(ToolDefinition(name="apple", server_key="s1"))
    registry.register(ToolDefinition(name="mango", server_key="s1"))
    assert registry.get_tool_names("s1") == ["apple", "mango", "zebra"]
```

This deliberately registers in *non*-sorted order to prove the guarantee is enforced by `get_tool_names()` itself, not merely inherited from a caller that happens to insert sorted (directly targeting the requirement's concern about undocumented, accidental ordering).
