# Plan: Remove RAG-specific DB operations from AgentSession

## Goal

Move RAG-specific document operations out of the agent layer into `rag-pipeline-mcp`, remove dead code, and update docs so the session layer focuses solely on sessions, messages, and notes.

## Scope

**In scope:**
- `scripts/agent/document_repo.py` — remove dead code (not imported since req 21)
- `scripts/agent/services/db_maintenance_service.py` — remove `list_documents()`, `delete_document()` methods
- `scripts/agent/services/models.py` — remove `DbStats.docs`, `DbStats.chunks` if no longer needed; remove `DocumentRow` if unused
- `scripts/agent/commands/cmd_db.py` — update `_db_clean`, `_db_list_urls`, `_db_stats` to call rag-pipeline-mcp client instead of `DbMaintenanceService`
- `scripts/mcp/rag_pipeline/tools.py` — add `rag_list_documents`, `rag_delete_document` tool schemas
- `scripts/mcp/rag_pipeline/service.py` — implement `list_documents()`, `delete_document()` using `SQLiteHelper("rag")`
- `scripts/mcp/rag_pipeline/server.py` — register new tool handlers
- `docs/05_agent_09_data-layer.md` — update boundary description, remove stale "Future refactoring" note
- `docs/05_agent_90_inconsistencies_and_known_issues.md` — remove stale AgentSession RAG issue (resolved by req 21)
- `docs/05_agent_12_reference-api.md` — update open question about `/db clean` and `/db stats`
- `docs/05_agent_07_cli-and-commands.md` — verify `/db` command descriptions are accurate

**Out of scope:**
- Splitting `DbMaintenanceService` itself (only removing document methods)
- Changing session/session message/note operations in `AgentSession` (already correct)
- Adding RAG ingestion tools to rag-pipeline-mcp
- Changing the `_db_rebuild_fts`, `_db_health`, `_db_checkpoint`, `_db_vacuum`, `_db_purge`, `_db_recover` commands (they stay in `DbMaintenanceService`)
- Changes to `db/maintenance.py` (low-level ops stay)

## Assumptions

1. `document_repo.py` is truly dead code — `DocumentRepository` is not imported anywhere. Verified by grep across all `scripts/`.
2. The `DbStats.docs` and `DbStats.chunks` fields are used only by `DbMaintenanceService.stats()` and `_db_stats` display. If stats move to the MCP path, these fields can be removed.
3. The agent REPL has an MCP client that can call rag-pipeline-mcp tools. `ToolExecutor.execute()` is the dispatch path for MCP tools.
4. req 21 (commit `fecfdb1`) already removed document operations from `AgentSession`; the remaining work is moving them from `DbMaintenanceService` to `rag-pipeline-mcp`.
5. `rag-pipeline-mcp` runs as an internal HTTP MCP server and is accessible from the agent layer.

- Register both tools in `server.py`

### Step 4: Make `_cmd_db` and sub-handlers async

**File:** `scripts/agent/commands/cmd_db.py`

- Change `_cmd_db(self, args)` → `async def _cmd_db(self, args)`
- Change all sub-handlers that need MCP access: `_db_clean`, `_db_list_urls` → `async def`
- Update `CommandRegistry.dispatch()` to support async handlers (check if `_cmd_db` is callable via `inspect.iscoroutinefunction`)
- `_db_clean(url)`: call `rag_delete_document` via `ToolExecutor.execute()` instead of `DbMaintenanceService.delete_document(url)`
- `_db_list_urls(rest)`: call `rag_list_documents` via `ToolExecutor.execute()` instead of `DbMaintenanceService.list_documents(lang, limit)`
- Keep `_db_stats()` sync if it only uses `DbMaintenanceService.stats()` for non-document counts, or make it async too

**Completion criterion:** `/db clean <url>` and `/db urls` work through rag-pipeline-mcp via async command handlers.
