# 危険操作の承認モデル強化 — Work Plan

Source instruction: `instructions/20260606_06_instruction.md`

---

## Goal

ツール名ベースの承認から、引数内容・対象リソース・操作種別を含む危険度ベース承認モデルへ強化し、安全性と運用性を両立する。

---

## Scope

**In:**
- args と resource_scope を評価する危険度判定ロジックの実装
- 危険操作の事前プレビュー・dry_run 強制
- 承認結果の audit.log への構造化保存

**Out:**
- UI ベースの承認フロー変更（CLI のみ対象）
- 非同期承認（人間が不在の場合の自動承認）

## Assumptions

1. 現在の `tool_approval.py` は tool 名 + `approval_risk_rules` + path/branch escalation で動作中（コード確認済み）。
2. `approval_dry_run_tools` と `approval_protected_paths` は既に設定可能。
3. `approval_resource_keys` で path_keys / branch_keys が設定可能。

1. `tool_approval.py` を読み、現在の承認フローを完全に理解する
