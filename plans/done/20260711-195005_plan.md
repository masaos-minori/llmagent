# GitHub MCP Server Circular Import Fix — Work Plan

## Goal

Resolve the GitHub MCP server startup failure caused by circular import between `server.py` and `server_common.py` by converting module-level imports to lazy imports.

## Scope

**In-Scope:**
- Convert `_service` and `logger` imports in `server_common.py` from module-level to lazy imports inside functions
- Verify the GitHub MCP server starts without ImportError

**Out-of-Scope:**
- Modifying any endpoint function signatures in `server_file.py`, `server_issues.py`, `server_pull_requests.py`, `server_repository.py`
- Creating `shared_state.py` (long-term architectural improvement, not part of this plan)
- Adding CI circular import detection
- Addressing GITHUB_TOKEN missing (separate issue)

## Assumptions

1. Python's module caching ensures lazy imports have negligible performance impact after the first call
2. The existing `noqa: PLC0415` suppression on the original import line is valid and should be preserved
3. `_service` and `logger` are guaranteed to be initialized before any endpoint function is called at runtime
4. The fix applies only to `/opt/llm/scripts/mcp_servers/github/server_common.py` (production copy), not the source repo copy under `/home/sugimoto/llmagent/`
