# Plan: Add docs consistency check for RAG CLI option drift

## Goal
Add a `check_crawler_cli_options()` function to `check_docs_consistency.py` that detects
when docs reference crawler CLI options (`--force`, `--targets-file`) that are not defined
in `crawler.py` argparse, while permitting options marked as `planned` or `not implemented`.

## Scope

**In**:
- `scripts/checks/check_docs_consistency.py` — add `check_crawler_cli_options()` and register it in `check_all()`
- `docs/03_rag_05_configuration_and_operations.md` — fix two stale crawler option references (lines 266 and 291: `--targets-file`); add `planned` markers or remove
- `scripts/rag/ingestion/crawler.py` — read-only; extract argparse option list as ground truth

**Out**:
- `chunk_splitter.py` and `ingester.py` CLI drift checks (deferred, noted as future extension)
- New modules or packages
- Database schema changes
- CI workflow changes (CI already picks up `check_docs_consistency.py` via `rag-docs-quality.yml`)

## Assumptions

1. `crawler.py` argparse defines exactly two options: `--url` and `--lang`. Confirmed by reading `main()` (lines 443–463).
2. Docs use inline code spans (backtick-wrapped) for CLI options, e.g. `` `--targets-file` ``. The checker should scan for backtick-wrapped `--*` tokens.
3. A `<!-- planned: --foo -->` HTML comment or a `(planned)` / `(not implemented)` inline label near the option mention is an acceptable opt-out marker for planned options.
4. `check_docs_consistency.py` is called without file arguments in CI (`python scripts/checks/check_docs_consistency.py`), which defaults to all `docs/*.md`. The new check must work in that mode.
5. CI is already wired: `.github/workflows/rag-docs-quality.yml` runs `check_docs_consistency.py` on push/PR touching `docs/03_rag_*.md` or the checker itself. No CI changes needed.

## Implementation Steps

### Phase 1: Scope grep and decision

- [ ] Run `grep -n "targets-file\|crawler.*--force" docs/03_rag_*.md` to confirm all stale references
- [ ] Decide static allowlist vs. dynamic argparse import (recommended: static allowlist `{"--url", "--lang"}` — zero import risk)
- [ ] Decide `planned` marker syntax (recommended: `<!-- planned -->` HTML comment on the same line as the option mention)

### Phase 2: Add the checker function

- [ ] In `scripts/checks/check_docs_consistency.py`, add constant:
  ```python
  # CLI options currently implemented in crawler.py argparse (update when crawler.py changes)
  CRAWLER_CLI_OPTIONS: frozenset[str] = frozenset({"--url", "--lang"})
  ```
- [ ] Add function `check_crawler_cli_options(content: str, filename: str) -> list[str]`:
  - Scan only files matching `03_rag_*.md`
  - Extract all backtick-wrapped tokens matching `--[a-z][-a-z]*` via `re.findall(r'\`(--[a-z][-a-z]*)\`', line)`
  - For each token not in `CRAWLER_CLI_OPTIONS`: check the line for a `planned` or `not implemented` marker (case-insensitive); if absent, emit an issue
  - Skip lines inside fenced code blocks (reuse the `in_fenced_block` pattern already used in other check functions)
  - Issue format: `"{filename}:{i}: undocumented or unimplemented crawler CLI option — '{option}'"`
- [ ] Register `check_crawler_cli_options` in `check_all()`:
  ```python
  issues.extend(check_crawler_cli_options(content, filename))
  ```

### Phase 3: Fix stale doc references

- [ ] In `docs/03_rag_05_configuration_and_operations.md`, for each `--targets-file` occurrence (lines 266, 291):
  - Option A (remove): replace the example command with the current `--url`/`--lang` usage pattern
  - Option B (mark planned): add `<!-- planned -->` comment after the line
  - Preferred: Option A — `--targets-file` is not implemented; showing it as usage is misleading
- [ ] Verify `crawler.py --force` row in the `--force` behavior table (line 137–139): the table already says "Not applicable" for crawler — confirm the new checker does not false-positive on that table row
