## Goal
Remove private attribute access (`_db`) from external modules and recover all pending approvals instead of just the latest one.

## Scope
**In**: `scripts/agent/state_store.py`, `scripts/agent/startup.py` — add public accessor method, update all callers
**Out**: Changes to the underlying database layer or query logic itself

## Assumptions
1. A simple getter method is sufficient — no need for complex abstraction
2. Multiple pending approvals can exist simultaneously and should all be recovered

## Unknowns
1. ~~Is there a specific scenario where multiple pending approvals can exist?~~ Resolved: yes, concurrent operations can create multiple pending approvals before any is resolved.

## Affected areas
- State management
- Workflow recovery
- Cross-module communication

## Design
Add a public `get_connection()` method to StateStore that returns the underlying DB connection. Update all callers to use this method instead of accessing `_db` directly. Also fix the incomplete recovery by finding all pending approvals.

## Implementation steps
1. Add `get_connection()` method to StateStore with clear documentation
2. Update startup.py to use `store.get_connection()` instead of `store._db`
3. Run tests to verify no regressions
