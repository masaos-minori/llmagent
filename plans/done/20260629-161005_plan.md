## 1. Goal

Harden the existing incremental `refresh_index` implementation by closing gaps between the requirement spec and the current code, and ensure full test coverage for all acceptance criteria.

## 2. Scope

- **In-Scope**:
  - `scripts/mcp/mdq/indexer.py` — change-detection logic (`refresh_paths`, `_delete_file_from_index`, `_index_single_file`)
  - `scripts/mcp/mdq/service.py` — `refresh_index` method and `_format_refresh_summary`
  - `scripts/mcp/mdq/models.py` — `RefreshIndexRequest`, `RefreshIndexResponse` (already defined; verify completeness)
  - `tests/test_mdq_incremental_refresh.py` — add missing coverage for `content_hash` + `size_bytes` change detection
- **Out-of-Scope**:
  - RAG pipeline (`rag/` layer) — MDQ uses its own SQLite DB (`mdq.sqlite`)
  - `docs/` updates (separate task)
  - Hybrid search / embedding layer
  - `workflow.sqlite` / `session.sqlite` schemas

## 3. Assumptions

- The MDQ SQLite schema (`documents`, `chunks`, `index_state` tables) introduced in the production-schema migration is already deployed at `/opt/llm/db/mdq.sqlite`.
- `mtime_ns` alone is the primary change-detection signal. `size_bytes` and `content_hash` are stored but not yet used for skip/re-index decisions — this is the primary gap to close.
- The `_delete_file_from_index` trigger DROP/RECREATE approach is fragile; the `ON DELETE CASCADE` FK on `chunks.doc_id` is a safer alternative that already exists in the schema.
- Tests run in `tmp_path` isolation; no production DB is touched during testing.

## 6. Implementation Steps

1. **Phase 1: Fix data integrity issues**
   - [ ] `indexer.py` `_index_single_file`: compute whole-file `content_hash` (`hashlib.sha256(path.read_bytes()).hexdigest()`) before the section loop; use it for the `documents` upsert (fixes UNK-04).
   - [ ] `indexer.py` `_delete_file_from_index`: remove manual DROP/RECREATE of `chunks_ad` trigger; rely on `ON DELETE CASCADE` FK (`chunks.doc_id → documents.doc_id`) to cascade deletes. Delete `documents` row only; chunks and FTS entries are cleaned up via FK cascade and existing `chunks_ad` trigger (fixes UNK-01).
   - [ ] Verify `ON DELETE CASCADE` is enabled — `_get_db_connection` must set `PRAGMA foreign_keys=ON` (currently absent in `service.py`; add it).

2. **Phase 2: Extend change-detection to use size_bytes + content_hash**
   - [ ] `indexer.py` `refresh_paths` incremental path: after mtime check, additionally compare `size_bytes` and `content_hash` from `documents` table. Skip only if all three match current filesystem values. This satisfies requirement "Track file metadata: mtime_ns, size_bytes, content_hash".
   - [ ] Extract skip-decision logic into a private helper `_file_is_unchanged(conn, doc_id, path) -> bool` to reduce CC complexity of `refresh_paths` (currently grade E=34).
   - [ ] Extract per-file indexing-with-state-update into `_refresh_single_file(service, conn, path, indexed_count, failed_count) -> tuple[int, int]` to further reduce nesting.

3. **Phase 3: Output format — return structured summary as JSON-serializable dict**
   - [ ] `service.py` `refresh_index`: current `_format_refresh_summary` returns `list[str]` which is joined to plain text. Requirement says "Return structured refresh summary as text or JSON string." The `_refresh_paths` function already returns a `dict`; expose option to return JSON string. Add `output_format: Literal["text", "json"] = "text"` to `RefreshIndexRequest` (models.py) and branch in `service.py`.
   - [ ] Update `tools.py` `refresh_index` inputSchema to document `output_format` property.

4. **Phase 4: Tests**

   - [ ] Add test: skip logic uses `size_bytes` — create file, index, append byte, restore mtime; expect re-index.
   - [ ] Add test: `output_format=json` returns valid JSON string with all 5 fields.
   - [ ] Add test: verify `_delete_file_from_index` via CASCADE does not leave orphan FTS5 rows (fts_consistency_check passes after deletion).
   - [ ] Run `uv run pytest tests/test_mdq_incremental_refresh.py -v` — all tests must pass.

5. **Phase 5: Deployment & Verification**

## 8. Risks & Mitigations

- **Risk**: Removing `index_state` as primary skip signal (UNK-02) could break the directory-scan deletion detection which relies on `mtime:` prefix keys. → **Mitigation**: Keep `index_state` writes for now; add `documents` table comparison as *additional* check. Full consolidation is a separate refactor task.
- **Risk**: Adding `PRAGMA foreign_keys=ON` to `_get_db_connection` changes connection behavior for all queries, potentially exposing latent FK violations. → **Mitigation**: Audit existing INSERT/DELETE statements for FK safety before enabling; run full test suite after change.
- **Risk**: `output_format` field addition is a public interface change (MCP tool schema). → **Mitigation**: Field is optional with default `"text"`; existing callers are unaffected. Mark as additive in changelog.
- **Risk**: `refresh_paths` CC complexity E (34) means high cognitive load for reviewers and regression risk. → **Mitigation**: Phase 2 extraction of `_file_is_unchanged` and `_refresh_single_file` reduces complexity before adding new logic.
