# Plan: Unify the memory JSONL configuration name and ownership

Source: `requires/20260626_05_require.md`

---

## Goal

Ensure docs, config loading, status output, and error messages all consistently use `memory_jsonl_dir` (the field name in code and config), and remove any doc references to a non-existent `memory_jsonl_path` key.

---

## Scope

**In-Scope**
- Audit all doc references to memory JSONL configuration key (`memory_jsonl_dir` vs `memory_jsonl_path`)
- Update docs to use the correct canonical name `memory_jsonl_dir`
- Add tests for correct JSONL path resolution: `memory_jsonl_dir` + `/memories.jsonl` = full path used by factory.py
- Confirm that config_builders.py, factory.py, and status display all use `memory_jsonl_dir` consistently

**Out-of-Scope**
- Replacing JSONL as the canonical source
- Refactoring memory storage subsystem
- Adding backward-compat alias for `memory_jsonl_path` (no known external callers)

---

## Assumptions

1. Code canonical name is `memory_jsonl_dir` (confirmed: config_dataclasses.py:299, config_builders.py:204, factory.py:285).
2. Full path = `memory_jsonl_dir + "/memories.jsonl"` (confirmed: factory.py:285).
3. `memory_jsonl_path` exists only in docs (not in code); no alias is needed.
4. Status display already uses `memory_jsonl_dir` (confirmed: cmd_config_display.py:159).
