## Implementation Steps

### Step 1: `agent/memory/retriever.py` — スコアリング設定可能化
- `_FTS_CANDIDATE_LIMIT` / `_RRF_K` / recency 設定値を `MemoryConfig` から注入できるようにする
- `AgentConfig.memory` に `memory_fts_limit`, `memory_rrk_k`, `memory_recency_days` 等のフィールドを追加
- `MemoryRetriever.__init__()` でこれらの値を受け取るよう変更
- `python-refactoring` スキルを使用する

### Step 2: `agent/memory/jsonl_store.py` — 書き込み直列化
- `asyncio.Lock` を導入して `write()` の並行呼び出しを直列化する
- `read_all()` の malformed line 処理に retry または skip カウンターを追加する
- `python-refactoring` スキルを使用する

### Step 3: `agent/memory/extract.py` — 抽出スコープ拡張
- `_try_extract_from_assistant()` に加えて `_try_extract_from_user()` を追加
- ツール出力（`tool` ロール）からの抽出を検討
- 重要度優先によるエントリーキャップ (`MAX_ENTRIES = 20`) の実装確認・改善
- `python-refactoring` スキルを使用する

### Step 4: `mcp/models.py` — バリデーション層の検討（Optional）
- `CallToolRequest.args` に name-based バリデーション層が必要かを判断
- 影響範囲が大きい場合は別プランとして分離する

### Step 5: バリデーション実行
- `uv run ruff check scripts/`
- `uv run mypy scripts/`
- `lint-imports`
- `uv run pytest`
- `diff-cover coverage.xml` (changed lines ≥ 90%)

### Step 6: instructions/done/ へ移動
- `instructions/20260603_01_instruction.md` を `instructions/done/` に移動する

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| retriever の設定変更が既存のメモリ検索品質を劣化させる | medium | 変更前後で同じデフォルト値を使用し、設定注入は optional にする |
| extract.py のスコープ拡張がノイズメモリを増やす | medium | 新抽出パスのデフォルトを importance=0.3 以下に設定し、既存のキャップ制限を維持する |
| jsonl_store.py の Lock 追加が既存の async テストに影響する | low | テスト実行時は単一タスクのみのため影響なし |
