# Plan: M-7 — Review SIGTERM behavior while waiting for blocking input

## Goal

Race `_read_input()` against `_shutdown_event.wait()` in `AgentREPL` so that a SIGTERM received while the REPL is waiting for user input causes `_read_input()` to return `None` promptly, enabling graceful shutdown without waiting for the next user keystroke.

## Scope

**In**: `scripts/agent/repl.py` — `_read_input()` method: add a race between `loop.run_in_executor(input)` and `_shutdown_event.wait()`.

**Out**: No changes to SIGTERM handler setup, `_repl_loop()`, `startup.py`, `factory.py`, or any other module.

## Assumptions

1. `_shutdown_event` is an `asyncio.Event` set on SIGTERM (confirmed: line 456, 461 in `repl.py`).
2. The blocking `input()` call in `run_in_executor` cannot be cancelled; the executor thread continues running until the user types a line. The race returns `None` on SIGTERM, leaving the executor thread alive until the next keystroke — this is acceptable (thread exits after user types, REPL loop is already shut down).
3. `asyncio.wait({input_task, shutdown_task}, return_when=FIRST_COMPLETED)` is the correct pattern for this race.
4. The input executor task is not cancelled (cancellation of `run_in_executor` does not interrupt `input()`); let it drain naturally.
5. SIGTERM may arrive before `_shutdown_event` is initialized (before `run()` sets it at line 456); guard with `if self._shutdown_event is None`.

## Implementation Steps

1. **Update `_read_input()` to race input against shutdown event**
   - Replace the current `run_in_executor` await with:
     ```python
     async def _read_input(self, loop: asyncio.AbstractEventLoop) -> str | None:
         input_task = loop.run_in_executor(None, lambda: input(self._prompt))
         if self._shutdown_event is not None:
             shutdown_task = asyncio.ensure_future(self._shutdown_event.wait())
             done, pending = await asyncio.wait(
                 {asyncio.ensure_future(input_task), shutdown_task},
                 return_when=asyncio.FIRST_COMPLETED,
             )
             # Cancel shutdown_task if input won
             for t in pending:
                 t.cancel()
             if shutdown_task in done:
                 return None
             raw = input_task.result()
         else:
             raw = await input_task
         # ... existing multiline handling
     ```
   - Preserve existing `EOFError`/`KeyboardInterrupt` handling.
   - Criterion: SIGTERM while idle returns `None` from `_read_input()` promptly.

2. **Add tests**
   - Test: `_shutdown_event.set()` before `_read_input()` completes → returns `None`.
   - Test: normal input with shutdown event unset → returns input string.
   - Test: EOF still returns `None`.
   - Test: `_shutdown_event is None` (early call) → falls back to blocking await safely.
   - Criterion: `uv run pytest tests/ -k "repl" -v` passes.
