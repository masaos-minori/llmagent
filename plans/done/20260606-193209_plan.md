# 履歴圧縮の重要度ベース化 — Work Plan

Source instruction: `instructions/20260606_04_instruction.md`

---

## Goal

`HistoryManager` の固定 `compress_turns * 2` 圧縮を廃止し、importance / pinned / 圧縮禁止区分を持つ重要度ベース圧縮に変更する。

---

## Scope

**In:**
- `LLMMessage` に importance / pinned 相当のメタデータ追加
- `HistoryManager.compress()` の重要度ベース選別ロジック実装
- 圧縮対象外区分の定義（system, note, tool error, 方針修正発話）
- 「失われた重要項目一覧」の別管理

**Out:**
- セッションをまたぐ記憶（メモリ層の役割）
- UI での importance 設定機能（後続タスク）

---

## Assumptions

1. 現在の `HistoryManager` は `compress_turns * 2` の固定方式（`history.py:23`）。
2. `LLMMessage` は `role: str, content: str, ...` の構造（`shared/types.py`）。
3. `agent/session.py` の `messages` テーブルに importance を追加する場合はスキーマ migration が必要。

## Implementation Steps

1. `shared/types.py` の `LLMMessage` に `importance: float = 0.5` と `pinned: bool = False` を追加（後方互換デフォルト）
2. `HistoryManager` に重要度ベースの圧縮対象選別ロジックを実装
   - `role="system"` / `role="tool"` でエラーを含む / 方針修正キーワードを含む → 圧縮対象外
   - `importance >= threshold` または `pinned=True` → 圧縮対象外
   - 残余を compress して要約
3. `force_compress()` も同様のポリシーに対応
4. 「失われた重要項目一覧」を `HistoryManager` が返す構造化結果に含める
5. `python-test-and-fix` スキルでテスト追加
6. バリデーション実行 + instructions/done/ へ移動
