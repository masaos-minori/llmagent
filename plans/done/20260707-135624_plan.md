# Plan: M-3 ‚Äî Make /mcp status avoid false HEALTHY fallback when health registry is missing

## Goal

Replace the `McpServerHealthState.HEALTHY` fallback in `_resolve_health_state()` with an `UNKNOWN` state added to `McpServerHealthState`, so that `/mcp status` clearly distinguishes a tracked-healthy server from an untracked server.

## Scope

**In**: `scripts/shared/mcp_health.py` ‚Äî add `UNKNOWN = "unknown"` to `McpServerHealthState`; `scripts/agent/services/mcp_status.py` ‚Äî return `UNKNOWN` when registry is `None`.

**Out**: No changes to `cmd_mcp.py` renderer beyond handling the new `"UNKNOWN"` health string (already rendered via `.value.upper()`). No changes to `McpProbeResult` (health field is already `str`). No changes to `factory.py`, `context.py`, or DB schema.

## Assumptions

1. `McpServerHealthState` is a plain `Enum` (not `StrEnum`); adding a new value is backward-compatible for existing comparisons.
2. `_resolve_health_state()` returns a `McpServerHealthState` whose `.value.upper()` is used as the health string in `McpProbeResult.health` ‚Äî adding `UNKNOWN` with value `"unknown"` renders as `"UNKNOWN"` in the table.
3. H-2 (shared health registry) ensures the registry is non-None after normal startup; `UNKNOWN` only appears in degraded or test scenarios.
4. No code currently compares the health string `"HEALTHY"` from `McpProbeResult.health` directly ‚Äî it's only displayed.

## Implementation Steps

1. **Add `UNKNOWN` to `McpServerHealthState` in `mcp_health.py`**
   - Add `UNKNOWN = "unknown"` to the `McpServerHealthState` enum.
   - Criterion: `McpServerHealthState.UNKNOWN.value == "unknown"`.

2. **Update `_resolve_health_state()` in `mcp_status.py`**
   - Change: `return McpServerHealthState.HEALTHY` ‚Üí `return McpServerHealthState.UNKNOWN` when `registry is None`.
   - Criterion: `_resolve_health_state()` with `registry=None` returns `McpServerHealthState.UNKNOWN`.

3. **Add tests**
   - Test: `_resolve_health_state()` with `registry=None` ‚Üí `McpServerHealthState.UNKNOWN`.
   - Test: `_resolve_health_state()` with registry present ‚Üí returns registry's reported state (not UNKNOWN).
   - Test: `McpProbeResult.health` is `"UNKNOWN"` when registry is missing (integration with `probe_all()`).
   - Criterion: `uv run pytest tests/ -k "mcp_status" -v` passes.
