# Plan: Generate or validate command documentation from command metadata

## Goal

Create a script that reads `_COMMANDS` and generates/validates command documentation to prevent drift between implementation and docs.

## Scope

### Files to modify
- New file: `scripts/generate_command_docs.py` — script to generate command docs from `_COMMANDS`
- Optional CI integration: add validation step to CI pipeline

### Files NOT to modify (per requirement spec)
- `agent/commands/command_defs_list.py`
- `agent/commands/registry.py`
- Runtime command handlers
- Memory implementation files

## Design Decisions

### Decision 1: Generation + Validation Mode
The script should support two modes:
- **Generate**: Render full command documentation from `_COMMANDS`
- **Validate**: Compare generated docs against existing docs and report drift

This is more useful than just one mode — it allows both creation and ongoing verification.

### Decision 2: Output Format
Output should be Markdown tables matching the existing documentation style:
```markdown
| Command | 副作用 | 関連する状態 |
|---|---|---|
```

### Decision 3: Include SubcommandSpec Entries
When `SubcommandSpec.entries` is populated, render them as nested sub-tables under the parent command.

### Step 2: Implement `generate_docs()`

1. Import `_COMMANDS` from `agent.commands.command_defs_list`
2. For each `CommandDef`:
   - Extract: name, prefix behavior (exact match vs prefix), async flag, help text
   - If `SubcommandSpec.entries` exists, render nested sub-table
3. Format as markdown table with columns: Command, Side effects, Related state
4. Write to specified output path

### Step 3: Implement `validate_docs()`

1. Run `generate_docs()` in memory (no write)
2. Read `expected_path` content
3. Compare line-by-line or token-by-token
4. Report differences: added commands, removed commands, changed descriptions

### Step 4: Add CI drift check (optional)

1. Add a CI step that runs `generate_command_docs.py --mode validate --expected <path>`
2. Fail CI if any drift is detected
3. Recommend updating via `generate_command_docs.py --mode generate --output <path>`

## Acceptance Criteria Verification

1. Documentation can be generated from the implementation source — verify by running generate mode
2. Drift between `_COMMANDS` and documentation is detectable — verify by temporarily modifying `_COMMANDS` and running validate mode
3. This issue does not change runtime command behavior — no changes to handler code
