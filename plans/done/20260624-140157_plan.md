## Goal

rag/session/workflow の 3 DB スキーマ初期化・migration を統一ライフサイクルに揃え、rag/session と同じパターン (`SQLiteHelper` + 専用 `_migrate_*` 関数 + `create_schema.py` 統合) を workflow にも適用する。

## Scope

**In**:
- `scripts/db/schema_sql.py` — workflow DDL を `_WORKFLOW_SCHEMA` + `build_workflow_schema_sql()` として追加
- `scripts/db/workflow_schema.py` — `init_schema()` を `SQLiteHelper` 使用に変更、migration を `_migrate_workflow_schema()` に分離
- `scripts/db/create_schema.py` — `create_workflow_schema()` 追加、`create_schema()` に統合

**Out**:
- フル DB migration フレームワークの導入
- ワークフローテーブル設計の変更
- `workflow_schema.py` の `__main__` エントリポイント削除

## Assumptions

1. `SQLiteHelper("workflow")` が `workflow.sqlite` を開くことは確認済み (helper.py:65 参照)。
2. `workflow.sqlite` は session/rag と異なり embedding を使わないため `dims` パラメータは不要。
3. `workflow_schema.py` は `PYTHONPATH=scripts python -m db.workflow_schema` として `init_db.sh` から呼ばれている。この `__main__` エントリポイントは保持する。
4. `create_schema.py` の `create_schema()` は rag + session の両方を初期化する便利ラッパーだが、workflow を追加する場合は `create_all_schemas()` と分離するか、`create_schema()` を拡張するかを決める。拡張する場合は後方互換。

## Design

### 統一ライフサイクルモデル

```
schema_sql.py          — DDL テンプレート文字列 (build_*_schema_sql 関数)
create_schema.py       — create_*_schema() + _migrate_*_schema() 関数
workflow_schema.py     — workflow 専用 __main__ エントリポイント
helper.py              — SQLiteHelper で DB open/close
```

### schema_sql.py 追加

```python
_WORKFLOW_SCHEMA: str = """
    PRAGMA journal_mode=WAL;
    PRAGMA foreign_keys=ON;
    CREATE TABLE IF NOT EXISTS tasks (...);
    ...
"""

def build_workflow_schema_sql() -> str:
    return _WORKFLOW_SCHEMA
```

(`dims` 不要のため引数なし)

### workflow_schema.py 変更

```python
def _migrate_workflow_schema(conn: sqlite3.Connection) -> None:
    """Add missing workflow columns idempotently."""
    columns = {row[1] for row in conn.execute("PRAGMA table_info(tasks)").fetchall()}
    if "workflow_id" not in columns:
        conn.execute("ALTER TABLE tasks ADD COLUMN workflow_id TEXT")

def init_schema(db_path: str) -> None:
    """Create workflow tables using SQLiteHelper for WAL/FK/timeout policy."""
    from db.helper import SQLiteHelper
    with SQLiteHelper("workflow").open(write_mode=True) as db:
        db.executescript(build_workflow_schema_sql())
        _migrate_workflow_schema(db.conn)
```

### create_schema.py 追加

```python
def create_workflow_schema() -> None:
    """Create workflow.sqlite tables."""
    from db.workflow_schema import init_schema
    from db.config import build_db_config
    cfg = build_db_config()
    init_schema(cfg.workflow_db_path)
    logger.info("Workflow schema created successfully.")
```
