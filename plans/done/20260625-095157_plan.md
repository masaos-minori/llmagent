## 1. Integration Testing Architecture

### Test Environment Design Principles

**Principle 1: Boundary mocking, not unit mocking.**
Mock at the transport boundary (HTTP, stdio, LLM SSE), not at the service boundary. This tests real code paths through `ToolExecutor`, `StdioTransport`, `HttpTransport`, and `LLMClient` without starting actual servers.

**Principle 2: Real SQLite, in-memory databases.**
Use `SQLiteHelper` with an in-memory SQLite path (`":memory:"`) or a temp-file path via `tmp_path`. Do NOT mock `SQLiteHelper` — test the actual WAL + FK + `begin_immediate()` behavior.

**Principle 3: pytest-asyncio for all async tests.**
Use `@pytest.mark.asyncio` with `asyncio_mode = "auto"` in `pyproject.toml`. All async components (`ToolExecutor.execute()`, `StdioTransport.call()`, `LLMClient.complete()`) require an event loop.

### Component Mocking Strategy

#### 1a. MCP Server (HTTP transport)

Use `httpx.MockTransport` or `respx` to mock HTTP responses:

```python
import httpx
import respx

@respx.mock
async def test_tool_call_http_timeout():
    respx.post("http://localhost:8080/v1/call_tool").mock(
        side_effect=httpx.TimeoutException("timeout", request=None)
    )
    # ... test code
```

For transport-level failures (5xx, connection reset), use `side_effect=httpx.ConnectError`.
For tool-level errors (`is_error=True`), return a `{"result": "...", "is_error": true}` JSON body.

#### 1b. MCP Server (stdio transport)

Simulate a stdio server by spawning a minimal subprocess that reads one JSON-RPC request and writes one JSON-RPC response:

```python
# conftest.py
import asyncio
import subprocess

@pytest.fixture
async def stdio_echo_server():
    """Minimal stdio MCP server that echoes requests."""
    proc = await asyncio.create_subprocess_exec(
        "python", "-c",
        'import sys, json; line=sys.stdin.readline(); r=json.loads(line); sys.stdout.write(json.dumps({"id": r["id"], "result": "ok", "is_error": False}) + "\\n"); sys.stdout.flush()',
        stdin=asyncio.subprocess.PIPE, stdout=asyncio.subprocess.PIPE,
    )
    yield proc
    proc.terminate()
```

For pipe closure simulation: `proc.stdin.close()` before the call → `BrokenPipeError`.

#### 1c. LLM SSE Stream

Monkeypatch `LLMClient.complete()` with an async generator that raises `httpx.RemoteProtocolError` after N tokens:

```python
async def _partial_stream(tokens):
    for t in tokens:
        yield {"type": "content", "delta": t}
    raise httpx.RemoteProtocolError("peer disconnected", request=None)
```

#### 1d. SQLite Busy / Lock Simulation

Use SQLite's `PRAGMA locking_mode = EXCLUSIVE` in a separate thread to hold a write lock, forcing `SQLITE_BUSY` in the agent:

```python
import sqlite3, threading

def hold_write_lock(db_path, duration_sec):
    conn = sqlite3.connect(db_path, timeout=0)
    conn.execute("PRAGMA locking_mode = EXCLUSIVE")
    conn.execute("BEGIN EXCLUSIVE")
    time.sleep(duration_sec)
    conn.close()

lock_thread = threading.Thread(target=hold_write_lock, args=(db_path, 2.0))
lock_thread.start()
# Now attempt agent write — should see SQLITE_BUSY
```

## 3. Robustness & Chaos Injection Strategy

### 3A. ToolLoopGuard Chaos

**Target:** `agent/tool_loop_guard.py`

Scenarios to inject:
1. **Exact duplicate tool call:** Same tool name + same args in consecutive rounds → guard should detect and stop
2. **Near-duplicate with different args:** Verify guard does NOT fire on args variation
3. **Cycle detection:** Call sequence A → B → A → B → ... → guard fires on cycle
4. **Error escalation:** N consecutive tool errors → guard fires with "repeated_errors" reason
5. **Reset between turns:** Guard state clears at turn start → no cross-turn state leakage

Injection method:
```python
# Inject repeated tool calls via mock LLM
def _make_repeated_tool_response(tool_name, args, count=10):
    """Return a mock LLM stream that calls the same tool N times."""
    ...
```

### 3B. ErrorInjectionService Chaos

**Target:** `agent/error_injection_service.py`

Scenarios:
1. **Inject transport error at specific round:** Verify error propagates to `stat_transport_errors`
2. **Inject multiple error types:** Transport error + tool error in same turn → correct error type recorded
3. **Inject error during multi-tool batch:** One tool fails, others succeed → partial result handling

### 3C. SQLite Chaos

1. **Busy lock injection:** Hold external EXCLUSIVE lock for 1s → verify agent waits up to `busy_timeout`
2. **Disk full simulation:** Use `unittest.mock.patch` on `sqlite3.connect()` to raise `sqlite3.OperationalError("disk I/O error")`
3. **WAL checkpoint failure:** Simulate WAL file read-only → verify graceful degradation

### 3D. Network Chaos

1. **Intermittent 502s:** Server alternates between 502 and 200 → retry logic handles correctly
2. **Connection reset:** `httpx.RemoteProtocolError` → treated as transport error
3. **Partial response body:** HTTP 200 with truncated JSON body → `ValueError` in `_parse_http_response` → `TransportError`

"""tests/integration/test_agent_mcp_integration.py"""
from __future__ import annotations

import asyncio
import json
import pytest
import httpx
from unittest.mock import AsyncMock, MagicMock, patch

from shared.mcp_config import McpServerConfig
from shared.tool_executor import ToolExecutor, StdioTransport, TransportError, ToolCallResult


# ── Fixtures ─────────────────────────────────────────────────────────────────

@pytest.fixture
def server_cfg() -> McpServerConfig:
    return McpServerConfig(
        key="test_server",
        url="",
        transport="stdio",
        tool_names=["echo_tool"],

@pytest.fixture
async def stdio_echo_proc():

# ── Test: TC-A06 — stdio pipe closes during call ─────────────────────────────

@pytest.mark.asyncio
async def test_stdio_pipe_close_raises_transport_error(stdio_echo_proc, server_cfg):
    """Closing stdin before the call should produce a TransportError."""
    async with httpx.AsyncClient() as http:
        executor = ToolExecutor(
            http=http,
            cache_ttl=0.0,
            server_configs={"test_server": server_cfg},
        )
        transport = StdioTransport(
            cmd=[],
            server_key="test_server",
        )
        transport._proc = stdio_echo_proc
        executor.set_transport("test_server", transport)

        # Close stdin to simulate pipe closure
        stdio_echo_proc.stdin.close()

        result = await executor.execute("echo_tool", {})
        assert result.is_error
        assert result.error_type == "transport"
        # transport error counter incremented
        errors = executor.get_error_counters()
        assert errors.get("test_server", {}).get("transport", 0) >= 1


# ── Test: TC-A09 — response ID mismatch (stdio) ──────────────────────────────

@pytest.mark.asyncio
async def test_stdio_response_id_mismatch(server_cfg):
    """Server sends back a response with wrong id — TransportError expected."""
    bad_script = (
        "import sys, json\n"
        "for line in sys.stdin:\n"
        "    req = json.loads(line)\n"
        "    # Send response with wrong id (99 instead of req id)\n"
        "    resp = {'id': 99, 'result': 'wrong', 'is_error': False}\n"
        "    sys.stdout.write(json.dumps(resp) + '\\n')\n"
        "    sys.stdout.flush()\n"
    )
    proc = await asyncio.create_subprocess_exec(
        "python", "-c", bad_script,
        stdin=asyncio.subprocess.PIPE, stdout=asyncio.subprocess.PIPE,
    )
    try:
        async with httpx.AsyncClient() as http:
            executor = ToolExecutor(
                http=http, cache_ttl=0.0,
                server_configs={"test_server": server_cfg},
            )
            transport = StdioTransport(cmd=[], server_key="test_server")
            transport._proc = proc
            executor.set_transport("test_server", transport)

            result = await executor.execute("echo_tool", {})
            assert result.is_error
            assert result.error_type == "transport"
            assert "mismatch" in result.output.lower()
    finally:
        proc.terminate()
        await proc.wait()


# ── Test: TC-B04 — concurrent SQLite writes ──────────────────────────────────

@pytest.mark.asyncio
async def test_concurrent_memory_store_writes(tmp_path):
    """Two coroutines writing MemoryEntry simultaneously must not corrupt data."""
    import uuid
    from agent.memory.store import MemoryStore
    from agent.memory.types import MemoryEntry, SourceType
    from agent.memory.enums import MemoryType

            session_db_path=str(tmp_path / "session.db"),

        def _make_entry(n: int) -> MemoryEntry:
            now = "2026-01-01T00:00:00"
            return MemoryEntry(
                memory_id=str(uuid.uuid4()),
                memory_type=MemoryType.EPISODIC,
                source_type=SourceType.CONVERSATION,

            asyncio.to_thread(store.add, e) for e in entries

        count = store.count_entries()
        assert count == 20, f"Expected 20 entries, got {count}"

    3. Increment stat_partial_completions
