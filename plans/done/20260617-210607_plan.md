# Plan: Clarify config ownership and loading rules in docs

## Goal

Add one clear config ownership section to shared docs so developers and AI agents can determine at a glance which config files are loaded by `load_all()`, which are loaded separately, and which layer owns each file.

## Scope

**In scope:**
- `docs/06_shared_03_runtime_and_execution.md` — add config ownership section (§2 or new §15)
- `docs/06_shared_01_overview.md` — update §9 Major Constraints table with config loading ownership summary
- `docs/06_shared_04_db_architecture_and_schema.md` — update §3 DbConfig and §9 Constraint list to reference the new ownership section
- `docs/06_shared_00_document-guide.md` — update AI Query Routing table entry for `load_all()` + `common.toml`

**Out of scope:**
- Code changes to `shared/config_loader.py` (req 61 handles that separately)
- Adding new config files or changing config file structure
- Refactoring how modules load config at runtime
- Documentation in non-shared layers (agent/, rag/, mcp/)

## Assumptions

1. The config loading behavior will not change during this doc update cycle (req 61 is a separate task).
2. The plan requires documenting the current state accurately, regardless of whether `common.toml` is eventually added to `load_all()`.
3. A single "Config Ownership" section can serve all four docs via cross-reference rather than duplicating content.

## Implementation Steps

1. **Draft the Config Ownership section content** in `06_shared_03_runtime_and_execution.md`
   - Create a table with columns: File | Loaded By | Owned By Layer | Notes
   - Rows for all 12 config files (11 from `_BASE_CONFIG_FILES` + `common.toml`)
   - For each file, state clearly: loaded by `load_all()` (yes/no), loaded separately (which modules), owning layer

2. **Insert the section into `06_shared_03_runtime_and_execution.md`**
   - Place as new §2a between existing §2 (ConfigLoader) and §3 (Logger), or append after §14 as §15
   - Add cross-reference at the top of §2 (ConfigLoader) pointing to the ownership section

3. **Update `06_shared_01_overview.md`** §9 Major Constraints table
   - Replace current `common.toml` row (`NOT included in load_all() — loaded separately`) with a cross-reference: "See [06_shared_03](06_shared_03_runtime_and_execution.md) §Config Ownership for full ownership table"
   - Add a line referencing the new Config Ownership section as the canonical source

4. **Update `06_shared_04_db_architecture_and_schema.md`**
   - In §3 DbConfig: change "common.toml is loaded separately (NOT via ConfigLoader.load_all())" to reference the ownership section
   - In §9 Constraint list: update `common.toml` loading row to reference ownership section instead of duplicating details

5. **Update `06_shared_00_document-guide.md`** AI Query Routing table
   - Update row: "Does `load_all()` include `common.toml`?" from "**No** — `06_shared_03` §2 + `06_shared_90` CONFIG-01" to point to the new ownership section

6. **Review all config-related sections for consistency**
   - Scan all four docs for any remaining statements about config loading that might conflict with the new ownership section
   - Update or remove redundant/conflicting statements (e.g., `06_shared_03` §12 "Execution Flow Summary" config loading flow diagram)

7. **Validate no stale references remain**
   - Run grep for `load_all()` and `common.toml` across all four docs to confirm consistent messaging
   - Ensure CONFIG-01/02/03 entries in `06_shared_90` are updated or marked as "Documentation gap — covered in new ownership section"
