# Plan: Improve /reload feedback for non-reloadable settings

## Goal

Make `/reload` output clearly distinguish between settings that took effect immediately vs those requiring restart. Reduce user confusion about which changes are active.

## Scope

**In scope:**
- Add visual indicators to /reload output (e.g., `[OK]`, `[RESTART]`, `[SKIPPED]`)
- Reorder sections to put "Restart required" first (most important for users to notice)
- Add explicit warning message when restart-required items exist
- Improve section headers with counts (e.g., "Restart required (2):")
- Consider adding a summary line at the top indicating if restart is needed

**Out of scope:**
- Changing which settings are reloadable vs restart-required (behavioral change, out of scope)
- Adding automatic restart prompts or commands
- Adding config diff preview before reload
- Changing the underlying ConfigReloadService classification logic

## Assumptions

1. Current `/reload` output already has three sections: Applied, Skipped, Restart required.
2. `ConfigReloadOutcome` has three lists: `applied`, `needs_restart`, `skipped`.
3. Users primarily need to know: "Did my change take effect, or do I need to restart?"
4. The existing plain-text format is functional but lacks visual hierarchy.

## Unknowns

## Implementation Steps

1. **Add section headers with counts**
   - Change "Applied (runtime):" → "Applied (runtime): [N items]"
   - Change "Skipped:" → "Skipped: [N items]"
   - Change "Restart required:" → "Restart required: [N items]"
   - Completion criterion: each section shows item count

2. **Add visual indicators per item**
   - Applied items: `[OK] - {item}`
   - Skipped items: `[SKIP] - {item}`
   - Restart-required items: `[RESTART] - {item}`
   - Completion criterion: each line has a clear status tag

3. **Reorder sections for priority**
   - New order:
     1. "Restart required:" (most important — users must act)
     2. "Applied (runtime):" (informational — changes active)
     3. "Skipped:" (informational — why some items weren't applied)
   - Completion criterion: restart-required section appears first

4. **Add warning banner when restart is needed**
   - If `needs_restart` is non-empty, add a line before the sections:
     `"WARNING: Some settings require restart to take effect."`
   - Use `write_warning()` or similar if available, otherwise plain text with clear wording
   - Completion criterion: explicit warning shown when restart is needed

5. **Improve top-level summary**
   - Change "Config reloaded (all base config files)" to include status:
     - If only applied: `"Config reloaded — all changes applied"`
     - If restart needed: `"Config reloaded — some changes require restart"`
     - If no changes: keep "No changes detected."
   - Completion criterion: header reflects actual outcome

6. **Update tests**
   - Update assertion strings in `test_config_commands.py` for new output format
   - Add test case verifying warning banner appears when restart is needed
