# Plan: Fix `deploy/init_db.sh` to initialize the current Event Bus schema

## Goal
`deploy/init_db.sh` が `scripts/eventbus/schema.sql` を正規ソースとして参照し、インラインDDLを持たない状態で Event Bus DB を正しく初期化できるようにする。

## Scope
**In-Scope**:
- `deploy/init_db.sh` のインラインDDL削除と `schema.sql` 参照への置き換え
- `deploy/init_db.sh` のシェル変数定義追加（`EVENTBUS_SCHEMA` パス変数）
- `deploy/init_db.sh` の `workflow.sqlite` テーブル確認コマンドに `approvals` 期待値を明示

**Out-of-Scope**:
- `scripts/eventbus/schema.sql` の変更（正規ソースとして現状維持）
- `scripts/eventbus/db.py` の変更（`_migrate()` の挙動は変更しない）
- Agent 統合・publisher/subscriber 動作の変更
- `docs/02_deployment.md` の更新（説明は現状で正確）

## Assumptions
- `deploy/deploy.sh` が `scripts/` を `rsync` で `/opt/llm/scripts/` へ同期するため、`init_db.sh` 実行時点で `/opt/llm/scripts/eventbus/schema.sql` が存在する（`deploy.sh` が先に実行されていることが前提）
- `sqlite3` コマンドは `.read <file>` サブコマンドで外部 SQL ファイルを読み込める
- Event Bus DB の初回作成は `init_db.sh` が担い、以降の実行時マイグレーションは `db.py:_migrate()` が担う（役割分担は変更しない）
- `scripts/eventbus/schema.sql` の内容は現時点で最新（`delivery_failure_count`, `dlq_requeue_count`, `idx_events_dlq_at`, `idx_events_dlq_seq` を含む）
