## Goal

Add a standalone CLI command that validates a workflow-definition JSON file (parseable, required fields, required stages, no duplicate stage IDs, valid retry policy) without starting any agent/MCP/LLM service, and call it from `deploy/deploy.sh` so deployment aborts on validation failure.

## Scope

**In-Scope:**
- New module `scripts/agent/workflow/validate.py` exposing a CLI: `python -m agent.workflow.validate <path-to-json>`
- Wire the CLI call into `deploy/deploy.sh`, aborting the whole script on non-zero exit
- Reuse the existing validation logic already implemented in `scripts/agent/workflow/workflow_loader.py` (`WorkflowLoader.load()` / `_validate()`) — do not duplicate the validation rules

**Out-of-Scope:**
- Rewriting or relaxing any existing validation rule in `_validate()` — all required checks (parseable JSON, required top-level fields `name`/`version`/`stages`/`retry_policy`, required stages `plan`/`execute`/`verify`, duplicate stage IDs, required stage fields, `max_attempts >= 1`, `backoff` in `{fixed, exponential}`, `backoff_sec >= 0`) are already implemented and confirmed correct by direct code read — this plan only adds a path-based CLI wrapper around them
- Starting the agent, any MCP server, or any LLM service from the validation command (explicitly forbidden by the requirement's Notes)
- Runtime (non-deploy-time) workflow validation — out of scope per the requirement's Background, which explicitly distinguishes deploy-time validation from (already-existing) runtime startup validation

## Assumptions

1. `scripts/agent/workflow/workflow_loader.py`'s `_validate()` function (lines 54-92) already implements every validation rule listed in the requirement's Tasks — confirmed by direct read at planning time. No new validation logic needs to be written; the task is purely to expose it as a path-based CLI.
2. `WorkflowLoader.__init__(workflows_dir: Path | None = None)` combined with `.load(name: str = "default")` builds the target path as `workflows_dir / f"{name}.json"` — an arbitrary file path (e.g. `config/workflows/default.json`) can be validated without modifying `WorkflowLoader`'s public API by calling `WorkflowLoader(workflows_dir=Path(path).parent).load(name=Path(path).stem)`. This avoids any change to the existing, already-tested `workflow_loader.py`.
3. `WorkflowLoader.load()` performs no network I/O, no service startup, and no import of `agent.repl`/`agent.factory`/any MCP/LLM client module (confirmed: its only imports are `orjson`, `pathlib`, `logging`, and `agent.workflow.models`) — it already satisfies the "must not start the agent or any MCP/LLM service" constraint without further isolation work.
4. Running `python -m agent.workflow.validate <path>` requires `PYTHONPATH=scripts` (matching this project's existing convention for all `agent.*`/`shared.*` module invocations, e.g. `PYTHONPATH=scripts uv run lint-imports` in `rules/toolchain.md`) — the deploy.sh call must set this explicitly rather than assume it is already exported in the deploy environment's shell.

## Design

### `scripts/agent/workflow/validate.py`

```python
#!/usr/bin/env python3
"""agent/workflow/validate.py
Standalone CLI to validate a workflow-definition JSON file at deploy time.
Does not start the agent or any MCP/LLM service.

Usage: python -m agent.workflow.validate <path-to-workflow.json>
"""
from __future__ import annotations

import argparse
import sys
from pathlib import Path

from agent.workflow.workflow_loader import WorkflowLoadError, WorkflowLoader


def validate_path(path: Path) -> None:
    """Raise WorkflowLoadError if the workflow file at `path` is invalid."""
    loader = WorkflowLoader(workflows_dir=path.parent)
    loader.load(name=path.stem)


def main() -> int:
    parser = argparse.ArgumentParser(description=__doc__)
    parser.add_argument("path", type=Path, help="Path to a workflow definition JSON file")
    args = parser.parse_args()

    try:
        validate_path(args.path)
    except WorkflowLoadError as exc:
        print(f"[FATAL] Invalid workflow definition {args.path}: {exc}", file=sys.stderr)
        return 1

    print(f"OK: {args.path} is a valid workflow definition")
    return 0


if __name__ == "__main__":
    sys.exit(main())
```

This deliberately delegates all validation logic to the existing, already-tested `WorkflowLoader`/`_validate()` — the new module contributes only argument parsing, path-to-name translation, and exit-code/message formatting.

### `deploy/deploy.sh` integration

Immediately after Requirement 1's pre-copy existence check (which will already be in place from the prior plan in this session), add:
```bash
if ! PYTHONPATH="${REPO_ROOT}/scripts" uv run python -m agent.workflow.validate \
     "${REPO_ROOT}/config/workflows/default.json"; then
  echo "[FATAL] Workflow definition failed validation; aborting deployment." >&2
  exit 1
fi
```
