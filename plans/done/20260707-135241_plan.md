# Plan: H-9 — Implement LifecycleState tracking in ServerLifecycleRouter

## Goal

Add a `_states: dict[str, LifecycleState]` map to `_ServerLifecycleRouter` and update `start_http_subprocess()`, `restart()`, and `shutdown_all()` to transition states (`STARTING` → `RUNNING`/`FAILED`, `STOPPED`), so that `get_transport_state()` returns meaningful values instead of always `UNKNOWN`.

## Scope

**In**: `scripts/agent/factory.py` — `_ServerLifecycleRouter.__init__()`, `start_http_subprocess()`, `restart()`, `shutdown_all()`, `get_transport_state()`. Import `LifecycleState` from `agent.lifecycle`.

**Out**: No changes to `lifecycle.py` (`assert_valid_transition()` is optional; use if convenient). No changes to `http_lifecycle.py`, `context.py`, config, or DB schema.

## Assumptions

1. `LifecycleState` and `assert_valid_transition()` are already in `scripts/agent/lifecycle.py` (confirmed).
2. State tracking is at the `_ServerLifecycleRouter` level, not inside `HttpServerLifecycleManager`, because the router controls the full lifecycle API.
3. `ensure_ready()` does not set state directly — it delegates to `_http_mgr.start()` when needed (H-4), and that triggers `start_http_subprocess()` or an inline start. If `ensure_ready()` calls `_http_mgr.start()` directly, add state tracking there too.
4. `assert_valid_transition()` is used as a guard in `_set_state()` helper for correctness but should not raise in production — wrap with `try/except` and log on invalid transition.

## Implementation Steps

1. **Import `LifecycleState` and `assert_valid_transition` in `factory.py`**
   - Add `from agent.lifecycle import LifecycleState, assert_valid_transition` to imports.
   - Criterion: import resolves; `mypy` passes.

2. **Add `_states` to `__init__()` and implement `_set_state()` helper**
   - Add `self._states: dict[str, LifecycleState] = {}` to `_ServerLifecycleRouter.__init__()`.
   - Add private helper:
     ```python
     def _set_state(self, server_key: str, new_state: LifecycleState) -> None:
         current = self._states.get(server_key, LifecycleState.UNKNOWN)
         try:
             assert_valid_transition(current, new_state)
         except ValueError:
             _logger.warning(
                 "Lifecycle: invalid state transition %r -> %r for %r",
                 current, new_state, server_key,
             )
         self._states[server_key] = new_state
     ```
   - Criterion: helper validates transitions without raising in production.

3. **Update `start_http_subprocess()` to track state**
   - Before `await self._http_mgr.start(...)`: `self._set_state(server_key, LifecycleState.STARTING)`.
   - After success: `self._set_state(server_key, LifecycleState.RUNNING)`.
   - On exception: `self._set_state(server_key, LifecycleState.FAILED)`; re-raise.
   - Criterion: state is `RUNNING` after a successful start.

4. **Update `restart()` to track state**
   - Before `await self._http_mgr.restart(...)`: `self._set_state(server_key, LifecycleState.STARTING)`.
   - After success: `self._set_state(server_key, LifecycleState.RUNNING)`.
   - On exception: `self._set_state(server_key, LifecycleState.FAILED)`; re-raise.
   - Criterion: state is `RUNNING` after a successful restart.

5. **Update `shutdown_all()` to track state**
   - After `await self._http_mgr.shutdown_all()`: set each known server to `STOPPED`.
   - If exception: set each to `FAILED` for known servers; re-raise.
   - Criterion: state is `STOPPED` for all servers after successful shutdown.

6. **Implement `get_transport_state()` using `_states`**
   - Return `self._states.get(server_key, LifecycleState.UNKNOWN)`.
   - Criterion: returns `UNKNOWN` for unknown keys; meaningful values for known keys.

7. **Add tests**
   - Test: after `start_http_subprocess()`, `get_transport_state()` returns `RUNNING`.
   - Test: `start_http_subprocess()` raises → `get_transport_state()` returns `FAILED`.
   - Test: after `restart()`, state is `RUNNING`.
   - Test: after `shutdown_all()`, state is `STOPPED`.
   - Test: unknown key returns `UNKNOWN`.
   - Criterion: `uv run pytest tests/test_agent_factory.py -v` passes.
