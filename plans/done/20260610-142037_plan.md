## Assumptions

1. `validate_server_name()` の戻り値 `str | None` を廃止し、無効時は `ValueError` を投げる。呼び出し元 `installer_writer.py` は現在 `if err := validate_server_name(...)` で受けており、例外伝播に変更する。
2. `dispatch.py` の `except Exception` を廃止すると、`server.py` の `call_tool()` が handler 例外を受け取る。`server.py` 側で分類する。
3. `installer_port.py` の `except Exception: pass` は TOML パース失敗を握りつぶしている。`tomllib.TOMLDecodeError` と `OSError` を個別に扱い、その他は伝播させる。
4. `audit.py` の `Any` を `logging.Logger` に変更する（duck-type 互換は維持）。
5. `installer_writer.py` の原子的書き込みは `tempfile + rename` 戦略を採用する。ただし今回は検証レイヤ例外伝播と `chmod` 判定修正を優先し、完全な原子的書き込みは今後の issue へ。

## Design

### D-1. `validate_server_name` — 例外送出

```python
def validate_server_name(server_name: str) -> str:
    """Return normalised name; raise ValueError on invalid input."""
    if not server_name:
        raise ValueError("Server name must not be empty.")
    if not _NAME_RE.match(server_name):
        raise ValueError(
            f"Invalid server name {server_name!r}. "
            "Use lowercase letters, digits, and hyphens; must start with a letter."
        )
    return server_name
```

### D-2. `installer_port.py` — except 分類

```python
try:
    with agent_toml.open("rb") as f:
        cfg = tomllib.load(f)
except OSError as e:
    raise OSError(f"Cannot read {agent_toml}: {e}") from e
except tomllib.TOMLDecodeError as e:
    raise ValueError(f"Invalid TOML in {agent_toml}: {e}") from e
```

### D-3. `dispatch.py` — except Exception 廃止

`dispatch_tool()` の `try/except Exception` を削除。handler が送出する例外はそのまま伝播させる。`_handle_tool_exception()` も削除。呼び出し元 `server.py` の `call_tool()` / `call_tool_stdio()` が例外を受けて HTTP 応答 / JSON エラーレスポンスに変換する。

### D-4. `audit.py` — Any 廃止

```python
import logging

def _audit_log(
    server_logger: logging.Logger,
    ...
) -> None:
```

### D-5. `installer_templates.py` — 不正 role 即例外

```python
_ROLE_CONFIGS = {"shell": ..., "sqlite": ..., "rag": ...}

def generate_config_toml_for_role(role: str, ...) -> str:
    if role not in _ROLE_CONFIGS:
        raise ValueError(f"Unknown role {role!r}. Must be one of {sorted(_ROLE_CONFIGS)}")
    ...
```
