## Design

### When to use stdio
- Single-user local tools where concurrency is not a concern
- Development and testing of new MCP servers before HTTP mode is ready
- Tools where process isolation model aligns with per-call subprocess

### When not to use stdio
- Multi-user deployments (per-instance serialization blocks concurrent requests)
- High-frequency tool calls (HTTP amortizes connection overhead)
- Production agent deployments (current default: all production servers are HTTP)
