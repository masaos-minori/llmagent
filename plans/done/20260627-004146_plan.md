## Goal

Clean up deprecated Event Bus configuration fields (`poll_interval_ms` and `offset_checkpoint_interval`) to reduce operator confusion by either removing them or retaining with explicit warnings.

## Scope

**In-Scope**:
- Decide whether deprecated fields should be removed from EventBusConfig or retained with explicit warning on load
- If retained, document them as no-op compatibility fields
- Emit a warning when deprecated fields are set in TOML config
- Add config validation tests for deprecated field handling
- Update reference API docs

**Out-of-Scope**:
- Reintroducing polling subscribe delivery
- Reintroducing automatic offset checkpointing

## Assumptions

1. `poll_interval_ms` is deprecated because subscribe delivery switched from polling to EventBroker push (push-mode)
2. `offset_checkpoint_interval` is deprecated because offset checkpointing was replaced with ack-only model
3. Removing these fields would break existing TOML configs that set them — need compatibility consideration

## Implementation Steps

1. **Phase 1: Decide removal vs retention strategy**
   - Option A (remove): Remove both fields from EventBusConfig and TOML config loading
   - Option B (retain with warning): Keep fields but emit warnings when set, document as no-op
   - Recommendation: Option B (retain with warning) for backward compatibility

2. **Phase 2: Add deprecation warnings**
   - In config.py `__post_init__`: add warnings.warn() when poll_interval_ms or offset_checkpoint_interval are set to non-default values
   - Update comments on both fields to clarify they are no-op

3. **Phase 3: Update TOML config example**
   - Remove `poll_interval_ms = 500` from config/eventbus.toml (line 7) — deprecated field no longer needed
   - Add comment if retention is chosen

4. **Phase 4: Update reference API docs**
   - In `06_eventbus_06_reference_api.md`: add deprecation notes for both fields
   - Document that setting these fields has no runtime effect

5. **Phase 5: Add config validation tests**
   - Test: EventBusConfig loads without deprecated fields (no error)
   - Test: EventBusConfig emits warning when deprecated fields are set to non-default values
   - Test: EventBusConfig accepts default values without warning
