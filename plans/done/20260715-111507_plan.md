# Work Plan: Unify Command Metadata as Single Source of Truth

## Goal

Eliminate duplicate slash command lists by making `_COMMANDS` the single source of truth for dispatch, `/help`, and tab completion. Define `/exit` as an explicit REPL-reserved command.

## Scope

- **In scope:**
  - `agent/repl.py`: Replace `SLASH_COMMANDS` with dynamic generation from `_COMMANDS` + REPL-reserved commands
  - `agent/commands/registry.py`: Review `_COMMANDS` usage for completeness
  - `agent/commands/command_defs.py`: Review existing command definitions
  - `agent/commands/command_defs_list.py`: Review existing command list
- **Out of scope:**
  - `agent/commands/cmd_memory.py` and `memory_*` modules
  - `agent/config_*` files
  - `agent/tool_policy.py`
  - `agent/tool_approval.py`
  - Documentation files
  - Test files (except command metadata tests if required)
  - `/memory` subcommand behavior changes

## Assumptions

1. `_COMMANDS` in `CommandRegistry` contains all built-in MCP tool commands but may not include REPL-reserved commands like `/exit`.
2. `AgentREPL.SLASH_COMMANDS` is a hardcoded list that only includes some commands (missing some from `_COMMANDS`).
3. `/exit` is handled specially in `repl.py` before command dispatch â€” it should remain outside `CommandRegistry.dispatch()`.
4. Tab completion needs to include both `_COMMANDS` commands and REPL-reserved commands.
5. `/help` output should include `/exit` as a REPL control command.

## Unknowns

All unknowns resolved through source inspection:

1. **SLASH_COMMANDS vs _COMMANDS gap**: `AgentREPL.SLASH_COMMANDS` contains 19 commands but is missing 6 from `_COMMANDS`: `/plan`, `/memory`, `/audit`, `/plugin`, `/skill`, `/mdq`. Conversely, `/exit` is in SLASH_COMMANDS but not in _COMMANDS.
2. **/exit interaction**: `/exit` is handled directly in `repl.py:_should_exit()` before `CommandRegistry.dispatch()` is called. No overlap or conflict exists.
3. **Other REPL-reserved commands**: Only `/exit` is REPL-reserved. All other commands are managed by `CommandRegistry`.
4. **Commands in _COMMANDS**: `_COMMANDS` includes all 23 built-in commands (`/help`, `/config`, `/stats`, `/context`, `/plan`, `/undo`, `/reload`, `/compact`, `/mcp`, `/session`, `/clear`, `/export`, `/history`, `/system`, `/db`, `/set`, `/memory`, `/debug`, `/audit`, `/approve`, `/reject`, `/plugin`, `/skill`, `/mdq`).

## Design

### 1. Define REPL-reserved commands

Add a new constant in `repl.py`:

```python
_REPL_RESERVED_COMMANDS = frozenset(["/exit"])
```

### 2. Create helper functions

Add helper functions in `repl.py` (module-level):

```python
def builtin_command_names() -> set[str]:
    """Return names of all built-in commands from _COMMANDS."""
    from agent.commands.command_defs_list import _COMMANDS
    return frozenset(cmd.name for cmd in _COMMANDS)

def reserved_repl_command_names() -> frozenset[str]:
    """Return names of REPL-reserved commands."""
    return _REPL_RESERVED_COMMANDS

def completion_command_names() -> frozenset[str]:
    """Return all commands available for tab completion."""
    return builtin_command_names() | reserved_repl_command_names()
```

Use lazy import inside `builtin_command_names()` to avoid circular imports at module level. Return `frozenset` instead of `set` for immutability.

### 3. Replace SLASH_COMMANDS

Replace the hardcoded `SLASH_COMMANDS` class attribute with a cached property:

```python
from functools import cached_property

@cached_property
def SLASH_COMMANDS(self) -> frozenset[str]:
    """Tab completion candidates derived from _COMMANDS + REPL-reserved commands."""
    return completion_command_names()
```

Using `cached_property` avoids regenerating the set on every access while keeping it dynamic enough for potential future changes.

### 4. Update CLIView initialization

Update `CLIView.__init__` call in `AgentREPL.__init__` â€” no change needed since `SLASH_COMMANDS` is accessed as an attribute.

### 5. Update /help output

The existing `_cmd_help` method in `registry.py` iterates over `_COMMANDS` and displays each command's name and help text. This already works correctly. Add a note about `/exit` being a REPL control command (not a dispatchable command).

### 6. Keep /exit outside dispatch

Maintain the current behavior where `/exit` is handled directly in `repl.py:_should_exit()` before `CommandRegistry.dispatch()` is called. No change needed.

## Risks

1. **Missing commands in _COMMANDS**: If `_COMMANDS` doesn't contain all expected commands, tab completion will be incomplete. Mitigation: Compare `_COMMANDS` against known command list after implementation. Currently confirmed: `_COMMANDS` has 23 commands, `SLASH_COMMANDS` has only 19 (missing `/plan`, `/memory`, `/audit`, `/plugin`, `/skill`, `/mdq`).
2. **Circular import**: Adding helper functions to `repl.py` that reference `_COMMANDS` may cause circular imports. Mitigation: Use lazy import inside `builtin_command_names()` to defer the import until function call time.
3. **Performance impact**: Generating completion candidates on every call may have minor overhead. Mitigation: Use `functools.cached_property` to cache the result per instance.
4. **Test breakage**: Tests that depend on `SLASH_COMMANDS` being a static list may fail. Mitigation: Update tests accordingly. Check for test files referencing `SLASH_COMMANDS`.
5. **Type mismatch**: `SLASH_COMMANDS` was previously a `list[str]`, now becomes `frozenset[str]`. The `CLIView` constructor may expect a specific type. Mitigation: Verify `CLIView.__init__` signature accepts `Iterable[str]` or update accordingly.
