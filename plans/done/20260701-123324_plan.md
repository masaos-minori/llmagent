## Goal
`restore_session()` でシステムメッセージが重複する問題と `_memory_injected` メッセージのリーク問題を修正する。DB から取得したメッセージのシステムメッセージを排除し、現在の正規システムプロンプト (`ctx.conv.system_prompt_content`) のみを先頭に配置する。

## Scope
**In-Scope**:
- `scripts/agent/services/session_restore.py` — `restore_session()` の history 再構築ロジック

**Out-of-Scope**:
- `fetch_messages()` / `SessionMessageRepository` の変更
- `_setup_prompt()` の変更

## Assumptions
1. `ctx.conv.system_prompt_content` が正規システムプロンプトの単一の真実の源
2. DB からのシステムメッセージは `ctx.conv.system_prompt_content` で上書きすれば十分
3. セッションリロード後に `_sync_system_prompt()` が次のターン開始時に呼ばれる

## Design
```python
# Before
system_msgs = [m for m in ctx.conv.history if m["role"] == "system"]
ctx.conv.history = system_msgs + messages

# After
system_msgs = (
    [{"role": "system", "content": ctx.conv.system_prompt_content}]
    if ctx.conv.system_prompt_content
    else []
)
non_system_messages = [m for m in messages if m["role"] != "system"]
ctx.conv.history = system_msgs + non_system_messages
```

DB からのシステムメッセージを除外し、現在の正規プロンプトのみを先頭に配置する。
`_memory_injected` 付きのインメモリシステムメッセージも使わないため、リーク問題も解消。
