# Plan: [Medium][eventbus][io] JSONL・オフセット・DLQ ファイル書き込みのイベントループオフロード

Source: `requires/20260625_18_require.md`

## Goal

`_append_jsonl()`、`write_offset()`、`_atomic_write()` の同期ファイル I/O をイベントループから `asyncio.to_thread()` 経由でオフロードし、SSE レイテンシへの影響を除去する。

## Scope

**In-Scope**
- `scripts/eventbus/app.py` の `_append_jsonl()` オフロード
- `scripts/eventbus/offsets.py` の `write_offset()` / `read_offset()` オフロード
- `scripts/eventbus/dlq.py` の `promote_to_dlq()` → `_atomic_write()` パスのオフロード
- ファイル I/O オフロードのリグレッションテスト追加

**Out-of-Scope**
- SQLite 操作のオフロード (req #14 の対象)
- DB ファサードの導入 (req #15 の対象)
- 接続モデルの変更 (req #16 の対象)

## Assumptions

1. `_append_jsonl()` は `publish()` からのみ呼ばれる (確認済み: app.py L119)
2. `write_offset()` は `subscribe()` 内の SSE ジェネレータから呼ばれる (確認済み: app.py L197, L211)
3. `read_offset()` は `subscribe()` 開始時に1回呼ばれる (確認済み: app.py L161) — 起動時なのでオフロード優先度は低いが対応する
4. `promote_to_dlq()` は `_dlq_loop()` (非同期タスク) から呼ばれる (確認済み: app.py L62)

## Unknowns

## Affected Areas

| File | Change | Blast Radius |
|---|---|---|
| `scripts/eventbus/app.py` | `_append_jsonl()` を async 化 + `asyncio.to_thread` ラップ | 低 (publish のみ) |
| `scripts/eventbus/offsets.py` | `read_offset()` / `write_offset()` を同期のまま維持し、呼び出し側でオフロード | 低 |
| `scripts/eventbus/dlq.py` | `_atomic_write()` を同期のまま維持し、`promote_to_dlq()` 呼び出し側でオフロード | 低 |

## Design

**パターン A**: ファイル操作関数を同期のまま維持し、非同期の呼び出し側で `asyncio.to_thread()` を使う。

```python
# app.py: _append_jsonl をオフロード
async def _append_jsonl_async(body: dict, seq: int) -> None:
    await asyncio.to_thread(_append_jsonl, body, seq)

# app.py: write_offset をオフロード (subscribe 内)
await asyncio.to_thread(write_offset, _cfg.offsets_dir, consumer_id, current_seq)

# app.py: promote_to_dlq をオフロード (_dlq_loop 内)
await asyncio.to_thread(promote_to_dlq, _db, _cfg.deadletter_dir, _cfg.max_retry)
```

`offsets.py` と `dlq.py` のコードは変更しない (ピュア同期関数のまま保持)。

## Implementation Steps

1. `app.py` の `_append_jsonl()` 呼び出しを `asyncio.to_thread` でラップ
2. `subscribe()` 内の `write_offset()` 呼び出しを `asyncio.to_thread` でラップ
3. `subscribe()` 内の `read_offset()` 呼び出しを `asyncio.to_thread` でラップ
4. `_dlq_loop()` の `promote_to_dlq()` 呼び出しを `asyncio.to_thread` でラップ (req #14 と重複する場合は統合)
5. ファイル I/O がイベントループスレッドで実行されないことをテストで確認
6. テスト全通過確認

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `subscribe()` の SSE ジェネレータ内での `asyncio.to_thread` が yield と干渉する | Low | `asyncio.to_thread` は await 可能なので yield の前後で await するだけで問題なし |
| `os.fsync()` のオフロードで durability とパフォーマンスのトレードオフが変わる | Low | fsync はオフロード内で実行するため durability は維持。パフォーマンス改善が主目的 |
