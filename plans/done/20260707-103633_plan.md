## Goal

Add a startup preflight check that validates the workflow SQLite schema (`tasks`, `attempts`, `processed_events`, `artifacts`, `approvals` tables and their required columns) and aborts agent startup with a clear error message if any required structure is missing.

## Scope

**In**:
- Implement `check_workflow_schema()` function that:
  - Opens `workflow.sqlite` (or raises if the file cannot be opened)
  - Checks for existence of 5 required tables: `tasks`, `attempts`, `processed_events`, `artifacts`, `approvals`
  - Checks required columns exist in each table
  - Raises `RuntimeError` with a specific message identifying the missing table or column
  - Includes remediation text: "Run `create_workflow_schema()` or initialize the workflow database"
- Call `check_workflow_schema()` from `AgentStartup._check_workflow_definition()` (or a new `_check_workflow_schema()` startup step) before `Orchestrator` construction
- Place implementation in `scripts/agent/repl_health.py` (alongside `check_workflow_definition()`) or a new `scripts/agent/workflow/schema_check.py`
- Add to `deploy/deploy.sh` cp list if new module is created
- Add tests for each failure case: missing DB, missing table, missing column
- Add test for valid schema: no error

**Out**:
- Schema migration or backfill
- Partial compatibility with old workflow DBs
- Auto-creation of missing schema at startup (only check; report remediation)
- Changes to `StateStore` CRUD operations

## Assumptions

1. `workflow.sqlite` path is determined by `SQLiteHelper(target="workflow")` — the same path used by `StateStore`.
2. The 5 required tables are: `tasks`, `attempts`, `processed_events`, `artifacts`, `approvals`.
3. `create_workflow_schema()` already exists or can be referenced in the remediation message.
4. The preflight check runs before `Orchestrator.__init__()` and before `WorkflowLoader().load()`.
5. `AgentStartup._check_workflow_definition()` is the appropriate insertion point in `startup.py:125`.

2. **Phase 2: Implement check_workflow_schema()**
   - [ ] Place in `scripts/agent/repl_health.py` or `scripts/agent/workflow/schema_check.py`
   - [ ] Implementation outline:
     ```python
     REQUIRED_TABLES = {
         "tasks": ["task_id", "session_id", "workflow_id", "status", ...],
         "attempts": ["attempt_id", "task_id", "stage_id", "status", ...],
         "processed_events": [...],
         "artifacts": [...],
         "approvals": ["approval_id", "task_id", "status", ...],
     }

     def check_workflow_schema(db_path: Path | None = None) -> None:
         db = SQLiteHelper(target="workflow")
         db.open(write_mode=False, row_factory=False)
         try:
             tables = {row[0] for row in db.fetchall("SELECT name FROM sqlite_master WHERE type='table'")}
             for table, cols in REQUIRED_TABLES.items():
                 if table not in tables:
                     raise RuntimeError(
                         f"Workflow schema missing table {table!r}. "
                         "Run create_workflow_schema() or reinitialize the workflow database."
                     )
                 existing_cols = {row[1] for row in db.fetchall(f"PRAGMA table_info({table})")}
                 for col in cols:
                     if col not in existing_cols:
                         raise RuntimeError(
                             f"Workflow schema missing column {table}.{col}. "
                             "Run create_workflow_schema() or reinitialize the workflow database."
                         )
         finally:
             db.close()
     ```
