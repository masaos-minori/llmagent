## Goal
Add a configurable limit on memory snippets injected during session startup to prevent token explosion.

## Scope
**In**: `scripts/agent/startup.py`, `scripts/agent/config_dataclasses.py` — add snippet limit logic and config parameter
**Out**: Changes to memory retrieval logic itself, or changes to other system prompt injection mechanisms

## Assumptions
1. A simple list slice is sufficient — no need for complex token counting
2. Default of 10 snippets is reasonable for most use cases

## Unknowns
1. ~~What is a reasonable default limit?~~ Resolved: 10 snippets is a reasonable default based on typical LLM context window sizes.

## Affected areas
- Session initialization
- System prompt generation
- Memory retrieval

## Design
Add a configuration parameter `memory.max_retrieved_memories` with a default of 10. Slice the snippets list before joining to enforce the limit. Log a warning when truncation occurs.

## Implementation steps
1. Add `max_retrieved_memories: int | None = None` to ConfigDataclass
2. Replace bare float timeouts with `httpx.Timeout(timeout=MCPSERVER_HEALTH_TIMEOUT)` in mcp_status.py
3. Run tests to verify no regressions
