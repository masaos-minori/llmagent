## Goal
Make MCP tool discovery failures fatal under PRODUCTION security profile, preventing the agent from starting silently crippled when all tools will fail.

## Scope
**In-Scope**:
- Modify `scripts/agent/startup.py:_check_services()` step 4 exception handler to check security profile and add FATAL vs WARNING accordingly
- Add test coverage for discovery failure under both PRODUCTION and LOCAL profiles

**Out-of-Scope**:
- Changes to MCP server implementations or configurations
- Adding new health gates beyond what's needed for this fix
- Changes to any other startup checks beyond the discovery failure handling

## Assumptions
1. Option A (changing the exception handler in `_check_services()`) is preferred over Options B/C because it mirrors existing patterns in `startup.py`
2. Under LOCAL profile, WARNING level is sufficient — the agent can still start but the warning should be unmistakable
3. No existing tests directly cover the discovery failure path in `_check_services()`
