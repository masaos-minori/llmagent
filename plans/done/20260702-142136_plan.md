(`remote_empty`, 4xx, 5xx, JSON parse error) and verify that all five
diagnostic fields (`result_source`, `http_result_kind`, `remote_status_code`,
`remote_latency_ms`, `fallback_reason`) are correctly populated and asserted
in the existing test files.

   `fallback_reason` are already populated by `_run_http_augment()` in
   `pipeline.py:505-523`. No source changes needed.

   for `remote_empty` (only `remote_nonempty` path returns silently).
   Documentation is consistent with code. No change needed.
