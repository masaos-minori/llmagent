# Plan: Make production config templates safe-by-default

Source: `requires/20260625_11_require.md`

---

## Goal

production 向け MCP config テンプレートのデフォルト値を危険な値から安全な値に変更し、post-load audit に依存せず safe-by-default の状態を実現する。

---

## Scope

**In-Scope**
- `config/*.toml`: 全 production 向け MCP config テンプレートのレビュー
- workflow-capable サーバー（cicd, github）: 空 allowlist をデフォルトとしない
- shell: `sandbox_backend = "firejail"` をデフォルト化
- local dev vs production defaults の分離をコメントで明示
- docs 更新

**Out-of-Scope**
- runtime validation ロジックの変更（10 番で対応）
- config loader の変更

---

## Assumptions

1. `config/` ディレクトリに各 MCP サーバーの toml ファイルが存在する。
2. production config テンプレートと local dev config が同一ファイル or 分離している。
3. safe default への変更はユーザーの意図的な上書きなしには影響しない（空リスト → deny all なので既存ユーザーは影響を受ける可能性）。

## Implementation Steps

### Phase 1: Audit
- [ ] 1-1. `config/*.toml` の全ファイルの現行デフォルト値を列挙
- [ ] 1-2. 危険なデフォルト（空 allowlist、sandbox=none）を特定

### Phase 2: 変更
- [ ] 2-1. 各 toml ファイルの危険なデフォルト値を安全値に変更
- [ ] 2-2. local dev 向け例外値をコメントで明記（`# local dev only: sandbox_backend = "none"`）

### Phase 3: ドキュメント
- [ ] 3-1. `docs/02_deployment.md` に template policy を記述
