## Design

追加するセクション構成:

```markdown
## Runtime Metrics

### Recommended metrics

| Metric | How to collect | Alert threshold |
|---|---|---|
| publish P95 latency | POST /publish 応答時間をクライアント側で計測 | > 200ms |
| publish P99 latency | 同上 | > 500ms |
| SSE delivery lag | 最後に受信した event の published_at と現在時刻の差 | > 2s |
| DLQ loop duration | /health の dlq_last_run_age_s (req #19 実装後) | > 120s |
| Threadpool saturation (proxy) | CPU 使用率が高い状態での publish レイテンシ上昇 | publish P95 > 500ms かつ CPU > 80% |
| Event backlog | /health の event_count の増加率 | 1分あたり N 件以上の未配信積み上がり |

### Distinguishing saturation from DB unavailability

| Symptom | Saturation | DB unavailable |
|---|---|---|
| /health db | ok | unavailable |
| /health status | degraded (latency) | degraded (db unavail.) |
| publish latency | 増加 | タイムアウト/エラー |
| SSE delivery | 遅延するが継続 | 停止 |

### Alerting guidance

- **Critical**: /health status = unavailable → PagerDuty等で即時通知
- **Warning**: publish P95 > 200ms → 5分以上継続した場合に通知
- **Warning**: SSE delivery lag > 2s → 2分以上継続した場合に通知
- **Info**: DLQ loop duration > 120s → ログ確認を推奨
```
