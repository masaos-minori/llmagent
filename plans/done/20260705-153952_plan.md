## Goal

Codify `ToolExecutor` execution order in tests so that any future reordering of plugin check, cache, route resolve, health pre-flight, lifecycle, semaphore, transport, and health update is caught immediately.

## Scope

**In**: New test file `tests/test_tool_executor_order.py` with mocked components verifying execution order for all 9 cases in `ToolExecutor.execute()` / `_raw_execute()`.

**Out**: Rewriting ToolExecutor. Changing plugin public API. Documentation-only updates.

## Assumptions

1. `ToolExecutor.execute()` at line 336: `plugin_invoker.try_execute()` → if hit, return immediately (bypasses cache and MCP routing).
2. `_execute_with_cache()` at line 276: cache hit → return without calling `_raw_execute()`.
3. `_raw_execute()` at line 243: `_resolver.resolve()` → `_check_health()` → `lifecycle.ensure_ready()` → `transport.call()` → `_record_success()` / `_record_transport_error()`.
4. `_check_health()` at line 209: checks `health_registry.is_unavailable()` — BEFORE `ensure_ready()`.
5. Concurrency semaphore is acquired inside `_execute_with_semaphore()` — AFTER health and lifecycle.
6. Cache hits do NOT call `record_success()` on `HealthRegistry` (confirmed: `_execute_with_cache()` returns early without calling `_raw_execute()`).
7. Transport errors → `error_type="transport"`. Tool errors → `error_type="tool"`.

## Design

**Verified execution order from code:**
1. `plugin_invoker.try_execute()` — if returns non-None → done (bypass cache, MCP routing, lifecycle)
2. `_execute_with_cache()`:
   - cache key check → if hit → return (bypass `_raw_execute()`, no health update)
   - stampede protection check
3. `_raw_execute()`:
   - `_resolver.resolve()` → ValueError if unknown tool
   - `_check_health()` → error result if UNAVAILABLE
   - `lifecycle.ensure_ready()` → may raise
   - `_transports.get()` → error if missing
   - `_ensure_semaphores()` + `sem.acquire()`
   - `transport.call()` → result
   - `_record_success()` or `_record_transport_error()`

**Test structure using call-order recording:**
```python
call_order = []

mock_plugin.try_execute.side_effect = lambda name, args: (
    call_order.append("plugin") or None  # return None = miss
)
mock_cache.get.side_effect = lambda key: (call_order.append("cache") or None)
mock_resolver.resolve.side_effect = lambda name: (call_order.append("resolve") or "server1")
mock_health.is_unavailable.side_effect = lambda key: (call_order.append("health") or False)
mock_lifecycle.ensure_ready.side_effect = lambda key: call_order.append("lifecycle")
mock_transport.call.side_effect = lambda name, args: (call_order.append("transport") or result)
```
