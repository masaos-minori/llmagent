# Work Plan: `scripts/shared/` リファクタリング — DTO統一・except Exception撲滅・型境界厳格化

## Goal

`scripts/shared/` 配下の全 Python ファイルから `except Exception`・`dict[str, Any]` 境界外使用・無条件 `str()` 変換・後方互換コード・`tuple[str, bool, str]` 戻り値を排除し、typed DTO・具体的例外型・fail-fast バリデーションに統一する。

- `scripts/shared/llm_client.py` — `call()`/`stream()` 戻り値 → `LLMResponse` DTO; `_emit_usage()` → `_parse_usage()` typed; `except Exception` → 具体型; `_extract_message()` → `_parse_response()` に統合

1. `LLMClient.call()` と `stream()` の戻り値型を変更しても、呼び出し元 (`agent/llm_turn_runner.py` 等) は同時に更新する。

## Design

### `ToolCallResult` DTO

```python
@dataclass(frozen=True)
class ToolCallResult:
    output: str
    is_error: bool
    request_id: str   # x-request-id from HTTP; "" for stdio/plugin/cache
    server_key: str   # "" for plugin tools
```

`HttpTransport.call()`, `StdioTransport.call()`, `ToolExecutor.execute()` / `_raw_execute()` / `_execute_with_cache()` のすべてが `ToolCallResult` を返す。

### `TransportErrorInfo` DTO

```python
@dataclass(frozen=True)
class TransportErrorInfo:
    summary: str
    detail: str  # JSON string for audit log
```

`format_transport_error()` の戻り値を `dict[str, str]` から変更する。

### `LLMResponse` / `LLMUsage` DTO (`llm_types.py`)

```python
@dataclass(frozen=True)
class LLMUsage:
    prompt_tokens: int
    completion_tokens: int

@dataclass(frozen=True)
class LLMResponse:
    message: LLMMessage
    finish_reason: str | None
    usage: LLMUsage | None
```

### `TracerProtocol`

```python
class TracerProtocol(Protocol):
    def start_as_current_span(self, name: str, **kwargs: Any) -> Any: ...
```

`build_tracer() -> TracerProtocol` に変更し、呼び出し元の型が通る。
