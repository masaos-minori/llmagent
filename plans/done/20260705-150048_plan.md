## Goal

Make `/memory rebuild` default to dry-run mode, require explicit `--confirm` flag for actual rebuild, and display a pre-rebuild summary with warnings before proceeding.

## Scope

**In**: Change default behavior of `rebuild()` in `memory_rebuild_ops.py` (commands layer). Add pre-rebuild summary display (JSONL count, SQLite count, warning). Require `--confirm` for actual rebuild. Run consistency check automatically after rebuild. Add tests.

**Out**: Implementing full event-sourced memory state, changing memory retrieval logic, documentation-only updates.

7. This is a behavior change; operators expecting immediate rebuild will now need `--confirm`.

## Design

**New `rebuild()` behavior**:
```python
def rebuild(self, mem: MemoryServices, args: list[str]) -> None:
    dry_run = "--confirm" not in args  # DEFAULT is now dry-run
    
    # Pre-rebuild summary (always shown)
    jsonl_store = mem.ingestion._jsonl
    jsonl_count = jsonl_store.count_all()
    consistency = mem.store.check_consistency()
    current_sqlite_count = consistency.memories
    
    self._out.write(f"  JSONL archive records:  {jsonl_count}")
    self._out.write(f"  Current SQLite memories: {current_sqlite_count}")
    self._out.write(f"  Expected inserted:       {jsonl_count}")
    self._out.write("  WARNING: delete, pin, and unpin operations are NOT replayed.")
    self._out.write("  Deleted entries may be re-inserted from JSONL.")
    
    if dry_run:
        self._out.write("  [dry-run] No changes made. Add --confirm to proceed.")
        return
    
    # Actual rebuild
    jsonl_count, inserted = import_from_jsonl(jsonl_store, dry_run=False, ...)
    self._out.write_success(f"Imported {inserted} entries from {jsonl_count} records.")
    
    # Auto-run consistency check
    self.check_consistency(mem)
```

**Return value for testability**: Consider having `rebuild()` return a `RebuildResult` dataclass with `jsonl_count`, `sqlite_count`, `inserted`, `dry_run` fields.
