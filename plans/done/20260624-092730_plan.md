---

## Goal

Document the single-connection model in `eventbus/db.py`, add startup log showing the connection strategy, and add error logging that traces DB failures to their calling path.

---

## Scope

**In:**
- `scripts/eventbus/db.py` — add docstring explaining `check_same_thread=False` and single-connection rationale; add error logging on connection failure
- `scripts/eventbus/app.py` — add startup log: "Using single shared SQLite connection (WAL mode, check_same_thread=False)"
- Event Bus documentation — add connection-lifecycle section explaining the current model

**Out:**
- Replacing shared connection with per-request connections
- Replacing SQLite with another DB

---

## Assumptions

1. WAL mode serializes concurrent writers in SQLite; single shared connection with `check_same_thread=False` is safe for FastAPI's single-threaded async event loop.
2. No per-request connection churn is intended.
