## 1. Goal
- Expose index-backed outline data with full heading hierarchy metadata (chunk_id, level, heading_path, start_line, end_line) and correct stale-index detection, replacing the current plain-text output.

## 2. Scope
- **In-Scope**:
  - `service.py`: fix ORDER BY in outline SQL query (ordinal-only); fix stale-detection unit mismatch (mtime_ns vs indexed_at); change return value from plain text to structured JSON-lines or YAML-like metadata block
  - `tools.py`: add `max_depth` parameter to `outline` inputSchema (currently missing)
  - `server.py`: fix `/health` endpoint (still checks legacy `sections` table; must check `chunks` + `documents`)
  - `tests/test_mdq_service.py`: update `test_outline_returns_headings` to assert chunk_id / level / heading_path fields in output
- **Out-of-Scope**:
  - DB schema changes (schema already supports all required fields: heading_level, heading_path, chunk_id, start_line, end_line, ordinal)
  - RAG pipeline or agent layer changes
  - Hybrid search mode
  - New MCP tools

## 3. Assumptions
- `documents.indexed_at` is a Unix timestamp in **seconds** (float), and `documents.mtime_ns` is in **nanoseconds** â€” the comparison must normalize to the same unit before comparing
- Output format should remain a string (MCP tool response is always `str`) but embed structured metadata per heading (e.g., one JSON object per line, or a YAML-like block); LLM clients can parse it
- `max_depth` default (6) and `max_items` default (500) from config remain unchanged
- Files not yet indexed should return a `MdqNotFoundError` (no fallback to direct file parsing)
- The `OutlineResponse` Pydantic model in `models.py` already has the correct shape; only the serialization in `service.py` needs updating

## 8. Risks & Mitigations
- **Risk**: `cmd_mdq.py` parses outline output by splitting on whitespace or indentation â†’ **Mitigation**: Read `cmd_mdq.py` in Phase 1 before changing format; if it parses text, update display logic alongside service change
- **Risk**: stale detection fix (divide mtime_ns by 1e9) may cause floating-point imprecision for very recent files â†’ **Mitigation**: Use integer comparison: `doc_row["mtime_ns"] // 1_000_000_000 > int(doc_row["indexed_at"])` to stay in integer domain
- **Risk**: health endpoint fix might break `test_mdq_health_stale.py` if it expects legacy table names â†’ **Mitigation**: Run existing tests before and after; update test fixtures as needed
- **Risk**: JSON-lines output format breaks any agent prompt that expects indented text outline â†’ **Mitigation**: Keep indented heading text as `"heading_text"` field; add a `"display"` key with the indented string as fallback, or emit both a human-readable block and a machine-readable block
