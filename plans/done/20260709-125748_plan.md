# Plan: Remove MCP reload deferred behavior entirely

Source requirement: `requires/20260709_02_require.md`

## Goal

Remove `ConfigReloadOutcome.deferred` and all `/reload` `[DEFER]`/"Deferred
(next connection)" output handling. This is the follow-up cleanup that
`plans/done/20260709-100635_plan.md` (M-2, from the prior BUG-01 fix)
explicitly deferred: `auth_token`/`startup_mode` already report as
`needs_restart` (not `deferred`) since that fix landed, but the `deferred`
field/rendering code itself was left in place as dead weight.

## Scope

In scope:
- `scripts/agent/services/config_reload.py`: remove the `deferred` field
  from `ConfigReloadOutcome`.
- `scripts/agent/commands/cmd_config.py`: remove all `result.deferred` /
  `[DEFER]` rendering branches in `_cmd_reload()`.
- `docs/05_agent_08_configuration.md`: remove the "Deferred" classification
  bullet, the "Deferred settings" subsection, and the `deferred` row from
  the `ConfigReloadOutcome` fields table.
- `docs/05_agent_07_cli-and-commands.md`: remove the "Deferred" row from
  the Reload classification summary table.
- `tests/test_cmd_config_refactor.py`: delete `TestCmdReloadDeferred`
  (tests the now-removed rendering path).
- `tests/test_config_reload.py`: remove the 4 `result.deferred` assertions
  from `TestMcpServerChangeClassification` (the field no longer exists, so
  these would raise `AttributeError`, not just fail).

Out of scope (per requirement's Non-Goals):
- Removing `/reload` itself.
- LLM/RAG/tool-cache hot-reload behavior.
- Startup-only memory/plugin initialization behavior.
- `auth_token`/`startup_mode` reload classification itself — already
  `needs_restart` since the prior BUG-01 fix; nothing to change there.

## Assumptions

1. `auth_token` and `startup_mode` are **already** classified as
   `needs_restart`, not `deferred` — verified by reading
   `scripts/agent/services/config_reload.py`'s current
   `_classify_mcp_server_changes()` / `_diff_mcp_server_config()` (landed in
   the prior BUG-01 fix, commit `78602a3a`). This requirement's
   "Implementation" section (classify auth/startup as restart-required) is
   therefore already done; only the leftover `deferred` field/rendering
   cleanup remains.
2. `ConfigReloadOutcome.deferred` has zero remaining writers anywhere in
   `scripts/` — verified by `grep -rn "\.deferred\b\|deferred=" scripts/`
   (only reads in `cmd_config.py`, no writes anywhere). Safe to remove the
   field entirely rather than phase it out.
3. `grep -rn "\.deferred\b\|deferred=" scripts/ tests/` found exactly 4
   consumer sites of the field, all already enumerated in Scope:
   `cmd_config.py` (7 lines), `test_cmd_config_refactor.py:161`
   (`TestCmdReloadDeferred`, the only test of the rendering path being
   removed), and `test_config_reload.py` (4 assertion lines across
   `TestMcpServerChangeClassification`).
4. `tests/test_agent_cmd_config.py:477` (`assert "[DEFER]" not in out`)
   requires no change — it asserts *absence*, which remains trivially true
   once the code path is deleted.

## Design

**`config_reload.py`** — remove one line from `ConfigReloadOutcome`:

```python
@dataclass
class ConfigReloadOutcome:
    """Structured report of what changed after a /reload."""

    applied: list[str] = field(default_factory=list)
    needs_restart: list[str] = field(default_factory=list)
    skipped: list[str] = field(default_factory=list)
    """..."""
    source_files: list[str] = field(default_factory=list)
    startup_only: list[str] = field(default_factory=list)
```
(delete the `deferred: list[str] = field(default_factory=list)` line)

**`cmd_config.py`** — rewrite `_cmd_reload()`'s status/output block:

```python
            if not result.applied and not result.needs_restart:
                if result.startup_only:
                    self._out.write(
                        "Config reloaded — startup-only settings cannot apply without restart"
                    )
                else:
                    self._out.write("No changes detected.")
            elif result.needs_restart:
                self._out.write("Config reloaded — some changes require restart")
            else:
                self._out.write("Config reloaded — all changes applied")
            if result.needs_restart:
                self._out.write(
                    "WARNING: Some settings require restart to take effect."
                )
                count = len(result.needs_restart)
                self._out.write(f"Restart required: [{count} items]")
                for item in result.needs_restart:
                    self._out.write(f"  [RESTART] - {item}")
            if result.applied:
                count = len(result.applied)
                self._out.write(f"Applied (runtime): [{count} items]")
                for item in result.applied:
                    self._out.write(f"  [OK] - {item}")
            if result.skipped:
                count = len(result.skipped)
                self._out.write(f"Skipped: [{count} items]")
                for item in result.skipped:
                    self._out.write(f"  [SKIP] - {item}")
            if result.startup_only:
                count = len(result.startup_only)
                self._out.write(f"Startup-only (ignored): [{count} items]")
                for item in result.startup_only:
                    self._out.write(f"  [STARTUP-ONLY] - {item}")
            logger.info(
                "Config reloaded: applied=%s, needs_restart=%s",
                result.applied,
                result.needs_restart,
            )
```
(i.e.: drop `not result.deferred` from the first `if`; drop the
`elif result.deferred: ...` branch entirely; drop the
`if result.deferred: ... [DEFER] ...` block entirely; drop `deferred=%s` /
`result.deferred` from the trailing `logger.info` call.)

## Risks

- **Risk**: Any external tooling or dashboard that parses `/reload`'s
  structured log line (`"Config reloaded: applied=%s, needs_restart=%s,
  deferred=%s"`) by field position or name would break when `deferred=%s`
  is removed from the format string.
  → **Mitigation**: This is a `logger.info()` call, not a machine-readable
  API contract; `grep -rn "Config reloaded: applied"` outside this file
  found no other consumer. Low risk.
- **Risk**: Removing `ConfigReloadOutcome.deferred` is a breaking change to
  the dataclass's public shape; any future plan or doc (e.g. archived plan
  files under `plans/done/`) that still references `.deferred` becomes
  historically inaccurate.
  → **Mitigation**: Acceptable — `plans/done/*.md` are historical records
  of what was planned at the time, not living documentation; they are not
  expected to stay in sync with later changes.
