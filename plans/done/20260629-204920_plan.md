## Goal
RAG consistency check の出力に `chunk_id` / URL レベルの修復対象を明示し、`/db consistency` 出力からオペレーターが具体的な修復アクションを判断できるようにする。

## Scope
**In-Scope**:
- `scripts/db/maintenance.py` の `check_rag_consistency()` が `affected_chunk_ids`, `affected_doc_ids`, `affected_orphan_chunk_ids`, `affected_orphan_urls` を既に返していることを確認
- `/db consistency` コマンドの出力フォーマットがこれらの情報を表示しているかを確認
- `docs/03_rag_05_configuration_and_operations.md` の修復ガイダンス表が完全かを確認
- 不足している場合のみ: コマンド出力への情報追加

**Out-of-Scope**:
- 自動修復処理の実装
- consistency check をバックグラウンドジョブに変更
- `check_rag_consistency()` の read-only 性の変更

## Assumptions
1. `maintenance.py` の `ConsistencyReport` は `affected_chunk_ids`, `affected_doc_ids`, `affected_orphan_chunk_ids`, `affected_orphan_urls` をすでに保持（コード確認済み）
2. `maintenance.py` の `describe_consistency_issues()` は既に `/db rag rebuild-fts` ヒントを含む
3. `/db consistency` コマンド (`cmd_db.py`) がこれらの詳細情報を実際に表示するかは確認が必要
