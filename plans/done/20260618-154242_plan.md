# Plan: Define a Better Incremental Update Strategy for `file://` Ingestion

Derived from `01_plan.md` using requirement `requires/20260618_50_require.md`.

---

## Goal

Local-file ingestion has a clear incremental update strategy (mtime or hash-based) that avoids re-embedding unchanged files, and the strategy is documented.

---

## Scope

**In scope:**
- Strategy definition document (not implementation)
- `docs/03_rag_02_ingestion_pipeline.md` — update local-file ingestion section with the defined strategy
- `docs/03_rag_05_configuration_and_operations.md` — document operational workflow for local files

**Out of scope:**
- Code implementation of the strategy (mtime/hash checking, CLI changes, DB schema changes)
- Watch mode / filesystem watcher

---

## Assumptions

1. The strategy can be designed and documented independently of implementation.
2. File mtime is a reasonable freshness signal (sufficient for all use cases; hash for edge cases).
3. The strategy should integrate with the existing `etag`/`last_modified` pattern — extending it rather than creating a parallel mechanism.

---

## Proposed strategy (to document)

| Signal | Mechanism | Skip condition |
|---|---|---|
| **File mtime** | Store `last_modified` in `documents` (already exists, currently `NULL` for files) | If file mtime ≤ stored `last_modified`, skip |
| **Content hash** (SHA-256 of first chunk) | Add to `documents` schema alongside `etag` | If hash matches stored hash, skip |

The crawler should compute both and pass them through the chunk files. The ingester should compare before re-inserting.

   - Criterion: clear spec for future implementation

2. **Document in `docs/03_rag_02_ingestion_pipeline.md`**
   - Add a "Local file ingestion" subsection in the ingestion pipeline doc
   - Describe: `crawl_file()` → chunk → ingest flow
   - Describe: incremental strategy (mtime or hash-based skip)
   - Describe: how it differs from web (etag/conditional GET)
   - Criterion: reader understands the refresh model without reading source

3. **Document in `docs/03_rag_05_configuration_and_operations.md`**
   - Add operation workflow for repeated local-file ingestion
   - Criterion: operator knows how to re-ingest local files efficiently
