# Plan: Normalize configuration schema — remove legacy Memory keys and GitOps security fields

## Summary

Remove deprecated Memory configuration keys (`use_memory_layer`, `memory_jsonl_dir`, `memory_embed_enabled`) and clarify/remove GitOps security fields (`gitops_push_blocked`, `gitops_force_push_blocked`, `gitops_protected_branches`). Add removed-key validation, include original exception messages in `ConfigLoadError`, and add reload preflight validation.

## Affected Files

- `scripts/agent/config_dataclasses.py` — MemoryConfig fields, cross-field validation
- `scripts/agent/config_builders.py` — _build_memory_config(), removed-key validation
- `scripts/agent/services/config_reload.py` — _detect_startup_only(), reload logic
- `scripts/shared/config_errors.py` — ConfigLoadError enhancement

## Decision: GitOps Security Fields

### Removed (reject at load/reload time)
- `gitops_force_push_blocked` — unclear semantics, no clear runtime consumer
- `gitops_protected_branches` — unclear semantics, no clear runtime consumer

### Retained (hot-reloadable)
- `gitops_push_blocked` — explicitly has reload behavior per acceptance criteria

### Classification summary

| Field                      | Status       | Reload Behavior |
|---------------------------|--------------|-----------------|
| use_memory_layer          | removed      | —               |
| memory_jsonl_dir          | removed      | —               |
| memory_embed_enabled      | removed      | —               |
| gitops_force_push_blocked | removed      | —               |
| gitops_protected_branches | removed      | —               |
| gitops_push_blocked       | retained     | hot-reloadable  |

```python
class ConfigLoadError(RuntimeError):
    """Raised when configuration files cannot be loaded."""

    def __init__(self, message: str, cause: Exception | None = None) -> None:
        if cause is not None:
            full_message = f"Config load failed ({type(cause).__name__}: {cause}): {message}"
        else:
            full_message = message
        super().__init__(full_message)
```

Update `load_config()` to pass the original exception:

```python
    try:
        config: dict[str, Any] = ConfigLoader().load_all()
```

## Risks

1. **Other modules reference removed fields**: The requirement explicitly states not to modify `agent/memory/*`, `agent/tool_policy.py`, etc. These modules may still reference the removed fields. Mitigation: The removed-key validation will catch any remaining references at load/reload time. The modules themselves should be addressed in separate issues per the dependency note.

2. **Config file compatibility**: Existing config files with removed keys will fail at load time. This is intentional — the acceptance criteria require rejecting these keys. Users must update their config files.

3. **Cross-file import changes**: Adding `_REMOVED_KEYS` constant to `config_loader.py` and `config_reload.py` requires ensuring no circular imports. Since these are module-level constants used only within their respective modules, no circular import risk exists.

4. **ConfigLoadError signature change**: Changing `ConfigLoadError.__init__` to accept an optional `cause` parameter is backward-compatible since it's keyword-only with a default value.
