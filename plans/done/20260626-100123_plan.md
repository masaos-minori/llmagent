# Plan: Strengthen automatic context restoration for new sessions

Source: `requires/20260625_92_require.md`

---

## Goal

`on_session_start()` と `on_user_prompt()` の役割を明確に分離し、セッション開始時の pinned/high-importance semantic memory 注入と、最初のユーザープロンプト時のタスク特化 retrieval を非重複な責務として再定義する。デフォルト閾値の調整で無関係な injection を減らす。

---

## Scope

**In-Scope**
- `scripts/agent/memory/injection.py`: `InjectionPolicy` のデフォルト値見直し、`on_session_start` に pinned 優先ロジック追加
- `scripts/agent/memory/retriever.py`: `top_semantic` に `pinned_only` オプション（segment: startup injection）
- config defaults: `memory_max_inject_semantic`, `memory_max_inject_episodic`, `memory_min_importance`
- docs: `05_agent_12_memory.md` にセッション開始の restore モデルを説明

**Out-of-Scope**
- history compression 変更
- memory subsystem API 全体の再設計

---

## Assumptions

1. `on_session_start()` は pinned entries + 高 importance semantic に限定するのが自然（タスク非依存のプロジェクトルールを注入）。
2. `on_user_prompt()` はタスク依存の semantic+episodic retrieval（現状のまま）。
3. `InjectionPolicy.min_importance=0.3` → 0.5 に上げて不要な semantic injection を削減。
