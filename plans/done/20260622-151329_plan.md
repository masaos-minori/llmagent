# Plan: resolve() エラーケーステスト追加 — test_tool_executor_routing.py

## Goal

`tests/test_tool_executor_routing.py` の `TestResolverIntegration` に、
`ToolRouteResolver.resolve()` の None・空文字・不正な ID 形式のエラーケーステストを追加する。

## Scope

**In-Scope:**
- `tests/test_tool_executor_routing.py` — `TestResolverIntegration` への 3 テスト追加

**Out-of-Scope:**
- 実装コードの変更
- `ToolRouteResolver` の型チェック強化
- discovery map / registry パスのテスト

## Assumptions

1. `ToolRouteResolver.resolve("")` の実行パス:
   - `_discovery_map.get("")` → None (空文字はキーとして存在しない)
   - `_lookup_registry("")` → None (空文字はツール名として登録されていない)
   - `_config_map.get("")` → None
   - `_fallback_route("")`:
     - `"".startswith("github_")` → False
     - `""` は `_SET_ROUTES` のどのセットにも含まれない
     - `raise ValueError(f"Unknown tool: ''")`
2. `ToolRouteResolver.resolve("call_abc123")` の実行パス (ツールコール ID 形式):
   - ツール名として登録されていないため最終的に `ValueError` が送出される
3. `ToolRouteResolver.resolve(None)` の実行パス:
   - `_fallback_route(None)`:
     - `None.startswith("github_")` → `AttributeError: 'NoneType' object has no attribute 'startswith'`
   - または `_lookup_registry(None)` で `AttributeError`
   - type hint は `str` だが実行時チェックがないため `AttributeError` が期待される

## Design

### 追加テスト一覧

**Test 1: `test_resolver_raises_for_empty_string`**
- `resolver.resolve("")` → `ValueError("Unknown tool: ''")`
- `_make_executor()` の `_resolver` を直接使用

```python
def test_resolver_raises_for_empty_string(self) -> None:
    ex = _make_executor()
    with pytest.raises(ValueError, match="Unknown tool"):
        ex._resolver.resolve("")
```

**Test 2: `test_resolver_raises_for_tool_call_id_format`**
- `resolve("call_abc123")` — ツール名ではなくツールコール ID 形式の文字列
- → `ValueError("Unknown tool: 'call_abc123'")`

```python
def test_resolver_raises_for_tool_call_id_format(self) -> None:
    ex = _make_executor()
    with pytest.raises(ValueError, match="Unknown tool"):
        ex._resolver.resolve("call_abc123")
```

**Test 3: `test_resolver_raises_for_none_input`**
- `resolve(None)` — type hint 違反; `AttributeError` または `TypeError` が発生する
- `# type: ignore[arg-type]` コメントを付与（意図的な型違反テスト）

```python
def test_resolver_raises_for_none_input(self) -> None:
    ex = _make_executor()
    with pytest.raises((AttributeError, TypeError)):
        ex._resolver.resolve(None)  # type: ignore[arg-type]
```
