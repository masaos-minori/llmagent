# Plan: RAG Quality Degradation Visibility (req_33)

## 1. Goal

Distinguish "legitimate zero results" from "degraded retrieval or indexing" by adding
operator-visible counters and warnings for embedding failures, FTS errors, and
embedding-skip during ingestion — without removing graceful degradation.

## 2. Scope

**In-Scope:**
- Add `n_embed_failed: int = 0` to `IngestUrlResult` — tracks chunks whose embedding failed
- Aggregate and log `n_embed_failed` in `ingest_all()` summary line
- Add `SearchDiagnostics` dataclass to `rag/models_result.py` and store it in `PipelineContext`
- Populate `SearchDiagnostics` in `_search_all_queries()`:
  - `embed_ok`: queries that produced an embedding
  - `embed_failed`: queries whose embedding failed (exception or non-list result)
  - `fts_errors`: queries where FTS/vector search threw an exception
- Emit `logger.warning` in `SearchStage.run()` when `embed_failed > 0`
- Add `stat_embed_skip: int = 0` counter to `MemoryIngestionService` — incremented when
  embedding fails in `_persist_entry()` and `on_session_stop()`
- Log `stat_embed_skip` on session stop summary
- Add `logger.info` in `HybridRetriever.search()` when `vec_hits` is empty (FTS-only path)
  — reason: "vec returned empty" vs "embedding=None (disabled)"

**Out-of-Scope:**
- `/memory status` command (covered by req_20 plan)
- Changing any graceful-degradation or fallback semantics
- New DB schema columns

## 3. Confirmed Facts

- `IngestUrlResult` (`ingester.py:38-44`) has `n_success`, `n_failed`, `skipped` but NO `n_embed_failed`
- `_embed_and_store()` (`ingester.py:374`) logs `logger.warning("embedding failed for %s")` but
  returns `False` — counted as `n_failed` (same bucket as parse/DB errors)
- `_search_all_queries()` (`search.py:32-57`): embedding failures are `continue`d silently;
  FTS/vec exceptions are also `continue`d — no counter, no aggregate warning
- `PipelineContext` (`stage.py:25-35`) has `stage_results` but no diagnostics from search
- `HybridRetriever.search()` (`retriever.py:266-281`): falls back to `fts_hits` silently
  when `vec_hits` is empty OR `embedding is None`
- `MemoryIngestionService._persist_entry()` (`ingestion.py:186-190`): embedding failure
  stores entry with `embedding=None` — no counter, no warning

## 4. Assumptions

1. `n_embed_failed` is a separate bucket from `n_failed` — `n_failed` means
   insert/parse failure; `n_embed_failed` means embedding failed but chunk was otherwise valid
2. `SearchDiagnostics` is informational only — does not change search result contents
3. `stat_embed_skip` on `MemoryIngestionService` is cumulative per service instance
   (session-scoped), not persisted to DB
4. Logging `logger.info` for FTS-only fallback path in `HybridRetriever` is sufficient;
   the req_20 plan will surface this in `/memory status` — no duplication

## 7. Implementation Steps

1. **Add `SearchDiagnostics` to `models_result.py`**
   ```python
   @dataclass(frozen=True)
   class SearchDiagnostics:
       embed_ok: int
       embed_failed: int
       fts_errors: int
   ```

2. **Add `search_diagnostics` to `PipelineContext`**
   ```python
   search_diagnostics: SearchDiagnostics | None = None
   ```

3. **Update `_search_all_queries()` in `search.py`**
   - Track `embed_ok`, `embed_failed`, `fts_errors` per query
   - Return `(all_results, SearchDiagnostics(...))` (change return type)
   - `SearchStage.run()` stores diagnostics in `ctx.search_diagnostics`
   - If `embed_failed > 0`: `logger.warning("search degraded: %d/%d queries lacked embedding", ...)`
   - If `fts_errors > 0`: `logger.warning("search degraded: %d FTS/vec errors", ...)`

4. **Update `IngestUrlResult` and `_ingest_chunk_files()`**
   - `IngestUrlResult`: `n_embed_failed: int = 0`
   - Inside `_ingest_chunk_files()` exception handler: detect embed failure from
     `_embed_and_store` returning False due to embed vs other failures
   - Alternative: split embed failure detection in `_embed_and_store` return value
     — use `bool` → no, add `_embed_chunk_only()` returning embed+content separately,
     or: keep `_embed_and_store` returning bool, add separate `_get_embedding_only()` test
   - Simpler: `_embed_and_store` returns `tuple[bool, bool]` → `(success, embed_ok)`
     — but frozen dataclass change is risky. Prefer: add a thread-local counter or
     use a `collections.Counter` shared via `self._embed_stats: Counter`
   - **Decision:** Add `self._embed_stats: dict[str, int]` to `RagIngester`; reset in
     `_ingest_chunk_files()`; increment in `_embed_and_store()` on embed=None path;
     propagate to `IngestUrlResult`

5. **Update `ingest_all()` aggregate log**
   ```python
   total_embed_failed = sum(r.n_embed_failed for r in results)
   logger.info(
       "=== done: %d success, %d failed, %d skipped, %d embed_failed ===",
       total_success, total_failed, total_skipped, total_embed_failed
   )
   ```

6. **Update `MemoryIngestionService`**
   ```python
   self.stat_embed_skip: int = 0

   # in _persist_entry():
   if not embed_result.success:
       self.stat_embed_skip += 1
       logger.info("memory.embed_skip: stored without embedding (reason=%s)", embed_result.error_kind)

   # in on_session_stop() summary:
   logger.info("MemoryIngestionService: %d entries stored; %d embed_skipped", ...)
   ```

7. **Update `HybridRetriever.search()`**
   ```python
   if embedding is None:
       logger.info("retrieval: fts_only (reason=embedding_disabled_or_none)")
       return fts_hits
   # ...
   if not vec_hits:
       logger.info("retrieval: fts_only (reason=vec_returned_empty)")
       return fts_hits
   ```

8. **Tests**
   - `test_search_diagnostics.py`: `_search_all_queries` with embedding mock failure → `embed_failed=1`
   - `test_ingester_embed_failed.py`: `_embed_and_store` failure → `n_embed_failed` increments
   - `test_memory_embed_skip.py`: `_persist_entry` with failed embedding → `stat_embed_skip=1`

9. **Docs**
   - Add "Degradation indicators" section to RAG ingestion docs describing each counter

## 9. Risks

- **Risk:** `_search_all_queries` return type change (`list[list]` → `tuple[list[list], SearchDiagnostics]`)
  breaks callers.
  → **Mitigation:** Only `SearchStage.run()` calls it (private function); update together.
- **Risk:** Thread-safety of `self._embed_stats` in `RagIngester` under `ThreadPoolExecutor`
  → **Mitigation:** Use `threading.Lock` around counter, or use `concurrent.futures` + local
  counting per `_embed_and_store` call collecting results in `futures` dict.
  **Preferred:** keep `_embed_and_store` returning `tuple[bool, bool]` — `(chunk_ok, embed_ok)` —
  then aggregate in `_ingest_chunk_files()`. No shared mutable state.
- **Risk:** Adding `n_embed_failed` to frozen `IngestUrlResult` breaks tests that construct it positionally
  → **Mitigation:** Field has default `= 0`; keyword-only callers are unaffected; positional callers
  get 0. Audit test files for positional construction.
