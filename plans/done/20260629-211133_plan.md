## Goal
Event Bus のデリバリー・ヘルス・オフセット・DLQ のコア不変条件を網羅するオペレーショナル回帰テストセットの完全性を確認し、不足するテストシナリオを追加する。

## Scope
**In-Scope**:
- 以下のシナリオが既存テストでカバーされているかの確認と不足分の追加:
  1. `/health` 200 (ok) / 503 (degraded) — ステータスコードレベルの確認（`test_eventbus_health.py`）
  2. ack-only offset 進行（`test_eventbus_offsets.py`）
  3. 最後の ack 以降からの再接続 replay（`test_eventbus_restart_resume.py`）
  4. crash-before-ack → 再接続 → replay（`test_eventbus_crash_ack.py`）
  5. replay → live push 遷移の race-free（`test_eventbus_subscribe_transition.py`）
  6. slow consumer queue overflow → health 503 （`test_eventbus_slow_consumer.py`）
  7. DLQ promotion（nack ベース）（`test_eventbus_dlq_promotion.py`）
  8. DLQ requeue（`test_eventbus_dlq.py`）
  9. DLQ re-promotion: `delivery_failure_count >= max_retry` のまま requeue → 次 DLQ loop で再 promote（`test_eventbus_requeue_edge_cases.py`）
  10. unsafe public bind guard（`test_eventbus_config.py`, `test_eventbus_startup.py`）
  11. concurrent publish/ack/replay stress（`test_eventbus_concurrent.py`）
- すべてのテストが一時ディレクトリ（DB, JSONL, offsets, deadletter）を使用しており機械依存でないことの確認

**Out-of-Scope**:
- 大規模負荷テスト
- 分散マルチノードテスト
- スループットベンチマーク

## Assumptions
1. 以下のテストファイルが既に存在する（確認済み）:
   - `test_eventbus_health.py`: `test_health_ok`, `test_health_degraded_when_db_unavailable`
   - `test_eventbus_dlq_promotion.py`: nack→promote, requeue→dlq_imminent, DLQ loop re-promotion
   - `test_eventbus_concurrent.py`: concurrent publish/ack/replay/DLQ requeue
   - `test_eventbus_startup.py`, `test_eventbus_config.py`: public bind guard
   - `test_eventbus_slow_consumer.py`: queue overflow, health slow consumer count
   - `test_eventbus_subscribe_transition.py`: replay→live push, dedup
   - `test_eventbus_crash_ack.py`: unacked → replay on reconnect
   - `test_eventbus_restart_resume.py`
2. `test_health_ok` / `test_health_degraded_when_db_unavailable` が HTTP status code (200/503) を検証している可能性が高い
3. DLQ task stopped による health 503 テストが欠落している可能性がある（`test_eventbus_health.py` に 2 テストしか存在しないため）
4. deprecated config フィールド DeprecationWarning のテストが `test_eventbus_config.py` に欠落している可能性がある（_71 の未確認部分）

## Risks
| Risk | Likelihood | Mitigation |
|---|---|---|
| DLQ task stopped を simulate するテストが asyncio Task のモックを必要とし複雑 | Medium | `app.state.dlq_task = None` 直接設定 or fixture でキャンセル済み task を渡す |
| slow consumer threshold テストがタイミングに依存してフラキー | Low | queue depth を直接操作して threshold に達させる（決定論的）|
