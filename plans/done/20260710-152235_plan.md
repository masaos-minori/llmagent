## Goal

Make `config/workflows/default.json` a mandatory deployment artifact: `deploy/deploy.sh` must fail fast (before copying any files) if the source file is missing, and must verify the deployed copy exists after the workflow-files copy step.

## Scope

**In-Scope:**
- `deploy/deploy.sh`: add a pre-copy existence check for `${REPO_ROOT}/config/workflows/default.json` that aborts the whole script with a non-zero exit and a `[FATAL]` message before any `mkdir`/`cp`/`rsync` runs
- `deploy/deploy.sh`: add a post-copy existence check for `${DEPLOY_CONFIG}/workflows/default.json` immediately after the workflow-files copy block, aborting with non-zero exit if missing

**Out-of-Scope:**
- Any workflow-disable, fallback, or optional mode (explicitly forbidden by the requirement's Notes section)
- Validating the *contents* of `default.json` (schema validation) — only existence is in scope, per the requirement's Tasks list
- Changes to `deploy/init_db.sh`, `deploy/setup_services.sh`, or any other deploy script
- Changes to how the agent itself loads/validates `config/workflows/default.json` at runtime (out of scope; this requirement is deploy-time only)

## Assumptions

1. `config/workflows/default.json` already exists in this repository at the expected path (confirmed: `ls config/workflows/` shows `default.json`) — the pre-check is a defense against a future accidental deletion/rename, not a fix for a currently-missing file.
2. `deploy.sh` already runs under `set -euo pipefail` (confirmed, line 10) — any unhandled non-zero exit from a command already aborts the script; the new checks only need to add explicit, clearly-worded failure paths rather than relying on an incidental `cp` failure (whose error message would not match the requirement's specified actionable wording).
3. The existing workflow-files copy block (lines 83-86) uses `cp -r "${REPO_ROOT}/config/workflows/." "${DEPLOY_CONFIG}/workflows/"` — a recursive directory copy, not an itemized per-file copy list — so the post-copy check must independently verify the specific `default.json` path rather than relying on the copy command's own exit status (a directory copy could succeed even if `default.json` itself were absent from the source, though Assumption 1 makes this scenario currently moot).

## Design

Two checks are added to `deploy/deploy.sh`:

**1. Pre-copy check** (placed immediately after `REPO_ROOT` is resolved, before the `echo "=== deploy.sh: デプロイ開始 ==="` banner, so that failure happens before any side effect):
```bash
if [[ ! -f "${REPO_ROOT}/config/workflows/default.json" ]]; then
  echo "[FATAL] Missing required workflow definition: config/workflows/default.json" >&2
  echo "This agent requires a valid workflow definition and does not support workflow-disabled mode." >&2
  exit 1
fi
```

**2. Post-copy check** (placed immediately after the existing workflow-files copy block, lines 83-86):
```bash
if [[ ! -f "${DEPLOY_CONFIG}/workflows/default.json" ]]; then
  echo "[FATAL] Deployed workflow definition missing after copy: ${DEPLOY_CONFIG}/workflows/default.json" >&2
  exit 1
fi
```

Both checks use `[[ ! -f ... ]]` (bash conditional test for "regular file does not exist") rather than relying on `cp`/`rsync`'s own exit code, since the copy command's success does not guarantee the specific file's presence (Assumption 3).

## Implementation steps

1. **Add pre-copy check**
   - [ ] Insert the pre-copy existence check (Design §1) directly after the `REPO_ROOT="$(cd ...)"` line and before the "デプロイ開始" banner echo, so it is the very first substantive action `deploy.sh` takes.
   - [ ] Add a one-line comment above the check noting it validates `default.json` specifically, not the `config/workflows/` directory as a whole.
2. **Add post-copy check**
   - [ ] Insert the post-copy existence check (Design §2) directly after the `cp -r "${REPO_ROOT}/config/workflows/." "${DEPLOY_CONFIG}/workflows/"` line.
   - [ ] Add the same scoping comment above this check.
3. **Manual verification**
   - [ ] Run `bash -n deploy/deploy.sh` (syntax check only, no execution) to confirm the script still parses correctly.
   - [ ] Simulate the missing-source case: temporarily rename `config/workflows/default.json` (or point at a scratch copy of the repo), run `bash deploy/deploy.sh`, confirm it exits non-zero **before** any `mkdir`/`cp` output appears, and prints the exact `[FATAL] Missing required workflow definition: config/workflows/default.json` message.
   - [ ] Restore `default.json` and re-run `bash deploy/deploy.sh` end-to-end (or in a scratch `/opt/llm`-equivalent target directory) to confirm it now succeeds past both new checks.
