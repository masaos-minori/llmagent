# Refactoring: commands/ 共通基盤導入（Parser / Formatter / 例外細分化）— Work Plan

Source instruction: `instructions/20260608_05_instruction.md`

## Goal

commands/ 層に共通サブコマンドパーサーを導入し、`except Exception` を細分化することで、コマンドミックスインの重複コードを削減し型安全性を高める。

## Scope

**In:**
- `cmd_mcp.py`: `McpStatusService` / `McpInstallService` のインスタンス化を constructor injection に変更し、直接インポートを除去
- `cmd_memory.py`: `_as_memory_layer(mem: object)` の型を `ctx.services.memory: MemoryLayer | None` へのアクセスに変更
- `cmd_db.py` / `cmd_config.py`: broad `except Exception` を I/O・DB・設定 例外に細分化
- `commands/shared_parser.py`: サブコマンドパース共通関数（任意、小規模から開始）

**Out:**
- 全コマンドの Formatter/Presenter 層新規作成 — 大規模リファクタリング、別フェーズ
- `utils.py` の render 関数分離 — 別フェーズ
- `cmd_session.py` / `cmd_notes.py` / `cmd_tooling.py` のパース統一 — 別フェーズ
- Status Presenter の新規作成 — 別フェーズ

## Assumptions

1. `McpStatusService` / `McpInstallService` は `cmd_mcp.py` のモジュールトップで import されており、コンストラクタに渡す形に変更可能。
2. `_as_memory_layer(mem: object)` は `ctx.services.memory` が `MemoryLayer | None` として既に型付けされているため削除可能。
3. `cmd_db.py` / `cmd_config.py` の `except Exception` は 9 件あるが、すべてが細分化対象ではない（try/except の内容を確認してから判断）。
4. 既存テスト（`test_agent_cmd_session.py`/`test_agent_cmd_notes.py`/`test_agent_cmd_db.py`: 27 件）がガードとして機能する。

## Design

### cmd_mcp.py — DI 化

```python
# 現在:
svc = McpStatusService(ctx)
# 変更後:
# McpStatusService / McpInstallService はコンストラクタで受け取るか、
# 遅延インスタンス化のままとし、local import を除去するのみ（最小変更）
```

### cmd_memory.py — _as_memory_layer 削除

```python
# 現在:
layer = _as_memory_layer(mem)
# 変更後:
layer = ctx.services.memory  # 型は MemoryLayer | None
```

### cmd_db.py — except 細分化例

```python
# 現在:
except Exception as e:
    print(f"DB error: {e}")
# 変更後:
except sqlite3.OperationalError as e:
    print(f"DB operation error: {e}")
except Exception as e:
    print(f"Unexpected error: {e}")
```

## Implementation steps

1. `cmd_memory.py` を読み、`_as_memory_layer` を削除して `ctx.services.memory` に直接アクセスするよう変更する。
2. `cmd_db.py` と `cmd_config.py` の `except Exception` を確認し、DB/設定/I/O 例外に細分化できる箇所を変更する。
3. `cmd_mcp.py` のモジュールトップのローカル import を整理する（すでに service として定義済みのため最小変更）。
4. `ruff`/`mypy`/`lint-imports`/`pytest` でバリデーション。
