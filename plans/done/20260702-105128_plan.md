## Goal

Align `docs/05_agent_03_turn-processing-flow.md` with the actual runtime behavior of
`ToolLoopGuard`: the current code terminates the tool loop immediately (returning
`TurnResult(action="fail")`), whereas the docs claim it "inject[s] hint" to the LLM.
Resolve the discrepancy by choosing one of two alternatives and implementing the chosen
path.

---

## Scope

**In-Scope**
- Correcting `docs/05_agent_03_turn-processing-flow.md` lines 100-103 (Option A), OR
- Modifying `scripts/agent/tool_loop_guard.py` + `scripts/agent/llm_turn_runner.py` to
  actually inject hints into `ctx.conv.history` (Option B).

**Out-of-Scope**

### Phase 1A — Option A: Fix Documentation (if fail-fast is confirmed correct)

- [ ] Open `docs/05_agent_03_turn-processing-flow.md`.
- [ ] Replace lines 100-103:

  **Before:**
  ```
  - **Dedup:** same `(name, args)` seen > `tool_dedup_max_repeats` times → inject hint
  - **Cycle detection:** same tool sequence in last `tool_cycle_detect_window` rounds → warn
  - **Retry:** errored `(name, args)` called again > `tool_error_retry_max` → block
  - **Consecutive error:** all tools in a round errored `tool_error_max_consecutive` times → break loop
  ```

  **After:**
  ```
  - **Dedup:** same `(name, args)` seen ≥ `tool_dedup_max_repeats` times → terminate loop;
    exit message `"Repeated tool call detected."`; hint stored in `session_diagnostics`
    (`kind='guard_hint'`, `guard_type='dedup'`).
  - **Cycle detection:** same tool-call fingerprint repeated ≥ `tool_cycle_detect_window`
    rounds → terminate loop; exit message `"Cyclic tool call pattern detected."`;
    hint stored in `session_diagnostics` (`kind='guard_hint'`, `guard_type='cycle'`).
  - **Retry:** errored `(name, args)` called again → terminate loop;
    exit message `"Repeated failed tool call detected."`;
    hint stored in `session_diagnostics` (`kind='guard_hint'`, `guard_type='retry'`).
  - **Consecutive error:** all tools in a round errored `tool_error_max_consecutive` times
    → terminate loop; exit message `"Too many consecutive tool errors."`.

  > Guard hints are stored in `session_diagnostics` under `kind='guard_hint'` and are
  > **NOT** added to `ctx.conv.history`. The user sees only the exit message; the hint
  > text is available for offline diagnostics only.
  ```

- [ ] Verify no other occurrences: `grep -rn "inject hint" docs/`
- [ ] Commit: `docs: correct ToolLoopGuard behavior description (fail-fast, not hint injection)`

### Phase 1B — Option B: Implement LLM Hint Injection (if self-correction is confirmed desired)

  function, update assumption U2).
- [ ] In `scripts/agent/tool_loop_guard.py` — update all three guard check methods:

- [ ] Update docs to match new behavior (same section, different wording: "inject hint to

- [ ] Add tests:

  - `_sync_system_prompt()` removes the ephemeral message on next turn start.
  - Guard fires repeatedly → behavior after N injections (if a hard-stop counter is added).
