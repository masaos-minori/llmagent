## Design

### Document Structure

The document contains these sections:
- Purpose: Why this document exists
- Deprecated Configuration Files: Old config file names and their replacements
- Deprecated Concepts: Obsolete architectural concepts
- Deprecated Commands: Removed slash commands
- Deprecated Document References: Links to removed or superseded documents
- How to Refer to Deprecated Items: Guidelines for referencing deprecated content
- Maintenance Rule: Rules for adding/removing items

### Required Fields per Entry

Each deprecated item must include:
- Deprecated Name: The old name/reference
- Current Replacement: What replaced it (or "none" if fully removed)
- Status: Active / Needs confirmation / Verified
- Notes: Context about why it was deprecated
- Evidence: Where the deprecation can be verified

### Initial Candidate Items

Mark each as "Needs confirmation" unless verifiable from current source:
- `config/rag_pipeline.toml` — replacement unknown, needs confirmation
- `common.toml` — replacement unknown, needs confirmation
- workflow optional mode — replacement unknown, needs confirmation
- shared common config — replacement unknown, needs confirmation

### Maintenance Rule

- New deprecations must be added within one week of the change being made.
- Items cannot be removed without documenting what replaced them.
- "Needs confirmation" items must be reviewed quarterly.
