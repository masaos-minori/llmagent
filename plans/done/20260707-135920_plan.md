# Plan: L-1 — Update lifecycle-related comments and docstrings

## Goal

Fix inaccurate comments and docstrings in 4 lifecycle-related source files so that they accurately describe the current runtime behavior: `_ServerLifecycleRouter` as the protocol facade, no ondemand startup mode, `verify_running()` as check-only, and document stderr log redirect and process group shutdown.

## Scope

**In**: 4 source files — documentation changes only. No logic, interface, or behavior changes.
- `scripts/agent/lifecycle_protocol.py` — update `LifecycleManagerProtocol` docstring

**Out**: No logic changes, no test changes, no config or DB changes.

## Assumptions

1. `startup.py` line 101–102 comment: "Ondemand servers are excluded; they start on first tool call via ensure_ready()" — this is partially correct (ensure_ready handles on-demand starts), but "ondemand" is not a supported `startup_mode` value; the correct description is "persistent-mode servers start externally; subprocess-mode servers start via ensure_ready() on first tool call".
2. `verify_running()` docstring says "optionally restart it" — the function only returns a boolean; no restart occurs. Fix to "Returns True if running, False if the process is missing or has exited."
3. `LifecycleManagerProtocol` docstring says "HttpServerLifecycleManager satisfies this protocol structurally" — the actual satisfying implementation at runtime is `_ServerLifecycleRouter` in `factory.py`.
4. H-1 (stderr log redirect) and H-8 (process group shutdown) plans are being implemented; add forward-looking docstrings describing their intent even if not yet merged.
