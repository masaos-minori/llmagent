# Plan: Unify SQLite timestamp defaults to ISO-8601 UTC Z suffix

Source: `requires/20260701_57_require.md`

---

## Goal

`db/schema_sql.py` 内の `datetime('now')` DEFAULT を `strftime('%Y-%m-%dT%H:%M:%SZ', 'now')` に統一し、Event Bus DDL も同じポリシーを適用する。

---

## Scope

**In-Scope**
- `scripts/db/schema_sql.py` の `_RAG_SCHEMA_TEMPLATE` 内 `documents.fetched_at` の DEFAULT を修正
- `scripts/db/schema_sql.py` の `_SESSION_SCHEMA_TEMPLATE` 内以下のカラムを修正:
  - `sessions.created_at`
  - `messages.created_at`
  - `memories.created_at`
  - `memories.updated_at`
- `build_eventbus_schema_sql()` の `published_at` DEFAULT が ISO-8601 UTC Z suffix であることを確認（52番で対応済みの場合は確認のみ）
- `docs/90_shared_04_db_architecture_and_schema.md` の「既存フォーマット差異は保持する」旨の記述を削除・修正
- スキーマ SQL テスト: DEFAULT 式が期待値と一致することをアサートするテストを追加

**Out-of-Scope**
- 既存 DB データの timestamp 値の書き換え（DDL DEFAULT 変更は新規行にのみ適用）
- Event Bus ランタイム timestamp writer の実装
- `_WORKFLOW_SCHEMA` の timestamp（`tasks.created_at` など）: これらは DEFAULT なしの `NOT NULL` カラムであり対象外

---

## Assumptions

1. `tool_results.created_at` と `session_diagnostics.created_at` はすでに `strftime('%Y-%m-%dT%H:%M:%SZ', 'now')` を使用しているため変更不要。
2. DDL の DEFAULT 変更は `IF NOT EXISTS` により既存テーブルには適用されない（マイグレーションは不要）。
3. テスト環境では新規 SQLite メモリ DB を作成してスキーマを適用し、DEFAULT 値を検証する。
