## Goal

req #24–#26 の配信ライフサイクル再設計に対して、コンシューマー失敗・タイムアウト・リトライカウント・インライン DLQ 昇格・リキュー後の再配信をカバーする統合テストを追加する。

## Scope

**In-Scope**
- `tests/test_eventbus_delivery.py` (新規) への E2E テスト追加
- コンシューマー失敗 (nack) → failure count 増加
- 閾値到達 → 即時 DLQ 昇格
- DLQ からのリキュー → failure count リセットなしに再配信
- ack → acked_at がセットされること

**Out-of-Scope**
- 既存テストの変更 (回帰が出た場合は別対応)
- パフォーマンス・負荷テスト (req #17 の対象)

## Assumptions

1. req #24, #25, #26 が実装済みであること
2. `TestClient` を使った同期テストで十分 (非同期不要なフロー)
3. `max_retry = 2` を使ってテストを簡潔に保つ

## Design

テストシナリオ:

1. **test_nack_increments_failure_count**: publish → nack 1回 → GET /events/{id} で failure_count == 1 を確認
2. **test_nack_at_max_retry_promotes_to_dlq**: publish → nack × max_retry → GET /dlq に出現し、live event から消えることを確認
3. **test_ack_sets_acked_at**: publish → ack → acked_at がセットされること
4. **test_requeue_after_dlq**: nack × max_retry → DLQ → requeue → live event に戻ること
5. **test_background_loop_does_not_double_promote**: インライン昇格済みイベントにバックグラウンドループが実行されても重複昇格しないことを確認

## Implementation Steps

1. `tests/test_eventbus_delivery.py` を新規作成 (TestClient fixture は既存テストから流用)
2. 5つのテストシナリオを実装
3. `uv run pytest tests/test_eventbus_delivery.py -v` で通過確認
4. 全テストスイートで回帰がないことを確認
