# Plan: H-8 ‚ÄĒ Stop HTTP MCP subprocesses by process group

## Goal

Start HTTP MCP subprocesses with `start_new_session=True` and update `_terminate_with_timeout()` to send signals to the process group (`os.killpg()`) rather than only the subprocess PID, so that child processes spawned by MCP servers are also terminated.

## Scope

**In**: `scripts/agent/http_lifecycle.py` ‚ÄĒ `start()` (add `start_new_session=True`, capture `pgid`), `_terminate_with_timeout()` (switch to `os.killpg()` with PID fallback).

**Out**: No changes to `factory.py`, `startup.py`, `shutdown_all()` (except via `_terminate_with_timeout()`), config, or any other module.

## Assumptions

1. `os.killpg()` is available on Linux (the target platform per WSL2 deployment env).
2. `os.getpgid(proc.pid)` returns the process group ID immediately after `Popen()` without race conditions for subprocess-mode servers.
3. Fallback to `proc.terminate()` / `proc.kill()` is correct when `os.killpg()` raises `ProcessLookupError` or `OSError` (process group already gone).
4. `start_new_session=True` is compatible with the existing `env` dict injection.

## Unknowns

| Whether `os.getpgid()` can fail if the process exits before the call | Possible race | Wrap `os.getpgid()` in `try/except OSError`; fall back to `proc.pid` as group ID | No |

## Implementation Steps

1. **Add `import os` and `import signal` to `http_lifecycle.py`**
   - Both are stdlib; no new dependencies.
   - Criterion: imports resolve without errors.

2. **Update `start()` to use `start_new_session=True` and store `pgid`**
   - Add `self._http_pgids: dict[str, int] = {}` to `__init__()`.
   - In `start()`, add `start_new_session=True` to `subprocess.Popen()` kwargs.
   - After `proc = subprocess.Popen(...)`:
     ```python
     try:
         self._http_pgids[server_key] = os.getpgid(proc.pid)
     except OSError:
         self._http_pgids[server_key] = proc.pid
     ```
   - In `restart()` and `shutdown_all()`: pop from `_http_pgids` when removing entry.
   - Criterion: `self._http_pgids[server_key]` is populated after successful `start()`.

3. **Update `_terminate_with_timeout()` to use process group signaling**
   - Change signature: `_terminate_with_timeout(self, proc, server_key, timeout=3.0)`.
   - Try `os.killpg(self._http_pgids.get(server_key, proc.pid), signal.SIGTERM)` first.
   - On `(ProcessLookupError, OSError)`: fall back to `proc.terminate()`.
   - After timeout: try `os.killpg(..., signal.SIGKILL)`; fall back to `proc.kill()`.
   - Criterion: process group receives signal when `pgid` is known.

4. **Add tests**
   - Unit-test: mock `os.killpg()` called with SIGTERM on process group.
   - Unit-test: mock `os.killpg()` raising `ProcessLookupError` ‚Üí `proc.terminate()` fallback called.
   - Unit-test: second timeout ‚Üí `os.killpg()` called with SIGKILL.
   - Unit-test: `os.killpg()` not available ‚Üí fallback works.
   - Criterion: `uv run pytest tests/test_lifecycle.py -v` passes.
