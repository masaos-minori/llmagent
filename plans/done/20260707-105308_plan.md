## Goal

Extend `LLMTurnRunner._span_ctx()` to accept and set `workflow_id`, `stage_id`, and `attempt_id` as OTel span attributes alongside the existing `task_id`, `session_id`, and `model_url`, so that every LLM span in a workflow execution contains the complete workflow stage context.

## Scope

**In**:
- Add keyword-only parameters to `_span_ctx()` in `scripts/agent/llm_turn_runner.py`:
  - `workflow_id: str = ""`
  - `stage_id: str = ""`
  - `attempt_id: str = ""`
- Set OTel span attributes (conditionally, same pattern as existing `task_id`):
  - `workflow.workflow_id`
  - `workflow.task_id` (already set when `task_id` non-empty)
  - `workflow.stage_id`
  - `workflow.attempt_id`
  - `workflow.session_id` (already set)
  - `llm.model_url` (already set)
- Update all call sites to `_span_ctx()` within `llm_turn_runner.py` and `orchestrator.py` to pass the new parameters (coordinated with plan 20260707-103631 which adds them to `run()`)
- Ensure the no-op tracer path (`nullcontext(_NoOpSpan())`) works unchanged with the extended signature
- Add tests: span with all 6 attributes; no-op tracer path; missing context rejected before span creation if applicable

**Out**:
- Changing tracing backend or span exporter
- Changing existing `workflow.task_id` or `llm.model_url` attribute names
- Adding span attributes to tool execution spans (covered by req07)
- Migration support

## Assumptions

1. `_span_ctx()` is a private method used only within `LLMTurnRunner`; signature change does not affect external callers.
2. `_NoOpSpan.set_attribute()` is a no-op — no change needed for the no-op path.
3. Attribute names follow the existing convention: `workflow.*` prefix for workflow identifiers, `llm.*` for LLM metadata.
4. The call site in `orchestrator.py:423` (`with self._llm_runner._span_ctx("llm") as llm_span`) passes no keyword args currently; it will need updating after plan 20260707-103631 adds workflow context to `run()`.
5. This plan is Path A (≤3 files, internal method change, no DB/public interface changes).
