files (`04_mcp_90_...`, `90_shared_90_...`) are preserved as empty-but-present per-domain
"known issues" containers — consistent with the sibling domains (`03_rag_90_...`,
`05_agent_90_...`, `06_eventbus_90_...`) which all currently hold real content, confirming
this is a standing structural convention rather than dead scaffolding to remove.

addressed within this plan's scope by widening the grep to the full `docs/*.md` tree (see
Affected Areas evidence) rather than deferring it further; result: none found beyond the two
already fixed.
