# Plan: Improve extraction quality - separate rule/decision/failure/conversation memory

Source: `requires/20260625_91_require.md`

---

## Goal

`extract.py` の extraction ロジックを強化し、`RULE / DECISION / FAILURE / CONVERSATION` の source type 分類を精度高く行う。現在の `_classify_content` は DECISION を "decided" のみで検出しており、FAILURE も keyword 密度のみに依存している。

---

## Scope

**In-Scope**
- `scripts/agent/memory/extract.py`: `_classify_content`, `_SEMANTIC_KEYWORDS`, `_EPISODIC_FAILURE_KEYWORDS`, `_importance_from_content` の改善
- `scripts/agent/memory/enums.py`: `SourceType` の確認（必要なら追加）
- 抽出 tests の追加（`tests/agent/memory/test_extract.py`）

**Out-of-Scope**
- LLM ベースの extraction 導入
- canonical storage model の変更

---

## Assumptions

1. `SourceType.DECISION`, `SourceType.RULE`, `SourceType.FAILURE`, `SourceType.CONVERSATION` はすでに定義済み（enums.py にある）。
2. 抽出改善はルールベース（regex/keyword）のみ。
3. DECISION 検出: "decided", "rationale", "trade-off", "chose", "opted for" などのキーワードに拡張。
4. FAILURE 検出は `_EPISODIC_FAILURE_KEYWORDS` に `workaround`, `root cause`, `mitigation` を追加。

## Implementation Steps

- [ ] 1. `_SEMANTIC_KEYWORDS` を拡張:
  - DECISION 検出用: `\b(rationale|trade.off|chose|opted|decided|decision|tradeoff)\b`
  - RULE 検出用: 既存 keywords に `\b(mandatory|enforce|prohibit|forbidden)\b` を追加
- [ ] 2. `_DECISION_KEYWORDS` を別途定義し、`_classify_content` で DECISION vs RULE を優先順位付きで分類:
  ```
  if decision_hits >= 1 and semantic_hits >= 1: → SEMANTIC + DECISION
  elif semantic_hits >= threshold: → SEMANTIC + RULE
  ```
- [ ] 3. `_EPISODIC_FAILURE_KEYWORDS` を拡張:
  - `\b(root.cause|mitigation|workaround|regression|deadlock|memory.leak)\b` を追加
- [ ] 4. `_importance_from_content` を source_type で分岐:
  - DECISION: semantic より高い importance base = 0.55
  - FAILURE: episodic より高い importance base = 0.45（学習価値が高い）
- [ ] 5. `tests/agent/memory/test_extract.py` に分類テスト追加:
  - RULE: "You must always use uv run pytest" → `SEMANTIC + RULE`
  - DECISION: "We decided to use SQLite because..." → `SEMANTIC + DECISION`
  - FAILURE: "Fixed the bug by resetting..." → `EPISODIC + FAILURE`
  - CONVERSATION: long Q&A without keywords → `EPISODIC + CONVERSATION`
