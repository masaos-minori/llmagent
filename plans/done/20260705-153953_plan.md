## Goal

Create a narrow security-audit-only API for MCP server config access in the agent, enforce it with a static scan, and verify that MCP server processes cannot read other servers' config files.

## Scope

**In**: Create `agent/security_audit_config.py` as the narrow audit API. Add static scan test forbidding direct server config model imports outside the audit module. Verify `ConfigPermissionError` covers cross-server reads.

**Out**: Removing the security audit. Merging agent and server configs. Documentation-only updates.

## Assumptions

1. `audit_security_defaults()` in `repl_health.py` directly imports `mcp.shell.models.ShellConfig`, `mcp.git.models.GitConfig`, `mcp.github.models_config.GitHubConfig`, `mcp.cicd.models.CicdConfig`.
2. These imports are the "agent reading server config" pattern that needs to be isolated.
3. `ConfigPermissionError` was previously implemented — each process may only load its own config file.
4. The narrow API pattern: a single `agent/security_audit_config.py` module imports from server config models; all other agent code imports from this module instead.
5. Static scan must only target security-relevant server config modules: `mcp.shell.models`, `mcp.git.models`, `mcp.github.models_config`, `mcp.cicd.models`. RAG pipeline config (`mcp.rag_pipeline.models`) used in `cmd_rag_export.py` is a different pattern and out of scope for this isolation check.

## Design

**New `agent/security_audit_config.py` (narrow audit API):**
```python
"""agent/security_audit_config.py
Narrow API for security audit access to MCP server config.
This is the ONLY agent module that may import from MCP server config models.
All security audit checks must go through this module.
"""

from dataclasses import dataclass

@dataclass
class ShellAuditConfig:
    sandbox_backend: str
    command_allowlist: list[str]

@dataclass
class GitAuditConfig:
    allowed_repo_paths: list[str]

@dataclass
class GitHubAuditConfig:
    allowed_repos: list[str]
    allow_force_push: bool
    require_pr_review: bool

@dataclass
class CicdAuditConfig:
    workflow_allowlist: list[str]

def load_shell_audit_config() -> ShellAuditConfig | None:
    """Load shell config for audit. Returns None if config is unavailable."""
    try:
        from mcp.shell.models import ShellConfig
        cfg = ShellConfig.load()
        return ShellAuditConfig(
            sandbox_backend=cfg.shell_sandbox_backend,
            command_allowlist=list(cfg.command_allowlist),
        )
    except ImportError:
        return None
    except Exception as exc:
        raise RuntimeError(f"Security audit: failed to load shell config: {exc}") from exc

# Similar load_git_audit_config(), load_github_audit_config(), load_cicd_audit_config()
```

**Updated `repl_health.py`:**
Replace 4 `try: from mcp.*.models import *; ...` blocks with:


**Static scan test:**
```python
_FORBIDDEN_SECURITY_IMPORTS = [
    "from mcp.shell.models",
    "from mcp.git.models",
    "from mcp.github.models_config",
    "from mcp.cicd.models",
]

def test_only_audit_module_imports_security_server_config():
    AGENT_DIR = pathlib.Path("scripts/agent")
    AUDIT_MODULE = "security_audit_config.py"
    violations = []
    for py_file in AGENT_DIR.rglob("*.py"):
        if py_file.name == AUDIT_MODULE:
            continue
        content = py_file.read_text()
        for pattern in _FORBIDDEN_SECURITY_IMPORTS:
            if pattern in content:
                violations.append(f"{py_file}: {pattern}")
    assert not violations, f"Direct security server config imports outside audit module: {violations}"
```
