# Plan: Align `DocumentRow.lang` nullability with the database schema

## Goal
`DocumentRow.lang` の型注釈・実装・テストを `documents.lang NOT NULL` スキーマと完全に一致させ、テストスキーマも `NOT NULL` 制約を反映する。

## Scope
**In-Scope**:
- `scripts/db/store_impl.py`: `doc_get()` / `doc_list()` での `lang` 構築を `str(r[3])` に明示キャスト (NOT NULL前提の型安全化)
- `tests/test_db_store_impl.py`: `_DOCUMENT_SCHEMA` の `lang` カラムに `NOT NULL` 追加、`lang` が `str` であることを確認するテストケース追加

**Out-of-Scope**:
- `scripts/db/models.py`: `DocumentRow.lang` は既に `str` (変更不要)
- DBスキーマ自体の変更 (既に `NOT NULL` 定義済み)
- 他モジュール (`rag/`, `mcp/` 等) への波及変更

## Assumptions
- `documents.lang` は `NOT NULL CHECK(lang IN ('ja', 'en'))` であり、DBから `None` が返ることはない
- `store_impl.py` の `doc_get()` / `doc_list()` は既に `lang=r[3]` でNoneチェックを行っていないが、型安全のため `str(r[3])` キャストを追加する
- `models.py` の `DocumentRow.lang: str` は要件記述時点より既に修正済み — 本計画では変更対象外
