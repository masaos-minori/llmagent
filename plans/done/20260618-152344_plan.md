# Plan: Fix Config Loading and `/reload` Consistency

Derived from `01_plan.md` using requirement `requires/20260618_23_require.md`.

---

## Goal

Config loading is predictable: the same files are loaded at startup and on `/reload`; which changes need a restart is documented and matches behavior.

---

## Scope

**In scope:**
- `scripts/shared/config_loader.py` — `ConfigLoader.load_all()` and `load()` method alignment
- `scripts/agent/config_builders.py` — `build_agent_config()` and `load_config()` flow
- `scripts/agent/services/config_reload.py` — `ConfigReloadService.apply_config_dict()` hot-reload scope
- `scripts/agent/commands/cmd_config.py` — `_cmd_reload()` handler
- `docs/05_agent_08_configuration.md` — document loading model, `/reload` scope, restart-required list

**Out of scope:**
- Individual sub-config builder logic (e.g. `_build_llm_config`, `_build_rag_config`) — only their invocation
- MCP server configs (`shared/mcp_config.py`)
- Config file format changes or migration

---

## Assumptions

1. `ConfigLoader.load_all()` loads the correct set of base files at startup; only `/reload` is wrong.
2. `ConfigReloadService` diffs correctly when given the right full config dict — the only fix needed is feeding it the right input.
3. Some settings are intentionally restart-only (`auth_token`, `transport`, `startup_mode`) and `/reload` must not change them; these are already handled by `needs_restart`.

## Implementation Steps

1. **Read `ConfigReloadService.apply_config_dict()` fully**
   - List every key in `applied` vs `needs_restart`
   - Criterion: complete inventory of hot vs restart-only settings

2. **Fix `_cmd_reload()` to use `load_all()`**
   - Change `ConfigLoader().load("common.toml", "agent.toml")` to `ConfigLoader().load_all()`
   - Or add a `reload_all()` wrapper that loads all base configs
   - Criterion: same config dict shape as startup

3. **Verify sub-config builders handle full dict on reload**
   - Check each `_build_*` function for `None` fallback that might break when all files are present
   - Criterion: no regression on `/reload`

4. **Document config loading model**
   - Write: config file list, loading order, `/reload` scope, restart-required settings table
   - Update `docs/05_agent_08_configuration.md`
   - Criterion: docs clearly answer "does this need restart?"

5. **Lint, type, arch, test validation**
   - `ruff check scripts/` — 0 errors
   - `mypy scripts/` — no new errors
   - Existing config tests pass
