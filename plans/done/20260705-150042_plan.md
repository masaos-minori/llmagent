## Goal

Make `/approve` and `/reject` fail closed when multiple pending approvals exist globally, and add explicit `approval_id` support so operators always resolve the intended approval.

## Scope

**In**: Count pending approvals before resolving; fail if >1 without explicit ID. Add `/approve <approval_id> [reason]` and `/reject <approval_id> [reason]` syntax. Add audit event fields. Add tests for all pending-count cases.

**Out**: Redesigning the workflow engine, implementing multi-user approval queues, documentation-only updates.

## Assumptions

1. `find_latest_pending_approval()` selects `LIMIT 1` globally â€” when multiple pending approvals exist, it silently picks the most recent one.
2. Multiple pending approvals can exist across different sessions (e.g., session A and session B both have `pending_approval` tasks).
3. The "safe" fallback for zero pending approvals already exists (both handlers check `result is None`).
4. Single-pending-approval backward compatibility must be preserved per spec.
5. Audit events should include `approval_id`, `task_id`, `workflow_id`, `session_id`, `decision`, `reason`.

## Design

**New functions in `approval_ops.py`**:
- `count_pending_approvals(db) -> int`: count globally pending approvals
- `find_approval_by_id(db, approval_id) -> ApprovalRecord | None`: fetch by approval_id

**New `/approve` logic**:
```python
count = count_pending_approvals(store._db)
if count == 0:
    # report: no pending approval
    return
if count > 1 and not explicit_approval_id:
    # fail closed: require explicit ID
    self._out.write_validation_error(
        f"Multiple pending approvals ({count}). Use: /approve <approval_id> [reason]"
    )
    return
# resolve the target approval
```

**Parsing**: `/approve` arg is now `[<approval_id>] [reason]`. If arg looks like a UUID, treat as `approval_id`; rest is reason. For backward compat (single pending, no ID), legacy path works unchanged.

**Audit event**: emit to `audit_logger` with `approval_id`, `task_id`, `workflow_id`, `session_id`, `decision`, `reason`.
