# Prompt Injection / Retrieval Trust 対策 — Work Plan

Source instruction: `instructions/20260606_07_instruction.md`

---

## Goal

RAG パイプラインにドキュメントサニタイゼーション・ツール命令分離・信頼スコアリングを導入し、プロンプトインジェクション攻撃を防御する。

---

## Scope

**In:**
- RAG 文書のサニタイゼーション（system-role 擬似命令の除去）
- tool instruction isolation（RAG 結果とシステム指示の境界明示）
- retrieval trust scoring（ソース信頼度によるチャンクフィルタリング）
- citation separation（RAG 結果と直接回答の分離）

**Out:**
- LLM 出力のサニタイゼーション（入力サニタイゼーションのみ対象）
- 外部 URL 許可リストの実装（別タスク）

---

## Assumptions

1. 現在の `rag/pipeline.py` はサニタイゼーションなしで結果をプロンプトに注入している。
2. `rag/utils.py` に `augment()` 相当の処理がある。
3. RAG 文書は `[RAG Context]` ブロックとしてプロンプトに挿入されている。

## Implementation Steps

1. `rag/pipeline.py` と `rag/utils.py` を読み、現在の augment フローを理解する
2. `sanitize_document(text: str) -> str` を実装（`\n---SYSTEM_OVERRIDE---\n` 等のパターンを除去）
3. `DocumentTrustPolicy` を定義し、ソース信頼スコアに基づいてチャンクをフィルタリング
4. RAG 挿入ブロックに `[RAG_START]` / `[RAG_END]` マーカーを追加してシステム指示と分離
5. citation を本文から分離する `format_rag_result()` を実装
6. `python-test-and-fix` スキルでテスト追加
7. バリデーション実行 + instructions/done/ へ移動
