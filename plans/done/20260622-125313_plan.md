## Assumptions

1. `_assert_allowed_repo()` は `in` 演算子による完全一致のみを評価し、
   ワイルドカード展開・正規表現・部分一致は行わない。
2. 空文字列 `""` は `not self._repo_allowlist` (空リスト判定) ではなく
   allowlist の要素として扱われる。allowlist に `""` が含まれる場合に
   `_assert_allowed_repo("")` がパスするのは仕様上の動作。
3. `repo` 引数には `"owner/repo"` 形式を想定しているが、形式検証は
   `_assert_allowed_repo()` では行わない (別の `_parse_repo_slug()` が担当)。

## Design

`_assert_allowed_repo()` のロジック:
```python
if not self._repo_allowlist:          # 空リスト → fail-closed (deny all)
    raise CicdAuthorizationError(...)
if repo not in self._repo_allowlist:  # 完全一致のみ
    raise CicdAuthorizationError(...)
```

追加するテストケース:
1. **空文字列 repo → denied**: `_assert_allowed_repo("")` はリストに `""` がなければ denied
2. **空文字列 repo + allowlist に `""` → passes**: 完全一致するため通過
3. **ワイルドカード `"owner/*"` は展開されない**: allowlist に `"owner/*"` があっても `"owner/repo"` は denied
4. **複数スラッシュ `"owner/repo/extra"` は denied**: allowlist の `"owner/repo"` と完全一致しない
5. **二重スラッシュ `"owner//repo"` は denied**: `"owner/repo"` と完全一致しない

## Implementation Steps

  - `test_extra_slash_suffix_is_denied`: `"owner/repo/extra"` は `"owner/repo"` と完全一致しない
