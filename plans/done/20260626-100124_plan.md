# Plan: Formalize project/repo/branch boundaries for memory retrieval

Source: `requires/20260625_93_require.md`

---

## Goal

`retriever.py` の `top_semantic` と `HybridRetriever.search` に branch-aware フィルタリングを追加し、無関係な project/branch の memory が現在のセッションに注入されないようにする。

---

## Scope

**In-Scope**
- `scripts/agent/memory/retriever.py`:
  - `top_semantic`: `branch` パラメータと optional WHERE clause 追加
  - `HybridRetriever.search`: `branch` パラメータ追加（`_context_boost` を補完する hard filter）
  - `_context_boost` の `branch` 対応
- `scripts/agent/memory/injection.py`: `MemoryInjectionService` に `branch` を渡す
- `scripts/agent/memory/ingestion.py`: `MemoryIngestionService._branch` はすでに存在（line 53）
- tests: unrelated branch memory が注入されないことを確認するテスト
- docs: `05_agent_12_memory.md`

**Out-of-Scope**
- git integration 変更
- memory metadata 構造変更

---

## Assumptions

1. `MemoryEntry.branch` フィールドはすでに存在。
2. branch filtering は soft boost（`_context_boost`）から hard filter（WHERE clause）へ。
3. global rule memory（branch=""）は branch フィルタを透過させる（空文字列は条件を無視）。
4. `branch=""` の場合はフィルタしない（既存動作維持）。

## Implementation Steps

- [ ] 1. `_context_boost(entry, project, repo, branch)` に `branch` 追加:
  ```python
  if branch and entry.branch == branch:
      return _CONTEXT_MATCH_BOOST + 0.05  # branch match > repo match
  if (project and entry.project == project) or (repo and entry.repo == repo):
      return _CONTEXT_MATCH_BOOST
  ```
- [ ] 2. `top_semantic` に `branch: str = ""` と optional WHERE clause 追加:
  ```sql
  WHERE memory_type = 'semantic' AND importance >= ?
  AND (? = '' OR branch = '' OR branch = ?)
  ```
- [ ] 3. `HybridRetriever.__init__` / `search` / `top_semantic` に `branch` パラメータを追加してパイプスルー
- [ ] 4. `MemoryInjectionService.__init__` に `branch: str = ""` を追加
- [ ] 5. tests:
  - branch="feat/x" のセッションで branch="feat/y" の memory が返らないことを確認
  - branch="" の global memory は常に返ることを確認
