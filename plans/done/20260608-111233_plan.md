# Refactoring Phase 1: context.py / config.py 参照規則の強化 — Work Plan

Source instruction: `instructions/20260608_01_instruction.md`

## Goal

`context.py` の `TurnState` を拡張し、`config.py` の `DbConfig` を `db/` 層へ移動することで、状態参照パスを安定化させる。

## Scope

**In:**
- `context.py`: `TurnState` に `background_tasks: set[asyncio.Task]` と `last_error_kind: str | None` を追加
- `config.py`: `DbConfig` を `db/config.py` へ移動し、`config.py` から re-export する（後方互換）
- 関連テストの更新

**Out:**
- `orchestrator.py` / `llm_turn_runner.py` のサービス分割（Phase 2/3 で対応）
- `AgentConfig` のフラットアクセス除去（本セッションで大部分完了済み）
- `repl.py` の変更

## Assumptions

1. `TurnState` に `background_tasks` を追加しても既存コードは`background_tasks`を使わないため動作は変わらない。
2. `DbConfig` は `config.py` 外から `from agent.config import DbConfig` でインポートされている可能性がある。
3. `db/config.py` は現存しない（新規作成）。
4. 本セッションで `config.py` の `__getattr__`/`__setattr__`、`context.py` の backward-compat コメントは除去済み。

## Unknowns

## Design

`TurnState` は `orchestrator.py` が `ctx.turn.current_turn_id` を使用済み。`background_tasks` と `last_error_kind` を追加しても既存動作に影響なし。

`DbConfig` 移動は `db/config.py` に定義を移し、`agent/config.py` で `from db.config import DbConfig` として re-export する。import-linter の `db → agent` 依存は許可されていない（`db` は `agent` に依存できない）ため、逆方向（`agent/config.py` が `db/config.py` を読む）は問題なし。

## Implementation steps

1. `scripts/agent/context.py` の `TurnState` に `background_tasks` と `last_error_kind` フィールドを追加する。
2. `scripts/db/config.py` を新規作成し、`DbConfig` 定義を移動する。
3. `scripts/agent/config.py` から `DbConfig` を削除し、`from db.config import DbConfig` で re-export する。
4. `grep -rn "from agent.config import DbConfig"` で確認し、外部インポートサイトを修正する。
5. `ruff`/`mypy`/`lint-imports`/`pytest` でバリデーション。
