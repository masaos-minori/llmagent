# Plan: H-2 — Inject one shared MCP health registry into ToolExecutor and AppServices

## Goal

Create a single `McpServerHealthRegistry` instance in `build_agent_context()` and inject it into both `ToolExecutor` (via `set_health_registry()`) and `AppServices` (via the `health_registry` constructor parameter) so that `/mcp status` and transport dispatch observe the same health state.

## Scope

**In**: `scripts/agent/factory.py` — instantiate `McpServerHealthRegistry` in `_build_tool_executor()`, inject via `set_health_registry()`, and pass to `AppServices` in `build_agent_context()`.

**Out**: No changes to `context.py` (the `health_registry` field already exists in `AppServices`), `ToolExecutor`, `McpServerHealthRegistry`, or any other module.

## Assumptions

1. `McpServerHealthRegistry` is already importable from `shared.mcp_config` (confirmed by `context.py` import).
2. `ToolExecutor.set_health_registry()` is the correct injection point (confirmed: `scripts/shared/tool_executor.py` line 131).
3. `AppServices.health_registry` parameter already exists (confirmed: `context.py` line 142).
4. No test currently asserts the registry is shared; new tests are the primary behavior lock.

## Implementation Steps

1. **Import `McpServerHealthRegistry` in `factory.py`**
   - Add `from shared.mcp_config import McpServerHealthRegistry` to imports.
   - Criterion: import resolves cleanly (`mypy scripts/`).

2. **Update `_build_tool_executor()` to create and wire the registry**
   - Add `registry = McpServerHealthRegistry()` before constructing `ToolExecutor`.
   - Call `tools.set_health_registry(registry)` after construction.
   - Return `tools, lifecycle, registry` (change return signature).
   - Update callsite in `build_agent_context()` to unpack 3 values.
   - Criterion: `mypy scripts/` passes; registry is set on `ToolExecutor`.

3. **Pass registry to `AppServices` in `build_agent_context()`**
   - Unpack registry from `_build_tool_executor()`.
   - Pass `health_registry=registry` to `AppServices(...)`.
   - Criterion: `ctx.services_required.health_registry is tools._health_registry`.

4. **Add tests**
   - Test: `build_agent_context()` results in `ctx.services.health_registry is not None`.
   - Test: same registry object is injected into `ToolExecutor` and `AppServices`.
   - Test: simulated `_record_transport_error()` on `ToolExecutor` reflects in `ctx.services.health_registry`.
   - Criterion: `uv run pytest tests/test_agent_factory.py -v` passes.
