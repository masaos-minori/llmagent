# Plan: Move RAG DB operations out of AgentSession (req 21)

## Goal

Remove `DocumentRepository` from `AgentSession` so that RAG-layer table access (`rag.sqlite`) is handled exclusively through `DbMaintenanceService`, keeping `AgentSession` responsible only for session/message/note operations in `session.sqlite`.

## Scope

**In scope**:
- `agent/session.py` — remove `_document_repo` field and `list_documents()`/`delete_document()` methods
- `agent/services/db_maintenance_service.py` — add `list_documents()` and `delete_document()` methods
- `agent/services/models.py` — add `DocumentRow` model with `chunking_strategy` field
- `agent/commands/cmd_db.py` — delegate `_db_clean` and `_db_list_urls` to `DbMaintenanceService` instead of `AgentSession`
- `agent/repl.py` — replace direct `SQLiteHelper("rag")` in `_get_chunk_count()` with service call
- `tests/test_document_repo.py` — keep tests for `DocumentRepository` class (it still exists as a standalone module)
- `tests/test_db_maintenance.py` — add tests for new document methods on `DbMaintenanceService`

**Out of scope**:
- Moving other RAG operations (ingestion, embedding, retrieval) — those belong to the RAG pipeline
- Changing `/db stats` — already delegates to `DbMaintenanceService`
- Modifying `db.maintenance` module or DB schema

## Assumptions

1. `DocumentRepository` class stays as a standalone module (`agent/document_repo.py`) for backward compatibility with any external callers; it is simply no longer owned by `AgentSession`.
2. `DbMaintenanceService` is the canonical service layer for all DB operations (both session.sqlite and rag.sqlite).
3. `DocumentRow` model in `services/models.py` needs to include `chunking_strategy` to match what `list_documents()` returns.

## Implementation Steps

1. **Add `DocumentRow` with `chunking_strategy` to `services/models.py`**
   - Update existing `DocumentRow` dataclass to include `chunking_strategy: str = "text"`
   - This matches the output shape of `DocumentRepository.list_documents()`

2. **Add document methods to `DbMaintenanceService`**
   - `list_documents(lang, limit)` — query `rag.sqlite` documents table with chunk count join
   - `delete_document(url)` — delete chunks_vec + document by URL (same logic as DocumentRepository)
   - Use the same SQL patterns as `DocumentRepository` to ensure identical behavior

3. **Update `cmd_db.py` to use `DbMaintenanceService`**
   - `_db_clean`: call `DbMaintenanceService().delete_document(url)` instead of `self._ctx.session.delete_document(url)`
   - `_db_list_urls`: call `DbMaintenanceService().list_documents(lang, limit)` instead of `self._ctx.session.list_documents(lang, limit)`

4. **Remove document methods from `AgentSession`**
   - Remove `_document_repo` field from `__init__`
   - Remove `list_documents()` and `delete_document()` methods
   - Keep all other methods (session, message, note operations) unchanged

5. **Update `repl.py._get_chunk_count()`**
   - Replace direct `SQLiteHelper("rag")` call with `DbMaintenanceService().stats().chunks`
   - `stats()` already queries both DBs and returns a `DbStats` object with `chunks` field

6. **Add tests for new `DbMaintenanceService` document methods**
   - Test `list_documents()` returns correct shape with chunking_strategy
   - Test `delete_document()` returns True/False correctly
   - Test error handling (not found, DB errors)

7. **Update documentation**
   - Update `docs/05_agent_09_data-layer.md` to reflect new ownership boundaries
