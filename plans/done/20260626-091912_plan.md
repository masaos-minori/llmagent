# Plan: Enforce plugin tool contract validation at registration time

Source: `requires/20260625_05_require.md`

---

## Goal

plugin tool の signature（return type `tuple[str, bool]`）と shadow/conflict ルールを `@register_tool` デコレータ呼び出し時点で検証し、無効な plugin tool を startup 時に明示的にエラーとして拒否する。

---

## Scope

**In-Scope**
- `shared/plugin_registry.py`: `register_tool()` での return type チェックを warning → error に昇格、起動時に `_validate_tool_conflicts()` を fail-fast 化
- plugin 拒否時の startup ログ（明示的なエラーメッセージ）
- plugin extension ドキュメントの更新

**Out-of-Scope**
- plugin の引数型検証（args は dict[str, Any] で自由）
- plugin の実行時エラーハンドリング変更

---

## Assumptions

1. `plugin_registry.py:register_tool()` line 130 で return type の不一致は warn のみで登録は続行される。
2. `_validate_tool_conflicts()` は既に存在し、startup 時に呼ばれているが結果は warning のみ。
3. strict_mode フラグが `_validate_tool_conflicts()` に存在するが、デフォルト False。
