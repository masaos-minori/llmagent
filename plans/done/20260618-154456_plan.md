# Plan: Clarify `ArtifactEvent` Is Data-Only and Prepare Future Event-Bus Extension Points

Derived from `01_plan.md` using requirement `requires/20260618_52_require.md`.

---

## Goal

`ArtifactEvent` is clearly understood as a data-only payload. Future event-bus extension points are documented at a high level without committing to an implementation.

---

## Scope

**In scope:**
- `scripts/shared/events.py` — update docstring to add future event-envelope fields (optional)
- `docs/06_shared_02_types_and_protocols.md` — expand ArtifactEvent section with future-envelope design note
- `docs/06_shared_90_inconsistencies_and_known_issues.md` — update UNIMPL-01 with design direction

**Out of scope:**
- Event bus implementation
- Wiring ArtifactEvent producers/consumers

---

## Assumptions

1. Current docstrings already say "data only" — the gap is the absence of future extension design.
2. Future event envelope fields should be documented as aspirational only.

---

## Proposed future envelope (design note only)

```
event_id: str        # UUID v7
source: str          # module name (e.g. "mcp/github")
timestamp: str       # ISO-8601 UTC
correlation_id: str  # trace ID linking related events
```

These are aspirational — not implemented, not required.
