## Goal

Guarantee that routing drift is always detected at startup and that strict mode treats any drift as a fatal startup error, with structured validation results and safety tier coverage.

## Scope

**In**: Wire `validate_all_routing()` into `startup.py`'s `_check_services()` so strict mode raises. Add safety tier missing check in `tool_routing_validation.py`. Add CI tests.

**Out**: Changing routing authority away from `ToolRegistry`. Using live `/v1/tools` for routing. Documentation-only updates.

## Assumptions

1. `check_routing_drift()` in `startup.py:167` always runs but is treated as `[non-fatal]` regardless of strict mode.
2. `check_routing_drift_vs_live()` uses `tool_definitions_strict` for strict mode, not a dedicated routing-strict flag.
3. `validate_all_routing()` in `tool_routing_validation.py` combines config + live drift but is not wired in `startup.py`.
4. Safety tier validation exists in `config_dataclasses.py` but no startup drift check verifies that every registered tool has a tier in `tool_safety_tiers`.
5. `repl_health.py` contains all the relevant functions; `startup.py` calls them.
6. `production_mode` in `startup.py` controls auth enforcement; `tool_definitions_strict` controls tool definition strictness.

**`check_tool_safety_tiers()` in `tool_routing_validation.py`:**
```python
def check_tool_safety_tiers(
    registry: ToolRegistry | None = None,
    tool_safety_tiers: dict[str, str] | None = None,
) -> list[str]:
    """Return list of tool names registered but without a safety tier."""
    if registry is None:
        registry = get_registry()
    if not tool_safety_tiers:
        return []
    missing = [t for t in registry.get_all_tool_names() if t not in tool_safety_tiers]
    return [f"Tool {t!r} has no safety tier defined in tool_safety_tiers" for t in sorted(missing)]
```
