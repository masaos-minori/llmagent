## Goal

Document the complete field mapping from `ChunkJsonRaw` (produced by `ChunkSplitter`) through `pipeline_utils.py` and `ingester.py` to `rag.sqlite` tables (`documents`, `chunks`), and explicitly state the persistence status of every field including `artifact_type` and `fetched_at`.

## Scope

**In**: 
- Add a `ChunkJsonRaw` field → DB column mapping table to `docs/90_shared_04_db_architecture_and_schema.md`
- Add `chunk_type` and `source_file` columns to the `chunks` DDL in `scripts/db/schema_sql.py` (currently missing despite being used in `ingester.py` INSERT)
- Explicitly document `artifact_type` as not persisted (validation-only)
- Explicitly document `fetched_at` DB-side behavior (DEFAULT, not supplied by INSERT)

**Out**: 
- Changing ingestion behavior (unless the `schema_sql.py` DDL mismatch counts as a bug fix — see Unknowns)
- Changes to `ChunkSplitter.py`
- Any changes to query pipeline or embedding logic

## Assumptions

1. `chunk_type` and `source_file` columns exist in the production `rag.sqlite` schema (added by a migration not reflected in `schema_sql.py`), otherwise `ingester.py` would fail at runtime.
2. `fetched_at` in `documents` is set by the DEFAULT expression at INSERT time; the input `fetched_at` field from `ChunkJsonRaw` is read by `ingester.py` but passed only to `ETagManager.update()` (skip path), not to the INSERT statement.
3. `artifact_type` is consumed by `_validate_artifact()` and not written to any DB table.
4. The doc target is `docs/90_shared_04_db_architecture_and_schema.md`.
