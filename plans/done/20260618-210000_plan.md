# Plan: Improve truncation metadata to show actual visible bytes

## Goal

Include the actual visible byte count in truncation metadata, or update the truncation message so it is less confusing, while keeping compatibility.

## Scope

**In scope:**
- `scripts/mcp/server.py` — truncation logic and metadata
- Truncation metadata fields
- Response display behavior

**Out of scope:**
- Changing the truncation mechanism itself (character-based vs byte-based)
- Modifying truncation limits in config
- Changes to individual MCP server implementations
- Breaking changes to response format

## Assumptions

1. Current truncation metadata reports the configured limit, not the actual visible byte count — this is misleading during troubleshooting.
2. The truncation happens in `scripts/mcp/server.py` as noted in the requirement.
3. Operators need to know how many bytes are actually shown vs how many were truncated away.

4. **Verify compatibility**
