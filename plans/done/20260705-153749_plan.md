**Out**: Redesigning server config files. Removing intentional lockdown support. Documentation-only updates.

## Design

**Current (broken) pattern for shell audit (line 634):**
```python
    except Exception:
        import sys as _sys
        exc = _sys.exc_info()[1]
        if isinstance(exc, RuntimeError):
            raise
        pass  # silently skip all other exceptions including config load failures
```

**New pattern:**
```python
    except RuntimeError:
        raise
    except (FileNotFoundError, OSError, ValueError, TypeError) as exc:
        msg = f"Security audit: failed to load shell config: {exc}"
        if production_mode:
            logger.error(msg)
            raise RuntimeError(msg) from exc
        logger.warning(msg)
        warnings.append(msg)
    except ImportError:
        pass  # optional dependency; skip gracefully
```

Apply same pattern to `GitConfig`, `GitHubConfig`, `CicdConfig` load exceptions.

**`security_lockdown_enabled=True` behavior:**
- SUPPRESS: deny-all warnings (empty `command_allowlist`, `allowed_repo_paths`, etc.)
- DO NOT SUPPRESS: config load failures, production auth enforcement, shell sandbox enforcement.
Current code at line 591 sets `lockdown` flag which is only checked for deny-all warnings — this is already correct.

**GitHub write settings (line 714):**
- `allow_force_push=true` and `require_pr_review=false` are warnings, not fatal.
- Config load failure here should also be non-fatal (different from security failures).
- Keep as-is except log properly on unexpected exception.
