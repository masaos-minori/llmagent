## 8. Risks & Mitigations
- **Risk**: Adding missing fields to §1.1 / §1.2 JSON examples changes the schema spec visible to AI agents, which may reveal a discrepancy with the actual runtime schema → **Mitigation**: Cross-check fields against the actual `_save_crawl_file` and `process_file` implementations in `crawler.py` / `chunk_splitter.py` before committing; do not add fields that do not exist in the runtime payload.
- **Risk**: Markdown lint rules may conflict with the intentional Japanese mixed content in §2.0, causing false-positive errors → **Mitigation**: Scope lint to structural rules only (headings, code fences, table formatting); disable MD013 (line length) and MD033 (inline HTML) in `.markdownlint.json`.
- **Risk**: `markdownlint-cli` is not available in the environment and `npm install` is slow or unavailable → **Mitigation**: Fall back to `pymarkdown` (pure-Python Markdown linter installable via `uv add`) and document the alternative invocation.
