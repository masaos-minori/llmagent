## Goal

Add automated tests that detect drift between `_COMMANDS` (command definitions) and `CommandRegistry` handler methods, verify `/help` output is generated from definitions, and test plugin shadow rejection for all built-in commands.

## Scope

**In**: Tests for bidirectional sync (every CommandDef has a handler, every handler has a CommandDef). Tests for `/help` output consistency. Tests for exact vs prefix dispatch. Tests for plugin shadow rejection for all built-in commands.

**Out**: Redesigning the command system, changing user-facing command names, documentation-only updates.

## Assumptions

1. `registry.py` `__init__` already validates handler existence at startup (`hasattr(self, cmd.handler)`).
2. The reverse check (every mixin `_cmd_*` method has a `CommandDef`) is NOT currently enforced.
3. `_COMMANDS` in `command_defs_list.py` is the single source of truth for all built-in commands.
4. Plugin shadow rejection is handled by `plugin_conflicts.py:validate_command_conflicts()`.
5. The `CommandRegistry` inherits from 13 mixin classes; all mixin `_cmd_*` methods should have entries in `_COMMANDS`.

## Design

**Test 1: Every CommandDef has a handler**
```python
def test_every_commanddef_has_handler():
    registry = CommandRegistry(mock_ctx)
    for cmd in _COMMANDS:
        assert hasattr(registry, cmd.handler), f"Missing handler: {cmd.handler!r}"
```
(Already enforced at runtime — but explicit test locks it down.)

**Test 2: Every mixin _cmd_* has a CommandDef**
```python
def test_every_handler_has_commanddef():
    registry = CommandRegistry(mock_ctx)
    handler_names = {name for name in dir(registry) if name.startswith("_cmd_")}
    defined_handlers = {cmd.handler for cmd in _COMMANDS}
    undefined = handler_names - defined_handlers - INTENTIONAL_EXCLUSIONS
    assert not undefined, f"Handlers without CommandDef: {undefined}"
```

**Test 3: /help output matches _COMMANDS**
```python
def test_help_lists_all_commands():
    out = capture_output(registry._cmd_help)
    for cmd in _COMMANDS:
        assert cmd.name in out
```

**Test 4: Dispatch correctness**
```python
# Exact match: "/reload" matches only /reload, not /reloadsomething
# Prefix: "/memory list" matches /memory handler with args="list"
# Exact vs prefix conflict: exact wins over prefix for same prefix
```

**Test 5: Plugin shadow rejection**
```python
for cmd in _COMMANDS:
    # attempt to register plugin with same name
    # verify rejection
```

## Implementation Steps

1. **Phase 1: Identify intentional exclusions**
   - [ ] Grep all mixin files for `def _cmd_` to enumerate all handlers.
   - [ ] Cross-reference with `_COMMANDS.handler` values.
   - [ ] Document intentional exclusions (internal helpers not in `_COMMANDS`).

2. **Phase 2: Add bidirectional sync tests**
   - [ ] Test: every `CommandDef.handler` exists on `CommandRegistry`.
   - [ ] Test: every `_cmd_*` mixin method has a corresponding `CommandDef`.

3. **Phase 3: Add /help and dispatch tests**
   - [ ] Test: `/help` output lists all `cmd.name` values.
   - [ ] Test: exact command dispatch (`/reload` exact).
   - [ ] Test: prefix command dispatch (`/memory list` → handler receives "list").
   - [ ] Test: no prefix false-positive (`/mem` does NOT match `/memory`).
   - [ ] Test: prefix/exact distinction (if `/compact` and `/compact-heavy` both existed).

4. **Phase 4: Add plugin shadow rejection tests**
   - [ ] For each command in `_COMMANDS`, verify `validate_command_conflicts()` rejects a plugin with that name.
   - [ ] Test strict mode: `PluginLoadError` when shadow detected.
   - [ ] Test non-strict mode: shadow reported but other plugins load.

5. **Phase 5: Deployment & Verification**
   - [ ] Run `uv run pytest tests/ -x -q` and confirm all pass.
   - [ ] No changes to deploy.sh required.
