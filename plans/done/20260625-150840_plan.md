# Plan: [Medium][eventbus][config] poll_interval_ms をフォールバック/再同期制御に再分類

Source: `requires/20260625_40_require.md`

## Goal

`poll_interval_ms` を「ライブ配信のメイン遅延制御」から「ブローカー障害時の再同期ポーリング間隔 (fallback)」に再定義し、通常のライブ配信パスがこの設定値に依存しないことを明確にする。

## Scope

**In-Scope**
- `scripts/eventbus/config.py` の `poll_interval_ms` / `offset_checkpoint_interval` フィールドへの deprecated コメント追加
- `scripts/eventbus/app.py` の subscribe() から `poll_interval_ms` 参照の削除 (req #37 実装後)
- `docs/06_eventbus_05_configuration_deploy_and_operations.md` の config テーブル更新
- 新フィールド `resync_interval_ms` の追加 (必要な場合のみ)

**Out-of-Scope**
- フォールバックポーリングロジックの実装 (将来タスク)
- フィールド削除 (後方互換性維持のため deprecated に留める)

## Assumptions

1. req #37 (subscribe ハイブリッド化) 実装後、subscribe() は `poll_interval_ms` を参照しない
2. 現在のフォールバックポーリングは「ブローカーが利用不可になった場合」の将来機能であり、本タスクでは実装しない
3. `poll_interval_ms` は config ファイルで設定されていても subscribe() に影響を与えなくなる
4. `offset_checkpoint_interval` も同様に deprecated とする (req #29 でオフセットは ack ベースになったため)

## Design

**`config.py` の変更:**

```python
@dataclass(frozen=True)
class EventBusConfig:
    port: int
    db_path: str
    storage_dir: str
    offsets_dir: str
    deadletter_dir: str
    max_retry: int
    poll_interval_ms: int = 500          # deprecated: no longer used for live delivery (req #37)
    offset_checkpoint_interval: int = 10  # deprecated: offset is advanced by explicit ack (req #29)
```

**`app.py` の変更:**

現在 lifespan に:
```python
logger.info("eventbus: subscribe polling interval: %dms", _cfg.poll_interval_ms)
```
→ 削除またはコメントアウト。

subscribe() 内の `interval = _cfg.poll_interval_ms / 1000.0` と `asyncio.sleep(interval)` は req #37 で削除済みのはずだが、本タスクでも参照がゼロであることを確認する。

**docs の変更:**

config テーブルの `poll_interval_ms` 行:
```
| `poll_interval_ms` | int | 500 | **Deprecated** (req #37). Was: subscribe poll interval. Now unused. |
| `offset_checkpoint_interval` | int | 10 | **Deprecated** (req #29). Was: mid-stream checkpoint interval. Now unused. |
```

## Implementation Steps

1. `config.py` の 2 フィールドに deprecated コメントを追加
2. `app.py` の lifespan ログから `poll_interval_ms` 参照を削除
3. subscribe() 内に `poll_interval_ms` 参照が残っていないことを確認 (req #37 完了後)
4. `docs/06_eventbus_05_configuration_deploy_and_operations.md` の config テーブルを更新
