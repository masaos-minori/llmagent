# MCP Health Check with Degraded State Control — Work Plan

Source instruction: `instructions/20260606_08_instruction.md`

## Goal

MCP サーバごとに `healthy / degraded / unavailable` 状態を持ち、unavailable 時にツールを自動無効化する。

## Scope

**In:** `McpServerHealthState` enum + `McpServerHealthRegistry`、`ToolExecutor` 無効化ゲート、watchdog 状態更新、/mcp 表示追加
**Out:** 自動復旧起動、外部通知

## Assumptions

1. `repl_health.py::watchdog_loop()` は定期ヘルスチェック済み。
2. `ToolExecutor` は現在 health 状態を保持していない。

## Unknowns

なし — 実装ドキュメント作成済み。

## Affected areas

| File | Change |
|---|---|
| `scripts/shared/tool_executor.py` | `McpServerHealthRegistry` 追加、unavailable ゲート |
| `scripts/agent/repl_health.py` | watchdog で健全性状態を更新 |
| `scripts/agent/services/mcp_status.py` | /mcp 表示に健全性状態追加 |

実装ドキュメント: `implementations/20260606-195339_mcp_health_states.md`

## Design

連続失敗カウンターで degraded → unavailable に遷移。成功で healthy にリセット。unavailable 時は transport を呼ばずに即エラーを返す。
