## Goal

Change `AgentContext.services` field type from `Any` to `AppServices | None`, and migrate all direct `ctx.services.xxx` accesses in `orchestrator.py` and `llm_turn_runner.py` to use `ctx.services_required.xxx`. This restores static type checking for services access in the core agent turn pipeline.

---

## Scope

**In-Scope**
- `scripts/agent/context.py`: Change `self.services: Any = None` to `self.services: AppServices | None = None`; remove the "Typed as Any..." comment; drop `Any` from imports if no longer used
- `scripts/agent/orchestrator.py`: Replace all `ctx.services.xxx` accesses (lines 358, 362–363, 377–378, 396, 399, 491, 495, 513, 615–616) with `ctx.services_required.xxx`
- `scripts/agent/llm_turn_runner.py`: Replace `ctx.services.llm` access at lines 177 and 179 with `ctx.services_required.llm`

**Out-of-Scope**
- `scripts/agent/repl.py` — its `ctx.services.xxx` accesses involve legitimate None-possibility (session teardown, startup failure); keep as-is
- `scripts/agent/commands/` directory — deferred to a follow-up migration pass
- `scripts/agent/startup.py` — out of scope per requirement
- `scripts/agent/tool_runner.py`, `tool_audit.py`, `tool_approval.py` — out of scope
- `scripts/agent/factory.py` — correctly assigns `AppServices`; no change needed
- `services_required` property implementation — no change needed

---

## Assumptions

- `orchestrator.py` and `llm_turn_runner.py` are only ever called after `factory.build_agent_context()` completes, so `ctx.services` is guaranteed non-None at every existing `ctx.services.xxx` access site in those files — using `services_required` is safe.
- `repl.py:138` (`ctx.services.memory is not None`) and `repl.py:154–155` (`ctx.services is not None`) are genuine teardown guards where `services` may be `None` (startup failed mid-way).
- `repl.py:384` (`llm = ctx.services.llm`) is called from `_dispatch_line` which runs after `_init_components`; however the requirement explicitly excludes `repl.py` from this pass — no change.
- `Any` is still used in `context.py` at line 76 (`set[asyncio.Task[Any]]`), so the `Any` import must remain even after the `services` field change.
