1. `message.get("content", "")` は `str | None | list | object` を返す可能性がある（TypedDict total=False）。`str()` の代わりに `isinstance` を使い、str 以外は `SessionTitleGenerationError` を raise する。
2. `write_export(content, outfile, n_messages)` の呼び出し元は `cmd_ingest.py:40` のみ（確認済み）。`out` 引数を `None` から `_CliExportOutput()` に変更する。
3. `_CliExportOutput` はすでに module-private クラス。`cmd_ingest.py` は `export_formatter` から import して使う。

## Design

### `session_title.py` の修正

```python
# Before
title = str(message.get("content", "")).strip()
if not title:
    raise SessionTitleGenerationError("LLM returned empty title")

# After
content_raw = message.get("content")
if not isinstance(content_raw, str):
    raise SessionTitleGenerationError(
        f"LLM title content must be str, got {type(content_raw).__name__}"
    )
title = content_raw.strip()
if not title:
    raise SessionTitleGenerationError("LLM returned empty title")
```

### `export_formatter.py` の修正

```python
# Before
def write_export(
    content: str,
    outfile: str | None,
    n_messages: int,
    out: ExportOutputPort | None = None,
) -> None:
    """...
    When out is None, falls back to the built-in CliExportOutput behaviour
    for backward compatibility with call sites that have not been updated.
    """
    _out = out if out is not None else _CliExportOutput()

# After
def write_export(
    content: str,
    outfile: str | None,
    n_messages: int,
    out: ExportOutputPort = _CliExportOutput(),  # concrete default
) -> None:
    """Write export content to stdout or a file."""
    _out = out
```

Note: Using `_CliExportOutput()` as a default argument is idiomatic for a stateless
adapter class. Alternatively, keep the function body as-is but remove the None branch
and the backward compat comment.

### `cmd_ingest.py` の修正

```python
# Before
write_export(content, outfile, len(ctx.conv.history))

# After
from agent.services.export_formatter import _CliExportOutput
write_export(content, outfile, len(ctx.conv.history), _CliExportOutput())
```

Or simpler: the `export_formatter` can keep `out` optional but with `_CliExportOutput()`
as the concrete default (removing the `None` path and comment).
