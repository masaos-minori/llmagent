# Plan: Define clear HTTP status semantics for EventBus health endpoint

Source: `requires/20260625_52_require.md`

---

## Goal

EventBus の `/health` エンドポイントが degraded/unavailable 時に HTTP 503 を返すようにし、インフラのヘルスチェックと統合できるようにする。

---

## Scope

**In-Scope**
- `scripts/eventbus/app.py`: `/health` エンドポイントの HTTP status code 変更（degraded → 503）
- healthy / degraded / unavailable の 3 状態の定義
- 後方互換性の考慮（既存ヘルスチェック consumer への影響）
- テスト追加

**Out-of-Scope**
- `/health` の body フィールドの変更

---

## Assumptions

1. 現行: `/health` は常に HTTP 200 を返し、body の `"status"` フィールドが `"ok"` または `"degraded"` を示す（確認済み、line 91-99）。
2. degraded = `dlq_task` が stopped または DB 接続エラー。
3. HTTP 503 への変更は breaking change（ヘルスチェックが 200 only を期待している場合）。
