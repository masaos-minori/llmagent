## Goal

Install a SIGINT handler alongside the existing SIGTERM handler in `AgentREPL.run()` so that Ctrl-C triggers the same graceful shutdown path as SIGTERM, preventing orphaned MCP subprocesses and other resource leaks.

## Scope

**In:**
- `scripts/agent/repl.py`: Install SIGINT handler alongside SIGTERM handler in `AgentREPL.run()`

**Out:**
- Modifying `scripts/agent/http_lifecycle.py` shutdown-time SIGINT absorption logic
- Renaming `_sigterm_handler` to `_shutdown_handler` (out of scope for this change)
- Any other file modifications

## Assumptions

1. The existing `_sigterm_handler` function name and implementation are acceptable as-is; renaming it is deferred to a separate refactoring effort.
2. The `NotImplementedError` fallback branch for non-event-loop environments exists solely for compatibility and should mirror the event-loop handler installation with SIGINT.
3. The shutdown-time SIGINT absorption mechanism in `http_lifecycle.py` (`_absorb_sigint_during_shutdown`) does not interfere with the new normal-operation SIGINT handler because they operate in different time windows (normal operation vs. shutdown cleanup).

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| SIGINT handler interferes with shutdown-time SIGINT absorption in http_lifecycle.py | Low | The two mechanisms operate in different time windows: normal operation vs. shutdown cleanup. The shutdown absorption only activates during `shutdown_all()` and restores the original handler afterward, which would be the newly installed SIGINT handler. |
| SIGINT handler fires during shutdown_all() cleanup, causing double-cleanup | Low | The `_shutdown_event` is already set by the first SIGINT/SIGTERM handler invocation; subsequent calls to `_close_resources()` will see `shutdown_requested=True` and skip redundant cleanup. |
| Windows event loop does not support SIGINT signal handlers | Low | The existing `NotImplementedError` fallback at line 456 already handles this for SIGTERM. The same fallback applies to SIGINT automatically — no additional action needed beyond verifying the fallback works on Windows. |
