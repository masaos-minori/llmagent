## 1. Goal

Add an optional `strict_artifact_validation` mode to `RagIngester._validate_artifact()` that rejects chunk JSON files missing `schema_version`, `artifact_type`, or `created_by` fields, while keeping the current lenient behavior as the default (off) to preserve backward compatibility.

---

## 2. Scope

**In-Scope**:
- Add `strict_artifact_validation` config key to `config/rag_pipeline.toml` (default: `false`)
- Extend `RagIngester.__init__()` to read and store the flag
- Extend `_validate_artifact()` signature: add `strict: bool` parameter; enforce `schema_version`, `artifact_type`, `created_by` presence when `strict=True`
- Add `schema_version` and `created_by` as `NotRequired` fields to `ChunkJsonRaw` TypedDict in `pipeline_utils.py`
- Update call sites in `ingester.py` (`ingest_url_group`, `_embed_and_store`) to pass `self._strict_artifact_validation`
- Add unit tests covering: lenient-mode pass (missing fields accepted), strict-mode reject (each missing field), strict-mode pass (all fields present), wrong `artifact_type` still rejected in both modes
- Update docs: `docs/03_rag_02_ingestion_pipeline.md` §4.1 (`_validate_artifact` description), `docs/03_rag_04_data_model_and_interfaces.md` §1.2 (chunk file field table), `docs/03_rag_05_configuration_and_operations.md` §1.1 (new config param)

**Out-of-Scope**:
- Changes to `crawler.py` or `chunk_splitter.py` (they already write the fields correctly)
- DB schema changes
- Migration tooling to retroactively fix existing registered files
- `schema_version` value validation (only presence check; value range enforcement is a future concern)
- Validation of `created_by` values (only presence check, no allowlist)

---

## 3. Assumptions

1. `ChunkOutputPayload` in `chunk_splitter.py` already declares `schema_version`, `artifact_type`, `created_by` as required fields; all chunk files produced post-2024 include them.
2. Legacy chunk files in `rag-src/chunk/` (pre-artifact-format) do NOT include these fields; enabling `strict=True` on a live system with legacy files would cause ingestion failures — migration guidance in docs is sufficient, no automated fix required.
3. The `strict_artifact_validation` flag is a top-level `rag_pipeline.toml` key; no restart of the embed-llm or SQLite service is needed.
4. `_validate_artifact` is a `@staticmethod`; passing `strict` as a parameter (rather than `self._strict`) keeps the method independently testable and avoids changing it to an instance method — callers pass the flag explicitly.
5. `schema_version` valid value is `"1"` (the only version in use); the strict check enforces presence only, not value equality, to avoid blocking future version increments.

## 8. Risks & Mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| Operator enables `strict=True` on a system with legacy chunk files lacking `schema_version`/`created_by`; all ingestion silently fails | Low (default is off) | Doc §1.1 includes a pre-flight inspection command; strict mode must be opted in explicitly |
| `_validate_artifact` becoming an instance method breaks tests that call it as a static method | Low | Keep it as `@staticmethod`; pass `strict` as explicit kwarg — no `self` needed |
| `ChunkJsonRaw` TypedDict extension causes false-positive mypy errors in callers that spread `**payload` | Low | `NotRequired` fields are already used in the TypedDict; adding more does not break spread semantics |
| Two call sites in `ingester.py` diverge (one updated, one not) | Low | Both are in the same file; code review checklist includes grepping for `_validate_artifact` before merging |
