## Design

### D-1. `CommandDef` 拡張

現在は `name / prefix / is_async / handler / help` のみ。以下を追加する:

```python
@dataclass
class SubcommandSpec:
    name: str
    help: str

@dataclass
class CommandDef:
    name: str
    prefix: bool
    is_async: bool
    handler: str
    help: str
    subcommands: list[SubcommandSpec] = field(default_factory=list)
```

`/help` の subcommand 一覧はここから自動生成できるようにする。dispatch 方式は現状維持（変更なし）。

### D-2. Structured Parser

`utils.py` の `parse_flag_int` / `parse_flag_str` を使っている箇所に対して、段階的に `argparse.ArgumentParser` ライクな `CommandParser` を導入する。今回は最初にインタフェースを定義し、使用箇所を順次置換する。

```python
@dataclass
class ParsedArgs:
    subcommand: str | None
    positional: list[str]
    flags: dict[str, str | int | bool]
    error: str | None  # None = parse success
```

### D-3. Formatter モジュール

`scripts/agent/commands/formatter.py` に以下を定義:
- `print_success(msg)` / `print_error(msg)` / `print_no_data(msg)` / `print_validation_error(msg)`
- `print_table(headers, rows)` — `/db urls` / `/memory list` などの統一表示
- `print_kv_list(pairs)` — `/context` / `/db stats` などの key-value 表示

### D-4. `cmd_db.py` の exception 廃止

各 `_db_*` メソッドの `except sqlite3.Error / except Exception` を削除し、例外は呼び出し元 `_cmd_db()` まで伝播させる。`_cmd_db()` は structured error を `formatter.print_error()` で表示する。

### D-5. `/clear` と `/system` の service 化

現状: `ctx.conv.history = ctx.conv.history[:1]` (直接操作)
方針: `ConversationLifecycleService.clear_history(ctx, new_session=bool)` に委譲。サービスは `agent/services/` に配置または既存サービスを拡張する。
