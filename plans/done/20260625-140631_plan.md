## Design

**`docs/06_eventbus_03_persistence_schema_and_replay.md` の変更箇所:**

削除する文:
```
**Why `check_same_thread=False` is safe**: FastAPI runs on a single async event
loop thread. All DB operations execute on that thread. WAL mode serializes concurrent
writers at the SQLite level.
```

置換後:
```
**Why `check_same_thread=False` is safe**: All DB operations are executed via
`asyncio.to_thread()` and protected by a module-level `threading.Lock`. The lock
serializes concurrent DB access across threads; WAL mode also serializes concurrent
writers at the SQLite level.
```

**`docs/06_eventbus_06_reference_api.md` の変更箇所:**

| Variable | Type | Description |
の `_db` 行:

変更前: `Shared SQLite connection; set in lifespan`
変更後: `Shared SQLite connection (protected by _db_lock); set in lifespan`

**`scripts/eventbus/db.py` の docstring:**

変更前: `Uses check_same_thread=False because FastAPI runs on a single async event loop thread; WAL mode serializes concurrent writers at the SQLite level.`
変更後: `Uses check_same_thread=False. Concurrent access from asyncio.to_thread() calls is serialized by _db_lock in app.py; WAL mode serializes concurrent writers at the SQLite level.`
