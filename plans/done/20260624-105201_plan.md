## Goal

Document the behavior when a plugin registers an MCP tool that conflicts with a built-in tool: the plugin registration is rejected at startup with a clear error, not silently overridden.

---

## Assumptions

1. Plugin tool name conflicts with built-in MCP tool names cause a startup error.
2. The error message includes both the conflicting plugin name and the MCP tool name.
3. `strict_startup_validation = true` (default) means this is fatal; `false` downgrades to WARNING.

## Design

### Plugin MCP-Conflict Rejection Behavior

```
Condition: Plugin attempts to register tool_name already registered by MCP/built-in
Behavior (strict_startup_validation = true):
  - Startup fails with: PluginConflictError("Plugin '{plugin}' tool '{tool_name}' conflicts with built-in MCP tool")
  - Agent exits immediately

Behavior (strict_startup_validation = false):
  - WARNING emitted; plugin tool registration is skipped
  - Built-in MCP tool takes precedence

SPEC: PLUGIN-01
```
