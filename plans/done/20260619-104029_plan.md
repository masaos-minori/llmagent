# Plan: Strengthen the operational path for check_rag_consistency()

## Goal

Improve `check_rag_consistency()` to provide clearer warnings, better operator guidance, and stronger post-ingestion validation. Add severity classification, repair recommendations, and detect additional inconsistency patterns.

## Scope

**In scope:**
- Add severity classification to consistency issues (warning vs critical)
- Improve warning messages with repair recommendations
- Detect FTS > chunks case (not just chunks > fts)
- Propagate consistency results from `ingest_all()` to callers
- Add operator workflow guidance in error messages

**Out of scope:**
- Adding MCP tool for consistency checking
- Adding periodic/cron-based background checks
- Automating repair actions (FTS rebuild, targeted re-ingestion)
- Optimizing orphan detection query performance

## Assumptions

1. The current detection logic is correct; only the reporting and operational guidance need improvement.
2. Repair commands (`/db rebuild-fts`, `--force` re-ingestion) are already available; plan just references them in warnings.
3. Post-ingestion check should remain non-blocking (warnings only, no abort).

## Unknowns

| Unknown | Evidence missing | Resolution | Blocking |
|---|---|---|---|
| Whether callers of `ingest_all()` need consistency results | Need to check who calls `RagIngester.ingest_all()` | Plan adds optional return value; backward compatible | no |

## Implementation Steps

1. **Add severity classification to `RagConsistencyReport`**
   - Add a method or property that classifies issues by severity:
     - `warning`: FTS gap > 0, vec/chunks mismatch (minor)
     - `critical`: orphan_vec_count > 0, FTS > chunks (data loss risk)
   - Update `summarize_issues()` to include severity prefix in messages

2. **Improve warning messages with repair recommendations**
   - FTS gap: "FTS gap detected (chunks=100, fts=95, gap=5). Run `/db rebuild-fts` to repair."
   - Orphan vec rows: "Orphan vec rows detected (count=3). Re-run ingestion with `--force` for affected URLs."
   - Vec/chunks mismatch: "Vector count mismatch (chunks=100, vec=97). Check for failed ingestion steps."
   - FTS > chunks: "FTS index has more entries than chunks (fts=105, chunks=100). Run `/db rebuild-fts` to repair."

3. **Detect FTS > chunks case**
   - Current: `fts_gap = chunks - fts` (only detects chunks > fts)
   - Fix: Use absolute difference or add separate field `fts_orphan_count = max(0, fts - chunks)`
   - Update `is_consistent()` to check both directions

4. **Propagate consistency results from `ingest_all()`**
   - Add optional return value or set attribute on `RagIngester` instance
   - Keep backward compatible: existing callers that don't check results still work
   - Log severity-prefixed warnings (e.g., `[WARNING]`, `[CRITICAL]`)

5. **Add operator workflow guidance**
   - Document the repair workflow in docstrings and comments
   - Include in error messages: what to do next
   - Reference `/db rebuild-fts` and `--force` re-ingestion
