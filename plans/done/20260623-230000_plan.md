## Goal

コアエージェント制御ループ (`orchestrator.py`, `llm_turn_runner.py`, `tool_loop_guard.py`) のコードレビューで発見された Critical/High 問題を修正し、状態管理の安全性・ツールループガードの正確性・asyncio の健全性を改善する。

---

## Scope

**In-Scope:**
- `scripts/agent/orchestrator.py`
  - `_classify_and_inject_mode()`: `system_prompt_content` への永続追記をエフェメラル注入へ変更
  - `_append_user_message()`: `asyncio.create_task()` の fire-and-forget をタスク参照付きに変更
  - `_process_turn()`: `ctx.cfg.tool.allowed_tools` のグローバルミューテーション除去
  - `_handle_llm_turn()`: `_handle_llm_transport_error` 二重呼び出しパスの統一
  - `session_id` のスコープを `try` ブロック外に移動 (NameError リスク除去)
  - `_consolidate_mid_turn_errors` / `_inject_recovery_context` dead code 削除
- `scripts/agent/llm_turn_runner.py`
  - `_finalize_answer_text()`: 不要な `async` 宣言を `def` に変更
  - `_stream_llm()`: 未使用 `turn` パラメータをログに活用またはシグネチャ整理
- `scripts/agent/tool_loop_guard.py`
  - `check_dedup` / `check_retry` ハッシュキーを `_canonical_key()` に統一
  - `check_cycle()`: `round_fingerprints.count()` 二重呼び出しを1回に削減
  - `check_retry()` diagnostics の `DEDUP_HINT` を `RETRY_HINT` (新規定義) に修正
  - `update_errors()`: 部分失敗時のリセット条件を保守的に変更
- `tests/test_tool_loop_guard.py`: ハッシュ統一後の挙動変更に合わせてテスト修正・追加

**Out-of-Scope:**
- `partial_completion` 監査フィールドの完全実装 (別 PR 候補)
- `_classify_and_inject_mode` のモード解決ロジック変更
- `ToolLoopGuard` のスレッドセーフ化
- セッション永続化 (`ctx.session.save`) の責務分離リファクタリング

## Assumptions

1. `ctx.conv.history` への `{"role": "system", "_ephemeral": True}` 注入は既存の `hist_mgr` がエフェメラルエントリを除去するタイミングで処理される (または本 PR で除去ロジックを追加する)。
2. `asyncio.create_task()` のタスク参照保持には `set` ベースの弱参照パターンを使用する。GC 保護のみが目的で、タスクの完了を待機する必要はない。
3. `LLMTurnRunner.run()` が `LLMTransportError` を `TurnResult(exception=e)` として返す (raise しない) 実装に統一することで `except` ブロックを除去できる。テストのモック方式もこれに合わせて変更する。
4. `tool_hash_key(name, args)` は引数 dict のキーをソートして正規化する実装であり、引数の JSON 表現順序に依存しない (既存実装を前提とする)。
5. `_canonical_key()` の JSON parse 失敗時は空 dict `{}` を使用してハッシュを生成し、`None` を返さない。
