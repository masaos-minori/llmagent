1. `.md` / `.markdown` / `.mdx` のURL判定は拡張子チェックのみで十分（content-based heuristic は md_index_enable フラグ有効時のみ維持）

5. `document_repo.py` が `SQLiteHelper("session")` を使用しているのは既存バグ — Step 0 で `"rag"` に修正する

## Design

### chunking_strategy の受け渡し方針 (Approach A: URL 判定)

`RagIngester._read_chunk_json()` は `read_json_file()` → `dataclasses.asdict(ChunkDocument)` を経由するため、chunk JSON に追加フィールドを書いても `ChunkDocument` にないフィールドは**消える**。

`ChunkDocument` / `pipeline_utils.py` を変更しない最小変更案として、URL から `chunking_strategy` を直接判定する方式を採用する。

- 制約: `--snippets-only` で非 `.md` ファイルを処理した場合は `'text'` と記録される（実際は `'heading'`）
- 許容理由: `--snippets-only` はデバッグ・強制用オプションであり、DB への記録精度より実装シンプルさを優先する

### [BUG-1/2/3] ingester.py `_read_chunk_json()` — 3 つのバグの根本原因

**根本原因**: `_read_chunk_json()` (L237–L247) が `read_json_file()` → `dataclasses.asdict(ChunkDocument)` 経由で
chunk ファイルを読むため、`ChunkDocument` に存在しないフィールドがすべて消える。

3 つのバグはすべてこの 1 か所から派生する:

**BUG-1** `chunking_strategy` フィールドが消える:
```
chunk JSON: {"chunking_strategy": "heading", ...}
→ ChunkDocument(url, title, lang, content, code_blocks)  ← フィールドなし
→ first_data.get("chunking_strategy", "text")  →  常に "text"
```
影響: `/db urls` の Strategy が常に `text`。`.md` の heading chunking が DB に記録されない。

**BUG-2** `normalized_content` フィールドが消える:
`_embed_and_store()` (L255) が `normalized_content: str | None = None` とハードコードしており、
chunk JSON の `normalized_content` (日本語 Sudachi 正規化テキスト) を無視する。
影響: 日本語チャンクの FTS5 検索が正規化テキスト (`normalized_content`) ではなく
生テキスト (`content`) でインデックスされる — 検索精度低下。

**BUG-3** `chunk_index` フィールドが消える:
`_embed_and_store()` (L256–L260) に `try: idx = 0; except ...: idx = 0` という
例外が発生しえない dead code が残り、常に `idx = 0` が使われる。
chunk JSON に `chunk_index` フィールドがあるが無視される。
影響: 全チャンクが `chunk_index=0` で INSERT される — 順序情報が失われる。

### Step 3-fix — `_read_chunk_json()` の修正 (BUG-1/2/3 同時解消)

**変更 1: `_read_chunk_json()` を生 JSON 読み取りに変更 (L237–L247)**

```python
def _read_chunk_json(self, path: Path) -> "dict[str, Any] | None":
    """Read and parse a chunk JSON file; returns None on failure."""
```

変更後:
```python
def _read_chunk_json(self, path: Path) -> "dict[str, Any] | None":
    """Read and parse a chunk JSON file; returns None on failure."""
    try:
        data = orjson.loads(path.read_bytes())
    except (OSError, orjson.JSONDecodeError) as e:
        logger.warning(f"skip chunk {path.name}: {e}")
        return None
    if not isinstance(data, dict) or not data.get("url") or not data.get("content"):
        logger.warning(f"skip chunk {path.name}: missing url or content")
        return None
```
