## Assumptions

1. `.md` / `.markdown` / `.mdx` のURL判定は拡張子チェックのみで十分（content-based heuristic は md_index_enable フラグ有効時のみ維持）
2. SQLite の `ALTER TABLE ADD COLUMN` は既存行の DEFAULT 値が自動適用されるため、既存レコードの backfill は不要
3. `chunking_strategy` は chunk JSON には書かず、`RagIngester.ingest_url_group()` が URL から直接判定する（Approach A）
4. `_chunk_markdown_by_heading()` のフォールバック（`_chunk_english()`）は現状のままで十分
5. `document_repo.py` が `SQLiteHelper("session")` を使用しているのは既存バグ — Step 0 で `"rag"` に修正する

## Design

### chunking_strategy の受け渡し方針 (Approach A: URL 判定)

`RagIngester._read_chunk_json()` は `read_json_file()` → `dataclasses.asdict(ChunkDocument)` を経由するため、chunk JSON に追加フィールドを書いても `ChunkDocument` にないフィールドは**消える**。

`ChunkDocument` / `pipeline_utils.py` を変更しない最小変更案として、URL から `chunking_strategy` を直接判定する方式を採用する。

- 制約: `--snippets-only` で非 `.md` ファイルを処理した場合は `'text'` と記録される（実際は `'heading'`）
- 許容理由: `--snippets-only` はデバッグ・強制用オプションであり、DB への記録精度より実装シンプルさを優先する

## 未解決の改善事項 (2026-06-13 調査反映)

### [BUG-1] ingester.py: chunking_strategy が常に `'text'` になる (致命的)

**問題**:
`ingest_url_group()` は chunk JSON から `chunking_strategy` を読もうとしているが、
`_read_chunk_json()` が `read_json_file()` → `dataclasses.asdict(ChunkDocument)` の経路を通るため
`ChunkDocument` にないフィールドはすべて消える。
結果として `.md` ファイルでも常に `'text'` が DB に保存される。

```
chunk ファイルの JSON: { "chunking_strategy": "heading", ... }
                               ↓
_read_chunk_json() → read_json_file() → ChunkDocument(url,title,lang,content,code_blocks)
                               ↓ dataclasses.asdict()
first_data: { "url":..., "title":..., "lang":..., "content":..., "code_blocks":[] }
                               ↑ chunking_strategy が消えている
first_data.get("chunking_strategy", "text")  →  常に "text"
```

**影響**: `/db urls` の Strategy カラムが常に `text` を表示する — 機能不全

**修正方針**: `_read_chunk_json()` を生 JSON 読み取りに変更 (Option A):

```python
def _read_chunk_json(self, path: Path) -> "dict[str, Any] | None":
    """Read and parse a chunk JSON file; returns None on failure."""
    try:
        data = orjson.loads(path.read_bytes())
    except (OSError, orjson.JSONDecodeError) as e:
        logger.warning(f"skip chunk {path.name}: {e}")
        return None
    if not isinstance(data, dict) or not data.get("url") or not data.get("content"):
        logger.warning(f"skip chunk {path.name}: missing url or content")
        return None
    return data
```

`read_json_file()` を使わなくなるため `pipeline_utils` への import が不要になる。`ChunkDocument` は変更不要。

**対応 Step**: Step 3-fix
