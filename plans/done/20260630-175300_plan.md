## Design
N/A — small task

The three required test functions will be added to `tests/test_eventbus_ack_nack.py` as a new `TestAckHttpBehavior` class. They verify HTTP semantics via TestClient (not DB layer), complementing the existing `TestAckEvent` class (DB layer) in the same file.

Each function maps to one acceptance criterion:
- `test_ack_first_time_returns_200` — POST /events/{id}/ack returns 200 with `acked: true, seq: int`
- `test_ack_repeated_returns_200_already_acked` — second POST returns 200 with `acked: true, already_acked: true`
- `test_ack_unknown_event_returns_404` — POST with unknown event_id returns 404

The TestClient fixture pattern in `test_eventbus_ack_nack.py` does not currently include an HTTP-level client. It will need to follow the same pattern as `test_eventbus_ack_endpoint.py` (import `make_eventbus_client` from `eventbus_helpers` or inline a minimal fixture).
