## 3. Assumptions
- `tests/test_rag_refiner.py` is a new file; the requirement listed it as a target component even though it does not yet exist.
- The requirement's `"refiner_exception"` diagnostic field means a boolean or per-call flag (e.g., `True` when the latest Refiner call raised an exception), distinct from the cumulative `refiner_exception_count`.
- Alternatively, `"refiner_exception"` may be acceptable as an alias for `refiner_exception_count > 0`. Resolution requires reviewing the requirement's referenced "Issue 2" documentation.
- The FusionStage currently uses `logger.info` (not `logger.debug`) for dedup mode; the requirement says "debug text indicating rank signal is disabled". This is functionally observable but at a different log level. Treat as acceptable unless Issue 2 documents DEBUG level specifically.
- `uv run --no-sync pytest` is the correct test invocation in this environment (network is unavailable for package resolution).

## 8. Risks & Mitigations
- **Risk**: Adding `"refiner_exception"` key to `get_diagnostics()` may break tests that do exact dict-equality checks on the return value. → **Mitigation**: Search for `get_diagnostics()` call sites in tests before committing; update any exact-match assertions to include the new key.
- **Risk**: `augment()` end-to-end tests require mocking `SQLiteHelper`, `get_embedding`, and `_search_all_queries` across multiple layers. Complex mock setup may introduce test brittleness. → **Mitigation**: Follow the pattern already established in `test_rag_quality_regression.py` (`patch("rag.stages.search._search_all_queries")`); keep augment-level tests focused on fallback behavior, not hit ordering.
- **Risk**: `tests/test_rag_refiner.py` end-to-end augment tests may fail if `RagPipeline.__init__` config validation rejects the mock config. → **Mitigation**: Use `_ModuleConfig.get` patch (as in existing `_make_pipeline` helpers) and provide all required `RagConfig` fields via `SimpleNamespace`.
- **Risk**: The `"refiner_exception"` semantics (boolean vs. last exception string) may conflict with how callers parse `get_diagnostics()` in the MCP service or agent layer. → **Mitigation**: Use boolean type (safest, non-breaking for JSON serialization); document in the function docstring.
