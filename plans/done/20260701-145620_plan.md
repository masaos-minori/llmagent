# Work Plan: Convert MCP server audit logging to JSON-lines format

## Goal

Unify MCP server audit logs with the agent-side JSON-lines audit format by replacing key=value AUDIT output with structured JSON-lines.

## Scope

**In-Scope:**
- Modify `scripts/mcp/audit.py` to emit JSON-lines instead of key=value format
- Update tests in `tests/test_audit_log_format.py` (TestMcpAuditLogFormat) to validate JSON-lines
- Update tests in `tests/test_mcp_server_base.py` (TestAuditLog) to validate JSON-lines

**Out-of-Scope:**
- No per-server `audit.py` files (explicitly excluded by requirement)
- Changes to agent-side audit logging (already JSON-lines)
- Changes to `_write_audit_log` methods in shell/file delete services (different log file)
- Changes to `_write_github_audit_log` methods in github service (different log file)

## Assumptions

1. The `action` parameter in `_audit_log()` maps to the `tool` field in the new format
2. `detail` field may contain comma-separated key=value pairs — should be preserved as-is in JSON
3. Timestamp should use `time.time()` for epoch seconds
4. Empty strings for optional fields should be omitted from JSON output (not included as empty values)

## Unknowns

| ID | Unknown Description | Evidence Missing | Resolution Path | Blocking? |
|---|---|---|---|---|
| UNK-01 | Whether `detail` field contains comma-separated key=value pairs that need to be parsed into separate JSON fields | Need to check call sites for detail format | Read mdq/server.py, cicd/server.py, etc. call sites | No |
| UNK-02 | Whether the timestamp should be UTC or local time | Not specified in requirement | Use UTC (standard practice) | No |

## Design

### Step 1: Modify scripts/mcp/audit.py
- Add `import json, time`
- Replace key=value f-string with json.dumps() output
- Map `action` parameter to `tool` field in JSON
- Add `event: "mcp_tool_exec"` constant
- Add `source: "mcp_server"` constant
- Add `ts: time.time()` timestamp
- Omit empty optional fields (server_key, error_type) from JSON output

### Step 2: Update tests/test_audit_log_format.py
- Rewrite TestMcpAuditLogFormat class to validate JSON-lines format
- Change assertions from `"AUDIT"` and `"session=sess-1"` to `json.loads()` validation
- Remove `test_no_json_format` — MCP log now IS JSON
- Add `test_mcp_log_is_valid_json` — validates JSON parsing
- Keep field-level tests but use parsed dict instead of string matching

### Step 3: Update tests/test_mcp_server_base.py
- Change TestAuditLog assertions from `"AUDIT"` to JSON validation
- Use `json.loads()` to parse log message and validate fields

## Implementation Steps

1. **Phase 1: Rewrite audit.py**
   - Replace key=value output with json.dumps()
   - Add event/source/ts fields
   - Map action → tool field

2. **Phase 2: Update test_audit_log_format.py**
   - Rewrite TestMcpAuditLogFormat class
   - Change all string-matching assertions to JSON parsing + dict validation
   - Add new tests for JSON-specific behavior (valid JSON, timestamp present)

3. **Phase 3: Update test_mcp_server_base.py**
   - Change AUDIT string assertions to JSON parsing

4. **Phase 4: Validation**
   - Run `pytest tests/test_audit_log_format.py -v`
   - Run `pytest tests/test_mcp_server_base.py -v`
   - Run full MCP service tests

## Risks

- **Risk**: detail field contains comma-separated key=value pairs that may need separate JSON fields for downstream parsing | Likelihood: High | Mitigation: Keep detail as single string field; downstream consumers can parse if needed
- **Risk**: Empty optional fields (server_key, error_type) should be omitted — some callers may expect them in output | Likelihood: Low | Mitigation: Include empty strings for all fields to maintain backward compatibility with log parsers
- **Risk**: Existing log analysis tools expect key=value format and will break | Likelihood: Medium | Mitigation: Document the format change; provide migration guide for log parsing scripts
