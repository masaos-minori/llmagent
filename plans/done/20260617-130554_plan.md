# Plan: Add ingestion tests for BUG-1, BUG-2, and BUG-3

## Goal

Create `tests/test_ingester.py` that locks in correct behavior of `_read_chunk_json()`, `_embed_and_store()`, and `ingest_url_group()` — ensuring `chunking_strategy`, `normalized_content`, and `chunk_index` fields are preserved through ingestion, and `--force` reinsert behavior is tested.

## Assumptions

1. `_read_chunk_json()` in ingester.py already reads raw JSON via `orjson.loads(path.read_bytes())` (fix for BUG-1/2/3 already applied). Tests must verify this behavior.
2. Chunk JSON files contain these fields: `url`, `title`, `lang`, `source_file`, `chunk_index`, `chunk_type`, `content`, `normalized_content`, `etag`, `last_modified`, `chunking_strategy`.
3. The `ChunkDocument` dataclass (models_data.py:27) is missing `chunk_index`, `chunking_strategy`, `normalized_content`, `etag`, `last_modified`, `source_file`, `chunk_type` — which caused the original bugs.
4. Tests should use in-memory SQLite (pattern from test_document_repo.py) and mock the embed-llm HTTP client.
5. `_embed_and_store()` opens its own `SQLiteHelper().open(write_mode=True)` connection per thread — tests must account for this by mocking the helper.
6. No real embed-llm service (port 8003) is available during testing; embeddings will be mocked.

## Implementation Steps

1. **Create `tests/test_ingester.py`** with test harness:
   - `_FakeSQLiteHelper` class (adapt from test_document_repo.py pattern) for in-memory SQLite
   - `_make_chunk_json(tmp_path, chunk_index, content, normalized_content, chunking_strategy)` helper to create test chunk files
   - Mock embed-llm HTTP client (httpx.Client) returning fixed 384-dim embeddings

2. **Test `_read_chunk_json()` field preservation:**
   - `test_read_chunk_json_returns_raw_dict` — verify method returns dict with all fields
   - `test_read_chunk_json_preserves_chunking_strategy` — verify `chunking_strategy` is read from JSON (not default `'text'`)
   - `test_read_chunk_json_preserves_normalized_content` — verify `normalized_content` is read (not None) for JA chunks
   - `test_read_chunk_json_handles_missing_fields` — verify graceful handling when optional fields absent

3. **Test `_embed_and_store()` field passthrough:**
   - `test_embed_and_store_writes_correct_chunk_index` — verify chunk_index from JSON is stored in DB
   - `test_embed_and_store_writes_normalized_content` — verify normalized_content is stored (not NULL)
   - `test_embed_and_store_handles_embedding_failure` — verify False return on embed failure

4. **Test `ingest_url_group()` end-to-end:**
   - `test_ingest_url_group_inserts_document_with_chunking_strategy` — verify documents.chunking_strategy reflects JSON
   - `test_ingest_url_group_inserts_chunks_in_order` — verify chunks have correct chunk_index values
   - `test_ingest_url_group_moves_files_to_registered` — verify file lifecycle

5. **Test `--force` reinsert behavior:**
   - `test_force_deletes_existing_document_and_reinserts` — verify old chunks are removed, new ones inserted
   - `test_force_preserves_new_chunking_strategy` — verify updated chunking_strategy after reinsert
   - `test_no_force_skips_existing_url` — verify idempotency when force=False

6. **Validation:** run `uv run pytest tests/test_ingester.py -v` and confirm all tests pass.
