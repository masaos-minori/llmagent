## Goal

Improve visibility and documentation for disabled memory layer and embedding states in `/memory status` output and startup banner.

## Scope

- **In-Scope**: Fixing `/memory status` to distinguish "memory layer disabled", "memory enabled but embedding disabled", "FTS-only", "degraded", and "hybrid" modes; updating documentation; updating tests if display changes.
- **Out-of-Scope**: Enabling memory by default; changing retrieval ranking; changing memory schema.

## Assumptions

1. Current `/memory status` shows generic "disabled" when `mem is None`, losing the distinction between memory-layer-disabled vs embedding-disabled states.
2. `build_memory_status()` in `memory_status.py` returns `None` when `embed_client is None`, which prevents showing embedding-disabled state details.
3. `MemoryStatus` already has all necessary fields (`memory_layer_enabled`, `embedding_enabled`, `circuit_open`, `last_retrieval_mode`).
