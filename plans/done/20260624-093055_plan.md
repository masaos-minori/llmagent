# Plan: Add Config/Schema Path Overrides to Event Bus

**Requirement:** `requires/20260623_26_require.md`
**Task size:** Path B (adds env-var overrides to config loading)

---

## Goal

Allow `/opt/llm/config/eventbus.toml` and `/opt/llm/schemas/event_envelope.json` defaults to be overridden via environment variables so the service can run outside the fixed deployment layout.

---

## Scope

**In:**
- `scripts/eventbus/config.py` — read `EVENTBUS_CONFIG_PATH` and `EVENTBUS_SCHEMA_PATH` env vars; fall back to `/opt/llm/...` defaults
- `scripts/eventbus/app.py` — pass override paths from env to config loader
- Deployment docs — document env-var overrides

**Out:**
- Redesigning the config model

---

## Assumptions

1. `config.py` currently hard-codes `/opt/llm/config/eventbus.toml` (to be confirmed during read).
2. Env-var override pattern: `os.environ.get("EVENTBUS_CONFIG_PATH", "/opt/llm/config/eventbus.toml")`.
