# Plan: [High][eventbus][publish] publish() から broker へ post-commit 通知を追加

Source: `requires/20260625_36_require.md`

## Goal

`publish()` がSQLite commit 成功後に `_broker.publish()` を呼び出し、ライブ購読者へイベントを即座にプッシュする。

## Scope

**In-Scope**
- `scripts/eventbus/app.py` の `publish()` ハンドラーに broker 通知ステップを追加
- 通知は `asyncio.to_thread` から戻った後 (commit 確認後) のイベントループで実行
- 重複 publish (INSERT OR IGNORE) 時は通知をスキップ
- post-commit notify のテスト追加

**Out-of-Scope**
- broker 実装 (req #35 対象)
- subscribe() の live 配信パス変更 (req #35/37 対象)
- スキーマ変更

## Assumptions

1. req #35 の `EventBroker` が実装済みで `_broker` グローバルが存在すること
2. `_broker.publish()` は同期関数 (asyncio.Queue.put_nowait) のためスレッドオフロード不要
3. 通知データは `_row_to_dict()` 形式の dict (seq 含む) とし、subscribe SSE の payload と一致させる
4. INSERT OR IGNORE で rowcount == 0 (重複) の場合は通知しない (`seq` が変わらず購読者は既に受信済みのはずだが安全のためスキップ)

## Design

**現在の `publish()` 末尾 (概略):**

```python
cur = _db.execute("INSERT OR IGNORE INTO events ...", (...))
_db.commit()
seq = cur.lastrowid if (cur.rowcount > 0 ...) else _get_seq(event_id)
try:
    _append_jsonl(body, seq)
except OSError: ...
logger.info("publish event_id=%s topic=%s seq=%d", ...)
return {"event_id": event_id, "seq": seq}
```

**変更後:**

```python
# Insert + commit (asyncio.to_thread で実行 — req #14 pattern に合わせるか inline かは既存実装に従う)
cur = _db.execute(...)
_db.commit()
seq = ...
committed = cur.rowcount > 0  # 重複 publish は False

try:
    _append_jsonl(body, seq)
except OSError as exc:
    logger.warning(...)

# Post-commit notification (イベントループ上で実行; broker.publish() は同期)
if committed and _broker is not None:
    event_dict = {
        "seq": seq,
        "event_id": event_id,
        "topic": topic,
        "payload": body["payload"],
        "producer": producer,
        "published_at": published_at,
    }
    n = _broker.publish(event_dict)
    logger.debug("publish notify broker delivered=%d seq=%d", n, seq)

logger.info("publish event_id=%s topic=%s seq=%d", event_id, topic, seq)
return {"event_id": event_id, "seq": seq}
```

Note: `_db.execute` + `_db.commit()` はすでに `asyncio.to_thread` なしで直接呼んでいる (現在の app.py:104–110)。broker 通知もイベントループ上で同期的に実行するため問題ない。

## Implementation Steps

1. `app.py` の `publish()` 末尾に broker 通知ブロックを追加 (`committed` フラグ + `_broker.publish(event_dict)`)
2. テスト: publish → broker queue にイベントが入ること、重複 publish では入らないことを確認
