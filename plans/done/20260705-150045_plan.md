## Goal

Add `error_kind` and `error_detail` columns to the `attempts` table, add `workflow_id` and `attempt_number` to the `artifacts` table, and make idempotency keys include enough context for de-duplication; add tests for failed workflow state reconstruction.

## Scope

**In**: Schema migration for `attempts` (add `error_kind`, `error_detail`). Schema migration for `artifacts` (add `workflow_id`, `attempt_number`). Update `attempt_ops.py`, `state_store.py`, `artifact_ops.py` to persist new fields. Add `workflow_id` to `processed_events`. Add tests for failed-stage state reconstruction.

**Out**: Implementing full automatic workflow resume, changing approval semantics, documentation-only updates.

## Assumptions

1. Current `attempts` schema: `attempt_id`, `task_id`, `stage_id`, `status`, `started_at`, `ended_at`, `error_msg`. Missing: `error_kind`, `error_detail`.
2. `error_msg` is currently a single text field. The require distinguishes `error_kind` (classification e.g. "timeout", "exception", "external_error") from `error_detail` (full message).
3. Current `artifacts` schema: `artifact_id`, `task_id`, `stage_id`, `uri`, `created_at`. Missing: `workflow_id`, `attempt_number`.
4. Current `processed_events` schema: `event_id`, `task_id`, `stage_id`, `recorded_at`. Missing: `workflow_id`. The `event_id` is `{task_id}:{stage_id}:{attempt}` which already includes attempt context.
5. Schema changes must be additive (no breaking migrations).
6. `state_store.py` has duplicate `start_attempt`/`finish_attempt` methods alongside `attempt_ops.py` — both need updating.

## Design

**Schema additions**:
```sql
-- attempts table: add two columns
ALTER TABLE attempts ADD COLUMN error_kind TEXT;
ALTER TABLE attempts ADD COLUMN error_detail TEXT;

-- artifacts table: add two columns  
ALTER TABLE artifacts ADD COLUMN workflow_id TEXT;
ALTER TABLE artifacts ADD COLUMN attempt_number INTEGER;

-- processed_events table: add column
ALTER TABLE processed_events ADD COLUMN workflow_id TEXT;
```

**`finish_attempt()` signature change**:
```python
def finish_attempt(
    db, attempt_id, status,
    error_msg=None,
    error_kind=None,   # NEW: e.g. "timeout", "exception", "stage_failure"
    error_detail=None, # NEW: full error detail string
) -> None
```

**`workflow_engine.py` error classification**:
- `WorkflowTimeoutError` → `error_kind="timeout"`
- `WorkflowHaltError` → `error_kind="halt"`
- Other exceptions → `error_kind="exception"`
