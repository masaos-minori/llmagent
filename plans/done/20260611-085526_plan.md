# File MCP (Read/Write/Delete) 改修計画書

## Goal

`scripts/mcp/file/` 配下の Read/Write/Delete MCP サーバーを全面的に改修し、
`_Lazy*Service` 依存・`except Exception` fallback・circular import・
`ValueError` 混在を除去し、strict fail-fast / 型付き DI / thin server 層に切り替える。

## Assumptions

1. `ConfigLoader().load("file_*_mcp_server.toml")` は設定ファイル不備時に例外を送出する（fail-fast 済み）。
2. `mcp/server.py`・`mcp/dispatch.py` の公開 API は変更しない。
3. FastAPI の `@app.exception_handler(ExcType)` は GitHub MCP と同じパターンで使える。
4. Pydantic の `model_json_schema()` は MCP inputSchema 形式の `{type, properties, required}` に変換可能。
5. `_Lazy*Service` 削除後のサービス初期化はモジュールトップレベルで行う
   （`_service = ReadFileService.from_config(FileReadConfig.load())`）。
6. `write_models.py` の `WriteFileRequest._check_content_bytes` validator は
   service 層へ移動し、circular import を解消できる。
7. Atomic write は `target.parent / ".tmp_<name>"` に書いてから `os.replace()` で移動する方式を採る。

## Design

### 1. `_Lazy*Service` → モジュールトップ初期化

```python
# 変更前（lazy proxy）
_service: ReadFileService = _LazyReadFileService()

# 変更後（起動時 1 回ロード）
def _init_service() -> ReadFileService:
    cfg = FileReadConfig.load()
    return ReadFileService(
        allowed_dirs=[Path(d) for d in cfg.allowed_dirs],
        max_read_bytes=cfg.max_file_size_kb * 1024,
        max_tree_depth=cfg.max_depth,
        max_search_results=cfg.max_files_per_batch,
    )

_service: ReadFileService = _init_service()
```

- `_Lazy*` クラスと `type: ignore[assignment]` コメントを完全削除
- `FileReadMCPServer` は `__init__` でサービスを受け取るか、クロージャで捕捉

### 2. 統一例外ハンドラ（GitHub MCP パターン）

```python
from fastapi.responses import JSONResponse

@app.exception_handler(FileAuthorizationError)
async def _on_auth_error(request: Request, exc: FileAuthorizationError) -> JSONResponse:
    return JSONResponse({"detail": str(exc)}, status_code=403)

@app.exception_handler(FileNotFoundError)
async def _on_not_found(request: Request, exc: FileNotFoundError) -> JSONResponse:
    return JSONResponse({"detail": str(exc)}, status_code=404)

@app.exception_handler(FileValidationError)
async def _on_validation_error(request: Request, exc: FileValidationError) -> JSONResponse:
    return JSONResponse({"detail": str(exc)}, status_code=422)
```

### 3. circular import 解消

`WriteFileRequest._check_content_bytes` validator を削除し、
`WriteFileService.write_file()` 冒頭で明示的に byte 数チェックする:

```python
def write_file(self, req: WriteFileRequest) -> WriteFileResponse:
    raw_bytes = req.content.encode("utf-8")
    if len(raw_bytes) > self._max_write_bytes:
        raise FileValidationError(f"content exceeds {self._max_write_bytes} bytes write limit")
    ...
```

### 4. Atomic write

```python
import os, tempfile

tmp = target.parent / f".tmp_{target.name}"
try:
    tmp.write_text(modified, encoding="utf-8")
    os.replace(str(tmp), str(target))
except PermissionError as e:
    tmp.unlink(missing_ok=True)
    raise FileAuthorizationError(str(e))
except OSError as e:
    tmp.unlink(missing_ok=True)
    raise FileValidationError(str(e))
```

### 5. schema 単一ソース化（段階的）

Phase A（今回）: `read_tools.py` の `_MCP_TOOLS` を `model_json_schema()` 由来に変換するユーティリティを追加し、整合性チェックを `tests/test_mcp_tools_validation.py` に追記。

Phase B（将来）: write/delete の `_MCP_TOOLS` も同様に変換。
