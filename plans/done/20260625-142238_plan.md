## Goal

## Design

**現在の問題:**
`write_offset()` は SSE ジェネレーター内でイベントを `yield` した直後に呼ばれる。
これは「配信済み」であって「処理完了」ではない。
コンシューマーがクラッシュすると、オフセットは進んでいるが処理は失敗している。

**新設計: ack ベースのオフセット更新**

```
POST /events/{event_id}/ack?consumer_id=X
  1. acked_at を更新 (req #25)
  2. consumer_id が指定されていれば write_offset(_cfg.offsets_dir, consumer_id, seq) を呼ぶ
  3. 200 {"event_id": "...", "acked": true}
```

SSE ジェネレーター内の `write_offset()` は:
- **Option A (シンプル)**: mid-stream checkpoint を廃止。オフセットは ack のみで更新。
- **Option B (互換性維持)**: mid-stream checkpoint は「at-most-once」モード用に残し、ack ベースを「at-least-once」モードとして提供。

推奨: **Option A** (シンプル、語義が明確)。ただし UNK-01 解決後に確定。

## Implementation Steps

※ req #25 UNK-01 解決後に確定

1. `app.py` の `/ack` エンドポイントに `consumer_id: str = Query(default="")` を追加
2. `consumer_id` が指定されている場合に `write_offset()` を呼ぶ (asyncio.to_thread 経由)
3. `subscribe()` SSE ジェネレーター内の mid-stream checkpoint (`write_offset`) を除去 (Option A の場合)
4. テスト: ack → オフセット更新確認、クラッシュ後に再接続で正しい seq から再配信されることを確認
