# Plan: Improve /memory CLI for source-type and context-boundary inspection

Source: `requires/20260625_98_require.md`

---

## Goal

`/memory list` と `/memory search` に source type および project/repo/branch フィルタを追加し、人間が特定種別の memory を効率的に検査・修正できるようにする。

---

## Scope

**In-Scope**
- `/memory list [--source=RULE|DECISION|FAILURE|CONVERSATION]` フィルタ追加
- `/memory list [--branch=<branch>]`, `/memory search [--branch=<branch>]` フィルタ追加
- `/memory list --pinned` クイックビュー（既存の pin 機能を活用）
- `docs/05_agent_07_cli-and-commands.md` の更新

**Out-of-Scope**
- UI 全体の再設計
- memory data model 変更

---

## Assumptions

1. `/memory list` と `/memory search` のコマンドハンドラが `scripts/agent/commands/` 以下に存在。
2. source type フィルタは SQLite の `WHERE source_type = ?` で実装可能。
3. branch フィルタは `WHERE branch = ? OR branch = ''`（93 番の global rule 透過ルールと整合）。
