## Design

### `scripts/shared/tool_registry.py` docstring rewrite

Replace the module docstring's "Drift detection" section:
```
Drift detection:
  - compare_registry_vs_config(): validates config tool_names against registry
  - compare_registry_vs_live(): validates live /v1/tools responses against registry
  - compare_config_vs_live(): validates config tool_names against live responses
```
with:
```
Drift detection (see shared.tool_routing_validation â€” the canonical validation module):
  - validate_routing_against_config(): validates config tool_names against this registry
  - validate_routing_against_live(): validates live /v1/tools responses against this registry
  - validate_all_routing(): runs both of the above and aggregates results
```
Also add, near the top of the docstring: an explicit line stating ToolRegistry owns *tool-to-server ownership and routing* only, and is not a schema/description registry â€” LLM-visible tool schemas are exported by each server's own `tools.py` (`TOOL_LIST`), documented in `docs/04_mcp_07_tool_schema_export_policy.md`.
