# Plan: Add Fallback Title When Session Title Generation Fails

Derived from `01_plan.md` using requirement `requires/20260618_31_require.md`.

---

## Goal

Sessions always get a useful title ŌĆö if LLM generation fails, a deterministic fallback based on the first user input is used instead.

---

## Scope

**In scope:**
- `scripts/agent/commands/cmd_session.py` ŌĆö add fallback in `_generate_session_title()`
- `scripts/agent/services/session_title.py` ŌĆö optionally return a failure indicator for fallback
- `docs/05_agent_04_state-and-persistence.md` ŌĆö document fallback behavior

**Out of scope:**
- `session.start()` default title
- Session list UX changes beyond title content

---

## Assumptions

1. `first_input` passed to `_generate_session_title()` is always available and non-empty when the method is called.
2. Truncating `first_input` to `SESSION_TITLE_MAX_CHARS` produces a readable fallback.
3. The fallback should be distinguishable from LLM-generated titles (e.g. prefix with `[Fallback]` or similar).

## Implementation Steps

1. **Add fallback title in `_generate_session_title()`**
   - In the `except SessionTitleGenerationError` block, set fallback from `first_input`
   - Truncate to `SESSION_TITLE_MAX_CHARS`, append `...` if truncated
   - Call `ctx.session.set_title(fallback_title)`
   - Criterion: session always has a non-empty title after first input

2. **Document fallback behavior**
   - Update `docs/05_agent_04_state-and-persistence.md`
   - Criterion: behavior is predictable

3. **Lint, type, arch validation**
   - `ruff check scripts/` ŌĆö 0 errors
   - `mypy scripts/` ŌĆö no new errors
   - `lint-imports` ŌĆö 0 violations
