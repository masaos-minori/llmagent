## Goal

後方互換のためだけに残っている機能・設定・API エイリアスのうち、ユーザーが今回のセッションで承認した範囲(低リスク項目全件 + 個別に承認した高影響項目4件)を廃止し、コード上の `deprecated`/`backward compat` 表記と実態(使用状況)の乖離を解消する。

## Scope

- **In-Scope**
  - Phase 1(低リスク・即時): `preflight_deny_reason()` 削除、`probe_mcp_health()` 削除、`_WORKFLOWS_DIR` 内部エイリアス削除
  - Phase 2(呼び出し元書き換え・即時、ユーザー承認により対象化): `TurnResult.success` プロパティ削除、`scripts/rag/types.py` 経由の `shared.types` 再エクスポート削除
  - Phase 3(破壊的変更・即時、ユーザー承認により対象化): `use_tool_dag=false` のレガシー実行経路削除、`POST /ack`(eventbus 互換エイリアス)削除
  - Phase 4(DB スキーマ変更・本番データ移行なし、ユーザー承認により対象化): eventbus `events.retry_count` カラム削除
- **Out-of-Scope(今回は対象外、フォローアップ課題として記録)**
  - `github.allowed_repos_mode="fail_open"`(local/dev 限定の意図的な環境依存動作。今回のヒアリングで個別の承認を得ていないため対象外)
  - `scripts/eventbus/config.py` の `poll_interval_ms` / `offset_checkpoint_interval` no-op フィールド(eventbus 設定スキーマの変更を伴うため今回は対象外)
  - `tests/test_mcp_tool_schema_exports.py` の `_MCP_TOOLS` 旧名禁止テスト — これは compat shim ではなく回帰防止用のテストであり、削除対象ではない。維持する。

## Assumptions

1. `config/agent.toml` および eventbus 側 TOML の再読み込みはデプロイ手順(`skills/deploy/SKILL.md`)に従って行われ、今回のプラン自体には新規デプロイ手順の追加は不要(既存ファイルの内容変更のみ)。
2. 実行環境の `sqlite3.sqlite_version` は 3.46.1(開発環境で確認済み)であり、`ALTER TABLE events DROP COLUMN retry_count` は動作する(DROP COLUMN サポートは SQLite 3.35.0 以降)。本番環境(`/opt/llm/venv/`, Python 3.13)でも同等以上のバージョンであることをデプロイ前に確認する。
3. `POST /ack` を利用する外部クライアントは存在しない、または本プロジェクト管理下のクライアントのみである。ユーザーが「今回のサイクルで直接削除」を選択済みのため、この前提が誤りであった場合は別途 Issue で対応する。
4. `github.allowed_repos_mode="fail_open"` と eventbus no-op フィールド(#6, #8)は、今回「低リスク項目優先」の回答に対する明示的な上書き指示がなかったため、スコープ外として扱ってよい。
