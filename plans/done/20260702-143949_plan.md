## 1. Goal

Bring `docs/04_mcp_06_configuration_and_operations.md` into alignment with the current HTTP-only `McpServerConfig` dataclass by removing all stdio-specific fields and examples, and correcting the fields table and defaults table to match the live implementation.

---

## 2. Scope

**In-Scope**:
- Remove stdio fields (`cmd`, `idle_timeout_sec`, `working_dir`, `env`) from the McpServerConfig Fields table in `docs/04_mcp_06_configuration_and_operations.md`
- Remove `TransportType.STDIO` from the table
- Remove `startup_mode = "ondemand"` from the table (not present in `StartupMode` enum)
- Remove `healthcheck_mode = "process"` and `"ping_tool"` from the table (not present in `HealthcheckMode` enum)
- Remove the "stdio Subprocess Operation (ondemand)" section and its TOML example block
- Remove `stdio timeout` (60.0 sec) from the Major Default Values table
- Add `call_timeout_sec` (60.0 sec) to the Major Default Values table
- Remove the validation rule `transport="stdio" → cmd must be non-empty`
- Remove the validation rule `startup_mode="subprocess" only valid with transport="http" (ValueError otherwise)` (now the only transport is HTTP, so this constraint is vacuously satisfied)
- Fix the Validation rules block to match current `_validate_cross_fields` (only HTTP + url check)
- Remove `cmd` from the New MCP Server Addition Checklist (`[mcp_servers.<key>]` entry description)
- Update the `Startup failure checkpoints` table entry referencing `mcp_servers.toml` to remove stdio combo check
- Remove `cmd = []` from `config/mcp_servers.toml` — all 9 server entries still carry this legacy field
- Remove `cmd = []` (or `cmd` key) from test fixtures in `tests/test_mcp_config.py` (3 test methods pass `"cmd": []` in raw dicts — it is silently ignored by `_build_single_server`, but the fixture should not mislead readers)

**Out-of-Scope**:
- Changes to `scripts/shared/mcp_config.py` — the dataclass already has no `cmd`, `idle_timeout_sec`, `working_dir`, or `env` fields; no code change is needed
- Other docs files (routing.md, 04_mcp_03, etc.)
- New feature work or enum additions

- **Risk**: Test fixtures with `cmd` removed cause a test to assert that `cmd` is *not* accepted by `_build_single_server`. **Likelihood**: Very low — the current behavior silently ignores unknown keys. **Mitigation**: Review test assertions after edit; no assertion currently checks for `cmd` rejection.
