## Goal

Clarify whether the Event Bus bind host is controlled by TOML configuration or by the uvicorn start command, and remove the dead `host` config from the TOML example.

## Scope

**In-Scope**:
- Remove `host = "127.0.0.1"` from TOML config example in deployment docs (dead config, not loaded by EventBusConfig)
- Document host binding as a deployment/runtime process configuration (uvicorn CLI)
- Add startup log for effective bind address
- Add config validation tests

**Out-of-Scope**:
- Adding authentication
- Replacing uvicorn deployment

## Assumptions

1. The `host` field is NOT loaded by `EventBusConfig` — confirmed in config.py (no `host` field defined)
2. The TOML example with `host = "127.0.0.1"` is dead config and should be removed
3. Host binding is controlled via uvicorn CLI arguments (`--host 127.0.0.1`)
