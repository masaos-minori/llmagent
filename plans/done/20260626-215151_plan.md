## Goal

Harden SQLite connection and transaction management for MDQ by enabling WAL mode, setting busy_timeout, using explicit write transactions, preferring atomic replace per document during indexing, adding FTS consistency check, adding FTS rebuild operation, and ensuring connections are closed reliably.

## Scope

**In-Scope**:
- Enable WAL mode
- Set busy_timeout
- Use explicit write transactions
- Prefer atomic replace per document during indexing
- Add FTS consistency check
- Add FTS rebuild operation if needed
- Ensure connections are closed reliably

**Out-of-Scope**:
- Adding new tools or features
- Changes to other MCP servers' SQLite configuration

## Assumptions

1. WAL mode is supported by SQLite version used (3.7+)
2. busy_timeout = 5000ms (5 seconds) is reasonable for production use
3. FTS consistency check compares chunks row count with chunks_fts row count
4. FTS rebuild uses `INSERT INTO chunks_fts(chunks_fts, chunks_fts) VALUES ('rebuild')`

## Affected Areas & Tool Evidence

- **Affected Files**:
  - `scripts/mcp/mdq/service.py` — add WAL mode, busy_timeout, explicit transactions, FTS consistency/rebuild
  - `scripts/mcp/mdq/indexer.py` — update indexing to use atomic replace and explicit transactions
  - `scripts/mcp/mdq/models.py` — add FtsConsistencyCheck model for rebuild operation

- **Blast Radius**: Medium — changes SQLite connection behavior. Existing queries will still work; concurrent operations will be safer with WAL mode.

## Implementation Steps

1. **Phase 1: Enable WAL mode and busy_timeout**
   - Add `PRAGMA journal_mode=WAL` after connection creation in _get_db_connection()
   - Add `PRAGMA busy_timeout = 5000` after connection creation

2. **Phase 2: Use explicit write transactions**
   - Add BEGIN/COMMIT/ROLLBACK around write operations in service.py and indexer.py
   - Replace implicit commit() calls with explicit transaction control

3. **Phase 3: Implement atomic replace per document**
   - In indexer.py _index_single_file(), use INSERT OR REPLACE for documents table
   - Wrap DELETE + INSERT in single transaction

4. **Phase 4: Add FTS consistency check**
   - Add method to compare chunks row count with chunks_fts row count
   - Return ConsistencyResult with mismatch flag and counts

5. **Phase 5: Add FTS rebuild operation**
   - Add method to rebuild FTS index using `INSERT INTO chunks_fts(chunks_fts, chunks_fts) VALUES ('rebuild')`
   - Add to tools.py as new tool or admin command

6. **Phase 6: Ensure reliable connection closing**
   - Add try/finally with conn.close() in _migrate_from_legacy()
   - Verify all code paths close connections
