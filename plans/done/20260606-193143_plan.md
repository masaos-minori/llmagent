# Git / GitHub 危険操作ガードと artifact flow 整備 — Work Plan

Source instruction: `instructions/20260606_03_instruction.md`

---

## Goal

Git/GitHub 操作に対してオペレーションポリシーと安全ガードを実装し、Worker が commit まで、Orchestrator が push/PR/merge を担当する役割分離を実現する。

---

## Scope

**In:**
- Worker の Git 操作制限（commit まで）
- Orchestrator の Push/PR/Merge 権限管理
- protected branch、force push 禁止、allowed_repositories 設定
- artifact.updated イベント定義

**Out:**
- Git LFS 管理（後続タスク）
- clone/cache/cleanup の詳細実装（別プランで扱う）

---

## Assumptions

1. `orchestrator/gitops/` ディレクトリは未存在（新規作成）。
2. 現在の Git MCP (`mcp/git/server.py`) は `read_only` フラグで書き込み制御可能。
3. GitHub MCP (`mcp/github/server.py`) は既に bearer token 認証で動作中。

## Implementation Steps

1. `orchestrator.py` を読み、Worker 生成・ツール制限の仕組みを確認する
2. `agent/orchestrator/gitops/policy.py` を作成し、Push/PR/Merge をブロックする policy を定義する
3. Worker の allowed_tools に commit 以下の Git ツールのみ追加するロジックを実装する
4. artifact.updated イベント型を `shared/events.py` または相当モジュールに定義する
5. GitHub MCP に `allowed_repositories` の強制チェックを追加する
6. バリデーション実行 + instructions/done/ へ移動
