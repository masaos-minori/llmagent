# Plan: Separate deprecated Event Bus endpoints and config fields from active documentation

## Goal
3つのEvent Bus docファイルで、deprecated/compatibility APIと設定フィールドをactive APIから視覚的・構造的に明確に分離し、新クライアントが誤って deprecated パスを利用しないようにする。

## Scope
**In-Scope**:
- `docs/06_eventbus_02_http_api_and_runtime.md` — `POST /ack` セクションの deprecation 表示を強化し、新クライアント向けの正規パスへの誘導を明示
- `docs/06_eventbus_05_configuration_deploy_and_operations.md` — deprecated config fields セクションが TOML example に混在していないか確認し、TOML example から deprecated フィールドを除外
- `docs/06_eventbus_06_reference_api.md` — deprecated フィールドの `EventBusConfig` クラス定義表示が active フィールドと混在しないか確認し、クラス定義とdeprecated fields を明確に分離

**Out-of-Scope**:
- 実装コード (`scripts/eventbus/`) の変更
- テストの変更
- 新しいエンドポイントや設定フィールドの追加
- `docs/06_eventbus_01_system-overview.md` など他ドキュメントの変更

## Assumptions
- `docs/06_eventbus_05_configuration_deploy_and_operations.md` のTOML exampleには `poll_interval_ms` / `offset_checkpoint_interval` が含まれていない（現状確認済み: 含まれていない）
- `docs/06_eventbus_06_reference_api.md` の `EventBusConfig` クラス定義には deprecated フィールドが active フィールドと同一ブロックに記載されていない（現状確認済み: 別テーブルとして分離済み）
- `docs/06_eventbus_02_http_api_and_runtime.md` の `POST /ack` セクションにはすでに deprecated 注記が存在する（現状確認済み: ブロッククォートで記載済み）
- 3ドキュメントはすでに部分的に要件を満たしているため、差分作業は補強と細部の一貫性確保が中心
