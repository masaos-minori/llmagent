## Goal
`GET /health` が `status="ok"` 時のみ HTTP 200、`status="degraded"` 時は HTTP 503 を返すことを確認し、旧来の「常に HTTP 200」記述を docs から排除する。

## Scope
**In-Scope**:
- `scripts/eventbus/app.py` の health エンドポイントが 200/503 を正しく返すことの確認
- `docs/06_eventbus_02_http_api_and_runtime.md` の health 記述の一貫性確認
- `docs/06_eventbus_05_configuration_deploy_and_operations.md` の health 記述確認
- `docs/06_eventbus_90_inconsistencies_and_known_issues.md` の `/health Degraded State Returns HTTP 503` 項目を Docs-Only → Resolved に移動（コード・docs が既に整合している場合）
- `tests/test_eventbus_health.py` が 503 ケースをカバーしているかの確認

**Out-of-Scope**:
- `/health` への認証追加
- health レスポンス JSON スキーマの変更

## Assumptions
1. `app.py:L159`: `status_code = 200 if overall == "ok" else 503` — 実装は正しい（確認済み）
2. `06_eventbus_02:L147`: "HTTP 200 for `ok`, HTTP 503 for `degraded`/`unhealthy`" — 既に正しい
3. `06_eventbus_05:L94`: `503 | degraded` — 既に正しい
