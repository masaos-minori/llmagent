## Goal

Remove `workflow_require_approval` from `AgentConfig` and `require_approval` from `WorkflowEngine` so that the approval gate between execute and verify stages is always active for every workflow task.

## Scope

**In**:
- Remove `AgentConfig.workflow_require_approval: bool = False` from `scripts/agent/config_dataclasses.py` (line 530)
- Remove `workflow_require_approval` read from `scripts/agent/config_builders.py`
- Remove `require_approval: bool = False` parameter from `WorkflowEngine.__init__()` in `scripts/agent/workflow/workflow_engine.py` (line 99)
- Remove `self._require_approval = require_approval` from `WorkflowEngine.__init__()` (line 104)
- Remove `if self._require_approval:` guard from `WorkflowEngine.run()` (line 129); call `_gate_approval(task)` unconditionally
- Remove `require_approval=self._ctx.cfg.workflow_require_approval` from `Orchestrator._handle_workflow_engine()` (orchestrator.py:233)
- Reject config dicts that still contain `workflow_require_approval` key (raise `ConfigLoadError`)
- Remove tests that assume `require_approval=False` (approval disabled); add tests that verify approval is always requested
- Update `test_config_reload_classification.py` entries for `workflow_require_approval`

**Out**:
- Tool-level approval (`tool_approval.run_approval_checks`) — unchanged, operates independently
- Combining tool-level and workflow-level approval
- Any change to `_gate_approval()` logic itself (it already correctly creates a pending approval)
- Backward compatibility or gradual deprecation
- `workflow_mode` removal (plan 20260707-095938)

## Assumptions

1. `WorkflowEngine._gate_approval()` already implements the correct approval gate logic; only the conditional guard needs removal.
2. `Orchestrator._handle_workflow_engine()` passes `require_approval=self._ctx.cfg.workflow_require_approval` to `WorkflowEngine()`; this line is removed and `WorkflowEngine()` is constructed without the parameter.
3. After this change, every workflow task will raise `WorkflowPendingApprovalError` after execute unless already approved.
4. The `/approve` and `/reject` commands already exist and handle `WorkflowPendingApprovalError` correctly.
5. Config dict key `workflow_require_approval` should be rejected with `ConfigLoadError` after field removal.
