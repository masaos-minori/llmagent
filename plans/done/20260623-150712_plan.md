# Plan: Graceful SIGTERM Shutdown with Turn-Completion Wait and WAL Checkpoint

**Requirement:** `requires/20260623_03_require.md`

---

## Goal

Replace the immediate `raise SystemExit(0)` SIGTERM handler with a graceful shutdown that:
1. Waits for the current turn to complete (up to configurable timeout, default 10s)
2. Runs WAL checkpoint before exit
3. Falls back to forced exit after timeout

---

## Current State

**`scripts/agent/__main__.py:15-16`:**
```python
def _request_shutdown(_signum: int, _frame: object) -> None:
    raise SystemExit(0)

signal.signal(signal.SIGTERM, _request_shutdown)
```

**Behavior:** `SystemExit(0)` raised immediately from signal handler. If a turn is active (LLM call, tool execution), it is interrupted mid-execution. The `finally` block in `_run_repl_loop()` does run (Python semantics), calling `_close_resources()`, but no WAL checkpoint is run there.

**REPL already has `shutdown_requested`** (`ctx.conv.shutdown_requested`) checked in `_should_exit()` — but this is checked only between turns, not during them.

---

## Scope

**In:**
- `scripts/agent/__main__.py` — replace `signal.signal()` with `asyncio`-aware registration
- `scripts/agent/repl.py` — add `_graceful_shutdown()` coroutine; add turn-active tracking; add WAL checkpoint to `_close_resources()`; add signal handler setup in `run()`
- `scripts/agent/context.py` — (no change needed — `shutdown_requested` already exists)

**Out:**
- `os._exit()` unless graceful timeout expires (per requirement constraint)
- Persisting signal state across restarts
- Changing tool execution semantics

---

## Assumptions

1. `loop.add_signal_handler()` (asyncio-native) is the correct pattern for asyncio signal handling — `signal.signal()` is thread-unsafe with asyncio.
2. The turn duration is bounded by `orchestrator.handle_turn()` which includes all LLM + tool execution. The 10s timeout covers most normal turns; long-running operations may be aborted.
3. WAL checkpoint: `checkpoint_wal(db, "TRUNCATE")` on `session.sqlite` is sufficient. The requirement says "safe database checkpoint/commit" — we run it on all databases (session, RAG, workflow) via existing `DbMaintenanceService`.
4. `_close_resources()` in `repl.py:297-304` already calls `svc.lifecycle.shutdown_all()` and `svc.http.aclose()` — WAL checkpoint runs before these.
5. The `input()` in `_read_input()` blocks the executor thread — SIGTERM while waiting for user input won't interrupt `input()` itself, but the asyncio event is set and the REPL loop checks it before the next dispatch.

## Design

### Change 1: `__main__.py` — remove thread-unsafe signal handler

**Before:**
```python
def _request_shutdown(_signum: int, _frame: object) -> None:
    raise SystemExit(0)

signal.signal(signal.SIGTERM, _request_shutdown)

if __name__ == "__main__":
    asyncio.run(AgentREPL().run())
```

**After:**
```python
if __name__ == "__main__":
    asyncio.run(AgentREPL().run())
```

Move signal registration into `AgentREPL.run()` where the event loop is running.

### Change 2: `repl.py` — turn tracking + graceful shutdown + WAL checkpoint

**Add to `AgentREPL.__init__()`:**
```python
self._turn_active: bool = False
self._shutdown_event: asyncio.Event | None = None
```

**Modify `run()` — add signal handler after loop starts:**
```python
async def run(self) -> None:
    loop = asyncio.get_running_loop()
    self._shutdown_event = asyncio.Event()

    def _sigterm_handler() -> None:
        self._ctx.conv.shutdown_requested = True
        if self._shutdown_event is not None:
            self._shutdown_event.set()
        logger.info("SIGTERM received; graceful shutdown initiated")

    try:
        loop.add_signal_handler(signal.SIGTERM, _sigterm_handler)
    except NotImplementedError:
        # Windows / some platforms don't support add_signal_handler
        import signal as _signal
        _signal.signal(_signal.SIGTERM, lambda *_: _sigterm_handler())

    from agent.startup import StartupOrchestrator  # noqa: PLC0415
    startup = StartupOrchestrator(self._ctx, self._view)
    self._cmds, self._orchestrator = await startup.run()
    await self._run_repl_loop()
```

**Modify `_repl_loop()` — wrap dispatch with turn tracking:**
```python
async def _repl_loop(self) -> None:
    ctx = self._ctx
    ...
    _GRACEFUL_TIMEOUT: float = 10.0
    while True:
        line = await self._read_input(loop)
        if line is None:
            break
        if not line:
            continue
        if self._should_exit(line, ctx):
            break
        self._turn_active = True
        try:
            await asyncio.wait_for(
                self._dispatch_line(line, ctx),
                timeout=_GRACEFUL_TIMEOUT if ctx.conv.shutdown_requested else None,
            )
        except asyncio.TimeoutError:
            logger.warning(
                "Graceful shutdown: turn did not complete within %.1fs; forcing exit",
                _GRACEFUL_TIMEOUT,
            )
            break
        finally:
            self._turn_active = False
        if ctx.conv.shutdown_requested:
            break
```

**Modify `_close_resources()` — add WAL checkpoint:**
```python
async def _close_resources(self) -> None:
    """Close all session resources. Called in the run() finally block."""
    self._view.write_history()
    svc = self._ctx.services
    if svc is not None:
        # WAL checkpoint before closing connections
        try:
            from db.helper import SQLiteHelper  # noqa: PLC0415
            with SQLiteHelper("session").open(write_mode=True) as db:
                db.checkpoint("TRUNCATE")
            logger.info("WAL checkpoint completed on shutdown")
        except Exception as e:  # noqa: BLE001 — non-critical; checkpoint failure must not block shutdown
            logger.warning("WAL checkpoint failed on shutdown: %s", e)
        await svc.lifecycle.shutdown_all()
        await svc.http.aclose()
```

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `asyncio.wait_for()` with `timeout=None` (no shutdown requested) behaves identically to no timeout | Confirmed — `asyncio.wait_for(coro, timeout=None)` is equivalent to `await coro` | Low |
| WAL checkpoint in `_close_resources()` fails if `session.sqlite` is in use | Low — checkpoint is read-safe; only fails if DB is corrupted. `except Exception` guard prevents blocking shutdown | Low |
| `loop.add_signal_handler()` raises `NotImplementedError` on Windows | Handled via fallback to `signal.signal()` in the `except NotImplementedError` block | Low |
| Cancelling a tool call mid-execution via `asyncio.TimeoutError` leaves orphaned subprocesses | Low — `_close_resources()` calls `svc.lifecycle.shutdown_all()` which cleans up subprocesses | Moderate — inherent in forced shutdown |
