## Goal

Add explicit tests that verify the invariant: partial LLM completions are never appended to `ctx.conv.history` or persisted to the `messages` table, and that all three failure paths (in-stream partial, pre-stream, tool-continuation) behave correctly.

## Scope

**In**: Add tests for all three `LLMTransportError` paths. Verify `persist_as_assistant=False` for partial completions. Confirm `messages` table is not contaminated. Confirm `session_diagnostics` and `tool_result_store` are written consistently.

**Out**: Changing SSE parser behavior, changing LLM retry policy, documentation-only updates.

## Assumptions

1. `handle_partial_completion()` does NOT append to `ctx.conv.history` — verified in `llm_transport_errors.py`.
2. `_handle_llm_error()` in `llm_turn_runner.py` returns `TurnResult(persist_as_assistant=False)` — verified at line 141.
3. `orchestrator.py` only calls `ctx.session.save("assistant", ...)` when `result.persist_as_assistant=True`.
4. The invariant is currently correct in production code; the gap is lack of tests to lock it down.
5. The three distinct paths are: (a) in-stream failure with `partial_text != ""`, (b) pre-stream failure `partial_text == ""`, (c) tool-continuation failure.

## Design

**Three test paths**:

1. **In-stream partial** (`partial_text != ""`):
   - Mock `LLMTurnRunner.run()` to raise/return with `LLMTransportError(partial_text="partial...")`.
   - Assert: `ctx.conv.history` not extended with partial text.
   - Assert: `messages` table not inserted with partial text.
   - Assert: `session_diagnostics` has a `llm_transport_error` record.
   - Assert: `tool_result_store` has a `llm_partial_completion` record.
   - Assert: `stat_partial_completions` incremented.

2. **Pre-stream** (`partial_text == ""`):
   - Mock LLM to fail before any output.
   - Assert: last user message is not in `messages` (or is removed from in-memory history).
   - Assert: no assistant message persisted.

3. **Tool-continuation** (LLM fails during tool result submission):
   - Mock LLM to fail while processing tool results.
   - Assert: synthetic tool error is present in next LLM call context.
   - Assert: conversation can continue.
