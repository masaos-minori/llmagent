# Plan: Add indexes for DLQ promotion and DLQ listing queries

Source: `requires/20260625_48_require.md`

---

## Goal

EventBus の `schema.sql` に DLQ 関連クエリ（`dlq_at IS NOT NULL` フィルタ、retry_threshold フィルタ）を効率化する索引を追加する。

---

## Scope

**In-Scope**
- `scripts/eventbus/schema.sql`: `dlq_at` カラム・`delivery_failure_count` カラムへの index 追加
- `scripts/eventbus/db.py`: DLQ 関連クエリのプランを確認
- schema 初期化パスの更新（migration/init）
- テスト追加

**Out-of-Scope**
- DLQ の business logic 変更
- 既存 index の変更

---

## Assumptions

1. `schema.sql` に既存 index は `idx_events_topic` と `idx_events_seq` のみ（確認済み）。
2. DLQ 関連クエリは `WHERE dlq_at IS NOT NULL` と `WHERE delivery_failure_count >= threshold` を使用。
3. `db.py` の `fetch_dlq()` と DLQ promotion クエリが full scan になっている可能性がある。

## Implementation Steps

- [ ] 1. `schema.sql` に以下を追加:
  ```sql
  CREATE INDEX IF NOT EXISTS idx_events_dlq_at ON events(dlq_at);
  CREATE INDEX IF NOT EXISTS idx_events_failure_count ON events(delivery_failure_count);
  ```
- [ ] 2. `db.py` の DLQ クエリで EXPLAIN QUERY PLAN を実行し改善を確認
- [ ] 3. docs 更新
