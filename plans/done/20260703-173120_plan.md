## Goal

Fix `load_plugins()` to catch `ValueError` from `register_tool()` annotation validation, so that invalid plugin tools fail gracefully under both strict and non-strict plugin load modes, consistent with other plugin load errors.

## Scope

**In-Scope:**
- Add `ValueError` to the exception catch tuple in `load_plugins()` in `scripts/shared/plugin_auto_discover.py`
- Add tests confirming that a plugin file whose `@register_tool` decorator raises `ValueError` (missing/wrong annotation) is recorded as a `PluginFailure` entry in `PluginLoadResult.failed` rather than propagating to the caller
- Add a test confirming strict mode aggregates `ValueError`-based failures into `PluginLoadError`
- Update `docs/05_agent_11_extension-points.md` to explicitly document the registration-time fail-fast behavior and the non-strict vs. strict recovery path

**Out-of-Scope:**
- Introducing a new `PluginToolInvoker` class (scope of a separate task â€” `tool_executor.py` runtime validation is defensive and stays)
- Changing the `ToolExecutor.execute()` runtime guard (len != 2, isinstance checks) â€” these remain as defensive fallback
- Changing `register_tool()` exception type from `ValueError` to `RuntimeError`
- Changing plugin command registration
- Adding backward-compatible shims

## Assumptions

1. `load_plugins()` exception catch at `scripts/shared/plugin_auto_discover.py:62` is the single location that must be updated; no other caller silently swallows or depends on `ValueError` propagating.
2. Raising `ValueError` from `register_tool()` during plugin import is logically equivalent to an `ImportError` for the purposes of `load_plugins()` â€” both represent a failure to load the plugin.
3. `_tools[name] = (fn, ...)` in `register_tool` is only executed after both annotation checks pass, so the invalid tool is never inserted into the registry even under non-strict mode.
4. `TestSignatureWarning` tests in `tests/test_plugin_registry.py` call `register_tool` directly (not via `load_plugins`); new tests must use the `tmp_path`/file-writing pattern to test the `load_plugins` integration path.
5. No existing test asserts that a `ValueError` propagates from `load_plugins()` (which would break after this fix).

## Design

### Problem

`load_plugins()` in `plugin_auto_discover.py:62` catches:
```python
except (ImportError, SyntaxError, AttributeError, RuntimeError) as e:
```

`register_tool()` raises `ValueError` when annotation validation fails. `ValueError` is not in the catch tuple, so it propagates to the caller as an unhandled exception â€” bypassing the `PluginFailure` recording and `strict_mode` aggregation entirely.

### Fix

Add `ValueError` to the exception tuple:
```python
except (ImportError, SyntaxError, AttributeError, RuntimeError, ValueError) as e:
```

This makes `ValueError` from annotation contract violations behave identically to other plugin load errors:
- **Non-strict mode** (`strict_mode=False`): records a `PluginFailure` entry, logs `[plugin] skipped: <filename> (ValueError)`, continues loading other plugins.
- **Strict mode** (`strict_mode=True`): includes the failure in the aggregated `PluginLoadError` message.

No change to `register_tool()` itself â€” the existing `ValueError` messages are clear and correctly block tool insertion.

### Test integration pattern

Use `tmp_path` to write a plugin `.py` file that registers a tool with an invalid annotation, then call `load_plugins(tmp_path)` and assert the result.

```python
# tests/test_plugin_registry.py â€” TestPluginToolContractLoadIntegration

def test_missing_annotation_recorded_as_failure(tmp_path):
    (tmp_path / "bad.py").write_text(
        "from shared.plugin_registry import register_tool\n"
        "@register_tool('bad')\n"
        "async def handler(args): return 'x', False\n"
    )
    result = plugin_registry.load_plugins(tmp_path)
    assert result.loaded_count == 0
    assert len(result.failed) == 1
    assert "missing return type annotation" in result.failed[0].error

def test_wrong_annotation_recorded_as_failure(tmp_path):
    (tmp_path / "wrong.py").write_text(
        "from shared.plugin_registry import register_tool\n"
        "@register_tool('wrong')\n"
        "async def handler(args: dict) -> str: return 'x'\n"
    )
    result = plugin_registry.load_plugins(tmp_path)
    assert result.loaded_count == 0
    assert len(result.failed) == 1
    assert "expected return type" in result.failed[0].error

def test_annotation_failure_strict_mode_raises(tmp_path):
    (tmp_path / "strict_bad.py").write_text(
        "from shared.plugin_registry import register_tool\n"
        "@register_tool('strict_bad')\n"
        "async def handler(args: dict) -> str: return 'x'\n"
    )
    with pytest.raises(plugin_registry.PluginLoadError):
        plugin_registry.load_plugins(tmp_path, strict_mode=True)
```
