## Goal

Strengthen the RAG quality regression test suite with deterministic assertions for RRF scoring,
semantic cache hit/miss behavior, and top-N retrieval, while preserving all existing embed-failure
regression tests.

## Scope

**In**: `tests/test_rag_quality_regression.py` — add new test cases:
  - RRF score value assertions against fixed-vector known hits (via `_search_all_queries` patch)
  - Semantic cache hit path: second identical query returns cached result (verify `cache.lookup` returns non-None)
  - Top-N retrieval count assertions: `len(result.reranked) == min(rag_top_k, available_hits)`
  - Deterministic ordering assertions: highest-RRF-score hit is first in `result.merged`

**In**: `docs/03_rag_03_query_pipeline.md` — update section 8.1 table to list new tests

**Out**: `scripts/rag/pipeline.py` — no changes  
**Out**: `scripts/rag/cache.py` — no changes  
**Out**: `scripts/rag/repository.py` — no changes  
**Out**: DB schema — no changes  
**Out**: Any other test file — no changes

## Assumptions

1. The existing `_make_rag_cfg()` and `_make_pipeline()` helpers in the test file are sufficient
   for constructing pipelines for new test cases.
2. Patching `rag.stages.search._search_all_queries` with `AsyncMock` is the correct injection point
   for bypassing embed/DB calls in deterministic tests (confirmed by existing `test_rrf_vs_no_rrf_fusion_mode`).
3. `SemanticCache` can be tested directly (unit-level) without constructing a full `RagPipeline`,
   as confirmed by the existing `test_semantic_cache_generation_invalidation` and
   `test_diagnostics_semantic_cache_hits` patterns.
4. `RagScorer.rrf_merge` assigns `rrf_score = Σ 1/(rrf_k + rank)` — with `rrf_k=60` and a single
   result list of 2 hits, expected scores are `1/61 ≈ 0.01639` (rank 0) and `1/62 ≈ 0.01613` (rank 1).
5. No in-memory SQLite DB setup is needed for the new tests; `MagicMock()` for `db` is sufficient
   because all DB access is bypassed by patching `_search_all_queries`.
6. `conftest.py` has no RAG-specific fixtures; all fixtures are test-local (confirmed by reading
   `tests/conftest.py` — only `sys.path` and `_VEC_AVAILABLE` are defined there).
7. The `rag_top_k=3` config setting in `_make_rag_cfg()` controls the `reranked` slice;
   when `use_rerank=False`, `RerankStage` truncates `merged` to `rag_top_k` items.

## Implementation Steps

### Phase 1: Add deterministic RRF scoring assertions

- [ ] 1. In `TestRagQualityRegression`, add `test_rrf_score_values_with_known_hits`:
  - Patch `rag.stages.search._search_all_queries` to return two `RawHit` lists
    (list A: [chunk_id=1], list B: [chunk_id=2, chunk_id=1]).
  - Run pipeline with `use_rrf=True, rrf_k=60`.
  - Assert `result.merged[0].chunk_id == 1` (chunk_id=1 appears in both lists → higher RRF score).
  - Assert `result.merged[0].rrf_score > result.merged[1].rrf_score`.
  - Assert all `rrf_score > 0.0`.
  - Commit: `test: add deterministic RRF score value assertions`

### Phase 2: Add top-N retrieval count assertion

- [ ] 2. Add `test_top_n_retrieval_count`:
  - Patch `_search_all_queries` to return a single list of 5 `RawHit` objects.
  - Config: `rag_top_k=3, use_rrf=True, use_rerank=False`.
  - Assert `len(result.reranked) == 3` (sliced to `rag_top_k`).
  - Assert `len(result.merged) == 5` (merged before slice).
  - Commit: `test: add top-N retrieval count assertion`

### Phase 3: Add semantic cache hit path test

- [ ] 3. Add `test_semantic_cache_hit_returns_cached_result`:
  - Instantiate `SemanticCache(max_size=10, threshold=0.9)` directly.
  - Call `cache.put([1.0] * 384, "", "expected_context")`.
  - Call `cache.lookup([1.0] * 384, "")` and assert result == `"expected_context"`.
  - Assert `cache.size == 1`.
  - Commit: `test: add semantic cache hit path assertion`

### Phase 4: Add semantic cache miss path test

- [ ] 4. Add `test_semantic_cache_miss_below_threshold`:
  - Instantiate `SemanticCache(max_size=10, threshold=0.99)` (high threshold).
  - Put `[1.0] * 384`, then lookup `[0.0] + [1.0] * 383` (different vector → low cosine sim).
  - Assert result is `None`.
  - Commit: `test: add semantic cache miss path assertion`

### Phase 5: Add RRF ordering invariant test

- [ ] 5. Add `test_rrf_merged_order_is_descending`:
  - Patch `_search_all_queries` with a list of 3 known hits in a single list.
  - Assert `result.merged` is sorted descending by `rrf_score`
    (i.e., `all(result.merged[i].rrf_score >= result.merged[i+1].rrf_score for i in range(len-1))`).
  - Commit: `test: assert RRF merged output is descending by rrf_score`

### Phase 6: Update documentation

- [ ] 6. In `docs/03_rag_03_query_pipeline.md` section 8.1, add 5 rows to the test table for

  - Commit: `docs: update RAG query pipeline test table for new regression tests`
