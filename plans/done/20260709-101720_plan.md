# Plan: Update Agent Configuration documentation for MCP restart-required behavior

Source requirement: `requires/20260708_39_require.md` (L-1)

## Goal

Confirm `docs/05_agent_08_configuration.md` (and related docs) no longer
describe MCP URL as hot-reloadable or `auth_token`/`startup_mode` as
deferred, no longer document `github_server_url` as active config, and
clearly state that all MCP server definition changes are restart-required,
with concrete examples.

## Scope

This requirement's documentation fixes are already fully specified across
two earlier plans in this batch:
- M-2 (`plans/20260709-100635_plan.md`) — removes "hot-reloadable" /
  "deferred" wording for MCP URL/auth_token/startup_mode in
  `docs/05_agent_08_configuration.md` and `docs/05_agent_07_cli-and-commands.md`,
  and (per the edit just made while planning L-1) now includes the 5
  concrete restart-required examples this requirement asks for.
- H-10 (`plans/20260709-100244_plan.md`) — removes the `github_server_url`
  table row from `docs/05_agent_08_configuration.md` and confirms
  `mcp_servers.github.url` is documented as canonical.

In scope for this plan: verifying that combination covers every acceptance
criterion below, and confirming "related configuration docs" / "config
samples" have no additional stale references beyond what those two plans
already scope.

## Assumptions

1. `docs/05_agent_08_configuration.md` is the only Agent-layer doc
   describing MCP hot-reload/deferred classification — verified by
   `grep -rln "hot-reloadable\|deferred" docs/05_agent_*.md` matching only
   this file (and `docs/05_agent_07_cli-and-commands.md`, already in M-2's
   scope).
2. MCP-layer docs (`docs/04_mcp_*.md`) contain no conflicting claims about
   MCP hot-reload, deferred auth/startup, or `github_server_url` — verified
   by `grep -rln "hot-reloadable\|deferred\|github_server_url"
   docs/04_mcp_*.md` (no matches). "Align Agent docs with MCP docs on config
   ownership" is therefore already satisfied once the Agent-doc fixes land;
   no MCP-doc edit is needed.
3. `docs/04_mcp_06_configuration_and_operations.md` already documents
   `[mcp_servers.*]` in `config/agent.toml` as the sole ownership model for
   server transport settings (line 28, 58) with no mention of
   `github_server_url` — confirms the "GitHub MCP endpoint configured only
   through `mcp_servers.github.url`" statement L-1 asks for is already the
   MCP-doc's existing position; only the Agent-doc side (H-10's scope) needs
   the correction.
4. `config/agent.toml` (a "config sample") is already scoped for the
   `github_server_url` line removal in H-10's plan
   (`plans/20260709-100244_plan.md` § Affected areas). No other config
   sample file references it — verified by
   `grep -rln "github_server_url" config/` (single match: `agent.toml`).

## Risks

- **Risk**: Because this plan performs no code/test changes of its own, it
  could be skipped entirely if M-2 and H-10 are implemented — but if either
  of those two plans is implemented only partially (e.g. the docs half is
  postponed while the code half lands), this requirement's acceptance
  criteria would remain unmet with no dedicated tracking.
  → **Mitigation**: The grep sweep in Implementation step 3 / Validation
  plan is a cheap, explicit final check specifically for this requirement's
  four acceptance criteria — run it even if M-2/H-10 report themselves done.
- **Risk**: "Related configuration docs" in the requirement's target files
  is vague; there could be a doc outside `docs/05_agent_*.md` and
  `docs/04_mcp_*.md` that this search missed.
  → **Mitigation**: Assumption 1's grep was repo-wide in effect for the key
  phrases ("hot-reloadable", "deferred", "github_server_url" — see
  `plans/20260709-100244_plan.md` Assumption 1's broader
  `grep -rn "github_server_url" scripts/ tests/ config/ docs/`); low
  residual risk.
