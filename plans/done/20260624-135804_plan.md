## Goal

`SQLiteHelper.__init__()` の `_default_load_vec` と db_path 分岐を、正規化済みの `resolved` 変数から導出することで、`DbTarget` enum 入力と文字列入力の挙動を完全に一致させる。

## Scope

**In**:
- `scripts/db/helper.py` — `__init__()` 内の `target` 参照を `resolved` に変更
- `tests/test_sqlite_helper.py` — `DbTarget` enum 入力を使った回帰テスト追加

**Out**:
- `SQLiteHelper` のサブクラス化
- `load_vec=` 明示オーバーライドのセマンティクス変更
- `DbTarget` enum 定義の変更

## Assumptions

1. `DbTarget` は `StrEnum` を継承するため `DbTarget.RAG == "rag"` は現在 True。よってランタイムの観察可能な挙動差は生じていないが、コードの意図が不明確であり `StrEnum` 実装依存となっている。
2. 修正は「正規化後の値を参照する」というコード品質上の修正であり、外部から観察可能な動作変更は伴わない。
3. `DbTarget.WORKFLOW` の `_default_load_vec` は False であること（vec 拡張不要）を前提とする。
