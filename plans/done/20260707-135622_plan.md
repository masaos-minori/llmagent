# Plan: M-1 — Decide and implement production behavior for subprocess startup failures

## Goal

Update `StartupOrchestrator._start_servers()` to fail-fast (re-raise) on HTTP subprocess MCP startup failures when `SecurityProfile.PRODUCTION` is active, while keeping non-fatal warning behavior for the local profile.

## Scope

**In**: `scripts/agent/startup.py` — `_start_servers()` method: add production-profile fail-fast check. H-6's rollback wrapper in `run()` handles cleanup automatically.

**Out**: No changes to `factory.py`, `http_lifecycle.py`, `context.py`, config files, or any other module.

## Assumptions

1. `SecurityProfile` is already imported in `startup.py` (confirmed: line 13).
2. H-6's `try/except` wrapper in `run()` is already merged or will be merged alongside this change; rollback shutdown fires automatically when `_start_servers()` raises.
3. The production fail-fast behavior applies to all subprocess-mode HTTP servers equally; per-server opt-out is out of scope.
4. `_servers_started` is set to `True` after `_start_servers()` returns; if `_start_servers()` raises, rollback is skipped (no servers fully started yet — the raise propagates before the flag is set).

## Unknowns

| Unknown | Evidence missing | Resolution | Blocking |
|---|---|---|---|

## Implementation Steps

1. **Add `production_mode` check in `_start_servers()`**
   - Detect `ctx.cfg.mcp.security_profile == SecurityProfile.PRODUCTION` inside the `except (OSError, RuntimeError)` block.
   - Local profile: log warning and call `self._view.write_warning(...)` — existing behavior.
   - Production profile: log error, then `raise RuntimeError(f"[FATAL] MCP subprocess {key!r} failed to start: {e}") from e`.
   - User-facing message includes `server_key` and root cause.
   - Criterion: in production, the exception propagates out of `_start_servers()`.

2. **Add tests**
   - Test: local profile — startup failure → warning written, no exception.
   - Test: production profile — startup failure → `RuntimeError` raised with server key in message.
   - Test: production profile, partial start (server A starts, server B fails) — `RuntimeError` raised; H-6 rollback can shut down server A via `run()` wrapper.
   - Criterion: `uv run pytest tests/test_startup.py -v` passes.
