# Plan: [Medium][eventbus][dlq] DLQ バックグラウンドループをリカバリー・スイープ専用に縮小

Source: `requires/20260625_30_require.md`

## Goal

req #26 でインライン昇格が実装された後、`_dlq_loop()` から閾値ベースの一次昇格ロジックを取り除き、ループをオーファンリカバリーと整合性スイープのみに限定する。

## Scope

**In-Scope**
- `scripts/eventbus/app.py` の `_dlq_loop()` からの一次昇格呼び出し除去
- `scripts/eventbus/dlq.py` への `sweep_orphans()` 関数追加 (インライン昇格を漏れた件数を返す)
- 新しいループ責務のドキュメント化 (docstring)
- スイープ専用動作のテスト追加

**Out-of-Scope**
- インライン昇格の実装 (req #26 の対象)
- バックグラウンドループの廃止
- タイムアウトスイープ (timeout は現在スキーマに未実装)

## Assumptions

1. req #26 (インライン昇格) が実装済みであること
2. req #28 (加算的スキーマ: `delivery_failure_count`) が実装済みであること
3. ループは引き続き 60 秒間隔で動作し、インライン昇格を補完するセーフティネットとして機能する

## Unknowns

## Design

**現在の `_dlq_loop()`:**
```python
n = await asyncio.to_thread(promote_to_dlq, _db, _cfg.deadletter_dir, _cfg.max_retry)
```

**変更後:**
```python
n = await asyncio.to_thread(sweep_orphans, _db, _cfg.deadletter_dir, _cfg.max_retry)
if n:
    logger.warning("dlq_loop: swept %d orphan(s) missed by inline promotion", n)
```

**`dlq.py` に `sweep_orphans()` を追加:**
```python
def sweep_orphans(
    db: sqlite3.Connection,
    deadletter_dir: str,
    max_retry: int,
) -> int:
    """Sweep events that reached retry limit but were not promoted inline.

    This is a safety-net sweep only. Under normal operation (inline promotion
    working correctly), this returns 0. Non-zero indicates a bug in the inline
    promotion path.
    """
    rows = db.execute(
```

既存の `promote_to_dlq()` は req #26 の `promote_single()` からは引き続き使用できるが、`_dlq_loop()` からは呼ばれなくなる。

## Implementation Steps

1. `dlq.py` に `sweep_orphans()` を追加 (内部で `_atomic_write` と `dlq_at` 更新を使用)
2. `app.py` の `_dlq_loop()` を `sweep_orphans()` 呼び出しに差し替え
3. `_dlq_loop()` の docstring を「一次昇格なし、スイープ専用」に更新
4. テスト: 通常動作では sweep_orphans が 0 を返すこと、インライン昇格を模擬的に失敗させた場合にスイープが拾うことを検証
