## Goal
RAG インジェスト成果物ファイル拡張子を `.txt` から `.json` へ統一する。本番コードは既にクリーンだが、テストフィクスチャ（`tests/test_ingester.py`）に `c.txt` 参照が残っているため修正する。

## Scope
**In-Scope**:
- `tests/test_ingester.py` の `c.txt` フィクスチャを `c.json` に置き換え
- `tests/test_ingestion_freshness.py` の `test.txt` / `doc.txt` をテスト用 URL として維持確認（URL パス扱いのため対象外を確認）
- 関連ドキュメント（`docs/03_rag_02_ingestion_pipeline.md` など）に `.txt` 参照がないことを確認

**Out-of-Scope**:
- 本番コード（`scripts/rag/ingestion/`）への変更（すでに `.json` のみ使用）
- JSON ペイロードスキーマの変更
- `.jsonl` への移行

## Assumptions
1. `scripts/rag/ingestion/` の全 Python ファイルに `.txt` 参照は存在しない（grep 確認済み）
2. `docs/03_rag_02_ingestion_pipeline.md`, `docs/03_rag_04_data_model_and_interfaces.md`, `docs/03_rag_05_configuration_and_operations.md` に `.txt` 参照は存在しない（grep 確認済み）
3. `tests/test_ingester.py` の `c.txt` はチャンクファイル名として使われており、`.json` に変えても動作する（JSON コンテンツを書いているため）
4. `tests/test_ingestion_freshness.py` の `.txt` は URL フィクスチャであり本タスク対象外
