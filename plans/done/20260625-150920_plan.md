## Design

**追加するセクション:**

```markdown
---

## scripts/eventbus/broker.py

### EventBroker

In-memory pub/sub broker for live event delivery. One global instance per process,
initialized in lifespan and referenced as `_broker` in `app.py`.

#### Methods

| Method | Signature | Description |
|---|---|---|
| `subscribe` | `(topics: list[str]) -> _Subscriber` | Register a new subscriber. `topics=[]` means all topics. |
| `unsubscribe` | `(sub: _Subscriber) -> None` | Remove subscriber from the registry. Idempotent. |
| `publish` | `(event: dict[str, Any]) -> int` | Fan out event to matching subscribers. Returns delivery count. |
| `subscriber_count` | `() -> int` | Number of currently active subscribers. |
| `max_queue_depth` | `() -> int` | Largest queue depth across all subscribers. |
| `slow_consumer_count` | `() -> int` | Subscribers with queue depth >= threshold (default 100). |

#### Internal type: _Subscriber

| Field | Type | Description |
|---|---|---|
| `queue` | `asyncio.Queue[dict \| None]` | Per-subscriber event queue. `None` = shutdown sentinel. |
| `topics` | `list[str]` | Topic filter. Empty = subscribe to all topics. |

#### Interactions

- **publish()** is called from `app.py:publish()` after SQLite commit (req #36)
- **subscribe()** is called from `app.py:subscribe()` before SQLite replay (req #35)
- **unsubscribe()** is called in the `finally` block of the SSE generator (req #37)
- Queue depth is exposed in `GET /health` via `max_queue_depth()` and `slow_consumer_count()` (req #42)

Cross-references: `docs/06_eventbus_02_http_api_and_runtime.md`, `docs/06_eventbus_04_dlq_offsets_and_delivery_semantics.md`
```
