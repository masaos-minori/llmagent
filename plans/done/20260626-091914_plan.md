# Plan: Change cicd-mcp workflow_allowlist from fail-open to fail-closed

Source: `requires/20260625_07_require.md`

---

## Goal

`cicd-mcp` の `workflow_allowlist` が空のとき deny-all（fail-closed）として動作するよう変更し、`repo_allowlist` と同じセキュリティポリシーに統一する。

---

## Scope

**In-Scope**
- `scripts/mcp/cicd/service_guards.py`: `_check_workflow()` の空リスト動作を fail-open → fail-closed に変更
- `scripts/mcp/cicd/server.py`: コメントの更新（"empty = allow all" → "empty = deny all"）
- `config/cicd_mcp_server.toml`: デフォルト値・コメントの更新
- `agent/repl_health.py`: `workflow_allowlist` 空の場合を `fail_closed_empty` リストに追加（現状 `fail_open_empty` 扱いの可能性）
- リグレッションテスト追加

**Out-of-Scope**
- `repo_allowlist` の動作変更（既に fail-closed）
- github-mcp の変更

---

## Assumptions

1. `service_guards.py:_check_workflow()` の `if not self._workflow_allowlist: return` が fail-open の原因（確認済み）。
2. `server.py` コメント line 9: "empty = allow all" と記載されており変更が必要。
3. `repl_health.py` の startup 検査で `workflow_allowlist` が `fail_open_empty` または未検査の可能性。
