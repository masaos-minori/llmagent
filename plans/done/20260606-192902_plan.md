## Implementation Steps

### Step 1: 優先度高 — 型安全性の強化

#### 1a. `shared/mcp_config.py` — Literal 型強化
- `transport: Literal["http", "stdio"]`
- `startup_mode: Literal["persistent", "ondemand", "subprocess"]`
- `healthcheck_mode: Literal["http", "process", "ping_tool", ""]`
- `_build_mcp_servers()` の legacy fallback に deprecation warning 追加

#### 1b. `shared/types.py` — `LLMMessage.role` の Literal 化
- `role: Literal["user", "assistant", "tool", "system"]`
- 全呼び出しサイトの型チェック通過を確認

### Step 2: 優先度中 — モジュールグローバル状態の除去

#### 2a. `shared/token_counter.py` — `_warned_unavailable` のインスタンス化
- `_warned_unavailable` を `TokenCounter` インスタンス変数または呼び出し引数に移動

#### 2b. `shared/route_resolver.py` — fallback 警告モードの追加
- `ToolRouteResolver(strict=False, warn_on_fallback=True)` を追加
- fallback 使用時に `logger.warning()` を出力
- 既存動作への影響なし（デフォルト `strict=False`）

### Step 3: 優先度低 — 調査と対応

#### 3a. `shared/git_helper.py` の例外処理
- `except Exception` → reason-coded 結果型への変更を検討

#### 3b. `shared/plugin_registry.py` の load report
- `load_plugins()` の戻り値を structured report に変更を検討

### Step 4: バリデーション実行
- `uv run ruff check scripts/`
- `uv run mypy scripts/`
- `lint-imports`
- `uv run pytest`
- `diff-cover` (≥ 90% on changed lines)

### Step 5: instructions/done/ へ移動
- `instructions/20260603_04_instruction.md` を `instructions/done/` に移動する
