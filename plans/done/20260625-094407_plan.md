# Unify diagnostics and partial-completion persistence model

**Requirement:** `requires/20260624_65_require.md`

## Goal

Select `session_diagnostics` (via `DiagnosticStore`) as the single canonical store for runtime diagnostics. Route partial completions, transport failures, and tool-loop guard hints through it at time of occurrence. Reduce `diagnostics.jsonl` to a secondary archive. Keep `tool_results` table for tool results only.

## Current Fragmentation

| Diagnostic type | Current storage | Problem |
|---|---|---|
| Partial LLM completion | `tool_results` (kind=llm_partial_completion) + in-memory counter | Stored in tool_results, not in diagnostic table |
| Session-end summary | `session_diagnostics` + `diagnostics.jsonl` (both) | Written only at session end; not queryable mid-session |
| Tool serialization events | `session_diagnostics` (kind=serialization_event) | Correctly stored; already canonical |
| Transport failures | `stat_transport_errors` counter in `ToolExecutor` (in-memory) | Lost on restart; not persisted |
| Loop guard hints | `ToolLoopGuard` in-memory only | No persistence at all |

**Decision: canonical store = `session_diagnostics` table via `DiagnosticStore`.**

Reasons:
1. Already used for serialization events (established pattern)
2. Queryable via `DiagnosticStore.fetch()` without parsing JSONL
3. Tied to `session_id` for per-session audit
4. `diagnostics.jsonl` is write-once JSONL; not suited for runtime queries
5. `tool_results` stores LLM-facing data; diagnostic events are not LLM-facing

## Scope

**In:**
- `scripts/agent/orchestrator.py` — change `_handle_partial_completion()`: after saving to `tool_results`, also save to `DiagnosticStore` with `kind="partial_completion"` at time of occurrence
- `scripts/agent/tool_runner.py` — when `TransportError` occurs and is recorded (currently only `stat_transport_errors` counter in `ToolExecutor`), save `kind="transport_failure"` to `DiagnosticStore`; note: transport errors are caught in `ToolExecutor._record_transport_error()` which is in `shared/` — need to propagate to `agent/` layer
- `scripts/agent/tool_loop_guard.py` — when guard fires (exceeds threshold), save `kind="loop_guard_hint"` to `DiagnosticStore`
- `scripts/agent/repl.py` — update `_persist_session_diagnostics()` to not duplicate partial completion data (already in diagnostic store); keep session summary for completeness
- `scripts/agent/diagnostic_store.py` — add `save_partial_completion()`, `save_transport_failure()`, `save_loop_guard_hint()` convenience methods; add `fetch_by_kind()` for targeted queries
- `scripts/agent/commands/cmd_stats.py` or related — update `/stats` to query `DiagnosticStore.fetch_by_kind("partial_completion")` for count instead of relying solely on `llm.stat_partial_completions` counter
- `tests/test_diagnostic_store.py` — add tests for new `kind` entries and `fetch_by_kind()`

**Out:**
- Full artifact lineage redesign
- Workflow phase refactor
- Removing `diagnostics.jsonl` entirely (kept as session-end summary archive)
- Changing `tool_results` table schema
- Removing `stat_partial_completions` counter from `LLMClient` (kept as in-memory fast counter)

## Assumptions

1. `DiagnosticStore` already has `save(session_id, kind, content, ...)` — the canonical write path. New convenience methods are thin wrappers.
2. Transport failures in `ToolExecutor._record_transport_error()` (in `shared/`) must be propagated to the `agent/` layer for `DiagnosticStore` access. This is done by returning `is_error=True` in `ToolCallResult`; `execute_one_tool_call()` in `tool_runner.py` sees the result and can save to diagnostics there (not in `shared/`).
3. `ToolLoopGuard` is currently in `agent/` (`tool_loop_guard.py`) — can import `DiagnosticStore` directly.
4. `ctx.diagnostics` is `DiagnosticStore | None` per `AgentContext`. All callers check for `None`.
5. The in-memory `llm.stat_partial_completions` counter is kept as a fast counter for the REPL warning (`_dispatch_line()` comparison). It's NOT removed; the canonical record is in `session_diagnostics`.
6. `diagnostics.jsonl` is kept as a session-end archive with the full summary. Its format does not change; it becomes secondary.

## Design

### `DiagnosticStore` new methods

```python
def save_partial_completion(
    self,
    session_id: int | None,
    turn: int,
    reason: str,
    content_length: int,
) -> None:
    content = orjson.dumps({
        "turn": turn,
        "reason": reason,
        "content_length": content_length,
    }).decode()
    self.save(session_id=session_id, kind="partial_completion", content=content)

def save_transport_failure(
    self,
    session_id: int | None,
    tool_name: str,
    server_key: str,
    error_msg: str,
) -> None:
    content = orjson.dumps({
        "tool_name": tool_name,
        "server_key": server_key,
        "error": error_msg,
    }).decode()
    self.save(session_id=session_id, kind="transport_failure", content=content)

def save_loop_guard_hint(
    self,
    session_id: int | None,
    reason: str,
    turn_count: int,
) -> None:
    content = orjson.dumps({
        "reason": reason,
        "turn_count": turn_count,
    }).decode()
    self.save(session_id=session_id, kind="loop_guard_hint", content=content)

def fetch_by_kind(self, session_id: int, kind: str) -> list[dict[str, Any]]:
    """Return diagnostics for a specific kind, newest first."""
    with SQLiteHelper("session").open(row_factory=True) as db:
        rows = db.fetchall(
            "SELECT id, session_id, kind, content, created_at"
            " FROM session_diagnostics WHERE session_id = ? AND kind = ?"
            " ORDER BY created_at DESC",
            (session_id, kind),
        )
    return [dict(r) for r in rows]
```

### `orchestrator.py._handle_partial_completion()` addition

```python
def _handle_partial_completion(self, e: PartialCompletion, ctx: AgentContext) -> None:
    ...
    # existing: save to tool_results
    ctx.session.save(...)
    ctx.services.llm.stat_partial_completions += 1
    
    # NEW: save to canonical diagnostic store
    if ctx.diagnostics is not None:
        ctx.diagnostics.save_partial_completion(
            session_id=ctx.session.session_id,
            turn=ctx.turn.turn_id if hasattr(ctx.turn, 'turn_id') else 0,
            reason=str(e),
            content_length=len(e.content) if hasattr(e, 'content') else 0,
        )
```

### `tool_runner.py.execute_one_tool_call()` addition

```python
result = await ctx.services.tools.execute(name, args)

# NEW: persist transport failures to diagnostic store
if result.is_error and result.error_type == "transport" and ctx.diagnostics is not None:
    ctx.diagnostics.save_transport_failure(
        session_id=getattr(ctx.session, 'session_id', None),
        tool_name=name,
        server_key=result.server_key,
        error_msg=result.output[:500],
    )
```

### `tool_loop_guard.py` addition

When guard fires (in `check()` or equivalent method that signals the loop to stop):
```python
if ctx.diagnostics is not None:
    ctx.diagnostics.save_loop_guard_hint(
        session_id=getattr(ctx.session, 'session_id', None),
        reason=guard_reason,
        turn_count=self._turn_count,
    )
```
