# Plan: Make `/db` Command Scope Easier to Understand

Derived from `01_plan.md` using requirement `requires/20260618_29_require.md`.

---

## Goal

Users can immediately tell which database each `/db` subcommand targets, without reading external docs.

---

## Scope

**In scope:**
- `scripts/agent/commands/cmd_db.py` — add `help` subcommand, label each subcommand with target DB
- `scripts/agent/commands/command_defs.py` — populate `subcommands` for `/db` with `SubcommandSpec` entries
- `scripts/agent/commands/cmd_help.py` — optionally improve `/help` output to include inline target-DB annotation
- `docs/05_agent_07_cli-and-commands.md` — update `/db` table to match code

**Out of scope:**
- `/db` subcommand renaming or restructuring (e.g. `/db-session` vs `/db-rag`)
- Refactoring `DbMaintenanceService` method names
- Help formatter rewrite

---

## Assumptions

1. Adding `SubcommandSpec` entries for `/db` does not break existing command dispatch logic.
2. Users read `/db help` output before running subcommands.
3. The current subcommand names and behavior are correct — only their documentation and discoverability need improvement.

## Implementation Steps

1. **Read `SubcommandSpec` and `cmd_help.py`** to confirm rendering capability
   - Criterion: know exactly what metadata fields exist

2. **Add `SubcommandSpec` entries for each /db subcommand**
   - Each entry: `name`, `args`, `description` with target DB annotation (e.g. `[rag]` or `[session]`)
   - Criterion: programmatic help possible

3. **Add `_db_help()` handler**
   - Print a table: subcommand | args | target DB | description
   - Criterion: `/db help` or `/db --help` shows clear DB scope

4. **Update output in each subcommand handler**
   - Prepend or append target DB in result summary (e.g. `/db stats` shows DB file paths in output)
   - Criterion: output context matches help

5. **Update `docs/05_agent_07_cli-and-commands.md`**
   - Ensure the `/db` table includes a "Target DB" column
   - Criterion: docs reflect the updated display

6. **Lint, type, arch validation**
   - `ruff check scripts/` — 0 errors
   - `mypy scripts/` — no new errors
   - `lint-imports` — 0 violations
