# Plan: Integrate Event Bus DDL into db/schema_sql.py

Source: `requires/20260701_52_require.md`

---

## Goal

`db/schema_sql.py` に `_EVENTBUS_SCHEMA` 定数と `build_eventbus_schema_sql()` 関数を追加し、`scripts/eventbus/schema.sql` の DDL を一元管理する。

---

## Scope

**In-Scope**
- `scripts/db/schema_sql.py` に `_EVENTBUS_SCHEMA` 文字列定数を追加
- `build_eventbus_schema_sql() -> str` 関数を追加
- `events` テーブル DDL を `_EVENTBUS_SCHEMA` に移植
- 4件のインデックス (`idx_events_topic`, `idx_events_seq`, `idx_events_dlq_at`, `idx_events_dlq_seq`) を移植
- `retry_count` を deprecated コメント付きで保持
- `published_at` に ISO-8601 UTC Z suffix の DEFAULT を設定: `strftime('%Y-%m-%dT%H:%M:%SZ', 'now')`
- `acked_at`, `dlq_at` は nullable のまま維持
- タイムスタンプポリシーをコメントで明記
- スキーマ SQL テストを追加

**Out-of-Scope**
- Event Bus Python ランタイム統合（publisher/subscriber/dispatcher）
- `eventbus.sqlite` 作成処理（53 番で対応）
- `scripts/eventbus/schema.sql` の削除（53/54 完了後に別途対応）

---

## Assumptions

1. `scripts/eventbus/schema.sql` の内容が canonical であり、そこから `_EVENTBUS_SCHEMA` を構築する。
2. `PRAGMA journal_mode=WAL` は `_WORKFLOW_SCHEMA` と同様に `_EVENTBUS_SCHEMA` にも含める（`SQLiteHelper.open()` の PRAGMA 設定と重複するが無害）。
3. `published_at` には `NOT NULL` 制約があるため DEFAULT 値を追加しても既存制約に反しない。

## Risks & Mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| `PRAGMA journal_mode=WAL` を `_EVENTBUS_SCHEMA` に含めると `SQLiteHelper.executescript()` 内での重複実行が起きる | Low | `_WORKFLOW_SCHEMA` と同じパターンを踏襲。SQLite の WAL 設定は冪等であるため無害 |
| `published_at` に DEFAULT を追加することで既存データとの整合性が崩れる可能性 | Low | DEFAULT は新規行にのみ適用される。`IF NOT EXISTS` による冪等 DDL でも既存テーブルには影響しない |
