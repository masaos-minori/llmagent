## Goal

Create a centralized inventory of all "Needs confirmation" items found across the design documentation set, making unconfirmed statements trackable and actionable.

## Scope

Create `docs/00_governance_07_needs-confirmation-inventory.md` — a single document that:
- Extracts all "Needs confirmation" labeled statements from Markdown files under `docs/`
- Lists each item with structured metadata including ID, source location, question, evidence, impact, and required action
- Does not modify the source documents from which items are extracted

## Assumptions

- The eleven fields listed in the issue are sufficient for tracking each item.
- The NC-{NNN} ID format is acceptable.
- The five status values cover all lifecycle states needed.
- No source documents should be modified during extraction.

## Unknowns

- None identified. All scope items are defined in the issue.

## Design

### Inventory Entry Fields

Each entry must contain:
- **ID**: Unique identifier in format `NC-{NNN}` (e.g., NC-001, NC-002)
- **Source File**: Relative path to the document containing the statement
- **Section**: Heading or section where the statement appears
- **Quoted Summary**: Brief quote or summary of the original statement
- **Question**: What specifically needs to be verified?
- **Evidence**: What evidence exists (or doesn't exist) regarding this item
- **Impact**: Consequence if this item is incorrect
- **Required Action**: What action resolves the uncertainty
- **Target Document**: Where the verification result should be recorded
- **Status**: One of: open, investigating, resolved, moved-to-known-issues, deprecated
- **Review Timing**: When this item should be reviewed

### Status Values

- open: Item identified but not yet investigated
- investigating: Investigation underway
- resolved: Verification completed and confirmed
- moved-to-known-issues: Item escalated to Known Issues process
- deprecated: Item no longer relevant

### Extraction Process

1. Search all Markdown files under `docs/` for "Needs confirmation" text.
2. For each match, extract the surrounding context (section heading, quoted text).
3. Assign an ID starting from NC-001.
4. Populate all required fields based on available information.
5. Mark uncertain items with "Needs confirmation" status rather than guessing.

## Implementation steps

1. Create `docs/00_governance_07_needs-confirmation-inventory.md` with the sections listed below.
2. Scan all `docs/*.md` files for "Needs confirmation" occurrences.
3. For each occurrence, create an entry with all required fields.
4. Add non-goals section listing excluded topics.

## Validation plan

- Verify all entries have all eleven required fields populated.
- Verify IDs follow NC-{NNN} format sequentially.
- Verify no source documents were modified.
- Verify uncertain items retain "Needs confirmation" status rather than being guessed.

## Risks

- Medium risk: Some entries may lack sufficient evidence to populate all fields. Mitigation: Allow partial field population with explicit notes about missing information.
- Low risk: Items may become stale if never reviewed. Mitigation: Require Review Timing field and periodic cleanup.
