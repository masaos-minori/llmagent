## Goal

`build_db_config()` の config ソースを `load_all()` から `load("common.toml")` に限定し、DB 設定キーの所有ファイルを明示的に `common.toml` 一本に固定する。

## Scope

**In**:
- `scripts/db/config.py` — `build_db_config()` の config ロード方法変更、docstring 更新

**Out**:
- `config_loader.py` の変更
- config ファイル名や merge 優先順位の変更
- runtime config reload 動作の変更
- 呼び出し側 (`db/maintenance.py` 等) の変更

## Assumptions

1. DB 設定キー (`rag_db_path`, `session_db_path`, `sqlite_timeout`, `sqlite_busy_timeout_ms`, `sqlite_vec_so`, `embedding_dims`) はすべて `common.toml` に存在する（実際に確認済み）。
2. `workflow_db_path` は `common.toml` に存在しないが、`DbConfig` のデフォルト値 `"/opt/llm/db/workflow.sqlite"` が代替する（`cfg.get("workflow_db_path", ...)` のフォールバックが使われる）。
3. `load("common.toml")` は `common.toml` が存在しない場合 `ValueError` を raise する。これは `load_all()` が欠損ファイルをスキップするより望ましい（DB 設定は必須）。

## Design

### 変更点

```python
# 変更前
def build_db_config() -> DbConfig:
    """Construct DbConfig from common.toml via ConfigLoader.load_all()."""
    cfg = ConfigLoader().load_all()

# 変更後
def build_db_config() -> DbConfig:
    """Construct DbConfig from common.toml; raises ValueError if common.toml is missing or malformed."""
    cfg = ConfigLoader().load("common.toml")
```

他の行 (`DbConfig(...)` 構築部分) は変更なし。

### 根拠

- `common.toml` が DB 設定キーの唯一の所有ファイル（`grep` 確認済み）
- `workflow_db_path` は `common.toml` に存在しないが `cfg.get("workflow_db_path", "/opt/llm/db/workflow.sqlite")` のデフォルトで補完
- `load_all()` は 12 ファイルをマージするため、将来別ファイルに同名キーが追加されると黙って上書きされるリスクがある
