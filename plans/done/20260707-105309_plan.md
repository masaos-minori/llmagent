## Goal

Delete `WorkflowExecutionPolicy` entirely — remove the class file, all imports, all construction sites, and the dedicated test file — leaving no mode-based policy abstraction in the codebase.

## Scope

**In**:
- Delete `scripts/agent/workflow_execution_policy.py`
- Remove `from agent.workflow_execution_policy import WorkflowExecutionPolicy` from:
  - `scripts/agent/orchestrator.py` (line 47)
  - `scripts/agent/startup.py` (line 31)
  - `scripts/agent/repl_health.py` (line 35, inside `check_workflow_definition()`)
- Remove `self._policy = WorkflowExecutionPolicy(workflow_mode)` from `orchestrator.py` (line 121)
- Remove `WorkflowExecutionPolicy(ctx.cfg.workflow_mode)` from `startup.py` (line 130)
- Update `check_workflow_definition()` in `repl_health.py` to remove policy parameter and always-required behavior (aligned with plan 20260707-095938 and 20260707-095941)
- Remove `workflow_mode` parameter from `Orchestrator.__init__()` (line 112)
- Delete `tests/test_workflow_execution_policy.py`
- Remove `WorkflowExecutionPolicy` from `scripts/llmagent.egg-info/SOURCES.txt` if it is auto-generated; otherwise document in deploy.sh
- If a narrow `assert_workflow_ready()` helper is needed, create it in `scripts/agent/startup.py` or `scripts/agent/repl_health.py`

**Out**:
- Any replacement mode abstraction
- Mode compatibility
- Migration support

**Note**: This plan is the cleanup leaf of req01 (plan 20260707-095938). It is most naturally implemented as the final step of req01's Phase 2, not as a separate implementation. However, it is listed here as a standalone plan for completeness and traceability.
