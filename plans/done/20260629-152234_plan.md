## 1. Goal
- Improve startup validation clarity and operator guidance for `workflow_mode="required"` so that a missing or invalid workflow definition produces an actionable error message and is detectable before the agent fully initialises.

## 2. Scope
- **In-Scope**:
  - Add a preflight workflow-definition check inside `StartupOrchestrator._check_services()` (`scripts/agent/startup.py`)
  - Enrich error messages in `Orchestrator.__init__()` and `Orchestrator._log_fallback()` (`scripts/agent/orchestrator.py`) with: mode, expected file path, and remediation hint
  - Add a `check_workflow_definition()` helper in `scripts/agent/repl_health.py` (consistent with existing `check_readiness`, `check_routing_drift` patterns)
  - Update `docs/05_agent_08_configuration.md` — clarify production deployment contract under the `workflow_mode` section
  - Update `docs/05_agent_10_operations-and-observability.md` — add a "Workflow startup validation" sub-section with troubleshooting steps
  - Expand tests in `tests/test_orchestrator.py` and `tests/test_repl_health.py` to assert the new error messages and preflight path
- **Out-of-Scope**:
  - Changing `workflow_mode="required"` semantics or weakening the hard-error behaviour
  - Changing the production default in `common.toml`
  - Auto-fallback to `"auto"` on missing definition (requires explicit design decision)
  - Redesigning `WorkflowLoader` file-discovery logic

## 3. Assumptions
- `config/workflows/default.json` is the only supported workflow definition filename (named `"default"` in `WorkflowLoader.load()`).
- The `_WORKFLOWS_DIR` path computed in `workflow_loader.py` is authoritative; the preflight check must reference the same value.
- The existing `check_readiness()` pattern (warn only vs raise in production_mode) is the correct model for the new preflight check.
- `StartupOrchestrator._check_services()` is called before `_init_orchestrator()` — must verify ordering is correct or adjust call site.
- No DB schema changes are required.

## 8. Risks & Mitigations
- **Risk**: Preflight check runs after `Orchestrator.__init__()` making it redundant → **Mitigation**: UNK-01 must be resolved in Phase 1; insert check explicitly before `_init_orchestrator()` call.
- **Risk**: Exposing `WORKFLOWS_DIR` as a module-level constant introduces a coupling point that breaks tests relying on filesystem isolation → **Mitigation**: Accept `workflows_dir: Path | None` override in `check_workflow_definition()` parameter, consistent with `WorkflowLoader.__init__` pattern.
- **Risk**: Doc-only changes diverge from actual code behaviour → **Mitigation**: Write tests first; update docs only after tests pass to ensure docs reflect verified behaviour.
- **Risk**: Enriched error messages in `orchestrator.py` change the regex patterns in existing tests (`test_orchestrator.py` `match="mode=required"`) → **Mitigation**: Use `re.search` compatible patterns or update test match strings as part of the same PR.
