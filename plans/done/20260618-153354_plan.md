# Plan: Improve `/reload` Feedback for Non-Reloadable Settings

Derived from `01_plan.md` using requirement `requires/20260618_32_require.md`.

---

## Goal

`/reload` output clearly separates "applied now" from "restart required" so users cannot misunderstand which changes took effect.

---

## Scope

**In scope:**
- `scripts/agent/commands/cmd_config.py` — display `result.skipped`, add "Files not loaded (restart required)" section
- `scripts/agent/services/config_reload.py` — optionally improve tracking of which keys were skipped
- `docs/05_agent_08_configuration.md` — document output format

**Out of scope:**
- Per-key granularity within applied services (service-level is sufficient)
- Refactoring silent-mutation helpers (`_apply_*`)

---

## Assumptions

1. `result.skipped` is already populated correctly but not displayed — displaying it is the primary fix.
2. Users understand "restart required" means restarting the agent process.
3. The current two-category split (applied / needs_restart) is sufficient; only the display needs improvement.

## Implementation Steps

1. **Audit how `result.skipped` is populated**
   - Search `config_reload.py` for `outcome.skipped.append` or `skipped=` — confirm coverage
   - Criterion: know what is tracked vs what is missing

2. **Add "Files not loaded (restart required)" section to output**
   - Show files that were intentionally excluded (security.toml, tools_definitions.toml, etc.)
   - Criterion: user sees which files were skipped

3. **Display `result.skipped` items**
   - Add a "Skipped:" section between Applied and Restart-required
   - Criterion: no information is silently dropped

4. **Improve formatting**
   - Use clear section headers, possibly with hints (e.g. "llm ✓" vs "server1 ⚠ restart")
   - Criterion: visually scannable

5. **Document output format**
   - Update `docs/05_agent_08_configuration.md` with example output
   - Criterion: output is self-explanatory

6. **Lint, type, arch validation**
   - `ruff check scripts/` — 0 errors
   - `mypy scripts/` — no new errors
   - `lint-imports` — 0 violations
