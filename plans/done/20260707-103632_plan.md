## Goal

Ensure every LLM, tool, audit, and diagnostic event emitted during workflow execution carries the full workflow context (`workflow_id`, `task_id`, `stage_id`, `attempt_id`, `session_id`), and that code paths which can emit events with empty context are removed or made to fail explicitly.

## Scope

**In**:
- Add workflow context fields to `DiagnosticStore.save()` calls in `LLMTurnRunner._handle_llm_error()` (currently passes no workflow context: `scripts/agent/llm_turn_runner.py:119`)
- Change `tool_audit.py` `or ""` fallbacks (`ctx.workflow.workflow_id or ""`, `ctx.workflow.current_task_id or ""`) to assert non-empty values during workflow execution (lines 54, 66, 72, 170, 179)
- Change `workflow_engine.py` OTel span `task.workflow_id or ""` fallbacks (lines 123, 146, 203, 222) to assert non-empty
- Add workflow context to partial-completion diagnostic payloads in `orchestrator.py` (line 372+)
- Add `stage_id` and `attempt_id` to OTel span attributes for tool execution and LLM spans where not yet present
- Fail tests when any workflow-execution event has empty `workflow_id` or `task_id`
- Remove "direct-path placeholder ID" patterns (any `or ""` masking of empty context in production event paths)

**Out**:
- Changing `DiagnosticStore` database schema (no new columns)
- Changing audit log storage backend
- Changing diagnostic table storage backend
- Migration support for historical events

## Assumptions

1. `ctx.workflow.workflow_id` and `ctx.workflow.current_task_id` are set before any tool or LLM event is emitted during `_handle_workflow_engine()`.
2. `tool_audit.py` functions that receive `ctx` can assert the workflow context is populated rather than silently fall back to `""`.
3. `WorkflowEngine` span attributes with `task.workflow_id or ""` are safe to assert non-empty because `create_task()` is called with a non-None `workflow_id` before `WorkflowEngine.run()`.
4. After req03 (plan 20260707-095940), there is no non-workflow execution path; all events are workflow-context events.
5. `DiagnosticStore.save()` already accepts `workflow_id` and `task_id` as optional kwargs; no schema change needed, just always pass them.
