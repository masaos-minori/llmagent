## Design

New workflow file, modeled on the existing `.github/workflows/backward-compat-check.yml` structure:

```yaml
name: Deploy Scripts Lint

on:
  push:
    paths:
      - "deploy/*.sh"
  pull_request:
    paths:
      - "deploy/*.sh"

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  lint:
    name: Shell Syntax and Shellcheck
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: bash -n syntax check
        run: |
          set -euo pipefail
          for f in deploy/*.sh; do
            echo "Checking: $f"
            bash -n "$f"
          done

      - name: shellcheck
        uses: ludeeus/action-shellcheck@v2
        with:
          scandir: "./deploy"
```

`ludeeus/action-shellcheck` is used rather than a manual `apt-get install shellcheck` step, since it is a widely-used, pinned marketplace action that avoids depending on the runner image's package availability/version drift.
