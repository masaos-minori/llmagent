# RAG パイプラインの Stage 分割 — Work Plan

Source instruction: `instructions/20260606_09_instruction.md`

---

## Goal

`rag/pipeline.py` をモノリシックな実装から `PipelineStage.run(ctx)` 共通インターフェースを持つ独立 Stage に分割し、各 Stage の単体テストと latency 計測を追加する。

---

## Scope

**In:**
- `PipelineStage` 共通インターフェースの定義
- MQE / Search / Fusion(RRF) / Rerank / Refine / Augment の Stage 化
- 各 Stage の単体テスト追加
- latency 計測（OTel span または simple timer）
- `/rag search` コマンドの dry-run / debug 導線

**Out:**
- Stage の並列化（現状の逐次実行を維持）
- 外部 Stage プラグイン機構

---

## Assumptions

1. 現在の `rag/pipeline.py` は `run()` メソッドで MQE → Search → Rerank → Augment を逐次実行している。
2. OTel tracer が `shared/otel_tracer.py` に実装済みで span 計測が可能。
3. `/rag` コマンドが `agent/commands/` に存在する（要確認）。

## Implementation Steps

1. `rag/pipeline.py` の現在実装を読み、各処理ブロックを特定する
2. `PipelineStage` Protocol を定義（`rag/stage.py` または `rag/types.py`）
3. 既存処理を独立 Stage クラスに分割（`rag/stages/mqe.py`, `search.py`, `fusion.py`, `rerank.py`, `refine.py`, `augment.py`）
4. 各 Stage に OTel span 計測を追加
5. `python-test-and-fix` スキルで各 Stage の単体テストを追加
6. `/rag search` の debug 導線を `cmd_debug.py` または新規コマンドに追加
7. バリデーション実行 + instructions/done/ へ移動
