# Plan: Enforce local-only guarantee for Persistent Semantic Memory embedding

Source: `requires/20260625_90_require.md`

---

## Goal

`EmbeddingClient` が local エンドポイントのみを受け入れるよう技術的に強制する。`memory_local_only=True` 設定時に `embed_url` が非ローカルであれば startup で fail-fast する。`/memory status` に local-only 保証状態を表示する。

---

## Scope

**In-Scope**
- `scripts/agent/memory/embedding_client.py`: `EmbeddingClientConfig` に `local_only` フィールド追加
- `config/memory.toml`: `memory_local_only` 設定追加
- startup 検証: `embed_url` が localhost/127.0.0.1/::1 でない場合に例外
- `/memory status` に local-only 状態表示
- docs: `05_agent_08_configuration.md`, `05_agent_10_operations-and-observability.md`, `05_agent_12_memory.md`

**Out-of-Scope**
- embedding モデルの変更
- memory backend の変更

---

## Assumptions

1. `EmbeddingClientConfig.embed_url` が現在の唯一の外部通信エンドポイント。
2. local エンドポイントの定義: `http://localhost:*`, `http://127.0.0.1:*`, `http://[::1]:*`（HTTPS ローカルは稀だが許容）。
3. `memory_local_only=True` + 非ローカル `embed_url` は startup exception（ValidationError）。
4. `memory_local_only=False`（デフォルト）は既存動作を維持。

## Implementation Steps

- [ ] 1. `EmbeddingClientConfig` に `local_only: bool = False` を追加
- [ ] 2. `EmbeddingClient.__init__` または `__post_init__` でURL検証を実装:
  ```python
  _LOCAL_PREFIXES = ("http://localhost", "http://127.0.0.1", "http://[::1]", "https://localhost", "https://127.0.0.1")
  if self._config.local_only and self._config.embed_url:
      if not any(self._config.embed_url.startswith(p) for p in _LOCAL_PREFIXES):
          raise ValueError(f"memory_local_only=True but embed_url is not local: {self._config.embed_url!r}")
  ```
- [ ] 3. `config/memory.toml` に `memory_local_only = false` を追加
- [ ] 4. `EmbeddingClientStatus` に `local_only: bool` フィールドを追加し、`get_status()` で返す
- [ ] 5. `/memory status` 出力に "local-only: enabled/disabled" を追加
- [ ] 6. docs 3 ファイルに local-only 保証の設定・故障モードを記述
