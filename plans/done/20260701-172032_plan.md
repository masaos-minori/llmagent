## Goal

Remove all stdio transport code from the MCP runtime so that MCP servers operate via HTTP-only transport.

## Scope

**In-Scope**:
- `scripts/shared/mcp_config.py` — remove `TransportType.STDIO`, `StartupMode.ONDEMAND`, `StartupMode.SUBPROCESS`, `HealthcheckMode.PROCESS`, `HealthcheckMode.PING_TOOL`; remove `cmd`, `working_dir`, `env`, `idle_timeout_sec` fields
- `scripts/shared/tool_executor.py` — remove `StdioTransport` class
- `scripts/agent/stdio_lifecycle.py` — delete file
- `scripts/agent/lifecycle_protocol.py` — remove `StdioLifecycleProtocol`, `restart_stdio()`
- `scripts/agent/factory.py` — remove stdio_procs, StdioServerLifecycleManager, stdio branches
- `scripts/agent/repl_health.py` — remove `_fetch_stdio_tools()`, `__list_tools__`, `_watchdog_check_stdio()`, stdio branches
- `scripts/agent/context.py` — remove `stdio_procs` field
- `scripts/agent/services/mcp_status.py` — remove `stdio_procs` reference
- `scripts/mcp/server.py` — remove `run_stdio()`, `_process_stdio_line()`, `_setup_stdio_reader()`, `_write_stdio_response()`, `__list_tools__` handling
- 11 MCP server files — remove `--stdio` entrypoints: cicd, file/read, file/delete, file/write, github, shell, mdq/__main__, mdq/server, rag_pipeline, git, web_search

**Out-of-Scope**:
- `agent/config.py` re-export stub cleanup
- `McpServerConfig._cast_enums()` compatibility shim removal (separate issue)
- MCP documentation cleanup
- `_MCP_TOOLS` to `TOOL_LIST` migration (separate plan)
- Audit format migration (separate plan)
- Test file cleanup for StdioTransport test classes (separate plan: 20260701-144828)

## Assumptions

1. No production MCP server is currently running in stdio mode; all are HTTP.
2. `deploy/deploy.sh` does not pass `--stdio` to any MCP server startup command.
3. `StdioTransport._parse_stdio_response()` is not used outside `StdioTransport` itself.
4. Removing `cmd`, `working_dir`, `env`, `idle_timeout_sec` from `McpServerConfig` will not break existing TOML configs (HTTP servers do not set these fields).
5. `StartupMode.PERSISTENT` and `HealthcheckMode.HTTP` remain after removal.
6. `_ServerLifecycleRouter` in factory.py can be simplified to HTTP-only once StdioServerLifecycleManager is removed.

## Design

### mcp_config.py changes
- Remove `TransportType.STDIO = "stdio"` from `TransportType` enum.
- Remove `StartupMode.ONDEMAND`, `StartupMode.SUBPROCESS` from `StartupMode`.
- Remove `HealthcheckMode.PROCESS`, `HealthcheckMode.PING_TOOL` from `HealthcheckMode`.
- Remove fields: `cmd: list[str]`, `working_dir: str`, `env: dict[str, str]`, `idle_timeout_sec: int`.
- Remove stdio-specific validation in `__post_init__()`: cmd-empty check, subprocess/stdio check, ondemand/http check.
- Remove stdio branch in `_cast_enums()` default healthcheck logic.
- Remove stdio parsing in `build_mcp_servers()`.

### tool_executor.py changes
- Delete `StdioTransport` class (lines 219+), `_StdioRequest`, `_StdioResponse` dataclasses.
- Remove `_parse_stdio_response`, `_write_stdio_request` static methods.
- Keep `HttpTransport`, `ToolExecutor`, `ToolCallResult`, `TransportError`.

### stdio_lifecycle.py
- Delete file entirely.

### lifecycle_protocol.py
- Remove `StdioLifecycleProtocol` class.
- Remove `restart_stdio()` method signature.

### factory.py
- Remove `from agent.stdio_lifecycle import StdioServerLifecycleManager`.
- Remove `from shared.tool_executor import StdioTransport`.
- Remove `stdio_procs: dict[str, StdioTransport]` parameter from `_ServerLifecycleRouter.__init__()`.
- Remove all stdio branches in `ensure_ready()`, `shutdown_all()`, `shutdown_idle()`, `get_transport_state()`, `restart_stdio()`.
- Remove `stdio_procs: dict = {}` from AppContext assembly.
- Simplify `_build_tool_executor()` to remove stdio_procs parameter.

### repl_health.py
- Delete `_fetch_stdio_tools()` function.
- Remove `__list_tools__` probe logic from `check_server_health()`.
- Delete `_watchdog_check_stdio()` function.
- Remove stdio branch from `check_server_health()`.
- Remove `from shared.tool_executor import StdioTransport` import.

### context.py
- Remove `stdio_procs: dict[str, StdioTransport]` field.
- Remove `from shared.tool_executor import StdioTransport` import.

### mcp_status.py

### mcp/server.py
- Remove `run_stdio()`, `_process_stdio_line()`, `_setup_stdio_reader()`, `_write_stdio_response()`.
- Remove `__list_tools__` special-case dispatch.
