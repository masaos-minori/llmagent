# ツール実行の依存関係解析付きスケジューリング — Work Plan

Source instruction: `instructions/20260606_05_instruction.md`

---

## Goal

`asyncio.gather()` による全並列と `serial_tool_calls` による全直列の二択を廃止し、ツール単位・リソース単位の依存関係解析による部分並列化を実装する。

---

## Scope

**In:**
- `tool_definitions` への `side_effect`, `resource_scope`, `requires_serial` フィールド追加
- 依存関係グラフ構築と部分並列実行プランの生成
- `ToolExecutor.execute_all_tool_calls()` の dependency-aware 実装

**Out:**
- 動的依存関係学習（静的メタデータのみ）
- `serial_tool_calls` の完全削除（backward-compat 維持）

---

## Assumptions

1. 現在は `serial_tool_calls=True` で全直列、`False` で全並列（`asyncio.gather()`）の二択。
2. `tool_definitions` は `config/tools_definitions.toml` で管理されている。
3. 実装は `shared/tool_executor.py` の `execute_all_tool_calls()` を対象とする。

## Implementation Steps

1. `tools_definitions.toml` の形式と `execute_all_tool_calls()` の現在実装を確認
2. `ToolSpec` または tool definition スキーマに `side_effect`, `resource_scope`, `requires_serial` を追加
3. `ToolDependencyScheduler` クラスを `shared/tool_scheduler.py` に実装
   - 同一 resource_scope への write → 直列化
   - 独立ツール → 並列化
   - `requires_serial=True` → 全体直列
4. `ToolExecutor.execute_all_tool_calls()` を `ToolDependencyScheduler` に委譲
5. `python-test-and-fix` スキルでテスト追加
6. バリデーション実行 + instructions/done/ へ移動
