# Plan: Remove backward-compatible re-export stub `mcp/github/service.py`

**Source requirement**: `requires/20260702_54_require.md`
**Created**: 2026-07-02 20:28:57

---

## Goal

Remove `mcp/github/service.py` (the backward-compatibility re-export stub) and update all
callers to import directly from `mcp.github.service_dispatch` or `mcp.github.service_init`,
so that the `mcp.github.service` import path no longer exists.

---

## Scope

**In**:
- Delete `scripts/mcp/github/service.py`
- Update `scripts/mcp/github/server.py` to import directly from sub-modules
- Update `scripts/mcp/github/__init__.py` to remove the re-export stub dependency
- Update `scripts/mcp/github/server_common.py` to import `GitHubService` directly
- Update `scripts/mcp/github/server_file.py` to import `GitHubService` directly
- Update `scripts/mcp/github/server_issues.py` to import `GitHubService` directly
- Update `scripts/mcp/github/server_pull_requests.py` to import `GitHubService` directly
- Update `scripts/mcp/github/server_repository.py` to import `GitHubService` directly
- Update `tests/test_github_mcp_service.py` to import `GitHubService` directly
- Update documentation that references `mcp.github.service` (none found by grep)

**Out**:
- Changing `GitHubService` class behavior or methods
- Changing `build_service()` behavior
- Changing GitHub MCP tool definitions or HTTP endpoints
- Keeping deprecated import aliases anywhere

---

## Assumptions

1. `GitHubService` for type annotation purposes comes from `mcp.github.service_dispatch`
   (the dispatch layer extends business methods with `fmt_*` methods); this is what the stub
   was re-exporting.
2. `build_service` and `_GITHUB_TOKEN` come from `mcp.github.service_init`.
3. No external scripts outside `scripts/`, `tests/`, and `docs/` reference `mcp.github.service`
   (low risk; no other source dirs found in this project).
4. `mcp/github/__init__.py` re-exports are not consumed by external code via
   `from mcp.github import GitHubService`; the same direct sub-module import fixes both.
5. `docs/` contains no references to `mcp.github.service` (confirmed by grep — zero matches).
