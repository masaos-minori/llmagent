## Goal

`ConfigLoader.load_all()` が存在しないファイルをスキップしつつ、構文・読み取りエラーは明示的に raise するよう、`config_loader.py` に専用例外型を導入する。

## Scope

**In**:
- `scripts/shared/config_loader.py` — 例外クラス 3 つの追加、`_load_single()` の raise 型変更、`load_all()` の catch 対象を `ConfigMissingError` のみに変更
- `tests/test_config_loader.py` — `load_all()` の parse/read 失敗テスト追加

**Out**:
- 呼び出し側 (`config_builders.py`, `rag/pipeline.py`, `rag/ingestion/ingester.py`) のコード変更
- config ファイル名・マージ優先順位の変更
- DB スキーマ変更

## Assumptions

1. `ConfigMissingError`, `ConfigParseError`, `ConfigReadError` はいずれも `ValueError` を継承する（既存の `except (OSError, ValueError, ...)` が引き続き動作）。
2. `rag/pipeline.py` の `except (FileNotFoundError, ValueError)` は `ConfigMissingError(ValueError)` / `ConfigParseError(ValueError)` を補足できる。
3. `rag/ingestion/ingester.py` の `RagIngester.__init__` は `config=None` 時のみ `load_all()` を呼ぶ。デプロイ時は config を直接渡すため、parse エラーが伝播してもクラッシュは想定内。
4. base config ファイル (`common.toml` 等) はデプロイ時に正しいフォーマットで配置される前提。

## Design

### 例外クラス

```python
class ConfigMissingError(ValueError):
    """Config file does not exist."""

class ConfigParseError(ValueError):
    """Config file exists but cannot be parsed."""

class ConfigReadError(ValueError):
    """Config file exists but cannot be read (permission, I/O)."""
```

### `_load_single()` の変更

| 現在 | 変更後 |
|---|---|
| `FileNotFoundError` → `ValueError` | `FileNotFoundError` → `ConfigMissingError` |
| `TOMLDecodeError` → `ValueError` | `TOMLDecodeError` → `ConfigParseError` |
| `JSONDecodeError` → `ValueError` | `JSONDecodeError` → `ConfigParseError` |
| `OSError` → `ValueError` | `OSError` → `ConfigReadError` |
| 非 dict JSON → `ValueError` | 非 dict JSON → `ConfigParseError` |

### `load_all()` の変更

```python
def load_all(self) -> dict[str, Any]:
    merged: dict[str, Any] = {}
    for name in _BASE_CONFIG_FILES:
        try:
            merged.update(self._filter_meta_keys(self._load_single(name)))
        except ConfigMissingError:
            logger.debug("Config file not found: %s", name)
    return merged
```

`ConfigParseError` と `ConfigReadError` は catch しない → 呼び出し元へ伝播。

### `load()` の変更

不要。既に `_load_single()` の例外をそのまま伝播させる。
