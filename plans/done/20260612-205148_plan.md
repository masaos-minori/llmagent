## Assumptions

1. `assert ctx.services.X is not None` は「ファクトリが初期化を怠ったケース」を検出するための precondition check であり、`RuntimeError` への置換が適切。
2. `except Exception` のうち、MCP network 呼び出し・HTTP probe・stdio start に関するものは `(httpx.HTTPError, OSError, asyncio.TimeoutError)` または `(OSError, RuntimeError)` に絞れる。
3. `repl.py:305` の `except Exception` は `memory.on_session_stop()` の広域キャッチで、finally ブロック内のため伝播させたくない。`MemoryServiceError, RuntimeError` に絞る。
4. `config.py:29` の `except Exception` は `ConfigLoader().load_all()` を wrap しており、`OSError, ValueError, TypeError` に絞れる。
5. `history.py:203` の `except Exception` は LLM HTTP 呼び出し + JSON parse を wrap しており、`httpx.HTTPError, orjson.JSONDecodeError, KeyError, TypeError` に絞れる。
6. `tool_result_formatter.py` の `str(args.get(...))` は display-only の preview 用途であるが、args の値が非 str の場合は明示的に扱う。`None` を空文字列として扱わず、型を確認してから変換する。
7. `config.py` の `str(cfg.get("tokenize_url", ""))` 等は、デフォルト値が `""` のため Noneは返らない。冗長な `str()` 呼び出しを除去するだけでよい。
8. `tool_approval.py:118` の `except Exception as preflight_exc: # noqa: BLE001` は `PolicyViolationError` のみをキャッチすることが意図されており、`except PolicyViolationError as preflight_exc:` に変更して noqa を除去できる。

## Design

### assert → RuntimeError 置換パターン

```python
# Before
assert ctx.services.llm is not None

# After
if ctx.services.llm is None:
    raise RuntimeError("llm service not initialized")
```

### except Exception → specific exceptions パターン

```python
# Before (MCP health check)
except Exception as e:
    logger.warning(f"...")

# After
except (httpx.HTTPError, OSError, asyncio.TimeoutError) as e:
    logger.warning(f"...")
```

### except Exception (BLE001) → direct type

```python
# Before
except Exception as preflight_exc:  # noqa: BLE001 — PolicyViolationError from check_preflight
    from agent.tool_exceptions import PolicyViolationError
    if not isinstance(preflight_exc, PolicyViolationError):
        raise

# After
except PolicyViolationError as preflight_exc:
```

### 無条件 str() 変換の除去パターン

```python
# Before (tool_result_formatter.py)
owner = str(args.get("owner", ""))

# After: args は LLM 由来で dict[str, Any]。preview 目的でも型確認を経る
_owner = args.get("owner")
owner = _owner if isinstance(_owner, str) else ""
```

```python
# Before (config.py)
tokenize_url=str(cfg.get("tokenize_url", "")),

# After: cfg.get() のデフォルトが str なので str() は冗長
tokenize_url=cfg.get("tokenize_url", ""),
```

## Implementation Steps

1. **`orchestrator.py` / `llm_turn_runner.py` / `note_repo.py` — assert 置換**
   - 各 `assert X is not None` を `if X is None: raise RuntimeError(...)` に置換
   - `orchestrator.py:92`, `orchestrator.py:129`, `llm_turn_runner.py:133`, `note_repo.py:22`
   - 完了条件: `grep -n "assert " scripts/agent/orchestrator.py scripts/agent/llm_turn_runner.py scripts/agent/note_repo.py` がゼロ

2. **`repl_health.py` — assert 置換 + except Exception 絞り込み**
   - assert 3件 → RuntimeError
   - `probe_mcp_health` の `except Exception:` → `except (httpx.HTTPError, OSError, asyncio.TimeoutError):`
   - `check_service_health` の `except Exception as e:` → `except (httpx.HTTPError, OSError):`
   - `_collect_server_tool_names` の `except Exception as e:` → `except (httpx.HTTPError, OSError):`
   - watchdog 内の 2件の `except Exception as e:` → `except (OSError, RuntimeError):`
   - 完了条件: `grep -n "except Exception\|assert " scripts/agent/repl_health.py` がゼロ

3. **`repl.py` — assert 置換 + except Exception 絞り込み**
   - assert 2件 → RuntimeError
   - `_get_chunk_count` の `except Exception as e:` → `except (sqlite3.Error, OSError):`
   - `_start_mcp_servers` の 2件の `except Exception as e:` → `except (OSError, RuntimeError):`
   - `run()` finally ブロックの `except Exception:` → memory 例外型を確認後に絞り込み
   - 完了条件: `grep -n "except Exception\|assert " scripts/agent/repl.py` がゼロ

4. **`config.py` — except Exception 絞り込み + 冗長 str() 除去**
   - `load_config()` の `except Exception as e:` → `except (OSError, ValueError, TypeError):`
   - `_build_llm_config` の `str(cfg.get("tokenize_url", ""))` → `cfg.get("tokenize_url", "")`
   - `_build_memory_config` の `str(cfg.get("memory_jsonl_dir", ...))` → `cfg.get("memory_jsonl_dir", ...)`
   - 完了条件: mypy エラーなし

5. **`history.py` — except Exception 絞り込み**
   - コンテキスト圧縮 LLM 呼び出しの `except Exception as e:` → `except (httpx.HTTPError, orjson.JSONDecodeError, KeyError, TypeError):`
   - 完了条件: mypy エラーなし

6. **`http_lifecycle.py` — except Exception 絞り込み**
   - health check poll の `except Exception as e:` → `except (httpx.HTTPError, OSError):`
   - `shutdown_all` の `except Exception as e:` → `except (OSError, asyncio.TimeoutError):`
   - 完了条件: mypy エラーなし

7. **`tool_result_formatter.py` — 無条件 str() 除去**
   - `build_github_preview`: `str(args.get("owner", ""))` / `str(args.get("repo", ""))` → isinstance チェック経由
   - `build_preview`: `str(args.get("content") or ...)` / `str(args.get("path") or ...)` / `str(args.get("command", "?"))` → 同様
   - 完了条件: mypy エラーなし、既存テスト全 pass

8. **`tool_approval.py` — BLE001 noqa 除去 + PolicyViolationError 直接キャッチ**
   - `except Exception as preflight_exc:  # noqa: BLE001` → `except PolicyViolationError as preflight_exc:`
   - 内部の `isinstance` チェックと再 raise を削除
   - `from agent.tool_exceptions import PolicyViolationError` を先頭 import に移動
   - 完了条件: `grep "noqa" scripts/agent/tool_approval.py` がゼロ
