# Plan: Synchronize compressed conversation history with session persistence

Source: `requires/20260626_01_require.md`

---

## Goal

When `HistoryManager.compress()` replaces `ctx.conv.history` in-memory, persist the same compressed state to the `messages` table so that `/session load` restores a semantically equivalent history.

---

## Scope

**In-Scope**
- Define canonical persistence model: persist compressed history back to `messages` (delete replaced rows, insert summary row + retained rows with new message_ids)
- Update `session_restore.py` to reconstruct history correctly from the persisted compressed state
- Ensure `/history`, `/export`, and `/session load` all use the same DB-sourced history
- Add regression test: "compress → /session load → equivalent restored history"

**Out-of-Scope**
- Redesign of `HistoryManager` compression algorithm
- Changing compression policy (char_limit, compress_turns, etc.)
- `/session rename`, `/session list`, `/session delete`

---

## Assumptions

1. The canonical source of truth for conversation history is `messages` in SQLite. In-memory state is a cache derived from it.
2. Compression replaces N old messages with a system summary + remaining messages. The equivalent DB state is: delete those N message rows and insert one summary row.
3. Message IDs increase monotonically; session restore reads by session_id ORDER BY message_id — order is preserved after DB update.
4. The `[Conversation summary]` marker in the system message is sufficient to distinguish compressed summaries from original system prompts on reload.

## Implementation Steps

1. **Phase 1: DB persistence method**
   - [ ] Add `replace_compressed_messages(session_id, cutoff_message_id, summary_content, retained_messages)` to `SessionMessageRepository` — deletes all messages with message_id < cutoff, inserts summary row and retained rows in one transaction
   - [ ] Expose via `AgentSession.replace_compressed_messages()`

2. **Phase 2: Wire compression to persistence**
   - [ ] In `HistoryManager.compress()`, return (compressed_history, CompressResult, deleted_count) or identify deleted messages via a new return field
   - [ ] In the caller (orchestrator or llm_turn_runner), after assigning `ctx.conv.history = compressed`, call `ctx.session.replace_compressed_messages()`

3. **Phase 3: Verification**
   - [ ] Run `uv run pytest tests/test_agent_session.py -v`
   - [ ] Add regression test: start session, add N messages, trigger compress, call `restore_session`, assert restored history matches compressed in-memory history
