## Goal

`/audit` スラッシュコマンド (`tail`, `turn <id>`, `tool <name>` サブコマンド) を追加し、
CLI から audit.log の検索・表示を可能にすることで障害解析・承認追跡の運用効率を向上させる。

---

## Scope

**In**:
- `/audit tail [N]` — audit.log の末尾 N 行を表示 (デフォルト 20)
- `/audit turn <task_id>` — 指定 task_id に一致する全イベントを表示
- `/audit tool <name>` — 指定ツール名に一致するイベントを表示
- 新規ファイル `scripts/agent/commands/cmd_audit.py` の作成
- `scripts/agent/commands/registry.py` への `/audit` コマンド登録
- `docs/06_ref-agent-commands.md` のドキュメント更新

**Out**:
- `replay_turn()` 相当の実行再現機能 (将来対応)
- audit.log スキーマ変更 (既存フィールドを活用)
- MCP サーバー側 `mcp/audit.py` の変更
- `/debug audit` の削除・変更 (後方互換を維持)

---

## Assumptions

1. audit.log の各行は `tool_audit.py` が出力する JSONL 形式であり、`event`, `task_id`, `tool`, `ts` フィールドを必ず持つ。
2. `mcp/audit.py` が出力する通常テキスト形式行 (`AUDIT session=...`) が audit.log に混在する可能性がある。JSON パース失敗行はスキップする。
3. `ctx.cfg.obs.audit_log_file` のパスが常に有効であり、ファイルが存在すれば読み取り可能である。
4. `cmd_audit.py` は `_AuditMixin` として実装し、CommandRegistry の既存継承チェーンに追加する。
5. `/audit` は同期コマンド (prefix=True, is_async=False) として実装する。

## Design

### audit.log のフォーマット

```jsonl
{"event":"tool_approval","task_id":"uuid4","tool":"github__list_prs","operation_type":"read","resource_scope":{},"risk":"low","decision":"auto_approved","args_preview":{},"ts":1718400000.0}
{"event":"tool_exec","task_id":"uuid4","tool":"github__list_prs","operation_type":"read","resource_scope":{},"mcp_request_id":"req-1","is_error":false,"args_preview":{},"ts":1718400001.0}
```

### サブコマンド仕様

| サブコマンド | 動作 | フィルタ条件 |
|---|---|---|
| `tail [N]` | 末尾 N 行を raw 表示 (デフォルト 20) | なし |
| `turn <task_id>` | 全行を task_id でフィルタし表示 | `line["task_id"] == task_id` |
| `tool <name>` | 直近 1000 行を tool でフィルタし表示 | `line["tool"] == name` |
| 引数なし | `tail 20` と同じ動作 | なし |

### 実装クラス構成

```
cmd_audit.py
  _AuditMixin(MixinBase)
    _cmd_audit(args: str) → None
      _audit_tail(n: int) → None
      _audit_turn(task_id: str) → None
      _audit_tool(tool_name: str) → None
    _audit_log_path() → pathlib.Path
    _iter_audit_lines(path: Path) → Iterator[dict]   # ジェネレータ; JSON パース失敗行スキップ
```

## Implementation Steps

1. **`cmd_audit.py` 新規作成**
   - `_AuditMixin(MixinBase)` を実装
   - `_cmd_audit(args)` でサブコマンドを分岐
   - `_iter_audit_lines()` は行単位ジェネレータ (JSON パース失敗は `continue`)
   - `tail`: `deque(maxlen=N)` で末尾 N 行を収集して出力
   - `turn <id>`: 全行をストリームし `task_id` 一致行のみ出力
   - `tool <name>`: 末尾 1000 行をストリームし `tool` 一致行のみ出力
   - 完了条件: `_cmd_audit` が各サブコマンドで正しい出力を返す

2. **`registry.py` への `/audit` コマンド登録**
   - `_COMMANDS` リストに `/audit` の `CommandDef` を追加 (prefix=True, is_async=False)
   - `cmd_audit.py` の import を追加
   - `CommandRegistry` の継承に `_AuditMixin` を追加
   - 完了条件: `CommandRegistry()` 初期化が AttributeError を出さない

3. **`docs/06_ref-agent-commands.md` 更新**
   - `/audit` コマンドのサブコマンド仕様を記述
   - 完了条件: ドキュメントが実装と一致する

4. **テスト作成**
   - `tests/test_cmd_audit.py` を新規作成
   - `tail`, `turn`, `tool` の各サブコマンドをテスト
   - 空ファイル、JSON パース失敗行、audit ファイル不在の異常系をテスト
   - 完了条件: `uv run pytest tests/test_cmd_audit.py` が全件 pass

5. **Validation 実行** (Validation Plan 参照)

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| audit.log に JSONL 以外の行が混在し JSON パース失敗 | 中 (mcp/audit.py が通常テキスト形式で書く) | `_iter_audit_lines()` で `orjson.JSONDecodeError` を `continue` でスキップ |
