# Plan: Unify Config Loading, Reload Behavior, and Restart-Required Semantics

## 1. Goal

Produce one authoritative reload behavior matrix for all config settings,
add a "deferred" category to `ConfigReloadOutcome`, and update `/reload` output
to be explicit about applied / deferred / restart-required / unchanged.

## 2. Scope

**In-Scope:**
- Add `deferred: list[str]` to `ConfigReloadOutcome` for changes that are accepted
  but take effect only on the next connection/operation (e.g. `auth_token`, `startup_mode`)
- Update `_apply_mcp_url_reload()` to move `auth_token`/`startup_mode` changes from
  `needs_restart` to `deferred`
- Update `_cmd_reload` output: show deferred items separately from restart-required
- Add startup-only fields to the config classification table in
  `docs/05_agent_08_configuration.md`: `use_memory_layer`, `plugin_strict`, any field
  not touched by `apply_config_dict()`
- Add "unchanged" note: items not in `applied`/`deferred`/`needs_restart`/`skipped`
  were not changed (no new output line needed â€” absence is implicit)
- Clarify `common.toml` ownership: loaded by `/reload` same as other files; primary owner
  of `llm_url`, RAG defaults, observability defaults

**Out-of-Scope:**
- Redesigning config file structure or merging config files
- Making startup-only fields reloadable (requires architectural change)
- Changing `ConfigLoader.load_all()` behavior

## 3. Assumptions

1. "Deferred" means: the change is stored in `ctx.cfg` (persisted across the run),
   but the running sub-system (e.g. MCP stdio connector) won't see it until
   the next connection restart or session reload â€” not the same as "restart agent".
2. `auth_token` and `startup_mode` are confirmed as deferred (not immediately applied)
   because stdio MCP processes are already running â€” the doc note at line 76 confirms this.
3. Fields NOT in `apply_config_dict()` (e.g. `use_memory_layer`, `plugin_strict`) are
   startup-only â€” verified by scanning `config_reload.py` for the complete set of applied keys.
4. `/reload` output improvement is backward-compatible â€” operators who script the text
   output are not officially supported.

1. **Add `deferred` to `ConfigReloadOutcome`**
   - `config_reload.py`: `deferred: list[str] = field(default_factory=list)`

2. **Update `_apply_mcp_url_reload()` for `auth_token`/`startup_mode`**
   - When `auth_token` or `startup_mode` differs: classify as `result.deferred` (not `needs_restart`)
   - Requires reading the new and old value for these fields

3. **Update `_cmd_reload` output**
   - After "Applied (runtime):" block, add:
     ```

         self._out.write(f"Deferred (next connection): [{len(result.deferred)} items]")

     ```

   - Add `deferred` to the `ConfigReloadOutcome` fields table
   - Add `common.toml` ownership note: which keys live exclusively in `common.toml`

5. **Add test**
