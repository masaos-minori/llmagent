## Goal
`orchestrator.py` の `_handle_history_compression()` と `_sync_system_prompt()` を修正し、`_memory_injected` / `_ephemeral` マーカー付きの一時的なシステムメッセージが DB に永続化されないようにする。

## Scope
**In-Scope**:
- `scripts/agent/orchestrator.py`
  - `_handle_history_compression()`: `replace_messages()` に渡す前に `_memory_injected` / `_ephemeral` をフィルタ
  - `_sync_system_prompt()`: 既存の `_ephemeral` フィルタに `_memory_injected` を追加

**Out-of-Scope**:
- `undo_service.py` の変更
- `session_message_repo.replace_messages()` 本体の変更
- `fetch_messages()` の変更
- `_append_user_message()` の変更

## Assumptions
1. `ctx.conv.history` はカレントターン中は `_memory_injected` メッセージを保持し続ける（LLM コンテキストに必要）
2. DB スナップショット（`replace_messages()` 呼び出し）のみがフィルタ対象
3. `_sync_system_prompt()` はターン開始時に実行され、前ターンの ephemeral メッセージを除去する

## Design
```python
# _handle_history_compression() 修正
ctx.session.replace_messages(
    [m for m in ctx.conv.history if not m.get("_memory_injected") and not m.get("_ephemeral")]
)

# _sync_system_prompt() 修正
ctx.conv.history = [
    m for m in ctx.conv.history
    if not m.get("_ephemeral") and not m.get("_memory_injected")
]
```
