# Plan: Document DLQ promotion as inline-on-nack plus background safety sweep

## Goal
3つのEvent Bus docファイルで、DLQ昇格の一次パスが `/nack` によるインライン昇格であること、バックグラウンドDLQループが安全スイープであること、および関連する運用上の注意点が実装と一致して記述されている状態にする。

## Scope

- `docs/06_eventbus_04_dlq_offsets_and_delivery_semantics.md` — DLQ promotion path 記述の確認・補強
- `docs/06_eventbus_05_configuration_deploy_and_operations.md` — DLQファイル作成タイミング・backgroundループ役割の追記

## Assumptions
- `sweep_orphans()` はインライン昇格を補完する安全スイープとして実装されており、通常運用ではゼロ件を返す（実装確認済み: `app.py` L103-108）
- `requeue_event()` は `delivery_failure_count` をリセットしない（実装確認済み: `db.py` L226-234）
- requeueされたイベントで `delivery_failure_count >= max_retry` の場合、次の DLQ loop tick（60秒以内）で再昇格する（実装確認済み: `app.py` L364-386、インライン再昇格はない）
