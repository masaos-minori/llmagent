# Plan: LLMTransportError per-kind テスト追加 — test_orchestrator.py

## Goal

`tests/test_orchestrator.py` に `LLMTransportError` の各 `kind` 値ごとのテストを追加し、
接続エラー・タイムアウト・HTTP エラー・ストリームエラーの各パスが正しく処理されることを検証する。

## Scope

**In-Scope:**
- `tests/test_orchestrator.py` — `TestHandleTurnLLMTransportError` への 4 テスト追加
- `tests/test_orchestrator.py` — `TestRunTurnLLMTransportError` への 1 テスト追加

**Out-of-Scope:**
- 実装コードの変更
- `LLMTransportError` のサブクラス化 (サブクラスは存在しない; `kind` フィールドで識別)

## Assumptions

1. `LLMTransportError` にサブクラスは存在しない。`LLMErrorKind` Literal で 9 種類の kind を識別する:
   `HTTP_STATUS_RETRYABLE`, `HTTP_STATUS_FATAL`, `CONNECT_ERROR`, `READ_TIMEOUT`,
   `HEARTBEAT_TIMEOUT`, `MALFORMED_SSE_FRAME`, `UTF8_PARTIAL_DECODE_ERROR`,
   `PREMATURE_EOF`, `UNKNOWN_STREAM_ERROR`
2. `_handle_llm_transport_error()` の分岐は `e.partial_text` の有無のみ:
   - `partial_text` 非空 → `_diagnostic_store.save(..., "llm_transport_error", ...)` + `stat_partial_completions += 1`
   - `partial_text` 空 → `_diagnostic_store.save(..., "mid_turn_error", json)` + pre-stream 扱い
3. 既存テストで使用済みの kind: `PREMATURE_EOF` (in-stream + partial_text), `CONNECT_ERROR` (pre-stream), `HEARTBEAT_TIMEOUT` (tool continuation)
4. 未テストの kind: `READ_TIMEOUT`, `HTTP_STATUS_RETRYABLE`, `HTTP_STATUS_FATAL`, `MALFORMED_SSE_FRAME`
5. `_make_err()` ヘルパーは `status_code` パラメータを持たないため、`status_code` が必要なテストは `LLMTransportError(...)` を直接構築する。

## Design

### 追加テスト一覧

#### TestHandleTurnLLMTransportError への追加 (4 テスト)

**Test 1: `test_read_timeout_with_partial_increments_stat`**
- kind=`READ_TIMEOUT`, partial_text="chunk text", phase="in_stream"
- `handle_turn("hello")` を呼ぶ
- Assert: `ctx.services.llm.stat_partial_completions == 1`
- Assert: `orch._diagnostic_store.save` が "llm_transport_error" で呼ばれた

```python
@pytest.mark.asyncio
async def test_read_timeout_with_partial_increments_stat(self) -> None:
    ctx = _make_ctx()
    orch = _make_orchestrator(ctx)
    err = _make_err(kind="READ_TIMEOUT", partial_text="chunk text")

    with patch.object(orch._llm_runner, "run", AsyncMock(side_effect=err)):
        await orch.handle_turn("hello")

    assert ctx.services.llm.stat_partial_completions == 1
    call_args = orch._diagnostic_store.save.call_args_list
    assert any("llm_transport_error" in str(a) for a in call_args)
```

**Test 2: `test_http_retryable_prestream_saves_mid_turn_diagnostic`**

**Test 3: `test_http_fatal_with_status_code_saves_diagnostic`**
- kind=`HTTP_STATUS_FATAL`, status_code=503 (`LLMTransportError` 直接構築)
- Assert: `stat_partial_completions == 0`
- Assert: `_diagnostic_store.save` が呼ばれた

```python

    ctx = _make_ctx()

```

**Test 4: `test_malformed_sse_with_partial_saves_llm_transport_diagnostic`**

#### TestRunTurnLLMTransportError への追加 (1 テスト)

**Test 5: `test_read_timeout_on_first_turn_stores_in_diagnostic`**
- kind=`READ_TIMEOUT`, partial_text="" (pre-stream timeout)
- `_llm_runner.run()` を直接呼ぶ (TestRunTurnLLMTransportError スタイル)
- Assert: result.answer に "READ_TIMEOUT" が含まれる
- Assert: `ctx.diagnostics.save` が呼ばれた

```python
@pytest.mark.asyncio
async def test_read_timeout_on_first_turn_stores_in_diagnostic(self) -> None:
    ctx = _make_ctx()
    orch = _make_orchestrator(ctx)
    err = _make_err(kind="READ_TIMEOUT", partial_text="")

    async def _mock_stream(*_args: object, **_kwargs: object) -> dict:
        raise err

    ctx.services.llm.stream = _mock_stream

    result = await orch._llm_runner.run("http://llm-test")
    assert "READ_TIMEOUT" in result.answer
    ctx.diagnostics.save.assert_called_once()
```
