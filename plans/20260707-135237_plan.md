# Plan: H-5 — Convert lifecycle ensure_ready failures into ToolCallResult errors

## Goal

Wrap `await self._lifecycle.ensure_ready(server_key)` in `_raw_execute()` with a `try/except (OSError, RuntimeError)` block so that startup failures are returned as structured `ToolCallResult(is_error=True, error_type="transport")` instead of escaping the tool loop.

## Scope

**In**: `scripts/shared/tool_executor.py` — add error handling around `ensure_ready()` in `_raw_execute()`.

**Out**: No changes to `factory.py`, `http_lifecycle.py`, `startup.py`, transport code, or test files not related to `ToolExecutor`.

## Assumptions

1. `_record_transport_error()` takes a `TransportError`; for `OSError`/`RuntimeError` caught here, we use `_error_result()` directly with `error_type="transport"` — same semantics.
2. `_health_registry.record_failure()` is the correct call to increment failure count when a registry exists.
3. Not catching bare `Exception` — only `OSError` and `RuntimeError` as specified by the requirement, since lifecycle start raises only these.

## Implementation Steps

1. **Wrap `ensure_ready()` in `_raw_execute()` with try/except**
   - Surround lines 256–257 with:
     ```python
     if self._lifecycle is not None:
         try:
             await self._lifecycle.ensure_ready(server_key)
         except (OSError, RuntimeError) as e:
             msg = f"Lifecycle ensure_ready failed for {server_key!r}: {e}"
             logger.error(msg)
             if self._health_registry is not None:
                 self._health_registry.record_failure(server_key)
             return self._error_result(server_key, msg, error_type="transport")
     ```
   - Criterion: `_raw_execute()` never propagates `OSError`/`RuntimeError` from `ensure_ready()`.

2. **Add tests**
   - Mock `_lifecycle.ensure_ready()` to raise `RuntimeError("startup failed")`.
   - Assert `_raw_execute()` returns `ToolCallResult(is_error=True, error_type="transport")`.
   - Assert result content contains `"Lifecycle ensure_ready failed"`.
   - Assert `_health_registry.record_failure()` is called.
   - Assert transport `execute()` is NOT called after lifecycle failure.
   - Mock `ensure_ready()` to raise `OSError`; assert same behavior.
   - Criterion: `uv run pytest tests/ -k tool_executor -v` passes.
