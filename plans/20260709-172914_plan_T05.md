# Plan: Move `stat_partial_completions` to `RuntimeStats`

## Goal

Consolidate `stat_partial_completions` into `RuntimeStats` (`ctx.stats`) so it lives alongside all other session-level statistics, removing the design inconsistency where this single field lives on `LLMClient` and requires manual None-guards at every access point.

## Scope

- `scripts/agent/context.py` — add field to `RuntimeStats`
- `scripts/shared/llm_client.py` — remove `self.stat_partial_completions` from `__init__` and the increment in `_increment_stats`
- `scripts/agent/llm_transport_errors.py` — redirect increment to `ctx.stats`
- `scripts/agent/repl.py` — redirect two read sites to `stats.stat_partial_completions` (summary dict and `_dispatch_line`)
- `scripts/agent/services/context_view.py` — redirect read to `ctx.stats`
- `scripts/agent/commands/cmd_config_stats.py` — redirect stat display to `ctx.stats`

## Assumptions

- No other code (tests, scripts outside the listed files) references `stat_partial_completions` on `LLMClient`.
- Merging into `RuntimeStats` does not change the persistence path for this stat (it is not persisted — `RuntimeStats` is in-memory only).
- The partial-completion warning in `repl.py:410` can use `ctx.stats.stat_partial_completions` directly; no need to snapshot before the turn.

## Design

1. `RuntimeStats` gains `stat_partial_completions: int = 0`.
2. `LLMClient` loses its own `stat_partial_completions`. The `_increment_stats` method in `llm_client.py` will no longer touch this field; the increment happens in `llm_transport_errors.py` via `ctx.services` instead.
3. All reads move from `ctx.services.llm.stat_partial_completions` (with `is not None` guards) to `ctx.stats.stat_partial_completions` (unconditional access).

This is a mechanical relocation with no behavioural change.

## Implementation steps

1. Add `stat_partial_completions: int = 0` to `RuntimeStats` in `context.py`.
2. Remove `self.stat_partial_completions: int = 0` from `llm_client.py:79`.
3. Remove `self.stat_partial_completions += partial_completions` from `llm_client.py:200`.
4. Change `llm_transport_errors.py:44` to `ctx.stats.stat_partial_completions += 1`.

## Risks

- Low: `repl.py:241` references `stats` (the local variable) while `repl.py:408` references `ctx.stats`. Both must use the correct binding; confusing them would cause a `NameError` or wrong value.
- Mitigation: use `ctx.stats` consistently in `_dispatch_line` (already uses `ctx`); keep `stats` alias in the session-summary block.
