# Plan: Normalize workflow approval table naming to `approvals`

## Goal
`workflow_approvals` という名称がスクリプト・テスト・ドキュメントのいずれにも残っていないことを網羅的に検証し、発見された場合は `approvals` に統一する。

## Scope
**In-Scope**:
- `scripts/agent/commands/cmd_workflow.py` の全コメント・docstring の名称確認
- `docs/05_agent_09_data-layer.md` の表記確認
- `docs/90_shared_04_db_architecture_and_schema.md` の表記確認
- `docs/05_agent_07_cli-and-commands.md` の表記確認
- `docs/05_agent_06_tool-execution-and-approval.md` の表記確認
- `docs/05_agent_03_turn-processing-flow.md` の表記確認
- `docs/90_shared_01_overview.md` の表記確認
- `docs/05_agent_10_operations-and-observability.md` の表記確認
- `tests/test_workflow_state_store.py`, `tests/test_workflow_schema.py`, `tests/test_create_schema.py`, `tests/test_startup.py` のコメント・文字列確認

**Out-of-Scope**:
- `plans/done/`, `implementations/done/` 配下の過去の作業記録（歴史的記録として保持）
- DB スキーマ変更（テーブル名は既に `approvals` で確定済み）
- `workflow.approvals.created_at` 形式の完全修飾名（DB名.テーブル名 であり正しい表記）

## Assumptions
- `workflow.sqlite` の実テーブル名は `approvals` で確定済み（`scripts/agent/workflow/state_store.py` の DDL が `INSERT INTO approvals` / `UPDATE approvals` / `SELECT * FROM approvals` で実装されていることを確認済み）
- `plans/done/` および `implementations/done/` 内の過去記録の `workflow_approvals` 参照はhistoricalとして許容される（要件明示: "unless explicitly marked historical"）
