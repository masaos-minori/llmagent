## Goal
FTS5 rebuild と reconcile_url が `COALESCE(normalized_content, content)` を使うよう修正し、トリガーの挙動と一致させる。

## Scope
**In-Scope**:
- `scripts/rag/maintenance.py` — `RagDbMaintenanceService.rebuild_fts()` の修正
- `scripts/agent/services/rag_maintenance_service.py` — `RagMaintenanceService.rebuild_fts()` と `reconcile_url()` の修正
- `tests/test_db_maintenance.py` — rebuild_fts の COALESCE 挙動を確認するテスト追加
- `docs/03_rag_05_configuration_and_operations.md` — rebuild-fts の説明更新
- `docs/03_rag_90_inconsistencies_and_known_issues.md` — 関連記述の追記

**Out-of-Scope**: DB スキーマの変更、トリガーの変更

## Assumptions
1. FTS5 `INSERT INTO chunks_fts(chunks_fts) VALUES('rebuild')` は `content = 'chunks'` 設定の場合、`chunks.content` 列を直接読み込み、`normalized_content` を考慮しない
2. `delete-all` + 明示的 `INSERT` で正しく再構築できる

## Design
```sql
-- Before (FTS5 built-in rebuild — reads chunks.content only)
INSERT INTO chunks_fts(chunks_fts) VALUES('rebuild')

-- After (explicit rebuild with COALESCE)
INSERT INTO chunks_fts(chunks_fts) VALUES('delete-all');
INSERT INTO chunks_fts(rowid, content)
    SELECT chunk_id, COALESCE(normalized_content, content) FROM chunks;
```
