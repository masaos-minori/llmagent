# Work Plan: Remove Obsolete web_search_url from Agent Configuration

## Goal

Remove `web_search_url` from agent configuration and establish `mcp_servers.web_search.url` as the authoritative web search endpoint. Eliminate duplicate sources of truth.

## Scope

**In-Scope:**
- Remove `web_search_url` from `config/agent.toml`
- Remove `web_search_url` from agent config dataclasses if present
- Remove builder logic that reads or propagates `web_search_url`
- Update docs to state that web search uses `mcp_servers.web_search.url`
- Decide and implement handling for existing `web_search_url` keys (reject as forbidden key OR emit migration warning)

**Out-of-Scope:**
- Removing the web-search MCP server
- Adding another endpoint alias
- Routing web search outside MCP

## Assumptions

1. The confirmed specification is: `web_search_url` in `agent.toml` is unnecessary; web search must use the `mcp_servers.web_search` MCP server entry.
2. Web search is already routed through the MCP server entry `mcp_servers.web_search`.
3. Option A (reject as forbidden key) is preferred over Option B (migration warning).
4. The existing plan format (see `plans/20260714-143629_plan.md`) is the canonical template.

## Design

### Change Strategy

1. **Remove** `web_search_url` from `config/agent.toml`.
2. **Remove** `web_search_url` from any AgentConfig dataclass field.
3. **Remove** builder logic that reads or propagates `web_search_url`.
4. **Add** `web_search_url` to `_FORBIDDEN_KEYS` in `config_loader.py` to reject future usage.
5. **Update** docs to state: "Web search uses the `mcp_servers.web_search` MCP server entry. The `web_search_url` configuration key is obsolete."
6. **Add** tests covering config loading with and without `web_search_url`.

### Forbidden Key Handling Decision

**Decision: Option A — Reject as forbidden key.**

Rationale:
- Cleaner than migration warning (no ambiguity about whether the key should be used)
- Forces operators to update their configs immediately
- Aligns with the principle of eliminating duplicate sources of truth

### Consistency Rules

- After changes, every mention of web search endpoint must reference `mcp_servers.web_search.url`.
- No document should describe `web_search_url` as a valid config option.
- If a document previously described both `web_search_url` and `mcp_servers.web_search`, retain only `mcp_servers.web_search`.
