## Design

**`EventBroker.publish()` の topic フィルタロジック:**

```python
def publish(self, event: dict[str, Any]) -> int:
    """Fan out event to matching subscribers. Returns delivery count."""
    delivered = 0
    event_topic: str = event.get("topic", "")
    for sub in list(self._subscribers):  # snapshot to avoid mutation during iteration
        # topics=[] means "all topics"; non-empty means filtered
        if sub.topics and event_topic not in sub.topics:
            continue
        try:
            sub.queue.put_nowait(event)
            delivered += 1
        except asyncio.QueueFull:
            logger.warning(
                "broker: queue full subscriber=%d dropping seq=%s",
                id(sub),
                event.get("seq"),
            )
    return delivered
```

**ポイント:**
- `list(self._subscribers)` でスナップショットを取る: `unsubscribe()` が並行して呼ばれた場合に `list.remove()` によるイテレーション崩壊を防ぐ
- ブローカーはイベントループ上で動作するため `threading.Lock` は不要
- `event_topic not in sub.topics` は O(n) だが topics リストは短い (通常 1–5 トピック)

**`app.py` の subscribe() との連携:**

```python
# broker.subscribe() に topic リストを渡す
sub = _broker.subscribe(list(topic))  # topic: list[str] = Query(default=[])
```

これで、全購読者に broadcast する代わりに、topic フィルタに一致する購読者にのみイベントが届く。
