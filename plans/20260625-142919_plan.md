## Design

**`docs/06_eventbus_04_dlq_offsets_and_delivery_semantics.md` の新構成:**

```markdown
## Delivery State Machine

Events transition through these states:

LIVE → ACKED      (consumer calls POST /events/{id}/ack)
LIVE → RETRY      (consumer calls POST /events/{id}/nack; delivery_failure_count increases)
RETRY → DLQ       (delivery_failure_count reaches max_retry; inline promotion via nack endpoint)
RETRY → ACKED     (consumer eventually succeeds after partial failures)
DLQ → LIVE        (operator calls POST /dlq/{id}/requeue; dlq_requeue_count increases)

## Counters

| Counter | When incremented | Resets? |
|---|---|---|
| delivery_failure_count | On each POST /events/{id}/nack | No (accumulates) |
| dlq_requeue_count | On each POST /dlq/{id}/requeue | No (accumulates) |

Note: retry_count is deprecated. Use delivery_failure_count.

## DLQ Promotion

Promotion happens **inline** when delivery_failure_count reaches max_retry (via the nack endpoint).
The background loop runs every 60s as a safety-net sweep only.

## Offset semantics

Offsets are written when:
1. Consumer explicitly acks via POST /events/{id}/ack?consumer_id=X
2. On SSE disconnect (consumer_id present)

Mid-stream checkpoint writes are removed (req #29).
Offset now reflects "last successfully processed event", not "last delivered event".

## Delivery Guarantee

| Property | Value |
|---|---|
| Delivery guarantee | At-least-once (until acked) |
| Duplicate suppression on publish | Yes (UNIQUE event_id) |
| Re-delivery on crash | Up to last acked seq (not last delivered seq) |
```
