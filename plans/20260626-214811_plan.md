## Goal

Implement index-backed outline retrieval for MDQ by reading from indexed chunks instead of reparsing files, returning hierarchy metadata and chunk IDs.

## Scope

**In-Scope**:
- Read outline data from indexed chunks table
- Return hierarchy information: heading, level, heading_path, chunk_id, start_line, end_line
- Add max_depth parameter to limit heading depth
- Add max_items parameter to cap result count
- Return stale index warning if source file changed after indexing

**Out-of-Scope**:
- Adding new tools or features
- Changes to other MCP servers' outline behavior

## Assumptions

1. max_depth default = 6 (reasonable for most Markdown documents)
2. max_items default = 500 (cap for large documents)
3. Stale index warning: compare documents.mtime_ns with documents.indexed_at

## Affected Areas & Tool Evidence

- **Affected Files**:
  - `scripts/mcp/mdq/models.py` — update OutlineRequest with max_depth, max_items; update OutlineResponse with hierarchy metadata
  - `scripts/mcp/mdq/service.py` — rewrite outline method to query chunks table, add stale check
  - `scripts/mcp/mdq/tools.py` — update outline tool definition

- **Blast Radius**: Medium — changes outline response format from flat list to structured hierarchy. Existing consumers expecting plain text will need updates.

## Implementation Steps

1. **Phase 1: Update models.py**
   - OutlineRequest: add max_depth, max_items fields
   - OutlineResponse: replace headings list with list of heading metadata objects (heading, level, heading_path, chunk_id, start_line, end_line)

2. **Phase 2: Implement index-backed outline query**
   - Query chunks table for source_path matching req.path
   - Filter by max_depth if specified
   - Cap results at max_items

3. **Phase 3: Add stale index warning**
   - Compare documents.mtime_ns with documents.indexed_at for the file
   - If mtime > indexed_at, add stale warning to response

4. **Phase 4: Update tools.py schema**
   - Update outline tool definition to match new OutlineRequest/Response
