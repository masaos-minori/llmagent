## Design

### Phase A — 横断的改善（影響小・具体的）

**A1 — print() OutputPort 移行（項目1-4、16 件）**:

現状:
- `agent/tool_output.py`: 8 件（emit_tool_call, emit_tool_result, emit_tool_approval 等）
- `agent/repl.py`: 5 件（shutdown, unknown_command, warn 等）
- `agent/services/export_formatter.py`: 2 件
- `shared/plugin_registry.py`: 1 件 (pong)

対応方針:
- `tool_output.py` の print() は既存の `OutputPort` / `CliOutputPort` に移行、または `emit_*` 関数の引数に `output: OutputPort` を追加
- `repl.py` の print() は `cli_view.py` の `CLIView` 経由に移行
- `export_formatter.py` の print() は呼び出し側（`cmd_ingest.py`）の OutputPort に委譲
- `plugin_registry.py` の print("pong") は logging に変更

**A2 — `create_schema.py` の `migrate_schema()` 削除（項目4-2）**:

現状: `create_schema.py:5` コメントに "Existing tables are protected by IF NOT EXISTS for idempotent re-runs." と明記されており、`IF NOT EXISTS` は意図的な idempotent 設計。
`migrate_schema()` 関数（L198-211）は ALTER TABLE 試行による incremental migration であり、要望書の「旧スキーマ救済コード」にあたる。

対応方針:
- `migrate_schema()` 関数（L198-211）を削除するのみ
- `IF NOT EXISTS` はそのまま維持（idempotent 再実行の保証）
- 呼び出し元に `migrate_schema()` を使っている箇所がないか `rg` で確認してから削除
