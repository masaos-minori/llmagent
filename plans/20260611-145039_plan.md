# agent/memory/ レイヤー fail-fast 設計リファクタリング計画書

## Goal

`agent/memory/` 配下 10 ファイルから `except Exception`・`dict[str,Any]` 内部使用・`str(…)` 無条件変換・silent recovery・sentinel 返却を除去し、DTO / Enum / 専用例外による fail-fast 設計に置き換える。

---

## Scope

- `scripts/agent/memory/` 配下 10 ファイル (types.py / store.py / embedding_client.py / jsonl_store.py / mapper.py / extract.py / ingestion.py / retriever.py / injection.py / services.py)
- 新設 4 ファイル: `models.py` / `enums.py` / `exceptions.py` / `ports.py`
- 呼び出し元の軽微な追従修正: `agent/factory.py` / `agent/commands/cmd_memory.py` / `agent/context.py`
- 既存テスト修正 + 新規エラーパステスト追加

---

## Assumptions

1. `types.py` は現在存在する (`MemoryEntry`, `MemoryQuery`, `MemoryHit`, `EmbeddingResult`, `SourceType`, `EmbeddingErrorKind` が定義済み)。
2. `MemoryType` Enum が存在しないため新設が必要。`memory_type: str` フィールドは `MemoryType` Enum に移行する。
3. `MemoryEntry` を frozen dataclass にするか mutable のままにするか — `store.py` が `entry.created_at = now` / `entry.updated_at = now` で直接書き込んでいるため、frozen 化にはファクトリ関数を挟む必要がある。計画では frozen + factory helper を採用する。
4. `JsonlMemoryStore.read_all()` は `strict=False` / quarantine 分岐を持つが、instruction は "fail immediately on malformed JSONL" を要求する。`quarantine_path` パラメータを除去し、常に例外送出に変更する。
5. `mapper.py` の `.get(…, default)` パターン: DB 行の必須フィールド (`memory_id`, `content`) は KeyError を許容し、optional フィールドは None と空文字を区別した明示的マッピングにする。
6. `embedding_client.py` の `_fetch_embedding` にある `except Exception` を `except (httpx.RequestError, Exception)` から適切な型に絞る。`resp.json()` を `orjson.loads(resp.content)` に変更する。
7. `extract.py` の `LLMMessage` 依存を `HistoryMessage` DTO (memory-local) に置き換える。LLMMessage は TypedDict で typed access が可能だが、memory 層が `shared.types` に直接依存することを避ける。
8. `services.py` / `ingestion.py` の `LLMMessage` 依存も同様に `HistoryMessage` に置き換える。
9. `factory.py` は memory サービスの組み立てを行うため、API 変更に伴う修正が必要。
10. `mypy --strict` は現時点では未達成。本計画の完了条件は「新規 mypy エラーを増やさない + 本タスクの変更箇所は strict-compatible にする」とする。

## Design

### 新設ファイル概要

**`agent/memory/enums.py`**:
```python
class MemoryType(StrEnum):
    SEMANTIC = "semantic"
    EPISODIC = "episodic"

class RetrievalMode(StrEnum):
    FTS = "fts"
    KNN = "knn"
    HYBRID = "hybrid"

class ExtractionDecision(StrEnum):
    ACCEPT = "accept"
    REJECT_TOO_SHORT = "reject_too_short"
    REJECT_NO_KEYWORDS = "reject_no_keywords"
    REJECT_DEDUP = "reject_dedup"
```

**`agent/memory/exceptions.py`**:
```python
class MemorySchemaError(ValueError): ...          # 型・フィールド検証失敗
class MemoryStorageError(RuntimeError): ...       # DB 書き込み失敗
class JsonlFormatError(ValueError): ...           # JSONL パース失敗
class MemoryConsistencyError(RuntimeError): ...   # memories/fts/vec 不整合
class EmbeddingTransportError(RuntimeError): ...  # HTTP レベル失敗
class EmbeddingProtocolError(ValueError): ...     # レスポンスのスキーマ違反
class ExtractionError(RuntimeError): ...          # 抽出処理の失敗
class UnknownMemoryTypeError(ValueError): ...     # 未知の memory_type 値
class InjectionValidationError(ValueError): ...   # 無効なクエリ等
```

**`agent/memory/models.py`** (追加 DTO):
```python
@dataclass(frozen=True)
class HistoryMessage:
    role: str   # "user" | "assistant" | "system" | "tool"
    content: str

@dataclass(frozen=True)
class JsonlRecord:
    memory_id: str; memory_type: MemoryType; ...  # MemoryEntry の JSONL 表現

@dataclass(frozen=True)
class ConsistencyReport:
    memories: int; fts: int; vec: int

@dataclass(frozen=True)
class EmbeddingRequest:
    text: str; query_prefix: str

@dataclass(frozen=True)
class EmbeddingResponse:
    embedding: list[float]; model: str | None = None

@dataclass(frozen=True)
class InjectionSnippet:
    prefix: str; text: str; memory_id: str; memory_type: MemoryType
```

**`agent/memory/ports.py`**:
```python
class MemoryOutputPort(Protocol):
    def emit_persist(self, memory_id: str, memory_type: str) -> None: ...
    def emit_skip_dedup(self, memory_id: str) -> None: ...
```

### types.py 変更

- `MEMORY_TYPES: frozenset[str]` → `MemoryType` Enum に一本化
- `MemoryEntry.memory_type: str` → `MemoryType`
- `MemoryEntry` を frozen dataclass に変更; `store.py` の代入を `dataclasses.replace()` に変更

### mapper.py 変更

`row_to_entry(row)` を `row_to_entry(row: sqlite3.Row | dict[str, str | int | float | None])` に変更:
- `d["memory_id"]`, `d["content"]` — KeyError は MemorySchemaError に変換
- `d.get("source_type", "conversation")` → `d["source_type"]` (KeyError → MemorySchemaError)
- `float(d.get("importance", 0.5))` → `d["importance"]` を直接 float キャスト; None → MemorySchemaError

### embedding_client.py 変更

`_fetch_embedding` の `except Exception` → `except (httpx.RequestError, httpx.HTTPStatusError)` のみ。`resp.json()` → `orjson.loads(resp.content)`。Response body を `EmbeddingResponse` DTO で検証。

### store.py 変更

- `count_vec()` / `count_entries()` / `count_prunable()`: `except Exception` → `except (sqlite3.OperationalError, sqlite3.DatabaseError)`。0 の sentinel 返却を除去し例外伝播。
- `check_consistency()`: `dict[str,int]` → `ConsistencyReport` DTO。`except Exception` → `except sqlite3.OperationalError`。`-1` sentinel → MemoryConsistencyError。

### retriever.py 変更

`_recency_boost(created_at)` で `ValueError` → `MemorySchemaError` を raise。呼び出し元でハンドリング。`float(d.pop("distance", 999.0))` → `d.pop("distance")` (KeyError 許容 → MemorySchemaError)。

## Implementation Steps

各ステップは独立コミット可能。

1. **[新設] enums.py / exceptions.py / models.py / ports.py を作成**
   - 既存コードへの変更なし
   - コミット: `feat(memory): add enums, exceptions, models, ports foundation`

2. **[types.py] MemoryType Enum 追加 + frozen MemoryEntry**
   - `MemoryType` Enum を enums.py から import、`memory_type: str` → `MemoryType`
   - `MemoryEntry` を frozen dataclass に変更
   - `MemoryQuery.memory_type: str | None` → `MemoryType | None`
   - コミット: `refactor(memory): MemoryType Enum, frozen MemoryEntry`

3. **[mapper.py] .get(default) 除去 + 型付き行マッピング**
   - `dict[str,Any]` → 型付きアクセス、KeyError → MemorySchemaError
   - コミット: `refactor(memory): remove .get(default), fail-fast row mapping`

4. **[store.py] except Exception 除去 + ConsistencyReport DTO**
   - `count_vec/count_entries/count_prunable`: `except Exception` → `except (sqlite3.OperationalError, sqlite3.DatabaseError)`
   - `check_consistency()` → `ConsistencyReport` DTO
   - `frozen MemoryEntry` 対応: `dataclasses.replace()` を使った stamp helper
   - コミット: `refactor(memory): ConsistencyReport DTO, remove except Exception in store`

5. **[jsonl_store.py] strict=False / quarantine 除去**
   - `quarantine_path` パラメータ除去
   - `read_all(strict=True)` 固定 (パラメータ除去)
   - `JsonlRecord` DTO 使用
   - コミット: `refactor(memory): fail-fast JSONL, remove quarantine`

6. **[embedding_client.py] except Exception 除去 + EmbeddingResponse DTO**
   - `except Exception` → 特定例外
   - `resp.json()` → `orjson.loads(resp.content)` + `EmbeddingResponse` DTO 検証
   - コミット: `refactor(memory): EmbeddingResponse DTO, specific exceptions in embedding_client`

7. **[extract.py] LLMMessage → HistoryMessage + str() 除去**
   - `from shared.types import LLMMessage` を除去
   - `msg.get("content") or ""` + `str(content_raw)` → `msg.content` (型付きアクセス)
   - コミット: `refactor(memory): HistoryMessage DTO, remove str() conversion in extract`

8. **[ingestion.py] except Exception 除去 + embedding=None 継続除去**
   - `except Exception` → `except (sqlite3.OperationalError, sqlite3.IntegrityError)`
   - `embedding=None` 継続 → embedding 取得失敗時は `EmbeddingTransportError` を raise または明示的スキップ記録
   - `LLMMessage` → `HistoryMessage`
   - コミット: `refactor(memory): fail-fast ingestion, HistoryMessage`

9. **[retriever.py] 無効タイムスタンプ → MemorySchemaError**
   - `_recency_boost` の silent 0.0 返却 → `MemorySchemaError` raise
   - `d.pop("bm25_rank", 0.0)` → `d.pop("bm25_rank")` (KeyError → MemorySchemaError)
   - コミット: `refactor(memory): fail-fast timestamp/row access in retriever`

10. **[injection.py] 空クエリ → ValidationError + InjectionSnippet DTO**
    - `if not query.strip(): return []` → `raise InjectionValidationError(...)`
    - `embedding=None` 後の暗黙的フォールバックを明示化
    - `InjectionSnippet` DTO を返却リストに使用
    - コミット: `refactor(memory): InjectionSnippet DTO, fail-fast validation in injection`

11. **[services.py] LLMMessage → HistoryMessage + DTO ベース API**
    - `from shared.types import LLMMessage` 除去、`HistoryMessage` に変更
    - コミット: `refactor(memory): HistoryMessage in services facade`

12. **[callers] factory.py / cmd_memory.py 追従修正**
    - `ConsistencyReport` DTO へのアクセス変更
    - `MemoryType` Enum の使用
    - コミット: `fix(memory): update callers for DTO/Enum changes`

13. **[tests] エラーパステスト追加 + 既存テスト修正**
    - malformed JSONL, invalid memory row, invalid embedding response schema
    - invalid timestamp in retrieval, embedding timeout, memory_links insert failure
    - コミット: `test(memory): add error-path tests for memory layer`

14. **[静的検査] mypy / ruff / lint-imports / pytest 最終確認**

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `MemoryEntry` の frozen 化により `store.py` の `entry.created_at = now` 代入がビルドエラーになる | High | Step 4 で `_stamp_entry(entry, now)` factory helper を `dataclasses.replace()` で実装し、`add()` / `upsert()` を呼ぶ前に呼び出す |
| `retriever.py` の `_recency_boost` が `MemorySchemaError` を raise するようになると、DB に不正なタイムスタンプが混入している既存環境でテストが壊れる | Medium | Step 9 で例外を raise する前に既存データのタイムスタンプを `_verify_timestamps` 関数で検証するヘルパーを追加し、移行期は WARNING ログ + 0.0 返却の移行フラグを設ける。DOD には「既存テストが 0 件以上増加しない」を追加 |
| `injection.py` の `embedding=None` 暗黙フォールバック除去により、embedding サービス停止時に injection 全体が失敗する | Medium | `on_user_prompt` 側で `EmbeddingTransportError` を catch して FTS-only フォールバックに明示的に落とす。FTS フォールバックの使用はログに記録 |
| `LLMMessage` → `HistoryMessage` 置き換えが `factory.py` 側の `on_session_stop()` 呼び出しを壊す | Low | Step 12 の callers 修正で `factory.py` を同時更新。CI で pytest が guards として機能 |
| `mapper.py` の `.get(default)` 除去により、過去に保存された不完全な DB 行 (summary="" など) が `MemorySchemaError` になる | Low | summary は空文字を許容する optional フィールドとして `d.get("summary", "")` を `d.get("summary") or ""` に変更する（None vs 空文字の区別）。必須フィールドのみ KeyError を MemorySchemaError に変換する |
