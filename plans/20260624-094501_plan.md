# Plan: Shared Structured Log Keys Across RAG Lifecycle

**Requirement:** `requires/20260624_13_require.md`
**Task size:** Path B (multi-file logging standardization)

---

## Goal

Add consistent structured log fields (`url`, `doc_id`, `chunk_id`, `source_type`, `stage_name`) across crawler, chunk splitter, ingester, and pipeline logs so one URL can be correlated end-to-end.

---

## Current State

- `crawler.py:79,84,110,121` — logs `url`, `path`, `out.name` but inconsistently
- `chunk_splitter.py` — logs file paths without consistent `url`/`chunk_id` keys
- `ingester.py` — logs URL-level messages but no `doc_id`/`chunk_id` correlation
- `pipeline.py` — logs stage names but no `url`/`doc_id` in retrieval logs

---

## Scope

**In:**
- `scripts/rag/ingestion/crawler.py` — standardize: add `url=`, `source_type=`
- `scripts/rag/ingestion/chunk_splitter.py` — add `url=`, `chunk_id=`, `stage_name=chunking`
- `scripts/rag/ingestion/ingester.py` — add `doc_id=`, `chunk_id=`, `url=`
- `scripts/rag/pipeline.py` — add `stage_name=` to existing stage log lines
- Operational documentation — add logging guidance section

**Out:**
- Full observability stack integration
- Log shipping infrastructure

---

## Canonical Log Fields

| Field | Type | Populated by |
|---|---|---|
| `url` | str | crawler, chunk_splitter (from src filename), ingester, pipeline |
| `doc_id` | int | ingester (after upsert), pipeline |
| `chunk_id` | int | ingester, chunk_splitter |
| `source_type` | `"file"` / `"http"` | crawler |
| `stage_name` | str | pipeline stage |
