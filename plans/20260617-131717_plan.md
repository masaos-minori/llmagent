## Assumptions

1. `FusionStage` is instantiated in `rag/pipeline.py:182` as `FusionStage()` with no config passed ‚Äî it always uses default `rrf_k=60`.
2. `_dedup_hits()` (repository.py:307) is called nowhere ‚Äî only imported in repository.py line 11; it is dead code.
3. `use_rrf` flows through: config TOML ‚Üí `RagPipelineConfig` (MCP models) / `SearchConfig` (rag models) ‚Üí `build_rag_cfg_adapter()` ‚Üí `RagPipeline` cfg ‚Üí but never read by `FusionStage`.
4. The non-RRF fallback (`_dedup_hits`) performs chunk_id-based deduplication with `rrf_score=0.0` ‚Äî it does not re-rank, only deduplicates.
5. RRF is the standard fusion method; removing it would be a behavioral change only for users who explicitly set `use_rrf=False` (which currently has no effect anyway).
