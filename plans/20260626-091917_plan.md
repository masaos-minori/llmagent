# Plan: Convert dangerous production security warnings into hard startup failures

Source: `requires/20260625_10_require.md`

---

## Goal

`agent/repl_health.py` で現在 warning のみの危険な production security 状態をすべて列挙し、hard startup failure に昇格させて自動化環境での見落としを排除する。

---

## Scope

**In-Scope**
- `scripts/agent/repl_health.py`: warning-only の security チェックを production_mode=True で RuntimeError に昇格
- 対象: 空 `workflow_allowlist`（cicd）、`shell_sandbox_backend="none"`（既存: 確認済みでも補強）、その他 fail-open 状態
- ログメッセージ・オペレータ向け guidance の更新
- テスト追加

**Out-of-Scope**
- MCP サーバー自体の変更
- 開発モード（production_mode=False）の動作変更

---

## Assumptions

1. `repl_health.py` に `fail_closed_empty` / `fail_open_empty` リストが存在し、`fail_open_empty` にある項目が warning-only で留まっている（line 601 以降）。
2. production_mode=True でのみ hard error にする設計方針は維持する。
3. `shell_sandbox_backend="none"` は既に hard error（line 484-486）。
4. `cicd.workflow_allowlist` は現状 `fail_open_empty` または未チェック。
