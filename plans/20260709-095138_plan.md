# Plan: Remove HTTP URL hot reload for MCP servers

Source requirement: `requires/20260708_25_require.md` (H-3)

## Goal

Stop `/reload` from ever assigning `old_srv.url = new_srv.url`; instead
report MCP URL differences as `mcp/<server_key>.url` in
`ConfigReloadOutcome.needs_restart`, rendered as
`[RESTART] - mcp/<server_key>.url`.

## Scope

In scope:
- `scripts/agent/services/config_reload.py` — same change as H-1/H-2
  (`plans/20260709-094640_plan.md`, `plans/20260709-094926_plan.md`); this
  requirement is the URL-specific slice of that same change and fixes the
  exact `needs_restart` string format.
- Reload command tests (`tests/test_config_reload.py`).

Out of scope:
- `scripts/shared/http_transport.py` — verified below that `HttpTransport`
  already captures `base_url` once at construction time and is never updated
  by `config_reload.py`; no code change needed there, only a confirming test.

## Assumptions

1. `HttpTransport` snapshots its base URL at construction and is never
   touched by `config_reload.py` — verified by
   `grep -n "base_url\|_base_url" scripts/shared/http_transport.py` (no
   reload-triggered writes found) and by the fact that `config_reload.py`
   never holds a reference to any `HttpTransport` instance, only to
   `McpServerConfig` objects.
2. This requirement, H-1, and H-2 are satisfied by one shared implementation
   (`_classify_mcp_server_changes()`, revised in the H-1 plan to emit
   field-qualified entries). No separate function is needed for URL handling
   specifically.
3. `cmd_config.py:76` renders `needs_restart` items verbatim as
   `f"  [RESTART] - {item}"`, so the item string itself must equal
   `mcp/<server_key>.url` to satisfy the acceptance criterion — verified by
   reading `scripts/agent/commands/cmd_config.py:69-76`.

## Unknowns

None. Resolved by cross-referencing the H-1 plan design and `cmd_config.py`'s
rendering code; no blocking unknowns remain.

## Design

No new production design beyond H-1's `_classify_mcp_server_changes()`
(`plans/20260709-094640_plan.md` § Design, already revised to emit
`mcp/<key>.url` for URL changes and to never write `old_srv.url`).

This plan's test addition:

```python
def test_url_change_reports_field_qualified_restart_entry(self) -> None:
    svc, old_srv = self._make_svc()  # url="http://localhost:8080"
    new_srv = McpServerConfig(
        transport=TransportType.HTTP,
        url="http://127.0.0.1:9090",
        cmd=[],
    )
    result = self._run(svc, {"svc": new_srv})

    assert "mcp/svc.url" in result.needs_restart
    assert old_srv.url == "http://localhost:8080"  # unchanged
    assert not any("url" in item for item in result.applied)
    assert not any("url" in item for item in result.deferred)
```

## Risks

- **Risk**: If H-1's implementation lands with a different `needs_restart`
  string format than `mcp/<key>.url` (e.g. due to independent, out-of-order
  implementation), this requirement's acceptance criteria will silently fail
  only at test time, not at review time.
  → **Mitigation**: The H-1 plan file was updated in place (see
  `plans/20260709-094640_plan.md` § Design) to use the exact format this
  requirement needs, before any of the three plans were implemented — so
  there is only one design to implement, not three divergent ones.
- **Risk**: `http_transport.py` is declared out of scope based on static
  analysis; if a future refactor gives `HttpTransport` a live reference to
  `McpServerConfig`, this assumption would silently break.
  → **Mitigation**: Low likelihood; flagged in Assumptions for visibility if
  `http_transport.py` is touched later.
