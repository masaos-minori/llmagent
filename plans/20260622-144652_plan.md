# Plan: テスト間依存リスクの解消 — test_lifecycle.py

## Goal

`tests/test_lifecycle.py` の `TestStartHttpSubprocess` クラスで繰り返される
`MockClient`/`client_instance`/`mock_resp` の生成パターンをヘルパー関数に抽出し、
各テストが確実に独立した新しいモックインスタンスを使用するようにする。

## Scope

**In-Scope:**
- `tests/test_lifecycle.py` — `_make_http_client_mock()` ヘルパー関数の新設
- `TestStartHttpSubprocess` クラスのテスト (約 5 件) での適用

**Out-of-Scope:**
- 実装コードの変更
- `TestEnsureReady`・`TestShutdownIdle` 等の他クラスの変更 (既に独立したモックを使用)

## Assumptions

1. 現時点でモジュールレベルの共有 `mock_http` 変数は存在しない。各テストは既に独立した
   `MagicMock()` を生成しているため、実際の依存は発生していない。
2. 問題は「コードの重複」にある: `TestStartHttpSubprocess` の複数テストが同一のモック生成
   パターン (MockClient + client_instance + mock_resp の 3 段構成) をコピーペーストしている。
3. ヘルパー関数に抽出することで、各呼び出し時に新しいインスタンスを生成し、
   テスト間の意図しない共有を構造的に防止できる。

## Design

### 現状の重複パターン (テストごとにコピーペーストされている)

```python
client_instance = AsyncMock()
client_instance.get = AsyncMock(return_value=mock_resp)
MockClient.return_value.__aenter__ = AsyncMock(return_value=client_instance)
MockClient.return_value.__aexit__ = AsyncMock(return_value=False)
```

### 提案するヘルパー関数

```python
def _wire_http_client(MockClient, status_code: int = 200):
    """Wire up AsyncClient mock to return a response with given status code."""
    mock_resp = MagicMock()
    mock_resp.status_code = status_code
    client_instance = AsyncMock()
    client_instance.get = AsyncMock(return_value=mock_resp)
    MockClient.return_value.__aenter__ = AsyncMock(return_value=client_instance)
    MockClient.return_value.__aexit__ = AsyncMock(return_value=False)
    return client_instance, mock_resp
```

各テストで:
```python
with patch("agent.http_lifecycle.httpx.AsyncClient") as MockClient:
    _wire_http_client(MockClient, status_code=200)
    await mgr.start_http_subprocess("s", cfg)
```

### _make_mock_proc() ヘルパー関数

```python
def _make_mock_proc(exit_code: int | None = None) -> MagicMock:
    """Create a mock subprocess with configurable poll() return value."""
    mock_proc = MagicMock()
    mock_proc.poll.return_value = exit_code
    mock_proc.stderr = None
    return mock_proc
```
