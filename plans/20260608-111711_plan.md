# Refactoring Phase 3: repl/lifecycle/cli_view の責務分離 — Work Plan

Source instruction: `instructions/20260608_03_instruction.md`

## Goal

`lifecycle.py` に統一 `LifecycleState` を導入し、`http_lifecycle.py` の `StartupFailure` を構造化例外に昇格させ、`stdio_lifecycle.py` の状態管理を `TransportHandle` に統合することで、ライフサイクル層の診断性と一貫性を向上させる。

## Scope

**In:**
- `lifecycle.py`: `LifecycleState` enum を追加し、`get_transport_state()` の戻り値を統一する
- `http_lifecycle.py`: `HttpStartupError(RuntimeError)` を追加し、`StartupFailure` を wrapper として使用する
- `stdio_lifecycle.py`: `TransportHandle` dataclass で transport + state + last_error を統合する
- 上記 3 ファイルのテスト更新

**Out:**
- `repl.py` の `app_runner.py` 分割（大規模変更、別フェーズ）
- `repl_health.py` のファイル分割（`health_probe.py`/`tool_definition_validator.py`/`watchdog.py`）— 別フェーズ
- `cli_view.py` の `cli_presenter.py` 分割 — 別フェーズ
- `TransportState` enum の削除（後方互換のため残存）

## Assumptions

1. `TransportState` は `lifecycle.py` から `stdio_lifecycle.py` へインポートされている（既存）。
2. `StartupFailure` は現在 `dataclass` として実装済み（`http_lifecycle.py:24`）。
3. `lifecycle.py` の `get_transport_state()` は HTTP では `None` を返し、stdio では `TransportState` を返している（非対称）。
4. 既存テスト（57 件: `test_lifecycle.py`/`test_mcp_server_base.py` 等）がガードとして機能する。

## Design

### lifecycle.py

```python
class LifecycleState(Enum):
    RUNNING = "running"
    STOPPED = "stopped"
    FAILED  = "failed"
    UNKNOWN = "unknown"
```

`get_transport_state()` が `LifecycleState | None` を返すよう変更（または `LifecycleState` に統一し `None` を廃止）。

### http_lifecycle.py

```python
class HttpStartupError(RuntimeError):
    def __init__(self, failure: StartupFailure) -> None:
        self.failure = failure
        super().__init__(failure.reason)
```

`RuntimeError(failure.reason)` を `raise HttpStartupError(failure)` に置き換え。

### stdio_lifecycle.py

```python
@dataclass
class TransportHandle:
    transport: StdioTransport | None
    state: TransportState
    last_error: str | None = None
```

`_stdio_procs: dict[str, StdioTransport]` と `_transport_states: dict[str, TransportState]` を `dict[str, TransportHandle]` に統合。
