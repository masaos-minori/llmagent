## Title
[Medium] Document `/replay` pagination semantics

## Priority
Medium

## Labels
documentation, eventbus, replay, api

## Dependencies
H-04

## Background
`GET /replay` supports `limit` and `offset`. For `format=json`, the endpoint returns a paginated response object, not a raw array.

Existing tests already validate the paginated response shape.

## Scope
Update:
- `docs/06_eventbus_02_http_api_and_runtime.md`
- `docs/06_eventbus_06_reference_api.md`

## Required Changes
- Document query parameters:
  - `since_seq`
  - `format`
  - `limit`
  - `offset`
- Document validation:
  - `limit >= 1`
  - `limit <= 1000`
  - `offset >= 0`
- Document JSON response shape:
  - `total`
  - `limit`
  - `offset`
  - `items`
- Clarify SSE behavior when pagination parameters are used.

## Acceptance Criteria
- Replay documentation matches implementation and tests.
- Clients can implement paginated replay correctly.
