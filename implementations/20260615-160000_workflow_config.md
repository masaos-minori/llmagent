# Implementation Plan: Workflow Config (Step 2)

## Goal

Create `config/workflows/default.json` with stage definitions, timeouts, and retry policies for the agent turn execution workflow.

## Scope

**In:**
- `config/workflows/default.json`: Stage definitions (plan/execute/verify/retry), timeouts, retry policies

**Out:**
- Workflow loader implementation (Step 4)
- Multiple workflow files (Phase 2+)

## Assumptions

1. `ConfigLoader._resolve_path()` does not join with `config_dir` for Path objects (per plan Assumption 6).
2. The default workflow is the only one in Phase 1; future phases add environment-specific files.
3. JSON Schema validation uses the `jsonschema` package (already in dependencies per pyproject.toml).

### Procedure

#### Create `config/workflows/default.json`

Structure:
```json
{
  "version": "1.0",
  "name": "default",
  "stages": [
    {
      "id": "plan",
      "timeout_sec": 30,
      "retry": {
        "max_attempts": 1,
        "backoff_multiplier": 1.0,
        "retryable_errors": []
      }
    },
    {
      "id": "execute",
      "timeout_sec": 120,
      "retry": {
        "max_attempts": 3,
        "backoff_multiplier": 2.0,
        "retryable_errors": ["LLMTransportError", "HTTP_STATUS_RETRYABLE"]
      }
    },
    {
      "id": "verify",
      "timeout_sec": 60,
      "retry": {
        "max_attempts": 1,
        "backoff_multiplier": 1.0,
        "retryable_errors": []
      }
    },
    {
      "id": "retry",
      "timeout_sec": 120,
      "retry": {
        "max_attempts": 2,
        "backoff_multiplier": 2.0,
        "retryable_errors": ["LLMTransportError"]
      }
    }
  ],
  "global_timeout_sec": 300,
  "on_halt": "rollback"
}
```

### Details

- `version` tracks workflow definition changes; compared against `tasks.workflow_version` at execution time
- `global_timeout_sec` is the maximum total time for all stages combined
- `on_halt` specifies behavior when max retries exceeded: `"rollback"` (current default) or `"persist"` (Phase 2+)
- Retry policies: `max_attempts` includes initial run; `backoff_multiplier` controls exponential delay between retries
- `retryable_errors` lists exception class names that trigger retry; empty list = no retry
