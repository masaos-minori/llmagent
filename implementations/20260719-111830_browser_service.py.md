## Goal

Create `scripts/mcp_servers/browser/service.py` implementing `BrowserService` — the domain
logic for fetching a URL, enforcing a fail-closed domain allowlist plus IP-literal/loopback
rejection, parsing HTML to visible text, and truncating with a flag — following the same
**allowlist-and-reject** / **truncate-with-flag** shape as `ShellService` in
`scripts/mcp_servers/shell/service.py` (342 lines, verified).

## Scope

**In scope**
- `BrowserService.__init__(self, cfg: BrowserConfig) -> None` storing `allowed_domains`,
  `max_response_kb`, `timeout_sec`.
- `_check_domain(url: str) -> None` — parse hostname via `urllib.parse.urlsplit`, verify it (a)
  is not an IP literal and not loopback/link-local (defense in depth, per plan Risks — always
  enforced, not configurable), and (b) matches `allowed_domains` (fail-closed: empty list denies
  everything). Raises `BrowserAuthorizationError` on any failure.
- `async fetch(self, req: BrowserFetchRequest) -> BrowserFetchResponse` — the orchestrating
  method: domain check → `httpx.AsyncClient` GET with `timeout_sec` → HTML→text extraction via
  BeautifulSoup → truncation with flag.
- `_extract_text(html: str) -> str` — parse with `BeautifulSoup(html, "lxml")`, strip
  `<script>`/`<style>` tags, join visible block-level text.
- `_truncate(text: str, max_kb: int) -> tuple[str, bool]` — byte-based truncation with a
  `truncated: bool` flag, mirroring `SubprocessRunner.truncate_output`'s
  slice-bytes-before-decode approach (`shell/service.py:270-273` calls
  `self._subprocess_runner.truncate_output(...)` — the actual truncation implementation lives in
  `shell/subprocess_runner.py`, not `shell/service.py` itself; confirm exact byte-slicing logic
  there before writing Browser's equivalent if bit-for-bit parity with shell's UTF-8-safe slicing
  is wanted).
- `fmt_fetch(self, args: ToolArgs) -> str` — dispatch-handler wrapper formatting
  `BrowserFetchResponse` as plain text for the LLM, mirroring `fmt_run_command`/
  `_format_run_result` (`shell/service.py:294-321`).
- `get_dispatch_table(self) -> dict[str, Callable[[ToolArgs], Awaitable[str]]]` returning
  `{"browser_fetch": self.fmt_fetch}`, mirroring `shell/service.py:323-329`.
- `build_service(cfg: BrowserConfig) -> BrowserService` factory with a startup warning when
  `allowed_domains` is empty, mirroring `build_service(policy)` in `shell/service.py:332-342`.

**Out of scope**
- No JS execution / rendering (per plan Assumption 6 and Risks — explicitly a documented
  limitation, not a gap to fix here).
- No `WebCrawler` reuse from `scripts/rag/ingestion/crawler.py` (per plan Assumption 5 — Browser
  MCP gets its own small fetch+parse function; `crawler.py` is referenced for pattern only:
  confirmed at `crawler.py:25` (`from bs4 import BeautifulSoup, Tag`) and
  `crawler.py:309,378` (`BeautifulSoup(html, "lxml")`) that `lxml` is the parser already used
  project-wide for this purpose — reuse the same parser string for consistency, not a new
  dependency).
- No sandboxing/subprocess/resource-limit logic (shell-specific; not applicable to an outbound
  HTTP fetch).
- No audit-log file-writing (git-mcp precedent, per `docs/04_mcp_04_05_git.md`, has none either;
  standard call-level logging via `mcp_servers.audit._audit_log` in `server.py`'s `call_tool`
  endpoint is sufficient — no dedicated `_write_audit_log` method needed in `BrowserService`
  unless the plan is revisited to require one; not called for in Design/Affected areas).

## Assumptions

1. Verified: `shell/service.py:100-125` (`_check_command`) is the allowlist-and-reject template
   — parses input, compares against an allowlist, raises a typed `*AuthorizationError` on
   mismatch. Browser's `_check_domain` follows the same raise-on-mismatch shape but checks a
   hostname against `allowed_domains` instead of a command basename against
   `command_allowlist`.
2. Verified: `shell/service.py:271-273` truncates **before** decoding
   (`self._subprocess_runner.truncate_output(stdout_b, stderr_b, max_output_bytes)`, then decodes
   at `shell/service.py:276-277`) specifically to avoid slicing a multibyte UTF-8 character in
   half. Browser's `_truncate` must do the same: encode the extracted text to bytes, slice, then
   decode with `errors="replace"` (or slice on the original bytes response body before decoding,
   whichever is simpler given BeautifulSoup already needs decoded text to parse — if truncation
   is applied post-parse on the extracted text string, encode→slice→decode is the correct order,
   not naive `text[:n]` character slicing, to avoid the same multibyte-cutoff class of bug).
3. Verified: no existing MCP server has IP-literal/loopback-rejection logic
   (`resolve_safe` in `scripts/mcp_servers/file/common.py:28-42` only handles filesystem
   path-containment, not hostname/IP classification). This is genuinely new logic for Browser
   MCP — implement using stdlib `ipaddress.ip_address()` (try/except `ValueError` to detect
   non-IP hostnames) plus `.is_loopback`/`.is_link_local`/`.is_private` checks, rejecting when
   the hostname parses as an IP address in any of the non-global ranges, regardless of whether
   it also happens to appear in `allowed_domains` (defense in depth, per plan Risks — this check
   runs unconditionally, not gated on an `allowed_domains` miss).
4. `httpx.AsyncClient` is the required HTTP client (per `rules/coding.md`: "Use `httpx` (not
   `requests`) for HTTP"). Verified precedent for **async** HTTP calls from inside an MCP server
   package (as opposed to the sync ingestion-pipeline usage in `crawler.py`) at
   `scripts/mcp_servers/cicd/service_github_actions.py:16,42-50`: `GitHubActionsBackend.__init__`
   takes `http: httpx.AsyncClient` as a constructor parameter (dependency injection), not a
   module-global client. Browser's `BrowserService` should follow the same DI shape — accept an
   `httpx.AsyncClient` (or construct one internally per-request with `async with
   httpx.AsyncClient(timeout=self._timeout_sec) as client:` if no shared-client lifecycle is
   needed elsewhere) rather than reinventing a new pattern. Given `BrowserService` has no other
   async resource to share, a per-request `async with httpx.AsyncClient(...)` inside `fetch()` is
   simplest and avoids a `startup`/`shutdown` FastAPI lifecycle hook that no other server in this
   codebase currently uses either (verified: none of `shell`/`git`/`cicd`/`web_search`'s
   `server.py` register `@app.on_event`/lifespan hooks).
5. `BeautifulSoup(html, "lxml")` (Design step 1c, verified against `crawler.py:309,378`) requires
   the `lxml` parser package; already a project dependency (transitively required by
   `beautifulsoup4`'s lxml backend — confirm `lxml` itself is pinned in `pyproject.toml` or
   pulled in as an extra at implementation time; if absent, `BeautifulSoup(html, "html.parser")`
   (stdlib, no extra dependency) is the fallback, but prefer `"lxml"` for consistency with
   `crawler.py` unless its absence is confirmed).
6. No new `.importlinter` contract needed (plan Assumption 4/5, independently re-confirmed:
   `grep -in "git\|shell" .importlinter` still returns zero server-specific entries as of this
   investigation).

## Implementation

### Target file

`scripts/mcp_servers/browser/service.py` (new file).

### Procedure

1. Import `httpx`, `ipaddress`, `logging`, `urllib.parse.urlsplit`, `BeautifulSoup` (from
   `bs4`), `Awaitable`/`Callable` (from `collections.abc`), and the paired
   `mcp_servers.browser.models` symbols (`BrowserAuthorizationError`, `BrowserConfig`,
   `BrowserFetchRequest`, `BrowserFetchResponse`, `BrowserValidationError`) plus
   `mcp_servers.server.ToolArgs`.
2. Define `class BrowserService` with `__init__(self, cfg: BrowserConfig) -> None` storing
   `self._allowed_domains: set[str] = set(cfg.allowed_domains)`,
   `self._max_response_kb = cfg.max_response_kb`, `self._timeout_sec = cfg.timeout_sec`.
3. Implement `_check_domain(self, url: str) -> str` (returns the validated hostname; raises
   `BrowserAuthorizationError` or `BrowserValidationError`):
   - `urlsplit(url)`; require `scheme in ("http", "https")`, else `BrowserValidationError`.
   - Extract `hostname` (lowercased); if empty, `BrowserValidationError`.
   - Try `ipaddress.ip_address(hostname)`; if it parses, check
     `.is_loopback or .is_link_local or .is_private or .is_reserved or .is_multicast` — if any
     True, raise `BrowserAuthorizationError` (IP-literal targets rejected regardless of
     allowlist, per Assumption 3).
   - If `hostname not in self._allowed_domains` (fail-closed; empty set denies all), raise
     `BrowserAuthorizationError`.
   - Return `hostname`.
4. Implement `async def fetch(self, req: BrowserFetchRequest) -> BrowserFetchResponse`:
   - Call `self._check_domain(req.url)`.
   - Clamp `max_response_kb = min(req.max_response_kb or self._max_response_kb,
     self._max_response_kb)` (server ceiling always wins, mirroring
     `shell/service.py:257-258`'s `min(req.timeout_sec, self._max_timeout_sec)` pattern).
   - `async with httpx.AsyncClient(timeout=self._timeout_sec) as client: resp = await
     client.get(req.url)`.
   - Extract `text = self._extract_text(resp.text)`.
   - `text, truncated = self._truncate(text, max_response_kb)`.
   - Return `BrowserFetchResponse(text=text, truncated=truncated, url=req.url,
     status_code=resp.status_code, elapsed_sec=resp.elapsed.total_seconds())`.
5. Implement `_extract_text(self, html: str) -> str` (static or instance method):
   `soup = BeautifulSoup(html, "lxml")`; `for tag in soup(["script", "style"]): tag.decompose()`;
   return `soup.get_text(separator="\n", strip=True)`.
6. Implement `_truncate(self, text: str, max_kb: int) -> tuple[str, bool]`: encode to UTF-8,
   compare `len(encoded)` against `max_kb * 1024`; if over, slice bytes and decode with
   `errors="replace"`, return `(sliced_text, True)`; else `(text, False)` — per Assumption 2's
   encode-before-slice ordering.
7. Implement `fmt_fetch(self, args: ToolArgs) -> str`: build `BrowserFetchRequest(**args)`, call
   `await self.fetch(req)`, format via a `_format_fetch_result` static helper (status_code,
   truncated flag, then text body) — mirror `_format_run_result`'s "flags first, body last"
   layout (`shell/service.py:306-321`).
8. Implement `get_dispatch_table` and `build_service` per Scope, with a `logger.warning(...)`
   in `build_service` when `cfg.allowed_domains` is empty, mirroring
   `shell/service.py:334-341`'s pattern of one warning per empty-critical-list.

### Method

New-file creation adapting an existing template (`ShellService`); no new abstractions —
`BrowserService` is a plain class, no `Protocol`/`ABC`, consistent with every existing MCP
service module.

### Details

```python
class BrowserService:
    def __init__(self, cfg: BrowserConfig) -> None: ...
    def _check_domain(self, url: str) -> str: ...
    async def fetch(self, req: BrowserFetchRequest) -> BrowserFetchResponse: ...
    def _extract_text(self, html: str) -> str: ...
    def _truncate(self, text: str, max_kb: int) -> tuple[str, bool]: ...
    async def fmt_fetch(self, args: ToolArgs) -> str: ...
    def get_dispatch_table(self) -> dict[str, Callable[[ToolArgs], Awaitable[str]]]: ...


def build_service(cfg: BrowserConfig) -> BrowserService: ...
```
Failure paths covered: invalid URL scheme/hostname (422 via `BrowserValidationError`), disallowed
or IP-literal/loopback domain (403 via `BrowserAuthorizationError`), oversized response
(silent truncation with `truncated=True`, not an error — matches shell's non-fatal truncation
semantics), and network/timeout errors from `httpx` (let `httpx.HTTPError` subclasses propagate
unhandled to `server.py`, which per Design step 3 should map them to a 502/504-style response —
confirm exact status code choice in the `server.py` doc, not duplicated here).
