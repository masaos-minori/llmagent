## Implementation

### Target file

`scripts/mcp_servers/mdq/indexer.py`

### Procedure

1. Add `import json` to the import block (lines 9-25), in stdlib import-alphabetical position
   (after `import hashlib`, before `import logging`, matching existing order).
2. Add a new module-level helper function, placed above `_index_single_file` (before line 102):
   ```python
   def _estimate_token_count(text: str) -> int:
       """Approximate token count via a chars-per-token-~4 heuristic.

       This is not an exact tokenizer count â€” no live tokenizer endpoint is available inside
       mcp_servers/mdq without adding a new network dependency. Consistent in spirit with
       shared/token_estimation.py's own ratio-based approach, but without its
       chat-category/network dependencies (see plans/20260719-100727_plan.md, Assumption 5).
       """
       return max(1, len(text) // 4)
   ```
3. Update line 117 from
   `sections = await parse_markdown(service, ParseMarkdownRequest(path=str(path)))`
   to
   `sections, tags = await parse_markdown(service, ParseMarkdownRequest(path=str(path)))`.
4. After line 117 (after the `sections, tags = ...` call, and after the existing
   `if not sections:` early-return at lines 122-124), add:
   `tags_json = json.dumps(tags)` â€” computed once per file (frontmatter tags are file-level, so
   every chunk/section in this file shares the same `tags_json` value).
5. Inside the `for section in sections:` loop (line 149), immediately after line 154's
   `char_count = len(section["content"])`, add:
   `token_count = _estimate_token_count(section["content"])`.
6. In the INSERT params tuple (lines 165-182):
   - Replace the `None` at line 178 (the `token_count` position) with `token_count`.
   - Replace the `""` at line 180 (the `tags_json` position) with `tags_json`.

### Method

Direct, in-place edit of the existing procedural indexing function â€” no new classes. The
heuristic is a single module-level function (not a method, not a new file) since it has one
caller and no state, consistent with `skills/python-design/SKILL.md`'s "prefer simple, stable
functions over classes when state is not required" and "no speculative over-engineering."

### Details

- `tags_json` is computed once per file (outside the `for section in sections:` loop), not once
  per chunk â€” frontmatter applies to the whole document, and the plan explicitly notes "all
  sections/chunks in one document share the same document-level tags."
- `token_count` is computed once per chunk (inside the loop), since it depends on
  `section["content"]`, which differs per chunk.
- Do not store `tags` or `token_count` back onto the `section` dict â€” bind them as local
  variables used only when building the INSERT params tuple, matching the existing style of
  `char_count`/`content_hash`/`normalized_content` in the same loop.
- `_estimate_token_count`'s docstring must state explicitly that this is an approximation, per
  the source plan's Risks section â€” this satisfies the "non-null" requirement, not an "exact
  tokenizer count" requirement.
