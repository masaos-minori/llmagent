# Implementation: scripts/mcp_servers/mdq/mdq_service.py (propagate `(text, metadata)` tuples
in `search_docs`, `index_paths`, `refresh_index`, `grep_docs`)

Source plan: `plans/20260719-211216_plan.md` ("Replace fragile MDQ audit detail parsing with
structured metadata"), Design section ("Threading metadata through the tool handlers"),
Implementation step 5.

**Note on pre-existing docs for this file:** `mdq_service.py` already has two companion
implementation documents from sibling design cycles in the same MDQ batch:
`implementations/20260720-091248_mdq_service.py.md` (requirement 26: `MdqConfig` wiring, deny-all
startup warning, read-time authorization in `get_chunk`/`grep_docs`) and
`implementations/20260720-092036_mdq_service.py.md` (requirement 27: `outline()` stale-comparison
bug fix, `stats()` switched to shared `STALE_SQL_CONDITION`). Both were read in full. Neither
changes the return type of `search_docs`, `index_paths`, `refresh_index`, or `grep_docs`, and
neither discusses metadata propagation. This is a NEW, ADDITIONAL document covering this plan's
distinct concern: changing four wrapper methods' return shape from bare `str` to
`(str, <metadata type>)`. Per task instructions, this document combines all four method changes
(all in `MdqService`, all part of this one plan's Design) into one document.

## Goal

Change `MdqService.search_docs`, `index_paths`, `refresh_index`, `grep_docs` to return
`(text: str, metadata: <TypedDict>)` tuples instead of bare `str`, threading through the new
metadata each underlying module (`search.py`, `indexer.py`, `db_grep.py` — companion docs) now
produces, so `mdq_server.py::call_tool()` can read exact audit fields without parsing text.

## Scope

**In scope:**
- `search_docs()` (`mdq_service.py:129-136`): unpack `(text, metadata)` from
  `search.search_docs()` (now returning a tuple per companion doc
  `20260720-093551_search.py.md`); keep the existing `[WARNING: Index is being updated...]`
  suffix logic operating on `text` only (metadata unaffected by the warning suffix); return
  `(text, metadata)`.
- `index_paths()` (`mdq_service.py:253-263`): unpack `(text, metadata)` from
  `indexer.index_paths()` (now returning a tuple per companion doc
  `20260720-093552_indexer.py.md`); return `(text, metadata)` unchanged from what the indexer
  produced (the `_index_lock`/`_is_indexing` wrapping logic is unaffected).
- `refresh_index()` (`mdq_service.py:265-276`): keep the `summary: RefreshSummary` returned by
  `_refresh_paths()` instead of discarding it after `_format_refresh_summary()` builds the text;
  return `("\n".join(self._format_refresh_summary(summary)), summary)` — reusing `RefreshSummary`
  directly as the metadata type per the plan's explicit design choice (no new type).
- `grep_docs()` (`mdq_service.py:326-361`): unpack `(text, metadata)` from the module-level
  `grep_docs()` call (now returning a tuple per companion doc
  `20260720-093553_db_grep.py.md`); return `(text, metadata)`.

**Out of scope:**
- `get_chunk()`, `outline()`, `stats()` — the plan's per-tool metadata list does not request
  structured metadata for these tools; `mdq_server.py::call_tool()`'s existing audit-detail block
  only special-cases `search_docs`, `get_chunk` (a simple text-substring check, unchanged — see
  companion `mdq_server.py` doc), `grep_docs`, `index_paths`, `refresh_index`. These three methods
  keep returning bare `str`.
- `MdqConfig` wiring, startup warning, read-time authorization (covered by
  `20260720-091248_mdq_service.py.md`) — unaffected, this document's changes are to different
  parts of the same methods' bodies (the return statement, not the authorization logic that
  precedes it in `get_chunk`/`grep_docs` — note `get_chunk` itself is NOT changed by this
  document at all, only `grep_docs`'s return statement).
- `outline()`/`stats()` bug fixes (covered by `20260720-092036_mdq_service.py.md`) — unaffected,
  different methods.
- `_validate_paths()`, `_format_refresh_summary()` — unchanged (the latter is still called, just
  its result is no longer the only thing extracted from `summary`).

## Implementation

### Target file

`scripts/mcp_servers/mdq/mdq_service.py`

### Procedure

1. Change method signatures:
   - `async def search_docs(self, req: SearchDocsRequest) -> tuple[str, SearchDocsMetadata]:`
   - `async def index_paths(self, req: IndexPathsRequest) -> tuple[str, IndexPathsMetadata]:`
   - `async def refresh_index(self, req: RefreshIndexRequest) -> tuple[str, RefreshSummary]:`
   - `async def grep_docs(self, req: GrepDocsRequest) -> tuple[str, GrepDocsMetadata]:`
   Add `SearchDocsMetadata`, `IndexPathsMetadata`, `GrepDocsMetadata` to the
   `from mcp_servers.mdq.mdq_models import (...)` block (`mdq_service.py:29-41`).
2. `search_docs()` body: `text, metadata = await search_docs(self, req)` (module-level function,
   already imported at line 42); keep the existing `if self._is_indexing:` suffix append applied
   to `text` only; `return text, metadata`.
3. `index_paths()` body: inside the existing `try:`/`finally:` lock block,
   `text, metadata = await _index_paths(self, req)`; `return text, metadata`.
4. `refresh_index()` body: inside the existing `try:`/`finally:` lock block, keep
   `self._validate_paths(req.paths)`; `summary = await _refresh_paths(self, req)`; `text =
   "\n".join(self._format_refresh_summary(summary))`; `return text, summary`.
5. `grep_docs()` body: change `result: str = grep_docs(...)` to
   `text, metadata = grep_docs(conn, compiled, req.paths or [], max_matches, max_chars,
   ctx_before, ctx_after)`; `return text, metadata` (inside the existing `try:`/`finally:`
   connection-close block).

### Method

Four independent signature/body changes, each a mechanical unpack-and-forward of the tuple now
produced by the underlying module (`search.py`, `indexer.py`, `db_grep.py`) or, for
`refresh_index`, retention of an already-computed local (`summary`) that was previously discarded
after use. No new control flow, no new exception handling.

### Details

- `mdq_server.py`'s `_handle_*` functions (companion doc `20260720-093555_mdq_server.py.md`) are
  the only call sites; they are updated in that companion document to unpack the tuple and
  extract metadata via the mdq-local dispatch-result wrapper described there. This document does
  not touch `mdq_server.py`.
- The `search_docs()` warning-suffix append (`if self._is_indexing: result += "\n\n[WARNING...]"`)
  is unaffected in spirit but must be updated for the new variable name (`text` instead of
  `result`) — purely a rename, not a behavior change; the suffix is still appended only to the
  text component, never to metadata.
