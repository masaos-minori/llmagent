# Implementation: cmd_mcp.py — add watchdog status to /mcp status output

## Goal

Show watchdog enabled/disabled state, interval, and max_restarts in `/mcp status` output.

## Scope

- `scripts/agent/commands/cmd_mcp.py` — `_cmd_mcp_status()` method only.

## Assumptions

1. `ctx.cfg.mcp.mcp_watchdog_interval` and `ctx.cfg.mcp.mcp_watchdog_max_restarts` are accessible
   from the `_McpMixin._ctx` property.
2. The output format follows the existing `_cmd_mcp_status()` style:
   `f"  Key         value"` (two-space indent, padded label).
3. Watchdog line appears after the Servers summary line.

## Implementation

### Target file

`scripts/agent/commands/cmd_mcp.py`

### Procedure

In `_cmd_mcp_status()`, after the existing Servers line, append:

```python
interval = ctx.cfg.mcp.mcp_watchdog_interval
max_restarts = ctx.cfg.mcp.mcp_watchdog_max_restarts
if interval > 0:
    wd_status = f"enabled (interval={interval:.0f}s, max_restarts={max_restarts})"
else:
    wd_status = "disabled (interval=0) — no auto-restart"
self._out.write(f"  Watchdog    {wd_status}")
```
