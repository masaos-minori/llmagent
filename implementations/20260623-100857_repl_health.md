## Goal

Upgrade `audit_security_defaults()` in `scripts/agent/repl_health.py` so that `shell_sandbox_backend="none"` in production mode (`production_mode=True`) raises `RuntimeError` instead of only logging a warning. In non-production mode the existing warning behavior is preserved.

## Scope

**In-Scope:**
- `scripts/agent/repl_health.py` lines 453-456 ã change warning-only path to conditional RuntimeError
- Add a new test case to `tests/test_repl_health.py` (or the file that tests `audit_security_defaults`) covering `shell_sandbox_backend="none"` + `production_mode=True`

**Out-of-Scope:**
- No changes to `ShellConfig`, `ShellService`, or `service_static_helpers.py`
- No changes to other security checks in `audit_security_defaults()`
- No new config fields

## Assumptions

1. The existing pattern for auth_token production enforcement (lines 429-434: `if production_mode and violations: raise RuntimeError(...)`) serves as the model; the shell sandbox check follows the same structure.
2. `ShellConfig.load()` is already called inside a `try/except Exception: pass` block (line 451-466), so if `ShellConfig` is unavailable the check is silently skipped. This behavior must be preserved ã the new RuntimeError must only fire inside the `try` block when `shell_cfg` is successfully loaded.
3. The `production_mode` parameter is already in scope at the shell sandbox check location (line 453); no additional parameter threading is needed.
4. `tests/test_repl_health.py` must gain one new test: `shell_sandbox_backend="none"` + `production_mode=True` ã `RuntimeError`. The existing `test_shell_sandbox_none_warns` covers `production_mode=False` and must remain unchanged.

## Implementation

### Target file
`scripts/agent/repl_health.py`

### Procedure

1. Locate lines 453-456 (the `shell_sandbox_backend == "none"` branch inside the `try` block).
2. Insert a `production_mode` guard that raises `RuntimeError` before the warning/append path.

### Method

Single `Edit` operation.

### Details

**Current (lines 453-456):**
```python
        if shell_cfg.shell_sandbox_backend == "none":
            msg = "Security: shell_sandbox_backend=none (no sandbox for shell commands)"
            logger.warning(msg)
            warnings.append(msg)
```

**Replacement:**
```python
        if shell_cfg.shell_sandbox_backend == "none":
            msg = "shell_sandbox_backend=none is not permitted in production mode"
            if production_mode:
                raise RuntimeError(f"Production mode requires shell sandbox. {msg}")
            logger.warning("Security: %s", msg)
            warnings.append(f"Security: {msg}")
```

Changes:
- `production_mode=True` raises `RuntimeError` with an actionable message
- `production_mode=False` logs warning and appends to `warnings` list (unchanged behavior)
- Warning message format aligned with existing `logger.warning("Security: %s", v)` pattern
