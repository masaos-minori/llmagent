# Implementation: L-1 — Fix inaccurate lifecycle comments and docstrings (4 files)

Fix inaccurate docstrings and comments in 4 lifecycle-related source files to accurately
describe runtime behavior: `_ServerLifecycleRouter` as the protocol facade, no
"ondemand" startup mode, `verify_running()` as check-only, and forward-looking docs for
H-1 (stderr log) and H-8 (process group) behaviors.

- `scripts/agent/lifecycle_protocol.py`

**Steps covered**: Plan L-1 steps 1–5.

**Out of scope**: test files, config, DB schema. No logic changes.

## Implementation

### Target files and procedures

#### File 1: `scripts/agent/lifecycle_protocol.py`

**Current docstring** on `LifecycleManagerProtocol`:
```
HttpServerLifecycleManager satisfies this protocol structurally.
```

**Replace with**:
```
_ServerLifecycleRouter in factory.py is the production implementation of this protocol.
HttpServerLifecycleManager is the low-level subprocess manager it delegates to.
```

No other changes to this file.

---

#### File 2: `scripts/agent/startup.py`

**Current comment** in `_start_servers()` docstring or inline (line ~101–102):
```
Ondemand servers are excluded; they start on first tool call via ensure_ready().
```

**Replace with**:
```
Persistent-mode servers are externally managed and excluded here.
Subprocess-mode servers with startup_mode='subprocess' are started at agent init.
Other subprocess-mode servers start on first tool call via ensure_ready().
```

No other changes.

---

#### File 3: `scripts/agent/http_lifecycle.py`

**Change 1**: `verify_running()` docstring.

Current:
```python
def verify_running(self, server_key: str) -> bool:
    """Check if an HTTP subprocess server is running and optionally restart it.

    Returns True if running, False if not running (and restart was attempted).
    """
```

Replace with:
```python
def verify_running(self, server_key: str) -> bool:
    """Return True if the HTTP subprocess server is running, False if missing or exited."""
```

**Change 2**: `HttpServerLifecycleManager` class docstring — add stderr log and process
group notes.

Current:
```python
class HttpServerLifecycleManager:
    """Manages HTTP subprocess MCP servers: start, health-poll, restart, shutdown."""
```

Replace with:
```python
class HttpServerLifecycleManager:
    """Manages HTTP subprocess MCP servers: start, health-poll, restart, shutdown.

    When stderr log redirect is active (H-1), each subprocess writes stderr to a
    per-server log file at /opt/llm/logs/mcp/{server_key}.stderr.log instead of a pipe.

    When process group shutdown is active (H-8), subprocesses are started with
    start_new_session=True and terminated via os.killpg() to include child processes.
    """
```

---

#### File 4: `scripts/agent/factory.py`

**Current** `_ServerLifecycleRouter` class docstring:
```python
class _ServerLifecycleRouter:
    """Routes lifecycle calls to the HTTP lifecycle manager.

    Implements LifecycleManagerProtocol for HTTP-only MCP servers.
    """
```

**Replace with**:
```python
class _ServerLifecycleRouter:
    """Production implementation of LifecycleManagerProtocol for HTTP MCP servers.

    Delegates subprocess management to HttpServerLifecycleManager (_http_mgr) while
    adding:
    - Shutdown guard (_shutting_down): prevents start/restart after shutdown begins.
    - LifecycleState tracking (_states): provides get_transport_state() with real values.
    - Process snapshot API: get_process_info() and list_processes() for observability.
    """
```
