# Implementation: shared/plugin_registry.py — Async handler validation at registration time

## Goal

Add `asyncio.iscoroutinefunction()` check in `register_tool()` so non-async plugin tool handlers are rejected at registration time with a clear error message.

## Scope

**In**: One check in `register_tool()` decorator before annotation checks.

**Out**: Changes to `plugin_tool_invoker.py`, tests.

## Assumptions

1. `plugin_auto_discover.py` already catches `ValueError` from `register_tool()` — the new check can raise `ValueError`.
2. `asyncio.iscoroutinefunction()` returns `True` for `async def` functions. For coroutine-wrapped functions (e.g., via `functools.wraps`), it should still work.
3. The check order: async first, then annotation checks. This gives a clear error for the most common mistake (forgetting `async`).
4. `inspect.iscoroutinefunction()` is equivalent and can be used as an alternative; `asyncio.iscoroutinefunction()` is preferred (it handles coroutine-wrapped callables).

**Updated `register_tool()`:**
```python
import asyncio  # add at top of file if not present

def register_tool(name: str) -> Callable[[_F], _F]:
    def decorator(fn: _F) -> _F:
        # Async check must come first — clearest error for the most common mistake
        if not asyncio.iscoroutinefunction(fn):
            raise ValueError(
                f"[plugin] tool contract violation: '{name}' handler must be "
                f"an async function (defined with 'async def')"
            )
        hints = typing.get_type_hints(fn)
        return_hint = hints.get("return")
        if return_hint is None:
            raise ValueError(
                f"[plugin] tool contract violation: '{name}' missing return type annotation "
                f"(expected tuple[str, bool])"
            )
        if return_hint != tuple[str, bool]:
            raise ValueError(
                f"[plugin] tool contract violation: '{name}' expected return type "
                f"'tuple[str, bool]', got {return_hint}"
            )
        _tools[name] = (fn, _current_loading_module[0])
        logger.debug("[plugin] tool registered: %s", name)
        return fn
    return decorator
```

### Details

- `asyncio` is likely already imported elsewhere in the module; confirm before adding duplicate import.
- The error message includes the tool name for easy diagnosis.
- This `ValueError` is caught by `plugin_auto_discover.py`'s `except (..., ValueError)` block, which records the plugin as failed. In non-strict mode, loading continues with other plugins. In strict mode, `PluginLoadError` is raised.
