# Implementation: agent/history.py — Narrow except Exception in context compression

## Goal

Narrow the single `except Exception as e:` in the LLM-based context compression method of `scripts/agent/history.py` to the specific exception types that can realistically occur: HTTP errors, JSON decode errors, and key/type access errors on the response dict.

## Scope

**In:** `scripts/agent/history.py`
**Out:** No other files change.

## Assumptions

1. The wrapped code (`_call_llm_for_summary`) makes an HTTP call to the LLM endpoint and parses the JSON response. The realistic failure modes are:
   - `httpx.HTTPError` — network or HTTP-level failure
   - `orjson.JSONDecodeError` — malformed response body
   - `KeyError` — response dict missing expected keys (`choices`, `message`, `content`)
   - `TypeError` — unexpected type in response (e.g. `choices` is not subscriptable)
2. On any failure, the function logs a warning and returns `None` (compression is skipped). This graceful-degradation behaviour is preserved.
3. `httpx` and `orjson` are already imported in this file.

## Implementation

### Target file

`scripts/agent/history.py`

### Procedure

Change line ~203:
```python
# Before
except Exception as e:
    logger.warning(f"Context compression failed: {e}")
    return None

# After
except (httpx.HTTPError, orjson.JSONDecodeError, KeyError, TypeError) as e:
    logger.warning(f"Context compression failed: {e}")
    return None
```
