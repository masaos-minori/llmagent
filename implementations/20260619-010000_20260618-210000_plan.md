# Implementation: Improve truncation metadata to show actual visible bytes

## Plan Reference
- Plan file: `plans/20260618-210000_plan.md`
- Commit message: "Design: add actual_visible_bytes to truncation metadata"

## Changes Made

### 1. `scripts/mcp/server.py` — TruncationResult dataclass

**Before:**
```python
@dataclasses.dataclass(frozen=True)
class TruncationResult:
    """Metadata returned by _truncate_with_meta()."""

    text: str
    truncated: bool
    total_bytes: int
```

**After:**
```python
@dataclasses.dataclass(frozen=True)
class TruncationResult:
    """Metadata returned by _truncate_with_meta()."""

    text: str
    truncated: bool
    total_bytes: int
    actual_visible_bytes: int = 0
```

Added `actual_visible_bytes` field with default value of `0` for backward compatibility. When truncation does not occur, this field remains `0`. When truncation occurs, it contains the actual byte count of the shown text after UTF-8 decoding (which may be less than `max_bytes` due to dropped partial multi-byte characters).

### 2. `scripts/mcp/server.py` — `_truncate_with_meta()` function

**Before:**
```python
def _truncate_with_meta(
    text: str, max_bytes: int = MCP_MAX_RESPONSE_BYTES
) -> TruncationResult:
    """Return TruncationResult with truncated text and metadata."""
    encoded = text.encode("utf-8")
    total = len(encoded)
    if total <= max_bytes:
        return TruncationResult(text=text, truncated=False, total_bytes=total)
    shown = encoded[:max_bytes].decode("utf-8", errors="ignore")
    truncated_text = (
        shown + f"\n[TRUNCATED: {total:,} bytes total, showing {max_bytes:,} bytes]"
    )
    return TruncationResult(text=truncated_text, truncated=True, total_bytes=total)
```

**After:**
```python
def _truncate_with_meta(
    text: str, max_bytes: int = MCP_MAX_RESPONSE_BYTES
) -> TruncationResult:
    """Return TruncationResult with truncated text and metadata."""
    encoded = text.encode("utf-8")
    total = len(encoded)
    if total <= max_bytes:
        return TruncationResult(text=text, truncated=False, total_bytes=total)
    shown = encoded[:max_bytes].decode("utf-8", errors="ignore")
    actual_visible = len(shown.encode("utf-8"))
    truncated_text = (
        shown + f"\n[TRUNCATED: {total:,} bytes total, showing {actual_visible:,} bytes]"
    )
    return TruncationResult(text=truncated_text, truncated=True, total_bytes=total, actual_visible_bytes=actual_visible)
```

Key changes:
- Added `actual_visible = len(shown.encode("utf-8"))` to compute the real byte count after UTF-8 decoding. When `errors="ignore"` drops partial multi-byte characters (e.g., Japanese/Chinese characters), the decoded text may be fewer bytes than `max_bytes`.
- Updated truncation message to show `{actual_visible:,} bytes` instead of `{max_bytes:,} bytes`.
- Passed `actual_visible_bytes` to the `TruncationResult` constructor.

### 3. `scripts/mcp/server.py` — `run_stdio()` JSON response

**Before:**
```python
resp_id = 0
truncated = False
total_bytes = 0
# ... (inside try block after dispatch)
tr = _truncate_with_meta(dispatch_result.output)
is_error = dispatch_result.is_error
result = tr.text
truncated = tr.truncated
total_bytes = tr.total_bytes
# ... (in JSON response)
"truncated": truncated,
"total_bytes": total_bytes,
```

**After:**
```python
resp_id = 0
truncated = False
total_bytes = 0
actual_visible_bytes = 0
# ... (inside try block after dispatch)
tr = _truncate_with_meta(dispatch_result.output)
is_error = dispatch_result.is_error
result = tr.text
truncated = tr.truncated
total_bytes = tr.total_bytes
actual_visible_bytes = tr.actual_visible_bytes
# ... (in JSON response)
"truncated": truncated,
"total_bytes": total_bytes,
"actual_visible_bytes": actual_visible_bytes,
```

Added `actual_visible_bytes` variable initialization and assignment. The JSON response now includes the new field for consumers that need to know the exact byte count of visible content.

## Backward Compatibility

- `actual_visible_bytes` has a default value of `0`, so existing code that creates `TruncationResult` without this field continues to work.
- The new JSON field is additive — existing consumers that don't expect it will simply ignore it.
- No changes to truncation behavior, limits, or encoding logic.

```python
>>> long_text = 'こんにちは' * 100000  # 1,500,000 bytes total
>>> r = _truncate_with_meta(long_text)
>>> r.truncated
True
>>> r.total_bytes
1500000
>>> r.actual_visible_bytes
524286  # 2 bytes dropped due to partial character handling
>>> r.text.split('\n')[-1]
'[TRUNCATED: 1,500,000 bytes total, showing 524,286 bytes]'
```

The `actual_visible_bytes` correctly reflects the real byte count after UTF-8 decoding, which differs from `max_bytes` when partial multi-byte characters are dropped.
