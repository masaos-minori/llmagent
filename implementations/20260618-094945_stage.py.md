# Implementation: scripts/rag/stage.py

## Goal

Add `StageResult` TypedDict and `stage_results` field to `PipelineContext` so that
pipeline observers can inspect per-stage success/failure/fallback outcomes programmatically.

## Scope

- `scripts/rag/stage.py` — add `StageResult`, extend `PipelineContext`. No other files.

## Assumptions

1. `StageResult` uses `typing.TypedDict` (available in Python 3.13).
2. `stage_results` field is backward-compatible: default `default_factory=list` means
   all existing callers of `PipelineContext(query=...)` continue to work unchanged.
3. `fallback_reason: str | None` — `None` when status is "success".
4. `status` is `str` (not Literal) to avoid import-time overhead; valid values are
   "success" | "failure" | "fallback".

## Implementation

### Target file

`scripts/rag/stage.py`

### Procedure

1. Add `TypedDict` to the `typing` import.
2. Add `StageResult` class before `PipelineContext`.
3. Add `stage_results: list[StageResult]` field with `default_factory=list` to `PipelineContext`.

### Details

```python
from typing import Any, Protocol, TypedDict


class StageResult(TypedDict):
    """Per-stage outcome recorded by RagPipeline.run()."""

    stage_name: str
    status: str  # "success" | "fallback" | "failure"
    elapsed_seconds: float
    fallback_reason: str | None


@dataclasses.dataclass
class PipelineContext:
    """Mutable state passed between pipeline stages."""

    query: str
    history_context: str = ""
    queries: list[str] = dataclasses.field(default_factory=list)
    search_results: list[list[RawHit]] = dataclasses.field(default_factory=list)
    merged: list[RagHit] = dataclasses.field(default_factory=list)
    reranked: list[RagHit] = dataclasses.field(default_factory=list)
    augment_result: str = ""
    stage_results: list[StageResult] = dataclasses.field(default_factory=list)
```
