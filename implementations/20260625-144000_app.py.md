# Implementation: app.py — ack-based offset advancement, remove mid-stream checkpoint (req #29)

## Goal

Change offset semantics so that offsets advance on explicit `ack` rather than on delivery. Remove the mid-stream checkpoint `write_offset()` calls from the SSE generator.

## Scope

- Remove `events_since_checkpoint` counter and mid-stream `write_offset()` call from `subscribe()` SSE generator
- The disconnect handler `write_offset()` (on `CancelledError`) is also removed — offsets are now ack-driven only
- `POST /events/{event_id}/ack?consumer_id=X` (req #25) is the sole path for offset advancement
- `config.offset_checkpoint_interval` field becomes unused (can be deprecated)

## Assumptions

- req #25 ack endpoint is implemented first and already calls `write_offset()` when `consumer_id` is provided
- Consumers without `consumer_id` are stateless — no offset file, unchanged behavior
- "At-least-once" guarantee is maintained: after crash, consumer reconnects from last acked seq (not last delivered)
- Consumers that never call ack will not advance their offset (by design)

## Implementation

### Target file

`scripts/eventbus/app.py`

### Procedure

1. In `subscribe()` SSE generator, remove `events_since_checkpoint` counter initialization
2. Remove the mid-stream checkpoint block (`if events_since_checkpoint >= ...`)
3. Remove the `CancelledError` handler's `write_offset()` call (offset is now ack-driven)
4. Simplify the `CancelledError` handler to just log and exit

### Method

Edit the `subscribe()` function. Keep the `read_offset()` call at the start (used to restore position on reconnect).
