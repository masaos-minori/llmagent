# Design: Update data-layer doc for RAG boundary clarification

## Goal
Update `docs/05_agent_09_data-layer.md` to reflect that document maintenance operations (`/db urls`, `/db clean`) now go through rag-pipeline-mcp instead of `DbMaintenanceService`.

## Target File
- `docs/05_agent_09_data-layer.md`

## Current State (lines 61-80)

### Line 64 — stale access pattern description
```
The agent accesses them exclusively through `DbMaintenanceService` (for `/db` commands).
```

### Lines 77-79 — stale "Future refactoring" note
```
**Note:** `/db urls`, `/db clean`, `/db stats`, `/db rebuild-fts` are agent-layer commands
that access RAG data through `DbMaintenanceService`. Future refactoring may move these to
the rag-pipeline-mcp service. See [05_agent_12_reference-api.md](05_agent_12_reference-api.md).
```

## Implementation Steps

### Step 1: Update line 64
Change from:
```
The agent accesses them exclusively through `DbMaintenanceService` (for `/db` commands).
```
To:
```
The agent accesses them through rag-pipeline-mcp MCP tools (`rag_list_documents`, `rag_delete_document`) for document listing and deletion. Direct SQLite access is not used by the agent layer.
```

### Step 2: Update lines 73-75 (responsibility boundary)
Current text describes `DbMaintenanceService.delete_document(url)` — update to reflect MCP path:
```
**Responsibility boundary:** The agent deletes via `rag_delete_document` MCP tool (dispatched through `rag-pipeline-mcp`), which cascades from `documents` → `chunks` → `chunks_fts` (trigger). `chunks_vec` is a sqlite-vec virtual table with no FK constraint; the tool also deletes it explicitly.
```

### Step 3: Remove stale "Future refactoring" note (lines 77-79)
Delete the entire **Note:** paragraph since the refactoring is now implemented, not future.

### Step 4: Update Agent-Side Document Access Patterns table (lines 86-92)
Current:
| Path | Mechanism | When to use |
|---|---|---|
| MCP tools (primary) | `ToolRouteResolver` → MCP server (rag-pipeline-mcp or mdq-mcp) | Normal operation; all agent turns |
| `/db` commands (admin) | `DbMaintenanceService` → direct SQLite access | Admin maintenance tasks only |

Change to:
| Path | Mechanism | When to use |
|---|---|---|
| MCP tools (primary + admin) | `ToolRouteResolver` → rag-pipeline-mcp (`rag_list_documents`, `rag_delete_document`) | Document listing/deletion; normal and admin operation |
| `/db` commands (admin) | MCP tool call via `ToolExecutor.execute()` | Admin maintenance tasks only |

## Completion Criteria
- No references to "Future refactoring" remain in the document
- `DbMaintenanceService` is no longer described as the access path for document operations
- Responsibility boundary correctly describes MCP tool dispatch
