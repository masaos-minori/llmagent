2. The MDQ service uses a SQLite DB with FTS5 index; read `mcp/mdq/service.py` to

5. Separate the FTS5 query execution layer from the result-formatting layer.

# Implementation

## Target file

`scripts/mcp/mdq/search.py`
(read `scripts/mcp/mdq/service.py` first to understand existing DB access)

## Procedure

1. Read `mcp/mdq/service.py` to understand:
   - DB path / `SQLiteHelper` usage pattern
   - Existing table schema (confirm FTS5 table name and columns)
2. Implement `_fts_search(query: str, db_path: str, limit: int) -> list[str]`:
   ```python
   def _fts_search(query: str, db_path: str, limit: int = 10) -> list[str]:
       with SQLiteHelper("rag").open(row_factory=True) as db:
           rows = db.fetchall(
               "SELECT content FROM docs_fts WHERE docs_fts MATCH ? LIMIT ?",
               (query, limit),
           )
       return [r["content"] for r in rows]
   ```
3. Change the public search function to return `SearchDocsResult`:
   ```python
   def search_docs(query: str, limit: int = 10) -> SearchDocsResult:
       if not query.strip():
           return SearchDocsResult(query=query, results=[], total=0)
       results = _fts_search(query, limit=limit)
       return SearchDocsResult(query=query, results=results, total=len(results))
   ```
4. Remove placeholder comment.
5. Run ruff + mypy.
