# Implementation: Add workflow_allowlist startup warning to repl_health.py

Steps covered: Plan 20260626-091914 — Step 2-2

---

## Goal

Add a startup health check in `repl_health.py` that warns (or fails) when `workflow_allowlist` is empty, so operators know the cicd MCP server is in fail-closed mode before it blocks requests.

---

## Scope

- **In scope**: `scripts/agent/repl_health.py` (or the cicd MCP server equivalent) — add allowlist health check
- **Out of scope**: service_guards.py (step 1-1); config (step 2-1); tests (step 3-x)

---

## Assumptions

- `repl_health.py` has a pattern of health checks that run at startup and emit warnings/errors.
- The cicd MCP server has its own health check or uses `repl_health.py`.
- At startup, if `workflow_allowlist` is empty, emit a WARNING — not an ERROR — because fail-closed is a valid (intentional) config state.

---

## Implementation

### Target file
`scripts/agent/repl_health.py`

### Procedure
1. Read `repl_health.py` to understand the health check pattern.
2. Add a new check `_check_cicd_workflow_allowlist(config)`:
   ```python
   def _check_cicd_workflow_allowlist(config: CicdConfig) -> HealthCheckResult:
       if not config.workflow_allowlist:
           return HealthCheckResult(
               name="cicd_workflow_allowlist",
               status="warning",
               message="workflow_allowlist is empty — all workflow triggers will be denied (fail-closed). "
                       "Set cicd_mcp_server.toml:workflow_allowlist to allow workflows.",
           )
       return HealthCheckResult(name="cicd_workflow_allowlist", status="ok")
   ```
3. Register this check in the startup health check sequence.

### Method
Health check addition. Non-breaking: warning does not prevent startup.
