# Implementation: Add workflow-context suppression guard to tool_approval.py

Steps covered: Plan 20260626-090724 — Phase 2, Step 2-2

---

## Goal

Add an optional suppression guard so `run_approval_checks` can skip per-tool approval when a workflow is active and `workflow_require_approval=True` is set, avoiding redundant double-confirmation UX.

---

## Scope

- **In scope**: `scripts/agent/tool_approval.py` — add `skip_in_workflow_mode: bool` parameter to `run_approval_checks`
- **In scope**: `scripts/agent/repository_gateway.py` — pass `skip_in_workflow_mode` based on workflow context
- **Out of scope**: changing the approval UI or risk classification logic
- **Out of scope**: removing tool-level approval entirely

---

## Assumptions

1. The design decision (UNK-02 resolved): both layers coexist. The suppression guard is **optional** — it is only active when `workflow_require_approval=True` AND an explicit operator choice enables it.
2. Default behavior: `skip_in_workflow_mode=False` — no change for existing code paths.
3. `ctx.workflow.approval_pending` or `ctx.cfg.workflow_require_approval` can be checked to detect active workflow approval mode.
4. The guard is additive — it does not remove any existing logic, only short-circuits when conditions are met.

---

## Implementation

### Target file
`scripts/agent/tool_approval.py`

### Procedure
1. Read `scripts/agent/tool_approval.py` lines 156-200 (`run_approval_checks` definition).
2. Add a `skip_in_workflow_mode: bool = False` parameter to `run_approval_checks`.
3. At the top of the function body, add:
   ```python
   if skip_in_workflow_mode:
       logger.debug("run_approval_checks: skipping — workflow approval mode active")
       return tool_calls, []  # all approved, none denied
   ```
4. Read `scripts/agent/repository_gateway.py` lines 90-110 (the call site).
5. Update the call in `repository_gateway.py` to pass:
   ```python
   skip_in_workflow_mode=(
       ctx.cfg.workflow_require_approval
       and ctx.workflow.approval_pending is False  # not yet at the gate
   )
   ```
   Note: the suppression applies only if the operator has explicitly enabled workflow approval. If `workflow_require_approval=False`, the guard never fires.

### Method
Minimal additive change. New parameter with `False` default — zero behavior change for all existing callers.

### Details

`run_approval_checks` signature change:
```python
async def run_approval_checks(
    ctx: AgentContext,
    tool_calls: list[ToolCall],
    skip_in_workflow_mode: bool = False,  # NEW
) -> tuple[list[ToolCall], list[str]]:
```

Return convention when skipping: return `(tool_calls, [])` — all calls approved, no denied IDs.

Logging: emit a `DEBUG` log with reason `"workflow_approval_mode_active"` when suppressed.
