## Implementation

### Target files
`tests/test_eventbus_app.py`, `tests/test_eventbus_db.py`, `tests/test_eventbus_offsets.py`, `tests/test_eventbus_config.py`

### Procedure

**test_eventbus_app.py**:
1. `/health` returns 200 when ok, 503 when degraded (regression for plan 52).
2. `/replay` pagination: offset/limit produce non-overlapping pages (regression for plan 47).
3. `/dlq` pagination: same.
4. App state isolation: two test clients do not share state (regression for plan 53).

**test_eventbus_db.py**:
5. DLQ query uses index: `EXPLAIN QUERY PLAN` shows `USING INDEX idx_dlq_stream_status` (regression for plan 48).
6. `count_events()` and `count_dlq_entries()` return correct totals.

**test_eventbus_offsets.py**:
7. Same consumer_name + host + pid → same consumer_id (stability).
8. Collision detection: same ID + different name → `ConsumerIdCollisionError` (regression for plan 49).

**test_eventbus_config.py**:
9. Invalid port → `ValidationError` (regression for plan 51).
10. Valid config → no error.

### Method
Pytest unit tests with in-memory SQLite fixture. Use `httpx.AsyncClient` for API tests.
