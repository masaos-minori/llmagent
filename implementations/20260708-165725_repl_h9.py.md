# Implementation: H-9 — Remove tool_results SQL query from _persist_session_diagnostics()

## Goal

Remove the direct `SELECT ... FROM tool_results` query and the `ctx.tool_result_store` read
from `_persist_session_diagnostics()`, since the `tool_results` table is being dropped from the
schema entirely (see the companion `db/schema_sql.py` doc) and `ToolResultStore` is being
deleted (see the companion `db/tool_results.py` deletion doc).

## Scope

**Target**: `scripts/agent/repl.py` — the `_persist_session_diagnostics()` method.

**Supersedes**: `implementations/20260708-164245_repl_h7.py.md` (the H-7 doc for this same
method), which only removed the `tool_results = ctx.tool_result_store` guard variable and
simplified the `if` condition, while KEEPING the SQL query itself (since H-7 did not remove the
`tool_results` table). **H-9 goes further: the query itself must also be removed**, since running
`SELECT ... FROM tool_results` against a database that no longer has that table would raise
`sqlite3.OperationalError: no such table: tool_results` on every fresh database created after
the schema change lands (existing databases created before this rollout would still have the
table and the query would still work against them — but the query becomes meaningless dead code
either way, since nothing in the system writes to that table anymore after H-1/H-2/H-4).
**Implement this doc instead of H-7's for this method.**

**Depends on**: land together with `implementations/*_db_schema_sql.py.md` (DDL removal) and
`implementations/*_db_tool_results_delete.py.md` (class deletion) — though technically this
method would still run without error against a database that still has the (now-orphaned) table,
the query is being removed here because it no longer serves any purpose.

**Out of scope**: everything else in `_persist_session_diagnostics()` (the `latency_summary`
computation, the workflow/task/approval/retry counters, the final diagnostics payload
construction) — unchanged.

## Assumptions

1. `tool_result_summary` (the dict populated by the removed query) is still referenced later in
   the method at the final payload construction (`"tool_result_summary": tool_result_summary,`)
   — the plan does not ask to change the diagnostics JSON schema, so this key stays in the
   payload with a value that is now always `{}` (an empty dict), rather than removing the key
   entirely. This avoids changing the shape of a payload that other tooling might already parse.

## Implementation

### Target file

`scripts/agent/repl.py`

### Procedure

#### Step 1: Confirm current state

```bash
grep -n "tool_result_store\|tool_results\|tool_result_summary" scripts/agent/repl.py
```

Expected matches: the `tool_results = ctx.tool_result_store` read, the guard condition, the SQL
query block, the `tool_result_summary` dict declaration/population, and its use in the final
payload dict.

#### Step 2: Remove the query block, keep `tool_result_summary` as an always-empty dict

Current:

```python
            tool_results = ctx.tool_result_store
            session_id = ctx.session.session_id

            latency_summary = {}
            for step, samples in stats.stat_latency.items():
                if samples:
                    latency_summary[step] = {
                        "count": len(samples),
                        "mean_ms": round(sum(samples) / len(samples) * 1000, 2),
                        "max_ms": round(max(samples) * 1000, 2),
                    }

            tool_result_summary: dict[str, int] = {}
            if session_id is not None and tool_results is not None:
                try:
                    with SQLiteHelper("session").open(row_factory=True) as db:
                        rows = db.fetchall(
                            "SELECT COUNT(*) as cnt, SUM(is_error) as errs"
                            " FROM tool_results WHERE session_id = ?",
                            (session_id,),
                        )
                        if rows:
                            row = rows[0]
                            tool_result_summary = {
                                "total": row["cnt"],
                                "errors": row["errs"] or 0,
                            }
                except sqlite3.Error:
                    pass
```

Replace with:

```python
            session_id = ctx.session.session_id

            latency_summary = {}
            for step, samples in stats.stat_latency.items():
                if samples:
                    latency_summary[step] = {
                        "count": len(samples),
                        "mean_ms": round(sum(samples) / len(samples) * 1000, 2),
                        "max_ms": round(max(samples) * 1000, 2),
                    }

            # tool_results table removed (H-9); kept as an empty placeholder so the
            # session_diagnostics JSON payload shape is unchanged for existing consumers.
            tool_result_summary: dict[str, int] = {}
```

### Method

- Remove the `tool_results = ctx.tool_result_store` line, the `if session_id is not None and
  tool_results is not None:` guard, and the entire `try/except sqlite3.Error` query block inside
  it.
- Keep the `tool_result_summary: dict[str, int] = {}` declaration and its later use in the
  payload dict (`"tool_result_summary": tool_result_summary,`) completely unchanged — it is now
  simply always empty, with a comment explaining why.

### Details

- `session_id` remains in use by the workflow/task/approval/retry counter queries further down
  in the same method (`if session_id is not None:` block querying `workflow.sqlite`) — do not
  remove it.
- `sqlite3` import (used for `except sqlite3.Error` elsewhere in this method, e.g. the workflow
  query block and the WAL checkpoint code near the top of `_close_resources()`) stays in use —
  do not remove it even though this specific `except sqlite3.Error` is gone.
