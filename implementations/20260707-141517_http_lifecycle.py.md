# Implementation: H-1 — stderr log file redirect in http_lifecycle.py

## Goal

Replace `stderr=subprocess.PIPE` in `HttpServerLifecycleManager.start()` with per-server
log file redirection. Add `_stderr_files` and `_stderr_log_paths` dicts to manage file
handles. Close handles cleanly in `restart()` and `shutdown_all()`. Read tail from log
file when raising `HttpStartupError`.

## Scope

**Target**: `scripts/agent/http_lifecycle.py`

**Steps covered**: Plan H-1 steps 1, 2, 3.

**Out of scope**: test files, factory.py, config, DB schema.

## Assumptions

1. Log directory `/opt/llm/logs/mcp` is writable at agent startup.
2. `server_key` values are filesystem-safe; sanitize by replacing non-`[A-Za-z0-9_-]`
   chars with `_`.
3. 64 KiB tail cap is sufficient for diagnostic output.
4. File handles must be closed on every error path in `start()` (use try/finally).
5. `_stderr_log_paths` stores the log file path as `str` alongside the file handle in
   `_stderr_files`, so M-6 can read the path without inspecting the file handle.

## Implementation

### Target file

`scripts/agent/http_lifecycle.py`

### Procedure

#### Step 1: Add instance variables to `__init__()`

Add two new dicts to `HttpServerLifecycleManager.__init__()`:

```python
self._stderr_files: dict[str, IO[bytes]] = {}
self._stderr_log_paths: dict[str, str] = {}
```

Add `import re` and `from typing import IO` to the module imports (if not already
present; `IO` is from `typing`).

#### Step 2: Add `_open_stderr_log()` helper

Insert after `__init__()`:

```python
def _open_stderr_log(self, server_key: str) -> IO[bytes]:
    safe_key = re.sub(r"[^A-Za-z0-9_-]", "_", server_key)
    log_dir = Path("/opt/llm/logs/mcp")
    log_dir.mkdir(parents=True, exist_ok=True)
    log_path = log_dir / f"{safe_key}.stderr.log"
    fh = log_path.open("ab")
    self._stderr_log_paths[server_key] = str(log_path)
    return fh
```

Add `from pathlib import Path` to imports if not present.

#### Step 3: Update `start()` to use log file

Replace:
```python
proc = subprocess.Popen(
    cfg.cmd,
    stdout=subprocess.DEVNULL,
    stderr=subprocess.PIPE,
    env=env,
)
self._http_procs[server_key] = proc
```

With:
```python
stderr_fh = self._open_stderr_log(server_key)
self._stderr_files[server_key] = stderr_fh
try:
    proc = subprocess.Popen(
        cfg.cmd,
        stdout=subprocess.DEVNULL,
        stderr=stderr_fh,
        env=env,
    )
except Exception:
    stderr_fh.close()
    self._stderr_files.pop(server_key, None)
    self._stderr_log_paths.pop(server_key, None)
    raise
self._http_procs[server_key] = proc
```

#### Step 4: Replace `proc.stderr.read()` with log file tail reads

In the early-exit branch inside the health-poll loop:

Replace:
```python
stderr_full = (
    proc.stderr.read().decode(errors="replace")
    if proc.stderr
    else ""
)
```

With:
```python
stderr_full = self._read_stderr_tail(server_key)
stderr_fh = self._stderr_files.pop(server_key, None)
if stderr_fh is not None:
    stderr_fh.close()
self._stderr_log_paths.pop(server_key, None)
```

In the timeout branch after the loop:

Replace:
```python
stderr_full = proc.stderr.read().decode(errors="replace") if proc.stderr else ""
```

With:
```python
stderr_full = self._read_stderr_tail(server_key)
stderr_fh = self._stderr_files.pop(server_key, None)
if stderr_fh is not None:
    stderr_fh.close()
self._stderr_log_paths.pop(server_key, None)
```

#### Step 5: Add `_read_stderr_tail()` helper

```python
_STDERR_TAIL_BYTES = 64 * 1024

def _read_stderr_tail(self, server_key: str) -> str:
    log_path = self._stderr_log_paths.get(server_key)
    if not log_path:
        return ""
    try:
        with open(log_path, "rb") as f:
            f.seek(0, 2)
            size = f.tell()
            f.seek(max(0, size - self._STDERR_TAIL_BYTES))
            return f.read().decode(errors="replace")
    except OSError:
        return ""
```

#### Step 6: Update `restart()` — close handle before terminate

Insert at the top of `restart()`, before `proc = self._http_procs.pop(server_key, None)`:

```python
stderr_fh = self._stderr_files.pop(server_key, None)
if stderr_fh is not None:
    try:
        stderr_fh.close()
    except OSError:
        pass
self._stderr_log_paths.pop(server_key, None)
```

#### Step 7: Update `shutdown_all()` — close all handles after terminating

After the termination loop in `shutdown_all()`, add:

```python
for fh in list(self._stderr_files.values()):
    try:
        fh.close()
    except OSError:
        pass
self._stderr_files.clear()
self._stderr_log_paths.clear()
```

### Method

- Direct source edit: surgical replacement of the `subprocess.PIPE` line and associated
  `proc.stderr` reads.
- No new external dependencies; `Path`, `re`, `IO` are stdlib.
- `_open_stderr_log()` and `_read_stderr_tail()` are private; no protocol changes.

### Details

- File handle lifecycle: opened in `start()`, closed in the error paths of `start()`,
  in `restart()` (before re-start), and in `shutdown_all()`.
- On normal server operation the handle stays open because the subprocess writes to it
  continuously; it is closed only on restart or shutdown.
- `_stderr_log_paths` stores the string path separately from the file handle so future
  callers (e.g. M-6 process snapshot API) can surface the log path without needing the
  handle itself.
- `_STDERR_TAIL_BYTES = 64 * 1024` is a class-level constant (not instance-level).
