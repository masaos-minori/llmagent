# Implementation: mcp/github/models.py — GitHubConfig dataclass + domain exceptions

## Goal

Remove `_get_cfg()` module-level cache with `except Exception` + `{}` fallback from `models.py`. Add `GitHubConfig` dataclass for typed config injection and domain exception classes shared by service and server layers.

---

## Scope

**Target file:** `scripts/mcp/github/models.py`

**In:**
- Delete `_cfg` module variable, `_get_cfg()` function, and `_models_logger`
- Add `@dataclasses.dataclass` `GitHubConfig` with all config fields
- Add `GitHubConfig.from_dict(d: dict[str, Any]) -> GitHubConfig` classmethod
- Change `DEFAULT_PER_PAGE = 10` to a plain module constant (no config call)
- Add domain exception classes: `GitHubAuthorizationError`, `GitHubNotFoundError`, `GitHubValidationError`, `GitHubConflictError`, `GitHubUpstreamError`, `GitHubAuditError`
- Export `_get_cfg` as a backward-compat re-export for `service.py` (removed in Step 2)

**Out:**
- Pydantic request/response models unchanged
- `ConfigLoader` import retained (used by `GitHubConfig.from_dict`)

---

## Assumptions

1. `DEFAULT_PER_PAGE = 10` as a literal constant is safe — `service.py` already clamps per_page via `_clamp_per_page()`; the Pydantic Field default is cosmetic.
2. `GitHubConfig` fields mirror the keys currently read via `_get_cfg().get(...)` in service.py: `allowed_repos`, `allowed_repos_mode`, `path_denylist`, `protected_branches`, `max_file_size_kb`, `audit_log_path`, `allow_force_push`, `require_pr_review`, `default_per_page`, `max_per_page`, `llm_url`.
3. `audit_log_path = ""` means audit is disabled (no write attempted).
4. Domain exceptions are defined here (not a separate file) to keep `models.py` the single local dep of `service.py`.

## Implementation

### Target file
`scripts/mcp/github/models.py`

### Procedure

1. Remove `_models_logger`, `_cfg`, `_get_cfg()`.
2. Add `import dataclasses` at the top.
3. Add `GitHubConfig` dataclass after imports.

5. Change `DEFAULT_PER_PAGE: int = 10` (remove `_get_cfg()` call).

### Method

```python
import dataclasses
from typing import Any
from pydantic import BaseModel, Field
from shared.config_loader import ConfigLoader

DEFAULT_PER_PAGE: int = 10


@dataclasses.dataclass
class GitHubConfig:
    allowed_repos: list[str] = dataclasses.field(default_factory=list)
    allowed_repos_mode: str = "fail_closed"
    path_denylist: list[str] = dataclasses.field(default_factory=list)
    protected_branches: list[str] = dataclasses.field(default_factory=list)
    max_file_size_kb: int = 0
    audit_log_path: str = ""
    allow_force_push: bool = True
    require_pr_review: bool = False
    default_per_page: int = DEFAULT_PER_PAGE
    max_per_page: int = 100
    llm_url: str = ""

    @classmethod
    def from_dict(cls, d: dict[str, Any]) -> "GitHubConfig":
        return cls(
            allowed_repos=list(d.get("allowed_repos", [])),
            allowed_repos_mode=str(d.get("allowed_repos_mode", "fail_closed")),
            path_denylist=list(d.get("path_denylist", [])),
            protected_branches=list(d.get("protected_branches", [])),
            max_file_size_kb=int(d.get("max_file_size_kb", 0)),
            audit_log_path=str(d.get("audit_log_path", "")),
            allow_force_push=bool(d.get("allow_force_push", True)),
            require_pr_review=bool(d.get("require_pr_review", False)),
            default_per_page=int(d.get("default_per_page", DEFAULT_PER_PAGE)),
            max_per_page=int(d.get("max_per_page", 100)),
            llm_url=str(d.get("llm_url", "")),
        )

    @classmethod
    def load(cls) -> "GitHubConfig":
        """Load from github_mcp_server.toml; raises on failure (fail-fast)."""
        return cls.from_dict(ConfigLoader().load("github_mcp_server.toml"))


class GitHubAuthorizationError(RuntimeError):
    """Raised when a repo/path/branch policy check fails (HTTP 403)."""

class GitHubNotFoundError(RuntimeError):
    """Raised when a GitHub resource is not found (HTTP 404)."""

class GitHubValidationError(ValueError):
    """Raised on invalid input (HTTP 400/422)."""

class GitHubConflictError(RuntimeError):
    """Raised on a GitHub conflict (HTTP 409)."""

class GitHubUpstreamError(RuntimeError):
    """Raised on GitHub API 5xx or unexpected upstream failures."""

class GitHubAuditError(RuntimeError):
    """Raised when audit log writing fails and audit_log_path is configured."""
```

### Details

- `GitHubConfig.load()` is the canonical entry point for server startup; it lets `ConfigLoader` raise on missing file — no silent `{}` fallback.
- `from_dict` is the injection entry point for tests: pass a plain dict without touching the filesystem.
- `_get_cfg` backward-compat re-export is NOT needed: service.py will be updated in Step 2 to accept `GitHubConfig` directly.
