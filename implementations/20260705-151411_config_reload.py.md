# Implementation: agent/services/config_reload.py — Add workflow_mode and workflow_require_approval to _detect_startup_only()

## Goal

Extend `_detect_startup_only()` to also detect changes to `workflow_mode` and `workflow_require_approval`, so that `/reload` correctly reports them as startup-only instead of silently ignoring them.

## Scope

**In**: Add two field checks to `_detect_startup_only()`.

**Out**: Changes to hot-reload logic, tests, config dataclasses.

## Assumptions

1. `ctx.cfg.workflow.workflow_mode` holds the current runtime value (type `str`).
2. `ctx.cfg.workflow.workflow_require_approval` holds the current runtime value (type `bool`).
3. `new_cfg` is the parsed TOML dict from the new config file.
4. Helper functions `_get_str()` and `_get_bool()` already exist in `config_reload.py`.
5. The `[workflow]` section in TOML corresponds to `ctx.cfg.workflow` dataclass.

## Implementation

### Target file
`scripts/agent/services/config_reload.py`

### Procedure
1. Read `agent/config_dataclasses.py` to confirm `workflow_mode` and `workflow_require_approval` field paths.
2. Add `workflow_mode` check after existing startup-only checks.
3. Add `workflow_require_approval` check.

### Method

**Existing checks in `_detect_startup_only()` (for reference pattern):**
```python
v = _get_bool(new_cfg, "use_memory_layer")
if v is not None and v != ctx.cfg.use_memory_layer:
    changed.append("use_memory_layer")
v = _get_bool(new_cfg, "plugin_strict")
if v is not None and v != ctx.cfg.plugin_strict:
    changed.append("plugin_strict")
```

**New additions:**
```python
# workflow subsection is nested: [workflow] in TOML
wf_cfg = new_cfg.get("workflow") if isinstance(new_cfg.get("workflow"), dict) else {}

v_mode = wf_cfg.get("workflow_mode")
if v_mode is not None and v_mode != ctx.cfg.workflow.workflow_mode:
    changed.append("workflow_mode")

v_approval = wf_cfg.get("workflow_require_approval")
if v_approval is not None and v_approval != ctx.cfg.workflow.workflow_require_approval:
    changed.append("workflow_require_approval")
```

### Details

- If `workflow_mode` / `workflow_require_approval` are top-level keys (not nested), replace `wf_cfg.get(...)` with `_get_str(new_cfg, "workflow_mode")` / `_get_bool(new_cfg, "workflow_require_approval")`. Verify by reading the existing flat-key pattern in the function.
- The result is appended to `changed` list; callers place this in `ConfigReloadOutcome.startup_only`.
- No hot-reload logic changes — these fields are NOT added to `_sync_services()`.
