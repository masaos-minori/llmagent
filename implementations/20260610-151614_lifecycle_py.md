# Implementation: lifecycle.py

## Goal

Add explicit reachable-state and transition-rule documentation to `LifecycleState`, and add a `STARTING` state to reflect the subprocess startup phase that both HTTP and stdio managers go through. Transition rules prevent callers from making invalid assumptions.

## Scope

- Target: `scripts/agent/lifecycle.py`
- Add `STARTING` enum value
- Add module-level `_VALID_TRANSITIONS` mapping that codifies which state changes are legal
- Add `assert_valid_transition(from_state, to_state)` helper that raises `ValueError` on illegal transitions
- `TransportState` in `stdio_lifecycle.py` must be reconciled: merge or alias to `LifecycleState` so one enum covers all managers

## Assumptions

1. `LifecycleState` is the canonical state type shared across HTTP and stdio managers.
2. `TransportState` in `stdio_lifecycle.py` is a separate enum with identical values — it must be replaced by `LifecycleState`.
3. Callers that currently use `TransportState.RUNNING` etc. will be updated to use `LifecycleState.RUNNING`.

## Implementation

### Target file

`scripts/agent/lifecycle.py`

### Procedure

1. Add `STARTING = "starting"` to `LifecycleState`.
2. Define `_VALID_TRANSITIONS: dict[LifecycleState, frozenset[LifecycleState]]` with legal edges:
   - `STOPPED → {STARTING, FAILED}`
   - `STARTING → {RUNNING, FAILED, STOPPED}`
   - `RUNNING → {STOPPED, FAILED, STARTING}`
   - `FAILED → {STARTING, STOPPED}`
   - `UNKNOWN → {STARTING, RUNNING, STOPPED, FAILED}`
3. Add `assert_valid_transition(from_state: LifecycleState, to_state: LifecycleState) -> None` that raises `ValueError(f"Invalid transition {from_state!r} → {to_state!r}")`.
4. Delete `TransportState` from `stdio_lifecycle.py` and replace all usages with `LifecycleState` (same step to keep the diff atomic).

### Method

- Extend the Enum in-place; no class restructuring required.
- `_VALID_TRANSITIONS` and `assert_valid_transition` are module-level, not methods of the Enum.

### Details

```python
from __future__ import annotations
from enum import Enum


class LifecycleState(Enum):
    STARTING = "starting"
    RUNNING  = "running"
    STOPPED  = "stopped"
    FAILED   = "failed"
    UNKNOWN  = "unknown"


_VALID_TRANSITIONS: dict[LifecycleState, frozenset[LifecycleState]] = {
    LifecycleState.STOPPED:  frozenset({LifecycleState.STARTING, LifecycleState.FAILED}),
    LifecycleState.STARTING: frozenset({LifecycleState.RUNNING, LifecycleState.FAILED, LifecycleState.STOPPED}),
    LifecycleState.RUNNING:  frozenset({LifecycleState.STOPPED, LifecycleState.FAILED, LifecycleState.STARTING}),
    LifecycleState.FAILED:   frozenset({LifecycleState.STARTING, LifecycleState.STOPPED}),
    LifecycleState.UNKNOWN:  frozenset(LifecycleState),
}


def assert_valid_transition(from_state: LifecycleState, to_state: LifecycleState) -> None:
    if to_state not in _VALID_TRANSITIONS.get(from_state, frozenset()):
        raise ValueError(f"Invalid lifecycle transition: {from_state!r} -> {to_state!r}")
```

Update all `TransportState.*` references in `stdio_lifecycle.py` to `LifecycleState.*` in the same commit.
