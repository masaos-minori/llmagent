# Implementation: H-7 — Update docs for removed AgentContext.tool_result_store

## Goal

Update three documentation files that describe `ctx.tool_result_store` as a live
`AgentContext` field or describe `mark_turn_undone(...)` as the active write path for the
`tool_results.undone` column — both are no longer true after H-7 lands.

## Scope

**Targets**: `docs/05_agent_02_runtime-architecture.md`, `docs/05_agent_04_state-and-persistence.md`,
`docs/05_agent_09_data-layer.md`

**Depends on**: apply after all H-7 source-code docs have landed (this is a documentation-only
change with no code dependency, but should describe the FINAL state, not an intermediate one).

**Out of scope**: `db/tool_results.py`'s own documentation of the `ToolResultStore` class API
(unchanged — the class itself is not removed, only `AgentContext`'s ownership of an instance).

## Assumptions

1. `docs/05_agent_09_data-layer.md`'s "tool_results.undone column" section (both the Japanese
   section at line ~95-107 and its English counterpart at line ~245-257) currently describes
   `mark_turn_undone(session_id, turn)` as being called by `/undo` — this becomes false once
   `undo_service.py`'s H-7 change lands (the call is removed entirely; `undone` stays `0`
   forever for rows written before this rollout, and no new rows are written at all after
   H-1/H-2/H-4). This finding was not explicitly listed in the H-7 plan's "Docs" affected-areas
   list (which only named the `tool_result_store` field references) but directly follows from
   the same `undo_service.py` change and would leave stale, actively misleading documentation if
   not corrected together.

## Implementation

### Target files

`docs/05_agent_02_runtime-architecture.md`, `docs/05_agent_04_state-and-persistence.md`,
`docs/05_agent_09_data-layer.md`

### Procedure

#### File 3: `docs/05_agent_09_data-layer.md`

Update BOTH the Japanese and English "tool_results.undone column" sections to reflect that
`mark_turn_undone` is no longer called.

Japanese section (around line 104-107), current:

```markdown
#### tool_results.undone カラム

`undone INTEGER NOT NULL DEFAULT 0` — `/undo` 実行時に `mark_turn_undone(session_id, turn)` が `1` にセットする。
アーティファクトは**削除されない**。`/tool list` では `[undone]` アノテーションが付き、`/tool show <id>` では `[undone turn — artifact retained for audit]` が表示される。
```

Replace with:

```markdown
#### tool_results.undone カラム

`undone INTEGER NOT NULL DEFAULT 0` — H-7 以降、`/undo` は `mark_turn_undone(...)` を呼ばなくなった
(`AgentContext` が `tool_result_store` を保持しなくなったため)。既存行の `undone` フラグは変更されず、
新規のツール実行結果も `tool_results` テーブルへは書き込まれない (H-1/H-2/H-4 で書き込み経路が撤去済み)。
`/tool list` の `[undone]` アノテーション表示ロジック自体は残るため、ロールアウト前に `undone=1` で
書き込まれていた既存行があれば引き続き表示される。
```

English section (around line 254-257), current:

```markdown
#### tool_results.undone column

`undone INTEGER NOT NULL DEFAULT 0` — set to `1` by `mark_turn_undone(session_id, turn)` when `/undo` is called.
Artifacts are **never deleted**. `/tool list` shows a `[undone]` annotation; `/tool show <id>` displays `[undone turn — artifact retained for audit]`.
```

Replace with:

```markdown
#### tool_results.undone column

`undone INTEGER NOT NULL DEFAULT 0` — as of H-7, `/undo` no longer calls `mark_turn_undone(...)`
(`AgentContext` no longer owns a `tool_result_store`). Pre-existing rows keep whatever `undone`
value they already had; no new rows are written to `tool_results` at all (the write paths were
removed in H-1/H-2/H-4). The `/tool list` `[undone]` annotation display logic itself is
unchanged, so any row already marked `undone=1` from before this rollout still displays as such.
```
