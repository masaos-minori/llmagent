# Implementation: Enrich StageResult with Failure Tracking and Refiner/HTTP Fallback Reasons

## Goal

Make exception-level failures and fallback paths in the RAG pipeline visible in `last_stage_results`. Currently `StageResult.status` is only ever `"success"` or `"fallback"`; exceptions during stage execution are not captured. Similarly, the refiner and HTTP fallback paths in `augment()` have no representation in `last_stage_results`.

## Scope

- `scripts/rag/pipeline.py`
  - Steps 1+2: wrap per-stage `await stage.run(ctx, db=db)` in try/except to record `"failure"` status
  - Add `StageResult` entries for `_augment_refiner` and `_augment_http` paths

Out of scope:
- `stage.py` (type already has `"failure"` in the comment; no change needed)
- `pipeline_service.py` min_score/max_chunks fix (tracked separately)
- MCP debug path

## Assumptions

1. `StageResult.status: str` already has `"failure"` documented as a valid value (`stage.py` line 19).
2. No existing consumer checks for exact status strings other than `"fallback"` (line 264 in pipeline.py); adding `"failure"` entries is safe.
3. When a stage raises, the pipeline should continue to the next stage (best-effort) rather than aborting — consistent with current soft-failure behavior in `search_queries()`.
4. For `_augment_http`: record `StageResult` only when the HTTP call is actually attempted (i.e., `rag_service_url` is set); use `"fallback"` when returning `None` (in-process fallback triggered), `"success"` when returning a non-None value.
5. For `_augment_refiner`: record `"fallback"` when returning `None`, `"success"` otherwise.

## Implementation

### Target file

`scripts/rag/pipeline.py`

### Procedure

1. In `run()`, wrap `await stage.run(ctx, db=db)` (line 213) in try/except to catch `Exception` and record `"failure"` status with the exception message as `fallback_reason`.
2. In `augment()`, after calling `_augment_http()` and `_augment_refiner()`, append a `StageResult` to `self.last_stage_results` reflecting the outcome.

### Method

Minimal try/except addition. Re-raise is not appropriate (best-effort pipeline). The `augment()` method builds `last_stage_results` from `run()`'s output; refiner/HTTP results are appended directly to `self.last_stage_results` after `run()` completes.

### Details

**Step 1 — failure status in per-stage loop (pipeline.py, around line 211-224):**

Replace:
```python
for stage in pre_augment_stages:
    t0 = time.perf_counter()
    await stage.run(ctx, db=db)
    elapsed = time.perf_counter() - t0
    self.last_timings[stage.__class__.__name__] = elapsed
    status, reason = self._get_stage_status(stage, ctx)
    ctx.stage_results.append(StageResult(...))
```

With:
```python
for stage in pre_augment_stages:
    t0 = time.perf_counter()
    exc_msg: str | None = None
    try:
        await stage.run(ctx, db=db)
    except Exception as e:
        exc_msg = str(e)
        logger.warning("Stage %s failed: %s", stage.__class__.__name__, e)
    elapsed = time.perf_counter() - t0
    self.last_timings[stage.__class__.__name__] = elapsed
    if exc_msg is not None:
        status, reason = "failure", exc_msg
    else:
        status, reason = self._get_stage_status(stage, ctx)
    ctx.stage_results.append(StageResult(
        stage_name=stage.__class__.__name__,
        status=status,
        elapsed_seconds=elapsed,
        fallback_reason=reason,
    ))
```

**Step 2 — refiner/HTTP fallback in augment() (pipeline.py, around line 344-386):**

After `_augment_http()` call, append:
```python
if rag_url := self._cfg.rag_service_url:
    result = await self._augment_http(rag_url, query, history_context)
    http_status = "success" if result is not None else "fallback"
    self.last_stage_results.append(StageResult(
        stage_name="HttpAugment",
        status=http_status,
        elapsed_seconds=0.0,
        fallback_reason="in-process fallback" if result is None else None,
    ))
    if result is not None:
        return result
```

After `_augment_refiner()` call, append:
```python
if self._cfg.use_refiner:
    refined = await self._augment_refiner(reranked, query)
    refiner_status = "success" if refined is not None else "fallback"
    self.last_stage_results.append(StageResult(
        stage_name="Refiner",
        status=refiner_status,
        elapsed_seconds=0.0,
        fallback_reason="refiner returned None" if refined is None else None,
    ))
    if refined is not None:
        return refined
```

Note: `elapsed_seconds=0.0` is a placeholder; timing these requires wrapping the calls with `time.perf_counter()` as done in `run()`.
