## Implementation

### Target file

### Procedure

### Method

Code audit + targeted fixes. The main logic is written in req #35's app.py doc.

### Details

**Correctness checklist for subscribe() SSE generator:**

```
✓ _broker.subscribe(topics) called BEFORE _fetch_replay()
  → events published during replay go into the queue; replay_ceil filters duplicates

✓ replay_ceil initialized to start_seq (not 0)
  → avoids erroneously discarding events at seq 1 when start_seq > 0

✓ replay_ceil updated to max(row["seq"]) during replay loop
  → correctly marks the boundary between replay and live

✓ queue.get() returns None → break (shutdown sentinel)
  → graceful shutdown when lifespan tears down

✓ event["seq"] <= replay_ceil → continue (not yield)
  → prevents duplicate delivery at the replay-live boundary

✓ asyncio.CancelledError caught → log disconnect
  → no uncaught exception in SSE generator

✓ finally: _broker.unsubscribe(sub)
  → subscriber always removed even on exception or cancellation

✓ No asyncio.sleep in subscribe()
  → no polling latency; live events delivered immediately on queue.get()

✓ No write_offset in subscribe() body
  → offsets advanced only via POST /events/{id}/ack (req #29)
```

**Replay race safety verification (timing diagram):**
```
T0: sub = _broker.subscribe(topics)     ← queue starts receiving
T1: rows = await to_thread(_fetch_replay)  ← SQLite query (seq > start_seq)
T2: publish(A) → broker.publish(A) → sub.queue.put(A)  [seq=5, during replay fetch]
T3: replay yields rows including A (seq=5), replay_ceil = 5
T4: queue.get() → A (seq=5) → 5 <= replay_ceil=5 → skip ✓
T5: publish(B) → broker.publish(B) → sub.queue.put(B)  [seq=6]
T6: queue.get() → B (seq=6) → 6 > replay_ceil=5 → yield ✓
```
