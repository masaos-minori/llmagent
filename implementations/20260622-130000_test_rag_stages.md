# Implementation: test_rag_stages.py — TestAugmentStage 改善

## Goal

`tests/test_rag_stages.py` の `TestAugmentStage` クラスに対して、`RawHit` 生成の重複を
`_make_raw_hit()` ヘルパーで排除し、`_format_chunks` の直接 unit test と
`sanitize_document` 動作テストを追加する。

## Scope

**In-Scope:**
- `tests/test_rag_stages.py` — `_make_raw_hit()` ヘルパー追加、`TestAugmentStage` テスト追加

**Out-of-Scope:**
- 他のテストクラス (`TestSearchStage`, `TestMqeStage`, etc.) の変更
- `scripts/rag/stages/augment.py` の実装変更
- httpx.HTTPStatusError / Timeout テスト — 現在の `AugmentStage` 実装は HTTP 非使用のため適用不可

## Assumptions

1. `AugmentStage.run()` は HTTP リクエストを行わず、`_format_chunks()` + `sanitize_document()` のみを使用する。
2. `sanitize_document()` は `[REMOVED]` で injection pattern を置換する副作用があり、テストで確認可能。
3. `RawHit` は `chunk_id`, `content`, `url`, `title` (省略可) フィールドを持つ。
