# Implementation: Update `classify_risk()` in `scripts/agent/tool_policy.py`

## Goal

Add `tool_constants.py` classification as priority 3 fallback in `classify_risk()` so that delete
and shell tools are classified HIGH by default (without requiring explicit `tool_safety_tiers`
configuration), and write tools are classified MEDIUM. Explicit `approval_risk_rules` (priority 1)
and `tool_safety_tiers` (priority 2) continue to override.

## Scope

- In-Scope: Update `classify_risk()` in `scripts/agent/tool_policy.py`:
  - Add `from shared.tool_constants import DELETE_TOOLS, SHELL_TOOLS, WRITE_TOOLS` to imports.
  - Replace the single `tool_safety_tiers.get(tool_name, "WRITE_DANGEROUS")` fallback with
    a two-step check: (a) if in tiers, use tier; (b) else, apply constants-based classification.
- Out-of-Scope: No changes to `_special_case_risk()`, `_escalate_for_path()`, or
  `_escalate_for_github_branch()`. No new `ApprovalPolicy` class.

## Assumptions

1. `scripts/shared/tool_constants.py` exports `DELETE_TOOLS`, `SHELL_TOOLS`, `WRITE_TOOLS` as
   `frozenset[str]` (confirmed by the plan).
2. `agent → shared` import direction is allowed by `.importlinter`.
3. After the fix, `delete_directory` and `delete_file` (in `DELETE_TOOLS`) return `RiskLevel.HIGH`
   when not overridden by tiers or explicit rules.
4. `shell_run` (in `SHELL_TOOLS`) returns `RiskLevel.HIGH` by default.
5. `write_file` (in `WRITE_TOOLS`) returns `RiskLevel.MEDIUM` by default (no change from current
   "WRITE_DANGEROUS" tier behavior).

## Implementation

### Target file

`scripts/agent/tool_policy.py` (existing)

### Procedure

1. Read `classify_risk()` implementation (lines 120-140 approximately) to confirm the exact
   current logic.
2. Add `DELETE_TOOLS, SHELL_TOOLS, WRITE_TOOLS` to the `from shared.tool_constants import ...`
   line (or add a new import if `tool_constants` is not yet imported).
3. Replace the tier fallback block with the two-step check as shown below.
4. Run `uv run ruff check scripts/agent/tool_policy.py` — expect 0 errors.
5. Run `uv run mypy scripts/agent/tool_policy.py` — expect no new errors.

### Method

```python
# Add to imports:
from shared.tool_constants import DELETE_TOOLS, SHELL_TOOLS, WRITE_TOOLS

# Update classify_risk() — replace the if/base block:
# Before:
if base is None:
    tier = cfg.approval.tool_safety_tiers.get(tool_name, "WRITE_DANGEROUS")
    base = _TIER_TO_RISK.get(tier, RiskLevel.MEDIUM)

# After:
# Priority 2: tool_safety_tiers
if base is None and tool_name in cfg.approval.tool_safety_tiers:
    tier = cfg.approval.tool_safety_tiers[tool_name]
    base = _TIER_TO_RISK.get(tier, RiskLevel.MEDIUM)

# Priority 3: tool_constants.py classification
if base is None:
    if tool_name in DELETE_TOOLS or tool_name in SHELL_TOOLS:
        base = RiskLevel.HIGH
    elif tool_name in WRITE_TOOLS:
        base = RiskLevel.MEDIUM
    else:
        base = RiskLevel.MEDIUM  # Priority 4: unchanged default
```

### Details

- The change from `tool_safety_tiers.get(tool_name, "WRITE_DANGEROUS")` to a two-step check
  preserves the priority ordering: explicit rules > tiers > constants > default.
- `RiskLevel.MEDIUM` is the same result for `WRITE_TOOLS` and the default, so behavior is
  unchanged for write and unknown tools.
- Only DELETE_TOOLS and SHELL_TOOLS get a new HIGH classification.
