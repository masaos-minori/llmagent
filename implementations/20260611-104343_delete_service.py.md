# Implementation: mcp/file/delete_service.py — Recursive delete safety boundary

## Goal

Add a guard in `delete_directory()` that prevents deleting an `allowed_dirs` root
itself when `recursive=True`. Deleting an allowed root would wipe out the entire
configured workspace, which is an unintended and unrecoverable operation.

## Scope

**Target file**: `scripts/mcp/file/delete_service.py`
**Test file**: `tests/test_file_delete_mcp_service.py`

In scope:
- Add a guard check at the start of the actual-delete path in `delete_directory()`
- Unit tests confirming the guard fires correctly

Out of scope:
- `delete_file()` (files cannot be an allowed_dir root)
- dry-run path (no actual deletion occurs)
- Non-recursive directory deletion (ENOTEMPTY guard is sufficient)

## Assumptions

1. `resolve_safe()` already verified that `target` is under some allowed_dir.
2. The guard checks `target == allowed_dir.resolve()` for all configured dirs.
   This is a strict equality check, not a prefix check, to avoid false positives
   on similarly-named directories.
3. The check fires ONLY in the non-dry-run path. Dry-run must still be allowed
   for reporting purposes.
4. The guard raises `FileAuthorizationError` — semantically this is an authorization
   boundary violation, consistent with how `resolve_safe()` raises it.

## Implementation

### Target file

`scripts/mcp/file/delete_service.py`

### Procedure

Locate `delete_directory()` at line ~137. After the dry-run block ends and before
the actual `shutil.rmtree` / `target.rmdir()` call, add the guard:

**Before (simplified):**
```python
        # ... dry_run block ...

        try:
            if req.recursive:
                shutil.rmtree(str(target))
            else:
                target.rmdir()
```

**After:**
```python
        # ... dry_run block ...

        # Guard: never delete an allowed_dir root itself.
        if req.recursive:
            for allowed in self._allowed_dirs:
                if target == allowed.resolve():
                    raise FileAuthorizationError(
                        f"Deleting an allowed root directory is not permitted: {target}",
                    )

        try:
            if req.recursive:
                shutil.rmtree(str(target))
            else:
                target.rmdir()
```

### Method

Direct `Edit` tool. The guard block is inserted as a new `if req.recursive:` block
immediately before the existing `try:` block.

### Details

- `self._allowed_dirs` is a `list[Path]` already resolved at service construction time
  via `build_service()` in `delete_server.py` which calls `Path(d)` for each configured dir.
  Call `.resolve()` at check time to handle any remaining symlinks or relative refs.
- The guard only fires for `recursive=True` because non-recursive `target.rmdir()` would
  already fail with `OSError(ENOTEMPTY)` if the directory has contents.
- The plan's mitigation: "target == allowed_dir" comparison only — this is exactly what
  the guard implements.

New tests to add in `test_file_delete_mcp_service.py`:
- `test_delete_directory_recursive_allowed_root_raises`: set `target = allowed_dir` itself;
  assert `FileAuthorizationError` raised with `recursive=True`.
- `test_delete_directory_recursive_subdir_allowed`: set `target = allowed_dir / "subdir"`;
  assert deletion proceeds normally (guard must not fire for subdirectories).
- `test_delete_directory_non_recursive_allowed_root_no_guard`: non-recursive on root;
  guard must NOT fire (ENOTEMPTY will handle this if non-empty).
