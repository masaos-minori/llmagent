# Implementation: H-6 — tests for startup rollback (test_startup.py)

## Goal

Add tests verifying that `StartupOrchestrator.run()` calls `lifecycle.shutdown_all()`
when a step after `_start_servers()` raises, that rollback shutdown failures do not mask
the original exception, and that `shutdown_all()` is NOT called when `_initialize()` or
`_start_servers()` itself fails.

## Scope

**Target**: `tests/test_startup.py`

**Step covered**: Plan H-6 step 2.

**Out of scope**: source changes, other test files.

## Assumptions

1. `tests/test_startup.py` exists or can be created.
2. `StartupOrchestrator` can be constructed with a stub `AgentContext` and `CLIView`.
3. Individual `_start_servers()`, `_check_services()`, etc. can be patched with
   `monkeypatch.setattr()` or `AsyncMock` on the orchestrator instance.

## Implementation

### Target file

`tests/test_startup.py`

### Procedure

#### Setup helper

```python
def make_orchestrator():
    ctx = make_stub_ctx()
    view = MagicMock()
    orch = StartupOrchestrator(ctx, view)
    # Patch _initialize() to a no-op
    orch._initialize = MagicMock()
    # Patch lifecycle on ctx after _initialize() would have set services
    mock_lifecycle = AsyncMock()
    ctx.services = make_stub_services(lifecycle=mock_lifecycle)
    return orch, ctx, mock_lifecycle
```

#### Test 1: _check_services raises → shutdown_all() called

```python
@pytest.mark.asyncio
async def test_startup_rollback_on_check_services_failure():
    orch, ctx, mock_lifecycle = make_orchestrator()
    orch._start_servers = AsyncMock()
    orch._check_services = AsyncMock(side_effect=RuntimeError("health check failed"))
    orch._recover_pending_approvals = AsyncMock()
    orch._setup_prompt = AsyncMock()

    with pytest.raises(RuntimeError, match="health check failed"):
        await orch.run()

    mock_lifecycle.shutdown_all.assert_awaited_once()
```

#### Test 3: Rollback shutdown_all() raises → original exception preserved

```python
@pytest.mark.asyncio
async def test_startup_rollback_shutdown_failure_preserves_original_error():
    orch, ctx, mock_lifecycle = make_orchestrator()
    orch._start_servers = AsyncMock()
    orch._check_services = AsyncMock(side_effect=RuntimeError("original error"))
    orch._recover_pending_approvals = AsyncMock()
    orch._setup_prompt = AsyncMock()
    mock_lifecycle.shutdown_all.side_effect = OSError("shutdown failed")

    with pytest.raises(RuntimeError, match="original error"):
        await orch.run()
```

#### Test 4: _initialize raises → shutdown_all() NOT called

```python
@pytest.mark.asyncio
async def test_startup_no_rollback_on_initialize_failure():
    orch, ctx, mock_lifecycle = make_orchestrator()
    orch._initialize = MagicMock(side_effect=RuntimeError("init failed"))

    with pytest.raises(RuntimeError, match="init failed"):
        await orch.run()

    mock_lifecycle.shutdown_all.assert_not_awaited()
```

#### Test 5: _start_servers raises → shutdown_all() NOT called

```python
@pytest.mark.asyncio
async def test_startup_no_rollback_on_start_servers_failure():
    orch, ctx, mock_lifecycle = make_orchestrator()
    orch._start_servers = AsyncMock(side_effect=RuntimeError("server start failed"))
    orch._check_services = AsyncMock()

    with pytest.raises(RuntimeError, match="server start failed"):
        await orch.run()

    mock_lifecycle.shutdown_all.assert_not_awaited()
```

### Method

- `orch._initialize` is patched to a `MagicMock()` no-op; services are pre-wired on
  `ctx` via `ctx.services` assignment.
- `AsyncMock(side_effect=...)` raises the given exception when awaited.
- Tests use `assert_awaited_once()` / `assert_not_awaited()` to verify call count.

### Details

- `make_stub_ctx()` must return an `AgentContext`-compatible stub where
  `ctx.services_required.lifecycle` resolves to `mock_lifecycle`. If `services_required`
  raises on `None`, assign `ctx.services` directly using an `AppServices` stub.
- `make_stub_services(lifecycle=...)` returns an `AppServices`-like object with the
  given lifecycle. Use `MagicMock(spec=AppServices)` and set `lifecycle` attribute.
- Tests 1 and 2 share identical rollback assertion; test 3 verifies the additional
  constraint that rollback failures do not swallow the original exception.
