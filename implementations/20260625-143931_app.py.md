# Implementation: app.py — inline DLQ promotion in nack handler (req #26)

## Goal

Update the `POST /events/{event_id}/nack` endpoint to call `promote_single()` immediately when `delivery_failure_count` reaches `max_retry`, eliminating the 60-second DLQ promotion delay.

## Scope

- Update nack endpoint (created in req #24) to call `promote_single()` on threshold
- The nack + check + promote must all run inside a single `asyncio.to_thread()` call (same Lock)
- Background `_dlq_loop()` continues running as a safety-net sweep

## Assumptions

- req #24 nack endpoint and req #26 `promote_single()` are both implemented first
- `_cfg` is non-None after lifespan; assert before use
- The inline check `count >= _cfg.max_retry` uses the returned count from `nack_event()`
- `deadletter_dir` is available from `_cfg`

## Implementation

### Target file

`scripts/eventbus/app.py`

### Procedure

1. Import `promote_single` from `eventbus.dlq` (add alongside existing `promote_to_dlq` import)
2. Replace the simple `_nack()` closure in the nack endpoint with `_nack_and_maybe_promote()`
3. The new closure atomically increments failure count and conditionally promotes inline

### Method

Update the nack endpoint added in req #24. Import section update required.

### Details

**Updated import:**
```python
from eventbus.dlq import promote_single, promote_to_dlq
```

**Updated nack endpoint:**
```python
@app.post("/events/{event_id}/nack")
async def nack_event_endpoint(event_id: str) -> dict[str, Any]:
    def _nack_and_maybe_promote() -> dict[str, Any]:
        assert _db is not None
        assert _cfg is not None
        count = nack_event(_db, event_id)
        if count < 0:
            return {"found": False, "failure_count": -1, "promoted": False}
        promoted = False
        if count >= _cfg.max_retry:
            promoted = promote_single(_db, _cfg.deadletter_dir, event_id)
        return {"found": True, "failure_count": count, "promoted": promoted}

    result = await asyncio.to_thread(_nack_and_maybe_promote)
    if not result["found"]:
        raise HTTPException(status_code=404, detail="event not found")
    return {
        "event_id": event_id,
        "failure_count": result["failure_count"],
        "promoted": result["promoted"],
    }
```

The `nack_event()` call and `promote_single()` call run in the same `asyncio.to_thread()` worker, so both execute under the same `threading.Lock` (per req #16 connection model). This prevents a race between the nack counter increment and the inline promotion check.
