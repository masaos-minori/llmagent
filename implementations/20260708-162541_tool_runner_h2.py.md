# Implementation: H-2 — Remove ToolResultStore.store() call from _collect_tool_result_msgs()

## Goal

Stop writing tool output full text to the `tool_results` SQLite table. Remove the
`ctx.tool_result_store.store(...)` call and its now-unused `summarized`/`is_summarized()`
computation from `_collect_tool_result_msgs()`. Conversation-flow persistence (the `messages`
table via `ctx.session.save_many(...)`) and in-memory LLM history (`ctx.conv.history.append(...)`)
are unaffected.

## Scope

**Target**: `scripts/agent/tool_runner.py`

**Depends on**: the H-3 implementation step for this same file (`_apply_turn_char_limit()`
signature reduction removing `result_id` — see
`implementations/20260708-162416_tool_runner.py.md` or its `done/` copy). Apply that change
first; this step only removes the `.store()` call and `summarized` variable, which sit upstream
of the (already-reduced) `_apply_turn_char_limit()` call.

**Out of scope**:
- `scripts/db/tool_results.py` (the store/table implementation itself — untouched)
- `scripts/agent/context.py` (`ctx.tool_result_store` field stays; `/tool list` and
  `/tool show <id>` in `cmd_tooling.py` still read from it for pre-existing rows)
- `scripts/agent/commands/cmd_tooling.py` (no change)

## Assumptions

1. The H-3 step has already reduced `_apply_turn_char_limit()` to
   `(llm_text, turn_chars, limit) -> str` with no `result_id` parameter, and the call site no
   longer passes `result_id=result_id`. If H-3 has not landed yet, apply it first — this doc's
   diff assumes that starting state.
2. `is_summarized` (imported from `agent.tool_result_formatter` at line 30) has no other call
   site in `tool_runner.py` besides the one being removed (verify via grep in Procedure Step 1).
3. `summarized` is used only as the `summary=llm_text if summarized else None` argument to the
   removed `.store(...)` call — no other reference.

## Implementation

### Target file

`scripts/agent/tool_runner.py`

### Procedure

#### Step 1: Confirm `is_summarized` and `summarized` have no other use

```bash
grep -n "is_summarized\|\bsummarized\b" scripts/agent/tool_runner.py
```

Expect exactly two hits: the import (line 30) and the single call/assignment
(`summarized = is_summarized(...)`, line 195) being removed in Step 2.

#### Step 2: Remove the `summarized` computation and `.store()` call

Current state (post-H-3, i.e. `_apply_turn_char_limit()` already has no `result_id` param):

```python
        summarized = is_summarized(ctx.cfg, text, llm_text, is_error)
        result_id = ctx.tool_result_store.store(
            session_id=ctx.session.session_id,
            turn=turn,
            tool_name=name,
            args_masked=_json_dumps(masked),
            full_text=text,
            summary=llm_text if summarized else None,
            is_error=is_error,
        )
        llm_text = _apply_turn_char_limit(
            llm_text,
            turn_chars,
            limit=ctx.cfg.tool.tool_results_turn_max_chars,
        )
```

Replace with:

```python
        llm_text = _apply_turn_char_limit(
            llm_text,
            turn_chars,
            limit=ctx.cfg.tool.tool_results_turn_max_chars,
        )
```

#### Step 3: Remove the now-unused `is_summarized` import

Remove `is_summarized` from the import block at line 30 (import from
`agent.tool_result_formatter`). Keep any sibling imports from the same statement (e.g.
`mask_args`, `TURN_LIMIT_HINT` if imported there) — remove only the `is_summarized` name.

### Method

- Pure deletion; no new logic, no new imports.
- `_json_dumps` (used to build `args_masked` for the removed `.store()` call) may become unused
  in this function if it has no other call site — check before removing the import (see Details).

### Details

- `_json_dumps` is also used by `_log_and_emit_tool_call()` (`emit_tool_call(name,
  _json_dumps(masked))`) in the same file — its import stays; do not remove it.
- `ctx.session.session_id`, `turn`, `name`, `masked`, `text`, `is_error` remain in use elsewhere
  in the loop body (masked for `_log_and_emit_tool_call`, text for `_emit_tool_result`, etc.) —
  removing the `.store()` call does not orphan any of these.
- `ctx.tool_result_store` field itself is NOT removed from `AgentContext`/`context.py` — it is
  still read by `/tool list` and `/tool show <id>` in `cmd_tooling.py` for rows written before
  this change (or by any other writer, if one exists — none found in this codebase besides the
  call being removed here).
