# Implementation and Test Procedure: Markdown Context Compression Engine MCP Server

## Goal
Implement a dedicated MCP server that compresses large Markdown while preserving meaning and retrieves only the necessary parts for injection.

## Scope
This implementation will:
- Create a new MCP server called `mdq-mcp` with Markdown-specific indexing and retrieval capabilities
- Enable heading-level local search and local retrieval for large Markdown documents
- Integrate with existing MCP operating model (HTTP, OpenRC, `/mcp`, `/v1/tools`, watchdog)
- Update Agent usage rules to prefer `search_docs` over `read_file` for Markdown documents

## Assumptions
- Current ingestion pipeline includes `web_crawler.py`, `chunk_splitter.py`, `rag_ingester.py`
- `chunk_splitter.py` already supports Markdown snippet splitting by heading boundary via `md_index_enable`
- Current Agent assumes HTTP MCP servers using `/v1/call_tool` and `/v1/tools` endpoints
- Current `file-mcp` provides search-oriented access to large documents but lacks Markdown heading structure support
- Target documents include local Markdown files, ADRs, specifications, and SKILL.md files
- Index storage will be in a dedicated local DB equivalent to `.mdq/index.sqlite`
- Agent usage rule will be: use `search_docs` before `read_file`

## Implementation
The implementation will create the following files:
1. `scripts/mdq_mcp_server.py` - Main MCP server implementation
2. `scripts/mdq_indexer.py` - Indexing logic for Markdown files
3. `scripts/mdq_parser.py` - Markdown parsing functionality
4. `scripts/mdq_search.py` - Search functionality using FTS5
5. `scripts/mdq_models.py` - Pydantic models for request/response handling
6. `config/mdq_mcp_server.json` - Configuration file for the server
7. `init.d/mdq-mcp` - OpenRC service script

## Validation plan
1. Unit tests for each component (parser, indexer, search)
2. Integration tests for the complete workflow
3. Verify that the server responds to `/v1/tools` endpoint
4. Test that the server can be started and stopped properly
5. Validate that all required tools are available
6. Check that the MCP server integrates with the agent system
7. Confirm that the database is created and used correctly
