# Implementation: Add fusion_mode key to get_diagnostics() in pipeline.py

## Goal
Add `"fusion_mode": "rrf" | "dedup_only"` to the `get_diagnostics()` return dict in
`scripts/rag/pipeline.py`, making the active fusion mode available for programmatic inspection.

## Scope
- File: `scripts/rag/pipeline.py`
- Method: `get_diagnostics()` — one new key added to the return dict
- Additive, backward-compatible change

## Assumptions
- `self._cfg.use_rrf` is a `bool` accessible in `get_diagnostics()`
  (confirmed: `pipeline._cfg` is `RagConfig` which has `use_rrf`)
- Value mapping: `use_rrf=True` → `"rrf"`, `use_rrf=False` → `"dedup_only"`
- No type change needed for the dict return type

## Implementation

### Target file
`scripts/rag/pipeline.py`

### Procedure
Locate the `return {` block in `get_diagnostics()` and add the new key.

### Method
Single edit — add one key to the return dict.
