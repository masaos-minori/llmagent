# Implementation: scripts/shared/plugin_registry.py (lifecycle docs)

## Goal

Add a "Lifecycle contract" section to `plugin_registry.py`'s module docstring covering Startup, Repeated loads, Test isolation, and Reload semantics. Update global variable comments and `load_plugins()` / `_reset_for_testing()` docstrings.

## Scope

- Target: `scripts/shared/plugin_registry.py`
- Append lifecycle contract section to module docstring
- Add 1-line lifecycle annotations to `_commands`, `_tools`, `_pipeline_post`, `_last_load_result` variables
- Update `load_plugins()` docstring to mention "one-shot per process" intent and repeated-call behavior
- Update `_reset_for_testing()` docstring to emphasize test-only use

## Assumptions

1. Changes are docstring/comment only — no runtime behavior changes.
2. `load_plugins()` is currently called once at startup from `agent/factory.py`.
3. `_last_load_result` accumulates per repeated call (last-call wins).

## Implementation

### Target file
`scripts/shared/plugin_registry.py`

### Procedure

3. Update `load_plugins()` docstring.
4. Update `_reset_for_testing()` docstring.

### Method

Append to module docstring:
```
Lifecycle contract:
  Startup:
    Call register_builtin_commands() before load_plugins() so conflict detection works.
    Call load_plugins(plugin_dir) once at process startup; calling it multiple times is
    allowed but each call replaces _last_load_result and ADDS to existing registries.
    Existing registrations are NOT cleared between calls.
  Repeated loads:
    Calling load_plugins() more than once is safe but accumulates registrations.
    Duplicate command/tool names registered by a later call silently overwrite earlier ones.
    _last_load_result reflects only the most recent call.
  Test isolation:
    Call _reset_for_testing() at the start of each test that loads plugins.
    This is the ONLY supported way to clear global registry state.
    Do not call _reset_for_testing() in non-test code.
  Reload:
    No hot-reload is implemented. Restarting the process is the intended upgrade path.
```

Global variable comments (add after each definition):
```python
_commands: dict[str, ...] = {}  # global: cleared only by _reset_for_testing()
_tools: dict[str, ...] = {}     # global: cleared only by _reset_for_testing()
_pipeline_post: list[...] = []  # global: cleared only by _reset_for_testing()
_last_load_result: PluginLoadResult | None = None  # last load call result; None until first load
```

### Details
- Read the current module docstring carefully and append the lifecycle block after the last paragraph.
- Locate global variable definitions and add inline comments on the same line or the line above.
- `load_plugins()` docstring: add "Intended to be called once per process at startup." and note repeated-call behavior.
- `_reset_for_testing()` docstring: add "For test use only. Do not call from production code."
