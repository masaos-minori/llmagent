# Implementation: Set firejail as default sandbox in shell_mcp_server.toml

Steps covered: Plan 20260626-091916 — Steps 1-2, 2-1

---

## Goal

Update `config/shell_mcp_server.toml` to set `sandbox_backend = "firejail"` as the production default, replacing the previous default of `"none"` or similar.

---

## Scope

- **In scope**: `config/shell_mcp_server.toml` — `sandbox_backend` field and related settings
- **Out of scope**: repl_health.py (steps 1-1, 2-2); tests; docs

---

## Assumptions

- `config/shell_mcp_server.toml` has a `sandbox_backend` field currently set to `"none"` or absent.
- `firejail` is the intended production default.
- Development environments may not have firejail; the config should note this.

---

## Implementation

### Target file
`config/shell_mcp_server.toml`

### Procedure
1. Read `config/shell_mcp_server.toml`.
2. Step 1-2: Change `sandbox_backend`:
   ```toml
   # Production default: firejail. Requires firejail to be installed.
   # For development without firejail, set to "none" (WARNING: unsandboxed execution).
   sandbox_backend = "firejail"
   ```
3. Step 2-1: If there are other sandbox-related settings (`sandbox_profile`, `sandbox_args`), add sensible defaults with comments.

### Method
Config-only change.
