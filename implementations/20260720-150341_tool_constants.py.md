## Goal

Remove the standalone `BROWSER_TOOLS` frozenset and fold `"browser_fetch"` directly into
`WEB_SEARCH_TOOLS`, since both tools now live under the one `web_search` server_key.

## Scope

**In scope**: lines 123 (`WEB_SEARCH_TOOLS` definition), 126 (`BROWSER_TOOLS` definition), and 187
(its union member) in `scripts/shared/tool_constants.py`.
**Out of scope**: every other frozenset in this file (`READ_TOOLS`, `WRITE_TOOLS`, `MDQ_TOOLS`,
etc.) — untouched. This is the one non-`mcp_servers`/non-`config` code reference to `BROWSER_TOOLS`
confirmed by the plan's own Design section research ("routing itself is live-discovery-driven, not
hardcoded").

## Assumptions

1. `WEB_SEARCH_TOOLS`/`BROWSER_TOOLS` are both plain `frozenset[str]` module constants with no
   surrounding class or function wrapping them — a direct, low-risk textual edit.
2. Confirmed via direct read: line 187's union member `| BROWSER_TOOLS,` sits inside a larger
   frozenset union expression (likely `ALL_TOOLS` or similar, spanning lines ~180-188) alongside
   `WEB_SEARCH_TOOLS` — removing the `BROWSER_TOOLS` name from both its own definition and this
   union list, while keeping `browser_fetch` reachable via the now-widened `WEB_SEARCH_TOOLS`,
   preserves the union's total tool count (no tool silently drops out of the all-tools set).
3. No other file in `scripts/`/`tests/`/`config/` imports `BROWSER_TOOLS` by name outside the
   `mcp_servers.browser` package itself (which is deleted) and `config`/docs already covered by
   companion docs — confirmed by the plan's Design-section grep research; re-verify with a fresh
   grep at implementation time as a safety net (same caution as the browser-directory-deletion doc).

## Implementation

### Target file

`scripts/shared/tool_constants.py`

### Procedure

1. Change line 123 from `WEB_SEARCH_TOOLS: frozenset[str] = frozenset({"search_web"})` to
   `WEB_SEARCH_TOOLS: frozenset[str] = frozenset({"search_web", "browser_fetch"})`.
2. Delete line 126 (`BROWSER_TOOLS: frozenset[str] = frozenset({"browser_fetch"})`) and any

3. Delete the `| BROWSER_TOOLS,` union member (line 187) from the union expression it belongs to,
   since `WEB_SEARCH_TOOLS` (already a member of that same union, per line 185) now covers
   `browser_fetch` too.

### Method

Pure frozenset-literal edit (add one string to an existing set literal) plus removal of a
now-redundant frozenset and its one union reference — no function signature changes, no new
imports.
