# Implementation: cmd_context.py — Immediate history[0] sync in _cmd_system

## Goal

When the user runs `/system <name>`, update `history[0]` immediately instead of
waiting for the next `Orchestrator._sync_system_prompt()` call. Eliminates the
two-state inconsistency window between command execution and next LLM turn.

## Scope

- `scripts/agent/commands/cmd_context.py`: modify `_cmd_system()` only.

## Assumptions

1. `history[0]` is a system message when a session is active (enforced by `_load_session`
   and Orchestrator startup). The empty-history edge case (no history yet) must also be
   handled.
2. `Orchestrator._sync_system_prompt()` is idempotent — inlining the same logic here
   causes no double-apply issues.
3. The comment "Orchestrator syncs history[0] at next turn start." documents the old
   delayed behavior and must be removed.
4. The test in `tests/test_agent_cmd_context.py` that patches `_cmd_system` may need
   updating to assert immediate sync.

## Implementation

### Target file

`scripts/agent/commands/cmd_context.py`

### Procedure

In `_cmd_system()`, replace the block that ends at line 255:

**Before:**
```python
ctx.conv.system_prompt_name = name
# Update the canonical field; Orchestrator syncs history[0] at next turn start.
ctx.conv.system_prompt_content = ctx.cfg.tool.system_prompts[name]
logger.info(f"System prompt switched to '{name}'")
print(f"System prompt: {name}")
```

**After:**
```python
ctx.conv.system_prompt_name = name
ctx.conv.system_prompt_content = ctx.cfg.tool.system_prompts[name]
# Immediately sync history[0] so the new prompt takes effect in this turn.
if ctx.conv.history and ctx.conv.history[0]["role"] == "system":
    ctx.conv.history[0]["content"] = ctx.conv.system_prompt_content
elif ctx.conv.system_prompt_content:
    ctx.conv.history.insert(0, {"role": "system", "content": ctx.conv.system_prompt_content})
logger.info(f"System prompt switched to '{name}'")
print(f"System prompt: {name}")
```

### Method

Targeted edit: replace four lines with eight lines in `_cmd_system()`.
Remove the delayed-sync comment; add immediate-sync block.

### Details

Edge cases:
- `history` is empty: the `elif` branch inserts a new system message.
- `history[0]` is not a system message (e.g. after `/clear`): same `elif` path inserts.
- `system_prompt_content` is empty string: neither branch executes (no-op).
