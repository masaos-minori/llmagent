# Implementation: n_embed_failed tracking in ingester.py

## Goal
Separate embedding failures from other failures in `RagIngester`:
- Add `n_embed_failed: int = 0` to `IngestUrlResult`
- Change `_embed_and_store()` to return `tuple[bool, bool]` — `(chunk_ok, embed_ok)` —
  so callers can distinguish embed failure from parse/DB failure
- Update `_ingest_chunk_files()` to count `n_embed_failed` without shared mutable state
- Propagate `n_embed_failed` through `ingest_url_group()` and aggregate in `ingest_all()`

## Scope
- File: `scripts/rag/ingestion/ingester.py`
- No changes to DB schema or other files

## Assumptions
- `IngestUrlResult` is `frozen=True`; add `n_embed_failed` field with default `0`
- `_embed_and_store()` returning `tuple[bool, bool]` is a safe internal change
  (no external callers confirmed; method is private)
- Thread-safe counting is achieved by collecting per-future results (no shared mutable state)
- Positional `IngestUrlResult` construction in test files: verify and update if found

## Implementation

### Target file
`scripts/rag/ingestion/ingester.py`

### Procedure

### Method

### Details

**Step 1 — `IngestUrlResult`:**

**Step 2 — `_embed_and_store()` return type:**
Change return type from `bool` to `tuple[bool, bool]` — `(chunk_ok, embed_ok)`:
```python
def _embed_and_store(self, doc_id: int, path: Path) -> tuple[bool, bool]:
    """...; returns (chunk_ok, embed_ok).
    chunk_ok=False means total failure (parse/DB error or embed failure).
    embed_ok=False specifically means embedding failed (content was valid).
    """
    data = self._read_chunk_json(path)
    if data is None:
        return False, False
    # ... (content/idx extraction unchanged) ...
    embedding = self._get_embedding(content)
    if embedding is None:
        logger.warning("embedding failed for %s: %r", path.name, content[:60])
        return False, False   # chunk_ok=False, embed_ok=False → embed failure
    with SQLiteHelper().open(write_mode=True) as db:
        self._insert_chunk(db, doc_id, idx, content, normalized_content,

        db.commit()
    return True, True         # chunk_ok=True, embed_ok=True
```

**Step 3 — `_ingest_chunk_files()` return type and logic:**
Change from `tuple[int, int]` to `tuple[int, int, int]` — `(inserted, failed, embed_failed)`:
```python
def _ingest_chunk_files(self, doc_id: int, chunk_files: list[Path]) -> tuple[int, int, int]:
    """...; returns (n_inserted, n_failed, n_embed_failed)."""
    inserted = 0
    failed = 0
    embed_failed = 0
    with ThreadPoolExecutor(max_workers=self._embed_workers) as executor:
        futures = {
            executor.submit(self._embed_and_store, doc_id, path): path
            for path in chunk_files
        }
        for future in as_completed(futures):
            path = futures[future]
            try:
                chunk_ok, embed_ok = future.result()
                if chunk_ok:
                    inserted += 1
                else:
                    failed += 1
                    if not embed_ok:
                        embed_failed += 1
            except (httpx.HTTPStatusError, httpx.RequestError,
                    OSError, ValueError, TypeError) as e:
                logger.error("Failed to ingest %s: %s", path, e)
                failed += 1
    return inserted, failed, embed_failed
```

**Step 4 — `ingest_url_group()` propagation:**
Update the call site and `IngestUrlResult` construction:
```python
inserted, failed, embed_failed = self._ingest_chunk_files(doc_id, chunk_files)
# ...
return IngestUrlResult(
    url=url, n_success=inserted, n_failed=failed,
    skipped=False, n_embed_failed=embed_failed,

```

**Step 5 — `ingest_all()` aggregate log:**
