# Implementation Procedure: scripts/eventbus/db.py + scripts/eventbus/app.py

## Goal

グローバル SQLite 接続の設計意図を docstring と起動ログで明示する。

## Scope

**In:**
- `scripts/eventbus/db.py` — `check_same_thread=False` の理由を docstring に記述; 接続失敗時のエラーログ追加
- `scripts/eventbus/app.py` — lifespan 起動時に接続戦略ログを追加

**Out:** 接続モデルの変更

## Assumptions

1. FastAPI は async イベントループ上で動作 (単一スレッド); `check_same_thread=False` は安全
2. WAL モードが有効化されている

## Implementation

### db.py — docstring + error logging

```python
def get_connection(db_path: str) -> sqlite3.Connection:
    """Return a shared SQLite connection for the Event Bus.

    Uses check_same_thread=False because FastAPI runs on a single async event
    loop thread; WAL mode serializes concurrent writers at the SQLite level.
    A single shared connection avoids per-request churn and is safe here.
    """
    try:
        conn = sqlite3.connect(db_path, check_same_thread=False)
        conn.execute("PRAGMA journal_mode=WAL")
        return conn
    except sqlite3.Error as exc:
        logger.error("eventbus: failed to open SQLite connection: path=%s err=%s", db_path, exc)
        raise
```

### app.py — startup log

```python
# lifespan 内
logger.info(
    "eventbus: using shared SQLite connection (WAL mode, check_same_thread=False) path=%s",
    cfg.db_path,
)
```
