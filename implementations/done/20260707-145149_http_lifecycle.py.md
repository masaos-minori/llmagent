# Implementation: M-4 — Add get_process_snapshot() to HttpServerLifecycleManager (http_lifecycle.py)

## Goal

Add a `get_process_snapshot(server_key)` method to `HttpServerLifecycleManager` that
returns a `dict` with `pid`, `pgid`, and `running` for known subprocess servers.

## Scope

**Target**: `scripts/agent/http_lifecycle.py`

**Step covered**: Plan M-4 step 2.

**Out of scope**: `factory.py` (router exposure covered separately), test files.

## Assumptions

1. `_http_procs` exists (base dict); `_http_pgids` exists after H-8 (guarded).
2. `proc.poll() is None` means the process is running.
3. Returns `None` (not a dict) for unknown `server_key` values.

## Implementation

### Target file

`scripts/agent/http_lifecycle.py`

### Procedure

#### Step 1: Add `get_process_snapshot()` method

Add after `verify_running()`:

```python
def get_process_snapshot(self, server_key: str) -> dict | None:
    """Return {pid, pgid, running} for a managed subprocess server, or None if unknown."""
    proc = self._http_procs.get(server_key)
    if proc is None:
        return None
    running = proc.poll() is None
    pgid = getattr(self, "_http_pgids", {}).get(server_key)
    return {
        "pid": proc.pid,
        "pgid": pgid,
        "running": running,
    }
```

### Method

- New public method; no changes to existing methods.
- `getattr(self, "_http_pgids", {})` guards against pre-H-8 environments.
- Return type `dict | None` is intentionally untyped for simplicity; a `TypedDict`
  can be introduced later if mypy requires it.

### Details

- `proc.pid` is always set immediately after `Popen()` completes; safe to access even
  if the process has since exited.
- `running=False` and `proc.pid` both remain accessible for exited processes (until
  H-7 pops the entry); this allows `McpStatusService` to report `last_exit_code` via
  `proc.poll()` on the same process object.
- For `last_exit_code`: the caller (`mcp_status.py`) can call `proc.poll()` via the
  snapshot dict, but since `proc` is not exposed, add `last_exit_code` to the dict:
  ```python
  return {
      "pid": proc.pid,
      "pgid": pgid,
      "running": running,
      "last_exit_code": proc.poll(),
  }
  ```
  `proc.poll()` returns `None` if running, or the exit code if exited.
