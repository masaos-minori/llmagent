# tool_runner.py — route writes through gateway; remove batch approval

**Plan:** `plans/20260625-094121_plan.md` (req #63)
**Target:** `scripts/agent/tool_runner.py`

## What to change

### 1. `execute_one_tool_call()` — use gateway when available (line 96)

**Before:**
```python
    result = await ctx.services.tools.execute(name, args)
```

**After:**
```python
    if ctx.services.gateway is not None:
        result = await ctx.services.gateway.execute(ctx, name, args)
    else:
        result = await ctx.services.tools.execute(name, args)
```

### 2. `execute_all_tool_calls()` — remove batch approval (line 425)

**Before (line 425):**
```python
    approved_calls, denied_ids = await run_approval_checks(ctx, tool_calls)
```

**After:** Remove this line entirely. Approval is now handled per-call inside the gateway.

Also remove the `approved_calls` filtering that follows. Currently line 366 passes `approved_calls`
(the approved subset) to `_execute_standard()`. After removal, pass all tool calls:

```python
# Before (line 365-366):
results = await _execute_standard(ctx, approved_calls, turn)

# After:
results = await _execute_standard(ctx, tool_calls, turn)
```

### 3. Remove `from agent.tool_approval import run_approval_checks` (line 25)

After removing all usages, remove the import. If other callers exist (unlikely per plan's grep),
keep the import.

## Impact on DAG path

`_execute_with_dag()` at line 282 uses `approved_calls` from the batch check. After removal,
it will receive the full `tool_calls` list. Denied tools become error results from the gateway
(same final effect — denied tools produce `ToolCallResult(is_error=True, error_type="denied")`).
