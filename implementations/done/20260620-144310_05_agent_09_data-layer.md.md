# Implementation: docs/05_agent_09_data-layer.md (update — MDQ/RAG boundary wording)

## Goal

Fix an inconsistency in `docs/05_agent_09_data-layer.md` where an absolute claim
("the agent layer never imports sqlite3 against rag.sqlite or mdq.sqlite directly")
contradicts the actual code in `DbMaintenanceService`, which does access `rag.sqlite`
directly for admin-only `/db` maintenance operations.

## Scope

**In:**
- Update line 91 of `docs/05_agent_09_data-layer.md` to acknowledge the admin-only
  maintenance exception

**Out:**
- Code changes of any kind
- Changes to other documentation files
- Changes to `docs/04_mcp-mdq.md` (already correct)

## Assumptions

- The three-path table at lines 83-90 of the same file already lists `/db` commands
  as an admin path — the fix only corrects the prose that follows the table
- `DbMaintenanceService` using `SQLiteHelper("rag")` is the intentional admin exception,
  not a bug to fix
- The target sentence ends with `See [04_mcp-mdq.md](04_mcp-mdq.md) for the boundary
  between RAG and MDQ systems.`

## Implementation

### Target file

`docs/05_agent_09_data-layer.md`

### Procedure

1. Locate line 91 (exact text):
   ```
   MCP tools are the preferred and supported path. The agent layer never imports `sqlite3` against `rag.sqlite` or `mdq.sqlite` directly. See [04_mcp-mdq.md](04_mcp-mdq.md) for the boundary between RAG and MDQ systems.
   ```

2. Replace with:
   ```
   MCP tools are the preferred and supported path. Direct `sqlite3` imports against `rag.sqlite` or `mdq.sqlite` are not allowed in normal application code. The `/db` admin commands use `DbMaintenanceService` as an explicit maintenance exception (see [04_mcp-mdq.md](04_mcp-mdq.md) §Agent Access Patterns). See [04_mcp-mdq.md](04_mcp-mdq.md) for the boundary between RAG and MDQ systems.
   ```

### Method

Single `Edit` operation targeting the exact sentence to replace.

### Details

- The replacement wording:
  - Changes "never imports" (absolute claim) to "not allowed in normal application code"
    (accurately scoped prohibition)
  - Adds one sentence explicitly acknowledging `DbMaintenanceService` as the admin-only
    maintenance exception
  - Preserves the trailing link to `04_mcp-mdq.md`
- No heading level, list structure, or surrounding content is changed
- The edit does not affect the three-path table above or the MDQ section below
