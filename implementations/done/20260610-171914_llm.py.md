# Implementation: rag/llm.py ŌĆö Config Injection and Fail-Fast LLM Calls

## Goal

Remove the module-level `_get_cfg()` cache with `except Exception` + `{}` fallback; inject `llm_url`/`embed_url` via constructor; replace all broad `except Exception` with specific exception types; replace fail-soft MQE/rerank fallbacks with explicit error types.

---

## Scope

**Target file:** `scripts/rag/llm.py`

**In:**
- Delete `_get_cfg()` function and `_cfg` module-level variable
- `RagLLM.__init__`: accept `llm_url: str` (already does); remove internal `_get_cfg()` calls
- `get_embedding`: remove `_get_cfg()` call; accept `embed_url: str` as parameter
- `_parse_mqe_response`: on JSON error, raise `MqeParseError` instead of returning `[original_query]`
- `RagLLM.expand_queries`: catch `httpx.HTTPStatusError`, `httpx.RequestError`, `orjson.JSONDecodeError`, `MqeParseError`; raise `RagExpansionError` on any failure (remove silent fallback)
- `RagLLM.cross_encoder_rerank`: remove broad `except Exception`; raise `RagRerankError` on LLM/parse failure
- `RagLLM.summarize_tool_result`: remove `except Exception`; let exceptions propagate
- Add `MqeParseError`, `RagExpansionError`, `RagRerankError` exception classes

**Out:**
- `_extract_chat_content`, `_build_rerank_prompt`, `_apply_rerank_scores` ŌĆö logic unchanged
- `RagLLM.refine_context` ŌĆö already raises; no changes needed
- `summarize_tool_result` module-level function ŌĆö update to accept `embed_url` param

---

## Assumptions

1. All callers of `get_embedding` currently pass `client: httpx.AsyncClient`; they will also need to pass `embed_url: str`. Caller update is coordinated in Step 5 (`pipeline.py`).
2. `expand_queries` currently returns `[query]` on failure ŌĆö changing to raise `RagExpansionError` requires `MqeStage` (Step 5) to catch and record the failure.
3. `cross_encoder_rerank` currently returns `candidates[:top_k]` on failure ŌĆö changing to raise `RagRerankError` requires `RerankStage` (Step 5) to handle.
4. `summarize_tool_result` is called from `agent/tool_runner.py` ŌĆö that caller currently does not handle exceptions from this function; adding exception propagation may require minimal update there (out of scope of this step; document as risk).
5. Module-level `_cfg: dict | None = None` and `_get_cfg()` are completely removed. No module-state caching.

## Implementation

### Target file
`scripts/rag/llm.py`

### Procedure

1. Add exception class definitions at the top (after imports):
   - `class MqeParseError(ValueError): ...`
   - `class RagExpansionError(RuntimeError): ...`
   - `class RagRerankError(RuntimeError): ...`
2. Remove `_cfg` module variable and `_get_cfg()` function entirely.
3. Update `get_embedding` signature to `async def get_embedding(text: str, client: httpx.AsyncClient, embed_url: str) -> list[float]`.
4. Update `_mqe_prompt` to accept `llm_url` + config dict instead of calling `_get_cfg()` ŌĆö or accept the rendered template directly.
5. Update `_parse_mqe_response`: raise `MqeParseError` when JSON is malformed or result is not a list.
6. Update `RagLLM.expand_queries`: replace `except Exception` with specific types; raise `RagExpansionError` wrapping the cause.
7. Update `RagLLM.cross_encoder_rerank`: remove all fallback logic; raise `RagRerankError` on LLM or parse failure.
8. Update `RagLLM.summarize_tool_result`: remove `except Exception`; let exceptions propagate.
9. Update module-level `summarize_tool_result` wrapper: accept `llm_url: str` parameter; remove `_get_cfg()` call.

### Method

```python
class MqeParseError(ValueError):
    """Raised when the MQE LLM response cannot be parsed as a query list."""

class RagExpansionError(RuntimeError):
    """Raised when MQE query expansion fails."""

class RagRerankError(RuntimeError):
    """Raised when cross-encoder reranking fails."""

# get_embedding ŌĆö explicit url param
async def get_embedding(text: str, client: httpx.AsyncClient, embed_url: str) -> list[float]:
    resp = await client.post(embed_url, json={"content": f"query: {text}"})
    resp.raise_for_status()
    embedding = orjson.loads(resp.content).get("embedding")
    if not isinstance(embedding, list) or not embedding:
        raise ValueError("missing or empty 'embedding' field in embed response")
    return embedding

# expand_queries ŌĆö fail-fast
async def expand_queries(self, query: str, context: str = "") -> list[str]:
    try:
        raw = await self._call_llm(
            [{"role": "user", "content": _mqe_prompt(query, context, self._cfg)}],
            _MQE_TEMPERATURE,
            _MQE_MAX_TOKENS,
        )
        return _parse_mqe_response(raw, query)  # raises MqeParseError on bad JSON
    except (httpx.HTTPStatusError, httpx.RequestError, orjson.JSONDecodeError, MqeParseError) as e:
        raise RagExpansionError(f"MQE expansion failed: {e}") from e
```

### Details

- `_mqe_prompt` needs to accept the config dict from the caller instead of calling `_get_cfg()`. Simplest approach: accept `cfg: dict[str, Any]` parameter. `RagLLM` stores `self._cfg` (already present as `self._llm_url`? ŌĆö check constructor). If `RagLLM` does not currently store config, add `cfg: dict[str, Any]` to its constructor.
- `RagLLM.__init__` currently only takes `client` and `llm_url` ŌĆö add `cfg: dict[str, Any] = dataclasses.field(default_factory=dict)` or a plain `cfg: dict[str, Any] | None = None`.
- `_build_rerank_prompt` calls `_get_cfg().get("rerank_prompt_template", "")` ŌĆö this must become `cfg.get(...)` where cfg is passed from the caller.
- All `_get_cfg()` references in `llm.py` must be replaced with `cfg` parameter access.
