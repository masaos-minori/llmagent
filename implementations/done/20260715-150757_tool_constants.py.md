# Implementation Procedure: scripts/shared/tool_constants.py

Source plan: `plans/20260715-133548_plan.md`

## Goal

Add granular write/admin frozensets for the CI/CD, RAG, and MDQ tool families so
that side-effect classification (consumed by `tool_executor_helpers.py` and
`tool_runner.py`) can distinguish write/admin tools from read-only tools within
those families, instead of relying on the existing mixed `CICD_TOOLS` /
`RAG_TOOLS` / `MDQ_TOOLS` unions.

## Scope

**In scope:**
- Add `CICD_WRITE_TOOLS`, `CICD_READ_TOOLS` (CI/CD family split).
- Add `RAG_WRITE_TOOLS`, `RAG_READ_TOOLS` (RAG family split).
- Add `MDQ_WRITE_TOOLS`, `MDQ_SERIAL_TOOLS`, `MDQ_READ_TOOLS` (MDQ family split,
  three-way because `fts_rebuild` is a serial-barrier candidate, not a plain
  write).

**Out of scope:**
- No change to `CICD_TOOLS`, `RAG_TOOLS`, `MDQ_TOOLS` (must remain the same
  union values — still consumed by `get_all_mcp_tool_names()`).
- No change to `GIT_TOOLS` / `GITHUB_TOOLS` or any other existing frozenset.
- No change to `get_all_mcp_tool_names()` itself.

## Assumptions

- `trigger_workflow`, `rag_delete_document`, `index_paths`, `refresh_index`,
  `fts_rebuild` are the only write/admin tools in their respective families
  today (confirmed by reading the file in full; no other write-shaped tool
  names are present in `CICD_TOOLS` / `RAG_TOOLS` / `MDQ_TOOLS`).
- Follow the existing `GIT_READ_TOOLS` / `GIT_WRITE_TOOLS` / `GIT_TOOLS`
  precedent already in this file: define explicit member sets first, derive
  the "read" set by set-difference from the existing union constant so the
  union constant itself never needs to change.
