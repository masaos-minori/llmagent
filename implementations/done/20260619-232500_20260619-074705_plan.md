### Activation Gate

The memory layer has a 3-layer activation gate that controls when memory operations are executed:

**Layer 1: Config bypass**
- `use_memory_layer` config flag (default: `True`)
- When `False`, all memory service methods become no-ops
- Bypasses all downstream calls (injection, ingestion, retrieval)
- Implemented in: `services.py::MemoryServices`

**Layer 2: Embedding client enabled**
- `EmbeddingClient.enabled` property gates HTTP/embedding calls
- When disabled: FTS5-only fallback is used (keyword search only)
- When enabled: Full hybrid search (FTS5 + KNN + RRF merge)
- Implemented in: `embedding_client.py::EmbeddingClient`, `retriever.py::HybridRetriever`

**Layer 3: Service facade invocation**
- `MemoryServices` is the single entry point for all memory operations
- Session lifecycle hooks: `on_session_start()`, `on_user_prompt()`, `on_session_stop()`
- All layers must pass for full memory functionality

### Data Model

#### MemoryEntry (stored in JSONL + SQLite)

| Field | Type | Description |
|-------|------|-------------|
| `id` | `str` | UUID v4, primary key |
| `session_id` | `str` | Parent session identifier |
| `turn_index` | `int` | Turn number within session (0-based) |
| `role` | `str` | "user" or "assistant" |
| `content` | `str` | Message content text |
| `timestamp` | `str` | ISO 8601 UTC timestamp |
| `embedding_id` | `str \| None` | UUID for associated embedding vector |
| `metadata` | `dict \| None` | Optional key-value metadata |

**DB mapping:** Stored in `memory_entries` table. JSONL is append-only source of truth; SQLite is queryable index with FTS5 full-text search and sqlite-vec vector index.

#### MemorySnippet (returned to LLM context)

| Field | Type | Description |
|-------|------|-------------|
| `entry_id` | `str` | Reference to `MemoryEntry.id` |
| `session_id` | `str` | Source session identifier |
| `role` | `str` | "user" or "assistant" |
| `content` | `str` | Message content (truncated if too long) |
| `score` | `float` | Relevance score from search (RRF merge rank) |
| `timestamp` | `str` | ISO 8601 UTC timestamp |

### JSONL Format

Each line in the JSONL store is a single JSON object:

```json
{"id": "uuid-here", "session_id": "sess-123", "turn_index": 5, "role": "user", "content": "Hello", "timestamp": "2026-06-19T23:00:00Z", "embedding_id": null, "metadata": {}}
```

**Properties:**
- Append-only: entries are never modified or deleted
- One entry per line
- UTF-8 encoded
- No trailing comma, valid JSON per line
- File path controlled by `memory_jsonl_path` config

**Content to add:**

### Search Strategies

#### FTS5 (Full-Text Search)
- **Engine:** SQLite FTS5 with BM25 ranking
- **Index:** Tokenized content index on `memory_entries.content`
- **Fallback:** Used when `EmbeddingClient.enabled=False`
- **Strengths:** Exact keyword matching, fast on small datasets
- **Weaknesses:** No semantic understanding, language-specific tokenization

#### KNN (Vector Search)
- **Engine:** sqlite-vec extension with cosine similarity
- **Index:** Dense embedding vectors stored in `memory_embeddings` table
- **Requirement:** `EmbeddingClient.enabled=True` and valid embedding API config
- **Strengths:** Semantic similarity matching, language-agnostic
- **Weaknesses:** Requires embedding API calls, slower on large datasets

#### Hybrid (RRF Merge)
- **Engine:** Combines FTS5 + KNN results using Reciprocal Rank Fusion (RRF)
- **Algorithm:** `rrf_score = 1 / (k + rank)` where k=60
- **Result:** Ranked merge of both search results
- **Strengths:** Best of both worlds, robust across query types
- **Weaknesses:** Higher latency (two searches + merge)
