## Risks

| Risk | Mitigation |
|------|-----------|
| Output becomes too verbose for many small changes | Count per section keeps output compact; tags are short |
| Existing scripts/pipelines that parse /reload output break | Tags are in brackets; parsers can be updated to strip them |
| Warning banner may alarm users unnecessarily | Only shown when restart is actually needed; wording is factual not alarming |
