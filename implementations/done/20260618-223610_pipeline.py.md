# Implementation: Improve Ingestion and Query Diagnostics — Pipeline.py

## Goal

Add `"failure"` status tracking to `StageResult`, expose refiner/HTTP fallback reasons, and wire `debug_fn` through the REPL debug path.

## Scope

- `scripts/rag/pipeline.py` — failure status on exception, fallback reason propagation

## Assumptions

1. Adding `"failure"` status does not break existing consumers (none check specific status strings).
2. `_get_stage_status()` already distinguishes `"success"` / `"fallback"` — `"failure"` is the third state.
3. `StageResult` TypedDict already declares `status: str  # "success" | "fallback" | "failure"` (line 19 of stage.py).

## Current State

### Stage status tracking (`pipeline.py:122-142`)

`_get_stage_status()` returns status for each known stage:
- `MqeStage`: `"fallback"` if `use_mqe=False`, reason `"use_mqe=False"`
- `SearchStage`: `"fallback"` if no search results, reason `"no search results"`
- `FusionStage`: `"fallback"` if `use_rrf=False`, reason `"use_rrf=False"`
- `RerankStage`: `"fallback"` if `use_rerank=False`, reason `"use_rerank=False"`

**Gap:** No `"failure"` status. If a stage raises an exception, the `except` block in `run()` (lines 202-275) catches it, but the exception is NOT caught per-stage — the entire `run()` method exits to `finally`.

## Proposed Changes

### 1. Per-stage exception handling in `run()`

Wrap each stage in try/except to catch exceptions and record `"failure"` status:

```python
for stage in pre_augment_stages:
    t0 = time.perf_counter()
    try:
        await stage.run(ctx, db=db)
        status, reason = self._get_stage_status(stage, ctx)
    except Exception as e:
        elapsed = time.perf_counter() - t0
        self.last_timings[stage.__class__.__name__] = elapsed
        status = "failure"
        reason = f"{type(e).__name__}: {e}"
    # append StageResult...
```

### 2. Refiner fallback reason propagation

Change `refine_context()` to return a typed result with reason:

```python
from dataclasses import dataclass

@dataclass
class RefineResult:
    text: str | None
    reason: str | None  # "empty_output", "exception: ...", or None

async def refine_context(...) -> RefineResult:
    try:
        refined = await llm.refine_context(...)
        if refined:
            return RefineResult(text=refined, reason=None)
        logger.warning("Refiner returned empty output; falling back to chunks")
        return RefineResult(text=None, reason="refiner_returned_empty")
    except (httpx.HTTPStatusError, httpx.RequestError, ValueError) as e:
        logger.warning("Refiner failed, falling back to original chunks: %s", e)
        return RefineResult(text=None, reason=f"refiner_exception: {e}")
```

Update caller in `pipeline.py:379-382`:
```python
if self._cfg.use_refiner:
    refined = await self._augment_refiner(reranked, query)
    if refined.text is not None:
        return refined.text
    # fallback_reason now available via refined.reason
```
