and `max_search_results` from the config field list, and replace the
hybrid-search "非稼働" (non-operational)/stub-note prose with language
reflecting that hybrid search has been removed entirely, not merely left
unimplemented.

## Implementation

### Target file

`docs/04_mcp_04_04_mdq.md`

### Procedure

1. Open `docs/04_mcp_04_04_mdq.md`.
2. In the 設定フィールド line, remove `max_search_results`,
   `use_embedding`, `embedding_dims`（デフォルト 384）, `vector_table`,

   names (`max_results_limit`, `max_chars_per_chunk`, etc.) and the existing
   `audit_log_path`/`concurrency_limit` parenthetical untouched.
3. Locate the フェーズ1/フェーズ2 table (current lines 70-72):
   ```markdown
   | FTS5のみ（フェーズ1） | FTS5 による全文検索; 本番のベースライン | `use_embedding = false`（デフォルト） |
   | ハイブリッド（フェーズ2） | **非稼働** — FTS5 + セマンティックベクトル検索を RRF で統合 | `use_embedding = true` |
   ```

   there is no config toggle anymore):
   ```markdown
   | FTS5（BM25） | FTS5 による全文検索; 唯一のサポートされる検索モード | `search_docs`の`mode`パラメータ（`bm25`のみ） |
   ```

   ```markdown
   **ハイブリッド検索（フェーズ2）:**

   `use_embedding = true` の場合、MDQ はハイブリッド検索を試みるが、`_search_vector()` は空リストを返すため、結果は FTS5 のみとなる。
   ```
   Replace with:
   ```markdown
   **ハイブリッド検索は削除済み:**

   ハイブリッド/セマンティック検索の設定項目（`use_embedding`, `embedding_dims`, `vector_table`, `embedding_model`）およびコード（`_search_vector()`, `_merge_hybrid()`）は[実装日]に完全に削除された。実装されたことのない機能であり（`_search_vector()`は常に空リストを返す永続的なプレースホルダーだった）、`search_docs`の`mode`パラメータは`bm25`のみをサポートする。
   ```

