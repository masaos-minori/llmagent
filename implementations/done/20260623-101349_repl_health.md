## Goal

Add two new security checks to `audit_security_defaults()` in `scripts/agent/repl_health.py`:
1. **cicd workflow_allowlist check**: if `CicdConfig.workflow_allowlist` is empty and `production_mode=True`, raise `RuntimeError`; otherwise log a warning.
2. **GitHub write settings check**: if `GitHubConfig.allow_force_push=True` or `require_pr_review=False`, log a warning (never raise ‚Äî warning-only per plan design decision).

## Scope

**In-Scope:**
- `scripts/agent/repl_health.py` ‚Äî insert two new check blocks before the "Security posture summary" block (before line 538)

**Out-of-Scope:**
- No changes to `CicdConfig`, `GitHubConfig`, or any other production file
- Test changes are in a separate implementation doc (`test_repl_health.py`)

## Assumptions

1. `CicdConfig` is not yet imported at module level in `repl_health.py`; a lazy `from mcp.cicd.models import CicdConfig` inside a `try` block follows the established pattern (matching the `GitHubConfig` lazy import at line 501-502).
2. `GitHubConfig` is already lazily imported inside the "Check github allowed_repos" block (lines 501-502). The new GitHub write settings check must use a separate `try/except` block to avoid coupling with the existing `allowed_repos` check.
3. The `except RuntimeError: raise` pattern (from the plan Design section) must be used in the cicd block so that `RuntimeError` raised inside the `try` is not silently swallowed by `except Exception: pass`.
4. Both new blocks must be inserted between the last existing check (line 536 `tool.allowed_tools`) and the "Security posture summary" block (line 538), or between other existing blocks in the same area.

### Details

**Insertion point** ‚Äî insert before line 538 (`# Security posture summary`):

```python
    # Check cicd workflow_allowlist (fail-open ‚Äî empty = allow all workflows)
    try:
        from mcp.cicd.models import CicdConfig  # noqa: PLC0415 ‚Äî lazy import; cicd-mcp optional
        cicd_cfg = CicdConfig.load()
        if not cicd_cfg.workflow_allowlist:
            msg = "cicd.workflow_allowlist is empty (fail-open: all workflows allowed)"
            if production_mode:
                raise RuntimeError(
                    f"Production mode requires explicit workflow_allowlist. {msg}"
                )
            fail_open_empty.append("cicd.workflow_allowlist")
            logger.warning("Security: %s", msg)
            warnings.append(f"Security: {msg}")
    except RuntimeError:
        raise
    except Exception:
        pass

    # Surface GitHub write settings (warning only ‚Äî not a production-mode hard error)
    try:
        from mcp.github.models_config import (
            GitHubConfig as _GitHubConfig,  # noqa: PLC0415 ‚Äî lazy import; github-mcp optional
        )
        gh_write_cfg = _GitHubConfig.load()
        if gh_write_cfg.allow_force_push:
            msg = "github.allow_force_push=true (force push and rebase merge permitted)"
            logger.warning("Security: %s", msg)
            warnings.append(f"Security: {msg}")
        if not gh_write_cfg.require_pr_review:
            msg = "github.require_pr_review=false (PR merge without review permitted)"
            logger.warning("Security: %s", msg)
            warnings.append(f"Security: {msg}")
    except Exception:
        pass

```

Key decisions:
- `_GitHubConfig` alias avoids re-binding the name from the earlier `GitHubConfig` lazy import (lines 501-502), preventing shadowing issues if both blocks are active in the same call.
- `cicd.workflow_allowlist` is added to `fail_open_empty` (not just `warnings`) so it appears in the posture summary.
- `except RuntimeError: raise` in the cicd block prevents `RuntimeError` from being caught by the outer `except Exception`.
