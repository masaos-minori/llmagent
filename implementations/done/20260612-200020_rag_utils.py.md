# Goal

Formally document `cosine_sim`'s zero-vector behavior in the function contract,
and change `sanitize_document` to return a `SanitizeResult` DTO instead of a raw
string so the sanitization outcome is auditable.

# Scope

- `scripts/rag/utils.py`
- Any caller of `sanitize_document` that currently expects `str`

# Assumptions

1. `cosine_sim` already has a docstring saying it returns 0.0 on zero magnitude.
   The change is to add a `# Contract` comment or assert-like note to the function.
   No behavior change needed.
2. `sanitize_document(text: str) -> str` becomes `sanitize_document(text: str) -> SanitizeResult`.
3. `SanitizeResult` is defined in `rag/models.py` (add it there):
   ```python
   @dataclass(frozen=True)
   class SanitizeResult:
       text: str
       was_sanitized: bool
       patterns_detected: list[str]
   ```
4. Callers of `sanitize_document` that use the return value as a string must be
   updated to use `.text`.
5. If no callers currently use `sanitize_document`, the change is internal only.

# Implementation

## Target file

`scripts/rag/utils.py`, `scripts/rag/models.py` (add `SanitizeResult`)

## Procedure

1. Add `SanitizeResult` dataclass to `rag/models.py`.
2. Change `sanitize_document(text: str) -> str` to `-> SanitizeResult`.
3. Build a `patterns_detected` list while iterating patterns.
4. Return `SanitizeResult(text=cleaned, was_sanitized=len(patterns_detected) > 0, patterns_detected=patterns_detected)`.
5. Update all callers to use `.text`.
6. Add explicit docstring note to `cosine_sim`:
   ```
   # Contract: returns 0.0 when either vector has zero magnitude (not an error).
   ```
7. Run ruff + mypy.

## Method

Return type change + new DTO in models.py.

# Validation plan

- `uv run ruff check scripts/rag/utils.py`
- `uv run mypy scripts/rag/utils.py`
- `uv run pytest tests/ -k "utils or rag" --ignore=tests/test_create_schema.py -v`
