## Goal
Change `create_task()` in `task_ops.py` and `StateStore.create_task()` in `state_store.py` so `workflow_id` is a required `str` parameter (not `str | None = None`).

## Scope
**In**: `scripts/agent/workflow/task_ops.py` and `scripts/agent/workflow/state_store.py` — change `workflow_id: str | None = None` to `workflow_id: str` (required); update `INSERT INTO tasks` to always write `workflow_id`.
**Out**: Other `task_ops.py` and `state_store.py` functions; schema migration.

## Assumptions
- `create_task(db, session_id, turn_number, workflow_version, workflow_id: str | None = None)` is the current signature.
- `orchestrator.py:280` already passes `workflow_id=str(uuid.uuid4())` — the new signature is backward-compatible for that call site.
- All other callers (test files) currently pass `workflow_id=None` or omit it — those must be updated.
- `StateStore.create_task()` delegates to `task_ops.create_task()` or has parallel logic.

## Implementation

**Target files**: `scripts/agent/workflow/task_ops.py`, `scripts/agent/workflow/state_store.py`

**Procedure**:

1. **`task_ops.py`**:
   ```python
   # Before:
   def create_task(
       db: ...,
       session_id: str,
       turn_number: int,
       workflow_version: str,
       workflow_id: str | None = None,
   ) -> TaskRecord:

   # After:
   def create_task(
       db: ...,
       session_id: str,
       turn_number: int,
       workflow_version: str,
       workflow_id: str,
   ) -> TaskRecord:
   ```
   - Remove `workflow_id = workflow_id or str(uuid.uuid4())` auto-generation if present
   - The caller (orchestrator.py:280) is already responsible for generating the UUID

2. **`state_store.py`**:
   - Same change to `StateStore.create_task()` signature: `workflow_id: str | None = None` → `workflow_id: str`
   - Update `INSERT INTO tasks (task_id, session_id, workflow_id, ...)` to always include `workflow_id` in the column list (remove any `if workflow_id: ...` conditional)

3. **Update test callers**:
   - `grep -rn "create_task(" tests/` — find all callers
   - For each test that omits `workflow_id` or passes `None`: add `workflow_id="wf-test-" + str(uuid.uuid4())` or a fixed test UUID

4. **Run mypy**:
   - `uv run mypy scripts/agent/workflow/task_ops.py scripts/agent/workflow/state_store.py` — must be clean

**Method**: Parameter requirement enforcement. No SQL schema change (column already exists).

**Details**:
- If `task_ops.py` contains `workflow_id = workflow_id or str(uuid.uuid4())` auto-generation: remove it. The UUID must be generated by the caller, not by the DB layer.
- Verify: `grep -n "uuid.uuid4\|workflow_id.*uuid" scripts/agent/workflow/task_ops.py` → should be 0 after change.
- In orchestrator.py: `workflow_id = str(uuid.uuid4())` at line 280 (new task) stays; line 305 (resume) is fixed in orchestrator.py doc.
