## Goal
Prevent token explosion by adding a configurable limit on memory snippets injected into the system prompt during session startup.

## Scope
**In**: `scripts/agent/startup.py` — add snippet limit to `_setup_prompt()`; config parameter addition
**Out**: Changes to `on_session_start()` logic, memory retrieval algorithm, or other memory-related functionality

## Assumptions
1. A default limit of 10 snippets is sufficient for most use cases
2. The most recent snippets (first N items) are the most relevant
3. A config parameter is preferred over a hardcoded constant for flexibility

## Implementation

### Target file
`scripts/agent/startup.py`, `scripts/agent/config_dataclasses.py`

### Procedure
1. Add `max_retrieved_memories` config parameter to AgentConfig with default value 10
2. Update `_setup_prompt()` to slice snippets based on this limit
3. Add warning log when snippets are truncated
4. Update documentation/config reference

### Method
Add a new config field and apply slicing in the startup prompt setup.

### Details
Current code in `startup.py` line 331-347:
```python
async def _setup_prompt(self) -> None:
    """Inject semantic memories into the initial system prompt."""
    ctx = self._ctx
    initial_prompt = ctx.cfg.tool.system_prompts.get(
        ctx.conv.system_prompt_name,
        ctx.cfg.tool.system_prompt_tool,
    )
    if ctx.services_required.memory is not None:
        memory_snippets = ctx.services_required.memory.on_session_start(
            ctx.session.session_id,
        )
        if memory_snippets:
            memory_block = "\n\n[Relevant memories]\n" + "\n".join(
                f"- {snippet.text}" for snippet in memory_snippets
            )
            initial_prompt = initial_prompt + memory_block
    ctx.conv.system_prompt_content = initial_prompt
    ctx.conv.history = [{"role": "system", "content": initial_prompt}]
```

Problem: No limit is enforced on the number of memory snippets injected at startup. For long sessions with many stored memories, this can cause token explosion in the system prompt.

Fix in `config_dataclasses.py` — add to `AgentConfig`:
```python
# Max memory snippets injected at session startup
agent_memory_max_startup_snippets: int = 10
```

Fix in `startup.py`:
```python
async def _setup_prompt(self) -> None:
    """Inject semantic memories into the initial system prompt."""
    ctx = self._ctx
    initial_prompt = ctx.cfg.tool.system_prompts.get(
        ctx.conv.system_prompt_name,
        ctx.cfg.tool.system_prompt_tool,
    )
    if ctx.services_required.memory is not None:
        memory_snippets = ctx.services_required.memory.on_session_start(
            ctx.session.session_id,
        )
        if memory_snippets:
            max_snippets = ctx.cfg.agent_memory_max_startup_snippets
            if len(memory_snippets) > max_snippets:
                logger.warning(
                    "Startup: truncating %d memory snippets to %d for %r",
                    len(memory_snippets),
                    max_snippets,
                    ctx.session.session_id,
                )
                memory_snippets = memory_snippets[:max_snippets]
            memory_block = "\n\n[Relevant memories]\n" + "\n".join(
                f"- {snippet.text}" for snippet in memory_snippets
            )
            initial_prompt = initial_prompt + memory_block
    ctx.conv.system_prompt_content = initial_prompt
    ctx.conv.history = [{"role": "system", "content": initial_prompt}]
```

Key points:
- Default limit of 10 snippets balances context richness vs token budget
- Warning log when truncation occurs helps operators tune the limit
- Slicing preserves order (most recent first) since `on_session_start()` returns ranked results
- Config field name follows existing convention (`agent_*` prefix for agent-level settings)
