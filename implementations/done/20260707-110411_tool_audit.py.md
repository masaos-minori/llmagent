## Goal
Replace `or ""` fallbacks for `workflow_id` and `task_id` in `tool_audit.py` with assertions that fail explicitly when workflow context is missing during workflow execution.

## Scope
**In**: `scripts/agent/tool_audit.py` — replace `ctx.workflow.workflow_id or ""` and `ctx.workflow.current_task_id or ""` at lines 54, 66, 72, 170, 179 with assertions; add `stage_id` and `attempt_id` to audit payloads where not yet present.
**Out**: `tool_audit.py` functions called from non-workflow paths (if any); audit log storage backend.

## Assumptions
- `ctx.workflow.workflow_id` and `ctx.workflow.current_task_id` are always set before `tool_audit.py` functions are called during `_handle_workflow_engine()`.
- After Plan 03 (req03), there is no non-workflow execution path; all tool audit events occur in workflow context.
- Exception: `_build_turn_end_metadata()` in orchestrator.py may call audit after workflow deactivation — those `or ""` fallbacks are kept.
- `stage_id` and `attempt_id` are available as closure variables from `WorkflowEngine._run_stage()`.

## Implementation

**Target file**: `scripts/agent/tool_audit.py`

**Procedure**:
1. **Audit call graph first**:
   - `grep -n "def.*tool_audit\|from agent.tool_audit" scripts/agent/` — list all functions
   - `grep -rn "tool_audit\." scripts/agent/orchestrator.py scripts/agent/workflow/` — confirm call sites
   - Categorize: (a) called only from workflow execution → assert; (b) called from startup/non-workflow → keep `or ""`

2. **Replace `or ""` with assertions** in functions called only from workflow context:
   ```python
   # Before:
   workflow_id = ctx.workflow.workflow_id or ""
   task_id = ctx.workflow.current_task_id or ""

   # After:
   assert ctx.workflow.workflow_id, "workflow_id required: tool audit called outside workflow context"
   assert ctx.workflow.current_task_id, "task_id required: tool audit called outside workflow context"
   workflow_id = ctx.workflow.workflow_id
   task_id = ctx.workflow.current_task_id
   ```

3. **Add `stage_id` and `attempt_id` to payloads**:
   - If functions accept a `stage_id` parameter already, use it
   - If not, add `stage_id: str = ""` param; pass from call site
   - Add to JSON payload: `"stage_id": stage_id`

4. **Do NOT assert** in:
   - Functions called from `_build_turn_end_metadata()` (post-deactivation)
   - Functions called from startup or test fixtures without workflow context

**Method**: Targeted assertion replacement with call-graph verification first.

**Details**:
- Run the assertion-replacement only after confirming call graph in step 1.
- If uncertain about a function's call context, keep `or ""` and add a TODO comment.
