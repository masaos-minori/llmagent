# Implementation: history.py — expose compressed message boundary

Source plan: `plans/20260626-180401_plan.md` — Phase 2 step 1

---

## Goal

Extend `CompressResult` and `HistoryManager.compress()` / `force_compress()` to expose the `cutoff_message_id` boundary — the DB `message_id` of the last message included in the compression window — so that the call site can call `session.replace_compressed_messages()` with the correct boundary.

---

## Scope

**In-Scope**
- Add `cutoff_message_id: int | None = None` field to `CompressResult` dataclass
- Modify `_build_compressed_result()` to compute and return the cutoff boundary
- The cutoff is NOT a DB message_id directly (history.py has no DB access); it is the **index** into the original history list of the last compressed message, to be resolved by the caller

**Out-of-Scope**
- DB access from history.py (violates layer contract: agent → db is ok, but history.py should not directly access DB)
- Changes to compression policy or threshold logic

---

## Assumptions

1. `history.py` does not and must not access the DB directly — the `agent` layer calls `session.py`.
2. The caller (orchestrator) knows the current `message_id` values because it tracks saved messages; alternatively, `SelectionResult` contains the `to_compress` list from which the caller can determine the DB boundary after the fact.
3. A simpler approach: `CompressResult` gains `n_compressed: int` (already exists as `compressed_count`) and `n_system: int` (already exists as `protected_count`) — the caller uses these counts to derive the DB boundary from a pre-fetched message_id list.
4. Design decision: add `compressed_indices: list[int] | None = None` to `CompressResult` — list of indices (0-based) in the pre-compression history list that were compressed, allowing the caller to map them to DB `message_id` values.

---

## Implementation

### Target file
`scripts/agent/history.py`

### Procedure
1. Add `compressed_indices: list[int] | None = None` to `CompressResult` dataclass.
2. In `_build_compressed_result()`, record the indices of `split.to_compress` in the original history list.
3. No changes needed in `compress()` or `force_compress()` public signatures.

### Method

**Update `CompressResult`:**

```python
@dataclass
class CompressResult:
    compressed_count: int
    protected_count: int
    summary_added: bool
    is_fallback: bool = False
    compressed_indices: list[int] | None = None  # indices into pre-compression history
```

**Update `_build_compressed_result()`:**

The `split: SelectionResult` from `_selection_policy.select_turns_to_compress(history)` contains `split.to_compress` — the messages to compress. We need their indices in the original `history` list.

```python
def _build_compressed_result(
    self, split: SelectionResult, summary_text: str, original_history: list[LLMMessage]
) -> tuple[list[LLMMessage], CompressResult]:
    # existing logic...
    compressed_id_set = {id(m) for m in split.to_compress}
    compressed_indices = [
        i for i, m in enumerate(original_history) if id(m) in compressed_id_set
    ]
    result = CompressResult(
        compressed_count=n,
        protected_count=protected,
        summary_added=True,
        compressed_indices=compressed_indices,
    )
    return system_msgs + [summary_msg] + remaining, result
```

Update `compress()` to pass `original_history` to `_build_compressed_result()`:
- Change: `return self._build_compressed_result(split, summary_text)` → `return self._build_compressed_result(split, summary_text, history)`
