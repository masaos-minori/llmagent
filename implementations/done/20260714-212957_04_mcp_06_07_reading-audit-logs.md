# Implementation Procedure: Correct Three-Layer Audit Log Documentation in Reading-Audit-Logs Guide

## Goal

Correct the three-layer audit log documentation in `04_mcp_06_07_reading-audit-logs.md` based on actual implementation behavior discovered during Task 12 inspection. Eliminate contradictions between what the document says and what the code actually does.

## Scope

- `docs/04_mcp_06_07_reading-audit-logs.md`

**Out-of-Scope:**
- Other audit-related documents (handled separately)

## Assumptions

1. Implementation inspection findings are correct:
   - **Shared MCP audit writer** (`_audit_log()` → `/opt/llm/logs/audit.log`) used by: web-search-mcp, github-mcp, shell-mcp, git-mcp, cicd-mcp, mdq-mcp
   - **Per-server audit logs only**: file-delete-mcp (`/opt/llm/logs/delete_audit.log`)
   - **Both shared + per-server**: github-mcp, shell-mcp
   - **No audit logging**: file-read-mcp, file-write-mcp, rag-pipeline-mcp
   - **MDQ does NOT have a per-server audit log** — uses shared `_audit_log()` only
   - **git-mcp's `audit_log_path` is reserved/not implemented**

## Implementation

### Procedure

#### Step 1: Remove file-read-mcp and file-write-mcp from shared MCP audit log list

Current L27 incorrectly lists file-read-mcp and file-write-mcp as using the shared audit log. These servers do NOT call `_audit_log()`.

Replace L27:
```markdown
**共有audit log** (`/opt/llm/logs/audit.log`): web-search-mcp、file-read-mcp、file-write-mcp、rag-pipeline-mcp、cicd-mcpで使用される。
```

With:
```markdown
**共有audit log** (`/opt/llm/logs/audit.log`): web-search-mcp、github-mcp、shell-mcp、git-mcp、cicd-mcp、mdq-mcpで使用される。
```

#### Step 2: Correct MDQ audit log description

Current L48-49 incorrectly states MDQ has no dedicated audit log and shares app log. MDQ uses shared `_audit_log()` which writes to `/opt/llm/logs/audit.log`.

Replace L48-49:
```bash
# MDQ operations (JSON-lines形式、専用のaudit logファイルはない。app logと共通)
grep '"event":"mcp_tool_exec"' /opt/llm/logs/mdq-mcp.log
```

With:
```bash
# MDQ operations (共有audit log via _audit_log())
grep '"event":"mcp_tool_exec"' /opt/llm/logs/audit.log
```

#### Step 3: Clarify cicd-mcp/git-mcp/app-log vs audit-log distinction

Current L52 conflates app logs with audit logs. Need to clarify: cicd-mcp and git-mcp have NO dedicated app log files AND use shared audit log only (no per-server audit logs).

Replace L52:
```markdown
> **注記:** cicd-mcp、git-mcp、mdq-mcp には専用のaudit logファイルがない。cicd-mcp/git-mcp は `logging.getLogger(__name__)` のみを使用し、mdq-mcp は `_audit_log()` 経由でアプリログ(`mdq-mcp.log`)にJSON-linesで記録する。
```

With:
```markdown
> **注記:** cicd-mcp と git-mcp は専用アプリログファイルも専用auditログファイルも持たず、共有audit log(`/opt/llm/logs/audit.log`)のみを使用する。mdq-mcp は専用アプリログファイルを持つが、専用auditログファイルは持たず、共有audit log(`/opt/llm/logs/audit.log`)に `_audit_log()` 経由でJSON-linesで記録する。
```
