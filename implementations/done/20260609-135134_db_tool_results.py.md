# Implementation: db/tool_results.py — Add logger.exception to silent fallbacks

## Goal

Add `logger.exception(...)` calls to the `except Exception` blocks in `ToolResultStore`
so DB errors are visible in logs instead of being silently swallowed.

## Scope

- `scripts/db/tool_results.py`: add `logger.exception` to `store()`, `get()`, `list_recent()`.

## Assumptions

1. The existing `None` / empty-list return behavior is preserved — only logging is added.
2. `logger` is already defined at module level.
3. This is the minimal viable change; full result-type changes are Phase 2.

## Implementation

### Target file

`scripts/db/tool_results.py`

### Procedure

In each `except Exception as e:` block, add `logger.exception(...)` before the return:

**store():**
```python
except Exception as e:
    logger.exception("ToolResultStore.store failed (tool=%r)", tool_name)
    return None
```

**get():**
```python
except Exception as e:
    logger.exception("ToolResultStore.get failed (id=%r)", result_id)
    return None
```

**list_recent():**
```python
except Exception as e:
    logger.exception("ToolResultStore.list_recent failed (session_id=%r)", session_id)
    return []
```
