# Implementation: agent/tool_runner.py — Fix concurrent execution for same-scope write groups

## Goal

Fix the bug where same-resource-scope write tool calls are executed concurrently via `asyncio.gather`. Tool_runner.py must respect the `serialize_flags` in `ScheduledBatch` to run same-scope tools sequentially while keeping different scopes and reads concurrent.

## Scope

**In**: Update the `concurrent_groups` iteration loop to use `ScheduledBatch.serialize_flags`. Add audit event fields for scheduling decision.

**Out**: Changes to `tool_scheduler.py`, test files.

## Assumptions

1. `tool_scheduler.py` is updated first (dependency: `ScheduledBatch` dataclass with `serialize_flags`).
2. Current tool_runner loop at lines ~297-310 iterates `metadata.concurrent_groups` as `list[list[list[dict]]]`.
3. After the scheduler change, `metadata.concurrent_groups` is `list[ScheduledBatch]`.
4. The inner `asyncio.gather` over tools-in-a-group must be replaced with sequential execution when `serialize_flags[group_idx]=True`.
5. Audit event currently logs `resource_scope`, `is_write`, `requires_serial`, `scheduling_decision` via `_SerializationEvent` — these should be included in the per-tool audit emit.

## Implementation

### Target file
`scripts/agent/tool_runner.py`

### Procedure
1. Update the `concurrent_groups` iteration to use `ScheduledBatch`.
2. Replace inner `asyncio.gather` with sequential loop when `serialize_within` flag is `True`.
3. Emit audit event including new `_SerializationEvent` fields.

### Method

**Current loop (approx lines 297-310):**
```python
for concurrent_batch in metadata.concurrent_groups:
    batch_results = await asyncio.gather(
        asyncio.gather(*(execute_one_tool_call(ctx, tc, turn) for tc in group))
        for group in concurrent_batch
    )
```

**Updated loop:**
```python
for batch in metadata.concurrent_groups:  # batch: ScheduledBatch
    async def run_group(group: list[dict], serialize: bool) -> list[Any]:
        if serialize:
            # Sequential: run tools one at a time
            results = []
            for tc in group:
                r = await execute_one_tool_call(ctx, tc, turn)
                results.append(r)
            return results
        else:
            # Concurrent: gather all tools in group
            return list(await asyncio.gather(
                *(execute_one_tool_call(ctx, tc, turn) for tc in group)
            ))

    batch_results = await asyncio.gather(
        *(run_group(group, serialize)
          for group, serialize in zip(batch.groups, batch.serialize_flags))
    )
```

**Audit event update:**
After the loop, include `resource_scope`, `is_write`, `requires_serial`, `scheduling_decision` from `_SerializationEvent` in the audit event dict (lines ~314-324):
```python
for se in serialization_events:
    round_event = {
        ...existing fields...,
        "resource_scope": se.resource_scope,
        "is_write": se.is_write,
        "requires_serial": se.requires_serial,
        "scheduling_decision": se.scheduling_decision,
    }
```

### Details

- The `run_group` async function must be defined inside the loop body or as a module-level helper to avoid closure issues with `group` and `serialize` variables.
- Prefer `module-level` helper: `async def _run_group_calls(group, serialize, ctx, turn)` to avoid lambda capture issues.
- The outer `asyncio.gather` over groups within a batch is unchanged — different groups within the same batch still run concurrently (e.g., scope1 and scope2 groups run in parallel).
- Type annotations: `batch.groups: list[list[dict]]`, `batch.serialize_flags: list[bool]`.
