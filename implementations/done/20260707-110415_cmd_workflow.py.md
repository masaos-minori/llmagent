## Goal
Remove ID-less fallback from `/approve` and `/reject` (always require explicit `<approval_id>`); update `_emit_approval_audit()` to use `approval.workflow_id` directly instead of task join.

## Scope
**In**: `scripts/agent/commands/cmd_workflow.py` — `_cmd_approve()` (lines 80–133), `_cmd_reject()` (lines 135–186), `_emit_approval_audit()` (lines 54–78).
**Out**: `find_latest_pending_approval()` in `approval_ops.py` (kept for startup recovery); `find_approval_by_id()` (kept).

## Assumptions
- When `explicit_id is None`, both handlers currently check `count == 1` and use `find_latest_pending_approval()` — this conditional is removed; always require `explicit_id`.
- `find_latest_pending_approval` import is removed from `cmd_workflow.py`; kept in `approval_ops.py`.
- `_emit_approval_audit(task_id, ...)` currently joins through `store.get_task_by_id(task_id)` — changed to `_emit_approval_audit(approval: ApprovalRecord, ...)` using `approval.workflow_id`.
- After Plan 11 (req11), `ApprovalRecord.workflow_id` is always populated.

## Implementation

**Target file**: `scripts/agent/commands/cmd_workflow.py`

**Procedure**:

1. **Change `_cmd_approve()`** (lines 80–133):
   - Remove `count > 1 and explicit_id is None` guard
   - Replace with unconditional `explicit_id is None` guard:
     ```python
     if explicit_id is None:
         self._out.write_validation_error(
             "Approval ID required. Use: /approve <approval_id> [reason]\n"
             "Use '/workflow status' to list pending approval IDs."
         )
         return
     ```
   - Remove the `count_pending_approvals()` call if only used for the count-1 branch
   - Remove the `find_latest_pending_approval()` call and its import
   - Update docstring: `[approval_id]` → `<approval_id>` (required)

2. **Change `_cmd_reject()`** (lines 135–186):
   - Same pattern as `_cmd_approve()`

3. **Remove import** of `find_latest_pending_approval`:
   - `grep -n "find_latest_pending_approval" scripts/agent/commands/cmd_workflow.py` — find and remove from import line

4. **Update `_emit_approval_audit()`** (lines 54–78):
   - Change signature: `_emit_approval_audit(self, task_id: str, ...)` → `_emit_approval_audit(self, approval: ApprovalRecord, ...)`
   - Replace task join:
     ```python
     # Before:
     task = self._store.get_task_by_id(task_id)
     "workflow_id": task.workflow_id if task else None,

     # After:
     "workflow_id": approval.workflow_id,
     ```
   - Remove `self._store.get_task_by_id(task_id)` call
   - Update callers `_cmd_approve()` and `_cmd_reject()`: pass `approval` object instead of `task_id`

5. **Update startup recovery notice** (if present):
   - `grep -n "recover_pending\|pending.*approval" scripts/agent/startup.py` — confirm approval ID is displayed
   - If not: add `approval.approval_id` to the notice output

**Method**: Conditional removal + parameter change.

**Details**:
- `count_pending_approvals()` may be used elsewhere — check before removing: `grep -rn "count_pending_approvals" scripts/`.
- The `_emit_approval_audit` signature change breaks `_cmd_approve` and `_cmd_reject` as call sites — update all three in the same edit.
