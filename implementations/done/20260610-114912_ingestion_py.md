# Implementation: ingestion.py — design comments and score contract clarification

## Goal

Make the design intent explicit in code comments:
1. `_persist_entry()` — manual write path; dedup/link intentionally omitted.
2. `_has_near_duplicate()` / `_link_duplicates()` — clarify `distance = -hit.score` conversion.
3. `on_session_stop` / `_persist_entry` docstrings — explain the intentional separation.

## Scope

- **Target file**: `scripts/agent/memory/ingestion.py`
- **No interface changes**: all method signatures remain identical.

## Assumptions

1. `VectorRetriever.knn_search()` returns `MemoryHit` with `score = -distance` (higher is better).
   Callers that need raw distance must negate: `distance = -hit.score`.
2. The separation between auto-extracted (with dedup) and manual writes (without dedup) is intentional.
3. No callers depend on the absence of comments; adding comments is safe.
