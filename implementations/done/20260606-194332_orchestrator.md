# Implementation: Worker Role-Based `allowed_tools` Injection

## Goal

Allow different `allowed_tools` sets to be passed to `Orchestrator` at construction time, enabling future per-role or per-worker tool restrictions without modifying `AgentConfig` globally.

## Scope

**In:**
- Add an optional `allowed_tools: list[str] | None` parameter to `Orchestrator.__init__()`
- When set, override `ctx.cfg.tool.allowed_tools` for the lifetime of that `Orchestrator` instance
- Add a helper on `ToolPolicy` (or `ToolExecutor`) that checks the per-instance override first

**Out:**
- Creating separate Worker classes (not in scope; Orchestrator is the single execution unit)
- Changing `AgentConfig` schema (no config change needed; runtime injection only)
- Persistent per-role config in `agent.toml`

## Assumptions

- `scripts/agent/orchestrator.py`: `Orchestrator.__init__` receives `ctx: AgentContext`
- `scripts/agent/tool_policy.py`: `ToolPolicy.is_allowed(tool_name)` checks `ctx.cfg.tool.allowed_tools`
- `scripts/agent/config.py`: `ToolConfig.allowed_tools: list[str]` — empty means all tools allowed
- No separate Worker classes exist; role differentiation is done by passing different `Orchestrator` instances

## Implementation

### 1. `scripts/agent/orchestrator.py`

Add `allowed_tools` parameter to `Orchestrator.__init__()`:

```python
def __init__(
    self,
    ctx: AgentContext,
    *,
    allowed_tools: list[str] | None = None,   # <-- add this
    on_turn_start: Callable[[], None] | None = None,
    on_turn_end: Callable[[], None] | None = None,
    on_error: Callable[[Exception], None] | None = None,
    on_first_turn: Callable[[str], Any] | None = None,
    tracer: Any = None,
) -> None:
    self._ctx = ctx
    self._allowed_tools: list[str] | None = allowed_tools
    # ... existing init ...
```

In the `handle_turn()` method, before tool dispatch, temporarily shadow `ctx.cfg.tool.allowed_tools`:

```python
# Snapshot original and apply override for this turn
original_allowed = ctx.cfg.tool.allowed_tools
if self._allowed_tools is not None:
    ctx.cfg.tool.allowed_tools = self._allowed_tools
try:
    result = await self._run_turn(user_message)
finally:
    ctx.cfg.tool.allowed_tools = original_allowed  # always restore
```

### 2. `scripts/agent/tool_policy.py`

Verify that `ToolPolicy.is_allowed()` reads from `ctx.cfg.tool.allowed_tools` (not a cached copy). If it reads from a snapshot taken at construction, change it to always read `ctx.cfg.tool.allowed_tools` at call time.

### 3. No changes needed to:
- `AgentConfig` or `ToolConfig`
- `agent.toml` / `mcp_servers.toml`
- `factory.py`

The factory continues to build one `Orchestrator` with no `allowed_tools` override (full access). Callers that need restricted access pass the list explicitly.

## Validation plan

### Unit tests (`tests/test_orchestrator.py`)

```python
async def test_allowed_tools_override_restricts_dispatch():
    """Orchestrator with allowed_tools=['read_text_file'] blocks 'write_file'."""
    ctx = make_test_ctx()
    orch = Orchestrator(ctx, allowed_tools=["read_text_file"])
    # Simulate a turn that tries to call 'write_file'
    # Expect ToolPolicy.is_allowed("write_file") to return False
    assert not orch._is_tool_allowed("write_file")
    assert orch._is_tool_allowed("read_text_file")

async def test_allowed_tools_none_permits_all():
    """Orchestrator with no override allows all tools."""
    ctx = make_test_ctx()
    orch = Orchestrator(ctx, allowed_tools=None)
    assert orch._is_tool_allowed("write_file")
    assert orch._is_tool_allowed("delete_file")

async def test_original_config_restored_after_turn():
    """ctx.cfg.tool.allowed_tools is restored to original value after a turn."""
    ctx = make_test_ctx()
    ctx.cfg.tool.allowed_tools = []  # all allowed
    orch = Orchestrator(ctx, allowed_tools=["search_web"])
    await orch.handle_turn("test message")
    assert ctx.cfg.tool.allowed_tools == []  # restored
```

### Integration test

- Create an `Orchestrator` with `allowed_tools=["search_web"]`
- Issue a turn that the LLM would normally respond to with `write_file`
- Confirm the tool call is blocked and an appropriate error is returned to the LLM
