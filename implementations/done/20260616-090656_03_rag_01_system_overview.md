# Implementation: docs/03_rag_01_system_overview.md

## Goal

Create a top-level system overview of the RAG pipeline that gives both human engineers
and AI systems a coherent picture before reading detailed per-component files.

## Scope

- Content from: `03_spec_rag.md` sections 1-6, 8 (partial), 11
- Content from: `03_rag-ingestion-pipeline.md` (dataflow diagram)
- Output: `docs/03_rag_01_system_overview.md`
- Not covered: per-module API, configuration details, known issues (those go in other files)

## Assumptions

- Audience: human engineer and AI that has NOT read any other RAG doc
- Canonical source for spec conflicts: `ref-*` files, then `03_spec_rag.md`
- The pipeline has 3 ingestion scripts (crawler, chunk_splitter, ingester) and 5 query stages

## Implementation

### Target file

`docs/03_rag_01_system_overview.md`

### Procedure

1. State the system purpose (one sentence)
2. Define scope (in-scope modules; out-of-scope: MDQ, agent REPL)
3. Describe ingestion pipeline at one level above API: 3 scripts / 4 phases
4. Describe query pipeline at one level above API: 5 stages
5. List prerequisites (embed-llm port, sqlite-vec, config file)
6. State constraints table (language detection, chunk sizes, crawl limits)
7. Link to detailed chapters

### Method

- Use heading structure: Purpose / Scope / System Architecture / Ingestion Overview /
  Query Overview / Prerequisites / Constraints / Related Chapters
- Include ASCII text diagram of data flow (no external tools needed)
- Do NOT include API signatures or config parameter tables (those belong in 04 and 05)
- Do NOT include known bugs (those belong in 90)

### Details

Ingestion pipeline description:
- 3 scripts: crawler.py → chunk_splitter.py → ingester.py
- 4 phases: crawl → chunk → embed → store
- File lifecycle: rag-src/*.txt → rag-src/chunk/*.txt → rag-src/registered/*.txt

Query pipeline description (5 stages from 05_ref-rag.md):
- MQE → Search → Fusion(RRF) → Rerank(CrossEncoder) → Augment
- Entry point: RagPipeline.augment(query) → context block string
- Called from: mcp/rag_pipeline/service.py

Note on stage count discrepancy (03_spec_rag.md says "4段階" vs 03_rag-ingestion-run.md says "3ステップ"):
- Write: "3 scripts / 4 processing phases" and add a note pointing to 03_rag_90_inconsistencies.md
