# Implementation: Refactor `AgentSession.save_diagnostic()` to delegate to `DiagnosticStore`

## Goal

Replace `AgentSession.save_diagnostic()`'s dependency on `SessionMessageRepository` with a delegation to `DiagnosticStore`. Remove the `_diagnostic` parameter from `SessionMessageRepository.save()`.

## Scope

- `scripts/agent/session.py` — update `save_diagnostic()`
- `scripts/agent/session_message_repo.py` — remove `_diagnostic` param

**Out of scope:** Caller changes in orchestrator.py (separate doc).

## Assumptions

1. `DiagnosticStore` module exists before this change.
2. Existing `messages` rows with `role='diagnostic'` are not migrated.

## Implementation

### Target file

`scripts/agent/session.py`

### Procedure

1. Import `DiagnosticStore`.
2. Add `self._diagnostic_store = DiagnosticStore(self.session_id)` in `__init__()`.
3. Replace `save_diagnostic()` body to delegate to `self._diagnostic_store.save()`.

### Method

Surgical edit: change `save_diagnostic()` method and add import.

### Details

**Current code (session.py:51-57):**
```python
def save_diagnostic(self, content: str) -> None:
    """Persist a diagnostic-only message; not included in normal history retrieval."""
    self._message_repo.save(
        role=_DIAGNOSTIC_ROLE,
        content=content,
        _diagnostic=True,
    )
```

**Desired code:**
```python
def save_diagnostic(self, content: str) -> None:
    """Persist a diagnostic-only message; not included in normal history retrieval."""
    self._diagnostic_store.save(self.session_id, kind="llm_transport_error", content=content)
```

**Also in session_message_repo.py:**
- Remove `_diagnostic` parameter from `save()` `__init__` signature.
- Remove the `_diagnostic` role-validation bypass in `save()` (lines 48-56 of session_message_repo.py).
- Remove `_DIAGNOSTIC_ROLE` constant if no longer used anywhere else.
