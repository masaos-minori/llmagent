# Implementation: M-6 — Expose get_process_info() and list_processes() in _ServerLifecycleRouter (factory.py)

## Goal

Add `get_process_info()` and `list_processes()` to `_ServerLifecycleRouter` delegating
to `HttpServerLifecycleManager`, so `McpStatusService` can access process snapshots via
duck-typing.

## Scope

**Target**: `scripts/agent/factory.py`

**Step covered**: Plan M-6 step 4.

**Dependencies**: M-6 steps 2–3 (`get_process_info()` and `list_processes()` in
`http_lifecycle.py`); M-6 step 1 (`ProcessInfoSnapshot` DTO).

**Out of scope**: `lifecycle_protocol.py`, `mcp_status.py`.

## Assumptions

1. `ProcessInfoSnapshot` import is handled in `http_lifecycle.py`; `factory.py` only
   needs to type-hint the return if needed, or use `Any`.
2. `McpStatusService` calls these methods via `getattr` duck-typing; no protocol
   extension is strictly required.

## Implementation

### Target file

`scripts/agent/factory.py`

### Procedure

#### Step 1: Import ProcessInfoSnapshot (optional)

If type annotations are desired:
```python
from agent.services.models import ProcessInfoSnapshot
```

If this causes circular import issues, use `Any` or omit type hints on these methods.

#### Step 2: Add `get_process_info()` to `_ServerLifecycleRouter`

```python
def get_process_info(self, server_key: str):
    """Return ProcessInfoSnapshot for a managed subprocess server, or None."""
    return self._http_mgr.get_process_info(server_key)
```

#### Step 3: Add `list_processes()` to `_ServerLifecycleRouter`

```python
def list_processes(self):
    """Return list of ProcessInfoSnapshot for all managed subprocess servers."""
    return self._http_mgr.list_processes()
```

### Method

- Pure delegation; no logic in the router.
- Omit explicit return type annotations if `ProcessInfoSnapshot` cannot be imported
  without circular deps; mypy will infer from the `_http_mgr` methods.

### Details

- M-4 step 3 added `get_process_snapshot()` (returning `dict | None`); M-6 adds a
  richer `get_process_info()` (returning `ProcessInfoSnapshot | None`). The two methods
  coexist; M-6 is the preferred API for new callers.
- `McpStatusService._probe_single_server()` uses `getattr` to call
  `get_process_info()` if present, falling back to `get_process_snapshot()` if not.
