# Implementation: Profile-aware watchdog default in config_builders.py (req 26 — Step 1)

## Goal

Make `mcp_watchdog_interval` default to 30.0 when `security_profile=PRODUCTION` and 0.0 for LOCAL, by deriving the default from the profile before constructing `MCPConfig`.

## Scope

- `scripts/agent/config_builders.py`

## Assumptions

1. `SecurityProfile` is already imported from `shared.mcp_config`.
2. Explicit config values always override the profile default.
3. The derived `security_profile_val` is reused in `MCPConfig(security_profile=...)` to avoid double parse.

## Implementation

### Target file

`scripts/agent/config_builders.py`

### Procedure

In `build_agent_config()`, before the `MCPConfig(...)` call:
1. Parse `security_profile` from cfg once into `security_profile_val`.
2. Derive `watchdog_interval_default` from it.
3. Pass `security_profile_val` directly to `MCPConfig`.

### Method

Edit tool.

### Details

```python
security_profile_val = SecurityProfile(cfg.get("security_profile", "local"))
watchdog_interval_default = 30.0 if security_profile_val == SecurityProfile.PRODUCTION else 0.0

mcp=MCPConfig(
    mcp_servers=_build_mcp_servers(cfg),
    mcp_watchdog_interval=float(cfg.get("mcp_watchdog_interval", watchdog_interval_default)),
    mcp_watchdog_max_restarts=int(cfg.get("mcp_watchdog_max_restarts", 3)),
    github_url=cfg.get("github_server_url", "http://127.0.0.1:8006"),
    security_profile=security_profile_val,
),
```
