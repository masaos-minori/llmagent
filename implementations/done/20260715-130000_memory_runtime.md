# Implementation Procedure: Refactor Memory runtime construction to always build SQLite-backed hybrid MemoryServices

## Goal

Remove the optional-memory-layer pattern and make MemoryServices always available at startup. Eliminate JSONL persistence from the runtime path, remove normal embedding-disabled mode, and remove FTS-only fallback from the retriever.

## Scope

### Files to modify
- `agent/factory.py` â€” remove `use_memory_layer` gate, JSONL store construction, `enabled=` argument
- `agent/memory/services.py` â€” remove `use_memory_layer` param, update `get_activation_mode()`
- `agent/memory/ingestion.py` â€” remove `jsonl` parameter, remove JSONL write in `_persist_entry_with_embedding()`
- `agent/memory/embedding_client.py` â€” remove `enabled=` constructor arg, remove `EmbeddingErrorKind.DISABLED` path
- `agent/memory/retriever.py` â€” remove FTS-only fallback from `search()`, remove `last_retrieval_mode` and `fts_fallback_count`
- `scripts/agent/context.py` â€” change `AppServices.memory` type from `MemoryServices | None` to `MemoryServices`

### Additional files to modify (found during analysis)
- `scripts/agent/orchestrator.py` â€” remove `is not None` guard on `ctx.services_required.memory`
- `scripts/agent/services/context_view.py` â€” remove `is None` check in `_format_memory_status()`

### Out of scope
- `agent/commands/cmd_memory.py`
- `agent/commands/memory_data_ops.py`
- `agent/commands/memory_rebuild_ops.py`
- `agent/commands/memory_status.py`
- `agent/config_dataclasses.py`
- `agent/config_builders.py`
- `agent/services/config_reload.py`
- Documentation files
- Test files (except Memory runtime tests if required)
- `agent/memory/jsonl_store.py` â€” delete (migration-only via import_ops.py)

## Assumptions

1. Config key removal (`use_memory_layer`, `memory_embed_enabled`) is handled separately in the config issue.
2. CLI command behavior updates are handled separately in the Memory CLI issue.
3. `import_from_jsonl()` in `import_ops.py` remains usable for migration â€” it will still reference `JsonlMemoryStore` which becomes migration-only.
4. `memory_local_only` validation behavior must be preserved (already exists in `EmbeddingClientConfig.local_only`).
5. Circuit breaker behavior for degraded/unavailable states must be preserved.

## Unknowns

1. **What should happen when embedding fails at runtime?** Need to define explicit behavior for three cases:
   - Missing embedding (embedding generation fails)
   - Vector index unavailable (sqlite-vec table missing or corrupted)
   - Vector query succeeds but returns zero rows

## Design

### 1. Service construction (always-on)

Remove the `if not ctx.cfg.memory.use_memory_layer: return None` guard in `_build_memory_services()`. Always construct and return `MemoryServices`. Remove `MemoryServices | None` return type.

### 2. JSONL removal

- Remove `_build_jsonl_store()` function entirely from `factory.py`
- Remove `jsonl` parameter from `_build_ingestion_service()` call site
- Remove `jsonl` parameter from `MemoryIngestionService.__init__()` signature
- Remove `self._jsonl` assignment and `await self._jsonl.write(entry)` in `_persist_entry_with_embedding()`
- Delete `jsonl_store.py` file

### 3. Embedding-required model

- Remove `enabled=` argument from `client_cls(cfg, http, enabled=...)` in `_build_embedding_client()`
- Remove `enabled: bool = False` from `EmbeddingClient.__init__()` signature
- Remove `self._enabled` attribute from `EmbeddingClient`
- Remove the `if not self._enabled` early-return in `EmbeddingClient.fetch()` that returns `EmbeddingErrorKind.DISABLED`
- Keep circuit breaker behavior intact
- Keep `local_only` validation in `EmbeddingClient.__init__()`

### 4. Retriever behavior

- Remove FTS-only fallback from `HybridRetriever.search()`:
  - When `embedding is None`: raise `ValueError` instead of returning FTS-only hits
  - When `vec_hits` is empty: raise `RuntimeError` instead of returning FTS-only hits
- Remove `self.last_retrieval_mode` attribute from `HybridRetriever`
- Remove `self.fts_fallback_count` attribute from `HybridRetriever`
- Update `MemoryServices.get_stats()` to remove `last_retrieval_mode` and `fts_fallback_count` references

### 5. Activation mode simplification

Update `MemoryServices.get_activation_mode()`:
- Remove `if not self._use_memory_layer: return "disabled"` branch
- Remove `if embed_client is None: return "fts-only"` branch
- Simplify to: "degraded" (circuit open) / "hybrid" (normal)

### 6. Type changes

- Change `AppServices.memory` type annotation from `MemoryServices | None` to `MemoryServices`
- Update `context.py` docstring accordingly

## Implementation steps

### Step 1: Remove JSONL from ingestion path
1. Remove `jsonl` parameter from `MemoryIngestionService.__init__()`
2. Remove `self._jsonl = jsonl` assignment
3. Remove `await self._jsonl.write(entry)` in `_persist_entry_with_embedding()`
4. Remove `from agent.memory.jsonl_store import JsonlMemoryStore` import from ingestion.py

### Step 2: Remove JSONL store construction from factory.py
1. Remove `_build_jsonl_store()` function
2. Remove `jsonl = _build_jsonl_store(ctx, JsonlMemoryStore)` line
3. Remove `jsonl=jsonl` from `_build_ingestion_service()` call
4. Remove `JsonlMemoryStore` import

### Step 3: Remove `enabled=` from EmbeddingClient construction
1. Remove `enabled=ctx.cfg.memory.memory_embed_enabled` from `_build_embedding_client()`
2. Remove `enabled: bool = False` from `EmbeddingClient.__init__()`
3. Remove `self._enabled = enabled` from `EmbeddingClient.__init__()`
4. Remove `if not self._enabled` early-return in `fetch()`
5. Remove `enabled=self._enabled` from `get_status()` return

### Step 4: Remove FTS-only fallback from retriever
1. Replace `if embedding is None:` block in `search()` with `raise ValueError("embedding required")`
2. Replace `if not vec_hits:` block in `search()` with `raise RuntimeError("vector index returned no results")`
3. Remove `self.last_retrieval_mode` initialization and all assignments
4. Remove `self.fts_fallback_count` initialization and all increments
5. Remove `logger.info("retrieval: fts_only ...")` log lines

### Step 5: Clean up MemoryServices
1. Remove `use_memory_layer: bool = True` from `MemoryServices.__init__()`
2. Remove `self._use_memory_layer = use_memory_layer`
3. Remove `if not self._use_memory_layer: return "disabled"` from `get_activation_mode()`
4. Remove `if embed_client is None: return "fts-only"` from `get_activation_mode()`
5. Remove `"last_retrieval_mode": self.retriever.last_retrieval_mode` from `get_stats()`
6. Remove `"fts_fallback_count": self.retriever.fts_fallback_count` from `get_stats()`

### Step 6: Update AppServices type
1. Change `memory: MemoryServices | None` to `memory: MemoryServices` in `AppServices.__init__()`
2. Update class docstring to reflect memory is now always present

### Step 7: Update callers that check memory is None
1. In `orchestrator.py`: Remove `if ctx.services_required.memory is not None:` guard in `_handle_memory_injection()` â€” call `on_user_prompt()` unconditionally
2. In `context_view.py`: Remove `if ctx.services_required.memory is None:` check in `_format_memory_status()` â€” return enabled status directly

### Step 8: Delete jsonl_store.py
Delete the file since it's no longer needed at runtime.

## Validation plan

1. **Unit tests**: Run existing Memory runtime tests to verify no regressions
2. **Startup verification**: Confirm MemoryServices is always constructed after successful startup
3. **Ingestion verification**: Verify runtime writes go only to SQLite (check logs for absence of JSONL write attempts)
4. **Retrieval verification**: Verify retrieval always requires vector search; confirm error behavior for missing embeddings and unavailable vector indexes
5. **Status verification**: Confirm `get_activation_mode()` returns correct values ("degraded"/"hybrid")
6. **Type checking**: Run type checker to ensure no type errors from removing `| None` from memory field
7. **Import verification**: Ensure `import_from_jsonl()` in `import_ops.py` still works for migration purposes
