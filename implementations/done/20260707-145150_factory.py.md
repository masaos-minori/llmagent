# Implementation: M-4 — Surface get_process_snapshot() in _ServerLifecycleRouter (factory.py)

## Goal

Add `get_process_snapshot(server_key)` to `_ServerLifecycleRouter` delegating to
`HttpServerLifecycleManager.get_process_snapshot()`, so `McpStatusService` can access
process info via duck-typing without needing direct access to `_http_mgr`.

## Scope

**Target**: `scripts/agent/factory.py`

**Step covered**: Plan M-4 step 3.

**Out of scope**: `lifecycle_protocol.py` (covered separately), test files.

## Assumptions

1. `get_process_snapshot()` was added to `HttpServerLifecycleManager` (M-4 step 2).
2. `McpStatusService` calls `getattr(lifecycle, "get_process_snapshot", None)` to
   guard against lifecycle managers that don't implement this method.
3. `LifecycleManagerProtocol` is extended with an optional default `return None` in
   M-4 step for `lifecycle_protocol.py`.

## Implementation

### Target file

`scripts/agent/factory.py`

### Procedure

#### Step 1: Add `get_process_snapshot()` to `_ServerLifecycleRouter`

Add after `get_transport_state()`:

```python
def get_process_snapshot(self, server_key: str) -> dict | None:
    """Return process snapshot dict for a managed subprocess server, or None."""
    return self._http_mgr.get_process_snapshot(server_key)
```

### Method

- One-method addition; pure delegation to `_http_mgr`.
- No new imports.

### Details

- `_ServerLifecycleRouter` is the facade for all lifecycle operations; adding snapshot
  here keeps callers decoupled from `HttpServerLifecycleManager` internals.
- Return type `dict | None` matches `HttpServerLifecycleManager.get_process_snapshot()`.
- `McpStatusService` accesses via `getattr(lifecycle, "get_process_snapshot", None)`
  as a duck-typing call; no protocol extension required at this level.
