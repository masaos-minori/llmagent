# Implementation: Update open_db() docstring to reflect new connection model

Source plan: `plans/20260625-140631_plan.md` (req #21, db.py portion)

## Goal

Remove the outdated "single async event loop thread" justification from the `open_db()` docstring in `scripts/eventbus/db.py` and replace it with an accurate description of the shared + Lock model.

## Scope

- Update `open_db()` docstring only
- No logic changes
- Prerequisite: req #16 (threading.Lock) must be implemented; the new docstring references `_db_lock`

## Assumptions

1. req #16 is implemented; `_db_lock` and `get_db_lock()` exist in `db.py`
2. The existing docstring at lines ~13-17 of `db.py` contains the phrase "FastAPI runs on a single async event loop thread"

## Implementation

### Target file

`scripts/eventbus/db.py`

### Procedure

1. Locate the `open_db()` docstring (currently lines 13-17)
2. Replace the body of the docstring

### Method

String replacement — remove old justification, write new justification.

### Details

**Remove:**
```python
    """Return a shared SQLite connection for the Event Bus.

    Uses check_same_thread=False because FastAPI runs on a single async event
    loop thread; WAL mode serializes concurrent writers at the SQLite level.
    A single shared connection avoids per-request churn and is safe here.
    """
```

**Replace with:**
```python
    """Return a shared SQLite connection for the Event Bus.

    Uses check_same_thread=False. Concurrent access from asyncio.to_thread()
    calls is serialized by _db_lock (retrieve via get_db_lock()). WAL mode
    additionally serializes concurrent writers at the SQLite level.
    A single shared connection avoids per-request connection churn.
    """
```
