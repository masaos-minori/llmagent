## Goal

Add `fdisp_browser_fetch`, formatting a `BrowserFetchResponse` as plain text for the LLM, and
register it in the shared `_WEB_DISPATCH` table so `dispatch_web_tool("browser_fetch", args)`
routes correctly — per UNK-01, this formatter calls `service.fetch_browser`, not
`search_provider.fetch_browser` directly.

## Scope

**In scope**: add one function + one dict entry to
`scripts/mcp_servers/web_search/formatters.py` (current: 45 lines, one dispatch entry
`"search_web"`).
**Out of scope**: `fmt_search_result`/`fdisp_search_web`/`dispatch_web_tool` — untouched. Dict name
stays `_WEB_DISPATCH`, function name stays `dispatch_web_tool` (plan's Affected-areas note: "no
renames needed").

## Assumptions

1. `fdisp_search_web`'s shape (lines 28-35) is the template: `async def fdisp_search_web(args:
   dict[str, Any]) -> str: result = await service.search_web(args); ...; return <formatted text>`.
   `fdisp_browser_fetch` mirrors this exactly, substituting `service.fetch_browser`.
2. Unlike `fdisp_search_web` (which has an empty-results special case), `BrowserFetchResponse` has
   no analogous "empty" case to special-case — every successful fetch has a `text` field (possibly
   an empty string if the page had no visible text, which is still a valid, non-error result). No
   "No results found" branch is needed; format unconditionally.
3. `result.truncated` needs a visible marker in the formatted text, mirroring
   `browser_service.py`'s own `_format_fetch_result` (source lines 136-145: prepends
   `status_code=... elapsed=...s`, then `[RESPONSE TRUNCATED]` if truncated, then the text) — port
   this exact 3-part text layout so LLM-facing output is unchanged by the merge.

## Implementation

### Target file

`scripts/mcp_servers/web_search/formatters.py`

### Procedure

1. Extend the `mcp_servers.web_search.web_search_models` import to include `BrowserFetchResponse`
   (type hint only, for a private formatting helper's signature).
2. Add a private `_format_fetch_result(result: BrowserFetchResponse) -> str` (ported verbatim from
   `browser_service.py`'s `_format_fetch_result`, source lines 136-145).
3. Add:
   ```
   async def fdisp_browser_fetch(args: dict[str, Any]) -> str:
       """Format browser_fetch MCP tool result as plain text for LLM."""
       result = await service.fetch_browser(args)
       return _format_fetch_result(result)
   ```
4. Add `"browser_fetch": fdisp_browser_fetch,` to the `_WEB_DISPATCH` dict (line 38-40).

### Method

Two-function addition (formatter + private helper) plus one dict entry — no change to
`dispatch_web_tool`'s body (it is already generic over `_WEB_DISPATCH`'s keys via
`mcp_servers.dispatch.dispatch_tool`).

### Details

- `service` (the module, already imported at line 17: `from mcp_servers.web_search import
  service`) already exposes `fetch_browser` once the companion `service.py` doc lands — no new
  import needed here beyond the `BrowserFetchResponse` type hint.
- Final `_WEB_DISPATCH` shape:
  ```
  _WEB_DISPATCH: dict[str, Any] = {
      "search_web": fdisp_search_web,
      "browser_fetch": fdisp_browser_fetch,
  }
  ```
