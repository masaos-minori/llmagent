# Implementation: H-9 ÔÇö LifecycleState tracking in _ServerLifecycleRouter (factory.py)

## Goal

Add `_states: dict[str, LifecycleState]` to `_ServerLifecycleRouter` and a `_set_state()`
helper. Update `start_http_subprocess()`, `restart()`, `shutdown_all()`, and
`ensure_ready()` to transition states (`STARTING` Ôćĺ `RUNNING`/`FAILED`, `STOPPED`).
Implement `get_transport_state()` to return meaningful values from `_states`.

## Scope

**Target**: `scripts/agent/factory.py`

**Steps covered**: Plan H-9 steps 1ÔÇô6.

**Out of scope**: `lifecycle.py` (no changes to enum), `http_lifecycle.py`, `context.py`,
config.

## Assumptions

1. `LifecycleState` and `assert_valid_transition` are already in `scripts/agent/lifecycle.py`
   (confirmed). `LifecycleState` has `STARTING`, `RUNNING`, `STOPPED`, `FAILED`,
   `UNKNOWN`.
2. State tracking happens at the `_ServerLifecycleRouter` level, not inside
   `HttpServerLifecycleManager`.
3. H-4 `ensure_ready()` calls `_http_mgr.start()` directly; state tracking must be
   added there too (via a shared `_start_and_track()` helper or inline).
4. `assert_valid_transition()` raises `ValueError` on invalid transitions; the helper
   catches it, logs, and writes the new state anyway.

## Implementation

### Target file

`scripts/agent/factory.py`

### Procedure

#### Step 1: Import LifecycleState and assert_valid_transition

The existing import in `factory.py` already includes:
```python
from agent.lifecycle import LifecycleState
```

Extend to:
```python
from agent.lifecycle import LifecycleState, assert_valid_transition
```

#### Step 2: Add `_states` to `__init__()` and implement `_set_state()`

In `_ServerLifecycleRouter.__init__()`, after `_shutting_down`:
```python
self._states: dict[str, LifecycleState] = {}
```

Add private helper method:
```python
def _set_state(self, server_key: str, new_state: LifecycleState) -> None:
    current = self._states.get(server_key, LifecycleState.UNKNOWN)
    try:
        assert_valid_transition(current, new_state)
    except ValueError:
        _logger.warning(
            "Lifecycle: invalid state transition %r -> %r for %r",
            current,
            new_state,
            server_key,
        )
    self._states[server_key] = new_state
```

#### Step 3: Update `start_http_subprocess()` to track state

```python
async def start_http_subprocess(
    self,
    server_key: str,
    cfg: McpServerConfig,
) -> None:
    if self._shutting_down:
        _logger.debug(
            "Lifecycle: start_http_subprocess(%r) ignored ÔÇö shutting down", server_key
        )
        return
    self._set_state(server_key, LifecycleState.STARTING)
    try:
        await self._http_mgr.start(server_key, cfg)
        self._set_state(server_key, LifecycleState.RUNNING)
    except Exception:
        self._set_state(server_key, LifecycleState.FAILED)
        raise
```

#### Step 4: Update `restart()` to track state

```python
async def restart(self, server_key: str) -> None:
    if self._shutting_down:
        _logger.warning("Lifecycle: restart(%r) ignored ÔÇö shutting down", server_key)
        return
    cfg = self._server_configs.get(server_key)
    if cfg is None or cfg.startup_mode != StartupMode.SUBPROCESS:
        _logger.warning(
            "Lifecycle: restart %r: not a subprocess-mode server;"
            " manual restart required",
            server_key,
        )
        return
    self._set_state(server_key, LifecycleState.STARTING)
    try:
        await self._http_mgr.restart(server_key, cfg)
        self._set_state(server_key, LifecycleState.RUNNING)
    except Exception:
        self._set_state(server_key, LifecycleState.FAILED)
        raise
```

#### Step 5: Update `shutdown_all()` to set STOPPED state

```python
async def shutdown_all(self) -> None:
    self._shutting_down = True
    await self._http_mgr.shutdown_all()
    for key in self._server_configs:
        self._set_state(key, LifecycleState.STOPPED)
```

#### Step 6: Add state tracking to `ensure_ready()` (H-4 path)

In the `ensure_ready()` body, after `not self._http_mgr.verify_running(server_key)`
triggers the start call:
```python
self._set_state(server_key, LifecycleState.STARTING)
try:
    await self._http_mgr.start(server_key, cfg)
    self._set_state(server_key, LifecycleState.RUNNING)
except Exception:
    self._set_state(server_key, LifecycleState.FAILED)
    raise
```

#### Step 7: Implement `get_transport_state()` using `_states`

Replace:
```python
def get_transport_state(self, server_key: str) -> LifecycleState:
    return LifecycleState.UNKNOWN
```

With:
```python
def get_transport_state(self, server_key: str) -> LifecycleState:
    return self._states.get(server_key, LifecycleState.UNKNOWN)
```

### Method

- Net changes: 1 import extension, 1 dict field, 1 helper method, 5 method updates.
- No new external imports; `assert_valid_transition` is already in `agent.lifecycle`.

### Details

- `shutdown_all()` iterates `_server_configs` (not `_http_procs`) to set STOPPED state
  for all known servers, including those that may have already exited.
- `_set_state()` catches `ValueError` from `assert_valid_transition()` and logs a
  warning. It still writes the new state to allow recovery from unexpected transitions.
- The H-4 `ensure_ready()` path and the `start_http_subprocess()` path both call
  `_http_mgr.start()`; both must track state. An alternative is to extract a shared
  `_start_and_track()` method; this plan inlines the state transitions for clarity.
- `get_transport_state()` returns `LifecycleState.UNKNOWN` for server keys not in
  `_states` (e.g. persistent-mode servers that were never started by the router).
