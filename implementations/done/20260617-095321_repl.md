# Implementation: Log watchdog status at startup in repl.py (req 26 — Step 2)

## Goal

Log one info-level line at startup indicating whether the watchdog is enabled or disabled, so operators can verify the state without reading config files.

## Scope

- `scripts/agent/repl.py`

## Assumptions

1. Logging happens after MCP servers are started (inside `_start_watchdog` or just before it in the startup sequence).
2. Uses existing `logger` (Logger instance at module level).
3. The log call goes in `_start_watchdog()` so it runs at the point the decision is made.

## Implementation

### Target file

`scripts/agent/repl.py`

### Procedure

In `_start_watchdog()`, add logging before the conditional return.

### Method

Edit tool.

### Details

```python
async def _start_watchdog(self, ctx: AgentContext) -> "asyncio.Task | None":
    """Create the watchdog task if watchdog_interval > 0."""
    watchdog_interval = ctx.cfg.mcp.mcp_watchdog_interval
    if watchdog_interval > 0:
        logger.info(
            "Watchdog enabled: interval=%ss, max_restarts=%s",
            watchdog_interval,
            ctx.cfg.mcp.mcp_watchdog_max_restarts,
        )
        return asyncio.create_task(self._watchdog_loop())
    logger.info("Watchdog disabled (mcp_watchdog_interval=0)")
    return None
```
