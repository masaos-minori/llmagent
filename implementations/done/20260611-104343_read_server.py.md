# Implementation: mcp/file/read_server.py — Remove private attribute access _max_tree_depth

## Goal

Replace the direct private attribute access `_service._max_tree_depth` at line 125 of
`read_server.py` with a public property accessor on `ReadFileService`, eliminating
access to private state from the server layer.

## Scope

**Target files**:
- `scripts/mcp/file/read_service.py` (add property)
- `scripts/mcp/file/read_server.py` (use property)

In scope:
- Add `max_tree_depth` property to `ReadFileService`
- Update `read_server.py` line 125 to use the property

Out of scope:
- `_allowed_dirs` private attribute also accessed in `read_server.py:219` via
  `list_allowed_directories` endpoint — consider as optional follow-on

## Assumptions

1. `ReadFileService._max_tree_depth` is an `int` set in `__init__`.
2. The property is read-only (no setter needed).
3. `_service._allowed_dirs` in `list_allowed_directories` endpoint (line 219) is a
   separate concern not addressed in this step.

## Implementation

### Target files

1. `scripts/mcp/file/read_service.py`
2. `scripts/mcp/file/read_server.py`

### Procedure

**Change 1 — add property to ReadFileService**

In `read_service.py`, locate the `ReadFileService` class. After the `__init__` method,
add a property:

```python
    @property
    def max_tree_depth(self) -> int:
        """Maximum tree recursion depth configured for this service."""
        return self._max_tree_depth
```

**Change 2 — use property in read_server.py**

Locate line 125 in `read_server.py`:

**Before:**
```python
    depth = min(req.depth, _service._max_tree_depth)
```

**After:**
```python
    depth = min(req.depth, _service.max_tree_depth)
```

### Method

Two `Edit` operations. No imports changed.

### Details

- The property is a simple pass-through with no logic; it just exposes the read-only value.
- mypy will verify that `_service._max_tree_depth` access warning disappears after the change
  (though mypy doesn't enforce private attribute access by default, ruff may flag it).
- This change does not affect runtime behavior.
