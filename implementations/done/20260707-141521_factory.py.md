# Implementation: H-3 — Shutdown guard in _ServerLifecycleRouter (factory.py)

## Goal

Add a `_shutting_down: bool = False` flag to `_ServerLifecycleRouter.__init__()`. Set it
to `True` as the first statement in `shutdown_all()`. Guard `ensure_ready()`,
`start_http_subprocess()`, `restart()`, and `shutdown_idle()` so they return immediately
(with a log line) when `_shutting_down` is True.

## Scope

**Target**: `scripts/agent/factory.py`

**Steps covered**: Plan H-3 steps 1, 2, 3.

**Out of scope**: `http_lifecycle.py`, `context.py`, `LifecycleManagerProtocol`, config.

## Assumptions

1. All HTTP subprocess start/restart paths are routed through `_ServerLifecycleRouter`;
   `HttpServerLifecycleManager` is never called directly from outside `factory.py`.
2. The guard is idempotent: setting `_shutting_down = True` twice is safe.
3. `restart()` uses `_logger.warning` (higher severity than debug); all other guarded
   methods use `_logger.debug`.
4. `shutdown_idle()` is currently a no-op (`pass`); the guard replaces the no-op body.

## Implementation

### Target file

`scripts/agent/factory.py`

### Procedure

#### Step 1: Add `_shutting_down` to `__init__()`

In `_ServerLifecycleRouter.__init__()`, after the existing assignments, add:

```python
self._shutting_down: bool = False
```

Current `__init__()` body:
```python
self._server_configs = server_configs
self._http_mgr = HttpServerLifecycleManager()
```

New body:
```python
self._server_configs = server_configs
self._http_mgr = HttpServerLifecycleManager()
self._shutting_down: bool = False
```

#### Step 2: Set guard in `shutdown_all()`

Insert as the first statement in `shutdown_all()`:

```python
self._shutting_down = True
```

Full updated method:
```python
async def shutdown_all(self) -> None:
    self._shutting_down = True
    await self._http_mgr.shutdown_all()
```

#### Step 3: Guard `ensure_ready()`

Insert at the top of `ensure_ready()`:
```python
if self._shutting_down:
    _logger.debug("Lifecycle: ensure_ready(%r) ignored — shutting down", server_key)
    return
```

#### Step 4: Guard `start_http_subprocess()`

Insert at the top:
```python
if self._shutting_down:
    _logger.debug(
        "Lifecycle: start_http_subprocess(%r) ignored — shutting down", server_key
    )
    return
```

#### Step 5: Guard `restart()` (warning level)

Insert at the top:
```python
if self._shutting_down:
    _logger.warning(
        "Lifecycle: restart(%r) ignored — shutting down", server_key
    )
    return
```

#### Step 6: Guard `shutdown_idle()`

Replace the current `pass` body:
```python
async def shutdown_idle(self) -> None:
    if self._shutting_down:
        _logger.debug("Lifecycle: shutdown_idle ignored — shutting down")
        return
```

### Method

- Surgical edits: one field addition and six guard-block insertions.
- No new imports required (uses existing `_logger`).
- No protocol changes; `LifecycleManagerProtocol` is unmodified.

### Details

- The guard in `restart()` uses `_logger.warning` because an ignored restart is
  diagnostically more significant than an ignored `ensure_ready`. Operators may call
  `/restart` during shutdown and need visibility.
- `shutdown_all()` sets the flag before awaiting `_http_mgr.shutdown_all()`. Under the
  GIL, bool assignment is atomic; no lock is needed for this single-process async
  application.
- The guard in `shutdown_idle()` is defensive; the method is currently a no-op so the
  guard has no observable effect today but protects future additions to `shutdown_idle`.
