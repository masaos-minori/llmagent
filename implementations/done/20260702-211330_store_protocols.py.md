# Implementation: scripts/db/store_protocols.py

## Goal

Extend `DocumentStore.chunk_insert()` in the Protocol to accept `chunk_type` and `source_file` as optional keyword parameters, bringing the Protocol API into alignment with the canonical `chunks` table schema.

## Scope

- Target: `scripts/db/store_protocols.py`
- Add `chunk_type: str = ""` and `source_file: str = ""` to `DocumentStore.chunk_insert()` signature
- Update the method docstring to describe the two new parameters
- No other Protocol methods changed

## Assumptions

1. `SQLiteDocumentStore` is the only concrete implementation of `DocumentStore`; no other implementors exist.
2. Default values of `""` preserve backward compatibility — existing callers that pass at most 4 positional args continue to work.
3. `runtime_checkable` Protocol does not enforce default values at structural-subtype check time; adding optional params does not break `isinstance` checks for existing implementors.
4. `chunk_type` and `source_file` are plain `str` (not `Optional[str]`) — empty string is the sentinel for "not set", consistent with the existing INSERT in `RagIngester._insert_chunk()`.

## Implementation

### Target file

`scripts/db/store_protocols.py`

### Procedure

1. Locate the `DocumentStore` Protocol class and its `chunk_insert` method signature.
2. After the existing `normalized: str | None` parameter, add two keyword-only parameters with defaults:
   - `chunk_type: str = ""`
   - `source_file: str = ""`
3. Update the method docstring to document the new parameters.
4. Add a comment noting that the field set matches the `chunks` table INSERT in `RagIngester._insert_chunk()`.

### Method

Replace the existing `chunk_insert` signature:

```python
def chunk_insert(
    self,
    doc_id: int,
    chunk_index: int,
    content: str,
    normalized: str | None,
) -> int: ...
```

With the extended signature:

```python
def chunk_insert(
    self,
    doc_id: int,
    chunk_index: int,
    content: str,
    normalized: str | None,
    chunk_type: str = "",
    source_file: str = "",
) -> int:
    """Insert one chunk row; return the new chunk_id.

    Args:
        doc_id: FK to the parent document row.
        chunk_index: Zero-based position of this chunk within the document.
        content: Raw chunk text.
        normalized: Pre-normalized text used for FTS and vector indexing, or None.
        chunk_type: Content type label (e.g. "text", "code"). Defaults to "".
        source_file: Path to the originating source file. Defaults to "".

    Returns:
        The rowid of the newly inserted chunk row.

    Note:
        Field set matches RagIngester._insert_chunk() INSERT in rag/ingester.py.
    """
    ...
```

### Details

- Keep the existing `@runtime_checkable` decorator on the Protocol class unchanged.
- The two new parameters must come after `normalized` and must carry default values so that all current call sites (which pass 4 positional args) continue to work without modification.
- Do not add `*` (keyword-only separator) before the new params — the Protocol already uses positional args in existing tests.
