## Assumptions

1. The TOML style used by the 45 existing `[[tool_definitions]]` entries is the required style for the 19
   new entries: `[[tool_definitions]]` / `type = "function"` / `[tool_definitions.function]` with
   `name`/`description` / `[tool_definitions.function.parameters]` with `type = "object"` +
   `required = [...]` / one `[tool_definitions.function.parameters.properties.<param>]` sub-table per
   parameter. Confirmed by direct read of `config/agent.toml` lines 1058-1223 (`write_file`,
   `create_directory`, `delete_file`, `delete_directory`, `grep_files`, `read_multiple_files`,
   `get_file_info`).
2. Array-typed params use a `[...properties.<param>].items` sub-table for the item type (confirmed pattern:
   `read_multiple_files.paths` at lines 1202-1207 — flat string array).
3. Descriptions use Japanese, consistent with all existing 45 entries (e.g. `write_file`'s
   "ローカルファイルを新規作成または上書きする…").
4. Nested array-of-objects (`edit_file.edits`, each item having its own `properties`/`required`) and a
   zero-property schema (`stats`) are **new patterns** not present in any of the 45 existing entries —
   TOML dotted-table syntax for both must be verified via a `tomllib` round-trip before being considered
   correct.
5. Source parameter contracts (verified live against source on 2026-07-21):
   - `scripts/mcp_servers/file/write_tools.py` lines 37-73 (`edit_file`), 95-114 (`move_file`)
   - `scripts/mcp_servers/git/git_tools.py` lines 17-260 (all 10 tools)
   - `scripts/mcp_servers/mdq/mdq_tools.py` lines 25-212 (all 7 tools)
