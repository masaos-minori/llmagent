# Implementation: config_reload.py — Diff-apply and remove legacy comment

## Goal

1. Change all config helpers to diff-apply mode: only update `ctx.cfg` fields when the key
   is present in `new_cfg`; preserve current value otherwise.
2. Rename `sync_services()` → `_sync_services()` and remove the "directly by legacy code" comment.

## Scope

- `scripts/agent/services/config_reload.py`: modify `_apply_rag_tool_params`, `_apply_llm_prompt_params`,
  `_apply_sse_reload_params`; rename `sync_services` → `_sync_services`; update callers.

## Assumptions

1. `sync_services()` has no external callers — only `apply_config_dict()` calls it.
2. Diff-apply means: `if "key" in new_cfg: ctx.cfg.x = convert(new_cfg["key"])`.
   Fields absent from `new_cfg` retain their current `ctx.cfg` value.
3. `_reload_approval_settings` already uses `if "key" in new_cfg:` guards — no change needed.
4. `bool(new_cfg.get(...))` conversion is kept but only applied when key is present.

## Implementation

### Target file

`scripts/agent/services/config_reload.py`

### Procedure

**Step 1 — rename sync_services → _sync_services:**

- Rename method definition: `def sync_services` → `def _sync_services`.
- Update call in `apply_config_dict`: `self.sync_services` → `self._sync_services`.
- Remove the legacy comment on the section header.

**Step 2 — diff-apply _apply_rag_tool_params:**

Replace unconditional assignments with guarded ones. Example:
```python
# Before:
ctx.cfg.llm.context_char_limit = int(new_cfg.get("context_char_limit", 8000))

# After:
if "context_char_limit" in new_cfg:
    ctx.cfg.llm.context_char_limit = int(new_cfg["context_char_limit"])
```
Apply this pattern to ALL fields in `_apply_rag_tool_params`, `_apply_llm_prompt_params`,
`_apply_sse_reload_params`.

**Step 3 — keep _reload_approval_settings unchanged** (already guards with `if "key" in new_cfg:`).

### Method

Systematic find-and-replace pattern across three helper methods + rename.
