# Implementation: schema_sql.py — Add Z-suffix DEFAULT to Workflow Table DDL

**Plan source:** `plans/20260702-201038_plan.md` (Phase 2a)
**Target file:** `scripts/db/schema_sql.py`

---

## Goal

Add `DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` to all timestamp columns in the workflow table DDL (`_WORKFLOW_SCHEMA`). This ensures newly created databases emit Z-suffix timestamps from the DDL level, consistent with the RAG, Session, and Event Bus schemas already defined in the same file.

---

## Scope

**In:**
- `_WORKFLOW_SCHEMA` constant in `scripts/db/schema_sql.py`
- Columns: `tasks.created_at`, `tasks.updated_at`, `attempts.started_at`, `processed_events.recorded_at`, `artifacts.created_at`, `approvals.created_at`

**Out:**
- `attempts.ended_at` — nullable column; add DEFAULT only if the column is defined as `TEXT NOT NULL`. Because it is nullable (`TEXT`), do not add a DEFAULT.
- Any table outside the workflow schema
- Any Python runtime code
- Existing DB files (change is DDL only; `CREATE TABLE IF NOT EXISTS` is idempotent)

---

## Assumptions

1. The canonical format is `YYYY-MM-DDTHH:MM:SSZ` (Z suffix), as decided in plan Phase 1.
2. `CREATE TABLE IF NOT EXISTS` means the new DEFAULT has no effect on existing DBs. This is intentional.
3. All listed columns are currently defined as `TEXT NOT NULL` without a DEFAULT clause (confirmed by grep).
4. `attempts.ended_at` is currently `TEXT` (nullable); its DEFAULT is intentionally omitted.

---

## Implementation

### Target file

`scripts/db/schema_sql.py`

### Procedure

1. Locate the `_WORKFLOW_SCHEMA` string literal (the section containing `CREATE TABLE IF NOT EXISTS tasks`, `attempts`, `processed_events`, `artifacts`, `approvals`).
2. For each column listed below, append `DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` after `NOT NULL`.
3. Do not touch columns outside the workflow schema.
4. Do not reorder columns or alter NOT NULL constraints.

### Method

Targeted string replacement inside `_WORKFLOW_SCHEMA`. Use Edit tool for each column individually to minimize diff noise.

### Details

Columns to update (current → target):

| Table | Column | Before | After |
|---|---|---|---|
| `tasks` | `created_at` | `TEXT NOT NULL` | `TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` |
| `tasks` | `updated_at` | `TEXT NOT NULL` | `TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` |
| `attempts` | `started_at` | `TEXT NOT NULL` | `TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` |
| `processed_events` | `recorded_at` | `TEXT NOT NULL` | `TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` |
| `artifacts` | `created_at` | `TEXT NOT NULL` | `TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` |
| `approvals` | `created_at` | `TEXT NOT NULL` | `TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))` |

`attempts.ended_at` (`TEXT`, nullable) — leave unchanged.
