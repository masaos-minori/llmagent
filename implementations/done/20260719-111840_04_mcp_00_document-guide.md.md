## Goal

Update `docs/04_mcp_00_document-guide.md`'s File Index (add the new Browser MCP catalog page)
and add a Migration Notes bullet mapping the original "MCP統合プラグインシステム" Gateway-style
proposal names (`list_files`/`read_file`/`search_file`/`invoke_script`) to the actual current
tool names (`list_directory`/`read_text_file`/`search_files`/`shell_run`).

## Scope

**In scope**
- File Index edit (`docs/04_mcp_00_document-guide.md:76`, verified): extend the `04_mcp_04`
  catalog-pages row to include `_06` (browser), updating the "(5分割、_04=mdq)" parenthetical to
  "(6分割、_04=mdq)".
- One new Migration Notes bullet (section starts at line 101, verified) mapping the Gateway-
  style proposal names to real tool names, placed after the existing `POST /v1/search` bullet
  (lines 103-105, verified).

**Out of scope — the plan's own wording names an "AI Query Routing Table row for `04_mcp_04`,"
but verification shows no edit is actually needed there.** Direct read of
`docs/04_mcp_00_document-guide.md:36-48` (the full AI Query Routing Table) shows the existing
`04_mcp_04` row (line 43) already reads generically: "web-search/github/shell/mdq各mcpが提供する
toolは。mdq-mcpのFTS5検索は本番稼働可能、ハイブリッド検索は未実装" → `04_mcp_04`. This row
describes the *category* of question ("what tools do the catalog-page servers provide") and
routes to the whole `04_mcp_04` series by number, not to an enumerated list of every individual
server name — `web-search`/`github`/`shell`/`mdq` are named as *examples*, and `git` (added in
an earlier plan, per `04_mcp_04_05_git.md`'s existence) is notably **already absent** from this
example list despite being a real, current `04_mcp_04` catalog page. This confirms the row's
example-server list is deliberately non-exhaustive and has not been kept in lockstep with every
catalog-page addition (`git` was added without updating this line) — so adding `browser` here
would be inconsistent with the row's own established (non-)maintenance pattern, not a gap
specific to this plan. No edit to this line is required; the File Index row (line 76, which
*does* enumerate every catalog page number, `_01` through `_05`) is the correct, actually-
enumerated place for `browser`'s addition, and is already in this doc's Scope above.
