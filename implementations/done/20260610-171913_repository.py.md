# Implementation: rag/repository.py — Fail-Fast DB Access and Sudachi Provider

## Goal

Remove the Sudachi tokenization `except Exception` fallback, use `RawHit` TypedDict for row results, distinguish DB failure from empty results, and tighten `SemanticCache` dimension validation.

---

## Scope

**Target file:** `scripts/rag/repository.py`

**In:**
- `_build_fts_tokens_ja`: replace `except Exception` with specific exception types; remove non-ASCII fallback
- `_get_sudachi_tokenizer`: let `ImportError` propagate (Sudachi is optional — callers decide how to handle)
- `RagRepository.fts_search`: remove `except sqlite3.OperationalError` catch that returns `[]`; let it propagate
- `RagRepository.vector_search` / `fts_search`: cast rows to `RawHit` (imported from `rag.types`)
- `SemanticCache.lookup` / `put`: validate embedding dimension matches on put/lookup; raise `ValueError` on mismatch
- `deduplicate_chunks` / `_dedup_hits`: update type annotations to use `RawHit` / `MergedHit`

**Out:**
- SQL queries unchanged
- `RagScorer.rrf_merge` logic unchanged (just type annotations updated)
- `cosine_sim` unchanged
- Module-level standalone functions (`vector_search`, `fts_search`, `fetch_full_document`) — type annotations only

---

## Assumptions

1. `sudachipy` raises `RuntimeError` (not a custom exception) for tokenization failures on invalid input.
2. Callers of `fts_search` in `stages/search.py` and `pipeline.py` must catch `sqlite3.OperationalError` after this change — coordinated in Step 5.
3. `SemanticCache` entries all have the same dimension after the first `put` call — validate on every `put` and `lookup` against the stored dimension.
4. `fetch_full_document` can return an empty list when the chunk_id is not found — this is a valid "not found" result, not a failure.
5. `RawHit` is imported from `rag.types` (defined in Step 1).

---

## Implementation

### Target file
`scripts/rag/repository.py`

### Procedure

1. Add `from rag.types import MergedHit, RawHit` import.
2. Rewrite `_build_fts_tokens_ja`: catch only `RuntimeError` (sudachipy tokenization error); let `ImportError` from `_get_sudachi_tokenizer` propagate; remove the `re.findall` fallback.
3. In `fts_search`: remove the `except sqlite3.OperationalError` block; add `sqlite3` import if not present; let the error propagate.
4. In `vector_search` and `fts_search`: replace `cast("list[RagHit]", [dict(r) for r in rows])` with `[RawHit(**dict(r)) for r in rows]` or equivalent.
5. In `rrf_merge`: update input/output type annotations to use `list[RawHit]` → `list[MergedHit]`.
6. In `SemanticCache`: add `_dim: int | None = None` field; validate dimension on `put`; validate on `lookup`; raise `ValueError` if mismatch.
7. Update `deduplicate_chunks` signature: `hits: list[RawHit | MergedHit | RankedHit]`.

### Method

```python
# _build_fts_tokens_ja — specific exception only
def _build_fts_tokens_ja(text: str) -> list[str]:
    tkn, split_c = _get_sudachi_tokenizer()  # ImportError propagates
    try:
        return [
            m.normalized_form()
            for m in tkn.tokenize(text, split_c)
            if m.part_of_speech()[0] in _FTS_KEEP_POS and m.normalized_form().strip()
        ]
    except RuntimeError as e:
        raise RuntimeError(f"Sudachi tokenization failed: {e}") from e

# SemanticCache — dimension guard
class SemanticCache:
    def __init__(self, max_size: int = 100, threshold: float = 0.92) -> None:
        self._entries: list[dict[str, Any]] = []
        self._max_size = max_size
        self._threshold = threshold
        self._dim: int | None = None

    def put(self, embedding: list[float], context_str: str) -> None:
        if self._dim is None:
            self._dim = len(embedding)
        elif len(embedding) != self._dim:
            raise ValueError(
                f"SemanticCache dimension mismatch: expected {self._dim}, got {len(embedding)}"
            )
        self._entries.append({"embedding": embedding, "context_str": context_str})
        self.prune()

    def lookup(self, embedding: list[float]) -> str | None:
        if self._dim is not None and len(embedding) != self._dim:
            raise ValueError(
                f"SemanticCache dimension mismatch: expected {self._dim}, got {len(embedding)}"
            )
```

### Details

- Removing the Sudachi `except Exception` fallback is a breaking change: callers in `_build_fts_query` must propagate the error up to `fts_search`, which then propagates to the pipeline stage. The stage (Step 5) decides whether to skip FTS when Sudachi is unavailable.
- `RawHit` TypedDict construction from `dict(r)`: since `sqlite3.Row` supports `dict()` conversion, `RawHit(**dict(r))` works if `RawHit` fields match column names exactly. Column aliases in SQL match `RawHit` fields (`chunk_id`, `content`, `url`, `title`, `distance`, `bm25_score`).
- `rrf_merge` return type changes to `list[MergedHit]` — callers that assign to `list[RagHit]` remain valid since `MergedHit` is a subtype.
