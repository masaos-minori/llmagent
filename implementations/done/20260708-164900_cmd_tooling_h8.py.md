# Implementation: H-8 — Remove /tool list and /tool show, keep /plan

## Goal

Remove `/tool list` and `/tool show <id>` entirely from `_ToolingMixin`, leaving only
`_cmd_plan` (`/plan`). `ToolResultStore` itself stays defined in `db/tool_results.py` for any
other consumer, but `cmd_tooling.py` no longer references it at all.

Replace `scripts/agent/commands/cmd_tooling.py` in full with:

```python
#!/usr/bin/env python3
"""agent/commands/cmd_tooling.py
Plan-mode mixin for CommandRegistry.

Provides _ToolingMixin with:
  _cmd_plan  — /plan: toggle plan mode
"""

import logging

from agent.commands.mixin_base import MixinBase

logger = logging.getLogger(__name__)


class _ToolingMixin(MixinBase):
    """Plan-mode slash-command handler."""

    def _cmd_plan(self) -> None:
        """Toggle plan mode. In plan mode, plan_blocked_tools are automatically blocked.

        This prevents destructive file operations from being executed while the agent
        is drafting a plan, guarding against accidental writes before the user has
        reviewed the proposal.
        """
        ctx = self._ctx
        ctx.conv.plan_mode = not ctx.conv.plan_mode
        state = "ON" if ctx.conv.plan_mode else "OFF"
        logger.info("Plan mode toggled: %s", state)
        self._out.write(f"Plan mode: {state}")
        if ctx.conv.plan_mode and ctx.cfg.tool.plan_blocked_tools:
            self._out.write("  Blocked tools:")
            for t in ctx.cfg.tool.plan_blocked_tools:
                self._out.write(f"    - {t}")
```

This removes, relative to the original (pre-H-7) file: `import orjson`,
`from db.models import ToolResultRow`, `from shared.json_utils import dumps as _json_dumps`,
`from agent.commands.models import MaskedArgs, ToolResultView`, `DecodedArgs`, `_decode_args`,
`_to_tool_result_view`, `_tool_list`, `_tool_show`, `_cmd_tool`. If H-7 was applied first, it
additionally removes `from db.tool_results import ToolResultStore` (added by that doc) — either
starting point converges to the same file shown above.

### Method

- Full-file replacement rather than incremental deletion, since nearly everything except
  `_cmd_plan` is removed — this is simpler and less error-prone than describing a large diff
  against two possible starting states (pre-H-7 or post-H-7).
- `_cmd_plan()`'s body is copied verbatim (unchanged logic) from the current file.

### Details

- The module and class docstrings are updated to describe the mixin's new, narrower purpose
  (`/plan` only).
- `MixinBase` import and inheritance are unchanged.
- File name (`cmd_tooling.py`) and class name (`_ToolingMixin`) are kept as-is per the plan's
  U-2 resolution (Option A: keep in a slimmed-down `_ToolingMixin`, no file rename).
