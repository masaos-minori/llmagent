# Implementation Procedure: scripts/rag/ingestion/ingester.py

## Goal

Clarify and document `fetched_at` handling in `ingester.py`.
Specifically: confirm that `fetched_at` read from `ChunkJsonRaw` is intentionally excluded from
the `INSERT INTO documents` statement (DB DEFAULT fires instead), and if the requirement is to
persist the input value, add `fetched_at` to the INSERT column list.

## Scope

**In scope:**
- Trace `fetched_at` through `ingest_url_group` → `_get_or_create_document` → INSERT statement
- Confirm whether the DB DEFAULT behavior is intentional or a bug
- If persistence is required: add `fetched_at` to the INSERT columns and values in
  `_get_or_create_document`
- If DEFAULT behavior is intentional: no code change; document the decision explicitly

**Out of scope:**
- Any other field in `ChunkJsonRaw`
- `ETagManager` logic (already handles `fetched_at` for the skip/update path)
- `crawler.py`, `chunk_splitter.py`
- New columns or schema changes

## Assumptions

1. `documents.fetched_at` has `NOT NULL DEFAULT datetime('now')` — the column is always set on
   INSERT even when not supplied explicitly.
2. `fetched_at` from `ChunkJsonRaw` is an ISO-8601 UTC string or `None`.
3. The skip path (`_handle_existing_document`) already passes `fetched_at` to
   `ETagManager.update()` — that path is not changed here.
4. If persistence is required, the value from `ChunkJsonRaw` overrides the DB DEFAULT; the
   INSERT must then include `fetched_at` in both the column list and the parameter tuple.

## Implementation

### Target file

`scripts/rag/ingestion/ingester.py`

### Procedure

1. Confirm the current INSERT statement:
   ```
   grep -n "INSERT INTO documents" scripts/rag/ingestion/ingester.py
   ```
   Expected: the column list does NOT include `fetched_at`.

2. Confirm that `fetched_at` is read from `ChunkJsonRaw` and passed to `_get_or_create_document`:
   ```
   grep -n "fetched_at" scripts/rag/ingestion/ingester.py
   ```

3. Determine the requirement:
   - If **DEFAULT behavior is intentional** (i.e., `fetched_at` should record ingestion time,
     not crawl time): no code change. Go to the documentation step only.
   - If **input value should be persisted**: apply the change below, then go to the
     documentation step.

4. (Conditional — only if persistence is required) Modify `_get_or_create_document`:

   **Before:**
   ```python
   cur = db.execute(
       "INSERT INTO documents"
       " (url, title, lang, etag, last_modified, chunking_strategy)"
       " VALUES (?, ?, ?, ?, ?, ?)",
       (url, title, lang, etag, last_modified, chunking_strategy),
   )
   ```

   **After:**
   ```python
   if fetched_at is not None:
       cur = db.execute(
           "INSERT INTO documents"
           " (url, title, lang, etag, last_modified, chunking_strategy, fetched_at)"
           " VALUES (?, ?, ?, ?, ?, ?, ?)",
           (url, title, lang, etag, last_modified, chunking_strategy, fetched_at),
       )
   else:
       cur = db.execute(
           "INSERT INTO documents"
           " (url, title, lang, etag, last_modified, chunking_strategy)"
           " VALUES (?, ?, ?, ?, ?, ?)",
           (url, title, lang, etag, last_modified, chunking_strategy),
       )
   ```

   Alternatively, always include `fetched_at` — pass `fetched_at or None` and let the DB
   COALESCE with DEFAULT if needed (note: SQLite DEFAULT does not fire when a value is
   explicitly supplied, even `NULL`; use the conditional form above to keep DEFAULT behavior
   for the `None` case).

5. Run lint and type check:
   ```
   ruff check scripts/rag/ingestion/ingester.py
   mypy scripts/rag/ingestion/ingester.py
   ```

### Method

Targeted read and, if required, a single conditional INSERT change in `_get_or_create_document`.
No signature changes needed — `fetched_at: str | None = None` is already in the parameter list.

### Details

- `fetched_at` flow:
  1. `ingest_url_group` reads `first_data.get("fetched_at")` → local `fetched_at`
  2. Passes it to `_get_or_create_document(..., fetched_at=fetched_at)`
  3. `_get_or_create_document` passes it to `_handle_existing_document` (ETagManager path)
  4. Current INSERT does **not** include `fetched_at` → DB DEFAULT `datetime('now')` fires
- Risk if persistence is added: `ETagManager.update()` also sets `fetched_at` on the skip path
  (update-only). Verify that both paths converge to the same semantics after the change.
