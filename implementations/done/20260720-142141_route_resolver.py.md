# Implementation procedure: `scripts/shared/route_resolver.py` (RuntimeToolRegistry sole routing authority)

Source plan: `plans/done/20260720-134821_plan.md`, Implementation step Phase 2 (item 1).

No prior implementation doc covers this exact change: `implementations/20260615-110633_route_resolver.py.md`
(strict_mode addition) and `implementations/20260618-151134_route_resolver.py.md` (`_EXACT_ROUTES` removal +
`known_tools`/coverage-log addition) both target this file but for unrelated, already-landed prior
requirements — neither removes the `ToolRegistry` fallback branch. Confirmed by reading both docs in full.
This is a new document.

## Goal

Make `RuntimeToolRegistry` the sole routing authority inside `ToolRouteResolver`: remove the
`ToolRegistry` fallback entirely from `resolve()` and from `__init__`'s bootstrap, add a
`set_runtime_registry()` mutator, and correct the module docstring, `_raise_strict_error()`,
`warn_on_missing` message, and `_log_routing_coverage()` so none of them reference a second
registry or a "fallback"/"priority" model.

## Scope

**In scope** — `scripts/shared/route_resolver.py` only:
- Module docstring (lines 1-10).
- `ToolRouteResolver.__init__` (lines 61-105): remove the `_legacy_registry` field and its
  `get_registry()` bootstrap (lines 99-106 per current numbering, inside the `try/except`).
- `resolve()` (lines 106-122): remove the "Priority 2" branch that calls `_lookup_legacy_registry()`.
- `_lookup_legacy_registry()` (lines 138-142): delete the method entirely.
- `_raise_strict_error()` (lines 144-149): rewrite message to reference only `RuntimeToolRegistry`.
- `_log_routing_coverage()` (lines 151-180): rewrite to check only `_lookup_runtime_registry()`;
  rewrite its docstring's "mapped via either registry" language.
- Add a new `set_runtime_registry(self, registry: RuntimeToolRegistry | None) -> None` method that
  mutates `self._runtime_registry` in place.

**Out of scope**:
- `build_discovery_map()` (module-level function, lines 26-52) — unrelated, untouched.
- `discovery_map` constructor parameter and `self._discovery_map` field — retained as-is (plan's
  "Future work" defers deprecating this).
- `scripts/shared/tool_registry.py` itself (its class/functions) — separate doc.
- `scripts/shared/tool_executor.py` — separate doc (item 2 of this phase).

## Implementation

### Target file

`scripts/shared/route_resolver.py`

### Procedure

1. Rewrite the module docstring (current lines 1-10):
   - Remove "Routing priority: 1. Tool registry ...".
   - State plainly: `RuntimeToolRegistry` (populated from live `/v1/tools` discovery via
     `ToolExecutor.set_runtime_registry()`) is the sole routing authority; unresolved tools raise
     `ValueError` immediately; `ToolRegistry` is no longer consulted here.
2. In `__init__`:
   - Delete the `self._legacy_registry: ToolRegistry | None = None` field and the following
     `try: from shared.tool_registry import get_registry; self._legacy_registry = get_registry()
     except (ImportError, RuntimeError) as exc: ...` block.
   - Remove the `ToolRegistry` `TYPE_CHECKING` import if no longer referenced anywhere in the file.
   - Update the `runtime_registry` parameter's docstring line ("takes resolution priority over
     legacy ToolRegistry") to state it is the only routing source.
3. In `resolve()`:
   - Delete the `# Priority 2: Legacy ToolRegistry (drift-detection fallback).` block
     (`if (key := self._lookup_legacy_registry(tool_name)) is not None: return key`).
   - Renumber the remaining "Priority 1" comment to drop the priority framing (there is now only
     one lookup).
   - Update the `warn_on_missing` log message from `"tool %r not in any registry; add it to the
     appropriate frozenset in shared/tool_constants.py"` to reference `RuntimeToolRegistry`
     explicitly (per plan's Design section sample: `"tool %r not found in RuntimeToolRegistry;
     ensure MCP servers are healthy and discovery completed."`).
4. Delete `_lookup_legacy_registry()` in its entirety.
5. Rewrite `_raise_strict_error()`'s message to drop "any registry" wording in favor of
   `RuntimeToolRegistry`-specific wording (note: this changes the string that
   `test_strict_mode_error_message_mentions_any_registry` currently asserts against in both test
   files — coordinate with the test-rewrite docs so the new message and the new test expectation
   match exactly).
6. Rewrite `_log_routing_coverage()`:
   - Docstring: replace "Mapped means resolvable via either RuntimeToolRegistry or ToolRegistry"
     with "Mapped means resolvable via RuntimeToolRegistry."
   - Body: change `if (self._lookup_runtime_registry(tool_name) is not None or
     self._lookup_legacy_registry(tool_name) is not None):` to just
     `if self._lookup_runtime_registry(tool_name) is not None:`.
7. Add:
   ```
   def set_runtime_registry(self, registry: RuntimeToolRegistry | None) -> None:
       """Replace the RuntimeToolRegistry consulted by resolve(), in place."""
       self._runtime_registry = registry
   ```
   Place it near `_lookup_runtime_registry()` for locality.

### Method

Direct edits to the existing class body; no new file, no signature change to `resolve()` or
`__init__`'s public parameter list (all constructor parameters — including `discovery_map` and
`known_tools` — are retained even though some are now closer to vestigial per the plan's
"Future work" deferral). `ruff format`/`ruff check`/`mypy` should pass with the `ToolRegistry`
import removed (reduces one dependency edge on `shared/tool_registry.py`, which
`lint-imports` should reflect as a coupling reduction, not a violation).

### Details

- Post-edit `resolve()` shape (pseudocode, matches plan's Design section exactly):
  ```
  def resolve(self, tool_name: str) -> str:
      if (key := self._lookup_runtime_registry(tool_name)) is not None:
          return key
      if self._strict_mode:
          self._raise_strict_error(tool_name)
      if self._warn_on_missing:
          logger.warning("... not found in RuntimeToolRegistry ...", tool_name)
      raise ValueError(f"Unknown tool: {tool_name!r}")
  ```
- `class ToolRouteResolver` docstring (currently "Map tool_name → server_key using RuntimeToolRegistry
  or ToolRegistry as routing authority... Resolution priority: 1. RuntimeToolRegistry ... 2.
  ToolRegistry (drift-detection fallback)") must also be corrected — this is inside the class body,
  not just the module docstring; do not miss it when editing.
- After this edit, `ToolRegistry` has zero remaining references anywhere in
  `scripts/shared/route_resolver.py` — verify with
  `grep -n 'ToolRegistry' scripts/shared/route_resolver.py` returning no output (aside from,
  transitively, none — `RuntimeToolRegistry` substring-matches `ToolRegistry` so use a word-boundary
  grep, e.g. `grep -nE '(?<!Runtime)ToolRegistry' scripts/shared/route_resolver.py` with a
  Perl-compatible grep, or eyeball the few remaining hits).
