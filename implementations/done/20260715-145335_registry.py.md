# Implementation Procedure: registry.py

## Goal

Remove the `_DbMixin` import and its composition from `CommandRegistry`'s base-class
tuple (since `cmd_db.py` is deleted), and correct the module docstring's mixin-list
comment so it no longer references `cmd_db.py` or the already-removed `/rag` command.

## Scope

### In scope
- Remove `from agent.commands.cmd_db import _DbMixin` (current line 41).
- Remove `_DbMixin` from `CommandRegistry`'s base-class tuple (current line 66).
- Update the module docstring's mixin list (current lines 10-24):
  - Drop the `cmd_db.py         — _DbMixin:         /db` line entirely.
  - Correct the `cmd_rag_export.py — _RagExportMixin:  /export, /compact, /rag` line
    (already stale before this plan — no `_cmd_rag` handler exists in the current
    codebase) to `cmd_rag_export.py — _RagExportMixin:  /compact` (dropping both the
    already-nonexistent `/rag` and the now-removed `/export`).
  - Update the `cmd_config.py     — _ConfigMixin:     /config, /stats, /set, /reload`
    line to drop `/set`: `cmd_config.py     — _ConfigMixin:     /config, /stats, /reload`.
  - Update the `cmd_session.py    — _SessionMixin:    /session commands` line's
    implicit scope is unchanged in wording (it already says "/session commands"
    generically) — no edit needed there, but note it now covers the additional
    subcommands.

### Out of scope
- `dispatch()`, `_get_handler()`, `_dispatch_plugin()`, `_cmd_help()` — logic
  unchanged; they already iterate `_COMMANDS` generically and require no per-command
  special-casing.
- The fail-fast `hasattr` validation loop in `__init__` (current lines 87-92) —
  unchanged; it will simply validate fewer `CommandDef` entries once
  `command_defs_list.py`'s `/set`/`/db`/`/export` entries are removed.
- Every other mixin's import/composition (`_SessionMixin`, `_McpMixin`, `_ConfigMixin`,
  `_ContextMixin`, `_ToolingMixin`, `_DebugMixin`, `_AuditMixin`, `_RagExportMixin`,
  `_MemoryMixin`, `_WorkflowMixin`, `_PluginsMixin`, `_MdqMixin`, `_SkillMixin`) —
  unchanged (only their internal method sets shrink, handled in their own files).

## Assumptions

- `_ConfigSetMixin` is composed via `_ConfigMixin` in `cmd_config.py`, **not** directly
  in `registry.py` (verified: `registry.py` imports only `_ConfigMixin` as a whole,
  never `_ConfigSetMixin` directly) — so no direct edit to `registry.py`'s import list
  or base-class tuple is needed for the `/set` removal beyond the docstring line
  correction above.
- Removing `_DbMixin` from the base-class tuple does not break MRO resolution for the
  remaining 13 mixins — verified by reading the full class body: `_DbMixin` sits at
  position 5 of 14 in the tuple with no other mixin depending on it (each mixin only
  extends `MixinBase`).

### Method

Targeted text edit: one import removal, one base-class-tuple entry removal, three

### Details

- This edit must land in the same change-set as `cmd_db.py`'s deletion — otherwise
  `registry.py` would import a module that no longer exists.
- After this edit, confirm `CommandRegistry.__init__`'s fail-fast loop
  (`for _cmd in _COMMANDS: if not hasattr(self, _cmd.handler): raise AttributeError(...)`)
  still passes — it will, because `command_defs_list.py`'s corresponding
  `CommandDef` entries (`/set`, `/db`) are removed in the same change-set, so no
  `CommandDef` references `_cmd_set` or `_cmd_db` anymore.
