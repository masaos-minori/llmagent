# Implementation: memory/store.py — ON CONFLICT DO UPDATE + remove broad exceptions

## Goal

1. Replace `INSERT OR REPLACE INTO memories` with `INSERT INTO ... ON CONFLICT(memory_id) DO UPDATE SET`.
2. Remove `except Exception` in `_write_vec()` — vec write failure propagates.
3. Remove `except Exception` in `clear_by_session()` and `search_by_type()`.

## Scope

- `scripts/agent/memory/store.py` — primary
- `tests/test_memory_store.py` — update tests that expect silent failure on vec error

## Assumptions

1. `ON CONFLICT DO UPDATE` requires `memory_id TEXT PRIMARY KEY` constraint, which exists in the schema.
2. `_write_vec()` is only called when `embedding is not None`. If memories_vec table is absent,
   the exception propagates and signals a configuration error (acceptable fail-fast behaviour).
3. `add()` and `upsert()` share a transaction; if vec write fails, the whole transaction rolls back.
4. The FTS sync (`delete + insert`) inside `upsert()` is retained but its exception propagates.

```python
def _write_vec(self, db, memory_id: str, embedding: list[float]) -> None:
    db.execute(
        "INSERT OR REPLACE INTO memories_vec(memory_id, embedding) VALUES (?,?)",
        (memory_id, _floats_to_blob(embedding, self._embed_dim)),
    )
```
