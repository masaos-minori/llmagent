## Goal
Increase HTTP health check request timeout from 2 seconds to 5 seconds and upgrade timeout log level from DEBUG to INFO in http_lifecycle.py.

## Scope
**In**: 
- Change `timeout=2.0` → `timeout=httpx.Timeout(connect=5.0, read=5.0)` in `scripts/agent/http_lifecycle.py` line ~249
- Change `logger.debug(...)` → `logger.info(...)` for timeout exceptions at line ~278

**Out**:
- No changes to overall startup timeout (`cfg.startup_timeout_sec`)
- No changes to polling interval
- No test modifications beyond adding new tests

## Assumptions
1. The `httpx` module is already imported (confirmed at line 23)
2. `logger` variable exists in http_lifecycle.py (confirmed at line 28)
3. A 5-second timeout is sufficient for most MCP servers to start (require doc suggests 5–10 seconds)

## Implementation

### Target file
- `scripts/agent/http_lifecycle.py`

### Procedure
1. Locate the AsyncClient instantiation around line 249
2. Replace `timeout=2.0` with `timeout=httpx.Timeout(connect=5.0, read=5.0)`
3. Locate the timeout exception logging around line 278
4. Change `logger.debug(...)` to `logger.info(...)` for timeout exceptions
5. Run lint/typecheck to confirm no regressions

### Method
Use `httpx.Timeout(connect=5.0, read=5.0)` instead of bare float `2.0`. This creates a Timeout object with explicit connect/read timeouts rather than a single blanket timeout.

### Details

```python
# In scripts/agent/http_lifecycle.py, replace line 249:

# Before:
    async with httpx.AsyncClient(timeout=2.0) as client:

# After:
    async with httpx.AsyncClient(timeout=httpx.Timeout(connect=5.0, read=5.0)) as client:
```

```python
# In scripts/agent/http_lifecycle.py, replace approximately line 278:

# Before:
                    except (httpx.HTTPError, OSError) as e:
                        logger.debug(
                            "Lifecycle: health-check poll %r: %s", server_key, e

# After:
                    except (httpx.HTTPError, OSError) as e:
                        logger.info(
                            "Lifecycle: health-check poll %r: %s", server_key, e
```

Key points:
- Use `httpx.Timeout()` (fully qualified) since `httpx.Timeout` is not directly imported
- `connect=5.0` controls time to establish TCP connection
- `read=5.0` controls time to receive first byte of response
- Upgrade from DEBUG to INFO because timeout during startup is an operator-relevant event
- Only timeout-related debug messages are upgraded (not all poll failure messages)
