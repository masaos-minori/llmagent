# Implementation: Add errors field to HealthCheckResult (req 25 — Step 1)

## Goal

Extend `HealthCheckResult` with an `errors: list[ServiceWarning]` field to distinguish critical failures from warnings.

## Scope

- `scripts/agent/shared/health_models.py`

## Assumptions

1. `errors` defaults to empty list — backward-compatible.
2. `has_issues` returns True when either `warnings` or `errors` is non-empty.
3. `warning_messages()` returns only warning messages; error messages are handled separately by callers.
4. The dataclass remains `frozen=True`.

## Implementation

### Target file

`scripts/agent/shared/health_models.py`

### Procedure

Add `errors` field after `warnings`, update `has_issues`, add `error_messages()` helper.

### Method

Edit tool.

### Details

```python
@dataclass(frozen=True)
class HealthCheckResult:
    warnings: list[ServiceWarning] = field(default_factory=list)
    errors: list[ServiceWarning] = field(default_factory=list)

    @property
    def has_issues(self) -> bool:
        return bool(self.warnings or self.errors)

    def warning_messages(self) -> list[str]:
        return [w.message for w in self.warnings]

    def error_messages(self) -> list[str]:
        return [e.message for e in self.errors]
```
