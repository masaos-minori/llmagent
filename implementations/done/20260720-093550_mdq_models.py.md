## Goal

Add per-tool `TypedDict` metadata types to `mdq_models.py` so that `search.py`, `indexer.py`,
`db_grep.py`, and `mdq_service.py` can return exact, structured execution metadata
(`result_count`, `indexed_count`, `skipped_count`, `deleted_count`, `failed_count`,
`match_count`, `truncated`, `duration_ms`, etc.) instead of `mdq_server.py::call_tool()` inferring
these values by regex/substring-parsing the user-facing formatted text.

## Scope

**In scope:**
- Add `SearchDocsMetadata(TypedDict)` — fields: `query_preview: str`, `result_count: int`,
  `shown_count: int`, `truncated: bool`, `total_count: int`, `duration_ms: float`.
- Add `IndexPathsMetadata(TypedDict)` — fields: `input_path_count: int`, `indexed_count: int`,
  `skipped_count: int`, `failed_count: int`, `duration_ms: float`.
- Add `GrepDocsMetadata(TypedDict)` — fields: `pattern_preview: str`, `path_filter_count: int`,
  `match_count: int`, `truncated: bool`, `grep_enabled: bool`.
- Do **not** add a new type for `refresh_index` — reuse the existing `RefreshSummary` TypedDict
  already defined in `indexer.py` (per plan's explicit design choice; see Design section
  "reuse `RefreshSummary` directly ... rather than duplicating it").

**Out of scope:**
- `MdqConfig` (covered by `20260720-091213_mdq_models.py.md`).
- The stale-detection helper and `SearchResultResult`'s `matched_count`/`shown_count` rename
  (covered by `20260720-091958_mdq_models.py.md`) — this document's `SearchDocsMetadata` is a
  **separate** type consumed by `mdq_service.py`/`mdq_server.py` for audit purposes, not a
  replacement for `SearchResultResult` (the internal FTS5 row-fetch wrapper used inside
  `search.py`). The two may end up with similarly-named fields (`shown_count`) by coincidence of
  vocabulary; they are different types with different consumers — no code sharing/inheritance is
  introduced between them.
- Any change to existing TypedDicts/Pydantic models already in the file (`SearchResultItem`,
  `GrepDocMatch`, `RefreshIndexResponse`, exception classes, etc.).

## Implementation

### Target file

`scripts/mcp_servers/mdq/mdq_models.py`

### Procedure

1. Add the three new `TypedDict` classes near the existing `SearchResultResult` definition
   (end of file, after line 248), keeping the file's existing grouping style (domain
   TypedDicts together, near their nearest-related existing type).
2. No changes to `__all__` or imports needed — no other file currently imports `TypedDict`

### Method

no new imports required beyond `TypedDict` (already imported at line 9 for `ParsedSection`/etc.).

### Details

```python
class SearchDocsMetadata(TypedDict):
    query_preview: str
    result_count: int       # exact match count before truncation
    shown_count: int         # results actually included in the formatted text
    truncated: bool
    total_count: int         # kept distinct from result_count per requirement wording
    duration_ms: float


class IndexPathsMetadata(TypedDict):
    input_path_count: int
    indexed_count: int
    skipped_count: int
    failed_count: int
    duration_ms: float


class GrepDocsMetadata(TypedDict):
    pattern_preview: str
    path_filter_count: int
    match_count: int
    truncated: bool
    grep_enabled: bool
```

- **Interaction with sibling doc `20260720-092134_search.py.md`:** that document renames
  `SearchResultResult.total` to `matched_count`/`shown_count`. If both land, `search.py`'s
  producer of `SearchDocsMetadata.result_count`/`total_count` should read from
  `SearchResultResult["matched_count"]` (the renamed field) rather than a now-removed `total`
  key — this is a call-site detail for the companion `search.py` doc to handle, not a reason to
  change this document's `SearchDocsMetadata` field names (the audit-facing field names
  `result_count`/`total_count` are independent of whatever internal field name
  `_search_docs_structured()` happens to use).
- `query_preview`/`pattern_preview` are plain truncated strings (e.g. `query[:80]`), matching the
  existing truncation convention already used in `audit_target.py` (`str(...)[:80]`) — not a new
  convention.
