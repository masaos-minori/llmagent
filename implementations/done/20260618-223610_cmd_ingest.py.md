## Proposed Changes

### 1. Add `on_status` callback to `/ingest`

```python
async def _cmd_ingest(self, args):
    svc = IngestWorkflowService()
    
    # Progress callback
    def on_status(msg: str) -> None:
        self._out.write(f"  [ingest] {msg}")
    
    try:
        result = await svc.run(target, lang=lang, snippets_only=snippets_only)
        for msg in result.messages:
            self._out.write(f"  {msg}")
    except IngestStageError as e:
        logger.error("Ingest failed at stage=%s: %s", e.stage, e.detail)
        self._out.write(f"  [ingest] error ({e.stage}): {e.detail}")
```

**Note:** `IngestWorkflowService` currently does NOT accept an `on_status` callback. The plan says "wire `on_status` callbacks" — this requires adding the parameter to `IngestWorkflowService.run()`.

### 2. Show fallback reasons in ingestion output (if applicable)

Currently `/ingest` does not interact with RAG pipeline diagnostics. The fallback reason display is for `/rag search --debug`, handled separately.
