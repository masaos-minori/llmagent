# Implementation: Simplify web-search-mcp to DuckDuckGo Only

## Goal

Remove Brave and Bing provider code from `web-search-mcp` so that DuckDuckGo is the only search provider, eliminating API key handling, fallback logic, and multi-provider config complexity.

## Scope

**In-scope:**
- Remove `_search_brave()` and `_search_bing()` from `server.py`
- Remove `BRAVE_API_KEY` / `BING_API_KEY` env var reads and startup warnings from `server.py`
- Remove `_PROVIDER_FUNCS` dict and `_try_provider()` fallback loop from `server.py`
- Inline DuckDuckGo call directly in the `search()` endpoint
- Simplify `/health` to remove `brave_api_key` / `bing_api_key` dependency checks
- Remove `brave_search_url`, `bing_search_url`, `search_providers` fields from `WebSearchConfig`
- Remove `http_timeout` from `WebSearchConfig` and config TOML (no httpx calls remain)
- Remove `httpx` and `os` imports from `server.py`
- Remove `brave_search_url`, `bing_search_url`, `search_providers`, `http_timeout` keys from `config/web_search_mcp_server.toml`
- Update `docs/04_mcp_04_server_catalog.md` — Purpose, Provider fallback section, Config table, Health example
- Update `tests/test_web_search_models.py` — remove assertions referencing removed fields

**Out-of-scope:**
- Renaming the `search_web` MCP tool
- Replacing DuckDuckGo with another provider
- Changing `SearchResult.provider` or `SearchResponse.provider` schema fields (keep for informational value)
- Modifying `deploy/deploy.sh`

## Assumptions

1. `duckduckgo_search.DDGS` is already installed and available in the venv
2. `http_timeout` is unused after removing Brave/Bing httpx calls; DDGS uses its own internal timeout via `proxies`
3. `search_providers` config field exists solely to support multi-provider selection; removing it simplifies the config
4. No external service reads `BRAVE_API_KEY` or `BING_API_KEY` for web-search-mcp specifically

### Phase 2: `scripts/mcp/web_search/server.py`

**Procedure:** Remove Brave/Bing code and simplify search/health endpoints.

**Method:** Delete functions, env vars, imports; rewrite search() and health().

**Details:**

1. **Remove imports:**
```python

```

2. **Remove env var reads:**

3. **Remove functions:**
- `_search_brave()` — entire function
- `_search_bing()` — entire function
- `_PROVIDER_FUNCS` dict
- `_try_provider()` — entire function

4. **Rewrite `search()` endpoint:**
```python
async def search(req: SearchRequest) -> SearchResponse:
    """Search the web using DuckDuckGo."""
    try:
        results = await asyncio.to_thread(_search_duckduckgo, req.query, req.max_results)
    except Exception as e:
        raise WebSearchUpstreamError(f"DuckDuckGo search failed: {e}") from e

    if not results:
        raise WebSearchUpstreamError("No results returned from DuckDuckGo")

    return SearchResponse(
        query=req.query,
        results=results,
        provider="duckduckgo",
    )
```

5. **Rewrite `health()` endpoint:**
```python
async def health() -> dict:
    """Health check endpoint."""
    return {"status": "ok", "ready": True}
```

6. **Update docstring** — remove provider list from module docstring, keep only DuckDuckGo reference.
