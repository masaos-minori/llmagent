## Goal
Change `TaskRecord.workflow_id` type from `str | None` to `str` (required); add `workflow_id: str` field to `ApprovalRecord`.

## Scope
**In**: `scripts/agent/workflow/models.py` — change `TaskRecord.workflow_id` type; add `workflow_id: str` to `ApprovalRecord`.
**Out**: Other model fields; model serialization/deserialization.

## Assumptions
- `TaskRecord` is a dataclass in `models.py`; `workflow_id: str | None` is the current type.
- `ApprovalRecord` is at line 25 of `models.py`; has no `workflow_id` field currently.
- Changing `TaskRecord.workflow_id` to `str` will cause mypy errors at all construction sites where `workflow_id=None` is passed; those sites are fixed in their respective docs.
- `ApprovalRecord` field order: append `workflow_id: str` at the end to avoid constructor position changes.

## Implementation

**Target file**: `scripts/agent/workflow/models.py`

**Procedure**:

1. **Change `TaskRecord.workflow_id`**:
   ```python
   # Before:
   @dataclass
   class TaskRecord:
       task_id: str
       session_id: str
       workflow_id: str | None
       ...

   # After:
   @dataclass
   class TaskRecord:
       task_id: str
       session_id: str
       workflow_id: str  # required — see req09
       ...
   ```

2. **Add `workflow_id` to `ApprovalRecord`**:
   ```python
   # Before:
   @dataclass
   class ApprovalRecord:
       approval_id: str
       task_id: str
       status: str
       ...

   # After:
   @dataclass
   class ApprovalRecord:
       approval_id: str
       task_id: str
       status: str
       ...
       workflow_id: str = ""  # populated from approvals.workflow_id column after req11
   ```
   Note: use default `""` temporarily until all construction sites are updated; switch to required `str` after `approval_ops.py` update (see approval_ops.py doc).

3. **Run mypy as change inventory**:
   - `uv run mypy scripts/` — collect all errors from `TaskRecord.workflow_id` type change
   - Use error list to identify all construction sites that need `workflow_id: str` passed

**Method**: Dataclass field type change; mypy-driven propagation.

**Details**:
- Priority: change `TaskRecord` first (Plan 09 dependency), then `ApprovalRecord` (Plan 11 dependency).
- Do NOT add `workflow_id` to `TaskRecord.__post_init__` validation — let mypy enforce at construction.
- If `TaskRecord` is used in `from_row()` methods, update the row mapping to read `row["workflow_id"]` instead of `row.get("workflow_id")`.

## Validation plan
- `uv run mypy scripts/agent/workflow/models.py` — no errors in the file itself
- `uv run mypy scripts/` — error list shows only legitimate construction sites; fix each
- `uv run pytest tests/ -k "models or task_record or approval_record" -x -q`
