# Implementation: Phase A1 — logging f-string → %s 形式変換

## Goal

`scripts/` 全体の 174 件の `logger.*(f"...")` 呼び出しを
`logger.*("...", arg1, arg2)` の lazy-format 形式に変換し、
G004 エラーをゼロにする。

## Scope

**In**: `scripts/` 以下の全 `.py` ファイル（174 エラー箇所）
**Out**: `tests/` ファイル、docstring 内の文字列、コメント

## Assumptions

1. ruff G004 は `--fix` / `--unsafe-fixes` では自動修正不可。手動変換が必要。
2. 変換ルール:
   - `logger.info(f"msg {x}")` → `logger.info("msg %s", x)`
   - `logger.info(f"msg {x!r}")` → `logger.info("msg %r", x)`
   - 複数変数: `f"a={a} b={b}"` → `"a=%s b=%s", a, b`
   - 式が複雑な場合（三項演算子・メソッド呼び出し等）は変換後に手動確認が必要
3. コメント・docstring 内の f-string は対象外（logger 呼び出しのみ）。

## Implementation

### Target files

`scripts/` 以下 全ファイル（`ruff check scripts/ --select=G004` でリストアップ）

### Procedure

1. `uv run ruff check scripts/ --select=G004 --output-format=json` で全箇所を特定
2. ファイルごとに Edit ツールで変換（または `sed` スクリプト）
3. 変換後に `uv run ruff check scripts/ --select=G004` で残存 0 を確認

### Method

単純パターン（1変数・2変数）は `sed -E` で一括変換可能:

```bash
# 1変数: logger.xxx(f"... {expr}")  →  logger.xxx("... %s", expr)
# ただし sed では式の分離が難しいため、手動変換が主体

# 手動変換の指針:
# f"text {var}"          → "text %s", var
# f"text {var!r}"        → "text %r", var
# f"text {var:.2f}"      → "text %.2f", var
# f"text {a} {b}"        → "text %s %s", a, b
# f"text {obj.attr}"     → "text %s", obj.attr
# f"text {func()}"       → "text %s", func()
```

### Details

主要な変換パターン（`ruff check --select=G004` で確認済みの形式）:

```python
# 変換前
logger.info(f"Tool cache hit: {tool_name} (age=%.0fs)")
logger.warning(f"Partial LLM completion saved: {e.kind}")
logger.error(f"LLM transport error: {e.kind} status={e.status_code}")

# 変換後
logger.info("Tool cache hit: %s (age=%.0fs)", tool_name)
logger.warning("Partial LLM completion saved: %s", e.kind)
logger.error("LLM transport error: %s status=%s", e.kind, e.status_code)
```

## Validation plan

1. `uv run ruff check scripts/ --select=G004` → 0 errors
2. `uv run mypy scripts/` → no new errors
3. `uv run pytest -q` → no regressions
