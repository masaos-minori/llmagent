# Implementation Procedure: Make memory retrieval degrade to FTS-only when vector KNN fails

## Goal

When vector KNN search fails in `HybridRetriever.search()`, fall back to FTS-only results instead of propagating the error. This ensures memory injection continues working when vector tables are unavailable.

## Scope

- `scripts/agent/memory/retriever.py` — add exception handling around vector KNN call in `HybridRetriever.search()`
- `tests/test_memory_retriever.py` — add tests for vector failure fallback

## Assumptions

- `HybridRetriever` already has `last_retrieval_mode` and `fts_fallback_count` attributes defined (verified at lines 209-210 of retriever.py)
- `fts_hits` variable exists in `search()` method body before the vector KNN call (verified at line 226)
- `sqlite3.OperationalError` is the primary exception to catch; raised by `VectorRetriever.knn_search()` when memories_vec table is missing (documented at line 157)
- Direct `HybridRetriever.knn_search()` should remain fail-fast (no changes needed there)

## Implementation

### Target file

`scripts/agent/memory/retriever.py`

### Procedure

1. Locate the vector KNN call in `HybridRetriever.search()` at line 233:
   ```python
   vec_hits = self._vec.knn_search(...)
   ```

2. Wrap this call with `try/except sqlite3.OperationalError`:
   - Move the existing `if not vec_hits:` check inside the `try` block's else clause
   - On `sqlite3.OperationalError`, log warning and return `fts_hits`

3. Update the method structure as follows:
   ```python
   fts_hits = self._fts.search(query, project, repo, branch)
   if embedding is None:
       logger.info("retrieval: fts_only (reason=embedding_disabled_or_none)")
       self.last_retrieval_mode = "fts_only"
       self.fts_fallback_count += 1
       return fts_hits

   try:
       vec_hits = self._vec.knn_search(
           embedding, query.memory_type, self._fts.candidate_limit, branch
       )
   except sqlite3.OperationalError as e:
       logger.warning("Vector KNN failed, falling back to FTS-only: %s", e)
       self.last_retrieval_mode = "fts_only"
       self.fts_fallback_count += 1
       return fts_hits

   if not vec_hits:
       logger.info("retrieval: fts_only (reason=vec_returned_empty)")
       self.last_retrieval_mode = "fts_only"
       self.fts_fallback_count += 1
       return fts_hits

   self.last_retrieval_mode = "hybrid"
   merged = rrf_merge([fts_hits, vec_hits], k=self._rrf_k)
   result: list[Any] = merged[: query.limit]
   return result
   ```

### Method

Add `import sqlite3` at the top of the file (already imported in test file but not in retriever.py). Check current imports section (lines 15-28) — `sqlite3` is NOT currently imported in retriever.py.

### Details

- Only catch `sqlite3.OperationalError` initially (narrow scope per requirements)
- Log warning with exception class and message using `logger.warning()`
- Update both `last_retrieval_mode` and `fts_fallback_count` on fallback
- Return raw `fts_hits` without merging
- The existing `if not vec_hits:` check remains unchanged after the try/except block — it handles the case where knn_search succeeds but returns empty results (different from a failure)
