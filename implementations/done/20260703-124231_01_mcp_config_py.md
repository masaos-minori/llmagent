## Goal

Remove `TransportType.STDIO` and `StartupMode.ONDEMAND` from `scripts/shared/mcp_config.py`, and fix `_build_single_server()` to extract and pass `cmd` and `env` fields to `McpServerConfig`.

## Scope

- **In-Scope:**
  - Delete `STDIO = "stdio"` from `TransportType` (line 22)
  - Delete `ONDEMAND = "ondemand"` from `StartupMode` (line 30)
  - In `_build_single_server()` (lines 171â€“204), extract `cmd = list(v.get("cmd", []))` and `env = dict(v.get("env", {}))` and pass them to `McpServerConfig(...)`
- **Out-of-Scope:**
  - Any other enum members (`HTTP`, `PERSISTENT`, `SUBPROCESS`, `HealthcheckMode`, `SecurityProfile`)
  - `McpServerConfig` field definitions or `_validate_cross_fields` logic (already correct)
  - `McpServerHealthRegistry` and related classes
  - Actual subprocess launch logic

## Assumptions

1. No production caller outside `scripts/` constructs `McpServerConfig` with `TransportType.STDIO` or `StartupMode.ONDEMAND`; `config/mcp_servers.toml` uses only `transport="http"` and `startup_mode="persistent"`.
2. `StrEnum` lookup will raise `ValueError` automatically when a removed member's string value is passed to `TransportType("stdio")` or `StartupMode("ondemand")`, so `_validate_enum_types` in `McpServerConfig.__post_init__` will still reject those strings via the `isinstance` check.
3. The existing tests `test_stdio_transport_rejected` (line 107) and `test_ondemand_startup_mode_rejected` (line 112) pass bare strings (not enum members), so removing the enum members makes those tests pass without changing the test logic.
4. `McpServerConfig` already defines `cmd: list[str]` and `env: dict[str, str]` with `field(default_factory=...)` (lines 61â€“64); `_validate_cross_fields` already enforces `cmd` non-empty for `SUBPROCESS` mode (line 89).
5. Explicit `list()` and `dict()` casts on the raw TOML values are sufficient type coercion; `McpServerConfig.__post_init__` validates the fields after construction.

- `dict(v.get("env", {}))` pattern: provides a copy of the TOML dict, ensuring mutation safety.
