# Goal

Replace all three `except Exception` clauses in `create_schema.py` with specific
exception types (`sqlite3.OperationalError`/`sqlite3.DatabaseError`/`OSError`).

# Scope

- `scripts/db/create_schema.py`

# Assumptions

1. DDL execution (`executescript`) can raise `sqlite3.OperationalError` (e.g. syntax
   error, locked DB) or `sqlite3.DatabaseError` (generic DB failure). These are the
   only expected failures from schema creation.
2. The `if __name__ == "__main__"` block wraps `create_schema()` which can raise
   `RuntimeError` (from helper config load) or `sqlite3.DatabaseError`.
3. After narrowing, unexpected exceptions propagate to the caller — which is the
   correct fail-fast behavior.

# Implementation

## Target file

`scripts/db/create_schema.py`

## Procedure

1. `create_rag_schema()` inner except (line 172):
   ```python
   # Before
   except Exception as e:
       logger.error(f"Failed to execute RAG schema DDL: {e}")
       raise

   # After
   except (sqlite3.OperationalError, sqlite3.DatabaseError) as e:
       logger.error(f"Failed to execute RAG schema DDL: {e}")
       raise
   ```

2. `create_session_schema()` inner except (line 184): same change.

3. `if __name__ == "__main__"` except (line 200):
   ```python
   # Before
   except Exception as e:
       logger.exception(f"Schema creation failed: {e}")
       sys.exit(1)

   # After
   except (RuntimeError, sqlite3.OperationalError, sqlite3.DatabaseError, OSError) as e:
       logger.exception(f"Schema creation failed: {e}")
       sys.exit(1)
   ```

4. Run ruff + mypy.

## Method

Three one-line exception type changes. No logic changes.
