## Goal

Port `BrowserService.fetch()`/`_check_domain()`/`_extract_text()`/`_truncate()` from
`scripts/mcp_servers/browser/browser_service.py` (162 lines, read in full) into a new
`fetch_browser()` async function in `search_provider.py`, preserving the domain-allowlist,
IP-literal-rejection, and byte-safe-truncation logic exactly (verbatim porting per plan Risk
mitigation — no rewrite).

## Scope

**In scope**: add `fetch_browser()` plus its private helpers to
`scripts/mcp_servers/web_search/search_provider.py` (current: 69 lines, one function
`search_duckduckgo`).
**Out of scope**: `search_duckduckgo()` itself — untouched. Health/metrics recording — stays in
`service.py` (this module remains a pure I/O/parsing layer with no `health.py`/`metrics.py` import,
matching `search_duckduckgo`'s existing shape, which also does not touch health/metrics).

## Assumptions

1. Source `BrowserService.__init__` takes a `BrowserConfig` and stores 3 scalars
   (`_allowed_domains: set[str]`, `_max_response_kb`, `_timeout_sec`). Since `search_provider.py`
   currently has no class, only free functions, `fetch_browser()` is ported as a **free async
   function** taking the same scalars as parameters (or a `BrowserConfig`-shaped object, per the
   `web_search_models.py` doc's chosen `BrowserConfig` view) — not re-instantiated as a class. This
   matches `search_duckduckgo(query, max_results, search_timeout_sec)`'s existing free-function
   style in this module.
2. `_check_domain`'s IP-literal check (loopback/link-local/private/reserved/multicast, checked via
   `ipaddress.ip_address()` unconditionally before the allowlist check) must be preserved exactly,
   including the fail-closed "empty allowlist denies everything" behavior (`hostname not in
   self._allowed_domains` with no special-case for an empty set).
3. `_truncate`'s byte-slice-before-decode approach (`text.encode("utf-8")[:max_bytes].decode(...,
   errors="replace")`) must be preserved exactly — this is the specific logic the plan's Risk
   section calls out as security/correctness-sensitive (avoids splitting a multibyte UTF-8 char).
4. New third-party imports needed in this module: `ipaddress` (stdlib), `httpx` (already a project
   dependency per `rules/coding.md`'s "use httpx not requests"), `bs4.BeautifulSoup` (already a
   dependency, used by the source `browser_service.py`), `urllib.parse.urlsplit` (stdlib).

## Implementation

### Target file

`scripts/mcp_servers/web_search/search_provider.py`

### Procedure

1. Add imports: `ipaddress`, `from urllib.parse import urlsplit`, `import httpx`,
   `from bs4 import BeautifulSoup`, and
   `from mcp_servers.web_search.web_search_models import BrowserAuthorizationError,
   BrowserConfig, BrowserFetchRequest, BrowserFetchResponse, BrowserValidationError` (alongside the
   existing `SearchResult`/`WebSearch*Error` import block).
2. Add a module-private `_check_domain(url: str, allowed_domains: set[str]) -> str` function,
   ported verbatim from `BrowserService._check_domain` (source lines 44-77), taking
   `allowed_domains` as an explicit parameter instead of `self._allowed_domains`.
3. Add `_extract_text(html: str) -> str` and `_truncate(text: str, max_kb: int) -> tuple[str,
   bool]`, ported verbatim (source lines 108-126).
4. Add the public async entry point:
   ```
   async def fetch_browser(req: BrowserFetchRequest, cfg: BrowserConfig) -> BrowserFetchResponse:
       _check_domain(req.url, set(cfg.allowed_domains))
       max_response_kb = min(req.max_response_kb or cfg.max_response_kb, cfg.max_response_kb)
       async with httpx.AsyncClient(timeout=cfg.timeout_sec) as client:
           resp = await client.get(req.url)
       text = _extract_text(resp.text)
       text, truncated = _truncate(text, max_response_kb)
       return BrowserFetchResponse(text=text, truncated=truncated, url=req.url,
                                    status_code=resp.status_code,
                                    elapsed_sec=resp.elapsed.total_seconds())
   ```
   (Ported from `BrowserService.fetch`, source lines 79-106, with `self._allowed_domains`/
   `self._max_response_kb`/`self._timeout_sec` replaced by `cfg.*` field reads.)

### Method

Free-function port, no class introduced — matches this module's existing style. No new
`shared/` dependency; both `httpx` and `bs4` are already used elsewhere in the codebase per
`rules/coding.md`. `search_duckduckgo`'s existing `WebSearchTimeoutError`/`WebSearchNetworkError`/
`WebSearchProviderError` exception-classification pattern is **not** reused here — `fetch_browser`
raises `BrowserAuthorizationError`/`BrowserValidationError` from `_check_domain`, and lets any
`httpx` exception propagate un-classified to `service.fetch_browser` (per UNK-01's resolution, the
service layer — not this provider layer — owns classification/health/metrics for consistency with
`search_web`'s split).

### Details

- `_check_domain` raises `BrowserValidationError` for bad scheme/missing hostname,
  `BrowserAuthorizationError` for IP-literal or non-allowlisted hostname — order of checks
  (scheme → hostname-present → IP-literal → allowlist) must match the source exactly, since a
  malformed-URL case (scheme/hostname) must surface as 422, not 403.
- No caller-supplied timeout override exists in the source (`_timeout_sec` is server-configured
  only) — do not add one; `req.max_response_kb` is the only caller-tunable field, matching
  `BrowserFetchRequest`'s existing shape.
