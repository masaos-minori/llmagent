## Goal

Promote workflow engine to the primary execution path in `Orchestrator.handle_turn()` by removing the fallback-to-direct-execution paths when `workflow_def` is loaded and `StateStore` construction fails.

## Scope

- Remove the `if self._workflow_def is None:` fallback block (lines ~114-117)
- Remove the `try/except RuntimeError` around `StateStore()` direct-execution fallback (lines ~120-127)
- Keep `if self._workflow_mode == "disabled":` opt-out path (lines ~108-112)
- Keep `self._log_fallback()` behavior for `workflow_mode="required"` error raising
- No new public API changes

## Assumptions

- When `workflow_def` fails to load at startup, `_workflow_def` is already `None` and `_log_fallback()` is called
- The `workflow_mode="disabled"` path must remain as the only explicit opt-out
- `StateStore()` construction failure should raise rather than silently fall back, when `workflow_mode != "disabled"`

## Implementation

### Target file

`scripts/agent/orchestrator.py`

### Procedure

1. Remove the `if self._workflow_def is None:` block that falls back to `_process_turn()` direct call
2. Remove the `store: StateStore | None = None` + `try/except RuntimeError` block that falls back to `_process_turn()` on StateStore failure
3. Replace with: `store = StateStore()` (direct construction, no try/except for fallback)
4. Update `finally: store.close()` to handle the case where `store` was never assigned (use `if store is not None: store.close()` or restructure with a context manager)

### Method

Direct edit.

### Details

The target structure after change:

```python
async def handle_turn(self, line: str) -> None:
    ctx = self._ctx
    turn_started_at = time.perf_counter()

    await self._handle_turn_start(line)

    answer: str = ""
    error_kind: str | None = None

    if self._workflow_mode == "disabled":
        logger.info("Workflow mode=disabled — direct execution")
        answer, error_kind = await self._process_turn(line, ctx, turn_started_at)
        await self._handle_turn_end(line, answer, turn_started_at, error_kind)
        return

    if self._workflow_def is None:
        self._log_fallback("workflow definition not loaded")
        answer, error_kind = await self._process_turn(line, ctx, turn_started_at)
        await self._handle_turn_end(line, answer, turn_started_at, error_kind)
        return

    store = StateStore()
    try:
        session_id = (
            str(ctx.session.session_id) if ctx.session.session_id else "none"
        )
        task = store.create_task(
            session_id=session_id,
            turn_number=ctx.stats.stat_turns,
            workflow_version=self._workflow_def.version,
        )
        audit_workflow_start(ctx, task.task_id, self._workflow_def.version)
        engine = WorkflowEngine(self._workflow_def, store, tracer=self._tracer)

        # ... plan_fn, execute_fn, verify_fn closures ...

        await engine.run(task, plan_fn, execute_fn, verify_fn)
    except WorkflowPendingApprovalError as exc:
        # ... existing handler ...
    except WorkflowHaltError as exc:
        # ... existing handler ...
    finally:
        store.close()
```

Key change: the `store: StateStore | None = None` guard and its `try/except RuntimeError` fallback are removed. `store = StateStore()` is now called unconditionally (when not disabled). The `finally: store.close()` no longer needs a None-check since `store` is always assigned before the try block.
