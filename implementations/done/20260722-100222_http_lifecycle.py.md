## Goal
Standardize HTTP timeout parameter usage across the codebase by using `httpx.Timeout(timeout=X)` consistently, fixing the broken `connect`/`read` only pattern in `http_lifecycle.py`.

## Scope
**In**: `scripts/agent/services/mcp_tool_discovery.py`, `scripts/agent/http_lifecycle.py`, `scripts/agent/services/mcp_status.py` — unify timeout parameter format
**Out**: Any changes to timeout values themselves, or changes to other HTTP clients outside these three files

## Assumptions
1. `httpx.Timeout(timeout=X)` provides equivalent functionality to `timeout=X` but is more explicit
2. The `httpx.Timeout(connect=5.0, read=5.0)` pattern is indeed broken per httpx API requirements
3. A shared constant would improve maintainability

## Implementation

### Target file
`scripts/agent/services/mcp_tool_discovery.py`, `scripts/agent/http_lifecycle.py`, `scripts/agent/services/mcp_status.py`

### Procedure
1. Identify all locations where HTTP timeout parameters are used inconsistently
2. Create a shared constant for the timeout value
3. Replace bare float timeouts with `httpx.Timeout(timeout=X)` pattern
4. Update both files to use the shared constant

### Method
Create a module-level constant and use `httpx.Timeout(timeout=X)` consistently across all files.

### Details
Current code locations:

1. `scripts/agent/http_lifecycle.py` line 249:
```python
async with httpx.AsyncClient(timeout=httpx.Timeout(timeout=5.0)) as client:
```
Already correct — no change needed.

2. `scripts/agent/services/mcp_tool_discovery.py` line 124:
```python
resp = await self._ctx.services_required.http.get(
    f"{cfg.url}/v1/tools", timeout=5.0
)
```
Fix:
```python
resp = await self._ctx.services_required.http.get(
    f"{cfg.url}/v1/tools", timeout=httpx.Timeout(timeout=MCPSERVER_HEALTH_TIMEOUT)
)
```

3. `scripts/agent/services/mcp_status.py` line 63:
```python
async with httpx.AsyncClient(timeout=5.0) as probe:
```
Fix:
```python
async with httpx.AsyncClient(timeout=httpx.Timeout(timeout=MCPSERVER_HEALTH_TIMEOUT)) as probe:
```

Add shared constant in `http_lifecycle.py` module level:
```python
MCPSERVER_HEALTH_TIMEOUT: float = 5.0
```

Then import in both affected files:
```python
from scripts.agent.http_lifecycle import MCPSERVER_HEALTH_TIMEOUT
```

Key points:
- Define constant in `http_lifecycle.py` since it's the primary consumer of this timeout value
- Import in `mcp_tool_discovery.py` and `mcp_status.py`
- Use `httpx.Timeout(timeout=MCPSERVER_HEALTH_TIMEOUT)` instead of bare float
- Verify layer contract: `services/` can import from `agent/` (both are peer layers under `scripts/`)
