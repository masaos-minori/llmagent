# Implementation: Add SubcommandSpec entries for /db subcommands

## Goal

Populate the `subcommands` field on the `/db` `CommandDef` with `SubcommandSpec` entries so that programmatic help output and user-facing documentation can be derived from metadata rather than hardcoded strings.

## Scope

- `scripts/agent/commands/command_defs.py` — add `subcommands=` kwarg to the `/db` `CommandDef`

**Out of scope:** Help rendering changes (separate doc), output labeling changes (separate doc).

## Assumptions

1. `SubcommandSpec` already has `name` and `help` fields (confirmed — `command_defs.py:18-23`).
2. Adding `subcommands` does not break command dispatch — the registry already stores and ignores the field.
3. The command help strings are in `_build_usage_string()` in `registry.py`.

## Implementation

### Target file

`scripts/agent/commands/command_defs.py`

### Procedure

Add a `subcommands=` argument to the existing `CommandDef("/db", ...)` entry. Each `SubcommandSpec` must include a bracketed target-DB tag in the help text (e.g. `[rag]`, `[session]`).

### Method

Single edit to the `_COMMANDS` list.

### Details

**Current entry (line 141-146):**

```python
    CommandDef(
        "/db",
        True,
        False,
        "_cmd_db",
        "stats | urls [--lang ja|en] [--limit N] | clean <url> | rebuild-fts | health | checkpoint | vacuum | purge | recover",
    ),
```

**Desired entry:**

```python
    CommandDef(
        "/db",
        True,
        False,
        "_cmd_db",
        "stats | urls [--lang ja|en] [--limit N] | clean <url> | rebuild-fts | health | checkpoint | vacuum | purge | recover",
```
