# Implementation: H-4 — Update /context partial-completion display in cmd_context.py

## Goal

Change the `/context` command's partial-completion line to describe where the data is actually
stored (`session_diagnostics`) instead of the removed `tool_result` /
`tool_name='llm_partial_completion'` reference.

## Scope

**Target**: `scripts/agent/commands/cmd_context.py`

**Depends on**: `implementations/20260708-163427_llm_transport_errors.py.md` (display-string
consistency dependency, same rationale as the `repl.py` and `cmd_config_stats.py` docs).

**Out of scope**: the `if state.partial_completions > 0:` guard and the rest of `write_kv(...)`
key/value pairs in this method — unchanged.

## Assumptions

1. The string lives inside a `write_kv([...])` call, as the value paired with the
   `"Partial compl   "` key, inside `if state.partial_completions > 0:` — confirm exact line
   numbers with `grep -n "partial_completions" -A6 scripts/agent/commands/cmd_context.py` before
   editing.
2. No other file duplicates this exact string (verify via grep in Procedure Step 1).

## Implementation

### Target file

`scripts/agent/commands/cmd_context.py`

### Procedure

#### Step 1: Confirm single occurrence of the old string

```bash
grep -rn "stored as tool_result(tool_name='llm_partial_completion')" scripts/
```

Expect exactly one match, in this file.

#### Step 2: Replace the display string

Current:

```python
        if state.partial_completions > 0:
            self._out.write_kv(
                [
                    (
                        "Partial compl   ",
                        f"{state.partial_completions} stored as tool_result(tool_name='llm_partial_completion')",
                    ),
                ]
            )
```

Replace with:

```python
        if state.partial_completions > 0:
            self._out.write_kv(
                [
                    (
                        "Partial compl   ",
                        f"{state.partial_completions} (stored in session_diagnostics)",
                    ),
                ]
            )
```

### Method

- Single f-string content replacement inside the existing `write_kv([...])` call; no
  control-flow or key-label change (`"Partial compl   "` key stays identical, including its
  trailing spaces for column alignment).

### Details

- Preserve the exact trailing-space padding on the `"Partial compl   "` key label — it aligns
  with sibling keys in the same `write_kv` call elsewhere in this method (e.g.
  `"Compress limit  "`, `"Fallback trunc  "`).
- `state.partial_completions` (the value being displayed) is unaffected by the companion
  `llm_transport_errors.py` change — it reads `ctx.services_required.llm.stat_partial_completions`
  (or an equivalent state snapshot field), whose increment logic is preserved.
