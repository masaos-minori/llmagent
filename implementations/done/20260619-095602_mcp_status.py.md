# Implementation: Expose Serialization Stats in /mcp status

## Goal

Include serialization statistics in the `/mcp status` output so operators can see total serialization events and tools-affected counts without parsing logs.

## Scope

- `scripts/agent/services/mcp_status.py` — read `ctx.services.serialization_events` and `serialization_tools_affected`, include in status output
- `scripts/agent/commands/cmd_mcp.py` — if the status display function needs updating

Out of scope:
- Adding new HTTP endpoints
- Persisting stats across sessions

## Assumptions

1. `McpStatusService` already collects MCP server health status (confirmed at `cmd_mcp.py` line 18).
2. After Step 2, `ctx.services` has `serialization_events: int` and `serialization_tools_affected: int`.
3. The `/mcp status` output format is a text table or key-value display (read `mcp_status.py` to confirm format before implementing).
4. Adding serialization stats is a small additive change to the existing status output.

## Implementation

### Target file

`scripts/agent/services/mcp_status.py`

### Procedure

1. Read `mcp_status.py` to understand the current status collection and rendering pattern.
2. Add a section that reads `ctx.services.serialization_events` and `serialization_tools_affected`.
3. Include them in the output alongside existing server health stats.

### Method

Read `ctx.services` counter fields (set to 0 if not present via `getattr(..., 0)`). Append to existing status output string.

### Details

**mcp_status.py — in the status collection method (locate it first):**
```python
serial_events = getattr(ctx.services, "serialization_events", 0)
serial_affected = getattr(ctx.services, "serialization_tools_affected", 0)
```

**Output addition (format to match existing style):**
```
--- Tool Scheduling ---
Serialization events this session: 5
Tools affected by serialization:   12
```

Or if output is JSON-like:
```json
{
  "serialization": {
    "total_events": 5,
    "tools_affected": 12
  }
}
```

Match the format used by the rest of `/mcp status` output.

**cmd_mcp.py** — If `_cmd_mcp_status()` builds the display inline, add the serialization lines there. If `McpStatusService` builds a string, add it to the service method.
