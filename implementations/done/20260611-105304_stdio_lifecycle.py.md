# Implementation: agent/stdio_lifecycle.py — except Exception removal + LifecycleConfigurationError

## Goal

Replace the two `except Exception` blocks in `stdio_lifecycle.py` with specific
exception types, and raise `LifecycleConfigurationError` when `_start()` is called
with missing configuration, instead of silently setting FAILED state.

## Scope

**Target file**: `scripts/agent/stdio_lifecycle.py`

In scope:
- `_start()`: `cfg is None or not cfg.cmd` → raise `LifecycleConfigurationError`
- `_start()`: `except Exception as e:` → `except (ValueError, OSError) as e:`
- `_stop_stdio()`: `except Exception as e:` → `except OSError as e:`

Out of scope:
- `ensure_ready()` no-op behavior (intentional skip for non-stdio/non-ondemand configs)
- `restart()` callers

## Assumptions

1. `StdioTransport.start()` raises `ValueError` (bad working_dir) or `OSError` (bad cmd).
2. `StdioTransport.stop()` handles all exceptions internally; propagates only `OSError` in edge cases.
3. `cfg is None` in `_start()` should be an error because `restart()` calls `_start()` directly,
   bypassing the `ensure_ready()` guard.
4. `cfg.cmd` empty/missing is also a configuration error.

## Implementation

### Target file

`scripts/agent/stdio_lifecycle.py`

### Procedure

**Import addition** (top of file):
```python
from agent.tool_exceptions import LifecycleConfigurationError
```

**Change 1 — _start(): cfg.cmd missing → LifecycleConfigurationError**

Before:
```python
        cfg = self._server_configs.get(server_key)
        if cfg is None or not cfg.cmd:
            logger.warning(
                f"Lifecycle: cannot start {server_key!r}: no cmd configured",
            )
            self._handles[server_key] = TransportHandle(
                transport=None,
                state=LifecycleState.FAILED,
                last_error="no cmd configured",
            )
            return
```

After:
```python
        cfg = self._server_configs.get(server_key)
        if cfg is None:
            raise LifecycleConfigurationError(
                f"Lifecycle: no config for server {server_key!r}",
            )
        if not cfg.cmd:
            raise LifecycleConfigurationError(
                f"Lifecycle: cannot start {server_key!r}: no cmd configured",
            )
```

**Change 2 — _start(): except Exception → specific exceptions**

Before:
```python
        except Exception as e:
            logger.error(
                f"Lifecycle: failed to start ondemand server {server_key!r}: {e}",
            )
            self._handles[server_key] = TransportHandle(
                transport=None, state=LifecycleState.FAILED, last_error=str(e)
            )
```

After:
```python
        except (ValueError, OSError) as e:
            logger.error(
                f"Lifecycle: failed to start ondemand server {server_key!r}: {e}",
            )
            self._handles[server_key] = TransportHandle(
                transport=None, state=LifecycleState.FAILED, last_error=str(e)
            )
```

**Change 3 — _stop_stdio(): except Exception → OSError**

Before:
```python
        except Exception as e:
            logger.warning("Lifecycle: error stopping stdio %r: %s", key, e)
```

After:
```python
        except OSError as e:
            logger.warning("Lifecycle: error stopping stdio %r: %s", key, e)
```

### Method

`Edit` tool. Add import at file top.

### Details

- The `except (ValueError, OSError)` in `_start()` covers both misconfiguration
  errors from `StdioTransport` and OS-level subprocess errors.
- `LifecycleConfigurationError` is a `RuntimeError` subclass — callers should not
  normally catch it; it signals a programming/config error.
