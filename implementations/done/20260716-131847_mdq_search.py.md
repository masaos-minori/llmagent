# Implementation: scripts/mcp_servers/mdq/search.py (remove hybrid branch, `_search_vector`, `_merge_hybrid`, `_RRF_K`)

Source plan: `plans/20260716-131500_plan.md`

## Goal

Remove all code reachable only through the `mode == "hybrid"` path: the
branch itself, `_search_vector()` (a permanent stub that always returns
`[]`), `_merge_hybrid()` (RRF merge logic with no remaining caller), and
the `_RRF_K` constant.

## Scope

**In:**
- Module docstring (line 4): update
  `"""Search functionality using FTS5 and optional embedding/hybrid search."""`
  to drop the hybrid mention.
- Delete `_RRF_K = 60` and its comment (lines 24-25).
- In `_search_docs_structured()`: replace the `if mode == "hybrid" and
  service.use_embedding: ... else: results = fts_results` block
  (lines 125-130) with a direct `results = fts_results` assignment.
- Delete the now-dead `mode = getattr(req, "mode", None) or "bm25"` line
  (line 89) *only if* `mode` has no remaining reader after step above —
  re-check before deleting (see Details).
- Delete `_search_vector()` (lines 151-158) in full.
- Delete `_merge_hybrid()` (lines 161-193) in full.

**Out:**
- `search_docs()` (the public async entry point, lines 28-74) — unchanged;
  it calls `_search_docs_structured()` and applies result-size limits,
  neither of which reference hybrid/vector logic.
- `_build_search_where()` (lines 196-214) — unrelated, unchanged.
- The `sqlite3.OperationalError`/`sqlite3.Error` exception handling in
  `_search_docs_structured()` — unchanged, unrelated to the hybrid branch.

## Assumptions

1. `_search_vector()` is a permanent, explicitly-labeled placeholder
   (comment: "Placeholder: actual embedding generation and search would
   require an LLM API call ... For now, return empty results") — not
   in-progress work — confirmed by direct read of `search.py:151-158`. No
   functionality is lost by removing it.
2. `_merge_hybrid()`'s only caller is the `mode == "hybrid"` branch being
   removed — confirmed via `rg -n "_merge_hybrid" scripts/` showing only
   the definition and the one call site in `_search_docs_structured()`.
3. `_RRF_K`'s only readers are inside `_merge_hybrid()` — confirmed via
   `rg -n "_RRF_K" scripts/` showing only the definition (line 25) and two
   uses inside `_merge_hybrid()` (lines 172, 175); safe to delete alongside
   the function.
4. After removing the hybrid branch, `mode` (computed at line 89) has no
   remaining reader in this function — `mode` was only consulted in the
   `if mode == "hybrid" and ...` condition being deleted. Confirm this via
   `grep -n "\bmode\b" scripts/mcp_servers/mdq/search.py` after the branch
   edit and before deciding whether to delete line 89; if some other use of
   the local `mode` variable is found (none is expected per current read),
   keep the assignment and only remove the actual hybrid conditional.
5. This change must land in the same commit as `models.py`'s
   `Literal["bm25"]` restriction (companion doc) — otherwise a request
   could still theoretically carry `mode="hybrid"` at the type level while
   the branch that handled it no longer exists (it would simply fall
   through to `results = fts_results`, which is harmless, but the model
   restriction closes this more cleanly per the overall plan).



6. Delete `_search_vector()` in full (current lines 151-158):
   ```python
   def _search_vector(
       service: MdqService, conn: sqlite3.Connection, req: SearchDocsRequest
   ) -> list[SearchResultItem]:
       """Run vector similarity search on the vector table."""
   ```

7. Delete `_merge_hybrid()` in full (current lines 161-193, the entire

8. Confirm no blank-line gaps larger than the file's existing 2-line

### Method

Sequential deletions within one file — first the branch/constant, then the
now-dead helper functions, then re-verify no dangling reference to `mode`,
`service.use_embedding`, `_search_vector`, `_merge_hybrid`, or `_RRF_K`
remains anywhere in the file.

### Details

- `service.use_embedding` itself is removed from `MdqService` in the
  companion `service.py` doc for this same plan — after both changes land,
  `search.py` no longer references `service.use_embedding` at all (it was
  only read in the now-deleted `if mode == "hybrid" and service.use_embedding:`
  condition).
- Do not remove `SearchResultItem`/`SearchResultResult` imports — both
  remain used by `_search_docs_structured()`'s return value and
  `fts_results` construction.
