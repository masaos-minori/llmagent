### Method
Split the try block so that `signal.getsignal()` and `signal.signal()` are each in their own try-except. This ensures that if `getsignal()` succeeds but `signal.signal()` raises ValueError, we still have the original handler saved and can attempt to restore it later.

### Details

```python
# In scripts/agent/http_lifecycle.py, replace lines 335-341:

# Before:
        old_sigint: object | None = None
        try:
            old_sigint = signal.getsignal(signal.SIGINT)
            signal.signal(signal.SIGINT, self._absorb_sigint_during_shutdown)
        except ValueError:
            # Not on the main thread — proceed without the guard rather than fail shutdown.
            old_sigint = None

# After:
        old_sigint: object | None = None
        try:
            old_sigint = signal.getsignal(signal.SIGINT)
        except ValueError:
            # Not on the main thread — proceed without the guard rather than fail shutdown.
            old_sigint = None

        if old_sigint is not None:
            try:
                signal.signal(signal.SIGINT, self._absorb_sigint_during_shutdown)
            except ValueError:
                # signal.signal() failed — do not install the guard handler.
                old_sigint = None
```

Key points:
- First try block captures the original handler via `signal.getsignal()`
- Second try block installs the new handler only if `getsignal()` succeeded
- If either operation raises ValueError, `old_sigint` is set to None (no handler to restore)
- If `getsignal()` succeeds but `signal.signal()` fails, we correctly avoid installing the guard handler
- The finally block at line 371 will still work correctly because it checks `if old_sigint is not None`
