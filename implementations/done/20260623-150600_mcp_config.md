## 変更 1: `_build_single_server()` — healthcheck_mode バグ修正

### 問題 (lines 162-164)

```python
healthcheck_mode=HealthcheckMode(v.get("healthcheck_mode", ""))
if v.get("healthcheck_mode")
else HealthcheckMode.HTTP,
```

TOML に `healthcheck_mode` が設定されていない場合、`HealthcheckMode.HTTP` を直接渡す。これは stdio サーバーにも `HealthcheckMode.HTTP` を設定してしまう。

`McpServerConfig.__post_init__()` の lines 69-77 では:
- `healthcheck_mode` が `HealthcheckMode` インスタンスでない場合のみ transport から解決する
- `HealthcheckMode.HTTP` を直接渡すと「すでに `HealthcheckMode` インスタンス」と判定され `__post_init__` の解決ロジックをスキップする

### 変更後

```python
healthcheck_mode=v.get("healthcheck_mode", ""),  # type: ignore[arg-type] — __post_init__ resolves
```

`""` を渡すことで `__post_init__` の `if not self.healthcheck_mode:` 分岐に入り、transport に応じて自動解決される:
- `transport = "http"` → `HealthcheckMode.HTTP`
- `transport = "stdio"` → `HealthcheckMode.PROCESS`
