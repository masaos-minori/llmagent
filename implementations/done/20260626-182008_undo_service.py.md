# Implementation: undo_service.py + session.py — mark tool_results obsolete on undo

Source plan: `plans/20260626-180403_plan.md` — Phase 2

---

## Goal

After `undo_last_turn()` removes messages from DB, mark the corresponding `tool_results` rows as `obsolete=1` so they are hidden from `/tool list` and `/tool show` by default.

---

## Scope

**In-Scope**
- Add `AgentSession.mark_tool_results_obsolete(from_message_id)` — sets `obsolete=1` for all `tool_results` rows in the current session whose `turn` aligns with the undone range
- `undo_service.undo_last_turn()`: after `ctx.session.undo_last_turn()`, call `ctx.session.mark_tool_results_obsolete()`
- `AgentSession.undo_last_turn()`: return the `last_user_id` boundary (currently returns `int` deleted count; add boundary as second return or separate method)

**Out-of-Scope**
- Changes to `cmd_tooling.py` (separate doc)
- Hard-delete of tool_results rows

---

## Assumptions

1. `tool_results` rows are NOT linked to `messages` by `message_id`. They use `(session_id, turn)` where `turn` is a monotonic counter in `ctx.stats.stat_turns`.
2. The undo removes the last user message and everything after it. The `turn` value for the undone turn is `ctx.stats.stat_turns` before decrement.
3. `undo_service.py` decrements `ctx.stats.stat_turns` by 1 (line 45: `max(0, ctx.stats.stat_turns - 1)`). The undone turn number is `ctx.stats.stat_turns` before decrement.

## Implementation

### Target files
- `scripts/agent/services/undo_service.py`
- `scripts/agent/session.py`
- `scripts/agent/session_message_repo.py` (add helper)

### Procedure
1. In `session.py`, add `mark_tool_results_obsolete(turn: int)` delegation.
2. In `session_message_repo.py`, add `mark_tool_results_obsolete_for_turn(session_id, turn)` that sets `obsolete=1`.
3. In `undo_service.py`, capture `undone_turn = ctx.stats.stat_turns` before decrement, then call `ctx.session.mark_tool_results_obsolete(undone_turn)`.

### Method

**`session_message_repo.py`** — new method:

```python
def mark_tool_results_obsolete_for_turn(self, session_id: int, turn: int) -> int:
    """Set obsolete=1 on all tool_results for (session_id, turn). Returns rows updated."""
    with SQLiteHelper("session").open(write_mode=True) as db:
        cursor = db.execute(
            "UPDATE tool_results SET obsolete = 1 WHERE session_id = ? AND turn = ?",
            (session_id, turn),
        )
        db.commit()
    return cursor.rowcount
```

**`session.py`** — delegation:

```python
def mark_tool_results_obsolete(self, turn: int) -> None:
    """Mark all tool_results for the given turn as obsolete in the current session."""
    if self.session_id is None:
        return
    count = self._message_repo.mark_tool_results_obsolete_for_turn(self.session_id, turn)
    if count:
        logger.info("Marked %d tool_results obsolete for session=%s turn=%s", count, self.session_id, turn)
```

**`undo_service.py`** — updated `undo_last_turn()`:

```python
def undo_last_turn(ctx: AgentContext) -> UndoResult:
    # ... existing logic up to ...
    undone_turn = ctx.stats.stat_turns  # capture before decrement
    ctx.stats.stat_turns = max(0, ctx.stats.stat_turns - 1)
    ctx.session.undo_last_turn()
    ctx.session.mark_tool_results_obsolete(undone_turn)  # NEW
    logger.info("Undo: removed %s messages from history", removed)
    return UndoResult(n_removed=removed)
```
