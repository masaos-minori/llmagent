# Implementation: H-3 — Remove result_id/id_hint from _apply_turn_char_limit()

## Goal

Remove the `result_id` parameter and `id_hint` generation from
`_apply_turn_char_limit()`, and update its caller `_collect_tool_result_msgs()`, so the
per-turn tool-result overflow hint no longer embeds a `result_id` that pointed at
`/tool show <id>` retrieval (removed in the companion `tool_result_formatter.py` step).

## Scope

**Target**: `scripts/agent/tool_runner.py`

**Depends on**: companion implementation step for `scripts/agent/tool_result_formatter.py`
(new `TURN_LIMIT_HINT` text, no trailing `id=` placeholder to fill in).

**Out of scope**:
- `ToolResultStore` itself (`ctx.tool_result_store.store(...)` call at line 196 stays — it is
  removed in a separate future task, not this one)
- `/tool show` command implementation (`cmd_tooling.py`)
- per-turn character-limit threshold logic (`limit > 0 and (turn_chars + len(llm_text)) > limit`
  stays exactly as-is)

## Assumptions

1. `result_id = ctx.tool_result_store.store(...)` (line 196) continues to run — only its
   downstream use inside `_apply_turn_char_limit()` is removed. `result_id` becomes an unused
   local in `_collect_tool_result_msgs()` after this change unless still needed elsewhere in
   that function (checked in Procedure Step 1 below).
2. No test in `tests/test_tool_runner.py` calls `_apply_turn_char_limit()` directly with a
   `result_id` keyword argument (confirmed absent via grep — see Validation plan).
3. `TURN_LIMIT_HINT` has already been updated by the companion `tool_result_formatter.py`
   implementation step to plain text with no `id=` placeholder.

#### Step 2: Remove `result_id` from `_apply_turn_char_limit()` signature and body

Current (lines 250-267):

```python
def _apply_turn_char_limit(
    llm_text: str,
    turn_chars: int,
    limit: int,
    result_id: int,
) -> str:
    """Apply per-turn char limit; return hint if exceeded."""
    if limit > 0 and (turn_chars + len(llm_text)) > limit:
        id_hint = f" (id={result_id})"
        logger.info(
            "Per-turn tool result limit reached: %s chars > %s;"
            " result replaced with hint (id=%s)",
            turn_chars + len(llm_text),
            limit,
            result_id,
        )
        return TURN_LIMIT_HINT.replace("]", f"{id_hint}]")
    return llm_text
```

Replace with:


### Method

- Pure signature/body reduction; no new imports, no new branches.
- `TURN_LIMIT_HINT` is now returned directly instead of via `.replace(...)`, since it no longer
  has a placeholder to fill.

### Details

- After this change, `result_id` in `_collect_tool_result_msgs()` is assigned but only used for
  its side effect via `.store(...)` — mypy will not flag this as unused since the name is read
  nowhere, but ruff's `F841` (unused variable) could theoretically trigger if the variable is
  never referenced at all. Since `result_id` is still bound as the return value of `.store()`
  and no longer read elsewhere in this function, verify `ruff check` does not raise `F841`; if it
  does, either keep the binding as `ctx.tool_result_store.store(...)` (drop the assignment
  entirely, since the return value has no remaining use) or add `# noqa: F841` with justification.
  Prefer dropping the assignment (`ctx.tool_result_store.store(...)` as a bare statement) if
  ruff flags it — this is the simplest fix consistent with `rules/coding.md`'s suppression
  governance (no `# noqa` without inline justification, and only when truly needed).
- Logging: the log message drops `result_id` from its `%s` args since the variable is no longer
  available in this function scope; the message is shortened to match the plan's Design section
  exactly (`"Per-turn tool result limit reached: %s chars > %s; result replaced with hint"`).
