# Implementation: repl_health.py

## Goal

Replace all `except Exception` handlers in `repl_health.py` with specific exception types (`httpx.RequestError`, `asyncio.TimeoutError`, `orjson.JSONDecodeError`, `OSError`) so unexpected errors propagate to the caller.

## Scope

- Target: `scripts/agent/repl_health.py`
- `probe_mcp_health()` line ~33: replace `except Exception` with `except (httpx.RequestError, httpx.HTTPStatusError)`
- `check_service_health()` line ~60: replace `except Exception` with `except (httpx.RequestError, httpx.HTTPStatusError)`
- `_fetch_stdio_tools()` line ~83: replace `except Exception` with `except (asyncio.TimeoutError, orjson.JSONDecodeError, RuntimeError)`
- `_collect_server_tool_names()` line ~107: replace `except Exception` with `except (httpx.RequestError, httpx.HTTPStatusError)`
- `_watchdog_check_http()` line ~186: replace `except Exception` with `except (RuntimeError, OSError)`
- `_watchdog_check_stdio()` line ~236: replace `except Exception` with `except (RuntimeError, OSError)`

## Assumptions

1. Health check failures (HTTP) are always `httpx.RequestError` (connection refused) or `httpx.HTTPStatusError` (non-200) — other exceptions are bugs.
2. `asyncio.wait_for()` raises `asyncio.TimeoutError` on timeout — already handled by the except clause.
3. `transport.call()` raises `RuntimeError` on RPC failure.
4. Watchdog restart (`lifecycle.restart()`, `lifecycle.restart_stdio()`) can raise `RuntimeError` or `OSError` on subprocess failure.

### Details

Key change in `probe_mcp_health()` — currently returns `False` on any exception, which is correct behavior; now only for connection/HTTP errors:

```python
async def probe_mcp_health(http: httpx.AsyncClient, base_url: str) -> bool:
    try:
        resp = await http.get(f"{base_url}/health", timeout=5.0)
        return resp.status_code == 200
    except (httpx.RequestError, httpx.HTTPStatusError):
        return False
    # Other exceptions (e.g. MemoryError) propagate — they are bugs
```
