## Goal

Fix `scripts/agent/startup.py` to call `check_tool_definitions_startup()` instead of `check_tool_definitions_runtime()` at startup. This enables the `tool_definitions_strict` config flag, which was previously ignored because `check_tool_definitions_runtime()` always uses `strict=False`.

## Scope

**In-Scope:**
- `scripts/agent/startup.py` line 22 — add `check_tool_definitions_startup` to the import
- `scripts/agent/startup.py` line 166 — replace the function call

**Out-of-Scope:**
- No changes to `repl_health.py` (covered in separate doc)
- No removal of `check_tool_definitions_runtime` from the import (it may be used elsewhere)

## Assumptions

1. `check_tool_definitions_startup` is defined in `agent.repl_health` (added in the `repl_health.py` implementation doc).
2. `check_tool_definitions_runtime` may or may not be used elsewhere in `startup.py` — check before deciding whether to remove it from the import.
3. When `tool_definitions_strict=False` (the default), `check_tool_definitions_startup` behaves identically to `check_tool_definitions_runtime` — no behavioral change for existing deployments.
