# Implementation: scripts/db/schema_sql.py (workflow schema)

## Goal

Add `_WORKFLOW_SCHEMA` string constant and `build_workflow_schema_sql()` function to `schema_sql.py`, migrating the workflow DDL from `workflow_schema.py` to the canonical DDL location.

## Scope

- Target: `scripts/db/schema_sql.py`
- Add `_WORKFLOW_SCHEMA: str` constant (extracted from `workflow_schema.py`)
- Add `build_workflow_schema_sql() -> str` function
- No existing constants or functions changed

## Assumptions

1. The workflow DDL (`_DDL` in `workflow_schema.py`) is the authoritative source — copy it verbatim.
2. `build_workflow_schema_sql()` takes no arguments (workflow DB uses no `embedding_dims` unlike RAG).
3. `workflow_schema.py` will import `build_workflow_schema_sql` from `schema_sql` after this change.

## Implementation

### Target file
`scripts/db/schema_sql.py`

### Procedure
1. Read the current `_DDL` content from `scripts/db/workflow_schema.py`.
2. Add `_WORKFLOW_SCHEMA = _DDL_content` at the end of `schema_sql.py` (after existing schema constants).
3. Add `def build_workflow_schema_sql() -> str: return _WORKFLOW_SCHEMA`.

### Method

```python
_WORKFLOW_SCHEMA: str = """
    PRAGMA journal_mode=WAL;
    PRAGMA foreign_keys=ON;
    CREATE TABLE IF NOT EXISTS tasks (
        ...
    );
    ...
"""

def build_workflow_schema_sql() -> str:
    return _WORKFLOW_SCHEMA
```

### Details
- Copy the exact DDL from `workflow_schema.py:_DDL` — do not paraphrase.
- Keep PRAGMA lines in `_WORKFLOW_SCHEMA` even if they duplicate `SQLiteHelper` setup (idempotent).
- Function has no parameters (unlike `build_session_schema_sql(dims)` which needs embedding dims).
