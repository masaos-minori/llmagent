# Implementation: rules/toolchain.md (update — add boundary check to validation sequence)

## Goal

Add `test_mdq_rag_boundary.py` to the standard validation sequence in
`rules/toolchain.md` so the boundary check runs as part of the normal test step.

## Scope

**In:**
- Update step 6 (Tests) in `rules/toolchain.md` to explicitly mention the boundary test
- Update the Completion checklist to include the boundary check

**Out:**
- Changes to other steps in the toolchain
- Adding a new top-level step (boundary check fits within the existing "Tests" step)

## Assumptions

- The boundary test is a normal pytest test in `tests/`; it runs automatically with
  `uv run pytest -v` — no separate invocation is needed
- The update is documentation only; no new CI script is required

## Implementation

### Target file

`rules/toolchain.md`

### Procedure

1. In step 6 (Tests), after the existing `pytest` commands, add a note:
   ```
   # boundary enforcement (MDQ/RAG cross-DB and agent-layer direct access)
   uv run pytest tests/test_mdq_rag_boundary.py -v
   ```

2. In the Completion checklist, add one bullet after the `uv run pytest` line:
   ```
   - `uv run pytest tests/test_mdq_rag_boundary.py` passes (MDQ/RAG boundary clean)
   ```

### Method

Two targeted `Edit` operations — one in the step 6 block, one in the checklist.

### Details

- The boundary test is included in the full `uv run pytest -v` run automatically;
  the explicit call in step 6 is for visibility and targeted re-runs after mdq/rag changes
- No change to step ordering (boundary check is a subset of the test step)
