# Implementation: RAG Documentation Quality Gate

## Goal

Add a Markdown/documentation quality gate (CI check + local script) that detects malformed Markdown, stale RAG artifact references, and non-canonical command names before they regress into the `docs/` directory.

## Scope

- **In-Scope**:
  - Extend `scripts/checks/check_docs_consistency.py` (already exists with partial implementation) to cover all required checks
  - Add a new GitHub Actions workflow `.github/workflows/rag-docs-quality.yml` covering `docs/**/*.md` and the check script
  - Optionally add a markdownlint configuration file (`.markdownlint.json`) for structural heading/table/code-block checks
  - Update `docs/03_rag_00_document-guide.md` and `docs/03_rag_90_inconsistencies_and_known_issues.md` to pass all checks after fixes from dependent issues (Issues 3, 8, 9)
  - Document local usage in `docs/03_rag_00_document-guide.md` or a `CONTRIBUTING.md` section
- **Out-of-Scope**:
  - Full documentation site generation
  - Heavy semantic validation of content meaning
  - Modifying any Python package under `scripts/agent/`, `scripts/db/`, `scripts/rag/`, `scripts/mcp/`, `scripts/shared/`

## Assumptions

- `scripts/checks/check_docs_consistency.py` already implements broken-heading, malformed-table, unclosed-inline-code, JSON-not-wrapped, stale-pattern, resolved-in-active, and stale-routing checks; the task is to verify completeness, fix gaps, and wire up CI.
- The existing `mcp-docs-consistency.yml` workflow covers `scripts/check_mcp_docs_consistency.py` for a different check scope; a separate workflow is warranted for RAG docs.
- Issues 3, 8, and 9 (dependencies) will fix actual content in `docs/03_rag_*.md` so that the new checks pass without requiring exceptions in the checker itself.

### Target file: `.github/workflows/rag-docs-quality.yml` (new)

#### Procedure

Create a new GitHub Actions workflow following the pattern of `mcp-docs-consistency.yml`, triggered on `push`/`pull_request` for `docs/**/*.md` and `scripts/checks/**`.

#### Method

Create new file.

#### Details

```yaml
name: RAG Docs Quality

on:
  push:
    paths:
      - "docs/03_rag_*.md"
      - "scripts/checks/check_docs_consistency.py"
  pull_request:
    paths:
      - "docs/03_rag_*.md"
      - "scripts/checks/check_docs_consistency.py"

# Cancel in-progress runs for the same workflow + branch
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  check:
    name: RAG Docs Quality
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.13"

      - name: Run RAG docs quality checks
        run: |
          set -euo pipefail
          python scripts/checks/check_docs_consistency.py
```

**Notes:**
- Scoped strictly to `docs/03_rag_*.md` paths to avoid redundancy with `mcp-docs-consistency.yml`
- No markdownlint step by default (pure-Python checker is sufficient); can be added later if team approves Node.js dependency

### Running checks locally

Run the RAG documentation consistency checker from the project root:

```bash
python scripts/checks/check_docs_consistency.py
```

To check specific files:

```bash
python scripts/checks/check_docs_consistency.py docs/03_rag_01_system_overview.md
```

The checker runs 8 checks: broken headings, malformed tables, unclosed inline code, JSON not wrapped in fenced code blocks, stale artifact references (`.txt` → `.json`), non-canonical command names, resolved issues under active sections, and stale issue ID routing. Historical markers (`legacy`, `historical`, `archive only`, `resolved`, `was:`, `removed`) exempt lines from stale-pattern failures.
```
