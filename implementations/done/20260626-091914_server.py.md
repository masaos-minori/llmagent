# Implementation: Wire workflow_allowlist into cicd MCP server startup in server.py

Steps covered: Plan 20260626-091914 — Step 1-2

---

## Goal

Ensure the cicd MCP `server.py` passes `workflow_allowlist` from config to `ServiceGuards` (or `_check_workflow()`), so the fail-closed behavior implemented in `service_guards.py` has correct input.

---

## Scope

- **In scope**: `scripts/mcp/cicd/server.py` — config loading, `ServiceGuards` initialization
- **Out of scope**: service_guards.py changes (step 1-1); repl_health.py; tests

---

## Assumptions

- `server.py` loads `cicd_mcp_server.toml` and initializes `ServiceGuards`.
- `workflow_allowlist` is currently loaded but may not be passed to `ServiceGuards`, or it may be passed but with an incorrect default.
- The fix is: ensure `workflow_allowlist` from config is always passed, and that an empty config list is not converted to `None` (which would silently revert to old behavior).

---

## Implementation

### Target file
`scripts/mcp/cicd/server.py`

### Procedure
1. Read `scripts/mcp/cicd/server.py` to find config loading and `ServiceGuards` initialization.
2. Locate where `workflow_allowlist` is read from config.
3. Ensure it is passed correctly:
   ```python
   guards = ServiceGuards(
       workflow_allowlist=config.workflow_allowlist,  # may be empty list []
   )
   ```
4. If `config.workflow_allowlist` defaults to `None`, change to `[]`:
   - In config model: `workflow_allowlist: list[str] = field(default_factory=list)`
   - `None` would bypass the fail-closed check; `[]` triggers it correctly.

### Method
Config wiring fix. No new logic.
