# Implementation procedure: `config/agent.toml` (fix 8 stale MCP server `cmd` paths)

Source plan: `plans/20260720-073534_plan.md`, Implementation step Phase 1.

`config/agent.toml` has many prior implementation docs under `implementations/done/`. Checked by
content (`grep -l 'cmd.*server\.py\|20260720-070458'`): the one hit,
`implementations/done/20260719-111834_agent.toml.md`, addresses the original browser-mcp
*registration* (adding a new `[mcp_servers.browser]` block with `cmd = [..., "browser/server.py"]`,
before the `<name>_server.py` rename convention existed) тАФ a different concern from fixing the
already-registered 8 servers' now-stale `cmd` paths. No genuine overlap. This is a new document.

## Goal

Every `[mcp_servers.<name>]` block for the 8 servers affected by the `ca6b7bfe`/`abcf0820` rename
(`shell`, `git`, `web_search`, `github`, `cicd`, `rag_pipeline`, `mdq`, `browser`) launches its actual
`<name>_server.py` file instead of the deleted bare `server.py`.

## Scope

**In scope**
- Change only the last path segment of `cmd` in 8 blocks (currently lines 293, 305, 314, 358, 367, 376,
  388, 397 тАФ re-verify exact line numbers at implementation time since other unrelated edits may have
  shifted them since this plan was written).

**Out of scope**
- `startup_mode`, `url`, `auth_token`, `role`, `tool_names` тАФ unchanged.
- `file_delete`/`file_write`/`file_read` servers (`delete_server.py`/`write_server.py`/`read_server.py`)
  тАФ different naming convention, never affected by the rename.
- Any `scripts/mcp_servers/**/*.py` change тАФ the 8 `<name>_server.py` files and their `app_module`
  self-references are already correct.

## Assumptions

1. Verified directly (`grep -n 'cmd\s*=' config/agent.toml`): all 8 entries still read
   `.../<name>/server.py`, and `ls scripts/mcp_servers/<name>/*_server.py` confirms exactly one file per
   subpackage named `<name>_server.py` in each case тАФ the fix is a mechanical suffix change, not a
   judgment call.
2. `grep -n 'models\.py\|tools\.py\|service\.py' config/agent.toml` returns no matches тАФ no other key in
   this file references a stale bare filename; only the 8 `cmd` entries are wrong.

## Implementation

### Target file

`config/agent.toml`

### Procedure

1. For each of the 8 blocks, change the final path element of `cmd` from `<name>/server.py` to
   `<name>/<name>_server.py`. No other field in the array or the surrounding block changes.
2. Re-run `grep -n 'cmd\s*=' config/agent.toml` after editing and manually confirm all 8 lines now end
   in `_server.py` matching their own server key (e.g. the `[mcp_servers.web_search]` block's `cmd` ends
   in `web_search_server.py`, not e.g. `websearch_server.py`).
3. Final gate: `grep -n 'models\.py\|tools\.py\|service\.py\|/server\.py"' config/agent.toml` тАФ must
   return zero matches (confirms no stale bare-filename reference remains anywhere in the file, not just
   the 8 known entries тАФ see the plan's own Risk about a possible 9th missed server).

### Method

Pure text substitution inside 8 TOML array literals; no schema change, no new/removed key.

```toml
# before (one representative block; apply the same suffix change to all 8)
cmd = ["uv", "run", "--directory", "/opt/llm", "python", "/opt/llm/scripts/mcp_servers/web_search/server.py"]
# after
cmd = ["uv", "run", "--directory", "/opt/llm", "python", "/opt/llm/scripts/mcp_servers/web_search/web_search_server.py"]
```

### Details

- Applies identically (substituting the server-specific name) to: `shell`, `git`, `web_search`,
  `github`, `cicd`, `rag_pipeline`, `mdq`, `browser`.
- `deploy/deploy.sh` copies `config/agent.toml` via a plain `cp` (confirmed in a prior cycle) тАФ no
  `deploy.sh` change needed.
- This is a config-content-only edit тАФ no `scripts/` file changes, so `ruff`/`mypy`/`lint-imports`/
  `bandit` have nothing new to check for this specific file; the regression test (see the sibling
  `tests/test_mcp_server_cmd_paths.py` procedure document) and the full pytest suite are the load-bearing
  gates.
