# Implementation: H-5 — ensure_ready failure → ToolCallResult error (tool_executor.py)

## Goal

Wrap `await self._lifecycle.ensure_ready(server_key)` in `_raw_execute()` with
`try/except (OSError, RuntimeError)` so that lifecycle startup failures are returned as
`ToolCallResult(is_error=True, error_type="transport")` instead of propagating as
unhandled exceptions through the tool loop.

## Scope

**Target**: `scripts/shared/tool_executor.py`

**Step covered**: Plan H-5 step 1.

**Out of scope**: `factory.py`, `http_lifecycle.py`, `startup.py`, all other files.

## Assumptions

1. `_raw_execute()` currently calls `ensure_ready()` at lines 256–257 (confirmed by
   source: inside `if self._lifecycle is not None:` guard).
2. `HttpStartupError` is a subclass of `RuntimeError`; `subprocess.Popen` raises
   `OSError` on command-not-found. Both are covered by `(OSError, RuntimeError)`.
3. `self._error_result(server_key, msg, error_type="transport")` is the correct method
   to return a transport-type error result (consistent with `_record_transport_error`).
4. `self._health_registry.record_failure(server_key)` is the correct method to call;
   confirm method name in `shared/mcp_health.py` before implementing.
5. Transport `execute()` must NOT be called after an `ensure_ready` failure; the early
   return achieves this.

## Implementation

### Target file

`scripts/shared/tool_executor.py`

### Procedure

#### Step 1: Locate the ensure_ready call in `_raw_execute()`

Current code (lines 255–257):
```python
# Lifecycle ensure_ready
if self._lifecycle is not None:
    await self._lifecycle.ensure_ready(server_key)
```

#### Step 2: Replace with try/except block

```python
# Lifecycle ensure_ready
if self._lifecycle is not None:
    try:
        await self._lifecycle.ensure_ready(server_key)
    except (OSError, RuntimeError) as e:
        msg = f"Lifecycle ensure_ready failed for {server_key!r}: {e}"
        logger.error(msg)
        if self._health_registry is not None:
            self._health_registry.record_failure(server_key)
        return self._error_result(server_key, msg, error_type="transport")
```

No other changes to `_raw_execute()`.

### Method

- Surgical replacement: 2 lines become 9 lines.
- `(OSError, RuntimeError)` is a narrow catch: only lifecycle startup exceptions are
  expected here. All other exception types continue to propagate.
- `record_failure()` is guarded with `if self._health_registry is not None` to handle
  the case where no registry is configured (pre-H-2 environments).

### Details

- `_error_result(server_key, msg, error_type="transport")` must accept `server_key` as
  the first argument. Read the method signature in `tool_executor.py` before
  implementing; adjust argument order if needed.
- Confirm `McpServerHealthRegistry.record_failure(server_key: str)` is the correct
  method name by reading `scripts/shared/mcp_health.py`. Alternative names:
  `record_error()`, `set_state()`.
- The `error_type="transport"` flag distinguishes this from tool-level errors
  (`error_type="tool"`) in audit logs and caller retry logic.
- After the early return, the transport `execute()` path (lines 269–274) is unreachable,
  which is correct behavior.
