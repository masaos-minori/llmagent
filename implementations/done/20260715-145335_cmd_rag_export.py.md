# Implementation Procedure: cmd_rag_export.py

## Goal

Remove `_cmd_export` from `_RagExportMixin` (the standalone `/export` handler is
being folded into `/session export`), while keeping `_cmd_compact` and the file itself
fully intact — `/compact` is unaffected by this plan.

## Scope

### In scope
- Delete the `_cmd_export` method (current lines 27-42).
- Update the module docstring (current lines 6-8) to drop the `_cmd_export` line.
- Check whether any import in this file becomes unused once `_cmd_export` is removed.

### Out of scope
- `_cmd_compact` (current lines 44-73) — unchanged, keeps using `HistoryCompressionError`.
- The file's existence — it survives (still hosts `_cmd_compact`).
- `registry.py`'s composition of `_RagExportMixin` into `CommandRegistry` — unchanged
  (still required for `/compact`).

## Assumptions

- `render_export` / `write_export` (imported from `agent.services.export_formatter`)
  are used only by `_cmd_export` in this file — verified by reading the full file
  (73 lines): `_cmd_compact` does not reference them. Once `_cmd_export` is deleted,
  the import `from agent.services.export_formatter import render_export, write_export`
  becomes unused here and must be removed (`_cmd_session`'s new `export` branch in
  `cmd_session.py` imports these two functions itself instead).
- The stale docstring reference to `/rag` (module docstring, current line 6, does not
  actually mention `/rag` in this file's current content but `registry.py`'s own
  docstring does — see that file's implementation doc) is corrected where it actually
  appears.

## Implementation

### Target file

`scripts/agent/commands/cmd_rag_export.py`

### Procedure

1. Delete the `_cmd_export` method:
   ```python
   def _cmd_export(self, args: str) -> None:
       """Export the current conversation history to Markdown or JSON.

       Usage: /export [md|json] [filename]
       """
       ctx = self._ctx
       parts = args.strip().split()
       fmt = "md"
       outfile: str | None = None
       for part in parts:
           if part in ("md", "json"):
               fmt = part
           else:
               outfile = part
       content = render_export(ctx.conv.history, fmt)
       write_export(content, outfile, len(ctx.conv.history))
   ```
2. Remove the now-unused import:
   ```python
   from agent.services.export_formatter import render_export, write_export
   ```
3. Update the module docstring (current lines 1-9):
   ```python
   """agent/commands/cmd_rag_export.py

   Export and compact mixin for CommandRegistry.

   Provides _RagExportMixin with:
     _cmd_export            — /export: dump conversation to Markdown or JSON
     _cmd_compact           — /compact: force immediate context compression
   """
   ```
   → change to:
   ```python
   """agent/commands/cmd_rag_export.py

   Compact mixin for CommandRegistry.

   Provides _RagExportMixin with:
     _cmd_compact           — /compact: force immediate context compression
   """
   ```
   (Class name `_RagExportMixin` and its file name are left unchanged — renaming
   either is out of scope / not required by the acceptance criteria; only the
   docstring text is corrected.)

### Method

Targeted deletion of one method plus its now-dead import, and a docstring edit.
`_cmd_compact` and its `HistoryCompressionError` import are left byte-for-byte
untouched.

### Details

- `ruff check --fix` will flag the unused `render_export`/`write_export` import if
  left in place (`F401`); remove it in the same edit rather than relying on
  autofix, to keep the diff intentional.
- `logging` import stays (used by `logger = logging.getLogger(__name__)`, referenced
  nowhere in the current file body but kept for parity with sibling mixins — verify
  with `ruff check` whether it is flagged as unused; if so, it was already unused
  before this change and is out of this plan's scope to touch).
