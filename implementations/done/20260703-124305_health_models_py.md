## Goal

Add the `McpHealthProbeResult` frozen dataclass to `agent/shared/health_models.py` to carry the structured result of a `/health` GET probe.

## Scope

- In-Scope:
  - Add `McpHealthProbeResult` dataclass with five fields: `reachable`, `status_code`, `restart_recommended`, `operator_action_required`, `body`
  - No existing classes or functions are modified
- Out-of-Scope:
  - Changes to `HealthCheckResult` or `ServiceWarning`
  - Any usage of the new dataclass (wiring into `repl_health.py` is covered in Step 4)

## Assumptions

1. `health_models.py` already imports `dataclasses` via `from dataclasses import dataclass, field`; the new class uses the same import.
2. `int | None` syntax is valid because the file uses `from __future__ import annotations`.
3. `dict[str, object]` is the correct body type — the body can contain mixed value types (str, bool, dict) as returned by the MCP health endpoint.
4. `frozen=True` is appropriate because the probe result is an immutable value object.

```python
@dataclass(frozen=True)
class McpHealthProbeResult:
    """Structured result of a single /health GET probe to an MCP server.

    Fields:
        reachable:               True if an HTTP response was received (any status code).
        status_code:             HTTP status code, or None if connection failed.
        restart_recommended:     Body field `restart_recommended`; False if absent or parse fails.
        operator_action_required: Body field `operator_action_required`; False if absent or parse fails.
        body:                    Parsed JSON body dict; empty dict if parse failed or unreachable.
    """

    reachable: bool
    status_code: int | None
    restart_recommended: bool
    operator_action_required: bool
    body: dict[str, object]
```

4. No other changes to the file.

### Method

- Standard `@dataclass(frozen=True)` pattern matching `ServiceWarning` and `HealthCheckResult` in the same file.
- `int | None` requires `from __future__ import annotations` (already present at line 1).
- `body` field has no `field(default_factory=...)` because `McpHealthProbeResult` is always constructed with explicit arguments by the caller; there is no sensible default for `body`.
