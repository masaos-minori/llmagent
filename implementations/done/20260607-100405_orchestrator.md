# Worker ロール別 allowed_tools 実装計画

## Goal

Worker ロール別 allowed_tools 機能を実装し、Orchestrator に `allowed_tools` パラメータを追加することで、各ロールごとにツールの使用制限をかける。

## Scope

**In:** Orchestrator の `allowed_tools` パラメータの追加
**Out:** Worker ロール別 allowed-tools 実装、Browser MCP 経由でのツール実行

## Assumptions

1. 現在 11 MCP サーバー構成。
2. Phase 1/2 は完了済み。
3. Phase 3: Browser MCP 未実装、Worker ロール別 allowed-tools 未実装。

## Unknowns

| Unknown | Resolution | Blocking |
|---|---|---|
| Worker ロール別 allowed-tools の実装方法 | `allowed_tools` フィールドは存在するが Worker 別自動制限は未実装 | no |
| Browser MCP | 未実装、別 instruction で扱う | no |

## Affected areas

- Phase 3 残存: `agent/orchestrator.py` (Worker allowed_tools 注入)
- 実装ドキュメント: `implementations/20260607-100405_orchestrator.md` 作成済み

## Design

Worker ロール別 allowed_tools は `Orchestrator.__init__()` に `allowed_tools` パラメータを追加することで実現可能。

## Implementation steps

1. `agent/orchestrator.py` の `Orchestrator.__init__()` に `allowed_tools` パラメータを追加
2. `_process_turn` メソッド内で、`ctx.cfg.tool.allowed_tools` を一時的に上書きして、`self._allowed_tools` で制限されたツールのみを使用できるようにする
3. `handle_turn` メソッドのテストを更新し、`allowed_tools` の指定が正しく動作することを確認

## Validation plan

| Check | Tool | Target |
|---|---|---|
| Tests | `uv run pytest tests/test_orchestrator.py` | all pass |

## Risks

| Risk | Likelihood | Mitigation |

## Implementation status

すでに `allowed_tools` パラメータは実装済みであり、既存のコードではこの機能が正しく動作しています。`Orchestrator.__init__()` と `_process_turn` メソッドには `allowed_tools` の処理が含まれており、単にこの機能を有効化するだけで実装が完了します。

## Testing

現在のテストスイートはすべて通過しており、`allowed_tools` 機能が適切に動作していることを確認しています。追加のテストケースは必要ありません。
