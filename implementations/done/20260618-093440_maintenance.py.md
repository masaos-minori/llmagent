# Implementation: scripts/db/maintenance.py

## Goal

Add RAG consistency check functions to `maintenance.py` so that trigger-based sync failures
and orphan records in `chunks_fts` and `chunks_vec` can be detected early.

## Scope

- `scripts/db/maintenance.py` — add `RagConsistencyReport` dataclass, `check_rag_consistency()`,
  `is_consistent()`, `summarize_issues()`. No other production files changed.

## Assumptions

1. `chunks_fts` uses `content_rowid = 'chunk_id'` (confirmed in `schema_sql.py:40`).
   `SELECT COUNT(*) FROM chunks_fts` returns currently indexed (non-deleted) row count.
2. `chunks_vec` has `chunk_id INTEGER PRIMARY KEY` (no FK constraint — sqlite-vec limitation).
   Orphan detection: `SELECT COUNT(*) FROM chunks_vec WHERE chunk_id NOT IN (SELECT chunk_id FROM chunks)`.
3. `db` argument is an open `SQLiteHelper` instance with `row_factory` not required (scalar queries).
4. `fts_gap = chunks - fts` — positive means FTS has fewer rows than chunks (trigger missed inserts);
   negative means FTS has more rows (impossible with current schema, but include for completeness).
5. `vec == chunks` is expected in a consistent state (vec and chunks are 1:1 via trigger).
6. The dataclass is `frozen=True` to follow the pattern of `RecoveryResult` and `WalCheckpointCounts`.

**`RagConsistencyReport` dataclass:**

```python
@dataclass(frozen=True)
class RagConsistencyReport:
    """Counts from chunks, chunks_fts, and chunks_vec for consistency verification."""

    chunks: int
    fts: int
    vec: int
    orphan_vec_count: int
    fts_gap: int  # chunks - fts; 0 = consistent
```

**`check_rag_consistency(db)` function:**
```python
def check_rag_consistency(db: SQLiteHelper) -> RagConsistencyReport:
    """Return row counts from chunks, chunks_fts, and chunks_vec for consistency verification.

    Queries (all read-only):
      SELECT COUNT(*) FROM chunks           → chunks count
      SELECT COUNT(*) FROM chunks_fts       → FTS5 indexed row count
      SELECT COUNT(*) FROM chunks_vec       → vec row count
      SELECT COUNT(*) FROM chunks_vec WHERE chunk_id NOT IN (SELECT chunk_id FROM chunks) → orphan vec count
    """
    chunks = db.fetchone("SELECT COUNT(*) FROM chunks")[0]
    fts = db.fetchone("SELECT COUNT(*) FROM chunks_fts")[0]
    vec = db.fetchone("SELECT COUNT(*) FROM chunks_vec")[0]
    orphan_vec_count = db.fetchone(
        "SELECT COUNT(*) FROM chunks_vec WHERE chunk_id NOT IN (SELECT chunk_id FROM chunks)"
    )[0]
    return RagConsistencyReport(
        chunks=chunks,
        fts=fts,
        vec=vec,
        orphan_vec_count=orphan_vec_count,
        fts_gap=chunks - fts,
    )
```

**`is_consistent(report)` function:**
```python
def is_consistent(report: RagConsistencyReport) -> bool:
    """Return True when all consistency conditions are met."""
    return report.fts_gap == 0 and report.orphan_vec_count == 0 and report.vec == report.chunks
```

**`summarize_issues(report)` function:**
```python
def summarize_issues(report: RagConsistencyReport) -> list[str]:
    """Return human-readable descriptions of any consistency issues found."""
    issues: list[str] = []
    if report.fts_gap != 0:
        issues.append(
            f"FTS gap: chunks={report.chunks}, fts={report.fts}, gap={report.fts_gap}"
        )
    if report.orphan_vec_count > 0:
        issues.append(
            f"Orphan vec rows: {report.orphan_vec_count} chunk_id(s) not in chunks"
        )
    if report.vec != report.chunks:
        issues.append(
            f"Vec count mismatch: chunks={report.chunks}, vec={report.vec}"
        )
    return issues
```
