# Implementation: scripts/agent/startup.py — render SKIPPED outcomes in CLI

Source plan: `plans/20260721-031357_plan.md` ("Make MCP tool-discovery failure visible/fatal
(sole-authority routing follow-up)"), Implementation step 2 (Design Change 2).

Not already implemented: verified against current source (`scripts/agent/startup.py`, method
`_display_pipeline_results()`, lines 293-299+ read directly) — the method has only `WARNING` and
`FATAL` branches; no `elif outcome.status == StartupCheckStatus.SKIPPED:` branch exists, so
`SKIPPED` outcomes (produced today by both `mcp_tool_discovery`'s and `rag_consistency`'s except
blocks) are silently dropped from CLI output. Checked all 16 filename-matching docs under
`implementations/done/*startup.py.md`; none add a `SKIPPED` rendering branch to this method — the
one doc that created this method (`20260706-105836_startup.py.md`, "Refactor _check_services()
with StartupValidationPipeline") only introduced `WARNING`/`FATAL` handling, not `SKIPPED`.

## Goal

Make `SKIPPED` startup-validation outcomes actually visible to the interactive CLI user (today
they only reach `logger.error` if some *other* step is *also* fatal), closing the visibility gap
the source requirement calls out.

## Scope

**In scope:**
- `scripts/agent/startup.py`, `StartupOrchestrator._display_pipeline_results()` (currently lines
  293-299).

**Out of scope:**
- The except-block production-mode branching in step 4 — separate item, see
  `implementations/20260721-162337_startup.py.md`.
- Any change to which sources produce `SKIPPED` (`mcp_tool_discovery`, `rag_consistency`) — this
  item only adds *rendering* for an existing status value, not new producers.

## Assumptions

1. `OutputTag.SKIPPED = "[skipped]"` is already defined at `scripts/agent/output_tags.py:27` and
   currently unused elsewhere — using it here introduces no new dependency or enum change.
2. This rendering branch applies to **all** `SKIPPED` outcomes project-wide (there are two
   producers today: `mcp_tool_discovery`'s and `rag_consistency`'s except blocks), not only
   `mcp_tool_discovery`'s. The plan's Design section explicitly accepts this as the simpler,
   more general fix and flags it as a slightly broader blast radius than the requirement's literal
   ask — if review prefers a narrower `mcp_tool_discovery`-only condition, that is a one-line
   change to add an `outcome.source == "mcp_tool_discovery"` guard.
3. `self._view.write_warning(...)` is the correct sink (same sink already used for `WARNING` and
   `FATAL` branches) — no new CLI view method is introduced.

## Implementation

### Target file

`scripts/agent/startup.py`

### Procedure

1. Open `scripts/agent/startup.py`, locate `_display_pipeline_results()` (currently lines
   293-299).
2. Add a new `elif` branch after the existing `FATAL` branch (which includes its nested
   `remediation` write) for `StartupCheckStatus.SKIPPED`, writing
   `f"{OutputTag.SKIPPED} {outcome.message}"` via `self._view.write_warning(...)`.
3. Do not alter the existing `WARNING`/`FATAL` branches.

### Method

Direct text edit — add one `elif` clause to an existing `for`/`if`/`elif` chain. No signature
change, no new import (`OutputTag` and `StartupCheckStatus` are already imported in this file for
the existing branches).

### Details

Current code (lines 293-299, from source read):

```python
    def _display_pipeline_results(self, pipeline: StartupValidationResult) -> None:
        """Display startup validation warnings and fatal errors via the CLI view."""
        for outcome in pipeline.outcomes:
            if outcome.status == StartupCheckStatus.WARNING:
                self._view.write_warning(f"{OutputTag.NON_FATAL} {outcome.message}")
            elif outcome.status == StartupCheckStatus.FATAL:
                self._view.write_warning(f"{OutputTag.FATAL} {outcome.message}")
                if outcome.remediation:
                    self._view.write_warning(f"  Remediation: {outcome.remediation}")
```

Add, immediately after the `FATAL` branch's body (i.e. after the `if outcome.remediation:` block,
still inside the same `for` loop):

```python
            elif outcome.status == StartupCheckStatus.SKIPPED:
                self._view.write_warning(f"{OutputTag.SKIPPED} {outcome.message}")
```
