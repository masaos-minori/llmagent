# HistoryManager 重要度ベースの履歴圧縮実装計画

## Goal

HistoryManager に重要度ベースの履歴圧縮機能を実装し、`LLMMessage` に `importance` と `pinned` フィールドを追加することで、重要なメッセージを圧縮時に保持できるようにする。

## Scope

- `LLMMessage` に `importance: float` と `pinned: bool` フィールドを追加
- `HistoryManager` の圧縮ロジックを重要度/固定ベースの圧縮に更新し、固定の `compress_turns * 2` ではなく重要度に基づく圧縮を実現

## Assumptions

1. `LLMMessage` は `TypedDict(total=False)` なのでフィールド追加は後方互換性あり
2. `HistoryManager` は現在固定の `compress_turns * 2` 圧縮を使用している

## Unknowns

| Unknown | Resolution | Blocking |
|---|---|---|
| HistoryManager の重要度ベース圧縮の実装方法 | 実装済み | no |

## Affected areas

- `scripts/shared/types.py`: LLMMessage に importance/pinned フィールドを追加
- `scripts/agent/history.py`: _classify_importance 関数と _select_turns_to_compress メソッドを更新

## Design

重要度ベースの圧縮は以下の手順で実装：
1. LLMMessage に importance (float) と pinned (bool) フィールドを追加
2. _classify_importance 関数でメッセージの重要度を評価
3. _select_turns_to_compress メソッドで重要度に基づいて圧縮候補を選定

## Implementation steps

1. `scripts/shared/types.py` に `importance: float` と `pinned: bool` フィールドを追加
2. `scripts/agent/history.py` に `_classify_importance()` 関数を追加
3. `HistoryManager` クラスの `compress()` メソッドを重要度ベースの圧縮に更新

## 実装状況

以下の変更を実装しました：

1. `scripts/shared/types.py` に `importance: float` と `pinned: bool` フィールドを追加
2. `scripts/agent/history.py` に `_classify_importance()` 関数を追加し、重要度ベースのメッセージ分類ロジックを実装
3. `_select_turns_to_compress` メソッドを更新して、重要度に基づく圧縮選定を行うように変更

## テスト結果

すべてのテストが正常に通過しました。これにより、以下の機能が実装されました：
- LLMMessage に importance と pinned フィールドの追加
- 重要度ベースでのメッセージ分類と圧縮処理
- メッセージの重要度を考慮した圧縮優先順位の実装

これらの変更により、重要なメッセージ（例：システムメッセージ、重要な会話内容）は圧縮対象から除外され、保持されるようになりました。
