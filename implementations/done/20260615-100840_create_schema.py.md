# Implementation: create_schema.py (BUG-1)

## Goal

Replace the module-level `Logger(...)` call in `scripts/db/create_schema.py` with
lazy initialization so that importing the module no longer requires `common.toml` to
exist, fixing the test collection failure for `tests/test_create_schema.py`.

## Scope

**In**: `scripts/db/create_schema.py` only.
**Out**: No changes to tests or other production files.

## Assumptions

1. `_get_schema_log_path()` at L26 reads `common.toml` via `ConfigLoader`. This call
   must be deferred to first use, not at import time.
2. `logger.` is referenced at exactly 6 lines: L175, L177, L187, L189, L213, L225.
3. The `# noqa: PLC0415` on the `ConfigLoader` import (L18) is already present;
   no additional suppression is needed.
4. Comments and log strings must remain in English (coding.md rule).

## Implementation

### Target file

`scripts/db/create_schema.py`

### Procedure

1. Remove L34: `logger = Logger(__name__, _get_schema_log_path())`
2. After L34's position, insert:
   ```python
   _logger: Logger | None = None


   def _get_logger() -> Logger:
       global _logger
       if _logger is None:
           _logger = Logger(__name__, _get_schema_log_path())
       return _logger
   ```
3. Replace all 6 `logger.` usages with `_get_logger().`:
   - L175: `logger.error(...)` → `_get_logger().error(...)`
   - L177: `logger.info(...)` → `_get_logger().info(...)`
   - L187: `logger.error(...)` → `_get_logger().error(...)`
   - L189: `logger.info(...)` → `_get_logger().info(...)`
   - L213: `logger.info(...)` → `_get_logger().info(...)`
   - L225: `logger.exception(...)` → `_get_logger().exception(...)`

### Method

Use Edit tool for the deletion + insertion, then 6 targeted replacements.

### Details

After the change, `create_schema.py` can be imported without `common.toml` present.
`Logger` is only constructed when the first logging call is made (i.e., when
`create_rag_schema()` or `create_session_schema()` actually runs).
