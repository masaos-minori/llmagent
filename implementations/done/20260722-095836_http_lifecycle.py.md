## Goal
Ensure stderr resources are properly cleaned up when `os.getpgid()` fails after `Popen()` succeeds in `McpServerLifecycle.start_server()`.

## Scope
**In**: `scripts/agent/http_lifecycle.py` — refactor `getpgid()` error handling path
**Out**: Any changes to `Popen()` behavior, subprocess management logic, or other lifecycle methods

## Assumptions
1. A process whose pgid cannot be obtained should not be managed by the lifecycle manager
2. The `OSError` from `getpgid()` indicates the process has already exited or is in an unrecoverable state
3. Logging a warning is sufficient — no need to retry or alert on this transient condition

## Implementation

### Target file
`scripts/agent/http_lifecycle.py`

### Procedure
1. Identify the `getpgid()` call site in `start_server()` method (around line 241)
2. Wrap the `getpgid()` call in a try/except block
3. On `OSError`, close the stderr file handle and remove tracking entries before logging
4. Remove the process reference from `_http_procs` and `_http_pgids`
5. Return early from `start_server()` to prevent health check attempts

### Method
Refactor the bare `except OSError: pass` into a proper cleanup-and-return pattern.

### Details
Current code at lines 240-244:
```python
try:
    self._http_pgids[server_key] = os.getpgid(proc.pid)
except OSError:
    pass
self._http_procs[server_key] = proc
```

Problem: When `getpgid()` raises `OSError`, the stderr file handle (`stderr_fh`) opened at line 225 is never closed. The file descriptor leaks because:
- The `except OSError: pass` silently ignores the error
- No cleanup of `_stderr_files` or `_stderr_log_paths` occurs
- The process reference remains in `_http_procs`, so subsequent operations may attempt to use a stale handle

Fix:
```python
try:
    self._http_pgids[server_key] = os.getpgid(proc.pid)
except OSError:
    logger.warning(
        "Lifecycle: getpgid() failed for %r pid=%d; cleaning up",
        server_key,
        proc.pid,
    )
    stderr_fh.close()
    self._stderr_files.pop(server_key, None)
    self._stderr_log_paths.pop(server_key, None)
    self._http_procs.pop(server_key, None)
    self._http_pgids.pop(server_key, None)
    return
```

Key points:
- Close `stderr_fh` explicitly before removing from tracking dicts
- Pop from `_http_procs` and `_http_pgids` to avoid orphaned references
- Early return prevents health check loop from starting on invalid process
- Log a WARNING level message for observability
