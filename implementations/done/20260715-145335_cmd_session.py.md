# Implementation Procedure: cmd_session.py

## Goal

Add `export`, `stats`, `health`, `checkpoint`, `vacuum`, `purge`, `recover`
subcommands to `_cmd_session`, reusing `render_export`/`write_export` (from
`agent.services.export_formatter`) and `DbSessionOps`/`_db_session_stats` unchanged,
so `/session export markdown|json [file]` and
`/session stats|health|checkpoint|vacuum|purge|recover` produce output identical to
the removed `/export md|json [file]` and `/db session ...` commands.

## Scope

### In scope
- Add a `DbSessionOps` instance to `_SessionMixin.__init__` (mirrors the deleted
  `_DbMixin.__init__`'s `self._session_ops = DbSessionOps(self._ctx, self._out)`).
- Add a dict-based dispatch table for the 7 new subcommands (per the plan's UNK-02
  resolution: dict dispatch to avoid growing `_cmd_session`'s if/elif chain past a
  `radon` C-grade).
- Add a new `_session_export(args: str) -> None` helper implementing the `export`
  subcommand (maps `markdown` → `"md"` per the plan's UNK-01 resolution; keeps
  `"md"`/`"json"` as the internal `ExportFormat`-compatible strings unchanged).
- Wire the 6 DB-op branches (`stats`, `health`, `checkpoint`, `vacuum`, `purge`,
  `recover`) to `_db_session_stats` (relocated from `db_stats_display.py` — see that
  file's own implementation doc) and `DbSessionOps` methods, unchanged internally.
- Extend `_cmd_session`'s final `write_validation_error` usage message to mention the
  new subcommands.

### Out of scope
- The existing `list`/`load`/rename`/`delete` if/elif branches (current lines 126-152)
  — left as-is; not migrated to dict dispatch (explicitly out of scope per the plan's
  UNK-02 resolution — "the existing branches can stay if/elif... out of scope to
  refactor working code").
- `_session_list`, `_session_load_safe`, `_session_delete`, `_load_session`,
  `_generate_session_title` — unchanged.
- `DbSessionOps` (`db_session_ops.py`) — reused verbatim, no source change.
- `render_export`/`write_export` (`export_formatter.py`) — reused verbatim, no
  source change (both are plain string/enum comparisons on `fmt`, confirmed by
  reading the full file: `render_export` only branches on
  `fmt == ExportFormat.JSON or fmt == "json"`, else Markdown).

## Assumptions

- `DbSessionOps.__init__(self, ctx: Any, out: Any)` (verified: `db_session_ops.py:17-19`)
  takes only `ctx`/`out`, identical to what `_SessionMixin` already has as
  `self._ctx`/`self._out` via `MixinBase`. No adapter needed.
- `DbSessionOps.health()`, `.vacuum()` take no extra args; `.checkpoint(mode: str | None)`,
  `.purge(rest: str)`, `.recover(backup_path: str | None)` take one arg each — all
  synchronous (no `await` anywhere in `db_session_ops.py`), so `_cmd_session` can
  remain a **synchronous** method (`CommandDef("/session", ..., is_async=False, ...)`
  in `command_defs_list.py` stays `False` — no change needed to that field).
- `_db_session_stats` (currently in `db_stats_display.py:29-38`) has no dependency on
  `_DbMixin`-specific state — it only calls `DbMaintenanceService().stats()` and
  `self._out.write_kv(...)`, both available on `_SessionMixin` via `MixinBase`.


   ```python
   def __init__(self, *args, **kwargs) -> None:

       self._title_gen = SessionTitleGen(self._ctx, self._out)
       self._db_session_ops = DbSessionOps(self._ctx, self._out)
   ```

2. Add `_session_export`:
   ```python
   def _session_export(self, args: str) -> None:
       """Export the current conversation history to Markdown or JSON.

       Usage: /session export [markdown|json] [filename]
       """
       ctx = self._ctx
       parts = args.strip().split()
       fmt = "md"
       outfile: str | None = None
       for part in parts:
           if part in ("markdown", "md"):
               fmt = "md"
           elif part == "json":
               fmt = "json"
           else:
               outfile = part
       content = render_export(ctx.conv.history, fmt)
       write_export(content, outfile, len(ctx.conv.history))
   ```
3. Add a dict-dispatch table for the 7 new subcommands, built once (module-level
   function or inline in `_cmd_session`, implementer's choice — a `dict[str, Callable[[str], None]]`
   keyed by subcommand name, values being zero/one-arg closures over `parsed.positional`/
   `parsed.flags`, mirroring the deleted `_cmd_db_session`'s pattern):
   ```python
   def _cmd_session(self, args: str) -> None:
       """Handle /session list [n] | load <id> | rename <title> | delete <id>
       | export markdown|json [file] | stats|health|checkpoint|vacuum|purge|recover.
       """
       parsed = parse_command_args(args.strip().split())
       sub = parsed.subcommand or "list"

       if sub == "list":
           self._session_list(parsed.positional[0] if parsed.positional else "20")
           return
       if sub == "load":
           arg = parsed.positional[0] if parsed.positional else ""
           self._session_load_safe(arg)
           return
       if sub == "rename":
           title = " ".join(parsed.positional)
           if not title:
               self._out.write_validation_error("/session rename <title>")
               return
           self._ctx.session.set_title(title)
           self._out.write_success(f"Session renamed: {title[:50]!r}")
           return
       if sub == "delete":
           arg = parsed.positional[0] if parsed.positional else ""
           self._session_delete(arg)
           return

       rest = " ".join(parsed.positional)
       db_dispatch: dict[str, Callable[[], None]] = {
           "export": lambda: self._session_export(rest),
           "stats": self._db_session_stats,
           "health": self._db_session_ops.health,
           "checkpoint": lambda: self._db_session_ops.checkpoint(rest.strip().upper() or None),
           "vacuum": self._db_session_ops.vacuum,
           "purge": lambda: self._db_session_ops.purge(rest),
           "recover": lambda: self._db_session_ops.recover(rest.strip() or None),
       }
       handler = db_dispatch.get(sub)
       if handler is not None:
           handler()
           return

       self._out.write_validation_error(
           "/session list [n] | /session load <id>"
           " | /session rename <title> | /session delete <id>"
           " | /session export markdown|json [file]"
           " | /session stats|health|checkpoint|vacuum|purge|recover"
       )
   ```
   Note: `rest = " ".join(parsed.positional)` reconstructs the raw remainder string
   `parse_command_args` already split into tokens — verify this is equivalent to the
   old `_cmd_db_session`'s `rest = parts[1] if len(parts) == DB_PARTS_COUNT else ""`
   (raw, unsplit remainder) for `purge`'s `--max-sessions N --max-age-days N` flag
   parsing (`DbSessionOps.purge` re-tokenizes via `parse_command_args(rest.split())`
   internally, so a re-joined-then-re-split string round-trips correctly as long as
   no flag value itself contains embedded whitespace that `parse_command_args`
   already stripped — confirmed safe for this use case).
4. Import `Callable` from `collections.abc` at the top of the file if not already
   present.
5. Add `_db_session_stats` as a method on `_SessionMixin` — moved from
   `db_stats_display.py` (see that file's own implementation doc for the exact
   relocation mechanics: either import a plain helper function or inline the method
   body directly here).

### Method

Additive edits to `_cmd_session` (new tail branches after the existing 4), plus one
new helper method (`_session_export`) and one new `__init__` line. No existing
branch's behavior changes.

### Details

- Cyclomatic complexity guard (per the plan's Risk section): `_session_list` is
  already graded C (CC 14) by `radon cc scripts/agent/commands/cmd_session.py -s -n C`.
  Using dict dispatch for the 7 new branches (rather than extending the if/elif chain
  to 11 total branches) keeps `_cmd_session`'s own complexity from crossing into a
  new C-or-worse grade — re-run `radon cc` after this change to confirm.
- `parse_command_args` (from `agent.commands.utils`, already imported in this file)
  splits `--flag value` pairs into `parsed.flags`; for `purge`, prefer passing the
  raw un-parsed remainder (`rest`) to `DbSessionOps.purge`, which does its own
  `parse_command_args(rest.split())` — do not double-parse flags into `parsed.flags`
  at the `_cmd_session` level for the `purge` subcommand, since `parse_command_args`
  as called on `args.strip().split()` up front will already have consumed
  `--max-sessions`/`--max-age-days` into `parsed.flags` rather than `parsed.positional`
  if the subcommand token itself is `purge`. Verify by tracing a concrete example:
  `/session purge --max-sessions 5` → `args.strip().split()` = `["purge", "--max-sessions", "5"]`
  → `parse_command_args(...)` sets `subcommand="purge"`, `flags={"max-sessions": "5"}`,
  `positional=[]`. Since `purge`'s flags land in `parsed.flags`, not `parsed.positional`,
  reconstruct `rest` for the DB-op branches from the **raw substring after the
  subcommand token** instead of `" ".join(parsed.positional)` — e.g.
  `rest = args.strip()[len(sub):].strip()` computed once before the `sub == "list"`
  checks, or equivalently slice the original `args` string. This is a correction to
  the naive `" ".join(parsed.positional)` sketch above and must be validated against
  a `purge --max-sessions N --max-age-days N` test case before considering this
  branch complete.
