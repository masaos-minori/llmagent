# Implementation: Show RRF Config in write_debug_rag()

## Goal

When `/rag debug` output is displayed, show the active `use_rrf` flag and `rrf_k` value alongside the RRF merge section so operators can confirm which fusion config was in effect.

## Scope

- `scripts/agent/cli_view.py` — update `write_debug_rag()` to print `use_rrf` and `rrf_k` from the data dict

Out of scope:
- Callers that build the `data` dict passed to `write_debug_rag()` (they need a separate update to include `use_rrf` and `rrf_k` keys)
- `pipeline.py` `debug_fn` signature (covered elsewhere)

## Assumptions

1. `write_debug_rag(self, data: dict)` receives a plain dict; the caller is responsible for including `use_rrf` and `rrf_k` keys.
2. If the keys are absent, display should degrade gracefully (use `.get()` with sensible defaults) rather than raise `KeyError`.
3. The existing print-based output format is preserved; no color/rich rendering.
4. Existing test `tests/test_cli_view.py::test_write_debug_rag` must pass after the change (may need updating to include the new keys).

## Implementation

### Target file

`scripts/agent/cli_view.py`

### Procedure

1. In `write_debug_rag()`, before the RRF merge section (currently line 144), extract `use_rrf` and `rrf_k` from `data`.
2. Print a fusion config line showing both values.
3. Update `tests/test_cli_view.py::test_write_debug_rag` to pass `use_rrf` and `rrf_k` in the fixture data and assert the new output line.

### Method

Minimal additive change to the print block. Use `data.get("use_rrf", True)` and `data.get("rrf_k", 60)` to avoid breaking callers that have not yet been updated.

### Details

**cli_view.py** — inside `write_debug_rag()`, insert before the existing RRF merge print (line 144):
```python
use_rrf = data.get("use_rrf", True)
rrf_k = data.get("rrf_k", 60)
print(f"  [debug] fusion: use_rrf={use_rrf} rrf_k={rrf_k}")
```

Full updated block (lines 144–150 area):
```python
use_rrf = data.get("use_rrf", True)
rrf_k = data.get("rrf_k", 60)
print(f"  [debug] fusion: use_rrf={use_rrf} rrf_k={rrf_k}")
print(f"  [debug] RRF merge: {len(merged)} unique candidates (top 5):")
for c in merged[:5]:
    print(
        f"    chunk_id={c.get('chunk_id')}"
        f" rrf={c.get('rrf_score', 0):.4f}"
        f" url={str(c.get('url', ''))[:60]}",
    )
```

**tests/test_cli_view.py** — update the fixture dict passed to `write_debug_rag` to include:
```python
data = {
    ...,
    "use_rrf": True,
    "rrf_k": 60,
}
```
And add an assertion:
```python
assert "fusion: use_rrf=True rrf_k=60" in captured.out
```
