# Implementation: Update _check_services() in repl.py (req 25 — Step 4)

## Goal

Replace the `self._check_service_health()` call in `_check_services()` with `check_readiness()` so production mode fails fast on service unavailability.

## Scope

- `scripts/agent/repl.py`

## Assumptions

1. `check_readiness` is imported from `agent.repl_health`.
2. `production_mode` is already derived from `security_profile == SecurityProfile.PRODUCTION`.
3. The `_check_service_health()` method is removed or kept unused — remove the direct call only.
4. The `_check_service_health()` private method itself can be removed if no other callers exist.

## Implementation

### Target file

`scripts/agent/repl.py`

### Procedure

1. Add `check_readiness` to the import from `agent.repl_health`.
2. In `_check_services()`, replace `await self._check_service_health()` with `await check_readiness(self._ctx, production_mode=production_mode)`.
3. Remove the `_check_service_health()` private method if it has no other callers.

### Method

Edit tool.

### Details

Current `_check_services()`:
```python
async def _check_services(self) -> None:
    production_mode = (
        self._ctx.cfg.mcp.security_profile == SecurityProfile.PRODUCTION
    )
    audit_security_defaults(self._ctx, production_mode=production_mode)
    await self._check_service_health()
    await self._check_tool_definitions()
```

New `_check_services()`:
```python
async def _check_services(self) -> None:
    production_mode = (
        self._ctx.cfg.mcp.security_profile == SecurityProfile.PRODUCTION
    )
    audit_security_defaults(self._ctx, production_mode=production_mode)
    await check_readiness(self._ctx, production_mode=production_mode)
    await self._check_tool_definitions()
```
