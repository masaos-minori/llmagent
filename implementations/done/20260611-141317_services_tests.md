# Implementation: tests/test_services_error_paths.py — Error-path tests for services layer

## Goal

Add error-path unit tests for the refactored service files:
- `IngestStageError` on crawl/split/ingest failure
- Unknown MCP tier → `McpProbeError`
- HTTP error / response mismatch → `SessionTitleGenerationError`
- Write error → `ExportWriteError`
- Invalid field type → `ConfigReloadValidationError`

## Scope

**Target test files**:
- `tests/test_agent_cmd_ingest.py` (extend existing file)
- `tests/test_agent_mcp_status.py` (extend existing file)
- `tests/test_session_title.py` (new file)
- `tests/test_export.py` (new file)
- `tests/test_config_reload.py` (new file)

In scope: New test methods only. Existing tests must not be modified.
Out of scope: Integration tests; tests requiring live DB.

## Assumptions

1. Foundation files (enums, exceptions, models) are already implemented.
2. Each target service file is implemented before its tests are written.
3. Async tests use `pytest.mark.asyncio`.
4. Mocking uses `unittest.mock.patch` / `AsyncMock`.
5. `ExportOutputPort` is tested via a `FakeExportOutput` test double that records calls.

def test_ingest_crawl_failure_raises_stage_error():
    """OSError during crawl raises IngestStageError with stage=CRAWL."""
    from agent.services.enums import IngestStage
    from agent.services.exceptions import IngestStageError
    from agent.services.ingest_workflow import IngestWorkflowService
    import asyncio
    from unittest.mock import patch, AsyncMock

    svc = IngestWorkflowService()
    with patch("rag.ingestion.crawler.WebCrawler") as MockCrawler:
        MockCrawler.return_value.crawl_site = AsyncMock(side_effect=OSError("network fail"))
        with pytest.raises(IngestStageError) as exc_info:
            asyncio.run(svc.run("http://example.com"))
    assert exc_info.value.stage == IngestStage.CRAWL
```

import pytest
from unittest.mock import MagicMock, patch

def test_write_export_os_error_raises_export_write_error(tmp_path):
    from agent.services.export_formatter import write_export
    from agent.services.exceptions import ExportWriteError

    class FakeOut:
        def write(self, content): pass
        def write_file(self, content, path, n): pass

    with patch("pathlib.Path.write_text", side_effect=OSError("disk full")):
        with pytest.raises(ExportWriteError, match="disk full"):
            write_export("content", str(tmp_path / "out.txt"), 5, FakeOut())

def test_write_export_no_file_calls_write():
    from agent.services.export_formatter import write_export

    class FakeOut:
        def __init__(self): self.written = []
        def write(self, content): self.written.append(content)
        def write_file(self, content, path, n): pass

    out = FakeOut()
    write_export("hello", None, 1, out)
    assert out.written == ["hello"]
```

    with pytest.raises(ConfigReloadValidationError):
        svc.apply_config(req)

### Method

`Edit` tool to append to existing test files.
`Write` tool for new test files.

### Details

- Each new test file starts with the `from __future__ import annotations` and `import pytest`.
- `AsyncMock` requires `pytest-asyncio` with `asyncio_mode = auto` (already in `pyproject.toml`).
- `FakeOut` test doubles implement `ExportOutputPort` protocol without explicit inheritance.
- Import the foundation files inside test functions (lazy import pattern) to avoid
  test collection errors if the files don't exist yet.
