# Implementation: mcp_config.py (Phase 5 — enum hardening)

## Goal

Replace `transport: str`, `startup_mode: str`, and `healthcheck_mode: str` in
`McpServerConfig` with `StrEnum` subclasses, catching invalid values at the type layer
rather than in `_validate_*` methods. Eliminate 33 string-literal comparisons across
10 files.

## Scope

**In**:
- `scripts/shared/mcp_config.py` — add `TransportType`, `StartupMode`, `HealthcheckMode` enums;
  update field types; simplify or remove `_validate_*` methods

**Out**:
- No changes to MCP protocol, config file schema, or TOML keys.
  The string values ("http", "stdio" etc.) remain unchanged on disk and in config.

## Assumptions

1. Python 3.13 is available; `StrEnum` is in stdlib since Python 3.11.
2. `str(TransportType.HTTP)` returns `"http"` and `TransportType.HTTP == "http"` is `True`
   with `StrEnum`. This maintains backward compatibility with existing string comparisons.
3. `mcp_config.py` already imports `from enum import Enum` (for `McpServerHealthState`).
4. Callers that build `McpServerConfig(transport="http", ...)` continue to work because
   `StrEnum("http")` accepts the string value as constructor argument.
5. The 33 comparison sites across 10 files must be updated to use enum constants or left
   as-is (StrEnum handles equality with plain strings at runtime, but mypy will flag
   Literal["http"] vs TransportType comparisons if strict).

## Implementation

### Target file

`scripts/shared/mcp_config.py`

### Procedure

1. Add `StrEnum` import: `from enum import StrEnum`.
2. Define three new enums before `McpServerConfig`:
   ```python
   class TransportType(StrEnum):
       HTTP = "http"
       STDIO = "stdio"

   class StartupMode(StrEnum):
       PERSISTENT = "persistent"
       ONDEMAND = "ondemand"
       SUBPROCESS = "subprocess"

   class HealthcheckMode(StrEnum):
       HTTP = "http"
       PROCESS = "process"
       PING_TOOL = "ping_tool"
       EMPTY = ""   # default before __post_init__ sets it
   ```
   Note: `StrEnum` with empty string `""` is valid for the default unset state.
3. Update `McpServerConfig` field types:
   ```python
   transport: TransportType
   startup_mode: StartupMode = StartupMode.PERSISTENT
   healthcheck_mode: HealthcheckMode = HealthcheckMode.EMPTY
   ```
4. Simplify `_validate_transport()` and `_validate_startup_mode()` — the enum
   constructor already rejects invalid values. Keep only the cross-field checks
   (url/cmd presence, subprocess+stdio conflict).
5. Update `__post_init__` default logic for `healthcheck_mode`:
   ```python
   if not self.healthcheck_mode:
       self.healthcheck_mode = (
           HealthcheckMode.HTTP if self.transport == TransportType.HTTP
           else HealthcheckMode.PROCESS
       )
   ```
6. Update `build_mcp_servers()` / loader to cast strings to enum on parse:
   ```python
   transport=TransportType(v["transport"]),
   startup_mode=StartupMode(v.get("startup_mode", "persistent")),
   ```

### Method

Edit `mcp_config.py` with targeted replacements. Then update comparison sites
in the 10 affected files (identified: `mcp_config.py`, `lifecycle.py`, `cmd_mcp.py`,
`factory.py`, `mcp/server.py`, `installer_templates.py`, `repl_health.py`,
plus test files). Use `rg '\.transport\b\|startup_mode\|healthcheck_mode'` to confirm
all sites before and after.

### Details

Impact summary (all confirmed via `rg`):
- `transport` comparisons: 17 sites across 9 files
- `startup_mode` comparisons: 7 sites
- `healthcheck_mode` comparisons: 9 sites (3 files: `mcp_config.py`, `repl_health.py`,
  `installer_templates.py`)

With `StrEnum`, existing `== "http"` comparisons remain valid at runtime. The main
benefit is mypy catching `cfg.transport = "bad_value"` assignments and IDE completion.
