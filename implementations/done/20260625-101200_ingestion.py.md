# memory/ingestion.py — flip write order: SQLite first, JSONL second

**Plan:** `plans/20260625-093754_plan.md` (req #62)
**Target:** `scripts/agent/memory/ingestion.py`

## What to change

Two write sites need the write order flipped. In both cases, currently JSONL is written first
(lines 104, 207), then SQLite. After the change, SQLite is written first (authoritative), then
JSONL (archive, best-effort).

### Site 1: `_persist_entry_with_dedup()` (lines 104-105)

**Before:**
```python
        await self._jsonl.write(entry)
        self._store.upsert(entry, embedding=embedding)
```

**After:**
```python
        self._store.upsert(entry, embedding=embedding)
        try:
            await self._jsonl.write(entry)
        except Exception:
            logger.warning(
                "memory.jsonl_write_failed memory_id=%r — entry saved in SQLite only",
                entry.memory_id,
            )
```

### Site 2: `_persist_entry()` (lines 207-208)

**Before:**
```python
        await self._jsonl.write(entry)
        self._store.upsert(entry, embedding=embedding)
```

**After:**
```python
        self._store.upsert(entry, embedding=embedding)
        try:
            await self._jsonl.write(entry)
        except Exception:
            logger.warning(
                "memory.jsonl_write_failed memory_id=%r — entry saved in SQLite only",
                entry.memory_id,
            )
```

## Rationale

If SQLite write succeeds but JSONL write fails, the entry is in the authoritative store.
The JSONL archive may be missing the entry but that divergence is tolerable — JSONL is
no longer the source of truth.

If SQLite write fails (exception propagates), JSONL is never written, which is correct
(no partial state in the archive for a failed authoritative write).
