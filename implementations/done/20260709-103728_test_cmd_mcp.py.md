# Implementation: M-5 — reload-classification isolation test for /mcp status

Source plan: `plans/20260709-101110_plan.md` (M-5, Implementation step 3).

## Goal

Prove `/mcp status` continues to show the pre-reload server set, URL, and
auth state after a reload classification pass — mirroring the H-8 watchdog
guarantee for this second consumer of `ctx.cfg.mcp.mcp_servers`.

## Scope

**Target**: `tests/test_cmd_mcp.py` (deviates from the plan's suggested
`tests/test_agent_mcp_status.py` — see Assumption 1 for why).

Depends on `implementations/20260709-103709_config_reload.py.md`.

## Assumptions

1. The plan named `tests/test_agent_mcp_status.py` OR `tests/test_cmd_mcp.py`
   as acceptable targets. `test_agent_mcp_status.py` has no full-`ctx` /
   mocked-HTTP-client fixture (it only tests narrow helpers like
   `_tier_for_server` via `McpStatusService.__new__(McpStatusService)`,
   bypassing `__init__`). `test_cmd_mcp.py` already has the exact fixtures
   needed (`_Ctx`, `_Mcp`, `_http()`, `_mock_client()` — verified by reading
   `tests/test_cmd_mcp.py:17-86`), so this doc targets that file instead, to
   avoid duplicating those fixtures.
2. `_Ctx(mcp_servers=...)` (constructor param) directly sets
   `self.cfg.mcp.mcp_servers` — mutating the dict after construction (to
   simulate what `_classify_mcp_server_changes` would see) works the same
   way it does for the direct dict-literal construction used by every other
   test in this file.

## Implementation

### Target file

`tests/test_cmd_mcp.py`

### Procedure

#### Step 1: Add the test to `TestCmdMcpStatus`

Insert after `test_http_connection_error` (or any convenient position in
`class TestCmdMcpStatus`):

```python
    @pytest.mark.asyncio
    async def test_status_unchanged_after_reload_classification(
        self, capsys: pytest.CaptureFixture
    ) -> None:
        from unittest.mock import patch as _patch

        from agent.services.config_reload import ConfigReloadService

        old_srv = _http(url="http://localhost:8080")
        ctx = _Ctx({"svc": old_srv})
        mcp = _Mcp(ctx)

        new_srv = McpServerConfig(
            transport=TransportType.HTTP, url="http://127.0.0.1:9999", cmd=[]
        )
        reload_svc = ConfigReloadService(ctx)  # type: ignore[arg-type]
        with _patch(
            "agent.config_builders._build_mcp_servers",
            return_value={"svc": new_srv, "new_one": new_srv},
        ):
            outcome = reload_svc._classify_mcp_server_changes(ctx, {})  # type: ignore[arg-type]

        resp = MagicMock()
        resp.status_code = 200
        mc = _mock_client()
        mc.get = AsyncMock(return_value=resp)
        with patch("agent.services.mcp_status.httpx.AsyncClient", return_value=mc):
            await mcp._cmd_mcp_status()

        out = capsys.readouterr().out
        assert "new_one" not in out  # pending server not shown until restart
        assert "svc" in out
        assert "http://localhost:8080" not in out or "9999" not in out  # old URL probed, not new
        assert "mcp/svc.url" in outcome.needs_restart
        assert "mcp/new_one (new server)" in outcome.needs_restart
```

### Method

- One additive async test in the existing `TestCmdMcpStatus` class; reuses
  every existing fixture/helper in this file (`_Ctx`, `_Mcp`, `_http`,
  `_mock_client`) with zero new fixtures.
- The URL assertion is written defensively (`"..." not in out or "..." not
  in out`) because `_format_mcp_table`'s exact endpoint column formatting
  was not re-verified at planning time — tighten to a single precise
  assertion once the real table output is confirmed during implementation.
