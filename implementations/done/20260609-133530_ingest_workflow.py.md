# Implementation: ingest_workflow.py — Add logger.exception to exception handlers

## Goal

Use the already-defined `logger` for exception logging in each stage's `except Exception` block,
so failures produce a stack trace in the log file.

## Scope

- `scripts/agent/services/ingest_workflow.py`: add `logger.exception(...)` calls in two `except` blocks.

## Assumptions

1. `logger = logging.getLogger(__name__)` is already defined at module level.
2. `result.error = str(e)` is kept — the command layer still reads it.
3. Adding `logger.exception` does not change observable behavior for tests.

## Implementation

### Target file

`scripts/agent/services/ingest_workflow.py`

### Procedure

**_crawl() except block:**
```python
except Exception as e:
    logger.exception("Ingest crawl stage failed for %r", target)
    result.stage = "crawl"
    result.error = str(e)
    return False
```

**_split_and_ingest() split except block:**
```python
except Exception as e:
    logger.exception("Ingest split stage failed")
    result.stage = "split"
    result.error = str(e)
    return
```

**_split_and_ingest() ingest except block:**
```python
except Exception as e:
    logger.exception("Ingest ingest stage failed")
    result.stage = "ingest"
    result.error = str(e)
```

### Method

Three targeted insertions of `logger.exception(...)` before each `result.stage = ...` line.
