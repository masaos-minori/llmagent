## Goal

Wrap the memory injection call in `StartupOrchestrator._setup_prompt()` in a try/except block so that memory failures do not prevent the agent from starting.

## Scope

**In:**
- `scripts/agent/startup.py`: Add try/except around `ctx.services_required.memory.on_session_start()` in `_setup_prompt()`

**Out:**
- Modifying `session.start()` behavior
- Changing the memory layer itself
- Any other file modifications

## Assumptions

1. The memory layer is explicitly optional — disabled via `use_memory_layer=False` means no memory functionality at all.
2. On failure, the agent should continue with the base system prompt (without memory snippets) rather than failing hard.
3. The warning should use `OutputTag.NON_FATAL` to distinguish from fatal errors.
4. The existing `logger.warning()` call for truncated snippets (line 355) provides an acceptable logging pattern to follow.

## Unknowns

Resolved: No unresolved unknowns remain.

- **UNK-01** (resolved): Whether the outer `except Exception as setup_err` handler in `StartupOrchestrator.run()` would still catch the re-raised exception. Since we are catching the exception inside `_setup_prompt()` and not re-raising it, the outer handler will never see this exception — the agent starts successfully even when memory injection fails.

  ```python
  if ctx.services_required.memory is not None:
      try:
          memory_snippets = ctx.services_required.memory.on_session_start(
              ctx.session.session_id,
          )
          if memory_snippets:
              # ... existing truncation and prompt injection logic ...
      except Exception as exc:
          logger.warning(
              "Memory injection failed during startup: %s; continuing without memory",
              exc,
          )
          self._view.write_warning(f"{OutputTag.NON_FATAL} Memory injection failed: {exc}")
  ```
