# Implementation: Decide archive/remove policy for legacy MCP source docs

Steps covered: Plan 20260626-095917 — Steps 1-4

---

## Goal

Determine whether legacy MCP source documents should be archived (`docs/archive/`) or deleted, and document the policy in `docs/04_mcp_00_document-guide.md`. Then execute the policy.

---

## Scope

- **In scope**:
  - `docs/04_mcp_00_document-guide.md`: policy decision entry
  - Legacy MCP source docs (to be identified): archive or delete
- **Out of scope**: restructured (canonical) docs changes

---

## Assumptions

- Legacy MCP source docs are identified by the document guide as "retained as-is".
- Policy decision: **delete** (preferred; git history enables recovery). Archive only if content is referenced by external systems.

---

## Implementation

### Target file (primary)
`docs/04_mcp_00_document-guide.md`

### Procedure
1. Read `docs/04_mcp_00_document-guide.md` to find legacy doc list.
2. Identify legacy files (e.g., `docs/mcp_old_*.md` or similar).
3. Policy decision:
   ```
   ### Legacy MCP Source Docs — Archive Policy

   **Policy: Delete.** Git history preserves all content. Archive is not required.
   Legacy docs were retained as-is through the restructuring phase (plans 71-76).
   As of 2026-06-26, they are deleted. If recovery is needed, use `git log --all`.

   Deleted files:
   - [list deleted files here after deletion]
   ```
4. Delete the identified legacy docs (or move to `docs/archive/` if needed).
5. Update `docs/04_mcp_00_document-guide.md` to remove "retained as-is" language.

### Method
File deletion or move + documentation update. Use `git rm` for tracked files.

---

## Validation plan

- Pre-commit: `pre-commit run --all-files` — markdown lint must pass.
- Confirm: `grep -rn "retained.*as-is" docs/04_mcp_00_document-guide.md` returns 0.
- Confirm: deleted files no longer exist in `docs/`.
