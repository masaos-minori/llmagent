# Implementation: session.py ‚Äî expose strict_mode and counters

## Goal

Pass `strict_mode` through the `AgentSession` facade to the underlying repository,
and expose skip counters as properties on `AgentSession`.

## Scope

`scripts/agent/session.py` only.

## Assumptions

1. `AgentSession.__init__` currently takes no parameters; adding `strict_mode=False` is backward compatible.
2. `start()` creates a new `SessionMessageRepository` ‚Äî it must also pass `strict_mode`.
3. Counter properties delegate directly to `_message_repo`.

## Implementation

### Target file

`scripts/agent/session.py`

### Procedure

1. Add `strict_mode: bool = False` parameter to `AgentSession.__init__()`.
2. Store as `self._strict_mode` and pass to initial `SessionMessageRepository` construction.
3. In `start()`, pass `strict_mode=self._strict_mode` when reconstructing the repo.
4. Add two `@property` accessors: `skipped_no_session_count` and `skipped_invalid_role_count`.

### Method

Additive only. No existing behavior changes.

### Details

```python
def __init__(self, *, strict_mode: bool = False) -> None:
    self.session_id: int | None = None
    self._strict_mode = strict_mode
    self._message_repo = SessionMessageRepository(self.session_id, strict_mode=strict_mode)
    self._note_repo = NoteRepository()
```

In `start()`:

```python
self._message_repo = SessionMessageRepository(self.session_id, strict_mode=self._strict_mode)
```

New properties after existing delegation methods:

```python
@property
def skipped_no_session_count(self) -> int:
    return self._message_repo.stat_skipped_no_session

@property
def skipped_invalid_role_count(self) -> int:
    return self._message_repo.stat_skipped_invalid_role
```
