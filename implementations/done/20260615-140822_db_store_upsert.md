# Implementation: Phase C — db/store.py INSERT OR REPLACE → ON CONFLICT DO UPDATE

## Goal

`db/store.py` の `doc_insert()` で使っている `INSERT OR REPLACE` を
`INSERT ... ON CONFLICT(url) DO UPDATE SET` に変更し、
既存行の `doc_id` と `chunking_strategy` を保持する。

## Scope

**In**: `scripts/db/store.py` の `doc_insert()` メソッドのみ
**Out**: 他のメソッド・他のファイル

## Assumptions

1. `documents` テーブルの `url` カラムに UNIQUE 制約がある（確認済み）。
2. `INSERT OR REPLACE` は `DELETE + INSERT` のため `doc_id` が変わり、
   CASCADE DELETE が chunks / chunks_fts / chunks_vec に波及する問題がある。
3. `ON CONFLICT(url) DO UPDATE SET` は既存行を in-place 更新するため
   `doc_id` と `chunking_strategy` が保持される。

## Implementation

### Target file

`scripts/db/store.py`

### Method

```python
# 変更前
cur = self._db.execute(
    "INSERT OR REPLACE INTO documents (url, title, lang, etag, last_modified)"
    " VALUES (?, ?, ?, ?, ?)",
    (url, title, lang, etag, last_modified),
)

# 変更後
cur = self._db.execute(
    "INSERT INTO documents (url, title, lang, etag, last_modified)"
    " VALUES (?, ?, ?, ?, ?)"
    " ON CONFLICT(url) DO UPDATE SET"
    "  title = excluded.title,"
    "  lang = excluded.lang,"
    "  etag = excluded.etag,"
    "  last_modified = excluded.last_modified,"
    "  fetched_at = strftime('%Y-%m-%dT%H:%M:%SZ', 'now')",
    (url, title, lang, etag, last_modified),
)
```
