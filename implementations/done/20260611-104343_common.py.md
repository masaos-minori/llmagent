# Implementation: mcp/file/common.py — FileValidationError in check_size_limit

## Goal

Replace `raise ValueError(...)` at line 61 of `scripts/mcp/file/common.py` with
`raise FileValidationError(...)` so that all size-limit violations use the domain
exception consistently, enabling callers and HTTP handlers to catch the right type.

## Scope

**Target file**: `scripts/mcp/file/common.py`
**Test file**: `tests/test_file_common.py`

In scope:
- Change one `raise ValueError(...)` to `raise FileValidationError(...)` in `check_size_limit()`
- Update assertions in `test_file_common.py` that expect `ValueError` from this path

Out of scope:
- Other files; this is a one-line fix
- `resolve_safe()` symlink strictness (separate task)

## Assumptions

1. `FileValidationError` is already defined as `class FileValidationError(ValueError)` in `common.py`.
2. Because `FileValidationError` is a subclass of `ValueError`, existing callers that catch `ValueError`
   will continue to work without modification.
3. The test file has assertions expecting `ValueError`; they must be updated to `FileValidationError`.

## Implementation

### Target file

`scripts/mcp/file/common.py`

### Procedure

1. Open `common.py` and locate `check_size_limit()` (lines 54–64).
2. Change line 61–63:

**Before:**
```python
    if size > max_bytes:
        raise ValueError(
            f"File size exceeds the limit ({max_bytes} bytes): {size} bytes",
        )
```

**After:**
```python
    if size > max_bytes:
        raise FileValidationError(
            f"File size exceeds the limit ({max_bytes} bytes): {size} bytes",
        )
```

3. Open `tests/test_file_common.py` and find every assertion that expects `ValueError`
   from `check_size_limit`. Update to `FileValidationError`.

### Method

Direct `Edit` tool; no AST transformation needed — single-character class name change.

### Details

- `FileValidationError` is already imported within `common.py` (defined in the same file).
- No import additions required.
- The change is backward-compatible for callers catching `ValueError` because `FileValidationError(ValueError)`.

## Validation plan

```bash
uv run pytest tests/test_file_common.py -v
uv run ruff check scripts/mcp/file/common.py
uv run mypy scripts/mcp/file/common.py
```

All tests in `test_file_common.py` must pass. `ruff` and `mypy` must report no new errors.
