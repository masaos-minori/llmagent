# Design: Add error_type attribute to tool_exec audit events

## Goal
Add `error_type` discriminator to audit log entries so operators can distinguish transport errors from tool errors in aggregated logs.

## Target File
- `scripts/agent/tool_audit.py`

## Current State (lines 127-150)
```python
def audit_tool_exec(
    ctx: AgentContext,
    tool_name: str,
    args: dict,
    is_error: bool,
    mcp_request_id: str,
) -> None:
    """Write a tool_exec event with mcp_request_id to the audit log."""
    if ctx.services.audit_logger is None or not mcp_request_id:
        return
    masked = mask_args(args, ctx.cfg.tool.masked_fields)
    resource_scope = _extract_resource_scope(ctx, masked)
    evt = ToolExecEvent(
        event="tool_exec",
        task_id=ctx.turn.current_turn_id or "",
        tool=tool_name,
        operation_type=classify_operation_type(tool_name),
        resource_scope=resource_scope,
        mcp_request_id=mcp_request_id,
        is_error=is_error,
        args_preview=masked,
        ts=time.time(),
    )
    ctx.services.audit_logger.info(_json_dumps(dataclasses.asdict(evt)))
```

Also need to check `ToolExecEvent` model definition.
