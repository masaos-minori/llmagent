# Implementation: Offload all SQLite operations from async endpoints to threadpool

Source plan: `plans/20260625-135630_plan.md` (req #14)

## Goal

Wrap every synchronous SQLite call inside `scripts/eventbus/app.py` with `asyncio.to_thread()` so that no blocking DB I/O executes on the FastAPI event loop thread.

## Scope

- All direct `_db.execute()`, `_db.commit()`, and `.fetchall()` calls inside async functions in `app.py`
- Affected functions: `health()`, `publish()`, `replay()`, `subscribe()` (SSE generator), `dlq_list()`, `dlq_requeue()`, `_get_seq()`
- Out of scope: connection strategy change (req #16), DB facade (req #15), file I/O offload (req #18)

## Assumptions

1. `asyncio.to_thread()` is available (Python 3.9+, confirmed Python 3.13 target)
2. The existing shared `_db` connection with `check_same_thread=False` is retained temporarily (thread-safety addressed in req #16)
3. `_get_seq()` is a sync helper called only from `publish()`; it must become a sync callable passed to `asyncio.to_thread`

## Implementation

### Target file

`scripts/eventbus/app.py`

### Procedure

1. Add `import asyncio` if not already present (already imported at line 3)
2. Wrap `health()` DB call
3. Wrap `publish()` INSERT + commit + `_get_seq()` sequence as a single callable
4. Wrap `replay()` SELECT + fetchall as a single callable
5. Wrap `subscribe()` inner poll loop SELECT + fetchall inside the async generator
6. Wrap `dlq_list()` SELECT + fetchall
7. Wrap `dlq_requeue()` UPDATE + commit
8. Wrap `_dlq_loop()` call to `promote_to_dlq()`

### Method

Use `await asyncio.to_thread(callable)` pattern where `callable` is an inner function (or lambda) that groups all DB calls that must execute atomically (INSERT + commit, UPDATE + commit).

Do **not** split INSERT and commit into two separate `asyncio.to_thread` calls — they must stay in the same thread call to avoid partial commits.

### Details

**health():**
```python
async def health() -> dict[str, str]:
    def _check() -> str:
        try:
            _db.execute("SELECT 1")  # type: ignore[union-attr]
            return "ok"
        except Exception:
            return "unavailable"
    db_status = await asyncio.to_thread(_check)
    ...
```

**publish() — INSERT + commit + seq fetch must be one callable:**
```python
async def publish(request: Request) -> dict[str, Any]:
    ...
    def _insert() -> tuple[int, bool]:
        cur = _db.execute("INSERT OR IGNORE INTO events ...", (...))
        _db.commit()
        inserted = cur.rowcount > 0
        seq = cur.lastrowid if (inserted and cur.lastrowid) else _get_seq(event_id)
        return seq, inserted
    seq, _ = await asyncio.to_thread(_insert)
    ...
```

**replay() — SELECT + fetchall as one callable:**
```python
def _fetch() -> list:
    return _db.execute("SELECT ... FROM events WHERE seq > ? ...", (since_seq,)).fetchall()
rows = await asyncio.to_thread(_fetch)
```

**subscribe() — wrap poll inside async generator:**
```python
async def _sse_gen() -> Any:
    ...
    while True:
        if topic:
            def _poll() -> list:
                placeholders = ",".join("?" for _ in topic)
                return _db.execute(f"SELECT ... WHERE seq > ? AND topic IN ({placeholders}) ...",
                                   (current_seq, *topic)).fetchall()
        else:
            def _poll() -> list:
                return _db.execute("SELECT ... WHERE seq > ? ...", (current_seq,)).fetchall()
        rows = await asyncio.to_thread(_poll)
        ...
        await asyncio.sleep(interval)
```

**dlq_list():**
```python
def _list() -> list:
    return _db.execute("SELECT ... FROM events WHERE dlq_at IS NOT NULL ...").fetchall()
rows = await asyncio.to_thread(_list)
```

**dlq_requeue():**
```python
def _requeue() -> int:
    cur = _db.execute("UPDATE events SET retry_count = retry_count + 1, dlq_at = NULL WHERE event_id = ?", (event_id,))
    _db.commit()
    return cur.rowcount
rowcount = await asyncio.to_thread(_requeue)
```

**_dlq_loop():**
```python
n = await asyncio.to_thread(promote_to_dlq, _db, _cfg.deadletter_dir, _cfg.max_retry)
```
