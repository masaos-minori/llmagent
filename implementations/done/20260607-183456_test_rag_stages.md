# Implementation and Test Procedure: RAG Stages Unit Tests

## Goal
Add unit tests for all RAG pipeline stages to improve test coverage and code quality.

## Scope
This implementation will:
- Create a new test file `tests/test_rag_stages.py`
- Add unit tests for SearchStage, MqeStage, FusionStage, RerankStage, and AugmentStage
- Test normal operation cases and error conditions
- Ensure all pipeline stages properly update PipelineContext state

## Assumptions
- All stage classes are in `scripts/rag/stages/` directory
- Each stage implements the `PipelineStage` protocol with a `run()` method
- PipelineContext has the expected structure (queries, search_results, etc.)
- The test environment can mock external dependencies like LLM calls and database operations

## Implementation
The implementation will create:
1. `tests/test_rag_stages.py` - Main test file for all RAG stages
2. Unit tests for each stage class covering:
   - Normal operation with valid inputs
   - Error handling scenarios
   - PipelineContext state updates
   - Mocked external service calls

## Validation plan
1. Run pytest on `tests/test_rag_stages.py` to ensure all tests pass
2. Verify code coverage for rag/stages/ is ≥ 80%
3. Confirm existing tests still pass
4. Run linting and type checking tools (ruff, mypy)
