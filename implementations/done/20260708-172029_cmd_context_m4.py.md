# Implementation: M-4 — Rename tool_results references/labels to tool_messages in cmd_context.py

## Goal

Rename `breakdown.tool_results`/`.token_tool_results` attribute reads and the `"tool_results"`
display-label strings in the Budget breakdown / Token estimate sections of `/context` to
`tool_messages`, matching the companion `ContextBudget` field rename.

## Scope

**Target**: `scripts/agent/commands/cmd_context.py` — `_cmd_context()` only, specifically the
`total_bd` calculation and the "Budget breakdown:"/"Token estimate:" display loops.

**Depends on / must land together with**: `implementations/20260708-171916_models_m4.py.md`
(the `ContextBudget` field rename) and `implementations/20260708-171946_context_view_m4.py.md`.

**Not part of this doc — do not touch**: the `"Partial compl"` line at line ~71
(`f"{state.partial_completions} stored as tool_result(tool_name='llm_partial_completion')"`) —
that is a COMPLETELY UNRELATED string (about partial-completion persistence, not budget category
naming) already covered by `implementations/20260708-163547_cmd_context.py.md` (an H-4 doc). Do
not conflate the two changes; this M-4 doc touches only the four `tool_results`/
`token_tool_results` occurrences related to `ContextBudget`.

## Assumptions

1. `breakdown.tool_results` appears at line ~45 (`total_bd` sum) and as a display-loop tuple
   entry `("tool_results", breakdown.tool_results)` at line ~90; `breakdown.token_tool_results`
   appears at line ~97 and as a display-loop tuple entry `("tool_results", tt)` at line ~104 (note:
   the SAME string label `"tool_results"` is reused for both the char-budget and token-estimate
   loops — both must be renamed).

## Implementation

### Target file

`scripts/agent/commands/cmd_context.py`

### Procedure

#### Step 1: Confirm all four occurrences

```bash
grep -n "breakdown\.tool_results\|breakdown\.token_tool_results\|\"tool_results\"" scripts/agent/commands/cmd_context.py
```

Expected: four matches within `_cmd_context()` (the `total_bd` line, two display-loop tuples,
and the `tt = breakdown.token_tool_results` assignment).

#### Step 2: Update the `total_bd` calculation

Current (line ~45):

```python
        total_bd = (breakdown.system + breakdown.history + breakdown.tool_results) or 1
```

Replace with:

```python
        total_bd = (breakdown.system + breakdown.history + breakdown.tool_messages) or 1
```

#### Step 3: Update the "Budget breakdown:" display loop

Current (lines ~87-91):

```python
        for cat, n in [
            ("system", breakdown.system),
            ("history", breakdown.history),
            ("tool_results", breakdown.tool_results),
        ]:
```

Replace with:

```python
        for cat, n in [
            ("system", breakdown.system),
            ("history", breakdown.history),
            ("tool_messages", breakdown.tool_messages),
        ]:
```

#### Step 4: Update the token-estimate variable and display loop

Current (lines ~95-105):

```python
            ts = breakdown.token_system
            th = breakdown.token_history
            tt = breakdown.token_tool_results
            if ts is not None and th is not None and tt is not None:
                total_tokens = ts + th + tt
                self._out.write("Token estimate:")
                for cat, n in [
                    ("system", ts),
                    ("history", th),
                    ("tool_results", tt),
                ]:
```

Replace with:

```python
            ts = breakdown.token_system
            th = breakdown.token_history
            tt = breakdown.token_tool_messages
            if ts is not None and th is not None and tt is not None:
                total_tokens = ts + th + tt
                self._out.write("Token estimate:")
                for cat, n in [
                    ("system", ts),
                    ("history", th),
                    ("tool_messages", tt),
                ]:
```

### Method

- Four targeted renames within `_cmd_context()`; the surrounding calculation logic (percentage
  math, `write`/`write_kv` calls), the "Partial compl" line, and every other method in this file
  are untouched.
