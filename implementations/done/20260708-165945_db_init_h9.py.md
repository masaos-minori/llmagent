# Implementation: H-9 — Remove ToolResultStore/ToolResultRow exports from db/__init__.py

## Goal

Remove `ToolResultStore` and `ToolResultRow` from `db/__init__.py`'s imports, `__all__` list,
and the module docstring's "Public API" table entry — both are being deleted from the codebase.

## Scope

**Target**: `scripts/db/__init__.py`

**Depends on**: land together with `implementations/20260708-165904_db_tool_results_delete.py.md`
and `implementations/20260708-165927_db_models_h9.py.md` — importing a deleted module/name here
would raise `ImportError` at package-import time (i.e. as soon as anything does `import db` or
`from db import ...`), which is far more disruptive than a normal unused-import lint warning.

**Out of scope**: every other import/export in this file (`DbConfig`, `SQLiteHelper`,
`DocumentRow`, `MessageRow`, etc.) — unchanged.

## Assumptions

1. `db/__init__.py` re-exports `ToolResultStore` from `db.tool_results` and `ToolResultRow` from
   `db.models` — both modules are having those names/files removed by companion H-9 docs.

## Implementation

### Target file

`scripts/db/__init__.py`

### Procedure

#### Step 1: Update the module docstring's Public API table

Current (line 11):

```
  - db.tool_results: ToolResultStore
```

Remove this line entirely from the docstring's bullet list.

#### Step 2: Remove `ToolResultRow` from the `db.models` import

Current (lines 25-35):

```python
from db.models import (
    DbHealthMetrics,
    DocumentRow,
    MessageRow,
    PurgeCounts,
    RagConsistencyReport,
    RecoveryResult,
    SessionRow,
    ToolResultRow,
    WalCheckpointCounts,
)
```

Replace with:

```python
from db.models import (
    DbHealthMetrics,
    DocumentRow,
    MessageRow,
    PurgeCounts,
    RagConsistencyReport,
    RecoveryResult,
    SessionRow,
    WalCheckpointCounts,
)
```

#### Step 3: Remove the `db.tool_results` import line entirely

Current (line 59):

```python
from db.tool_results import ToolResultStore
```

Remove this line entirely.

#### Step 4: Remove both names from `__all__`

Current (lines 80-81, inside the `__all__` list):

```python
    "ToolResultRow",
    "ToolResultStore",
```

Remove both lines.

### Method

- Four small, mechanical removals across the docstring, one import statement, one whole import
  line, and the `__all__` list — no other exports in this file change.

## Risks

- This file MUST be updated in the exact same commit as the deletion of `db/tool_results.py` and
  the removal of `ToolResultRow` from `db/models.py` — leaving stale imports here after either

  transitively imports `db` would fail at import time, not just callers of the removed
  functionality).
