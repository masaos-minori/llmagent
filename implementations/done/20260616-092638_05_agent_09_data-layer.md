# 09 Data Layer — Agent Documentation Restructuring

## Goal
セッション永続化・メッセージ保存・エクスポート/インポートのデータ層仕様を1章にまとめる。

## Scope
- AgentSession のストレージ実装詳細
- メッセージのシリアライズ/デシリアライズ
- セッション管理操作(一覧・削除・エクスポート)

## Assumptions
- 05_ref-agent-session.md が本章の主ソース
- 05_agent-impl-flow.md §3.3 (セッション保存) と §3.6 (エクスポート処理) を補足として使用
## Implementation

### Target file
`docs/05_agent/09_data-layer.md`

### Procedure
- 05_ref-agent-session.md 全体から AgentSession のAPIと実装notesを抽出
- 05_ref-agent-session.md のスキーマ定義(テーブル構造 or JSON構造)を抽出
- 05_agent-impl-flow.md §3.3 からセッション自動保存のトリガーとタイミングを抽出
- 05_agent-impl-flow.md §3.6 からエクスポート/インポートの処理ステップを抽出

### Method
- H2: AgentSession概要 / ストレージスキーマ / CRUD操作 / エクスポート・インポート
- CRUD操作は「メソッド(シグネチャ) — 説明・副作用」の箇条書き
- スキーマはフィールド名・型・用途の箇条書き

### Details
- ストレージ: SQLiteファイル(session_dir/sessions.db)
- セッションスキーマ: session_id(UUID), created_at, updated_at, metadata(JSON), messages(JSON配列)
- 主要メソッド: save(), load(session_id), list(), delete(session_id), export_json(), import_json()
- 自動保存: ターン終了後に非同期でsave()呼び出し
- エクスポート形式: JSON(messages配列 + metadata)、インポート時はUUID再生成

## Validation plan
- AgentSessionの全パブリックメソッドが05_ref-agent-session.mdと一致していること
- セッションスキーマのフィールドが05_ref-agent-session.mdの定義と矛盾しないこと
- 自動保存のタイミングが05_agent-impl-flow.md §3.3と一致していること
