# Implementation: M-1 — Remove tool-summarize args and forbid the config keys

## Goal

Remove `use_tool_summarize`/`tool_summarize_threshold` from `_build_tool_config()`'s
`ToolConfig(...)` construction, and add both keys to `build_agent_config()`'s
`_FORBIDDEN_KEYS` set so any config file that still contains them fails fast with a clear error
at startup — matching the existing precedent for `workflow_mode`/`workflow_require_approval`.

## Scope

**Target**: `scripts/agent/config_builders.py`

**Depends on / must land together with**: `implementations/20260708-171057_config_dataclasses.py.md`
(the `ToolConfig` field removal this constructor call must match), and
`implementations/*_config_agent_toml_m1.md` (removes `use_tool_summarize`/
`tool_summarize_threshold` from `config/agent.toml` itself — this repo's config file explicitly
sets both keys, so adding them to `_FORBIDDEN_KEYS` without also editing `config/agent.toml`
breaks agent startup entirely; see Details).

**Out of scope**: every other field/kwarg in `_build_tool_config()` and every other function in
this file.

## Assumptions

1. `_FORBIDDEN_KEYS` is currently a local variable inside `build_agent_config()`, not a
   module-level constant (confirmed: `_FORBIDDEN_KEYS = {"workflow_mode",
   "workflow_require_approval"}` appears inside the function body at line 262) — extend it
   in-place rather than converting it to a module-level constant (scope is not part of this
   plan).

## Implementation

### Target file

`scripts/agent/config_builders.py`

### Procedure

#### Step 1: Remove the two kwargs from `_build_tool_config()`

Current (lines 169-179, showing relevant lines):

```python
def _build_tool_config(cfg: dict[str, Any], system_prompt_tool: str) -> ToolConfig:
    return ToolConfig(
        tool_cache_ttl=float(cfg.get("tool_cache_ttl", 300)),
        tool_cache_max_size=int(cfg.get("tool_cache_max_size", 200)),
        serial_tool_calls=bool(cfg.get("serial_tool_calls", False)),
        use_tool_summarize=bool(cfg.get("use_tool_summarize", False)),
        tool_summarize_threshold=int(cfg.get("tool_summarize_threshold", 3000)),
        tool_definitions_strict=bool(cfg.get("tool_definitions_strict", False)),
        ...
```

Remove the `use_tool_summarize=...` and `tool_summarize_threshold=...` lines. The
`serial_tool_calls=...` and `tool_definitions_strict=...` lines become adjacent.

#### Step 2: Extend `_FORBIDDEN_KEYS` in `build_agent_config()`

Current (line 262):

```python
    _FORBIDDEN_KEYS = {"workflow_mode", "workflow_require_approval"}
```

Replace with:

```python
    _FORBIDDEN_KEYS = {
        "workflow_mode",
        "workflow_require_approval",
        "use_tool_summarize",
        "tool_summarize_threshold",
    }
```

### Method

- One two-line deletion inside `_build_tool_config()`, one set-literal extension inside
  `build_agent_config()` — the existing fail-fast mechanism (`found = _FORBIDDEN_KEYS &
  set(cfg.keys())`, raising `ConfigLoadError` listing the offending keys) is reused unchanged.

### Details

- This makes any config file still containing `use_tool_summarize`/`tool_summarize_threshold`
  fail at startup with `ConfigLoadError: Config contains removed keys: ['tool_summarize_threshold',
  'use_tool_summarize']. Remove them from your config file.` (or similar, depending on which keys
  are present) — the chosen backward-compatibility policy per the plan (fail-closed, not
  silent-ignore).
- **`config/agent.toml` in THIS repository explicitly sets both keys** (`use_tool_summarize =
  false` and `tool_summarize_threshold = 3000`, confirmed present). Once `_FORBIDDEN_KEYS`
  includes these, `build_agent_config()` will raise `ConfigLoadError` on every agent startup
  against this file until they are removed. **This is a hard, must-fix-together dependency, not
  an optional follow-up** — see the companion `implementations/*_config_agent_toml_m1.md` doc,
  which removes both lines from `config/agent.toml`. Land that doc in the SAME commit as this one
  and `config_dataclasses.py`'s field removal, or the agent will fail to start immediately after
  this change lands.
