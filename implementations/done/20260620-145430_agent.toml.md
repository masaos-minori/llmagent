# Implementation: config/agent.toml (update — add mdq_rag_mode option)

## Goal

Add `mdq_rag_mode = "auto"` to the `[mcp_servers]` section of `config/agent.toml`
so the MDQ/RAG routing mode can be configured per deployment.

## Scope

**In:**
- Add one key `mdq_rag_mode = "auto"` to the `[mcp_servers]` section (line 304 area)
  or as a standalone entry at the top of the section

**Out:**
- Changes to any `[mcp_servers.*]` subsection
- Schema changes to `MCPConfig` dataclass (a separate step if needed)

## Assumptions

- `[mcp_servers]` section exists at line 304 of `config/agent.toml`
- The config dataclass (`MCPConfig` or `AgentConfig`) will be updated separately to
  expose `mdq_rag_mode` as a typed field; until then `getattr(ctx.cfg, "mdq_rag_mode", None)`
  in the orchestrator handles the missing field gracefully
- Valid values are `"auto"` (default), `"mdq"`, `"rag"` (documented via comment)

## Implementation

### Target file

`config/agent.toml`

### Procedure

1. Locate the `[mcp_servers]` section header (around line 304)
2. Add the following line immediately after the section header, before `[mcp_servers.web_search]`:

```toml
# MDQ/RAG routing mode: "auto" (classifier), "mdq" (force MDQ), "rag" (force RAG)
mdq_rag_mode = "auto"
```

### Method

Single `Edit` targeting the `[mcp_servers]` section header line, inserting the new
key-value pair and comment below it.

### Details

- The comment explains valid values so operators can change without reading source code
- `"auto"` is the safe default — existing behavior is preserved (LLM decides via
  tool descriptions) when the classifier finds no Markdown-structural keywords
- No existing keys in `[mcp_servers]` are affected
