# 04_mcp_99_source_mapping — Implementation Procedure

## Goal
各ソースファイルのセクションと出力ドキュメントのマッピングを明示する。ドキュメント更新時にどのソースを参照すべきかを即座に判断できるようにする。

## Scope
- `docs/04_spec_mcp.md`, `docs/06_ref-mcp.md`, `docs/04_mcp-protocol.md` 全セクション
- `docs/04_mcp-*.md` 各サーバードキュメント (11ファイル)

## Assumptions
- ソースファイルは現行（変更前）ドキュメント; 出力は再構成後のドキュメント
- 1ソースセクションが複数出力ファイルにまたがる場合は全て列挙する

## Implementation

### Target file
`docs/04_mcp_99_source_mapping.md`

### Procedure
- ソースファイル別にセクション→出力ファイルのマッピングテーブルを作成する
- 逆引きテーブル（出力→ソース）と削除候補セクション一覧も作成する

### Method
各ソースファイルの目次と各出力ファイルの Scope セクションを突合してマッピングを構築する。

### Details
- 04_spec_mcp.md §1-5 → 01; §6-8 → 02
- 06_ref-mcp.md §1-2 → 02; §3 → 03
- 04_mcp-protocol.md §1 → 02 (重複確認要); §2 → 03
- 04_mcp-{server}.md (11ファイル) → 04 (各セクション)
- 全サーバーファイルのセキュリティ設定記述 → 05; 運用手順記述 → 06
- 削除候補: 重複した概要説明、古いポート番号記述

## Validation plan
- 全ソースファイルの全セクションがいずれかの出力ファイルにマッピングされていることを確認する
- マッピングテーブルと実際の出力ファイルの Scope セクションが一致することを確認する
- 削除候補セクションが出力に含まれていないことを確認する
