## Goal

Correct `scripts/shared/tool_registry.py`'s module docstring so it no longer describes `ToolRegistry`
as providing "(b) backward-compatible routing fallback when the registry is empty/unavailable" —
that fallback path no longer exists in `route_resolver.py` after this plan's core change lands.
Keep the docstring's accurate description of `ToolRegistry`'s one remaining role: drift-detection
seed data.

## Scope

**In scope**: `scripts/shared/tool_registry.py` module docstring only (current lines 2-9, read
directly from source).

**Out of scope**: Any code in this file (`ToolDefinition`, `ToolRegistry` class, `get_registry()`,
etc.) — the plan explicitly keeps this module structurally untouched; only prose describing its role
changes.
