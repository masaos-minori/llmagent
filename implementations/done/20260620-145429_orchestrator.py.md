# Implementation: scripts/agent/orchestrator.py (update — MDQ/RAG mode injection)

## Goal

Add `_classify_and_inject_mode()` to `Orchestrator` and call it before each LLM turn
so the resolved MDQ/RAG mode is injected as a system prompt hint, guiding tool selection.

## Scope

**In:**
- Add import: `from agent.mdq_rag_classifier import MdqRagMode, resolve_mode`
- Add method `_classify_and_inject_mode(self, query: str) -> None`
- Call `_classify_and_inject_mode(line)` in the main turn loop, before `_sync_system_prompt()`
- Fallback: if MDQ mode is resolved but mdq-mcp tools are unavailable, log warning and
  fall back to RAG hint

**Out:**
- Changes to `ToolRouteResolver`, `ToolPolicy`, or any other module
- Changes to `_sync_system_prompt()` internals

## Assumptions

- `ctx.conv.system_prompt_content` is the mutable system prompt string that
  `_sync_system_prompt()` writes into history[0]
- MDQ tool availability can be checked via `ctx.cfg.mcp_servers` or by checking
  whether any MDQ tool names are in the configured tool set
- The injected hint is appended to (not replaces) the existing system prompt content
- `ctx.cfg` exposes `mdq_rag_mode: str | None` from the new config option
  (see `20260620-145430_agent.toml.md` for config addition)
- The hint text is short (~20-40 tokens): "Use MDQ tools for Markdown-structural queries."
  or "Use RAG tools for semantic/general retrieval."

## Implementation

### Target file

`scripts/agent/orchestrator.py`

### Procedure

1. Add import at top of file (after existing agent imports):
   ```python
   from agent.mdq_rag_classifier import MdqRagMode, resolve_mode
   ```

2. Add method after `_sync_system_prompt()` (around line 370):
   ```python
   def _classify_and_inject_mode(self, query: str) -> None:
       """Inject MDQ/RAG routing hint into system prompt based on query classification."""
       ctx = self._ctx
       config_mode = getattr(ctx.cfg, "mdq_rag_mode", None)
       mode = resolve_mode(query, config_mode)
       if mode == MdqRagMode.MDQ:
           # Verify MDQ tools are available; fall back to RAG if not
           mdq_available = any(
               "search_docs" in (srv.tool_names or [])
               for srv in ctx.cfg.mcp_servers.values()
           )
           if not mdq_available:
               logger.warning("MDQ mode selected but mdq-mcp tools unavailable; falling back to RAG")
               mode = MdqRagMode.RAG
       hint = _mode_hint(mode)
       if hint and ctx.conv.system_prompt_content:
           ctx.conv.system_prompt_content = ctx.conv.system_prompt_content + "\n" + hint
   ```

3. Add module-level helper (pure function, no I/O):
   ```python
   def _mode_hint(mode: MdqRagMode) -> str:
       if mode == MdqRagMode.MDQ:
           return "For this query, prefer MDQ tools (search_docs, outline, get_chunk) for Markdown-structural retrieval."
       if mode == MdqRagMode.RAG:
           return "For this query, prefer RAG tools (rag_run_pipeline) for semantic/general retrieval."
       return ""
   ```

4. In the main turn loop, call `_classify_and_inject_mode(line)` before `_sync_system_prompt()`:
   - Locate the block around line 306-307 where `_handle_memory_injection` and
     `_handle_note_injection` are called
   - Add the call after those injections, before `_sync_system_prompt()` at line 375

### Method

Three targeted edits:

3. `_mode_hint()` helper insertion + call site in turn loop

### Details

- The hint is appended with `"\n"` separator, not replacing the system prompt
- `getattr(ctx.cfg, "mdq_rag_mode", None)` is used to safely access the new config
  field before the config dataclass is updated (avoids AttributeError in transition)
- `logger` is already defined in `orchestrator.py` — no new import needed
- The MDQ availability check uses `srv.tool_names` from the existing server config
  structure; the exact attribute path must be verified against `MCPServerConfig`
