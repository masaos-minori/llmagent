# Implementation: session.py

## Goal

Replace all `except Exception` handlers in `AgentSession` with `sqlite3.Error` (for direct DB calls) and let repository-delegated exceptions propagate, removing `[]`/`False`/`0` fallback returns on error.

## Scope

- Target: `scripts/agent/session.py`
- Direct `SQLiteHelper` calls remain in `start()`, `set_title()`, `list_sessions()`, `delete_last_turn()`, `undo_last_turn()`, `delete_session()` â€” each broad `except Exception` replaced with `sqlite3.Error`
- `start()` raises `sqlite3.Error` on session creation failure instead of silently setting `session_id = None`
- `list_sessions()` raises `sqlite3.Error` instead of returning `[]`
- `delete_last_turn()`, `undo_last_turn()`, `delete_session()` raise `sqlite3.Error` instead of warning+returning `False`/`0`
- Delegated calls (`add_note`, `list_notes`, etc.) already propagate after repository changes

## Assumptions

1. All callers of `session.start()` are in `repl.py._run_repl_loop`; it must catch `sqlite3.Error` and decide whether to continue without session persistence.
2. `cmd_session.py` calls `session.list_sessions()` and `session.delete_session()`; it must handle `sqlite3.Error` for user-facing display.
3. `session.save()` delegates to `session_message_repo.save()`; after `session_message_repo.py` change, `sqlite3.Error` propagates through.

## Implementation

### Target file

`scripts/agent/session.py`

### Procedure

1. Add `import sqlite3` at the top.
2. In `start()`: replace `except Exception as e: logger.warning(...); self.session_id = None` with `except sqlite3.Error: raise`. Update caller in `repl.py` to handle `sqlite3.Error` â€” if it raises, log a warning and continue without persistence (session_id stays None).
3. In `set_title()`: replace `except Exception as e: logger.warning(...)` with `except sqlite3.Error: raise`. Caller `cmd_session.py` catches and displays.
4. In `list_sessions()`: replace `except Exception as e: logger.warning(...); return []` with `except sqlite3.Error: raise`.
5. In `delete_last_turn()`, `undo_last_turn()`, `delete_session()`: same pattern â€” replace broad except with `except sqlite3.Error: raise`.
6. Keep the `if self.session_id is None: return` early-exit guards (not error cases).

### Method

All methods that currently swallow errors become fail-fast. The REPL (`repl.py`) and commands (`cmd_session.py`) become the error display boundary.

### Details

`start()` is the most critical: it creates the session record at REPL startup. If it fails, the REPL should continue with `session_id = None` (no persistence). The exception should be caught at the call site in `repl.py._run_repl_loop`, not inside `start()`:

```python
# repl.py._run_repl_loop
try:
    ctx.session.start()
except sqlite3.Error as e:
    logger.warning(f"Session create failed; history will not be persisted: {e}")
    # session_id remains None; REPL continues
```

All other `session.*` methods used inside commands should propagate `sqlite3.Error` to command handlers, which display an error message.
