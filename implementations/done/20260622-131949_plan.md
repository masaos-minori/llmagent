## Assumptions

1. `_rrf_merge` は `rrf_scores[mid] += 1.0 / (k + rank + 1)` でスコアを累積するため、
   同一リスト内で同じ `memory_id` が複数回出現した場合、両方のランク貢献が加算される。
2. `by_id[mid] = hit` で最後の出現が `MemoryHit` 参照を上書きするが、
   スコア累積は変わらない (`hit.score = rrf_scores[mid]` で上書き済みの hit に適用される)。
3. 同一 RRF スコアの場合の sort 順は Python の `sorted()` の安定性に依存するが、
   テストではスコア同一性の確認のみを行い、順序には依存しない。
4. `k=60` (デフォルト `_RRF_K`) を使用。
5. IEEE 754 浮動小数点では `a + b == b + a` が成立するため、
   `1/61 + 1/62 == 1/62 + 1/61` は `True`。

## Design

### _rrf_merge の実装詳細 (scripts/agent/memory/retriever.py:109-128)

```python
for lst in hit_lists:
    for rank, hit in enumerate(lst):
        mid = hit.entry.memory_id
        rrf_scores[mid] = rrf_scores.get(mid, 0.0) + 1.0 / (k + rank + 1)
        by_id[mid] = hit          # 最後の出現が MemoryHit 参照を上書き
for mid, hit in by_id.items():
    hit.score = rrf_scores[mid]   # 累積スコアを代入
return sorted(by_id.values(), key=lambda h: h.score, reverse=True)
```

### 追加するテストケース

**1. `test_duplicate_memory_id_within_single_list`**
- `hits = [_make_hit("dup"), _make_hit("other"), _make_hit("dup")]`
  (rank 0 と rank 2 に同じ memory_id "dup")
- 期待:
  - 結果は 2 件のみ ("dup" は重複排除される)
  - `score("dup") = 1/(60+0+1) + 1/(60+2+1) = 1/61 + 1/63` (両ランクの累積)
  - `score("other") = 1/(60+1+1) = 1/62`
  - `score("dup") > score("other")` (dup は 2 ランク分の貢献がある)

**2. `test_same_rrf_score_both_entries_present`**
- `list1 = [_make_hit("a"), _make_hit("b")]`
- `list2 = [_make_hit("b"), _make_hit("a")]`
- "a": list1 で rank 0 (`1/61`)、list2 で rank 1 (`1/62`) → `score = 1/61 + 1/62`
- "b": list1 で rank 1 (`1/62`)、list2 で rank 0 (`1/61`) → `score = 1/61 + 1/62`
- 期待:
  - 結果は 2 件 (両方が含まれる)
  - `score("a") == score("b")` (浮動小数点加算の可換性による)
  - 順序には依存しない (stable sort だが同スコアの順序は不定)
