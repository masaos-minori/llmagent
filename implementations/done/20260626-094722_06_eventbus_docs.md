### Procedure

**Architecture doc (06_eventbus_02)**:
```
### Storage Model
SQLite is the canonical event store. JSONL files are write-only (archiving/debugging).
Never read primary data from JSONL; use SQLite queries exclusively.

### Consumer ID Stability
Consumer IDs are generated as `sha256(name:hostname:pid)[:16]`. This ensures:
- Same consumer process always gets the same ID (stable offsets across restarts within a process)
- Different processes get different IDs (collision-resistant)
- Collision detection: duplicate ID with different consumer name raises `ConsumerIdCollisionError`
```

**Operations doc (06_eventbus_05)**:
```
### /health Endpoint Semantics
- HTTP 200: `{"status": "ok"}` — all systems nominal
- HTTP 503: `{"status": "degraded"}` — connected but degraded (slow queries, etc.)
- HTTP 503: `{"status": "unhealthy"}` — DB disconnected or critical failure

Monitoring tools MUST use HTTP status code, not JSON body, for alerting.
```
