# Implementation: Fix `_log_routing_coverage()` in `route_resolver.py`

## Goal

Fix `_log_routing_coverage()` in `scripts/shared/route_resolver.py` to check the registry (Priority 2) before falling back, preventing registry-routed tools from being logged as unmapped.

## Scope

- **In-Scope:** Modify `_log_routing_coverage()` lines 186-194 — insert `_lookup_registry()` check between config_map check and `_fallback_route()` call
- **Out-of-Scope:** Changes to any other method

## Assumptions

1. `_lookup_registry(tool_name)` returns `str | None` — returns the server key string or None (confirmed from source line 160).
2. The method is on `self` — `self._lookup_registry(tool_name)` is valid.
3. No exception can be raised by `_lookup_registry` — it returns None on miss.

## Verified current state

Lines 160-164 of `scripts/shared/route_resolver.py` (confirmed by Read):

```python
def _lookup_registry(self, tool_name: str) -> str | None:
    """Look up a tool in the registry; returns server key or None."""
    if self._registry is not None:
        return self._registry.get_server_for_tool(tool_name)
    return None
```

Lines 182-205 of `scripts/shared/route_resolver.py` (confirmed by Read):

```python
def _log_routing_coverage(self, known_tools: frozenset[str]) -> None:
    """Log routing coverage for all known tools at startup."""
    mapped: list[str] = []
    unmapped: list[str] = []
    for tool_name in sorted(known_tools):
        if tool_name in self._discovery_map or tool_name in self._config_map:
            mapped.append(tool_name)
            continue
        try:
            self._fallback_route(tool_name)
            mapped.append(tool_name)
        except ValueError:
            unmapped.append(tool_name)
    ...
```
