# Implementation: H-8 — tests for process group shutdown (test_lifecycle.py)

## Goal

Add unit tests for `_terminate_with_timeout()` verifying that `os.killpg()` is called
with `SIGTERM` on the process group, that `proc.terminate()` is used as fallback on
`ProcessLookupError`, and that `SIGKILL` is sent after the second timeout.

## Scope

**Target**: `tests/test_lifecycle.py`

**Step covered**: Plan H-8 step 4.

**Out of scope**: source changes, other test files.

## Assumptions

1. `os.killpg` can be monkeypatched in the `http_lifecycle` module namespace.
2. `asyncio.wait_for` is patched or the tests use a real asyncio event loop via
   `pytest-asyncio`.
3. `proc.wait` can be patched to return immediately (simulating fast termination).

## Implementation

### Target file

`tests/test_lifecycle.py`

### Procedure

#### Helper: make_running_proc

```python
def make_running_proc(pid: int = 99999) -> MagicMock:
    proc = MagicMock(spec=subprocess.Popen)
    proc.poll.return_value = None
    proc.pid = pid
    proc.wait = MagicMock(return_value=0)
    return proc
```

#### Test 1: os.killpg called with SIGTERM on process group

```python
@pytest.mark.asyncio
async def test_terminate_uses_killpg_sigterm(monkeypatch):
    import signal as _signal
    import scripts.agent.http_lifecycle as mod

    killed = []

    def fake_killpg(pgid, sig):
        killed.append((pgid, sig))

    monkeypatch.setattr(mod.os, "killpg", fake_killpg)

    mgr = HttpServerLifecycleManager()
    proc = make_running_proc(pid=42)
    mgr._http_pgids["srv"] = 42

    # Patch asyncio.wait_for to complete immediately
    async def fast_wait_for(coro, timeout):
        return await coro

    monkeypatch.setattr(mod.asyncio, "wait_for", fast_wait_for)

    await mgr._terminate_with_timeout(proc, "srv", timeout=1.0)

    assert (42, _signal.SIGTERM) in killed
    proc.terminate.assert_not_called()
```

#### Test 3: Second timeout → os.killpg with SIGKILL

```python
@pytest.mark.asyncio
async def test_terminate_sigkill_on_second_timeout(monkeypatch):
    import signal as _signal
    import scripts.agent.http_lifecycle as mod

    killed = []
    monkeypatch.setattr(mod.os, "killpg", lambda pgid, sig: killed.append((pgid, sig)))

    mgr = HttpServerLifecycleManager()
    proc = make_running_proc(pid=55)
    mgr._http_pgids["srv"] = 55

    call_count = 0

    async def timeout_first_then_ok(coro, timeout):
        nonlocal call_count
        call_count += 1
        if call_count == 1:
            raise asyncio.TimeoutError()
        return await coro

    monkeypatch.setattr(mod.asyncio, "wait_for", timeout_first_then_ok)

    await mgr._terminate_with_timeout(proc, "srv", timeout=1.0)

    assert any(sig == _signal.SIGKILL for _, sig in killed)
```

#### Test 4: start() populates _http_pgids

```python
@pytest.mark.asyncio
async def test_start_populates_http_pgids(tmp_path, monkeypatch):
    import scripts.agent.http_lifecycle as mod

    # Patch os.getpgid to return a known value
    monkeypatch.setattr(mod.os, "getpgid", lambda pid: 9999)
    # Patch _open_stderr_log if H-1 is merged
    _patch_open_to_tmp(monkeypatch, tmp_path)

    mgr = HttpServerLifecycleManager()
    cfg = make_test_cfg(
        cmd=["python", "-c", "import time; time.sleep(60)"],
        startup_timeout_sec=1,
    )

    # Stub health poll to fail immediately so we can catch the error
    with pytest.raises(HttpStartupError):
        await mgr.start("srv", cfg)

    # pgid is captured before health poll starts
    assert mgr._http_pgids.get("srv") == 9999
```

### Method

- `monkeypatch.setattr(mod.os, "killpg", ...)` patches `os.killpg` in the
  `http_lifecycle` module namespace, not the global `os` module.
- `asyncio.wait_for` is patched in the same module namespace for precise control over
  timeouts.
- Tests avoid real subprocess launch where possible; Test 4 starts a real subprocess
  (sleep) but immediately causes health-poll timeout to minimize test duration.

### Details

- Test 1 and 2 do not test the full `start()` path; they call `_terminate_with_timeout`
  directly with a pre-populated `_http_pgids`.
- Import `signal` via `import signal as _signal` to avoid shadowing the `signal` name
  in the test module.
- If `_open_stderr_log` patch helper `_patch_open_to_tmp` was introduced in H-1 tests,
  reuse it here; otherwise provide a local equivalent.
