# Implementation: Document shell-mcp Sandbox Production Guidance

## Goal

Make `sandbox_backend = "none"` explicitly local-only and add a firejail production configuration example across 3 MCP docs.

## Scope

**In:**
- `docs/04_mcp_04_server_catalog.md` — add local vs production config table
- `docs/04_mcp_05_security_and_safety_model.md` — add sandbox_backend=none local-only note
- `docs/04_mcp_06_configuration_and_operations.md` — add firejail example and startup-fail note

**Out:** No code changes.

## Assumptions

1. `sandbox_backend = "none"` disables sandboxing entirely — acceptable only for local/dev use.
2. `sandbox_backend = "firejail"` is the recommended production configuration.
3. Starting without a production sandbox should emit a WARNING or be blocked by strict mode.

## Implementation

### Target file

`docs/04_mcp_04_server_catalog.md`, `docs/04_mcp_05_security_and_safety_model.md`, `docs/04_mcp_06_configuration_and_operations.md`

### Procedure

1. Read the shell-mcp entry in `docs/04_mcp_04_server_catalog.md` (search for `shell` section).
2. Add sandbox config comparison table to shell-mcp server entry.
3. Read the security model section in `docs/04_mcp_05_security_and_safety_model.md`.
4. Add note: "`sandbox_backend = \"none\"` = local/dev only. Do not use in production."
5. Read the configuration section in `docs/04_mcp_06_configuration_and_operations.md` for shell-mcp.
6. Add firejail production example and startup note.

### Method

Read then Edit tool patches.

### Details

**Sandbox config table for `04_mcp_04` shell-mcp entry:**

```markdown
| sandbox_backend | Meaning | Use case |
|---|---|---|
| `"none"` | No sandboxing | Local development only |
| `"firejail"` | firejail process isolation | Production recommended |
| `"docker"` | Docker container isolation | Production alternative |

**Production example (`config/shell_mcp.toml`):**
```toml
[shell_mcp]
sandbox_backend = "firejail"
firejail_profile = "/etc/firejail/shell-mcp.profile"
allowed_commands = ["ls", "cat", "grep"]
```
```

**Security note for `04_mcp_05`:**
```markdown
> **Warning:** `sandbox_backend = "none"` disables all process isolation.
> This is acceptable for local development only. For production, set `sandbox_backend = "firejail"` or `"docker"`.
```

**Startup note for `04_mcp_06`:**
```markdown
If shell-mcp starts with `sandbox_backend = "none"` and `tool_definitions_strict = true`, the agent logs a WARNING.
To enforce production-only sandbox, set `require_sandbox = true` in shell_mcp config — agent will exit if sandbox is unavailable.
```
