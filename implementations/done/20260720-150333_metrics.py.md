## Goal

Track `browser_fetch` query counts/latency/error state independently of `search_web`'s metrics, so
`browser_fetch` traffic/failures don't skew `search_web`'s aggregate stats in `/health`'s
`metrics.snapshot()` output (companion to the `health.py` doc's degradation-isolation goal).

## Scope

**In scope**: add a second `WebSearchMetrics` singleton (`_browser_metrics`) and a
`record_browser_query()`/`browser_snapshot()` pair, in
`scripts/mcp_servers/web_search/metrics.py` (current: 78 lines).
**Out of scope**: `WebSearchMetrics` dataclass shape, and the existing `record_query`/`snapshot`/
`reset` functions for `_metrics` (DuckDuckGo/search_web) ‚Äî untouched, external contract stable.

## Assumptions

1. Same two-singleton choice as `health.py` (see that doc's Assumption 1) ‚Äî for the same reasons
   (only two tools exist in this package; no dict-keyed generalization needed).
2. `record_browser_query(success: bool, latency_ms: float, error_type: str = "")` mirrors
   `record_query`'s exact signature ‚Äî `service.fetch_browser()` (per that doc) calls this instead of
   `record_query` for `browser_fetch` outcomes.
3. `WebSearchMetrics`'s existing invariant "never accepts a query string anywhere" (module
   docstring, lines 15-17) extends identically to the browser case: `record_browser_query` must
   never accept a URL string either ‚Äî only `success`/`latency_ms`/`error_type`, consistent with the
   file's stated privacy/security property (no full request content ever enters the metrics layer).

## Implementation

### Target file

`scripts/mcp_servers/web_search/metrics.py`

### Procedure

1. Add `_browser_metrics = WebSearchMetrics()` alongside the existing `_metrics = WebSearchMetrics()`
   (line 46).
2. Add `record_browser_query(success: bool, latency_ms: float, error_type: str = "") -> None`,
   body identical to `record_query` but mutating `_browser_metrics`.
3. Add `browser_snapshot() -> dict[str, object]`, identical shape to `snapshot()` but reading
   `_browser_metrics`'s fields.
4. Add `reset_browser() -> None` (test helper, mirrors `reset()`).
5. Update the module docstring's "never accepts a query string" claim to note it applies to
   `record_browser_query` too (one-line addition, per Assumption 3).

### Method

Straight duplication of the existing `record_query`/`snapshot`/`reset` trio against a second
`WebSearchMetrics()` instance ‚Äî same no-generalization rationale as the companion `health.py` doc.
