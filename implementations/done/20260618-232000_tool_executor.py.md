# Design: Add transport vs tool error distinction in ToolExecutor metrics

## Goal
Distinguish transport-level errors from tool-level errors in metrics/tracing, add per-server tool error counters to `ToolExecutor`.

## Target File
- `scripts/shared/tool_executor.py`

## Current State (lines 386-606)
- `ToolExecutor.__init__()`: creates `_transports`, `_resolver`, health registry reference
- `_raw_execute()`: calls transport, catches `TransportError` → returns `is_error=True`; successful transport with `is_error=True` in response body treated as success for health purposes (lines 516-521)
- Health registry tracks success/failure but only for transport failures
- No error_type attribute anywhere
- No per-server error counters
- Audit log via `audit_tool_exec()` has `is_error: bool` but no `error_type`

## Implementation Steps

### Step 1: Add error type counter tracking to ToolExecutor.__init__()

Add after line 431 (after `_resolver` init):
```python
# Per-server error counters: server_key -> {"transport": int, "tool": int}
self._error_counters: dict[str, dict[str, int]] = {
    key: {"transport": 0, "tool": 0} for key in server_configs
}
```

### Step 2: Add error_type attribute to ToolCallResult

Modify `ToolCallResult` dataclass (lines 49-57):
```python
@dataclass(frozen=True)
class ToolCallResult:
    """Typed result from a single tool call execution."""

    output: str
    is_error: bool
    request_id: str  # x-request-id from HTTP transport; "" for stdio/plugin/cache
    server_key: str  # server key that handled the call; "" for plugin tools
    error_type: str = ""  # "transport" | "tool" | "" (none for success)
```

### Step 3: Set error_type in HttpTransport.call()

In `_parse_http_response()` (lines 101-125), add `error_type` to return:
```python
return ToolCallResult(
    output=result_val,
    is_error=is_error_val,
    request_id=x_request_id,
    server_key="",  # set by caller
    error_type="tool" if is_error_val else "",
)
```

In `HttpTransport.call()` (lines 127-168), transport errors already raise `TransportError` — no change needed for error_type there (set in `_raw_execute`).

### Step 4: Set error_type in StdioTransport.call()

Similar to HttpTransport — in `_parse_stdio_response()` (lines 228-240):
```python
return ToolCallResult(
    output=str(resp["result"]),
    is_error=bool(resp["is_error"]),
    request_id="",
    server_key="",  # set by caller
    error_type="tool" if resp.get("is_error") else "",

```

### Step 5: Increment counters and set error_type in _raw_execute()

Replace lines 514-534:
```python
try:
    result = await _do_call()
    # Transport succeeded — record success regardless of tool-level errors.
    if self._health_registry is not None:
        self._health_registry.record_success(server_key)
    return result
except TransportError as e:
    # Transport-level failure — increment counter.
    counters = self._error_counters.get(server_key, {"transport": 0, "tool": 0})
    counters["transport"] += 1
    if self._health_registry is not None:
        state = self._health_registry.record_failure(server_key)
        logger.warning(
            "transport failure for %r: %s (state=%s)",
            server_key,
            e,
            state.value,
        )
    return ToolCallResult(
        output=str(e), is_error=True, request_id="", server_key=server_key, error_type="transport"
    )
```

Add tool error counting in the success path (after `result = await _do_call()`):
```python
if result.is_error and result.error_type == "tool":
    counters = self._error_counters.get(server_key, {"transport": 0, "tool": 0})
    counters["tool"] += 1
```

### Step 6: Add getter method for error counters

Add after `clear_cache()` (line 605):
```python
def get_error_counters(self) -> dict[str, dict[str, int]]:
    """Return per-server error counters: {server_key: {transport: N, tool: N}}."""
    return {k: dict(v) for k, v in self._error_counters.items()}
```

## Completion Criteria
- `ToolCallResult` has `error_type` field (default "")
- Transport errors set `error_type="transport"`
- Tool errors set `error_type="tool"`
- Per-server counters track both types
- `get_error_counters()` returns current counts
