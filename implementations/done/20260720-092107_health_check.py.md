### Procedure

   the return-value handling on lines 35-38 are all unchanged.)

### Method

change to the function's `try`/`except`/return shape. Since the substituted SQL text is
byte-identical, this is a refactor with zero expected behavioral delta; the validation plan below
confirms this via the existing `tests/test_mdq_health_stale.py` continuing to pass unmodified.

### Details

- Keep the bare `except Exception: return None` in `_check_stale_documents()` as-is â€” it is
  pre-existing and out of scope for this plan (no constraint in this plan concerns broad
  exception handling in this function; the project's "no bare `except:`" constraint from
  `rules/coding.md` only prohibits a bare `except:` with no exception type at all, which this is
  not â€” `except Exception:` names a type).
- Confirm after the edit that `health_check.py` has zero remaining occurrences of the raw literal
  `"mtime_ns > CAST(indexed_at * 1e9 AS INTEGER)"`.
