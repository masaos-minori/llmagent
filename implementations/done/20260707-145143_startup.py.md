# Implementation: M-1 — Production fail-fast for subprocess startup failures (startup.py)

## Goal

Update `StartupOrchestrator._start_servers()` to re-raise on MCP subprocess startup
failure when `SecurityProfile.PRODUCTION` is active, while keeping the existing
non-fatal warning behavior for the local profile.

## Scope

**Target**: `scripts/agent/startup.py`

**Step covered**: Plan M-1 step 1.

**Out of scope**: `factory.py`, `http_lifecycle.py`, `context.py`, config files.

**Dependency**: H-6 rollback wrapper in `run()` must be merged first (or simultaneously),
so that `shutdown_all()` fires automatically when `_start_servers()` raises.

## Assumptions

1. `SecurityProfile` is already imported at line 13 of `startup.py` (confirmed).
2. The `except (OSError, RuntimeError)` block in `_start_servers()` currently catches
   per-server startup failures and writes a warning — confirmed by source.
3. `ctx.cfg.mcp.security_profile` holds the active `SecurityProfile` value.
4. The `raise` propagates from `_start_servers()` through H-6's wrapper in `run()`,
   which calls `lifecycle.shutdown_all()` before re-raising.

## Implementation

### Target file

`scripts/agent/startup.py`

### Procedure

#### Step 1: Locate the except block in `_start_servers()`

Find the loop over subprocess-mode servers and its `except (OSError, RuntimeError)` handler.
The existing handler writes a warning and continues:
```python
except (OSError, RuntimeError) as e:
    logger.warning("Lifecycle: %r failed to start: %s", key, e)
    self._view.write_warning(f"[WARNING] MCP server {key!r} failed to start: {e}")
```

#### Step 2: Add production-profile check

Replace the handler body with:
```python
except (OSError, RuntimeError) as e:
    if self._ctx.cfg.mcp.security_profile == SecurityProfile.PRODUCTION:
        msg = f"[FATAL] MCP subprocess {key!r} failed to start: {e}"
        logger.error(msg)
        raise RuntimeError(msg) from e
    logger.warning("Lifecycle: %r failed to start: %s", key, e)
    self._view.write_warning(f"[WARNING] MCP server {key!r} failed to start: {e}")
```

The production `raise` propagates out of `_start_servers()` and is caught by H-6's
`run()` wrapper, which calls `lifecycle.shutdown_all()` then re-raises.

### Method

- Minimal: insert one `if` check before the existing `logger.warning` line.
- No new imports needed (`SecurityProfile` is already imported).

### Details

- `ctx.cfg.mcp.security_profile` type: confirm it is `SecurityProfile` enum (not `str`)
  before implementing. If it's a `str`, use `== SecurityProfile.PRODUCTION.value` or
  add the enum conversion.
- The `from e` chaining preserves the root cause in the traceback.
- Error message includes `key` (server name) for operator diagnostics.
