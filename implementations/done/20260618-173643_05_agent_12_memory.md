# Implementation: Create `docs/05_agent_12_memory.md`

## Goal

Create a new documentation file that serves as an API reference for all 14 memory layer modules. A developer should understand each module's responsibility, public API surface, and disabled behavior without reading source code.

## Scope

**In scope:**
- `docs/05_agent_12_memory.md` — create new file

**Out of scope:**
- Code changes
- Dashboard/UI documentation
- Test documentation

## Assumptions

1. Each module's `__all__` or `__init__.py` exports represent the intended public API.
2. Disabled behavior follows the `ctx.services.memory is None` pattern (factory gating), not per-module flags.

## Implementation

### Target file

`docs/05_agent_12_memory.md`

### Procedure

1. Document module responsibility map as an ASCII/table diagram.
2. For each of the 14 modules, write a section with: responsibility, public classes/functions, key parameters, failure modes.
3. Include a dedicated "Disabled Behavior" section explaining the gating pattern.

### Method

Write the doc following the conventions of existing `docs/05_agent_*.md` files (half-width alphanumeric, Markdown, concise).

### Details

**Document structure:**

```
# Memory Layer — Module Reference

## Overview

(Table of all 14 modules with one-line responsibility)

## Module Reference

### 1. `__init__.py` — Public API barrel

Re-exports all public symbols. List key categories (types, enums, services, etc.).
Notable: includes internal mapper utils (`_floats_to_blob`) in `__all__`.

### 2. `types.py` — Core runtime types

- `MemoryEntry`: persisted entry (memory_id, memory_type, source_type, session_id, content, summary, tags, importance, pinned, timestamps, project/repo/branch)
- `MemoryQuery`: search input (query text, memory_type, limit)
- `MemoryHit`: ranked result (entry + score)
- `EmbeddingResult`: success/failure + embedding vector + error_kind
- `EmbeddingErrorKind`: DISABLED, TIMEOUT, HTTP_ERROR, CIRCUIT_OPEN, INVALID_RESPONSE, UNKNOWN_ERROR
- `SourceType`: RULE, CONVERSATION, FAILURE, MANUAL, etc.

### 3. `enums.py` — Domain enums

- `MemoryType`: semantic, episodic
- `DedupAction`: SKIP_NEW, ALWAYS_WRITE
- `DedupPolicy`: action + threshold

### 4. `exceptions.py` — Exception hierarchy

- `MemorySchemaError` — invalid data schema
- `MemoryConsistencyError` — FTS/vec consistency failure
- `ExtractionError` — extraction failure
- `InjectionValidationError` — invalid injection input
- `EmbeddingProtocolError` / `EmbeddingTransportError` — embedding client errors
- `JsonlFormatError` — malformed JSONL
- `UnknownMemoryTypeError` — unrecognized type string

### 5. `models.py` — Frozen DTOs

- `HistoryMessage`: (role, content)
- `MemorySnippet`: (text, source)
- `ConsistencyReport`: (memories, fts, vec) row counts

### 6. `store.py` — CRUD layer

Class `MemoryStore`:
- `add(entry, embedding?)` — insert + FTS sync; BEGIN IMMEDIATE
- `upsert(entry, embedding?)` — insert-or-replace + FTS sync
- `delete(memory_id)` → bool
- `clear_by_session(session_id)` → int
- `search_by_type(memory_type, limit, min_importance)` → list[MemoryEntry]
- `count_by_type()` → dict[str, int]
- `count_vec()` → int
- `check_consistency()` → ConsistencyReport
- `pin(memory_id)` / `unpin(memory_id)` → bool
- `get_by_id(memory_id)` → MemoryEntry | None
- `count_entries()` → int
- `count_prunable(days)` → int

Failure modes: sqlite3.OperationalError (DB locked, missing vec table), MemoryConsistencyError (FTS count fails).

### 7. `retriever.py` — Search (FTS5 + KNN + Hybrid)

Classes: `FtsRetriever`, `VectorRetriever`, `HybridRetriever`.

`HybridRetriever` (primary interface):
- `search(query, embedding?, project?, repo?)` → FTS-only when no embedding, RRF merge when embedding present
- `knn_search(embedding, memory_type, limit)` → delegate to VectorRetriever
- `top_semantic(limit, min_importance, project?, repo?)` → direct SQL (no FTS)

Scoring formula:
- score = -bm25_rank + importance_boost + pin_boost + recency_decay + context_match
- Semantic: importance_weight=1.0, recency_weight=0.5
- Episodic: importance_weight=0.5, recency_weight=1.0

Failure modes: `sqlite3.OperationalError` (vec table missing → KNN returns []), `MemorySchemaError` (invalid created_at).

### 8. `injection.py` — Lifecycle injection service

Class `MemoryInjectionService`:
- `on_session_start()` → list[MemorySnippet] — top semantic entries
- `on_user_prompt(query, session_id)` → list[MemorySnippet] — semantic + episodic search with embedding

Uses `InjectionPolicy`: max_semantic (5), max_episodic (3), min_importance (0.3), format prefixes.

Failure modes: `InjectionValidationError` (empty query).

### 9. `ingestion.py` — Extraction + dedup + persist

Class `MemoryIngestionService`:
- `on_session_stop(session_id, history, turn_id?)` — extract, dedup, persist
- `write_semantic(session_id, content)` / `write_episodic(session_id, content)` — manual persist (no dedup)

Uses `DedupPolicy`: SKIP_NEW + threshold-based KNN dedup.

Failure modes: `sqlite3.OperationalError` (memory_links insert swallowed), embedding failures handled gracefully.

### 10. `extract.py` — Rule-based extraction

- `extract_memories(history, ...)` → list[MemoryEntry] — entry point
- `ExtractionPolicy`: min_content_chars (80), min_turns (2), max_entries (20)
- Classification: semantic (rules/decisions), episodic (failures, Q&A)
- Importance heuristic: 0.4 base + length_bonus + keyword_bonus

### 11. `jsonl_store.py` — JSONL source of truth

Class `JsonlMemoryStore`:
- `write(entry)` — async append (asyncio.Lock)
- `read_all()` → list[MemoryEntry] — sync read

Failure modes: `JsonlFormatError` on malformed lines.

### 12. `embedding_client.py` — HTTP embedding client

Class `EmbeddingClient`:
- `fetch(text)` → EmbeddingResult — with retry + circuit breaker

`EmbeddingClientConfig`: embed_url, timeout (5s), max_retries (2), circuit_open_after (3), circuit_reset_sec (60).

Disabled behavior: when `enabled=False`, `fetch()` returns `DISABLED` error immediately.

Failure modes: non-DISABLED errors propagate as `EmbeddingResult` with appropriate error_kind.

### 13. `services.py` — MemoryServices facade

Class `MemoryServices`:
- `injection`, `ingestion`, `store`, `retriever` — public attributes
- `on_session_start(session_id?)` → delegates to injection
- `on_session_stop(session_id, history, turn_id?)` → delegates to ingestion
- `on_user_prompt(query, session_id)` → delegates to injection

### 14. `mapper.py` — Row conversion utils

- `row_to_entry(dict)` → MemoryEntry — SQLite row to MemoryEntry
- `_floats_to_blob(list[float])` → bytes — embedding to sqlite-vec blob
- `_stamp_entry(entry, now)` → MemoryEntry — set created_at/updated_at
- `_now_iso()` → str — UTC ISO timestamp

## Disabled Behavior

When `config.memory.use_memory_layer` is `False`:
1. `factory._build_memory_services()` returns `None` (not called in factory).
2. `ctx.services.memory` is `None`.
3. All callers check `ctx.services.memory is None` before access (e.g., `cmd_memory.py`, `context.py`).
4. `EmbeddingClient` has its own `enabled` flag — when disabled, returns `DISABLED` error without HTTP call.
5. `cli_view.py` shows `Memory layer` status as disabled.

No individual module implements its own enabled/disabled gate — the factory-level None pattern covers all operations.
