## 変更 2: `run_stdio()` — `asyncio.CancelledError` キャッチ + エラーレスポンス書き込み後に再 raise

### 変更箇所: 最後の `except` ブロック (line 253-263) の後、`write()` の前に追加

`asyncio.CancelledError` は Python 3.8+ では `BaseException` を継承するため、現状の `except` ブロックではキャッチされない。キャンセル時にエラーレスポンスを書かずにループを抜けると、呼び出し元が応答を受け取れないまま詰まる。

**追加内容:** 既存の `except` ブロック全体を以下に変更:

```python
            except asyncio.CancelledError:
                result = "Server cancelled"
                is_error = True
                # Write error response before re-raising
                resp = _json_dumps(
                    {
                        "id": req_id,
                        "result": result,
                        "is_error": is_error,
                        "truncated": False,
                        "total_bytes": 0,
                        "actual_visible_bytes": 0,
                    },
                )
                loop.run_in_executor(None, _write_stdout, resp + "\n")
                raise
            except orjson.JSONDecodeError as e:
                ...  # 既存
```

## 変更 3: stdout 書き込みを非同期に変更

### UNK-01 調査が必要な箇所

`asyncio.loop.connect_write_pipe()` がWSL2 Python 3.11で動作するかをスモークテストで確認:

```bash
uv run python -c "
import asyncio, sys

async def test():
    loop = asyncio.get_running_loop()
    transport, proto = await loop.connect_write_pipe(
        asyncio.BaseProtocol, sys.stdout.buffer
    )
    transport.write(b'ok\n')
    transport.close()

asyncio.run(test())
"
```

### connect_write_pipe が使える場合の実装

`run_stdio()` の先頭でトランスポートを取得:

```python
        loop = asyncio.get_running_loop()
        reader = asyncio.StreamReader()
        protocol = asyncio.StreamReaderProtocol(reader)
        await loop.connect_read_pipe(lambda: protocol, sys.stdin.buffer)

        # stdout async writer (UNK-01: verify WSL2 compatibility)
        writer_transport, _ = await loop.connect_write_pipe(
            asyncio.BaseProtocol, sys.stdout.buffer
        )
```

書き込み箇所:
```python
            writer_transport.write((resp + "\n").encode())
```

### connect_write_pipe が使えない場合のフォールバック

```python
def _write_stdout(data: str) -> None:
    sys.stdout.write(data)
    sys.stdout.flush()

# loop.run_in_executor(None, _write_stdout, resp + "\n")
```

**実装方針:** UNK-01 スモークテスト結果に応じて以下を選択:
- `connect_write_pipe` 成功 → `writer_transport.write()`
- 失敗 → `loop.run_in_executor(None, _write_stdout, resp + "\n")`
