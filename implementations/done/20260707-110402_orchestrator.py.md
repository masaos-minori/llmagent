## Goal
Remove all `WorkflowExecutionPolicy`-based branching from `Orchestrator`: delete the `workflow_mode` constructor parameter, the `self._policy` field, the disabled-mode direct-execution branch in `handle_turn()`, the `_log_fallback()` policy checks, and all error messages suggesting `workflow_mode=disabled`.

## Scope
**In**: `scripts/agent/orchestrator.py` — lines 47, 112, 121, 132–142, 146–161, 163–173, 195–208; all `self._policy.*` calls; all strings containing `"set workflow_mode=disabled in config"`.
**Out**: `_handle_workflow_engine()`, `_process_turn()`, `_handle_turn_end()` internals remain unchanged. `require_approval=` kwarg to `WorkflowEngine` handled separately (req05).

## Assumptions
- After this change, `handle_turn()` always goes to `_handle_workflow_engine()` or raises if `self._workflow_def is None`.
- `WorkflowCreationError` class can be removed (no direct-execution fallback remains).
- `workflow_status()` returns only `{"tracking": "enabled"|"not_loaded"}` — remove `mode` key.
- `self._workflow_def = WorkflowLoader().load()` becomes unconditional (no policy guard).

## Implementation

**Target file**: `scripts/agent/orchestrator.py`

**Procedure**:

2. Remove `workflow_mode: str = "auto"` parameter from `__init__` (line 112)
