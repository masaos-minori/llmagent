# Implementation: tool_cache.py (Phase 3 — responsibility separation)

## Goal

Extract `_CacheEntry` and `_execute_with_cache()` from `scripts/shared/tool_executor.py`
into a new `scripts/shared/tool_cache.py` module, reducing `tool_executor.py` from
597 lines to ~530 and giving the cache logic its own test surface.

## Scope

**In**:
- New file `scripts/shared/tool_cache.py`
- `scripts/shared/tool_executor.py` — remove `_CacheEntry` and `_execute_with_cache`,
  import them from `tool_cache`

**Out**: No changes to callers of `ToolExecutor`. No public API change.

## Assumptions

1. `_CacheEntry` is a `@dataclass(frozen=True)` with `output: str`, `is_error: bool`,
   `cached_at: float` (confirmed at `tool_executor.py:63-73`).
2. `_execute_with_cache()` is an `async` method on `ToolExecutor`; it calls
   `self._raw_execute()`, `self._cache` (an `OrderedDict`), `self._cache_ttl`,
   `self._cache_max_size`, `self.stat_cache_hits`. Extracting it as a free function
   would require passing these as arguments — too invasive. Keep it as a method but move
   `_CacheEntry` to `tool_cache.py`.
3. `tool_cache.py` must stay within `shared/` — no `agent/` or `db/` imports (lint-imports).

## Implementation

### Target file

`scripts/shared/tool_cache.py` (new), `scripts/shared/tool_executor.py` (modify)

### Procedure

1. Create `scripts/shared/tool_cache.py` with `_CacheEntry` dataclass.
2. In `tool_executor.py`: remove the `_CacheEntry` class; add
   `from shared.tool_cache import _CacheEntry`.
3. Run lint and mypy.

### Method

**`scripts/shared/tool_cache.py`**:

```python
#!/usr/bin/env python3
"""shared/tool_cache.py
Cache entry dataclass for ToolExecutor's LRU result cache.
"""

from __future__ import annotations

from dataclasses import dataclass


@dataclass(frozen=True)
class _CacheEntry:
    """Internal LRU cache entry storing a successful tool call result."""

    output: str
    is_error: bool
    cached_at: float
```

**`tool_executor.py`** change:
- Remove lines 63-72 (`@dataclass(frozen=True)\nclass _CacheEntry:...`)
- Add after the `TransportErrorInfo` import block:
  `from shared.tool_cache import _CacheEntry`

### Details

`_execute_with_cache()` stays in `ToolExecutor` as a method (it accesses `self._cache`,
`self._cache_ttl`, etc.) — moving it to a free function would require 5+ extra parameters
with no net simplification. The split boundary is `_CacheEntry` (pure data) vs the cache
management logic (stateful, stays in `ToolExecutor`).
