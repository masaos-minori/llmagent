### Details

**New file `scripts/agent/turn_result.py`:**
```python
"""agent/turn_result.py
TurnResult frozen dataclass for LLM turn lifecycle.

Defined separately from orchestrator.py so that llm_turn_runner.py
can import TurnResult without creating a circular dependency.
"""
from __future__ import annotations

import dataclasses
from typing import TYPE_CHECKING, Literal

if TYPE_CHECKING:
    from shared.llm_client import LLMTransportError


@dataclasses.dataclass(frozen=True)
class TurnResult:
    """Typed result of one LLM turn."""
    action: Literal["continue", "fail"]
    answer: str = ""
    error_kind: str | None = None
    reason: str = ""
    exception: LLMTransportError | None = dataclasses.field(default=None, compare=False)

    @property
    def success(self) -> bool:
        """Backward-compatible: True when action == 'continue'."""
        return self.action == "continue"
```

**`orchestrator.py` changes:**
- Remove lines 30-36 (`class TurnResult` and its `__init__`).
- Add to imports: `from agent.turn_result import TurnResult`
- Change any `TurnResult(success=True, ...)` → `TurnResult(action="continue", ...)` if present
  (Step 3 will handle `_handle_llm_turn` body, but the import must be correct here).

**Key design decisions:**
- `exception: LLMTransportError | None` with `compare=False` — exception objects are not
  equality-comparable; frozen dataclasses would fail on `==` without this.
- `TYPE_CHECKING` guard — `LLMTransportError` is only needed for type annotations, not at
  runtime. Avoids import overhead and potential circular imports.
- `@property success` — backward compatibility for `_process_turn` which checks `result.success`.
