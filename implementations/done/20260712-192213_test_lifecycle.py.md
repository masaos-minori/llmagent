Companion implementation doc: `implementations/20260712-192149_http_lifecycle.py.md`

## Goal

`tests/test_lifecycle.py` has a unit test proving that `shutdown_all()` completes
termination of all managed servers even when a second SIGINT arrives mid-loop, and that the
SIGINT handler is restored to its prior state afterward — without sending any real OS-level
signal.

## Scope

**In scope:** one new test method added to `TestShutdownAllCleanup`
(`tests/test_lifecycle.py:616-674`).

**Out of scope:**
- Any other test class in the file
- Real-signal (`os.kill`/`signal.raise_signal`) based integration tests — explicitly deferred
  by the requirement due to CI stability risk; this test simulates the interrupt by invoking
  the installed handler function directly

## Assumptions

1. `HttpServerLifecycleManager`, `_make_mock_proc`, `AsyncMock`, `MagicMock`, and `pytest` are
   already imported at the top of `tests/test_lifecycle.py` (used by every existing test in
   `TestShutdownAllCleanup`) — no new imports needed beyond `signal`, which must be added if
   not already imported in this test file.
2. The companion implementation in `scripts/agent/http_lifecycle.py` installs the temporary
   handler via `signal.signal(signal.SIGINT, self._absorb_sigint_during_shutdown)` before the
   loop and restores it via `signal.signal(signal.SIGINT, old_sigint)` in a `finally` — this
   test's assertions are written against that exact contract.

## Implementation

### Target file

`tests/test_lifecycle.py`

### Procedure

1. Add `import signal` near the top of the file if not already present (check existing
   imports first; do not duplicate).
2. In `TestShutdownAllCleanup` (after `test_shutdown_all_continues_after_terminate_error`,
   matching its style), add:
   ```python
   @pytest.mark.asyncio
   async def test_shutdown_all_survives_second_sigint(self) -> None:
       mgr = HttpServerLifecycleManager()
       mgr._http_procs["srv1"] = _make_mock_proc(exit_code=None)
       mgr._http_procs["srv2"] = _make_mock_proc(exit_code=None)

       original_handler = signal.getsignal(signal.SIGINT)
       call_count = 0

       async def terminate_and_interrupt(
           proc: MagicMock, key: str, timeout: float = 3.0
       ) -> None:
           nonlocal call_count
           call_count += 1
           if call_count == 1:
               # Simulate a second Ctrl-C arriving mid-loop by invoking whatever
               # handler shutdown_all() has currently installed for SIGINT.
               current_handler = signal.getsignal(signal.SIGINT)
               current_handler(signal.SIGINT, None)

       mgr._terminate_with_timeout = terminate_and_interrupt

       await mgr.shutdown_all()

       assert call_count == 2  # both servers still processed despite the mid-loop "interrupt"
       assert len(mgr._http_procs) == 0
       assert signal.getsignal(signal.SIGINT) is original_handler
   ```
3. Do not remove or alter any of the four existing tests in `TestShutdownAllCleanup`.

### Method

Simulate the interrupt by directly invoking the handler object returned by
`signal.getsignal(signal.SIGINT)` at the point `shutdown_all()` has (per the companion
implementation) already installed its absorbing handler — this exercises the same code path
a real second `SIGINT` would reach, without invoking `os.kill`/`signal.raise_signal`, which
would risk affecting the test runner process itself (the exact class of bug already fixed
historically for this file — see `issues/20260709_00_issue.md` section B, now archived —
where a stray real `os.killpg` call in this same test file broadcast a signal to unrelated
processes).

### Details

- `call_count == 2` after the loop proves the loop was not aborted: both `srv1` and `srv2`
  were still passed to `_terminate_with_timeout`, even though the "interrupt" happened during
  the first call.
- `signal.getsignal(signal.SIGINT) is original_handler` after `shutdown_all()` returns proves
  the `finally`-based restoration in the companion implementation runs on the success path.
- This test intentionally does not assert on the log message emitted by the absorbing
  handler — log-content assertions are brittle and out of scope for what this test needs to
  prove (loop completion + handler restoration).
- If `test_shutdown_all_twice_is_safe` (existing test, line 666) still passes unmodified after
  the companion implementation lands, that is additional evidence the restore-on-`finally`
  behavior does not leak state across repeated calls — no new test is needed for that case
  since the existing one already covers it.
