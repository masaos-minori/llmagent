# 04_mcp_04_server_catalog — Implementation Procedure

## Goal
11台の全MCPサーバーを1ファイルにカタログ化する。各サーバーのツール一覧、ポート、設定パラメータ、制約を標準フォーマットで記載する。

## Scope
- `docs/04_mcp-{web-search,file-read,file-write,file-delete,github,shell,rag-pipeline,sqlite,cicd,mdq,git}.md` (11ファイル)

## Assumptions
- 各セクションは統一フォーマット（概要・ポート・ツール一覧・設定・制約）で記載する
- ツール一覧は引数と戻り値の型を簡潔に示す

## Implementation

### Target file
`docs/04_mcp_04_server_catalog.md`

### Procedure
- セクションヘッダ「## {サーバー名} (port {N})」形式で統一し、ポート昇順 (8004→8014) に並べる
- 各セクション: 概要1行、ツール名テーブル（ツール名・引数・説明）、設定パラメータ、制約を記載する

### Method
各 04_mcp-*.md ファイルから情報を抽出し、統一フォーマットに変換して結合する。

### Details
- web-search(8004): search_web, get_webpage; file-read(8005): read_file, list_dir / allowed_dirs
- file-write(8007): write_file / read_only=false 必須; file-delete(8008): delete_file / path_denylist
- github(8006): create_issue, create_pr, list_issues / repo_allowlist
- shell(8009): run_command / command_allowlist; shell_run_bg 未実装
- rag-pipeline(8010): ingest, query_rag / workflow_allowlist; sqlite(8011): query_sqlite / db_allowlist
- cicd(8012): trigger_workflow / workflow_allowlist; mdq(8013): プレースホルダー実装
- git(8014): git_log, git_diff, git_status / repo_allowlist

## Validation plan
- 11サーバー全セクションが存在することを確認する
- 各セクションに概要・ツール・設定・制約が含まれることを確認する
- ツール引数がソースコードの型定義と一致することを確認する
