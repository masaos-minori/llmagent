# Implementation Procedure: Remove _REMOVED_KEYS from config_loader.py

Source plan: `plans/done/20260719-150531_plan.md`

## Goal

Remove `_REMOVED_KEYS` constant and `_filter_removed_keys()` static method from `config_loader.py`. This silent filter is redundant because Pydantic dataclasses already ignore unknown fields.

## Scope

**In-Scope:**
- Remove `_REMOVED_KEYS` constant (lines 31-39)
- Remove `_filter_removed_keys()` static method (lines 167-169)
- Remove calls to `_filter_removed_keys()` in `load()` (line 79) and `load_all()` (line 107)

**Out-of-Scope:**
- Changes to other source files (handled in separate documents)
- Changes to Pydantic dataclass definitions

## Assumptions

1. `_filter_removed_keys()` is only called within `config_loader.py` — verified by grep search.
2. `_REMOVED_KEYS` is only referenced within `config_loader.py` — verified by grep search.
3. Removing this silent filter has no functional impact since Pydantic ignores unknown fields anyway.

## Implementation

### Target file

`scripts/shared/config_loader.py`

### Procedure

1. **Phase 1: Verify scope**
   - Confirm `_filter_removed_keys()` is not called from outside `config_loader.py`
   - Confirm `_REMOVED_KEYS` is not imported/exported from this module

2. **Phase 2: Remove calls to `_filter_removed_keys()`**
   - In `load()` method (line 79): change `filtered = self._filter_removed_keys(self._filter_meta_keys(self._load_single(name)))` to `filtered = self._filter_meta_keys(self._load_single(name))`
   - In `load_all()` method (line 107): remove the `_filter_removed_keys()` call similarly

3. **Phase 3: Remove constant and method**
   - Delete `_REMOVED_KEYS` constant (lines 31-39)
   - Delete `_filter_removed_keys()` static method (lines 167-169)

4. **Phase 4: Cleanup**
   - Run `uv run ruff format scripts/ && uv run ruff check scripts/ --fix`

### Method

Remove the method and its constant, then update all call sites to remove the method invocation while preserving the surrounding logic (`_filter_meta_keys()` and `_load_single()` calls remain unchanged).
