# Implementation: Make require_approval config-driven in orchestrator.py

Steps covered: Plan 20260626-090724 — Phase 2, Step 2-1

---

## Goal

Replace the implicit `require_approval=False` default in `WorkflowEngine` instantiation (orchestrator.py line 204) with an explicit value read from `AgentConfig`, so the setting can be changed without modifying source code.

---

## Scope

- **In scope**: `scripts/agent/orchestrator.py` — change `WorkflowEngine(...)` call to pass `require_approval` from config
- **In scope**: verify `AgentConfig` has (or add) a `workflow_require_approval: bool = False` field
- **Out of scope**: changing `WorkflowEngine.__init__` signature (it already accepts `require_approval`)
- **Out of scope**: enabling workflow approval by default

---

## Assumptions

- `AgentConfig` is accessible via `ctx.cfg` in `orchestrator.py`.
- A new config field `workflow_require_approval: bool = False` will be added to `AgentConfig` (or an existing nested config class).
- The default value `False` preserves the current behavior.
- `WorkflowEngine` constructor already accepts `require_approval: bool = False` (confirmed: workflow_engine.py line 83).

---

## Implementation

### Target file
`scripts/agent/orchestrator.py`

### Procedure
1. Read `scripts/agent/orchestrator.py` lines 195-215 to confirm the exact `WorkflowEngine(...)` call site.
2. Read `scripts/agent/app_config.py` (or wherever `AgentConfig` is defined) to find the right class to add `workflow_require_approval`.
3. Add `workflow_require_approval: bool = False` to the config class.
4. In `orchestrator.py`, change:
   ```python
   engine = WorkflowEngine(self._workflow_def, store, tracer=self._tracer)
   ```
   to:
   ```python
   engine = WorkflowEngine(
       self._workflow_def,
       store,
       require_approval=self._ctx.cfg.workflow_require_approval,
       tracer=self._tracer,
   )
   ```
5. Update `config/agent.toml` (if applicable) to document the new field.

### Method
Minimal targeted change. Add one config field, thread it into the `WorkflowEngine` constructor call.

### Details

Config class location: find with `grep -rn "class AgentConfig\|workflow_require" scripts/agent/` to identify the right file.

New field:
```python
workflow_require_approval: bool = False
# When True, WorkflowEngine requires /approve before the verify stage runs.
# Disabled by default; enable in production after testing the approval flow.
```

Orchestrator change location: line 204 (approx.) in `orchestrator.py`.

No behavioral change when `workflow_require_approval=False` (default).
