## Goal

Make `scripts/shared/tool_executor.py` importable again by adding the missing
`from __future__ import annotations` import, eliminating the startup-breaking
`NameError: name 'RuntimeToolRegistry' is not defined` that currently crashes every import of
`agent.factory`, `agent.context`, and `agent.repository_gateway`.

## Scope

**In scope:**
- Add `from __future__ import annotations` as the first import line in
  `scripts/shared/tool_executor.py`, placed immediately after the module docstring and before all
  other imports.

**Out of scope:**
- No change to the `TYPE_CHECKING` guard, the `RuntimeToolRegistry` import, or the
  `set_runtime_registry` method signature.
- No `from __future__ import annotations` audit/rollout to other files.
- No fix for unrelated pre-existing lint/architecture/bandit findings.
- No refactor of `ToolExecutor` / `ToolRouteResolver` / `RuntimeToolRegistry` routing logic.
- No change to `tests/test_import_smoke.py` (covered by a separate, already-tracked implementation
  item: `implementations/20260721-123247_test_import_smoke.py.md`).

## Assumptions

1. `scripts/shared/tool_executor.py` currently has no `from __future__ import annotations` and raises
   `NameError` on import — confirmed directly (2026-07-21) via
   `PYTHONPATH=scripts .venv/bin/python3 -c "import shared.tool_executor"`.
2. Adding `from __future__ import annotations` only defers annotation evaluation; it does not change
   runtime behavior for any annotation in this file, since no caller in `scripts/` or `tests/` calls
   `get_type_hints()` or `inspect.signature()` against `ToolExecutor` or `tool_executor.py` (confirmed
   via `rg -n "get_type_hints|inspect\.signature" scripts/ tests/` in the source plan's Design section
   — the only matches are in `tests/test_command_registry_consistency.py`, unrelated to this module).
3. mypy already treats annotations as deferred by default, so no new mypy errors are expected from this
   change.
4. `scripts/agent/tool_policy.py` (line 9) and `scripts/agent/repository_gateway.py` (line 13) already
   have `from __future__ import annotations` placed directly after their module docstring, before any
   other import — this is the placement convention to mirror exactly.

### Method

- This is a single-line, additive change with no structural refactor: one new import statement, no
  deletions, no reordering of existing imports beyond making room for the new first line.
- `ruff`'s import-order rule (`I`, isort-compatible, per `rules/coding.md`) sorts `__future__` imports
  first automatically; placing it directly after the docstring and before `import asyncio` already
  matches expected isort ordering, so no `ruff check --fix` reordering surprise is expected.
- No test file changes are needed as part of this procedure — the regression test that proves this fix
  works (and that it was needed) is `tests/test_import_smoke.py`, implemented separately per
  `implementations/20260721-123247_test_import_smoke.py.md`. Implementers should apply both procedures
  in the same pass (the test file is expected to fail before this fix lands and pass after), per that
  document's Assumption 3.
