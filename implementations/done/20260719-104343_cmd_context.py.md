## Goal

Implement `_cmd_diff(self) -> None` as a new async method on `_ContextMixin`
(`scripts/agent/commands/cmd_context.py`), which scans `ctx.conv.history` for `write_file`/`edit_file`
tool calls made this session, groups the touched file paths by containing git repository, calls the
`git_diff` MCP tool once per distinct repository, and prints each touched file's diff (or a clear
per-file notice when a diff is unavailable) — without aborting the whole command if one repo's
`git_diff` call fails or is denied.

## Implementation

### Target file

`scripts/agent/commands/cmd_context.py`.

### Procedure

1. Add `import json` and `import git` (GitPython) to the module's import block (`cmd_context.py:18-28`);
   `pathlib` is not strictly needed if using `git.Repo(path, search_parent_directories=True)` directly
   on the string path (GitPython accepts `str` paths).
2. Update the module docstring (`cmd_context.py:6-11`) to add a line: `_cmd_diff — /diff: show diffs
   for files written/edited this session`.
3. Add `_cmd_diff` as a new `async def` method on `_ContextMixin`, placed after `_cmd_system` (the last
   existing method, ending at `cmd_context.py:178`):
   ```python
   async def _cmd_diff(self) -> None:
       """Show working-tree diffs for files written/edited via tool calls this session."""
       ctx = self._ctx
       paths = self._collect_diff_touched_paths()
       if not paths:
           self._out.write("No files written or edited this session.")
           return
       if ctx.services is None or ctx.services.tools is None:
           self._out.write("MCP tool executor not available.")
           return
       by_repo, outside_repo = self._group_paths_by_repo(paths)
       for path in outside_repo:
           self._out.write(f"{path}: not inside a git repository (skipped)")
       for repo_root, repo_paths in by_repo.items():
           result = await ctx.services.tools.execute(
               "git_diff", {"repo_path": repo_root, "commit": ""}
           )
           if result.is_error or result.output.startswith("[DENIED]"):
               self._out.write(f"[{repo_root}] git diff unavailable: {result.output}")
               continue
           self._print_repo_diffs(repo_root, repo_paths, result.output)
   ```
   (Method decomposition below keeps each piece independently testable and readable; exact private
   helper names are this doc's own choice, not dictated by the plan, since the plan's Design section
   describes behavior/steps, not a specific decomposition.)
4. Add three small private helpers on `_ContextMixin` (or as module-level functions if preferred —
   either is consistent with this file's style, which currently has no module-level helper functions
   apart from the two service-layer imports; keeping them as methods matches `_cmd_context`'s existing
   style of doing all logic inline, adapted here only because the logic is long enough to warrant
   splitting for test coverage and readability, per `rules/coding.md`'s function-length guidance — not
   independently re-read in this pass, but consistent with the plan's own step-lettered breakdown 2a-2f):
   - `_collect_diff_touched_paths(self) -> list[str]`: walk `ctx.conv.history`; for messages with
     `role == "assistant"` and a truthy `tool_calls` list, for each call where
     `call["function"]["name"] in ("write_file", "edit_file")`, `json.loads(call["function"]["arguments"])`
     and collect `args["path"]`; guard `json.JSONDecodeError`/`KeyError`/`TypeError` per malformed entry
     (skip that one call, do not abort the scan — no precedent elsewhere in this file for this exact
     guard, but consistent with `_cmd_history`'s defensive `msg.get(...)` style at `cmd_context.py:157`).
     Deduplicate preserving first-seen order (e.g. via `dict.fromkeys(paths)`).
   - `_group_paths_by_repo(self, paths: list[str]) -> tuple[dict[str, list[str]], list[str]]`: for each
     path, attempt `git.Repo(path, search_parent_directories=True)` (or on its parent directory if the
     path itself is a not-yet-created/deleted file — use `os.path.dirname(path)` or the path's parent as
     the search start, since GitPython's `search_parent_directories` walks from the given directory, not
     a file), catch `git.InvalidGitRepositoryError` (and `git.NoSuchPathError` if the file was later
     deleted) to route into the "outside any git repository" list; on success, use
     `repo.working_tree_dir` as the canonical repo-root key so distinct touched paths under the same repo
     group together even if resolved via different subdirectories.
   - `_print_repo_diffs(self, repo_root: str, touched_paths: list[str], diff_text: str) -> None`: split
     `diff_text` on a regex anchored to `^diff --git a/(.+?) b/(.+?)$` (per file hunk header, `re.MULTILINE`),
     build a `dict[str, str]` mapping the header's relative path to its hunk body; for each path in
     `touched_paths`, compute its path relative to `repo_root` (`os.path.relpath`), look it up in that
     dict, and print either the matching hunk under an `f"--- {path} ---"`-style header, or
     `f"{path}: no working-tree diff"` if not found (covers both the clean-repo `"(no diff)"` case and
     the "touched but no hunk for this file" case from Assumption 3/plan Design 2e — both collapse to the
     same user-facing message, matching the plan's own wording).
5. Keep all four new/changed pieces (`_cmd_diff` + 3 helpers) inside `_ContextMixin`; no new mixin, no
   change to `MixinBase` or `registry.py`'s MRO (per Scope).

### Method

New async method + three small private synchronous helpers, added to an existing mixin class. No new
`Protocol`/`ABC`; reuses the existing `OutputPort`/`_out` and `AgentContext`/`_ctx` interfaces already
declared on `MixinBase` (`scripts/agent/commands/mixin_base.py:33-46`, re-verified: `_ctx:
AgentContext` and `_out: OutputPort = CliOutputPort()` class attributes).

### Details

- New method signature: `async def _cmd_diff(self) -> None`. Registered with `is_async=True` in the
  paired `CommandDef` doc — this is a hard requirement, since `CommandRegistry` dispatches based on
  the `CommandDef.is_async` flag (not independently re-read here beyond the plan's own citation, but
  consistent with every other `async def _cmd_*` method in this codebase, e.g. `_cmd_mdq*` in
  `cmd_mdq.py`, `_cmd_compact`).
- `self._out` calls used: `self._out.write(...)` only (matches `OutputPort.write(text: str) -> None`,
  `output_port.py:14-16`) — no `write_kv`/`write_table` needed since output is a simple per-path text
  stream, consistent with `_cmd_history`'s plain `self._out.write(f"[{msg['role']}] {preview}")` style
  (`cmd_context.py:162`).
- `ctx.services`/`ctx.services.tools.execute(name, args)` is the pattern to use (see Assumption 6
  correction above) — `await`ed, returns `ToolCallResult` (`output: str`, `is_error: bool`), never
  raises for tool-level failures.
- No new module-level constants needed beyond what already exists (`CONTEXT_PREVIEW_LENGTH` is unrelated
  to this feature and untouched).
- Edge case (from the plan's Risks): a touched path containing a space must still match its
  `diff --git a/<path> b/<path>` header correctly — the regex in `_print_repo_diffs` must not assume
  whitespace-delimited tokens after `a/`/`b/`; capturing everything up to ` b/` (non-greedy) and to
  end-of-line respectively (as sketched in Procedure step 4) handles this without extra quoting logic,
  since `git diff`'s default header format is `diff --git a/<path> b/<path>` verbatim (no quoting)
  for paths without special/non-ASCII characters — a fully quoted-path edge case (e.g.
  `core.quotepath` interactions) is not separately handled here; if the dedicated space-containing-path
  test (required by the paired tests doc) reveals a mismatch, the regex must be tightened at
  implementation time rather than assumed correct from this design alone.
