# Implementation: scripts/agent/session.py — Wrap sessions INSERT to raise RuntimeError on missing schema

**Plan source:** `plans/20260702-202919_plan.md` (Phase 2 (session))
**Target file:** `scripts/agent/session.py`

---

## Goal

AgentSession.start() 内の `INSERT INTO sessions` 呼び出しを try/except で囲み、sessions テーブルが存在しない場合に分かりやすい RuntimeError を送出するようにする。

---

## Scope

**In:**
- AgentSession.start() の INSERT INTO sessions を sqlite3.OperationalError でキャッチする処理の追加
- エラーメッセージに "no such table" が含まれる場合に RuntimeError として再送出する
- RuntimeError のメッセージ: `"Session schema missing. Run: bash deploy/init_db.sh to initialize the database."`

**Out:**
- sessions テーブル以外の OperationalError のハンドリング変更
- write_fatal の呼び出し (repl.py の Phase 2 で対応)
- データベース接続エラーのハンドリング

---

## Assumptions

1. AgentSession.start() 内で sqlite3 を直接使用して INSERT INTO sessions を実行している
2. sqlite3.OperationalError の args[0] または str(e) に "no such table" が含まれる
3. RuntimeError は呼び出し元 (repl.py) でキャッチされる前提 (Phase 2 (repl) で対応)

---

## Implementation

### Target file

`scripts/agent/session.py`

### Procedure

1. session.py を読み込み、AgentSession.start() の INSERT INTO sessions 実装を確認する
2. INSERT 実行箇所を `try:` ブロックで囲む
3. `except sqlite3.OperationalError as e:` を追加し、`"no such table"` が str(e) に含まれるか判定する
4. 条件が True の場合は `raise RuntimeError("Session schema missing. Run: bash deploy/init_db.sh to initialize the database.")` を送出する
5. それ以外の OperationalError は `raise` で再送出する (既存動作を維持)
6. lint/mypy を確認する

### Method

Edit tool でコード変更

### Details

変更後のパターン:
```python
try:
    cursor.execute("INSERT INTO sessions ...", ...)
except sqlite3.OperationalError as e:
    if "no such table" in str(e):
        raise RuntimeError(
            "Session schema missing. Run: bash deploy/init_db.sh to initialize the database."
        ) from e
    raise
```

sqlite3 は標準ライブラリのため import 追加不要 (既存 import を確認)。
