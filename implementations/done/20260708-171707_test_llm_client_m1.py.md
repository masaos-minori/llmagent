# Implementation: M-1 — Remove forbidden config keys from test_llm_client.py's _build()

## Goal

Remove `use_tool_summarize`/`tool_summarize_threshold` from
`TestAgentConfigSseValidation._build()`'s `base` dict. Mandatory fix: `_build()` calls
`build_agent_config(base)`, and once these keys are added to `_FORBIDDEN_KEYS` (per
`config_builders.py`'s M-1 doc), every test in `TestAgentConfigSseValidation` would fail with
`ConfigLoadError`.

## Scope

**Target**: `tests/test_llm_client.py` — `TestAgentConfigSseValidation._build()`'s `base` dict
only (lines ~603-604).

**Depends on / must land together with**: `implementations/20260708-171134_config_builders.py.md`.

**Out of scope**: every other test class in this large file — only
`TestAgentConfigSseValidation` constructs a real `AgentConfig` via `build_agent_config()`; other
classes use different fixtures (mocked LLM clients, etc.) unaffected by this config change.

## Assumptions

1. `_build()`'s `base` dict is used by every test in `TestAgentConfigSseValidation` (via
   `self._build(**overrides)`), all of which test SSE-related validation
   (`sse_heartbeat_timeout`, etc.) unrelated to summarization.

## Implementation

### Target file

`tests/test_llm_client.py`

### Procedure

#### Step 1: Confirm the current lines

```bash
grep -n "use_tool_summarize\|tool_summarize_threshold" tests/test_llm_client.py
```

Expected: two matches inside `TestAgentConfigSseValidation._build()`'s `base` dict.

#### Step 2: Remove both keys

Current (within the `base` dict):

```python
            "serial_tool_calls": False,
            "use_tool_summarize": False,
            "tool_summarize_threshold": 3000,
            "use_semantic_cache": False,
```

Replace with:

```python
            "serial_tool_calls": False,
            "use_semantic_cache": False,
```

### Method

- Two-key removal from a shared fixture dict local to one test class.
