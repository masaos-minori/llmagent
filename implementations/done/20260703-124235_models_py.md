# Step 2 — Fix `scripts/agent/shared/models.py`: add `source` field to `ToolExecEvent`

## Goal

Add a `source: str = "agent"` field to the `ToolExecEvent` frozen dataclass so that agent-side tool execution audit events are identifiable by source in the unified JSON-lines log.

## Scope

- In-Scope:
  - Add `source: str = "agent"` as a defaulted field in `ToolExecEvent` (after the last required field, before or after `error_type`)
- Out-of-Scope:
  - Adding `source` to `ToolApprovalEvent` or `ApprovalDecisionEvent` (not required by the spec)

  - Modifying callers of `ToolExecEvent` in `tool_audit.py` — the default value makes the field appear automatically in `dataclasses.asdict()`

1. `ToolExecEvent` is a frozen dataclass; defaulted fields must come after all non-defaulted (required) fields. 2. Adding a defaulted field does not change the call sites in `audit_tool_exec` — the default `"agent"` is correct for all current callers.
3. `dataclasses.asdict(ToolExecEvent(...))` automatically includes the `source` field with value `"agent"`.
