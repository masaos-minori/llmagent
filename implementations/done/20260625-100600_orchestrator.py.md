# orchestrator.py — fail fast at construction in required mode

**Plan:** `plans/20260625-093349_plan.md` (req #61)
**Target:** `scripts/agent/orchestrator.py`

## What to change

Currently (line 118-122), when `WorkflowLoader().load()` fails, the exception is caught and
a warning is logged regardless of `workflow_mode`:

```python
if self._workflow_mode != "disabled":
    try:
        self._workflow_def = WorkflowLoader().load()
    except (WorkflowLoadError, Exception):
        logger.warning("WorkflowLoader failed — workflow tracking disabled")
```

Add a check for `required` mode in the except handler so that construction raises immediately
instead of silently setting `_workflow_def = None`.

**After:**
```python
if self._workflow_mode != "disabled":
    try:
        self._workflow_def = WorkflowLoader().load()
    except (WorkflowLoadError, Exception) as exc:
        if self._workflow_mode == "required":
            raise RuntimeError(
                f"[workflow] mode=required but WorkflowLoader failed: {exc}. "
                "Check workflow definition file or set workflow_mode=auto in config."
            ) from exc
        logger.warning("WorkflowLoader failed — workflow tracking disabled")
```

## Impact on existing tests

The existing test `test_required_mode_no_workflow_def_raises` (line 743) currently:
1. Creates `Orchestrator(ctx, workflow_mode="required")` without catching exceptions
2. Asserts `orch._workflow_def is None` — construction succeeds
3. Expects `handle_turn("hello")` to raise `RuntimeError`

After this change, step 1 will raise at construction time, so the test must be updated to
match the new behavior:

```python
def test_required_mode_raises_at_construction_when_loader_fails(self) -> None:
    ctx = _make_ctx()
    with (
        patch("agent.orchestrator.WorkflowLoader") as mock_loader,
        pytest.raises(RuntimeError, match="mode=required"),
    ):
        mock_loader.return_value.load.side_effect = Exception("not found")
        Orchestrator(ctx, workflow_mode="required")
```

Remove or update the old `test_required_mode_no_workflow_def_raises` test since its premise
(construction succeeds → handle_turn raises) no longer holds.
