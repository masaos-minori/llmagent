## Goal

Add an `obsolete INTEGER NOT NULL DEFAULT 0` column to the `tool_results` table DDL so that undo operations can soft-delete tool result rows without destroying audit history.

---

## Scope

**In-Scope**
- Add `obsolete` column to `_SESSION_SCHEMA_TEMPLATE` in `schema_sql.py`
- Add `ALTER TABLE` migration path for existing DBs (via `create_schema.py` or a separate migration call)
- Update `ToolResultRow` dataclass in `db/models.py` to include `obsolete: bool = False`

**Out-of-Scope**
- Changing existing query logic in `cmd_tooling.py` (handled in separate doc)
- Any data migration of existing tool_results rows

---

## Assumptions

1. SQLite supports `ALTER TABLE tool_results ADD COLUMN obsolete INTEGER NOT NULL DEFAULT 0` for existing DBs â€” safe for DEFAULT columns.
2. New column defaults to `0` (not obsolete) so existing rows remain unaffected.
3. `create_schema.py` uses `CREATE TABLE IF NOT EXISTS` â€” new DBs get the column; existing DBs need the ALTER.
4. `ToolResultRow.obsolete` should be `bool` in the Python dataclass (mapped from SQLite INTEGER 0/1).

### Details

**`schema_sql.py` change** â€” in `_SESSION_SCHEMA_TEMPLATE`, update `tool_results`:

```sql
CREATE TABLE IF NOT EXISTS tool_results (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER REFERENCES sessions(session_id) ON DELETE CASCADE,
    turn       INTEGER NOT NULL,
    tool_name  TEXT    NOT NULL,
    args_masked  TEXT,
    full_text  TEXT    NOT NULL,
    summary    TEXT,
    is_error   INTEGER NOT NULL DEFAULT 0,
    obsolete   INTEGER NOT NULL DEFAULT 0,
    created_at TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);
```

**`models.py` change** â€” add field to `ToolResultRow`:

```python
@dataclass(frozen=True)
class ToolResultRow:
    id: int
    tool_name: str
    is_error: bool
    summary: str | None = None
    session_id: int | None = None
    turn: int = 0
    args_masked: str = ""
    full_text: str = ""
    created_at: str = ""
    obsolete: bool = False  # True when the owning turn was undone
```

**`create_schema.py` migration** â€” add after schema creation:

```python
# Migrate existing session.sqlite: add obsolete column to tool_results if absent
with SQLiteHelper("session").open(write_mode=True) as db:
    try:
        db.execute(
            "ALTER TABLE tool_results ADD COLUMN obsolete INTEGER NOT NULL DEFAULT 0"
        )
        db.commit()
    except Exception:
        pass  # column already exists; safe to ignore
```
