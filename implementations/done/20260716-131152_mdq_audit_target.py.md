# Implementation: scripts/mcp_servers/mdq/audit_target.py (trim admin-tool branch to `"stats"` only)

Source plan: `plans/20260716-123031_plan.md`

## Goal

Remove `"fts_consistency_check"` and `"fts_rebuild"` from the tuple-membership
check in `extract_audit_target()`, keeping `"stats"` since it remains a live
tool.

## Scope

**In:**
- Line 30 of `scripts/mcp_servers/mdq/audit_target.py`:
  `if tool_name in ("stats", "fts_consistency_check", "fts_rebuild"):`
  → `if tool_name in ("stats",):`

**Out:**
- Any other branch in `extract_audit_target()` (`search_docs`, `get_chunk`,
  `outline`, `index_paths`/`refresh_index`, `grep_docs`).
- The function's return value/signature — unchanged.

## Assumptions

1. `extract_audit_target()` is called with `tool_name` values only from the
   live `TOOL_LIST` (companion doc removes `fts_consistency_check`/
   `fts_rebuild` from there) — after this change, the function is never
   invoked with those two names in practice, so removing them from the
   branch is a safe simplification, not a behavior change for any reachable
   input.
2. No test asserts `extract_audit_target("fts_consistency_check", {})` or
   `extract_audit_target("fts_rebuild", {})` directly — confirmed via
   `rg -n "extract_audit_target" tests/`; if such a test is found during
   implementation, it must be removed/updated as part of this same change
   (not deferred).

## Implementation

### Target file

`scripts/mcp_servers/mdq/audit_target.py`

### Procedure

1. Open `scripts/mcp_servers/mdq/audit_target.py`.
2. Locate line 30:
   ```python
   if tool_name in ("stats", "fts_consistency_check", "fts_rebuild"):
   ```
3. Replace with a single-name tuple:
   ```python
   if tool_name in ("stats",):
   ```
   (Retain the trailing comma inside the single-element tuple — required
   Python syntax to distinguish it from a plain parenthesized string.)
4. Leave the `return "mdq-mcp"` line beneath it unchanged.

### Method

Single-line conditional simplification — tuple membership test narrowed
from 3 elements to 1.

### Details

- Because only one name remains, an equality check (`if tool_name == "stats":`)
  would also be valid Python, but keep the tuple form (`("stats",)`) to
  minimize the diff and preserve the function's existing style of using
  tuple-membership checks for multi-branch dispatch (matches the
  `if tool_name in ("index_paths", "refresh_index"):` pattern a few lines
  above it in the same function).
