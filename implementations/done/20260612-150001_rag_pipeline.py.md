# Goal

Fix two issues in `scripts/rag/pipeline.py`:
1. Replace `assert isinstance(result, list)` with an explicit warning+continue guard.
2. Replace `str(body.get("context", ""))` with a schema-validated access that
   raises `ValueError` on unexpected type.

# Scope

- `scripts/rag/pipeline.py` only

# Assumptions

1. Line 133: `result` at this point has already been filtered by the preceding
   `isinstance(result, Exception)` check. The only remaining unexpected case is a
   non-list return from `get_embedding()`. Correct behavior is to log and skip,
   not to raise (the outer `asyncio.gather(return_exceptions=True)` context means
   raising here would not propagate correctly anyway).

2. Line 208: `body["context"]` should always be `str` when the RAG HTTP service
   returns a valid response. If it is not a str, this is a schema violation and
   should raise `ValueError` so the surrounding `except` block catches it and falls
   back to in-process RAG.

# Implementation

## Target file

`scripts/rag/pipeline.py`

## Procedure

1. Replace `assert isinstance(result, list)` (line 133):

```python
# Before
assert isinstance(result, list)

# After
if not isinstance(result, list):
    logger.warning(
        f"Unexpected embedding result type for '{q}': {type(result).__name__}"
    )
    continue
```

2. Replace `return str(body.get("context", ""))` (line 208):

```python
# Before
return str(body.get("context", ""))

# After
context_raw = body.get("context")
if context_raw is None:
    return ""
if not isinstance(context_raw, str):
    raise ValueError(
        f"RAG service 'context' field must be str, got {type(context_raw).__name__}"
    )
return context_raw
```

3. Run ruff + mypy.

## Method

Two targeted line replacements. No logic or interface change.
