# Implementation: Make consumer_id collision-resistant in offsets.py

Steps covered: Plan 20260626-094717 — Steps 2-1, 2-2

---

## Goal

Ensure `consumer_id` values in the offsets table are collision-resistant. Replace any auto-increment or user-supplied consumer ID with a UUID-based or hash-based ID that cannot accidentally collide between different consumer processes.

---

## Scope

- **In scope**: `scripts/eventbus/offsets.py` — `consumer_id` generation/storage
- **Out of scope**: docs (step 3-2); tests (step 3-1)

---

## Assumptions

- `offsets.py` manages per-consumer stream offsets (position tracking).
- `consumer_id` is currently a user-supplied string or auto-increment int.
- The fix: generate `consumer_id` as `uuid4()` if not supplied, or hash `(consumer_name + host + pid)` for stable-but-unique IDs.
- Step 2-1: Stable ID generation (same consumer process gets same ID across restarts if name is stable).
- Step 2-2: Collision detection (if two consumers register the same ID, raise an error).

---

## Implementation

### Target file
`scripts/eventbus/offsets.py`

### Procedure
1. Read `scripts/eventbus/offsets.py` to understand `consumer_id` handling.
2. Step 2-1 (Stable ID): Generate `consumer_id` as:
   ```python
   import hashlib, socket, os
   
   def _make_consumer_id(consumer_name: str) -> str:
       stable_key = f"{consumer_name}:{socket.gethostname()}:{os.getpid()}"
       return hashlib.sha256(stable_key.encode()).hexdigest()[:16]
   ```
   This is stable within a process (same name+host+pid = same ID), but unique across processes.
3. Step 2-2 (Collision detection): In `register_consumer()`, check if the generated ID is already in use by a different `consumer_name`:
   ```python
   existing = await self._db.get_consumer_by_id(consumer_id)
   if existing and existing.name != consumer_name:
       raise ConsumerIdCollisionError(
           f"consumer_id {consumer_id!r} already registered by {existing.name!r}"
       )
   ```

### Method
ID generation change. Existing consumers with old IDs will get new IDs (one-time offset loss risk — document in migration note).
