# Implementation: Add visible_bytes to TruncationResult

## Goal

Fix the truncation message in `_truncate_with_meta()` so it reports the actual number of visible bytes shown, not the configured limit. The current message says "showing {max_bytes:,} bytes" but `decode("utf-8", errors="ignore")` may discard bytes at a multi-byte character boundary, so the real visible count can be less than `max_bytes`.

## Scope

- `scripts/mcp/server.py` — `TruncationResult` dataclass + `_truncate_with_meta()` only

Out of scope:
- Changing truncation limits or mechanism
- Modifying individual MCP server implementations
- Breaking changes to existing `text`, `truncated`, `total_bytes` fields

## Assumptions

1. `TruncationResult` is a `dataclasses.dataclass(frozen=True)` — adding a field with a default keeps backward compatibility for call sites that construct by keyword.
2. `shown = encoded[:max_bytes].decode("utf-8", errors="ignore")` may produce fewer visible bytes than `max_bytes` at a multi-byte boundary. `len(shown.encode("utf-8"))` gives the accurate count.
3. Step 3 (alternative: update message only, no new field) is not chosen because adding `visible_bytes: int` to the dataclass gives callers precise data without repeated encoding in the message.
4. All callers of `_truncate_with_meta()` consume only `text` and `truncated`; `total_bytes` is used for logging/debug. No caller currently reads a `visible_bytes` field — the addition is purely additive.

## Implementation

### Target file

`scripts/mcp/server.py`

### Procedure

1. Add `visible_bytes: int` field to `TruncationResult` (after `total_bytes`).
2. In `_truncate_with_meta()`, compute `visible = len(shown.encode("utf-8"))` after decoding.
3. Update the truncation message to use `visible` instead of `max_bytes`.
4. Pass `visible_bytes=visible` to the `TruncationResult` constructor.
5. In the non-truncated path, set `visible_bytes=total` (all bytes are visible).

### Method

Minimal additive change: two lines added to dataclass, two lines changed in function body.

### Details

**`TruncationResult` — add field after `total_bytes`:**

```python
@dataclasses.dataclass(frozen=True)
class TruncationResult:
    """Metadata returned by _truncate_with_meta()."""

    text: str
    truncated: bool
    total_bytes: int
    visible_bytes: int  # actual bytes shown after UTF-8 boundary alignment
```

**`_truncate_with_meta()` — full updated function:**

```python
def _truncate_with_meta(
    text: str, max_bytes: int = MCP_MAX_RESPONSE_BYTES
) -> TruncationResult:
    """Return TruncationResult with truncated text and metadata."""
    encoded = text.encode("utf-8")
    total = len(encoded)
    if total <= max_bytes:
        return TruncationResult(text=text, truncated=False, total_bytes=total, visible_bytes=total)
    shown = encoded[:max_bytes].decode("utf-8", errors="ignore")
    visible = len(shown.encode("utf-8"))
    truncated_text = (
        shown + f"\n[TRUNCATED: {total:,} bytes total, showing {visible:,} bytes]"
    )
    return TruncationResult(text=truncated_text, truncated=True, total_bytes=total, visible_bytes=visible)
```

Key changes:
- Non-truncated path: `visible_bytes=total`
- Truncated path: `visible = len(shown.encode("utf-8"))` — re-encodes decoded string to get accurate byte count after boundary alignment
- Message: `showing {visible:,} bytes` (was `showing {max_bytes:,} bytes`)
