# Implementation: routing.md + slash-command docs — document `/skill`

Source plan: `plans/20260712-174955_plan.md` (Implementation Steps, Phase 4)

## Goal

Document the new `/skill` command so both `routing.md` (the AI-facing skill-loading
reference) and the human-facing CLI reference docs describe its behavior accurately.

## Scope

**In scope**
- `routing.md`: one added sentence noting `/skill <name>` as the runtime-invocable
  equivalent of `Skill("...")`.
- `docs/05_agent_07_11_cli-and-commands-slash-commands-memory-other.md`: a new
  "Skillカテゴリ" table section, inserted between the existing "Pluginカテゴリ" and
  "Otherカテゴリ" sections.

**Out of scope**
- `docs/05_agent_07_01_cli-and-commands-cli-reference.md` — this file is a chapter
  table-of-contents (front-matter `related:` list only, no command detail tables); the
  requirement's original suggestion to edit this file was corrected during planning (see
  `plans/20260712-174955_plan.md` UNK-01). No edit needed here beyond what already exists
  (it already lists `05_agent_07_11_...md` in its index table and `related:` front matter).

## Assumptions

1. `routing.md` line 7 (`Skills can be invoked as slash commands...`) is the correct
   anchor point since it is the existing sentence describing the `Skill(...)` tool-call
   form; appending the `/skill` note directly after it keeps both invocation forms
   documented together.
2. `docs/05_agent_07_11_...md`'s "Pluginカテゴリ" section (currently ends at line 62,
   right before "### Otherカテゴリ" at line 64) is the correct insertion point, matching
   the existing per-category table pattern used for Memory/MDQ/Plugin/Other.
3. No change to `docs/05_agent_07_01_cli-and-commands-cli-reference.md`'s index tables is
   needed — it already references `05_agent_07_11_cli-and-commands-slash-commands-memory-other.md`
   generically ("スラッシュコマンド: Memory, MDQ, Plugin, Other"); a new subsection inside
   that file does not require a new index row category (Skill fits under the existing
   "Memory, MDQ, Plugin, Other" grouping title, or the grouping title itself may be left
   as-is since the plan does not require an index-title change).

## Implementation

### Target file

`routing.md` and `docs/05_agent_07_11_cli-and-commands-slash-commands-memory-other.md`

### Procedure

1. Edit `routing.md` line 7: append one sentence after the existing sentence about
   `Skill("python-implementation")`.
2. Edit `docs/05_agent_07_11_cli-and-commands-slash-commands-memory-other.md`: insert a
   new `### Skillカテゴリ` section between the end of `### Pluginカテゴリ` (line 62) and
   the start of `### Otherカテゴリ` (line 64).
3. Do not touch `docs/05_agent_07_01_cli-and-commands-cli-reference.md`.

### Method

Both edits via `Edit`, anchored on exact existing text.

### Details

#### A. `routing.md` edit

Before (current line 7):
```markdown
Skills can be invoked as slash commands (e.g. `/python-implementation`) or via `Skill("python-implementation")`. The command reads `skills/*/SKILL.md` automatically.
```

After:
```markdown
Skills can be invoked as slash commands (e.g. `/python-implementation`) or via `Skill("python-implementation")`. The command reads `skills/*/SKILL.md` automatically.
`/skill <name> [args]` is the equivalent runtime-invocable form inside AgentREPL; `/skill` with no argument lists available skill names.
```

#### B. `docs/05_agent_07_11_cli-and-commands-slash-commands-memory-other.md` edit

Before (current lines 58-64, "Pluginカテゴリ" through the start of "Otherカテゴリ"):
```markdown
### Pluginカテゴリ

| Command | 副作用 | 関連する状態 |
|---|---|---|
| `/plugin status` | なし | プラグインの読み込み結果(loaded、failed、conflicts)を表示 |

### Otherカテゴリ
```

After:
```markdown
### Pluginカテゴリ

| Command | 副作用 | 関連する状態 |
|---|---|---|
| `/plugin status` | なし | プラグインの読み込み結果(loaded、failed、conflicts)を表示 |

### Skillカテゴリ

| Command | 副作用 | 関連する状態 |
|---|---|---|
| `/skill` | なし | `skills/`配下のディレクトリ名一覧を表示(`DESIGN.md`等のファイル単体エントリは除外)。LLM呼び出しは発生しない |
| `/skill <name> [args]` | `ctx.conv.history`にephemeral systemメッセージを追加(`_ephemeral: True`, `_skill_ephemeral: True`) | `skills/<name>/SKILL.md`の内容が次のLLMターンに渡される。同一セッション内で再実行すると前回分は置き換わる。次のターン開始時に`_sync_system_prompt()`で自動的に除去される |

> **注記:** `_skill_ephemeral: True`は`/skill`由来のメッセージのみを識別するための専用フラグであり、`mode_classification.py`や`memory injection`由来の`_ephemeral: True`メッセージには影響しない。実装は`agent/commands/cmd_skill.py`の`_SkillMixin`。

### Otherカテゴリ
```
