# Implementation: M-1 — tests for production fail-fast (test_startup.py)

## Goal

Add tests verifying that `_start_servers()` raises in production profile on subprocess
startup failure but only warns in local profile, and that a partial start in production
still propagates the exception (so H-6 rollback fires).

## Scope

**Target**: `tests/test_startup.py`

**Step covered**: Plan M-1 step 2.

**Out of scope**: source changes, other test files.

## Assumptions

1. `tests/test_startup.py` exists or can be extended.
2. `StartupOrchestrator` can be instantiated with a stub context that has a configurable
   `security_profile`.
3. H-6 tests cover rollback behavior; M-1 tests focus only on fail-fast vs. warn.

## Implementation

### Target file

`tests/test_startup.py`

### Procedure

#### Setup helpers

```python
def make_orchestrator_with_profile(profile: SecurityProfile):
    ctx = make_stub_ctx()
    ctx.cfg.mcp.security_profile = profile
    # Pre-populate _http_procs to simulate a subprocess server
    ctx.cfg.mcp.mcp_servers = {
        "srv": McpServerConfig(
            transport=TransportType.HTTP,
            url="http://localhost:9999",
            startup_mode=StartupMode.SUBPROCESS,
            cmd=["echo"],
        )
    }
    view = MagicMock()
    orch = StartupOrchestrator(ctx, view)
    orch._initialize = MagicMock()
    build_agent_context(ctx, view)  # or stub services directly
    return orch, view
```

#### Test 1: Local profile — startup failure → warning only, no exception

```python
@pytest.mark.asyncio
async def test_start_servers_local_profile_warns_on_failure(monkeypatch):
    orch, view = make_orchestrator_with_profile(SecurityProfile.LOCAL)

    # Patch _http_mgr.start to raise
    async def fail_start(key, cfg):
        raise RuntimeError("server error")

    orch._ctx.services_required.lifecycle._http_mgr.start.side_effect = fail_start

    await orch._start_servers()  # must NOT raise

    view.write_warning.assert_called()
```

#### Test 2: Production profile — startup failure → RuntimeError raised

```python
@pytest.mark.asyncio
async def test_start_servers_production_profile_raises_on_failure(monkeypatch):
    orch, view = make_orchestrator_with_profile(SecurityProfile.PRODUCTION)

    async def fail_start(key, cfg):
        raise RuntimeError("server error")

    orch._ctx.services_required.lifecycle._http_mgr.start.side_effect = fail_start

    with pytest.raises(RuntimeError, match="FATAL.*srv"):
        await orch._start_servers()
```

#### Test 3: Production profile — error message contains server key

```python
@pytest.mark.asyncio
async def test_production_failure_message_contains_server_key(monkeypatch):
    orch, view = make_orchestrator_with_profile(SecurityProfile.PRODUCTION)

    async def fail_start(key, cfg):
        raise OSError("no such file")

    orch._ctx.services_required.lifecycle._http_mgr.start.side_effect = fail_start

    with pytest.raises(RuntimeError) as exc_info:
        await orch._start_servers()

    assert "srv" in str(exc_info.value)
```

### Method

- `SecurityProfile.LOCAL` and `SecurityProfile.PRODUCTION` are importable from
  `shared.mcp_config`.
- Mock the lifecycle manager's `start()` method to raise for the test subprocess server.
- The mock approach depends on how `_start_servers()` calls lifecycle — adjust to match
  actual implementation (direct `lifecycle.start_http_subprocess()` call or through
  `_http_mgr`).

### Details

- Read `_start_servers()` source before writing tests to confirm how it iterates
  servers and calls start.
- If `_start_servers()` calls `_ServerLifecycleRouter.start_http_subprocess()`, patch
  at that level instead of `_http_mgr.start`.
