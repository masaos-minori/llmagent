# Implementation: app.py — broker health metrics in health() (req #42)

## Goal

Add `active_subscribers`, `max_queue_depth`, and `slow_consumers` fields to the `GET /health` response using `EventBroker` metrics methods.

## Scope

- Add 3 broker metric fields to health() response dict
- Add `broker_queue_backlog_high` and `slow_consumers_detected` to `degraded_reasons`
- Thresholds: `max_queue_depth >= 500` → backlog high; `slow_consumer_count() > 0` → slow consumers
- `_broker is None` case: metrics default to 0

## Assumptions

- req #42 broker.py metrics methods are implemented (subscriber_count, max_queue_depth, slow_consumer_count)
- health() currently returns status, degraded_reasons, uptime, etc.

## Implementation

### Target file

`scripts/eventbus/app.py`

### Procedure

1. In health(), compute broker metrics if `_broker is not None`
2. Add to degraded_reasons list as appropriate
3. Add fields to response dict

### Method

Edit health() function body.

### Details

**Addition to health() (after existing degraded_reasons collection):**
```python
# Broker health metrics
active_subscribers = 0
max_queue_depth = 0
slow_consumers = 0
if _broker is not None:
    active_subscribers = _broker.subscriber_count()
    max_queue_depth = _broker.max_queue_depth()
    slow_consumers = _broker.slow_consumer_count()

if max_queue_depth >= 500:
    degraded_reasons.append("broker_queue_backlog_high")
if slow_consumers > 0:
    degraded_reasons.append("slow_consumers_detected")
```

**Addition to response dict:**
```python
return {
    "status": "ok" if not degraded_reasons else "degraded",
    "degraded_reasons": degraded_reasons,
    "active_subscribers": active_subscribers,
    "max_queue_depth": max_queue_depth,
    "slow_consumers": slow_consumers,
    # ... existing fields ...
}
```
