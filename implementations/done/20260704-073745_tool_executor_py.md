# Implementation: Update `scripts/shared/tool_executor.py` to delegate to `PluginToolInvoker`

## Goal

Refactor `ToolExecutor` so that the inline plugin block in `execute()` is replaced by a delegation
to `PluginToolInvoker.try_execute()`, making `execute()` a thin dispatcher. Remove the
now-redundant `_PLUGIN_RESULT_TUPLE_LENGTH` constant and `from shared import plugin_registry` import.

## Scope

- In-Scope:
  - Add `from shared.plugin_tool_invoker import PluginToolInvoker` to imports.
  - Remove `from shared import plugin_registry` (line 21) — no longer needed here.
  - Remove `_PLUGIN_RESULT_TUPLE_LENGTH = 2` constant (line 44) and its comment (line 43).
  - Add `self._plugin_invoker = PluginToolInvoker()` to `__init__()`.
  - Replace inline plugin block in `execute()` (lines 344-381) with `try_execute` delegation.
- Out-of-Scope: No changes to `_raw_execute`, `_execute_with_cache`, cache logic, or any other method.

## Assumptions

1. `plugin_tool_invoker.py` has been created first (prerequisite; see `implementations/20260704-073719_plugin_tool_invoker_py.md`).
2. `from shared import plugin_registry` is used ONLY in the inline plugin block of `execute()`
   and in `_reset_for_testing()`. Verify with `grep -n "plugin_registry" scripts/shared/tool_executor.py`
   before removing. If used elsewhere, retain the import.
3. `_PLUGIN_RESULT_TUPLE_LENGTH` is not imported directly by any test; verify with
   `grep -rn "_PLUGIN_RESULT_TUPLE_LENGTH" tests/` before deleting.
4. Existing `TestPluginReturnValidation` tests in `tests/test_tool_executor.py` call `execute()`
   via the `ToolExecutor.__new__` pattern and will continue to work after the refactor.
5. The noqa: F401 re-export comments for `ToolCallResult` and `TransportErrorInfo` in the
   `from shared.transport_dto import (...)` block must be retained.
