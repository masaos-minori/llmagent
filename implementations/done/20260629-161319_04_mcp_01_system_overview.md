# Implementation: MCP Tool Count Drift Elimination

## Goal

Eliminate tool count drift across tool_constants.py, mcp_servers.toml, agent.toml, docs, and tests so that all sources agree on rag-pipeline-mcp owning 4 tools and the total MCP tool count is documented accurately.

## Scope

- **In-Scope**:
  - `config/mcp_servers.toml` — add rag_list_documents, rag_delete_document to rag_pipeline tool_names
  - `config/agent.toml` — add rag_list_documents, rag_delete_document to rag_pipeline tool_names + tool_safety_tiers + tool_definitions
  - `config/tools_definitions.toml` — add rag_list_documents, rag_delete_document entries
  - `docs/04_mcp_01_system_overview.md` — correct "43 tools" to actual total (66 including github-mcp)
  - `docs/04_mcp_04_server_catalog.md` — confirm rag-pipeline-mcp shows 4 tools (already correct)
  - `tests/test_tool_constants.py` — fix test_no_overlapping_tools to include SHELL_TOOLS, SQLITE_TOOLS, WEB_SEARCH_TOOLS in total count assertion
  - New test file `tests/test_tool_registry_counts.py` — verify registry counts match docs-declared counts
- **Out-of-Scope**:
  - Redesigning RAG pipeline tools or removing tools
  - Adding github tools to tool_constants.py frozensets
  - Modifying shared/tool_constants.py or shared/tool_registry.py (no functional changes needed)
  - Live /v1/tools endpoint testing (runtime dependency not available in CI)

## Assumptions

- All 4 RAG tools (rag_run_pipeline, rag_debug_pipeline, rag_list_documents, rag_delete_document) are production-ready (confirmed by tools.py status fields and server_catalog.md)
- The github-mcp server's 21 tools are intentionally absent from tool_constants.py; routing uses prefix matching (_GITHUB_PREFIX in route_resolver.py) rather than frozensets — this is by design
- "totaling 43 tools" in system_overview.md is stale documentation; the actual count from tool_constants frozensets is 45, and including github-mcp the full server table total is 66
