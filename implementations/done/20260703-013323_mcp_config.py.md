# Implementation: scripts/shared/mcp_config.py — Resolve UNK-03 and remove _cast_enums

**Plan source:** `plans/20260702-202739_plan.md` (Phase 1 + Phase 5)
**Target file:** `scripts/shared/mcp_config.py`

---

## Goal

Confirm that no caller of `McpServerConfig` passes raw string arguments relying on implicit enum coercion, then remove the `_cast_enums` method and its `__post_init__` call from `McpServerConfig`.

---

## Scope

**In:**
- grep for `McpServerConfig(` with raw string arguments in `tests/` and `scripts/`
- Run `uv run pytest tests/ -k mcp_config -v` to enumerate tests exercising `_cast_enums`
- If raw-string callers exist, update them to pass enum values
- Remove `_cast_enums` method body from `McpServerConfig`
- Remove the `_cast_enums()` call from `__post_init__`

**Out:**
- Refactoring unrelated code in `mcp_config.py`
- Changing MCP tool behavior
- Introducing new tests beyond covering changed import paths

---

## Assumptions

1. `_cast_enums` is called only from `__post_init__`; removing it is safe only when the config loader always passes correctly-typed enum values.
2. `uv run pytest` is the authoritative test runner for this project.

---

## Implementation

### Target file

`scripts/shared/mcp_config.py`

### Procedure

1. Run `grep -rn "McpServerConfig(" tests/ scripts/` and inspect each call site for raw string arguments (e.g. `transport="stdio"` instead of `McpTransport.STDIO`).
2. Run `uv run pytest tests/ -k mcp_config -v` to see which tests exercise `_cast_enums` implicitly.
3. For each raw-string call site found, update the argument to pass the enum value directly (e.g. `McpTransport.STDIO`).
4. Run `uv run pytest -x -q` to confirm no regressions after caller updates.
5. Open `scripts/shared/mcp_config.py` and locate the `_cast_enums` method and the `self._cast_enums()` line in `__post_init__`.
6. Remove both: the `self._cast_enums()` call in `__post_init__` and the full `_cast_enums` method definition.
7. Run `uv run pytest -x -q` to confirm tests pass after removal.

### Method

Edit tool for code changes in `scripts/shared/mcp_config.py` and any caller files that pass raw strings.

### Details

- `_cast_enums` is located at approximately line 69 of `scripts/shared/mcp_config.py`.
- The method performs implicit string-to-enum coercion for fields such as `transport`.
- After removal, callers must pass enum instances directly, e.g. `McpTransport.STDIO` instead of `"stdio"`.
- The `__post_init__` call site to remove is `self._cast_enums()`.
