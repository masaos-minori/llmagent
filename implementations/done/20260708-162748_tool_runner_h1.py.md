# Implementation: H-1 — Remove LLM-based tool result summarization from execute_one_tool_call()

## Goal

Remove the `summarize_tool_result` LLM-summarization branch from `execute_one_tool_call()`.
Long tool output is always truncated by `tool_result_max_llm_chars` — no LLM call is made to
condense it. Remove the now-unused `summarize_tool_result` import and the
`use_tool_summarize` / `tool_summarize_threshold` / `ctx.services_required.http` dependency
from this function.

## Scope

**Target**: `scripts/agent/tool_runner.py`

**Depends on / overlaps with**: the H-2 implementation step for this same file
(`implementations/20260708-162541_tool_runner_h2.py.md` or its `done/` copy) already removes
`_collect_tool_result_msgs()`'s `is_summarized()` call, `summarized` variable, and the
`is_summarized` import entirely (a superset of this plan's original item "1d: change
`summary=llm_text if summarized else None` to `summary=None`", which is moot once the H-2 step
removes the `.store()` call it belonged to). **This doc covers only the remaining,
non-overlapping part of the H-1 plan: the `execute_one_tool_call()` summarize branch and its
`summarize_tool_result` import.** Do not re-apply the `is_summarized`/`summarized` removal here
if the H-2 step has already landed — it is a no-op at that point (nothing left to remove).

**Out of scope** (per the H-1 plan itself):
- `rag/llm_client.py`'s `summarize_tool_result` function body (dead code after this change;
  removal is a separate future task)
- `scripts/agent/config_dataclasses.py` / `config_builders.py` / `services/config_reload.py`
  (`use_tool_summarize`, `tool_summarize_threshold` fields stay; removal is a separate future
  task)
- `scripts/agent/tool_result_formatter.py` (`is_summarized()` definition stays — still
  referenced by its own dedicated unit tests in `tests/test_tool_result_formatter.py`)
- `cmd_config_display.py` (no change)

## Assumptions

1. `summarize_tool_result` (imported from `rag.llm_client` at line 19) has no call site in
   `tool_runner.py` other than the one being removed (line 158) — confirm via grep in Procedure
   Step 1.
2. `ctx.services_required.http` has no other reference in `tool_runner.py` besides the summarize
   branch's condition (`ctx.services_required.http is not None`, line 156) — confirmed by the
   plan's own U-2 analysis; re-verify via grep before removing any related dead code paths (no
   removal of `services_required.http` itself is needed — it is a shared context field used
   elsewhere in the codebase, only this function's reference to it goes away).
3. `ctx.cfg.tool.use_tool_summarize` and `ctx.cfg.tool.tool_summarize_threshold` fields remain
   defined in `AgentConfig` — only their read sites in this function are removed.

## Implementation

### Target file

`scripts/agent/tool_runner.py`

### Procedure

#### Step 1: Confirm `summarize_tool_result` has exactly one call site

```bash
grep -n "summarize_tool_result" scripts/agent/tool_runner.py
```

Expected: the import (line 19) and the single call (line 158).

#### Step 2: Remove the `rag.llm_client` import

Remove line 19:

```python
from rag.llm_client import summarize_tool_result
```

#### Step 3: Collapse the summarize branch in `execute_one_tool_call()`

Current (lines 152-172):

```python
    if (
        ctx.cfg.tool.use_tool_summarize
        and not is_error
        and len(text) > ctx.cfg.tool.tool_summarize_threshold
        and ctx.services_required.http is not None
    ):
        llm_text = await summarize_tool_result(
            text, name, args, ctx.services_required.http
        )
        logger.info(
            "Tool result %s summarized: %s → %s chars",
            name,
            len(text),
            len(llm_text),
        )
    else:
        llm_text = (
            text[: ctx.cfg.tool.tool_result_max_llm_chars] + "\n... (truncated)"
            if len(text) > ctx.cfg.tool.tool_result_max_llm_chars
            else text
        )
```

Replace with:

```python
    llm_text = (
        text[: ctx.cfg.tool.tool_result_max_llm_chars] + "\n... (truncated)"
        if len(text) > ctx.cfg.tool.tool_result_max_llm_chars
        else text
    )
```

#### Step 4: Update the function docstring

Current docstring line: `"""Parse, execute, and optionally summarize one tool_call dict.`

Replace with: `"""Parse, execute, and truncate one tool_call dict.` — the function no longer
"optionally summarizes"; it always truncates when over the char limit.

### Method

- Pure branch removal; the `else` clause becomes the only path, so it is unindented and the `if`
  wrapper is deleted.
- No signature change — `execute_one_tool_call()`'s parameters and return tuple shape are
  unaffected.

### Details

- `logger` (module-level logger used by the removed `logger.info("Tool result %s summarized...")`
  call) has other call sites in this file (e.g. `_log_and_emit_tool_call`,
  `_apply_turn_char_limit`) — do not remove the `logger` definition or its import.
- `ctx.services_required.http` remains a valid, populated field on `AppServices` used by other
  modules (e.g. `agent/repl_health.py`) — this change only removes the one read of it inside
  `execute_one_tool_call()`.
- If this doc is applied AFTER the H-2 doc has already removed the `is_summarized` import from
  the same import block, do not attempt to remove it again — the import block will already be
  reduced to `from agent.tool_result_formatter import (TURN_LIMIT_HINT, mask_args)` or similar
  at that point.

  test exercising the summarize path in `tool_runner.py` — only `test_tool_result_formatter.py`
  tests `is_summarized()` directly, which is unaffected since that function definition stays.
  Re-run `grep -rn "use_tool_summarize" tests/` before finalizing to confirm this still holds.
