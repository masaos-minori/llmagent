# Implementation: types.py + mapper.py — Centralize validation, remove silent fallbacks

## Goal

Add `updated_at` validation to `MemoryEntry.__post_init__` in `types.py`, and remove all
silent `source_type → CONVERSATION` and `tags → []` fallbacks from `mapper.py`.
`MemoryEntry.__post_init__` becomes the single validation point for all fields.

## Scope

- `scripts/agent/memory/types.py`: add `updated_at` format validation.
- `scripts/agent/memory/mapper.py`: remove `source_type` fallback; let `MemoryEntry()` raise; remove `tags` silent fallback.

Note: `jsonl_store.py` fallback removal is covered in the `jsonl_store.py` implementation doc.

## Assumptions

1. `types.py.__post_init__` already validates `created_at` with `_ISO8601_RE`. The same regex applies to `updated_at`.
2. `updated_at` may be an empty string (valid; the existing `created_at` check allows empty string via `if self.created_at and ...`).
3. In `mapper.py`, removing the `source_type` fallback means passing the raw string directly into `MemoryEntry()`; `__post_init__` will coerce or raise `ValueError`.
4. For `tags`: instead of silently catching `orjson.loads` errors and returning `[]`, propagate the exception so the caller (`row_to_entry`) raises `ValueError` and the row is skipped higher up. This makes the failure explicit.
5. No existing production data has malformed `tags` (accepted risk; see plan Risks section).

## Implementation

### Target files

- `scripts/agent/memory/types.py`
- `scripts/agent/memory/mapper.py`

### Procedure

**types.py — add updated_at validation in __post_init__:**
```python
if self.updated_at and not _ISO8601_RE.match(self.updated_at):
    raise ValueError(
        f"updated_at must be ISO-8601 UTC (YYYY-MM-DDTHH:MM:SSZ), got {self.updated_at!r}",
    )
```
Add immediately after the `created_at` check.

**mapper.py — remove source_type fallback:**

Before:
```python
raw_source = d.get("source_type", "conversation")
try:
    source_type = SourceType(raw_source)
except ValueError:
    source_type = SourceType.CONVERSATION
return MemoryEntry(
    ...
    source_type=source_type,
    ...
)
```

After:
```python
return MemoryEntry(
    ...
    source_type=d.get("source_type", "conversation"),
    ...
)
```
Remove the `raw_source` variable, the `try/except ValueError`, and the `source_type` local.
Remove the `SourceType` import from `mapper.py` (no longer needed directly).

**mapper.py — remove tags fallback:**

Before:
```python
tags_raw = d.get("tags", "[]")
try:
    tags: list[str] = (
        orjson.loads(tags_raw) if isinstance(tags_raw, str) else list(tags_raw)
    )
except Exception:
    tags = []
```

After:
```python
tags_raw = d.get("tags", "[]")
tags: list[str] = (
    orjson.loads(tags_raw) if isinstance(tags_raw, str) else list(tags_raw)
)
```
Remove the `try/except` wrapper. If `orjson.loads` fails, the exception propagates to the caller,
which skips the row.

### Method

Minimal line removals. No new imports needed. `SourceType` import in `mapper.py` becomes unused
after removing the explicit `SourceType(...)` call — remove it.

### Details

After these changes, `mapper.py` no longer imports `SourceType` directly:
```python
# Remove this line:
from agent.memory.types import MemoryEntry, SourceType
# Becomes:
from agent.memory.types import MemoryEntry
```
