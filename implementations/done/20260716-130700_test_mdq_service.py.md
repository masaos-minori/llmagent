# Implementation: tests/test_mdq_service.py (regression test for removed `audit_log_path`)

Source plan: `plans/20260716-121714_plan.md`

## Goal

Add a regression test asserting that `MdqService` has no `audit_log_path`
attribute after the field removal in `scripts/mcp_servers/mdq/service.py`,
and that construction still succeeds from a config dict/file that lacks the
key (it already does, since `config/mdq_mcp_server.toml` has not had the key
since 2026-07-13).

## Scope

**In:**
- One new test function in `tests/test_mdq_service.py` that:
  1. Constructs an `MdqService` (reuse the existing `service` fixture at
     `tests/test_mdq_service.py:36-47`, or construct a bare instance).
  2. Asserts `not hasattr(service, "audit_log_path")`.
  3. Asserts construction did not raise (implicit — fixture/constructor
     succeeding is the assertion).

**Out:**
- Any other `MdqService` field or method.
- Modifying the existing `service` fixture's behavior.

## Assumptions

1. The `service` fixture at `tests/test_mdq_service.py:36-47` already
   constructs `MdqService(db_path=db)` without an `audit_log_path` config
   key present — it can be reused as-is for this new test with no fixture
   changes.
2. This test must run *after* the `service.py` change in the companion
   implementation doc (`implementations/20260716-130659_service.py.md`) —
   before that change, `hasattr(service, "audit_log_path")` is `True` and
   this test fails, which is the expected red-then-green sequence.

## Implementation

### Target file

`tests/test_mdq_service.py`

### Procedure

1. Open `tests/test_mdq_service.py`.
2. Locate the fixtures section (lines 33-47, the `# ── fixtures ──` block
   containing the `service` fixture).
3. Find the section containing `MdqService.__init__`-focused tests (search
   for existing tests that construct `MdqService` directly or assert on its
   attributes — if no such section exists, add the new test immediately
   after the fixtures block, before the first tool-level test).
4. Add a new test function:
   ```python
   def test_service_has_no_audit_log_path_attribute(service: MdqService) -> None:
       """audit_log_path was a dead config field; MdqService must not carry it."""
       assert not hasattr(service, "audit_log_path")
   ```
5. Confirm the `service` fixture parameter type hint matches the existing
   fixture's return type (`MdqService`), matching the style of other tests
   in the file that consume the same fixture.

### Method

Single new test function using the existing `service` fixture — no new
fixtures, no parametrization, no mocking required (construction from a
config dict without the key is already the fixture's default behavior).

### Details

- Place the test near other `MdqService`-construction-level tests (not
  interspersed with `search_docs`/`grep_docs` tool-behavior tests) to keep
  related assertions grouped.
- Do not assert on the *value* of any config field the field might have had
  (e.g. do not assert `service.audit_log_path is None`) — the point is the
  attribute must not exist at all, per the plan's stated design (no
  replacement field, pure removal).
- Keep the docstring in English per `rules/coding.md` ("comments and log
  output: English only").
