# Implementation: H-4 — Update /stats partial-completion hint in cmd_config_stats.py

## Goal

Change the `/stats` command's partial-completion line to describe where the data is actually
stored (`session_diagnostics`) instead of the removed `tool_result` / `llm_partial_completion`
reference.

## Scope

**Target**: `scripts/agent/commands/cmd_config_stats.py`

**Depends on**: `implementations/20260708-163427_llm_transport_errors.py.md` (same rationale as
the `repl.py` doc — a display-string consistency dependency, not a hard code dependency).

**Out of scope**: the `stats.llm_partial_completions > 0` branching logic and the `else` branch
printing `"  Partial compl : 0"` — unchanged.

## Assumptions

1. The hint text lives in the `_cmd_config_stats` (or equivalently named) display method, inside
   the `if stats.llm_partial_completions > 0:` branch — confirm exact line numbers with
   `grep -n "Partial compl" -A2 scripts/agent/commands/cmd_config_stats.py` before editing.
2. No other file duplicates this exact string (verify via grep in Procedure Step 1).

## Implementation

### Target file

`scripts/agent/commands/cmd_config_stats.py`

### Procedure

#### Step 1: Confirm single occurrence of the old hint

```bash
grep -rn "stored as tool_result, tool_name='llm_partial_completion'" scripts/
```

Expect exactly one match, in this file.

#### Step 2: Replace the hint string

Current:

```python
        if stats.llm_partial_completions > 0:
            self._out.write(
                f"  Partial compl : {stats.llm_partial_completions}"
                "  (stored as tool_result, tool_name='llm_partial_completion')"
            )
        else:
            self._out.write("  Partial compl : 0")
```

Replace with:

```python
        if stats.llm_partial_completions > 0:
            self._out.write(
                f"  Partial compl : {stats.llm_partial_completions}"
                "  (stored in session_diagnostics)"
            )
        else:
            self._out.write("  Partial compl : 0")
```

### Method

- Single string-literal replacement inside the existing `if`/`else`; no control-flow change.

### Details

- The `else` branch (`"  Partial compl : 0"`) is unaffected — it does not reference
  `tool_result`/`llm_partial_completion` and needs no change.
- `stats.llm_partial_completions` (the counter itself) is read-only here and unaffected by the
  companion `llm_transport_errors.py` change (the counter increment logic is preserved there).
