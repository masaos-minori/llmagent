# Implementation: Add Per-Server Tool Error Counters

## Goal

Track the count of tool-level errors (`is_error=True`) per MCP server so operators can query how many times each server's tools have failed without scanning log files.

## Scope

- `scripts/shared/tool_executor.py`
  - Add `stat_tool_errors: dict[str, int]` counter dict (per server_key)
  - Increment on each `is_error=True` result in `_raw_execute()`
  - Expose via a read property or existing stats interface

Out of scope:
- Transport error counters (already tracked via `_health_registry`)
- Prometheus/external export

## Assumptions

1. `stat_cache_hits: int = 0` (line 403) is the existing stats pattern; extend with a dict for per-server counters.
2. `server_key` is available in `_raw_execute()` (confirmed — it is a parameter).
3. Callers of `ToolExecutor` may read `stat_cache_hits` directly; the new `stat_tool_errors` dict follows the same pattern.
4. Thread safety is not required — `ToolExecutor` runs in a single async event loop.

## Implementation

### Target file

`scripts/shared/tool_executor.py`

### Procedure

1. In `__init__`, add `self.stat_tool_errors: dict[str, int] = {}`.
2. In `_raw_execute()`, after detecting `result.is_error`, increment the counter.
3. Optionally add a `tool_error_total` property that sums the dict values.

### Method

Minimal: one new field in `__init__`, one increment in `_raw_execute()`.

### Details

**`__init__` addition (after line 403):**
```python
self.stat_cache_hits: int = 0
self.stat_tool_errors: dict[str, int] = {}   # server_key → count of is_error=True results
```

**`_raw_execute()` — after line 521 (successful transport, tool error check):**
```python
if result.is_error:
    self.stat_tool_errors[server_key] = (
        self.stat_tool_errors.get(server_key, 0) + 1
    )
    logger.debug(
        "tool error from %r: %s (error_type=tool)",
        server_key,
        result.output[:120],
    )
```

(The logger.debug line may already be present from Step 1 — combine if implementing together.)

**Optional read property:**
```python
@property
def stat_tool_errors_total(self) -> int:
    return sum(self.stat_tool_errors.values())
```

**Usage by operators (via session diagnostics or a `/db health`-style command):**
```python
executor.stat_tool_errors          # {"rag_pipeline": 3, "github": 0}
executor.stat_tool_errors_total    # 3
```
