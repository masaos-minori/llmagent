# Implementation: Update StdioTransport mock specs in test_lifecycle.py, test_repl_health.py, test_startup.py

## Goal

Remove StdioTransport references from test_lifecycle.py, test_repl_health.py, and test_startup.py so that no tests depend on the removed stdio_lifecycle module or StdioTransport class.

## Scope

- Target files:
  - tests/test_lifecycle.py
  - tests/test_repl_health.py
  - tests/test_startup.py
- Deletion of stdio-dependent test classes and skipping of remaining stdio tests
- Removal of `cmd` parameter from McpServerConfig constructors in tests

## Assumptions

- StdioTransport class no longer exists in shared.tool_executor
- agent.stdio_lifecycle module removed — TransportHandle import needs stub or skip
- `_ServerLifecycleRouter.__init__()` takes 2 positional arguments (server_configs, tool_executor) — no stdio_procs parameter
- McpServerConfig no longer accepts `cmd` parameter — all test config constructors need updating
- HttpSubprocess tests that depend on cfg.cmd cannot be fixed without source code changes — skip with pytest.skip

## Implementation

### Target file: tests/test_lifecycle.py

#### Procedure

1. Replace TransportHandle import with try/except stub class for remaining tests
2. Remove `cmd=[]` from `_http_cfg()` function
3. Remove `cmd` parameter and `cmd=cmd or [...]` from `_http_subprocess_cfg()`, `_stdio_ondemand()`, `_stdio_ondemand_idle()`, `_ondemand_echo_cfg()`
4. Remove `cmd=["python", "s.py"]` from all McpServerConfig constructors in test functions
5. Remove third argument (stdio_procs) from all `_ServerLifecycleRouter()` constructor calls
6. Skip stdio-dependent tests in TestEnsureReady, TestShutdownIdle, TestEnsureReadyIntegration, TestShutdownIdleIntegration
7. Skip HttpSubprocess tests that depend on cfg.cmd (source code issue — cannot fix without source changes)
8. Skip TestRestart tests that depend on cfg.cmd
9. Skip TestTransportHandle.test_default_last_error_is_none (StdioTransport import removed)

#### Method

- Read each deletion target in full before modifying to confirm no shared logic is embedded
- Use pytest.skip() for tests that depend on removed stdio_lifecycle module or cfg.cmd attribute
- After each modification, verify the file still parses (no dangling references)

### Target files: tests/test_repl_health.py, tests/test_startup.py

#### Procedure

1. Remove StdioTransport imports
2. Remove _stdio_persistent_cfg() and _stdio_ondemand_cfg() helper functions
3. Update _http_subprocess_cfg() to remove cmd parameter
4. Remove all tests that depend on stdio transports
