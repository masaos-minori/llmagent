# Implementation: Add get_activation_mode() to memory services.py

Steps covered: Plan 20260626-100125 — Steps 1-5

---

## Goal

Add `get_activation_mode() -> str` to `MemoryServices` in `scripts/agent/memory/services.py`, returning one of `"disabled"` / `"fts-only"` / `"degraded"` / `"hybrid"`. Display this in startup banner and `/memory status`.

---

## Scope

- **In scope**:
  - `scripts/agent/memory/services.py`: `get_activation_mode()` method
  - startup banner: memory mode display
  - `/memory status` handler: mode line
  - docs: `05_agent_08_configuration.md`, `05_agent_10_operations-and-observability.md`
- **Out of scope**: auto memory setup

---

## Assumptions

- Mode determination:
  - `use_memory_layer=False` → `"disabled"`
  - `use_memory_layer=True` + embedding disabled → `"fts-only"`
  - `use_memory_layer=True` + embedding enabled + circuit open → `"degraded"`
  - `use_memory_layer=True` + embedding enabled + circuit closed → `"hybrid"`
- `EmbeddingClientStatus.enabled` exists (plan 90 adds `local_only`).

---

## Implementation

### Target file
`scripts/agent/memory/services.py`

### Procedure
1. Read `scripts/agent/memory/services.py` to find the class structure.
2. Step 1: Add `get_activation_mode()`:
   ```python
   def get_activation_mode(self) -> str:
       if not self._config.use_memory_layer:
           return "disabled"
       embed_status = self.embedding_client.get_status()
       if not embed_status.enabled:
           return "fts-only"
       if embed_status.circuit_open:
           return "degraded"
       return "hybrid"
   ```
3. Step 2: In startup banner, add:
   ```python
   mode = memory_services.get_activation_mode()
   logger.info("[memory] mode=%s", mode)
   ```
4. Step 3: In `/memory status` handler, add:
   ```
   mode: hybrid (embed: circuit_open=False, fail_count=0)
   ```
5. Steps 4-5: Update docs with activation mode table.

### Method
Method addition. No behavioral change.
