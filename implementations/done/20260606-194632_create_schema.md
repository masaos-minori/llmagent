# Implementation: Remove Legacy Memory Tables from Session Schema DDL

## Goal

Remove the legacy `memory_entries` and `memory_vec` tables from `_SESSION_SCHEMA_TEMPLATE` in `scripts/db/create_schema.py`. These tables were superseded by `memories`, `memories_fts`, `memory_links`, and `memories_vec`; the baseline DDL should not create them.

## Scope

**In:**
- Delete the `CREATE TABLE IF NOT EXISTS memory_entries (...)` block (lines ~127-133)
- Delete the `CREATE VIRTUAL TABLE IF NOT EXISTS memory_vec USING vec0(...)` block (lines ~134-137)
- Remove the now-unnecessary `DROP TABLE IF EXISTS memory_entries` and `DROP TABLE IF EXISTS memory_vec` migration statements (lines ~250-251)

**Out:**
- Changing existing production databases (tables already dropped by prior migrations; no data impact)
- Changing the new `memories*` schema

## Assumptions

- `scripts/db/create_schema.py`: `_SESSION_SCHEMA_TEMPLATE` string (line ~90) contains both old and new memory table DDL
- `_SESSION_MIGRATE_SQL: list[str]` contains `"DROP TABLE IF EXISTS memory_entries"` and `"DROP TABLE IF EXISTS memory_vec"` as cleanup migrations
- New tables (`memories`, `memories_fts`, `memory_links`, `memories_vec`) remain in `_SESSION_SCHEMA_TEMPLATE`
- `db/migrate.py` does not exist; migration logic lives inside `create_schema.py`

## Implementation

### `scripts/db/create_schema.py`

**Step A — Remove from `_SESSION_SCHEMA_TEMPLATE`:**

Delete these lines from the template string:
```sql
    CREATE TABLE IF NOT EXISTS memory_entries (
        entry_id   INTEGER PRIMARY KEY AUTOINCREMENT,
        session_id INTEGER,
        mem_type   TEXT NOT NULL CHECK (mem_type IN ('long_term', 'task')),
        content    TEXT NOT NULL,
        created_at TEXT NOT NULL DEFAULT (datetime('now'))
    );
    CREATE VIRTUAL TABLE IF NOT EXISTS memory_vec USING vec0(
        entry_id  INTEGER PRIMARY KEY,
        embedding float[DIMS]
    );
```

The template still retains: `memories`, `memories_fts`, `memory_links`, `memories_vec`.

**Step B — Remove DROP migrations from `_SESSION_MIGRATE_SQL`:**

Remove the two statements that are no longer needed once the baseline no longer creates the old tables:
```python
"DROP TABLE IF EXISTS memory_entries",
"DROP TABLE IF EXISTS memory_vec",
```

These were cleanup migrations; after removing the tables from the baseline DDL, they remain harmless for existing DBs (DROP IF EXISTS) but are no longer semantically meaningful. Remove them to keep the migration list clean.

## Validation plan

```bash
uv run ruff check scripts/db/create_schema.py
uv run mypy scripts/
uv run pytest tests/test_create_schema.py -v
```

Specific checks:
1. A fresh DB created via `_build_session_schema_sql()` does NOT contain `memory_entries` or `memory_vec` tables
2. A fresh DB DOES contain `memories`, `memories_fts`, `memory_links`, `memories_vec` tables
3. Running `_run_migrations()` on an existing DB with old tables still present should be tested with `DROP TABLE IF EXISTS` to confirm it still works (the DROP in migrations can stay if needed for backward compat, or be removed with a test verifying no errors on a DB that never had the old tables)

Add to `tests/test_create_schema.py`:
```python
def test_session_schema_no_legacy_memory_tables(tmp_path):
    """Newly created session DB should not have memory_entries or memory_vec."""
    db_path = tmp_path / "session.sqlite"
    with SQLiteHelper(str(db_path), db_type="session") as db:
        cur = db.conn.execute(
            "SELECT name FROM sqlite_master WHERE type='table' AND name IN "
            "('memory_entries', 'memory_vec')"
        )
        assert cur.fetchall() == [], "Legacy tables must not exist in new schema"

def test_session_schema_has_new_memory_tables(tmp_path):
    """New memory tables must be present."""
    db_path = tmp_path / "session.sqlite"
    with SQLiteHelper(str(db_path), db_type="session") as db:
        cur = db.conn.execute(
            "SELECT name FROM sqlite_master WHERE type IN ('table','shadow') "
            "AND name IN ('memories', 'memory_links')"
        )
        names = {row[0] for row in cur.fetchall()}
        assert "memories" in names
        assert "memory_links" in names
```
