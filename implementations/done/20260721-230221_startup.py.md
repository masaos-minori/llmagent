## Goal
Change startup rollback failure handling from WARNING to ERROR level and raise RuntimeError when shutdown_all() fails during rollback in startup.py.

## Scope
**In**: 
- Change log level from `logger.warning(...)` → `logger.error(...)` in `scripts/agent/startup.py` around line 69-71
- Add `raise RuntimeError(...)` after the error log at line ~71

**Out**:
- No changes to behavior when shutdown_all() succeeds
- No changes to exception propagation when no subprocesses were started
- No test modifications beyond adding new tests

## Assumptions
1. The custom `Logger` class supports `.error()` method (confirmed via `__getattr__` delegation to stdlib logger)
2. `logger` variable exists in startup.py (confirmed at line 37)
3. Raising RuntimeError is acceptable — will propagate up through the startup sequence

## Implementation

### Target file
- `scripts/agent/startup.py`

### Procedure
1. Locate the rollback exception handling block around line 64-72
2. Replace `logger.warning(...)` with `logger.error(...)`
3. Add `raise RuntimeError(...)` after the error log
4. Run lint/typecheck to confirm no regressions

### Method
Replace the existing warning-only rollback failure handling with an error-level log followed by raising a RuntimeError that chains the original shutdown exception.

### Details

```python
# In scripts/agent/startup.py, replace lines 64-72:

# Before:
        except Exception:
            if _servers_started:
                try:
                    await self._ctx.services_required.lifecycle.shutdown_all()
                except Exception as shutdown_err:
                    logger.warning(
                        "Startup rollback: shutdown_all() failed: %s", shutdown_err
                    )
            raise

# After:
        except Exception:
            if _servers_started:
                try:
                    await self._ctx.services_required.lifecycle.shutdown_all()
                except Exception as shutdown_err:
                    logger.error(
                        "CRITICAL: Startup rollback FAILED — subprocesses may be orphaned: %s",
                        shutdown_err,
                    )
                    raise RuntimeError(
                        f"Startup rollback failed: {shutdown_err}. "
                        "Subprocesses may be orphaned."
                    ) from shutdown_err
            raise
```

Key points:
- Use `logger.error()` (not `_logger.error()`) — startup.py uses `logger` per convention
- Message prefix "CRITICAL:" makes the severity immediately visible in logs
- RuntimeError chains the original shutdown_err via `from shutdown_err` preserving the traceback
- RuntimeError message explicitly mentions potential orphaned subprocesses for operator awareness
- Only applies when `_servers_started=True`; when False, original exception propagates unchanged
