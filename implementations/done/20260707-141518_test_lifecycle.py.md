# Implementation: H-1 — tests for stderr log file redirect (test_lifecycle.py)

## Goal

Add tests for the new stderr log file redirect behavior added to
`HttpServerLifecycleManager` in H-1. Verify that: the subprocess does not block on pipe
buffer fill, `HttpStartupError.failure.stderr_full` is read from the log file on early
exit and timeout, and the log file handle is closed on restart.

## Scope

**Target**: `tests/test_lifecycle.py`

**Step covered**: Plan H-1 step 4.

**Out of scope**: source changes, other test files.

## Assumptions

1. `tests/test_lifecycle.py` already exists with tests for `HttpServerLifecycleManager`.
2. Tests can write to a temp directory using `tmp_path` (pytest fixture); monkeypatching
   `_open_stderr_log()` to redirect to `tmp_path` avoids `/opt/llm/logs/mcp` side effects.
3. `subprocess.PIPE` no longer appears in `start()` after H-1; tests must not assume it.

### Procedure

#### Test 1: Large stderr output does not block subprocess

```python
def test_start_large_stderr_does_not_block(tmp_path, monkeypatch):
    """Subprocess writing > 64 KiB to stderr must not stall start()."""
    # Patch _open_stderr_log to use tmp_path
    original_open = HttpServerLifecycleManager._open_stderr_log

    def patched_open(self, server_key):
        log_path = tmp_path / f"{server_key}.stderr.log"
        fh = log_path.open("ab")
        self._stderr_log_paths[server_key] = str(log_path)
        return fh

    monkeypatch.setattr(HttpServerLifecycleManager, "_open_stderr_log", patched_open)

    mgr = HttpServerLifecycleManager()
    cfg = make_test_cfg(
        cmd=["python", "-c", "import sys; sys.stderr.write('x' * 200_000); sys.exit(1)"],
        startup_timeout_sec=5,
    )
    with pytest.raises(HttpStartupError) as exc_info:
        asyncio.run(mgr.start("test_server", cfg))
    # Must complete without hanging; stderr_full is truncated to 64 KiB
    assert len(exc_info.value.failure.stderr_full) <= 64 * 1024 + 10
```

#### Test 4: Restart closes old log file handle

```python
def test_restart_closes_old_stderr_handle(tmp_path, monkeypatch):
    """restart() must close the old stderr file handle before re-starting."""
    _patch_open_to_tmp(monkeypatch, tmp_path)

    mgr = HttpServerLifecycleManager()
    # Pre-populate _http_procs with a fake process and a file handle
    mock_proc = _make_mock_proc(running=True)
    mgr._http_procs["srv"] = mock_proc
    fh = (tmp_path / "srv.stderr.log").open("ab")
    mgr._stderr_files["srv"] = fh
    mgr._stderr_log_paths["srv"] = str(tmp_path / "srv.stderr.log")

    # Stub start() to avoid actually launching a subprocess
    async def fake_start(server_key, cfg):
        pass

    monkeypatch.setattr(mgr, "start", fake_start)

    asyncio.run(mgr.restart("srv", make_test_cfg()))
    assert fh.closed
    assert "srv" not in mgr._stderr_files
```

### Method

- Use `monkeypatch.setattr` to redirect `_open_stderr_log` to `tmp_path`; avoids
  real filesystem side effects.
- Helper `_patch_open_to_tmp(monkeypatch, tmp_path)` extracts the monkeypatch logic.
- `make_test_cfg(...)` is assumed to be an existing helper or created as a local
  factory using `McpServerConfig(transport=TransportType.HTTP, url="http://localhost:9999", ...)`.

### Details

- Tests must import `asyncio`, `pytest`, `HttpServerLifecycleManager`,
  `HttpStartupError`, `McpServerConfig`, `TransportType`.
- Tests that launch real subprocesses run in `asyncio.run()` to avoid event loop
  leakage between tests.
- `_patch_open_to_tmp` should be a module-level helper (not fixture) so tests can call
  it inline.
