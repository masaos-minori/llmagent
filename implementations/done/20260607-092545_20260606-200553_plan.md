# Goal

Implement MCP health check with degraded state control to automatically disable tools when MCP servers are unavailable.

# Scope

- Add `McpServerHealthState` enum and `McpServerHealthRegistry` 
- Update `ToolExecutor` to include an unavailable gate
- Implement watchdog to update health status
- Add health state display to /mcp endpoint

# Assumptions

1. The `repl_health.py::watchdog_loop()` already performs periodic health checks.
2. `ToolExecutor` currently does not maintain health status.
3. Implementation documentation exists in `implementations/20260606-195339_mcp_health_states.md`.

# Implementation

1. Create `McpServerHealthState` enum and `McpServerHealthRegistry` in `shared/tool_executor.py`
2. Add unavailable gate logic to `ToolExecutor`
3. Update watchdog in `repl_health.py` to update health states
4. Modify `/mcp` display in `mcp_status.py` to show health states
