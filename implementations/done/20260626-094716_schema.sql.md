# Implementation: Add DLQ indexes to schema.sql

Steps covered: Plan 20260626-094716 — Step 1

---

## Goal

Add indexes to the `dlq` table in `schema.sql` to support efficient DLQ queries (by `status`, `stream`, `created_at`).

---

## Scope

- **In scope**: `scripts/eventbus/schema.sql` (or equivalent) — add CREATE INDEX statements for DLQ table
- **Out of scope**: db.py (step 2); docs (step 3); tests (step 4)

---

## Assumptions

- `dlq` table has columns: `id`, `stream`, `event_id`, `status` (`pending`/`promoted`/`dropped`), `created_at`, `attempts`.
- Common queries: filter by `status`, filter by `stream + status`, sort by `created_at`.
- Existing schema has no DLQ indexes (or only a primary key index on `id`).

---

## Implementation

### Target file
`scripts/eventbus/schema.sql`

### Procedure
1. Read `schema.sql` to confirm the `dlq` table structure.
2. Add indexes after the `dlq` table definition:
   ```sql
   -- DLQ query optimization indexes
   CREATE INDEX IF NOT EXISTS idx_dlq_status ON dlq (status);
   CREATE INDEX IF NOT EXISTS idx_dlq_stream_status ON dlq (stream, status);
   CREATE INDEX IF NOT EXISTS idx_dlq_created_at ON dlq (created_at);
   CREATE INDEX IF NOT EXISTS idx_dlq_event_id ON dlq (event_id);
   ```
3. Ensure `IF NOT EXISTS` is used so schema.sql is idempotent.

### Method
Schema-only change. Indexes will be created on next DB initialization. Existing DBs will need a migration or re-init.
