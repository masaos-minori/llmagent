# Implementation: Add security_lockdown_enabled Config Option

## Goal

Add a `security_lockdown_enabled` boolean config option that, when `true`, suppresses deny-all warnings in `audit_security_defaults()`. This allows operators who have intentionally configured a deny-all state to acknowledge it without noisy startup warnings.

## Scope

- `scripts/agent/config_dataclasses.py` — add `security_lockdown_enabled: bool = False` to `AgentConfig`
- `scripts/agent/config_builders.py` — read `security_lockdown_enabled` from TOML
- `scripts/agent/repl_health.py` — check the flag before logging deny-all warnings

Out of scope:
- Changing enforcement logic
- Hot-reload of the lockdown flag
- Per-server lockdown flags (single global flag)

## Assumptions

1. `AgentConfig` is the correct home for this flag (it drives `audit_security_defaults()` via `ctx.cfg`).
2. `security_lockdown_enabled: bool = False` means warnings are ON by default — opt-in suppression.
3. The flag is read from `agent.toml` or `common.toml` (verify which TOML `AgentConfig` reads from).
4. The flag suppresses ALL deny-all warnings, not individual ones.
5. When `security_lockdown_enabled=True`, a single INFO log entry should confirm lockdown mode is active (so the operator can verify the flag is being read).

## Implementation

### Target file

`scripts/agent/repl_health.py` (read path); `scripts/agent/config_dataclasses.py` (definition)

### Procedure

1. In `config_dataclasses.py`, add `security_lockdown_enabled: bool = False` to `AgentConfig`.
2. In `config_builders.py`, populate it from `cfg.get("security_lockdown_enabled", False)`.
3. In `audit_security_defaults()`, at the start of the fail-closed checks, read `ctx.cfg.security_lockdown_enabled`. If `True`, log an INFO message and skip all deny-all warnings; return early from the deny-all block.

### Method

Single flag check before each deny-all warning. Wrap in a guard at the block level (not individual per-check guards) for simplicity.

### Details

**config_dataclasses.py — AgentConfig (find the security section):**
```python
security_lockdown_enabled: bool = False
```

**config_builders.py:**
```python
security_lockdown_enabled=bool(cfg.get("security_lockdown_enabled", False)),
```

**repl_health.py — at the start of the fail-closed section (after line 440):**
```python
lockdown = getattr(ctx.cfg, "security_lockdown_enabled", False)
if lockdown:
    logger.info(
        "Security: security_lockdown_enabled=True — deny-all warnings suppressed "
        "(intentional lockdown acknowledged)"
    )
    # Skip fail-closed deny-all checks — operator has acknowledged them
else:
    # Check shell sandbox and command_allowlist
    try:
        ...  # existing and new deny-all check blocks
    ...
    # Security posture summary
    if fail_closed_empty or fail_open_empty:
        ...
```

Alternatively, wrap each individual check in `if not lockdown:` for finer granularity — but the single block guard is simpler and matches the plan's intent.

**agent.toml / common.toml documentation:** Add a comment in the template config (if one exists):
```toml
# security_lockdown_enabled = false  # Set to true to suppress deny-all startup warnings
```
