# Implementation: M-3 — tool_executor.py apply_config docstring

Source plan: `plans/20260709-100825_plan.md` (M-3, Implementation step 1).

## Goal

Make `ToolExecutor.apply_config()`'s narrow hot-reload scope (cache TTL
only) explicit, so a future contributor doesn't assume it can be extended
to hot-reload MCP server config.

## Scope

**Target**: `scripts/shared/tool_executor.py`, `apply_config()` method
(lines 123-126). No behavior change — docstring only.

## Assumptions

1. `apply_config()` currently only mutates `self._cache_ttl` — verified by
   reading the method body while planning M-3; `_transports`,
   `_server_configs`, `_resolver`, `_concurrency_limits`, `_semaphores` are
   all set only in `__init__` and never touched elsewhere in the class.

## Implementation

### Target file

`scripts/shared/tool_executor.py`

### Procedure

#### Step 1: Expand the docstring

Current:
```python
    def apply_config(self, *, cache_ttl: float | None = None) -> None:
        """Update hot-reloadable configuration fields without recreating the instance."""
        if cache_ttl is not None:
            self._cache_ttl = cache_ttl
```

Replace with:
```python
    def apply_config(self, *, cache_ttl: float | None = None) -> None:
        """Update hot-reloadable configuration fields without recreating the instance.

        Only `cache_ttl` is hot-reloadable here. The following are restart-required
        and are never touched by this method: server configs (`_server_configs`),
        HTTP transports (`_transports`), the route resolver (`_resolver`), server
        add/remove, and concurrency limits/semaphores (`_concurrency_limits`,
        `_semaphores`). MCP server definitions are restart-time snapshots — see
        `ConfigReloadService._classify_mcp_server_changes()`.
        """
        if cache_ttl is not None:
            self._cache_ttl = cache_ttl
```

### Method

- Docstring-only change; no logic touched.
