# Implementation: mcp/github/tools.py — Model-Derived Schema Generation

## Goal

Replace the 500-line hand-written `_MCP_TOOLS` list with a `build_tool_list()` function that generates MCP tool schemas from Pydantic request models via `model_json_schema()`. The JSON output seen by MCP clients must remain identical.

---

## Scope

**Target file:** `scripts/mcp/github/tools.py`

**In:**
- Keep `_MCP_TOOLS` as the authoritative list of `(tool_name, description, RequestModel)` tuples
- Add `build_tool_list() -> list[dict]` that generates the full `inputSchema` from each model
- The `inputSchema` format: `{"type": "object", "properties": {...}, "required": [...]}`
- Add a module-level `TOOL_LIST = build_tool_list()` constant replacing `_MCP_TOOLS` direct use
- Update `server.py` to use `TOOL_LIST` instead of `_MCP_TOOLS`

**Out:**
- Tool descriptions unchanged
- Tool names unchanged
- No changes to models.py request model field definitions

---

## Assumptions

1. `model_json_schema()` (Pydantic v2) generates `{"type": "object", "properties": {...}, "required": [...]}` which matches the structure of the existing `inputSchema` entries.
2. Pydantic's `$defs` for nested models (e.g. `PushFile` in `PushFilesRequest`) needs to be inlined/resolved before use as `inputSchema`.
3. The `description` in the generated schema comes from `Field(description=...)` — matching the existing hand-written descriptions.
4. Fields with `default=` are NOT in `required` (Pydantic behavior matches existing hand-written schemas).
5. `dry_run` field is optional in all write-operation models — no schema change needed.

---

## Implementation

### Target file
`scripts/mcp/github/tools.py`

### Procedure

1. Define `_TOOL_SPECS: list[tuple[str, str, type[BaseModel]]]` — `(name, description, RequestModel)` for all 20 tools.
2. Implement `build_tool_list() -> list[dict]`:
   - For each `(name, desc, model_cls)`, call `model_cls.model_json_schema()`.
   - Extract `properties` and `required` from the schema.
   - Resolve `$defs` for nested models (inline them into `properties`).
   - Return `{"name": name, "description": desc, "inputSchema": {...}}`.
3. Set `TOOL_LIST = build_tool_list()` at module level.
4. Keep `_MCP_TOOLS` as a backward-compat alias: `_MCP_TOOLS = TOOL_LIST`.

### Method

```python
from __future__ import annotations
from pydantic import BaseModel
from mcp.github.models import (
    SearchRepositoriesRequest,
    GetFileContentsRequest,
    # ... all request models
)

_TOOL_SPECS: list[tuple[str, str, type[BaseModel]]] = [
    ("search_repositories", "Search GitHub repositories...", SearchRepositoriesRequest),
    ("get_file_contents", "Get the contents of a file...", GetFileContentsRequest),
    # ... all 20 tools
]


def _resolve_schema(schema: dict) -> dict:
    """Inline $defs into properties for nested models (e.g. PushFile)."""
    defs = schema.pop("$defs", {})
    props = schema.get("properties", {})
    for key, prop in props.items():
        if "$ref" in prop:
            ref_name = prop["$ref"].split("/")[-1]
            if ref_name in defs:
                props[key] = defs[ref_name]
        elif prop.get("type") == "array" and "$ref" in prop.get("items", {}):
            ref_name = prop["items"]["$ref"].split("/")[-1]
            if ref_name in defs:
                prop["items"] = defs[ref_name]
    return schema


def build_tool_list() -> list[dict]:
    tools = []
    for name, description, model_cls in _TOOL_SPECS:
        raw = model_cls.model_json_schema()
        schema = _resolve_schema(raw)
        tools.append({
            "name": name,
            "description": description,
            "inputSchema": {
                "type": "object",
                "properties": schema.get("properties", {}),
                "required": schema.get("required", []),
            },
        })
    return tools


TOOL_LIST = build_tool_list()
_MCP_TOOLS = TOOL_LIST  # backward compat
```

### Details

- Pydantic v2's `model_json_schema()` returns `{"title": ..., "type": "object", "properties": {...}, "required": [...]}`. We drop `title` and extract the rest.
- For `PushFilesRequest`, the `files` field uses `list[PushFile]`. Pydantic generates a `$defs` section. `_resolve_schema` inlines this so the output matches the hand-written schema.
- The `dry_run` field has `default=False` so it is not in `required` — identical to hand-written behavior.
- Before removing `_MCP_TOOLS`, add a test (Step 6) that asserts `TOOL_LIST == _old_MCP_TOOLS` to validate no regression.
