# Implementation: scripts/db/store_impl.py (doc_upsert RETURNING doc_id)

## Goal

Change `doc_upsert()` to use `RETURNING doc_id` instead of `cur.lastrowid` so both the INSERT and DO UPDATE paths return the correct `doc_id` explicitly.

## Scope

- Target: `scripts/db/store_impl.py`
- `doc_upsert()` SQL: append `RETURNING doc_id`
- Replace `cur.lastrowid` check with `cur.fetchone()` result
- Update error message string
- Public return type `-> int` unchanged

## Assumptions

1. SQLite version >= 3.35.0 (RETURNING support). Confirmed: 3.46.1 in deployment.
2. `SQLiteHelper.execute()` returns `sqlite3.Cursor` — `cur.fetchone()` is available.
3. `RETURNING doc_id` works on both the INSERT path and the `ON CONFLICT DO UPDATE` path, returning the affected row's `doc_id`.

## Implementation

### Target file
`scripts/db/store_impl.py`

### Procedure
1. In `doc_upsert()`, append ` RETURNING doc_id` to the end of the SQL string (before the closing quote).
2. Replace `if cur.lastrowid is None: raise RuntimeError(...)` with `row = cur.fetchone(); if row is None: raise RuntimeError(...)`.
3. Replace `return int(cur.lastrowid)` with `return int(row[0])`.
4. Update the error message to reference "RETURNING doc_id".

### Method

```python
# Before (lines ~72-81)
cur = self._db.execute(
    "INSERT INTO documents (...)"
    " VALUES (?,...)"
    " ON CONFLICT(url) DO UPDATE SET ...",
    (...),
)
if cur.lastrowid is None:
    raise RuntimeError(
        "doc_insert: INSERT ... ON CONFLICT did not produce a lastrowid"
    )
return int(cur.lastrowid)

# After
cur = self._db.execute(
    "INSERT INTO documents (...)"
    " VALUES (?,...)"
    " ON CONFLICT(url) DO UPDATE SET ..."
    " RETURNING doc_id",
    (...),
)
row = cur.fetchone()
if row is None:
    raise RuntimeError("doc_upsert: RETURNING doc_id returned no row")
return int(row[0])
```

### Details
- `RETURNING doc_id` must be the last clause, after `DO UPDATE SET ...`.
- The `lastrowid` pattern for ON CONFLICT DO UPDATE is unreliable in some SQLite versions — RETURNING is explicit and correct.
