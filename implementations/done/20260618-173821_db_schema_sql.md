# Implementation: Add `session_diagnostics` table to session schema

## Goal

Add the `session_diagnostics` table DDL to `db/schema_sql.py` so it is created on next schema initialization.

## Scope

- `scripts/db/schema_sql.py` — add CREATE TABLE to `_SESSION_SCHEMA_TEMPLATE`

**Out of scope:** Data migration from `messages` table's diagnostic rows.

## Assumptions

1. The existing `_SESSION_SCHEMA_TEMPLATE` is executed by `create_schema.py` on startup.
2. `CREATE TABLE IF NOT EXISTS` makes the migration idempotent.

## Implementation

### Target file

`scripts/db/schema_sql.py`

### Procedure

Add the `session_diagnostics` table definition after the `memory_links` table (before the `memories_vec` virtual table, or after all tables).

### Method

Add a `CREATE TABLE IF NOT EXISTS` block to `_SESSION_SCHEMA_TEMPLATE`.

### Details

**Add to `_SESSION_SCHEMA_TEMPLATE`:**
```sql
CREATE TABLE IF NOT EXISTS session_diagnostics (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER REFERENCES sessions(session_id) ON DELETE CASCADE,
    kind       TEXT    NOT NULL,
    content    TEXT    NOT NULL,
    created_at TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);
CREATE INDEX IF NOT EXISTS idx_session_diagnostics_session
    ON session_diagnostics(session_id);
```

Place after the `memory_links` table definition and before `memories_vec`.
