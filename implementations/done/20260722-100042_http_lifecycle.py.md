## Goal
Preserve the original SIGINT signal handler when `signal.signal()` fails due to threading constraints during `McpServerLifecycle.shutdown_all()`.

## Scope
**In**: `scripts/agent/http_lifecycle.py` — fix `shutdown_all()` SIGINT handler management
**Out**: Any changes to signal handling logic outside of shutdown_all(), or changes to `_absorb_sigint_during_shutdown()` behavior

## Assumptions
1. A `ValueError` from `signal.signal()` always indicates a threading constraint (not on main thread)
2. The original handler should be restored even when the guard handler could not be installed
3. Debug-level logging is sufficient — no need for warning/error level

## Implementation

### Target file
`scripts/agent/http_lifecycle.py`

### Procedure
1. Identify the `shutdown_all()` method (around line 329)
2. Find the inner `except ValueError:` block at line 345-347
3. Remove the `old_sigint = None` assignment at line 347
4. Add optional debug log when guard handler cannot be installed
5. Verify `finally` block restores correct handler in both paths

### Method
Remove the erroneous reset of `old_sigint` in the inner except block. Keep the original handler reference intact so the finally block can restore it.

### Details
Current code at lines 335-347:
```python
old_sigint: object | None = None
try:
    old_sigint = signal.getsignal(signal.SIGINT)
except ValueError:
    # Not on the main thread — proceed without the guard rather than fail shutdown.
    old_sigint = None

if old_sigint is not None:
    try:
        signal.signal(signal.SIGINT, self._absorb_sigint_during_shutdown)
    except ValueError:
        # signal.signal() failed — do not install the guard handler.
        old_sigint = None
```

Problem: When `getsignal()` succeeds (returns the original handler) but `signal.signal()` raises `ValueError` (e.g., called from non-main thread), line 347 sets `old_sigint = None`. This causes the finally block at line 377-378 to skip restoring the original handler:
```python
finally:
    if old_sigint is not None:
        signal.signal(signal.SIGINT, old_sigint)
```

Fix:
```python
old_sigint: object | None = None
try:
    old_sigint = signal.getsignal(signal.SIGINT)
except ValueError:
    # Not on the main thread — proceed without the guard rather than fail shutdown.
    old_sigint = None

if old_sigint is not None:
    try:
        signal.signal(signal.SIGINT, self._absorb_sigint_during_shutdown)
    except ValueError:
        # signal.signal() failed — do not install the guard handler.
        # Keep old_sigint intact so the finally block can restore the original handler.
        logger.debug(
            "Lifecycle: could not install SIGINT guard handler: %s",
            sys.exc_info()[1],
        )
```

Key points:
- Remove `old_sigint = None` from the inner except block
- Keep the original handler reference so finally block can restore it
- Add debug log for observability of why guard handler wasn't installed
- The outer `except ValueError` (line 338-340) correctly handles the case where `getsignal()` itself fails — in that case, there's no original handler to restore, so `old_sigint = None` is appropriate
