# Implementation: Fallback Title in _generate_session_title()

## Goal

When LLM-based session title generation fails, set a deterministic fallback title derived from the first user input. This ensures every session always has a non-empty, meaningful title.

## Scope

- `scripts/agent/commands/cmd_session.py` — modify `_generate_session_title()` to set fallback title in the `except` block

## Assumptions

1. `first_input` is always available and non-empty when `_generate_session_title()` is called.
2. Truncating `first_input` to `SESSION_TITLE_MAX_CHARS` (32) produces a readable fallback.
3. Existing constants `SESSION_TITLE_MAX_CHARS = 32` and `SESSION_TITLE_TRUNCATE_AT = 29` are already defined in the file.

## Implementation

### Target file

`scripts/agent/commands/cmd_session.py`

### Procedure

1. Read `_generate_session_title()` method (lines 28-41).
2. In the `except SessionTitleGenerationError` block, add fallback title logic after the existing `logger.warning` call.
3. Truncate `first_input` to `SESSION_TITLE_TRUNCATE_AT` chars; append `...` if truncated.
4. Call `self._ctx.session.set_title(fallback_title)` to persist the fallback.

### Method

Inline edit in the existing `except` block. No new imports or dependencies needed.

### Details

Current code (lines 39-40):
```python
        except SessionTitleGenerationError as e:
            logger.warning("Session title generation failed: %s", e)
```

Replace with:
```python
        except SessionTitleGenerationError as e:
            logger.warning("Session title generation failed: %s", e)
            truncated = first_input[:SESSION_TITLE_TRUNCATE_AT]
            if len(first_input) > SESSION_TITLE_MAX_CHARS:
                fallback_title = truncated + "..."
            else:
                fallback_title = truncated
            self._ctx.session.set_title(fallback_title)
```
