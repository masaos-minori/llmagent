# 00 Document Guide — Agent Documentation Restructuring

## Goal
Agentドキュメント群の全体ナビゲーションガイドを作成する。読者が目的のトピックへ最短で到達できるようにする。

## Scope
- 各章の概要と対象読者の明示
- 章間クロスリファレンスの整理
- 既存ソースドキュメント(05_agent*.md / 05_ref-agent-*.md)からの情報統合

## Assumptions
- 最終ドキュメントは11章構成(01〜11)で確定している
- 読者はPythonおよびLLMエージェントの基礎知識を持つ開発者

## Implementation

### Target file
`docs/05_agent/00_document-guide.md`

### Procedure
- 各出力ファイル(01〜12)のGoal/Scopeを参照して章概要を1〜2行で要約
- 05_agent.md §1-3 から全体アーキテクチャの俯瞰図を引用
- 05_agent-impl-class.md §1.7 のクラス一覧を章対応表として活用

### Method
- 冒頭に「本ドキュメントの読み方」セクションを設ける
- 章番号・ファイル名・概要・主な対象読者の4列テーブルを作成
- 典型的な3ユースケース(初回セットアップ / デバッグ / 拡張)ごとの推奨読順を箇条書きで示す

### Details
- 章一覧テーブル: 章番号 / タイトル / 概要(1行) / 読むべきロール
- ユースケース別読順: 初回導入→01→08→10、デバッグ→03→05→10、拡張→02→11→12
- 用語集へのポインタ(12_reference-api参照)
- ドキュメント更新ポリシー(ソースファイルと同期する旨)

## Validation plan
- 全12章ファイルへのリンクが正しく解決できること
- テーブルの章番号が抜けなく01〜12を網羅していること
- 3ユースケースの読順に矛盾がないこと
- 40行以内に収まっていること
