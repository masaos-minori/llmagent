## Goal

Update the MDQ server catalog doc so it advertises 7 tools (not 9) and drops
the "2ツールが `admin`" mention, since `fts_consistency_check`/`fts_rebuild`
no longer exist anywhere in the schema/registry/config after this plan.

## Scope

**In:**
- The line: `**ツール（9個）:** ...` → update count to 7 and remove
  `fts_consistency_check`, `fts_rebuild` from the tool-name list.
- The line: `**ツールステータス:** 7ツールが \`production\`...2ツール
  （\`fts_consistency_check\`, \`fts_rebuild\`）が \`admin\`。...` → simplify
  to state all 7 tools are `production`, removing the admin-tier mention.

**Out:**
- The **本番稼働範囲** sentence about `use_embedding=true` hybrid search
  being stub — unrelated, no change.
- The 設定フィールド (config fields) line — already covered by the
  companion `audit_log_path`-focused doc
  (`implementations/20260716-130701_04_mcp_04_04_mdq.md.md`); do not
  duplicate that edit here.
- Any other section of this doc (health response, DB path, log path, audit,
  usage scenario, パス制御 table below).

   same file's change (same target file, no separate doc needed).
