# Implementation: Enforce plugin tool contract validation in plugin_registry.py

Steps covered: Plan 20260626-091912 — Steps 1-1, 1-2, 1-3

---

## Goal

Make `plugin_registry.py:register_tool()` (line ~130) raise `ValueError` instead of emitting a warning when a plugin tool contract is violated. Enable `strict_mode=True` in `_validate_tool_conflicts()` to fail-fast on conflict detection.

---

## Scope

- **In scope**: `scripts/agent/plugin_registry.py` — `register_tool()`, `_validate_tool_conflicts()`
- **Out of scope**: tests (steps 2-1, 2-2); docs (step 3-1)

---

## Assumptions

- `register_tool()` currently calls `logger.warning(...)` for contract violations (missing required fields, invalid inputSchema, etc.) and continues registration.
- `_validate_tool_conflicts()` has a `strict_mode` parameter or can be made to raise on conflicts.
- A "tool contract" violation means: missing `name`, missing `description`, `inputSchema` not a valid JSON Schema object, or duplicate tool name.

---

## Implementation

### Target file
`scripts/agent/plugin_registry.py`

### Procedure
1. Read `scripts/agent/plugin_registry.py` lines 120-150 to find `register_tool()` and the warning call.
2. Step 1-1: Change warning → raise:
   ```python
   # Before:
   logger.warning("tool contract violation: %s — %s", tool_name, reason)
   # After:
   raise ValueError(f"plugin tool contract violation for {tool_name!r}: {reason}")
   ```
3. Step 1-2: In `_validate_tool_conflicts()`, add strict_mode guard:
   ```python
   def _validate_tool_conflicts(self, tool: ToolDef, *, strict_mode: bool = True) -> None:
       if conflict := self._find_conflict(tool):
           if strict_mode:
               raise ValueError(f"tool conflict: {tool.name!r} conflicts with {conflict!r}")
           logger.warning("tool conflict (ignored): %s vs %s", tool.name, conflict)
   ```
   Call `_validate_tool_conflicts(tool, strict_mode=True)` in `register_tool()`.
4. Step 1-3: Audit inputSchema validation — if `inputSchema` is not a `dict` with `"type": "object"`, raise:
   ```python
   if not isinstance(tool.inputSchema, dict) or tool.inputSchema.get("type") != "object":
       raise ValueError(f"tool {tool.name!r}: inputSchema must be a JSON Schema object")
   ```

### Method
Behavioral change: registration now raises on violations rather than warning. Existing tests that expect warnings may need to be updated (covered in step 2-x).
