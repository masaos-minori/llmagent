# Implementation: scripts/rag/ingestion/ingester.py

## Goal

Extract `_delete_existing_document()` logic into a module-level function
`delete_document_chain(db, doc_id)` so the delete order is explicit and reusable.

## Scope

- `scripts/rag/ingestion/ingester.py` — add `delete_document_chain()`, thin-wrap `_delete_existing_document()`

## Assumptions

1. `delete_document_chain` is a module-level function (not a method), placed just before `RagIngester`.
2. `_delete_existing_document()` becomes a one-liner that delegates to `delete_document_chain()`.
3. Delete order: chunks_vec → chunks → documents.
4. No external callers import `delete_document_chain` from this module in this iteration.

## Implementation

### Target file

`scripts/rag/ingestion/ingester.py`

### Procedure

1. Add `delete_document_chain(db, doc_id)` function before the `RagIngester` class.
2. Rewrite `_delete_existing_document()` to call `delete_document_chain(db, doc_id)`.

### Details

```python
def delete_document_chain(db: SQLiteHelper, doc_id: int) -> None:
    """Delete chunks_vec → chunks → documents for doc_id.

    chunks_vec has no FK to chunks, so it must be deleted first.
    chunks deletion fires FTS5 triggers automatically (via ON DELETE trigger on chunks).
    documents deletion cascades to any remaining chunks rows.
    """
    db.execute(
        "DELETE FROM chunks_vec"
        " WHERE chunk_id IN (SELECT chunk_id FROM chunks WHERE doc_id = ?)",
        (doc_id,),
    )
    db.execute("DELETE FROM chunks WHERE doc_id = ?", (doc_id,))
    db.execute("DELETE FROM documents WHERE doc_id = ?", (doc_id,))
```

Modified `_delete_existing_document`:
```python
def _delete_existing_document(self, db: SQLiteHelper, doc_id: int) -> None:
    """Delete a document and its chunks; chunks_vec removed first because it has no FK constraint to chunks."""
    delete_document_chain(db, doc_id)
```
