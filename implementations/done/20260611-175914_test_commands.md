# Goal

Add error-path tests to existing command test files and create two new test files
(`test_agent_cmd_mcp.py`, `test_agent_cmd_tooling.py`) to cover the new DTO/exception behavior
introduced in Steps 1-10.

# Scope

Existing files to update:
- `tests/test_agent_cmd_config.py` — DB config load error, reload error
- `tests/test_agent_cmd_context.py` — unknown preset → `UnknownPresetError` (if applicable); None content handling
- `tests/test_agent_cmd_db.py` — invalid flag → graceful handling; OutputPort capture
- `tests/test_agent_cmd_memory.py` — unknown subcommand → `UnknownSubcommandError`
- `tests/test_agent_cmd_notes.py` — unknown subcommand → `UnknownSubcommandError`
- `tests/test_agent_cmd_ingest.py` — print → OutputPort capture check

New files to create:
- `tests/test_agent_cmd_tooling.py` — `ToolResultView` DTO, invalid JSON fallback
- `tests/test_agent_cmd_mcp.py` — `McpInstallRequest` validation

# Assumptions

1. Existing tests use `capsys` for stdout capture or mock `OutputPort`; after OutputPort migration, tests should capture via `capsys` (which still works because `CliOutputPort` calls `print()`).
2. `UnknownSubcommandError` propagates out of `_cmd_memory` and `_cmd_notes` — tests verify `pytest.raises(UnknownSubcommandError)`.
3. `ToolResultView` tests use a mock store returning a known dict; the helper `_to_tool_result_view` is tested indirectly via `_tool_list` / `_tool_show`.
4. `McpInstallRequest` validation test mocks `validate_server_name` to verify it's called.
5. Tests use the project's existing fixture pattern (e.g. `make_ctx` or similar factory).

# Implementation

## Target file

Multiple test files (see Scope above)

## Procedure

1. Add `test_cmd_config_db_config_error` — mock `_build_db_cfg` to raise `ValueError`; verify error message in output.
2. Add `test_cmd_config_reload_oserror` — mock `ConfigLoader.load` to raise `OSError`; verify "Reload failed (I/O error)" in output.
3. Add `test_cmd_context_none_content` — mock history with a message where `content=None`; verify no crash.
4. Add `test_cmd_memory_unknown_sub_raises` — call `_cmd_memory("badcmd")` → expect `UnknownSubcommandError`.
5. Add `test_cmd_notes_unknown_sub_raises` — call `_cmd_note("badcmd")` → expect `UnknownSubcommandError`.
6. Create `test_agent_cmd_tooling.py`:
   - `test_tool_list_with_entries` — mock `list_recent` returning a dict; verify `ToolResultView` attribute use.
   - `test_tool_show_invalid_json_args` — mock `get` returning dict with unparseable `args_masked`; verify output shows empty args, no exception.
   - `test_tool_show_not_found` — mock `get` returning `None`; verify "not found" message.
7. Create `test_agent_cmd_mcp.py`:
   - `test_mcp_install_invalid_name` — mock `validate_server_name` to raise `ValueError`; verify error output.
   - `test_mcp_install_builds_request` — mock service to capture `McpInstallRequest.server_name`.

## Method

Use `pytest.raises` for exception-path tests. Use `capsys.readouterr()` for output capture.
Mock `CliOutputPort` methods or capture `capsys` — both work since `CliOutputPort.write*` calls `print()`.

## Details

Example for `test_agent_cmd_memory.py`:

```python
import pytest
from agent.commands.exceptions import UnknownSubcommandError

def test_memory_unknown_subcommand_raises(registry_with_memory_enabled):
    with pytest.raises(UnknownSubcommandError) as exc_info:
        registry_with_memory_enabled._cmd_memory("badcmd")
    assert exc_info.value.sub == "badcmd"
    assert "list" in exc_info.value.valid
```

Example for `test_agent_cmd_tooling.py`:

```python
def test_tool_show_invalid_json_does_not_raise(registry, capsys):
    registry._ctx.tool_result_store.get = lambda _: {
        "id": 1, "tool_name": "test_tool", "args_masked": "NOT_JSON",
        "full_text": "content", "summary": None, "is_error": False,
    }
    registry._tool_show("1")
    captured = capsys.readouterr()
    assert "Args: {}" in captured.out
```
