# Implementation: scripts/db/store_impl.py (delete_memories_before begin_immediate)

## Goal

Wrap the multi-table DELETE block in `delete_memories_before()` with `self._db.begin_immediate()` context manager and remove the explicit `self._db.commit()` call, making atomicity explicit in code.

## Scope

- Target: `scripts/db/store_impl.py`
- `delete_memories_before()`: wrap DELETE block with `with self._db.begin_immediate():`
- Remove `self._db.commit()` inside the method
- No other methods changed

## Assumptions

1. `SQLiteHelper.begin_immediate()` is a context manager that issues `BEGIN IMMEDIATE` on enter and `COMMIT` on clean exit (or `ROLLBACK` on exception). Already used in `agent/memory/store.py`.
2. `self._db.fetchall(SELECT ...)` does not start a Python sqlite3 implicit transaction â€” safe to call before `begin_immediate()`.
3. `self._db.conn` is non-None when `delete_memories_before()` is called (opened with `write_mode=True` in `maintenance.py`).

## Implementation

### Target file
`scripts/db/store_impl.py`

### Procedure
1. Wrap lines from `cur = self._db.execute(f"DELETE FROM memories ...")` through `self._db.commit()` with `with self._db.begin_immediate():`.
2. Remove `self._db.commit()` line.
3. Keep `deleted = cur.rowcount` inside the `with` block.
4. Move `return MemoryDeleteResult(deleted=deleted)` outside the `with` block.

### Method

```python
def delete_memories_before(self, older_than_days: int) -> MemoryDeleteResult:
    rows = self._db.fetchall(
        "SELECT memory_id FROM memories WHERE created_at < datetime('now', ?)",
        (f"-{older_than_days} days",),
    )
    if not rows:
        return MemoryDeleteResult(deleted=0)

    mids = [row[0] for row in rows]
    placeholders = ",".join("?" * len(mids))
    with self._db.begin_immediate():
        cur = self._db.execute(
            f"DELETE FROM memories WHERE memory_id IN ({placeholders})",
            tuple(mids),
        )
        deleted = cur.rowcount
        for mid in mids:
            self._db.execute("DELETE FROM memories_fts WHERE memory_id=?", (mid,))
        for mid in mids:
            self._db.execute("DELETE FROM memories_vec WHERE memory_id=?", (mid,))
    return MemoryDeleteResult(deleted=deleted)
```

### Details
- Remove the `self._db.commit()` line that previously followed the loop â€” `begin_immediate()` commits on exit.
- The `# nosec B608` comment on the dynamic SQL should be preserved.
- `deleted = cur.rowcount` must stay inside the `with` block (after `execute`), then referenced outside for the return value.
