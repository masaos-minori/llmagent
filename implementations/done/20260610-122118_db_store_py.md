# Implementation: db/store.py — remove fallback, true UPSERT, atomic delete

## Goal

1. `get_embedding_dims()`: remove `except Exception: return 384` fallback; propagate exception.
2. `SQLiteRagStore.doc_upsert()`: replace SELECT→UPDATE/INSERT two-step with a single
   `INSERT OR REPLACE INTO documents` statement.
3. `SQLiteMemoryDeleteStore.delete_memories_before()`: replace partial-success pattern with
   full-transaction atomicity; any failure rolls back the entire deletion.

## Scope

- `scripts/db/store.py` — primary target
- No test files changed in this step (existing tests should still pass after the refactor)

## Assumptions

1. `doc_upsert()` currently checks if the document exists and does UPDATE or INSERT.
   `INSERT OR REPLACE` is semantically equivalent when `url` has a UNIQUE constraint,
   which it does (`UNIQUE` on `documents.url`). The `doc_id` changes on replace, which
   is acceptable since chunks are re-inserted anyway in the ingestion pipeline.
2. `delete_memories_before()` currently catches per-table exceptions and sets
   `vec_skipped=True`. The new design: wrap all three DELETE statements in a single
   transaction context; if any raises, roll back and re-raise.
3. `get_embedding_dims()` callers (`create_schema.py`, `validate_embedding_blob()`) will
   now receive the exception on config failure. This is the intended fail-fast behaviour.

## Implementation

### Target file

`scripts/db/store.py`

### Procedure

1. In `get_embedding_dims()`: remove the `try/except` wrapper; call `build_db_config()`
   directly and return `.embedding_dims`.
2. In `SQLiteRagStore.doc_upsert()`: read the current implementation, then replace the
   SELECT → conditional UPDATE/INSERT logic with:
   ```python
   cur = db.execute(
       "INSERT OR REPLACE INTO documents"
       " (url, title, lang, etag, last_modified)"
       " VALUES (?, ?, ?, ?, ?)",
       (url, title, lang, etag, last_modified),
   )
   assert cur.lastrowid is not None
   return cur.lastrowid
   ```
3. In `SQLiteMemoryDeleteStore.delete_memories_before()`: wrap the three delete operations
   in `db.begin_immediate()` and remove the per-table try/except; let exceptions propagate.
   Remove `vec_skipped` / `vec_error` fields from `MemoryDeleteResult` or keep them as
   False/None (verify what callers use).

### Method

Direct textual edit.

### Details

`get_embedding_dims()` after:
```python
def get_embedding_dims() -> int:
    """Return embedding dimensions from DbConfig; raises on config error."""
    return build_db_config().embedding_dims
```

`doc_upsert()` after (simplified):
```python
def doc_upsert(self, url, title, lang, etag, last_modified) -> int:
    with SQLiteHelper("rag").open(write_mode=True) as db:
        cur = db.execute(
            "INSERT OR REPLACE INTO documents"
            " (url, title, lang, etag, last_modified)"
            " VALUES (?, ?, ?, ?, ?)",
            (url, title, lang, etag, last_modified),
        )
        assert cur.lastrowid is not None
        doc_id = cur.lastrowid
        db.commit()
    return doc_id
```

`delete_memories_before()` — check current implementation first, then remove
`vec_skipped` partial-success path.
