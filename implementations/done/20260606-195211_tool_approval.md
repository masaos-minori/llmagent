# Implementation: Dangerous Operation Approval Model Enhancement

## Goal

Extend `scripts/agent/tool_approval.py` with content-aware risk escalation based on argument values and resource scope, and strengthen structured audit logging for approval decisions.

## Scope

**In:**
- Add `_escalate_by_args(tool_name, args, cfg) -> str | None` that detects high-risk arg patterns (protected paths, dangerous flags like `recursive=True`, `force=True`)
- Add `ApprovalDecision` TypedDict to unify the return contract
- Strengthen `tool_audit.py` to log structured approval decisions with `decision`, `risk_level`, `reason`, `tool`, `args_summary`

**Out:**
- Changing the interactive prompt format
- Adding new config fields (uses existing `ApprovalConfig` fields)
- Changing how `run_approval_checks()` is called

## Assumptions

- `scripts/agent/tool_approval.py`: `run_approval_checks()` returns `(approved_calls, denied_ids)`; uses `_get_risk_level()` + `_needs_approval()`; performs dry-run via `_build_preview_with_dry_run()`
- `scripts/agent/tool_audit.py`: `log_tool_call()` logs tool execution; approval decision logging may be absent
- `scripts/agent/config.py`: `ApprovalConfig.approval_protected_paths`, `approval_resource_keys`, `approval_dry_run_tools`, `approval_risk_rules` all exist

## Implementation

### 1. `scripts/agent/tool_approval.py` — add `_escalate_by_args()`

```python
def _escalate_by_args(
    tool_name: str,
    args: dict[str, Any],
    cfg: ApprovalConfig,
) -> tuple[str, str] | None:
    """Return (escalated_risk_level, reason) if arg content warrants escalation.

    Returns None when no escalation is needed.
    Checks:
    - Recursive deletion (delete_directory with recursive=True or non-empty dir)
    - Force/overwrite flags
    - Dangerous shell command prefixes (already partially covered by approval_shell_safe_prefixes)
    """
    # Check recursive delete escalation
    if tool_name == "delete_directory" and args.get("recursive"):
        return "high", "recursive directory deletion requested"

    # Check for dangerous explicit flags
    for flag in ("force", "overwrite", "clobber"):
        if args.get(flag) is True:
            return "high", f"dangerous flag {flag}=True in args"

    # Check path escalation (extends existing approval_protected_paths logic)
    resource_keys = cfg.approval_resource_keys
    path_keys = resource_keys.get("path_keys", [])
    for key in path_keys:
        path = str(args.get(key, ""))
        if path and any(path.startswith(p) for p in cfg.approval_protected_paths):
            return "high", f"path {path!r} is in a protected directory"

    return None
```

Integrate into the existing `_get_risk_level()` or call before the final approval decision.

### 2. `ApprovalDecision` TypedDict

```python
class ApprovalDecision(TypedDict, total=False):
    """Structured result of a single tool approval evaluation."""
    tool_name: str
    risk_level: str          # "none" | "medium" | "high"
    decision: str            # "approved" | "denied" | "dry_run" | "preview_only"
    escalation_reason: str   # why the risk was escalated (or "" if no escalation)
    preview: str             # dry_run result text (or "")
```

### 3. `scripts/agent/tool_audit.py` — log approval decisions

Add a `log_approval_decision(ctx, decision: ApprovalDecision) -> None` function:

```python
def log_approval_decision(ctx: AgentContext, decision: ApprovalDecision) -> None:
    """Write a structured approval decision to the audit log."""
    if ctx.services.audit_logger is None:
        return
    ctx.services.audit_logger.info(
        orjson.dumps({
            "event": "approval_decision",
            "tool": decision.get("tool_name"),
            "risk_level": decision.get("risk_level"),
            "decision": decision.get("decision"),
            "escalation_reason": decision.get("escalation_reason", ""),
        }).decode()
    )
```

Call `log_approval_decision()` for every tool call that goes through the approval flow, after the decision is made.

def test_recursive_delete_escalates_to_high():
