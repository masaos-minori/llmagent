### Shell MCP Sandbox Policy

The shell MCP server supports pluggable sandbox backends:

| Backend | Use Case | Production? |
|---------|----------|-------------|
| `firejail` | Production — process isolation, restricted filesystem | Yes (required) |
| `none` | Development only — no isolation | No (WARNING) |
| `docker` | Alternative to firejail (not default) | Optional |

**Production rule**: `sandbox_backend` MUST be `"firejail"` (or equivalent). Using `"none"`
in production is a security risk and triggers a startup WARNING.

**Why firejail?**
- Restricts file system access to declared paths
- Limits network access
- Limits process capabilities
- Does not require root (unlike Docker for some operations)
```

After installation, verify: `firejail --version`
```
