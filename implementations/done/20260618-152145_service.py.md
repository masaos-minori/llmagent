# Goal

Add a `sandbox_backend` read-only property to `ShellService` in `scripts/mcp/shell/service.py` so that `server.py` and tests can access the resolved sandbox backend without using the private `_sandbox_backend` attribute.

# Scope

- **File:** `scripts/mcp/shell/service.py`
- **Change:** Add `@property sandbox_backend(self) -> str` after `__init__`
- **Out of scope:** `_sandbox_backend` initialization logic, `init_sandbox()` helper

# Assumptions

1. `self._sandbox_backend` is a `str` set once in `__init__` and never mutated.
2. The property name `sandbox_backend` matches the config key and health response field name.
3. No existing code accesses `service._sandbox_backend` from outside the class except tests.

# Implementation

## Target file

`scripts/mcp/shell/service.py`

## Procedure

Add after `__init__` (after line ~88):
```python
@property
def sandbox_backend(self) -> str:
    """Return the resolved sandbox backend name ('firejail' or 'none')."""
    return self._sandbox_backend
```

## Details

- No import changes needed.
- Existing internal `self._sandbox_backend` references in `_apply_sandbox` remain as-is.
