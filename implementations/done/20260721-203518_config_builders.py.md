# Implementation Procedure: Unify startup config validation with reload validation and fix missing defaults

## Goal

Make startup config loading use the same strict typed validators as runtime reload, eliminating silent coercion bugs and ensuring deterministic configuration behavior.

## Scope

- `scripts/agent/config_builders.py` — replace all implicit `bool()`, `list()`, `dict()` conversions with typed validators
- `scripts/agent/services/typed_validators.py` — may need minor additions for startup-specific needs
- `scripts/agent/config_dataclasses.py` — verify defaults for `tool_concurrency_limits` and `tool_safety_tiers`
- Tests: `tests/test_agent_config*.py` — add/update tests for strict startup validation

## Assumptions

- `ConfigReloadValidationError` can be reused for startup errors (or a new `ConfigLoadError` should be introduced)
- All typed validators already handle the edge cases correctly (bool excluded from int, etc.)
- Existing test fixtures will need updates to reflect stricter validation

## Implementation

### Target file

`scripts/agent/config_builders.py`

### Procedure

1. Import typed validators into `config_builders.py`:
   ```python
   from agent.services.typed_validators import (
       _get_bool, _get_int, _get_float, _get_str, _get_list, _get_dict
   )
   ```

2. Replace all `bool(cfg.get("key", default))` patterns with `_get_bool(cfg, "key") or default`

3. Replace all `list(cfg.get("key", default))` patterns with `_get_list(cfg, "key") or default`

4. Replace all `dict(cfg.get("key", default))` patterns with `_get_dict(cfg, "key") or default`

5. Add business rule validations after typed validator checks pass:
   - `memory_retention_days >= 1`
   - `memory_embed_dim >= 1`
   - `memory_embed_timeout_sec > 0`
   - `memory_max_inject_semantic >= 0`
   - `memory_max_inject_episodic >= 0`
   - `tool_safety_tiers` values must be one of: `READ_ONLY`, `WRITE_SAFE`, `WRITE_DANGEROUS`, `ADMIN`

6. Update tests covering:
   - Invalid bool type rejection
   - Invalid list type rejection
   - Invalid dict type rejection
   - Bool-as-int rejection
   - Missing concurrency limits / safety tiers
   - Invalid safety tier values
   - Invalid memory numeric values
   - Minimal valid config acceptance

### Method

Use targeted edits to replace each implicit conversion pattern with the corresponding typed validator. Use `_get_bool`, `_get_int`, `_get_float`, `_get_str`, `_get_list`, `_get_dict` from `typed_validators.py`. For dict/list fields with non-trivial defaults, use `_get_dict(cfg, "key") or default` pattern.

### Details

- Recommend reusing `ConfigReloadValidationError` for both startup and reload contexts (Option A) — consistent error type simplifies caller logic
- Typed validators (`_get_bool`, `_get_int`, `_get_list`, `_get_dict`, etc.) already exist in `typed_validators.py` but are never imported or used in `config_builders.py`
- After migration, grep for remaining `bool(`, `list(`, `dict(` patterns in `config_builders.py` to ensure complete coverage
- Blast radius: High — many existing tests will fail due to stricter validation
