# Goal

Extend `_cmd_mcp_status` in `scripts/agent/commands/cmd_mcp.py` to show a server availability summary (N configured, N ok, N unreachable) in the `/mcp status` output.

# Scope

- **File:** `scripts/agent/commands/cmd_mcp.py`
- **Function:** `_cmd_mcp_status`, `_format_mcp_table` (or add a new summary line)
- **Out of scope:** `McpStatusService.probe_all()` behavior, `McpProbeResult` model

# Assumptions

1. `McpProbeResult.availability` uses `McpAvailability` enum with values: `OK`, `FAIL`, `HTTP_ERROR`, `NO_URL`, `STOPPED`, `NOT_STARTED`, `DEAD`.
2. "Unreachable" = `FAIL | HTTP_ERROR | DEAD` (server was contacted or expected to be running but failed).
3. The summary line is added after the table in `_cmd_mcp_status`, not inside `_format_mcp_table`.
4. This is an additive output change â€” no existing lines removed.

# Implementation

## Target file

`scripts/agent/commands/cmd_mcp.py`

## Procedure

1. Import `McpAvailability` in the import block.
2. After `self._out.write(_format_mcp_table(rows))` in `_cmd_mcp_status`, add a summary line.

## Method

Add to imports:
```python
from agent.services.enums import McpAvailability
```

Update `_cmd_mcp_status`:
```python
async def _cmd_mcp_status(self) -> None:
    """Print MCP server status table."""
    ctx = self._ctx
    svc = McpStatusService(ctx)
    self._out.write(
        f"  Tools       {len(ctx.cfg.tool.tool_definitions)} (from config/agent.toml)"
    )
    self._out.write("")
    rows = await svc.probe_all()
    self._out.write(_format_mcp_table(rows))
    _UNREACHABLE = {McpAvailability.FAIL, McpAvailability.HTTP_ERROR, McpAvailability.DEAD}
    ok_count = sum(1 for r in rows if r.availability == McpAvailability.OK)
    unreachable_count = sum(1 for r in rows if r.availability in _UNREACHABLE)
    self._out.write(
        f"\n  Servers     {len(rows)} configured"
        f" ({ok_count} ok, {unreachable_count} unreachable)"
    )
```

## Details

- `STOPPED` and `NOT_STARTED` are not counted as unreachable (ondemand/not-yet-started servers are expected to be inactive).
- `NO_URL` is also not unreachable (misconfigured, not failed).
- The summary is a single added line at the end of the status output.
