# Implementation: Add _MCP_TOOLS pattern to check_no_compat.py

## Goal

Register `_MCP_TOOLS` as a banned compatibility pattern in `COMPAT_PATTERNS` so that
the CI check rejects any future re-introduction of the old export name.

## Scope

- Target file: scripts/checks/check_no_compat.py
- Only the `COMPAT_PATTERNS` dict is modified; no logic changes.

## Assumptions

- Steps 1-3 (tools.py, server.py, test files) are complete before this step;
  adding the pattern before removing all references would cause the CI check to fail immediately.
- The existing COMPAT_PATTERNS values are plain regex strings (confirmed by reading lines 24-40).
- The check scans all .py and .md files under ROOT_DIR; no exclusion list is needed.

## Implementation

### Target file: scripts/checks/check_no_compat.py

#### Procedure

1. Read the COMPAT_PATTERNS dict (lines 24 onward) to find the correct insertion point.
2. Add one entry at the end of COMPAT_PATTERNS (before the closing `}`):
   ```python
   "_MCP_TOOLS export": r"\b_MCP_TOOLS\b",
   ```
3. Verify the regex does not produce false positives by checking the pattern against
   the current codebase (should match zero lines after Steps 1-3 are done).

#### Method

Targeted insertion into the COMPAT_PATTERNS dict literal.

#### Details

Pattern rationale:
- `\b_MCP_TOOLS\b` matches the bare identifier in any context (definition, import, usage).
- The `\b` word boundary prevents partial matches (e.g., `_MCP_TOOLS_EXTRA`).
- The key string `"_MCP_TOOLS export"` follows the naming convention of other entries
  (descriptive phrase, not just the symbol name).

Insertion location: after the last existing entry in COMPAT_PATTERNS and before the
closing `}`. Match the surrounding indentation (4 spaces).
