# Implementation: route_resolver.py (Phase 6 — strict mode)

## Goal

Add `strict_mode: bool` to `ToolRouteResolver` that raises `ValueError` when a tool
is not found in the config map, preventing silent static-fallback routing for tools
that should be explicitly configured. This enforces config-first routing in production
while allowing existing fallback in test environments.

## Scope

**In**: `scripts/shared/route_resolver.py` only.
**Out**: No changes to `ToolExecutor` or callers. `warn_on_fallback` is retained for
backward compatibility; `strict_mode` is the stronger enforcement option.

## Assumptions

1. `warn_on_fallback: bool = False` already exists; `strict_mode` is additive.
2. `strict_mode=True` should raise `ValueError` when `_fallback_route()` would be called,
   before the static lookup runs.
3. Default `strict_mode=False` preserves current behavior entirely.
4. The `ToolExecutor` constructor passes `warn_on_fallback` from config; `strict_mode`
   will need to be wired through `AgentConfig` in a subsequent step (out of scope here).

## Implementation

### Target file

`scripts/shared/route_resolver.py`

### Procedure

1. Add `strict_mode: bool = False` parameter to `ToolRouteResolver.__init__`.
2. In `resolve()`, when the config map misses and `strict_mode` is True, raise before
   calling `_fallback_route()`.

### Method

Two targeted edits to `route_resolver.py`.

### Details

```python
def __init__(
    self,
    server_configs: dict[str, McpServerConfig],
    *,
    warn_on_fallback: bool = False,
    strict_mode: bool = False,
) -> None:
    self._config_map: dict[str, str] = {}
    for key, cfg in server_configs.items():
        for tool_name in cfg.tool_names:
            self._config_map[tool_name] = key
    self._warn_on_fallback = warn_on_fallback
    self._strict_mode = strict_mode

def resolve(self, tool_name: str) -> str:
    if (key := self._config_map.get(tool_name)) is not None:
        return key
    if self._strict_mode:
        raise ValueError(
            f"ToolRouteResolver: tool {tool_name!r} not in config map "
            f"and strict_mode=True; add it to tool_names in mcp_servers config"
        )
    if self._warn_on_fallback:
        logger.warning(
            "ToolRouteResolver: tool %r not in config map; using static fallback. "
            "Add tool_names to mcp_servers config to suppress this warning.",
            tool_name,
        )
    return self._fallback_route(tool_name)
```
