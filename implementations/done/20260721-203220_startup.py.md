# Implementation Procedure: Make discovery failure fatal under PRODUCTION security profile

## Goal

Prevent the agent from starting silently crippled when MCP tool discovery fails, by making the failure fatal under `PRODUCTION` security profile.

## Scope

- `scripts/agent/startup.py` — change discovery exception handler to use `add_fatal` under PRODUCTION
- Tests: `tests/test_startup.py` — add test for discovery failure under both profiles

## Assumptions

- `SecurityProfile` enum has `PRODUCTION` and `LOCAL` values
- `pipeline.add_fatal()` and `pipeline.add_warning()` methods exist and work correctly
- The existing `production_mode` variable computed at line 192 can be reused directly

## Implementation

### Target file

`scripts/agent/startup.py`

### Procedure

1. Locate the discovery exception handler in `_check_services()` at approximately lines 245-248
2. Modify the handler to branch based on `production_mode`:
   ```python
   except Exception as exc:  # noqa: BLE001
       if production_mode:
           pipeline.add_fatal(
               "mcp_tool_discovery", f"MCP tool discovery failed: {exc}"
           )
       else:
           pipeline.add_warning(
               "mcp_tool_discovery", f"MCP tool discovery skipped: {exc}"
           )
   ```

3. Add tests in `tests/test_startup.py`:
   - PRODUCTION profile: Mock `McpToolDiscoveryService.discover_all()` to raise an exception → assert `pipeline.has_fatal` is True → assert startup raises RuntimeError
   - LOCAL profile: Same mock setup → assert `pipeline.has_fatal` is False → assert startup proceeds normally → assert warning message contains discovery failure details

### Method

Use targeted edit to add one conditional branch to the existing exception handler. The `production_mode` variable is already computed at line 192: `production_mode = ctx.cfg.mcp.security_profile == SecurityProfile.PRODUCTION`. This mirrors the existing `SecurityProfile.PRODUCTION` fatal-on-subprocess-failure pattern in `_start_servers()`.

### Details

- This change prevents the agent from starting silently crippled when MCP tool discovery fails
- Under PRODUCTION: discovery failure is fatal, agent refuses to start
- Under LOCAL: discovery failure is a warning, agent continues (mirrors existing behavior with clearer semantics — SKIPPED → WARNING)
- Blast radius: Low — only affects startup behavior when discovery fails; normal operation unchanged
