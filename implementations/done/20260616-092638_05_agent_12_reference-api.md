# 12 Reference API — Agent Documentation Restructuring

## Goal
全クラスのAPIサマリーテーブルと各章へのクロスリファレンスを1章にまとめる。

## Scope
- 全主要クラスの名前・モジュール・主要メソッドの早見表
- 各クラスの詳細記述がある章への参照
- 用語集

## Assumptions
- 05_agent-impl-class.md §1.7 のクラスサマリーテーブルが本章の骨格
- 各参照先章(01〜11)が確定済みであること

## Implementation

### Target file
`docs/05_agent/12_reference-api.md`

### Procedure
- 05_agent-impl-class.md §1.7 から全クラス名・モジュールパス・主要メソッドを抽出
- 各クラスが詳細記述されている章番号を対応付け(02章→AgentREPL/Orchestrator等)
- 全ソースドキュメント(05_ref-agent-*.md)から用語定義を収集して用語集を構築

### Method
- H2: クラスAPIサマリー / 章クロスリファレンス / 用語集
- クラスサマリーは「クラス名 | モジュール | 主要メソッド(3件まで) | 詳細章」の4列テーブル
- 用語集は「用語 — 定義(1行)」のアルファベット順箇条書き

### Details
- テーブル対象クラス: AgentREPL, Orchestrator, ToolExecutor, LLMClient, CLIView, AgentContext, ConversationState, TurnState, RuntimeStats, AppServices, HistoryManager, AgentSession, CommandRegistry, PluginRegistry
- 章クロスリファレンス: 「トピック — 参照章」形式で15件程度
- 用語集収録例: turn, session, tool_use, SSE, compress, plugin, MCP, HealthRegistry
- 索引として機能させるため用語はソース英語表記を基準とする

## Validation plan
- テーブルの全14クラスが05_agent-impl-class.md §1.7と一致していること
- 全クラスに詳細章番号が割り当てられていること
- 用語集が8件以上収録されていること
- 40行以内に収まっていること
