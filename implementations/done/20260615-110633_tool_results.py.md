# Implementation: tool_results.py (Phase 2 — DI)

## Goal

Add optional constructor-level dependency injection to `ToolResultStore` so that tests
can supply an in-memory or temporary `SQLiteHelper` without relying on a real
session.sqlite. Each of the three public methods currently hard-codes
`SQLiteHelper("session")` inside the `with` block; a `_db_factory` callable makes this
replaceable.

## Scope

**In**: `scripts/db/tool_results.py` only.
**Out**: No changes to callers (`agent/context.py`, `tool_runner.py` etc.) — the default
arg preserves backward compatibility.

## Assumptions

1. `SQLiteHelper` is imported from `db.helper` and its constructor/open() contract is unchanged.
2. Passing `db_factory=None` (default) must behave identically to the current code.
3. The DI injection point is a zero-arg callable `Callable[[], SQLiteHelper]` so callers
   control both target DB and open mode.
4. All log messages and comments must remain in English (coding.md).

## Implementation

### Target file

`scripts/db/tool_results.py`

### Procedure

1. Add `from collections.abc import Callable` import.
2. Add `_db_factory` field to `ToolResultStore.__init__`:
   ```python
   def __init__(
       self,
       db_factory: Callable[[], SQLiteHelper] | None = None,
   ) -> None:
       self._db_factory = db_factory
   ```
3. Add `_open_write` and `_open_read` helpers that delegate to factory or default:
   ```python
   def _open_write(self) -> SQLiteHelper:
       return self._db_factory() if self._db_factory else SQLiteHelper("session").open(write_mode=True)
   ```
   Wait — `SQLiteHelper.open()` returns `self` and is used as a context manager.
   Better approach: introduce `_session_db_write` / `_session_db_read` properties that
   return a context-manager-compatible object.

   Actually simplest approach: factory returns a pre-opened `SQLiteHelper` context manager.
   Use `contextlib.contextmanager` pattern instead.

   Simplest of all: just use `_db_target` (str | SQLiteHelper) override:

   ```python
   def __init__(self, db_target: str = "session") -> None:
       self._db_target = db_target
   ```

   And replace every `SQLiteHelper("session")` with `SQLiteHelper(self._db_target)`.
   This allows `ToolResultStore(db_target="/tmp/test.sqlite")` in tests.

   But `SQLiteHelper` takes a `target: str` ("rag" | "session") not a path.
   Check `SQLiteHelper.__init__` signature.

4. After checking: `SQLiteHelper` accepts `target: str = "rag"` where the target is used
   to look up `db_cfg.rag_db_path` or `db_cfg.session_db_path`. Path injection requires
   patching `build_db_config`.

   **Final approach**: Add a `_helper_factory` callable that defaults to
   `lambda mode: SQLiteHelper("session").open(**mode)`. Not clean.

   **Cleanest approach aligned with plan**: Replace inline `SQLiteHelper("session")`
   with a private method `_helper(write_mode=False, row_factory=False)` that can be
   overridden in tests:

   ```python
   def _make_helper(self, *, write_mode: bool = False, row_factory: bool = False) -> SQLiteHelper:
       return SQLiteHelper("session").open(write_mode=write_mode, row_factory=row_factory)
   ```

   In tests, subclass `ToolResultStore` and override `_make_helper`.

### Method

1. Add `_make_helper()` private method to `ToolResultStore`.
2. Replace the 3 inline `SQLiteHelper("session").open(...)` with `self._make_helper(...)`.
3. No change to callers.

### Details

```python
# Add this method to ToolResultStore:
def _make_helper(self, *, write_mode: bool = False, row_factory: bool = False) -> "SQLiteHelper":
    return SQLiteHelper("session").open(write_mode=write_mode, row_factory=row_factory)

# store():
with self._make_helper(write_mode=True) as db:  # was: SQLiteHelper("session").open(write_mode=True)

# get():
with self._make_helper(row_factory=True) as db:  # was: SQLiteHelper("session").open(row_factory=True)

# list_recent():
with self._make_helper(row_factory=True) as db:  # was: SQLiteHelper("session").open(row_factory=True)
```
