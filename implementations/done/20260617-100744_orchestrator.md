# Implementation: Remove history injection from _handle_llm_transport_error() (req 27 — Step 3)

## Goal

In `orchestrator.py`, remove `ctx.conv.history.append()` for incomplete LLM output and route it to `ctx.session.save_diagnostic()` only.

## Scope

- `scripts/agent/orchestrator.py`

## Assumptions

1. `ctx.session.save_diagnostic()` is already implemented (Step 2).
2. `tool_result_store.store()` call is kept unchanged — provides `/tool show` access.
3. The log message is updated to reflect "diagnostic" instead of "saved".

## Implementation

### Target file

`scripts/agent/orchestrator.py`

### Procedure

In `_handle_llm_transport_error()`, replace lines that inject into history with `save_diagnostic()`.

### Method

Edit tool.

### Details

Old:
```python
if e.partial_text:
    incomplete_msg = f"{e.partial_text}\n[INCOMPLETE: {e.kind}]"
    ctx.conv.history.append(LLMMessage(role="assistant", content=incomplete_msg))
    ctx.session.save("assistant", incomplete_msg)
    ...
```

New:
```python
if e.partial_text:
    incomplete_msg = f"{e.partial_text}\n[INCOMPLETE: {e.kind}]"
    # Store in diagnostic channel only — do NOT add to conversation history
    ctx.session.save_diagnostic(incomplete_msg)
    ...
```

Remove `LLMMessage` import if it becomes unused after this change.
