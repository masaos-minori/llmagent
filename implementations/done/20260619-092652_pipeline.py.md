# Implementation: Plumb rrf_k from Config to FusionStage

## Goal

Pass the `rrf_k` parameter from runtime config to `FusionStage` so the k value is no longer hardcoded to `_DEFAULT_RRF_K` inside the stage. This enables operators to tune RRF scoring by editing config without touching source code.

## Scope

- `scripts/shared/types.py` — add `rrf_k: int` to `RagConfig` protocol
- `scripts/agent/config_dataclasses.py` — add `rag_rrf_k: int = 60` field
- `scripts/agent/config_builders.py` — read `rag_rrf_k` from TOML, populate `AgentConfig`
- `scripts/mcp/rag_pipeline/models.py` — add `rrf_k: int = 60` to `RagPipelineConfig` and `build_rag_cfg_adapter()`
- `scripts/rag/pipeline.py` — update `FusionStage(...)` call to pass `rrf_k`

Out of scope:
- `FusionStage` itself (already accepts `rrf_k: int = _DEFAULT_RRF_K`)
- Runtime `/set` override
- Config hot-reload path

## Assumptions

1. The plan assumed `self._cfg.fusion.rrf_k` (nested `PipelineConfig` structure). The actual `self._cfg` is `RagConfig`, a flat Protocol. The nested path does not exist.
2. `FusionStage.__init__` signature is `(rrf_k: int = _DEFAULT_RRF_K, use_rrf: bool = True)` — already correct.
3. Default `rrf_k = 60` matches `FusionStage._DEFAULT_RRF_K` and `FusionConfig.rrf_k`.
4. `AgentConfig` uses `memory_rrf_k` for the memory retriever (separate concern); this adds `rag_rrf_k` for the RAG pipeline.

## Implementation

### Target file

Primary: `scripts/rag/pipeline.py`
Supporting: `scripts/shared/types.py`, `scripts/agent/config_dataclasses.py`, `scripts/agent/config_builders.py`, `scripts/mcp/rag_pipeline/models.py`

### Procedure

1. Add `rrf_k: int` to `RagConfig` protocol.
2. Add `rag_rrf_k: int = 60` field to `AgentConfig` with a validator (`>= 1`).
3. In `config_builders.py`, read `rag_rrf_k` from TOML and populate the field.
4. In `RagPipelineConfig`, add `rrf_k: int = 60`; update `from_dict()` to read it; update `build_rag_cfg_adapter()` to expose it.
5. In `pipeline.py` line 208, change the call to `FusionStage(use_rrf=self._cfg.use_rrf, rrf_k=self._cfg.rrf_k)`.

### Method

Minimal additive change. No new abstractions. The `RagConfig` protocol extension requires all concrete implementors (`AgentConfig`, `SimpleNamespace` from `build_rag_cfg_adapter`) to carry the field; failing to add it to any implementor will be caught by mypy.
