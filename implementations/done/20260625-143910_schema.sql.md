# Implementation: schema.sql — delivery_failure_count field (req #24)

## Goal

Add `delivery_failure_count` field to the `events` table to track per-event consumer nack count, replacing `retry_count` as the DLQ promotion trigger.

## Scope

- Add `delivery_failure_count INTEGER NOT NULL DEFAULT 0` column to the CREATE TABLE statement
- Keep `retry_count` as deprecated (additive migration, per req #28 decision)
- Add migration SQL for existing databases

## Assumptions

- req #28 is implemented first (schema.sql change is shared between req #24 and #28)
- SQLite 3.25.0+ is available (Python 3.13 ships with SQLite 3.46+)
- `_init_schema()` in `db.py` calls `conn.executescript(sql)` so new columns are picked up automatically for new DBs

## Implementation

### Target file

`scripts/eventbus/schema.sql`

### Procedure

1. Add `delivery_failure_count INTEGER NOT NULL DEFAULT 0` after the `retry_count` line in the CREATE TABLE statement
2. Add comment marking `retry_count` as deprecated
3. Add migration block at the end of schema.sql for existing production databases

### Method

Edit the CREATE TABLE block and append migration SQL.

### Details

**Current CREATE TABLE (lines 3–13):**
```sql
CREATE TABLE IF NOT EXISTS events (
    seq          INTEGER PRIMARY KEY AUTOINCREMENT,
    event_id     TEXT    NOT NULL UNIQUE,
    topic        TEXT    NOT NULL,
    payload      TEXT    NOT NULL,
    producer     TEXT    NOT NULL,
    published_at TEXT    NOT NULL,
    acked_at     TEXT, -- reserved; not currently set by any code path
    retry_count  INTEGER NOT NULL DEFAULT 0,
    dlq_at       TEXT
);
```

**Target state:**
```sql
CREATE TABLE IF NOT EXISTS events (
    seq                    INTEGER PRIMARY KEY AUTOINCREMENT,
    event_id               TEXT    NOT NULL UNIQUE,
    topic                  TEXT    NOT NULL,
    payload                TEXT    NOT NULL,
    producer               TEXT    NOT NULL,
    published_at           TEXT    NOT NULL,
    acked_at               TEXT,
    retry_count            INTEGER NOT NULL DEFAULT 0, -- deprecated; use delivery_failure_count
    delivery_failure_count INTEGER NOT NULL DEFAULT 0,
    dlq_requeue_count      INTEGER NOT NULL DEFAULT 0,
    dlq_at                 TEXT
);
```

**Migration block to append:**
```sql
-- Additive migration for existing databases (idempotent via IF NOT EXISTS)
ALTER TABLE events ADD COLUMN IF NOT EXISTS delivery_failure_count INTEGER NOT NULL DEFAULT 0;
ALTER TABLE events ADD COLUMN IF NOT EXISTS dlq_requeue_count INTEGER NOT NULL DEFAULT 0;
```

Note: `ALTER TABLE ADD COLUMN IF NOT EXISTS` is supported since SQLite 3.37.0. Older SQLite (3.25–3.36) requires catching `OperationalError: duplicate column name`. Since Python 3.13 ships SQLite 3.46+, this is safe in production but add a startup check if portability matters.
