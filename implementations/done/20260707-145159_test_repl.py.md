# Implementation: M-7 — tests for SIGTERM-during-idle behavior (test_repl.py)

## Goal

Add tests verifying that `AgentREPL._read_input()` returns `None` promptly when
`_shutdown_event` is set, returns the input string on normal input, and falls back
safely when `_shutdown_event is None`.

## Scope

**Target**: `tests/test_repl.py` (or the equivalent REPL test file)

**Step covered**: Plan M-7 step 2.

**Out of scope**: source changes, other test files.

## Assumptions

1. `AgentREPL` can be instantiated with a stub context in tests.
2. `asyncio.AbstractEventLoop` can be obtained via `asyncio.get_event_loop()` or
   `asyncio.new_event_loop()` within the test.
3. `_read_input()` can be tested without launching a real REPL loop.

## Implementation

### Target file

`tests/test_repl.py`

### Procedure

#### Setup helper

```python
def make_repl():
    ctx = make_stub_ctx()
    view = MagicMock()
    repl = AgentREPL(ctx, view)
    repl._prompt = "> "
    repl._shutdown_event = asyncio.Event()
    return repl
```

#### Test 1: _shutdown_event.set() before input → returns None

```python
@pytest.mark.asyncio
async def test_read_input_returns_none_on_shutdown_event():
    repl = make_repl()
    repl._shutdown_event.set()  # signal shutdown immediately

    loop = asyncio.get_event_loop()
    result = await repl._read_input(loop)

    assert result is None
```

#### Test 2: shutdown event set while awaiting → returns None promptly

```python
@pytest.mark.asyncio
async def test_read_input_returns_none_when_shutdown_event_fires():
    repl = make_repl()
    loop = asyncio.get_event_loop()

    async def set_event_soon():
        await asyncio.sleep(0.05)
        repl._shutdown_event.set()

    asyncio.ensure_future(set_event_soon())
    # _read_input blocks on input(); shutdown event fires after 50ms
    # We can't easily test the blocking input() path without stdin mocking,
    # so mock input() to block for 5s
    with patch("builtins.input", side_effect=lambda p: (asyncio.sleep(5), "never")[1]):
        result = await asyncio.wait_for(repl._read_input(loop), timeout=1.0)

    assert result is None
```

#### Test 3: _shutdown_event is None → falls back to blocking await

```python
@pytest.mark.asyncio
async def test_read_input_fallback_when_no_shutdown_event(monkeypatch):
    repl = make_repl()
    repl._shutdown_event = None

    monkeypatch.setattr("builtins.input", lambda p: "hello")
    loop = asyncio.get_event_loop()

    result = await repl._read_input(loop)

    assert result == "hello"
```

#### Test 4: EOF with _shutdown_event → returns None

```python
@pytest.mark.asyncio
async def test_read_input_eof_returns_none(monkeypatch):
    repl = make_repl()
    monkeypatch.setattr("builtins.input", lambda p: (_ for _ in ()).throw(EOFError()))

    loop = asyncio.get_event_loop()
    result = await repl._read_input(loop)

    assert result is None
```

### Method

- `asyncio.Event` is pre-set for Test 1 (no timing dependency).
- Test 2 requires mocking `builtins.input` in the executor thread. Use `monkeypatch`
  or `unittest.mock.patch` as a context manager.
- Test 3 verifies the `else` branch with `_shutdown_event = None`.
- `asyncio.wait_for(..., timeout=1.0)` in Test 2 ensures the test does not hang if
  the race fails to return `None`.

### Details

- Patching `builtins.input` must happen in the test process before the executor thread
  picks it up; since `input()` is called inside the executor, the patch must be in
  place before `_read_input()` is called.
- Tests that mock `input()` must restore the original after the test; use
  `monkeypatch.setattr("builtins.input", ...)` (auto-restored) or `patch` as context
  manager.
- Test 2 may be flaky if the 50ms timer and the mock input() don't race correctly.
  Consider increasing the delay or using `asyncio.Event` directly to trigger shutdown
  after a known sync point.
