# Implementation Procedure: scripts/agent/startup.py

## Goal

Wire `check_routing_drift_live()` into `_check_services()` in startup.py, after existing `check_routing_drift(ctx)` call.

## Scope

**In**:
- Import `check_routing_drift_live` from `agent.repl_health`
- Add call to `check_routing_drift_live(ctx)` in `_check_services()` after the existing `check_routing_drift(ctx)` call

**Out**:
- No changes to any other files

## Assumptions

1. `_check_services()` is the single call site for all startup health functions (confirmed from source).
2. The new function should be called after `check_routing_drift(ctx)` — config drift first, then live drift.
3. Output pattern matches existing: iterate messages and write warnings via `self._view.write_warning()`.

### Method

- Follow the exact same pattern as the existing `check_routing_drift(ctx)` call — iterate messages and write warnings.
- Call after config drift check, before RAG consistency check.
- No error handling needed — `check_routing_drift_live()` already swallows exceptions internally.

### Details

- The async call is appropriate because `_check_services()` is an async method.
- The order matters: config drift → live drift → RAG consistency (as specified in the plan).
