# Implementation: Add Skipped-Chunk Counter to ingest_url_group()

## Goal

After `ingest_all()` completes, operators can see how many chunks failed to embed or insert (e.g., due to embedding service errors or DB failures). Currently the log only shows `inserted N/M chunks` per URL but has no aggregate failure count.

## Scope

- `scripts/rag/ingestion/ingester.py`
  - `ingest_url_group()`: change return type from `None` to a named result (or tuple) carrying `(n_inserted, n_failed)`
  - `_ingest_chunk_files()`: track failed chunk count and return it alongside the inserted count
  - `ingest_all()`: log aggregate counts across all URL groups

Out of scope:
- Full `IngestResult` DTO (plan explicitly excludes this — counter only)
- Per-chunk error details in return value
- Changes to callers outside `ingester.py` (ingester is the leaf; callers only call `ingest_all()`)

## Assumptions

1. `ingest_url_group()` currently returns `None` (confirmed, line 107). Changing it to return a value is safe — `ingest_all()` calls it via `_process_url_groups()` and discards the return.
2. `_ingest_chunk_files()` currently returns `inserted: int` (line 323-343). It can be extended to also return `n_failed`.
3. The failing case in `_embed_and_insert_chunks()` returns `False` per chunk (line 299, 316); aggregating `False` counts gives `n_failed`.
4. Skipping an already-registered URL (force=False path) is NOT a failure — it should be counted separately as `n_skipped_urls`, but the plan only asks for chunk-level failures.

## Implementation

### Target file

`scripts/rag/ingestion/ingester.py`

### Procedure

1. In `_ingest_chunk_files()`, count chunks where `_embed_and_insert_chunks()` returns `False` and return `(inserted, failed)`.
2. In `ingest_url_group()`, capture the `(inserted, failed)` tuple and return it.
3. In `_process_url_groups()` or `ingest_all()`, accumulate totals and log at completion.

### Method

Minimal: change `_ingest_chunk_files` return from `int` to `tuple[int, int]`. Update two call sites. No new classes.

### Details

**`_ingest_chunk_files()` (around line 323-343) — current:**

**Updated:**
```python
def _ingest_chunk_files(
    self, doc_id: int, chunk_files: list[Path]
) -> tuple[int, int]:
    """Embed and insert chunk files; returns (n_inserted, n_failed)."""
    with ThreadPoolExecutor(...) as executor:
        results = list(executor.map(
            lambda p: self._embed_and_insert_chunk(doc_id, p), chunk_files
        ))
    inserted = sum(1 for r in results if r)
    failed = sum(1 for r in results if not r)
    return inserted, failed
```

**`ingest_url_group()` — updated return:**
```python
def ingest_url_group(
    self, db: SQLiteHelper, url: str, chunk_files: list[Path], force: bool
) -> tuple[int, int]:
    """Returns (n_inserted, n_failed)."""
    ...
    inserted, failed = self._ingest_chunk_files(doc_id, chunk_files)
    logger.info(
        "inserted %s/%s chunks (%s failed): %s",
        inserted, len(chunk_files), failed, url,
    )
    self._move_to_registered(chunk_files)
    return inserted, failed
```

Return `(0, 0)` on early returns (no chunk files, doc_id is None).

**`ingest_all()` — aggregate logging:**
```python
def ingest_all(self, force: bool = False) -> None:
    ...
    total_inserted, total_failed = 0, 0
    with SQLiteHelper().open(write_mode=True) as db:
        for url, chunks in url_groups.items():
            n_in, n_fail = self.ingest_url_group(db, url, chunks, force)
            total_inserted += n_in
            total_failed += n_fail
    logger.info(
        "=== done: %s URLs, %s chunks inserted, %s chunks failed ===",
        len(url_groups), total_inserted, total_failed,
    )
```

Note: if `_process_url_groups()` is a separate method that iterates the groups, update that method to accumulate and return the totals, then log in `ingest_all()`.
