# Implementation: scripts/agent/repl_health.py — Add [non-fatal] prefix to unreachable LLM/embed warnings

**Plan source:** `plans/20260702-202919_plan.md` (Phase 3 (repl_health))
**Target file:** `scripts/agent/repl_health.py`

---

## Goal

repl_health.py の check_service_health() で unreachable な LLM/embed サービスに対する警告メッセージ文字列に `[non-fatal]` プレフィックスを付加し、非致命的エラーであることを明示する。

---

## Scope

**In:**
- check_service_health() 内の LLM/embed 到達不能時の警告メッセージ文字列を `f"[non-fatal] {label} unreachable at {health_url}: {e}"` 形式に更新

**Out:**
- 警告の送出方法 (write_warning 呼び出し自体) の変更
- 致命的エラー (fatal) への格上げ
- LLM/embed 以外のサービスチェックの変更

---

## Assumptions

1. check_service_health() 内で `{label} unreachable at {health_url}` のような形式の警告文字列が既に存在する
2. `label`、`health_url`、`e` の変数名が実際のコードで使用されている (確認後に調整)
3. write_warning の呼び出し形式はそのまま維持し、渡すメッセージ文字列のみ変更する

---

## Implementation

### Target file

`scripts/agent/repl_health.py`

### Procedure

1. repl_health.py を読み込み、check_service_health() の実装と既存の警告メッセージ文字列を確認する
2. unreachable 判定箇所の警告メッセージ文字列の変数名を特定する
3. メッセージ文字列を `f"[non-fatal] {label} unreachable at {health_url}: {e}"` に変更する (実際の変数名に合わせる)
4. lint/mypy を確認する

### Method

Edit tool でコード変更

### Details

変更前 (例):
```python
write_warning(f"{label} unreachable at {health_url}: {e}")
```

変更後:
```python
write_warning(f"[non-fatal] {label} unreachable at {health_url}: {e}")
```

実際の変数名は repl_health.py を読んで確認する。メッセージ文字列のプレフィックス追加のみで、ロジックは変更しない。
