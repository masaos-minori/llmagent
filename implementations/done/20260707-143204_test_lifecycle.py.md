# Implementation: H-7 — tests for shutdown_all cleanup (test_lifecycle.py)

## Goal

Add tests verifying that `HttpServerLifecycleManager.shutdown_all()` empties
`_http_procs`, handles already-exited processes gracefully, continues cleanup when one
server's termination raises, and is safe to call twice.

## Scope

**Target**: `tests/test_lifecycle.py`

**Step covered**: Plan H-7 step 2.

**Out of scope**: source changes, other test files.

## Assumptions

1. `HttpServerLifecycleManager` can be instantiated directly in tests.
2. Fake `subprocess.Popen` objects can be created with `MagicMock(spec=subprocess.Popen)`.
3. `proc.poll()` can be set to `None` (running) or `0` (exited) via `MagicMock.return_value`.

## Implementation

### Target file

`tests/test_lifecycle.py`

### Procedure

#### Helper: make_mock_proc

```python
def make_mock_proc(running: bool = True) -> MagicMock:
    proc = MagicMock(spec=subprocess.Popen)
    proc.poll.return_value = None if running else 0
    proc.pid = 12345
    return proc
```

#### Test 1: Two running processes → shutdown_all() → _http_procs is empty

```python
@pytest.mark.asyncio
async def test_shutdown_all_clears_http_procs():
    mgr = HttpServerLifecycleManager()
    mgr._http_procs["srv1"] = make_mock_proc(running=True)
    mgr._http_procs["srv2"] = make_mock_proc(running=True)

    # Patch _terminate_with_timeout to a no-op
    mgr._terminate_with_timeout = AsyncMock()

    await mgr.shutdown_all()

    assert len(mgr._http_procs) == 0
```

#### Test 2: Already-exited process → removed without terminate, no error

```python
@pytest.mark.asyncio
async def test_shutdown_all_removes_exited_proc_without_terminate():
    mgr = HttpServerLifecycleManager()
    mgr._http_procs["exited"] = make_mock_proc(running=False)
    mgr._terminate_with_timeout = AsyncMock()

    await mgr.shutdown_all()

    assert len(mgr._http_procs) == 0
    mgr._terminate_with_timeout.assert_not_awaited()
```

#### Test 3: One termination raises OSError → second server still cleaned

```python
@pytest.mark.asyncio
async def test_shutdown_all_continues_after_terminate_error():
    mgr = HttpServerLifecycleManager()
    mgr._http_procs["srv1"] = make_mock_proc(running=True)
    mgr._http_procs["srv2"] = make_mock_proc(running=True)

    call_count = 0

    async def flaky_terminate(proc, key, timeout=3.0):
        nonlocal call_count
        call_count += 1
        if key == "srv1":
            raise OSError("terminate failed")

    mgr._terminate_with_timeout = flaky_terminate

    await mgr.shutdown_all()  # must not raise

    assert len(mgr._http_procs) == 0
    assert call_count == 2  # both servers attempted
```

#### Test 4: shutdown_all() twice → no error, _http_procs still empty

```python
@pytest.mark.asyncio
async def test_shutdown_all_twice_is_safe():
    mgr = HttpServerLifecycleManager()
    mgr._http_procs["srv"] = make_mock_proc(running=True)
    mgr._terminate_with_timeout = AsyncMock()

    await mgr.shutdown_all()
    await mgr.shutdown_all()  # must not raise

    assert len(mgr._http_procs) == 0
```

### Method

- All tests construct `HttpServerLifecycleManager()` directly and populate `_http_procs`
  with mock processes.
- `_terminate_with_timeout` is patched at the instance level to avoid real process
  interaction.
- Test 3 uses a closure to count calls and raise on the first server only.

### Details

- Import `subprocess` in the test file for `MagicMock(spec=subprocess.Popen)`.
- Tests do not start real subprocesses; they exercise only the loop/cleanup logic.
- If H-1 is merged, `_stderr_files` cleanup is covered by H-1 test 4 (restart closes
  handle); add an additional assertion in Test 1 here if `_stderr_files` is present:
  `assert len(mgr._stderr_files) == 0`.
