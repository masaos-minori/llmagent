# Implementation: H-7 — Remove ToolResultStore reference from undo_last_turn()

## Goal

Remove the `ctx.tool_result_store.mark_turn_undone(...)` call from `undo_last_turn()`.
`UndoResult.n_artifacts_marked` becomes always `0` — the field stays in the model (no schema
change) but the value is now a fixed constant instead of a real count.

## Scope

**Target**: `scripts/agent/services/undo_service.py`

**Depends on**: land together with `implementations/20260708-163935_context.py.md` and the
other H-7 companion docs.

**Out of scope**: `UndoResult` dataclass definition (`agent/services/models.py`) — the
`n_artifacts_marked` field is NOT removed, only always set to `0` from this call site (per the
plan's U-3 resolution: "解決: n_artifacts_marked=0 固定. `UndoResult` モデルの変更は不要").

## Assumptions

1. `ctx.tool_result_store.mark_turn_undone(...)` (lines 50-53) has exactly one call site in this
   file (confirm via grep in Procedure Step 1).
2. `turn_to_mark` (captured before decrementing `stat_turns`) has no other use besides this call
   and can be left in place even though it becomes otherwise unused-for-tool-result-purposes — it
   documents intent and costs nothing to keep, OR can be removed if it has no other reader (see
   Details).

## Implementation

### Target file

`scripts/agent/services/undo_service.py`

### Procedure

#### Step 1: Confirm the call site

```bash
grep -n "tool_result_store" scripts/agent/services/undo_service.py
```

Expected: exactly one match, spanning lines 50-53 (the `if` guard and `.mark_turn_undone(...)`
call).

#### Step 2: Remove the guard block, fix n_marked

Current (lines 49-53):

```python
    n_marked = 0
    if turn_to_mark > 0 and ctx.tool_result_store is not None:
        n_marked = ctx.tool_result_store.mark_turn_undone(
            ctx.session.session_id, turn_to_mark
        )
```

Replace with:

```python
    n_marked = 0
```

(i.e. remove the `if turn_to_mark > 0 and ctx.tool_result_store is not None:` block and its
body entirely; `n_marked = 0` stays as the sole remaining line, now unconditional.)

### Method

- Pure deletion of the conditional mark-as-undone call; `n_marked` becomes a constant `0` passed
  to `UndoResult(n_removed=removed, n_artifacts_marked=n_marked)` at the end of the function
  (unchanged call site, now always receiving `0`).

### Details

- `turn_to_mark` (line 46, `turn_to_mark = ctx.stats.stat_turns`) becomes unused after this
  change since its only consumer was the removed `.mark_turn_undone(...)` call. Check with
  `ruff check` whether this triggers `F841` (unused variable); if so, remove the
  `turn_to_mark = ctx.stats.stat_turns` assignment on line 46 as well, since it no longer serves
  any purpose (the comment above it, "Capture turn BEFORE decrementing stat_turns — equals the
  last stored turn," only explained why `turn_to_mark` needed to be captured before the
  decrement for the now-removed call).
- If `turn_to_mark` is removed, also remove its explanatory comment (line 45) since it no longer
  applies to anything in the function.
- `ctx.session.undo_last_turn()` (line 48) and `ctx.stats.stat_turns` decrement (line 47) are
  unaffected and stay in their current order relative to the removed block.
