# Implementation: utils.py + cmd_db.py тАФ Consolidate flag-parsing helpers

## Goal

Move `_parse_flag_int()` and `_parse_flag_str()` from `cmd_db.py` into the shared
`utils.py` so all command mixins share a single, tested implementation.

## Scope

- `scripts/agent/commands/utils.py`: add `parse_flag_int` and `parse_flag_str` (no leading underscore).
- `scripts/agent/commands/cmd_db.py`: delete the two module-level helpers; import from utils instead.

## Assumptions

1. `_parse_flag_int` / `_parse_flag_str` exist only in `cmd_db.py` and are not referenced
   by any other module (grep confirmed).
2. `utils.py` has no circular import risk тАФ it imports only from `shared.types`, `orjson`,
   `logging`, and `pathlib`.
3. Renaming from `_parse_flag_*` (private) to `parse_flag_*` (public) is intentional:
   utils helpers are meant to be shared across mixins.

### Procedure

**utils.py тАФ append two functions after `write_export`:**

```python
def parse_flag_int(tokens: list[str], flag: str) -> int | None:
    """Return the integer value following `flag` in tokens, or None."""
    for i, t in enumerate(tokens):
        if t == flag and i + 1 < len(tokens):
            try:
                return int(tokens[i + 1])
            except ValueError:
                pass
    return None


def parse_flag_str(tokens: list[str], flag: str) -> str | None:
    """Return the string value following `flag` in tokens, or None."""
    for i, t in enumerate(tokens):
        if t == flag and i + 1 < len(tokens):
            return tokens[i + 1]
    return None
```
