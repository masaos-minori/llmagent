# Implementation: injection.py — pass branch to on_user_prompt search calls

Source plan: `plans/20260626-180404_plan.md` — Phase 1

---

## Goal

Fix `MemoryInjectionService.on_user_prompt()` to pass `branch=self._branch` to both `HybridRetriever.search()` calls so that branch context is used as an active scoring boundary during task-specific retrieval.

---

## Scope

**In-Scope**
- Add `branch=self._branch` to the two `self._retriever.search()` calls in `on_user_prompt()` (lines ~84–104)
- No structural changes — this is a two-line fix

**Out-of-Scope**
- Changing HybridRetriever.search() signature (already accepts `branch`)
- Adding strict branch exclusion (boost-only policy is intentional)
- on_session_start() — already passes branch correctly

---

## Assumptions

1. `self._branch` is set in `__init__` and available in `on_user_prompt()` (confirmed: injection.py:42).
2. `HybridRetriever.search()` signature: `search(query, embedding, project, repo, branch)` — `branch=""` default (confirmed: retriever.py:276).
3. FtsRetriever applies `_context_boost` which gives `_CONTEXT_MATCH_BOOST + 0.05 = 0.15` for branch match (confirmed: retriever.py:74–78).
4. The fix does not break any existing test (branch defaults to `""` if not set, same behavior as today).
