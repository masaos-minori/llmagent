# Implementation: M-4 — Populate new McpProbeResult fields in McpStatusService (mcp_status.py)

## Goal

Update `McpStatusService._probe_single_server()` to populate the 11 new fields added to
`McpProbeResult` in M-4 step 1: `lifecycle_state`, `pid`, `pgid`, `running`,
`managed`, `last_exit_code`, and `restart_recommended`.

## Scope

**Target**: `scripts/agent/services/mcp_status.py`

**Step covered**: Plan M-4 step 4.

**Dependencies**: M-4 step 1 (fields in `McpProbeResult`), M-4 step 3 (`get_process_snapshot()` on router).

**Out of scope**: `cmd_mcp.py` renderer (covered separately), test files.

## Assumptions

1. `_probe_single_server()` already has access to `ctx.services_required.lifecycle`
   and `server_key`.
2. `get_transport_state(server_key).value` returns a string (`"running"`, `"stopped"`, etc.).
3. `get_process_snapshot(server_key)` returns `dict | None` (None for persistent servers).
4. `restart_recommended` is set to `True` when `lifecycle_state == "failed"`.

## Implementation

### Target file

`scripts/agent/services/mcp_status.py`

### Procedure

#### Step 1: Read lifecycle state

In `_probe_single_server()`, after obtaining `server_key`, add:
```python
lifecycle = ctx.services_required.lifecycle
lifecycle_state = lifecycle.get_transport_state(server_key).value
```

#### Step 2: Get process snapshot

```python
snapshot_fn = getattr(lifecycle, "get_process_snapshot", None)
snapshot = snapshot_fn(server_key) if snapshot_fn is not None else None
```

#### Step 3: Populate new fields in McpProbeResult construction

Add to the `McpProbeResult(...)` constructor call:
```python
managed=snapshot is not None,
pid=snapshot.get("pid") if snapshot else None,
pgid=snapshot.get("pgid") if snapshot else None,
running=snapshot.get("running") if snapshot else None,
last_exit_code=snapshot.get("last_exit_code") if snapshot else None,
lifecycle_state=lifecycle_state,
restart_recommended=(lifecycle_state == LifecycleState.FAILED.value),
```

Add import at the top of the file:
```python
from agent.lifecycle import LifecycleState
```

### Method

- All new fields are computed from existing lifecycle APIs; no new network calls.
- `getattr` guard makes `get_process_snapshot` optional (duck-typing pattern).

### Details

- `lifecycle_state == LifecycleState.FAILED.value` compares string `"failed"` to
  string `"failed"` (since `.value` returns the enum's string value).
- For persistent-mode servers: `snapshot` is `None` → `managed=False`, all process
  fields `None`.
- `stderr_log` and `last_shutdown_result` are left as defaults `""` in this step;
  M-6 populates them via `ProcessInfoSnapshot`.
- `restart_recommended` and `operator_action_required` from `/health` body are
  populated by M-5; this step only sets `restart_recommended` from lifecycle state.
