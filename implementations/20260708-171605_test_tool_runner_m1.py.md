# Implementation: M-1 — Remove forbidden config keys from test_tool_runner.py's _cfg()

## Goal

Remove `use_tool_summarize`/`tool_summarize_threshold` from `_cfg()`'s `defaults` dict. This is
mandatory, not optional cleanup: `_cfg()` calls `build_agent_config(defaults)`, and once
`config_builders.py`'s `_FORBIDDEN_KEYS` includes these keys (per its M-1 doc), passing them
would make **every test in this file** fail with `ConfigLoadError`.

## Scope

**Target**: `tests/test_tool_runner.py` — the `_cfg()` helper's `defaults` dict only (lines
~42-43).

**Distinct from**: `implementations/20260708-162748_tool_runner_h1.py.md` and
`implementations/20260708-162842_test_tool_runner_h1.py.md` (H-1 docs for this same test file),
which added new tests for `execute_one_tool_call()`'s truncation-only behavior but did NOT touch
`_cfg()`'s defaults dict — those docs' new tests also call `_cfg()`, so they benefit from this
fix too. Also distinct from `implementations/20260708-162621_test_tool_runner.py.md` (H-2 doc),
which removed `ctx.tool_result_store` mock setup, unrelated to this config-key issue.

**Depends on / must land together with**: `implementations/20260708-171134_config_builders.py.md`
(adds the keys to `_FORBIDDEN_KEYS`, which is what makes this fix mandatory).

## Assumptions

1. `_cfg()`'s `defaults` dict (lines ~24-65) is used by every test class in this file via
   `_make_ctx(cfg)` — fixing it once benefits all tests uniformly, including the new
   `TestExecuteOneToolCallTruncation` class added by the H-1 doc.

## Implementation

### Target file

`tests/test_tool_runner.py`

### Procedure

#### Step 1: Confirm the current lines

```bash
grep -n "use_tool_summarize\|tool_summarize_threshold" tests/test_tool_runner.py
```

Expected: two matches inside `_cfg()`'s `defaults` dict (lines ~42-43).

#### Step 2: Remove both keys

Current (within the `defaults` dict):

```python
        "serial_tool_calls": False,
        "use_tool_summarize": False,
        "tool_summarize_threshold": 3000,
        "use_semantic_cache": False,
```

Replace with:

```python
        "serial_tool_calls": False,
        "use_semantic_cache": False,
```

### Method

- Two-key removal from a shared fixture dict; every test in the file that goes through `_cfg()`
  is fixed by this one change.
