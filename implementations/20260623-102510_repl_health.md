## Goal

Fix two issues in `scripts/agent/repl_health.py`:

1. **Bug fix** — `_check_tool_definitions()` currently logs INFO and returns an empty `HealthCheckResult` when all servers are unreachable, even when `strict=True`. Add a `RuntimeError` raise for the `strict=True` + all-unreachable case.

2. **New function** — Add `check_tool_definitions_startup()` that reads `ctx.cfg.tool.tool_definitions_strict` and calls `_check_tool_definitions()` with the correct `strict` flag. This is the function `startup.py` will call.

## Scope

**In-Scope:**
- `scripts/agent/repl_health.py` lines 210-217 — add `if strict: raise RuntimeError(...)` branch
- `scripts/agent/repl_health.py` after line 248 — add `check_tool_definitions_startup()` function

**Out-of-Scope:**
- No changes to `check_tool_definitions_runtime()` (remains non-strict)
- No changes to `startup.py` (covered in separate doc)
- No test file changes

## Assumptions

1. The `not server_names` branch (lines 210-217) fires when `_collect_server_tool_names()` returns an empty set. This covers both "all servers unreachable" (unreachable list non-empty) and "no servers configured" (unreachable list empty).
2. The `RuntimeError` should only be raised in the `if unreachable:` sub-branch — if there are simply no servers configured, strict mode silently passes (no servers = no tool definitions to validate).
3. `ctx.cfg.tool.tool_definitions_strict` returns `bool`; `getattr(..., False)` handles the case where the attribute is missing.
4. `check_tool_definitions_startup` is a public function (no leading underscore) to match the naming convention of `check_tool_definitions_runtime`.

## Implementation

### Target file
`scripts/agent/repl_health.py`

### Procedure

1. Locate lines 210-217 (`if not server_names:` block).
2. Replace with the corrected version that raises `RuntimeError` when `strict=True` and servers are unreachable.
3. After line 248 (end of `check_tool_definitions_runtime`), add `check_tool_definitions_startup()`.

### Method

Two `Edit` operations.

### Details

**Change 1 — `_check_tool_definitions()` bug fix (lines 210-217):**

Current:
```python
    if not server_names:
        if unreachable:
            msg = f"All MCP servers unreachable during strict validation: {sorted(set(unreachable))}; skipping tool definition check"
            logger.info(msg)
        else:
            msg = "No tool definitions in config and no servers reachable; skipping validation"
            logger.info(msg)
        return HealthCheckResult()
```

Replacement:
```python
    if not server_names:
        if unreachable:
            if strict:
                msg = (
                    f"Strict mode: all MCP servers unreachable — cannot validate tool definitions. "
                    f"Unreachable servers: {sorted(set(unreachable))}."
                )
                logger.error(msg)
                raise RuntimeError(msg)
            msg = f"All MCP servers unreachable; skipping tool definition check. Unreachable: {sorted(set(unreachable))}"
            logger.info(msg)
        else:
            msg = "No tool definitions in config and no servers reachable; skipping validation"
            logger.info(msg)
        return HealthCheckResult()
```

**Change 2 — new `check_tool_definitions_startup()` (insert after line 248):**

```python
async def check_tool_definitions_startup(ctx: AgentContext) -> HealthCheckResult:
    """Startup validation: respects tool_definitions_strict config flag."""
    strict = getattr(ctx.cfg.tool, "tool_definitions_strict", False)
    return await _check_tool_definitions(ctx, strict=strict)
```
