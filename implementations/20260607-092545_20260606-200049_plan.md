# Goal

Implement and test the MCP transport transparency and dual startup modes feature.

# Scope

- Implement the MCP transport transparency feature that allows switching between HTTP and stdio transports.
- Implement the dual startup mode that supports both HTTP and stdio modes.
- Ensure all existing MCP servers (12 total) work with both transport modes.
- Validate the implementation through tests.

# Assumptions

1. The configuration file `config/mcp_servers.toml` contains all MCP server configurations.
2. The `transport` field in the configuration can be set to either "http" or "stdio".
3. All existing MCP servers are already implemented and functional in HTTP mode.
4. stdio mode is supported by existing infrastructure.

# Implementation

1. Extend McpServerConfig to include transport type
2. Update ToolExecutor to handle both transport modes transparently
3. Implement ServerLifecycleManager for dual-mode server startup
4. Add --stdio flag support to all MCP servers
5. Add tool definition validation
6. Add watchdog functionality
7. Ensure REPL connections work with both transports

# Validation plan

- Run tests: `uv run pytest tests/test_lifecycle.py tests/test_mcp_server_base.py` and ensure they all pass.
- Manually verify that all 12 MCP servers can be started in both HTTP and stdio modes.
