## Assumptions

1. `McpServerHealthState` is already imported in `tool_executor.py` from `shared.mcp_config`. Verify before adding a new import line.
2. The `HALF_OPEN` check must come **before** the `is_unavailable()` call — once the registry transitions a server from `UNAVAILABLE` to `HALF_OPEN` inside `is_unavailable()`, the `get_state()` call in this method would return `HALF_OPEN`, so the guard `if state == HALF_OPEN: return None` must fire first.
3. `self._health_registry` may be `None` — the existing null guard must be preserved.

**Replacement:**
```python
    def _check_health(self, server_key: str) -> ToolCallResult | None:
        """Return an error result if the server is unavailable; None if healthy."""
        if self._health_registry is None:
            return None
        state = self._health_registry.get_state(server_key)
        if state == McpServerHealthState.HALF_OPEN:
            logger.info("Health: %r is HALF_OPEN — allowing trial dispatch", server_key)
            return None
        if self._health_registry.is_unavailable(server_key):
            msg = f"MCP server {server_key!r} is currently unavailable (health check failed)"
            logger.warning(msg)
            return self._error_result(server_key, msg, error_type="tool")
        return None
```

Key changes:
- Early `return None` when `_health_registry is None` (cleaner than `is not None and ...`)
- `HALF_OPEN` guard fires before `is_unavailable()` — if the registry already set state to `HALF_OPEN`, `is_unavailable()` would return `False` anyway, but the explicit check enables the INFO log
- `error_type="tool"` preserved (not changed to "transport" here — the existing behavior is unchanged for the UNAVAILABLE path)
