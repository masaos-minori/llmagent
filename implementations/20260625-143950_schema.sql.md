# Implementation: schema.sql — additive migration for delivery state fields (req #28)

## Goal

Add `delivery_failure_count` and `dlq_requeue_count` columns to the `events` table via additive migration, keeping `retry_count` as deprecated. This is the foundational schema change that req #24, #25, #26 depend on.

## Scope

- Add `delivery_failure_count INTEGER NOT NULL DEFAULT 0`
- Add `dlq_requeue_count INTEGER NOT NULL DEFAULT 0`
- Mark `acked_at` comment as "used by ack endpoint" (remove "reserved; not currently set")
- Mark `retry_count` comment as deprecated
- Add migration SQL block for existing databases

## Assumptions

- This is the FIRST change in the req #24–#30 implementation order (all others depend on it)
- Additive strategy confirmed by user (keep `retry_count`, add new columns)
- `_init_schema()` in `db.py` runs `executescript(sql)` which includes the migration ALTER TABLE statements
- `ALTER TABLE ADD COLUMN IF NOT EXISTS` requires SQLite 3.37.0+ (Python 3.10+ ships SQLite 3.37+; Python 3.13 ships 3.46+)
- This file is the same target as req #24's schema.sql change — req #28 owns the additive migration; req #24 references it

## Implementation

### Target file

`scripts/eventbus/schema.sql`

### Procedure

1. Update the CREATE TABLE block: add two new columns, update comments
2. Append migration SQL at the end of the file

### Method

Edit the existing `schema.sql` file. One change covers both req #28's schema addition and req #24's dependency.

### Details

**Full target file content:**
```sql
PRAGMA journal_mode=WAL;

CREATE TABLE IF NOT EXISTS events (
    seq                    INTEGER PRIMARY KEY AUTOINCREMENT,
    event_id               TEXT    NOT NULL UNIQUE,
    topic                  TEXT    NOT NULL,
    payload                TEXT    NOT NULL,   -- JSON string
    producer               TEXT    NOT NULL,
    published_at           TEXT    NOT NULL,
    acked_at               TEXT,               -- set by POST /events/{id}/ack
    retry_count            INTEGER NOT NULL DEFAULT 0, -- deprecated; use delivery_failure_count
    delivery_failure_count INTEGER NOT NULL DEFAULT 0, -- incremented on each nack
    dlq_requeue_count      INTEGER NOT NULL DEFAULT 0, -- incremented on each DLQ requeue
    dlq_at                 TEXT                        -- set when promoted to DLQ
);

CREATE INDEX IF NOT EXISTS idx_events_topic ON events(topic);
CREATE INDEX IF NOT EXISTS idx_events_seq   ON events(seq);

-- Additive migration for existing databases
ALTER TABLE events ADD COLUMN IF NOT EXISTS delivery_failure_count INTEGER NOT NULL DEFAULT 0;
ALTER TABLE events ADD COLUMN IF NOT EXISTS dlq_requeue_count INTEGER NOT NULL DEFAULT 0;
```

Note: When `_init_schema()` calls `executescript(sql)`, the `ALTER TABLE` statements run after `CREATE TABLE IF NOT EXISTS`. For a new DB, `CREATE TABLE` creates all columns and `ALTER TABLE` no-ops (IF NOT EXISTS). For an existing DB, `CREATE TABLE` no-ops and `ALTER TABLE` adds the columns. This is idempotent.
