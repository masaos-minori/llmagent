# Implementation: Protect all DB operations with threading.Lock

Source plan: `plans/20260625-135650_plan.md` (req #16, app.py portion)

## Goal

Wrap every `asyncio.to_thread` DB callable in `scripts/eventbus/app.py` with `get_db_lock()` so that concurrent threadpool workers cannot access the shared SQLite connection simultaneously.

## Scope

- Apply `get_db_lock()` inside every sync callable passed to `asyncio.to_thread` in: `health()`, `publish()`, `replay()`, `subscribe()`, `dlq_list()`, `dlq_requeue()`, `_dlq_loop()`
- Out of scope: adding the lock itself (req #16 db.py doc), facade function bodies (req #15)

## Assumptions

1. req #14 (asyncio.to_thread wrapping) and req #16 db.py (add threading.Lock) are implemented first
2. `get_db_lock()` is imported from `eventbus.db`
3. All DB callables are already inner sync functions (established in req #14); this change adds `with get_db_lock():` inside each

## Implementation

### Target file

`scripts/eventbus/app.py`

### Procedure

1. Add import: `from eventbus.db import get_db_lock` (or extend existing import)
2. For every inner sync callable that performs DB operations, wrap the body in `with get_db_lock():`

### Method

Each sync callable passed to `asyncio.to_thread` acquires the lock at the start of its body using `with get_db_lock():`. This ensures mutual exclusion without changing the caller structure.

### Details

Pattern applied to every DB callable:
```python
def _check() -> bool:
    with get_db_lock():
        return check_db(_db)  # type: ignore[arg-type]

def _insert() -> tuple[int, bool]:
    with get_db_lock():
        return insert_event(_db, event_id, topic, payload_str, producer, published_at)  # type: ignore[arg-type]

def _fetch() -> list:
    with get_db_lock():
        return fetch_events_since(_db, since_seq)

def _poll() -> list:
    with get_db_lock():
        return fetch_events_since(_db, current_seq, topic or None)

def _list() -> list:
    with get_db_lock():
        return fetch_dlq(_db)

def _requeue() -> bool:
    with get_db_lock():
        return requeue_event(_db, event_id)
```

`_dlq_loop()` wraps the `promote_to_dlq` call with the same pattern. `promote_to_dlq` in `dlq.py` contains its own `db.execute` + `db.commit` calls, so the lock must be held for the entire `promote_to_dlq` duration:
```python
def _promote() -> int:
    with get_db_lock():
        return promote_to_dlq(_db, _cfg.deadletter_dir, _cfg.max_retry)  # type: ignore[arg-type]
n = await asyncio.to_thread(_promote)
```
