# Implementation: agent/services/mcp_install.py — InstallIOPort + role enum validation

## Goal

1. Replace `print()` / `input()` calls in `CliInstallQA` and `print_next_steps()` with
   `InstallIOPort` / `StatusRenderPort` protocol delegation.
2. Validate role in `ask_role()` using an enum; unknown roles raise `ValueError` instead
   of silently falling back to "generic".
3. Move `print_next_steps()` rendering to a `StatusRenderPort` implementor in the command layer.

## Scope

**Target file**: `scripts/agent/services/mcp_install.py`

In scope:
- `CliInstallQA.ask_role()`: `print("Role [...]")` → removed (prompt text goes in `InstallIOPort`)
- `CliInstallQA.ask_role()`: unknown role → raise `ValueError("Unknown role: {raw!r}")`
- `McpInstallService.print_next_steps()` → accept `StatusRenderPort` argument or move to command layer
- `InstallQA` Protocol renamed to `InstallIOPort` to match `io_ports.py` definition

Out of scope:
- `McpInstallService.run()` business logic
- `CliInstallQA.ask_port()` / `ask_confd()` (no print/input issues there)

## Assumptions

1. `InstallIOPort(Protocol)` in `io_ports.py`:
   `async def ask_port(self, default: int) -> int: ...`
   `async def ask_role(self) -> str: ...`
   `async def ask_confd(self) -> bool: ...`
2. `VALID_ROLES = frozenset({"generic", "sqlite", "shell", "git", "ci"})` defines acceptable roles.
3. Unknown role raises `ValueError` — no fallback to "generic".
4. `CliInstallQA` continues to use `input()` for interactive prompts; the `print("Role [...]:")`
   line is removed (the prompt text is part of the `input()` call itself).
5. `McpInstallService.print_next_steps()` is replaced by accepting a callable/port that
   renders next steps; or the method is moved to the command layer entirely.
6. `StatusRenderPort.render_next_steps(result: ScaffoldResult) -> str` is called by
   the command handler to render and print the next steps string.

## Implementation

### Target file

`scripts/agent/services/mcp_install.py`

### Procedure

**Remove `InstallQA` Protocol** — use `InstallIOPort` from `io_ports.py` instead.
Update `McpInstallService.run()` signature to accept `InstallIOPort`.

**Update `CliInstallQA.ask_role()`**:
```python
_VALID_ROLES: frozenset[str] = frozenset({"generic", "sqlite", "shell", "git", "ci"})

async def ask_role(self) -> str:
    if self._role is not None:
        if self._role not in _VALID_ROLES:
            raise ValueError(f"Unknown role: {self._role!r}. Valid: {sorted(_VALID_ROLES)}")
        return self._role
    raw = (await asyncio.to_thread(input, "Role [generic|sqlite|shell|git|ci] (default: generic): ")).strip().lower()
    if not raw:
        return "generic"
    if raw not in _VALID_ROLES:
        raise ValueError(f"Unknown role: {raw!r}. Valid: {sorted(_VALID_ROLES)}")
    return raw
```

**Update `McpInstallService.print_next_steps()`**:
Extract body to return a string (pure rendering), remove all `print()` calls:
```python
def format_next_steps(self, result: ScaffoldResult) -> str:
    """Format post-install checklist as a string (no I/O side effects)."""
    lines: list[str] = ["", "Next steps:"]
    lines.append(f"  1. Edit scripts/mcp/{result.module}/server.py ...")
    # ... build all lines ...
    return "\n".join(lines)
```

The command handler calls `svc.format_next_steps(result)` and prints the result.

### Method

`Edit` tool. Remove the `print()` calls from `print_next_steps()` and rename to `format_next_steps()`.
Update `ask_role()` to validate against `_VALID_ROLES`.

### Details

- Keep `InstallQA` Protocol in the file for backward compatibility, but add a type alias
  pointing to `InstallIOPort`: `InstallQA = InstallIOPort`
- The `format_next_steps()` method returns `str`; the caller (command handler) does `print(svc.format_next_steps(result))`.
- `ast-grep` check for `print($$$)` in this file should return 0 after the change.
