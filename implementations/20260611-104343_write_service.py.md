# Implementation: mcp/file/write_service.py — Atomic write + dry-run silent error fix

## Goal

1. Replace the non-atomic `target.write_text(...)` in `write_file()` with a
   temp-file + `os.replace()` pattern to prevent partial writes.
2. Remove the silent `except (UnicodeDecodeError, PermissionError): pass` in the
   `dry_run` branch of `write_file()`, replacing it with explicit failure.

## Scope

**Target file**: `scripts/mcp/file/write_service.py`
**Test file**: `tests/test_file_write_mcp_service.py`

In scope:
- `WriteFileService.write_file()` atomic write path (non-dry-run)
- `WriteFileService.write_file()` dry-run read-error handling
- Test coverage for both scenarios

Out of scope:
- `edit_file()` and `move_file()` (separate operations)
- `_write_if_changed()` helper (used by `edit_file`; separate concern)

## Assumptions

1. `os.replace()` is atomic on POSIX when source and destination are on the same filesystem.
   Placing the temp file in `target.parent` guarantees this.
2. Temp file name is `.tmp_{target.name}` — must be cleaned up on failure.
3. In the dry-run branch, if reading the existing file fails (`UnicodeDecodeError` or
   `PermissionError`), the correct behavior is to raise `FileValidationError` /
   `FileAuthorizationError` instead of silently producing an empty diff.

## Implementation

### Target file

`scripts/mcp/file/write_service.py`

### Procedure

**Change 1 — atomic write (non-dry-run path)**

Locate the non-dry-run block in `write_file()` (currently lines ~128–139):

**Before:**
```python
        try:
            # Ensure parent directory exists before writing
            target.parent.mkdir(parents=True, exist_ok=True)
            target.write_text(req.content, encoding="utf-8")
        except PermissionError as e:
            raise FileAuthorizationError(str(e))
        except OSError as e:
            logger.error(f"write_file: OS error writing '{target}': {e}")
            raise FileValidationError(str(e))
```

**After:**
```python
        import os  # stdlib; add to module-level imports instead
        tmp = target.parent / f".tmp_{target.name}"
        try:
            target.parent.mkdir(parents=True, exist_ok=True)
            tmp.write_text(req.content, encoding="utf-8")
            os.replace(str(tmp), str(target))
        except PermissionError as e:
            tmp.unlink(missing_ok=True)
            raise FileAuthorizationError(str(e))
        except OSError as e:
            tmp.unlink(missing_ok=True)
            logger.error(f"write_file: OS error writing '{target}': {e}")
            raise FileValidationError(str(e))
```

Add `import os` at module level (already present if needed).

**Change 2 — dry-run read-error handling**

Locate the dry-run block in `write_file()` (lines ~112–128):

**Before:**
```python
            if target.exists():
                try:
                    original = target.read_text(encoding="utf-8")
                    diff, _ = WriteFileService._write_if_changed(
                        target, original, req.content, True,
                    )
                except (UnicodeDecodeError, PermissionError):
                    pass
```

**After:**
```python
            if target.exists():
                try:
                    original = target.read_text(encoding="utf-8")
                    diff, _ = WriteFileService._write_if_changed(
                        target, original, req.content, True,
                    )
                except UnicodeDecodeError:
                    raise FileValidationError("Existing file cannot be decoded as UTF-8")
                except PermissionError as e:
                    raise FileAuthorizationError(str(e))
```

### Method

Direct `Edit` tool. Module-level `import os` may already be present; confirm before adding.

### Details

- The `os` module is part of stdlib; check if already imported at module top.
- `.tmp_{name}` naming convention matches the plan. If `target.name` contains a leading dot
  already, the tmp name becomes `.tmp_.something` — acceptable.
- `tmp.unlink(missing_ok=True)` uses Python 3.8+ `missing_ok` parameter; Python 3.13 in use, no issue.
