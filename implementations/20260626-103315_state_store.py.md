### Details

Expected schema (to confirm):
```sql
CREATE TABLE workflow_approvals (
    approval_id TEXT PRIMARY KEY,
    task_id     TEXT NOT NULL,
    status      TEXT NOT NULL DEFAULT 'pending',  -- pending | approved | rejected
    reason      TEXT,
    created_at  TEXT NOT NULL,
    resolved_at TEXT
);
```

Key invariants to confirm:
- Only one `pending` approval can exist at a time (enforced by `find_latest_pending_approval`).
- `resolve_approval` sets `status` to `approved` or `rejected` and records `reason`.
- `cmd_workflow.py:44` sets `ctx.turn.pending_approval_id = None` and `ctx.workflow.approval_pending = False` after resolution — confirm these mirror the DB state correctly.
