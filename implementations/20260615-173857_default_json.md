# Implementation: config/workflows/default.json

## Goal

Create `config/workflows/default.json` — the workflow definition file that declares the four pipeline stages (plan, execute, verify, retry), per-stage timeouts, and retry policy.

## Scope

**In:**
- `config/workflows/default.json` — new file

**Out:**
- Any Python source changes
- Validation schema file (schema is embedded in `workflow_loader.py`, step 4)

## Assumptions

1. Stages map exactly to `orchestrator.py` handlers: `plan`, `execute`, `verify`, `retry`.
2. `version` field is a semver string; `workflow_engine.py` compares it against `tasks.workflow_version`.
3. Timeout values are in seconds (integer).
4. `max_attempts` is the total number of attempts before the task is halted (including the first attempt).
5. Backoff is `fixed` initially; exponential backoff is a Phase 2 extension.

## Implementation

### Target file

`config/workflows/default.json`

### Procedure

Create the file with the structure below.

### Method

```json
{
  "name": "default",
  "version": "1.0.0",
  "stages": [
    {
      "id": "plan",
      "description": "Prepare RAG context and memory injection",
      "timeout_sec": 30,
      "retryable": false
    },
    {
      "id": "execute",
      "description": "LLM call and tool execution loop",
      "timeout_sec": 120,
      "retryable": true
    },
    {
      "id": "verify",
      "description": "Result validation and loop guard check",
      "timeout_sec": 10,
      "retryable": false
    },
    {
      "id": "retry",
      "description": "Transport error retry gate",
      "timeout_sec": 5,
      "retryable": false
    }
  ],
  "retry_policy": {
    "max_attempts": 3,
    "backoff": "fixed",
    "backoff_sec": 1
  }
}
```

### Details

- `retryable: true` means the `workflow_engine` may re-enter this stage after a failure.
- `retryable: false` stages propagate failure directly to the halt state.
- `max_attempts` covers the `execute` stage only (the only retryable stage in v1.0.0).
- `backoff_sec` is the fixed wait between retries; ignored when `backoff != "fixed"`.
