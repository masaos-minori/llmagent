# Implementation: L-3 — Remove residual `tool_result_store`/Tool-Result-Summarization docs for diagnostics-only partial completions

## Goal

Remove the remaining `tool_result_store` / `/tool show llm_partial_completion` / "Tool Result
Summarization" documentation across three files, aligning docs with the diagnostics-only
(`session_diagnostics`) reality of partial-completion storage, and deleting the no-longer-active
Tool Result Summarization feature description entirely.

### 3. `docs/05_agent_06_tool-execution-and-approval.md`

```markdown
## Tool Result Summarization

When `use_tool_summarize=True` and result length > `tool_summarize_threshold` (default 3000 chars):

1. `summarize_tool_result(text, tool_name, args)` calls the LLM with a summarization prompt
2. Summarized result is stored in `ctx.conv.history` (LLM context)
3. Full result is stored in `ctx.tool_result_store`

### `is_summarized(cfg, text, llm_text, is_error)`

Returns `True` when `llm_text` represents a summarized (not truncated) form of `text`.

Conditions for returning `False`:
- `use_tool_summarize` is disabled or `is_error` is True
- `text` length ≤ `tool_summarize_threshold`
- `llm_text == text` (identical — means truncation, not summarization)

Returns `True` when all above conditions are False and `llm_text != truncated_version`.
```

`### build_preview(tool_name, args)` section and everything after it (the preview-format table,
`TURN_LIMIT_HINT` section) stay untouched, becoming the content that immediately follows whatever
section preceded "Tool Result Summarization".
