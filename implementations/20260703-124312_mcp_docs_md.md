2. Add the three new fields to the table (or JSON example):

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `status` | string | — | `"ok"` when all deps healthy; `"degraded"` otherwise |

**`04_mcp_03_routing_lifecycle_and_execution.md`:**

1. Find the Watchdog section.
2. Replace the description of restart logic with:
   - The watchdog calls `GET /health` on each HTTP-transport server at `mcp_watchdog_interval` seconds.
   - **Restart is gated on `restart_recommended`:**
     - `reachable=False` (no HTTP response): attempt restart if `startup_mode=subprocess` and under `mcp_watchdog_max_restarts`.
     - `reachable=True` and `restart_recommended=true`: attempt restart as above.
     - `reachable=True` and `restart_recommended=false`: no restart; if `operator_action_required=true`, log a WARNING.
   - Externally-managed (non-subprocess) servers: warning-only regardless of probe result.
3. Remove any sentence stating "the watchdog checks only the HTTP status code" and replace with the new behavior.
