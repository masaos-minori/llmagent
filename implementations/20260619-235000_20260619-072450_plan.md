## Implementation Order

Steps should be implemented in order 1-11 (bottom-up dependency):
1. DB schema (step 1) — foundation for all persistence
2. Data models (steps 2, 3, 4) — types flow through the system
3. Audit writers (step 5) — consume DTOs from step 4
4. OTel spans (step 6) — consume workflow_id from step 3
5. LLM runner wiring (step 7) — independent but uses workflow_id
6. Orchestrator (step 8) — central hub; wires steps 1-7 together
7. Logger context (step 9) — consumes workflow_id from step 3
8. Diagnostics (step 10) — consumes from steps 2, 8
9. Validation (step 11) — runs after all above
