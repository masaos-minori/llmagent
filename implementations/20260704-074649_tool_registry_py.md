# Implementation: Add re-export stubs to `scripts/shared/tool_registry.py`

## Goal

Add backward-compatible re-exports of `validate_routing_against_config`,
`validate_routing_against_live`, and `validate_all_routing` from `shared.tool_registry` pointing
to `shared.tool_routing_validation`. This fixes broken imports in `repl_health.py` and
`test_tool_registry.py` which currently import these functions from `shared.tool_registry` where
they are not defined.

## Scope

- In-Scope: Append a re-export block to `scripts/shared/tool_registry.py` importing the three
  validation functions from `shared.tool_routing_validation`.
- Out-of-Scope: No changes to function definitions or `ToolRegistry` / `ToolDefinition` APIs.
  Removing these re-exports is a follow-on step (require-29).

## Assumptions

1. `scripts/shared/tool_routing_validation.py` exists (prerequisite: rename doc `20260704-074648`).
2. `tool_routing_validation.py` does NOT import from `tool_registry.py` at module level (uses
   `TYPE_CHECKING` guard), so no circular import is introduced.
3. `shared → shared` direction is allowed by `.importlinter`.
4. The `# noqa: E402,F401` comment pattern is consistent with project conventions.

## Implementation

### Target file

`scripts/shared/tool_registry.py` (existing)

4. Run `uv run ruff check scripts/shared/tool_registry.py` — expect 0 errors.

### Method

Append to end of `scripts/shared/tool_registry.py`:

```python
# Validation helpers — defined in tool_routing_validation; re-exported here for
# backward compatibility with callers that import them from shared.tool_registry.
```

### Details

- The `# noqa: E402` suppresses "module level import not at top of file" if the append falls
  after other code.
- The `# noqa: F401` suppresses "imported but unused" since these are re-exports.
- `validate_routing_against_config` is used by `repl_health.py:272`.
- All three functions are used by `test_tool_registry.py:17-19`.
