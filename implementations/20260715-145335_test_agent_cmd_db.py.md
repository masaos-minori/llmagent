# Implementation Procedure: test_agent_cmd_db.py

## Goal

Delete `tests/test_agent_cmd_db.py` (974 lines) in full, after migrating the
session-scoped test coverage it holds (health, checkpoint, vacuum, purge,
session-scope, session-half of backward-compat) into `tests/test_agent_cmd_session.py`
as rewritten (not copy-pasted) equivalents targeting `/session ...` instead of
`/db session ...`.

## Scope

### In scope
- Delete the file `tests/test_agent_cmd_db.py` in full (18 test classes:
  `TestCmdDbClean`, `TestCmdDbStats`, `TestCmdDbUnknownSubcommand`, `TestParseFlagInt`,
  `TestParseFlagStr`, `TestDbStats`, `TestDbListUrls`, `TestDbRebuildFts`,
  `TestDbHealth`, `TestDbCheckpoint`, `TestDbVacuum`, `TestDbPurge`, `TestDbRecover`,
  `TestDbHelp`, `TestCmdDbRagScope`, `TestCmdDbSessionScope`, `TestCmdDbBackwardCompat`,
  `TestCmdDbFlatAliasesInvalid`).
- Before deleting: extract and rewrite the **session-relevant** subset into
  `tests/test_agent_cmd_session.py` (see that file's own implementation doc for the
  destination-side procedure): `TestDbHealth` (2 tests), `TestDbCheckpoint` (2),
  `TestDbVacuum` (2), `TestDbPurge` (4), `TestCmdDbSessionScope` (3), and the
  session-only half of `TestCmdDbBackwardCompat` (`test_session_recover_success`/
  `test_session_recover_failure`, 2) — 15 tests total.

### Out of scope
- `TestCmdDbClean`, `TestDbListUrls`, `TestDbRebuildFts`, `TestCmdDbRagScope` —
  these test the already-removed `/db rag` family (confirmed dead code path in
  `cmd_db.py` per that file's own implementation doc — reading the current
  `cmd_db.py` shows no `rag` branch exists in `_cmd_db`'s dispatch, only `session`
  and `help`). These tests are presumably already failing today (pre-existing,
  unrelated to this plan) and are simply deleted along with the rest of the file,
  not migrated anywhere.
- `TestDbStats` / `TestDbHelp` — `_db_stats` is dead code (deleted, not migrated,
  per `db_stats_display.py`'s own implementation doc); `/db help`'s table has no
  required replacement (per this plan's Design step 4 — `/session`'s existing
  unknown-subcommand fallback message suffices).
- `TestParseFlagInt` / `TestParseFlagStr` — see Risks below; these test
  `agent/commands/utils.py::parse_flag_int`/`parse_flag_str`, which have **zero**
  production callers anywhere in `scripts/` (verified via
  `grep -rn "parse_flag_int\|parse_flag_str" scripts/ tests/` — only this test file
  and the function definitions themselves). Deleting this file drops their only test
  coverage. `utils.py` is explicitly out of this plan's scope to touch (see Risks in
  the plan) — leave `parse_flag_int`/`parse_flag_str` in place, un-tested, and note
  the coverage loss in the commit/PR description rather than expanding scope. This is
  a pre-existing dead-code situation (these functions are unused by `DbSessionOps`/
  `cmd_session.py`, which use `parse_command_args` instead), not something introduced
  by this plan.
- `TestCmdDbFlatAliasesInvalid` — tests that already-removed flat `/db <subcmd>`
  aliases (pre-dating even the `/db rag`/`/db session` scoped forms) show a usage
  message; irrelevant once `/db` itself is gone. Deleted, not migrated.

## Assumptions

- Running `uv run pytest tests/test_agent_cmd_db.py -q` today shows 22 pre-existing
  failures (the `/db rag`-scoped tests, broken by the earlier unrelated RAG-removal
  change) and 37 passing tests — confirmed by the plan's own Affected Areas table.
  The baseline for "no new failures" after this plan explicitly excludes those 22
  (they are deleted, not fixed).
- `TestCmdDbSessionScope` (current line 798) and the session half of
  `TestCmdDbBackwardCompat` (current line 840) construct their command object via a
  `_DbMixin`-based test double calling `_cmd_db("session ...")` — the migrated
  equivalents in `test_agent_cmd_session.py` must instead construct a `_SessionMixin`
  test double (matching the existing `_make_cmd()` helper pattern already in that
  file) and call `_cmd_session("...")` directly (no `"session "` prefix, since the
  scope is now implicit).

## Implementation

### Target file

`tests/test_agent_cmd_db.py`

### Procedure

1. Read every test method in `TestDbHealth`, `TestDbCheckpoint`, `TestDbVacuum`,
   `TestDbPurge`, `TestCmdDbSessionScope`, and `TestCmdDbBackwardCompat` (lines
   462-508, 508-553, 553-592, 592-685, 798-840, 840-896 respectively) in full,
   noting exact assertions, mock setups, and edge cases (e.g. `TestDbPurge`'s 4 tests
   likely cover: default no-flags, `--max-sessions` only, `--max-age-days` only, both
   flags together).
2. For each, write a **rewritten** (not copy-pasted) equivalent in
   `tests/test_agent_cmd_session.py` using that file's existing `_make_cmd()` /
   `_make_session()` helper pattern, targeting `cmd._cmd_session("health")`,
   `cmd._cmd_session("checkpoint FULL")`, `cmd._cmd_session("purge --max-sessions 5")`,
   etc., instead of `_cmd_db("session health")` etc. Preserve the same assertions
   (same expected output substrings, same mock call assertions) — the goal is
   behavioral equivalence, not textual equivalence.
3. Keep a scratch copy of the original 6 test classes during development (per the
   plan's Risk mitigation) so old and new tests can be diffed line-by-line and run
   side-by-side once before deleting the source file, to catch any dropped assertion.
4. Once the migrated tests are green in `tests/test_agent_cmd_session.py`, delete
   `tests/test_agent_cmd_db.py` in full: `git rm tests/test_agent_cmd_db.py`.

### Method

Manual read-and-rewrite (not a mechanical copy-paste) of 15 test functions across 6
classes, followed by whole-file deletion of the 974-line source file. This must
happen **after** `cmd_session.py`'s new branches are implemented (this test file's
replacement tests depend on that code existing) and can happen **before or
concurrently with** `cmd_db.py`'s deletion (since `cmd_db.py`'s deletion is what
makes this old test file's imports break anyway).

### Details

- `TestCmdDbUnknownSubcommand` (current line 248) — check whether this tests `/db`'s
  top-level unknown-subcommand behavior (distinct from `/db session`'s) — if so, it
  has no direct `/session`-side equivalent to migrate (the plan does not require one;
  `/session`'s own unknown-subcommand fallback is already covered by
  `TestCmdSessionUsage::test_unknown_subcommand_shows_usage` in
  `test_agent_cmd_session.py`, which is unrelated pre-existing coverage).
- Confirm the exact line ranges for each of the 6 classes to be migrated by
  re-reading the file just before extraction (line numbers given here are from the
  `grep -n "^class "` output taken at design time and may shift slightly if earlier
  classes in the file are edited first).

  — confirm the migrated tests in `test_agent_cmd_session.py` cover the new

- Note in the commit/PR description: `parse_flag_int`/`parse_flag_str` in

  unchanged per this plan's explicit scope boundary.
