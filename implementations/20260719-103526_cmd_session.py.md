## Goal

Add a `/session rag-rebuild-fts` subcommand to `scripts/agent/commands/cmd_session.py`, giving
operators an on-demand replacement for the removed `/db rag rebuild-fts` command, backed by the
existing `RagMaintenanceService().rebuild_fts()` (already used internally as a delete-all +
re-insert rebuild of `chunks_fts`). This is a thin dispatch + print wrapper, the same shape as the
sibling `_rag_consistency` addition documented in `implementations/20260719-102923_cmd_session.py.md`
(source plan `plans/20260719-091140_plan.md`).

**Builds on prior doc**: `implementations/20260719-102923_cmd_session.py.md` already specifies
adding a module-level `RagMaintenanceService` import, a `_rag_consistency` method, a
`"rag-consistency": self._rag_consistency` `db_dispatch` entry, and `|rag-consistency` appended to
the docstring and `write_validation_error` usage strings. This document's Procedure assumes those
changes are already applied and adds `_rag_rebuild_fts` / `"rag-rebuild-fts"` / `|rag-rebuild-fts`
alongside them — it does not repeat or re-specify the `rag-consistency` changes.

## Scope

**In scope**
- Add a `_rag_rebuild_fts` method to the class containing `_cmd_session` that calls
  `RagMaintenanceService().rebuild_fts()` and prints a success message.
- Add `"rag-rebuild-fts": self._rag_rebuild_fts` to the `db_dispatch` dict inside `_cmd_session`.
- Update the `_cmd_session` docstring and the `write_validation_error` fallback usage string to
  include `rag-rebuild-fts`.

**Out of scope**
- The `RagMaintenanceService` import itself, the `_rag_consistency` method, and the
  `"rag-consistency"` dispatch entry — all added by the prior doc
  (`implementations/20260719-102923_cmd_session.py.md`); not repeated here. If, at implementation
  time, that prior doc's changes have NOT yet been applied, the implementer must apply them first
  (per that doc) before applying this one, per plan Assumption 1 / Implementation step 1's branch.
- Any change to `RagMaintenanceService.rebuild_fts()` itself
  (`scripts/agent/services/rag_maintenance_service.py:31-44`) — reused as-is, verified below.
- The module-level docstring comment at `cmd_session.py:7` — same reasoning as the prior doc's
  Assumption 4 (a comment, not a runtime usage string; optional, not required).

## Implementation

### Target file

`scripts/agent/commands/cmd_session.py`.

### Procedure

Performed *after* `implementations/20260719-102923_cmd_session.py.md`'s changes are applied (i.e.
after the `RagMaintenanceService` import, `_rag_consistency` method, and `"rag-consistency"`
dispatch/usage-string entries already exist in the file):

1. Add a new `_rag_rebuild_fts` method directly after `_rag_consistency`:
   ```python
   def _rag_rebuild_fts(self) -> None:
       """Rebuild the RAG FTS5 index (chunks_fts) from current chunk content."""
       RagMaintenanceService().rebuild_fts()
       self._out.write_success("RAG FTS index rebuilt. [RAG]")
   ```
2. In the `db_dispatch` dict, add one line directly after `"rag-consistency": self._rag_consistency,`:
   `"rag-rebuild-fts": self._rag_rebuild_fts,`.
3. Update the `_cmd_session` docstring to append `|rag-rebuild-fts` after `rag-consistency`, e.g.
   `... | stats|health|checkpoint|vacuum|purge|recover|rag-consistency|rag-rebuild-fts.`
4. Update the `write_validation_error` fallback message the same way, appending
   `|rag-rebuild-fts` to the final line's subcommand list.
5. Run `uv run ruff format scripts/agent/commands/cmd_session.py` and
   `uv run ruff check scripts/agent/commands/cmd_session.py --fix`.

Plain method addition + one dict entry + two string edits, identical in shape to the prior doc's
`_rag_consistency` addition. No new abstractions, no signature changes to any existing method.
