# Implementation: Improve extraction quality in extract.py

Steps covered: Plan 20260626-100122 — Steps 1-5

---

## Goal

Strengthen `_classify_content`, `_SEMANTIC_KEYWORDS`, `_EPISODIC_FAILURE_KEYWORDS`, and `_importance_from_content` in `scripts/agent/memory/extract.py` to more accurately distinguish RULE / DECISION / FAILURE / CONVERSATION source types.

---

## Scope

- **In scope**: `scripts/agent/memory/extract.py`, `tests/agent/memory/test_extract.py`
- **Out of scope**: LLM-based extraction; storage model changes

---

## Assumptions

- `SourceType.DECISION`, `RULE`, `FAILURE`, `CONVERSATION` exist in `enums.py`.
- DECISION: "rationale", "trade-off", "chose", "opted for", "decided", "decision", "tradeoff".
- FAILURE: "root cause", "mitigation", "workaround", "regression", "deadlock", "memory leak".
- `_importance_from_content`: DECISION base = 0.55; FAILURE base = 0.45.

---

## Implementation

### Target files
`scripts/agent/memory/extract.py`, `tests/agent/memory/test_extract.py`

### Procedure
1. Read `scripts/agent/memory/extract.py` to find `_classify_content`, `_SEMANTIC_KEYWORDS`, `_EPISODIC_FAILURE_KEYWORDS`.
2. Step 1 (Extend keywords):
   ```python
   _DECISION_KEYWORDS = re.compile(
       r'\b(rationale|trade.off|chose|opted|decided|decision|tradeoff)\b', re.IGNORECASE
   )
   _SEMANTIC_KEYWORDS = re.compile(
       r'\b(mandatory|enforce|prohibit|forbidden|must|always|never|rule|policy)\b', re.IGNORECASE
   )
   _EPISODIC_FAILURE_KEYWORDS = re.compile(
       r'\b(root.cause|mitigation|workaround|regression|deadlock|memory.leak|bug|error|failed)\b',
       re.IGNORECASE,
   )
   ```
3. Step 2 (`_classify_content`): Add DECISION priority:
   ```python
   if _DECISION_KEYWORDS.search(text) and _SEMANTIC_KEYWORDS.search(text):
       return SourceType.DECISION
   elif _SEMANTIC_KEYWORDS.search(text):
       return SourceType.RULE
   elif _EPISODIC_FAILURE_KEYWORDS.search(text):
       return SourceType.FAILURE
   else:
       return SourceType.CONVERSATION
   ```
4. Step 3 (Failure keywords already handled above).
5. Step 4 (`_importance_from_content`): Add source_type-based branching.
6. Step 5 (Tests): Add 4 classification tests (RULE, DECISION, FAILURE, CONVERSATION).

### Method
Regex extension + classification logic update. No ML changes.
