### `_sync_system_prompt()` へのエフェメラル除去ロジック追加

`_sync_system_prompt()` の先頭にエフェメラルエントリを history から除去する処理を追加する:

**変更前 (lines 374-384):**
```python
    def _sync_system_prompt(self) -> None:
        """Sync history[0] from ctx.conv.system_prompt_content before each turn."""
        ctx = self._ctx
        if not ctx.conv.system_prompt_content:
            return
        if ctx.conv.history and ctx.conv.history[0]["role"] == "system":
            ctx.conv.history[0]["content"] = ctx.conv.system_prompt_content
        else:
            ctx.conv.history.insert(
                0, {"role": "system", "content": ctx.conv.system_prompt_content}
            )
```

**変更後:**
```python
    def _sync_system_prompt(self) -> None:
        """Sync history[0] from ctx.conv.system_prompt_content before each turn.

        Also removes ephemeral system messages injected in the previous turn.
        """
        ctx = self._ctx
        # Remove ephemeral entries from the previous turn before rebuilding prompt
        ctx.conv.history = [
            m for m in ctx.conv.history if not m.get("_ephemeral")  # type: ignore[typeddict-item]
        ]
        if not ctx.conv.system_prompt_content:
            return
        if ctx.conv.history and ctx.conv.history[0]["role"] == "system":
            ctx.conv.history[0]["content"] = ctx.conv.system_prompt_content
        else:
            ctx.conv.history.insert(
                0, {"role": "system", "content": ctx.conv.system_prompt_content}
            )
```

## 変更 3: `_append_user_message()` — asyncio.create_task タスク参照保持

### 変更箇所: line 413

`asyncio.create_task()` の戻り値を参照しないと GC に回収される可能性がある。

**前提:** クラスに `_background_tasks: set[asyncio.Task[object]]` フィールドを追加する。

**クラス `__init__` への追加:**
```python
        self._background_tasks: set[asyncio.Task[object]] = set()
```

**変更前:**
```python
        if ctx.stats.stat_turns == 1 and self._on_first_turn is not None:
            asyncio.create_task(self._on_first_turn(line))
```

**変更後:**
```python
        if ctx.stats.stat_turns == 1 and self._on_first_turn is not None:
            _task = asyncio.create_task(self._on_first_turn(line))
            self._background_tasks.add(_task)
            _task.add_done_callback(self._background_tasks.discard)
```

## 変更 5: `_handle_llm_turn()` — 二重 `_handle_llm_transport_error` パスの整理

### 現状 (lines 290-304)

```python
                if result.exception is not None:
                    # run() caught LLMTransportError internally; propagate callbacks
                    self._handle_llm_transport_error(result.exception, ctx)
                    if self._on_error:
                        self._on_error(result.exception)
                ...
        except LLMTransportError as e:
            # Reached when run() is mocked with side_effect=e (tests) or re-raises
            self._handle_llm_transport_error(e, ctx)
            if self._on_error:
                self._on_error(e)
            return TurnResult(action="fail", answer="", error_kind=str(e), exception=e)
```

### 問題

`result.exception is not None` パスと `except LLMTransportError` パスが重複。テスト時に `mock.side_effect=e` で `except` に到達するのは既存テストの都合によるものだが、production では `run()` が `TurnResult(exception=e)` を返す。

### 修正方針

`_llm_runner.run()` が常に `TurnResult` を返す (raise しない) ことを前提として `except LLMTransportError` ブロックを削除し、`result.exception` パスに一本化する。

**変更後:**
```python
            with self._llm_runner._span_ctx("llm") as llm_span:
                llm_span.set_attribute("model_url", llm_url)
                result = await self._llm_runner.run(llm_url)
                logger.info("LLM response: %s", result.answer)
                ctx.session.save("assistant", result.answer)
                if result.exception is not None:
                    self._handle_llm_transport_error(result.exception, ctx)
                    if self._on_error:
                        self._on_error(result.exception)
                else:
                    if self._on_turn_end:
                        self._on_turn_end()
                return result
```

`except LLMTransportError` ブロックを削除する。テストは `run()` が `TurnResult(exception=e)` を返すように修正する。

**注意:** `_llm_runner.run()` の実装を確認し、実際に raise しないことを確認してから実施する (`grep -n "raise\|LLMTransportError" scripts/agent/llm_turn_runner.py`)。
