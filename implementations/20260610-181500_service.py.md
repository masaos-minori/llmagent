# Implementation: mcp/github/service.py — Config Injection + Fail-Fast (HTTPException removal)

## Goal

Remove `_get_cfg()` references and `HTTPException` from `service.py`. Inject `GitHubConfig` via constructor. Replace all `HTTPException` raises with domain exceptions defined in Step 1. Redesign `_handle_github_error()` to raise domain exceptions instead of `HTTPException`.

## Assumptions

1. `_assert_allowed_path` and `_assert_allowed_branch` are currently `@staticmethod` — they call `_get_cfg()`. After change, they become instance methods (or accept `cfg` parameter). Instance methods is simpler.
2. `_assert_max_file_size` is a staticmethod that calls `_get_cfg()` — becomes instance method.
3. `_write_github_audit_log` is a staticmethod that calls `_get_cfg()` — becomes instance method.
4. `_LazyGitHubService` lazily creates `GitHubService`; it should call `GitHubConfig.load()` at first access. Config load failure raises immediately (no `{}` fallback).
5. Tests that monkeypatch `_get_cfg` will break; they will be fixed in Step 6.

## Implementation

### Target file
`scripts/mcp/github/service.py`

### Procedure

1. Replace `from fastapi import HTTPException` with domain exception imports.
2. Remove `_get_cfg` from the models import list.
3. Add `GitHubConfig` to the models import list.
4. Update `GitHubService.__init__` signature: `def __init__(self, gh: Github, cfg: GitHubConfig) -> None`.
5. Convert static methods that reference `_get_cfg()` to instance methods referencing `self._cfg`.
6. Replace each `HTTPException(status_code=403, ...)` with `GitHubAuthorizationError(detail)`.
7. Replace each `HTTPException(status_code=404, ...)` with `GitHubNotFoundError(detail)`.
8. Replace each `HTTPException(status_code=400, ...)` with `GitHubValidationError(detail)`.
9. Replace each `HTTPException(status_code=409, ...)` with `GitHubConflictError(detail)`.
10. Replace each `HTTPException(status_code=502, ...)` with `GitHubUpstreamError(detail)`.
11. Rewrite `_handle_github_error` to map GithubException.status to domain exceptions.
12. Rewrite `_write_github_audit_log`: if `self._cfg.audit_log_path` is empty, return; on OSError raise `GitHubAuditError`.
13. Update `_LazyGitHubService._instance` creation: `GitHubService(_gh, GitHubConfig.load())`.

### Method

```python
# Updated _handle_github_error
@staticmethod
def _handle_github_error(e: GithubException) -> NoReturn:
    if e.status == 404:
        raise GitHubNotFoundError("Resource not found")
    if e.status == 403:
        raise GitHubAuthorizationError(
            "GitHub API rate limit exceeded or access denied"
        )
    if e.status == 409:
        raise GitHubConflictError(f"GitHub API conflict (status={e.status})")
    if e.status in (400, 422):
        raise GitHubValidationError(f"GitHub API validation error (status={e.status})")
    raise GitHubUpstreamError(f"GitHub API error (status={e.status})")

# Updated _write_github_audit_log
def _write_github_audit_log(self, op: str, **kwargs: Any) -> None:
    if not self._cfg.audit_log_path:
        return
    ts = datetime.now(tz=UTC).isoformat()
    fields = " ".join(f"{k}={v!r}" for k, v in kwargs.items())
    record = f"{ts} op={op} {fields}\n"
    try:
        with open(self._cfg.audit_log_path, "a", encoding="utf-8") as fh:
            fh.write(record)
    except OSError as e:
        raise GitHubAuditError(
            f"Audit log write failed ({self._cfg.audit_log_path}): {e}"
        ) from e

# Updated __init__
def __init__(self, gh: Github, cfg: GitHubConfig) -> None:
    self._gh = gh
    self._cfg = cfg
    self._default_per_page = cfg.default_per_page
    self._max_per_page = cfg.max_per_page

# Updated _LazyGitHubService
class _LazyGitHubService:
    _instance: GitHubService | None = None
    def __getattr__(self, name: str) -> Any:
        if _LazyGitHubService._instance is None:
            _LazyGitHubService._instance = GitHubService(_gh, GitHubConfig.load())
        return getattr(_LazyGitHubService._instance, name)
```

### Details

- `_assert_allowed_repo` currently uses `mode = _get_cfg().get("allowed_repos_mode", "fail_closed")`. Default in `GitHubConfig` is `"fail_closed"` — behavior unchanged.
- `_assert_allowed_path`, `_assert_allowed_branch` were `@staticmethod` — changing to instance methods requires removing the decorator and adding `self` parameter. All callers in the same class pass `self.*` so this is safe.
- The `_clamp_per_page` usage `self._max_per_page` already comes from constructor — no change.
