# Implementation: scripts/db/schema_sql.py

## Goal

Add `REFERENCES sessions(session_id) ON DELETE CASCADE` FK constraint to `tool_results.session_id` in the DDL so new databases are created with the FK from the start.

## Scope

- Target: `scripts/db/schema_sql.py`
- Change `tool_results.session_id INTEGER` to `tool_results.session_id INTEGER REFERENCES sessions(session_id) ON DELETE CASCADE`
- No other tables or files changed

## Assumptions

1. `sessions` table is defined in the same schema and its DDL appears before `tool_results` in the `executescript` call (FK target must exist).
2. `session_id` remains `INTEGER` (nullable) — SQLite excludes NULL values from FK enforcement, so existing `store(session_id=None)` callers are unaffected.
3. This DDL change only affects new installations; existing databases need the migration in `create_schema.py` (separate plan).

## Implementation

### Target file
`scripts/db/schema_sql.py`

### Procedure
1. Locate the `tool_results` DDL block (around lines 86-98).
2. Replace `session_id INTEGER,` with `session_id INTEGER REFERENCES sessions(session_id) ON DELETE CASCADE,`.

### Method

```sql
-- Before
CREATE TABLE IF NOT EXISTS tool_results (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER,
    ...
);

-- After
CREATE TABLE IF NOT EXISTS tool_results (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER REFERENCES sessions(session_id) ON DELETE CASCADE,
    ...
);
```

### Details
- Only the `session_id` line changes — all other columns and indexes are unchanged.
- `CREATE TABLE IF NOT EXISTS` means this DDL only runs on fresh databases; migration handles existing ones.
