# Implementation: Add `round_exec` event type to tool_audit.py

## Goal

Add `write_round_exec()` function to `tool_audit.py` so that round-wide serialization events can be logged to the audit trail with timing and trigger-tool metadata.

## Scope

- `scripts/agent/tool_audit.py` — add `write_round_exec()` function

**Out of scope:** Instrumentation in tool_runner.py (separate doc), cmd_audit.py changes (events appear in tail naturally).

## Assumptions

1. The audit logger follows the same guard pattern (`if ctx.services.audit_logger is None: return`) — confirmed.
2. No new dataclass is needed — the plan specifies a raw dict like `audit_stage_completed`.
3. `round_id` can be derived from existing context (turn number or a UUID).

## Implementation

### Target file

`scripts/agent/tool_audit.py`

### Procedure

Add `write_round_exec()` function following the existing `audit_stage_completed()` pattern.

### Method

Single function addition.

### Details

Add after `audit_tool_exec()` (line ~135):

```python
def write_round_exec(
    ctx: AgentContext,
    *,
    round_id: str,
    tool_count: int,
    mode: str,
    has_side_effect: bool,
    trigger_tool: str | None,
    elapsed_ms: float,
) -> None:
    """Log a round-wide execution event, capturing serialization impact."""
    if ctx.services.audit_logger is None:
        return
    ctx.services.audit_logger.info(
        "round_exec",
        extra={
            "round_id": round_id,
            "tool_count": tool_count,
            "mode": mode,
            "has_side_effect": has_side_effect,
            "trigger_tool": trigger_tool,
            "elapsed_ms": round(elapsed_ms, 1),
        },
    )
```
