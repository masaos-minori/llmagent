# Implementation: Track Per-Session Serialization Metrics

## Goal

Accumulate per-session serialization counts in `AgentContext` or `ToolExecutor` so operators can query total serialization events and tools-affected counts without parsing logs.

## Scope

- `scripts/agent/tool_runner.py` ‚Äî increment session-level counters when serialization is detected in `_execute_with_dag()` and `_execute_standard()`
- Counter storage: `ctx.services` or a new lightweight field in `AgentContext`

Out of scope:
- Persisting counters to DB
- Resetting counters mid-session
- Per-tool-name breakdown (aggregate only)

## Assumptions

1. `AgentContext` (or `ctx.services`) is the correct place for session-scoped counters ‚Äî it is passed to all tool execution paths.
2. Two counters are sufficient:
   - `serialization_events: int` ‚Äî total number of rounds that ran serially due to side effects or write conflicts
   - `serialization_tools_affected: int` ‚Äî total number of tool calls that ran serially (summed across all serialization events)
3. `ctx.services` may not have these fields currently; they can be added as a simple counter dict or as direct int attributes.
4. The counters are readable via `/mcp status` after Step 3 is implemented.

## Implementation

### Target file

`scripts/agent/tool_runner.py`

### Procedure

1. Identify where `ctx.services` or `AgentContext` counters live; add `serialization_events: int = 0` and `serialization_tools_affected: int = 0`.
2. In `_execute_with_dag()`, after detecting serial groups, increment counters.
3. In `_execute_standard()`, when `use_serial=True` due to side effects, increment counters.

### Method

Simple integer increment. No new data structures; piggyback on existing `ctx.services` or add to a `stats` dict if one exists.

### Details

**Counter definition ‚Äî check if `ctx.services` has a stats field. If not, add to a simple location:**

If `ctx.services` is a dataclass, add:
```python
serialization_events: int = 0
serialization_tools_affected: int = 0
```

If adding to dataclass is not feasible, use a `defaultdict` or plain dict on an existing object.

**tool_runner.py ‚Äî `_execute_with_dag()` (after line 178-184):**
```python
serial_groups = [g for g in groups if len(g) < len(approved_calls)]
# Count groups that are NOT the final parallel group as serialization events.
# More precise: count groups that serialize multiple tools together.
serial_tool_count = sum(len(g) for g in groups if _is_serial_group(g, tool_meta))
if serial_tool_count > 0:
    ctx.services.serialization_events += 1
    ctx.services.serialization_tools_affected += serial_tool_count
```

Helper `_is_serial_group(group, tool_meta)` returns True if the group is a serial barrier or write-conflict group (not a pure parallel group).

**tool_runner.py ‚Äî `_execute_standard()` (after line 203-210):**
```python
if use_serial and has_side_effect:
    ctx.services.serialization_events += 1
    ctx.services.serialization_tools_affected += len(approved_calls)
```
