# Implementation: orchestrator.py

## Goal

Remove the two `except Exception` handlers in `orchestrator.py` so that unclassified errors propagate to the REPL rather than being silently converted to `TurnResult(success=False)`.

## Scope

- Target: `scripts/agent/orchestrator.py`
- Remove `except Exception` in `_handle_llm_turn` (line ~152)
- Remove `except Exception` in `_handle_llm_transport_error` (line ~260, `ToolResultStore.store`)
- `TurnResult` keeps its current shape but `error_kind` becomes `str | None` only for `LLMTransportError`; unknown exceptions are no longer captured
- `_handle_llm_turn` only catches `LLMTransportError` explicitly; all other exceptions propagate
- Update `repl.py._run_repl_loop` to handle exceptions from `handle_turn`

## Assumptions

1. `repl.py._repl_loop` calls `handle_turn(line)` without catching; after this change it must wrap in `try/except` for display.
2. `LLMTransportError` is the only classified failure type; other failures are bugs or infrastructure failures that should surface.
3. `ctx.tool_result_store.store()` failure in `_handle_llm_transport_error` is a persistence side-effect; losing it is acceptable â€” log warning and continue (no silent swallowing via `except Exception`).

## Implementation

### Target file

`scripts/agent/orchestrator.py`

### Procedure

1. In `_handle_llm_turn`, remove the `except Exception as e:` block (keep only `except LLMTransportError as e:`).
2. In `_handle_llm_transport_error`, replace `except Exception:` around `tool_result_store.store()` with `except (RuntimeError, OSError) as e: logger.warning(...)` â€” i.e. catch only known DB/persistence errors.
3. In `repl.py._repl_loop`, add `except Exception as e: self._view.write_llm_error(e)` around `await self._orchestrator.handle_turn(line)` so unexpected errors are displayed without crashing the REPL loop.
4. Remove `Any` import from orchestrator.py if `on_first_turn` callback type can be narrowed to `Callable[[str], Coroutine[Any, Any, None]] | None`.

### Method

Targeted exception handler removal. The REPL becomes the last-resort catch boundary.

### Details

```python
# _handle_llm_turn â€” BEFORE
except Exception as e:
    self._handle_general_llm_error(e, ctx)
    ...
    return TurnResult(success=False, answer="", error_kind=str(e))

# _handle_llm_turn â€” AFTER
# (removed â€” exception propagates to _process_turn â†’ handle_turn â†’ repl._repl_loop)
```

```python
# repl.py._repl_loop â€” addition

    await self._orchestrator.handle_turn(line)
except Exception as e:
    self._view.write_llm_error(e)
```

Remove `_handle_general_llm_error` method if it is no longer called after the `except Exception` removal.
