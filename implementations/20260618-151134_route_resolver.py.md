# Goal

Three changes to `scripts/shared/route_resolver.py`:
1. Remove `_EXACT_ROUTES` — move `shell_run` and `search_web` to `_SET_ROUTES` using new `SHELL_TOOLS`/`WEB_SEARCH_TOOLS` constants.
2. Add `known_tools` optional parameter to `ToolRouteResolver.__init__` for startup validation.
3. Log routing coverage summary at `__init__` time when `known_tools` is provided.

# Scope

- **File:** `scripts/shared/route_resolver.py`
- **Changes:**
  - Add `SHELL_TOOLS`, `WEB_SEARCH_TOOLS` to imports from `tool_constants`
  - Append 2 entries to `_SET_ROUTES`
  - Delete `_EXACT_ROUTES` dict
  - Remove `_EXACT_ROUTES` lookup from `_fallback_route()`
  - Add `known_tools: frozenset[str] | None = None` param to `__init__`
  - Add `_log_routing_coverage()` private method
- **Out of scope:** `build_discovery_map()`, `_GITHUB_PREFIX`, existing `__init__` params

# Assumptions

1. `SHELL_TOOLS = frozenset({"shell_run"})` and `WEB_SEARCH_TOOLS = frozenset({"search_web"})` are already added to `tool_constants.py` (prerequisite).
2. `_SET_ROUTES` search is O(n×m) — with 10 entries and 50 tools this is negligible; no performance concern.
3. `known_tools` is optional — callers that don't pass it get identical behavior to current code.
4. Coverage log uses `logger.info` (not `logger.debug`) so it appears in startup output when log level is INFO.
5. Tests for `_EXACT_ROUTES` removal: `test_shell_run` and `test_search_web` must still pass (behavior unchanged).

### 4. Add `known_tools` param and `_log_routing_coverage()`

```python
def __init__(
    self,
    server_configs: dict[str, McpServerConfig],
    *,
    discovery_map: dict[str, str] | None = None,
    warn_on_fallback: bool = False,
    strict_mode: bool = False,
    known_tools: frozenset[str] | None = None,
) -> None:
    self._discovery_map: dict[str, str] = discovery_map or {}
    self._config_map: dict[str, str] = {}
    for key, cfg in server_configs.items():
        for tool_name in cfg.tool_names:
            self._config_map[tool_name] = key
    self._warn_on_fallback = warn_on_fallback
    self._strict_mode = strict_mode
    if known_tools:
        self._log_routing_coverage(known_tools)

def _log_routing_coverage(self, known_tools: frozenset[str]) -> None:
    """Log routing coverage for all known tools at startup."""
    mapped: list[str] = []
    unmapped: list[str] = []
    for tool_name in sorted(known_tools):
        try:
            self.resolve(tool_name)
            mapped.append(tool_name)
        except ValueError:
            unmapped.append(tool_name)
    total = len(known_tools)
    if unmapped:
        logger.warning(
            "Routing: %d/%d tools mapped; %d unmapped: %s",
            len(mapped),
            total,
            len(unmapped),
            unmapped,
        )
    else:
        logger.info("Routing: %d/%d tools mapped", total, total)
```

## Details

- `_log_routing_coverage` calls `self.resolve()` which respects `strict_mode` and `warn_on_fallback`. Pass `warn_on_fallback=False` (default) to suppress noisy fallback warnings during the coverage scan — or call `_fallback_route` directly to avoid double-warning. Better: call `_fallback_route` directly in `_log_routing_coverage` to avoid side effects.
- Docstring update: remove `_EXACT_ROUTES` reference from class docstring "Static fallback (SET_ROUTES, EXACT_ROUTES, prefix matching)".
