## Goal

Add three minimal, additive frozensets — `RAG_WRITE_TOOLS`, `CICD_WRITE_TOOLS`,
`MDQ_WRITE_TOOLS` — isolating the tools that must bypass `ToolExecutor`
caching, so that `is_side_effect()` in `tool_executor_helpers.py` can classify
them as side-effecting without redefining the existing flat `RAG_TOOLS` /
`CICD_TOOLS` / `MDQ_TOOLS` unions (whose exact membership and count are
pinned by other tests).

`MDQ_WRITE_TOOLS` covers `fts_rebuild`, `index_paths`, and `refresh_index`.
The requirement (`requires/done/20260715_20_require.md`) named only
`fts_rebuild`; `index_paths`/`refresh_index` were added per the resolution of
the follow-up issue `issues/20260715-141104_risks.md`, which found that both
tools already write to `mdq.sqlite` (`scripts/mcp_servers/mdq/indexer.py`)
and are already self-classified as write tools locally
(`scripts/mcp_servers/mdq/tools.py:177`'s `_WRITE_TOOLS`, plus
`is_write=True`/`requires_serial=True` in their tool schemas) — leaving them
out of `shared/tool_constants.py`'s `MDQ_WRITE_TOOLS` would have left that
inconsistency in place. The same issue found `rag_run_pipeline`/
`rag_debug_pipeline` do **not** warrant inclusion: both are read-only
retrieval pipelines with no database writes (confirmed in
`scripts/rag/pipeline.py` and `scripts/mcp_servers/rag_pipeline/service.py`),
so they are intentionally left out of `RAG_WRITE_TOOLS` and remain cacheable.
