# Implementation: H-10 — config_builders.py remove wiring + add rejection

Source plan: `plans/20260709-100244_plan.md` (H-10, Implementation step 2).

## Goal

Stop constructing `MCPConfig.github_server_url`, and reject any config that
still contains the removed `github_server_url` key with an actionable error
message.

## Scope

**Target**: `scripts/agent/config_builders.py`, `build_agent_config()`
(lines 253-292).

## Assumptions

1. `build_agent_config()` already has a `_FORBIDDEN_KEYS` mechanism (lines
   260-271) that raises `ConfigLoadError` for other removed keys
   (`workflow_mode`, `workflow_require_approval`, `use_tool_summarize`,
   `tool_summarize_threshold`) with the message
   `f"Config contains removed keys: {sorted(found)}. Remove them from your
   config file."` — this is the established precedent; `github_server_url`
   gets its own dedicated check (not folded into `_FORBIDDEN_KEYS`) so the
   more specific, actionable replacement guidance
   ("Use [mcp_servers.github].url instead") is preserved verbatim.

## Implementation

### Target file

`scripts/agent/config_builders.py`

### Procedure

#### Step 1: Add the dedicated rejection check

After the existing `_FORBIDDEN_KEYS` check (around line 271), add:

```python
    if "github_server_url" in cfg:
        raise ConfigLoadError(
            "Config contains removed key: github_server_url. "
            "Use [mcp_servers.github].url instead."
        )
```

#### Step 2: Remove the field construction

In the `MCPConfig(...)` construction (around line 282-291), delete:
```python
            github_server_url=cfg.get("github_server_url", "http://127.0.0.1:8006"),
```

### Method

- Two small edits in the same function; do both in one pass since the
  second depends on
  `implementations/20260709-103716_config_dataclasses.py.md` having removed
  the field (otherwise this edit alone would leave the field at its
  dataclass default forever, silently, rather than removed).

| Full suite (after all H-10 docs land) | `uv run pytest tests/test_config_builders.py -v` | all pass |
