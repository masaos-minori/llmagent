# Implementation: Graceful SIGTERM Shutdown with Turn-Completion Wait and WAL Checkpoint

## Goal

Replace the immediate `raise SystemExit(0)` SIGTERM handler with a graceful shutdown that:
1. Waits for the current turn to complete (up to configurable timeout, default 10s)
2. Runs WAL checkpoint before exit
3. Falls back to forced exit after timeout

## Scope

**In:**
- `scripts/agent/__main__.py` — remove remaining SIGTERM handler remnants
- `scripts/agent/repl.py` — add asyncio-native signal handler via `loop.add_signal_handler()`; add turn-active tracking; add WAL checkpoint to `_close_resources()`

**Out:**
- Configurable timeout (hardcoded 10s default)
- Changing tool execution semantics

## Assumptions

1. `loop.add_signal_handler()` is the correct pattern for asyncio signal handling
2. The 10s timeout covers most normal turns; long-running operations may be aborted
3. WAL checkpoint: `checkpoint_wal(db, "TRUNCATE")` on session.sqlite is sufficient
4. `_close_resources()` in `repl.py` already calls `svc.lifecycle.shutdown_all()` and `svc.http.aclose()` — WAL checkpoint runs before these

## Implementation

### Target file: `scripts/agent/__main__.py`

**Procedure:** Remove the SIGTERM handler that was partially removed (the function definition may remain but is no longer called).

**Method:** Delete `_request_shutdown` function and any remaining `signal.signal(signal.SIGTERM, _request_shutdown)` call.

**Details:**
- The SIGTERM handler was already removed from `__main__.py` in a prior commit (no `signal` or `SIGTERM` references remain)
- No changes needed here — signal registration moves to `repl.py` where the event loop is running

### Target file: `scripts/agent/repl.py`

**Procedure:** Add asyncio-native signal handler, turn tracking, and WAL checkpoint.

**Method:** Modify `AgentREPL.__init__()`, `run()`, `_repl_loop()`, and `_close_resources()`.

**Details:**

1. **`__init__()` — add fields:**
```python
self._turn_active: bool = False
self._shutdown_event: asyncio.Event | None = None
```

2. **`run()` — add signal handler after loop starts:**
```python
async def run(self) -> None:
    loop = asyncio.get_running_loop()
    self._shutdown_event = asyncio.Event()

    def _sigterm_handler() -> None:
        self._ctx.conv.shutdown_requested = True
        if self._shutdown_event is not None:
            self._shutdown_event.set()
        logger.info("SIGTERM received; graceful shutdown initiated")

    try:
        loop.add_signal_handler(signal.SIGTERM, _sigterm_handler)
    except NotImplementedError:
        import signal as _signal
        _signal.signal(_signal.SIGTERM, lambda *_: _sigterm_handler())

    from agent.startup import StartupOrchestrator
    startup = StartupOrchestrator(self._ctx, self._view)
    self._cmds, self._orchestrator = await startup.run()
    await self._run_repl_loop()
```

3. **`_repl_loop()` — wrap dispatch with turn tracking:**
```python
async def _repl_loop(self) -> None:
    ctx = self._ctx
    loop = asyncio.get_running_loop()
    _GRACEFUL_TIMEOUT: float = 10.0
    while True:
        line = await self._read_input(loop)
        if line is None:
            break
        if not line:
            continue
        if self._should_exit(line, ctx):
            break
        self._turn_active = True
        try:
            await asyncio.wait_for(
                self._dispatch_line(line, ctx),
                timeout=_GRACEFUL_TIMEOUT if ctx.conv.shutdown_requested else None,
            )
        except asyncio.TimeoutError:
            logger.warning(
                "Graceful shutdown: turn did not complete within %.1fs; forcing exit",
                _GRACEFUL_TIMEOUT,
            )
            break
        finally:
            self._turn_active = False
        if ctx.conv.shutdown_requested:
            break
```

4. **`_close_resources()` — add WAL checkpoint:**
```python
async def _close_resources(self) -> None:
    """Close all session resources. Called in the run() finally block."""
    self._view.write_history()
    svc = self._ctx.services
    if svc is not None:
        # WAL checkpoint before closing connections
        try:
            from db.helper import SQLiteHelper
            with SQLiteHelper("session").open(write_mode=True) as db:
                db.checkpoint("TRUNCATE")
            logger.info("WAL checkpoint completed on shutdown")
        except Exception as e:
            logger.warning("WAL checkpoint failed on shutdown: %s", e)
        await svc.lifecycle.shutdown_all()
        await svc.http.aclose()
```
