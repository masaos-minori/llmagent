# Implementation: scripts/eventbus/config.py — deprecate poll_interval_ms (req #40)

## Goal

Mark `poll_interval_ms` and `offset_checkpoint_interval` as deprecated in `EventBusConfig` to signal that they are no longer used for subscribe delivery.

## Scope

- Add deprecation comments to `poll_interval_ms` and `offset_checkpoint_interval` fields
- Retain fields (do not remove) for backward compatibility with existing config files
- No behavioral changes; fields are now ignored by subscribe()

## Assumptions

- `EventBusConfig` is a dataclass in `scripts/eventbus/config.py`
- `poll_interval_ms` default = 500; `offset_checkpoint_interval` default = 10
- No runtime warning on init needed; comment in source is sufficient

## Implementation

### Target file

`scripts/eventbus/config.py`

### Procedure

1. Find `poll_interval_ms` field definition
2. Add `# deprecated: no longer used; push-mode delivery via EventBroker` comment
3. Find `offset_checkpoint_interval` field definition
4. Add `# deprecated: offset checkpointing removed in req #29; ack-only offset model` comment

### Method

Edit field comment lines.
