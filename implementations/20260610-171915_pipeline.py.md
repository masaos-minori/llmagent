# Implementation: rag/pipeline.py and rag/stages/ — Safe Wrapper Removal and Stage Separation

## Goal

Remove all `except Exception` safe-wrapper patterns from `pipeline.py` and all `stages/` files; fix the `db._http` bug in `stages/search.py`; remove `SQLiteHelper` direct instantiation from `augment()`; resolve the circular import in `stages/augment.py`; update `_get_cfg()` removal effects.

---

## Scope

**Target files:**
- `scripts/rag/pipeline.py`
- `scripts/rag/stages/mqe.py`
- `scripts/rag/stages/search.py`
- `scripts/rag/stages/rerank.py`
- `scripts/rag/stages/augment.py`

**In:**
- `pipeline.py`: delete `expand_queries_safe()`; delete `_get_cfg()` and `_cfg`; pass `cfg` dict and `embed_url` to `RagLLM` and `get_embedding`; pass `http` client to `SearchStage`; update `search_queries()` and `rerank_candidates()` to remove `except Exception`; update `augment()` to propagate `sqlite3.OperationalError` as `RagPipelineError`; update `_augment_http()` / `_augment_refiner()` broad exceptions to specific types
- `stages/mqe.py`: remove `except Exception`; propagate `RagExpansionError`
- `stages/search.py`: fix `db._http` — pass `http: httpx.AsyncClient` to `SearchStage.__init__`; pass it to `_search_all_queries`; remove `except Exception`; remove `# type: ignore`
- `stages/rerank.py`: remove `except Exception`; propagate `RagRerankError`
- `stages/augment.py`: move `sanitize_document` import from `rag.pipeline` to `rag.utils` (also update `rag/utils.py` to export it, or move it there)

**Out:**
- SQL queries in repository.py unchanged
- `mcp/rag_pipeline/service.py` — add `except (sqlite3.OperationalError, RagPipelineError)` around `run_pipeline()` call only

## Assumptions

1. `_get_cfg()` in `pipeline.py` is removed following Step 4; `llm_url` and `embed_url` come from `RagConfig` fields or from `ConfigLoader` resolved by the constructor caller.
2. `RagConfig` (from `shared/types.py`) currently has `llm_url`? — check; if not, add `llm_url: str` and `embed_url: str` fields to `RagConfig` or resolve them from config in `RagPipeline.__init__`.
3. `SearchStage` receives `http: httpx.AsyncClient` from `RagPipeline.__init__` via `SearchStage(self._cfg.__dict__, self._http)`.
4. `sanitize_document` is currently in `pipeline.py` — move to `utils.py` and update `augment.py` import.
5. `RagPipelineError` is a new exception defined in `pipeline.py` (or a separate `rag/exceptions.py`) wrapping `sqlite3.OperationalError` and other pipeline-level failures.

## Implementation

### Target file
`scripts/rag/pipeline.py`

### Procedure

1. Remove `_cfg` module variable and `_get_cfg()` function.
2. In `RagPipeline.__init__`: resolve `llm_url` and `embed_url` from `cfg` (RagConfig or config dict); store as `self._llm_url` and `self._embed_url`.
3. Delete `expand_queries_safe()`.
4. Update `run()`: pass `http=self._http` when creating `SearchStage`; remove the outer `try/except` (keep `finally: self._on_clear()`).
5. Update `search_queries()`: replace `except Exception` with `except (httpx.HTTPStatusError, httpx.RequestError, ValueError)`.
6. Update `rerank_candidates()`: remove `except Exception`; propagate `RagRerankError`.
7. Update `augment()`: replace `except Exception` around `SQLiteHelper().open()` with `except (sqlite3.OperationalError, sqlite3.DatabaseError) as e: raise RagPipelineError(...) from e`.
8. Update `_augment_http()`: replace `except Exception` with `except (httpx.HTTPStatusError, httpx.RequestError, orjson.JSONDecodeError)`.
9. Update `_augment_refiner()`: replace `except Exception` with `except (httpx.HTTPStatusError, httpx.RequestError, ValueError)`.
10. Move `sanitize_document` and `_INJECTION_PATTERNS` to `utils.py`.
11. Add `class RagPipelineError(RuntimeError): ...` to `pipeline.py` (or `rag/exceptions.py`).

### Method (stages/search.py fix)

```python
# stages/search.py — fix db._http bug
async def _search_all_queries(
    queries: list[str],
    db: SQLiteHelper,
    cfg: dict[str, Any],
    http: httpx.AsyncClient,
    embed_url: str,
) -> list[list[RawHit]]:
    raw = await asyncio.gather(
        *(get_embedding(q, http, embed_url) for q in queries),
        return_exceptions=True,
    )
    ...

class SearchStage(PipelineStage):
    def __init__(self, cfg: dict[str, Any], http: httpx.AsyncClient, embed_url: str) -> None:
        self._cfg = cfg
        self._http = http
        self._embed_url = embed_url

    async def run(self, ctx: PipelineContext, db: SQLiteHelper | None = None, **kwargs: Any) -> None:
        if db is None:
            raise ValueError("SearchStage.run requires db argument")
        ctx.search_results = await _search_all_queries(ctx.queries, db, self._cfg, self._http, self._embed_url)
```

### Details

- `RagConfig` currently does not have `llm_url`/`embed_url` — these come from `_get_cfg()` at runtime. After Step 4 removes `_get_cfg()`, `RagPipeline.__init__` must load them from `ConfigLoader().load("common.toml", "agent.toml")` directly (no module-level caching) or they must be added to `RagConfig`.
- Simplest approach: in `RagPipeline.__init__`, call `ConfigLoader().load(...)` once and store the results as `self._llm_url` and `self._embed_url`. This replaces the module-level cache with instance-level initialization.
- `mcp/rag_pipeline/service.py` wraps `pipeline.augment()` in `run_pipeline()` — add `except RagPipelineError` there to return a graceful error response.
