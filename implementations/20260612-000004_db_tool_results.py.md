# Goal

Change `ToolResultStore.get()` and `list_recent()` to return typed `ToolResultRow` DTO
instances instead of raw `dict` / `list[dict]`.

# Scope

- `scripts/db/tool_results.py`

# Assumptions

1. `ToolResultRow` is defined in `agent/services/models.py` (Step 1 prerequisite).
   Import path from `db/tool_results.py` would violate import-linter contracts
   (`db` must not import from `agent`). Therefore define `ToolResultRow` in `db/models.py`
   (Step 2 creates that file), and have `agent/services/models.py` re-export it.
   `db/tool_results.py` imports from `db.models`.
2. `get()` currently returns `dict(rows[0])` for the full schema:
   `id, session_id, turn, tool_name, args_masked, full_text, summary, is_error, created_at`.
3. `list_recent()` currently returns only `id, tool_name, summary, is_error`.
   Use `ToolResultRow` with partial fields — `session_id=None`, `turn=0`, `args_masked=""`,
   `full_text=""`, `created_at=""` as defaults. This is correct because `list_recent` is
   display-only and those fields are not needed at list level.
4. `ToolResultRow.is_error` maps from integer DB value: `bool(row["is_error"])`.

# Implementation

## Target file

`scripts/db/tool_results.py`

## Procedure

1. Add import: `from db.models import ToolResultRow`.
2. Change `get()` signature: `-> ToolResultRow | None`.
   Replace `return dict(rows[0])` with:
   ```python
   r = rows[0]
   return ToolResultRow(
       id=r["id"],
       session_id=r["session_id"],
       turn=r["turn"],
       tool_name=r["tool_name"],
       args_masked=r["args_masked"] or "",
       full_text=r["full_text"] or "",
       summary=r["summary"],
       is_error=bool(r["is_error"]),
       created_at=r["created_at"] or "",
   )
   ```
3. Change `list_recent()` signature: `-> list[ToolResultRow]`.
   Replace `return [dict(r) for r in reversed(rows)]` with:
   ```python
   return [
       ToolResultRow(
           id=r["id"],
           tool_name=r["tool_name"],
           summary=r["summary"],
           is_error=bool(r["is_error"]),
       )
       for r in reversed(rows)
   ]
   ```
4. Run ruff + mypy.

## Method

Minimal change: only the return construction changes. SQL queries and connection logic
are unchanged.
