# Implementation Procedure: cmd_config_set.py

## Goal

Delete `scripts/agent/commands/cmd_config_set.py` in full, since the `/set` command
it implements (`_ConfigSetMixin._cmd_set`, `_set_temperature`, `_set_max_tokens`) is
being removed from the product entirely (`llm_temperature`/`llm_max_tokens` remain
configurable only via `config/agent.toml` + `/reload`).

## Scope

### In scope
- Delete the file `scripts/agent/commands/cmd_config_set.py` (94 lines: `_ConfigSetMixin`
  class with `_set_temperature`, `_set_max_tokens`, `_cmd_set`).
- Remove its sole consumer's import/composition (`cmd_config.py` — see that file's own
  implementation doc).

### Out of scope
- `llm_temperature` / `llm_max_tokens` field names, defaults, validators, or `/reload`
  behavior in `config_dataclasses.py` / `config_builders.py` / `config_reload.py` —
  these fields keep working exactly as before, just without a runtime `/set` override.
- `MAX_TEMPERATURE` / `SET_PARTS_COUNT` constants — deleted along with the file; no
  other module references them (verified: they are module-local to this file).

## Assumptions

- `cmd_config.py` is the only importer of `_ConfigSetMixin` (verified via
  `grep -rn "_ConfigSetMixin\|cmd_config_set" scripts/` — only `cmd_config.py` imports
  it and composes it into `_ConfigMixin`'s base-class tuple).
- No test outside `tests/test_cmd_config_refactor.py` imports from `cmd_config_set`
  directly (that test file is deleted in its own step).

## Implementation

### Target file

`scripts/agent/commands/cmd_config_set.py`

### Procedure

1. Confirm no other importer exists: `grep -rn "cmd_config_set\|_ConfigSetMixin" scripts/ tests/`.
2. Delete the file: `git rm scripts/agent/commands/cmd_config_set.py`.
3. In `cmd_config.py`, remove:
   - `from agent.commands.cmd_config_set import _ConfigSetMixin  # noqa: E402` (import line).
   - `_ConfigSetMixin` from `_ConfigMixin`'s base-class tuple.
   - The module docstring line `cmd_config_set   — _set_temperature, _set_max_tokens, _cmd_set`.
   (This edit belongs to `cmd_config.py`'s own implementation step — cross-referenced here
   because the two changes must land together to avoid a dangling import.)

### Method

Whole-file deletion via `git rm`; no partial edits needed since every symbol in the
file (`_ConfigSetMixin`, `_set_temperature`, `_set_max_tokens`, `_cmd_set`,
`MAX_TEMPERATURE`, `SET_PARTS_COUNT`) is being removed from the product.

### Details

- Deleting this file removes the `/set` handler (`_cmd_set`) that
  `command_defs_list.py`'s `CommandDef("/set", ...)` (removed in that file's own step)
  referenced by name — the two deletions must be coordinated so `registry.py`'s
  fail-fast `hasattr` check in `__init__` never runs against a stale `CommandDef`.
- `deploy/deploy.sh` needs no edit: it `rsync -av --delete`s the whole `scripts/` tree
  (verified: `deploy/deploy.sh:68-73`), so file removal is picked up automatically.
