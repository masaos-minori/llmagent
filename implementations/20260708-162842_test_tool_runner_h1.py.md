# Implementation: H-1 — Add tests for execute_one_tool_call() truncation-only behavior

## Goal

Add test coverage proving `execute_one_tool_call()` always truncates long tool output and never
calls `summarize_tool_result`, per the companion source change
(`implementations/20260708-162748_tool_runner_h1.py.md` or its `done/` copy).

## Scope

**Target**: `tests/test_tool_runner.py`

**Depends on**: `scripts/agent/tool_runner.py`'s H-1 change already applied (summarize branch
removed from `execute_one_tool_call()`).

**Out of scope**: `tests/test_tool_result_formatter.py` (per the H-1 plan, `is_summarized()`
tests there are unaffected and require no changes — verify only, no edits).

## Assumptions

1. `execute_one_tool_call` is not currently imported in this test file (only
   `_compute_serial_overhead`, `_estimate_parallel_time`, `_execute_with_dag`,
   `execute_all_tool_calls` are imported from `agent.tool_runner` — see current import block).
   Add `execute_one_tool_call` to that import.
2. `_make_ctx()` / `_cfg()` (module-level helpers, lines 23-92) already provide
   `tool_result_max_llm_chars` (default `4000`) and `use_tool_summarize` (default `False`) in
   `_cfg()`'s defaults dict — reusable as-is for the new tests; no helper changes needed.
3. `ctx.services_required.tools.execute` (set by `_make_ctx()`) returns a fixed
   `ToolCallResult(output="result", ...)` — new tests override this per-test via
   `ctx.services_required.tools.execute = AsyncMock(return_value=ToolCallResult(output=<long or
   short text>, is_error=False, request_id="req-1", server_key=""))` to control `text` length.

#### Step 2: Add a new test class with the four required cases

Add near the end of the file (after `TestExecuteStandardSerialization`, or in a location
consistent with the file's existing class ordering):

```python
class TestExecuteOneToolCallTruncation:
    """execute_one_tool_call() always truncates; no LLM summarization path exists."""

    @pytest.mark.asyncio
    async def test_long_output_is_truncated(self) -> None:
        cfg = _cfg(tool_result_max_llm_chars=10)
        ctx = _make_ctx(cfg)
        long_text = "x" * 50
        ctx.services_required.tools.execute = AsyncMock(
            return_value=ToolCallResult(
                output=long_text, is_error=False, request_id="req-1", server_key=""
            )
        )

        _tc_id, _name, _args, text, is_error, llm_text = await execute_one_tool_call(
            ctx, _tc("read_text_file"), 0
        )

        assert text == long_text  # full text is preserved in the return tuple
        assert llm_text == long_text[:10] + "\n... (truncated)"
        assert not is_error

    @pytest.mark.asyncio
    async def test_short_output_passes_through_unchanged(self) -> None:
        cfg = _cfg(tool_result_max_llm_chars=4000)
        ctx = _make_ctx(cfg)
        short_text = "short result"
        ctx.services_required.tools.execute = AsyncMock(
            return_value=ToolCallResult(
                output=short_text, is_error=False, request_id="req-1", server_key=""
            )
        )

        _tc_id, _name, _args, _text, _is_error, llm_text = await execute_one_tool_call(
            ctx, _tc("read_text_file"), 0
        )

        assert llm_text == short_text

    @pytest.mark.asyncio
    async def test_error_output_truncated_only_no_summarize(self) -> None:
        cfg = _cfg(tool_result_max_llm_chars=10, use_tool_summarize=True)
        ctx = _make_ctx(cfg)
        long_error_text = "e" * 50
        ctx.services_required.tools.execute = AsyncMock(
            return_value=ToolCallResult(
                output=long_error_text, is_error=True, request_id="req-1", server_key=""
            )
        )

        with patch(
            "agent.tool_runner.summarize_tool_result",
            side_effect=AssertionError("summarize_tool_result must not be called"),
        ):
            _tc_id, _name, _args, _text, is_error, llm_text = await execute_one_tool_call(
                ctx, _tc("read_text_file"), 0
            )

        assert is_error
        assert llm_text == long_error_text[:10] + "\n... (truncated)"

    @pytest.mark.asyncio
    async def test_summarize_tool_result_never_called_even_when_enabled(self) -> None:
        """use_tool_summarize=True must have no effect — the summarize path is removed."""
        cfg = _cfg(
            use_tool_summarize=True,
            tool_summarize_threshold=1,
            tool_result_max_llm_chars=4000,
        )
        ctx = _make_ctx(cfg)
        text_over_old_threshold = "y" * 100  # > tool_summarize_threshold, < max_llm_chars
        ctx.services_required.tools.execute = AsyncMock(
            return_value=ToolCallResult(
                output=text_over_old_threshold,
                is_error=False,
                request_id="req-1",
                server_key="",
            )
        )

        with patch(
            "agent.tool_runner.summarize_tool_result",
            side_effect=AssertionError("summarize_tool_result must not be called"),
        ):
            _tc_id, _name, _args, _text, _is_error, llm_text = await execute_one_tool_call(
                ctx, _tc("read_text_file"), 0
            )

        assert llm_text == text_over_old_threshold  # under max_llm_chars, passed through as-is
```

Note: `test_summarize_tool_result_never_called_even_when_enabled`'s `patch(...)` target
(`"agent.tool_runner.summarize_tool_result"`) will fail with `AttributeError` once the companion
source-code doc removes the `summarize_tool_result` import from `tool_runner.py` — `patch()`
cannot target a name that no longer exists in the module namespace. **If the source change has
already removed the import when this test is written, drop the `patch(...)` context manager
entirely** (the whole point of the removed import is that the module can no longer reach
`summarize_tool_result` at all, making the patch both unnecessary and impossible). In that case
the two tests simplify to a plain `await execute_one_tool_call(...)` call with no patch, and the
assertion becomes purely about the returned `llm_text` value — the "not called" guarantee is
then structural (no import exists to call) rather than behaviorally asserted via a mock.

### Method

- New test class, additive only — no existing test is modified in this doc (see the companion
  H-2 doc for the one existing assertion removal, which is a separate concern).
- Follows the file's existing `_cfg()`/`_make_ctx()`/`_tc()` helper conventions rather than
  introducing new fixtures.

### Details

- `tool_result_max_llm_chars=10` is deliberately small in the truncation tests to make the
  `"\n... (truncated)"` suffix easy to assert against without constructing very long strings.
- The four cases map directly to the H-1 plan's Implementation Steps 2a-2d:
  - 2a → `test_long_output_is_truncated`
  - 2b → `test_short_output_passes_through_unchanged`
  - 2c → `test_error_output_truncated_only_no_summarize`
  - 2d → `test_summarize_tool_result_never_called_even_when_enabled` (and the `patch(...)` in
    `test_error_output_truncated_only_no_summarize`)
