## Goal
Change `LLMTurnRunner.run()` to require workflow context (`workflow_id`, `task_id`, `stage_id`, `attempt_id`), extend `_span_ctx()` with OTel workflow attributes, and update `_handle_llm_error()` to include workflow context in DiagnosticStore.

## Scope
**In**: `scripts/agent/llm_turn_runner.py` — change `run()` signature; update `_handle_llm_error()`; extend `_span_ctx()` with `workflow_id`, `stage_id`, `attempt_id`.
**Out**: LLM streaming protocol; tool runner; DiagnosticStore schema changes.

## Assumptions
- `run(self, llm_url: str)` is at line 58; no workflow context params.
- `_handle_llm_error()` at line 114 takes no workflow context.
- `_span_ctx(name, task_id="", session_id="", model_url="")` at line 144 — missing workflow_id, stage_id, attempt_id.
- `ctx.diagnostics.save()` accepts `workflow_id` and `task_id` as optional kwargs already.
- All production callers of `run()` go through `Orchestrator._handle_llm_turn()`.

## Implementation

**Target file**: `scripts/agent/llm_turn_runner.py`

**Procedure**:

1. **Change `run()` signature**:
   ```python
   # Before:
   async def run(self, llm_url: str) -> TurnResult:

   # After:
   async def run(
       self,
       llm_url: str,
       *,
       workflow_id: str,
       task_id: str,
       stage_id: str,
       attempt_id: str,
   ) -> TurnResult:
       if not (workflow_id and task_id and stage_id and attempt_id):
           raise RuntimeError(
               "LLMTurnRunner.run() requires non-empty workflow context: "
               f"workflow_id={workflow_id!r}, task_id={task_id!r}, "
               f"stage_id={stage_id!r}, attempt_id={attempt_id!r}"
           )
   ```

2. **Extend `_span_ctx()` signature**:
   ```python
   # Before:
   def _span_ctx(self, name: str, task_id: str = "", session_id: str = "", model_url: str = "") -> Any:

   # After:
   def _span_ctx(
       self,
       name: str,
       task_id: str = "",
       session_id: str = "",
       model_url: str = "",
       workflow_id: str = "",
       stage_id: str = "",
       attempt_id: str = "",
   ) -> Any:
   ```
   Add conditional attribute setting in the body:
   ```python
   if workflow_id:
       attrs["workflow.workflow_id"] = workflow_id
   if stage_id:
       attrs["workflow.stage_id"] = stage_id
   if attempt_id:
       attrs["workflow.attempt_id"] = attempt_id
   ```

3. **Update `_handle_llm_error()` signature**:
   ```python
   def _handle_llm_error(self, ..., workflow_id: str = "", task_id: str = "") -> None:
   ```
   Add to `ctx.diagnostics.save()` call:
   ```python
   ctx.diagnostics.save(
       ctx.session.session_id,
       "mid_turn_error",
       ...,
       workflow_id=workflow_id,
       task_id=task_id,
   )
   ```

4. **Pass context from `run()` to internal calls**:
   - Pass `workflow_id`, `task_id`, `stage_id`, `attempt_id` to `_span_ctx()` call inside `run()`
   - Pass `workflow_id`, `task_id` to `_handle_llm_error()` call inside `run()`

**Method**: Keyword-only parameter addition + guard at top of run().

**Details**:
- Plans 06 and 12 must be implemented together (both touch `_span_ctx` and `run()`).
- After signature change, call site in `orchestrator.py` must be updated (see orchestrator.py doc).
