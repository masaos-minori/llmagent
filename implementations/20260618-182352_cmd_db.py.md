# Implementation: Add `_db_help()` subcommand and target-DB labels to cmd_db.py

## Goal

Users can type `/db help` to see a table of subcommands with their target database, argument syntax, and description. Each subcommand handler also appends a target-DB annotation to its output so the user knows which database was affected.

## Scope

- `scripts/agent/commands/cmd_db.py` — add `"help"` to dispatch dict, add `_db_help()` method, label output per subcommand

**Out of scope:** Schema changes, command_defs.py metadata changes (separate doc).

## Assumptions

1. `SubcommandSpec` entries for `/db` are already populated in `command_defs.py` before this change.
2. The `_out` object (from `MixinBase`) has `write_table()` — confirmed usage in `_db_list_urls`.
3. The error fallback (else branch in `_cmd_db`) should remain unchanged as a safety net for unknown subcommands.

## Implementation

### Target file

`scripts/agent/commands/cmd_db.py`

### Procedure

1. Add `"help": self._db_help` to the dispatch dict in `_cmd_db()`.
2. Implement `_db_help(self) -> None` that prints a table: subcommand, target DB, args, description.
3. Modify each subcommand handler to append or prepend a target-DB label in its output.

### Method

Surgical edits: add one dispatch entry, add one method, modify output lines.

### Details

#### Change A — Dispatch entry (line 39-49)

Add after the `"stats"` line or before the `"urls"` line:

```python
        dispatch = {
            "help": self._db_help,
            "stats": self._db_stats,
            ...
```

#### Change B — New method

Add after `_cmd_db()`:

```python
    def _db_help(self) -> None:
        """Print a help table for /db subcommands."""
        rows = [
            ["stats",       "rag + session", "",                "Record counts"],
            ["urls",        "rag",           "--lang --limit",  "List document URLs"],
            ["clean",       "rag",           "<url>",           "Delete a document"],
            ["rebuild-fts", "rag",           "",                "Rebuild FTS5 index"],
            ["health",      "rag + session", "",                "Integrity check / size"],
            ["checkpoint",  "session",       "[MODE]",          "WAL checkpoint"],
            ["vacuum",      "session",       "",                "Reclaim free pages"],
            ["purge",       "session",       "--max-sessions N\n--max-age-days N", "Purge old sessions"],
            ["recover",     "session",       "[backup-path]",   "Integrity check / restore"],
        ]
        self._out.write_table(
            ["Subcommand", "Target DB", "Arguments", "Description"],
            rows,
        )
```

#### Change C — Output labels

Update output lines in each subcommand handler to include a target-DB tag like `[rag]` or `[session]` at the end of the success/result line:

- `_db_stats`: Change to `f"documents={result.docs:,}, chunks={result.chunks:,}, sessions={result.sessions:,}, messages={result.messages:,} [rag+session]"` (or a multi-line format that includes the label)
- `_db_clean`: `f"Document deleted: {url} [rag]"`
- `_db_list_urls`: No change needed (table output is self-explanatory)
- `_db_rebuild_fts`: `"FTS5 index rebuilt [rag]"`
- `_db_health`: Change to include `[rag+session]` in output
- `_db_checkpoint`: Append `[session]` after pages_written
- `_db_vacuum`: Append `[session]` after "VACUUM complete"
- `_db_purge`: Append `[session]` after "session(s) removed"
- `_db_recover`: Append `[session]` or `[rag]` as appropriate
