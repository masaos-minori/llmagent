# Implementation: Add branch-aware filtering to retriever.py

Steps covered: Plan 20260626-100124 — Steps 1-5

---

## Goal

Add `branch` parameter to `top_semantic()` and `HybridRetriever.search()` in `scripts/agent/memory/retriever.py`, adding a hard SQL WHERE filter so memories from unrelated branches are not injected.

---

## Scope

- **In scope**:
  - `scripts/agent/memory/retriever.py`: `_context_boost`, `top_semantic`, `HybridRetriever.__init__/search`
  - `scripts/agent/memory/injection.py`: `MemoryInjectionService` — add `branch` field
  - `tests/agent/memory/` — branch boundary tests
  - `docs/05_agent_12_memory.md` — context boundary docs
- **Out of scope**: git integration changes; memory metadata changes

---

## Assumptions

- `MemoryEntry.branch` field exists.
- Global rule memories have `branch=""` and must always be returned (WHERE clause: `branch = '' OR branch = ?`).
- Default `branch=""` maintains existing behavior.

---

## Implementation

### Target file
`scripts/agent/memory/retriever.py`

### Procedure
1. Read retriever.py lines 60-80 (`_context_boost`) and lines 310-340 (`top_semantic`).
2. Step 1 (`_context_boost`): Add branch boost:
   ```python
   if branch and entry.branch == branch:
       return _CONTEXT_MATCH_BOOST + 0.05
   if (project and entry.project == project) or (repo and entry.repo == repo):
       return _CONTEXT_MATCH_BOOST
   ```
3. Step 2 (`top_semantic`): Add `branch: str = ""` parameter and WHERE clause:
   ```sql
   WHERE memory_type = 'semantic' AND importance >= ?
   AND (? = '' OR branch = '' OR branch = ?)
   ```
4. Step 3: Pass `branch` through `HybridRetriever.__init__` → `search` → `top_semantic`.
5. Step 4: Add `branch: str = ""` to `MemoryInjectionService.__init__`.
6. Step 5 (Tests): Add test — branch="feat/x" session does not return branch="feat/y" memories; global (branch="") memories always returned.

### Method
SQL WHERE extension + parameter threading. Default `branch=""` ensures backward compatibility.
