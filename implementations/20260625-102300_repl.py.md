# repl.py — use orchestrator.workflow_status() instead of private attrs

**Plan:** `plans/20260625-094257_plan.md` (req #64)
**Target:** `scripts/agent/repl.py`

## What to change

`_get_workflow_status()` at lines 402-411 accesses private attributes `_workflow_mode`
and `_workflow_def`. Replace with the new `workflow_status()` public API.

**Before (lines 402-411):**
```python
def _get_workflow_status(self) -> str:
    """Return a human-readable workflow status string for the startup banner."""
    if self._orchestrator is None:
        return "unknown"
    mode = self._orchestrator._workflow_mode
    if mode == "disabled":
        return "disabled"
    if self._orchestrator._workflow_def is not None:
        return f"{mode} (tracking enabled)"
    return f"{mode} (definition not loaded)"
```

**After:**
```python
def _get_workflow_status(self) -> str:
    """Return a human-readable workflow status string for the startup banner."""
    if self._orchestrator is None:
        return "unknown"
    status = self._orchestrator.workflow_status()
    mode = status["mode"]
    if mode == "disabled":
        return "disabled"
    if status["tracking"] == "enabled":
        return f"{mode} (tracking enabled)"
    return f"{mode} (definition not loaded)"
```
