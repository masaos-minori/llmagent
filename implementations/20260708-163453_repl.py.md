# Implementation: H-4 — Update partial-completion warning message in repl.py

## Goal

Remove the `tool_results` / `tool_name='llm_partial_completion'` retrieval hint from the
user-facing warning shown after a partial LLM completion, matching the companion removal of
`ToolResultStore` persistence in `llm_transport_errors.py`.

## Scope

**Target**: `scripts/agent/repl.py`

**Depends on**: `implementations/20260708-163427_llm_transport_errors.py.md` (or its `done/`
copy) — not a hard code dependency (this is just a string literal), but both changes describe
the same user-facing feature and should land together to avoid a stale hint pointing at data
that is no longer written.

**Out of scope**: the surrounding `stat_partial_completions` comparison logic (`_prev_partial`,
the `if llm is not None and llm.stat_partial_completions > _prev_partial:` guard) — unchanged.

## Assumptions

1. The warning is emitted via `self._view.write_warning(...)` inside the line-dispatch method
   (around lines 426-435) — confirm exact line numbers with
   `grep -n "Partial LLM completion stored" -A3 scripts/agent/repl.py` before editing, since line
   numbers shift as the file changes.
2. No other module constructs this exact string (verify via grep in Procedure Step 1) — this is
   a single-site string literal change.

## Implementation

### Target file

`scripts/agent/repl.py`

### Procedure

#### Step 1: Confirm single occurrence

```bash
grep -rn "Partial LLM completion stored" scripts/
```

Expect exactly one match, in this file.

#### Step 2: Replace the warning message

Current:

```python
                self._view.write_warning(
                    "Partial LLM completion stored."
                    " Use /stats to see count or query tool_results"
                    " (tool_name='llm_partial_completion')."
                )
```

Replace with:

```python
                self._view.write_warning(
                    "Partial LLM completion stored. Use /stats to see count."
                )
```

### Method

- Single string-literal replacement; no control-flow change.
- Collapses the multi-line implicit string concatenation into one line since the retrieval
  clause is gone (matches the plan's Design section wording exactly).

### Details

- `self._view.write_warning(...)` call itself, its trigger condition
  (`llm.stat_partial_completions > _prev_partial`), and the surrounding `if`/`else` structure
  are all unchanged — only the string argument's content changes.
