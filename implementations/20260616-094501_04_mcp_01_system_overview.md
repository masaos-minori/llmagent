# 04_mcp_01_system_overview — Implementation Procedure

## Goal
MCPシステム全体像を1ファイルで把握できるドキュメントを作成する。11サーバーの一覧、ポート割り当て、起動モード、トランスポート種別を整理する。

## Scope
- `docs/04_spec_mcp.md` §1〜5
- サーバーポートテーブル (8004〜8014)
- startup_mode テーブル (always/on-demand/subprocess)
- トランスポート種別 (http/stdio)

## Assumptions
- サーバー総数は11台（web-search, file-read, file-write, file-delete, github, shell, rag-pipeline, sqlite, cicd, mdq, git）
- ポートは 8004〜8014 の連番に割り当てられている
- startup_mode と transport の組み合わせは有限であり列挙可能

## Implementation

### Target file
`docs/04_mcp_01_system_overview.md`

### Procedure
- システム構成図（テキストアスキーアート or Mermaid）でエージェント→MCPクライアント→サーバー群の関係を示す
- サーバーポートテーブルを作成する（サーバー名・ポート・transport・startup_mode）
- 起動モード別の動作差異を説明する
- トランスポート種別（HTTP vs stdio）の使い分けを説明する

### Method
04_spec_mcp.md の §1〜5 を精読し、テーブルと説明文に再構成する。冗長な説明は省き要点のみ記載する。

### Details
- ポートテーブル: web-search:8004, file-read:8005, github:8006, file-write:8007, file-delete:8008, shell:8009, rag-pipeline:8010, sqlite:8011, cicd:8012, mdq:8013, git:8014
- startup_mode: always=常時起動, on-demand=初回呼び出し時起動, subprocess=エージェントが子プロセス起動
- transport: http=HTTP/SSE, stdio=標準入出力

## Validation plan
- 11サーバー全てがテーブルに存在することを確認する
- ポート番号の重複・抜けがないことを確認する
- startup_mode × transport の組み合わせが全ケース記載されていることを確認する
