## Implementation

### Target file

### Procedure

1. Locate the existing table (`docs/04_mcp_04_04_mdq.md:54-58`):
   ```
   | ツール | パス入力 | 認可 |
   |------|------------|---------------|
   | `index_paths` | インデックス対象のディレクトリ/ファイル | `authorize_path()` — `allowed_dirs` 外の場合 `MdqAuthorizationError` を発生させる |
   | `refresh_index` | 更新対象のパス | パス認可 — いずれかのパスが拒否された場合 `MdqAuthorizationError` を発生させる |
   | `outline` | outline 対象のファイル | `authorize_path()` — `allowed_dirs` 外の場合 `MdqAuthorizationError` を発生させる |
   ```
2. Append three new rows immediately after the `outline` row (line 58) and before the blank
   line preceding the bullet list (line 59):
   ```
   | `search_docs` | インデックス済みチャンクの `source_path`（読み取り時に再チェック） | `authorize_path()` — `allowed_dirs` 外のチャンクは結果から無音で除外される（件数にも計上されない） |
   | `get_chunk` | 取得対象チャンクの `source_path`（読み取り時に再チェック） | `authorize_path()` — `allowed_dirs` 外の場合 `MdqAuthorizationError` を発生させる |
   | `grep_docs` | 明示指定パス、または全インデックス済み `source_path` のうち認可済みの集合（読み取り時に再チェック） | `authorize_path()` — 明示指定パスが未認可の場合は `MdqAuthorizationError` を発生させ全体を拒否；フィルタ未指定時は未認可パスを無音で除外 |
   ```
3. Leave the bullet list (`..` トラバーサル / シンボリックリンク / 空の `allowed_dirs`) as-is
   — those bullets describe `authorize_path()`'s internal mechanics, still accurate verbatim
   for the three new tools since they all delegate to the same `authorize_path()` function.

### Method

Direct Markdown table-row insertion — three new rows appended to an existing table, matching
its exact column count and Markdown pipe-table syntax. No restructuring of the table itself or
the surrounding section.

### Details

- Distinguish the two authorization *behaviors* precisely in the new rows' 認可 column, per
  the companion code docs: `get_chunk` fails the whole call (raises); `search_docs` silently
  drops unauthorized rows (no raise, no leaked count); `grep_docs` does either depending on
  whether `paths` was explicitly supplied (raises) or omitted (silently filters) — do not
  flatten this distinction into one generic "authorizes" statement, since it is operationally
  significant (a caller passing an unauthorized path explicitly gets a hard error; a caller
  doing an unscoped search/grep gets quietly filtered results).
- Keep the Japanese terminology/style consistent with the rest of this table (e.g. "無音で除外
  される" mirrors this file's existing style of describing effects precisely; "計上されない"
  echoes the plan's Risk item about not surfacing hidden-result counts).
