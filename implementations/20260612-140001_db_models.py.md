# Goal

Add `DbHealthMetrics`, `DocumentRow`, `SessionRow`, and `MessageRow` frozen
dataclasses to `scripts/db/models.py` alongside the existing DTOs.

# Scope

- `scripts/db/models.py` — append-only; no existing code is changed

# Assumptions

1. Fields for each DTO match the columns returned by the respective SQL queries
   in `store.py` and `helper.py`.
2. `SessionRow.is_current` is a bool derived from comparing `session_id` to the
   current session — the store already has this logic.
3. All DTOs use `@dataclass(frozen=True)` per project convention.
4. `db/models.py` may only import from stdlib (no circular imports).

# Implementation

## Target file

`scripts/db/models.py`

## Procedure

1. Append the four new dataclasses after the existing `ToolResultRow` class.
2. Run ruff + mypy.

## Method

Append-only. No existing code modified.

## Details

```python
@dataclass(frozen=True)
class DbHealthMetrics:
    """DB health metrics returned by SQLiteHelper.health_check()."""

    journal_mode: str
    integrity: str
    page_count: int
    page_size: int
    freelist_count: int
    db_size_bytes: int


@dataclass(frozen=True)
class DocumentRow:
    """One row from the documents table (list_documents query result)."""

    url: str
    lang: str | None
    chunk_count: int
    fetched_at: str


@dataclass(frozen=True)
class SessionRow:
    """One row from the sessions table (list_sessions query result)."""

    session_id: int
    title: str | None
    created_at: str
    is_current: bool


@dataclass(frozen=True)
class MessageRow:
    """One row from the messages table (list_messages query result)."""

    message_id: int
    session_id: int
    role: str
    content: str
    created_at: str
```
