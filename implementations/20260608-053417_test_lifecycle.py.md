# Implementation: tests/test_lifecycle.py — ServerLifecycleManager reference update

## Goal

Update all 50+ references to `ServerLifecycleManager` in `tests/test_lifecycle.py` to use `HttpServerLifecycleManager` and `StdioServerLifecycleManager` directly instead.

## Scope

- `tests/test_lifecycle.py`
- All test functions that instantiate or mock `ServerLifecycleManager`.

## Assumptions

- `HttpServerLifecycleManager` and `StdioServerLifecycleManager` are the two concrete implementations.
- Tests can be split: HTTP-specific tests use `HttpServerLifecycleManager`, stdio-specific tests use `StdioServerLifecycleManager`.
- Shared fixture behavior is factored into helper functions.

## Implementation

### Target file

`tests/test_lifecycle.py`

### Procedure

1. Replace `from agent.lifecycle import LifecycleState, ServerLifecycleManager` with imports for concrete managers.
2. For each test that uses `ServerLifecycleManager`:
   - Identify if the test exercises HTTP subprocess lifecycle → use `HttpServerLifecycleManager`
   - Identify if the test exercises stdio lifecycle → use `StdioServerLifecycleManager`
   - Update type references, mock targets, and assertions accordingly.
3. Run `ruff format`.
4. Run `uv run pytest tests/test_lifecycle.py` — all pass.

### Method

Systematic find-and-replace across the test file. Use `HttpServerLifecycleManager` constructor (no args) and `StdioServerLifecycleManager(server_configs, tool_executor, stdio_procs, last_called)` signature.

### Details

`HttpServerLifecycleManager.__init__()` takes no arguments.
`StdioServerLifecycleManager.__init__()` takes `(server_configs, tool_executor, stdio_procs, last_called)`.

Update mock paths:
- `patch("agent.lifecycle.ServerLifecycleManager")` → `patch("agent.http_lifecycle.HttpServerLifecycleManager")` or `patch("agent.stdio_lifecycle.StdioServerLifecycleManager")`
