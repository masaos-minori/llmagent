# Implementation: agent/shared/health_models.py — Add StartupValidationResult types

## Goal

Add `StartupCheckStatus`, `StartupCheckOutcome`, and `StartupValidationResult` to `scripts/agent/shared/health_models.py` to support the startup validation pipeline that collects all check results before raising.

## Scope

**In**: Append the 3 new types to `health_models.py`. No changes to existing `ServiceWarning`, `HealthCheckResult`, or `McpHealthProbeResult`.

**Out**: Changes to `startup.py` or tests. Those are separate phases.

## Assumptions

1. `health_models.py` currently exports `ServiceWarning`, `HealthCheckResult`, `McpHealthProbeResult`. New types are appended without altering existing ones.
2. `StartupCheckStatus` uses `str` mixin enum for easy serialisation.
3. `StartupCheckOutcome` is frozen — outcomes are immutable once created.
4. `StartupValidationResult` is mutable (accumulates outcomes during startup) — not frozen.
5. Python 3.13; `from __future__ import annotations` already present.
6. `dataclasses.field(default_factory=list)` for the mutable `outcomes` list.

## Implementation

### Target file

`scripts/agent/shared/health_models.py`

### Procedure

1. Add `from enum import Enum` import (or `StrEnum` if Python 3.11+ pattern used elsewhere).
2. Append `StartupCheckStatus(str, Enum)` with 4 values: `OK`, `WARNING`, `FATAL`, `SKIPPED`.
3. Append `StartupCheckOutcome` frozen dataclass with fields: `source: str`, `status: StartupCheckStatus`, `message: str = ""`, `remediation: str = ""`.
4. Append `StartupValidationResult` mutable dataclass with `outcomes: list[StartupCheckOutcome]` and methods: `add_fatal`, `add_warning`, `add_ok`, `add_skipped`, `has_fatal` property, `fatal_messages()`.

### Method

```python
from enum import Enum


class StartupCheckStatus(str, Enum):
    OK = "ok"
    WARNING = "warning"
    FATAL = "fatal"
    SKIPPED = "skipped"


@dataclass(frozen=True)
class StartupCheckOutcome:
    source: str
    status: StartupCheckStatus
    message: str = ""
    remediation: str = ""


@dataclass
class StartupValidationResult:
    outcomes: list[StartupCheckOutcome] = field(default_factory=list)

    def add_fatal(self, source: str, message: str, remediation: str = "") -> None:
        self.outcomes.append(
            StartupCheckOutcome(source, StartupCheckStatus.FATAL, message, remediation)
        )

    def add_warning(self, source: str, message: str) -> None:
        self.outcomes.append(
            StartupCheckOutcome(source, StartupCheckStatus.WARNING, message)
        )

    def add_ok(self, source: str) -> None:
        self.outcomes.append(StartupCheckOutcome(source, StartupCheckStatus.OK))

    def add_skipped(self, source: str, message: str = "") -> None:
        self.outcomes.append(
            StartupCheckOutcome(source, StartupCheckStatus.SKIPPED, message)
        )

    @property
    def has_fatal(self) -> bool:
        return any(o.status == StartupCheckStatus.FATAL for o in self.outcomes)

    def fatal_messages(self) -> list[str]:
        return [o.message for o in self.outcomes if o.status == StartupCheckStatus.FATAL]

    def warning_messages(self) -> list[str]:
        return [o.message for o in self.outcomes if o.status == StartupCheckStatus.WARNING]
```

### Details

- `str` mixin on `StartupCheckStatus` allows `status == "fatal"` comparisons in tests.
- `add_skipped` is added for the live routing check when all servers are unreachable in non-strict mode.
- `warning_messages()` is symmetric with `fatal_messages()` for use in `_display_pipeline_results()`.
- No imports outside stdlib (`dataclasses`, `enum`).
