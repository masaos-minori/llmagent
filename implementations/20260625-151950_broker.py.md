### Details

**Topic filter semantics:**
- `sub.topics == []` (empty list): subscriber receives ALL events regardless of topic
- `sub.topics != []` (non-empty): subscriber receives event only if `event["topic"] in sub.topics`
- Topic comparison is exact string match (case-sensitive, no glob or prefix matching)
- Consistent with SQLite `IN (...)` clause used in replay queries

**Snapshot pattern in `publish()`:**
```python
for sub in list(self._subscribers):   # snapshot prevents mutation errors
    if sub.topics and event_topic not in sub.topics:
        continue
    try:
        sub.queue.put_nowait(event)
        delivered += 1
    except asyncio.QueueFull:
        logger.warning(
            "broker: queue full sub=%d dropping seq=%s",
            id(sub),
            event.get("seq"),
        )
```

Rationale for `list(self._subscribers)` snapshot:
- `subscribe()` and `unsubscribe()` may run concurrently on the same event loop
- Iterating over a list being modified raises `RuntimeError: list changed size during iteration`
- Snapshot is O(n) copy but n is bounded by active SSE connections (typically < 100)

**Why put_nowait() instead of put():**
- `put()` is a coroutine that suspends the fan-out loop if any queue is full
- One slow consumer would block delivery to all fast consumers
- `put_nowait()` drops to the slow consumer but fast consumers are unaffected
- Slow consumer reconnects and replays from last acked offset via SQLite

**Metrics methods (used by health endpoint in req #42):**
```python
def subscriber_count(self) -> int:
    return len(self._subscribers)

def max_queue_depth(self) -> int:
    if not self._subscribers:
        return 0
    return max(sub.queue.qsize() for sub in self._subscribers)

def slow_consumer_count(self, threshold: int = 100) -> int:
    return sum(1 for sub in self._subscribers if sub.queue.qsize() >= threshold)
```
