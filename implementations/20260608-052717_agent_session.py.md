# Implementation: agent/session.py — re-export removal

## Goal

Remove `DocumentRepository` / `NoteRepository` / `SessionMessageRepository` re-export lines from `agent/session.py`. These are no longer needed as public exports since callers should import directly from the repository modules.

## Scope

- Delete the 3 `# noqa: F401` import lines that serve as re-exports.
- `AgentSession` internal usage of these classes is unchanged (uses `_message_repo`, `_document_repo`, `_note_repo` private attributes via regular imports).

## Assumptions

- No external code uses `from agent.session import DocumentRepository` (grep confirmed 0 callers).
- `AgentSession` class itself continues to work because the classes are used via the instance attribute pattern, not via re-export.

## Implementation

### Target file

`scripts/agent/session.py`

### Procedure

1. Delete lines 11–15 (the 3 re-export import lines).
2. Preserve the regular imports for `SessionMessageRepository`, `DocumentRepository`, `NoteRepository` that are used inside `AgentSession.__init__()` — actually these ARE the same imports. Wait: currently lines 11-15 ARE the only imports of these classes. They are used in `__init__` via `self._message_repo = SessionMessageRepository(...)` etc. So these imports must remain for the class to work. The fix is to remove the `# noqa: F401 — re-export` comments but keep the imports as private/normal imports.
3. Run `ruff format` on the file.

### Method

Convert public re-exports to private internal imports:

```python
# Before:
from agent.document_repo import DocumentRepository  # noqa: F401 — re-export
from agent.note_repo import NoteRepository  # noqa: F401 — re-export
from agent.session_message_repo import (
    SessionMessageRepository,  # noqa: F401 — re-export
)

# After:
from agent.document_repo import DocumentRepository
from agent.note_repo import NoteRepository
from agent.session_message_repo import SessionMessageRepository
```

The `# noqa: F401` and `— re-export` comments are removed. The imports themselves are kept because `AgentSession.__init__` uses these classes.

### Details

The plan's scope mentions "private importに戻す" — but since these are used in `__init__` the imports must remain as regular imports. Only the `# noqa: F401` comments and re-export labels are removed.
