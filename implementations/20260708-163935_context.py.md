# Implementation: H-7 — Remove AgentContext.tool_result_store field

## Goal

Remove the `tool_result_store` field and its `ToolResultStore` import from `AgentContext`.
`ToolResultStore` stays defined in `db/tool_results.py`; `AgentContext` simply stops owning an
instance of it.

## Scope

**Target**: `scripts/agent/context.py`

**Depends on / must land together with**: every other H-7 file that reads `ctx.tool_result_store`
— removing this field first, before updating the callers, would break the whole codebase at
runtime (though not at import time, since Python attribute access is duck-typed). Coordinate
this change with the companion docs for `tool_runner.py` (already superseded by earlier H-2/H-3
docs), `llm_transport_errors.py` (already covered by the H-4 doc), `error_injection_service.py`,
`undo_service.py`, `cmd_tooling.py`, and `repl.py`'s `_persist_session_diagnostics` — see the
sibling `implementations/*_h7*.md` docs created alongside this one.

**Out of scope**: `scripts/db/tool_results.py` (the `ToolResultStore` class itself — untouched),
`scripts/db/__init__.py` exports (untouched), `tests/test_tool_result_store.py` (untouched).

## Assumptions

1. Every other H-7-scoped file's own design doc removes its `ctx.tool_result_store` reference
   independently — this doc does not need to (and should not) touch those files.
2. No other field or method in `AgentContext` reads `self.tool_result_store` internally
   (verify via grep in Procedure Step 1) — it is purely a passthrough field for external callers.

## Implementation

### Target file

`scripts/agent/context.py`

### Procedure

#### Step 1: Confirm no internal use within context.py itself

```bash
grep -n "tool_result_store" scripts/agent/context.py
```

Expected (pre-change): line 28 (import) and line 180 (assignment) only.

#### Step 2: Remove the import

Remove line 28:

```python
from db.tool_results import ToolResultStore
```

#### Step 3: Remove the field assignment and its comment

Current (lines 179-180):

```python
        # Persistent store for full tool results; /tool show <id> retrieves them
        self.tool_result_store = ToolResultStore()
```

Remove both lines entirely.

### Method

- Pure deletion; `AgentContext.__init__()`'s remaining assignments
  (`self.conv`, `self.turn`, `self.stats`, `self.workflow`, `self.cfg`, `self.session`,
  `self.diagnostics`, `self.services`) are unaffected and keep their relative order.

### Details

- `db.tool_results.ToolResultStore` remains importable and instantiable by any module that needs
  it directly (e.g. `cmd_tooling.py`, per its own H-7 doc, instantiates it locally instead of
  reading it off `ctx`).
- This change alone does not break any test that constructs a REAL `AgentContext()` and reads
  `.tool_result_store` — such a read would now raise `AttributeError`. Tests using `MagicMock()`
  stand-ins for `ctx` are unaffected by this specific file's change (MagicMock auto-creates any
  attribute), but should still be cleaned up per their own H-7 test docs for clarity.

## Risks

- Landing this file's change BEFORE the companion caller-file changes (tool_runner.py,
  llm_transport_errors.py, error_injection_service.py, undo_service.py, cmd_tooling.py, repl.py)
  will cause `AttributeError: 'AgentContext' object has no attribute 'tool_result_store'` at
  runtime wherever a REAL `AgentContext` instance is used (tests using `MagicMock()` for `ctx`
  will not catch this). Apply all H-7 file changes in the same commit/PR, not incrementally.
