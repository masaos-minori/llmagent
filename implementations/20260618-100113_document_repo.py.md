# Implementation: scripts/agent/document_repo.py

## Goal

Update `DocumentRepository.delete_document()` to use explicit 3-step delete
(chunks_vec → chunks → documents) matching ingester.py's pattern.

## Scope

- `scripts/agent/document_repo.py` — modify `delete_document()` to add explicit `DELETE FROM chunks`

## Assumptions

1. Option B: duplicate the 3-step logic (no cross-module import from rag.ingestion.ingester).
2. Current code skips explicit `DELETE FROM chunks`, relying on ON DELETE CASCADE from documents.
   Adding explicit chunks delete is safe and makes order explicit.
3. No behavior change for callers — same rows deleted in same net effect.

## Implementation

### Target file

`scripts/agent/document_repo.py`

### Procedure

Replace the 2-step delete (chunks_vec, documents) with 3-step (chunks_vec, chunks, documents).

### Details

Current:
```python
db.execute(
    "DELETE FROM chunks_vec"
    " WHERE chunk_id IN"
    " (SELECT chunk_id FROM chunks WHERE doc_id = ?)",
    (doc_id,),
)
db.execute("DELETE FROM documents WHERE doc_id = ?", (doc_id,))
```

New:
```python
db.execute(
    "DELETE FROM chunks_vec"
    " WHERE chunk_id IN (SELECT chunk_id FROM chunks WHERE doc_id = ?)",
    (doc_id,),
)
db.execute("DELETE FROM chunks WHERE doc_id = ?", (doc_id,))
db.execute("DELETE FROM documents WHERE doc_id = ?", (doc_id,))
```
