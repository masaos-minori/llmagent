## Goal

Add a new, fast (no server process, no `TestClient`), pytest-parametrized test module that statically
validates every tool dict in the file-server and git-server `TOOL_LIST` exports against the
post-requirement-14 schema contract: `name`/`description`/`inputSchema`/`status` well-typed and
non-empty, `config_dependent` present as `bool`, and `"requires_config"` absent. This is the static
half of requirement 18's two acceptance criteria (the request-time `enabled`/`disabled_reason` pair
is covered by the companion file, `tests/test_tools_endpoint.py`, since those fields do not exist on
`TOOL_LIST` itself — see Assumptions).

`mcp_servers.file.read_tools`, `mcp_servers.file.write_tools`, `mcp_servers.file.delete_tools`,

## Implementation

### Target file

`/home/sugimoto/llmagent/tests/test_tool_schema.py` (new file, does not exist yet)

### Procedure

1. Create the new file with a module docstring stating its purpose and the cross-reference to
   `tests/test_mcp_tools_validation.py`'s narrower, broader-coverage `config_dependent` check (per
   plan step 4).
2. Define `_SCHEMA_MODULES: list[tuple[str, str]] = [("mcp_servers.file.read_tools", "TOOL_LIST"),
   ("mcp_servers.file.write_tools", "TOOL_LIST"), ("mcp_servers.file.delete_tools", "TOOL_LIST"),
   ("mcp_servers.git.tools", "TOOL_LIST")]`, mirroring `test_mcp_tool_schema_exports.py`'s
   `_TOOL_MODULES` constant.
3. Write one `@pytest.mark.parametrize("module_path,attr_name", _SCHEMA_MODULES)`-decorated test
   function that imports the module via `importlib.import_module`, retrieves the `TOOL_LIST`
   attribute, and for every tool dict in the list asserts (per the landing-check gate's current
   "not landed" branch):
   - Hard assertions (pass today): `name` is a non-empty `str`; `description` is a non-empty `str`;
     `inputSchema` is a `dict`; `status` is a non-empty `str`.
   - `xfail(strict=True)`-wrapped assertions (fail today, expected): `config_dependent in tool` and
     `isinstance(tool["config_dependent"], bool)`; `"requires_config" not in tool`.
4. Since `pytest.mark.xfail` cannot wrap a single assertion line inside an already-parametrized test
   function directly, structure this as either (a) a second, separately-parametrized test function
   dedicated to the `config_dependent`/`requires_config` checks, marked `xfail(strict=True)` at the
   function level via `@pytest.mark.xfail(...)` above the `@pytest.mark.parametrize` decorator, or
   (b) a single test function with a local `try/except AssertionError` pattern — prefer option (a)
   (a separate function) for clarity and to match `strict=True`'s intended semantics (the whole
   parametrized case is expected to fail, not part of it).
5. Once requirement 14 lands (per the landing-check gate doc), remove the `xfail` marker from the
   dedicated function and merge its assertions back into the main per-tool loop if desired (optional
   cleanup, not required for correctness).

### Method

Pseudocode (no production code blocks per design-doc convention):

```
_SCHEMA_MODULES: list[tuple[str, str]] = [
    ("mcp_servers.file.read_tools", "TOOL_LIST"),
    ("mcp_servers.file.write_tools", "TOOL_LIST"),
    ("mcp_servers.file.delete_tools", "TOOL_LIST"),
    ("mcp_servers.git.tools", "TOOL_LIST"),
]

@pytest.mark.parametrize("module_path,attr_name", _SCHEMA_MODULES)
def test_static_schema_required_fields(module_path, attr_name) -> None:
    mod = importlib.import_module(module_path)
    tool_list = getattr(mod, attr_name)
    for tool in tool_list:
        assert isinstance(tool["name"], str) and tool["name"]
        assert isinstance(tool["description"], str) and tool["description"]
        assert isinstance(tool["inputSchema"], dict)
        assert isinstance(tool["status"], str) and tool["status"]

@pytest.mark.xfail(reason="depends on requirement 14 landing in scripts/mcp_servers/", strict=True)
@pytest.mark.parametrize("module_path,attr_name", _SCHEMA_MODULES)
def test_config_dependent_field_present_and_requires_config_absent(module_path, attr_name) -> None:
    mod = importlib.import_module(module_path)
    tool_list = getattr(mod, attr_name)
    for tool in tool_list:
        assert "config_dependent" in tool
        assert isinstance(tool["config_dependent"], bool)
        assert "requires_config" not in tool
```

### Details

- Import style: module-level `import importlib`, `import pytest`, matching
  `test_mcp_tool_schema_exports.py`'s existing style exactly (no lazy per-test imports needed here,
  since this file, unlike `test_mcp_tools_validation.py`, has no server-process side effects to avoid).
- Do not import `TOOL_LIST` directly at module level for all four modules — use
  `importlib.import_module` + `getattr` inside the test function body, matching the existing
  parametrized pattern, so a single bad module doesn't break collection of the whole file.
- Module docstring must state: "See also `tests/test_mcp_tools_validation.py` (requirement 14) for
  a broader-server, narrower-field `config_dependent`/`requires_config` check across all 13 MCP tool
  modules; this file is the authoritative full-field-set check for the 4 file+git modules only."
- Keep this file dependency-free of any `TestClient`/FastAPI import — the companion
  `tests/test_tools_endpoint.py` owns all endpoint-level testing; this file only imports
  `TOOL_LIST` modules.
