# Implementation: Add source-type and branch filters to /memory CLI commands

Steps covered: Plan 20260626-100129 — Steps 1-6

---

## Goal

Add `--source`, `--branch` filter options to `/memory list` and `/memory search` commands, enabling targeted inspection of memory by type and branch context.

---

## Scope

- **In scope**:
  - `/memory` command handler (in `scripts/agent/commands/`): `--source`, `--branch` options
  - `scripts/agent/memory/store.py`: `list_entries()` filter parameters
  - `docs/05_agent_07_cli-and-commands.md`: updated command reference
- **Out of scope**: UI redesign; data model changes

---

## Assumptions

- `/memory list` and `/memory search` exist as sub-commands.
- `MemoryStore.list_entries()` currently has no filter parameters.
- SQLite WHERE clause: `WHERE source_type = ? AND (branch = '' OR branch = ?)`.

---

## Implementation

### Target files
`scripts/agent/commands/cmd_memory.py` (or equivalent), `scripts/agent/memory/store.py`

### Procedure
1. Read `scripts/agent/memory/store.py` — find `list_entries()`.
2. Step 1 (`store.py`): Add filter parameters:
   ```python
   def list_entries(
       self,
       source_type: str | None = None,
       branch: str | None = None,
   ) -> list[MemoryEntry]:
       where_clauses = []
       params = []
       if source_type:
           where_clauses.append("source_type = ?")
           params.append(source_type)
       if branch is not None:
           where_clauses.append("(branch = '' OR branch = ?)")
           params.append(branch)
       ...
   ```
3. Step 2 (`cmd_memory.py`): Add `--source RULE|DECISION|FAILURE|CONVERSATION` to `/memory list`.
4. Step 3: Add `--branch <branch>` to `/memory list` and `/memory search`.
5. Step 4: Add `/memory list --branch` → passes `branch` filter to `list_entries()`.
6. Step 5: Help text for `--pinned` (existing feature).
7. Step 6: Update docs command reference.

### Method
Argument parsing addition + SQL WHERE extension.
