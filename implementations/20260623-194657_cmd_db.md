# Implementation: Admin Annotations for /db Help Tables

## Goal

Add "(admin)" annotations to admin-only subcommands in `/db help` tables and update alias note from "Compatibility aliases" to "Legacy aliases (prefer /db rag/session forms)".

## Scope

**In:**
- `scripts/agent/commands/cmd_db.py` — annotate admin-only commands in `_db_help()`, `_db_help_rag()`, `_db_help_session()`; update alias note

**Out:**
- Removing compatibility aliases
- Changing routing logic

## Assumptions

1. Admin subcommands: vacuum, checkpoint, recover, purge, health (require SQLite superuser-level access)
2. Normal subcommands: stats, urls, clean, rebuild-fts, consistency
3. Compatibility aliases must be preserved — only the note text changes

## Implementation

### Target file: `scripts/agent/commands/cmd_db.py`

### Procedure

1. Add "(admin)" suffix to admin-only rows in `_db_help()` table (lines 120-145)
2. Update alias note from "Compatibility aliases" to "Legacy aliases (prefer /db rag/session forms)" (line 151)
3. Add "(admin)" to recover row in `_db_help_rag()` (line 165)
4. Add "(admin)" to admin rows in `_db_help_session()` (lines 173-182)

### Method

Direct string modification of the Description column in each help table.
