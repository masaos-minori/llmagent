# Implementation: Add command-shadowing detection and return-type validation to plugin_registry

## Goal

`load_plugins()` warns when a plugin command shadows a built-in command; `@register_tool` validates return-type annotation at registration time; existing plugins continue loading without false positives.

## Scope

**In scope:**
- `scripts/shared/plugin_registry.py` — add `_validate_command_conflicts()`, add return-type check in `register_tool()`

**Out of scope:**
- Runtime command management CLI
- Plugin directory configurability

## Assumptions

1. `_COMMANDS` from `agent.commands.command_defs` lists all built-in command names. The validation function can import it.
2. The existing `plugins/example.py` has correct `tuple[str, bool]` annotation on `tool_echo` — confirmed.
3. `register_tool` runs at import time (module level), so the annotation is available via `typing.get_type_hints()`.

## Implementation

### Target file

`scripts/shared/plugin_registry.py`

### Procedure

Two additive changes. Both are independent.

---

### Change A: Command-shadowing detection

**Add a new function `_validate_command_conflicts()`:**

```python
def _validate_command_conflicts(strict_mode: bool = False) -> dict[str, bool]:
    """Warn when a plugin command shadows a built-in command name.

    Returns a dict mapping command_name to True when shadowing was detected.
    """
    try:
        from agent.commands.command_defs import _COMMANDS  # noqa: PLC0415 — deferred: avoid circular import at module level
    except ImportError:
        return {}

    builtin_names = frozenset(cmd.name for cmd in _COMMANDS)
    shadowed: dict[str, bool] = {}
    for name in list(_commands.keys()):
        if name in builtin_names:
            shadowed[name] = True
            msg = (
                f"Plugin command {name!r} shadows built-in command."
                " The built-in command will take precedence."
            )
            if strict_mode:
                logger.error(msg)
            else:
                logger.warning(msg)
    return shadowed
```

**Integration:** Call `_validate_command_conflicts(strict_mode)` at the end of `load_plugins()`, after `_validate_tool_conflicts()`.

**Why deferred import:** `plugin_registry.py` is in `shared/` layer. Importing from `agent/` at module level would violate the import-linter contract (`shared` must not import from `agent`). The deferred import inside the function avoids this because `load_plugins()` is called at runtime from `agent/factory.py`, which is in the agent layer.

### Change B: Return-type validation on `@register_tool`

**Modify `register_tool()` to check return annotation:**

```python
def register_tool(name: str) -> Callable[[_F], _F]:
    """Register a local async function as a tool handler; bypasses MCP entirely."""

    def decorator(fn: _F) -> _F:
        _tools[name] = fn
        hints = typing.get_type_hints(fn)
        return_hint = hints.get("return")
        if return_hint is not None and return_hint != tuple[str, bool]:
            logger.warning(
                "Plugin tool %r: expected return type 'tuple[str, bool]', got %s",
                name,
                return_hint,
            )
        logger.debug("Plugin tool registered: %s", name)
        return fn

    return decorator
```

**Add import**: `import typing` at the top of the file.

**Rationale**: Warning-only (non-blocking). This prevents false-positive validation failures for existing plugins.
