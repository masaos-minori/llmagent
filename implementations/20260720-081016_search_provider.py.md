# Implementation Procedure: scripts/mcp_servers/web_search/search_provider.py

Source plan: `plans/20260719-193357_plan.md` (Phase 2, Implementation steps item
"Update `search_duckduckgo()` in `search_provider.py`: accept/derive
`search_timeout_sec`, wrap the provider call in `asyncio.wait_for`, classify
exceptions ..., validate raw result item shape ...").

## Goal

`search_duckduckgo()` enforces an explicit, configurable timeout around the
blocking DuckDuckGo call, raises classified domain exceptions
(`WebSearchTimeoutError`, `WebSearchNetworkError`, `WebSearchProviderError`,
`WebSearchParseError`) instead of a single generic `WebSearchUpstreamError`, and
validates that each raw result item is a `dict` before constructing a
`SearchResult`, while leaving well-formed partial results (missing individual
string sub-fields) formatted exactly as today.

## Scope

**In scope:**
- `search_duckduckgo()`'s signature: add a `search_timeout_sec: float` parameter
  (or accept a `WebSearchConfig`/similarly-typed value) so the caller
  (`formatters.py:search_web()`) can thread the configured timeout in.
- Wrap `asyncio.to_thread(_sync_search)` in `asyncio.wait_for(..., timeout=...)`.
- Replace the current single `except (RuntimeError, TimeoutError)` clause with
  three-way classification: timeout → `WebSearchTimeoutError`; network-ish
  failures (`OSError`, and the existing `RuntimeError` case) →
  `WebSearchNetworkError`; anything else → `WebSearchProviderError` (fallback,
  per plan's UNK-01 resolution).
- Add a shape-validation step: for each item in the raw provider result list, if
  the item is not a `dict`, raise `WebSearchParseError` (do not silently skip).
- Import the four new exception classes from `web_search_models.py` (added there
  by a separate procedure document) instead of `WebSearchUpstreamError` alone.

**Out of scope:**
- No change to `SearchResult`'s field shape or to the module docstring's
  dependency-direction comment beyond what the new imports require.
- No change to the empty-list-is-not-an-error behavior — an empty `raw` list
  already naturally produces `[]` via the existing list comprehension; no new
  code path needed for that case (already correct today).
- No change to `formatters.py` (covered by its own procedure document).

## Assumptions

1. Current file (verified by direct read, 44 lines) imports `SearchResult` and
   `WebSearchUpstreamError` only from `web_search_models`; the new classes
   (`WebSearchTimeoutError`, `WebSearchNetworkError`, `WebSearchProviderError`,
   `WebSearchParseError`) are assumed already added there by the
   `web_search_models.py` procedure (already covered by an existing document,
   `implementations/20260720-080006_web_search_models.py.md` — not re-specified
   here).
2. `asyncio.wait_for(asyncio.to_thread(_sync_search), timeout=search_timeout_sec)`
   mirrors `scripts/mcp_servers/mdq/search.py`'s `search_docs()` pattern
   (confirmed: that function wraps `asyncio.to_thread(_search_docs_structured, ...)`
   in `asyncio.wait_for(..., timeout=service.search_timeout_sec)` and raises a
   timeout-specific error on `TimeoutError`) — same tradeoff (cancelling
   `to_thread` does not kill the underlying OS thread) is accepted here too.
3. `TimeoutError` here refers to Python's builtin `TimeoutError`, which
   `asyncio.wait_for` raises on expiry in Python 3.11+ (this repo targets
   `py313` per `rules/coding.md`) — `asyncio.TimeoutError` is an alias of the
   builtin in 3.11+, so a single `except TimeoutError` clause catches both call
   sites' timeout signal.
4. Malformed-item validation is lenient for missing sub-fields (existing
   `r.get("title", "")` etc. defaulting stays) and strict only for
   structurally-wrong entries (`raw` item is not a `dict`, e.g. `None` or a
   scalar) — per plan Assumption 5.

## Implementation

### Target file

`scripts/mcp_servers/web_search/search_provider.py`

### Procedure

1. Update the import block to pull in the four new exception classes alongside
   `SearchResult` (drop the now-unused-for-this-purpose bare
   `WebSearchUpstreamError` import if nothing else in the file still needs it —
   check whether `WebSearchUpstreamError` is referenced anywhere else in the
   file after the rewrite; if not, remove it to avoid an unused-import lint
   error).
2. Change `search_duckduckgo`'s signature to accept the timeout value as a
   parameter.
3. Replace the single `try/except` around `await asyncio.to_thread(_sync_search)`
   with `await asyncio.wait_for(asyncio.to_thread(_sync_search), timeout=...)`
   inside a `try` block with three `except` clauses in this order: `TimeoutError`
   first (most specific), then `(RuntimeError, OSError)`, then a catch-all
   `Exception` fallback — each re-raising the corresponding classified
   exception `from e`.
4. After the raw list is obtained, iterate once to validate shape before the
   existing list-comprehension construction: if any item is not a `dict`, raise
   `WebSearchParseError` (with the offending item's `type(item).__name__` or
   similar in the message) instead of letting `r.get(...)` raise
   `AttributeError` deep inside the comprehension.
5. Leave the final `SearchResult(...)` construction and the
   `for r in raw` comprehension otherwise unchanged (defaults for missing string
   fields stay via `.get(key, "")`).

### Method

Pure control-flow rewrite of one `async def` function; no new classes, no new
module-level state. Signature change is the only public-API-visible change —
callers (`formatters.py:search_web()`) must pass the timeout value explicitly
(covered by the `formatters.py` procedure document).

### Details

Illustrative signature and control flow (pseudocode, not production code):

```python
async def search_duckduckgo(
    query: str, max_results: int, search_timeout_sec: float
) -> list[SearchResult]:
    def _sync_search() -> list[dict]:
        with DDGS() as ddgs:
            return list(ddgs.text(query, max_results=max_results))

    try:
        raw = await asyncio.wait_for(
            asyncio.to_thread(_sync_search), timeout=search_timeout_sec
        )
    except TimeoutError as e:
        raise WebSearchTimeoutError(f"... timed out after {search_timeout_sec}s") from e
    except (RuntimeError, OSError) as e:
        raise WebSearchNetworkError(f"... failed: {e}") from e
    except Exception as e:  # fallback classification, UNK-01
        raise WebSearchProviderError(f"... failed: {e}") from e

    for item in raw:
        if not isinstance(item, dict):
            raise WebSearchParseError(f"unexpected result item type: {type(item).__name__}")

    return [SearchResult(title=r.get("title", ""), url=r.get("href", ""), body=r.get("body", ""), provider="duckduckgo") for r in raw]
```

Note: `except Exception` as a fallback is intentionally broad per the plan's
UNK-01 recommended default (no bare `except:` — this is `except Exception`,
which passes the `ast-grep --pattern 'except: $$$'` constraint check since that
pattern matches only the bare form).
