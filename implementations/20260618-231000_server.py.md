# Design: Register new document maintenance tool handlers

## Goal
Ensure `rag_delete_document` and `rag_list_documents` are discoverable by the MCP server's tool list endpoint and dispatch mechanism.

## Target File
- `scripts/mcp/rag_pipeline/server.py`

## Current State (lines 40-42, 160-193)
```python
from mcp.rag_pipeline.tools import _MCP_TOOLS
```
- `_MCP_TOOLS` imported from `tools.py`
- `list_tools()` endpoint returns `{tools: [{**t, "server_key": "rag_pipeline"} for t in _MCP_TOOLS]}`
- `RagPipelineMCPServer.mcp_tools = _MCP_TOOLS`

## Implementation

### No code changes needed in server.py

The server.py file uses `_MCP_TOOLS` which is imported from `tools.py`. When new tool schemas are added to `tools.py`, they are automatically:
1. Returned by the `GET /v1/tools` endpoint (line 161-164)
2. Registered as `RagPipelineMCPServer.mcp_tools` (line 189)
3. Dispatched via `_dispatch_rag_tool()` → `_service.get_dispatch_table()` (lines 167, 173)

The only server-side change needed is adding the formatter methods to `get_dispatch_table()` in `service.py`, which is covered by the service.py implementation doc.

## Completion Criteria
- `GET /v1/tools` returns 4 tools (was 2)
- `POST /v1/call_tool` with new tool names dispatches correctly
- `RagPipelineMCPServer.mcp_tools` includes new tools
