# Implementation: agent/workflow/approval_ops.py — Add count and find-by-id helpers

## Goal

Add `count_pending_approvals()` and `find_approval_by_id()` to `approval_ops.py` so that `cmd_workflow.py` can check the count before resolving and target a specific approval by ID.

## Scope

**In**: Two new functions: `count_pending_approvals(db)` and `find_approval_by_id(db, approval_id)`.

**Out**: Changes to `cmd_workflow.py`, tests.

## Assumptions

1. `approvals` table has: `approval_id`, `task_id`, `stage_id`, `status`, `reason`, `created_at`, `resolved_at`.
2. `count_pending_approvals()` counts all rows where `status='pending'` across all tasks/sessions.
3. `find_approval_by_id()` returns `ApprovalRecord | None` for the given `approval_id`.
4. Both functions follow the same pattern as existing `find_latest_pending_approval()`.

## Implementation

### Target file
`scripts/agent/workflow/approval_ops.py`

### Procedure
1. Add `count_pending_approvals()`.
2. Add `find_approval_by_id()`.

### Method

```python
def count_pending_approvals(db: SQLiteHelper) -> int:
    """Return the count of globally pending approvals."""
    rows = db.fetchall(
        """
        SELECT COUNT(*)
        FROM tasks t
        JOIN approvals a ON t.task_id = a.task_id
        WHERE t.status = 'pending_approval'
          AND a.status = 'pending'
        """,
        (),
    )
    return int(rows[0][0])


def find_approval_by_id(db: SQLiteHelper, approval_id: str) -> ApprovalRecord | None:
    """Return the ApprovalRecord for the given approval_id, or None if absent."""
    rows = db.fetchall(
        "SELECT * FROM approvals WHERE approval_id=?",
        (approval_id,),
    )
    if not rows:
        return None
    r = rows[0]
    return ApprovalRecord(
        approval_id=r["approval_id"],
        task_id=r["task_id"],
        stage_id=r["stage_id"],
        status=r["status"],
        reason=r["reason"],
        created_at=r["created_at"],
        resolved_at=r["resolved_at"],
    )
```

### Details

- `count_pending_approvals()` uses a JOIN with `tasks` to ensure only pending-approval tasks are counted (same condition as `find_pending_approval_by_session()`).
- `find_approval_by_id()` does not join with tasks — it just fetches by primary key. The caller (`cmd_workflow.py`) must verify that the approval belongs to the correct context if needed.
