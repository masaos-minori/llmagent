# Implementation: H-2 — Inject shared McpServerHealthRegistry in factory.py

## Goal

Create a single `McpServerHealthRegistry` instance in `_build_tool_executor()`, wire it
into `ToolExecutor` via `set_health_registry()`, and pass it to `AppServices` in
`build_agent_context()` so that `/mcp status` and transport dispatch observe the same
health state object.

## Scope

**Target**: `scripts/agent/factory.py`

**Steps covered**: Plan H-2 steps 1, 2, 3.

**Out of scope**: `context.py` (field already exists), `ToolExecutor`, `McpServerHealthRegistry`
internals, any other file.

## Assumptions

1. `McpServerHealthRegistry` is importable from `shared.mcp_config` (it is re-exported
   there from `shared.mcp_health`; confirmed by `shared/mcp_config.py` line 16).
2. `McpServerHealthRegistry()` takes no required constructor arguments (zero-arg
   construction; confirmed by `shared/mcp_health.py` — to be verified before edit).
3. `ToolExecutor.set_health_registry(registry)` exists and sets `_health_registry`
   (confirmed: `scripts/shared/tool_executor.py` line 131).
4. `AppServices(health_registry=...)` accepts the keyword argument (confirmed:
   `scripts/agent/context.py` line 142).
5. Only one callsite unpacks `_build_tool_executor()`; the return-type change is local
   to factory.py.

## Implementation

### Target file

`scripts/agent/factory.py`

### Procedure

#### Step 1: Add import

In the `from shared.mcp_config import McpServerConfig` line, extend to:

```python
from shared.mcp_config import McpServerConfig, McpServerHealthRegistry
```

Alternatively, add a separate import line immediately after:

```python
from shared.mcp_health import McpServerHealthRegistry
```

Prefer `shared.mcp_health` over `shared.mcp_config` for clarity of source.

#### Step 2: Update `_build_tool_executor()` signature and body

Current signature:
```python
def _build_tool_executor(
    ctx: AgentContext,
    http: httpx.AsyncClient,
) -> tuple[ToolExecutor, LifecycleManagerProtocol]:
```

New signature:
```python
def _build_tool_executor(
    ctx: AgentContext,
    http: httpx.AsyncClient,
) -> tuple[ToolExecutor, LifecycleManagerProtocol, McpServerHealthRegistry]:
```

Insert after `tools = ToolExecutor(...)` construction:
```python
registry = McpServerHealthRegistry()
tools.set_health_registry(registry)
```

Update the return statement:
```python
return tools, lifecycle, registry
```

#### Step 3: Update the callsite in `build_agent_context()`

Change:
```python
tools, lifecycle = _build_tool_executor(ctx, http)
```

To:
```python
tools, lifecycle, health_registry = _build_tool_executor(ctx, http)
```

Pass `health_registry` to `AppServices`:
```python
ctx.services = AppServices(
    http=http,
    llm=llm,
    tools=tools,
    lifecycle=lifecycle,
    hist_mgr=hist_mgr,
    audit_logger=audit_logger,
    memory=memory,
    gateway=gateway,
    health_registry=health_registry,
)
```

### Method

- Minimal change: 3 edits to factory.py (import, `_build_tool_executor` body + return
  type, `build_agent_context` unpack + AppServices constructor call).
- No new modules, no protocol changes, no config keys.

### Details

- `McpServerHealthRegistry` zero-arg constructor: verify in `shared/mcp_health.py`
  before implementing. If constructor requires args, read them from
  `ctx.cfg.mcp.mcp_servers`.
- `ToolExecutor.set_health_registry()` must be called before `set_lifecycle()` is
  called, because lifecycle callbacks may observe health state during startup. Current
  order in `_build_tool_executor()`: construct `ToolExecutor` → `_ServerLifecycleRouter`
  → `tools.set_lifecycle(lifecycle)`. Insert `registry` creation and
  `tools.set_health_registry(registry)` between `ToolExecutor` construction and
  `_ServerLifecycleRouter` construction.
- The registry instance is shared by reference: `ctx.services.health_registry is
  tools._health_registry` must hold after build.
