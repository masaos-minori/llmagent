# 04_mcp_00_document-guide — Implementation Procedure

## Goal
MCPドキュメント群のナビゲーションガイドを作成する。AIクエリルーティングテーブルと正規ソースルールを提供し、読者が目的の情報に最短でたどり着けるようにする。

## Scope
- 既存の全 04_mcp_*.md ドキュメント（完成後）
- 04_spec_mcp.md, 06_ref-mcp.md, 04_mcp-protocol.md など現行ソース

## Assumptions
- ドキュメントセットは 01〜06 + 90 + 99 の計8ファイルで構成される
- AI向けクエリルーティングは「質問の意図」×「対象ファイル」のマトリクスで表現する
- 正規ソースルール: 仕様は 01、プロトコル詳細は 02、ルーティングは 03、サーバー仕様は 04、セキュリティは 05、運用は 06

## Implementation

### Target file
`docs/04_mcp_00_document-guide.md`

### Procedure
- ドキュメント一覧テーブル（ファイル名・タイトル・概要）を作成する
- AIクエリルーティングテーブルを作成する（質問カテゴリ→参照先ファイル番号）
- 正規ソースルール（どの情報がどのファイルに属するか）を明文化する
- 関連ドキュメントへのクロスリファレンスリンクを設ける

### Method
既存の 00_document-guide パターン（RAG版、Agent版）を参考に同一フォーマットで作成する。

### Details
- クエリルーティングテーブルの行: サーバー一覧/ポート→04、認証/セキュリティ→05、起動/運用→06、プロトコル仕様→02、ルーティングロジック→03、既知バグ→90
- 正規ソースルール: 重複記述を避け、各トピックの「唯一の真実の源」を宣言する
- ドキュメント改訂時の更新対象ファイルを明示する

## Validation plan
- 全8ファイルへのリンク/参照が網羅されていることを確認する
- クエリルーティングテーブルが実際の質問パターンを網羅していることをレビューする
- 40行以内に収まっていることを確認する
