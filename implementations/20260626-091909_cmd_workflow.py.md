# Implementation: Show restored pending approval in cmd_workflow.py

Steps covered: Plan 20260626-091909 — Step 2-2

---

## Goal

When a pending approval is restored at startup (step 2-1), display a startup notice and update `/workflow status` output to show the restored pending state.

---

## Scope

- **In scope**: `scripts/agent/commands/cmd_workflow.py` — startup notice + status output
- **Out of scope**: startup.py changes (step 2-1); docs (step 3-1)

---

## Assumptions

- `ctx.workflow.approval_pending` is set by `startup.py:_recover_pending_approvals()` before REPL starts.
- `cmd_workflow.py` handles `/workflow status` and displays workflow state.
- A startup banner or "on_session_start" hook exists in the REPL where a one-time notice can be emitted.

---

## Implementation

### Target file
`scripts/agent/commands/cmd_workflow.py`

### Procedure
1. Read `scripts/agent/commands/cmd_workflow.py` to find: (a) the `/workflow status` handler, (b) any on-startup print hook.
2. In the `/workflow status` handler, add a restored-pending branch:
   ```python
   if ctx.workflow.approval_pending:
       task_id = ctx.workflow.pending_approval_task_id or "unknown"
       print(f"[workflow] Pending approval restored from prior session — task: {task_id}")
       print("  Run /approve <reason> or /reject <reason> to resolve.")
   ```
3. If there is a startup notice mechanism, emit once:
   ```
   [startup] Pending workflow approval detected (task: {task_id}). Use /approve or /reject.
   ```

### Method
Display-only change. No DB writes. Uses existing `ctx.workflow` fields.
