# Implementation Procedure: Normalize configuration schema — remove legacy Memory keys and GitOps security fields

## Goal

Remove deprecated Memory configuration keys (`use_memory_layer`, `memory_jsonl_dir`, `memory_embed_enabled`) and clarify/remove GitOps security fields (`gitops_force_push_blocked`, `gitops_protected_branches`). Add removed-key validation, include original exception messages in `ConfigLoadError`, and add reload preflight validation.

## Assumptions

1. Removed keys should be rejected at both load time (via `_FORBIDDEN_KEYS`) and reload time (via preflight validation).
2. `gitops_push_blocked` is retained as hot-reloadable; `gitops_force_push_blocked` and `gitops_protected_branches` are removed.
3. `ConfigLoadError` signature change adding optional `cause` parameter is backward-compatible.
4. No circular import risk when adding `_REMOVED_KEYS` constant to `config_loader.py` and `config_reload.py`.

## Implementation

### Target file

`scripts/shared/config_errors.py`

### Procedure

#### Step 1: Enhance ConfigLoadError with original exception message

Update `ConfigLoadError` in `config_errors.py` to include the original error:

```python
class ConfigLoadError(RuntimeError):
    """Raised when configuration files cannot be loaded."""

    def __init__(self, message: str, cause: Exception | None = None) -> None:
        if cause is not None:
            full_message = f"Config load failed ({type(cause).__name__}: {cause}): {message}"
        else:
            full_message = message
        super().__init__(full_message)
```

Update `load_config()` to pass the original exception:

```python
def load_config() -> dict[str, Any]:
    try:
        config: dict[str, Any] = ConfigLoader().load_all()
        return config
    except (OSError, ValueError, TypeError) as e:
        raise ConfigLoadError(f"Config load failed: {e}", cause=e) from e
```

### Method

- Use keyword-only `cause` parameter with default `None` for backward compatibility
- Include type name and message of original exception for debugging context
- Chain exceptions using `from e` for proper exception chaining

### Details

**Type safety considerations:**
- `cause: Exception | None` uses union type annotation compatible with Python 3.13
- Backward-compatible: callers that don't pass `cause` get the same behavior as before

**Import considerations:**
- No new imports required; `Exception` is built-in

**Backward compatibility:**
- `ConfigLoadError(message)` without `cause` works exactly as before
- Callers that pass `cause` get enhanced error messages
