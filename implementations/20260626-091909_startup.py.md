# Implementation: Restore pending approvals at startup in startup.py

Steps covered: Plan 20260626-091909 — Steps 1-1, 2-1

---

## Goal

At agent startup, detect any `workflow_approvals` DB record left in `pending` state from a prior session and restore `WorkflowContext.approval_pending = True`, so `/approve` and `/reject` work without requiring the user to re-trigger the workflow.

---

## Scope

- **In scope**: `scripts/agent/startup.py` — add `_recover_pending_approvals()` call in startup sequence
- **Out of scope**: workflow re-execution logic; cmd_workflow.py display (step 2-2)

---

## Assumptions

- `StateStore.find_latest_pending_approval()` exists and returns the most recent `WorkflowApproval` with `status="pending"`, or `None`.
- `WorkflowContext` has a settable `approval_pending: bool` field and a `pending_approval_task_id: str | None` field.
- `ctx.workflow` is initialized before `_recover_pending_approvals()` is called.
- Existing `startup.py:184` has a `_recover_pending_approvals()` stub or hook point.

---

## Implementation

### Target file
`scripts/agent/startup.py`

### Procedure
1. Read `scripts/agent/startup.py` lines 175-200 to understand the startup sequence and the hook point.
2. Implement `_recover_pending_approvals(ctx: AgentContext) -> None`:
   ```python
   def _recover_pending_approvals(ctx: AgentContext) -> None:
       pending = ctx.state_store.find_latest_pending_approval()
       if pending:
           ctx.workflow.approval_pending = True
           ctx.workflow.pending_approval_task_id = pending.task_id
           logger.info("Restored pending approval: task_id=%s", pending.task_id)
   ```
3. Call `_recover_pending_approvals(ctx)` in the startup sequence after `StateStore` is initialized and before REPL starts.

### Method
Minimal logic: one DB query, two field assignments, one log line. No new dependencies.

### Details

Failure modes to consider:
- DB unavailable at startup → `StateStore.find_latest_pending_approval()` raises → let it propagate (startup already fails-fast on DB errors).
- Multiple pending approvals → `find_latest_pending_approval()` returns the most recent → only the latest is restored; older ones remain in DB as `pending` but are stale.
