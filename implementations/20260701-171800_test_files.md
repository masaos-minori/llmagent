# Implementation: Update _MCP_TOOLS references in 7 test files

## Goal

Replace all `_MCP_TOOLS` references in test files with `TOOL_LIST` so that tests
import from the renamed canonical export.

## Scope

- Target files:
  - tests/test_mdq_metadata_consistency.py
  - tests/test_cicd_mcp_service.py (lines 577, 579, 585, 587)
  - tests/test_github_config_consistency.py — already uses TOOL_LIST; verify only
  - tests/test_mdq_routing.py (lines 19-22, 67-71)
  - tests/test_github_tool_registry.py — already uses TOOL_LIST; verify only
  - tests/test_mcp_tools_validation.py
  - tests/test_tool_registry_counts.py

## Assumptions

- tools.py and server.py renames (Steps 1-2) are complete before this step;
  importing `_MCP_TOOLS` will fail at runtime once tools.py is updated.
- test_github_config_consistency.py and test_github_tool_registry.py already use
  TOOL_LIST; no code change needed, only a grep check.
- test_mdq_routing.py line 71 contains a string literal referencing `_MCP_TOOLS`
  in an error message: update that string to `TOOL_LIST` as well.

## Implementation

### Procedure

For each target file requiring changes:
1. Replace `from mcp.<name>.tools import _MCP_TOOLS` → `from mcp.<name>.tools import TOOL_LIST`.
2. Replace all variable usages `_MCP_TOOLS` → `TOOL_LIST` in the file body.
3. Update any string literals that reference `_MCP_TOOLS` (e.g., error messages, docstrings).

For verify-only files (test_github_config_consistency.py, test_github_tool_registry.py):
- Run `grep _MCP_TOOLS <file>`; expect zero matches; no edit needed.

### Method

Use replace_all for each file: `_MCP_TOOLS` → `TOOL_LIST`.
