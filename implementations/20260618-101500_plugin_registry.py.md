# Implementation: shared/plugin_registry.py — structured plugin load result

## Goal

Change `load_plugins()` return type from `int` to `PluginLoadResult` (frozen dataclass)
so callers can programmatically check failures, not just parse logs.

## Changes

### `scripts/shared/plugin_registry.py`
- Add `from dataclasses import dataclass` to imports
- Add `PluginFailure(path: str, error: str)` frozen dataclass before `load_plugins()`
- Add `PluginLoadResult(loaded_count: int, failed: tuple[PluginFailure, ...])` frozen dataclass
- Change `load_plugins()` return type `int` → `PluginLoadResult`
- Add `failures: list[PluginFailure] = []` before import loop
- In `except` block: append `PluginFailure(path=py_file.name, error=f"Plugin load failed ({py_file.name}): {type(e).__name__}: {e}")` to `failures`
- Return `PluginLoadResult(loaded_count=loaded, failed=tuple(failures))` instead of `return loaded`

### `scripts/agent/factory.py`
- `n_plugins = plugin_registry.load_plugins(...)` → `result = plugin_registry.load_plugins(...)`
- `if n_plugins:` → `if result.loaded_count:`
- `"Loaded %s plugin(s)"` with `n_plugins` → `"Loaded %d plugin(s)"` with `result.loaded_count`

### `tests/test_plugin_registry.py`
- Rename all `n = plugin_registry.load_plugins(...)` → `result = ...`
- Replace all `assert n == X` → `assert result.loaded_count == X`
- Add `test_failure_details_included` — bad plugin → `result.failed` has `PluginFailure`
- Add `test_empty_failed_when_all_succeed` — good plugin → `result.failed == ()`
- Add `test_strict_mode_does_not_collect_failures` — strict mode raises before returning
