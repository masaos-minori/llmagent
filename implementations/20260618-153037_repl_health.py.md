# Implementation: repl_health.py — expand audit_security_defaults with fail-open/fail-closed summary

## Goal

Expand `audit_security_defaults()` in `scripts/agent/repl_health.py` to:
1. Add checks for db_allowlist (sqlite-mcp), command_allowlist (shell-mcp), and allowed_repo_paths (git-mcp).
2. Append a structured fail-open/fail-closed summary at the end of the warning list and log output.

## Scope

- `scripts/agent/repl_health.py` only.
- No changes to AgentConfig, McpServerConfig, or any MCP server files.
- No changes to `/mcp` status display (plan step 3 is covered by existing servers/ok summary).

## Assumptions

1. `agent` layer may import from `mcp` layer (enforced by `.importlinter`; existing example: `cmd_ingest.py` imports `mcp.rag_pipeline.models`).
2. `SqliteConfig.load()`, `ShellConfig.load()`, `GitConfig.load()` each call `ConfigLoader().load("<file>.toml")` and can be called independently at agent startup time.
3. Loading individual server config files at startup is acceptable (lightweight TOML read); exceptions from load() are caught and treated as "unable to check".
4. The existing `getattr(ctx.cfg, "shell_policy", None)` check always returns None (no such field on AgentConfig); it should be replaced with a direct `ShellConfig.load()` call.
5. The existing `github.allowed_workflows` check also always returns None; it should remain as-is (no `github` field on AgentConfig) — it's a no-op placeholder.

## Implementation

### Target file

`scripts/agent/repl_health.py`

### Procedure

1. Add imports at the top of the file (with existing mcp imports or after stdlib imports):
   ```python
   from mcp.git.models import GitConfig
   from mcp.shell.models import ShellConfig
   from mcp.sqlite.models import SqliteConfig
   ```

2. Inside `audit_security_defaults()`, replace the shell sandbox block with a direct `ShellConfig.load()` call.

3. Add three new check blocks (after auth_token checks, before summary):
   - shell command_allowlist (fail-closed when empty — all commands denied)
   - sqlite db_allowlist (fail-closed when empty — all DB queries denied)
   - git allowed_repo_paths (fail-closed when empty — all repo access denied)

4. Collect fail-open and fail-closed items into two lists, then append a summary section to `warnings`.

### Method

Replace the entire `audit_security_defaults()` function body below the auth_token block with the expanded version. Keep the function signature unchanged.

### Details

New logic structure (pseudocode):
```
fail_closed_empty: list[str] = []  # settings that deny when empty
fail_open_empty: list[str] = []    # settings that allow when empty

# shell sandbox
try:
    shell_cfg = ShellConfig.load()
    if shell_cfg.shell_sandbox_backend == "none":
        warnings.append("Security: shell_sandbox_backend=none (no sandbox)")
    # shell command_allowlist
    if not shell_cfg.command_allowlist:
        fail_closed_empty.append("shell.command_allowlist")
        warnings.append("Security: shell.command_allowlist is empty (fail-closed: all shell commands denied)")
except Exception:
    pass  # config file unavailable; skip check

# sqlite db_allowlist
try:
    sqlite_cfg = SqliteConfig.load()
    if not sqlite_cfg.db_allowlist:
        fail_closed_empty.append("sqlite.db_allowlist")
        warnings.append("Security: sqlite.db_allowlist is empty (fail-closed: all DB queries denied)")
except Exception:
    pass

# git allowed_repo_paths
try:
    git_cfg = GitConfig.load()
    if not git_cfg.allowed_repo_paths:
        fail_closed_empty.append("git.allowed_repo_paths")
        warnings.append("Security: git.allowed_repo_paths is empty (fail-closed: all repo access denied)")
except Exception:
    pass

# cicd workflow_allowlist (fail-open)
github_cfg = getattr(ctx.cfg, "github", None)
if github_cfg is not None:
    allowed_workflows = getattr(github_cfg, "allowed_workflows", None)
    if isinstance(allowed_workflows, (list, tuple)) and len(allowed_workflows) == 0:
        fail_open_empty.append("github.allowed_workflows")
        warnings.append("Security: github.allowed_workflows is empty (fail-open: all workflows allowed)")

# allowed_tools (fail-open)
tool_cfg = getattr(ctx.cfg, "tool", None)
if tool_cfg is not None:
    allowed_tools = getattr(tool_cfg, "allowed_tools", None)
    if isinstance(allowed_tools, (list, tuple)) and len(allowed_tools) == 0:
        fail_open_empty.append("tool.allowed_tools")
        warnings.append("Security: tool.allowed_tools is empty (all tools allowed)")

# summary
if fail_closed_empty or fail_open_empty:
    fc_str = ", ".join(fail_closed_empty) if fail_closed_empty else "none"
    fo_str = ", ".join(fail_open_empty) if fail_open_empty else "none"
    summary = (
        f"Security posture summary — "
        f"fail-closed (deny when empty): {fc_str}; "
        f"fail-open (allow when empty): {fo_str}"
    )
    warnings.append(summary)
    logger.warning(summary)
```

Note: remove the `for w in warnings: logger.warning(w)` block at the end — each warning is now logged where it is appended, to avoid double-logging the summary.
