## Goal

Fix `watchdog_loop()` in `agent/repl_health.py` to return immediately when `mcp_watchdog_interval <= 0`, eliminating the current infinite spin when the interval is zero.

## Scope

- In-Scope:
  - Add an early `return` after the disabled-interval log in `watchdog_loop()`
  - Move `while True: await asyncio.sleep(interval)` to run only when `interval > 0`
- Out-of-Scope:
  - Changes to `_watchdog_check_http()` (covered in Step 5)
  - Changes to `repl.py`'s `_start_watchdog()` guard

## Assumptions

1. Current code (lines 346-379): `watchdog_loop()` logs a warning when `interval <= 0` but does NOT return; it falls through to `while True: await asyncio.sleep(interval)` which with `interval=0` calls `asyncio.sleep(0)` in an infinite coroutine-spin.
2. `repl.py:_start_watchdog()` already checks `watchdog_interval > 0` before creating the task, so in normal operation `watchdog_loop(interval=0)` is never called. The fix is defense-in-depth.
3. The early `return` must come after the log message so the disabled state is observable in logs.
4. The `while True` loop structure remains unchanged for the normal `interval > 0` path.

Current code:

```python
    max_restarts = ctx.cfg.mcp.mcp_watchdog_max_restarts

```

Replace with:
```python
async def watchdog_loop(ctx: AgentContext) -> None:
    interval = ctx.cfg.mcp.mcp_watchdog_interval
    max_restarts = ctx.cfg.mcp.mcp_watchdog_max_restarts
    restart_counts: dict[str, int] = {}
    if interval <= 0:
        logger.warning(
            "Watchdog: disabled (interval=%.0f) — failed servers will not be auto-restarted",
            interval,
        )
        return
    logger.info(
        "Watchdog: enabled (interval=%.0fs, max_restarts=%d)",
        interval,
        max_restarts,
    )
    while True:
        await asyncio.sleep(interval)
        for key, srv_cfg in ctx.cfg.mcp.mcp_servers.items():
            if srv_cfg.transport == "http":
                await _watchdog_check_http(
                    ctx,
                    key,
                    srv_cfg,
                    restart_counts,
                    max_restarts,
                )
        if ctx.services_required.lifecycle is not None:
            await ctx.services_required.lifecycle.shutdown_idle()
```

5. Verify the `while True` block body (the `for` loop and `lifecycle.shutdown_idle()` call) is preserved exactly.

### Method

- Invert the `if interval > 0` guard to `if interval <= 0:` with an early return.
- This restructuring avoids the `if/else` split that leaves the `while True` reachable in both branches.
- The log message text is unchanged from the original (same wording for observability continuity).

### Details

- Original lines 356-379 in `watchdog_loop()`.
- The `while True:` loop starts at line 367 in the original; after restructuring it runs only when the function did not return early.
- `asyncio.sleep(0)` would yield control once per event loop tick but loop forever, consuming CPU; the early return fully prevents this.
- No new imports required.
