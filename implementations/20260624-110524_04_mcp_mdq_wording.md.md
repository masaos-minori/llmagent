# Implementation: Standardize mdq-mcp Wording Across MCP Docs

## Goal

Replace all inconsistent mdq-mcp status terms ("stub", "not implemented", "placeholder", "experimental") with the approved terminology across 4 MCP documents. Add an explanation of `stub: true` config field.

## Scope

**In:**
- `docs/04_mcp_00_document-guide.md`
- `docs/04_mcp_04_server_catalog.md`
- `docs/04_mcp_07_mdq_rag_boundary.md`
- `docs/04_mcp_90_inconsistencies_and_known_issues.md`

**Out:** No code changes.

## Assumptions

1. mdq-mcp FTS5 search/index is implemented but not production-validated.
2. `stub: true` means "not auto-started in production", not "unimplemented".
3. Current docs use mixed terms; all should be replaced with approved terms.

## Implementation

### Target file

`docs/04_mcp_00_document-guide.md`, `docs/04_mcp_04_server_catalog.md`, `docs/04_mcp_07_mdq_rag_boundary.md`, `docs/04_mcp_90_inconsistencies_and_known_issues.md`

### Procedure

1. Run audit command to find all current non-standard terms:
   ```bash
   grep -rn "stub\|experimental\|not implemented\|placeholder" docs/04_mcp_*.md | grep -i mdq
   ```
2. For each match, determine the correct approved term based on context:
   - Implemented feature but not prod-validated → "experimental but functional"
   - General mdq-mcp label → "not production-validated"
   - True stub with no implementation → "placeholder / missing implementation"
3. Apply Edit tool patches for each file.
4. In `docs/04_mcp_04_server_catalog.md`, add an explanation of `stub: true`:
   ```
   `stub: true` — サーバーは自動起動しない (not auto-started in production)。実装がないという意味ではない。
   ```

### Method

Bash audit → Read affected lines → Edit tool patches per file.

### Details

**Approved terminology mapping:**

| Term to replace | Replace with | Context |
|---|---|---|
| `stub` (as status) | `experimental but functional` | mdq-mcp FTS5 search |
| `not implemented` | `experimental but functional` or `placeholder / missing implementation` | depends on whether code exists |
| `experimental` (standalone) | `experimental but functional` | where functionality exists |
| `placeholder` | `placeholder / missing implementation` | true stub tools |

**`stub: true` explanation (add to `04_mcp_04` server entry):**
```markdown
> **Note:** `stub: true` は本番環境で自動起動しない設定を意味する。実装がないという意味ではない。
> mdq-mcp の FTS5 検索・インデックス機能は実装済みだが、本番検証は未実施 (experimental but functional)。
```
