# Implementation: cmd_skill.py — `_SkillMixin` handler + registry wiring

Source plan: `plans/20260712-174955_plan.md` (Implementation Steps, Phase 2)

## Goal

Implement the `/skill` command handler: list `skills/` subdirectory names with no
argument, or inject a skill's `SKILL.md` content as an ephemeral system message
(`_ephemeral: True`, `_skill_ephemeral: True`) with a given name, replacing any prior
`/skill`-injected message in the same conversation.

## Scope

**In scope**
- New file `scripts/agent/commands/cmd_skill.py` defining `_SkillMixin(MixinBase)` with
  `_skills_dir()` and `_cmd_skill(args)`.
- Edits to `scripts/agent/commands/registry.py`: import `_SkillMixin`, add it to
  `CommandRegistry`'s base class list, update the module docstring's mixin table.

**Out of scope**
- The `CommandDef` registration itself — covered in
  `implementations/20260712-180631_command_defs_list.md` (Phase 1); both must land
  together (see that document's Assumptions §1).
- Test file — covered in a separate document (Phase 3).
- Any `deploy/deploy.sh` change — confirmed unnecessary; `deploy/deploy.sh:68-73` rsyncs
  `scripts/` wholesale (plan's UNK-05).

## Assumptions

1. `repo_root` is resolved from `scripts/agent/commands/cmd_skill.py` via
   `Path(__file__).resolve().parent.parent.parent.parent` — 4 `.parent` calls climb
   `commands -> agent -> scripts -> repo_root`. This mirrors `config_loader.py:49`'s
   `parent.parent.parent` (which climbs from `scripts/shared/`, one directory shallower
   than `scripts/agent/commands/`).
2. `MixinBase` (`scripts/agent/commands/mixin_base.py`) provides `_ctx: AgentContext` and
   `_out: OutputPort` type annotations/defaults; `_SkillMixin` only needs a pass-through
   `__init__` like every other mixin (e.g. `_PluginsMixin`).
3. `ctx.conv.history` is a plain `list[dict]` — appending/filtering it directly (as
   `mode_classification.py` and `orchestrator.py._handle_memory_injection` already do) is
   the established pattern; no new helper method is needed on `ConversationState`.
4. `_sync_system_prompt()` (`orchestrator.py:500-519`) already strips any message where
   `m.get("_ephemeral")` or `m.get("_memory_injected")` is truthy — adding
   `"_ephemeral": True` to the skill message is sufficient for automatic removal next turn;
   no change to `_sync_system_prompt()` is needed.
5. Skill names are directory names only; `is_dir()` is the sole membership test, correctly
   excluding loose files like `skills/DESIGN.md`.

### Details

#### A. Full content of `scripts/agent/commands/cmd_skill.py`

```python
"""agent/commands/cmd_skill.py
/skill slash command: list skills or inject a skill's SKILL.md as ephemeral system context.
"""

from __future__ import annotations

from pathlib import Path
from typing import Any

from agent.commands.mixin_base import MixinBase


class _SkillMixin(MixinBase):
    def __init__(self, *args: Any, **kwargs: Any) -> None:
        super().__init__(*args, **kwargs)

    def _skills_dir(self) -> Path:
        """repo_root/skills — repo_root is 4 levels above this file."""
        return Path(__file__).resolve().parent.parent.parent.parent / "skills"

    def _cmd_skill(self, args: str = "") -> None:
        """Handle /skill [name] [args]."""
        args = args.strip()
        if not args:
            for name in sorted(
                p.name for p in self._skills_dir().iterdir() if p.is_dir()
            ):
                self._out.write(name)
            return

        name, _, rest = args.partition(" ")
        skill_dir = self._skills_dir() / name
        if not skill_dir.is_dir():
            self._out.write(f"Unknown skill: {name}")
            return

        content = (skill_dir / "SKILL.md").read_text(encoding="utf-8")
        rest = rest.strip()
        if rest:
            content = f"{content}\n\nInvocation args: {rest}"

        ctx = self._ctx
        ctx.conv.history = [
            m for m in ctx.conv.history if not m.get("_skill_ephemeral")
        ]
        ctx.conv.history.append(
            {
                "role": "system",
                "content": content,
                "_ephemeral": True,
                "_skill_ephemeral": True,
            }
        )
```

Design notes carried over from the plan:
- No `write_table` — plain `self._out.write(name)` per directory name, since the
  requirement specifies names only, no descriptions.
- Invocation-args label is English (`"Invocation args: ..."`), confirmed with the user
  during plan review, to stay consistent with `rules/coding.md`'s English-only convention
  even though this text is LLM-facing content rather than a code comment or log line.
