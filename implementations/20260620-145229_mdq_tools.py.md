# Implementation: scripts/mcp/mdq/tools.py (update — tool descriptions)

## Goal

Update all 7 MDQ tool description strings to emphasize Markdown-only scope,
structure-aware retrieval, and experimental status, making the MDQ/RAG distinction
visible at the point of tool selection.

## Scope

**In:**
- 7 `"description"` values in `scripts/mcp/mdq/tools.py`
- Tool-level descriptions only (top-level `"description"` key per tool)

**Out:**
- Parameter-level `"description"` values (left unchanged)
- Tool names (must not change)
- Any logic or schema

## Assumptions

- Each tool definition is a dict with a top-level `"description"` key
- Parameter-level descriptions (nested under `"properties"`) are unchanged
- Description strings are single-line (no multiline format needed for conciseness)

## Implementation

### Target file

`scripts/mcp/mdq/tools.py`

### Procedure

Apply 7 targeted replacements, one per tool:

| Line | Old description | New description |
|---|---|---|
| 13 | `"Search indexed Markdown sections by query."` | `"Search indexed Markdown documents using BM25/FTS5. Markdown-only, structure-aware retrieval. Experimental."` |
| 42 | `"Retrieve a Markdown chunk by its ID."` | `"Retrieve a Markdown chunk by ID, with optional adjacent heading context. Markdown-only, experimental."` |
| 58 | `"Get the heading structure of a Markdown file."` | `"Get the heading hierarchy of a Markdown file. Structure-aware, Markdown-only. Experimental."` |
| 68 | `"Index a set of paths into the in-process SQLite DB."` | `"Index Markdown file paths into the MDQ store. Markdown-only ingestion. Experimental."` |
| 80 | `"Incrementally refresh the index for a set of paths."` | `"Incrementally refresh the index for changed Markdown files. Markdown-only. Experimental."` |
| 92 | `"Return document/chunk count and index metadata."` | `"Return document/chunk counts and FTS5 index metadata for the Markdown store. Experimental."` |
| 98 | `"Search Markdown chunks with a regex pattern."` | `"Search Markdown chunks with a regex pattern. Structure-aware, Markdown-only. Experimental."` |

### Method

7 individual `Edit` operations. Each replaces only the top-level description string
of that tool; surrounding keys and parameter descriptions are untouched.

### Details

- Line numbers refer to the current file state; confirm each with a read before editing
- All new descriptions end with `"Experimental."` to signal MDQ is not production-validated
- `outline` description is changed despite being noted as "(good)" in the plan —
  adding `"Experimental."` suffix maintains consistency across all MDQ tools
