# Implementation: search_diagnostics field in stage.py

## Goal
Add `search_diagnostics: SearchDiagnostics | None` field to `PipelineContext` in
`scripts/rag/stage.py` so the search stage can store per-query diagnostics for
downstream consumers.

## Scope
- Modify `PipelineContext` dataclass only; no changes to `StageResult` or `PipelineStage`
- File: `scripts/rag/stage.py`

## Assumptions
- `SearchDiagnostics` is defined in `rag.models_result` (implemented first)
- Default value is `None` — no search has run yet; stages that do not set it leave it None
- `PipelineContext` is mutable (not frozen); adding an optional field is safe

## Implementation

### Target file
`scripts/rag/stage.py`

### Procedure
1. Add import for `SearchDiagnostics` from `rag.models_result`
2. Add `search_diagnostics` field to `PipelineContext` with default `None`

### Method
Two-step edit: import then field.

### Details

**Step 1 — Add import (after existing imports):**
```python
from rag.models_result import SearchDiagnostics
```

**Step 2 — Add field to `PipelineContext`:**
```python
@dataclasses.dataclass

    """Mutable state passed between pipeline stages."""
```
