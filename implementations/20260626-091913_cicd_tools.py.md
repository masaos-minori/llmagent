# Implementation: Expose dry_run in inputSchema for cicd MCP tools

Steps covered: Plan 20260626-091913 — Steps 1-1, 1-2, 2-1

---

## Goal

Add `dry_run: bool` to the `inputSchema` of `trigger_workflow` and other applicable cicd MCP tools so that callers can request a dry-run without it being silently ignored.

---

## Scope

- **In scope**: `scripts/mcp/cicd/` tool definitions — `trigger_workflow` inputSchema and any other tools that accept `dry_run` implicitly
- **Out of scope**: tests (step 3-1); backend implementation of dry_run logic (separate concern)

---

## Assumptions

- `trigger_workflow` is defined in `scripts/mcp/cicd/tools.py` or similar.
- The `inputSchema` is a Python dict that maps to JSON Schema.
- `dry_run` is currently accepted by the backend but not declared in the schema.
- Step 1-1: identify `trigger_workflow`; step 1-2: identify other cicd tools with implicit dry_run; step 2-1: add to all identified schemas.

---

## Implementation

### Target file
`scripts/mcp/cicd/tools.py` (or equivalent cicd tool definition file)

### Procedure
1. Read the cicd tool definitions to find `trigger_workflow` and related tools.
2. Step 1-1 (Identify `trigger_workflow`): Find its `inputSchema` dict.
3. Step 1-2 (Identify other tools): `grep -n "dry_run" scripts/mcp/cicd/` to find any tool that uses `dry_run` in its implementation but not its schema.
4. Step 2-1 (Add to schemas): For each identified tool, add to `inputSchema.properties`:
   ```python
   "dry_run": {
       "type": "boolean",
       "description": "If true, validate and plan the action but do not execute it.",
       "default": False,
   }
   ```
   Add `"dry_run"` to `inputSchema.required` only if it is always required (usually not — keep it optional).

### Method
Schema-only change. No backend logic changes.
