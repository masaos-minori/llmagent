# Implementation and Test Procedure: scripts/db/helper.py (reuse_connection)

## Goal

Add `reuse_connection: bool = False` parameter to `SQLiteHelper.open()` so callers can skip reconnect/close for long-lived connection patterns.

## Scope

**In:**
- `scripts/db/helper.py` — add `_reuse_connection` flag and guard in `open()` and `__exit__()`

**Out:**
- Changing the existing per-query open/close behavior (default stays the same)
- Test file creation (handled separately in `test_db_performance.py`)

## Assumptions

1. `SQLiteHelper.__init__` accepts `db_name` and config.
2. `SQLiteHelper.open()` currently always creates a new connection.
3. `SQLiteHelper.__exit__` currently always calls `self.close()`.
4. Adding `reuse_connection=True` means: if already connected, skip reconnect in `open()` and skip `close()` in `__exit__`.

## Implementation

### Target file
`scripts/db/helper.py`

### Procedure
1. Add `_reuse_connection: bool = False` to `__init__`.
2. Add `reuse_connection: bool = False` parameter to `open()`.
3. In `open()`: if `reuse_connection=True`, set `self._reuse_connection = True`; if already connected, return `self` without re-opening.
4. In `__exit__`: if `self._reuse_connection` is True, skip `close()`.

### Details

```python
# __init__ addition:
self._reuse_connection: bool = False

# open() signature change:
def open(self, write_mode: bool = True, reuse_connection: bool = False) -> "SQLiteHelper":
    if reuse_connection:
        self._reuse_connection = True
    if self._reuse_connection and self._conn is not None:
        return self  # skip reconnect
    # ... existing open logic ...
    return self

# __exit__ change:
def __exit__(self, *args: object) -> None:
    if not self._reuse_connection:
        self.close()
```
