## Goal

Implement stable chunk IDs for MDQ chunks to replace integer row IDs with deterministic identifiers based on normalized source path, heading path, ordinal, and content hash. Update all affected code paths to use the new `chunks`/`chunks_fts` schema instead of legacy `sections`/`sections_fts`.

## Scope

**In-Scope**:
- Generate stable `chunk_id` from deterministic inputs (normalized source path, heading path, ordinal, content hash)
- Update `search_docs` to query `chunks_fts` and return `chunk_id` in results
- Update `get_chunk` to accept stable `chunk_id` (str type)
- Update `grep_docs` to query `chunks` table and return `chunk_id` in matches
- Update `stats` to query `chunks`/`documents` tables
- Update models.py type annotations for chunk_id fields
- Update tools.py tool schemas for chunk_id type

**Out-of-Scope**:
- Token counting implementation (placeholder for now)
- Adding new tools or features
- Migrating existing data (migration already exists in service.py)

## Assumptions

1. chunk_id = `chunk_{SHA-256 hash of (normalized source path + heading_path + ordinal)}` â€” consistent with migration code in service.py line 220
2. normalized_content = content with extra whitespace collapsed
3. backward compatibility is not required â€” prefer stable IDs over legacy integer IDs
