# Implementation: Update `scripts/agent/startup.py` to use `WorkflowExecutionPolicy`

## Goal

Update `scripts/agent/startup.py` to create a `WorkflowExecutionPolicy` instance from
`ctx.cfg.workflow_mode` and pass it to `check_workflow_definition()` instead of the raw string.

## Scope

- In-Scope: One call site in `startup.py` where `check_workflow_definition(ctx.cfg.workflow_mode, ...)`
  is called — wrap it with `WorkflowExecutionPolicy(ctx.cfg.workflow_mode)`.
- Out-of-Scope: No changes to `AgentConfig`, no changes to `check_workflow_definition()` signature
  (that's in `repl_health_py.md`). No other changes to `startup.py`.

## Assumptions

1. `scripts/agent/workflow_execution_policy.py` exists (prerequisite: `workflow_execution_policy_py.md`).
2. `scripts/agent/repl_health.py:check_workflow_definition()` accepts both `str` and
   `WorkflowExecutionPolicy` (backward compat, per `repl_health_py.md`).
3. `startup.py` calls `check_workflow_definition(ctx.cfg.workflow_mode, ...)` exactly once.
4. `ctx.cfg.workflow_mode` is a `str`.

## Implementation

### Target file

`scripts/agent/startup.py` (existing)

### Procedure

1. Read `startup.py` to find the `check_workflow_definition()` call site.
2. Add import: `from agent.workflow_execution_policy import WorkflowExecutionPolicy`.
3. Replace `check_workflow_definition(ctx.cfg.workflow_mode, ...)` with
   `check_workflow_definition(WorkflowExecutionPolicy(ctx.cfg.workflow_mode), ...)`.
4. Run `uv run ruff check scripts/agent/startup.py` — expect 0 errors.
5. Run `uv run mypy scripts/agent/startup.py` — expect no new errors.

### Method

```python
# Add import (lazy or top-level):
from agent.workflow_execution_policy import WorkflowExecutionPolicy

# Change call site:
# Before:
check_workflow_definition(ctx.cfg.workflow_mode, workflows_dir=...)

# After:
check_workflow_definition(WorkflowExecutionPolicy(ctx.cfg.workflow_mode), workflows_dir=...)
```

### Details

- If `check_workflow_definition()` takes `workflow_mode` as a positional argument, the change
  is a drop-in replacement.
- The `WorkflowExecutionPolicy` constructor validates the mode at construction time; an invalid
  `workflow_mode` in `AgentConfig` will raise `ValueError` at startup rather than silently using
  a wrong default.
