# Implementation procedure: `scripts/shared/runtime_tool.py`

Source plan: `plans/done/20260717-124020_plan.md` (requirement `requires/20260717_02_require.md`), Implementation step 2.

## Goal

Create `scripts/shared/runtime_tool.py`, a new module defining `RuntimeTool` — a frozen dataclass that
normalizes tool execution metadata (routing, LLM schema, scheduler metadata, side-effect detection,
safety-tier, approval requirement) into a single, typed shape. This is the foundational data type that
`RuntimeToolRegistry` (separate implementation doc) will store and operate on. No consumer wiring, no
MCP discovery — this step only defines the shape and its safe-default construction logic.

## Scope

**In scope**
- New file `scripts/shared/runtime_tool.py` only.
- `RuntimeTool` frozen dataclass, 13 fields per the plan/requirement.
- A local `AgentSafetyTier` type alias (4-value `Literal`) declared in this file.
- A factory function that applies safe defaults for omitted annotation fields.

**Out of scope**
- `RuntimeToolRegistry` (separate file/doc).
- Any change to `scripts/shared/tool_spec.py`, `scripts/shared/tool_registry.py`,
  `scripts/shared/tool_executor_helpers.py`, `scripts/agent/tool_policy.py`, or `scripts/agent/tool_enums.py`.
- Any change to `scripts/agent/context.py` (tracked separately; per this workflow's filename-match check,
  docs already exist under `implementations/done/` for `context.py`, so it is treated as already covered).
- MCP tool discovery / population of live data (requirement 03).

## Assumptions

1. **`agent_safety_tier`'s literal vocabulary must match `scripts/agent/tool_policy.py:42-47`'s
   `_TIER_TO_RISK` dict keys** — confirmed verbatim:
   ```
   43: _TIER_TO_RISK: dict[str, RiskLevel] = {
   44:     "READ_ONLY": RiskLevel.NONE,
   45:     "WRITE_SAFE": RiskLevel.NONE,
   46:     "WRITE_DANGEROUS": RiskLevel.MEDIUM,
   47:     "ADMIN": RiskLevel.HIGH,
   ```
   These 4 tier-name strings are bare string literals in `tool_policy.py` (not enum members) — only the
   risk-level *value* side is an enum (`agent.tool_enums.RiskLevel`).

2. **Correction to the plan's Risk-mitigation text**: the plan's Risks section suggests importing the
   tier vocabulary from `tool_enums.py` "since `RiskLevel`/`OperationType` already live there." Direct
   investigation shows this is imprecise: `RiskLevel`/`OperationType` live in **`scripts/agent/tool_enums.py`**
   (agent layer), not `scripts/shared/tool_enums.py` (no such file exists). Per `.importlinter`'s
   `shared-is-leaf` contract (`shared` must not import from `agent`), `scripts/shared/runtime_tool.py`
   **must not** import `agent.tool_enums`. Therefore `AgentSafetyTier` must be declared as a **local**
   `Literal["READ_ONLY", "WRITE_SAFE", "WRITE_DANGEROUS", "ADMIN"]` type alias inside `runtime_tool.py`
   itself — duplicating the 4 string values (acceptable, since the whole point per the plan's own
   Assumption 1 is that this is a temporary, additive, not-yet-unified module).

3. `ToolSpec` (`scripts/shared/tool_spec.py:12-30`) is confirmed unmodified and has 6 fields:
   `call_id: str`, `name: str`, `args: dict[str, object]`, `resource_scope: str = ""`,
   `requires_serial: bool = False`, `is_write: bool = False`. `RuntimeTool` is a superset for the
   `is_write`/`requires_serial`/`resource_scope` fields (used later by `RuntimeToolRegistry.tool_spec_map()`).

4. Raw MCP tool shape (`MCPToolSchema`, `scripts/mcp_servers/mdq/tools.py:12-18`) has `name`,
   `description`, `inputSchema`, `status`, plus optional `is_write`/`requires_serial` — this is the shape
   `raw_definition` holds verbatim, and the shape the factory function's parameters are normalized from.
   `server_key` is attached separately by `scripts/mcp_servers/server.py:128-136`'s
   `list_tools_with_server_key()`, not part of the per-tool dict itself.

5. No default value is specified by the plan for `is_write` itself when omitted (the plan's "Safe
   defaults" section only fixes `agent_safety_tier`, `requires_approval`, `enabled_for_llm`, and
   `requires_serial`). Decision for this implementation: default `is_write=False` when not explicitly
   supplied (least-surprising — "not known to be a write" rather than assuming write), while
   `requires_serial` still defaults to `True` whenever `is_write` was *not explicitly supplied* (per the
   plan's literal wording: "`requires_serial` defaults to `True` whenever `is_write` is also
   unannotated/defaulted"). This means the factory function needs an internal "was `is_write` explicitly
   passed" signal — implemented by giving the factory's `is_write` parameter a sentinel default of `None`
   (not `False`), distinct from the dataclass field's own type (`bool`, no `None`).

## Implementation

### Target file

`scripts/shared/runtime_tool.py` (new).

### Procedure

1. Add module docstring stating: this module defines the normalized runtime tool-metadata shape;
   it does not yet have any consumer (requirements 03-07 wire it in); `agent_safety_tier`'s literal
   values intentionally duplicate (not import) `scripts/agent/tool_policy.py`'s tier vocabulary to
   respect the `shared`-is-leaf import-layer contract.
2. Define `AgentSafetyTier = Literal["READ_ONLY", "WRITE_SAFE", "WRITE_DANGEROUS", "ADMIN"]`.
3. Define `RuntimeTool` as `@dataclass(frozen=True)` with the 13 fields (see Details).
4. Define a module-level factory function `build_runtime_tool(...)` (not a classmethod, to keep
   `RuntimeTool` itself a plain data container per the "Design for Immutability" rule) that accepts
   optional annotation parameters and applies the safe-default rules from Assumption 5 and the plan's
   "Safe defaults" section before constructing the frozen instance.
5. No runtime validation of `agent_safety_tier` against the `Literal` is added (mypy/pyright enforce it
   statically only) — do not add an `if value not in (...)` runtime guard unless a caller path proves
   untrusted string input needs validation at a boundary (none identified in this step's scope; MCP
   discovery/config parsing, where such a boundary would live, is requirement 03/07's concern).

### Method

Plain dataclass + module-level factory function. No `Protocol`/`ABC` needed (no polymorphism required at
this step — `RuntimeToolRegistry` will hold concrete `RuntimeTool` instances only).

```
    name: str,
    server_key: str,
    server_url: str = "",
    description: str = "",
    input_schema: dict[str, object] | None = None,
    raw_definition: dict[str, object] | None = None,
    status: str = "active",
    is_write: bool | None = None,          # None = unannotated -> safe-default path
    requires_serial: bool | None = None,   # None = unannotated -> derived from is_write per Assumption 5
    resource_scope: str = "",
    agent_safety_tier: AgentSafetyTier | None = None,  # None -> "WRITE_DANGEROUS"
    requires_approval: bool | None = None,             # None -> True
    enabled_for_llm: bool | None = None,                # None -> False
) -> RuntimeTool:
    # resolved_is_write = is_write if is_write is not None else False
    # resolved_requires_serial = (
    #     requires_serial if requires_serial is not None
    #     else True if is_write is None else False
    # )
    # resolved_tier = agent_safety_tier if agent_safety_tier is not None else "WRITE_DANGEROUS"
    # resolved_requires_approval = requires_approval if requires_approval is not None else True
    # resolved_enabled_for_llm = enabled_for_llm if enabled_for_llm is not None else False
    # return RuntimeTool(name=name, server_key=server_key, ..., is_write=resolved_is_write, ...)
```

Empty-mutable-default handling: `input_schema`/`raw_definition` parameters default to `None` and are
resolved to `{}` inside the factory (never a shared mutable default object) before being passed to the
frozen dataclass constructor.
