# Implementation: H-6 — Remove UndoResult.n_artifacts_marked field

## Goal

Remove the `n_artifacts_marked: int = 0` field from the `UndoResult` dataclass. `undo_last_turn()`
no longer produces this value (per the companion `undo_service.py` H-6 doc), so the field is
dead weight on the model.

## Scope

**Target**: `scripts/agent/services/models.py`

**Depends on**: land together with `implementations/20260708-165242_undo_service_h6.py.md` (the
call-site update) — see that doc's Risks section for the safe application order.

**Out of scope**: every other dataclass/model in this file — only `UndoResult` is touched.

## Assumptions

1. `n_artifacts_marked` has exactly three referencing locations in the whole codebase (confirmed
   via `grep -rn "n_artifacts_marked" scripts/ tests/`): the field definition itself
   (`models.py`), the one call site in `undo_service.py` (covered by its own H-6 doc), and
   `tests/test_undo_artifact_consistency.py` (deleted entirely — see its own H-6 doc).
2. `cmd_context.py`'s `_cmd_undo` (or equivalent) reads only `result.n_removed`, never
   `result.n_artifacts_marked` — confirmed by grep (`n_removed` appears at line 128 of
   `cmd_context.py`; `n_artifacts_marked` does not appear in that file at all).

## Implementation

### Target file

`scripts/agent/services/models.py`

### Procedure

#### Step 1: Confirm the current dataclass definition

```bash
grep -n "class UndoResult" -A 3 scripts/agent/services/models.py
```

Expected:

```python
@dataclass(frozen=True)
class UndoResult:
    n_removed: int
    n_artifacts_marked: int = 0
```

#### Step 2: Remove the field

Replace with:

```python
@dataclass(frozen=True)
class UndoResult:
    n_removed: int
```

### Method

- Single-field deletion from a frozen dataclass; no other fields, decorators, or methods on
  `UndoResult` change.

### Details

- Since `n_artifacts_marked` had a default value (`= 0`), it was not a required constructor
  argument before this change — removing it is backward-compatible with any call site that
  already omitted it (there are none currently, since the sole call site in `undo_service.py`
  always passed it explicitly until its own H-6 doc updates it in the same rollout).
