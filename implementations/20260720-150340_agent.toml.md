## Goal

Remove the `[mcp_servers.browser]` transport block entirely and update `[mcp_servers.web_search]`
so `tool_names` includes `"browser_fetch"` and `cmd` points at the correct, current
`web_search_server.py` path — consolidating both tools under the one `web_search` server entry.

## Scope

**In scope**: `config/agent.toml` lines 306-313 (`[mcp_servers.web_search]`) and lines 389-396
(`[mcp_servers.browser]`) only, both confirmed via direct read during this design cycle.
**Out of scope**: `tool_safety_tiers` (lines 222-223: `search_web = "READ_ONLY"`,
`browser_fetch = "READ_ONLY"` — both already correct, no edit) and both tools'
`[[tool_definitions]]` blocks (lines 400-417 for `search_web`, 1441-1459 for `browser_fetch` —
confirmed via direct read: schemas already correct/independent of the server-registration change,
no edit needed per plan's own note "no functional change to their values, just consolidation").
Does **not** re-plan or duplicate the separate, still-unimplemented fix in
`plans/done/20260720-073534_plan.md` for the other 7 servers' stale `cmd` paths.

## Implementation

### Target file

`config/agent.toml`

### Procedure

1. Delete the entire `[mcp_servers.browser]` block (lines 389-396: `transport`, `url`,
   `startup_mode`, `auth_token`, `role`, `tool_names`, `cmd`, plus the section header and the blank
   line separating it from the following `# ── Tool definitions ──` comment — confirm no blank-line
   double-up results after removal).
2. In `[mcp_servers.web_search]` (lines 306-313):
   - Change `tool_names = ["search_web"]` (line 312) to
     `tool_names = ["search_web", "browser_fetch"]`.
   - Change `cmd = [..., "/opt/llm/scripts/mcp_servers/web_search/server.py"]` (line 313) to
     `cmd = [..., "/opt/llm/scripts/mcp_servers/web_search/web_search_server.py"]` (per Assumption 1
     — re-verify this hasn't already been fixed by the separate issue's landing, per Assumption 3).
   - `url = "http://127.0.0.1:8004"` (line 308), `auth_token = ""` (line 310, this is
     `search_web`'s own top-level MCP-transport auth token — distinct from the new
     `browser_auth_token` config key living inside `web_search_mcp_server.toml`, per that doc's
     Assumption; do not conflate the two), `role = "search"` (line 311) — all unchanged.
3. Leave `tool_safety_tiers` (lines 222-223) and both `[[tool_definitions]]` blocks (lines 400-417,
   1441-1459) untouched — already correct per Scope.

### Method

Two-part TOML edit: one block deletion, one existing-block field update (`tool_names` list append,
`cmd` path fix) — no new tables introduced, no renumbering of other `[mcp_servers.*]` blocks needed
(TOML tables are unordered; removing one doesn't require touching neighbors).

### Details

- Post-edit `[mcp_servers.web_search]` shape:
  ```toml

  transport        = "http"
  url              = "http://127.0.0.1:8004"
  startup_mode     = "subprocess"
  auth_token       = ""
  role             = "search"
  tool_names       = ["search_web", "browser_fetch"]
  cmd              = ["uv", "run", "--directory", "/opt/llm", "python", "/opt/llm/scripts/mcp_servers/web_search/web_search_server.py"]
  ```
- After this edit, `grep -n "mcp_servers.browser" config/agent.toml` returns no output.
