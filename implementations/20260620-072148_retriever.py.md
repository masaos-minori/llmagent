# Implementation: FTS-only fallback logging in retriever.py

## Goal
Add `logger.info` calls in `HybridRetriever.search()` when the search falls back to
FTS-only, with a reason string distinguishing "embedding=None (disabled or not provided)"
from "vec returned empty".

## Scope
- File: `scripts/agent/memory/retriever.py`
- Modify `HybridRetriever.search()` only — two early-return paths

## Assumptions
- The two FTS-only paths are:
  1. `embedding is None` → embedding was not provided / disabled
  2. `not vec_hits` → embedding was provided but KNN returned no results
- `logger.info` (not warning) is appropriate — FTS-only is a graceful degradation,
  not a failure; warning would be noise in normal operation
- No state change to `HybridRetriever` class needed for this step
  (req_20 plan adds `last_retrieval_mode` as a separate enhancement)

## Implementation

### Target file
`scripts/agent/memory/retriever.py`

### Procedure
1. Locate `HybridRetriever.search()` (currently at lines 266-281)
2. Add `logger.info` before each early return in the FTS-only paths

### Method
Single-step edit of the `search()` method body.

### Details

**Current `search()` method (lines 266-281):**
```python
def search(self, query, embedding=None, project="", repo=""):
    fts_hits = self._fts.search(query, project, repo)
    if embedding is None:
```

**Updated `search()` method:**
```python
def search(
    self,
    query: MemoryQuery,
    embedding: list[float] | None = None,
    project: str = "",
    repo: str = "",
) -> list[MemoryHit]:
    fts_hits = self._fts.search(query, project, repo)
    if embedding is None:
        logger.info("retrieval: fts_only (reason=embedding_disabled_or_none)")
        return fts_hits

    vec_hits = self._vec.knn_search(
        embedding, query.memory_type, self._fts._fts_limit
    )
    if not vec_hits:
        logger.info("retrieval: fts_only (reason=vec_returned_empty)")
        return fts_hits

    merged = _rrf_merge([fts_hits, vec_hits], k=self._rrf_k)
    return merged[: query.limit]
```
