# Implementation Procedure: `--targets-file` argument + `crawl()` `file://` dispatch

**Target file:** `scripts/rag/ingestion/crawler.py`

---

## Goal

1. Add `--targets-file PATH` CLI argument to `main()` in `crawler.py`.
2. Enforce mutual exclusion: `--url` and `--targets-file` cannot be used together.
3. Extend `WebCrawler.crawl()` to dispatch `file://` URLs to `crawl_file()` instead of `crawl_site()`.

---

## Scope

**In-scope:**
- Add `--targets-file PATH` to `argparse` in `main()`
- Add conflict check: `if args.url and args.targets_file: parser.error(...)`
- Call `parse_targets_file()` when `--targets-file` is given; pass result to `crawler.crawl()`
- Extend `crawl()` loop with `if url.startswith("file://"):` branch calling `self.crawl_file()`

**Out-of-scope:**
- Modifying `WebCrawler.__init__()`, `crawl_site()`, `crawl_file()` signatures
- Adding `--force` argument (crawler does not support `--force`)
- Changing the `CrawlPayload` TypedDict
- Any change to `--url` or `--lang` behavior

---

## Assumptions

1. `parse_targets_file` is imported from `rag.ingestion.crawler_utils` (the new function added in `20260702-143340_crawler_utils.md`).
2. `crawl_file()` is a synchronous method; it is safe to call directly inside the `async def crawl()` body without `await`.
3. `file://` URL path extraction: `Path(url[len("file://"):])` strips the `file://` prefix and converts the remainder to a `Path`. This is consistent with how `crawl_file()` constructs the stored URL (`f"file://{path.resolve()}"`).
4. Exceptions from `parse_targets_file()` (`FileNotFoundError`, `ValueError`) are caught in `main()` and converted to `parser.error()` calls.
5. The `targets` parameter passed to `crawl()` may contain mixed `http://`, `https://`, and `file://` entries.

Replace the `try/except` block to add `file://` dispatch before calling `crawl_site()`:

```python
async def crawl(self, targets: list[tuple[str, str]] | None = None) -> None:
    """Crawl all given targets, or config target_urls when targets is None."""
    for url, lang in targets or self._target_urls:
        logger.info("=== start: %s (lang=%s) ===", url, lang)
        try:
            if url.startswith("file://"):
                self.crawl_file(Path(url[len("file://"):]), lang)
            else:
                await self.crawl_site(url, lang)
        except (httpx.RequestError, httpx.HTTPStatusError, OSError) as _crawl_err:
            logger.exception("crawl failed: %s: %s", url, _crawl_err)
        logger.info("=== done:  %s ===", url)
```

Note: `crawl_file()` raises `OSError` on file read failure; this is already covered by the `except` clause. The log message changes from `"crawl_site failed"` to `"crawl failed"` to be accurate for both dispatch paths.

### Method details

**`--targets-file` argument:**
- `dest` is `args.targets_file` (argparse converts hyphens to underscores automatically)
- `metavar="PATH"` makes the help output read `--targets-file PATH`
- No `type=` conversion; path is kept as a string and converted to `Path` in the call to `parse_targets_file()`

**Mutual exclusion:**
- Checked before URL validation to fail fast with a clear message
- Uses `parser.error()` (not `sys.exit()`) so argparse prints usage before exiting

**`file://` dispatch in `crawl()`:**
- `url[len("file://"):]` strips exactly 7 characters; equivalent to `url[7:]` but self-documenting
- `Path(...)` accepts absolute paths like `/path/to/file.py`; no additional resolution needed since `crawl_file()` calls `path.resolve()` internally when writing the URL field
- No `await` on `self.crawl_file()` — it is a synchronous method; calling sync code inside an `async def` is valid as long as it is not I/O-blocking in a way that would block the event loop for a long time. Local file reads are fast and this matches the existing design.
