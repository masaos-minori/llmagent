# Goal

Remove the backward-compatibility `LLMMessage` re-export and `RagHit` union alias
from `rag/types.py`, and fix all callers that import `LLMMessage` or `RagHit` from
this module.

# Scope

- `scripts/rag/types.py` — remove re-export and alias
- Any file in `scripts/` that imports `LLMMessage` or `RagHit` from `rag.types`

# Assumptions

1. `LLMMessage` is re-exported from `shared.types`; callers should import directly
   from `shared.types`.
2. `RagHit = RawHit | MergedHit | RankedHit` is a union alias. After removal, callers
   must use the explicit union or a per-use type annotation.
3. Run `grep -rn "from rag.types import.*LLMMessage\|from rag.types import.*RagHit"
   scripts/` before editing to enumerate all callers.

# Implementation

## Target file

`scripts/rag/types.py`

## Procedure

1. Remove line `from shared.types import LLMMessage`.
2. Remove `"LLMMessage"` from `__all__`.
3. Remove line `RagHit = RawHit | MergedHit | RankedHit`.
4. Remove `"RagHit"` from `__all__`.
5. For each caller found by grep:
   - Replace `from rag.types import LLMMessage` → `from shared.types import LLMMessage`
   - Replace `from rag.types import RagHit` → inline the union `RawHit | MergedHit | RankedHit`
     or import the individual types.
6. Run ruff + mypy.

## Method

Import line deletion + caller update. No logic changes.

# Validation plan

- `grep -rn "from rag.types import.*LLMMessage" scripts/` → 0 hits
- `grep -rn "from rag.types import.*RagHit" scripts/` → 0 hits
- `uv run ruff check scripts/rag/types.py`
- `uv run mypy scripts/rag/types.py`
