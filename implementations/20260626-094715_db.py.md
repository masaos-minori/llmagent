# Implementation: Add pagination queries for /replay and /dlq to db.py

Steps covered: Plan 20260626-094715 — Step 2-1 (db layer)

---

## Goal

Add `limit` + `offset` parameters to `db.get_events()` and `db.get_dlq_entries()` (or equivalent) so `app.py` can pass pagination to the SQLite queries.

---

## Scope

- **In scope**: `scripts/eventbus/db.py` — `get_events()`, `get_dlq_entries()`, add `count_events()` / `count_dlq_entries()`
- **Out of scope**: app.py (covered separately)

---

## Assumptions

- `db.py` uses `aiosqlite` or `sqlite3` for SQLite queries.
- `get_events(stream, limit, offset)` → `SELECT * FROM events WHERE stream=? ORDER BY id ASC LIMIT ? OFFSET ?`
- `count_events(stream)` → `SELECT COUNT(*) FROM events WHERE stream=?`

---

## Implementation

### Target file
`scripts/eventbus/db.py`

### Procedure
1. Read `scripts/eventbus/db.py` — find `get_events()` and related functions.
2. Update `get_events()`:
   ```python
   async def get_events(
       self,
       stream: str,
       limit: int = 100,
       offset: int = 0,
   ) -> list[Event]:
       rows = await self._db.execute_fetchall(
           "SELECT * FROM events WHERE stream = ? ORDER BY id ASC LIMIT ? OFFSET ?",
           (stream, limit, offset),
       )
       return [Event.from_row(r) for r in rows]
   ```
3. Add `count_events()`:
   ```python
   async def count_events(self, stream: str) -> int:
       row = await self._db.execute_fetchone(
           "SELECT COUNT(*) FROM events WHERE stream = ?", (stream,)
       )
       return row[0] if row else 0
   ```
4. Repeat for `get_dlq_entries()` and `count_dlq_entries()`.

### Method
SQL LIMIT/OFFSET pagination. Standard pattern for SQLite.
