# Implementation: M-5 — Align /mcp status health parsing with watchdog (mcp_status.py)

## Goal

Replace `_get_http_status()` with `_probe_mcp_health_detail()` from `repl_health.py`
in `McpStatusService` so that `/mcp status` parses `restart_recommended`,
`operator_action_required`, `reason`, and `message` from the `/health` body —
consistent with the watchdog.

## Scope

**Target**: `scripts/agent/services/mcp_status.py`

**Steps covered**: Plan M-5 steps 1–3 (including tests).

**Dependencies**: M-4 (fields added to `McpProbeResult`).

**Out of scope**: `repl_health.py` (no changes), `cmd_mcp.py`, other files.

## Assumptions

1. `_probe_mcp_health_detail()` in `repl_health.py` is the canonical health body parser;
   it parses `restart_recommended`, `operator_action_required`, and captures `body` dict.
2. Import from `agent.repl_health` is permitted by `lint-imports`; check after adding.
3. The return value of `_probe_mcp_health_detail()` is a probe object with attributes:
   `reachable`, `status_code`, `body`, `restart_recommended`, `operator_action_required`.
4. Existing `McpAvailability` semantics (OK / HTTP_ERROR / FAIL) are preserved.

## Implementation

### Target file

`scripts/agent/services/mcp_status.py`

### Procedure

#### Step 1: Import `_probe_mcp_health_detail`

```python
from agent.repl_health import _probe_mcp_health_detail
```

If `lint-imports` rejects this import (cross-module in `agent` layer), extract
`_probe_mcp_health_detail` to `scripts/agent/shared/health_probe.py` and import from
there.

#### Step 2: Replace `_get_http_status()` call with `_probe_mcp_health_detail()`

In `_resolve_endpoint()` (or wherever `_get_http_status()` is called), replace:
```python
availability, sandbox_backend = await _get_http_status(cfg.url)
```

With:
```python
probe = await _probe_mcp_health_detail(http_client, cfg.url)
if probe.reachable and probe.status_code == 200:
    availability = McpAvailability.OK
elif probe.reachable:
    availability = MccAvailability.HTTP_ERROR
else:
    availability = McpAvailability.FAIL
sandbox_backend = (probe.body or {}).get("details", {}).get("sandbox_backend", "")
```

Return `probe` alongside `availability` for use in `_probe_single_server()`.

#### Step 3: Populate new health fields in `_probe_single_server()`

```python
restart_recommended=probe.restart_recommended,
operator_action_required=probe.operator_action_required,
health_reason=str(
    (probe.body or {}).get("reason", "")
    or (probe.body or {}).get("message", "")
),
```

These override the `restart_recommended` set by lifecycle_state in M-4 step 4 (M-5
value is more authoritative — from the server itself).

### Method

- Replace one async call with another; no new external HTTP calls (same URL).
- `_probe_mcp_health_detail` already handles malformed JSON gracefully.

### Details

- Read `repl_health.py`'s `_probe_mcp_health_detail()` signature before implementing
  to confirm parameters (likely `(client: httpx.AsyncClient, url: str) -> ProbeResult`).
- The function may already pass an `httpx.AsyncClient`; verify whether to pass the
  shared client from `McpStatusService` or create a new one.
- `probe.body` may be `None` if the endpoint was unreachable; guard all `.get()` calls
  with `(probe.body or {})`.
