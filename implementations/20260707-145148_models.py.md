# Implementation: M-4 — Add optional fields to McpProbeResult (models.py)

## Goal

Add 11 optional lifecycle and health detail fields to `McpProbeResult` dataclass with
safe defaults so existing construction call sites are unaffected.

## Scope

**Target**: `scripts/agent/services/models.py`

**Step covered**: Plan M-4 step 1.

**Out of scope**: `mcp_status.py`, `cmd_mcp.py`, other files.

## Assumptions

1. `McpProbeResult` is `@dataclass(frozen=True)` — all new fields must have defaults.
2. All new fields are placed after existing fields to maintain construction compatibility.
3. `bool | None` and `int | None` with `= None` defaults are valid in frozen dataclasses.

## Implementation

### Target file

`scripts/agent/services/models.py`

### Procedure

#### Step 1: Locate McpProbeResult dataclass

Find `@dataclass(frozen=True)` class `McpProbeResult` and identify the last existing
field (to append after it).

#### Step 2: Add 11 new optional fields

Append to `McpProbeResult` after all existing fields:
```python
# Lifecycle and process observability (added by M-4)
managed: bool = False
pid: int | None = None
pgid: int | None = None
running: bool | None = None
lifecycle_state: str = ""
last_exit_code: int | None = None
last_shutdown_result: str = ""
restart_recommended: bool = False
operator_action_required: bool = False
health_reason: str = ""
stderr_log: str = ""
```

### Method

- Append-only; no existing fields are modified or reordered.
- No new imports needed (all types are stdlib).

### Details

- `managed: bool = False` — `True` when the server is managed by `HttpServerLifecycleManager`.
- `pid: int | None = None` — subprocess PID; `None` for persistent servers.
- `pgid: int | None = None` — process group ID (populated after H-8).
- `running: bool | None = None` — `True` if `proc.poll() is None`; `None` for persistent.
- `lifecycle_state: str = ""` — `.value` of `LifecycleState`; `""` before H-9.
- `last_exit_code: int | None = None` — exit code of a terminated process.
- `last_shutdown_result: str = ""` — human-readable shutdown outcome.
- `restart_recommended: bool = False` — from `/health` body (M-5) or lifecycle FAILED.
- `operator_action_required: bool = False` — from `/health` body (M-5).
- `health_reason: str = ""` — `reason` or `message` from `/health` body.
- `stderr_log: str = ""` — path to stderr log file (H-1).
