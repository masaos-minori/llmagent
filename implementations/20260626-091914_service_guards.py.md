# Implementation: Make workflow_allowlist fail-closed in service_guards.py

Steps covered: Plan 20260626-091914 — Steps 1-1, 2-1

---

## Goal

Change `_check_workflow()` in `service_guards.py` so that an empty `workflow_allowlist` raises `CicdAuthorizationError` (fail-closed) instead of silently allowing all workflows.

---

## Scope

- **In scope**: `scripts/mcp/cicd/service_guards.py` — `_check_workflow()` method
- **Out of scope**: server.py (step 1-2); repl_health.py (step 2-2); config (step 2-1 config doc); tests (step 3-x); docs (step 3-3)

---

## Assumptions

- `_check_workflow()` currently has: `if not self._workflow_allowlist: return` (allow-all when empty).
- `CicdAuthorizationError` exists in the module or nearby error definitions.
- Changing to fail-closed: empty allowlist = deny all.
- `_workflow_allowlist` is populated from config `cicd_mcp_server.toml:workflow_allowlist`.

---

## Implementation

### Target file
`scripts/mcp/cicd/service_guards.py`

### Procedure
1. Read `scripts/mcp/cicd/service_guards.py` — find `_check_workflow()` and the allowlist check.
2. Step 1-1 (Audit): Confirm the current allow-all-when-empty behavior.
3. Change:
   ```python
   # Before (fail-open):
   if not self._workflow_allowlist:
       return  # allow all
   
   # After (fail-closed):
   if not self._workflow_allowlist:
       raise CicdAuthorizationError(
           "workflow_allowlist is empty — all workflow triggers denied (fail-closed). "
           "Add allowed workflow patterns to config."
       )
   ```
4. Step 2-1 (Config guard): If `workflow_allowlist` is loaded from config, add a startup validation in `service_guards.py.__init__()` or in `repl_health.py`:
   - Log a WARNING at startup if `workflow_allowlist` is empty.
   - The runtime check (step 1-1) is the enforcement point.

### Method
Behavioral change. Existing code that relies on empty allowlist = allow-all will break. Tests must be updated (covered in step 3-1).
