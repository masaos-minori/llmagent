# Implementation: H-6 — Startup rollback shutdown in startup.py

## Goal

Wrap the post-`_initialize()` steps in `StartupOrchestrator.run()` with a try/except
block that calls `lifecycle.shutdown_all()` on any failure after `_start_servers()` has
succeeded, then re-raises the original exception.

## Scope

**Target**: `scripts/agent/startup.py`

**Step covered**: Plan H-6 step 1.

**Out of scope**: `factory.py`, `http_lifecycle.py`, `_initialize()` logic, other files.

## Assumptions

1. Services (including `lifecycle`) exist after `_initialize()` returns successfully.
2. `_initialize()` failures are NOT rolled back — servers may not exist yet.
3. Rollback shutdown failures are logged as warnings, not re-raised, to preserve the
   original exception context.
4. `_servers_started` flag is set only after `_start_servers()` returns normally;
   partial starts inside `_start_servers()` are still cleaned by `shutdown_all()`.
5. `ctx.services_required.lifecycle` is the correct accessor for the lifecycle manager
   (confirmed: `AgentContext` raises on uninitialized services via `services_required`).

## Implementation

### Target file

`scripts/agent/startup.py`

### Procedure

#### Step 1: Replace `run()` body with rollback-guarded version

Current `run()` body (lines 52–63):
```python
async def run(self) -> tuple[CommandRegistry, Orchestrator]:
    """Execute full startup sequence; return (cmds, orchestrator)."""
    self._initialize()
    await self._start_servers()
    await self._check_services()
    await self._recover_pending_approvals()
    await self._setup_prompt()
    if self._cmds is None or self._orchestrator is None:
        raise RuntimeError(
            "StartupOrchestrator.run() failed to initialize cmds/orchestrator"
        )
    return self._cmds, self._orchestrator
```

New `run()` body:
```python
async def run(self) -> tuple[CommandRegistry, Orchestrator]:
    """Execute full startup sequence; return (cmds, orchestrator)."""
    self._initialize()
    _servers_started = False
    try:
        await self._start_servers()
        _servers_started = True
        await self._check_services()
        await self._recover_pending_approvals()
        await self._setup_prompt()
    except Exception:
        if _servers_started:
            try:
                await self._ctx.services_required.lifecycle.shutdown_all()
            except Exception as shutdown_err:
                logger.warning(
                    "Startup rollback: shutdown_all() failed: %s", shutdown_err
                )
        raise
    if self._cmds is None or self._orchestrator is None:
        raise RuntimeError(
            "StartupOrchestrator.run() failed to initialize cmds/orchestrator"
        )
    return self._cmds, self._orchestrator
```

### Method

- Single method replacement; existing behavior unchanged on the happy path.
- The `_servers_started` flag is a local variable; no new instance state required.
- No new imports needed; `logger` is already defined at module level.

### Details

- `_initialize()` is called before the try block intentionally: if `_initialize()`
  fails, services do not exist and `shutdown_all()` cannot be safely called.
- `_servers_started = True` is set after `await self._start_servers()` returns. If
  `_start_servers()` raises (e.g. a critical server fails to start), `_servers_started`
  remains `False` and rollback is skipped. `_start_servers()` internally catches
  per-server errors (non-fatal); an exception propagating out of `_start_servers()`
  means a configuration-level failure, not a server startup failure.
- The `raise` at the end of the `except` block re-raises the original exception with
  its original traceback, ensuring the caller sees the root cause.
- `logger.warning("Startup rollback: ...")` is sufficient; no need for `logger.error`
  since the original exception is the primary failure signal.
