# Goal

Add `"stub": True` to the `/health` endpoint response dict in `scripts/mcp/mdq/server.py` so that health check consumers can detect that this server is non-production.

# Scope

- **File:** `scripts/mcp/mdq/server.py`
- **Lines affected:** `/health` endpoint return dict (~line 154)
- **Out of scope:** Other endpoints, service logic, other MCP servers

# Assumptions

1. Additive field change only — existing consumers will not break.
2. The `"stub"` field is a boolean constant `True`; no config lookup needed (hardcoded intent).
3. Health response schema is not shared/standardized across servers (each defines its own).

# Implementation

## Target file

`scripts/mcp/mdq/server.py`

## Procedure

1. Locate the `health()` function return dict (lines ~137-158).
2. Add `"stub": True` to the returned dict alongside the existing `"status"`, `"ready"`, `"dependencies"`, `"details"` fields.

## Method

Edit return statement of `async def health()`.

Current return:
```python
return {
    "status": "ok",
    "ready": ready,
    "dependencies": deps,
    "details": {"service": "mdq-mcp"},
}
```

After:
```python
return {
    "status": "ok",
    "ready": ready,
    "stub": True,
    "dependencies": deps,
    "details": {"service": "mdq-mcp"},
}
```

## Details

- `True` (Python bool) serializes to `true` in JSON via FastAPI/orjson automatically.
- Field placed after `"ready"` for logical grouping (status → ready → stub indicator → details).
