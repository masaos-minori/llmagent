# Implementation: Add firejail sandbox production default to repl_health.py

Steps covered: Plan 20260626-091916 — Steps 1-1, 2-2

---

## Goal

Make `sandbox_backend="firejail"` the production default. In `repl_health.py`, add a startup check that warns when `sandbox_backend` is not `"firejail"` in production mode, and add a `_require_firejail_available()` check that fails-hard if firejail is required but not installed.

---

## Scope

- **In scope**: `scripts/agent/repl_health.py` — `sandbox_backend` health checks
- **Out of scope**: `config/shell_mcp_server.toml` (steps 1-2, 2-1); tests (step 3-1); docs (step 3-2)

---

## Assumptions

- `repl_health.py` has a pattern of `HealthCheckResult`-returning checks.
- `sandbox_backend` is read from shell MCP server config.
- `firejail` availability can be tested with `shutil.which("firejail")`.
- In production mode (determined by config `environment="production"` or similar), non-firejail sandbox is a hard failure.

---

## Implementation

### Target file
`scripts/agent/repl_health.py`

### Procedure
1. Read `repl_health.py` to understand existing check patterns.
2. Step 1-1: Add `_check_sandbox_backend(config)`:
   ```python
   def _check_sandbox_backend(config: ShellMcpConfig) -> HealthCheckResult:
       backend = config.sandbox_backend
       if backend != "firejail":
           return HealthCheckResult(
               name="sandbox_backend",
               status="warning",
               message=f"sandbox_backend={backend!r}; production default is 'firejail'. "
                       "Update config/shell_mcp_server.toml.",
           )
       return HealthCheckResult(name="sandbox_backend", status="ok")
   ```
3. Step 2-2: Add `_require_firejail_available(config)`:
   ```python
   def _require_firejail_available(config: ShellMcpConfig) -> HealthCheckResult:
       if config.sandbox_backend == "firejail":
           if not shutil.which("firejail"):
               return HealthCheckResult(
                   name="firejail_binary",
                   status="error",
                   message="sandbox_backend=firejail but firejail binary not found. "
                           "Install firejail or change sandbox_backend.",
               )
       return HealthCheckResult(name="firejail_binary", status="ok")
   ```
4. Register both checks in the startup sequence.

### Method
Two new health checks. Uses `shutil.which()` (stdlib). Non-breaking: warning does not prevent startup; error in `_require_firejail_available` may or may not fail startup depending on the existing severity policy.
