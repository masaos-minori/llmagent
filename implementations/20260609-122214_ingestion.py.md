# Implementation: ingestion.py — Remove DedupAction.LINK_ONLY + adopt EmbeddingResult

## Goal

Remove `DedupAction.LINK_ONLY` from `ingestion.py`, fixing the dedup policy to the single
`SKIP_NEW` behavior. Also update `_has_near_duplicate()` and `_link_duplicates()` to use
the public `VectorRetriever.knn_search()` API (prerequisite: retriever.py split in step 6).

## Scope

- `scripts/agent/memory/ingestion.py`:
  - Remove `DedupAction.LINK_ONLY` enum value; leave only `SKIP_NEW`.
  - Update `on_session_stop()` to remove the `action == DedupAction.SKIP_NEW` conditional
    (now always the only action) and use `EmbeddingResult`.
  - Replace `self._retriever._vec_search(...)` with `self._retriever.knn_search(...)`.
- `tests/test_memory_layer.py`: update LINK_ONLY test to test SKIP_NEW behavior instead.

## Assumptions

1. `DedupAction.LINK_ONLY` is used only in `tests/test_memory_layer.py` (grep confirmed; zero production callers).
2. After `DedupAction` has only one value (`SKIP_NEW`), the class can be simplified or kept as `StrEnum` with one member for future extensibility.
3. This step depends on step 3 (EmbeddingResult) and step 6 (retriever split) being completed first. The `knn_search()` public API must exist on `VectorRetriever` before this step is implemented.
4. The `DedupPolicy.action` field remains for forward compatibility but effectively always equals `SKIP_NEW`.

## Implementation

### Target files

- `scripts/agent/memory/ingestion.py`

### Procedure

**ingestion.py — remove LINK_ONLY:**
```python
class DedupAction(StrEnum):
    SKIP_NEW = "skip_new"  # skip new entry when a near-duplicate already exists
```
Remove `LINK_ONLY = "link_only"` and its comment.

**ingestion.py — simplify on_session_stop():**

Before:
```python
if (
    self._dedup_policy.action == DedupAction.SKIP_NEW
    and embedding is not None
    and self._has_near_duplicate(entry.memory_id, embedding)
):
    logger.debug("memory.skip_dup memory_id=%r", entry.memory_id)
    continue
```

After (using EmbeddingResult):
```python
embed_result = await self._embed_client.fetch(entry.content)
embedding = embed_result.embedding if embed_result.success else None
if (
    embed_result.success
    and self._has_near_duplicate(entry.memory_id, embed_result.embedding)  # type: ignore[arg-type]
):
    logger.debug("memory.skip_dup memory_id=%r", entry.memory_id)
    continue
```

**ingestion.py — replace _vec_search with knn_search:**

`_has_near_duplicate()`:
```python
def _has_near_duplicate(self, memory_id: str, embedding: list[float]) -> bool:
    neighbors = self._retriever.knn_search(embedding, memory_type=None, limit=5)
    return any(
        -h.score < self._dedup_policy.threshold and h.entry.memory_id != memory_id
        for h in neighbors
    )
```

`_link_duplicates()`:
```python
def _link_duplicates(self, memory_id: str, embedding: list[float]) -> None:
    neighbors = self._retriever.knn_search(embedding, memory_type=None, limit=5)
    ...
```

Note: `self._retriever` type annotation changes from `MemoryRetriever` to `VectorRetriever`
(or `HybridRetriever` if it exposes `knn_search()` as a delegation).

**tests/test_memory_layer.py — rewrite LINK_ONLY test:**

The test `"""LINK_ONLY dedup policy persists entry even when near-duplicate exists."""`
should be replaced with a test that verifies `SKIP_NEW` correctly skips a near-duplicate.

### Method

Sequential edits: complete steps 3 and 6 first; then edit `ingestion.py`.

### Details

The `_link_duplicates()` method itself is unchanged in behavior; only the call site changes
from `_vec_search` to `knn_search`. The method continues to run whenever `embedding is not None`
(i.e., `embed_result.success`), regardless of whether the entry was skipped.
