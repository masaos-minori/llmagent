# Implementation: H-7 — Clean up _http_procs entries in shutdown_all (http_lifecycle.py)

## Goal

Refactor `HttpServerLifecycleManager.shutdown_all()` to pop and remove every entry from
`_http_procs` (including already-exited processes) so that repeated `shutdown_all()`
calls are safe no-ops and stale records do not accumulate.

## Scope

**Target**: `scripts/agent/http_lifecycle.py`

**Step covered**: Plan H-7 step 1.

**Out of scope**: `factory.py`, `startup.py`, `restart()`, all other methods.

## Assumptions

1. After H-1, `shutdown_all()` must also close `_stderr_files` handles; this plan
   includes a `getattr`-guarded cleanup for H-1 compatibility.
2. Already-exited processes (`proc.poll() is not None`) do not need `terminate()`;
   just pop and log.
3. One server's termination failure must not prevent cleanup of the remaining servers.
4. `list(self._http_procs.keys())` snapshot before the loop avoids mutation-during-
   iteration issues.

## Implementation

### Target file

`scripts/agent/http_lifecycle.py`

### Procedure

#### Step 1: Replace `shutdown_all()` body

Current implementation:
```python
async def shutdown_all(self) -> None:
    """Terminate all HTTP subprocess servers."""
    for key, proc in list(self._http_procs.items()):
        if proc.poll() is None:
            try:
                await self._terminate_with_timeout(proc, key, timeout=5.0)
            except (OSError, TimeoutError) as e:
                logger.warning(
                    "Lifecycle: error stopping HTTP subprocess %r: %s",
                    key,
                    e,
                )
```

New implementation:
```python
async def shutdown_all(self) -> None:
    """Terminate all HTTP subprocess servers and clear internal state."""
    keys = list(self._http_procs.keys())
    for key in keys:
        proc = self._http_procs.pop(key, None)
        if proc is None:
            continue
        if proc.poll() is not None:
            logger.debug("Lifecycle: %r already exited; removing entry", key)
        else:
            try:
                await self._terminate_with_timeout(proc, key, timeout=5.0)
            except (OSError, TimeoutError) as e:
                logger.warning(
                    "Lifecycle: error stopping HTTP subprocess %r: %s", key, e
                )
        # Close stderr file handle (H-1 compatibility guard)
        stderr_fh = getattr(self, "_stderr_files", {}).pop(key, None)
        if stderr_fh is not None:
            try:
                stderr_fh.close()
            except OSError as close_err:
                logger.warning(
                    "Lifecycle: error closing stderr log for %r: %s", key, close_err
                )
    # Clear log paths if H-1 is present
    if hasattr(self, "_stderr_log_paths"):
        self._stderr_log_paths.clear()
```

### Method

- Direct replacement of the existing `shutdown_all()` body.
- `getattr(self, "_stderr_files", {})` safely accesses the H-1 attribute without
  failing if H-1 has not yet been merged.
- No new imports required.

### Details

- The outer loop iterates over a snapshot of `_http_procs.keys()` taken before the
  loop begins; `pop()` inside the loop is safe because we are not iterating the dict.
- After the loop, `_http_procs` is empty: all entries are popped regardless of whether
  termination succeeded. This makes a second `shutdown_all()` call a no-op.
- `_http_pgids` cleanup (H-8): if H-8 is merged, add `self._http_pgids.pop(key, None)`
  inside the loop alongside the `_stderr_files` cleanup. This plan does not require H-8
  but is compatible with it.
