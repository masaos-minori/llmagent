# Implementation: agent/repl_health.py — Replace direct MCP server config imports with audit API

## Goal

Remove the module-level `from mcp.git.models import GitConfig` and `from mcp.shell.models import ShellConfig` imports from `repl_health.py`, and update `audit_security_defaults()` to call the loader functions from `agent.security_audit_config` instead of importing server config models directly.

## Scope

**In**: Remove lines 17–18 (module-level MCP imports). Update `audit_security_defaults()` to use `load_shell_audit_config()`, `load_git_audit_config()`, `load_github_audit_config()`, `load_cicd_audit_config()`.

**Out**: Changes to exception handling patterns (separate phase already documented in `20260705-154859_repl_health_security_audit.py.md`). No changes to other functions in `repl_health.py`.

## Assumptions

1. `repl_health.py` lines 17–18 import `GitConfig` and `ShellConfig` at module level; these are used only inside `audit_security_defaults()`.
2. `audit_security_defaults()` currently accesses config fields such as `cfg.shell_sandbox_backend`, `cfg.command_allowlist`, `cfg.allowed_repo_paths`, `cfg.allowed_repos`, `cfg.allow_force_push`, `cfg.require_pr_review`, `cfg.workflow_allowlist`.
3. The new audit DTOs (`ShellAuditConfig`, `GitAuditConfig`, etc.) expose the same fields.
4. `audit_security_defaults()` has 4 try/except blocks for github and cicd (lazy imports at lines ~659, ~688, ~707) that will be replaced by the corresponding loader calls.
5. After this change, no agent file other than `security_audit_config.py` may import `from mcp.shell.models`, `from mcp.git.models`, `from mcp.github.models_config`, or `from mcp.cicd.models`.

## Implementation

### Target file

`scripts/agent/repl_health.py`

### Procedure

1. Remove lines 17–18 (`from mcp.git.models import GitConfig` and `from mcp.shell.models import ShellConfig`).
2. Add import at top of `audit_security_defaults()` (or at module level under `from agent.*`):
   ```python
   from agent.security_audit_config import (
       load_cicd_audit_config,
       load_git_audit_config,
       load_github_audit_config,
       load_shell_audit_config,
   )
   ```
3. Replace the shell config loading block:
   - Old: `cfg = ShellConfig.load()` → access `cfg.shell_sandbox_backend`, `cfg.command_allowlist`
   - New: `shell_cfg = load_shell_audit_config()` → access `shell_cfg.sandbox_backend`, `shell_cfg.command_allowlist`
   - Wrap loader call in try/except RuntimeError (re-raise or handle based on production_mode)
4. Replace the git config loading block:
   - Old: `cfg = GitConfig.load()` → access `cfg.allowed_repo_paths`
   - New: `git_cfg = load_git_audit_config()` → access `git_cfg.allowed_repo_paths`
5. Replace GitHub lazy import block (line ~659):
   - Old: `from mcp.github.models_config import GitHubConfig; cfg = GitHubConfig.load()`
   - New: `github_cfg = load_github_audit_config()` → `if github_cfg is None: pass` (optional)
6. Replace second GitHub block (line ~707, write settings):
   - Old: `from mcp.github.models_config import GitHubConfig`
   - New: same `github_cfg` from step 5 (reuse or call loader again)
7. Replace CI/CD lazy import block (line ~688):
   - Old: `from mcp.cicd.models import CicdConfig; cfg = CicdConfig.load()`
   - New: `cicd_cfg = load_cicd_audit_config()` → `if cicd_cfg is None: pass`

### Method

**Pattern for each replaced block:**
```python
# Before (module-level or lazy import)
from mcp.shell.models import ShellConfig
...
cfg = ShellConfig.load()
sandbox = cfg.shell_sandbox_backend

# After
from agent.security_audit_config import load_shell_audit_config
...
shell_cfg = load_shell_audit_config()
if shell_cfg is None:
    pass  # optional dependency not installed
else:
    sandbox = shell_cfg.sandbox_backend
```

**RuntimeError propagation for production mode:**
```python
try:
    shell_cfg = load_shell_audit_config()
except RuntimeError:
    if production_mode:
        raise
    logger.warning("Security audit: shell config load failed")
    shell_cfg = None
```

### Details

- `load_shell_audit_config()` returns `None` on `ImportError`; raises `RuntimeError` on config load failure. The caller (`audit_security_defaults()`) must handle both.
- The existing `try/except Exception: pass` blocks around lazy imports are replaced by `if cfg is None: pass` guards, simplifying control flow.
- After this change, `ruff check` must pass with no unused import warnings for the removed MCP imports.
- The existing `repl_health_security_audit.py.md` doc (replacing except Exception handlers) is a companion change; both may be applied independently.
