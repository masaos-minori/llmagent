# Implementation: agent/workflow/workflow_engine.py — Pass error classification and attempt context

## Goal

Pass `error_kind`, `error_detail`, `workflow_id`, and `attempt_number` to the updated ops functions when finishing attempts and recording artifacts.

## Scope

**In**: Update `_run_stage()` and `_run_execute_with_retry()` to pass new fields. Add error classification constants.

**Out**: Changes to schema, ops files, tests.

## Assumptions

1. `attempt_ops.py`, `artifact_ops.py`, `idempotency_ops.py` are updated first.
2. Error classification: `WorkflowTimeoutError` → `"timeout"`, `WorkflowHaltError` → `"halt"`, other exceptions → `"exception"`.
3. `task.workflow_id` provides the workflow ID for denormalization.
4. Attempt number comes from the `attempt` parameter in `_run_execute_with_retry()` (already tracked).
5. `begin_stage_if_new()` needs `workflow_id` from the task record.

## Implementation

### Target file
`scripts/agent/workflow/workflow_engine.py`

### Procedure
1. Define error kind constants.
2. Update `_run_stage()` to pass `error_kind`, `error_detail`, `workflow_id`, `attempt_number`.
3. Update `_run_execute_with_retry()` to pass `attempt` number through to `_run_stage()`.

### Method

**Error kind constants (top of file):**
```python
_ERROR_KIND_TIMEOUT = "timeout"
_ERROR_KIND_HALT = "halt"
_ERROR_KIND_EXCEPTION = "exception"
```

**Updated `_run_stage()` signature:**
```python
async def _run_stage(
    self,
    task: TaskRecord,
    stage_id: str,
    fn: StageCallback,
    attempt: int = 1,
) -> None:
```
(Already has `attempt` parameter — no change needed there.)

**Updated `_run_stage()` error handling:**
```python
try:
    artifact_uri = await asyncio.wait_for(fn(), timeout=timeout)
except TimeoutError as exc:
    self._store.finish_attempt(
        attempt_rec.attempt_id,
        "failed",
        error_msg=f"timeout after {timeout}s",
        error_kind=_ERROR_KIND_TIMEOUT,
        error_detail=str(exc),
    )
    raise WorkflowTimeoutError(...)
except Exception as exc:
    self._store.finish_attempt(
        attempt_rec.attempt_id,
        "failed",
        error_msg=str(exc),
        error_kind=_ERROR_KIND_EXCEPTION,
        error_detail=repr(exc),
    )
    raise
```

**Updated `record_artifact()` call:**
```python
if artifact_uri:
    record_artifact(
        self._store._db,
        task.task_id,
        stage_id,
        artifact_uri,
        workflow_id=task.workflow_id,
        attempt_number=attempt,
    )
```

**Updated `begin_stage_if_new()` call:**
```python
attempt_rec = begin_stage_if_new(
    self._store._db,
    event_id,
    task.task_id,
    stage_id,
    workflow_id=task.workflow_id,  # NEW
)
```

### Details

- `state_store.finish_attempt()` is called instead of `attempt_ops.finish_attempt()` in `workflow_engine.py` — update the `StateStore.finish_attempt()` signature accordingly (already covered in `workflow_ops.py` doc).
- `error_kind=_ERROR_KIND_HALT` is set in `WorkflowHaltError` path (already in `run()` — update if `update_task_status("halted")` is called there).
