# Implementation: M-1 — Remove tool-summarize reload handling from config_reload.py

## Goal

Remove the `_apply_bool(..., "use_tool_summarize", ...)` and
`_apply_int(..., "tool_summarize_threshold", ...)` blocks from
`ConfigReloadService._apply_tool_params()`, matching the field removal from `ToolConfig`.

## Scope

**Target**: `scripts/agent/services/config_reload.py`

**Depends on / must land together with**: `implementations/20260708-171057_config_dataclasses.py.md`
(the `ToolConfig` fields these `setattr(cfg.tool, ...)` lambdas target).

**Out of scope**: every other `_apply_*` call in `_apply_tool_params()` (`tool_cache_ttl`,
`serial_tool_calls`, `tool_definitions_strict`, and whatever follows) — unchanged.

## Assumptions

1. `_apply_bool`/`_apply_int` are generic helper functions (defined elsewhere in this file or a
   shared reload-helpers module) that no-op when the key is absent from `new_cfg` — removing
   the two calls simply means `/reload` never touches `cfg.tool.use_tool_summarize`/
   `.tool_summarize_threshold`, which is correct since those attributes no longer exist on
   `ToolConfig` after the companion doc lands.

## Implementation

### Target file

`scripts/agent/services/config_reload.py`

### Procedure

#### Step 1: Confirm the current blocks

```bash
grep -n "use_tool_summarize\|tool_summarize_threshold" scripts/agent/services/config_reload.py
```

Expected: two `_apply_*(...)` call blocks inside `_apply_tool_params()`.

#### Step 2: Remove both blocks

Current (within `_apply_tool_params()`):

```python
        _apply_bool(
            new_cfg,
            "use_tool_summarize",
            lambda v: setattr(cfg.tool, "use_tool_summarize", v),
        )
        _apply_int(
            new_cfg,
            "tool_summarize_threshold",
            lambda v: setattr(cfg.tool, "tool_summarize_threshold", v),
        )
```

Remove this entire two-block section. The `_apply_bool(..., "serial_tool_calls", ...)` call
before it and the `_apply_bool(..., "tool_definitions_strict", ...)` call after it become
adjacent.

### Method

- Two-block deletion; no other reload-parameter handling in this method changes.

### Details

- Per the plan's R-3 risk note: if a user's live config still contains these keys and calls
  `/reload`, the keys are now simply ignored (no error, no effect) — this is intentionally
  different from `build_agent_config()`'s hard-fail behavior at startup, since `/reload` does not
  go through the `_FORBIDDEN_KEYS` check. This asymmetry is the plan's accepted, documented
  behavior (see the plan's R-3 mitigation: "Acceptable behavior... hard-fail only applies at
  startup").
