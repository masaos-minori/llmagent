# Goal

Add `SHELL_TOOLS` and `WEB_SEARCH_TOOLS` to `scripts/shared/tool_constants.py` as single-element frozensets, matching the existing pattern for `SQLITE_TOOLS`. These are prerequisite constants for removing `_EXACT_ROUTES` from `route_resolver.py`.

# Scope

- **File:** `scripts/shared/tool_constants.py`
- **Change:** Add 2 new frozenset constants after `SQLITE_TOOLS`
- **Out of scope:** Existing sets (READ_TOOLS, WRITE_TOOLS, etc.), `get_all_mcp_tool_names()`

# Assumptions

1. Naming convention: `<UPPERCASE_SERVER_KEY>_TOOLS` — consistent with `SQLITE_TOOLS`.
2. `shell_run` is the only tool on the shell server; `search_web` is the only tool on web_search server.
3. `get_all_mcp_tool_names()` must be updated to include the new sets.

# Implementation

## Target file

`scripts/shared/tool_constants.py`

## Procedure

1. After `SQLITE_TOOLS`, add `SHELL_TOOLS` and `WEB_SEARCH_TOOLS`.
2. Add both sets to the union in `get_all_mcp_tool_names()`.

## Method

Insert after `SQLITE_TOOLS` (after line 96):
```python
# Shell execution tools (shell-mcp)
SHELL_TOOLS: frozenset[str] = frozenset({"shell_run"})

# Web search tools (web-search-mcp)
WEB_SEARCH_TOOLS: frozenset[str] = frozenset({"search_web"})
```

Update `get_all_mcp_tool_names()` to include both new sets in the union.

## Details

- `route_resolver.py` will import and use these in `_SET_ROUTES`.
- No behavior change for existing code — these are new symbols only.
