# Implementation: Offload JSONL, offset, and DLQ file I/O from the event loop

Source plan: `plans/20260625-135710_plan.md` (req #18)

## Goal

Wrap all synchronous file I/O operations in `scripts/eventbus/app.py` — JSONL archive append, offset reads/writes, and DLQ `promote_to_dlq` file writes — with `asyncio.to_thread()` so that durability-sensitive file operations no longer block the event loop.

## Scope

- `_append_jsonl()` call in `publish()`: offload the entire function via `asyncio.to_thread`
- `read_offset()` call in `subscribe()` startup: offload
- `write_offset()` calls in `subscribe()` SSE generator: offload
- `promote_to_dlq()` in `_dlq_loop()`: already covered by req #14/#16; confirm `_atomic_write` inside is also offloaded
- `offsets.py` and `dlq.py` source functions remain synchronous — only the calling wrappers in `app.py` change
- Out of scope: DB operation offload (req #14), lock application (req #16)

## Assumptions

1. req #14 (asyncio.to_thread for DB ops) is already implemented; file I/O offload follows the same pattern
2. `_append_jsonl()` is a module-level sync function; it can be passed directly to `asyncio.to_thread`
3. `os.fsync()` inside `_append_jsonl()` must remain (durability requirement); it executes in the thread, not the event loop
4. `write_offset()` is called inside an `async def` SSE generator (`_sse_gen`); it can be awaited with `asyncio.to_thread`

## Implementation

### Target file

`scripts/eventbus/app.py`

### Procedure

1. Wrap `_append_jsonl(body, seq)` call in `publish()` with `asyncio.to_thread`
2. Wrap `read_offset(...)` call in `subscribe()` startup with `asyncio.to_thread`
3. Wrap both `write_offset(...)` calls in `subscribe()` SSE generator with `asyncio.to_thread`
4. Confirm `_dlq_loop()` wraps `promote_to_dlq` with `asyncio.to_thread` (req #14/#16); `_atomic_write` inside `dlq.py` is then automatically offloaded

### Method

Use `await asyncio.to_thread(func, *args)` for each file I/O call. For `_append_jsonl`, the OSError catch block must remain in the async caller.

### Details

**publish() — wrap _append_jsonl:**
```python
try:
    await asyncio.to_thread(_append_jsonl, body, seq)
except OSError as exc:
    logger.warning("eventbus: JSONL append failed (event still committed): %s", exc)
```

**subscribe() — wrap read_offset at startup:**
```python
if consumer_id and start_seq == 0:
    assert _cfg is not None
    start_seq = await asyncio.to_thread(read_offset, _cfg.offsets_dir, consumer_id)
```

**subscribe() SSE generator — wrap write_offset calls:**
```python
# mid-stream checkpoint
if events_since_checkpoint >= _cfg.offset_checkpoint_interval:
    await asyncio.to_thread(write_offset, _cfg.offsets_dir, consumer_id, current_seq)
    events_since_checkpoint = 0

# on disconnect
except asyncio.CancelledError:
    ...
    if consumer_id:
        await asyncio.to_thread(write_offset, _cfg.offsets_dir, consumer_id, current_seq)
```

Note: the `subscribe()` function itself is an async generator (`async def _sse_gen()`). `asyncio.to_thread` can be awaited inside an async generator body.
