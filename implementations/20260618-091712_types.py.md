# Implementation: scripts/shared/types.py (+ models.py cascade)

## Goal

Add `rag_auth_token: str` to `RagConfig` Protocol in `types.py`, propagate through `mcp/rag_pipeline/models.py` (dataclass + load + MCP adapter), and pass the token to `call_rag_service()` via `pipeline.py`.

## Scope

- `scripts/shared/types.py` ‚Äî add `rag_auth_token: str` field to `RagConfig` Protocol
- `scripts/mcp/rag_pipeline/models.py` ‚Äî add `rag_auth_token: str = ""` to `RagPipelineConfig` dataclass + `load()` method + MCP adapter `from_agent_cfg()`
- `scripts/rag/pipeline.py` ‚Äî pass `auth_token=self._cfg.rag_auth_token` to `call_rag_service()`

## Assumptions

1. `rag_auth_token` defaults to `""` (empty string = no auth); existing deployments are unaffected.
2. The token is passed as an `X-RAG-Token` header by `call_rag_service()` (implemented in `pipeline_service.py` doc).
3. `RagConfig` is a `Protocol` (structural subtyping) ‚Äî adding a field with a default at implementation sites still satisfies implementors that don't set it if they use `SimpleNamespace` adapters.
4. `RagPipelineConfig.load()` reads from `rag_pipeline_mcp_server.toml`; the new field is optional with default `""`.
5. `RagPipelineConfig.from_agent_cfg()` converts `AgentConfig`; `AgentConfig` does NOT need `rag_auth_token` ‚Äî the field is MCP-server-specific.

## Implementation

### Target file

`scripts/shared/types.py`, `scripts/mcp/rag_pipeline/models.py`, `scripts/rag/pipeline.py`

### Procedure

1. Add `rag_auth_token: str` to `RagConfig` Protocol in `types.py` (after `rag_service_url`).
2. Add `rag_auth_token: str = ""` field to `RagPipelineConfig` dataclass in `models.py`.
3. Add `rag_auth_token=str(d.get("rag_auth_token", ""))` to `RagPipelineConfig.load()`.
4. Add `rag_auth_token=""` in `RagPipelineConfig.from_agent_cfg()` (explicit empty default).
5. Update `RagPipeline._augment_http()` in `pipeline.py` to pass `auth_token=self._cfg.rag_auth_token` to `call_rag_service()`.
6. Run ruff ‚Üí mypy ‚Üí lint-imports ‚Üí pytest.

### Method

Direct field additions; no new abstractions. The `RagConfig` Protocol only adds one attribute. Existing `SimpleNamespace` adapters that don't set `rag_auth_token` must be updated wherever they're created ‚Äî search with `rg "SimpleNamespace"` in `scripts/`.

### Details

**types.py change:**
```python
class RagConfig(Protocol):
    ...
    rag_service_url: str
    rag_auth_token: str       # <- add here; empty string = no auth
    use_refiner: bool
    ...
```

**models.py dataclass:**
```python
@dataclass
class RagPipelineConfig:
    ...
    rag_service_url: str = ""
    rag_auth_token: str = ""   # <- add
    ...
```

**models.py load():** add `rag_auth_token=str(d.get("rag_auth_token", ""))` alongside `rag_service_url`.

**models.py from_agent_cfg():** add `rag_auth_token=""` (MCP server does not inherit agent auth token).

**pipeline.py _augment_http():**
```python
return await call_rag_service(
    self._http,
    rag_url,
    query,
    history_context,
    auth_token=self._cfg.rag_auth_token,
    set_fetch_result=...,
)
```

**SimpleNamespace adapters** (search with `rg "SimpleNamespace" scripts/`): add `rag_auth_token=""` wherever a `SimpleNamespace` is constructed to satisfy `RagConfig`.
