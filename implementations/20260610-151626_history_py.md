# Implementation: history.py

## Goal

Replace the `except Exception` handler in `_call_compress_llm()` with specific exception types and raise on compression failure instead of returning `None`, so that the caller (`compress()`) propagates the failure to the Orchestrator.

## Scope

- Target: `scripts/agent/history.py`
- Replace `except Exception as e:` in `_call_compress_llm()` with `except (httpx.RequestError, httpx.HTTPStatusError) as e:`
- Change `_call_compress_llm()` return type from `str | None` to `str` (raise on failure instead of returning `None`)
- Update `compress()` to not check `if summary_text is None:` — if LLM call fails, raise; caller handles
- Update `orchestrator.py._handle_history_compression()` to handle the new exception

## Assumptions

1. LLM compression HTTP failures are `httpx.RequestError` or response errors after `resp.raise_for_status()`.
2. Empty LLM response (`raw_content` is None/empty) is treated as a failure — raise `ValueError("LLM returned empty summary")`.
3. `compress()` callers are only in `orchestrator.py._handle_history_compression()`. After this change, that method must handle the exception (decide: propagate or fall back).
4. `force_compress()` calls `compress()` — it will also propagate the exception.

## Implementation

### Target file

`scripts/agent/history.py`

### Procedure

1. In `_call_compress_llm()`:
   - Replace `except Exception as e: logger.warning(...); return None` with `except (httpx.RequestError, httpx.HTTPStatusError) as e: raise`.
   - After parsing: `if not raw_content: raise ValueError("LLM returned empty summary")`.
   - Change return type from `str | None` to `str`.
2. In `compress()`:
   - Remove the `if summary_text is None: return history, _no_op` check — `_call_compress_llm` no longer returns None.
   - `compress()` itself does not catch compression failures; they propagate.
3. In `orchestrator.py._handle_history_compression()`:
   - Wrap `await ctx.services.hist_mgr.compress(...)` in `try/except (httpx.RequestError, ValueError) as e: logger.warning(...); return` (skip compression on failure, do not abort the turn).
   - This is the single catch boundary for compression failure.

### Method

Raise-on-failure in the LLM call; single catch boundary at the orchestrator level.

### Details

```python
# _call_compress_llm — AFTER
async def _call_compress_llm(self, history_text: str) -> str:
    resp = await self._http.post(self._llm_url, json={...})
    resp.raise_for_status()
    data = resp.json()
    choices = data.get("choices") or []
    raw_content = choices[0].get("message", {}).get("content") if choices else None
    if not raw_content:
        raise ValueError("Context compression: LLM returned empty summary")
    return str(raw_content).strip()
    # httpx.RequestError / httpx.HTTPStatusError propagate to compress()
```

```python
# orchestrator.py._handle_history_compression — AFTER
async def _handle_history_compression(self) -> None:
    try:
        ctx.conv.history, _ = await ctx.services.hist_mgr.compress(ctx.conv.history)
    except (httpx.RequestError, ValueError) as e:
        logger.warning(f"History compression skipped: {e}")
```
