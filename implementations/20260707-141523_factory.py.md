# Implementation: H-4 — ensure_ready calls start() for missing subprocess servers (factory.py)

## Goal

Update `_ServerLifecycleRouter.ensure_ready()` to call `await self._http_mgr.start()`
when `verify_running()` returns False for subprocess-mode HTTP servers, so that missing
or crashed processes are restarted on the next tool call.

## Scope

**Target**: `scripts/agent/factory.py`

**Steps covered**: Plan H-4 steps 1, 2 (step 1 is a read-only inspection of
`shared/mcp_config.py`; results are incorporated here).

**Out of scope**: `http_lifecycle.py` (reuses existing `start()`), `context.py`,
`LifecycleManagerProtocol`, config.

## Assumptions

1. `TransportType` and `StartupMode` are `StrEnum` in `shared/mcp_config.py` (confirmed:
   `TransportType.HTTP = "http"`, `StartupMode.SUBPROCESS = "subprocess"`).
2. Current `ensure_ready()` uses string literals `"http"` and `"subprocess"` for
   comparison; these must be replaced with enum members.
3. `_shutting_down` guard (H-3) is applied before any `start()` call; include the guard
   in this implementation regardless of H-3 merge state.
4. `HttpServerLifecycleManager.start()` is idempotent: calling it when the process is
   already running logs a message and returns immediately.
5. `HttpStartupError` from `start()` propagates through `ensure_ready()`; callers handle
   it.

## Implementation

### Target file

`scripts/agent/factory.py`

### Procedure

#### Step 1: Confirm enum imports

Ensure `TransportType` and `StartupMode` are imported from `shared.mcp_config`:

```python
from shared.mcp_config import McpServerConfig, TransportType, StartupMode
```

If H-2 is already implemented, `McpServerHealthRegistry` is also in this import line.

#### Step 2: Replace `ensure_ready()` body

Current implementation:
```python
async def ensure_ready(self, server_key: str) -> None:
    cfg = self._server_configs.get(server_key)
    if cfg is None:
        return
    if cfg.transport == "http" and cfg.startup_mode == "subprocess":
        self._http_mgr.verify_running(server_key)
```

New implementation:
```python
async def ensure_ready(self, server_key: str) -> None:
    if self._shutting_down:
        _logger.debug("Lifecycle: ensure_ready(%r) ignored — shutting down", server_key)
        return
    cfg = self._server_configs.get(server_key)
    if cfg is None:
        return
    if cfg.transport != TransportType.HTTP or cfg.startup_mode != StartupMode.SUBPROCESS:
        return
    if not self._http_mgr.verify_running(server_key):
        _logger.info(
            "Lifecycle: %r not running; starting via ensure_ready", server_key
        )
        await self._http_mgr.start(server_key, cfg)
```

### Method

- Single method replacement; 4 lines become 10 lines.
- The `_shutting_down` guard is included here; if H-3 is already merged, this guard is
  redundant but not harmful (idempotent bool check).
- String literals `"http"` and `"subprocess"` are replaced with enum members
  `TransportType.HTTP` and `StartupMode.SUBPROCESS`.

### Details

- `verify_running()` logs a warning when the process is not running; `ensure_ready()`
  adds an INFO log before calling `start()` for traceability.
- Persistent-mode servers (`StartupMode.PERSISTENT`) are excluded by the
  `startup_mode != StartupMode.SUBPROCESS` check; their lifecycle is externally managed.
- Unknown server keys (not in `_server_configs`) return early via `if cfg is None`.
- `start()` raises `HttpStartupError` if the server fails to start; this propagates
  through `ensure_ready()` to the caller (`_raw_execute` in `tool_executor.py`). The
  H-5 plan adds a try/except in `_raw_execute` for this case.
- If H-3 is not yet merged, `_shutting_down` does not exist on the instance; copy the
  guard from H-3 step 1 verbatim so this implementation is self-contained.
