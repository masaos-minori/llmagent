## Goal

Wire `ctx.workflow` (the new `WorkflowState` field from step 1) into `Orchestrator.handle_turn()` so the agent's runtime state reflects the current workflow execution state.

## Scope

- Set `ctx.workflow.current_task_id`, `ctx.workflow.current_workflow_version`, `ctx.workflow.active` on task creation
- Clear `ctx.workflow.active` and `ctx.workflow.current_task_id` on completion or failure
- Set `ctx.workflow.approval_pending = True` in `WorkflowPendingApprovalError` handler
- Clear `ctx.workflow.approval_pending = False` on successful completion

## Assumptions

- `WorkflowState` is already added to `context.py` (step 1 completed)
- All workflow state assignments are in `Orchestrator.handle_turn()` only
- `ctx.workflow` is transient; no persistence needed

## Implementation

### Target file

`scripts/agent/orchestrator.py`

### Procedure

1. After `task = store.create_task(...)`, add:
   ```python
   ctx.workflow.current_task_id = task.task_id
   ctx.workflow.current_workflow_version = self._workflow_def.version
   ctx.workflow.active = True
   ```
2. After `await engine.run(...)` succeeds (i.e., before the `except` blocks), add:
   ```python
   ctx.workflow.active = False
   ctx.workflow.current_task_id = None
   ```
3. In the `except WorkflowPendingApprovalError` handler, add:
   ```python
   ctx.workflow.approval_pending = True
   ```
4. In the `except WorkflowHaltError` handler, add:
   ```python
   ctx.workflow.active = False
   ctx.workflow.current_task_id = None
   ```
5. In the `finally` block, ensure cleanup only if not pending approval:
   - `ctx.workflow.active` is already cleared in both normal and halt paths; pending approval leaves `active=True` intentionally (turn is suspended, not ended)

### Method

Direct edit.

### Details

After the `engine.run()` call and before the except blocks, the structure becomes:

```python
await engine.run(task, plan_fn, execute_fn, verify_fn)
ctx.workflow.active = False
ctx.workflow.current_task_id = None
```

In WorkflowPendingApprovalError handler:
```python
except WorkflowPendingApprovalError as exc:
    logger.info(...)
    audit_approval_requested(ctx, exc.task_id, exc.approval_id)
    ctx.turn.pending_approval_id = exc.approval_id
    ctx.workflow.approval_pending = True
```

In WorkflowHaltError handler:
```python
except WorkflowHaltError as exc:
    logger.error("Turn halted by workflow engine: %s", exc)
    ctx.workflow.active = False
    ctx.workflow.current_task_id = None
    if self._on_error:
        self._on_error(exc)
```

`last_session_id` on `WorkflowState` is not set by the orchestrator (it is set externally or left None for now; the plan does not specify a setter).
