# Implementation Procedure: Normalize configuration schema — remove legacy Memory keys and GitOps security fields

## Goal

Remove deprecated Memory configuration keys (`use_memory_layer`, `memory_jsonl_dir`, `memory_embed_enabled`) and clarify/remove GitOps security fields (`gitops_force_push_blocked`, `gitops_protected_branches`). Add removed-key validation, include original exception messages in `ConfigLoadError`, and add reload preflight validation.

## Scope

### In scope
- `scripts/shared/config_loader.py` — _filter_removed_keys()

### Out of scope
- `agent/memory/*` modules
- `agent/tool_policy.py`
- `agent/tool_approval.py`
- Documentation files

## Assumptions

1. A post-load filter method in `config_loader.py` will silently drop removed keys before they reach builders.
2. The filter should be applied after `_filter_meta_keys()` in both `load()` and `load_all()`.
3. `_REMOVED_KEYS` constant should match the set defined in `config_reload.py` and `config_builders.py`.

## Implementation

### Target file

`scripts/shared/config_loader.py`

### Procedure

#### Step 1: Add removed-key filter

Add a post-load filter method in `config_loader.py`:

```python
_REMOVED_KEYS = frozenset((
    "use_memory_layer",
    "memory_jsonl_dir",
    "memory_embed_enabled",
    "gitops_force_push_blocked",
    "gitops_protected_branches",
))

@staticmethod
def _filter_removed_keys(data: dict[str, Any]) -> dict[str, Any]:
    """Remove deprecated keys that are no longer part of the schema."""
    return {k: v for k, v in data.items() if k not in _REMOVED_KEYS}
```

#### Step 2: Apply filter in load() and load_all()

Apply this filter after `_filter_meta_keys()` in both `load()` and `load_all()`:

```python
# In load():
data = self._filter_meta_keys(data)
data = self._filter_removed_keys(data)
# ... rest of existing logic

# In load_all():
result = self._filter_meta_keys(result)
result = self._filter_removed_keys(result)
# ... rest of existing logic
```

### Method

- Use dictionary comprehension for efficient key filtering
- Define `_REMOVED_KEYS` as module-level `frozenset` for immutability and performance
- Apply filter immediately after meta-key filtering to ensure consistent ordering

### Details

**Type safety considerations:**
- `_REMOVED_KEYS` uses `frozenset` for immutability — prevents accidental mutation
- Dictionary comprehension preserves value types — no type coercion occurs

**Import considerations:**
- No new imports required for this file

**Cross-file consistency:**
- `_REMOVED_KEYS` in this file must match the sets defined in `config_reload.py` and `config_builders.py`
- Consider consolidating into a shared constant to avoid duplication across three files

**Behavioral note:**
- Unlike `config_builders.py` which rejects removed keys at load time, this filter silently drops them
- This provides backward compatibility for users who haven't updated their configs yet
- However, the `_FORBIDDEN_KEYS` approach in `config_builders.py` will still catch these during validation
