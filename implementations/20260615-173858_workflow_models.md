# Implementation: scripts/agent/workflow/models.py

## Goal

Define the core dataclasses used by the workflow subsystem: `TaskRecord`, `AttemptRecord`, `ArtifactRef`, and `WorkflowDef` (plus its nested types `StageDefinition` and `RetryPolicy`).

## Scope

**In:**
- `scripts/agent/workflow/__init__.py` — new, empty package marker
- `scripts/agent/workflow/models.py` — new file

**Out:**
- Any database access
- Any file I/O

## Assumptions

1. All fields map 1-to-1 to the `workflow.sqlite` schema defined in `db/workflow_schema.py`.
2. Datetime values are stored as ISO-8601 strings in SQLite and represented as `str` in Python (no `datetime` objects to avoid timezone complexity).
3. `WorkflowDef`, `StageDefinition`, and `RetryPolicy` mirror the JSON structure of `config/workflows/default.json`.
4. `models.py` has no imports from outside `agent/` or stdlib — it is a pure data definition module.

## Implementation

### Target files

- `scripts/agent/workflow/__init__.py`
- `scripts/agent/workflow/models.py`

### Procedure

1. Create `scripts/agent/workflow/__init__.py` (empty).
2. Create `scripts/agent/workflow/models.py` with the dataclasses below.

### Method

```python
#!/usr/bin/env python3
"""agent/workflow/models.py
Dataclasses for the Metadata DB entities and workflow definition schema.
"""
from __future__ import annotations
from dataclasses import dataclass, field


@dataclass
class TaskRecord:
    task_id: str
    session_id: str
    turn_number: int
    workflow_version: str
    status: str          # pending | running | completed | failed | halted
    idempotency_key: str
    created_at: str      # ISO-8601
    updated_at: str      # ISO-8601


@dataclass
class AttemptRecord:
    attempt_id: str
    task_id: str
    stage_id: str
    status: str          # running | completed | failed
    started_at: str      # ISO-8601
    ended_at: str | None = None
    error_msg: str | None = None


@dataclass
class ArtifactRef:
    artifact_id: str
    task_id: str
    stage_id: str
    uri: str
    created_at: str      # ISO-8601


@dataclass
class StageDefinition:
    id: str
    description: str
    timeout_sec: int
    retryable: bool


@dataclass
class RetryPolicy:
    max_attempts: int
    backoff: str         # "fixed" | "exponential"
    backoff_sec: int


@dataclass
class WorkflowDef:
    name: str
    version: str
    stages: list[StageDefinition] = field(default_factory=list)
    retry_policy: RetryPolicy = field(
        default_factory=lambda: RetryPolicy(max_attempts=3, backoff="fixed", backoff_sec=1)
    )

    def get_stage(self, stage_id: str) -> StageDefinition | None:
        """Return the StageDefinition for the given id, or None."""
        return next((s for s in self.stages if s.id == stage_id), None)
```

### Details

- `status` values are deliberately plain strings (not enums) to avoid import cycles and simplify SQLite round-trips.
- `get_stage()` is the only method on `WorkflowDef`; all logic lives in `workflow_engine.py`.
- `field(default_factory=...)` is required for mutable defaults on dataclasses.
