# Implementation: mcp/file/read_tools.py — Schema single-source via model_json_schema()

## Goal

Replace the hand-written `_MCP_TOOLS` list in `read_tools.py` with schemas generated
from the Pydantic request models in `read_models.py`. This eliminates the risk of
the tool schema drifting out of sync with the model definitions.

## Scope

**Target files**:
- `scripts/mcp/file/read_tools.py` (generate schema from models)
- `tests/test_mcp_tools_validation.py` (add consistency test)

In scope:
- Replace hand-written schemas with `model_json_schema()`-derived schemas
- Add a test that asserts the generated schema matches the hand-written one

Out of scope:
- write and delete tools (Phase B per plan)
- Modifying the Pydantic models themselves

## Assumptions

1. Pydantic v2's `model_json_schema()` returns a JSON Schema dict.
   For simple models without nested `$defs`, the output directly contains
   `type`, `properties`, and `required` keys usable as MCP `inputSchema`.
2. For `ReadMultipleFilesRequest` (has a `list[str]` field), `model_json_schema()`
   does NOT produce `$defs` — the schema is flat.
3. Some tools (`list_directory_with_sizes`) share the same request model as
   `list_directory` (`ListDirectoryRequest`). These must be handled by calling
   `model_json_schema()` once and referencing it for multiple tool entries.
4. The generated schema must match the existing hand-written schema exactly for
   consistency check purposes. If fields differ, update the hand-written schema first.

**Step 2 — implement schema generation utility**

Replace the hand-written `_MCP_TOOLS` with a generator function:

```python
from __future__ import annotations

from typing import Any

from mcp.file.read_models import (
    DirectoryTreeRequest,
    GetFileInfoRequest,
    GrepFilesRequest,
    ListDirectoryRequest,
    ReadMediaFileRequest,
    ReadMultipleFilesRequest,
    ReadTextFileRequest,
    SearchFilesRequest,
)


def _tool(name: str, description: str, model: type, **overrides: Any) -> dict[str, Any]:
    """Build an MCP tool entry from a Pydantic model's JSON schema."""
    schema = model.model_json_schema()
    # Extract only type/properties/required; drop $defs, title, description if present
    input_schema: dict[str, Any] = {"type": "object"}
    if "properties" in schema:
        input_schema["properties"] = schema["properties"]
    if "required" in schema:
        input_schema["required"] = schema["required"]
    input_schema.update(overrides)
    return {"name": name, "description": description, "inputSchema": input_schema}


_MCP_TOOLS = [
    _tool("list_directory",
          "Return immediate entries of the specified directory",
          ListDirectoryRequest),
    _tool("list_directory_with_sizes",
          "Return directory entries with sizes including stat size for directories",
          ListDirectoryRequest),
    _tool("directory_tree",
          "Recursively return the tree structure of a directory",
          DirectoryTreeRequest),
    _tool("read_text_file",
          "Read file contents as UTF-8 text. head/tail options limit the number of lines returned",
          ReadTextFileRequest),
    _tool("read_media_file",
          "Return a media file (image/audio etc.) base64-encoded with its MIME type",
          ReadMediaFileRequest),
    _tool("read_multiple_files",
          "Retrieve multiple files in a single request. Continues even if individual errors occur",
          ReadMultipleFilesRequest),
    _tool("search_files",
          "Recursively search for files matching a glob pattern within a directory",
          SearchFilesRequest),
    _tool("grep_files",
          "Search file contents under a directory using a regex pattern",
          GrepFilesRequest),
    _tool("get_file_info",
          "Return metadata (size, timestamps, permissions) for a file or directory",
          GetFileInfoRequest),
]
```

### Method

Full file rewrite using the `Write` tool.

### Details

- `model_json_schema()` may include Pydantic-specific annotations (e.g., `title` on
  properties). The `_tool()` helper strips to just `type`/`properties`/`required`.
- If a model has `ge`/`le` constraints (e.g., `GrepFilesRequest.max_matches`), they
  will appear in the generated schema as `minimum`/`maximum` — this is valid JSON Schema
  and compatible with MCP.
- The `list_directory_with_sizes` tool intentionally uses the same `ListDirectoryRequest`
  model as `list_directory`.
- After implementing, run the consistency test to confirm the old and new schemas match
  for all 9 tools.

```python
def test_read_tools_schema_matches_hand_written():
    """Generated schema must have same keys as the original hand-written schema."""
    from mcp.file.read_tools import _MCP_TOOLS
    names = [t["name"] for t in _MCP_TOOLS]
    assert names == [
        "list_directory", "list_directory_with_sizes", "directory_tree",
        "read_text_file", "read_media_file", "read_multiple_files",
        "search_files", "grep_files", "get_file_info",
    ]
    for tool in _MCP_TOOLS:
        assert "inputSchema" in tool
        assert tool["inputSchema"]["type"] == "object"
        assert "properties" in tool["inputSchema"]
```
