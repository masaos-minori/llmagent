# Goal

Separate `LLMMessage`'s always-required `role` field from optional fields using
TypedDict inheritance, and add `ToolCallDict` / `ToolCallFunction` TypedDicts to
replace `list[dict[str, Any]]` for tool call structures.

# Scope

- `scripts/shared/types.py`

# Assumptions

1. Splitting `LLMMessage` into `_LLMMessageRequired` (role) and `LLMMessage`
   (optional fields) is backward-compatible: existing `m["role"]` accesses still
   work because `role` is still a key on `LLMMessage`.
2. `ToolCallFunction` and `ToolCallDict` are new TypedDicts that can be referenced
   in `llm_client.py` (Step 8). They do not replace existing `list[dict[str, Any]]`
   in this step — just define the types.
3. `RagConfig` Protocol is unchanged.

# Implementation

## Target file

`scripts/shared/types.py`

## Procedure

1. Split `LLMMessage`:
   ```python
   class _LLMMessageRequired(TypedDict):
       role: Literal["user", "assistant", "tool", "system"]

   class LLMMessage(_LLMMessageRequired, total=False):
       content: str | None
       tool_calls: list[dict[str, Any]]
       tool_call_id: str
       name: str
       importance: float
       pinned: bool
   ```

2. Add after `LLMMessage`:
   ```python
   class ToolCallFunction(TypedDict):
       name: str
       arguments: str

   class ToolCallDict(TypedDict):
       id: str
       type: Literal["function"]
       function: ToolCallFunction
   ```

3. Update `__all__` to include new types.

4. Run ruff + mypy on the full codebase (types.py is imported by 14 files).

## Method

Append-only plus split of existing class. No runtime behavior change.

## Details

The split requires removing the `Required` import from `typing` (or keeping it if
used elsewhere). Check with `grep -n "Required" scripts/shared/types.py` before editing.
