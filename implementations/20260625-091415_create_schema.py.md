# Implementation: scripts/db/create_schema.py

## Goal

Add `_migrate_session_schema(conn)` to `create_schema.py` and call it from `create_session_schema()` to add the `REFERENCES sessions(session_id) ON DELETE CASCADE` FK constraint to `tool_results.session_id` in existing databases.

## Scope

- Target: `scripts/db/create_schema.py`
- Add `_migrate_session_schema(conn: sqlite3.Connection) -> None`
- Call `_migrate_session_schema(db.conn)` at end of `create_session_schema()`
- No changes to `ToolResultStore` API or other callers

## Assumptions

1. `ALTER TABLE ... ADD CONSTRAINT` is unavailable in SQLite — FK must be added via table recreation.
2. `PRAGMA foreign_key_list(tool_results)` returns rows if FK already exists; use this to guard against re-running migration.
3. `conn` has `PRAGMA foreign_keys = ON` already set or migration sets it locally.
4. Existing `tool_results` data must be preserved — use `INSERT INTO ... SELECT ... FROM` before dropping the old table.
5. Migration runs inside a transaction so a crash leaves the database intact.

## Implementation

### Target file
`scripts/db/create_schema.py`

### Procedure
1. Add `_migrate_session_schema(conn: sqlite3.Connection) -> None` function.
2. At end of `create_session_schema()`, after `db.executescript(...)`, call `_migrate_session_schema(db.conn)`.

### Method

```python
def _migrate_session_schema(conn: sqlite3.Connection) -> None:
    """Add FK constraint to tool_results.session_id if not already present."""
    fk_info = conn.execute("PRAGMA foreign_key_list(tool_results)").fetchall()
    if fk_info:
        return  # FK already in place; skip

    conn.execute("PRAGMA foreign_keys = ON")
    conn.executescript("""
        BEGIN;
        CREATE TABLE tool_results_new (
            id         INTEGER PRIMARY KEY AUTOINCREMENT,
            session_id INTEGER REFERENCES sessions(session_id) ON DELETE CASCADE,
            turn       INTEGER NOT NULL,
            tool_name  TEXT    NOT NULL,
            args_masked  TEXT,
            full_text  TEXT    NOT NULL,
            summary    TEXT,
            is_error   INTEGER NOT NULL DEFAULT 0,
            created_at TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
        );
        INSERT INTO tool_results_new
            SELECT id, session_id, turn, tool_name, args_masked, full_text, summary, is_error, created_at
            FROM tool_results;
        DROP TABLE tool_results;
        ALTER TABLE tool_results_new RENAME TO tool_results;
        CREATE INDEX IF NOT EXISTS idx_tool_results_session ON tool_results(session_id);
        COMMIT;
    """)
```

### Details
- The `BEGIN`/`COMMIT` inside `executescript` makes the migration atomic.
- Column list in `INSERT INTO ... SELECT` must match the old table exactly — verify against current `schema_sql.py` before committing.
- `PRAGMA foreign_key_list` check makes the function idempotent.
- `db.conn` in `create_session_schema()` must be the live connection after `executescript`.
