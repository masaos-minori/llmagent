## Goal

Replace `ctx.services.llm` accesses in `scripts/agent/llm_turn_runner.py` with `ctx.services_required.llm`, and remove the now-redundant explicit None guard. This is Phase 3 (final code phase) of the ctx.services type migration plan.

## Scope

**In**: `scripts/agent/llm_turn_runner.py` lines 177–179 only.

**Out**: All other files. Specifically, `repl.py` lines 154–155, 384 are excluded (legitimate None guards).

## Assumptions

1. `ctx.services.llm` at lines 177–179 is inside `_stream_llm()`, called only after full agent initialization. `services_required` is safe.
2. The existing explicit `if ctx.services.llm is None: raise RuntimeError(...)` guard is redundant because `services_required` raises its own `RuntimeError` when `services is None`. Removing the explicit guard simplifies the code without changing observable behavior.
3. This phase depends on Phase 1 (`context.py`) being completed first; otherwise mypy will not benefit from the type narrowing.

## Implementation

### Target file
`scripts/agent/llm_turn_runner.py`

### Procedure
1. Read lines 169–185 to confirm exact current structure.
2. Simplify the None-check + access into a single `services_required` call.
3. Run mypy + ruff.

### Method
Direct file edit.

### Details

**Lines 177–179** — current code (approximate):

```python
# Before
if ctx.services.llm is None:
    raise RuntimeError("LLM client not initialized")
response = await ctx.services.llm.stream(...)
```

**After** — remove the explicit guard; `services_required` already raises `RuntimeError` on None:

```python
# After
response = await ctx.services_required.llm.stream(...)
```

The removal of the explicit guard is safe because:
- `services_required` property raises `RuntimeError("AgentContext.services not initialized")` when `services is None`.
- The error type (RuntimeError) and the fail behavior are preserved.
- No caller catches `RuntimeError` from this specific guard, so the change is transparent.
