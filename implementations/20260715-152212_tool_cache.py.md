# Implementation Procedure: scripts/shared/tool_cache.py

Source plan: `plans/20260715-140914_plan.md`

## Prior-doc divergence note

`implementations/done/20260615-110633_tool_cache.py.md` targets the same file
but is from an unrelated, already-completed refactor (extracting the cache
entry dataclass out of `tool_executor.py` into this module, phase 3 of a
responsibility-separation task; that plan's `_CacheEntry` had no `server_key`
field and pre-dates the current, already-public `CacheEntry` name and
`ToolResultCache` class). It does not address this plan's requirement at all
(adding a `server_key` attribution field). Confirmed against current source
(`scripts/shared/tool_cache.py`, read in full): `CacheEntry` today has exactly
`output: str`, `is_error: bool`, `cached_at: float` â€” no `server_key` field â€”
and `ToolResultCache.get_result()` hard-codes `server_key=""` on every
cache-hit `ToolCallResult` (line 61). This document is written fresh for this
plan.

## Goal

Add a `server_key: str = ""` field to `CacheEntry` and thread it through
`ToolResultCache.store_if_success()` / `get_result()` so cache-hit results
carry the `server_key` of the server that originally produced them, and mark
successful cache hits with `source="cache"` for observability.

## Scope

**In scope**
- `CacheEntry` â€” add `server_key: str = ""` field.
- `ToolResultCache.store_if_success()` â€” persist `result.server_key` into the
  new field.
- `ToolResultCache.get_result()` â€” return `server_key=cached.server_key` and
  `source="cache"` on the reconstructed `ToolCallResult` (only for successful
  entries; error entries are never stored, see Assumptions).

**Out of scope**
- No change to `ToolResultCache.make_key()` or `clear()`.
- No change to the module's "not currently used by ToolExecutor" status note
  in the docstring â€” still accurate; `ToolExecutor` keeps its own internal
  `OrderedDict`-based cache (see
  `implementations/20260715-152212_tool_executor.py.md`), which also gains a
  `server_key`-carrying `CacheEntry` via the same dataclass defined here.
- No new production caller is introduced for `ToolResultCache` by this plan.

## Assumptions

- `CacheEntry` is shared between `ToolResultCache` (this file) and
  `ToolExecutor`'s internal cache dict (`scripts/shared/tool_executor.py`,
  which already imports `CacheEntry` from this module at line 28) â€” adding the
  field here automatically makes it available to both; `ToolExecutor`'s own
  `_store_and_evict()` / cache-hit branch construct `CacheEntry` and
  `ToolCallResult` independently and are updated separately in
  `implementations/20260715-152212_tool_executor.py.md`.
- `store_if_success()` early-returns on `result.is_error` (current line 67-68)
  before ever constructing a `CacheEntry`, so no error entry ever needs a
  `server_key` â€” `source="cache"` is therefore only ever set on the success
  path in `get_result()`.
- `ToolCallResult` (from `shared.transport_dto`) already has both `server_key`
  and `source` fields (confirmed by reading `transport_dto.py`); no DTO change
  needed here, only how this module populates them (see also
  `implementations/20260715-152212_transport_dto.py.md` for the doc-comment
  update on `source`).

## Implementation

### Target file

`scripts/shared/tool_cache.py`

### Procedure

1. Add `server_key: str = ""` as the fourth field on `CacheEntry` (after
   `cached_at`, current lines 27-32).
2. In `store_if_success()` (current lines 65-73), pass
   `server_key=result.server_key` into the `CacheEntry(...)` construction.
3. In `get_result()` (current lines 47-63), change the returned
   `ToolCallResult(...)` to use `server_key=cached.server_key` (in place of
   the hard-coded `""`) and `source="cache"` in place of the omitted default.

### Method

Additive dataclass field + two small constructor-argument changes; no
signature changes to any public method.

### Details

```python
@dataclass(frozen=True)
class CacheEntry:
    """LRU cache entry storing a successful tool call result."""

    output: str
    is_error: bool
    cached_at: float
    server_key: str = ""
```

`get_result()`:

```python
        self._cache.move_to_end(key)
        return ToolCallResult(
            output=cached.output,
            is_error=cached.is_error,
            request_id="",
            server_key=cached.server_key,
            source="cache",
            error_type="tool" if cached.is_error else "",
        )
```

`store_if_success()`:

```python
        self._cache[key] = CacheEntry(
            output=result.output,
            is_error=result.is_error,
            cached_at=time.time(),
            server_key=result.server_key,
        )
```
