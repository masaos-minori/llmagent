# Implementation: tool_audit.py — workflow event writers

## Goal

Add structured audit log writers for workflow lifecycle events.

## Scope

- `scripts/agent/tool_audit.py`
- `scripts/agent/orchestrator.py` — call new audit functions

## New audit functions

```python
def audit_workflow_start(ctx, task_id, workflow_version) -> None:
    """Write workflow_start event to audit log."""

def audit_stage_completed(ctx, task_id, stage_id, elapsed_ms) -> None:
    """Write stage_completed event to audit log."""

def audit_approval_requested(ctx, task_id, approval_id) -> None:
    """Write approval_requested event to audit log."""
```

All use `ctx.services.audit_logger.info(...)` pattern matching existing events.

Event schema (JSON-lines):
```json
{"event": "workflow_start", "task_id": "...", "workflow_version": "...", "ts": 1234.56}
{"event": "stage_completed", "task_id": "...", "stage_id": "plan", "elapsed_ms": 12.3, "ts": 1234.56}
{"event": "approval_requested", "task_id": "...", "approval_id": "...", "ts": 1234.56}
```
