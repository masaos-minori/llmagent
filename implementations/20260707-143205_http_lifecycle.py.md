# Implementation: H-8 — Process group shutdown in http_lifecycle.py

## Goal

Start HTTP MCP subprocesses with `start_new_session=True` and store the process group
ID (`pgid`) in `_http_pgids`. Update `_terminate_with_timeout()` to send `SIGTERM` /
`SIGKILL` to the process group via `os.killpg()`, falling back to `proc.terminate()` /
`proc.kill()` on `ProcessLookupError` or `OSError`.

## Scope

**Target**: `scripts/agent/http_lifecycle.py`

**Steps covered**: Plan H-8 steps 1, 2, 3.

**Out of scope**: `factory.py`, `startup.py`, `shutdown_all()` (affected only indirectly
through `_terminate_with_timeout()`), config.

## Assumptions

1. `os.killpg()` is available on Linux (WSL2 target platform).
2. `os.getpgid(proc.pid)` is called immediately after `Popen()` with no significant
   race; wrapped in `try/except OSError` for the rare case the process exits first.
3. Fallback to `proc.terminate()` / `proc.kill()` is correct when the process group is
   gone (`ProcessLookupError`) or pgid is unknown.
4. `start_new_session=True` does not affect `env` dict injection or working directory.
5. H-1's `_stderr_files` dict does not interfere; H-8 only touches `_http_pgids`.

## Implementation

### Target file

`scripts/agent/http_lifecycle.py`

### Procedure

#### Step 1: Add imports

Add to the existing stdlib imports block:
```python
import os
import signal
```

`os` is likely already imported; `signal` may need to be added.

#### Step 2: Add `_http_pgids` to `__init__()`

In `HttpServerLifecycleManager.__init__()`, add:
```python
self._http_pgids: dict[str, int] = {}
```

#### Step 3: Update `start()` to set `start_new_session=True` and capture pgid

In the `subprocess.Popen(...)` call, add `start_new_session=True`:
```python
proc = subprocess.Popen(  # nosec B603
    cfg.cmd,
    stdout=subprocess.DEVNULL,
    stderr=stderr_fh,       # if H-1 merged; else stderr=subprocess.PIPE
    env=env,
    start_new_session=True,
)
```

After `proc = subprocess.Popen(...)` and before `self._http_procs[server_key] = proc`:
```python
try:
    self._http_pgids[server_key] = os.getpgid(proc.pid)
except OSError:
    self._http_pgids[server_key] = proc.pid
```

#### Step 4: Clean up `_http_pgids` in `restart()` and `shutdown_all()`

In `restart()`, before the `_http_procs.pop()` call (or alongside it):
```python
self._http_pgids.pop(server_key, None)
```

In `shutdown_all()`, inside the per-server loop (compatible with H-7 refactor):
```python
self._http_pgids.pop(key, None)
```

If H-7 is not yet merged and `shutdown_all()` still uses `list(self._http_procs.items())`
iteration, add a `self._http_pgids.clear()` at the end of `shutdown_all()`.

#### Step 5: Update `_terminate_with_timeout()` to use `os.killpg()`

Replace:
```python
proc.terminate()
```
With:
```python
pgid = self._http_pgids.get(server_key, proc.pid)
try:
    os.killpg(pgid, signal.SIGTERM)
except (ProcessLookupError, OSError):
    proc.terminate()
```

Replace the kill-on-second-timeout:
```python
proc.kill()
```
With:
```python
pgid = self._http_pgids.get(server_key, proc.pid)
try:
    os.killpg(pgid, signal.SIGKILL)
except (ProcessLookupError, OSError):
    proc.kill()
```

The signature of `_terminate_with_timeout()` becomes:
```python
async def _terminate_with_timeout(
    self,
    proc: subprocess.Popen[bytes],
    server_key: str,
    timeout: float = 3.0,
) -> None:
```

(Signature already matches; `server_key` was already a parameter.)

### Method

- Adds two new stdlib imports: `os` and `signal`.
- Adds one new instance dict `_http_pgids`.
- Modifies `start()`, `restart()`, `shutdown_all()`, and `_terminate_with_timeout()`.
- No new external dependencies.

### Details

- `start_new_session=True` creates a new process group where the subprocess is the
  leader. `os.getpgid(proc.pid)` returns `proc.pid` in this case (a process is its own
  group leader in a new session). The `try/except` guard handles the degenerate case
  where the process exits before `getpgid()` completes.
- `os.killpg(pgid, SIGTERM)` sends the signal to every process in the group, catching
  child processes that the MCP server may have spawned (e.g. a Node.js process spawning
  worker threads with separate pids).
- The `ProcessLookupError` fallback handles the case where the process group already
  exited between `getpgid()` and `killpg()`.
- `bandit` may flag `os.killpg`; add `# nosec B603` if needed, consistent with the
  existing `subprocess.Popen` comment.
