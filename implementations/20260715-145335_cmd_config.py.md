# Implementation Procedure: cmd_config.py

## Goal

Remove the `_ConfigSetMixin` import and its composition from `_ConfigMixin`'s
base-class tuple, since `cmd_config_set.py` (and the `/set` command it implements)
is being deleted. `_cmd_reload` and all other `_ConfigMixin` behavior are unaffected.

## Scope

### In scope
- Remove `from agent.commands.cmd_config_set import _ConfigSetMixin  # noqa: E402`
  (current line 23).
- Remove `_ConfigSetMixin` from `_ConfigMixin(_ConfigStatsMixin, _ConfigDisplayMixin, _ConfigSetMixin)`
  (current lines 29-33).
- Update the module docstring (current lines 6-9) to drop the
  `cmd_config_set   — _set_temperature, _set_max_tokens, _cmd_set` line.

### Out of scope
- `_cmd_reload` (current lines 39-103) — untouched, still reads `config/*.toml` via
  `ConfigLoader`/`ConfigReloadService` exactly as today.
- `_ConfigStatsMixin` / `_ConfigDisplayMixin` and their source files
  (`cmd_config_stats.py`, `cmd_config_display.py`) — untouched.

## Assumptions

- `_ConfigMixin.__init__` (current lines 36-37, just `super().__init__(*args, **kwargs)`)
  needs no change beyond the base-class tuple edit — the MRO chain to
  `_ConfigStatsMixin`/`_ConfigDisplayMixin`/`MixinBase` (via `mixin_base.py`) is
  unaffected by removing one mixin from the tuple, since Python's C3 linearization
  handles the shorter tuple transparently.
- `registry.py`'s composition of `_ConfigMixin` into `CommandRegistry` needs no direct
  edit for this change (it composes `_ConfigMixin` as a whole, not its sub-mixins).

## Implementation

### Target file

`scripts/agent/commands/cmd_config.py`

### Procedure

1. Delete the import line:
   ```python
   from agent.commands.cmd_config_set import _ConfigSetMixin  # noqa: E402
   ```
2. Change the class declaration from:
   ```python
   class _ConfigMixin(
       _ConfigStatsMixin,
       _ConfigDisplayMixin,
       _ConfigSetMixin,
   ):
   ```
   to:
   ```python
   class _ConfigMixin(
       _ConfigStatsMixin,
       _ConfigDisplayMixin,
   ):
   ```
3. Update the module docstring's sub-module list (current lines 6-9) to remove the
   `cmd_config_set` line, leaving:
   ```
   Split into sub-modules for single-responsibility:
     cmd_config_stats   — _collect_stats, _cmd_stats
     cmd_config_display — all _print_* helpers, _cmd_config
   ```

### Method

Targeted text edit: one import removal, one base-class-tuple entry removal, one
docstring line removal. No behavioral logic in this file changes.

### Details

- This edit must land in the same change-set as `cmd_config_set.py`'s deletion —
  otherwise `cmd_config.py` would import a module that no longer exists
  (`ModuleNotFoundError` at import time, which would fail every test that imports
  `agent.commands.registry` transitively).
- After this edit, `_ConfigMixin` no longer provides `_cmd_set`, `_set_temperature`,
  or `_set_max_tokens` — confirm no other mixin or `registry.py` code path still
  references them (it does not; `_cmd_set` was `/set`'s sole handler).

## Validation plan

- `uv run python -m compileall -q scripts/agent/commands/cmd_config.py` — syntax check.
- `uv run pytest tests/test_agent_cmd_config.py -v` — existing config/stats/display
  tests still pass (they do not exercise `_cmd_set`).
- `uv run pytest tests/test_command_def_sync.py -v` — `test_every_handler_has_commanddef`
  no longer needs to special-case `_cmd_set` (it was never in
  `INTENTIONAL_HANDLER_EXCLUSIONS`, so nothing to update there; it simply ceases to exist).
- `PYTHONPATH=scripts uv run lint-imports` — clean; `_ConfigMixin` still only depends on
  `shared`/`agent` layers within contract.
- `uv run ruff check scripts/agent/commands/cmd_config.py` — no unused-import lint errors
  remain (the `# noqa: E402` comments on the two surviving imports are untouched).
