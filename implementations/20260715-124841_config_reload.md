# Implementation Procedure: Normalize configuration schema — remove legacy Memory keys and GitOps security fields

## Goal

Remove deprecated Memory configuration keys (`use_memory_layer`, `memory_jsonl_dir`, `memory_embed_enabled`) and clarify/remove GitOps security fields (`gitops_force_push_blocked`, `gitops_protected_branches`). Add removed-key validation, include original exception messages in `ConfigLoadError`, and add reload preflight validation.

## Scope

### In scope
- `scripts/agent/services/config_reload.py` — _detect_startup_only(), reload logic

### Out of scope
- `agent/memory/*` modules
- `agent/tool_policy.py`
- `agent/tool_approval.py`
- Documentation files

## Assumptions

1. `_detect_startup_only()` should no longer track `use_memory_layer` changes since that key is removed.
2. Preflight validation in `apply_config_dict()` should reject any config containing removed keys before mutating runtime config.
3. `_REMOVED_KEYS` constant in `config_reload.py` must match the set defined in `config_builders.py`.

## Implementation

### Target file

`scripts/agent/services/config_reload.py`

### Procedure

Remove `use_memory_layer` tracking from `_detect_startup_only()`:

#### Step 2: Add reload preflight validation

Add a preflight validation step before applying changes in `apply_config_dict()`:

```python
_REMOVED_KEYS = frozenset((
    "use_memory_layer",
    "memory_jsonl_dir",
    "memory_embed_enabled",
    "gitops_force_push_blocked",
    "gitops_protected_branches",
))

def apply_config_dict(self, new_cfg: dict[str, Any]) -> ConfigReloadOutcome:
    ctx = self._ctx
    # Preflight: reject removed keys before mutating runtime config
    found_removed = _REMOVED_KEYS & set(new_cfg.keys())
    if found_removed:
        raise ConfigReloadValidationError(
            f"Reload rejected: config contains removed keys: {sorted(found_removed)}"
        )
    # ... rest of existing logic
```

### Method

- Use comment markers (`# REMOVED:`) to indicate removed lines for traceability
- Define `_REMOVED_KEYS` as module-level `frozenset` for immutability and performance
- Use set intersection (`&`) to find overlap between removed keys and incoming config keys efficiently

### Details

**Type safety considerations:**
- `_REMOVED_KEYS` uses `frozenset` for immutability — prevents accidental mutation
- Set intersection operation returns another `frozenset` — safe to check truthiness

**Import considerations:**
- `ConfigReloadValidationError` must already be imported; verify import exists
- No new imports required beyond what's already present

**Cross-file consistency:**
- `_REMOVED_KEYS` in this file must match the set used in `config_builders.py` `_FORBIDDEN_KEYS`
- Consider consolidating into a shared constant to avoid duplication

**Runtime behavior change:**
- Failed reload now raises an exception instead of silently ignoring removed keys
- This is intentional — prevents partial config mutations
