# Implementation: agent/tool_audit.py — Replace dict-based audit with DTO-based

## Goal

Rewrite `scripts/agent/tool_audit.py` so that the three audit functions (`audit_approval`, `log_approval_decision`, `audit_tool_exec`) construct typed frozen dataclasses (`ToolApprovalEvent`, `ApprovalDecisionEvent`, `ToolExecEvent`) before serializing, instead of building inline dicts.

This enforces that all required fields are present at construction time and eliminates `dict[str, Any]` at the serialization boundary.

## Scope

**In:** `scripts/agent/tool_audit.py`
**Out:** `agent/shared/models.py` (new DTOs) and `agent/tool_models.py` (`ApprovalOutcome`) are used but not modified here. Caller signatures are unchanged.

## Assumptions

1. The three new DTOs (`ToolApprovalEvent`, `ApprovalDecisionEvent`, `ToolExecEvent`) have been added to `agent/shared/models.py` in the previous step.
2. `dataclasses.asdict()` produces a plain dict suitable for `orjson.dumps()`.
3. `resource_scope` values extracted by `_extract_resource_scope` are always strings (path/branch keys); typing it as `dict[str, str]` is valid.
4. `risk` and `decision` are passed as `RiskLevel | str` and `ApprovalDecisionType | str` respectively; `str(risk)` / `str(decision)` coercion is explicitly allowed here because these are `StrEnum` subclasses — `.value` is equivalent to `str()` for StrEnum.
5. Caller signatures (`audit_approval`, `log_approval_decision`, `audit_tool_exec`) do not change; no callers need updating.

## Implementation

### Target file

`scripts/agent/tool_audit.py`

### Procedure

1. Add import: `import dataclasses` at the top.
2. Add import: `from agent.shared.models import ApprovalDecisionEvent, ToolApprovalEvent, ToolExecEvent`.
3. Rewrite `audit_approval`:
   - Construct `ToolApprovalEvent(event="tool_approval", ...)` from existing locals.
   - Serialize with `orjson.dumps(dataclasses.asdict(event)).decode()`.
4. Rewrite `log_approval_decision`:
   - Construct `ApprovalDecisionEvent(event="approval_decision", ...)` from `outcome` fields.
   - Serialize with `orjson.dumps(dataclasses.asdict(event)).decode()`.
5. Rewrite `audit_tool_exec`:
   - Construct `ToolExecEvent(event="tool_exec", ...)` from existing locals.
   - Serialize with `orjson.dumps(dataclasses.asdict(event)).decode()`.
6. Remove the now-unused inline dict literals from all three functions.

### Method

Targeted rewrite of function bodies; imports updated; no interface changes.

**Note on `resource_scope` typing:**
`_extract_resource_scope` returns `dict` (untyped). After this change, it must return `dict[str, str]` to satisfy `ToolApprovalEvent.resource_scope: dict[str, str]`. Update the return annotation in `_extract_resource_scope` as part of this step.

```python
def _extract_resource_scope(ctx: AgentContext, masked: dict) -> dict[str, str]:
    path_keys = set(ctx.cfg.approval.approval_resource_keys.get("path_keys", []))
    branch_keys = set(ctx.cfg.approval.approval_resource_keys.get("branch_keys", []))
    return {k: str(v) for k, v in masked.items() if k in path_keys | branch_keys}
```

Adding `str(v)` coercion is safe because resource_scope values are always file paths or branch names.
