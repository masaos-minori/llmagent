# Implementation: db/schema_sql.py — Workflow schema extensions for stage-level persistence

## Goal

Add `error_kind` and `error_detail` columns to `attempts`, add `workflow_id` and `attempt_number` to `artifacts`, and add `workflow_id` to `processed_events`. All changes are additive.

## Scope

**In**: Schema additions for three workflow tables. Idempotent migration pattern.

**Out**: Changes to ops files, workflow_engine.py, tests.

## Assumptions

1. SQLite `ALTER TABLE ... ADD COLUMN` is the standard migration approach; it is idempotent if wrapped in `try/except OperationalError`.
2. The schema constants are in `_WORKFLOW_SCHEMA` string in `schema_sql.py`.
3. New columns have `NULL` default (not `NOT NULL`) for backward compatibility with existing rows.
4. `attempts.error_msg` remains for backward compatibility; `error_kind` and `error_detail` are new columns.

## Implementation

### Target file
`scripts/db/schema_sql.py`

### Procedure
1. Add new column definitions to `CREATE TABLE IF NOT EXISTS attempts`.
2. Add new column definitions to `CREATE TABLE IF NOT EXISTS artifacts`.
3. Add new column definition to `CREATE TABLE IF NOT EXISTS processed_events`.
4. Add migration SQL to handle existing databases.

### Method

**Updated `attempts` table:**
```sql
CREATE TABLE IF NOT EXISTS attempts (
    attempt_id   TEXT PRIMARY KEY,
    task_id      TEXT NOT NULL REFERENCES tasks(task_id) ON DELETE CASCADE,
    stage_id     TEXT NOT NULL,
    status       TEXT NOT NULL DEFAULT 'running',
    started_at   TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    ended_at     TEXT,
    error_msg    TEXT,
    error_kind   TEXT,    -- NEW: "timeout"|"halt"|"exception"|null
    error_detail TEXT     -- NEW: full detail string; may overlap with error_msg
);
```

**Updated `artifacts` table:**
```sql
CREATE TABLE IF NOT EXISTS artifacts (
    artifact_id    TEXT PRIMARY KEY,
    task_id        TEXT NOT NULL REFERENCES tasks(task_id) ON DELETE CASCADE,
    stage_id       TEXT NOT NULL,
    uri            TEXT NOT NULL,
    created_at     TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    workflow_id    TEXT,    -- NEW: denormalized for easy querying
    attempt_number INTEGER  -- NEW: which attempt produced this artifact
);
```

**Updated `processed_events` table:**
```sql
CREATE TABLE IF NOT EXISTS processed_events (
    event_id    TEXT PRIMARY KEY,
    task_id     TEXT NOT NULL REFERENCES tasks(task_id) ON DELETE CASCADE,
    stage_id    TEXT NOT NULL,
    recorded_at TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    workflow_id TEXT  -- NEW: denormalized for easy querying
);
```

**Migration SQL (for existing databases that already have the tables):**
Add to a `_WORKFLOW_MIGRATIONS` constant or a separate migration function:
```python
_WORKFLOW_MIGRATIONS: list[str] = [
    "ALTER TABLE attempts ADD COLUMN error_kind TEXT",
    "ALTER TABLE attempts ADD COLUMN error_detail TEXT",
    "ALTER TABLE artifacts ADD COLUMN workflow_id TEXT",
    "ALTER TABLE artifacts ADD COLUMN attempt_number INTEGER",
    "ALTER TABLE processed_events ADD COLUMN workflow_id TEXT",
]
```

Apply migrations in `SQLiteHelper.ensure_schema()` or `db/helper.py` init with:
```python
for stmt in _WORKFLOW_MIGRATIONS:
    try:
        db.execute(stmt)
    except sqlite3.OperationalError:
        pass  # column already exists
```

### Details

- The `CREATE TABLE IF NOT EXISTS` already handles fresh databases correctly.
- The migration statements handle existing databases.
- `error_kind` values: `"timeout"`, `"halt"`, `"exception"` — defined as string constants in `workflow_engine.py`.
- `attempt_number` is the retry count (1-based) from `_run_execute_with_retry()`.
