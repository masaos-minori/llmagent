# Implementation: scripts/agent/repl.py — Catch RuntimeError from session.start() and call write_fatal

**Plan source:** `plans/20260702-202919_plan.md` (Phase 2 (repl))
**Target file:** `scripts/agent/repl.py`

---

## Goal

repl.py の _run_repl_loop() 内で session.start() が送出する RuntimeError をキャッチし、write_fatal で致命的エラーを表示してから終了する。

---

## Scope

**In:**
- _run_repl_loop() 内での RuntimeError キャッチ (汎用ハンドラより先に配置)
- キャッチ時に `self._view.write_fatal(str(e))` を呼び出す
- RuntimeError を re-raise または graceful exit (エージェント終了)

**Out:**
- session.py の変更 (Phase 2 (session) で対応済み)
- write_fatal メソッドの実装 (Phase 1 で対応済み)
- session.start() 以外の RuntimeError ハンドリング変更

---

## Assumptions

1. _run_repl_loop() 内に session.start() の呼び出しが存在する
2. 既存の汎用 except ハンドラ (Exception など) が存在し、RuntimeError はその前に配置する必要がある
3. `self._view` が CLIView インスタンスで write_fatal メソッドを持つ (Phase 1 完了後)
4. re-raise した場合、呼び出し元がプロセス終了を担う

---

## Implementation

### Target file

`scripts/agent/repl.py`

### Procedure

1. repl.py を読み込み、_run_repl_loop() の全体構造と既存の except ハンドラを確認する
2. session.start() 呼び出しを含む try ブロックを特定する
3. 既存の汎用 except より手前に `except RuntimeError as e:` ブロックを追加する
4. ブロック内で `self._view.write_fatal(str(e))` を呼び出す
5. その後 `raise` で re-raise するか、graceful exit (return/sys.exit) を実装する (既存の終了パターンに合わせる)
6. lint/mypy を確認する

### Method

Edit tool でコード変更

### Details

変更後のパターン (汎用ハンドラの前に挿入):
```python
except RuntimeError as e:
    self._view.write_fatal(str(e))
    raise
```

graceful exit が望ましい場合:
```python
except RuntimeError as e:
    self._view.write_fatal(str(e))
    return  # またはループを抜ける
```

既存コードの終了パターン (sys.exit vs return vs raise) を確認してから決定する。
