# Goal

Surface the shell-mcp sandbox backend in `/mcp status` output by:
1. Adding `sandbox_backend: str = ""` field to `McpProbeResult`
2. Fetching the `/health` body in `McpStatusService._get_http_status` and extracting `sandbox_backend`
3. Appending sandbox info to the role column in `_format_mcp_table` when non-empty

# Scope

- **Files:**
  - `scripts/agent/services/models.py` — add `sandbox_backend` field to `McpProbeResult`
  - `scripts/agent/services/mcp_status.py` — fetch health body, populate sandbox_backend
  - `scripts/agent/commands/cmd_mcp.py` — show sandbox_backend in role column
- **Out of scope:** `McpAvailability` enum, other probe result fields

# Assumptions

1. `McpProbeResult` is a `frozen=True` dataclass — new field needs a default value.
2. Only HTTP servers can return sandbox_backend via `/health` body parsing.
3. Non-shell servers will have `sandbox_backend = ""` (empty) — not shown in table.
4. Appending to role column avoids changing table column count (no layout breakage).

# Implementation

## Target files

### models.py — Add field

```python
@dataclass(frozen=True)
class McpProbeResult:
    key: str
    transport: str
    startup_mode: str
    auth: bool
    tier: McpTier
    role: str
    availability: McpAvailability
    health: str
    endpoint: str
    sandbox_backend: str = ""  # new — populated from /health body when available
```

### mcp_status.py — Fetch health body

Refactor `_get_http_status` to return `(McpAvailability, str)` where str is sandbox_backend:
```python
async def _get_http_status(
    self, probe: httpx.AsyncClient, url: str
) -> tuple[McpAvailability, str]:
    if not url:
        return McpAvailability.NO_URL, ""
    try:
        r = await probe.get(f"{url}/health")
        if r.status_code == HTTPStatus.OK:
            body = r.json() if r.headers.get("content-type", "").startswith("application/json") else {}
            sandbox = body.get("sandbox_backend", "") if isinstance(body, dict) else ""
            return McpAvailability.OK, str(sandbox) if sandbox else ""
        return McpAvailability.HTTP_ERROR, ""
    except (httpx.RequestError, httpx.HTTPStatusError):
        return McpAvailability.FAIL, ""
```

Update `_resolve_endpoint` to pass sandbox_backend through, and update `_probe_single_server` to set `sandbox_backend`.

### cmd_mcp.py — Show in role column

In `_format_mcp_table`, build role display:
```python
role_display = r.role
if r.sandbox_backend:
    role_display = f"{r.role} [sb:{r.sandbox_backend}]".strip()
```

Use `role_display` instead of `r.role` in the format call.

## Details

- `r.json()` may raise for non-JSON responses — guard with try/except.
- The `[sb:none]` annotation in the role column makes sandbox state visible without a new column.
- Only shell-mcp returns `sandbox_backend` in `/health`, so other servers show `""` and the role column is unchanged.
