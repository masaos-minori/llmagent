# Implementation: M-7 — Race _read_input() against _shutdown_event (repl.py)

## Goal

Update `AgentREPL._read_input()` to race the blocking `input()` executor task against
`_shutdown_event.wait()` so that SIGTERM received while waiting for user input causes
`_read_input()` to return `None` promptly without waiting for the next keystroke.

## Scope

**Target**: `scripts/agent/repl.py`

**Step covered**: Plan M-7 step 1.

**Out of scope**: SIGTERM handler setup, `_repl_loop()`, `startup.py`, `factory.py`.

## Assumptions

1. `_shutdown_event` is an `asyncio.Event` set on SIGTERM (confirmed: lines 456, 461).
2. `input()` in a `run_in_executor` thread cannot be cancelled; the thread lives until
   the user types. This is acceptable — the REPL loop exits, and the orphaned thread
   terminates when the process exits.
3. `asyncio.wait(FIRST_COMPLETED)` is the correct pattern for this race.
4. `_shutdown_event` may be `None` before `run()` initializes it; guard with
   `if self._shutdown_event is not None`.

## Implementation

### Target file

`scripts/agent/repl.py`

### Procedure

#### Step 1: Locate `_read_input()` current implementation

Find the method; it currently does something like:
```python
async def _read_input(self, loop: asyncio.AbstractEventLoop) -> str | None:
    try:
        raw = await loop.run_in_executor(None, lambda: input(self._prompt))
    except EOFError:
        return None
    # ... multiline handling
```

#### Step 2: Replace with race implementation

```python
async def _read_input(self, loop: asyncio.AbstractEventLoop) -> str | None:
    input_future = loop.run_in_executor(None, lambda: input(self._prompt))
    if self._shutdown_event is not None:
        input_task = asyncio.ensure_future(input_future)
        shutdown_task = asyncio.ensure_future(self._shutdown_event.wait())
        try:
            done, pending = await asyncio.wait(
                {input_task, shutdown_task},
                return_when=asyncio.FIRST_COMPLETED,
            )
        except Exception:
            for t in (input_task, shutdown_task):
                t.cancel()
            raise
        # Cancel the losing task
        for t in pending:
            t.cancel()
        if shutdown_task in done:
            return None
        try:
            raw = input_task.result()
        except EOFError:
            return None
        except KeyboardInterrupt:
            return None
    else:
        try:
            raw = await input_future
        except EOFError:
            return None
        except KeyboardInterrupt:
            return None
    # ... existing multiline handling continues here
```

### Method

- Wraps existing `run_in_executor` with a race using `asyncio.wait(FIRST_COMPLETED)`.
- The `else` branch (when `_shutdown_event is None`) preserves existing behavior.
- `asyncio.ensure_future()` wraps both the `Future` and the coroutine for `asyncio.wait`.

### Details

- `input_future` from `run_in_executor` is a `concurrent.futures.Future` wrapped by
  asyncio into an `asyncio.Future`. `asyncio.ensure_future()` accepts this directly.
- `shutdown_task.cancel()` is safe when `_shutdown_event.wait()` is already done; the
  cancel is a no-op for completed tasks.
- `input_task.cancel()` does NOT interrupt the executor thread's `input()` call; the
  thread continues blocking until user types. Cancellation only affects the asyncio
  future wrapper.
- Preserve all existing `EOFError` / `KeyboardInterrupt` handling after the race.
- Multiline handling (if `_read_input` supports multi-line input via continuation) must
  remain intact after the `raw = input_task.result()` line.
