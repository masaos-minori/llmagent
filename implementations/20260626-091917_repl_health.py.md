# Implementation: Convert repl_health.py warnings to hard startup failures

Steps covered: Plan 20260626-091917 — Steps 1-1, 1-2, 2-1, 2-2, 3-1

---

## Goal

Identify all `HealthCheckResult(status="warning")` returns in `repl_health.py` that should be hard failures, and convert them to `status="error"` with startup abort. Enforce a "no silent warnings in production" policy.

---

## Scope

- **In scope**: `scripts/agent/repl_health.py` — all health checks, startup failure policy
- **Out of scope**: docs (covered in a separate doc step)

---

## Assumptions

- `repl_health.py` has a `run_health_checks()` function that collects all check results.
- The startup sequence respects `status="error"` as a hard failure (raises or exits).
- Checks that were "warning" but should be "error":
  - Missing required API keys
  - Config validation failures
  - DB connection failures
  - (New in plan 09) firejail missing when required
- Checks that should remain "warning":
  - Optional performance settings not configured
  - `workflow_allowlist` empty (fail-closed is OK, not an error)

---

## Implementation

### Target file
`scripts/agent/repl_health.py`

### Procedure
1. Read `scripts/agent/repl_health.py` fully — list all checks and their current severity.
2. Step 1-1 (Audit): For each `status="warning"` check, determine if it should be `"error"`.
3. Step 1-2 (Categorize): Create two lists:
   - Must-be-error: anything that makes the agent non-functional (missing key, broken DB, etc.)
   - May-remain-warning: optional/advisory settings.
4. Step 2-1 (Convert): Change identified warnings to errors:
   ```python
   # Before:
   return HealthCheckResult(name="missing_api_key", status="warning", message="...")
   # After:
   return HealthCheckResult(name="missing_api_key", status="error", message="...")
   ```
5. Step 2-2 (Startup abort): In `run_health_checks()`, ensure the startup sequence raises or exits when any check returns `status="error"`:
   ```python
   errors = [r for r in results if r.status == "error"]
   if errors:
       for e in errors:
           logger.error("startup check FAILED: %s — %s", e.name, e.message)
       raise StartupHealthCheckError(f"{len(errors)} startup check(s) failed")
   ```
6. Step 3-1 (Remaining warnings): Add a `HealthCheckResult(status="warning")` summary line at startup for any remaining warnings.

### Method
Severity escalation. Primary risk: previously-deployed environments with misconfigured settings will now fail to start. This is the intended behavior (fail-fast).
