# Implementation: scripts/agent/cli_view.py — Add write_fatal to CLIViewProtocol and CLIView

**Plan source:** `plans/20260702-202919_plan.md` (Phase 1)
**Target file:** `scripts/agent/cli_view.py`

---

## Goal

CLIViewProtocol および CLIView クラスに write_fatal メソッドを追加し、致命的エラーを "[fatal] {msg}" 形式で出力できるようにする。

---

## Scope

**In:**
- CLIViewProtocol (Protocol stub) に `write_fatal(self, msg: str) -> None` のシグネチャを追加
- CLIView クラスに `write_fatal(self, msg: str) -> None` の実装を追加 (出力: `[fatal] {msg}`)

**Out:**
- 既存の `write_warning` メソッドへの変更
- 呼び出し側 (repl.py 等) の変更 (Phase 2 以降で対応)

---

## Assumptions

1. CLIViewProtocol は cli_view.py 冒頭付近に定義された Protocol クラスである
2. write_warning の実装パターン (print または stderr 出力) に倣って write_fatal を実装する
3. mypy がプロトコル適合チェックを行うため、Protocol とクラス双方に追加が必要

---

## Implementation

### Target file

`scripts/agent/cli_view.py`

### Procedure

1. cli_view.py を読み込み、CLIViewProtocol の定義箇所と CLIView クラスの write_warning メソッドを確認する
2. CLIViewProtocol に `write_fatal(self, msg: str) -> None` シグネチャを追加する (write_warning の直後が望ましい)
3. CLIView クラスに `write_fatal(self, msg: str) -> None` を実装する。本文: `print(f"[fatal] {msg}")`
4. ruff check で lint エラーがないことを確認する
5. mypy で型エラーがないことを確認する

### Method

Edit tool でコード変更 (2 箇所の Edit)

### Details

追加するシグネチャ (Protocol):
```python
def write_fatal(self, msg: str) -> None: ...
```

追加する実装 (CLIView):
```python
def write_fatal(self, msg: str) -> None:
    print(f"[fatal] {msg}")
```

write_warning の出力先 (stdout/stderr) を確認し、write_fatal も同じストリームに出力する。
