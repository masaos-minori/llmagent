# Implementation: Remove unsafe direct-execution fallback from orchestrator.py

Steps covered: Plan 20260626-091910 — Steps 1-1, 2-1, 2-2, 2-3

---

## Goal

Audit and remove the direct-execution fallback path in `orchestrator.py` that bypasses approval checks. When a workflow cannot be created (e.g., missing plan), raise a clear error rather than falling back to unapproved direct execution.

---

## Scope

- **In scope**: `scripts/agent/orchestrator.py` — locate fallback path, add audit log + CLI notification before removal, then remove/replace with error raise
- **Out of scope**: factory.py (step 1-2); docs (step 3-1)

---

## Assumptions

- A fallback path exists in `orchestrator.py` that calls `_log_fallback()` or similar before executing directly.
- `audit_logger` is accessible from the orchestrator.
- A `WorkflowCreationError` or similar exception is appropriate when no workflow plan is available.
- CLI notification means printing a user-visible message before raising.

---

## Implementation

### Target file
`scripts/agent/orchestrator.py`

### Procedure
1. Read `scripts/agent/orchestrator.py` — find the fallback execution path (search for `fallback`, `direct`, or the direct tool call site).
2. Step 1-1 (Audit): Confirm the fallback path's call sites and what triggers it.
3. Step 2-1 (Add audit log): Before the fallback executes, add:
   ```python
   audit_logger.warning("direct-execution fallback triggered — bypassing workflow approval checks", task=task_id)
   ```
4. Step 2-2 (Add CLI notification): Print to user:
   ```python
   ctx.output.print("[WARNING] Direct execution fallback activated. Approval checks may be bypassed.")
   ```
5. Step 2-3 (Remove fallback or raise): Replace the fallback body with:
   ```python
   raise WorkflowCreationError(
       f"Cannot create workflow for task {task_id!r}. Direct execution fallback is disabled."
   )
   ```
   If a `WorkflowCreationError` does not exist, define it in `scripts/agent/errors.py`.

### Method
Behavioral change: fallback path becomes an error path. Requires `WorkflowCreationError` to exist or be added.

### Details

The goal is fail-closed: if the orchestrator cannot build a workflow, it must not silently execute directly. The audit log and CLI notification serve as forensic markers for the transition period. The final state is: fallback path is gone, replaced by a clear error.
