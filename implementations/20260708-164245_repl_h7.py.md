# Implementation: H-7 — Remove ctx.tool_result_store guard from _persist_session_diagnostics()

## Goal

Remove the `tool_results = ctx.tool_result_store` read and its `is not None` guard clause from
`_persist_session_diagnostics()`. The function's actual tool-result count query already goes
directly through `SQLiteHelper("session")`, not through `ctx.tool_result_store` — the field was
only ever used as a `None`-check guard here (per the plan's U-4 analysis).

## Scope

**Target**: `scripts/agent/repl.py` — the `_persist_session_diagnostics()` method
(distinct from the H-4 change to this same file, which touches the partial-completion warning
message in a different method; see
`implementations/20260708-163453_repl.py.md` for that unrelated change).

**Depends on**: land together with `implementations/20260708-163935_context.py.md` (field
removal).

**Out of scope**: the `SQLiteHelper("session").open(...)` query itself (lines 176-187), the
`tool_result_summary` dict construction, and everything else in
`_persist_session_diagnostics()` outside the guard condition.

## Assumptions

1. `tool_results = ctx.tool_result_store` (line 161) is used ONLY as the `is not None` half of
   the guard at line 174 — it has no other reader in this function (confirmed by the plan's U-4
   analysis: "L.174: `if session_id is not None and tool_results is not None:` — ガード... ガード
   条件を `if session_id is not None:` に変更するだけで削除できる").
2. `session_id is None` alone is a sufficient guard to skip the query safely — this was already
   true before the change (a session with no `session_id` cannot have a meaningful
   `tool_results` row count query regardless of whether `ctx.tool_result_store` exists).

## Implementation

### Target file

`scripts/agent/repl.py`

### Procedure

#### Step 1: Confirm the two reference points

```bash
grep -n "tool_result_store\|tool_results = " scripts/agent/repl.py
```

Expected (pre-change): line 161 (`tool_results = ctx.tool_result_store`) and line 174 (the guard
using `tool_results is not None`). The `tool_result_summary` dict variable (line 173) and the
raw SQL string `"...FROM tool_results WHERE..."` (line 179) are unrelated name collisions on the
word "tool_result(s)" — do not touch those.

#### Step 2: Remove the field read

Remove line 161:

```python
            tool_results = ctx.tool_result_store
```

#### Step 3: Simplify the guard condition

Current (line 174):

```python
            if session_id is not None and tool_results is not None:
```

Replace with:

```python
            if session_id is not None:
```

### Method

- Two-line deletion/simplification; no change to the query itself, its error handling
  (`except sqlite3.Error: pass`), or the surrounding `try/except (RuntimeError, sqlite3.Error,
  OSError)` at the method's outer scope.

### Details

- `tool_result_summary` (the dict populated by the query, used later in the method to build the
  final diagnostics payload) is unaffected — it is a different variable from `tool_results`
  (the now-removed field read) despite the similar name.
- The raw SQL string literal `"SELECT COUNT(*) as cnt, SUM(is_error) as errs FROM tool_results
  WHERE session_id = ?"` queries the `tool_results` TABLE directly via `SQLiteHelper` — this is
  independent of `AgentContext.tool_result_store` and stays exactly as-is. The table itself is
  not removed by H-7 (per the plan's "変更しないファイル" / "変更しないもの" — `tool_results`
  table schema is untouched).
