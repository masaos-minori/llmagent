# Goal

Delete the legacy `scripts/agent.py` entry-point file and update the OpenRC init
script so the service uses `python -m agent` instead.

# Scope

- `scripts/agent.py` — delete
- `init.d/llama-agent` — update `command_args` from `agent.py` to `-m agent`
- `deploy/deploy.sh` — confirm no copy of `agent.py` (no change expected)

# Assumptions

1. `init.d/llama-agent` line 8: `command_args="agent.py"` is the only production
   reference to `scripts/agent.py`.
2. `python -m agent` (run from `scripts/`) is the correct replacement; `agent/__main__.py`
   or equivalent already exists and starts the REPL.
3. No other file in `deploy/`, `conf.d/`, or `scripts/` references `agent.py` by name
   (verified by grep before deletion).

# Implementation

## Target file

- `scripts/agent.py` (deleted)
- `init.d/llama-agent` (modified)

## Procedure

1. Confirm no remaining references:
   ```bash
   grep -rn "agent\.py" deploy/ init.d/ conf.d/ scripts/ --include="*.sh" --include="*.py" --include="*.conf"
   ```
2. Delete `scripts/agent.py`.
3. In `init.d/llama-agent` line 8, change:
   ```
   command_args="agent.py"
   ```
   to:
   ```
   command_args="-m agent"
   ```
   (Ensure `command` is `python` or the venv python path, not `agent.py` directly.)
4. Verify `scripts/agent/__main__.py` exists and starts the REPL correctly.

## Method

File deletion + one-line init script change. No Python code changes.

## Details

After deletion, `ls scripts/agent.py` must return non-zero. The comment in
`init.d/llama-agent` line 3 ("agent.py は FastAPI アプリではなく対話型 REPL のため")
can remain or be updated to reflect the new invocation style.

# Validation plan

- `ls scripts/agent.py` → file not found (exit code 1)
- `grep -rn "agent\.py" deploy/ init.d/ conf.d/` → 0 hits (comments OK)
- `uv run ruff check scripts/` — no change expected
- `uv run mypy scripts/` — no change expected
- Service smoke-test (if available): `init.d/llama-agent start` with new `command_args`
