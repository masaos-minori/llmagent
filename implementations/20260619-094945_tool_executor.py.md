# Implementation: Detect and Warn on Repeated Tool Failures

## Goal

Log a warning when the same MCP server accumulates tool-level errors (`is_error=True`) beyond a configurable threshold, making repeated failures operator-visible without requiring log file scanning.

## Scope

- `scripts/shared/tool_executor.py`
  - After incrementing `stat_tool_errors` (Step 2), check against a warning threshold
  - Log `logger.warning(...)` when the per-server count crosses the threshold
  - Threshold: configurable, default 3

Out of scope:
- Time-windowed counters (plan says "same tool failing 3+ times in N minutes" but implementing a sliding window requires a time-tracked list; use a simple cumulative counter with a configurable threshold for the initial version)
- Alerting to external systems
- Resetting counters (counters accumulate for the process lifetime; reset on restart)

## Assumptions

1. `stat_tool_errors: dict[str, int]` is added by Step 2; this step extends the increment logic.
2. The threshold should be configurable via a constructor parameter with a default of 3.
3. Warning should log once per threshold crossing (at 3, 6, 9, ... or just at 3 for simplicity).
4. Simple approach: warn every time the count is a multiple of the threshold (e.g., warn at 3, 6, 9); avoids a one-time flag per server.
5. `mcp/server.py` does not need changes; the detection lives in `tool_executor.py` which has per-server visibility.

## Implementation

### Target file

`scripts/shared/tool_executor.py`

### Procedure

1. Add `repeated_tool_error_threshold: int = 3` parameter to `ToolExecutor.__init__`.
2. Store as `self._tool_error_threshold = repeated_tool_error_threshold`.
3. After incrementing `stat_tool_errors[server_key]`, check if count % threshold == 0.

### Method

Single conditional block after the counter increment. No new data structures.

### Details

**`__init__` parameter addition:**
```python
def __init__(
    self,
    ...
    repeated_tool_error_threshold: int = 3,
) -> None:
    ...
    self._tool_error_threshold = repeated_tool_error_threshold
```

**`_raw_execute()` — extend the is_error block (Step 2 increment + Step 3 threshold check):**
```python
if result.is_error:
    count = self.stat_tool_errors.get(server_key, 0) + 1
    self.stat_tool_errors[server_key] = count
    logger.debug(
        "tool error from %r: %s (error_type=tool)",
        server_key,
        result.output[:120],
    )
    if count % self._tool_error_threshold == 0:
        logger.warning(
            "repeated tool errors from %r: %d failures (error_type=tool)",
            server_key,
            count,
        )
```

This warns at count = 3, 6, 9, ... — each multiple of the threshold triggers a new warning.

**Callers that want to reset counters** (e.g., after a server restart) can call:
```python
executor.stat_tool_errors.pop(server_key, None)
```
No special reset method is needed.
