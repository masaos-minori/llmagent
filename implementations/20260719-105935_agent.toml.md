## Goal

Set `config/agent.toml`'s `use_memory_layer`, `memory_embed_enabled`, and `memory_local_only` values so
the committed config file's text is consistent with the new enabled-by-default memory layer, and so the
`memory_local_only`/`embed_url` conflict (Option B, user-confirmed) is resolved by disabling
local-only mode rather than by broadening `_LOCAL_PREFIXES`.

## Scope

**In scope**
- `use_memory_layer = false` ŌåÆ `true` (line 82).
- `memory_embed_enabled = false` ŌåÆ `true` (line 87).
- `memory_local_only = true` ŌåÆ `false` (line 90).

**Out of scope**
- `embed_url` (line 10) ŌĆö untouched; already a non-empty, non-local-prefix address
  (`http://192.168.11.238:8081`), which is exactly why `memory_local_only` must become `false` (Option B)
  rather than staying `true`.
- `memory_max_content_chars = 500` (line 92) ŌĆö untouched; this plan reinterprets its meaning
  (documentation-only change, per the paired `extract.py` doc) but does not change its numeric value.
- `scripts/agent/memory/embedding_client.py`'s `_LOCAL_PREFIXES` ŌĆö untouched (Option B chosen, not
  Option A).
- All other `config/agent.toml` sections/keys ŌĆö untouched.

## Assumptions

1. Confirmed by direct read (`grep -n "embed_url\|use_memory_layer\|memory_embed_enabled\|memory_local_only\|memory_max_content_chars" config/agent.toml`):
   `embed_url = "http://192.168.11.238:8081"` at line 10; `use_memory_layer = false` at line 82;
   `memory_embed_enabled = false` at line 87; `memory_local_only = true` at line 90;
   `memory_max_content_chars = 500` at line 92. Matches the plan's own Assumption 6 exactly ŌĆö no drift.
2. **Significant finding, carried over from the paired `config_dataclasses.py` implementation doc's
   Assumption 3 ŌĆö repeated here because it directly affects what this file's edit actually accomplishes**:
   `shared/config_loader.py`'s `_REMOVED_KEYS` (lines 31-39) includes `use_memory_layer` and
   `memory_embed_enabled`; `ConfigLoader.load_all()` strips both from the loaded dict before
   `agent/config_builders.py::_build_memory_config()` ever sees it, and that builder function has the
   corresponding constructor arguments commented out (`config_builders.py:197,202`, `# REMOVED:`
   prefix) ŌĆö it never reads these two keys from any config dict, regardless of value. **Practical
   consequence: editing this file's `use_memory_layer`/`memory_embed_enabled` lines has no runtime
   effect today** ŌĆö the actual default is controlled solely by `scripts/agent/config_dataclasses.py`'s
   `MemoryConfig` field defaults (see the paired doc, which is what makes the plan's Goal true). This
   edit is still worth making for source-of-truth/documentation consistency (a reader of
   `config/agent.toml` should see values matching the intended production behavior, and
   `tools/check_agent_docs_consistency.py` or a future contributor removing the dead
   `config_loader.py`/`config_builders.py` wiring would otherwise re-inherit stale `false` values) ŌĆö but
   it must not be mistaken for the mechanism that achieves the Goal.
3. `memory_local_only` is **not** on `_REMOVED_KEYS`/`_FORBIDDEN_KEYS` and **is** read live by
   `config_builders.py:211` (`memory_local_only=bool(cfg.get("memory_local_only", False))`) ŌĆö this
   file's `memory_local_only = false` edit is genuinely functional (confirmed by direct read of both
   `config_loader.py` and `config_builders.py`, not by executing `_filter_removed_keys` itself, but by
   its absence from the frozenset literal).
4. `scripts/agent/memory/embedding_client.py`'s `_LOCAL_PREFIXES`-based check (around line 133-137:
   `if config.local_only and config.embed_url: if not any(config.embed_url.startswith(p) for p in
   _LOCAL_PREFIXES): raise ValueError("memory_local_only=True but embed_url is not a local address: ...")`)
   is the exact `ValueError` the plan's Goal names. With `memory_local_only` set to `false` here, the
   `if config.local_only and ...` guard short-circuits and the `ValueError` cannot be raised ŌĆö this is
   the mechanism satisfying "starts without a `ValueError`". `EmbeddingClientConfig.local_only` is wired
   from `ctx.cfg.memory.memory_local_only` at `factory.py:342`, confirmed by direct read.

## Implementation

### Target file

`config/agent.toml`

### Procedure

1. Re-run `grep -n "use_memory_layer\|memory_embed_enabled\|memory_local_only" config/agent.toml`
   immediately before editing to reconfirm lines 82/87/90 have not shifted.
2. Change line 82 from `use_memory_layer = false` to `use_memory_layer = true`.
3. Change line 87 from `memory_embed_enabled = false` to `memory_embed_enabled = true`.
4. Change line 90 from `memory_local_only = true` to `memory_local_only = false`.
5. Leave the `# ŌöĆŌöĆ Memory ŌöĆŌöĆ...` section header and all surrounding comments/keys unchanged.

### Method

Three-line TOML value edit. No structural change to the file (no new keys, no new sections, no
key removal/reordering).

convention ŌĆö confirmed by reading the surrounding lines, e.g. `otel_enabled = false` at line 97 uses the
same lowercase form).
