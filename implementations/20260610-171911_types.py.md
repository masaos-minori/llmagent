# Implementation: rag/types.py — Stage-Separated Hit Types

## Goal

Replace the single loosely-typed `RagHit` TypedDict with stage-specific types (`RawHit`, `MergedHit`, `RankedHit`) and add pipeline-level types (`RagQuery`, `PipelineStageResult`). Retain `RagHit` as a Union alias for backward compatibility with existing callers.

---

## Scope

**Target file:** `scripts/rag/types.py`
**Also update:** `scripts/rag/stage.py` (typed `PipelineContext` fields and observer callback)

**In:**
- Define `RawHit`, `MergedHit`, `RankedHit` as `TypedDict` with `total=True` (required fields only)
- Define `RagHit = RawHit | MergedHit | RankedHit` as Union alias
- Define `RagQuery` dataclass (query string + optional metadata)
- Define `PipelineStageResult` dataclass (stage name, success flag, failure_reason, elapsed_s)
- Update `PipelineContext` in `stage.py`: replace `list[Any]` fields with typed lists; replace untyped `observers` with `list[Callable[[str, PipelineStageResult], None]]`

**Out:**
- No changes to `repository.py`, `llm.py`, `pipeline.py`, or `stages/` in this step
- No `__all__` changes to existing external exports except additions

---

## Assumptions

1. `RawHit` requires `chunk_id: int`, `content: str`, `url: str`, `title: str`. Distance/BM25 fields are optional (`total=False` or separated subclass) since vector hits have `distance` and FTS hits have `bm25_score`.
2. `MergedHit` extends `RawHit` fields plus `rrf_score: float`.
3. `RankedHit` extends `MergedHit` fields plus `rerank_score: float`.
4. Callers that currently do `hit["rrf_score"]` will get a type error if they receive `RawHit`; this is intentional (forces correct stage sequencing).
5. `PipelineStageResult` is a dataclass (not TypedDict) so it can carry optional fields without `total=False` ambiguity.
6. `observers` list in `PipelineContext` is currently always empty in production — typed change is safe.

## Implementation

### Target file
`scripts/rag/types.py`

### Procedure

1. Keep `LLMMessage` re-export unchanged.
2. Define `RawHit` as `TypedDict` with `total=True` for required fields; add `distance` and `bm25_score` as optional via inheritance trick or `total=False` subclass.
3. Define `MergedHit(RawHit, total=False)` adding `rrf_score: float`.
4. Define `RankedHit(MergedHit, total=False)` adding `rerank_score: float`.
5. Define `RagHit = RawHit | MergedHit | RankedHit` — Union alias for backward compat.
6. Define `@dataclasses.dataclass` `PipelineStageResult` with fields: `stage: str`, `success: bool`, `failure_reason: str | None = None`, `elapsed_s: float = 0.0`.
7. Update `__all__` to export all new names.

### Method

TypedDict inheritance for stage separation:

```python
from __future__ import annotations
import dataclasses
from typing import TypedDict
from shared.types import LLMMessage

class RawHit(TypedDict, total=False):
    chunk_id: int      # required; mark via Required[] if Python 3.11+
    content: str
    url: str
    title: str
    distance: float    # vector search only
    bm25_score: float  # fts search only

class MergedHit(RawHit):
    rrf_score: float

class RankedHit(MergedHit):
    rerank_score: float

RagHit = RawHit | MergedHit | RankedHit

@dataclasses.dataclass
class PipelineStageResult:
    stage: str
    success: bool
    failure_reason: str | None = None
    elapsed_s: float = 0.0
```

Note: Use `typing.Required` / `typing.NotRequired` (Python 3.11+) to mark mandatory fields precisely within `total=False` TypedDicts. Since project uses Python 3.13, this is available.

### Details

- `RawHit` with `total=False` and `Required[chunk_id]`, `Required[content]`, etc. — ensures chunk_id/content/url/title are always present while distance/bm25_score remain optional.
- Callers doing `hit.get("rrf_score")` still work; callers doing `hit["rrf_score"]` will get a type error on `RawHit` only, which is correct.
- `PipelineContext` update in `stage.py`:
  - `search_results: list[list[RawHit]]`
  - `merged: list[MergedHit]`
  - `reranked: list[RankedHit]`
  - `observers: list[Callable[[str, PipelineStageResult], None]]`
- Import `from collections.abc import Callable` in `stage.py`.
