# Implementation: H-2 — Update test_tool_runner.py for removed ToolResultStore.store() call

## Goal

Remove test setup/assertions that depend on `ctx.tool_result_store.store(...)` being called by
`_collect_tool_result_msgs()`, matching the companion source change that removes that call (see
`implementations/20260708-162541_tool_runner_h2.py.md` or its `done/` copy).

## Scope

**Target**: `tests/test_tool_runner.py`

**Depends on**: `scripts/agent/tool_runner.py` H-2 change already applied (or applied together
with this doc — order between the two does not matter for correctness, but tests will fail
against unmodified source until both land).

**Out of scope**: `tests/test_tool_result_store.py` (tests the `ToolResultStore` class itself,
unaffected by this change since the class is not modified).

## Assumptions

1. `_make_ctx()` (test helper, around line 85-86) sets up
   `ctx.tool_result_store.store = MagicMock(return_value=1)` purely to support the old
   `result_id = ctx.tool_result_store.store(...)` call path. Since that call is removed from
   production code, this mock configuration is unnecessary but harmless to leave in place UNLESS
   it causes an unused-mock lint warning (mocks are not statically checked, so no action is
   strictly required here — see Method).
2. Only one test method (`test_approved_calls_executed_and_collected`) asserts
   `ctx.tool_result_store.store.assert_called_once()` — confirmed by grep (single match at line
   303; the other three `tool_result_store` references at lines 85, 86, 289 are mock setup, not
   assertions).

## Implementation

### Target file

`tests/test_tool_runner.py`

### Procedure

#### Step 1: Confirm the current reference set

```bash
grep -n "tool_result_store" tests/test_tool_runner.py
```

Expected (pre-change):
```
85:    ctx.tool_result_store = MagicMock()
86:    ctx.tool_result_store.store = MagicMock(return_value=1)
289:        ctx.tool_result_store.store = MagicMock(return_value=1)
303:        ctx.tool_result_store.store.assert_called_once()
```

#### Step 2: Remove the failing assertion

In `TestExecuteAllToolCalls::test_approved_calls_executed_and_collected` (around line 280-304),
remove line 303:

```python
        ctx.tool_result_store.store.assert_called_once()
```

Leave `ctx.services_required.tools.execute.assert_awaited_once_with(...)` (lines 300-302) and
`ctx.session.save_many.assert_called_once()` (line 304) — these verify behavior that is
explicitly preserved by the plan (`ctx.session.save_many(...)` persists the `messages` row).

#### Step 3: Remove now-pointless mock setup (optional cleanup)

Lines 86 and 289 (`ctx.tool_result_store.store = MagicMock(return_value=1)`) configure a mock
return value that nothing consumes anymore, since production code no longer calls `.store(...)`.
Leaving them is harmless (mocks that are never called do not fail a test unless explicitly
asserted), but for clarity remove them:

- Line 86 inside `_make_ctx()`: remove `ctx.tool_result_store.store = MagicMock(return_value=1)`.
  Keep `ctx.tool_result_store = MagicMock()` itself (line 85) — `cmd_tooling.py` tests elsewhere
  may still rely on `ctx.tool_result_store` existing as a MagicMock attribute, and this test file
  is not the only consumer of `_make_ctx()`'s context object shape.
- Line 289 inside `test_approved_calls_executed_and_collected`: remove
  `ctx.tool_result_store.store = MagicMock(return_value=1)` (now redundant with the base
  `_make_ctx()` setup and unused either way).

### Method

- Deletion-only change; no new test cases required by this plan (per its Acceptance Criteria,
  which only lists the absence of the `.store()` call and continuity of `messages`
  persistence/stat counters — all already covered by existing assertions once the failing one is
  removed).
- Do not add a new assertion asserting `ctx.tool_result_store.store.assert_not_called()` unless
  requested — the plan's scope is removal of the stale assertion, not addition of a negative
  test. (If the user wants regression coverage against the store call reappearing, that would be
  a reasonable follow-up, but it is not in this plan's Implementation Steps.)

### Details

- `_make_ctx()` is a shared fixture helper used by many test classes in this file beyond
  `TestExecuteAllToolCalls` — verify no other test in the file relies on
  `ctx.tool_result_store.store` returning `1` specifically (grep in Step 1 confirms only the one
  assertion at line 303 and the two mock-setup lines exist; no other test reads the return value).
