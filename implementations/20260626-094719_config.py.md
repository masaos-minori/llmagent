# Implementation: Add validation to EventBus config in config.py

Steps covered: Plan 20260626-094719 — Steps 1, 2

---

## Goal

Add Pydantic (or dataclass-based) validation to `scripts/eventbus/config.py` so that invalid configuration values raise a clear `ValidationError` at startup rather than causing cryptic runtime errors.

---

## Scope

- **In scope**: `scripts/eventbus/config.py` — `EventBusConfig` validation
- **Out of scope**: tests (step 3)

---

## Assumptions

- `EventBusConfig` is a dataclass or Pydantic model.
- Validations needed: `port` in range [1024, 65535], `host` is a valid IP or hostname, `max_batch_size >= 1`, `retention_days >= 1`.
- Current config has no validation (ValueError only at use-time).

---

## Implementation

### Target file
`scripts/eventbus/config.py`

### Procedure
1. Read `scripts/eventbus/config.py` to see the current model structure.
2. Step 1 (Add validators):
   If using Pydantic:
   ```python
   from pydantic import validator, ValidationError
   
   class EventBusConfig(BaseModel):
       host: str = "127.0.0.1"
       port: int = 8765
       max_batch_size: int = 100
       retention_days: int = 90
       
       @validator("port")
       def port_in_range(cls, v):
           if not 1024 <= v <= 65535:
               raise ValueError(f"port must be 1024-65535, got {v}")
           return v
       
       @validator("max_batch_size", "retention_days")
       def positive(cls, v, field):
           if v < 1:
               raise ValueError(f"{field.name} must be >= 1")
           return v
   ```
   If using dataclass, use `__post_init__` for the same checks.
3. Step 2 (Startup fail-fast): In `app.py` or server startup, catch `ValidationError` and exit with a clear message.

### Method
Validation-only change. No behavioral change for valid configs.
