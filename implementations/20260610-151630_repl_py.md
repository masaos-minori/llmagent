# Implementation: repl.py

## Goal

Replace all `except Exception` handlers in `repl.py` with specific exception types, route `print()` calls through `self._view`, and add a catch boundary in `_repl_loop` for exceptions propagated from `handle_turn()`.

## Scope

- Target: `scripts/agent/repl.py`
- `_get_chunk_count()` line ~100: replace `except Exception` with `except sqlite3.Error`
- `_start_subprocess_servers()` lines ~205, ~230: replace `except Exception` with `except (OSError, RuntimeError, HttpStartupError)`
- `_run_repl_loop()` line ~306: replace `except Exception` with `except (RuntimeError, asyncio.CancelledError)`
- Replace direct `print()` in `_print_startup_banner()` and `_start_subprocess_servers()` with `self._view.write_*` calls
- Add `except Exception as e: self._view.write_llm_error(e)` around `await self._orchestrator.handle_turn(line)` in `_repl_loop`

## Assumptions

1. `self._view.write_warning(msg)` is the appropriate method for startup warnings (already exists in `CLIView`).
2. `self._view.write_llm_error(e)` is the appropriate method for unexpected turn errors (already exists in `CLIView`).
3. `_get_chunk_count()` returns `"?"` on DB error — this is acceptable for startup banner display; change to `sqlite3.Error` catch.
4. `HttpStartupError` is imported from `agent.http_lifecycle`.
5. The `_run_repl_loop` `except Exception` around `memory.on_session_stop` is the last-resort cleanup boundary; replace with `except (RuntimeError, OSError)`.

## Implementation

### Target file

`scripts/agent/repl.py`

### Procedure

1. Add imports: `import sqlite3`, `from agent.http_lifecycle import HttpStartupError`.
2. `_get_chunk_count()`: `except Exception as e:` → `except sqlite3.Error as e:`.
3. `_start_subprocess_servers()` HTTP server: `except Exception as e:` → `except (HttpStartupError, OSError) as e:`. Replace `print(f"[warn] HTTP ...")` with `self._view.write_warning(...)`.
4. `_start_subprocess_servers()` stdio server: `except Exception as e:` → `except (OSError, RuntimeError) as e:`. Replace `print(f"[warn] stdio ...")` with `self._view.write_warning(...)`.
5. `_print_startup_banner()`: replace `print(f"DB: ...")` with `self._view.write_startup_banner(chunk_count, self._n_tools)`. Add `write_startup_banner(chunk_count: str, n_tools: int) -> None` to `CLIView` (see `cli_view_py.md`).
6. `_repl_loop()`: replace `print()` (blank line on EOFError/KeyboardInterrupt) with `self._view.write_turn_end()`. Replace `print("\nShutdown requested...")` with `self._view.write_warning("Shutdown requested, exiting...")`. Replace `print(f"Unknown command: ...")` with `self._view.write_warning(...)`. Add around `await self._orchestrator.handle_turn(line)`:
   ```python
   try:
       await self._orchestrator.handle_turn(line)
   except Exception as e:
       self._view.write_llm_error(e)
   ```
7. `_run_repl_loop()` `except Exception:` around `memory.on_session_stop`: → `except (RuntimeError, OSError):`.

### Method

Systematic `print()` → `view` migration plus exception type narrowing. The `try/except Exception` around `handle_turn` is intentional: it's the REPL's last-resort display boundary.

### Details

The only remaining `except Exception` in `repl.py` after this change is the `handle_turn` catch, which is the correct REPL-level safety net. All other catches are narrowed.

```python
# _repl_loop — handle_turn error boundary
try:
    await self._orchestrator.handle_turn(line)
except Exception as e:
    self._view.write_llm_error(e)
```
