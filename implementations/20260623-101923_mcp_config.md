## Goal

Add `HALF_OPEN` state and cooldown-based automatic recovery to `McpServerHealthRegistry` in `scripts/shared/mcp_config.py`. After a server becomes `UNAVAILABLE` and 30 seconds elapse, `is_unavailable()` transitions it to `HALF_OPEN` and returns `False`, allowing one trial dispatch. A successful trial reverts to `HEALTHY`; a failed trial resets to `UNAVAILABLE` with a fresh cooldown timestamp.

## Scope

**In-Scope:**
- `scripts/shared/mcp_config.py` — all changes are within this file:
  - Add `import time` at module level
  - Add `HALF_OPEN = "half_open"` to `McpServerHealthState`
  - Add `half_open_cooldown_sec: float = 30.0` parameter to `__init__()`
  - Add `_unavailable_since: dict[str, float]` dict to `__init__()`
  - Update `is_unavailable()` — check cooldown, transition UNAVAILABLE → HALF_OPEN on expiry
  - Update `record_failure()` — handle HALF_OPEN → UNAVAILABLE, set `_unavailable_since`
  - Update `record_success()` — clear `_unavailable_since`, log when recovering from HALF_OPEN

**Out-of-Scope:**
- No changes to `tool_executor.py` (covered in separate doc)
- No changes to test files
- No changes to `agent/config_dataclasses.py` (cooldown is a constructor arg, not a config field)

## Assumptions

1. `time.monotonic()` is used (not `time.time()`) to avoid wall-clock skew during tests.
2. The `is_unavailable()` method is the only place that triggers the UNAVAILABLE → HALF_OPEN transition — callers that want to gate dispatch call `is_unavailable()`.
3. `_unavailable_since` is populated only when a server enters `UNAVAILABLE` state (both from threshold in `record_failure()` and from HALF_OPEN probe failure). It is cleared by `record_success()`.
4. The HALF_OPEN state does not itself set `_unavailable_since` — the timestamp remains from when the server first became unavailable, which is correct for the "cooldown elapsed once → allow one probe" model.
5. Existing tests use `McpServerHealthRegistry(failure_threshold=3)` without `half_open_cooldown_sec`; the default value of `30.0` means those tests are unaffected unless they call `is_unavailable()` after time has elapsed.

```python
    def is_unavailable(self, server_key: str) -> bool:
        state = self.get_state(server_key)
        if state != McpServerHealthState.UNAVAILABLE:
            return False
        since = self._unavailable_since.get(server_key, 0.0)
        if time.monotonic() - since >= self._half_open_cooldown_sec:
            self._states[server_key] = McpServerHealthState.HALF_OPEN
            logger.info(
                "Health: %r transitioning UNAVAILABLE → HALF_OPEN (trial probe)",
                server_key,
            )
            return False
        return True
```

Replacement:
```python
    def record_failure(self, server_key: str) -> McpServerHealthState:
        was_half_open = self.get_state(server_key) == McpServerHealthState.HALF_OPEN
        count = self._failure_counts.get(server_key, 0) + 1
        self._failure_counts[server_key] = count
        if was_half_open or count >= self._failure_threshold:
            self._states[server_key] = McpServerHealthState.UNAVAILABLE
            self._unavailable_since[server_key] = time.monotonic()
            if was_half_open:
                logger.warning(
                    "Health: %r trial probe failed → UNAVAILABLE (cooldown reset)",
                    server_key,
                )
            return McpServerHealthState.UNAVAILABLE
        self._states[server_key] = McpServerHealthState.DEGRADED
        return McpServerHealthState.DEGRADED
```

Replacement:

```python
    def record_success(self, server_key: str) -> None:
        prev = self.get_state(server_key)
        self._states[server_key] = McpServerHealthState.HEALTHY
        self._failure_counts[server_key] = 0
        self._unavailable_since.pop(server_key, None)
        if prev == McpServerHealthState.HALF_OPEN:
            logger.info("Health: %r trial probe succeeded → HEALTHY", server_key)
```
