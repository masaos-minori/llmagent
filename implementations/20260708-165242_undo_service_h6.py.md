# Implementation: H-6 — Remove n_artifacts_marked entirely from undo_last_turn()

## Goal

Remove `turn_to_mark`, `n_marked`, and the `mark_turn_undone(...)` call block from
`undo_last_turn()`, and stop passing `n_artifacts_marked` to `UndoResult` at all (the field
itself is removed from the model — see the companion `models.py` doc).

## Scope

**Target**: `scripts/agent/services/undo_service.py`

**Supersedes**: `implementations/20260708-164037_undo_service.py.md` (the H-7 doc for this same
file, which only fixed `n_marked` to always be `0` while keeping the field on `UndoResult`). H-6
goes further: the field is removed from `UndoResult` entirely (see the companion `models.py`
doc), so this function must stop passing it at all, not just pass a constant `0`.
**Implement this doc's version instead of H-7's** if both are pending — H-6's end state is a
strict superset of cleanup over H-7's for this file.

**Depends on**: land together with `implementations/*_models_h6.py.md` (the `UndoResult` field
removal) — removing the keyword argument here without removing the field from the dataclass
would still work (extra `0` field with a default), but removing the field without updating this
call site would raise `TypeError: unexpected keyword argument 'n_artifacts_marked'`. Apply both
in the same commit.

**Out of scope**: `scripts/db/tool_results.py` (`ToolResultStore.mark_turn_undone()` method
itself — stays defined, simply never called from here anymore), the `tool_results.undone` column
schema (unchanged — per the plan's explicit note that column removal is a separate future issue).

## Assumptions

1. `ctx.tool_result_store` may or may not exist on `AgentContext` depending on whether H-7's
   `context.py` doc has landed yet — this doc's change does not read `ctx.tool_result_store` at
   all, so it is safe regardless of H-7's status.
2. `UndoResult(n_removed=removed)` (single positional/keyword argument) is a valid call once the
   companion `models.py` doc removes `n_artifacts_marked` (which currently has a `= 0` default,
   so it is already optional today — removing it from the dataclass is what makes passing it an
   error, not the reverse).

## Implementation

### Target file

`scripts/agent/services/undo_service.py`

### Procedure

#### Step 1: Confirm current state

```bash
grep -n "turn_to_mark\|n_marked\|mark_turn_undone\|n_artifacts_marked" scripts/agent/services/undo_service.py
```

Expected (pre-change, matching the plan's Affected Areas): `turn_to_mark = ctx.stats.stat_turns`,
`n_marked = 0`, the `if turn_to_mark > 0 and ctx.tool_result_store is not None:` block, and
`return UndoResult(n_removed=removed, n_artifacts_marked=n_marked)`.

#### Step 2: Remove all four elements

Current (lines 45-55, per the module's current numbering):

```python
    # Capture turn BEFORE decrementing stat_turns — equals the last stored turn
    turn_to_mark = ctx.stats.stat_turns
    ctx.stats.stat_turns = max(0, ctx.stats.stat_turns - 1)
    ctx.session.undo_last_turn()
    n_marked = 0
    if turn_to_mark > 0 and ctx.tool_result_store is not None:
        n_marked = ctx.tool_result_store.mark_turn_undone(
            ctx.session.session_id, turn_to_mark
        )
    logger.info("Undo: removed %s messages from history", removed)
    return UndoResult(n_removed=removed, n_artifacts_marked=n_marked)
```

Replace with:

```python
    ctx.stats.stat_turns = max(0, ctx.stats.stat_turns - 1)
    ctx.session.undo_last_turn()
    logger.info("Undo: removed %s messages from history", removed)
    return UndoResult(n_removed=removed)
```

#### Step 3: Update the function docstring

Current: `"""Roll back the last user+assistant turn from history and DB.` followed by mentions of
"decrements stat_turns, and calls session.undo_last_turn()". Remove any reference to artifact
marking if present (the current docstring, per the earlier H-7 read of this file, does not
explicitly mention marking, so this step may be a no-op — verify against the live file).

### Method

- Pure deletion of the `turn_to_mark`/`n_marked`/`mark_turn_undone` block; the function's other
  logic (finding `last_user_idx`, stripping memory-injection markers, trimming history,
  decrementing `stat_turns`, calling `ctx.session.undo_last_turn()`) is entirely unchanged.

### Details

- `ctx.tool_result_store` is not referenced anywhere in this function after this change — this
  file no longer has ANY dependency on that field, regardless of whether H-7's `context.py`
  removal has landed.
- This is a stricter cleanup than H-7's version of this file: H-7 kept `n_marked = 0` as an
  explicit local variable passed to `UndoResult(..., n_artifacts_marked=n_marked)`; H-6 removes
  the concept of "artifacts marked" from this function's vocabulary entirely.
