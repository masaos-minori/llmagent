# Implementation: Improve /reload Feedback for Non-Reloadable Settings

## Goal

Display `result.skipped` items in `/reload` output so users see which settings require a restart.

## Scope

- `scripts/agent/commands/cmd_config.py` — add "Skipped:" section to reload output

## Assumptions

1. `ConfigReloadOutcome.skipped` is already populated by `ConfigReloadService`.
2. The existing two-section format (Applied / Restart required) should be extended with a Skipped section.

## Implementation

### Target file

`scripts/agent/commands/cmd_config.py`

### Procedure

Add a "Skipped:" section between "Applied" and "Restart required" in `_cmd_reload()`.

### Details

Current code (lines 55-64):
```python
            if result.applied:
                self._out.write("Applied (runtime):")
                for item in result.applied:
                    self._out.write(f"  - {item}")
            if result.needs_restart:
                self._out.write("Restart required:")
                for item in result.needs_restart:
                    self._out.write(f"  - {item}")
```

Replace with:
```python
            if result.applied:
                self._out.write("Applied (runtime):")
                for item in result.applied:
                    self._out.write(f"  - {item}")
            if result.skipped:
                self._out.write("Skipped:")
                for item in result.skipped:
                    self._out.write(f"  - {item}")
            if result.needs_restart:
                self._out.write("Restart required:")
                for item in result.needs_restart:
                    self._out.write(f"  - {item}")
```
