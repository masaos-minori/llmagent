# Implementation: Replace module-global state with app state in EventBus app.py

Steps covered: Plan 20260626-094721 — Steps 1, 2, 3, 4

---

## Goal

Move all module-level global variables in `scripts/eventbus/app.py` to `app.state` (FastAPI/Starlette lifespan pattern) so that tests can run isolated instances without shared global state between test cases.

---

## Scope

- **In scope**: `scripts/eventbus/app.py` — all module-level globals (db, config, subscribers dict, etc.)
- **Out of scope**: db.py; config.py; tests

---

## Assumptions

- `app.py` has module-level globals like `_db: Optional[Database] = None`, `_subscribers: dict = {}`.
- FastAPI lifespan context manager is the standard way to initialize/teardown resources.
- After this change, `app.state.db`, `app.state.config`, etc. replace module globals.

---

## Implementation

### Target file
`scripts/eventbus/app.py`

### Procedure
1. Read `scripts/eventbus/app.py` — list all module-level globals.
2. Step 1 (Lifespan context): Add a lifespan context manager:
   ```python
   from contextlib import asynccontextmanager
   
   @asynccontextmanager
   async def lifespan(app: FastAPI):
       app.state.config = load_config()
       app.state.db = await Database.connect(app.state.config.db_path)
       app.state.subscribers = {}
       yield
       await app.state.db.close()
   
   app = FastAPI(lifespan=lifespan)
   ```
3. Step 2 (Remove globals): Delete all module-level `_db`, `_config`, `_subscribers` etc.
4. Step 3 (Update handlers): In each route handler, access via `request.app.state.db` instead of the module global.
5. Step 4 (Dependency injection): Use FastAPI `Depends` for clean injection:
   ```python
   def get_db(request: Request) -> Database:
       return request.app.state.db
   ```

### Method
Refactor: globals → app state. No behavioral change. Test isolation improves significantly.
