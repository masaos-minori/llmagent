## Goal

Add a single shared helper function `availability_flags(allowed_dirs: list[str]) ->
tuple[bool, str]` to `scripts/mcp_servers/file/common.py`, so the three file servers
(read/write/delete) can compute their `/v1/tools` `enabled`/`disabled_reason` fields from
one place instead of duplicating the same boolean check three times.

## Scope

**In scope**: add one new pure function to `scripts/mcp_servers/file/common.py`. No
changes to existing functions/classes in this file.

**Out of scope**: the three `list_tools()` handlers that will call this helper (covered by
their own docs: `read_server.py`, `write_server.py`, `delete_server.py`), and the git
server (which has a different precedence rule and does not use this helper).

## Assumptions

- All three file servers (`read_server.py`, `write_server.py`, `delete_server.py`) already
  import from `mcp_servers.file.common` (confirmed: each has a multi-line
  `from mcp_servers.file.common import (...)` block importing `FileAuthorizationError`,
  `FileValidationError`, `_health`, `_on_auth_error`, `_on_not_found`,
  `_on_validation_error`), so adding one more name to each of those three import blocks
  requires no new import edge and stays `.importlinter`-compliant (`mcp_servers -> shared`
  is unaffected; this is an intra-package `mcp_servers.file` import, already allowed).
- The function must be pure (no I/O, no side effects) and take a plain `list[str]` so it
  works uniformly across `FileReadConfig.allowed_dirs`, `FileWriteConfig.allowed_dirs`, and
  `FileDeleteConfig.allowed_dirs` (all three are `list[str]` per their respective
  `*_models.py`, confirmed by the plan's Assumptions section).
- Reason string must be the exact literal `"allowed_dirs is empty"` (case/wording matters
  per the plan's Risks table — Agent-side parsing may do exact string matching).
