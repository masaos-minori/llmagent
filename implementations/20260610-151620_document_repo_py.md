# Implementation: document_repo.py

## Goal

Replace `except Exception` handlers with `sqlite3.Error` and remove `[]`/`False` fallback returns so DB failures propagate to the caller.

## Scope

- Target: `scripts/agent/document_repo.py`
- Replace `except Exception` in `list_documents()` and `delete_document()` with `except sqlite3.Error`
- Remove `return []` and `return False` fallback returns on DB error
- Remove `from typing import Any` — use `list[str | int]` for SQL params instead

## Assumptions

1. `AgentSession.list_documents()` delegates to `DocumentRepository.list_documents()`; it must handle `sqlite3.Error` after this change.
2. `AgentSession.delete_document()` delegates to `DocumentRepository.delete_document()`; same.
3. Command handlers (`cmd_db.py`) that call these methods must be updated to catch `sqlite3.Error` and display an error message.

## Implementation

### Target file

`scripts/agent/document_repo.py`

### Procedure

1. Add `import sqlite3` at the top.
2. Remove `from typing import Any`.
3. In `list_documents()`: change `params: list[Any]` to `params: list[str | int]`. Remove the `try/except` block; let `sqlite3.Error` propagate.
4. In `delete_document()`: remove the `try/except` block; let `sqlite3.Error` propagate. Change the `if row is None: return False` early-return to raising `KeyError(f"Document not found: {url!r}")` (or keep `return False` for the not-found case — this is domain logic, not an error).

### Method

Remove broad catches. The not-found case (`row is None`) remains a `return False` since it is expected domain behavior, not a failure.

### Details

```python
import sqlite3

class DocumentRepository:
    def list_documents(self, lang=None, limit=20) -> list[dict]:
        sql = "SELECT ..."
        params: list[str | int] = []
        if lang:
            sql += " WHERE d.lang = ?"
            params.append(lang)
        params.append(limit)
        with SQLiteHelper("session").open(row_factory=True) as db:
            rows = db.fetchall(sql, tuple(params))
        return [{...} for r in rows]
        # sqlite3.Error propagates to AgentSession.list_documents()

    def delete_document(self, url: str) -> bool:
        with SQLiteHelper("session").open(write_mode=True) as db:
            row = db.execute("SELECT doc_id FROM documents WHERE url = ?", (url,)).fetchone()
            if row is None:
                return False  # not found — domain result, not an error
            doc_id = row[0]
            db.execute("DELETE FROM chunks_vec WHERE chunk_id IN (...)", (doc_id,))
            db.execute("DELETE FROM documents WHERE doc_id = ?", (doc_id,))
            db.commit()
        return True
```
