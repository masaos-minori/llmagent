# Implementation: scripts/db/models.py

## Goal

Add `tool_call_id: str | None = None` field to `MessageRow` dataclass so that the DTO correctly represents the OpenAI tool-call ID associated with a tool result message.

## Scope

- Target: `scripts/db/models.py`
- Add `tool_call_id: str | None = None` to `MessageRow`
- No other dataclasses changed

## Assumptions

1. `MessageRow` is a `@dataclass(frozen=True)` — adding a field with a default value does not break existing instantiations that omit `tool_call_id`.
2. `tool_call_id` is placed after `tool_calls` for logical grouping (both are tool-call related).
3. The column `tool_call_id TEXT` already exists in the `messages` DDL (`schema_sql.py`).

## Implementation

### Target file
`scripts/db/models.py`

### Procedure
1. Locate `MessageRow` dataclass definition.
2. Add `tool_call_id: str | None = None` after the `tool_calls` field.

### Method

```python
@dataclass(frozen=True)
class MessageRow:
    role: str
    content: str
    tool_calls: str | None
    tool_call_id: str | None = None  # add this line
```

### Details
- `tool_call_id` must have a default value (`= None`) so existing `MessageRow(role=..., content=..., tool_calls=...)` calls remain valid.
- No import changes needed.
