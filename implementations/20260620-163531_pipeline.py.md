# Implementation: HTTP result classification in pipeline.py (augment + get_diagnostics)

## Goal
1. Set `fallback_reason="http_remote_empty"` on `StageResult` for `HttpAugment` when
   `call_rag_service()` returns `""` (valid empty result, not a failure).
2. Add `_http_result_kind()` helper and `"http_result_kind"` key in `get_diagnostics()`.

## Scope
- File: `scripts/rag/pipeline.py`
- Two locations: `augment()` HTTP block (lines ~429-442) and `get_diagnostics()` return dict
- Additive changes only; existing behavior is preserved

## Assumptions
- `result is not None and not result` is the correct check for empty-string result (confirmed:
  `call_rag_service()` returns `str | None`; `""` is `not result` when `result is not None`)
- `fallback_reason` on `status="success"` is intentional per plan (pragmatic overload)
- `_http_result_kind()` reads `self.last_stage_results` directly — safe, called after `augment()`
- `rag_service_url` empty → no `HttpAugment` entry → `_http_result_kind()` returns `None`

**Edit 2 — add `_http_result_kind()` method (insert before or after `get_diagnostics()`):**
```python
    def _http_result_kind(self) -> str | None:
        """Classify the HTTP RAG result for the last augment() call.

        Returns None when HTTP mode was not used (rag_service_url not set).
        """
        http_stage = next(
            (r for r in self.last_stage_results if r["stage_name"] == "HttpAugment"),
            None,
        )
        if http_stage is None:
            return None
        if http_stage["status"] == "fallback":
            return "in_process_fallback"
        if http_stage.get("fallback_reason") == "http_remote_empty":
            return "remote_empty"
        return "remote_nonempty"
```
