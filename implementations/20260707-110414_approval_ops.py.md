## Goal
Add `workflow_id: str` to `request_approval()` and populate it in all `ApprovalRecord` row-mapping functions in `approval_ops.py`; add `workflow_id TEXT NOT NULL` to the `approvals` table schema.

## Scope
**In**: `scripts/agent/workflow/approval_ops.py` — change `request_approval()` to accept `workflow_id: str`; update all row-to-`ApprovalRecord` conversions; add `workflow_id` column to `CREATE TABLE approvals`.
**Out**: Removing `find_latest_pending_approval()` (kept for startup recovery); migration support; changing approval status logic.

## Assumptions
- `request_approval(db, task_id, stage_id=None)` is the current signature.
- `WorkflowEngine._gate_approval(task)` is the only caller (confirmed by `grep -rn "request_approval(" scripts/`).
- After Plan 09 (req09), `task.workflow_id` is a required `str`; passing it to `request_approval()` is always safe.
- `CREATE TABLE approvals` is defined in a Python function (not .sql file) — add `workflow_id TEXT NOT NULL DEFAULT ''` initially; tighten to `TEXT NOT NULL` on fresh schema init.
- `ApprovalRecord` is updated in `models.py` doc to include `workflow_id: str` field.

## Implementation

**Target file**: `scripts/agent/workflow/approval_ops.py`

**Procedure**:

1. **Locate `CREATE TABLE approvals`**:
   - `grep -n "CREATE TABLE approvals\|CREATE.*approvals" scripts/agent/workflow/approval_ops.py scripts/` — find definition
   - Add column: `workflow_id TEXT NOT NULL DEFAULT ''`
   - Note: `DEFAULT ''` allows existing rows without workflow_id to remain valid; enforce non-empty at app layer

2. **Change `request_approval()` signature**:
   ```python
   # Before:
   def request_approval(db, task_id: str, stage_id: str | None = None) -> ApprovalRecord:

   # After:
   def request_approval(db, task_id: str, workflow_id: str, stage_id: str | None = None) -> ApprovalRecord:
   ```
   - Add `workflow_id` to the `INSERT INTO approvals` statement:
     ```sql
     INSERT INTO approvals (approval_id, task_id, workflow_id, status, stage_id, created_at)
     VALUES (?, ?, ?, 'pending', ?, ?)
     ```
   - Pass `workflow_id` in the VALUES tuple

3. **Update row-to-`ApprovalRecord` conversions** in all finder functions:
   - `get_pending_approval()`: add `workflow_id=row["workflow_id"]` to `ApprovalRecord(...)` constructor
   - `find_approval_by_id()`: same
   - `find_latest_pending_approval()`: same (keep function; only update record construction)
   - `find_pending_approval_by_session()`: same

4. **Update caller in `workflow_engine._gate_approval()`** (see workflow_engine.py doc):
   ```python
   approval = request_approval(self._store._db, task.task_id, task.workflow_id, stage_id)
   ```

**Method**: Parameter addition + INSERT statement update + row-mapping updates.

**Details**:
- Execution order: implement after `workflow_models.py` change (ApprovalRecord.workflow_id field) and Plan 09 (task.workflow_id required).
- If the schema definition is in a separate function (`create_workflow_schema()`), find it with `grep -rn "CREATE TABLE approvals" scripts/`.
- The `DEFAULT ''` clause allows the preflight check (req08/Plan 08) to distinguish new rows (non-empty) from legacy rows (empty) without a migration.
