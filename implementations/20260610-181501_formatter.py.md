# Implementation: mcp/github/formatter.py + mapper.py — Formatter/Mapper Extraction

## Goal

Extract `fmt_*` formatter methods, `_dry_run_preview`, and `_issue_to_info`/`_pr_to_info` mapper methods from `service.py` into dedicated `formatter.py` and `mapper.py` modules. `service.py` retains only business logic.

---

## Scope

**New files:**
- `scripts/mcp/github/formatter.py` — `fmt_*` methods (20 methods) + `_dry_run_preview`
- `scripts/mcp/github/mapper.py` — `_issue_to_info`, `_pr_to_info`

**Update:**
- `scripts/mcp/github/service.py` — remove `fmt_*` + `_issue_to_info` + `_pr_to_info` + `_dry_run_preview`; import from new modules
- `scripts/mcp/github/server.py` — update `get_dispatch_table` call if it references `fmt_*` directly

**Out:**
- Request/response model changes — none
- `tools.py` — unchanged in this step

---

## Assumptions

1. `fmt_*` methods are async and take `(self, args: ToolArgs) -> str`. After extraction to `formatter.py`, they become standalone async functions taking `(service: GitHubService, args: ToolArgs) -> str`.
2. `_dry_run_preview` is a `@staticmethod` that takes `(preview: str) -> str` — becomes a module-level function.
3. `_issue_to_info` and `_pr_to_info` are `@staticmethod` — become module-level functions in `mapper.py`.
4. `get_dispatch_table` in `service.py` builds a dict of `{tool_name: fmt_method}`. After extraction, it maps to `formatter.py` functions bound with service.
5. Backward-compat: `service.py` re-exports `fmt_*` as instance methods calling the extracted functions, so `server.py` dispatch table stays valid.

---

## Implementation

### Target files
`scripts/mcp/github/formatter.py` (new), `scripts/mcp/github/mapper.py` (new), `scripts/mcp/github/service.py` (update)

### Procedure

**formatter.py:**
1. Create `scripts/mcp/github/formatter.py`.
2. Move `_dry_run_preview(preview: str) -> str` as a module-level function.
3. Move all `fmt_*` as module-level async functions: `async def fmt_search_repositories(service: "GitHubService", args: ToolArgs) -> str`.
4. Import `GitHubService` using `TYPE_CHECKING` guard to avoid circular import.

**mapper.py:**
1. Create `scripts/mcp/github/mapper.py`.
2. Move `_issue_to_info(issue: Any) -> IssueInfo` as module-level function.
3. Move `_pr_to_info(pr: Any) -> PullRequestInfo` as module-level function.
4. Import `IssueInfo`, `PullRequestInfo` from `mcp.github.models`.

**service.py:**
1. Remove moved methods.
2. Add imports: `from mcp.github.mapper import issue_to_info, pr_to_info`.
3. Update all callers of `_issue_to_info` and `_pr_to_info` to use `issue_to_info`, `pr_to_info`.
4. Keep `get_dispatch_table` in service.py but update it to use `formatter.py` functions.

### Method

```python
# mapper.py
from typing import Any
from mcp.github.models import IssueInfo, PullRequestInfo

def issue_to_info(issue: Any) -> IssueInfo:
    return IssueInfo(
        number=issue.number,
        title=issue.title,
        state=issue.state,
        url=issue.html_url,
        body=issue.body,
        created_at=issue.created_at.isoformat(),
        updated_at=issue.updated_at.isoformat(),
        labels=[lb.name for lb in issue.labels],
        assignees=[a.login for a in issue.assignees],
    )

def pr_to_info(pr: Any) -> PullRequestInfo:
    return PullRequestInfo(
        number=pr.number,
        title=pr.title,
        state=pr.state,
        url=pr.html_url,
        body=pr.body,
        head_ref=pr.head.ref,
        base_ref=pr.base.ref,
        created_at=pr.created_at.isoformat(),
        updated_at=pr.updated_at.isoformat(),
        draft=pr.draft,
    )


# formatter.py
from __future__ import annotations
from typing import TYPE_CHECKING, Any
if TYPE_CHECKING:
    from mcp.github.service import GitHubService
from mcp.server import ToolArgs

def dry_run_preview(preview: str) -> str:
    return f"[DRY RUN] The following would be executed:\n{preview}"

async def fmt_search_repositories(service: "GitHubService", args: ToolArgs) -> str:
    # delegate to service.search_repositories, format result
    ...
```

### Details

- The `TYPE_CHECKING` guard in `formatter.py` prevents the circular import: `formatter → service → formatter`.
- `get_dispatch_table` in `service.py` should return lambdas that call formatter functions with `self` bound: `{"search_repositories": lambda args: fmt_search_repositories(self, args)}`.
- All 20 `fmt_*` methods need to be converted; their bodies only call `self.<method>(req)` and format the result — the pattern is uniform.
- After extraction, `service.py` no longer imports `fmt_md_link` or formats strings (imports that only existed for formatters are removed).
