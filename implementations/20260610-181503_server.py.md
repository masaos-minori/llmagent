## Implementation

### Target file
`scripts/mcp/github/server.py`

### Procedure

1. Add domain exception imports from `mcp.github.models`.
2. Add `TOOL_LIST` import from `mcp.github.tools`.
3. Add `@app.exception_handler` decorators after `app = FastAPI(...)`:
   - `GitHubAuthorizationError` ظْ 403
   - `GitHubNotFoundError` ظْ 404
   - `GitHubValidationError` ظْ 400
   - `GitHubConflictError` ظْ 409
   - `GitHubUpstreamError` ظْ 502
   - `GitHubAuditError` ظْ 500
4. Add `lifespan` context manager that calls `GitHubConfig.load()` and stores in `app.state.github_cfg`.
5. Update `list_tools()` route to return `{"tools": TOOL_LIST}`.

### Method

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
from mcp.github.models import (
    GitHubAuthorizationError, GitHubNotFoundError, GitHubValidationError,
    GitHubConflictError, GitHubUpstreamError, GitHubAuditError, GitHubConfig,
)
from mcp.github.tools import TOOL_LIST


@asynccontextmanager
async def lifespan(app: FastAPI):
    app.state.github_cfg = GitHubConfig.load()
    yield


app = FastAPI(lifespan=lifespan)


@app.exception_handler(GitHubAuthorizationError)
async def handle_auth_error(request: Request, exc: GitHubAuthorizationError):
    return JSONResponse(status_code=403, content={"detail": str(exc)})

@app.exception_handler(GitHubNotFoundError)
async def handle_not_found(request: Request, exc: GitHubNotFoundError):
    return JSONResponse(status_code=404, content={"detail": str(exc)})

@app.exception_handler(GitHubValidationError)
async def handle_validation(request: Request, exc: GitHubValidationError):
    return JSONResponse(status_code=400, content={"detail": str(exc)})

@app.exception_handler(GitHubConflictError)
async def handle_conflict(request: Request, exc: GitHubConflictError):
    return JSONResponse(status_code=409, content={"detail": str(exc)})

@app.exception_handler(GitHubUpstreamError)
async def handle_upstream(request: Request, exc: GitHubUpstreamError):
    return JSONResponse(status_code=502, content={"detail": str(exc)})

@app.exception_handler(GitHubAuditError)
async def handle_audit(request: Request, exc: GitHubAuditError):
    return JSONResponse(status_code=500, content={"detail": str(exc)})


@app.get("/v1/tools")
async def list_tools() -> dict:
    return {"tools": TOOL_LIST}
```

### Details

- The existing `lifespan` in `server.py` (if any) needs to be checked before adding a new one.
- `app.state.github_cfg` is stored but NOT yet used by routes (which still use `_service`). This is preparatory work for future DI migration.
- `from mcp.github.service import _GITHUB_TOKEN, _service` stays ظ¤ the singleton is still the production path.
- Exception handlers do NOT conflict with Pydantic `ValidationError` (422) ظ¤ FastAPI handles those separately.
