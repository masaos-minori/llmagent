# Step 3 — Fix `scripts/agent/tool_audit.py`: remove empty-request-id guard and convert `write_round_exec` to JSON-lines

## Goal

Remove the `or not mcp_request_id` guard in `audit_tool_exec` so transport-error events with empty `mcp_request_id` are written to the audit log, and convert `write_round_exec` to emit a single JSON-lines string instead of passing keyword `extra=` to `logger.info`.

## Scope

- In-Scope:
  - `audit_tool_exec`: remove `or not mcp_request_id` from the early-return guard (line 162)
  - `write_round_exec`: replace `logger.info("round_exec", extra={...})` with `logger.info(_json_dumps({...}))` — the dict must include `"event": "round_exec"`, `"source": "agent"`, `"ts": time.time()`, and all existing fields as top-level keys
- Out-of-Scope:
  - Changing the function signatures of `audit_tool_exec` or `write_round_exec`
  - Modifying `audit_approval`, `log_approval_decision`, `audit_workflow_start`, `audit_stage_completed`, `audit_approval_requested` — those are already correct
  - Importing additional modules beyond what is already imported (`time`, `_json_dumps` are already present)

## Assumptions

1. `audit_tool_exec` is called with `mcp_request_id=""` specifically for transport errors; removing the guard is the intended behavior per the plan.
2. `cmd_audit.py` does NOT filter on `mcp_request_id`; it filters only on `task_id` and `tool` fields. Confirmed by inspection: `_iter_audit_lines` skips non-JSON lines via `orjson.JSONDecodeError` catch, and no path in `cmd_audit.py` checks `mcp_request_id`. Empty `mcp_request_id` events are safe.
3. `write_round_exec` currently uses `logger.info("round_exec", extra={...})` (line 201–219). After the change, the positional argument must be a single JSON string, matching the pattern used by all other audit writers in the same file.
4. `time` is already imported in `tool_audit.py` (confirmed at top of file via `import time`).
5. `_json_dumps` is already imported from `shared.json_utils` (line 13).

### Method

- Guard removal: simple line edit (remove `or not mcp_request_id` fragment)
- `write_round_exec` conversion: replace `logger.info("round_exec", extra=dict)` pattern with `logger.info(_json_dumps(dict))` — the same pattern used by `audit_workflow_start` (lines 88–99), `audit_stage_completed` (lines 112–125), and `audit_approval_requested` (lines 138–149)
- The `event` key becomes `"round_exec"` (was the positional message arg)
- Add `"source": "agent"` and `"ts": time.time()` to the dict (not present in the old `extra=` dict)
- All other field names stay identical

### Details
