# Implementation: Wire REPL Debug Path for /rag search --debug

## Goal

The `/rag search --debug` command currently shows only per-stage timings. This implementation wires the `debug_fn` callback into `pipeline.augment()` and surfaces fallback reasons in the debug output alongside timings. It also ensures `write_debug_rag()` in `cli_view.py` is called from the REPL path.

## Scope

- `scripts/agent/commands/cmd_ingest.py`
  - Pass `debug_fn` to `pipeline.augment()` when `--debug` is set
  - Show `last_stage_results` fallback reasons in `_print_rag_timings()`
- `scripts/agent/cli_view.py`
  - `write_debug_rag()` is already defined; caller in `cmd_ingest.py` must invoke it

Out of scope:
- MCP debug path (already works via `capture_fn` in `mcp/rag_pipeline/service.py`)
- Changes to `write_debug_rag()` signature (covered in separate cli_view.py doc)

## Assumptions

1. `pipeline.augment()` accepts an optional `debug_fn: Callable[[list[str], list[list[RawHit]], list[RagHit], list[RagHit]], None] | None` (confirmed, line 327-330 of pipeline.py).
2. `write_debug_rag(data: dict)` in `cli_view.py` takes a plain dict; a shim is needed to adapt the positional-arg `debug_fn` signature to the dict-based `write_debug_rag()` call.
3. The REPL `/rag search` path in `cmd_ingest.py` calls `pipeline.augment(query)` without `debug_fn` (confirmed line 120).
4. `cmd_ingest.py` already has access to `self._out` (a `CLIView`-like interface with `write_debug_rag()`). Verify the concrete type of `self._out` before implementing.

## Implementation

### Target file

`scripts/agent/commands/cmd_ingest.py`

### Procedure

1. When `debug=True`, create a `debug_fn` closure that collects `(queries, all_results, merged, reranked)` and calls `self._out.write_debug_rag(data)`.
2. Pass `debug_fn=debug_fn` to `pipeline.augment(query, debug_fn=debug_fn)`.
3. In `_print_rag_timings()`, also print fallback reasons from `pipeline.last_stage_results`.

### Method

A closure captures the pipeline's debug args and calls `write_debug_rag` immediately (synchronous). No async needed; `debug_fn` is called synchronously by `augment()`.

Replace with:
```python
pipeline = RagPipeline(ctx.services.http, rag_cfg)

debug_fn = None
if debug:
    def debug_fn(
        queries: list,
        all_results: list,
        merged: list,
        reranked: list,
    ) -> None:
        self._out.write_debug_rag({
            "queries": queries,
            "all_results": all_results,
            "merged": [dict(c) for c in merged],
            "reranked": [dict(c) for c in reranked],
        })

context = await pipeline.augment(query, debug_fn=debug_fn)
...
if debug:
    self._print_rag_timings(pipeline.last_timings, pipeline.last_stage_results)
```

**cmd_ingest.py — update `_print_rag_timings()` signature and body:**

```python
def _print_rag_timings(
    self,
    timings: dict[str, float],
    stage_results: list | None = None,
) -> None:
    if not timings:
        return
    self._out.write("\n--- Stage timings ---")
    for stage_name, elapsed in timings.items():
        self._out.write(f"  {stage_name}: {elapsed * 1000:.1f} ms")
    if stage_results:
        fallbacks = [r for r in stage_results if r["status"] in ("fallback", "failure")]
        if fallbacks:
            self._out.write("\n--- Fallbacks / Failures ---")
            for r in fallbacks:
                self._out.write(
                    f"  {r['stage_name']} [{r['status']}]: {r['fallback_reason']}"
                )
```

**Note on self._out type:**
Verify that `self._out` implements `write_debug_rag(data: dict)`. If `self._out` is typed as a protocol or abstract class, ensure `write_debug_rag` is declared there. If not, add it to the protocol definition.
