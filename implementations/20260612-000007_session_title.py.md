# Goal

Strengthen `session_title.py` by adding `isinstance(choices, list)` validation and
replacing chained silent `.get()` calls on the first choice with explicit None/type checks.

# Scope

- `scripts/agent/services/session_title.py`

# Assumptions

1. Current line 54: `choices = data.get("choices", [])` — if `data["choices"]` exists
   but is not a list (e.g. `null` or `int`), the code falls through to `if not choices`
   which would be False for a truthy non-list, leading to an unexpected `KeyError`.
2. Current line 57: `choices[0].get("message", {}).get("content", "")` — silently returns
   `""` when `choices[0]` lacks "message" or when "content" is absent, producing an
   empty title that is then caught by `if not title` on line 58. However if `choices[0]`
   is not a dict, `.get()` raises `AttributeError` which is caught by the broad
   `except (orjson.JSONDecodeError, KeyError)` only if it's a KeyError — AttributeError
   would escape to `SessionTitleGenerationError` wrapping at the outer except.
3. The fix should not change observable behavior for well-formed responses — only make
   unexpected shapes raise `SessionTitleGenerationError` with a clear message.
4. `data` comes from `orjson.loads(resp.content)` which returns `Any`.

# Implementation

## Target file

`scripts/agent/services/session_title.py`

## Procedure

1. Replace lines 54-59 with explicit type-safe access:

```python
choices = data.get("choices")
if not isinstance(choices, list) or not choices:
    raise SessionTitleGenerationError("LLM response has no choices")
first = choices[0]
if not isinstance(first, dict):
    raise SessionTitleGenerationError("LLM choices[0] is not a dict")
message = first.get("message")
if not isinstance(message, dict):
    raise SessionTitleGenerationError("LLM choices[0].message is not a dict")
title = str(message.get("content", "")).strip()
if not title:
    raise SessionTitleGenerationError("LLM returned empty title")
```

2. Run ruff + mypy.

## Method

Replace chained `.get()` defaults with explicit type guards at each level.
Each structural deviation raises `SessionTitleGenerationError` immediately.
