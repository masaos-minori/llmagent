# Implementation: M-4 — Rename ContextBudget.tool_results to tool_messages

## Goal

Rename `ContextBudget.tool_results` → `tool_messages` and `.token_tool_results` →
`.token_tool_messages`. Pure rename to avoid confusion with the deleted `tool_results` DB table
(H-9/H-10); no calculation logic changes.

## Scope

**Target**: `scripts/agent/services/models.py` — the `ContextBudget` dataclass only.

**Depends on / must land together with**: `implementations/*_context_view_m4.py.md` and
`implementations/*_cmd_context_m4.py.md` (both construct/read `ContextBudget` by these field
names) — renaming the field here without updating both callers breaks the build immediately
(`TypeError: unexpected keyword argument` at construction, `AttributeError` at read time).

**Out of scope**: `system`, `history`, `token_system`, `token_history` fields — unchanged. Not
related to the `tool_results` DB table removal (H-9/H-10) in any code-dependency sense — this is
purely a naming clarification for an in-memory dataclass, motivated BY that removal but not
technically coupled to it.

## Assumptions

1. `ContextBudget` is a frozen dataclass with no external serialization (confirmed by the plan's
   own assumption) — renaming fields has no on-disk/JSON compatibility concern.
2. `ContextBudget` is constructed and read only within `context_view.py` and `cmd_context.py`
   (confirmed by the plan's own grep-based analysis, Unknown 1 resolution) — no other module
   references these two fields.

## Implementation

### Target file

`scripts/agent/services/models.py`

### Procedure

#### Step 1: Confirm current field names and docstring

```bash
grep -n "class ContextBudget" -A 15 scripts/agent/services/models.py
```

Expected:

```python
class ContextBudget:
    """Per-category character and token counts for /context budget breakdown.

    Character counts (*system*, *history*, *tool_results*) are always present.
    Token estimates are ``None`` when the source is exact (LLM usage or
    /tokenize endpoint); they are populated only when a category-aware fallback
    estimate is used.
    """

    system: int
    history: int
    tool_results: int
    token_system: int | None = None
    token_history: int | None = None
    token_tool_results: int | None = None
```

#### Step 2: Rename both fields and update the docstring

Replace with:

```python
class ContextBudget:
    """Per-category character and token counts for /context budget breakdown.

    Character counts (*system*, *history*, *tool_messages*) are always present.
    Token estimates are ``None`` when the source is exact (LLM usage or
    /tokenize endpoint); they are populated only when a category-aware fallback
    estimate is used.
    """

    system: int
    history: int
    tool_messages: int
    token_system: int | None = None
    token_history: int | None = None
    token_tool_messages: int | None = None
```

### Method

- Two field renames plus one docstring word substitution (`*tool_results*` → `*tool_messages*`);
  field order, types, and defaults are otherwise unchanged.
