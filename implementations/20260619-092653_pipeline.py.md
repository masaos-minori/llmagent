# Implementation: Log Active Fusion Config at Pipeline Init

## Goal

Emit a startup log line from `RagPipeline.__init__` that shows whether RRF is active and what `rrf_k` value is in use. This makes the fusion mode visible in log output without running a query.

## Scope

- `scripts/rag/pipeline.py` — add one `logger.info(...)` call in `__init__`

Out of scope:
- Changes to `RagConfig` protocol (covered by the rrf_k plumbing doc)
- Log format changes elsewhere

## Assumptions

1. This step assumes `self._cfg.rrf_k` is already available (depends on the plumbing step in `20260619-092652_pipeline.py.md`).
2. The log should fire unconditionally at `INFO` level on every pipeline instantiation.
3. No structured logging (JSON) is required; the existing `logging.getLogger(__name__)` pattern is sufficient.

## Implementation

### Target file

`scripts/rag/pipeline.py`

### Procedure

1. In `RagPipeline.__init__`, after `self._embed_url` is set (line 120), add the log statement.

### Method

Single `logger.info(...)` call. No new imports needed; `logger` is already defined at module level (line 61).

### Details

After line 120 (`self._embed_url = _module_cfg.get(...)`), insert:
```python
logger.info(
    "RagPipeline init: use_rrf=%s rrf_k=%d",
    self._cfg.use_rrf,
    self._cfg.rrf_k,
)
```

The log message follows the existing pattern in the file (`logger.warning("key=%s", val)`).

If `rrf_k` is not yet on `RagConfig` (because the plumbing step has not been applied), substitute `self._cfg.rrf_k` with a hardcoded fallback or defer this step until after the plumbing doc is implemented.
