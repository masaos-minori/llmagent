# Implementation: M-3 — Return UNKNOWN when health registry is None (mcp_status.py)

## Goal

Update `_resolve_health_state()` in `McpStatusService` to return
`McpServerHealthState.UNKNOWN` instead of `McpServerHealthState.HEALTHY` when the
health registry is `None`, so that untracked servers are clearly distinguished from
confirmed-healthy servers.

## Scope

**Target**: `scripts/agent/services/mcp_status.py`

**Steps covered**: Plan M-3 steps 2, 3 (including tests).

**Dependency**: M-3 step 1 (`McpServerHealthState.UNKNOWN` added to `mcp_health.py`).

## Assumptions

1. `_resolve_health_state()` currently returns `McpServerHealthState.HEALTHY` as a
   fallback when `registry is None`.
2. The `health` field in `McpProbeResult` is set from `_resolve_health_state().value`
   (or `.name`); `UNKNOWN.value == "unknown"` renders correctly.
3. Existing tests may assert `health == "healthy"` for the no-registry case; those must
   be updated to `"unknown"`.

## Implementation

### Target file

`scripts/agent/services/mcp_status.py`

### Procedure

#### Step 1: Locate `_resolve_health_state()`

Find the fallback return:
```python
if registry is None:
    return McpServerHealthState.HEALTHY
```

#### Step 2: Replace fallback return

```python
if registry is None:
    return McpServerHealthState.UNKNOWN
```

No other changes.

### Method

- Single token replacement.
- `McpServerHealthState.UNKNOWN` must be importable (after M-3 step 1 is applied).

### Details

- If `_resolve_health_state()` uses `McpServerHealthState.HEALTHY` as a default in
  multiple branches (not just the `registry is None` branch), only change the
  `registry is None` branch.
- The `HEALTHY` return for `registry.get_state(server_key)` when the registry exists
  but the server key is not tracked is correct behavior — leave that branch unchanged.

## Tests

### Target test file

`tests/` (mcp_status tests — locate by `pytest --collect-only -k mcp_status`)

### Test procedures

#### Test 1: registry=None → UNKNOWN health state

```python
def test_resolve_health_state_returns_unknown_when_no_registry():
    from agent.services.mcp_status import McpStatusService  # or wherever the function lives

    result = _resolve_health_state(registry=None, server_key="any")

    assert result == McpServerHealthState.UNKNOWN
    assert result.value == "unknown"
```

#### Test 2: registry present → returns registry's state (not UNKNOWN)

```python
def test_resolve_health_state_uses_registry_when_present():
    registry = McpServerHealthRegistry()
    registry.record_failure("srv")  # DEGRADED after first failure

    result = _resolve_health_state(registry=registry, server_key="srv")

    assert result != McpServerHealthState.UNKNOWN
    assert result in (McpServerHealthState.DEGRADED, McpServerHealthState.UNAVAILABLE)
```

#### Test 3: McpProbeResult.health is "unknown" when registry missing

```python
@pytest.mark.asyncio
async def test_probe_result_health_unknown_without_registry():
    # Build McpStatusService with no health_registry on ctx
    service = make_service_without_registry()
    results = await service.probe_all()

    for r in results:
        assert r.health.lower() == "unknown"
```
