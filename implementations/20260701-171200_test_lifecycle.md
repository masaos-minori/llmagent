# Implementation: Update StdioTransport mock specs in test_lifecycle.py, test_repl_health.py, test_startup.py

## Goal

Replace all `spec=StdioTransport` and `patch(...StdioTransport...)` references in three test files
with generic MagicMock specs or HTTP-transport equivalents so that these files no longer depend
on StdioTransport.

## Scope

- Target files:
  - tests/test_lifecycle.py
  - tests/test_repl_health.py
  - tests/test_startup.py

## Assumptions

- HttpTransport exists in shared.tool_executor and can be used as a generic mock spec.
- Tests using `spec=StdioTransport` only use it as a duck-type contract; the specific
  class does not matter for these mocks.
- `patch("agent.stdio_lifecycle.StdioTransport")` in test_lifecycle.py patches the
  stdio lifecycle module; if that module is removed later, the patch target changes.
- `patch("agent.startup.StdioTransport")` in test_startup.py patches startup module;
  verify the patch target string after source changes.
