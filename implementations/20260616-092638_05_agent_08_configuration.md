# 08 Configuration — Agent Documentation Restructuring

## Goal
AgentConfigの全7サブ設定・McpServerConfig・HealthRegistryの仕様を1章にまとめる。

## Scope
- AgentConfig の構造と全フィールド
- 7つのサブ設定クラスそれぞれの役割とキーパラメータ
- McpServerConfig と HealthRegistry の概要

## Assumptions
- 05_ref-agent-config.md 全体が本章の唯一のソース
- 設定値のデフォルト値は05_ref-agent-config.mdに記載のものを使用
## Implementation

### Target file
`docs/05_agent/08_configuration.md`

### Procedure
- 05_ref-agent-config.md の AgentConfig クラス定義と全フィールドを抽出
- 05_ref-agent-config.md の7サブ設定クラス(LLMConfig, HistoryConfig, ToolConfig, SessionConfig, ViewConfig, CommandConfig, OpsConfig等)を全件抽出
- 05_ref-agent-config.md の McpServerConfig を抽出
- 05_ref-agent-config.md の HealthRegistry を抽出
### Method
- H2: AgentConfig全体構造 / 各サブ設定クラス / McpServerConfig / HealthRegistry
- 各サブ設定は「フィールド名: 型 = デフォルト値 — 説明」の箇条書き
- 設定ファイル(TOML/YAML)での記述例を1例掲載
### Details
- AgentConfig: 7サブ設定への参照フィールド + グローバルフィールド(profile, log_level等)
- LLMConfig: model, max_tokens, temperature, max_retries, timeout_sec
- HistoryConfig: max_messages, compress_threshold, compression_model
- ToolConfig: always_allow リスト, tool_timeout_sec, max_iterations
- SessionConfig: session_dir, auto_save, session_ttl_days
- McpServerConfig: name, command, args, env の各フィールド
- HealthRegistry: サービスのliveness/readinessチェック登録API

## Validation plan
- 7サブ設定クラスが05_ref-agent-config.mdの定義と一致していること
- McpServerConfigの全フィールドが記述されていること
- デフォルト値が05_ref-agent-config.mdと一致していること
