# Remove unused MCP installer validation helpers if no longer referenced

## Priority
Medium

## Labels
mcp, cleanup, dead-code

## Dependencies
- Remove or isolate `agent/services/mcp_install.py`

## Background
`mcp/installer_validation.py` provides helper functions for validating and converting MCP server names. These helpers are used by the `/mcp install` scaffold workflow.

## Task
Delete `mcp/installer_validation.py` if it is only used by the removed MCP installer workflow.

## Implementation Requirements
- Search the repository for:
  - `validate_server_name`
  - `name_to_module`
  - `name_to_class`
  - `installer_validation`
- If all references are installer-only, delete `mcp/installer_validation.py`.
- If any non-installer workflow uses these helpers, move them to the appropriate non-installer module or leave them with clear ownership.

## Acceptance Criteria
- No Agent REPL runtime path imports `mcp.installer_validation`.
- If the file is deleted, all imports and tests are updated.
- If retained, its purpose is no longer tied to `/mcp install`.

## Source Context
`installer_validation.py` defines `validate_server_name()`, `name_to_module()`, and `name_to_class()`, which are used by the installer workflow for server name validation and scaffold naming.
