# Add migration note for `/ingest` removal

## Priority
Low

## Labels
docs, migration, rag, ingest, breaking-change

## Dependencies
- Update help text and documentation for `/ingest` removal

## Background
The `/ingest` command is removed without backward-compatible aliases. Users need a concise migration note.

## Task
Add a migration note explaining the removal of `/ingest`.

## Implementation Requirements
Add a short migration note containing:

Removed command:
- `/ingest <url|path> [lang] [--snippets-only]`

Reason:
- RAG ingestion is an operational pipeline concern, not an Agent REPL command.
- The Agent REPL should provide RAG search/debugging, not mutate the RAG DB through `/ingest`.

Replacement:
- Run the RAG ingestion pipeline outside the Agent REPL.
- Use `/rag search <query> [--debug]` for retrieval and debugging from the Agent.

## Acceptance Criteria
- Migration note clearly states `/ingest` is removed.
- No backward-compatible alias is documented.
- Replacement workflow is clear.

## Source Context
The removed command previously triggered crawl, split, and DB ingest from inside the Agent REPL.
