# Remove `/ingest` command registration from CommandRegistry

## Priority
High

## Labels
agent, commands, rag, ingest, cleanup, breaking-change

## Dependencies
None

## Background
The Agent currently registers `/ingest` as a built-in slash command. This command starts RAG ingestion from inside the Agent REPL. We want to remove this REPL mutation path while keeping `/rag search`, `/export`, and `/compact`.

## Task
Remove `/ingest` from the built-in command registry.

## Implementation Requirements
- Remove the `/ingest` `CommandDef` from `registry.py`.
- Remove the `/ingest` help description from the command list.
- Update the command group comment that currently describes `_IngestMixin` as handling `/ingest`.
- Do not remove `_IngestMixin` yet if it still owns `/export`, `/compact`, or `/rag`.

## Acceptance Criteria
- `/ingest https://example.com` is no longer dispatched.
- `/ingest /path/to/file` is no longer dispatched.
- `/ingest something --snippets-only` is no longer dispatched.
- `/rag search`, `/export`, and `/compact` remain registered.

## Source Context
`registry.py` currently registers `/ingest` as a prefix async command, while `/rag`, `/export`, and `/compact` are also registered separately.
