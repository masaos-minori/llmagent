## Title
[Medium] Document that Agent integration is intentionally not implemented yet

## Priority
Medium

## Labels
documentation, architecture, eventbus, agent

## Dependencies
None

## Background
Event Bus is designed as an internal event service for the broader agent system, but Agent runtime integration is intentionally not implemented at this time.

There should be no implementation work for Agent-side publisher, subscriber, or topic taxonomy in this change set.

## Scope
Update:
- Event Bus system overview documentation
- Known issues / future work documentation if applicable

## Required Changes
- Add a clear note:
  - Agent integration is intentionally deferred.
  - Event Bus currently operates as an independent internal HTTP event service.
  - No Agent-side publisher or subscriber is expected in the current implementation.
- Remove or avoid wording that implies Agent runtime is already integrated with Event Bus.
- Optionally add a “Future Integration” section without implementation tasks.

## Acceptance Criteria
- Documentation accurately reflects current scope.
- Future implementers do not assume Agent/Event Bus runtime integration already exists.
- No Agent-side implementation is added as part of this issue.
