## Title
[Medium] Document Event Bus public bind guard behavior

## Priority
Medium

## Labels
documentation, security, configuration, eventbus

## Dependencies
None

## Background
Event Bus has no authentication or ACL. The configuration therefore includes a public bind guard.

The implementation rejects wildcard/public binds unless `allow_public_bind=true`. Tests cover loopback, private IPs, wildcard addresses, and hostname behavior.

## Scope
Update:
- `docs/06_eventbus_05_configuration_deploy_and_operations.md`
- `docs/06_eventbus_06_reference_api.md`
- Config examples if present

## Required Changes
- Document `host`.
- Document `allow_public_bind`.
- Document behavior for:
  - `127.0.0.1`
  - `::1`
  - private IPs
  - `0.0.0.0`
  - `::`
  - hostnames
- Explain that hostnames are treated as public unless explicitly allowed.
- Explain that using public bind without authentication is unsafe.

## Acceptance Criteria
- Configuration documentation matches `EventBusConfig`.
- Operators understand how to configure safe binding.
- Public exposure requires explicit opt-in.
