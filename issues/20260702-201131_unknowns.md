## UNK-01: Confirm canonical timestamp format for new DBs

**Labels:** question, database, schema

**Description:**

The requirement (`requires/20260702_48_require.md`) lists two candidate formats but does not make a decision:

- Option A: `YYYY-MM-DDTHH:MM:SSZ` (Z suffix) — already used in all RAG/Session DDL defaults and in the Event Bus DDL
- Option B: Python ISO format with `+00:00` — currently produced by `datetime.now(UTC).isoformat()` in `state_store._now()`

**Impact:**

- If Option A (Z suffix): `schema_sql.py` workflow DDL needs DEFAULT clauses added; `state_store._now()` needs to use `strftime('%Y-%m-%dT%H:%M:%SZ')` instead of `isoformat()`.
- If Option B (`+00:00`): No code changes needed; documentation update only.

**Question for user:**

Which format should be the canonical standard for newly recreated DBs?
- [ ] Option A: `YYYY-MM-DDTHH:MM:SSZ` (Z suffix, consistent with RAG/Session/EventBus)
- [ ] Option B: Python ISO format `+00:00` (consistent with current workflow runtime behavior)

**Resolution path:** User decision. Once confirmed, update `docs/90_shared_04_db_architecture_and_schema.md` §7a and (if Option A) update `schema_sql.py` + `state_store.py`.

**Blocking:** Yes — implementation steps in Phase 2 depend on this decision.
