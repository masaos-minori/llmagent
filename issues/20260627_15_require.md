# [orchestrator][scope] Freeze the Phase 1 scope

## Summary
Phase 1 needs a clear scope freeze.

## Problem
The plan includes many tools and roles.
Without a clear scope, implementation can grow too large.

## Scope
- Phase 1 roadmap
- MUST / SHOULD / LATER decision

## Task
- define Phase 1 MUST items
- define Phase 1 SHOULD items
- define Phase 2 or later items

## Phase 1 Scope (from spec section 4)

### MUST
- `orchestrator.py` ‚Äî workflow orchestration core
- `workflow_engine.py` ‚Äî phase state machine
- `task_scheduler.py` ‚Äî DAG-based task scheduling with parallel execution
- `retry_controller.py` ‚Äî retry policy per role/error class
- `state_manager.py` ‚Äî workflow/task/artifact/retry/approval persistence
- `approval_gate.py` ‚Äî GitHub write approval gating
- `policy_engine.py` ‚Äî repository/branch/path/action constraints
- `repository_gateway.py` ‚Äî unified repository I/O abstraction
- `mcp_registry.py` ‚Äî MCP endpoint registration and health monitoring
- `audit_logger.py` ‚Äî approval/retry/write audit logging
- `orchestrator_barrier.py` ‚Äî phase barrier synchronization
- `planner_mcp_server.py` ‚Äî task graph generation
- `retriever_mcp_server.py` ‚Äî RAG execution and evidence collection
- `patch_worker_mcp_server.py` ‚Äî file-level patch generation with worktree isolation
- `integrator_mcp_server.py` ‚Äî patch_set integration
- `validator_mcp_server.py` ‚Äî syntax/lint/unit test validation
- `publisher_mcp_server.py` ‚Äî branch/commit/push/Draft PR publishing
- `worker_runtime.py` ‚Äî reusable worker runtime
- `worker_launcher.py` ‚Äî worker process management
- `worktree_manager.py` ‚Äî worktree lifecycle management
- `orchestrator_models.py` ‚Äî DTO definitions
- `workflow.sqlite` schema (reuse existing)
- MCP tool definitions: `orchestrate_task`, `get_run_status`, `get_run_artifacts`, `retry_phase`, `abort_run`, `list_workers`

### SHOULD
- OpenRC service integration (`init.d/orchestrator-mcp`)
- Watchdog monitoring integration
- `/mcp` status display for orchestrator-mcp
- Worker log file output (`logs/worker_<id>.log`)
- Artifact storage in `artifacts/<workflow_id>/`

### LATER (Phase 2+)
- `Reviewer` role MCP server
- `Researcher` role MCP server
- `Memory Manager` role MCP server
- External artifact store separation from DB inline
- Advanced retry backoff strategies
- Dynamic concurrency limits per role

The team knows exactly what is required for the first working release.
