## [validator][execution] Define the Validator execution contract

### Summary

Validator role は patch_set 全体に対する syntax / lint / unit test 実行結果を、orchestrator が機械的に扱える形で返さなければならない。

### Problem

validation 結果が自由形式だと、retry_controller が deterministic / transient を判断できず、publish gate も誤動作する。

### Scope
- validator runtime design
- execution path
- result format
- retry_hint contract

### Task
- Validator の実行経路を定義する
- result format を定義する
- retry_hint format を定義する
- retry_controller との責務境界を明記する

### Validation Execution Path
- Validator は shell-mcp を用いて syntax / lint / unit test を実行する
- 実行対象は integrated patch_set であり、file 単位ではない
- 専用の repository write capability は持たない

### Result Format
{
  "syntax": { "status": "pass|fail", "details": [] },
  "lint": { "status": "pass|fail", "details": [] },
  "unit_test": { "status": "pass|fail", "details": [] },
  "failed_cases": [
    {
      "test_name": "...",
      "error": "...",
      "traceback": "..."
    }
  ],
  "logs": [],
  "summary": "...",
  "retry_hint": {
    "error_class": "transient|deterministic|fatal",
    "suggested_action": "...",
    "retry_allowed": true
  }
}

### Responsibility Split
- Validator: 実行・観測・構造化結果生成
- retry_controller: retry_hint とポリシーに基づく最終 retry 判定
- orchestrator: phase 遷移 / abort / publish gate 判定

### Expected Result

orchestrator は validator results を機械的に解釈し、retry / abort / continue を安全に決定できる。

### Acceptance Criteria
- validator execution path が定義されている
- validator result format が定義されている
- retry_hint format が定義されている
- retry_controller との責務境界が定義されている
