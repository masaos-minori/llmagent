# テストエラー調査報告 (2026-06-21)

## エラー概要

```
tests/test_route_resolver.py::TestStaticFallbackRouting::test_read_tools
ERROR at setup of TestStaticFallbackRouting.test_read_tools

ValueError: '' is not a valid StartupMode
```

## 原因

`scripts/shared/mcp_config.py:68` の `McpServerConfig.__post_init__()` で、`startup_mode` が空文字列の場合に Enum 変換が失敗する。

```python
def __post_init__(self) -> None:
    if not isinstance(self.startup_mode, StartupMode):
        self.startup_mode = StartupMode(self.startup_mode)  # ← ここで ValueError
```

テストの `_http()` helper が空文字列を渡しているのが原因:

```python
def _http(key: str, url: str = "http://127.0.0.1:8000") -> McpServerConfig:
    return McpServerConfig("http", url, [], "")  # ← startup_mode="" が渡される
```

## 既存設定ファイルとの比較

実設定ファイル `config/mcp_servers.toml` では正しく `"startup_mode = "persistent"` が設定されており、本番環境では問題が発生しない。

## 改善案

### 案1: テスト helper を修正する（推奨）

テストの `_http()` helper で明示的にデフォルト値を渡す:

```python
def _http(key: str, url: str = "http://127.0.0.1:8000") -> McpServerConfig:
    return McpServerConfig("http", url, [], StartupMode.PERSISTENT)
```

**メリット**: テストが本番の設定に近い状態になり、テストの信頼性が向上する。

### 案2: `__post_init__` に空文字列のフォールバックを追加する

```python
def __post_init__(self) -> None:
    if not isinstance(self.transport, TransportType):
        self.transport = TransportType(self.transport)
    if not isinstance(self.startup_mode, StartupMode):
        self.startup_mode = StartupMode(self.startup_mode) if self.startup_mode else StartupMode.PERSISTENT
    # ...
```

**メリット**: 設定ファイルの値が欠落した場合でもデフォルト値が適用される。

**デメリット**: エラーになることで「設定の欠落に気づける」というメリットを失う。

### 案3: テストで明示的に startup_mode を指定する

テストメソッド内で各サーバーの設定を明示:

```python
def setup_method(self) -> None:
    configs = {
        "file_read": McpServerConfig("http", "http://127.0.0.1:8000", [], StartupMode.PERSISTENT),
        # ...
    }
```

**メリット**: テストが明示的になる。

**デメリット**: コード量が増える。

## 推奨案

**案1** を推奨する。テスト helper の修正は最小限で、本番設定に近い状態をテストできる。
