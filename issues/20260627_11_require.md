## [mcp-registry][tools] Define how role MCP servers fit into /v1/tools and startup validation

### Summary

orchestrator 自体は MCP server ではないが、Planner / Retriever / PatchWorker / Integrator / Validator / Publisher の role MCP servers は既存の capability verification flow に適合しなければならない。

### Problem

role MCP servers の /v1/tools 形式や startup validation が不統一だと、mcp_registry.py による health / capability 管理や起動時検証が失敗する。

### Scope
- role MCP tool metadata
- tool definitions config format
- startup compatibility rules

### Task
- role MCP servers の /v1/tools output format を定義する
- matching tool definition format を定義する
- mcp_registry.py が行う startup validation を定義する
- persistent / managed startup rules を定義する

### /v1/tools Output Format
{
  "tools": [
    {
      "name": "<tool_name>",
      "description": "...",
      "input_schema": { ... }
    }
  ]
}

### Required Server Classes
- planner-mcp
- retriever-mcp
- patchworker-mcp
- integrator-mcp
- validator-mcp
- publisher-mcp

### Tool Definition Rules
- 各 tool は name / description / input_schema を持つ
- mcp_registry.py が config 上の定義と /v1/tools 応答を照合する
- capability mismatch は startup warning または hard fail として扱う

### Startup Compatibility Rules
- role MCP servers は health endpoint と /v1/tools を提供する
- startup_mode は空値にしない
- managed startup / persistent monitoring の運用方針を config で定義する
- orchestrator は Agent 内部コンポーネントであり、この検証対象には含めない

### Expected Result

role MCP servers は既存の MCP startup validation flow と整合し、mcp_registry.py から一貫して扱える。

### Acceptance Criteria
- /v1/tools format が定義されている
- matching tool definitions が定義されている
- startup validation rules が定義されている
- orchestrator を検証対象から除外する方針が明記されている
