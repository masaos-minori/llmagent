# [orchestrator-mcp][ops] Define how orchestrator-mcp is added to /mcp, OpenRC, and watchdog

## Summary
The new orchestrator server must fit into the current MCP operations model.

## Problem
We need a consistent way to:
- start it
- monitor it
- show it in `/mcp`
- restart it with watchdog/OpenRC

## Scope
- MCP server config
- service integration
- operational visibility

## Task
- define the service name
- define the MCP server config entry
- define `/mcp` status behavior
- define watchdog and restart expectations

## Expected Result
The new server works like other MCP services in operations.

## Acceptance Criteria
- [ ] service naming is defined
- [ ] MCP config entry shape is defined
- [ ] `/mcp` integration behavior is defined
- [ ] watchdog/OpenRC expectations are defined
