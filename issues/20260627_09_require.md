## [orchestrator][storage] Decide whether to reuse workflow.sqlite or add orchestrator.sqlite

### Summary

Phase 1 の workflow metadata / task state / artifact metadata / retry / approval は、既存 workflow.sqlite を再利用して一元管理する。

### Problem

workflow.sqlite と orchestrator.sqlite の二重 DB 構成は、状態分散・運用複雑化・整合性確認コストを増やす。特に approval / audit / MCP health を跨ぐ横断参照が難しくなる。

### Scope
- DB reuse vs new DB analysis
- artifact storage strategy
- state_manager persistence API direction

### Task
- workflow.sqlite 再利用と新規 DB 追加を比較する
- Phase 1 の推奨方針を決定する
- artifact inline storage の方針を定義する
- state_manager の最低限 API を定義する

### Storage Decision
- **Recommended**: workflow.sqlite を再利用する
- 既存 logical tables を拡張し、workflow / task / dependencies / artifacts / retries / approvals / events / MCP metadata を一元管理する
- orchestrator 専用の別 DB は Phase 1 では追加しない

### Artifact Storage Strategy (Phase 1)
- DB inline 対象:
  - diff
  - patch_set
  - validator logs（要サイズ制限）
  - evidence bundle summary
  - Draft PR payload
- file system 併用対象:
  - 大きなログ
  - worker raw outputs
  - long evidence dumps

### Risk Mitigation
- artifact size limits を設ける
- large fields の truncation / compression policy を定義する
- 将来の external artifact store への移行余地を残す

### Minimum state_manager APIs
- create_workflow
- get_workflow
- update_workflow_state
- register_task
- update_task_state
- register_artifact
- append_event
- record_retry
- record_approval
- list_artifacts

### Policy Alignment Notes

This requirement must follow the Phase 1 integration policy below.

1. **Do not create `orchestrator-mcp`.**
   - The central Orchestrator is an internal Agent component, not an MCP server.
   - Role MCP services may be added and monitored via MCP health/capability checks, but the Orchestrator itself must not be registered as an MCP server.
2. **Add internal Orchestrator APIs.**
   - REPL / command layers should call internal APIs such as `start_workflow`, `resume_workflow`, `cancel_workflow`, `get_workflow_status`, `get_workflow_artifacts`, and `retry_phase`.
3. **Use `state_manager.py` as the source of truth.**
   - Workflow / task / artifact / retry / approval state must be persisted and read through `state_manager.py`.
   - REPL, `AgentContext`, WorkerRuntime, JSONL logs, and UI code must not own canonical workflow state.
4. **Extend `workflow.sqlite`.**
   - Phase 1 must reuse and extend `workflow.sqlite` for workflow metadata, task state, artifact metadata references, retries, approvals, and events.
   - Do not introduce a separate `orchestrator.sqlite` in Phase 1.

### Expected Result

プロジェクトは一つの明確な orchestration metadata storage plan を持ち、state_manager を通じて一貫した永続化を行える。

### Acceptance Criteria
- reuse vs new DB decision が定義されている
- decision reasons が記述されている
- artifact storage strategy が定義されている
- state_manager minimum APIs が定義されている
