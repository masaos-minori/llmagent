# [Issue] Audit log format: mcp/audit.py uses key=value, policy says JSON-lines

## Section: UNK-03 from 20260701_41 plan

## Problem

`requires/20260701_41_require.md` の受け入れ基準に「All audit logs are JSON-lines」とある。
しかし、現行の `mcp/audit.py` の実装は key=value フォーマット（`AUDIT session=... request=... action=...`）を使用している。

```python
# scripts/mcp/audit.py (現行)
server_logger.info(
    f"AUDIT session={session_id or '-'} request={request_id or '-'} "
    f"action={action} target={target} outcome={outcome} detail={detail} "
    f"server_key={server_key} error_type={error_type}"
)
```

このまま `docs/04_mcp_02_protocol_and_transport.md` と `docs/04_mcp_06_configuration_and_operations.md` の audit log フォーマット記述を JSON-lines に書き換えると、実装とドキュメントが乖離する。

## Required Decision

以下のいずれかを決定する必要がある:

1. **`mcp/audit.py` を JSON-lines フォーマットに変更する** — コード変更を伴う新規計画が必要。
   ```python
   # 変更後イメージ
   server_logger.info(json.dumps({
       "event": "AUDIT",
       "session_id": session_id or "-",
       "request_id": request_id or "-",
       "action": action,
       "target": target,
       "outcome": outcome,
       "detail": detail,
       "server_key": server_key,
       "error_type": error_type,
   }))
   ```
   - 影響: `mcp/audit.py`, 全 MCP サーバーのディスパッチハンドラ, 既存ログパーサ/grep パターン。

2. **ドキュメントの文言を「JSON-structured audit log line」に変更する** — JSON-lines ではなく「構造化ログの1行」として記述し、フォーマット例は現行の key=value を維持する。

## Acceptance Criteria

- `mcp/audit.py` の実装とドキュメント記述が一致している。
- `AUDIT session=` の key=value フォーマットが docs に残る場合、`check_no_compat.py` の禁止パターン追加対象から除外することを明記する。
