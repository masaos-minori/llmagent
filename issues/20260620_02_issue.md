## Event Bus の導入・永続化・Replay・DLQ

- 改善案: Event Bus の導入・永続化・Replay・DLQ
- 実装手順概要:
  - HTTP publish + SSE subscribe + append-only log + SQLite を採用する。
  - event envelope、events テーブル、since_seq による Replay、DLQ を定義する。
  - 各ホストに eventbus-node を配置し replication を実装する。
  - at-least-once、event_id 冪等性、lease / retry / stale cleanup を整備する。
- 実装対象:
  - eventbus/app.py
  - eventbus/schema.sql
  - schemas/event_envelope.json
  - storage/events.jsonl
  - offsets/
  - deadletter/
