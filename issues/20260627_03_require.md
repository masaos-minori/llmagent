# [worker-runtime][roles] Define role presets for Planner / PatchWorker / Validator / Integrator

## Summary
Each worker role needs a clear preset.

## Problem
The system needs role-specific behavior for:
- prompt
- allowed tools
- denied tools
- LLM endpoint
- input and output rules

## Scope
- role configuration design
- worker runtime role presets

## Task
- define Planner preset
- define PatchWorker preset
- define Validator preset
- define Integrator preset
- define Publisher preset

## Planner Preset

- **MCP server**: `planner_mcp_server.py`
- **Transport**: HTTP (`http://127.0.0.1:8009`)
- **Input**: user request, session context, repository context
- **Output**: goal, change_sets, tasks, dependencies, acceptance_criteria
- **Tool policy**: read-only tools; no write operations
- **Design constraints**: task graph design and dependency definition only

## PatchWorker Preset

- **MCP server**: `patch_worker_mcp_server.py`
- **Transport**: HTTP (`http://127.0.0.1:8009`)
- **Input**: file task, evidence/context, repository target, constraints
- **Output**: file patch, diff, rationale, changed file summary
- **Tool policy**: read tools + edit tools scoped to worktree; no direct github-mcp/file-mcp access
- **Design constraints**: isolated per worktree; write execution not included; file-level isolation

## Validator Preset

- **MCP server**: `validator_mcp_server.py`
- **Transport**: HTTP (`http://127.0.0.1:8009`)
- **Input**: `patch_set`, repository state, scope
- **Output**: syntax result, lint result, unit test result, failed case list, logs, summary
- **Tool policy**: shell-mcp for execution; read tools for analysis
- **Design constraints**: validates `patch_set` as a whole; not file-level validation

## Integrator Preset

- **MCP server**: `integrator_mcp_server.py`
- **Transport**: HTTP (`http://127.0.0.1:8009`)
- **Input**: file patch group, repository context, validation prerequisites
- **Output**: `patch_set`, integrated diff, validation artifact, integration summary
- **Tool policy**: read tools only; no write operations
- **Design constraints**: limited to `patch_set` integration; no Git operations

## Publisher Preset

- **MCP server**: `publisher_mcp_server.py`
- **Transport**: HTTP (`http://127.0.0.1:8009`)
- **Branch creation**: Creates a new branch from the approved base
- **Commit**: Generates commit based on validated `patch_set`
- **Push**: Pushes the branch to remote repository
- **Draft PR**: Creates a Draft PR with summary
- **Final summary**: Produces final summary of all changes
- **Input**: validation-verified `patch_set`, repository context, approval/policy state
- **Output**: branch name, commit plan, push result, Draft PR payload, final summary
- **Tool policy**: write API via `repository_gateway.py` only; no direct github-mcp/file-mcp access
- **Design constraints**: limited to publishing responsibility; only handles approved artifacts; no merge responsibility

## WorkerRuntime Concept

WorkerRuntime is a reusable runtime that launches role-specific workers without copying the full agent implementation.

### Shared parts across all roles
- MCP HTTP transport (`http://127.0.0.1:8009`)
- Health check via `GET /health` and capability discovery via `GET /v1/tools`
- Tool execution via `POST /v1/call_tool`
- State persistence to `workflow.sqlite`
- Event logging to JSONL
- Retry controller integration
- Barrier synchronization
- Audit logging

### Role-specific parts
- Prompt configuration
- Allowed/denied tool lists
- LLM endpoint configuration
- Input/output schema
- Worktree isolation (PatchWorker only)
- Concurrency limits per role

## Expected Result
The orchestrator can launch and manage all worker roles.
