# [orchestrator-mcp][response] Define structured MCP response handling for orchestration data

## Summary
The orchestration server needs more structured output than normal MCP tools.

## Problem
The current MCP pattern is mostly:
- `result`
- `is_error`

But orchestration needs:
- text summary
- structured run data
- metadata

## Scope
- orchestrator response format
- client-side result handling

## Task
- define how `result_text` works
- define how `result_data` works
- define how `meta` works
- define compatibility with the current MCP result model

## Expected Result
The agent can read human-friendly text and machine-friendly data from orchestration tools.

## Acceptance Criteria
- [ ] structured response fields are defined
- [ ] compatibility rules are defined
- [ ] client-side handling strategy is defined
