# Unknowns — Plan 20260702-201040 (ChunkSplitter-to-DB field mapping)

## UNK-02: Should input `fetched_at` be persisted to `documents.fetched_at`?

**Context:**

`ChunkJsonRaw.fetched_at` is read by `ingester.py::ingest_url_group()` (line 181) and passed to `_get_or_create_document()` (line 193), which in turn passes it to `ETagManager.update()` on the skip path. However, the INSERT statement at line 436–439 of `ingester.py` does **not** include `fetched_at` in the column list:

```python
cur = db.execute(
    "INSERT INTO documents"
    " (url, title, lang, etag, last_modified, chunking_strategy)"
    " VALUES (?, ?, ?, ?, ?, ?)",
    (url, title, lang, etag, last_modified, chunking_strategy),
)
```

The `documents.fetched_at` column has a DEFAULT expression `strftime('%Y-%m-%dT%H:%M:%SZ', 'now')`, so it is always set to the current server time at INSERT.

**Question:**

Should the input `fetched_at` value from `ChunkJsonRaw` (the crawl timestamp from the crawler) be explicitly written to `documents.fetched_at`, or should the DEFAULT (INSERT time) be used?

- **Option A: Use DEFAULT** — `fetched_at` = time of ingestion (current behavior). The field in `ChunkJsonRaw` is informational only and should be documented as "not persisted on INSERT; only propagated via ETagManager on skip".
- **Option B: Persist input value** — Add `fetched_at` to the INSERT column list in `ingester.py`. The crawler timestamp would be preserved. Requires confirming that `ETagManager.update()` on the skip path already handles this correctly.

**Impact if unresolved:** The documentation will be ambiguous about `fetched_at` behavior. If Option B is correct, there is also a latent bug in `ingester.py`.
