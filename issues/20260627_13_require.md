## [orchestrator][response] Define structured internal result DTO handling for workflow data

### Summary

MCP 互換の result/is_error 形式ではなく、内部 Orchestrator の workflow 実行結果を表現する structured DTO を定義する。

### Problem

workflow 実行では、human-friendly summary だけでなく、phase / artifact / approval / retry の machine-readable data が必要である。MCP result 形式に寄せると内部 API 契約が不明瞭になる。

### Scope
- internal response DTO format
- REPL / command side result handling
- structured workflow result fields

### Task
- workflow_result DTO を定義する
- workflow_status DTO を定義する
- artifact reference DTO を定義する
- UI 向け summary と machine-readable data の関係を定義する

### workflow_result DTO
- workflow_id
- workflow_state
- current_phase
- summary
- final_summary
- artifact_refs
- approval_state
- retry_state
- next_action_hint
- emitted_at

### workflow_status DTO
- workflow_id
- workflow_state
- current_phase
- running_tasks
- blocked_tasks
- last_event
- updated_at

### artifact_ref DTO
- artifact_id
- artifact_type
- content_hash
- storage_kind
- location
- size_bytes

### Handling Rules
- REPL / command は summary を表示用に使う
- structured fields は status / retry / artifact 表示や command 連携に使う
- DTO は state_manager と整合する field 命名を採用する
- backward compatibility のために MCP result 形式へ寄せない

### Expected Result

Agent 内部で human-friendly 表示と machine-friendly 処理を両立しつつ、内部 API 契約を明確化できる。

### Acceptance Criteria
- workflow_result DTO が定義されている
- workflow_status DTO が定義されている
- artifact_ref DTO が定義されている
- UI 表示用 summary と structured data の関係が定義されている
