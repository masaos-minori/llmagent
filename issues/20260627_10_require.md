## [worker-runtime][tools] Define role-specific tool policies

### Summary

各 role には責務に応じた allow / deny / approval boundary を持つ tool policy が必要である。

### Problem

tool policy が曖昧だと、PatchWorker や Integrator が publish 経路を迂回し、approval / policy / audit をすり抜ける危険がある。

### Scope
- role-based tool policy design
- allowlist / denylist
- approval boundary and repository write control

### Task
- Planner の tool policy を定義する
- Retriever の tool policy を定義する
- PatchWorker の tool policy を定義する
- Integrator の tool policy を定義する
- Validator の tool policy を定義する
- Publisher の tool policy を定義する

### Role Tool Policies

#### Planner
- **Allowed**: read/search/context summarization tools
- **Denied**: repository write, publish actions, destructive shell actions
- **Approval boundary**: not applicable

#### Retriever
- **Allowed**: retrieval/search/read tools, metadata extractors
- **Denied**: repository write, publish actions, destructive shell actions
- **Approval boundary**: not applicable

#### PatchWorker
- **Allowed**: read tools, worktree-scoped change proposal tools, diff helpers
- **Denied**: repository write API, direct github-mcp, direct unrestricted file-mcp, publish actions
- **Approval boundary**: not applicable

#### Integrator
- **Allowed**: read tools, merge/consistency analysis tools
- **Denied**: repository write API, git push, PR creation, direct unrestricted github/file writes
- **Approval boundary**: not applicable

#### Validator
- **Allowed**: shell-mcp for syntax/lint/unit test execution, read tools
- **Denied**: repository write, publish actions
- **Approval boundary**: not applicable

#### Publisher
- **Allowed**: repository_gateway.py 経由の write API のみ
- **Denied**: direct github-mcp, direct unrestricted file-mcp, approval bypass operations
- **Approval boundary**: approval-passed patch_set のみ対象、merge responsibility なし
- **Audit logging**: publish operations は必ず audit_logger に記録する

### Common Rules
- deny by default を基本とする
- role policy は config-driven とする
- write capability は Publisher のみに付与する
- policy_engine / approval_gate を通らない publish 系操作は禁止する

### Expected Result

各 worker role は安全で明確な tool policy を持ち、approval / policy / audit 境界を破らない。

### Acceptance Criteria
- Planner tool policy が定義されている
- Retriever tool policy が定義されている
- PatchWorker tool policy が定義されている
- Integrator tool policy が定義されている
- Validator tool policy が定義されている
- Publisher tool policy が定義されている
- Publisher の write 境界が明記されている
