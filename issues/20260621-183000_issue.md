## datetime.datetime.UTC 非互換エラーの調査

### エラー概要

Python 3.14で`datetime.datetime.UTC`が非推奨となり、`datetime.timezone.utc`に置き換えが必要。これにより24件のテストが失敗していた。

### エラー原因

```python
# NG - Python 3.14で削除
now = datetime.datetime.now(datetime.UTC)

# OK - 互換性のある書き方
from datetime import timezone
now = datetime.datetime.now(timezone.utc)
```

### 影響を受けたファイル

| ファイル | 行数 | 修正内容 |
|----------|------|----------|
| scripts/agent/memory/ingestion.py | 173 | `datetime.UTC` → `datetime.timezone.utc` |
| scripts/agent/memory/extract.py | 259 | `datetime.UTC` → `timezone.utc` (from import) |
| scripts/agent/memory/retriever.py | 58 | `datetime.UTC` → `datetime.timezone.utc` |
| scripts/agent/memory/mapper.py | 23 | `datetime.UTC` → `datetime.timezone.utc` |

### テスト結果

- 修正前: 24件失敗（test_memory_layer.py, test_memory_extract.py）
- 修正後: 2551件パス、12件失敗（test_cmd_mcp.py - 既存のテスト分離問題、datetime.UTCとは無関係）

### 改善提案

1. `scripts/agent/memory/` 配下の全ファイルで `datetime.timezone.utc` に統一
2. `scripts/agent/memory/mapper.py` の `_now_iso()` helper関数を `scripts/shared/` に移し、全モジュールから共有
3. Python 3.14対応のCIチェックを追加（`datetime.UTC`の使用を検知）
