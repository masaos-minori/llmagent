# [worker-runtime][extensibility] Define extension points for future roles

## Summary
Future roles are planned for Phase 2 and later.

## Problem
If role handling is hardcoded for only 4 roles, later extension will be harder.

## Scope
- role registry design
- future role extension design

## Task
- define how new roles are added
- define how role configuration is extended
- define where future role policies should live

## Expected Result
The Phase 1 design stays extensible.

## Acceptance Criteria
- [ ] role extension rule is defined
- [ ] new role config path is defined
- [ ] current design does not hardcode only 4 roles
