# [orchestrator][state] Define the run-oriented state model

## Summary
The current system has session-level and turn-level state.
The multi-agent design needs run-level state.

## Problem
We need a new state model for:
- run_id
- phase
- task graph
- worker assignments
- barrier state
- retry counts
- artifacts

## Scope
- new orchestrator state design
- DB and JSONL persistence design

## Task
- define the run-level state structure
- define artifact-level state
- define knowledge-level state
- decide what is stored in SQLite
- decide what is stored in JSONL logs

## Expected Result
The system can track one multi-agent run as one logical unit.

## Acceptance Criteria
- [ ] run-level state fields are defined
- [ ] phase state is defined
- [ ] worker assignment state is defined
- [ ] artifact state is defined
- [ ] persistence strategy is defined
