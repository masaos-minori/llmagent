# Issue: resolve_safe() が null byte パスの ValueError をキャッチしない

## Bug Report

**File:** `scripts/mcp/file/common.py`
**Function:** `resolve_safe()`
**Severity:** Medium

## 問題

`resolve_safe()` は `except OSError` のみでエラーをキャッチするが、
Python 3.13 では null byte を含むパス (`"/tmp/file\x00.txt"`) に対して
`Path.resolve()` が `OSError` ではなく `ValueError` を投げる。
このため `ValueError` がそのまま呼び出し元に伝播し、
`FileValidationError` に変換されない。

## 再現方法

```python
from mcp.file.common import resolve_safe, FileValidationError
from pathlib import Path

# Python 3.13 では ValueError が発生する (OSError ではない)
try:
    resolve_safe("/tmp/file\x00.txt", [Path("/tmp")])
except FileValidationError:
    print("OK: FileValidationError")
except ValueError as e:
    print("BUG: ValueError leaked:", e)  # ← こちらが実行される
```

## 期待動作

null byte パスは `FileValidationError("Invalid path")` として報告されるべき。

## 修正案

```python
def resolve_safe(raw_path: str, allowed_dirs: list[Path]) -> Path:
    try:
        resolved = Path(raw_path).resolve()
    except (OSError, ValueError):          # ← ValueError を追加
        raise FileValidationError("Invalid path")
    ...
```
