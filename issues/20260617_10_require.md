# [worker-runtime][tools] Define role-specific tool policies

## Summary
Each role needs a different tool policy.

## Problem
The current agent has tool control features, but the new system needs role-based tool control.

## Scope
- role tool policy design
- approval and allowlist reuse

## Task
- define allowed tools per role
- define denied tools per role
- define whether approval rules still apply per role
- define plan-mode interaction if needed

## Expected Result
Each worker role has a safe and clear tool policy.

- [ ] PatchWorker tool policy is defined
- [ ] Validator tool policy is defined
