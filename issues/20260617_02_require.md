# [worker-runtime][design] Design a reusable WorkerRuntime

## Summary
The current agent is built for one interactive REPL.
The new system needs reusable worker runtimes.

## Problem
We need one shared runtime that can run different roles:
- Planner
- PatchWorker
- Validator
- Integrator

## Scope
- worker runtime design
- role-specific runtime settings

## Task
- define the WorkerRuntime concept
- define which parts are shared across roles
- define which parts are different per role
- separate UI-specific code from runtime logic

## Expected Result
The system can launch role-specific workers without copying the full agent implementation.

## Acceptance Criteria
- [ ] shared worker runtime design is defined
- [ ] role-specific settings are defined
- [ ] UI-dependent parts are separated from runtime logic
