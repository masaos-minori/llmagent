# [orchestrator][barrier] Define barrier rules between phases

## Summary
The orchestrator needs barrier logic to control when a phase can continue.

## Problem
Some phases must wait for other workers to finish first.

## Scope
- barrier logic design
- worker synchronization rules

## Task
- define the plan barrier
- define the implement barrier
- define the integrate barrier
- define what data is needed to open each barrier

## Expected Result
The orchestrator can block or release phase transitions safely.

## Acceptance Criteria
- [ ] planner completion barrier is defined
- [ ] patchworker completion barrier is defined
- [ ] integrator start condition is defined
- [ ] barrier state is part of run state
