# [validator][execution] Define the Validator execution contract

## Summary
The Validator role needs a clear execution and result contract.

## Problem
Validation must produce machine-readable results for the orchestrator.

## Scope
- validator runtime design
- command execution path
- validation result format

## Task
- define how Validator runs tests and checks
- define the result format:
  - pass/fail
  - summary
  - logs
  - retry hint
- define whether shell-mcp is enough or a dedicated execution path is needed

## Expected Result
The orchestrator can understand validator results and react automatically.

## Acceptance Criteria
- [ ] validator execution path is defined
- [ ] validator result format is defined
- [ ] retry hint format is defined
