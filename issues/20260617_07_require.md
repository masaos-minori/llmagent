# [orchestrator][retry] Define retry and abort rules

## Summary
The orchestration flow needs clear retry and abort behavior.

## Problem
When validation fails or a phase gets stuck, the system must know what to do next.

## Scope
- retry policy
- abort policy
- stuck handling

## Task
- define when a phase can be retried
- define max retry rules
- define abort conditions
- define cleanup behavior after abort

## Expected Result
The orchestrator can recover or stop safely.

## Acceptance Criteria
- [ ] retry rules are defined
- [ ] abort rules are defined
- [ ] stuck detection rules are defined
