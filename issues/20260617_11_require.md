# [orchestrator-mcp][tools] Define how orchestrator-mcp fits into /v1/tools and tool definitions

## Summary
The new MCP server must match the current tool-definition verification flow.

## Problem
The current agent checks `/v1/tools` against configured tool definitions.
The new server must follow that rule.

## Scope
- orchestrator MCP tool metadata
- tool definitions config
- startup tool check compatibility

## Task
- define the `/v1/tools` output format for orchestrator-mcp
- define the matching tool definitions format
- make sure startup tool checks can pass

## `/v1/tools` Output Format (from spec section 3.4)

### Tool list endpoint
```json
{
  "tools": [
    {
      "name": "orchestrate_task",
      "description": "Execute an orchestration task",
      "input_schema": { ... }
    },
    {
      "name": "get_run_status",
      "description": "Get current workflow run status",
      "input_schema": { ... }
    },
    {
      "name": "get_run_artifacts",
      "description": "Get artifacts for a workflow run",
      "input_schema": { ... }
    },
    {
      "name": "retry_phase",
      "description": "Retry a failed phase",
      "input_schema": { ... }
    },
    {
      "name": "abort_run",
      "description": "Abort the current workflow run",
      "input_schema": { ... }
    },
    {
      "name": "list_workers",
      "description": "List active workers for this workflow",
      "input_schema": { ... }
    }
  ]
}
```

### Tool definitions format
- Each tool must have `name`, `description`, `input_schema`
- Tools follow the same definition format as other MCP servers in the system
- Tool definitions are registered via `mcp_registry.py`

## Startup Compatibility (from spec section 3.6)

### Startup mode enforcement
- All orchestrator-mcp servers MUST use `startup_mode: persistent`
- Empty string or omitted `startup_mode` is rejected by `McpServerConfig.__post_init__()`
- This ensures the server survives single-session crashes

## Expected Result
The orchestrator server works with the existing MCP startup validation flow.

## Acceptance Criteria
- [ ] `/v1/tools` format is defined
- [ ] matching tool definitions are defined
- [ ] startup compatibility is verified
