# MCP Re-export Stub Cleanup

## Background

Two backward-compatibility re-export stub modules exist in the codebase.
Both are marked as stubs and have zero active importers.
This issue tracks their removal.

## Affected Stub Files

### 1. `scripts/mcp/github/service.py`

**Type:** Re-export stub for backward compatibility

**Current re-exports:**
```python
from mcp.github.service_dispatch import GitHubService  # noqa: F401
from mcp.github.service_init import _GITHUB_TOKEN, build_service  # noqa: F401
```

**Replacement import paths:**
```python
from mcp.github.service_business import GitHubService   # business methods
from mcp.github.service_dispatch import GitHubService   # with dispatch formatters
from mcp.github.service_init import build_service       # client init + singleton
from mcp.github.service_init import _GITHUB_TOKEN
```

**Current importers (scripts/ + tests/):** None — `rg "from mcp\.github\.service import" scripts/ tests/` returns zero matches.

**Test impact:** Zero — no test imports from `mcp.github.service` directly.

**check_no_compat.py:** Pattern `"_GITHUB_PREFIX": r"_GITHUB_PREFIX"` already bans a related GitHub compatibility remnant. Consider adding a pattern for `from mcp.github.service import` after removal.

---

### 2. `scripts/agent/config.py`

**Type:** Re-export stub for backward compatibility

**Current re-exports:** AgentConfig, LLMConfig, RAGConfig, build_agent_config, load_config, ConfigLoadError, and related symbols from:
- `agent.config_dataclasses`
- `agent.config_builders`

**Replacement import paths:**
```python
from agent.config_dataclasses import AgentConfig, LLMConfig, RAGConfig, ...
from agent.config_builders import build_agent_config, load_config, ConfigLoadError
```

**Current importers (scripts/ + tests/):** None — `rg "from agent\.config import" scripts/ tests/` returns zero matches (excluding config_dataclasses, config_builders, config_loader).

**Test impact:** Zero — no test imports from `agent.config` stub directly.

---

## Cleanup Plan

1. Confirm zero importers in `/opt/llm/` (production deployment) before deletion:
   ```bash
   grep -rn "from mcp.github.service import\|from agent.config import" /opt/llm/
   ```
2. Delete `scripts/mcp/github/service.py`.
3. Delete `scripts/agent/config.py`.
4. Add `check_no_compat.py` patterns to ban re-introduction:
   - `"mcp.github.service stub": r"from mcp\.github\.service import"`
   - `"agent.config stub": r"from agent\.config import"`
5. Run full test suite: `uv run pytest tests/ -x -q`.
6. Run `pre-commit run --all-files`.

## Acceptance Criteria

- `scripts/mcp/github/service.py` does not exist.
- `scripts/agent/config.py` does not exist.
- check_no_compat.py blocks re-introduction of both import paths.
- All tests pass.
