# 添付要件ファイルへの方針反映レポート

## 反映した共通方針

全添付ファイルに、以下の 4 方針を `Policy Alignment Notes` として反映する前提で整理した。

1. `orchestrator-mcp` は作らない
2. internal orchestrator API を追加する
3. `state_manager.py` を正本にする
4. `workflow.sqlite` を拡張する

---

# 共通追記ブロック

以下のブロックを、各ファイルの `#### Expected Result` の直前、または同等の設計方針セクションの末尾に追記する。

```markdown
## Policy Alignment Notes

This requirement must follow the Phase 1 integration policy below.

1. Do not create `orchestrator-mcp`.
   - The central Orchestrator is an internal Agent component, not an MCP server.
   - Role MCP services may be added and monitored via MCP health/capability checks, but the Orchestrator itself must not be registered as an MCP server.
2. Add internal Orchestrator APIs.
   - REPL / command layers should call internal APIs such as `start_workflow`, `resume_workflow`, `cancel_workflow`, `get_workflow_status`, `get_workflow_artifacts`, and `retry_phase`.
3. Use `state_manager.py` as the source of truth.
   - Workflow / task / artifact / retry / approval state must be persisted and read through `state_manager.py`.
   - REPL, `AgentContext`, WorkerRuntime, JSONL logs, and UI code must not own canonical workflow state.
4. Extend `workflow.sqlite`.
   - Phase 1 must reuse and extend `workflow.sqlite` for workflow metadata, task state, artifact metadata references, retries, approvals, and events.
   - Do not introduce a separate `orchestrator.sqlite` in Phase 1.

```

# ファイル別の反映箇所

## 20260627_01_require.md — orchestrator[state] Define the workflow-oriented state model

### 反映箇所

- `State Model Rules / Source of truth`
- `Persistence split`
- `Prohibited Patterns`

### 反映内容

- 既存の「workflow / task / artifact / retry / approval の正本は state_manager.py のみ」という方針を維持し、今回の方針 3 として明示的に強化する。
- `workflow.sqlite` が Phase 1 の正本 DB であり、state_manager.py 経由で読み書きすることを追記する。
- REPL / AgentContext / WorkerRuntime / UI / JSONL に canonical workflow state を持たせないことを再確認する。

## 20260627_02_require.md — worker-runtime[design] Design a reusable WorkerRuntime for role execution

### 反映箇所

- `Explicit non-responsibilities`
- `Integration Rules`

### 反映内容

- WorkerRuntime は internal orchestrator API を直接所有せず、orchestrator から呼び出される stateless executor であると明記する。
- 永続 state の読み書きは state_manager.py 経由のみとする既存方針を強化する。
- WorkerRuntime から workflow.sqlite を直接更新しない方針を追記する。

## 20260627_03_require.md — worker-runtime[roles] Define role presets

### 反映箇所

- `Common Preset Rules`
- 各 role preset の prohibited / responsibility

### 反映内容

- role は orchestrator-mcp ではなく internal orchestrator から起動される worker / role MCP として扱う。
- 各 role は state 正本を保持せず、state_manager.py にイベント・成果物・状態を登録する。
- role output は workflow.sqlite 上の task / artifact metadata に接続されることを追記する。

## 20260627_04_require.md — orchestrator[api] Define the minimum internal orchestrator API

### 反映箇所

- `Design Rules`
- `Required Internal API`

### 反映内容

- 「orchestrator-mcp は作らない」を最重要方針として明記する。
- `start_workflow`, `resume_workflow`, `cancel_workflow`, `get_workflow_status`, `get_workflow_artifacts`, `retry_phase` を internal API として固定する。
- すべての永続 state 読み書きは state_manager.py 経由とする。
- response DTO は workflow.sqlite / state_manager.py の正本と整合させる。

## 20260627_05_require.md — orchestrator[phase] Define the phase state machine

### 反映箇所

- `Workflow States`
- `Task States`
- `Invalid Transition Rules`

### 反映内容

- phase / task / workflow state の正本は state_manager.py 経由で workflow.sqlite に保存する。
- internal orchestrator API が state transition を開始し、REPL は直接 state を変更しない。
- terminal state からの復帰は internal API の `resume_workflow` / `retry_phase` のみとする。

## 20260627_06_require.md — orchestrator[barrier] Define barrier rules

### 反映箇所

- `Responsibility Split`
- `Barrier State Fields`

### 反映内容

- barrier state は state_manager.py 経由で workflow.sqlite に保存する。
- REPL / scheduler は barrier state の正本を持たない。
- barrier release / blocked 状態は internal orchestrator API の status 応答に反映する。

## 20260627_07_require.md — orchestrator[retry] Define retry and abort rules

### 反映箇所

- `Retry Rules`
- `Abort Conditions`
- `Cleanup After Abort`

### 反映内容

- retry / abort / stuck detection の正本は state_manager.py に集約する。
- retry record / abort reason / cleanup state は workflow.sqlite に保存する。
- retry / cancel は internal API (`retry_phase`, `cancel_workflow`) 経由で実行する。

## 20260627_08_require.md — worktree[design] Define the worktree lifecycle rules

### 反映箇所

- `Create and Cleanup Rules`
- `Artifact Collection Paths`
- `Conflict Handling Rules`

### 反映内容

- worktree lifecycle の state は state_manager.py に登録し、workflow.sqlite の workflow/task state と整合させる。
- artifact collection は workflow.sqlite の artifact metadata と file system path の組み合わせで扱う。
- orphaned worktree 検出は workflow.sqlite の active workflow state と file system の差分で行う。

## 20260627_09_require.md — orchestrator[storage] Decide whether to reuse workflow.sqlite or add orchestrator.sqlite

### 反映箇所

- `Storage Decision`
- `Artifact Storage Strategy`
- `Minimum state_manager APIs`

### 反映内容

- 方針 4 として、Phase 1 では workflow.sqlite を拡張することを強化する。
- `orchestrator.sqlite` は追加しないことを明記する。
- `state_manager.py` が workflow.sqlite 上の workflow/task/artifact/retry/approval/event の正本 API になることを明記する。
- Artifact metadata は workflow.sqlite に持ち、大容量 body は file system 参照にする。

## 20260627_10_require.md — worker-runtime[tools] Define role-specific tool policies

### 反映箇所

- `Common Rules`
- `Publisher`

### 反映内容

- role policy の許可・拒否・approval 境界は state_manager.py / workflow.sqlite の approval / policy state と整合するようにする。
- Publisher の write 操作は internal orchestrator API / approval state / policy state と連動する。
- role MCP services は作ってよいが、orchestrator-mcp は作らない。

## 20260627_11_require.md — mcp-registry[tools]

### 反映箇所

- `Startup Compatibility Rules`
- `Required Server Classes`

### 反映内容

- Orchestrator は internal component であり、MCP registry の server validation 対象から除外する。
- `/v1/tools` 対応は Planner / Retriever / PatchWorker / Integrator / Validator / Publisher に限定する。

## 20260627_12_require.md — validator[execution]

### 反映箇所

- `Responsibility Split`

- Validator result は state_manager.py 経由で workflow.sqlite の artifact / validation metadata に登録する。

- Validator は state 正本を持たない。

## 20260627_13_require.md — orchestrator[response]

### 反映箇所

- `workflow_result DTO`
- `workflow_status DTO`
- `artifact_ref DTO`
- `Handling Rules`

### 反映内容

- DTO は internal orchestrator API の response として扱い、MCP result 形式には寄せない。
- DTO の structured fields は state_manager.py / workflow.sqlite の canonical state と整合させる。
- `artifact_ref` は workflow.sqlite 上の artifact metadata 参照を表す。

## 20260627_14_require.md — ops[runtime]

### 反映箇所

- `Internal Orchestrator Visibility`
- `Role MCP Visibility`
- `Watchdog Expectations`

### 反映内容

- Orchestrator は MCP service として扱わない。
- workflow 状態は state_manager.py / internal API 経由で表示する。
- role MCP health は mcp_registry.py、workflow heartbeat / event lag は internal orchestrator 側で監視する。

## 20260627_15_require.md — [orchestrator] Freeze the Phase 1 scope

### 反映箇所

- `Phase 1 MUST`
- `Completion Criteria`

### 反映内容

- orchestrator-mcp を作らないことを Phase 1 scope の明示的な除外項目として維持する。
- internal APIs を Phase 1 MUST として維持する。
- `state_manager.py` と workflow.sqlite reuse/extension を Phase 1 MUST に含める。
- external artifact store separation は LATER のまま維持する。

## 20260627_16_require.md — worker-runtime[extensibility]

### 反映箇所

- `Role Registry Design`
- `How New Roles Are Added`
- `Key Non-hardcoding Rules`

### 反映内容

- role registry は role MCP services を対象にし、internal orchestrator は role registry / MCP validation 対象にしない。
- new role 追加時も state 正本は state_manager.py / workflow.sqlite に集約する。
- role 名・tool policy・schema は config-driven とし、orchestrator-mcp 追加で拡張しない。

# 反映後の整合方針

## 正本の所在

```text
workflow/task/artifact/retry/approval state
  -> state_manager.py
  -> workflow.sqlite
```

## 内部 API

```text
start_workflow
resume_workflow
cancel_workflow
get_workflow_status
get_workflow_artifacts
retry_phase
```

## MCP 対象

```text
MCP registry 対象:
- planner-mcp
- retriever-mcp
- patchworker-mcp
- integrator-mcp
- validator-mcp
- publisher-mcp

MCP registry 対象外:
- internal orchestrator
```

## DB 方針

```text
Phase 1:
- workflow.sqlite を拡張する
- orchestrator.sqlite は作らない
- Artifact metadata は workflow.sqlite に保存する
- 大容量 body は file system 参照にする
```
