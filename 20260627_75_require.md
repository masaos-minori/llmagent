# Stabilize `SQLiteHelper.open(reuse_connection=True)` lifecycle behavior

## Priority
High

## Labels
- db
- sqlite
- performance
- connection-lifecycle
- high-priority

## Dependencies
None

## Background
`SQLiteHelper.open()` supports `reuse_connection=True`, but `_reuse_connection` is only set when reuse mode is enabled. This makes lifecycle behavior unclear when a single helper instance mixes reuse and non-reuse opens.

Existing tests already cover basic reuse behavior, but additional state reset coverage is needed.

## Scope
Modify:
- `db/helper.py`
- `tests/test_db_performance.py`

## Required Changes
- Initialize `self._reuse_connection = False` in `SQLiteHelper.__init__()`.
- In `open(reuse_connection=False)`, explicitly set `self._reuse_connection = False`.
- Ensure context manager exit closes the connection only when reuse mode is disabled.
- Ensure reuse mode can be safely disabled after previously being enabled.

## Tests
Update `tests/test_db_performance.py`.

Add tests for:
- `reuse_connection=True` keeps the connection open after context exit.
- `reuse_connection=False` closes the connection after context exit.
- Calling `open(reuse_connection=True)` followed by `open(reuse_connection=False)` resets lifecycle behavior.
- `close()` sets `conn` back to `None`.

## Acceptance Criteria
- Reuse and non-reuse modes behave predictably.
- Existing connection reuse tests pass.
- No hidden persistent connection remains when reuse mode is disabled.
