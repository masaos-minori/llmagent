# Remove `/mcp install` dispatch from MCP command handler

## Priority
High

## Labels
agent, commands, mcp, cleanup, breaking-change

## Dependencies
None

## Background
The Agent REPL currently supports `/mcp install <server-name>` through `_McpMixin._cmd_mcp_install()` and the `install` branch inside `_cmd_mcp()`. This feature generates MCP server scaffold files from the runtime REPL, but `/mcp` should be limited to MCP status and health inspection.

## Task
Remove the `/mcp install` execution path from the Agent MCP command handler.

## Implementation Requirements
- Remove `_cmd_mcp_install()` from `cmd_mcp.py`.
- Remove the `sub == "install"` branch from `_cmd_mcp()`.
- Update `_cmd_mcp()` so only the following forms are valid:
  - `/mcp`
  - `/mcp status`
- Update `UnknownSubcommandError` valid subcommand list to include only `status`.
- Update `_cmd_mcp()` docstring to remove references to the install wizard.
- Keep `_cmd_mcp_status()` unchanged.

## Acceptance Criteria
- `/mcp` still displays MCP server status.
- `/mcp status` still displays MCP server status.
- `/mcp install test` is rejected as an unknown/invalid subcommand.
- No `_cmd_mcp_install()` method remains in `cmd_mcp.py`.

## Source Context
`cmd_mcp.py` currently defines `_cmd_mcp_install()` and dispatches to it when the subcommand is `install`. The same file also contains the status implementation that must remain.
