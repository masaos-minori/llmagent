# 00_llm_spec_tobe（追加機能候補整理）

## `/diff` コマンド（変更差分表示）

### 課題
- 実現優先度: 高
- 実装難易度: 中
- エージェントがセッション中に行ったファイル変更を一覧で確認する手段がない

### 仕様/実装方針
- エージェントが書き込んだファイルのパスを追跡し、`/diff` で git diff または内部 diff を表示する

### 効果
- セッション中の変更点を即座に可視化でき、レビュー/確認が容易。
- 意図しない変更の早期検知により、事故・手戻りを低減。
- 変更範囲の共有が容易になり、チーム内合意・引き継ぎに有効。

### 実装方式
- agent_repl.py: `_execute_one_tool_call()` で modified_files 更新、スナップショット保存
- agent_commands.py: `_cmd_diff(args)` 新規

### 該当ファイル名・関数名
- agent_repl.py
- agent_commands.py

## `/branch`（セッション分岐）

### 課題
- 実現優先度: 中
- 実装難易度: 低
- 別案を検討するとき、現在の会話履歴を壊さずに分岐して試したい

### 仕様/実装方針
- `/branch <name>` で現在の `session_id` と履歴をフォークし、新セッションとして実験できるようにする

### 効果
- 別案検討を並行して進められ、現行案の履歴を破壊せずに試行可能。
- 比較検討が容易になり、最終方針の品質向上。
- やり直しコスト（履歴再構築、再説明）を削減。

### 実装方式
- agent_commands.py: `_cmd_branch(args)` 新規

### 該当ファイル名・関数名
- agent_commands.py
- agent_session.py

## `/rewind`（巻き戻し）

### 課題
- 実現優先度: 中
- 実装難易度: 低
- `/undo` は直前 1 ターンのみ。複数ターン前の状態に戻したい場面がある

### 仕様/実装方針
- `/rewind <n>` で n ターン前まで履歴・DB をロールバックする

### 効果
- 誤誘導/誤前提の連鎖を早期に断ち切り、任意地点へ復旧可能。
- 長いセッションでの試行錯誤の安全弁となり、心理的コストを低減。
- 不要な履歴を除去でき、コンテキスト節約（トークン削減）に寄与。

### 実装方式
- agent_commands.py: `_cmd_rewind(args)` 新規

### 該当ファイル名・関数名
- agent_commands.py
- agent_session.py

## Skills 読み込み機構

### 課題
- 実現優先度: 高
- 実装難易度: 高
- 「コードレビュー」「テスト生成」「移行ガイド」など定型手順を毎回会話で指示するのはコスト高

### 仕様/実装方針
- `skills/<name>/SKILL.md` に手順テンプレートを定義し、`/skill <name> [args]` で呼び出せるようにする

### 効果
- 定型手順の再利用により、指示の反復とトークン消費を削減。
- 手順の標準化により、品質のばらつきを低減し再現性を向上。
- プロジェクト/個人スコープ分離により、運用拡張が容易。

### 実装方式
- agent_commands.py: `_cmd_skill(args)` 新規
- agent_repl.py: 一時システム追記を渡せる `extra_system_prompt` 経路追加(必要なら)
- config/agent.json: `skill_paths` 追加

### 該当ファイル名・関数名
- agent_repl.py
- agent_commands.py
- config/agent.json
