# 危険操作の承認モデル強化

- 改善案:
  `require_approval_tools` によるツール名ベース承認から、引数内容、対象リソース、操作種別を含む危険度ベース承認へ変更すること。

- 効果:
  `write_file`、削除系、GitHub 書き込み系などの危険操作に対して、より精密な承認制御ができること。安全性と運用性の両立。

- リスク:
  承認要求が増えすぎると操作性が低下すること。危険度閾値設計が必要になること。

- 優先度:
  Must。副作用の大きいツールが既に仕様上存在するため。

- 難易度:
  中。既存承認フローを拡張する構造で対応可能であるため。

- 実装方式:
  ツール名だけでなく `args` と `resource_scope` を見て承認判定し、可能な操作は事前プレビューや `dry_run` を強制し、承認結果を `audit.log` へ構造化保存する方式。

- 実装対象:
  `agent_repl_tool_exec.py` 相当、`agent_config.py`、`tool_executor.py` 相当、`audit_logger` 連携。
