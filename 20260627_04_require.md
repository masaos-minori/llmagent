## Summary
Remove entries from `03_rag_90_inconsistencies_and_known_issues.md` that are already resolved or no longer reflect the current documentation and implementation state.

## Background
The user wants resolved tasks removed from documentation. The known-issues document still contains active entries for items that appear to be already addressed or partially addressed in the latest docs, such as `.json` canonicalization, `/db rebuild-fts`, and HTTP `remote_empty` semantics.

## Problem
Keeping resolved issues in active known-issues documentation misleads AI implementers. It can cause agents to rework fixed problems or prioritize obsolete tasks.

## Required Changes
- Re-scan current docs and code for each known issue.
- Delete resolved entries from the known-issues document.
- Keep only active, unresolved, or explicitly `Needs confirmation` items.
- For each remaining issue, include:
  - Type
  - Impact
  - Safe interpretation
  - Recommended action
  - Source or evidence
- Remove stale references to old issue IDs that no longer exist.
- Do not keep a resolved-issues archive in this document unless explicitly required. Use Git history for historical recovery.

## Acceptance Criteria
- Resolved tasks are removed from the known-issues document.
- Active issues are not mixed with resolved issues.
- Remaining entries are actionable and current.
- AI implementers can trust `03_rag_90_inconsistencies_and_known_issues.md` as the current risk list.

## Out of Scope
- Maintaining a historical changelog inside the known-issues document.
- Fixing runtime behavior unless required by dependent issues.
