## Title
[Medium] Clarify and test `consumer_id` sanitization for offset files

## Priority
Medium

## Labels
eventbus, offsets, security, documentation

## Dependencies
None

## Background
Offset files are stored under `offsets_dir` using a sanitized `consumer_id`. The current implementation replaces `/` and `..`, but documentation has described different behavior in the past.

The behavior should be explicit and covered by tests to avoid path traversal risks.

## Scope
Update:
- `scripts/eventbus/offsets.py`
- `tests/test_eventbus_offsets.py`
- Event Bus offset documentation

## Required Changes
- Decide the canonical sanitization behavior.
- Either:
  - update documentation to match the current implementation, or
  - update implementation to match the intended documentation.
- Add tests for:
  - `/`
  - `..`
  - `.`
  - empty consumer ID, if applicable
  - long consumer IDs
- Ensure path traversal is not possible.

## Acceptance Criteria
- Offset filename generation is deterministic and safe.
- Documentation and implementation agree.
- Path traversal attempts cannot write outside `offsets_dir`.
