## Title
[High] Include tool name in `tool_hash_key()` to avoid cross-tool collisions

## Priority
High

## Labels
bug, tool-loop-guard, retry, deduplication

## Dependencies
None

## Background
`tool_hash_key(name, args)` is documented as hashing a tool name and argument pair. The current implementation hashes only arguments.

This can cause collisions between different tools with identical arguments, affecting retry suppression and failed-call tracking.

## Scope
Update:
- `shared/tool_executor.py`
- `agent/tool_loop_guard.py` tests, if affected
- Tool loop guard tests

## Required Changes
- Change hash input to include the tool name:
  - `f"{name}:{json_dumps(args)}"`
- Preserve stable hashing for identical tool name + identical args.
- Update any tests that assumed args-only hashing.

## Tests to Add
- `test_tool_hash_key_differs_for_different_tool_names`
- `test_tool_hash_key_same_for_same_tool_and_args`
- Regression test for failed-call tracking across different tools.

## Acceptance Criteria
- Different tools with identical args produce different hash keys.
- Same tool with identical args produces the same hash key.
- Tool retry suppression no longer collides across tools.
