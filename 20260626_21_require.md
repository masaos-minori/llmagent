# [Medium] Implement true incremental refresh_index behavior

## Priority
Medium

## Labels
`mcp`, `mdq`, `indexing`, `incremental`, `performance`

## Dependencies
- Redesign MDQ SQLite schema for production indexing
- Introduce stable chunk IDs for MDQ chunks

## Background
Current `refresh_index` delegates to full indexing behavior. Production MDQ requires efficient incremental indexing based on file changes.

## Task
Implement true incremental refresh for Markdown documents.

## Implementation Requirements
- Track file metadata:
  - `mtime_ns`
  - `size_bytes`
  - `content_hash`
- Skip unchanged files.
- Re-index changed files.
- Remove chunks for deleted files.
- Add `force` option for full re-index.
- Return structured refresh summary as text or JSON string.

## Acceptance Criteria
- Unchanged files are skipped.
- Changed files are re-indexed.
- Deleted files are removed from the index.
- Refresh output includes:
  - indexed count
  - skipped count
  - deleted count
  - failed count
  - elapsed time
