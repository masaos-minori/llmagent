# Define MCP tool capability naming convention

## Priority

Low

## Target files

- docs/tool_capabilities.md
- MCP server-specific tools.py files
- agent/services/mcp_tool_discovery.py
- shared/runtime_tool.py

## Reason

Capabilities can support future policy, grouping, and discoverability.

They should be named consistently before being added broadly to MCP tool schemas.

## Intent

Document a standard optional capability naming convention.

## Examples

- `filesystem.read`
- `filesystem.write`
- `filesystem.delete`
- `git.read`
- `git.write`
- `github.issue.write`
- `process.execute`
- `search.web`

## Scope

Define conventions for:

- domain prefix
- action suffix
- read/write/delete/admin distinction
- optional multiple capabilities per tool

## Acceptance criteria

- Capability naming convention is documented.
- MCP tool schemas may adopt capabilities consistently.
- Discovery service tolerates missing capabilities.
