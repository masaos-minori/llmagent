# M-7: Review SIGTERM behavior while waiting for blocking input

## Priority

Medium

## Background

`AgentREPL` sets `_shutdown_event` on SIGTERM, but `_read_input()` waits on blocking `input()` through `run_in_executor()`. The shutdown event is not currently used to interrupt or race the input wait.

This may delay graceful shutdown while the REPL is waiting for user input.

## Target files

- `agent/repl.py`

## Implementation tasks

- Review current SIGTERM behavior during idle input wait.
- Decide whether immediate wake-up is required.
- If required, race `_read_input()` with `_shutdown_event.wait()`.
- Ensure no orphaned executor tasks or leaked input futures.
- Document behavior if immediate wake-up is not implemented.

## Acceptance criteria

- SIGTERM graceful shutdown behavior is documented.
- If implemented, SIGTERM exits idle input wait promptly.
- Existing EOF, Ctrl-C, and `/exit` behavior remains unchanged.
- Watchdog shutdown and resource cleanup still run.

## Test plan

- Unit-test or integration-test SIGTERM while idle.
- Test EOF and `/exit` still work.
- Test watchdog cleanup still runs after SIGTERM.
