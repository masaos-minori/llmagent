# Decide strictness for plugin tool return tuple length

## Priority

Medium

## Labels

plugin, tools, validation, design

## Background

Plugin tools are declared with return type `tuple[str, bool]`.

Registration-time validation checks the return annotation. Runtime validation in `ToolExecutor.execute()` checks that the actual return value is a tuple with at least two elements and validates the first two elements.

This means a plugin can return a tuple with more than two elements, and the extra elements are ignored.

## Goal

Define whether plugin tool return values must be exactly two elements or whether extra tuple elements are allowed.

## Scope

### In scope

- Decide the plugin tool return value contract
- Update runtime validation accordingly
- Update documentation and tests

### Out of scope

- Changing plugin tool dispatch priority
- Changing MCP tool result format
- Changing plugin command behavior

## Policy options

### Option A: Strict two-element tuple

Require exactly:

- `tuple[str, bool]`

Reject tuples with more or fewer than two elements.

### Option B: Minimum two elements

Allow extra tuple elements but use only:

- index 0 as output
- index 1 as is_error

## Recommended option

Use Option A because the documented contract is `tuple[str, bool]`.

## Acceptance criteria

- Runtime validation matches the selected policy
- Registration-time annotation validation and runtime value validation are consistent
- Tests cover:
  - non-tuple return
  - one-element tuple
  - two-element valid tuple
  - three-element tuple
  - wrong output type
  - wrong `is_error` type

## Validation

Run plugin tool tests.

Expected result:

- Behavior matches selected policy
- Invalid plugin return values fail predictably

## Affected files

- `scripts/shared/tool_executor.py`
- `scripts/shared/plugin_registry.py`
- Plugin documentation
- Plugin tests

## Dependencies

None
