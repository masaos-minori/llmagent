# Add registry-level tests for removed `/note` command

## Priority
Medium

## Labels
tests, agent, commands, notes, breaking-change

## Dependencies
- Remove `/note` command registration from CommandRegistry

## Background
After removing `/note`, the command should no longer be recognized as a built-in command. This should be locked with registry-level tests instead of `_NotesMixin` tests.

## Task
Add tests ensuring `/note` is no longer dispatched.

## Implementation Requirements
- Add registry or command-dispatch tests for the following inputs:
  - `/note add hello`
  - `/note list`
  - `/note delete 1`
  - `/note pin 1`
  - `/note unpin 1`
  - `/note search hello`
- Expected behavior should match the project’s existing unknown-command behavior.
- Add a guard test confirming `/memory` commands are still registered.

## Acceptance Criteria
- `/note ...` is treated as unknown or not handled by built-in commands.
- `/memory ...` remains available.
- Tests do not depend on `_NotesMixin`.

## Source Context
`/note` is currently a built-in command in `registry.py`; `/memory` is a separate command group and should remain.
