# [High] Promote DESIGN-2 to a confirmed RAG design decision

## Goal

Promote DESIGN-2, "FTS5 uses `normalized_content`; LLM receives `content`", from a needs-confirmation item to a confirmed RAG design decision.

## Background

The current RAG docs consistently describe the separation between `content` and `normalized_content`: `content` is the original readable chunk text passed to the LLM, while `normalized_content` is Sudachi-normalized Japanese text used for FTS5 indexing. FTS5 indexes `COALESCE(normalized_content, content)`, and AugmentStage formats `content`, not `normalized_content`. 【1-54c80f】【2-58c0bb】【3-6a9d52】

## Scope

Documentation and design invariant clarification only.

Do not change runtime behavior in this issue unless tests reveal a contradiction.

- Change DESIGN-2 type from `Needs confirmation` to `Confirmed design decision`.
- Document the invariant:
  - `chunks.content` is the original chunk text.

  - FTS5 indexes `COALESCE(normalized_content, content)`.
  - Japanese chunks use Sudachi-normalized text for FTS5.
  - English/code chunks keep `normalized_content = NULL`, so FTS5 falls back to `content`.
