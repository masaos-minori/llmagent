# L-3: Update operations documentation for MCP server definition restart requirement

## Priority

Low

## Background

Operations documentation should clearly state that `/reload` does not apply MCP server definition changes to running processes.

Operators must restart the agent to apply MCP URL, token, startup mode, transport, command, or environment changes.

## Target files

- `05_agent_10_operations-and-observability.md`
- `04_mcp_06_configuration_and_operations.md`

## Implementation tasks

- Add an operations note:
  - MCP server definition changes require full agent restart.
- State that `/reload` detects MCP server definition changes but does not apply them.
- Clarify that `/mcp` status shows running server health, not pending config.
- Clarify that watchdog restart is not config reload.
- Add examples of restart-required changes:
  - URL
  - auth token
  - startup mode
  - transport
  - command
  - environment
- Document GitHub MCP endpoint source as `mcp_servers.github.url`.

## Acceptance criteria

- Operations docs clearly distinguish reload, watchdog restart, and full agent restart.
- Operators are not instructed to use `/reload` to apply MCP server URL or token changes.
- `/mcp` status semantics are clear.
- GitHub MCP endpoint configuration is no longer documented through `github_server_url`.

## Test plan

- Search operations docs for `/reload` near MCP troubleshooting.
- Search operations docs for `github_server_url`.
- Verify restart-required guidance is present.
