# Remove pinned notes injection from startup prompt setup

## Priority
High

## Labels
agent, startup, prompt, notes, cleanup

## Dependencies
- Remove note delegation APIs from `AgentSession`

## Background
During startup, `StartupOrchestrator._setup_prompt()` injects pinned notes into the initial system prompt when `ctx.cfg.tool.auto_inject_notes` is enabled.

## Task
Remove startup-time pinned note injection.

## Implementation Requirements
- Update `startup.py`.
- Remove the `ctx.cfg.tool.auto_inject_notes` branch from `_setup_prompt()`.
- Remove the call to `ctx.session.get_pinned_notes()`.
- Remove generation of the `[Pinned Notes]` prompt block.
- Keep memory-layer startup injection unchanged.

## Acceptance Criteria
- Startup no longer injects notes into the system prompt.
- The `[Pinned Notes]` block is never generated.
- Memory snippets are still injected through `ctx.services.memory.on_session_start()` when memory is enabled.

## Source Context
`startup.py` currently appends pinned notes to the initial system prompt before memory snippets are injected.
