# L-2: Update data-layer documentation after removing tool_results

## Priority

Low

## Background

Data-layer documentation currently lists `tool_results` as a session.sqlite table and describes the boundary between `messages` and `ToolResultStore`.

After removal, diagnostics and workflow artifacts should be documented as the remaining non-message persistence channels.

## Target files

- `05_agent_04_state-and-persistence.md`
- `05_agent_09_data-layer.md`

## Implementation tasks

- Remove `tool_results` from the session.sqlite table list.
- Remove the `messages vs ToolResultStore` responsibility boundary.
- Remove documentation for `tool_results.undone`.
- Document the new persistence boundaries:
  - `messages`: LLM-visible conversation history
  - `session_diagnostics`: diagnostic-only events
  - `workflow.artifacts`: workflow-produced artifact references
  - `audit.log`: operational trace
- Keep `messages` table behavior unchanged.

## Acceptance criteria

- Data-layer docs no longer describe `tool_results` as an active table.
- Docs no longer describe ToolResultStore as a runtime component.
- Diagnostic storage is clearly described.
- Workflow artifact storage remains documented separately.

## Test plan

- Search docs for `ToolResultStore`.
- Search docs for `tool_results.undone`.
- Search docs for `messages vs ToolResultStore`.
