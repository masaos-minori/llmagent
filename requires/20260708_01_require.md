# H-1: Remove Tool Result Summarization from tool execution path

## Priority

High

## Background

Tool Result Summarization is being removed as a backward-compatibility feature.

`agent/tool_runner.py` currently imports `summarize_tool_result` from `rag.llm_client` and conditionally summarizes long tool outputs when `ctx.cfg.tool.use_tool_summarize` is enabled. After this change, tool outputs should only be truncated before being injected into LLM-visible history.

## Target files

- `agent/tool_runner.py`
- `agent/tool_result_formatter.py`

## Implementation tasks

- Remove the `summarize_tool_result` import from `agent/tool_runner.py`.
- Remove the `ctx.cfg.tool.use_tool_summarize` branch from `execute_one_tool_call()`.
- Remove the `tool_summarize_threshold` check from tool execution.
- Remove the call to `summarize_tool_result(...)`.
- Remove summarization-specific logging.
- Set `llm_text` using only the existing truncation behavior:
  - if the tool output length is within `tool_result_max_llm_chars`, pass the text through unchanged
  - otherwise truncate to `tool_result_max_llm_chars` and append the truncation suffix
- Remove any dependency on `is_summarized()` from this execution path.

## Acceptance criteria

- Tool execution no longer calls an LLM to summarize tool output.
- `rag.llm_client.summarize_tool_result` is no longer imported by `agent/tool_runner.py`.
- Long tool outputs are still bounded before being injected into `ctx.conv.history`.
- Tool execution still returns tool-role messages to the LLM.
- No behavior depends on `use_tool_summarize` or `tool_summarize_threshold`.

## Test plan

- Add or update tests for a long successful tool output.
- Verify the LLM-visible tool message is truncated, not summarized.
- Verify no summarization LLM call is made.
- Verify normal short tool outputs remain unchanged.
- Verify error tool outputs are not summarized.
