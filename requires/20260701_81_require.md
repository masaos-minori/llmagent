# [Low] Strengthen RAG documentation checks for stale history and removed references

## Goal

Prevent stale legacy references and deleted-history notes from being reintroduced into active RAG documentation.

## Background

The active RAG docs should be the only current specification source. Legacy source files and migration notes should not be used as active guidance. 【6-960385】

## Scope

Documentation checks only.

## Implementation Tasks

- Update `scripts/checks/check_docs_consistency.py` or the relevant documentation checker.
- Add checks for stale RAG references:
  - `03_spec_rag.md`
  - `03_rag-ref-`
  - `03_rag-ingestion-`
  - `05_ref-rag.md`
  - `Migration Notes`
  - `removed`
  - `legacy source`
- Allow exceptions only if explicitly marked as non-active archival text.
- Prefer removing archival text from active docs rather than adding broad exemptions.
- Ensure DESIGN-2 and DESIGN-3 are not flagged as unresolved once promoted.

## Acceptance Criteria

- Stale legacy references are detected by docs checks.
- Deleted-history notes are not reintroduced silently.
- Confirmed design decisions are not flagged as active unresolved issues.

## Validation

- Run:

  python scripts/checks/check_docs_consistency.py

- Temporarily add a stale reference in a scratch branch and confirm the checker fails.
- Remove the scratch change and confirm the checker passes.
