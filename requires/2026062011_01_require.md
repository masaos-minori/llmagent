# Title
AgentSession と RAG 保守処理の責務境界を統一する

## Summary
`AgentSession` を `session.sqlite` のセッション / メッセージ / ノート管理専用に寄せ、RAG テーブル操作は `DbMaintenanceService` または MCP サービスへ一本化する。

## Background
文書上、`AgentSession` が RAG テーブルに依存している説明と、RAG 操作は `DbMaintenanceService` 経由であり `AgentSession` は session 層専用である説明が混在している。【1-3c0516】【7-69e5fd】【3-0d13a3】【2-b7a22f】

## Problem
責務境界が曖昧なままだと、RAG スキーマ変更時に agent/session 層まで影響範囲が拡散し、保守性が低下する。【1-3c0516】【2-b7a22f】

## Required Changes
- `agent/session.py` の責務を棚卸しし、RAG テーブル依存の有無を確認する。【1-3c0516】【2-b7a22f】
- `/db clean`、`/db stats`、`/db urls`、`/db rebuild-fts` などの RAG 系処理を `DbMaintenanceService` または MCP 呼び出しへ移す。【8-0b71a5】【2-b7a22f】【7-69e5fd】
- `05_agent_04`、`05_agent_09`、`05_agent_12`、`05_agent_90` の記述を統一する。【3-0d13a3】【2-b7a22f】【7-69e5fd】【1-3c0516】

## Acceptance Criteria
- `AgentSession` が session.sqlite 以外の schema 詳細に依存しない。【3-0d13a3】【2-b7a22f】
- RAG メンテナンス処理が session 層から分離されている。【2-b7a22f】【7-69e5fd】
- 関連ドキュメントの責務境界記述が一致している。【1-3c0516】【3-0d13a3】【2-b7a22f】【7-69e5fd】

## Out of Scope
- RAG パイプラインの検索アルゴリズム変更。【2-b7a22f】
- `/db` コマンドの全面廃止。【8-0b71a5】【2-b7a22f】
