# Title
Add a safe fallback strategy for failed history compression

## Summary
Add a non-LLM fallback path for history compression so that context growth remains controlled even when LLM-based summarization fails.

## Background
History compression currently depends on an LLM summarization step.
If summarization fails, the history is returned unchanged.

## Problem
If the compression step fails while the context is already too large, subsequent LLM calls may become even more fragile or expensive.

## Required Changes
- Add a deterministic fallback compression path when LLM compression fails.
- Preserve the protected recent turns.
- Prefer removing or shrinking lower-priority content first:
  - tool role messages
  - temporary reasoning artifacts
  - low-value old content
- Expose whether fallback compression was used through stats, logs, or context inspection.

## Acceptance Criteria
- Context size remains bounded even if LLM compression fails.
- Protected recent turns are not unintentionally lost.
- Operators can distinguish successful LLM compression from fallback compression.
- Existing history selection priorities remain respected.

## Out of Scope
- Replacing LLM-based compression entirely.
- Reworking the full history scoring algorithm.

## Notes
Preserve correctness and continuity before preserving verbosity.
