# [Low] Add CI check for removed compatibility leftovers

## Priority
Low

## Background
Compatibility leftovers were found across routing code, registry tests, GitHub MCP exports, and command documentation. A static check should prevent reintroduction.

## Goal
Add a CI check that blocks removed compatibility paths while allowing intentionally retained stdio transport.

## Scope
Affected files:
- `scripts/checks/check_no_agent_compat.py`
- CI workflow files
- `shared/`
- `agent/`
- `mcp/`
- `docs/`
- `tests/`

## Implementation Tasks
- Add forbidden pattern checks for:
  - `Static fallback`
  - `compatibility/emergency`
  - `_SET_ROUTES`
  - `_GITHUB_PREFIX`
  - `prefix-routed`
  - `github_ prefix`
  - `/mcp install`
  - `/debug audit`
  - `/note`
  - `/db urls`
  - `/db clean`
  - `/db rebuild-fts`
  - `/db recover`
  - `auto_inject_notes`
  - `notes table`
- Allow migration/archive sections explicitly.
- Do not flag valid stdio transport symbols:
  - `StdioTransport`
  - `TransportType.STDIO`
  - `StartupMode.ONDEMAND`
  - `HealthcheckMode.PING_TOOL`
  - `__list_tools__`
- Print file path and line number for violations.

## Acceptance Criteria
- CI catches removed routing fallback.
- CI catches removed command aliases.
- CI catches GitHub prefix routing language.
- CI does not fail on valid stdio transport usage.

## Validation
- Run: `python scripts/checks/check_no_agent_compat.py`
