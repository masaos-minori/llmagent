# [High] Unify diagnostics and partial-completion persistence model

## Summary
Define one canonical persistence model for diagnostics such as:
- partial LLM completions
- mid-turn transport failures
- tool-loop guard hints

## Problem
Diagnostics are spread across multiple stores:
- tool result store
- session diagnostics
- audit log
- session-end diagnostics JSONL

This weakens traceability and documentation consistency.

## Goal
Use one canonical diagnostics store and keep other stores as secondary views only.

## Scope
Target files may include:
- `agent/orchestrator.py`
- `agent/diagnostic_store.py`
- `agent/session.py`
- statistics and `/tool show` related code
- diagnostics output paths

## Required Changes
1. Select one canonical diagnostics store.
2. Define explicit roles for:
   - conversation history
   - full tool outputs
   - diagnostic records
   - audit events
3. Route partial completions and guard hints through the canonical path.
4. Update `/stats`, `/tool show`, and diagnostics lookup behavior to align with the new model.
5. Update tests and documentation.

## Acceptance Criteria
- Diagnostics ownership is unambiguous.
- Partial completion records are stored and retrieved consistently.
- Tool result archives remain separate from canonical diagnostics.
- Tests cover partial completion, transport failure, and guard hint persistence.

## Out of Scope
- Full artifact lineage redesign
- Workflow phase refactor
