# Title
Expose partial LLM completions through normal operator-facing views

## Summary
Make partial LLM completions visible in normal runtime views so that interrupted answers are easier to detect and investigate.

## Background
Partial completions are intentionally stored outside normal history to avoid polluting future context.
However, this also makes them less visible during normal operation.

## Problem
Operators may miss interrupted or incomplete responses unless they manually inspect specialized stores or debug commands.

## Required Changes
- Add a visible signal when a partial completion occurred in the current session.
- Surface this state in at least one normal operator-facing view such as:
  - `/stats`
  - `/context`
  - session banner / warning line
- Improve discoverability of the saved partial completion artifact.

## Acceptance Criteria
- Partial completion state is visible without requiring deep debug inspection.
- Diagnostic isolation from normal conversation history remains intact.
- Operators can discover the relevant stored artifact from normal runtime commands.

## Out of Scope
- Reinjecting partial assistant text into the main conversation history.
- Reworking the SSE transport model itself.

## Notes
Preserve diagnostic isolation, but improve observability.
