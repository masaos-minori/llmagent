# H-10: Stop creating tool_results table in new session schema

## Priority

High

## Background

The session schema currently creates the `tool_results` table and `idx_tool_results_session` index. New databases should no longer create this backward-compatibility table.

Existing databases may still contain the table, but runtime code should not use it.

## Target files

- `db/schema_sql.py`
- `db/create_schema.py`

## Implementation tasks

- Remove `CREATE TABLE IF NOT EXISTS tool_results` from the session schema template.
- Remove `CREATE INDEX IF NOT EXISTS idx_tool_results_session`.
- Update `create_session_schema()` docstring to remove `tool_results`.
- Update schema comments that describe session.sqlite contents.
- Do not add destructive DROP migration in this task.

## Acceptance criteria

- New session.sqlite databases no longer contain `tool_results`.
- New session.sqlite databases still contain:
  - `sessions`
  - `messages`
  - memory tables when applicable
  - `session_diagnostics`
- Schema creation remains idempotent.
- Existing databases with `tool_results` still open successfully.

## Test plan

- Create a fresh session.sqlite.
- Verify `.tables` does not include `tool_results`.
- Verify `sessions`, `messages`, and `session_diagnostics` exist.
- Run schema creation twice to verify idempotency.
