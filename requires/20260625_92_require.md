# Title
Strengthen automatic context restoration for new sessions

## Priority
High

## Labels
- enhancement
- memory
- retrieval
- session-start
- context-restore

## Dependencies
- Issue 2

## Summary
Improve automatic context restoration in new sessions by clarifying and strengthening the split between session-start memory injection and first-prompt task-specific retrieval. The current lifecycle already supports `on_session_start()` and `on_user_prompt()`, but the intended “restore relevant past context into a new session” experience needs to be made more intentional and explicit.

## Background
The current memory lifecycle is:
- session start → semantic injection
- user prompt → query-based semantic + episodic retrieval
- session stop → extraction + persistence

## Problem
The requirement calls for automatic restoration of relevant past context when a new session begins, but the current design splits this behavior between startup injection and first-prompt retrieval in a way that is not yet strong or explicit enough.

## Required Changes
- Clarify `on_session_start()` as the path for pinned / high-importance semantic memory.
- Clarify first user prompt retrieval as the main task-specific restoration path.
- Revisit injection defaults (`memory_max_inject_semantic`, `memory_max_inject_episodic`, `memory_min_importance`) to reduce irrelevant injection.
- Update docs to explain the new session restore model clearly.

## Acceptance Criteria
- Session-start injection and prompt-time retrieval have clear, non-overlapping roles.
- New-session context restoration is measurably more relevant.
- Docs explain how relevant past context is restored.

## Out of Scope
- History compression redesign
- Full memory subsystem API redesign

## Components
- `scripts/agent/memory/injection.py`
- `scripts/agent/memory/retriever.py`
- `scripts/agent/memory/services.py`
- memory config and docs

## AI Implementation Instruction
Implement this as a **cross-session recall quality task**.

Strict rules:
- Do not simply inject more memories.
- Keep startup injection and prompt-time retrieval conceptually separate.
- Avoid over-injecting unrelated high-importance memories.
- If ranking or restore policy is unclear, mark it as: `Needs confirmation`
