## Title
[High] Wire `memory_local_only` configuration into the embedding client

## Priority
High

## Labels
bug, memory, configuration, security

## Dependencies
None

## Background
`EmbeddingClientConfig` supports `local_only`, and `/memory status` displays local-only state. However, the memory configuration does not currently expose `memory_local_only`, and the factory does not pass it into `EmbeddingClientConfig`.

This makes the local-only safety control ineffective or misleading.

## Scope
Update:
- `agent/config_dataclasses.py`
- `agent/config_builders.py`
- `agent/factory.py`
- `config/memory.toml` if applicable
- Tests for memory config and embedding client construction

## Required Changes
- Add `memory_local_only: bool = False` to `MemoryConfig`.
- Read `memory_local_only` in `_build_memory_config()`.
- Pass `local_only=ctx.cfg.memory.memory_local_only` into `EmbeddingClientConfig`.
- Ensure non-local `embed_url` is rejected when `memory_local_only=True`.

## Tests to Add
- `test_memory_local_only_default_false`
- `test_memory_local_only_passed_to_embedding_client`
- `test_memory_local_only_rejects_non_local_embed_url`

## Acceptance Criteria
- `memory_local_only=true` is enforced.
- `/memory status` accurately reflects the configured state.
- Non-local embedding endpoints are rejected when local-only mode is enabled.
