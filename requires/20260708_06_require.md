# H-6: Remove tool_results undone marking from undo service

## Priority

High

## Background

`undo_last_turn()` currently marks tool result rows as undone by calling `ctx.tool_result_store.mark_turn_undone(...)`.

The `tool_results.undone` retention model is being removed. `/undo` should only roll back conversation history and persisted messages.

## Target files

- `agent/services/undo_service.py`
- `agent/services/models.py`

## Implementation tasks

- Remove the call to `ctx.tool_result_store.mark_turn_undone(...)`.
- Remove or simplify `turn_to_mark` if it becomes unused.
- Decide whether to remove `UndoResult.n_artifacts_marked` or keep it as a deprecated always-zero field.
- If kept, always return `n_artifacts_marked=0`.
- Update docstrings to remove artifact marking behavior.
- Ensure `/undo` still:
  - removes the last user turn and following messages from `ctx.conv.history`
  - strips preceding memory injection markers
  - decrements `ctx.stats.stat_turns`
  - calls `ctx.session.undo_last_turn()`

## Acceptance criteria

- `/undo` no longer touches `ToolResultStore`.
- `/undo` no longer updates `tool_results.undone`.
- `/undo` still rolls back in-memory conversation history.
- `/undo` still rolls back persisted session messages.
- Return type behavior is documented.

## Test plan

- Run `/undo` after a turn with tool calls.
- Verify history is trimmed.
- Verify session messages are rolled back.
- Verify no ToolResultStore method is called.
- Verify returned `UndoResult` matches the new behavior.
