# [mcp][startup] Define strict tool-definition verification behavior clearly

## Summary
The startup behavior for `tool_definitions_strict` is not fully clear when servers are unreachable.

## Problem
Tool mismatch behavior is known, but unreachable-server behavior is not documented clearly enough.
This makes startup failures harder to understand.

## Scope
- startup verification logic
- `/v1/tools` comparison
- startup documentation
- `/mcp` diagnostics

## Task
- define behavior for:
  - one server unreachable
  - multiple servers unreachable
  - all servers unreachable
  - tool name mismatch
- make startup reporting clearer
- document the final behavior

## Expected Result
Startup validation failures are easier to diagnose and reason about.

## Acceptance Criteria
- [ ] strict-mode behavior is documented clearly
- [ ] mismatch and unreachable cases are distinguished
- [ ] startup diagnostics are easier to interpret
