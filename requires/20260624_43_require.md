# Separate missing-config handling from parse/read failures in ConfigLoader.load_all()

## Summary
Refactor `shared/config_loader.py` so that `load_all()` treats missing config files differently from parse/read failures.

## Background
The current implementation converts file-not-found, TOML/JSON parse failures, and generic file read failures into `ValueError` in `_load_single()`. Then `load_all()` catches `ValueError` and silently skips the file with only a debug log.

This means a malformed config file is treated the same as an intentionally missing file.

## Objective
Make configuration loading fail predictably for malformed config while still allowing intentionally missing optional files to be skipped.

## In Scope
- `config_loader.py`

## Required Changes
- Introduce distinct error categories for:
  - file not found
  - parse failure
  - read failure
- Update `load_all()` to:
  - skip only missing files
  - surface parse/read failures clearly
- Keep `load(*names)` strict for direct file loading
- Preserve `_`-prefixed key filtering behavior

## Acceptance Criteria
- Missing optional config files can still be skipped by `load_all()`
- Malformed TOML/JSON is no longer silently ignored
- File read errors are distinguishable from parse errors
- Caller behavior is deterministic and testable

## Out of Scope
- Changing config file names
- Redesigning config merge precedence

## Implementation Notes
Do not use one generic exception type for all config-loading failures. The severity of “missing” vs “invalid” must remain distinguishable.
