# [MCP][MDQ] Implement real mdq-mcp search and indexing

## Summary
`mdq-mcp` needs real indexing and retrieval logic instead of placeholder service responses.

## Problem
Current mdq tools return stub strings and do not perform actual data operations.

## Scope
- `mcp/mdq/service.py`
- `mcp/mdq/indexer.py`
- `mcp/mdq/search.py`

## Task
- implement real behavior for:
  - `search_docs`
  - `get_chunk`
  - `index_paths`
  - `refresh_index`
  - `stats`
  - `grep_docs`
- keep `outline` aligned with actual indexed content once indexing exists
- implement in stages:
  1. real search
  2. real chunk retrieval
  3. indexing / refresh
  4. stats and grep

## Expected Behavior
- mdq tools return real indexed results
- index refresh changes actual state
- stats reflect real index contents

## Acceptance Criteria
- [ ] `search_docs` returns real matches
- [ ] `get_chunk` retrieves real stored chunks
- [ ] `index_paths` and `refresh_index` modify real index state
- [ ] `stats` returns real metadata about the index
