# [Agent][Ops] Add production startup readiness checks for required services

## Summary
Startup health checks are warning-only today, and the REPL continues even if important dependencies are unavailable.

## Problem
This allows the agent to start in a partially broken state.
In production, required services such as the LLM endpoint, embedding service, or critical MCP servers should be validated more strictly.

## Scope
- `AgentREPL._check_service_health()`
- startup validation flow
- operational startup policy

## Task
- define a production readiness mode
- fail startup when required dependencies are unavailable
- keep warning-only behavior only for development or degraded modes
- allow configuration of which dependencies are required

## Expected Behavior
- production startup fails fast when required services are down
- development startup can still allow warning-only behavior
- startup logging clearly shows which required dependency failed

## Acceptance Criteria
- [ ] required dependency checks can fail startup
- [ ] warning-only mode remains available as an explicit choice
- [ ] startup output clearly shows readiness status
- [ ] the readiness policy is documented
