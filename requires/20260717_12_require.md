# Add MCP tool schema versioning and validation tests

## Priority

Low

## Target files

- mcp_servers/server.py
- MCP server-specific server.py files
- MCP server-specific tools.py files
- agent/services/mcp_tool_discovery.py
- tests/mcp_servers/
- tests/agent/services/test_mcp_tool_discovery.py

## Reason

RuntimeToolRegistry depends on valid MCP `/v1/tools` responses.

A schema version and validation tests will improve compatibility and reduce the risk of malformed tool definitions entering runtime execution.

## Intent

Standardize and test MCP tool schema quality.

## Proposed /v1/tools response shape

{
  "schema_version": "1.0",
  "tools": [
    {
      "name": "tool_name",
      "description": "Tool description",
      "inputSchema": {
        "type": "object",
        "properties": {},
        "required": []
      },
      "status": "production",
      "is_write": false,
      "requires_serial": false,
      "resource_scope": ""
    }
  ]
}

## Scope

Add tests for:

- non-empty tool name
- description
- object input schema
- valid optional status
- valid optional is_write
- valid optional requires_serial
- valid optional resource_scope

## Acceptance criteria

- MCP servers may include `schema_version`.
- Agent tolerates missing schema_version during migration.
- Invalid schemas fail tests with clear server and tool identification.
