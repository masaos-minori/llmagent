# [High] Make ToolRegistry the single source of MCP tool ownership

## Priority
High

## Background
`tool_registry.py` describes `ToolRegistry` as the primary registry, but still lists config `tool_names` and static fallback in routing priority. This conflicts with the intended ownership model. 【6-424347】

## Goal
Make `ToolRegistry` the only non-discovery source of MCP tool ownership.

## Scope
Affected files:
- `shared/tool_registry.py`
- `shared/route_resolver.py`
- `shared/tool_constants.py`
- `tests/test_tool_registry_counts.py`
- `tests/test_route_resolver.py`
- MCP/Agent routing documentation

## Implementation Tasks
- Update `shared/tool_registry.py` docstring:
  - `ToolRegistry` owns MCP tool ownership.
  - Each tool belongs to exactly one server key.
  - `tool_constants.py` is registry seed/classification data only.
  - config `tool_names` is validation metadata only.
  - static fallback no longer exists.
- Ensure `ToolRouteResolver` uses `ToolRegistry` for registry routing.
- Add GitHub tools to `ToolRegistry`.
- Add or update tests for one-owner-per-tool behavior.
- Add duplicate ownership detection if not already present.

## Acceptance Criteria
- Tool ownership is defined by `ToolRegistry`.
- All known MCP tools, including GitHub tools, have exactly one owner.
- `route_resolver.py` does not route directly from `tool_constants.py`.
- Active docs do not describe config/static fallback as routing priority.
- Tests pass.

## Validation
- Run: `pytest tests/test_tool_registry_counts.py tests/test_route_resolver.py -v`
- Run: `rg -n "Static fallback|Config tool_names.*routing|primary routing.*fallback" shared docs tests`
