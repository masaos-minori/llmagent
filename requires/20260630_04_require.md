# [High] Remove `rag/models.py` backward compatibility re-export stub

## Priority
High

## Difficulty
Medium

## Background
`rag/models.py` explicitly states that it is a re-export stub for backward compatibility.

It re-exports DTOs from:

- `models_config.py`
- `models_data.py`
- `models_result.py`
- `models_audit.py`

## Goal
Remove the backward-compatible `rag.models` import path and require direct imports from canonical DTO modules.

## Scope
Affected files:

- `scripts/rag/models.py`
- `scripts/rag/models_config.py`
- `scripts/rag/models_data.py`
- `scripts/rag/models_result.py`
- `scripts/rag/models_audit.py`
- all imports under `scripts/`
- all imports under `tests/`
- related documentation

## Implementation Tasks
- Search for all imports from `rag.models`.
- Replace them with direct imports from:
  - `rag.models_config`
  - `rag.models_data`
  - `rag.models_result`
  - `rag.models_audit`
- Delete `scripts/rag/models.py`.
- Update documentation to remove references to `rag.models`.
- Add a static check or import test to prevent `from rag.models import ...`.

## Acceptance Criteria
- `scripts/rag/models.py` no longer exists.
- `rg -n "from rag\\.models|import rag\\.models" scripts tests docs` returns no active references.
- Tests pass using direct DTO imports only.
- No backward compatibility re-export layer remains.

## Validation
- Run unit tests.
- Run `rg -n "rag\\.models" scripts tests docs`.
