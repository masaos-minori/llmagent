# [rag][ingestion] Fix field loss in RagIngester._read_chunk_json()

## Summary
`RagIngester._read_chunk_json()` drops important fields from chunk JSON.

## Problem
The current read path can lose these fields:
- `chunking_strategy`
- `normalized_content`
- `chunk_index`

This can reduce retrieval quality and break document order handling.

## Scope
- `scripts/rag/ingestion/ingester.py`
- chunk JSON read path
- DB insert path

## Task
- preserve all required chunk fields when reading chunk JSON
- stop dropping metadata that exists in the chunk file
- make sure the preserved fields reach the DB insert stage

## Expected Result
Chunk metadata is preserved correctly from chunk files into the database.

## Acceptance Criteria
- [ ] `chunking_strategy` is preserved
- [ ] `normalized_content` is preserved
- [ ] `chunk_index` is preserved
- [ ] preserved values reach the DB correctly
