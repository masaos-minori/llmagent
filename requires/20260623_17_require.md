# Task: Implement Graceful Shutdown with Timeout Guards for SIGTERM

## Overview and Context
The current operational specification relies on translating `SIGTERM` signals into a `SystemExit(0)` exception, setting a `_shutdown_requested` flag, and waiting for the current REPL loop to complete its prompt input. However, if `SIGTERM` is intercepted while the agent is mid-stream with an LLM, actively communicating via an MCP server, or running parallel tools via `execute_all_tool_calls()`, the abrupt termination can lead to orphaned background processes or corrupted SQLite WAL states. A strict active guard and timeout limit must be introduced.

## Acceptance Criteria (AC)
- [ ] Intercept `SIGTERM` using an active guard that checks if a turn execution block (`execute_all_tool_calls` or DB transaction) is currently active.
- [ ] If active, defer termination until the current turn transaction completes or safely rolls back.
- [ ] Implement a hard 10-second maximum "timeout guard". If the processes fail to wrap up within 10 seconds, force close all MCP pipes and SQLite connections cleanly before exiting.

## Technical Specifications and Implementation Steps
1. Target Files: `agent.py`, `shared/tool_executor.py`, `shared/db/helper.py`
2. Step 1 (Active Flag): Introduce an execution context manager or state flag `ctx.turn.is_processing` to mark runtime transaction boundaries.
3. Step 2 (Signal Handler): Update the signal registration handler to launch an asynchronous cleanup wrapper with a `clamped timeout` of 10 seconds using `asyncio.wait_for`.
4. Step 3 (Teardown Orchestration): Ensure the timeout cleanup triggers explicit process termination for stdio-based external MCP servers.

## Validation and Testing Plan
- Command: `pytest tests/agent/test_graceful_shutdown.py`
- Required Test Cases:
  - `test_sigterm_during_active_tool_execution`: Simulates a `SIGTERM` signal while a long-running mock tool is executing, ensuring it finishes or cleanly rolls back without creating orphan processes.
  - `test_shutdown_timeout_enforcement`: Verifies that if a tool deadlocks, the 10-second timeout guard triggers a forced, clean connection termination.

## Constraints and Core Rules
- Never allow a bare `os._exit(0)` without attempting to close open SQLite connection references.
