# [Medium] Improve MCP production operations and security guidance

## Labels
priority/medium, area/mcp, area/security, area/operations, documentation

## Summary
Improve MCP production operations and security guidance.

This issue consolidates documentation and operational guidance for:

- migrating from local no-auth MCP setup to production auth
- GitHub MCP opt-in deny controls
- `mcp_watchdog_interval` local and production recommendations

## Background
Current local configuration can use `security_profile="local"` and empty MCP `auth_token` values. `auth_token=""` is documented as local/development compatibility behavior and is rejected in production. 【1-14e820】【3-5d2763】【4-8938fe】

GitHub MCP has opt-in deny controls such as `path_denylist` and `protected_branches`. Empty lists allow all paths or branches for those dimensions. 【3-5d2763】【1-62b04c】

`mcp_watchdog_interval=0` disables automatic restart and is suitable for local/manual operation, while production recommendation is a non-zero interval such as `30.0`. 【3-5d2763】【4-8938fe】

## Required Changes

### 1. Local no-auth to production auth migration
Document a migration checklist:

- switch `security_profile` from `local` to `production`
- set non-empty `auth_token` for all HTTP MCP servers
- avoid hardcoding secrets where env or secret injection is available
- restart the agent after changing MCP auth configuration
- verify `/mcp status`
- inspect startup failures for missing or mismatched auth tokens

Add troubleshooting cases:

- empty `auth_token`
- missing environment secret
- mismatched Bearer token
- MCP auth config changed but only `/reload` was run

### 2. GitHub MCP opt-in deny controls
Document operational guidance for:

- `path_denylist=[]`
- `protected_branches=[]`
- `allow_force_push`
- `require_pr_review`

- protect `master`
