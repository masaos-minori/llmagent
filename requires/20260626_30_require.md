# [Low] Add summary cache for large Markdown chunks

## Priority
Low

## Labels
`mcp`, `mdq`, `summary-cache`, `context-compression`, `enhancement`

## Dependencies
- Redesign MDQ SQLite schema for production indexing
- Enforce result size limits for MDQ tools

## Background
Large Markdown sections may still consume significant context even after heading-based retrieval. A summary cache can further reduce token usage.

## Task
Add optional summary caching for large chunks.

## Implementation Requirements
- Add `chunk_summaries` table.
- Store:
  - `chunk_id`
  - `summary`
  - `summary_model`
  - `content_hash`
  - `created_at`
- Invalidate summaries when content hash changes.
- Use summary when chunk exceeds configured threshold.
- Fall back to raw truncated content if summarization fails.

## Acceptance Criteria
- Summary cache is optional.
- Cached summaries are invalidated after content changes.
- Large chunk retrieval can return summary plus source metadata.
- Failure to summarize does not break `get_chunk`.
