# Update MCP documentation to remove `/mcp install` wizard

## Priority
Medium

## Labels
docs, mcp, breaking-change

## Dependencies
- Remove `/mcp install` dispatch from MCP command handler

## Background
The MCP documentation currently describes `/mcp install <server-name>` as a wizard/recommended option for adding new MCP servers. This is no longer valid after removing the command.

## Task
Update MCP documentation so new MCP server creation is documented as a manual developer workflow, not an Agent REPL command.

## Implementation Requirements
- Update:
  - `04_mcp_03_routing_lifecycle_and_execution.md`
  - `04_mcp_06_configuration_and_operations.md`
- Remove references to:
  - `/mcp install <server-name>`
  - `Wizard (recommended)`
  - install wizard scaffold generation
- Make the manual server addition checklist canonical.
- Keep `/mcp` only as a verification/status command after manual setup.

## Acceptance Criteria
- Documentation no longer presents `/mcp install` as an available command.
- New MCP server setup instructions are REPL-independent.
- `/mcp` is documented only for status/health/tool-list verification.

## Source Context
The MCP docs currently describe `/mcp install <server-name>` as a wizard for generating server scaffolds and config templates.
