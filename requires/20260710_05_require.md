# [High] Repair and harden deployment shell scripts

## Background

Deployment scripts must be reliable because workflow definition deployment is mandatory. Shell script syntax or variable expansion errors can prevent proper artifact deployment and validation.

## Goal

Fix broken shell script constructs and add shell validation to CI.

## Scope

- `deploy/deploy.sh`
- `deploy/setup_services.sh`
- CI configuration

## Tasks

- [ ] Fix `REPO_ROOT` assignment in deployment scripts.
- [ ] Fix broken variable expansions.
- [ ] Remove or correct invalid empty `mkdir` commands.
- [ ] Remove or correct invalid empty `cp` commands.
- [ ] Ensure all paths use quoted variables safely.
- [ ] Fix URL command examples so they are valid shell syntax.
- [ ] Add `bash -n` validation for deployment scripts.
- [ ] Add `shellcheck` validation for deployment scripts if available.
- [ ] Ensure scripts work with `set -euo pipefail`.

## Acceptance Criteria

- [ ] `bash -n deploy/deploy.sh` passes.
- [ ] `bash -n deploy/setup_services.sh` passes.
- [ ] `shellcheck deploy/deploy.sh deploy/setup_services.sh` passes, or documented exceptions are justified.
- [ ] Workflow artifact checks are not bypassed by shell errors.
- [ ] Deployment scripts fail fast on invalid workflow state.

## Notes

This task must be completed before relying on deploy-time workflow validation.
