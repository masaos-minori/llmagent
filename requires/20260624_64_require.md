# [High] Shrink REPL into a pure UI adapter

## Summary
Refactor the REPL so it only handles:
- user input
- progress display
- command display
- result rendering

It must not own workflow execution state.

## Problem
The REPL still depends on transient workflow/session state for workflow-facing commands and approval display behavior.

## Goal
Make the REPL a thin UI adapter over orchestrator and durable workflow storage.

## Scope
Target files may include:
- `agent/repl.py`
- `agent/cli_view.py`
- workflow-related command handlers
- `agent/orchestrator.py`

## Required Changes
1. Remove workflow authority from REPL state.
2. Make workflow-related commands query durable state instead of in-memory flags.
3. Ensure REPL only renders state returned by orchestrator/storage APIs.
4. Keep UI-oriented conveniences only if they are non-authoritative.

## Acceptance Criteria
- REPL does not own workflow execution state.
- Workflow and approval commands resolve state from durable storage.
- UI behavior remains functional after REPL restart.
- Tests verify command behavior without relying on transient UI state.

## Out of Scope
- Artifact-level approval redesign
- History compaction persistence
