# Strengthen production guidance for `shell-mcp` sandbox configuration

## Summary
Make the production requirements for `shell-mcp` explicit in both configuration examples and operational guidance.

## Background
The current documents state that `shell_sandbox_backend = "none"` is the default, while production mode requires sandboxing and rejects `none` at startup. This is correct but still easy to misconfigure.

## Objective
Make the local-only nature of `sandbox_backend = "none"` unmistakable and reduce production rollout mistakes.

## In Scope
- `config/shell_mcp_server.toml` documentation examples
- `04_mcp_04_server_catalog.md`
- `04_mcp_05_security_and_safety_model.md`
- `04_mcp_06_configuration_and_operations.md`

## Required Changes
- Add an explicit local vs production configuration comparison
- Mark `sandbox_backend = "none"` as local-only / non-production
- Add a production-ready config example using `firejail`
- Clarify startup failure behavior for invalid production configuration
- Ensure security audit documentation references the same rule set

## Acceptance Criteria
- Readers can see the local vs production difference at a glance
- Production-safe configuration is shown directly in an example
- Startup failure conditions for unsafe production shell configuration are clearly stated

## Out of Scope
- Changing shell sandbox implementation
- Adding new sandbox backends

## Notes
Use direct operator language. Treat production guidance as normative, not optional advice.
