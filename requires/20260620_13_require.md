# Title
Add deterministic fallback and visible failure handling for session title generation

## Summary
Improve session title generation so that a failed title-generation task never leaves the session with an empty or unusable title, and make failures operator-visible.

## Background
Session title generation currently runs asynchronously in a fire-and-forget task.
If it fails, the error is logged, but no fallback title is created and no user-visible signal is provided.

## Problem
This creates silent failure and leaves session management quality dependent on a best-effort background operation.

## Required Changes
- Add a deterministic fallback title when LLM-based title generation fails.
- Use a simple and stable fallback such as the first user input truncated to a safe length.
- Surface title-generation failures through audit, debug, or warning output.
- Ensure session list views never show empty titles.

## Acceptance Criteria
- Every session has a non-empty title even if LLM title generation fails.
- Title-generation failures are visible to operators.
- Existing successful title generation behavior remains intact.

## Out of Scope
- Adding complex title revision workflows
- Redesigning session management end to end

## AI Implementation Instruction
Implement this as a reliability and UX hardening task.

Strict rules:
- Do not remove asynchronous title generation unless necessary.
- Add a deterministic fallback, not another best-effort async retry chain.
- Preserve current successful generation behavior.
- Make failure visible in at least one operator-facing path.
- If there is uncertainty about fallback title length or formatting rules, mark it as: Needs confirmation

Deliverables:
1. Deterministic fallback title behavior
2. Failure visibility in runtime or audit output
3. Validation that session listings never show empty titles
