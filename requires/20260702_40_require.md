# Update `tests/test_create_schema.py` for DDL-only schema creation

## Priority

High

## Summary

Update `tests/test_create_schema.py` so it no longer assumes backward-compatible migration helpers exist.

## Background

`test_create_schema.py` currently contains comments that refer to migration helpers and backward-compatible schema additions.

This conflicts with the DDL-only schema creation policy.

The patched session schema in the test also lacks `tool_results.undone`, while the actual schema and tool result tests use it.

## Goal

Make schema creation tests validate only latest DDL creation and idempotent `CREATE ... IF NOT EXISTS` behavior.

## Scope

### In scope

- Remove migration-related comments.
- Update patched test schemas.
- Align test DDL with canonical DDL where practical.

### Out of scope

- Testing old DB upgrades.
- Testing `ALTER TABLE`.
- Testing schema migration behavior.

## Implementation tasks

- Remove comments mentioning:
  - `_migrate_rag_schema()`
  - `_migrate_session_schema()`
  - backward-compatible schema additions
- Update `_RAG_SCHEMA_NO_VEC0` to include:
  - `chunk_type TEXT NOT NULL DEFAULT 'text'`
  - `source_file TEXT NOT NULL DEFAULT ''`
- Update `_SESSION_SCHEMA_NO_VEC0` to include:
  - `undone INTEGER NOT NULL DEFAULT 0`
- Consider adding `session_diagnostics` to `_SESSION_SCHEMA_NO_VEC0` if the patched test schema should mirror the latest session schema.

## Acceptance criteria

- Test comments no longer mention migration helpers.
- Test schemas include:
  - `chunks.chunk_type`
  - `chunks.source_file`
  - `tool_results.undone`
  - `messages.tool_call_id`
- Tests pass without any migration helper.

## Validation

- Run:
```
uv run pytest tests/test_create_schema.py
```

## Confirmation required

- Confirm whether `session_diagnostics` should be included in the patched no-vec session schema.
- Confirm whether workflow patched schema should create all five workflow tables.
