# [mcp][security][ops] Make fail-open and fail-closed security settings visible at startup

## Summary
Security settings are not all fail-closed, and the differences are easy to miss.

## Problem
Some settings deny everything when empty, while others allow everything when empty.
This is risky if operators assume all allowlists behave the same way.

## Scope
- startup security summary
- `/mcp` status display
- security documentation

## Task
- show fail-open / fail-closed behavior clearly at startup
- expose important empty-list behaviors in `/mcp`
- improve operator-facing documentation for security defaults

## Expected Result
Security-sensitive defaults are easier to understand and audit.

## Acceptance Criteria
- [ ] startup output includes security mode summary
- [ ] `/mcp` shows important allow/deny defaults
- [ ] fail-open settings are clearly visible
