## Background
The current Event Bus implementation and tests define behavior that is not fully reflected in the documentation.

Confirmed implementation behavior includes:
- offsets advance only through ack endpoints
- `/subscribe` does not write offsets on disconnect
- canonical ack endpoint exists at `POST /events/{event_id}/ack`
- compatibility ack endpoint exists at `POST /ack`
- DLQ requeue endpoint is `POST /dlq/{event_id}/requeue`
- `/replay?format=json` returns a paginated object
- `/health` returns HTTP 503 when degraded

## Scope
Update:
- `docs/06_eventbus_02_http_api_and_runtime.md`
- `docs/06_eventbus_04_dlq_offsets_and_delivery_semantics.md`
- `docs/06_eventbus_05_configuration_deploy_and_operations.md`

## Required Changes
- Remove any statement that `/subscribe` writes offsets on disconnect.
- Document ack-only offset semantics.
- Document `POST /events/{event_id}/ack` as the canonical ack endpoint.
- Document `POST /ack` as a compatibility or deprecated endpoint.
- Document `POST /dlq/{event_id}/requeue`.
- Document `/replay?format=json` response shape:
  - `total`
  - `limit`
  - `offset`
  - `items`
- Document `/health` semantics:
  - `ok` returns HTTP 200
  - `degraded` returns HTTP 503
- Update error cases based on implementation and the decisions from H-02 and H-03.

## Acceptance Criteria
- API docs match `scripts/eventbus/app.py`.
- API docs match current Event Bus tests.
- A client implementer can correctly implement publish, replay, subscribe, ack, nack, DLQ list, and DLQ requeue from the documentation alone.
