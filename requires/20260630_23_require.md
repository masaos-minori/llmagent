# [Low] Consider renaming `tool_constants.py` after routing cleanup

## Priority
Low

## Background
`tool_constants.py` currently mixes several concepts: registry seed data, side-effect classification, and tool policy grouping. After removing routing fallback, the filename may no longer describe its reduced role clearly.

## Goal
Evaluate whether `tool_constants.py` should be renamed or split after routing cleanup.

## Scope
Affected files:
- `shared/tool_constants.py`
- `shared/tool_registry.py`
- `shared/tool_executor.py`
- `agent/tool_policy.py`

## Implementation Tasks
- Do not rename immediately as part of fallback removal.
- After routing cleanup, reassess responsibilities of `tool_constants.py`.
- Consider splitting into:
  - `tool_registry_seed.py`
  - `tool_classification.py`
  - `tool_policy_sets.py`
- Ensure resolver never imports this module for routing.
- Document the intended responsibility if the file remains.

## Acceptance Criteria
- A follow-up decision is documented.
- Routing does not depend on `tool_constants.py`.
- If no rename is done, the module docstring clearly states that it is not a routing fallback source.

## Validation
- Run `rg -n "tool_constants.*routing|route_resolver.*tool_constants" shared docs tests`.
