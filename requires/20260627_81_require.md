# Document the responsibility boundary between `SessionMessageRepository` and `SQLiteSessionStore`

## Priority
Medium

## Labels
- architecture
- db
- session
- documentation
- medium-priority

## Dependencies
None

## Background
`SessionMessageRepository` performs role validation, strict-mode behavior, skip counters, JSON encoding/decoding, and session-aware persistence. `SQLiteSessionStore` is a lower-level DB adapter.

This boundary should be documented to avoid duplicate validation and inconsistent persistence behavior.

## Scope
Modify documentation:
- Shared/DB documentation for session persistence
- DB API documentation
- Any relevant architecture notes

No code changes required unless documentation reveals inconsistencies.

## Required Documentation
Document that `SessionMessageRepository` owns:
- Role validation
- Strict mode behavior
- No-session skip handling
- `content=None` normalization
- Tool call JSON encoding/decoding

Document that `SQLiteSessionStore` owns:
- Thin DB insert/list operations
- Schema-aligned persistence
- Minimal validation only

## Acceptance Criteria
- The repository/store responsibility boundary is explicit.
- Future developers know where validation belongs.
- No duplicate validation is introduced unnecessarily.
