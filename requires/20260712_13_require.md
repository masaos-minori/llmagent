# [Tooling][MCP] check_mcp_docs_consistency.py のツール数整合性チェックが分割後に機能停止している

## 背景

`tools/check_mcp_docs_consistency.py` の `check_tool_counts()` は、ドキュメント記載の
ツール数とコード側の正準定義(`frozenset`)との乖離を検出する仕組みである。過去にはこの
仕組みで実際に乖離を検出・修正した実績がある(git log commit `47e5c2bf`
"fix: correct MDQ_TOOLS count (7→9) and audit_log_path defaults across MCP docs")。

しかし commit `49154b4a`(docs split)でドキュメントが分割されて以降、
`check_tool_counts()` が対象とする `04_mcp_04_server_catalog.md` は存在しなくなり、
ツール一覧は `04_mcp_04_01_web-search-file-read-github.md`〜`04_mcp_04_05_git.md` の
5ファイルに分割された。関数は分割後のファイル構成を認識しないため、現在は毎回 WARNING を
出すだけで実質的に何も検証していない。

## 目的

ドキュメント記載のMCPツール数とコード側 `frozenset` 定義との乖離を検出する仕組みを、
分割後のファイル構成に対応させて復旧する。

## 現状(調査結果)

- `tools/check_mcp_docs_consistency.py:455-472` の `check_tool_counts()` は、`files` の中から
  `doc.rel_path == "04_mcp_04_server_catalog.md"` に一致するファイルを探索している(462行目)。
- 分割後、該当ファイルは存在しないため `catalog_file is None` となり、
  `04_mcp_04_server_catalog.md not found — cannot verify tool counts.` という
  WARNING(472行目)のみを返して、以降の検証ロジック(474行目以降、`_TOOL_COUNT_RE` による
  ツール数抽出・`frozenset` との突合)がスキップされる。
- 実行結果: `uv run python tools/check_mcp_docs_consistency.py` は
  `Found 2 warning(s)` のうちの1つとしてこれを報告するのみで、終了コードには影響しない(exit 0)。
- ツール一覧・コード側の正準 `frozenset` 定義自体(web-search-mcp, file-read-mcp,
  file-write-mcp, shell-mcp, github-mcp, rag-pipeline-mcp, cicd-mcp, mdq-mcp, git-mcp 等)は
  同ファイル内に残っている。

## 提案設計

1. `check_tool_counts()` が単一ファイル(`04_mcp_04_server_catalog.md`)を探す実装から、
   `04_mcp_04_01_*.md`〜`04_mcp_04_05_*.md` の5ファイルすべてを走査してツール数を
   抽出する実装に変更する。
2. サーバー名とファイルの対応関係(どのサーバーがどのファイルに記載されているか)を明示的に
   定義するか、全5ファイルを横断して `_TOOL_COUNT_RE` を検索する形にする。
3. 対応後、既存の `frozenset` 定義との突合が実際に走ることを確認する。

## 効果

- ドキュメントとコードのツール数乖離を、分割後も引き続き自動検出できるようになる。
- 過去に実績のある検出能力(MDQ_TOOLS 7→9 のような乖離検出)を復旧する。
