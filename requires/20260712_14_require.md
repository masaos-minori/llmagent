# [Docs][MCP] 04_mcp_03_03_transport-and-health.md がサイズ上限を超過している

## 背景

本プロジェクトのdocsは、LLM/RAGでの参照コストを抑えるため、1ファイルあたり8KB(8192 byte)を
上限として `-partN` 形式に分割する運用を取っている(`tools/validate_docs_structure.py` の
`MAX_SIZE = 8192` で機械的に検証)。`docs/04_mcp_03_03_transport-and-health.md` はこの上限を
超過している。

## 目的

サイズ上限を超過しているファイルを、既存の分割運用(part1/part2)に揃えて分割する。

## 現状(調査結果)

- `uv run python tools/validate_docs_structure.py` を実行すると、以下の1件が検出される:
  - `04_mcp_03_03_transport-and-health.md: size 8721 bytes exceeds 8192 byte limit`
- 実測: 8721 byte、138行(`wc -c` / `wc -l` で確認)。上限を約6.5%超過している。
- 同系列の `04_mcp_03_01_dispatch-and-routing.md`〜`04_mcp_03_05_lifecycle-and-new-server.md` は
  既に分割・命名済みであり、このファイルのみが `part1`/`part2` 分割の対象として残っている。

## 提案設計

1. `docs/04_mcp_03_03_transport-and-health.md` の内容を、既存見出し(H2/H3境界)を基準に
   `04_mcp_03_03_transport-and-health-part1.md` / `-part2.md` に分割する
   (他ファイルの分割慣習に合わせる)。
2. 分割後、front matter (`related:`/`source:`)を実ファイル名に揃える(別課題「front matter の
   壊れたリンク解消」と同様の不具合を新たに作らないよう注意する)。
3. 当該ファイルを参照している他ドキュメント(索引ドキュメント等)のリンクを、分割後の
   ファイル名に更新する。

## 効果

- docs全体のファイルサイズ運用ルールへの準拠。
- 既存の分割済みファイル群との一貫性維持。
