# Remove `/note` command registration from CommandRegistry

## Priority
High

## Labels
agent, commands, notes, cleanup, breaking-change

## Dependencies
None

## Background
The Agent currently registers `/note` as a built-in slash command through `CommandRegistry`. The command is implemented by `_NotesMixin` and supports note operations such as add, list, delete, pin, unpin, and search.

## Task
Remove the `/note` command group from the command registry.

## Implementation Requirements
- Remove the `/note` `CommandDef` from `registry.py`.
- Remove the `_NotesMixin` import from `registry.py`.
- Remove `_NotesMixin` from the `CommandRegistry` inheritance list.
- Update the module-level command group comment in `registry.py` to remove the `cmd_notes.py` entry.
- Do not affect `/memory` commands.

## Acceptance Criteria
- `/note add test` is no longer dispatched.
- `/note list` is no longer dispatched.
- `/note delete 1` is no longer dispatched.
- `/note pin 1`, `/note unpin 1`, and `/note search foo` are also no longer dispatched.
- `/memory ...` commands remain available.

## Source Context
`registry.py` currently registers `/note` and includes `_NotesMixin` in `CommandRegistry`. `_NotesMixin` owns the `/note` command group.
