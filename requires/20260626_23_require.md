# [Medium] Implement path filtering and limits for grep_docs

## Priority
Medium

## Labels
`mcp`, `mdq`, `grep`, `search`, `performance`

## Dependencies
- Add path allowlist and index root jail to mdq-mcp
- Redesign MDQ SQLite schema for production indexing

## Background
`grep_docs` currently scans all sections and ignores the optional `paths` filter. This is not suitable for large Markdown collections.

## Task
Implement production-safe `grep_docs`.

## Implementation Requirements
- Apply `paths` filtering.
- Add `max_matches`.
- Add `max_chars_per_match`.
- Add optional context line controls if feasible.
- Return chunk IDs with matches.
- Prevent unbounded regex scans.
- Handle invalid regex patterns as tool errors.

## Acceptance Criteria
- `paths` filter limits search scope.
- Large result sets are capped.
- Invalid regex returns `is_error=true` or a clear tool-level error message.
- Each match includes `chunk_id`, heading path, and source path.
