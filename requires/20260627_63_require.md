## Title
[Low] Add regression tests for Event Bus publish envelope schema contract

## Priority
Low

## Labels
tests, eventbus, schema, api-contract

## Dependencies
H-05

## Background
`schemas/event_envelope.json` defines a strict publish envelope contract:
- UUID v4 `event_id`
- required `topic`
- object-only `payload`
- required `producer`
- date-time `published_at`
- no additional properties

Some publish tests already validate invalid schema bodies, but more explicit schema contract tests would help prevent accidental drift.

## Scope
Update:
- Event Bus publish tests
- Schema validation tests

## Required Tests
Add tests for:
- invalid non-UUID-v4 `event_id`
- missing required field
- non-object `payload`
- additional unexpected property
- invalid `published_at`
- optional `schema_version`

## Acceptance Criteria
- Publish envelope schema behavior is explicitly covered.
- Documentation and schema contract remain aligned.
- Existing publish tests continue to pass.
