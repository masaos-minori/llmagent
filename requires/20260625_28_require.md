## [Medium][eventbus][schema] Split delivery-state fields into explicit counters and state markers

### Summary
The current schema mixes unrelated meanings into `retry_count` and leaves `acked_at` unused.

### Problem
Using one counter for multiple lifecycle meanings makes delivery semantics hard to reason about.
Unused `acked_at` also suggests that the delivery state model is incomplete.

### Scope
- schema.sql
- persistence model
- delivery state machine
- retry/DLQ semantics

### Task
- redesign the delivery state model in the schema
- separate concepts such as:
  - delivery failure count
  - DLQ requeue count
  - ack timestamp
  - last error or delivery state
- update code paths to use the new schema fields consistently
- add migration logic if required
- update tests for the new state model

### Expected Result
The persistence schema clearly represents delivery attempts, failures, acknowledgements, and requeue behavior.

### Acceptance Criteria
- retry-related counters no longer mix unrelated meanings
- `acked_at` is either used properly or replaced with a clearer state model
- code uses the new fields consistently
- tests cover the new schema semantics
