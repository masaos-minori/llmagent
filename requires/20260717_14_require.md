# Replace requires_config with config_dependent in MCP tool definitions

## Priority

High

## Target files

- mcp_servers/file/read_tools.py
- mcp_servers/file/write_tools.py
- mcp_servers/file/delete_tools.py
- mcp_servers/git/tools.py
- Other mcp_servers/*/tools.py files
- tests/mcp_servers/test_tool_schema.py

## Reason

`requires_config` is ambiguous and does not correctly represent runtime availability.

File tools currently use `requires_config: False`, but they still depend on `allowed_dirs`. When `allowed_dirs` is empty, all file operations are rejected.

Git tools currently use `requires_config: True`, but runtime availability depends on multiple configuration values:

- `allowed_repo_paths`
- `read_only`

A single boolean field cannot represent:

- whether the tool depends on server configuration
- whether the tool is currently executable
- why the tool is currently unavailable

## Intent

Remove the ambiguous legacy field and introduce a clear static metadata field for configuration dependency.

The new model is:

- `config_dependent`: static metadata that indicates whether the tool depends on MCP server configuration
- `enabled`: dynamic runtime metadata returned by `/v1/tools`
- `disabled_reason`: dynamic runtime metadata returned by `/v1/tools`

No compatibility period is required.

## Scope

Remove `requires_config` from all MCP tool definitions.

Add `config_dependent` to all MCP tool definitions.

For the currently reviewed file and git servers:

- all file tools should use `config_dependent: true`
- all git tools should use `config_dependent: true`

## Implementation notes

`config_dependent` belongs in `tools.py` because it is a static property of the tool.

Do not keep compatibility fallback for `requires_config`.

Schema validation should fail if `requires_config` is still present.

## Acceptance criteria

- No MCP tool definition contains `requires_config`.
- All MCP tool definitions include boolean `config_dependent`.
- File-read, file-write, file-delete, and git tools all use `config_dependent: true`.
- `/v1/tools` schema validation fails if `requires_config` is present.
- Runtime discovery does not read or depend on `requires_config`.
