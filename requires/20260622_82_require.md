# Title
Improve visibility of partial LLM completions in normal operator workflows

## Summary
Keep partial completions isolated from main conversation history, but make their existence much easier to notice during normal operation.

## Background
Partial completions are intentionally stored outside normal history to avoid polluting future LLM context.
This is useful, but it also makes partial responses easy to miss.

## Problem
Operators may not notice interrupted or incomplete responses unless they already know where to look.

## Required Changes
- Add a normal runtime indicator that a partial completion occurred in the session.
- Surface this in a common operator-facing view such as `/stats` or `/context`.
- Improve discoverability of the stored partial-completion artifact.

## Acceptance Criteria
- Partial completions are easier to detect during normal use.
- Diagnostic isolation from main conversation history remains intact.
- Operators can find the stored artifact without deep debug knowledge.

## Out of Scope
- Reinjecting partial assistant output into main history
- Redesigning SSE transport

## AI Implementation Instruction
Implement this as an observability and usability task.

Strict rules:
- Preserve diagnostic isolation.
- Do not merge partial outputs back into main history.
- Focus on discoverability, not transport redesign.
- If the operator-facing surface is unclear, mark it as: Needs confirmation

Deliverables:
1. Runtime indicator for partial completion
2. Better discoverability of stored artifacts
3. Preserved isolation of diagnostic content
