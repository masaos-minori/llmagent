# [Low] Add documentation consistency checks for MCP routing, audit, health, and transport invariants

## Goal

Prevent stale MCP documentation patterns from being reintroduced.

## Scope

Documentation checks only.

Update:

- `scripts/checks/check_docs_consistency.py`
- or equivalent documentation validation tooling

## Checks to Add

Flag active MCP docs when they contain stale statements such as:

- live discovery overrides registry routing
- `/v1/tools` is routing authority
- config `tool_names` is a routing input
- shared audit log uses `AUDIT session=...` key=value format
- `HttpTransport` returns `is_error=True` for transport failures
- stdio is active transport when removed
- ondemand is active startup mode when removed
- watchdog restarts on every dependency failure
- strict validation skips all-unreachable servers if implementation is fail-fast

## Acceptance Criteria

- Stale routing-authority wording is detected.
- Stale audit-format wording is detected.
- Stale transport-error wording is detected.
- Stale health/watchdog wording is detected.
- The checker has narrow allowlist exceptions only when explicitly justified.

## Validation

Run:

    python scripts/checks/check_docs_consistency.py

Add temporary stale examples in a scratch branch and confirm the checker fails.
Remove temporary examples and confirm the checker passes.
