# Validate import boundaries after removing compatibility stubs and shims

## Priority

Low

## Summary

Validate import boundaries after deleting compatibility-only stubs and shims.

## Background

Removing compatibility import paths changes import locations. Import boundaries must remain valid after:

- deleting `agent/config.py`
- deleting `mcp/github/service.py`
- removing package-level service re-exports from `mcp/github/__init__.py`
- removing `McpServerConfig._cast_enums()`

## Goal

Ensure the cleanup does not introduce layer violations or broken imports.

## Scope

### In scope

- Run import-linter.
- Run tests.
- Check GitHub MCP server importability.
- Check direct import paths.

### Out of scope

- Refactoring unrelated dependency boundaries.
- Changing MCP tool behavior.
- Changing agent config behavior.

## Implementation tasks

- Run the full test suite.
- Run import-linter.
- Ensure `mcp/github/server.py` still imports successfully.
- Ensure `test_mcp_server_base.py` app module importability test passes.
- Confirm `shared/mcp_config.py` does not import from upper layers.
- Confirm direct imports point to the correct modules.

## Acceptance criteria

- All tests pass.
- Import-linter passes.
- No import references deleted modules:
  - `agent.config`
  - `mcp.github.service`
- `mcp.github` package no longer re-exports service symbols.
- `shared/mcp_config.py` remains a shared-layer module with no upper-layer dependency.

## Validation

- Run:
```bash
uv run pytest
```
- Run:
```bash
PYTHONPATH=scripts uv run lint-imports
```
- Run:
```bash
grep -R "from agent.config import\|import agent.config" -n scripts tests docs
```
- Run:
```bash
grep -R "from mcp.github.service import\|import mcp.github.service" -n scripts tests docs
```
- Run:
```bash
grep -R "from mcp.github import GitHubService\|from mcp.github import build_service\|from mcp.github import _GITHUB_TOKEN" -n scripts tests docs
```
