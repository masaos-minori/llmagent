# [High] Document chunks_vec deletion order as a RAG design invariant

## Goal

Document the required deletion order for `chunks_vec` to prevent orphan vector records during force re-ingestion and document deletion.

## Background

`chunks_vec` is a sqlite-vec virtual table and does not have standard foreign key constraints. The docs state that force reinsertion and deletion must delete `chunks_vec` before deleting `chunks` or `documents`. 【1-54c80f】【5-63b173】

## Scope

Documentation first.

Implementation changes are required only if current delete/re-ingest logic violates the documented order.
