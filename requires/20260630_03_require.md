# [High] Make RAG MCP selected hit serialization dataclass-safe

## Priority
High

## Difficulty
Low

## Background
`RagPipelineMCPService._make_capture_fn()` serializes dataclass hits using `dataclasses.asdict()`, but `run_pipeline()` and `run_debug_pipeline()` build `selected_hits` with `dict(h)`.

Since `RawHit`, `MergedHit`, and `RankedHit` are dataclasses, `dict(h)` may fail.

## Goal
Use one safe hit-to-dict conversion path for all RAG MCP responses.

## Scope
Affected files:

- `scripts/mcp/rag_pipeline/service.py`
- related tests

## Implementation Tasks
- Add a helper function in `service.py` to convert hit objects into dictionaries.
- The helper must support:
  - dataclass hit objects
  - dictionary hit objects
- Replace all `dict(h)` conversions for RAG hits with the helper.
- Reuse the helper in:
  - `_make_capture_fn()`
  - `run_pipeline()`
  - `run_debug_pipeline()`

## Acceptance Criteria
- `RawHit`, `MergedHit`, and `RankedHit` serialize correctly in:
  - `selected_hits`
  - `merged_hits`
  - `reranked_hits`
- No `TypeError` occurs when `_fetch.hits` contains dataclass instances.
- Debug and non-debug RAG MCP responses use the same serialization behavior.

## Validation
- Add unit tests with dataclass hit instances.
- Add unit tests with dict hit instances if remote or HTTP mode can return dictionaries.
- Run RAG debug pipeline tests and assert JSON-serializable output.
