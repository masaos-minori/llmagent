# Title
Improve extraction quality to separate abstract rule memory from raw Q&A memory

## Priority
High

## Labels
- enhancement
- memory
- extraction
- semantic-memory
- episodic-memory

## Dependencies
- Issue 1 is optional but recommended first if extraction changes depend on local-only embedding guarantees.

## Summary
Improve rule-based extraction so the memory layer cleanly separates abstract rule memory from raw Q&A memory, while also capturing decision rationale and failure cases more accurately. The current design already has `semantic` / `episodic` memory types and `RULE / CONVERSATION / DECISION / FAILURE` source types, but extraction quality should be strengthened to match the intended Persistent Semantic Memory behavior.

## Background
The current memory design already supports:
- `semantic` and `episodic` memory types
- `RULE`, `CONVERSATION`, `DECISION`, and `FAILURE` source types
- rule-based extraction from conversation history
- summary, tags, and importance fields per memory entry

## Problem
The current extraction logic is structurally close to the goal, but it does not yet strongly guarantee high-quality separation between:
- abstract project rules / invariants
- design decisions / rationale
- failure patterns / workarounds
- raw question-and-answer memory

## Required Changes
- Strengthen extraction rules in `extract_memories()`.
- Explicitly detect:
  - project rules / invariants / policy → semantic + RULE
  - decisions / rationale / trade-offs → semantic + DECISION
  - failures / error handling / workaround knowledge → episodic + FAILURE
  - raw Q&A or conversational lessons → episodic + CONVERSATION
- Revisit how `summary`, `tags`, and `importance` are assigned by source type.
- Add tests that lock down extraction behavior for “code change intent” and “design rationale” cases.

## Acceptance Criteria
- Abstract rule memory and raw Q&A memory are distinguishable in retrieval and inspection.
- `RULE / DECISION / FAILURE / CONVERSATION` classification is covered by tests.
- Docs explain the separation clearly.

## Out of Scope
- Full LLM-based extraction redesign
- Changing the canonical storage model

## Components
- `scripts/agent/memory/extract.py`

## AI Implementation Instruction
Implement this as an **extraction-quality improvement task**.

Strict rules:
- Do not blur semantic vs episodic memory.
- Do not treat source type as a passive label; it must match extraction logic.
- Prefer incremental rule improvements over speculative redesign.
- If a classification boundary is unclear, mark it as: `Needs confirmation`
