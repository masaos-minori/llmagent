## [High][workflow][recovery] Restore pending approvals from durable workflow state after restart

### Summary
Pending approvals are persisted in workflow storage, but the command flow relies on transient in-memory state only.

### Problem
After REPL restart or after transient turn state is lost, pending approvals may become difficult or impossible to resolve from the CLI.

### Scope
- agent/context.py
- agent/commands/cmd_workflow.py
- agent/workflow/state_store.py
- workflow approval command UX

### Task
- load unresolved approvals from workflow storage
- allow approval commands to target durable approval IDs
- restore pending approval context on startup or on demand
- add a CLI path to list unresolved approvals
- remove the assumption that approval can only be resolved from `ctx.turn.pending_approval_id`

### Expected Result
Pending approvals remain actionable after restart and do not depend only on transient turn memory.

### Acceptance Criteria
- restart does not lose the ability to approve or reject a pending workflow
- `/approve` and `/reject` can resolve a durable approval ID
- unresolved approvals can be listed from workflow storage
- restart/recovery behavior is documented
