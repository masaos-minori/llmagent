# [MCP][Tools] Add readiness/status metadata to /v1/tools

## Summary
`/v1/tools` does not expose whether a tool is production-ready, partial, stub, experimental, or config-dependent.

## Problem
Clients cannot distinguish usable tools from placeholder or partially implemented tools.

## Scope
- `/v1/tools` output model
- per-server tool metadata

## Task
- extend tool metadata with optional fields such as:
  - `status`
  - `experimental`
  - `requires_config`
  - `stub`
- mark tools that are not fully ready
- preserve backward compatibility if needed

## Acceptance Criteria
- [ ] clients can distinguish ready vs stub/partial tools
- [ ] config-dependent tools are marked clearly
- [ ] current consumers are not broken by the new metadata
