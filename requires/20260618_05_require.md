# [mcp][shell][security] Expose shell sandbox state in health and operations views

## Summary
`shell-mcp` is described as sandboxed, but the actual backend may be `none`.

## Problem
Operators and users may assume shell execution is always sandboxed, even when no sandbox backend is active.

## Scope
- `shell-mcp`
- `/health`
- `/mcp` status display
- shell operational docs

## Task
- expose the active shell sandbox backend in `/health`
- show sandbox state in `/mcp` or equivalent status output
- update docs to reflect actual sandbox behavior clearly

## Expected Result
The real shell isolation state is visible and not misleading.

## Acceptance Criteria
- [ ] shell health output shows active sandbox backend
- [ ] operations view shows sandbox status
- [ ] documentation matches real runtime behavior
