# [RAG][Config] Make embedding dimension use one source of truth

## Summary
The runtime embedding dimension and the dataclass default do not match.

## Problem
Production uses 384 dimensions, but the code default is 768.
This is confusing and can cause wrong assumptions or broken behavior.

## Scope
- ingestion config model
- `scripts/rag/ingestion/ingester.py`
- `rag/utils.py`

## Task
- make the embedding dimension come from one clear source
- update the default if needed
- validate that the embedding API response dimension matches the configured dimension

## Expected Result
The configured embedding dimension is clear and consistent everywhere.

## Acceptance Criteria
- [ ] config default and runtime value match
- [ ] embedding response dimension is validated
- [ ] wrong dimension fails fast
