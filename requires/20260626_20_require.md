# [High] Treat index_paths and refresh_index as side-effect tools

## Priority
High

## Labels
`mcp`, `mdq`, `safety`, `tool-scheduling`, `concurrency`

## Dependencies
- Redesign MDQ SQLite schema for production indexing

## Background
`index_paths` and `refresh_index` modify `mdq.sqlite`. They are side-effect tools and must not be treated as ordinary read-only tools.

## Task
Add safety, scheduling, and concurrency controls for MDQ indexing tools.

## Implementation Requirements
- Mark `index_paths` and `refresh_index` as write or admin tools in tool safety tiers.
- Add tool metadata for serialization:
  - `requires_serial: true`, or
  - `resource_scope: "mdq_index"`
- Configure concurrency limit for the `mdq` server if needed.
- Ensure indexing does not race with another indexing operation.
- Define behavior for search while indexing is running.
- Invalidate relevant tool cache after index updates.

## Acceptance Criteria
- `index_paths` and `refresh_index` are not classified as read-only.
- Concurrent index operations are serialized.
- Search does not observe corrupted or partially written index state.
- Cache invalidation occurs after successful index refresh.
