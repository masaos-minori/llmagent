# [Medium] Wire live routing drift validation into startup warnings

## Priority

Medium

## Goal

Detect live /v1/tools ownership drift at startup.

## Background

Live discovery can override ToolRegistry routing. Drift validation against live server tool lists exists conceptually, but it is documented as not wired into startup.

## Scope

Affected files:

- shared/tool_registry.py
- agent/repl_health.py
- tests/test_repl_health.py
- docs/04_mcp_03_routing_lifecycle_and_execution.md

## Tasks

- Call validate_routing_against_live() during startup validation.
- Emit warnings for live-vs-registry owner mismatches.
- Keep this warning-only for now.
- Do not make unreachable servers fatal.
- Keep this separate from _check_tool_definitions.
- Document the distinction:
  - tool_definitions check: LLM-visible tools vs live server tools
  - routing drift check: live server ownership vs ToolRegistry

## Unknowns / Open Questions

- Confirm where live /v1/tools results are collected.
- Confirm whether discovery map is built before or after startup validation.
- Confirm desired strict behavior for future production mode.

## Acceptance Criteria

- Startup logs warn when live ownership differs from ToolRegistry.
- The agent still starts unless a separate strict policy is implemented.
- Docs explain this validation separately from tool definition validation.

## Validation

Run:
```bash
uv run pytest tests/test_repl_health.py tests/test_tool_registry.py -v
```
