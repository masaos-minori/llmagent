# Reflect degraded-but-reachable MCP health in HealthRegistry

## Priority

High

## Type

Reliability / Implementation / Observability

## Background

The watchdog periodically probes MCP server `/health` endpoints.

Current behavior distinguishes:

- HTTP 200 healthy response
- Unreachable server
- Reachable degraded server where restart is not recommended
- Reachable degraded server where restart is recommended

The health registry tracks per-server availability and can block dispatch when a server is unavailable.

## Problem

When a server is reachable but degraded and `restart_recommended=false`, the watchdog does not restart it. This is correct.

However, if HealthRegistry is not updated for this degraded state, the agent's internal health state may diverge from the server's actual health response.

## Goal

Represent reachable degraded MCP servers in the health registry and surface this state in operator output.

## Scope

Affected areas:

- `agent/repl_health.py`
- `shared/mcp_health.py`
- `McpServerHealthRegistry`
- `/mcp status`
- Watchdog tests

## Implementation Tasks

- Add a method such as `record_degraded(server_key, reason=None)` to `McpServerHealthRegistry`.
- In watchdog checks, call this method for reachable degraded servers.
- Preserve current behavior that `restart_recommended=false` does not trigger restart.
- Track degraded reason if available from `/health`.
- Expose degraded state in `/mcp status`.
- Ensure per-call transport successes can still restore health appropriately.

## Acceptance Criteria

- HTTP 200 health response records success.
- HTTP 503 with `operator_action_required=true` records degraded state.
- HTTP 503 with `restart_recommended=false` does not trigger restart.
- HTTP 503 with `restart_recommended=true` can trigger restart.
- Unreachable server records failure.
- `/mcp status` shows degraded reason when available.

## Validation Plan

Add tests for:

- Healthy server.
- Degraded server requiring operator action.
- Degraded server with restart recommended.
- Unreachable server.
- Malformed health body.
- Recovery from degraded to healthy.
- Interaction between watchdog state and per-call success/failure updates.

## Risks

Introducing an explicit degraded state may require minor changes to displayed status and health state transitions.

## Out of Scope

- Changing watchdog restart policy.
- Restarting operator-action-required servers.
- Documentation-only updates.
