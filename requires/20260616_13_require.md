# [MCP][Security] Require authentication in production MCP deployments

## Summary
Authentication is optional for all HTTP MCP servers, which is too weak for production defaults.

## Problem
`auth_token=""` disables Bearer authentication completely.

## Scope
- `McpServerConfig.auth_token`
- deployment profiles
- startup validation / warnings

## Task
- define a production rule that requires `auth_token` for HTTP MCP servers
- keep auth optional only for explicit local/dev profiles
- add warnings or hard failures for production mode without auth

## Acceptance Criteria
- [ ] production deployments do not expose unauthenticated MCP endpoints
- [ ] dev and production auth behavior are clearly separated
- [ ] startup behavior clearly indicates when auth is disabled
