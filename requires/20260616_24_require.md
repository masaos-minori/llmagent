# [Agent][Plugins][RAG] Isolate plugin pipeline hook failures from the RAG pipeline

## Summary
Pipeline post-stage hooks currently run inside the RAG pipeline context, and exceptions from hooks propagate into the pipeline execution path.

## Problem
A bad plugin hook can break a normal RAG request.
This makes extension failures too disruptive.

## Scope
- plugin pipeline stage execution
- RAG pipeline post-stage hook integration
- error handling and logging

## Task
- wrap plugin pipeline stage execution with error isolation
- when a hook fails, log the plugin name, hook name, and query context
- skip the failed hook and continue the pipeline if safe
- optionally add a strict mode for testing that re-raises hook errors

## Expected Behavior
- plugin hook failures do not break the full RAG pipeline in normal runtime
- plugin failures are still visible in logs and diagnostics

## Acceptance Criteria
- [ ] hook failures are logged with enough diagnostic context
- [ ] normal runtime can continue after a hook failure
- [ ] strict mode can still surface hook failures during testing
- [ ] docs reflect the new failure behavior
