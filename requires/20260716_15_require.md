## Reason for change

The system needs a strict boundary between transient model-input context and durable user conversation state.

History compression and memory extraction are long-lived mechanisms. If they receive temporary routing hints, the agent can persist stale classification decisions and carry them into unrelated future turns.

This issue is necessary to protect session correctness, memory quality, and deterministic context management.

## Design intent

Treat message lifecycle explicitly:

- Ephemeral mode hints are transient current-turn inputs only.
- Ephemeral messages must not be persisted.
- Ephemeral messages must not be summarized.
- Ephemeral messages must not be extracted into memory.
- Ephemeral messages must not survive into the next turn.
- History compression must use deterministic retention priority.
- Fallback truncation must remove lower-priority temporary messages before durable facts.
- Memory injection and mode classification must be distinguishable.

## Implementation instructions

### Ephemeral message lifecycle

- Confirm all creation points for `_ephemeral` messages.
- Confirm all cleanup points for `_ephemeral` messages.
- Ensure ephemeral messages are removed before the next turn.
- Ensure ephemeral messages are not saved to SQLite session storage.
- Ensure ephemeral messages are excluded from memory extraction.
- Ensure ephemeral messages are excluded from history compression summaries.
- Define `/undo` behavior for ephemeral messages.
- Distinguish mode-classification hints from memory-injection system messages.
- Add tests that ephemeral hints do not accumulate during repeated tool-loop calls.

### History compression and fallback truncation

- Represent message category and retention priority explicitly in code.
- Ensure ephemeral messages are removed or ignored before compression.
- Verify that `history_protect_turns` protects recent user/assistant turn pairs.
- Define fallback truncation order when LLM compression fails.
- Ensure factual system messages have higher retention priority than temporary or tool messages.
- Track fallback truncation count in stats.
- Ensure diagnostics or `/context` output exposes fallback truncation state where applicable.
- Add tests for compression success, compression failure, protected turns, fallback order, and ephemeral exclusion.

## Acceptance criteria

- Ephemeral messages are not persisted to session storage.
- Ephemeral messages are not included in memory extraction.
- Ephemeral messages are not summarized into compressed history.
- Ephemeral messages do not survive into the next turn.
- Repeated LLM/tool-loop iterations do not duplicate ephemeral hints.
- Compression and fallback truncation order is deterministic.
- Recent protected turns are preserved.
- Factual system messages are not removed before lower-priority temporary content.
- Tests cover persistence, compression, memory extraction, undo behavior, next-turn cleanup, and fallback truncation.

## Non-goals

- Do not redesign the entire memory system.
- Do not remove persistent system prompts.
- Do not change MDQ/RAG classification logic except where needed to enforce lifecycle boundaries.
- Do not change session schema unless separately scoped.
