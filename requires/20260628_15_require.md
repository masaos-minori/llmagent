# Remove `auto_inject_notes` from configuration

## Priority
High

## Labels
agent, config, notes, cleanup, breaking-change

## Dependencies
- Remove pinned notes injection from startup prompt setup

## Background
`auto_inject_notes` exists to control startup injection of persistent notes. After removing `/note` and note injection, this setting becomes obsolete.

## Task
Remove `auto_inject_notes` from the Agent configuration model and runtime config handling.

## Implementation Requirements
- Remove `auto_inject_notes` from `ToolConfig`.
- Remove config loading/building logic for `auto_inject_notes`.
- Remove `/reload` handling for `auto_inject_notes`, if any.
- Remove `/config` display output for `auto_inject_notes`.
- Remove `auto_inject_notes` from TOML config examples/defaults.
- Update tests that construct config objects or mocks with `auto_inject_notes`.

## Acceptance Criteria
- `ctx.cfg.tool.auto_inject_notes` no longer exists.
- `/config` output does not include note-related settings.
- `/reload` does not read or apply `auto_inject_notes`.
- Existing config tests pass after removing this field.

## Source Context
`startup.py` uses `ctx.cfg.tool.auto_inject_notes` to decide whether to inject pinned notes, and config tests currently set `ctx.cfg.tool.auto_inject_notes = True`.
