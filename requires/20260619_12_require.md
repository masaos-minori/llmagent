# Title
Add `dry_run` support to `create_directory`

## Summary
Extend `create_directory` so that it supports `dry_run` consistently with other file-writing tools.

## Background
Most write-oriented file tools already support `dry_run`, but `create_directory` does not.
This makes the safety model inconsistent across the file-write MCP server.

## Problem
`create_directory` is a state-changing operation but currently lacks the same preview path available to other write tools.

## Required Changes
- Add `dry_run` to the request model for `create_directory`.
- Add `dry_run` to the MCP tool schema.
- Implement preview-only behavior in the service layer.
- Document the preview behavior consistently with other write tools.

## Acceptance Criteria
- `create_directory` accepts `dry_run=True`.
- Preview mode performs validation and reports the planned action without creating directories.
- The tool schema, model, and service behavior are consistent.
- Security and server catalog docs no longer describe `create_directory` as a dry-run exception.

## Out of Scope
- Broader redesign of file-write approval logic.
- File permission model changes.

## Notes
Keep behavior aligned with existing dry-run conventions used by other file mutation tools.
