# [Agent][Observability] Persist runtime diagnostics and tool-result summaries for later analysis

## Summary
Important runtime diagnostics are session-local and in-memory, such as tool result storage and several runtime stats.

## Problem
This is useful during a live session, but it makes post-session analysis weaker.
The current observability setup focuses on audit logs and optional OTel, while many useful diagnostics disappear at session end.

## Scope
- `ctx.tool_result_store`
- runtime stats summary
- session-end persistence path
- observability outputs

## Task
- persist a lightweight summary of runtime diagnostics at session end
- include enough metadata for later troubleshooting, such as:
  - turn counts
  - tool call counts
  - partial completion count
  - selected tool result metadata
- keep full raw payload persistence optional to avoid storing too much sensitive data

## Expected Behavior
- important runtime diagnostics survive beyond the current session
- operators can inspect failures after the REPL exits
- observability is useful even without live interactive inspection

## Acceptance Criteria
- [ ] a session-end diagnostic summary is persisted
- [ ] stored data is useful for troubleshooting
- [ ] full raw payload persistence remains optional
- [ ] docs explain where runtime summaries are stored
