## Title
[Low] Separate deprecated Event Bus endpoints and config fields from active documentation

## Priority
Low

## Labels
documentation, cleanup, eventbus, deprecation

## Dependencies
H-04

## Background
Event Bus currently exposes both:
- canonical ack endpoint: `POST /events/{event_id}/ack`
- compatibility ack endpoint: `POST /ack`

The configuration also includes deprecated no-op fields:
- `poll_interval_ms`
- `offset_checkpoint_interval`

Documentation should clearly separate active APIs/configuration from compatibility or deprecated surfaces.

## Scope
Update:
- `docs/06_eventbus_02_http_api_and_runtime.md`
- `docs/06_eventbus_05_configuration_deploy_and_operations.md`
- `docs/06_eventbus_06_reference_api.md`

## Required Changes
- Mark `POST /ack` as compatibility or deprecated.
- Make `POST /events/{event_id}/ack` the canonical endpoint.
- Move deprecated config fields to a dedicated section.
- Document that non-default values emit `DeprecationWarning`.
- Avoid presenting deprecated fields as active tuning knobs.

## Acceptance Criteria
- New clients use the canonical ack endpoint.
- Deprecated config fields are not mistaken for active settings.
- Compatibility behavior remains documented.
