# Remove or isolate `IngestWorkflowService`

## Priority
High

## Labels
agent, services, rag, ingest, cleanup, dead-code

## Dependencies
- Remove `_cmd_ingest()` from the Agent command mixin

## Background
`IngestWorkflowService` orchestrates the crawl → split → ingest workflow for the `/ingest` command. After removing `/ingest`, this service may become unused from the Agent runtime path.

## Task
Determine whether `IngestWorkflowService` is still used outside `/ingest`. If not, delete it. If it is needed for a non-REPL workflow, move or document ownership outside the Agent command layer.

## Implementation Requirements
- Search for:
  - `IngestWorkflowService`
  - `IngestStageError`
  - `IngestOutcome`
  - `IngestStage`
- If references exist only in `/ingest` code and `/ingest` tests, delete `ingest_workflow.py`.
- If a standalone developer or operations workflow uses it, move it outside the Agent REPL command path.
- Do not delete the standalone RAG ingestion pipeline modules.

## Acceptance Criteria
- Agent REPL has no path that starts RAG ingestion.
- `agent.services.ingest_workflow` is not imported by Agent command code.
- If unused, `ingest_workflow.py` is deleted.
- If retained, it is clearly not owned by the Agent REPL command layer.

## Source Context
`ingest_workflow.py` imports and orchestrates `WebCrawler`, `ChunkSplitter`, and `RagIngester` for the `/ingest` command.
