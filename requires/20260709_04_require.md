# [Medium] Detect unknown tool_safety_tiers keys

## Labels
priority/medium, area/config-validation, area/tool-safety, production-hardening

## Summary
Add validation for unknown `tool_safety_tiers` keys.

The validator must detect entries that exist in `tool_safety_tiers` but do not exist in ToolRegistry.

## Background
Current safety tier validation detects tools registered in ToolRegistry but missing from `tool_safety_tiers`.

It does not detect the reverse case:

- `tool_safety_tiers` contains an old key
- the key is not a real registered tool
- the configuration appears valid but has no effect

This allows stale server-level keys such as `mdq` to remain unnoticed.

## Required Changes

### Validation
Add validation for:

- `tool_safety_tiers.keys() - ToolRegistry.get_all_tool_names()`

Behavior:

- local/development: warning
- production: RuntimeError

### Validator Placement
Use the new `ProductionConfigValidator` for production hard failures.

Optionally expose a reusable helper from `shared/tool_routing_validation.py`.

### Config Cleanup
- Remove or replace server-level keys such as `mdq`.
- Replace them with actual tool names if needed.

### Documentation
- Update `05_agent_08_configuration.md`:
  - `tool_safety_tiers` must use actual tool names.
  - Unknown keys are warnings in local/development.
  - Unknown keys are fatal in production.
- Update `04_mcp_05_security_and_safety_model.md` to mention unknown key validation.

### Tests
Add tests for:

- all keys known => valid
- unknown key in local => warning
- unknown key in production => error
- missing registered tool tier behavior remains unchanged

## Acceptance Criteria
- Unknown `tool_safety_tiers` keys are detected.
- Production fails fast on unknown safety tier keys.
- Local/development mode warns only.
- Existing missing-tier validation still works.
- Docs describe both missing and unknown key behavior.

## Target Files
- `shared/tool_routing_validation.py`
- `shared/production_config_validator.py`
- `agent/repl_health.py`
- `agent/services/mcp_status.py`
- `config/agent.toml`
- `tests/test_production_config_validator.py`
- `05_agent_08_configuration.md`
- `04_mcp_05_security_and_safety_model.md`

## Non-Goals
- Do not change risk tier enum values.
- Do not change default behavior for tools absent from `tool_safety_tiers`.
- Do not make local/development unknown keys fatal.
