# [Medium] Add cross-server TOOL_LIST export tests

## Priority

Medium

## Goal

Ensure all MCP tool schema modules use TOOL_LIST as the canonical export.

## Background

GitHub MCP already combines domain tool lists into a canonical TOOL_LIST. The MCP export policy requires all MCP tool schema modules to do the same.

## Scope

Affected files:

- mcp/*/tools.py
- scripts/mcp/*/tools.py
- tests/test_mcp_tool_schema_exports.py
- docs/04_mcp_07_tool_schema_export_policy.md

## Tasks

- Add a test that imports each MCP tool schema module.
- Assert each module exports TOOL_LIST.
- Assert TOOL_LIST is a list of tool schema dictionaries.
- Assert each tool has a name.
- Assert legacy exports such as _MCP_TOOLS are not used by active server modules.
- Use GitHub mcp/github/tools.py as the reference pattern.

## Unknowns / Open Questions

- Confirm the full list of MCP tool schema modules.
- Confirm whether file read, file write, and file delete servers use mcp/ or scripts/mcp/ paths.
- Confirm whether admin tools should be included in the same export policy.

## Acceptance Criteria

- All active MCP tool schema modules export TOOL_LIST.
- Legacy _MCP_TOOLS active usage is detected by tests.
- Server /v1/tools implementations can consistently use TOOL_LIST.

## Validation

Run:
```bash
uv run pytest tests/test_mcp_tool_schema_exports.py -v
rg -n "_MCP_TOOLS" mcp scripts tests docs
```
