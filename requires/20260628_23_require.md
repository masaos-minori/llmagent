# Remove `_cmd_ingest()` from the Agent command mixin

## Priority
High

## Labels
agent, commands, rag, ingest, cleanup

## Dependencies
- Remove `/ingest` command registration from CommandRegistry

## Background
`cmd_ingest.py` currently contains `_cmd_ingest()`, which parses `/ingest <url|path> [ja|en] [--snippets-only]`, creates an `IngestWorkflowService`, and runs the crawl/split/ingest workflow from the Agent REPL.

## Task
Remove the `/ingest` command handler from `cmd_ingest.py`.

## Implementation Requirements
- Delete `_cmd_ingest()` from `cmd_ingest.py`.
- Remove `/ingest` usage text.
- Remove `/ingest` argument parsing:
  - `target`
  - `lang`
  - `snippets_only`
- Remove lazy imports used only by `_cmd_ingest()`:
  - `IngestStageError`
  - `IngestWorkflowService`
- Remove the `on_status()` callback used only for `/ingest` progress output.
- Remove temporary `RagPipeline` construction used only to invalidate semantic cache after `/ingest`.
- Keep the following methods unchanged:
  - `_cmd_export`
  - `_cmd_compact`
  - `_cmd_rag`
  - `_print_rag_debug`

## Acceptance Criteria
- `_cmd_ingest()` no longer exists.
- `/export` still works.
- `/compact` still works.
- `/rag search <query> [--debug]` still works.
- `cmd_ingest.py` no longer imports ingest workflow code.

## Source Context
`cmd_ingest.py` currently implements `_cmd_ingest()` alongside `/export`, `/compact`, and `/rag` handlers.
