# [MCP][Config] Strengthen McpServerConfig typing with Literal values

## Summary
`McpServerConfig` uses plain `str` for fields that only allow a small fixed set of values.

## Problem
Invalid config values are not caught by static type checkers and are only detected at runtime.

## Scope
- `shared/mcp_config.py`

## Task
- change `transport` to `Literal["http", "stdio"]`
- change `startup_mode` to a matching `Literal[...]`
- change `healthcheck_mode` to a matching `Literal[...]`
- align runtime validation with the type annotations

## Expected Behavior
- invalid config values are caught earlier
- code completion and static checks become more useful

## Acceptance Criteria
- [ ] `transport` uses `Literal["http", "stdio"]`
- [ ] `startup_mode` uses a finite literal type
- [ ] `healthcheck_mode` uses a finite literal type
- [ ] validation logic still rejects invalid values at runtime
