# [Docs][RAG] 既知課題ドキュメント OPEN-02 が実装と乖離(stale) — セマンティックキャッシュ無効化は既に実装済み

## 背景

`docs/03_rag_90_inconsistencies_and_known_issues-part2.md` の OPEN-02(46〜66行目)は、
「`rag_delete_document` 実行後もセマンティックキャッシュが無効化されない」ことを
未解決の設計上の疑問点として記録している。しかし実装を確認したところ、既に対応済みで
あることが判明した。

## 目的

既知課題ドキュメントの記載を実装の現状に合わせて更新し、解決済みの問題を「未解決」として
誤って参照し続けることによる誤解・二重対応を防ぐ。

## 現状(調査結果)

- `scripts/mcp_servers/rag_pipeline/service.py` の `RagPipelineMCPService.fmt_delete_document()`
  (203〜214行)は、`self._doc_mgr.delete_document(url)` が `True` を返した場合に
  `self._pipeline_or_raise().invalidate_cache()` を呼び出している(210〜213行)。
- これは OPEN-02 が推奨していた対応(`_pipeline.semantic_cache.invalidate()` の呼び出し)と
  実質的に同じ効果を持つ実装であり、既に取り込まれている。
- 一方、同ドキュメントの OPEN-01(CLI取り込みがセマンティックキャッシュを無効化しない件)は
  `scripts/rag/ingestion/ingester.py:619` の `main()` が `ingester.ingest_all(args.force)` を
  `on_ingest_complete` なしで呼んでいることを確認済みで、これは現在も未解決のまま正しい。
  **OPEN-01 は変更しないこと。**

## 提案設計

1. OPEN-02 のエントリ本文は完全削除せず、`docs/05_agent_90_inconsistencies_and_known_issues.md`
   の UNDOC-02 の前例(「2026-07-09に削除された。理由は〜」という短いイタリック注記に
   置き換える形式)に倣い、短い解決済み注記に置き換える。
   - 前例の文体: `*(現在追跡中の...はない。UNDOC-02「...」は2026-07-09に削除された。... もはや
     存在しない。)*`
   - OPEN-02 用の注記案: `*(OPEN-02「delete_document()はセマンティックキャッシュを無効化しない」は
     解決済み。scripts/mcp_servers/rag_pipeline/service.py の fmt_delete_document()(210〜213行)が
     削除成功時に self._pipeline_or_raise().invalidate_cache() を呼び出しており、このエントリが
     指摘していたギャップはもはや存在しない。)*`
2. 更新の際は、対応済みの実装箇所(`service.py:210-213`)を根拠として明記する。
3. OPEN-01 は変更しない。

## 効果

- 既に解決済みの問題を「未解決」として参照し続けることによる、将来の実装者・AIエージェントの
  誤解・重複調査コストを防ぐ。
