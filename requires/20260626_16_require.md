# [High] Replace line-based Markdown parser with hierarchy-aware parser

## Priority
High

## Labels
`mcp`, `mdq`, `parser`, `markdown`, `context-compression`

## Dependencies
- Add path allowlist and index root jail to mdq-mcp

## Background
The current parser splits sections using simple `line.startswith("#")` logic. This is not sufficient for production Markdown parsing because it can mis-handle code fences, heading hierarchy, frontmatter, and line ranges.

## Task
Implement a production-ready Markdown section parser.

## Implementation Requirements
The parser must produce section records with:
- heading
- heading level
- heading path
- content
- start line
- end line
- ordinal among same-level headings
- parent heading context

The parser must correctly handle:
- fenced code blocks
- content before the first heading
- repeated heading names
- nested heading hierarchy
- optional frontmatter
- malformed headings

Setext heading support may be implemented or explicitly documented as unsupported.

## Acceptance Criteria
- Code fence lines beginning with `#` are not treated as headings.
- Nested heading paths are generated correctly.
- Repeated headings produce distinct chunk identities.
- Parser output includes line ranges.
- Unit tests cover common Markdown structures.
