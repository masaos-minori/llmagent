# Add tests to lock down /reload configuration classification

## Priority

Medium

## Type

Implementation / Test

## Background

The agent supports `/reload` for configuration changes. Each setting belongs to one of the following categories:

- Hot-reloadable
- Deferred
- Restart-required
- Startup-only
- Skipped

Runtime services are updated only for settings that are safe to apply during a running session.

## Problem

If startup-only settings are applied at runtime, the process may enter an inconsistent state.

If hot-reloadable settings are not applied, operators may believe changes took effect when they did not.

## Goal

Add automated tests that freeze the intended reload behavior.

## Scope

Affected areas:

- `agent/services/config_reload.py`
- `shared/config_loader.py`
- `AgentConfig`
- `LLMClient.apply_config`
- `HistoryManager.apply_config`
- `ToolExecutor.apply_config`
- `/reload` command tests

## Implementation Tasks

- Add tests for hot-reloadable LLM settings.
- Add tests for hot-reloadable context settings.
- Add tests for hot-reloadable tool settings.
- Add tests for deferred MCP settings.
- Add tests for restart-required MCP changes.
- Add tests for startup-only settings.
- Snapshot `ConfigReloadOutcome`.

## Acceptance Criteria

The following classifications are tested:

- `llm_temperature`: hot-reloadable
- `llm_max_tokens`: hot-reloadable
- `context_char_limit`: hot-reloadable
- `tool_cache_ttl`: hot-reloadable
- MCP `auth_token`: deferred
- MCP `startup_mode`: deferred
- MCP transport change: restart-required
- New MCP server: restart-required or skipped
- `use_memory_layer`: startup-only
- `workflow_mode`: startup-only
- `workflow_require_approval`: startup-only
- `plugin_strict`: startup-only

## Validation Plan

- Unit-test `ConfigReloadService.apply_config_dict()`.
- Verify service instances are updated only for hot-reloadable settings.
- Verify startup-only settings are reported but not applied.
- Verify deferred settings are stored but not applied to existing connections.
- Verify restart-required changes are reported clearly.

## Risks

Tests may reveal currently ambiguous reload behavior. Prefer explicit classification over implicit behavior.

## Out of Scope

- Changing config file format.
- Implementing live activation of startup-only subsystems.
- Documentation-only updates.
