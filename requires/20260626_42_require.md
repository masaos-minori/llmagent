# Title
Remove Agent session tables from the RAG data model documentation

## Priority
High

## Labels
- bug
- rag
- data-model
- documentation
- responsibility-boundary

## Dependencies
- None

## Summary
Fix the RAG data model documentation so it only describes RAG-owned tables and does not present Agent session tables as part of `rag.sqlite`.

## Background
The RAG system overview defines Agent REPL as out of scope and states that the Agent uses RAG through MCP. The RAG data model currently includes `sessions` and `messages`, which belong to the Agent/session layer.

## Problem
Mixing Agent session tables into the RAG data model blurs DB ownership and may cause incorrect schema initialization, maintenance behavior, or documentation-driven implementation mistakes.

## Required Changes
- Remove `sessions` and `messages` from the RAG-owned table list.
- Keep the RAG data model focused on:
  - `documents`
  - `chunks`
  - `chunks_fts`
  - `chunks_vec`
- Add a cross-reference to the Agent/session DB schema if needed.
- Clarify that the RAG layer does not own Agent conversation history.
- Update known issues if this was an existing documentation inconsistency.

## Acceptance Criteria
- RAG docs no longer imply that `sessions` and `messages` are RAG-owned tables.
- DB ownership boundaries are explicit.
- RAG schema initialization cannot be misread as needing Agent session tables.

## Out of Scope
- Changing the actual Agent session schema.
- Changing MCP access behavior.
