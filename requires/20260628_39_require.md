# Update debug command tests after `/debug audit` removal

## Priority
Medium

## Labels
tests, agent, commands, debug, audit

## Dependencies
- Remove `/debug audit` branch from debug command handler
- Make unknown `/debug` subcommands explicit validation errors

## Background
Tests should no longer expect `/debug audit` to display audit log contents. Existing debug tests for `/debug verbose` and `/debug normal` should remain valid.

## Task
Update or add debug command tests for the new behavior.

## Implementation Requirements
- Remove any success-path tests for `/debug audit`.
- Add tests verifying:
  - `/debug audit` is rejected.
  - `/debug audit` does not read the audit log.
  - `/debug audit` does not toggle debug mode.
  - `/debug verbose` still works.
  - `/debug normal` still works.
  - `/debug` still toggles debug mode.
- If debug command tests are currently mixed into unrelated test files, consider moving them to a dedicated debug command test file.

## Acceptance Criteria
- No test expects `/debug audit` to succeed.
- Tests confirm `/debug audit` is invalid.
- Existing `/debug verbose|normal` behavior remains covered.
- Test suite passes.

## Source Context
The uploaded tests include audit command tests and a debug-log-level test class mixed into `test_agent_rag.py`; audit browsing tests should remain under `/audit`.
