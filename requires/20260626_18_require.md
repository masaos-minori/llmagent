# [High] Introduce stable chunk IDs for MDQ chunks

## Priority
High

## Labels
`mcp`, `mdq`, `chunk-id`, `indexing`, `api`

## Dependencies
- Redesign MDQ SQLite schema for production indexing

## Background
The current `get_chunk` API uses an integer SQLite row ID. This ID changes after re-indexing and is not suitable for stable references, caching, audit logs, or reproducible retrieval.

## Task
Replace integer row IDs with stable MDQ chunk IDs.

## Implementation Requirements
- Generate stable `chunk_id` values from deterministic inputs.
- Recommended inputs:
  - normalized source path
  - heading path
  - ordinal
  - content hash
- Update `search_docs` to return `chunk_id`.
- Update `get_chunk` to accept stable `chunk_id`.
- Update `grep_docs` to return `chunk_id`.
- Keep backward compatibility only if needed, but prefer stable IDs.

## Acceptance Criteria
- Re-indexing unchanged files preserves chunk IDs.
- Search results can be passed directly to `get_chunk`.
- `get_chunk` no longer depends on autoincrement row IDs.
- Tests verify chunk ID stability across refreshes.
