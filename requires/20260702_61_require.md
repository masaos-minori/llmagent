# H-1: Fix exact vs prefix command dispatch argument handling

## Priority

High

## Labels

bug, agent, cli, command-registry, startup-blocker

## Background

`CommandRegistry.dispatch()` currently passes an empty string argument to exact-match commands.

This breaks exact commands whose handlers take no argument, such as `_cmd_help()`.

Current failing behavior:

- User runs `/help`
- `/help` matches `CommandDef("/help", prefix=False, is_async=False, handler="_cmd_help", ...)`
- `dispatch()` calls `handler("")`
- `_cmd_help(self)` receives an unexpected argument
- Runtime error: `TypeError: CommandRegistry._cmd_help() takes 1 positional argument but 2 were given`

## Goal

Fix command dispatch so that:

- `prefix=False` commands are called with no argument
- `prefix=True` commands are called with the remaining argument string
- async handlers are awaited
- sync handlers are called normally

## Scope

### In scope

- Update `scripts/agent/commands/registry.py`
- Fix `CommandRegistry.dispatch()`
- Preserve built-in command priority over plugin commands
- Preserve existing prefix boundary matching behavior

### Out of scope

- `/exit` handling
- sqlite-vec path handling
- large command registry redesign

## Required behavior

For built-in commands:

- Exact command:
  - Match: `line == cmd.name`
  - Call:
    - `await handler()` when `cmd.is_async=True`
    - `handler()` when `cmd.is_async=False`

- Prefix command:
  - Match: `line == cmd.name or line.startswith(cmd.name + " ")`
  - Args:
    - `args = line[len(cmd.name):].strip()`
  - Call:
    - `await handler(args)` when `cmd.is_async=True`
    - `handler(args)` when `cmd.is_async=False`

## Implementation notes

Update the exact-match branch from passing `""` to passing no argument.

Do not change plugin command dispatch in this task.

## Acceptance criteria

- `/help` works without `TypeError`
- `/config` works

- Prefix commands such as `/db help`, `/mcp`, `/rag search test`, and `/mdq status` still receive argument strings correctly
- Built-in commands still take precedence over plugin commands
