# [High] Use ToolRegistry for plugin tool conflict detection

## Priority
High

## Background
Plugin conflict detection receives a known MCP tool set through `plugin_registry.load_plugins(known_tools=...)`. The factory currently imports `get_all_mcp_tool_names` from `shared.tool_constants`, which means conflict detection likely depends on `tool_constants.py` instead of `ToolRegistry`. 【4-17ef54】【1-ad543e】

## Goal
Detect plugin-vs-MCP tool conflicts using `ToolRegistry`.

## Scope
Affected files:
- `agent/factory.py`
- `shared/plugin_registry.py`
- `shared/tool_registry.py`
- `tests/test_plugin_registry.py`
- `docs/05_agent_11_extension-points.md`

## Implementation Tasks
- Replace `tool_constants.get_all_mcp_tool_names()` usage with `ToolRegistry`-based tool enumeration.
- Pass `ToolRegistry.get_all_tool_names()` or equivalent into `plugin_registry.load_plugins(known_tools=...)`.
- Ensure GitHub 21 tools are included in conflict detection.
- Preserve existing behavior:
  - `plugin_tool_override=false`: reject plugin tools that conflict with MCP tools.
  - `plugin_tool_override=true`: allow plugin tools to shadow MCP tools.
- Add plugin conflict tests for representative GitHub tools.
- Update docs to say plugin conflict detection uses `ToolRegistry`.

## Acceptance Criteria
- Plugin conflict detection no longer depends on `tool_constants.py`.
- Plugin conflicts are detected for all registered MCP tools, including GitHub tools.
- Existing override policy behavior is preserved.
- Tests pass.

## Validation
- Run: `pytest tests/test_plugin_registry.py -v`
- Run: `rg -n "known MCP tool names.*tool_constants|tool_constants.*plugin|load_plugins.*known_tools" agent shared docs tests`
