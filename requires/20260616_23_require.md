# [Agent][Plugins][Tools] Prevent plugin tools from silently shadowing MCP tools

## Summary
A plugin tool can use the same name as an MCP tool and will take priority over the MCP tool for the whole session.

## Problem
This makes plugin tools very powerful, but it also creates a high risk of accidental override or hidden behavior changes.
The current priority order checks plugin tools before cache and before MCP dispatch.

## Scope
- `shared/plugin_registry.py`
- `shared/tool_executor.py`
- plugin registration rules

## Task
- add an explicit override policy for plugin tools
- by default, reject registration when a plugin tool name conflicts with a known MCP tool
- allow override only if a dedicated config flag is enabled
- log all accepted overrides clearly at startup

## Expected Behavior
- plugin tools cannot silently replace MCP tools
- override behavior is explicit, auditable, and configurable

## Acceptance Criteria
- [ ] conflicting plugin tool names are rejected by default
- [ ] explicit override mode exists
- [ ] accepted overrides are logged clearly
- [ ] docs explain the precedence rule and override policy
