# H-8: Remove /tool list and /tool show while preserving /plan

## Priority

High

## Background

The `/tool` command is a UI wrapper around `ToolResultStore`. It supports `/tool list` and `/tool show <id>`. This command must be removed.

However, `/plan` is currently implemented in the same mixin. `/plan` must remain available.

## Target files

- `agent/commands/command_defs_list.py`
- `agent/commands/cmd_tooling.py`
- `agent/commands/registry.py`
- optionally `agent/commands/cmd_plan.py`

## Implementation tasks

- Remove `CommandDef("/tool", ...)` from `command_defs_list.py`.
- Remove `_cmd_tool()`.
- Remove `_tool_list()`.
- Remove `_tool_show()`.
- Remove `/tool`-only helper code:
  - `DecodedArgs`
  - `_decode_args()`
  - `_to_tool_result_view()`
  - `ToolResultRow` import
  - `ToolResultView` import
  - `/tool` usage strings
- Preserve `_cmd_plan()`.
- Either:
  - keep `_cmd_plan()` in a slimmed-down `_ToolingMixin`, or
  - move `_cmd_plan()` to a new `_PlanMixin`.
- Update `registry.py` mixin comments, imports, and base class list if a new mixin is introduced.
- Ensure `/help` no longer lists `/tool`.

## Acceptance criteria

- `/tool`, `/tool list`, and `/tool show` are no longer built-in commands.
- `/plan` still works.
- CommandRegistry fail-fast validation passes.
- `/help` output does not mention `/tool`.
- Tab completion no longer includes `/tool`.

## Test plan

- Dispatch `/tool list` and verify it is not handled as a built-in command.
- Dispatch `/tool show 1` and verify it is not handled as a built-in command.
- Dispatch `/plan` and verify plan mode toggles.
- Run `/help` and verify `/tool` is absent and `/plan` is present.
