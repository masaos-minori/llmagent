# [Medium] Formalize config/mdq_mcp_server.toml

## Priority
Medium

## Labels
`mcp`, `mdq`, `configuration`, `operations`

## Dependencies
- Add path allowlist and index root jail to mdq-mcp
- Enforce result size limits for MDQ tools

## Background
MDQ currently hardcodes the database path and lacks formal production configuration for paths, limits, and indexing behavior.

## Task
Define and load `config/mdq_mcp_server.toml`.

## Implementation Requirements
Add configuration fields for:
- `db_path`
- `allowed_dirs` or `index_roots`
- `include_globs`
- `exclude_globs`
- `max_results_limit`
- `max_chars_per_chunk`
- `max_total_result_chars`
- `max_outline_items`
- `max_grep_matches`
- `enable_grep`
- `enable_refresh`
- `audit_log_path`

## Acceptance Criteria
- MDQ service reads configuration from `config/mdq_mcp_server.toml`.
- No production path or limit is hardcoded.
- Missing config results in safe defaults.
- Empty allowlist denies indexing and file parsing.
