# [High] Convert MCP server audit logging to JSON-lines format

## Goal

Unify MCP server audit logs with the agent-side JSON-lines audit format.

## Background

MCP server audit logging currently emits key=value lines such as `AUDIT session=... request=... action=...`. Agent-side audit logging already writes structured JSON-lines. MCP server audit logs should use JSON-lines as well.

## Scope

- Modify MCP audit helper implementation.
- Update MCP server/service call sites if needed.
- Do not add per-server `audit.py` files.

The following files do not exist and must not be introduced for this change:

- `scripts/mcp/github/audit.py`
- `scripts/mcp/shell/audit.py`
- `scripts/mcp/file_delete/audit.py`
- `scripts/mcp/mdq/audit.py`

## Implementation Tasks

- Update `mcp/audit.py`.
- Replace key=value `AUDIT ...` output with JSON-lines.
- Use a distinct MCP server event name, for example `mcp_tool_exec`.
- Include `source = "mcp_server"` to distinguish MCP-side audit events from agent-side `tool_exec`.
- Include these fields:
  - `event`
  - `source`
  - `session_id`
  - `request_id`
  - `server_key`
  - `tool`
  - `target`
  - `outcome`
  - `error_type`
  - `detail`
  - `ts`
- Update MCP server/service call sites that invoke `_audit_log()`.
- Update tests expecting the old key=value format.

## Suggested Event Shape

    {
      "event": "mcp_tool_exec",
      "source": "mcp_server",
      "session_id": "42",
      "request_id": "abc-123",
      "server_key": "file_read",
      "tool": "read_text_file",
      "target": "/path/to/file",
      "outcome": "ok",
      "error_type": "",
      "detail": "",
      "ts": 1719500000.0
    }

## Acceptance Criteria

- MCP audit logs are valid JSON-lines.
- No active MCP audit output starts with `AUDIT session=`.
- MCP server audit events are distinguishable from agent-side audit events.
- Existing correlation fields are preserved:
  - `session_id`
  - `request_id`
  - `server_key`
  - `tool`
- Docs no longer describe mixed key=value and JSON-lines audit formats.

## Validation

- Add or update tests that parse MCP audit output as JSON.
- Run MCP service tests.
- Manually trigger one MCP tool call and confirm the audit line is valid JSON.
