# [High] Promote mdq-mcp from experimental/stub to production status

## Priority
High

## Labels
`mcp`, `mdq`, `production-readiness`, `metadata`, `documentation`

## Dependencies
None

## Background
`mdq-mcp` is currently marked as experimental/stub in tool metadata and documentation. For production use, this status must be removed or replaced with an explicit production-ready status.

## Task
Update `mdq-mcp` metadata and documentation so that it is treated as a production MCP server.

## Implementation Requirements
- Update all MDQ tool metadata in `tools.py`.
- Remove or replace `status: "stub"` with a production-ready value.
- Remove `Experimental` wording from tool descriptions.
- Remove root-level `stub: true` from the MDQ health response.
- Update MCP documentation that currently describes `mdq-mcp` as experimental or not production-ready.
- Ensure the server catalog describes MDQ as a production Markdown context compression server.

## Acceptance Criteria
- No MDQ tool is exposed with `status: "stub"`.
- `/health` no longer returns `stub: true`.
- Documentation no longer instructs users to treat MDQ as experimental.
- Existing tool names remain unchanged:
  - `search_docs`
  - `get_chunk`
  - `outline`
  - `index_paths`
  - `refresh_index`
  - `stats`
  - `grep_docs`
