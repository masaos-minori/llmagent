# [Medium] Redesign memory source-of-truth model

## Summary
Fix the source-of-truth model for memory persistence so delete/prune/rebuild semantics are consistent.

## Problem
The memory layer currently mixes:
- append-only JSONL as source of truth
- mutable SQLite state
- delete/prune/rebuild operations

This can lead to rebuild inconsistencies and resurrection of deleted memory entries.

## Goal
Choose and implement one consistent source-of-truth model.

## Scope
Target files may include:
- `agent/memory/jsonl_store.py`
- `agent/memory/store.py`
- `agent/memory/services.py`
- `agent/memory/ingestion.py`
- memory commands/tests

## Required Changes
Choose one direction and implement it consistently:
1. Keep JSONL as source of truth and add tombstones/event semantics, or
2. Make SQLite the source of truth and treat JSONL as export/archive

Also:
- ensure rebuild behavior respects delete/prune state
- add regression tests

## Acceptance Criteria
- Deleted memory entries do not reappear after rebuild.
- Source-of-truth ownership is explicit.
- Commands and rebuild logic follow the same model.
- Tests cover delete/prune/rebuild consistency.

## Out of Scope
- Embedding retrieval algorithm redesign
- New memory scoring heuristics
