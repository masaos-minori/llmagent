# Simplify plugin example score filter to avoid unreachable code

## Priority

Low

## Labels

plugin, example, cleanup, developer-experience

## Background

The example plugin contains a post-rerank score filter that is disabled by returning `hits` before the filtering return statement.

This works as a template, but it leaves intentionally unreachable code in the example.

A clearer template would use an explicit feature flag.

## Goal

Make the example plugin easier to read and safer to copy.

## Scope

### In scope

- Update `plugins/example.py`
- Replace unreachable return pattern with an explicit enable flag
- Preserve default disabled behavior

### Out of scope

- Changing plugin hook behavior
- Changing plugin registry behavior
- Changing production plugin loading

## Suggested implementation

Introduce a flag such as:

- `ENABLE_SCORE_FILTER = False`

Then implement:

- If disabled, return hits unchanged
- If enabled, return filtered hits

## Acceptance criteria

- `plugins/example.py` contains no intentional unreachable return
- Score filtering remains disabled by default
- The example remains easy to copy and modify
- Plugin loads successfully

## Validation

Run:

- `uv run python -m agent`

Expected result:

- Example plugin loads without import error
- No behavior change unless score filter is explicitly enabled

## Affected files

- `plugins/example.py`

## Dependencies

None
