# Title
Remove backward-compatibility re-exports from `utils.py`

## Summary
Delete legacy re-export paths from `scripts/agent/commands/utils.py` and update callers to import from the canonical module.

## Background
The refactoring plan explicitly states that backward-compatibility layers should be removed.
The `utils.py` re-export block is listed as one of those legacy layers.

## Problem
Legacy re-exports hide ownership, create maintenance noise, and make imports less explicit.

## Required Changes
- Remove the re-export import block from `utils.py`.
- Remove the re-exported names from `__all__`.
- Update any callers still importing those functions from `agent.commands.utils`.
- Use the canonical source module directly.

## Acceptance Criteria
- The backward-compatibility re-export block is removed.
- Callers import from the correct canonical module.
- No stale compatibility import path remains.

## Out of Scope
- Changing the behavior of export formatting functions
- Broader cleanup of unrelated utility functions

## Components
- `scripts/agent/commands/utils.py`
- Call sites importing those re-exported helpers

## AI Implementation Instruction
Implement this as a cleanup and ownership-clarity task.

Strict rules:
- Do not preserve compatibility shims unless a hard dependency still exists and is documented.
- Update all affected imports, not just the source file.
- If a caller cannot be safely migrated in the same change, mark it as: `Needs confirmation`
