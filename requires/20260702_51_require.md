# Clarify `rotate_db()` vs `rotate_all_dbs()` documentation

## Priority

Low

## Summary

Clarify the difference between `rotate_db()` and `rotate_all_dbs()` in DB maintenance documentation.

## Background

`rotate_db()` archives `rag.sqlite` and `session.sqlite`.

`rotate_all_dbs()` archives:

- `rag.sqlite`
- `session.sqlite`
- `workflow.sqlite`

## Goal

Avoid operational confusion when archiving DBs before recreation.

## Target files

- `90_shared_05_db_api_and_operations.md`
- Optionally `maintenance.py` docstrings

## Scope

### In scope

- Documentation update.
- Naming clarification.
- Recommendation for schema recreation workflow.

### Out of scope

- Renaming functions unless separately approved.
- Removing `rotate_db()`.

## Implementation tasks

- Document:
  - `rotate_db()` archives only rag/session DBs.
  - `rotate_workflow_db()` archives workflow DB only.
  - `rotate_all_dbs()` archives all three DBs.
- Recommend `rotate_all_dbs()` before DB recreation.
- Add a note that `rotate_db()` is a rag/session-specific helper.

## Acceptance criteria

- DB recreation docs recommend `rotate_all_dbs()`.
- Function scope is unambiguous.
- No doc implies `rotate_db()` covers workflow.

## Validation

- Search docs and code:
```bash
grep -R "rotate_db\|rotate_all_dbs\|rotate_workflow_db" docs scripts/db/maintenance.py -n
```

## Confirmation required

- Confirm whether `rotate_db()` should remain or be renamed in a separate issue.
