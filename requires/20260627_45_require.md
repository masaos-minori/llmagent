## Title
[Low] Document workflow approval recovery behavior covered by tests

## Priority
Low

## Labels
documentation, workflow, tests, operations

## Dependencies
None

## Background
Workflow command tests explicitly cover cross-session recovery behavior: approval records can be resolved from the database even when in-memory approval state is empty.

This behavior should be documented as an operational guarantee.

## Scope
Update:
- Workflow operations documentation
- Known issues / resolved behavior documentation if present

## Required Changes
- Document that `/approve` and `/reject` resolve the latest pending approval from the workflow database.
- Document that in-memory `pending_approval_id` is not required for approval resolution.
- Mention that startup recovery restores pending state for user visibility.

## Acceptance Criteria
- Workflow approval recovery behavior is documented.
- Operators understand how approval after restart works.
