# Normalize prefix command arguments in `CommandRegistry.dispatch()`

## Priority

Medium

## Labels

bug, cleanup, agent, cli, command-registry

## Background

Built-in prefix command arguments are currently sliced from the original user input.

For an input such as `/db help`, the argument string can include a leading space.

Many handlers defensively call `strip()`, but the dispatcher should provide a normalized argument contract.

## Goal

Ensure prefix command handlers receive normalized argument strings.

## Scope

### In scope

- Normalize built-in prefix command args
- Normalize plugin prefix command args if plugin dispatch is touched in the same file
- Preserve command boundary matching behavior

### Out of scope

- Changing individual command parsers
- `/exit` handling
- Plugin command conflict policy

## Required behavior

For built-in prefix commands:

- Input: `/db help`
- Command name: `/db`
- Handler args should be: `help`

Use:

- `args = line[len(cmd.name):].strip()`

For empty args:

- Input: `/mcp`
- Handler args should be: empty string

## Acceptance criteria

- Prefix handlers receive args without leading or trailing whitespace
- Exact command behavior is unchanged
- `/db help` still works
- `/session list` still works
- `/mdq status` still works
- Existing command tests pass

## Validation

Run the agent and manually verify:

- `/db help`
- `/session list`
- `/mcp`
- `/mdq status`

Expected result:

- Commands behave as before
- Handlers receive normalized args

## Affected files

- `scripts/agent/commands/registry.py`

## Dependencies

None
