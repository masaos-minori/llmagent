## [Medium][agent][tests] Remove backward-compatibility test coverage for deleted /db aliases and replace it with scoped-command tests

### Summary
Backward-compatibility behavior for `/db` aliases is currently part of the documented command surface.

### Problem
If tests still validate deleted alias behavior, the old interface may be reintroduced accidentally.

### Scope
- tests/*
- agent CLI command tests
- DB command tests
- recovery command tests

### Facts
- Flat `/db` commands are currently documented as compatibility aliases. 【1-819e5c】【2-3cddb8】
- `/db recover` is explicitly documented as a compatibility alias. 【3-4da049】

### Task
- remove tests that expect flat `/db` alias behavior
- remove tests that expect `/db recover`
- add tests for scoped commands only:
  - `/db rag ...`
  - `/db session ...`
- add negative tests that confirm deleted aliases now fail with a clear validation message

### Expected Result
The automated test suite protects the new no-compatibility-alias command surface.

### Acceptance Criteria
- no test depends on deleted `/db` alias behavior
- scoped command coverage exists
- deleted alias forms are covered by negative tests
