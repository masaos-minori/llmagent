# Title
Add strict input validation at the start of `registry.py::dispatch()`

## Summary
Harden `dispatch()` by explicitly validating that the input line is a string and handling empty input deterministically.

## Background
The plan identifies that `dispatch()` does not currently validate input type before processing.

## Problem
Without an explicit guard, invalid caller input can fail later in less clear ways.

## Required Changes
- Add a type check for `line`.
- Raise a clear `TypeError` for non-string input.
- Return `False` early for empty string input if that is the intended behavior.
- Keep downstream dispatch logic unchanged.

## Acceptance Criteria
- Non-string input is rejected explicitly.
- Empty-string input is handled consistently.
- Dispatch behavior for valid input remains unchanged.

## Out of Scope
- Refactoring command routing logic
- Redesigning the registry interface

## Components
- `scripts/agent/commands/registry.py`

## AI Implementation Instruction
Implement this as a precondition-hardening task.

Strict rules:
- Do not use `assert` for runtime validation.
- Use explicit exceptions for invalid type preconditions.
- Keep the valid dispatch path unchanged.
- If empty-input behavior is not already clear from current usage, mark it as: `Needs confirmation`
