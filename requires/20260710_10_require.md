# [Low] Update documentation wording to required deployment artifact

## Background

The workflow definition should be described consistently as a required deployment artifact, not as an optional configuration file.

## Goal

Align documentation terminology with mandatory workflow deployment.

## Scope

- Agent documentation
- Deployment documentation
- Configuration documentation
- Operations documentation

## Tasks

- [ ] Replace ambiguous wording such as `workflow definition file` where appropriate with `required workflow deployment artifact`.
- [ ] Clearly state the required runtime path: `/opt/llm/config/workflows/default.json`.
- [ ] Clearly state the source path: `config/workflows/default.json`.
- [ ] Document that the agent must not be deployed or started without the workflow definition.
- [ ] Document that there is no `workflow_mode=disabled`.
- [ ] Document that there is no `workflow_mode=auto`.
- [ ] Document that there is no fallback execution mode.
- [ ] Add workflow artifact checks to the deployment checklist.
- [ ] Cross-link workflow validation, DB initialization, and startup validation sections.

## Acceptance Criteria

- [ ] Documentation consistently treats workflow definition as mandatory.
- [ ] No documentation implies workflow can be disabled.
- [ ] Deployment checklist includes workflow definition validation.
- [ ] Startup troubleshooting points to the workflow deployment runbook.

## Notes

This is a documentation-only issue.
