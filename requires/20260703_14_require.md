# [High] Unify audit log JSON-lines schema and preserve transport-error audit events

## Goal

Make all records written to the shared audit log parseable JSON-lines with consistent correlation fields.

## Background

MCP server audit logging is already JSON-lines, but its field names do not match agent-side audit events.

Current mismatch:

- MCP server audit uses `session` and `request`.
- Agent-side tool audit uses `session_id` and `mcp_request_id`.

Also, agent-side `audit_tool_exec()` currently skips audit writing when `mcp_request_id` is empty. Transport errors normally have no MCP request ID, so transport-error tool executions may be omitted from the audit log.

## Scope

Update:

- `mcp/audit.py`
- `agent/tool_audit.py`
- `tests/test_audit_log_format.py`
- MCP docs:
  - `04_mcp_02_protocol_and_transport.md`
  - `04_mcp_06_configuration_and_operations.md`
  - `04_mcp_90_inconsistencies_and_known_issues.md`

## Required Schema

MCP server audit event:

    {
      "event": "mcp_tool_exec",
      "source": "mcp_server",
      "ts": 1719500000.0,
      "session_id": "42",
      "request_id": "abc-123",
      "server_key": "file_read",
      "tool": "read_text_file",
      "target": "/path/to/file",
      "outcome": "ok",
      "error_type": "",
      "detail": ""
    }

Agent-side tool execution event:

    {
      "event": "tool_exec",
      "source": "agent",
      "ts": 1719500000.0,
      "session_id": "42",
      "workflow_id": "",
      "task_id": "turn-123",
      "tool": "read_text_file",
      "mcp_request_id": "",
      "is_error": true,
      "error_type": "transport"
    }

## Implementation Tasks

- Change MCP server audit fields:
  - `session` -> `session_id`
  - `request` -> `request_id`
- Add `source="mcp_server"` consistently.
- Add `source="agent"` to agent-side audit events if missing.
- Allow `audit_tool_exec()` to write events even when `mcp_request_id == ""`.
- Ensure transport errors are audited with:
  - `is_error=true`
  - `error_type="transport"`
  - `mcp_request_id=""`
- Convert `write_round_exec()` into JSON-lines output instead of relying on logger `extra`.
- Add real test methods to `tests/test_audit_log_format.py`.
- Update documentation examples to use JSON-lines only.

## Acceptance Criteria

- All shared audit log records are JSON parseable.
- MCP server audit records use `session_id` and `request_id`.
- Agent-side transport errors are logged even without an MCP request ID.
- `round_exec` records are JSON-lines.
- Docs no longer describe key=value audit records as the active shared audit format.

## Validation

Run:

    uv run pytest tests/test_audit_log_format.py -v

Also verify manually:

    rg -n "AUDIT session=|key=value" docs mcp agent tests

No active shared-audit documentation should require key=value parsing.
