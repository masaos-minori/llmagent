# M-3: Update /context partial completion display to reference session_diagnostics

## Priority

Medium

## Background

`/context` currently displays partial completions as stored in `tool_result(tool_name='llm_partial_completion')`.

After ToolResultStore removal, this output must reference diagnostics instead.

## Target files

- `agent/commands/cmd_context.py`

## Implementation tasks

- Update the `Partial compl` output in `_cmd_context()`.
- Remove `stored as tool_result(tool_name='llm_partial_completion')`.
- Replace with diagnostics-oriented wording, for example:
  - `stored in session_diagnostics`
- Do not add `/tool show` references.
- Keep the partial completion count.

## Acceptance criteria

- `/context` no longer references `tool_result`.
- `/context` no longer references `llm_partial_completion`.
- `/context` still shows partial completion count when non-zero.
- Output is consistent with `/stats`.

## Test plan

- Render `/context` with non-zero partial completions.
