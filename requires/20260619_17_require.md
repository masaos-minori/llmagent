# Title
Fail fast in strict mode when all MCP servers are unreachable

## Summary
Change startup validation so that strict tool-definition mode does not silently skip validation when every MCP server is unreachable.

## Background
Current startup behavior skips tool-definition validation entirely when all MCP servers are unreachable, even in strict mode.

## Problem
Strict mode should provide strong safety against misconfiguration.
Skipping validation when nothing is reachable weakens its purpose.

## Required Changes
- In strict mode, treat "all MCP servers unreachable" as a startup failure or equivalent hard error.
- Preserve current behavior for non-strict mode if desired.
- Ensure the error message clearly distinguishes:
  - all unreachable
  - partial unreachable
  - tool mismatch

## Acceptance Criteria
- Strict mode no longer allows startup to proceed silently when zero MCP servers are reachable.
- Error output is actionable and distinguishes reachability failure from schema mismatch.
- Documentation reflects the new strict behavior.

## Out of Scope
- Redesigning all startup readiness rules.
- Changing non-strict mode behavior unless necessary.

## Notes
Strict mode should prioritize fail-fast integrity over convenience.
