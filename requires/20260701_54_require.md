# [High] Delete db/workflow_schema.py and use create_schema.py as the workflow schema entrypoint

## Goal

Remove the duplicate workflow schema initialization entrypoint and make `db/create_schema.py` the only schema initialization entrypoint.

## Background

`db/workflow_schema.py` defines `init_schema()` and initializes workflow tables using `build_workflow_schema_sql()`. `db/create_schema.py` already provides `create_workflow_schema()` using the same canonical DDL builder and migration helper, so the workflow schema entrypoint is duplicated. 【4-96deb3】【2-418bb2】

## Scope

- Remove duplicate workflow schema script
- Update references
- Update tests and docs

## Implementation Tasks

- Delete `db/workflow_schema.py`.
- Replace any import or invocation of:
  - `db.workflow_schema`
  - `workflow_schema.init_schema`
  - `python -m db.workflow_schema`
- Use `db.create_schema.create_workflow_schema()` instead.
- Prefer `python -m db.create_schema` for full schema initialization.
- Update tests that directly import `workflow_schema`.
- Update docs that describe `workflow_schema.py` as an initialization entrypoint.

## Acceptance Criteria

- `db/workflow_schema.py` is removed.
- No code imports `db.workflow_schema`.
- No script runs `python -m db.workflow_schema`.
- Workflow schema is initialized through `db/create_schema.py`.
- Docs no longer treat `workflow_schema.py` as a canonical entrypoint.

## Validation

- Run:

  rg -n "workflow_schema|db.workflow_schema|init_schema" .

- Confirm remaining matches are removed or unrelated.
- Run schema creation tests.
- Run deployment initialization tests if available.
