# M-1: Decide and implement production behavior for subprocess startup failures

## Priority

Medium

## Background

`StartupOrchestrator._start_servers()` currently treats HTTP subprocess MCP startup failures as non-fatal warnings in all profiles. For production deployments, continuing with missing subprocess MCP servers may lead to confusing tool failures later.

## Target files

- `agent/startup.py`

## Implementation tasks

- Decide production policy:
  - local profile: non-fatal warning
  - production profile: fail-fast
- If production fail-fast is selected:
  - detect `SecurityProfile.PRODUCTION`
  - re-raise startup failure as `RuntimeError`
- Preserve existing non-fatal behavior for local profile.
- Ensure rollback shutdown still runs when startup aborts after partial success.

## Acceptance criteria

- In local profile, subprocess startup failure is logged and shown as warning.
- In production profile, subprocess startup failure aborts startup.
- Any servers started before the fatal failure are shut down by rollback logic.
- User-facing error message identifies the server key and root cause.

## Test plan

- Test local mode startup failure remains non-fatal.
- Test production mode startup failure raises.
- Test partial startup followed by production fatal triggers rollback shutdown.
