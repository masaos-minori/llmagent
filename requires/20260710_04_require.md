# [High] Revalidate workflow artifacts and workflow DB before service startup

## Background

`setup_services.sh` runs after deployment and before normal operation. It should verify that workflow artifacts and workflow DB schema are present before starting services.

## Goal

Prevent services from starting when the mandatory workflow definition or workflow DB schema is missing or invalid.

## Scope

- `deploy/setup_services.sh`

## Tasks

- [ ] Check that `/opt/llm/config/workflows/default.json` exists.
- [ ] Run the workflow validation CLI against `/opt/llm/config/workflows/default.json`.
- [ ] Check that `workflow.sqlite` exists or is accessible through the configured DB path.
- [ ] Verify that the `tasks` table exists.
- [ ] Verify that the `attempts` table exists.
- [ ] Verify that the `processed_events` table exists.
- [ ] Verify that the `artifacts` table exists.
- [ ] Verify that the `approvals` table exists.
- [ ] Stop setup before starting Event Bus or other services if validation fails.
- [ ] Print clear remediation instructions.

## Acceptance Criteria

- [ ] `setup_services.sh` fails when the deployed workflow definition is missing.
- [ ] `setup_services.sh` fails when the deployed workflow definition is invalid.
- [ ] `setup_services.sh` fails when workflow DB schema is incomplete.
- [ ] No services are started after workflow validation failure.

## Notes

This is a pre-flight safety check. It must not modify workflow definition files.
