# Title
Replace `.isdigit()`-based session ID parsing in `cmd_session.py` with strict integer validation

## Summary
Refactor `cmd_session.py` so session IDs are parsed with explicit integer validation instead of `.isdigit()` checks.

## Background
The plan identifies multiple `.isdigit()` uses in session-related commands and recommends replacing them with `int(...)` parsing plus explicit validation.

## Problem
`.isdigit()` provides weak validation, does not express intent clearly, and can hide invalid cases that should be rejected explicitly.

## Required Changes
- Replace `.isdigit()`-based parsing in session-related command paths.
- Add a shared helper such as `_parse_session_id(...)` or equivalent.
- Reject invalid, non-positive, or malformed session IDs explicitly.
- Keep user-facing validation errors clear.

## Acceptance Criteria
- `.isdigit()` is no longer used for session ID parsing.
- Session IDs are validated through explicit integer parsing.
- Invalid input is handled predictably and clearly.

## Out of Scope
