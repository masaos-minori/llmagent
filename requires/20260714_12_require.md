# [Medium] Clarify MDQ production-ready scope and hybrid search status

## Labels
priority/medium, area/mdq, area/docs, consistency

## Summary
Clarify that MDQ is production-ready for implemented FTS5-based Markdown structural search, while hybrid embedding/vector search must not be presented as production-ready unless implementation confirms it.

## Background
The documentation describes MDQ as production-ready, but also states that hybrid embedding integration is not yet implemented or still returns empty vector results.

This makes the production-ready scope unclear.

## Required Changes

### 1. Inspect implementation
Determine whether MDQ hybrid/vector search is implemented.

Inspect:

- search implementation
- config model
- tests
- service behavior when `use_embedding=true`

### 2. Clarify production-ready scope
If hybrid/vector search is not implemented, document production-ready scope as limited to:

- FTS5 indexing
- Markdown structural search
- outline
- get_chunk
- grep_docs
- refresh/index operations
- stats and admin consistency operations, if implemented

### 3. Clarify non-production-ready scope
Document as unavailable, experimental, or not production-ready:

- hybrid semantic search
- embedding-backed vector search

### 4. Update guidance
Remove or soften recommendations that imply MDQ hybrid is production-ready.

Prefer language such as:

MDQ is production-ready for FTS5-based Markdown structural search. MDQ hybrid/vector search remains unavailable or experimental until embedding integration is implemented.

## Acceptance Criteria
- MDQ production-ready scope is explicit.
- Hybrid/vector search status is consistent across guide, catalog, boundary docs, and known issues.
- `use_embedding=true` behavior is documented accurately.
- Operators do not mistake unimplemented hybrid functionality for production-ready behavior.
- Summary sections qualify MDQ production readiness.
