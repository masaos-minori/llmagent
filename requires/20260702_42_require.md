# Remove backward-compatible DB migration documentation and replace it with DB recreation policy

## Priority

High

## Summary

Remove documentation that describes automatic schema migration or backward-compatible DB updates. Replace it with the DB recreation policy.

## Background

The project policy is that existing DB-compatible migration is not supported.

Existing DB updates must be handled by DB recreation.

Documentation still contains references to additive migration, `ALTER TABLE`, and duplicate-column suppression.

## Goal

Make documentation match the DDL-only schema creation policy.

## Target files

- `90_shared_04_db_architecture_and_schema.md`
- `90_shared_05_db_api_and_operations.md`
- `90_shared_00_document-guide.md`
- `create_schema.py`

## Scope

### In scope

- Remove migration support descriptions.
- Add DB recreation procedure.
- Clarify that `schema_sql.py` is the canonical schema source.

### Out of scope

- Implementing migration tools.
- Preserving old DB compatibility.

## Implementation tasks

- Remove or rewrite references to:
  - incremental migration
  - `ALTER TABLE`
  - duplicate column suppression
  - backward-compatible schema additions
  - existing deployment compatibility
- Add policy text:
  - `Schema migration is not supported.`
  - `When schema changes, recreate DB files from the latest DDL.`
  - `Existing DB updates are handled by DB recreation, not automatic migration.`
- Add operational guidance for DB archive and recreation.

## Acceptance criteria

- Documentation does not imply automatic DB migration support.
- DB recreation is documented as the standard schema update path.
- `schema_sql.py` is documented as the canonical latest schema definition.

## Validation

- Search for stale wording:
```bash
grep -R "migration\|ALTER TABLE\|backward-compatible\|duplicate column" docs scripts -n
```
- Any remaining occurrence must be intentional and consistent with the no-migration policy.

## Confirmation required

- Confirm the exact DB deletion and recreation commands for the deployment environment.
