# Consolidate tool startup validation into MCP discovery service

## Priority

Medium

## Target files

- agent/repl_health.py
- agent/startup.py
- agent/services/mcp_tool_discovery.py
- shared/tool_routing_validation.py
- tests/agent/services/test_mcp_tool_discovery.py

## Reason

Tool validation is currently split across static tool definition checks, config drift checks, live routing drift checks, and duplicate ownership checks.

Once RuntimeToolRegistry is introduced, discovery should own validation.

## Intent

Move startup tool validation into McpToolDiscoveryService.

## Scope

Migrate or replace:

- `check_tool_definitions_startup()`
- `check_routing_drift_vs_live()`
- duplicate live tool ownership checks
- malformed `/v1/tools` checks

## Acceptance criteria

- Discovery and validation happen in one service.
- Strict and production behavior are preserved.
- Duplicate MCP tool names are detected before registry construction completes.
- Startup validation output remains clear.
