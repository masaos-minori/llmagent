# [Low] Add CI check for MCP compatibility leftovers

## Priority
Low

## Background
Backward compatibility leftovers were found in routing code and RAG MCP service comments. A CI check should prevent reintroduction of removed compatibility paths while allowing stdio transport to remain.

## Goal
Add a static check that blocks active compatibility leftovers but does not flag retained stdio transport.

## Scope
Affected files:
- `scripts/checks/check_no_mcp_compat.py`
- CI workflow files
- `docs/`
- `shared/`
- `mcp/`
- `scripts/mcp/`
- `tests/`

## Implementation Tasks
- Add a checker for forbidden active patterns:
  - `compatibility/emergency`
  - `Static fallback`
  - `_SET_ROUTES`
  - `_fallback_route`
  - `_GITHUB_PREFIX`
  - `github_ prefix exception`
  - `rag.pipeline._cfg`
  - `module-level _cfg`
  - `/mcp install`
  - `POST /v1/search`
- Allow these patterns only in explicit archive or migration note sections.
- Do not flag retained stdio symbols:
  - `StdioTransport`
  - `TransportType.STDIO`
  - `StartupMode.ONDEMAND`
  - `HealthcheckMode.PING_TOOL`
  - `__list_tools__`
- Wire the checker into CI.

## Acceptance Criteria
- CI fails if static fallback routing is reintroduced.
- CI fails if GitHub prefix fallback is reintroduced.
- CI fails if RAG module-level `_cfg` override is reintroduced.
- CI does not fail on valid stdio transport usage.
- The checker prints file path and line number for violations.

## Validation
- Run `python scripts/checks/check_no_mcp_compat.py`.
- Add temporary forbidden text in a fixture and confirm the checker fails.
- Confirm stdio-related retained symbols do not fail the check.
