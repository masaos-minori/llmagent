# [High] Remove active stdio references from MCP documentation

## Priority

High

## Goal

Make MCP documentation HTTP-only.

## Background

The implementation and config model are HTTP-only, but MCP docs still describe stdio as an active transport.

## Scope

Affected files:

- docs/04_mcp_01_system_overview.md
- docs/04_mcp_02_protocol_and_transport.md
- docs/04_mcp_03_routing_lifecycle_and_execution.md
- docs/04_mcp_06_configuration_and_operations.md
- docs/04_mcp_90_inconsistencies_and_known_issues.md

## Tasks

- Remove active references to:
  - stdio
  - StdioTransport
  - --stdio
  - __list_tools__
  - healthcheck_mode="ping_tool"
  - startup_mode="ondemand"
  - stdio request and response examples
  - stdio lifecycle and idle timeout
  - stdio timeout
- Rewrite transport sections as HTTP-only.
- Rewrite lifecycle sections to cover only:
  - persistent HTTP
  - subprocess HTTP
- If historical notes are needed, clearly mark them as historical and unsupported.

## Unknowns / Open Questions

- Confirm whether release notes should preserve old stdio information as historical context.
- Confirm whether any deployment docs still reference stdio.

## Acceptance Criteria

- Active MCP docs do not instruct operators to use stdio.
- Active examples use only HTTP transport.
- Startup mode docs match McpServerConfig.

## Validation

Run:
```bash
rg -n "stdio|StdioTransport|--stdio|__list_tools__|ping_tool|ondemand|STDIO" docs/04_mcp_*.md
```
