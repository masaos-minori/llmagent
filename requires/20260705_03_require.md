# Guarantee partial LLM completions never enter LLM-visible history

## Priority

High

## Type

Implementation / Test / Safety

## Background

When an LLM stream fails after producing partial text, the partial output must be isolated from normal conversation history.

The intended model is:

- Store diagnostic information in `session_diagnostics`.
- Store full partial output in `tool_result_store`.
- Increment partial completion statistics.
- Do not append the partial text to `ctx.conv.history`.
- Do not persist the partial text into the `messages` table.

## Problem

If partial assistant output enters LLM-visible history, the next turn may treat incomplete content as a valid assistant answer. This causes context contamination and can lead to incorrect reasoning.

## Goal

Enforce the invariant that partial completions are diagnostic artifacts only and are never part of normal conversation history.

## Scope

Affected areas:

- `agent/orchestrator.py`
- `agent/diagnostic_store.py`
- `agent/session.py`
- Tool result store
- LLM transport error handling tests

## Implementation Tasks

- Inspect the `LLMTransportError` handling path.
- Confirm behavior for `partial_text != ""`.
- Confirm behavior for pre-stream failures where `partial_text == ""`.
- Confirm behavior for tool-continuation failures.
- Add explicit tests for all three paths.
- Ensure `session_diagnostics` and `tool_result_store` are written consistently.
- Ensure `messages` table is not contaminated.

## Acceptance Criteria

For `LLMTransportError(partial_text != "")`:

- Partial text is not appended to `ctx.conv.history`.
- Partial text is not inserted into the `messages` table.
- Diagnostic record is inserted into `session_diagnostics`.
- Partial output is available through `tool_result_store`.
- `stat_partial_completions` increments.

For pre-stream transport errors:

- The last user message is removed from history.
- No assistant message is persisted.

For tool-continuation transport errors:

- A synthetic tool error is added as a tool result.
- The conversation can continue safely.

## Validation Plan

Add tests for:

- In-stream LLM failure with partial text.
- Pre-stream LLM failure with no partial text.
- Tool-continuation LLM failure.
- Session reload after partial completion.
- `/tool show llm_partial_completion` equivalent retrieval.
- `messages` table inspection after failure.

## Risks

Changing error-handling behavior may affect existing tests that assume partial output is visible. Update those tests to reflect the intended safety invariant.

## Out of Scope

- Changing SSE parser behavior.
- Changing LLM retry policy.
- Documentation-only updates.
