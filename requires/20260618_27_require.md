# [agent][memory][docs] Add detailed API documentation for the memory layer

## Summary
The top-level memory service is documented, but the internal memory modules are not documented enough.

## Problem
It is still hard to understand the responsibilities of:
- store
- retriever
- injection
- ingestion
- embedding client

without reading source code directly.

## Scope
- memory layer documentation
- reference API docs

## Task
- document the main memory sub-modules
- explain the role of each module
- document activation and disabled behavior clearly

## Expected Result
Developers can understand the memory layer without opening many source files.

## Acceptance Criteria
- [ ] store responsibilities are documented
- [ ] retriever responsibilities are documented
- [ ] injection and ingestion responsibilities are documented
- [ ] disabled behavior is documented clearly
