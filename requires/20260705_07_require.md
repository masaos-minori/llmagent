# Add static checks for database ownership boundaries

## Priority

Medium

## Type

Architecture / Test / Safety

## Background

The agent architecture defines clear ownership boundaries for SQLite databases:

- `session.sqlite`: Agent layer
- `rag.sqlite`: RAG layer
- `mdq.sqlite`: mdq-mcp service
- `workflow.sqlite`: Workflow engine

Agent application code should access RAG and MDQ data through MCP tools or explicitly approved maintenance services.

## Problem

Direct database access across ownership boundaries creates hidden coupling and makes schema migration, maintenance, and recovery harder.

## Goal

Prevent accidental direct access to RAG and MDQ databases from the wrong layers.

## Scope

Affected areas:

- `agent/`
- `shared/`
- RAG MCP service
- MDQ MCP service
- Static analysis tests
- CI checks

## Implementation Tasks

- Add a static check that scans for direct `SQLiteHelper("rag")` usage outside approved modules.
- Add a static check that scans for direct `SQLiteHelper("mdq")` usage outside approved modules.
- Add a static check that prevents `shared/` from directly depending on RAG or MDQ database paths.
- Define an explicit whitelist for approved maintenance services.
- Add tests that verify `/db rag urls` and `/db rag clean` go through rag-pipeline-mcp.
- Add tests that verify `/mdq` commands go through mdq-mcp.

## Acceptance Criteria

- CI fails when unauthorized code directly accesses `rag.sqlite`.
- CI fails when unauthorized code directly accesses `mdq.sqlite`.
- Approved maintenance exceptions are explicit and reviewed.
- RAG document list/delete operations use MCP tools.
- MDQ commands use mdq-mcp tools.

## Validation Plan

- Add static scan tests using repository file content.
- Add command tests with mocked MCP tool executor.
- Add negative tests using temporary forbidden imports or helper usage.
- Verify approved exceptions do not trigger false positives.

## Risks

Existing code may violate the intended boundary. Fix violations rather than weakening the check unless there is a documented maintenance exception.

## Out of Scope

- Replacing SQLiteHelper.
- Changing database schema.
- Documentation-only updates.
