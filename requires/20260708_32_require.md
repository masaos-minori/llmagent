# H-10: Remove github_server_url legacy configuration

## Priority

High

## Background

`github_server_url` duplicates `mcp_servers.github.url`.

The canonical GitHub MCP endpoint must be `mcp_servers.github.url`. The legacy scalar `github_server_url` creates a second URL path and can be mistaken for a hot-reloadable GitHub MCP endpoint.

## Target files

- `agent/config_dataclasses.py`
- `agent/config_builders.py`
- `agent/services/config_reload.py`
- `agent/commands/cmd_config_display.py`
- documentation
- config samples
- tests

## Implementation tasks

- Remove `MCPConfig.github_server_url`.
- Remove `github_server_url=cfg.get(...)` from `build_agent_config()`.
- Remove `github_server_url` handling from `ConfigReloadService._apply_llm_prompt_params()`.
- Remove `github_server_url` from `/config` display output.
- Ensure GitHub MCP endpoint is documented as `mcp_servers.github.url`.
- Decide removed-key behavior for existing config:
  - preferred: startup/reload validation error
- Recommended error message:
  - `Config contains removed key: github_server_url. Use [mcp_servers.github].url instead.`
- Update tests and config examples.

## Acceptance criteria

- Runtime config no longer exposes `cfg.mcp.github_server_url`.
- `/config` no longer displays `github_server_url`.
- `/reload` no longer applies `github_server_url`.
- GitHub MCP URL has one canonical source: `mcp_servers.github.url`.
- Existing `github_server_url` config is rejected or explicitly handled as a removed key.

## Test plan

- Build config without `github_server_url`.
- Build config with `mcp_servers.github.url`.
- Verify GitHub transport uses `mcp_servers.github.url`.
- Verify `/config` output does not contain `github_server_url`.
- Verify reload does not handle `github_server_url`.
- Verify removed-key behavior for old config.
