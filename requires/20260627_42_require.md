## Title
[Medium] Improve workflow pending approval startup recovery notice

## Priority
Medium

## Labels
workflow, ux, startup, observability

## Dependencies
None

## Background
Startup recovery restores pending workflow approval state from the workflow database. It sets `ctx.workflow.approval_pending` and `ctx.turn.pending_approval_id`.

The current user-facing warning does not include task ID or approval ID, making it harder to inspect or debug pending approvals.

## Scope
Update:
- `agent/startup.py`
- Startup tests
- Workflow operations documentation

## Required Changes
- Include `task_id` and `approval_id` in the startup warning.
- Clarify that the latest global pending approval is recovered.
- Document multi-session behavior.
- Add a startup recovery test if missing.

## Tests to Add
- `test_startup_recovery_restores_pending_approval`
- `test_startup_recovery_warning_contains_task_and_approval_id`

## Acceptance Criteria
- Startup warning is actionable.
- Pending approval state is visible to the user.
- Multi-session recovery semantics are documented.
