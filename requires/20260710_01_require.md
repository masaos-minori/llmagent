# [High] Enforce required workflow definition existence in deploy.sh

## Background

The agent requires `config/workflows/default.json` as a mandatory workflow definition. Deployment should fail before copying files if this required artifact is missing.

## Goal

Make `config/workflows/default.json` a required deployment artifact.

## Scope

- `deploy/deploy.sh`

## Tasks

- [ ] Add a pre-deploy check for `config/workflows/default.json`.
- [ ] Exit with non-zero status if the file is missing.
- [ ] Print a clear fatal error message that includes the expected source path.
- [ ] After copying workflow files, verify that `/opt/llm/config/workflows/default.json` exists.
- [ ] Exit with non-zero status if the deployed file is missing.

## Suggested Error Message

`[FATAL] Missing required workflow definition: config/workflows/default.json`

`This agent requires a valid workflow definition and does not support workflow-disabled mode.`

## Acceptance Criteria

- [ ] Running `deploy.sh` without `config/workflows/default.json` fails before copying files.
- [ ] Running `deploy.sh` with `config/workflows/default.json` succeeds past the workflow artifact check.
- [ ] The deployed path `/opt/llm/config/workflows/default.json` is verified after copy.
- [ ] Error messages are actionable and include the required file path.

## Notes

Do not add any workflow disable, fallback, or optional mode.
