# Delete `NoteRepository`

## Priority
High

## Labels
agent, database, notes, cleanup, dead-code

## Dependencies
- Remove note delegation APIs from `AgentSession`

## Background
`note_repo.py` is a repository dedicated to the `notes` table. After removing `/note` and note injection, this repository should no longer be used.

## Task
Delete `note_repo.py` if no non-note command code uses it.

## Implementation Requirements
- Search for `NoteRepository` references.
- If only `AgentSession` and note tests use it, delete `note_repo.py`.
- Remove all note table access paths:
  - insert into `notes`
  - list from `notes`
  - delete from `notes`
  - pin/unpin updates
  - LIKE search
  - pinned note retrieval
  - all note contents retrieval

## Acceptance Criteria
- No runtime code imports `agent.note_repo`.
- No runtime code reads from or writes to the `notes` table.
- Lint/type checks pass without unused imports.

## Source Context
`note_repo.py` contains note-specific SQLite operations for `add_note`, `list_notes`, `delete_note`, `pin_note`, `unpin_note`, `get_pinned_notes`, `search_notes`, and `get_all_note_contents`.
