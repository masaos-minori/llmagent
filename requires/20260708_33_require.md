# M-1: Add field-level MCP server config diff helper

## Priority

Medium

## Background

MCP server definition changes should be reported field-by-field so operators can see exactly which setting requires restart.

A dedicated helper will keep reload classification simple and testable.

## Target files

- `agent/services/config_reload.py`
- optionally `shared/mcp_config.py`
- tests for config diff helper

## Implementation tasks

- Add a helper such as:
  - `_diff_mcp_server_config(old, new) -> list[str]`
- Compare these fields:
  - `transport`
  - `url`
  - `startup_mode`
  - `healthcheck_mode`
  - `call_timeout_sec`
  - `startup_timeout_sec`
  - `tool_names`
  - `auth_token`
  - `role`
  - `cmd`
  - `env`
- Return field names or fully formatted restart-required entries.
- Use output format:
  - `mcp/<server_key>.<field>`
- Treat `tool_names`, `cmd`, and `env` as value comparisons.
- Handle server add/remove outside the field diff helper.

## Acceptance criteria

- The diff helper returns deterministic field-level differences.
- The helper does not mutate old or new config.
- Reload classification uses the helper.
- Tests cover scalar, list, and dict field differences.

## Test plan

- Compare identical configs and expect no differences.
- Change each scalar field and expect one difference.
- Change `tool_names`, `cmd`, and `env` and expect one difference per field.
- Verify old and new objects remain unchanged.
