# H-7: Classify MCP server add, remove, and rename as restart-required

## Priority

High

## Background

ToolExecutor builds transports and routing at startup. Adding, removing, or renaming MCP server keys cannot be safely applied at runtime.

The current behavior may report new servers as both skipped and restart-required, which is ambiguous.

## Target files

- `agent/services/config_reload.py`
- reload command tests

## Implementation tasks

- Detect server keys present only in the new config.
- Report new servers as:
  - `mcp/<server_key> (new server)`
- Detect server keys present only in running config.
- Report removed servers as:
  - `mcp/<server_key> (removed server)`
- Treat server key rename as remove + add.
- Add these entries to `ConfigReloadOutcome.needs_restart`.
- Do not add or remove server keys from running config.
- Do not report MCP server add/remove as skipped.
- Avoid reporting the same change under both `[SKIP]` and `[RESTART]`.

## Acceptance criteria

- New MCP servers are restart-required.
- Removed MCP servers are restart-required.
- Renamed MCP servers appear as removed old key + new new key.
- Running config is unchanged.
- Reload output has no duplicate skip/restart semantics for the same change.

## Test plan

- Reload with one new MCP server.
- Reload with one existing MCP server removed.
- Reload with rename simulated by old key removed and new key added.
- Verify all are restart-required.
- Verify `skipped` does not contain the same MCP items.
