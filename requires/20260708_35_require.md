# M-3: Document ToolExecutor.apply_config hot-reload scope

## Priority

Medium

## Background

`ToolExecutor.apply_config()` only updates cache TTL. It does not rebuild transports, route resolver, server configs, or concurrency semaphores.

This should be explicit to prevent future accidental MCP server hot reload implementations.

## Target files

- `shared/tool_executor.py`
- tests for ToolExecutor apply_config
- relevant documentation

## Implementation tasks

- Update `ToolExecutor.apply_config()` docstring to state that only cache TTL is hot-reloadable.
- Explicitly document that these are restart-required:
  - server configs
  - transports
  - route resolver
  - server add/remove
  - concurrency limits
- Add or update tests proving `apply_config(cache_ttl=...)` does not recreate transports.
- Ensure `ConfigReloadService._sync_services()` still only applies cache TTL to ToolExecutor.

## Acceptance criteria

- ToolExecutor hot reload scope is documented.
- Applying cache TTL does not mutate transports.
- Applying cache TTL does not mutate server configs.
- Applying cache TTL does not recreate the route resolver.
- No MCP transport config is hot-reloaded through ToolExecutor.

## Test plan

- Instantiate ToolExecutor.
- Store identity of `_transports` and one `HttpTransport`.
- Call `apply_config(cache_ttl=<new value>)`.
- Verify cache TTL changed.
- Verify transport identities did not change.
