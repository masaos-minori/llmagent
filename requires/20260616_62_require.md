# [shared][docs][config] Clarify config ownership and loading rules in docs

## Summary
The current docs do not make the config loading boundaries clear enough.

## Problem
Some config files are loaded by `load_all()`.
Some important config files are loaded separately.
This is easy to misunderstand.

## Scope
- `06_shared_00_document-guide.md`
- `06_shared_01_overview.md`
- `06_shared_03_runtime_and_execution.md`
- `06_shared_04_db_architecture_and_schema.md`

## Task
- add one clear section that explains:
  - which config files are loaded by `load_all()`
  - which config files are loaded separately
  - which layer owns each config file

## Expected Result
- developers and AI agents can follow config loading without guessing

## Acceptance Criteria
- [ ] one clear config ownership section exists
- [ ] shared docs no longer imply that `load_all()` loads everything
- [ ] DB path and embedding path loading are easy to trace
