# Title
Refactor `cmd_memory.py` to use typed memory action selection and strict limit validation

## Summary
Replace string-based memory type detection and `.isdigit()`-based limit parsing in `cmd_memory.py` with typed and validated input handling.

## Background
The plan flags string comparison for memory type and `.isdigit()`-based limit extraction as weak points in the current command implementation.

## Problem
String-based action selection and weak numeric parsing reduce clarity and allow ambiguous or malformed input handling.

## Required Changes
- Replace string-based memory type selection with a typed enum or equivalent explicit model.
- Replace `.isdigit()` limit parsing with strict integer validation.
- Add explicit limit range validation.
- Reuse a shared validation helper if one is introduced elsewhere.

## Acceptance Criteria
- Memory type selection is explicit and typed.
- Limit parsing does not rely on `.isdigit()`.
- Out-of-range limits are rejected clearly.
- The command behavior remains stable for valid inputs.

## Out of Scope
- Refactoring memory retrieval logic itself
- Changing memory-layer semantics

## Components
- `scripts/agent/commands/cmd_memory.py`
- Shared validation helper module
- Relevant enum module if extended
