# Add validation for `k` in `SQLiteVectorStore.vec_search()`

## Priority
Low

## Labels
- db
- vector-search
- validation
- low-priority

## Dependencies
None

## Background
`SQLiteVectorStore.vec_search()` passes `k` directly to SQL `LIMIT`. The behavior for `k <= 0` should be explicit.

## Scope
Modify:
- `db/store_impl.py`
- `tests/test_db_store_impl.py`

## Required Changes
- If `k < 1`, return `[]` without querying the database.
- Keep existing behavior for valid positive `k`.
- Do not change embedding validation behavior.

## Tests
Update `tests/test_db_store_impl.py`.

Add tests for:
- `vec_search(..., k=0)` returns `[]`.
- `vec_search(..., k=-1)` returns `[]`.
- Valid `k` still delegates to DB query.

## Acceptance Criteria
- Invalid `k` values are handled safely.
- SQL is not executed for `k < 1`.
- Existing vector store tests pass.
