# Title
Allow config and schema path overrides instead of hard-coding `/opt/llm/...`

## Summary
Improve local development, tests, and multi-environment portability by allowing path overrides for config and schema locations while preserving current production defaults.

## Background
The current implementation uses fixed default paths such as `/opt/llm/config/eventbus.toml` and `/opt/llm/schemas/event_envelope.json`.

## Problem
Hard-coded paths are convenient for one deployment model, but they make local testing and alternate environments harder.

## Required Changes
- Add environment-variable or startup-argument overrides for config and schema paths.
- Keep `/opt/llm/...` as the default production path unless another issue changes that convention.
- Make test fixtures able to inject temporary paths cleanly.

## Acceptance Criteria
- The service can run outside the fixed `/opt/llm/...` layout when explicitly configured.
- Production defaults remain stable.
- Tests and local development can use temporary paths without patching source code.

## Out of Scope
- Redesigning the entire config model

## Components
- `scripts/eventbus/config.py`
- `scripts/eventbus/app.py`
- Deployment / setup docs
- Test fixtures and local dev docs
