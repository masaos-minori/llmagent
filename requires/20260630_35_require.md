# [Medium] Normalize `McpServerConfig.tool_names` documentation as validation metadata

## Priority
Medium

## Background
Some tests already show that known tools can route through the registry without config `tool_names`, but other tests still assume config fallback behavior. The documentation and tests must be aligned around the rule that `tool_names` is validation metadata only. 【2-7f91a3】

## Goal
Document and test `tool_names` as validation metadata, not routing input.

## Scope
Affected files:
- `shared/mcp_config.py`
- `shared/tool_registry.py`
- `tests/test_route_resolver.py`
- `docs/04_mcp_06_configuration_and_operations.md`
- `docs/05_agent_08_configuration.md`

## Implementation Tasks
- Remove all documentation that presents `tool_names` as routing input.
- Describe `tool_names` as optional drift validation metadata.
- Add or update tests showing routing works when `tool_names` is empty.
- Add or update tests showing config-only tools do not route.
- Ensure validation functions can still report config/registry drift.

## Acceptance Criteria
- `tool_names` is not needed for routing.
- `tool_names` is only used for validation/drift detection.
- Tests and docs are consistent.
- Tests pass.

## Validation
- Run: `pytest tests/test_route_resolver.py -v`
- Run: `rg -n "tool_names.*routing|tool_names.*fallback|Config tool_names" shared docs tests`
