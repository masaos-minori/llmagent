# [agent][architecture] Reduce AgentREPL to a UI and command adapter only

## Summary
`AgentREPL` still owns too much startup and runtime coordination logic.

## Problem
The current REPL is not only a UI layer.
It also handles:
- startup wiring
- MCP startup
- service health checks
- initial prompt setup
- shutdown coordination

This makes the UI layer too central.

## Scope
- `agent/repl.py`
- startup/bootstrap flow
- runtime wiring flow

## Task
- move startup orchestration out of `AgentREPL`
- keep `AgentREPL` focused on:
  - input loop
  - command dispatch
  - output display
- define a separate bootstrap/runtime setup path

## Expected Result
The REPL becomes a thin UI-facing layer.

## Acceptance Criteria
- [ ] `AgentREPL` is mostly UI and command handling
- [ ] startup orchestration is moved out
- [ ] docs and implementation match the new responsibility boundary
