# Fix public API export declarations in `db.store` and `db.__init__`

## Priority
High

## Labels
- db
- public-api
- import
- high-priority

## Dependencies
None

## Background
The DB package exposes a stable public API through `db/store.py` and `db/__init__.py`. These files currently define the export list using an invalid `**all**` variable instead of Python's standard `__all__`.

This can break wildcard imports and API surface validation.

## Scope
Modify:
- `db/store.py`
- `db/__init__.py`

Add or update tests:
- `tests/test_db_public_api.py`

## Required Changes
- Rename `**all**` to `__all__` in `db/store.py`.
- Rename `**all**` to `__all__` in `db/__init__.py`.
- Ensure every symbol listed in `__all__` exists and can be imported.
- Keep the intended stable public import surface unchanged.

## Tests
Add `tests/test_db_public_api.py`.

Test:
- `db.store.__all__` exists.
- `db.__all__` exists.
- Every symbol in `db.store.__all__` exists on `db.store`.
- Every symbol in `db.__all__` exists on `db`.
- Wildcard-import behavior does not fail.

## Acceptance Criteria
- `from db.store import *` works.
- `from db import *` works.
- No undefined symbol exists in either `__all__`.
- Existing DB tests continue to pass.
