# Remove unused `is_async` parameter from `CommandRegistry._get_handler()`

## Priority

Medium

## Labels

cleanup, agent, cli, command-registry

## Background

`CommandRegistry._get_handler()` currently accepts an `is_async` parameter, but the method does not use it.

Async dispatch decisions are handled by `CommandRegistry.dispatch()` using `cmd.is_async`.

Keeping an unused parameter makes the method contract misleading.

## Goal

Simplify `_get_handler()` by removing the unused `is_async` parameter.

## Scope

### In scope

- Update `_get_handler()` signature
- Update all call sites
- Preserve runtime behavior

### Out of scope

- Changing dispatch semantics
- Changing `CommandDef`
- Adding new validation behavior

## Required change

Change:

- `_get_handler(self, cmd: CommandDef, is_async: bool, /)`

To:

- `_get_handler(self, cmd: CommandDef)`

Update call sites from:

- `handler = self._get_handler(cmd, cmd.is_async)`

To:

- `handler = self._get_handler(cmd)`

## Acceptance criteria

- `_get_handler()` has no unused parameters
- Command dispatch behavior is unchanged
- Existing command validation still raises `AttributeError` for unknown handlers
- Tests pass

## Validation

Run:

- `uv run pytest tests/ -k command`
- Start the agent and run `/help`

Expected result:

- No behavior change
- No unused-argument warning

## Affected files

- `scripts/agent/commands/registry.py`

## Dependencies

None
