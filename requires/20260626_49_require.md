## Summary
Improve visibility and policy clarity for Refiner fallback behavior when `use_refiner=true`.

## Background
When the Refiner fails or returns empty output, the pipeline falls back to raw chunk formatting. Current docs state that refiner exceptions are not retried.

## Problem
Temporary LLM failures can cause raw fallback, but operators have limited visibility into how often this happens and whether retry should be used.

## Required Changes
- Add counters or diagnostics for:
  - `refiner_returned_empty`
  - `refiner_exception`
- Clarify in docs why retry is not performed, or add configurable retry if desired.
- If retry is added, introduce explicit config such as `refiner_retry`.
- Improve `/rag search --debug` visibility for refiner fallback.

## Acceptance Criteria
- Refiner fallback frequency is observable.
- The retry/no-retry policy is explicit.
- Operators can tell whether raw fallback is common.

## Out of Scope
- Replacing the Refiner model.
- Redesigning the entire Augment stage.

- Do not hide fallback frequency.
