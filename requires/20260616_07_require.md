# [MCP][Config][Docs] Document and enforce startup_mode × transport compatibility clearly

## Summary
The compatibility rules between `transport` and `startup_mode` are valid in code but not clearly documented in one authoritative place.

## Problem
Invalid combinations such as `startup_mode="subprocess"` with `transport="stdio"` are not obvious enough from the documentation.

## Scope
- config validation docs
- startup mode docs
- validation error messages

## Task
- create one authoritative compatibility matrix for `transport` and `startup_mode`
- update validation error messages to explain valid alternatives
- ensure all related docs reference the same compatibility source

## Expected Behavior
- users can understand valid combinations immediately
- invalid combinations fail fast with helpful guidance

## Acceptance Criteria
- [ ] compatibility matrix exists in one clear location
- [ ] invalid config errors suggest valid alternatives
- [ ] docs are consistent across overview, routing, and config sections
