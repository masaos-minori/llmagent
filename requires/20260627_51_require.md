## Title
[High] Fix `deploy/init_db.sh` to initialize the current Event Bus schema

## Priority
High

## Labels
bug, deployment, database, eventbus, schema

## Dependencies
None

## Background
`deploy/init_db.sh` contains an outdated inline Event Bus schema. It does not include the current Event Bus columns and indexes such as `delivery_failure_count`, `dlq_requeue_count`, `idx_events_dlq_at`, and `idx_events_dlq_seq`.

The canonical Event Bus schema is defined in `scripts/eventbus/schema.sql`. The deployment initializer should not create an outdated schema.

## Scope
Update:
- `deploy/init_db.sh`
- Event Bus deployment documentation if applicable

Do not change:
- Agent integration
- Agent publisher/subscriber behavior

## Required Changes
- Remove or replace the outdated inline Event Bus DDL in `deploy/init_db.sh`.
- Use `scripts/eventbus/schema.sql` as the canonical Event Bus schema source.
- Fix broken shell variable references in `init_db.sh`.
- Ensure `eventbus.sqlite` is initialized with:
  - `delivery_failure_count`
  - `dlq_requeue_count`
  - `idx_events_dlq_at`
  - `idx_events_dlq_seq`
- Clarify whether migration is handled by `eventbus.db._migrate()` or by deployment initialization.
- Update workflow table checks to include `approvals` if workflow DB validation is also touched.

## Tests / Validation
- Run `bash deploy/init_db.sh` in a clean environment.
- Verify that `eventbus.sqlite` includes all columns and indexes from `scripts/eventbus/schema.sql`.
- Verify that the script is idempotent for existing databases.
- Verify that no outdated retry-count-only Event Bus schema is created.

## Acceptance Criteria
- `deploy/init_db.sh` runs without shell variable expansion errors.
- Event Bus DB schema matches `scripts/eventbus/schema.sql`.
- Deployment initialization no longer creates stale Event Bus tables.
- Existing Event Bus tests continue to pass.
