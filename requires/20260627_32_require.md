## Title
[High] Use `memory_jsonl_dir` when constructing the JSONL memory store

## Priority
High

## Labels
bug, memory, configuration, persistence

## Dependencies
None

## Background
`MemoryConfig` defines `memory_jsonl_dir` as the canonical configuration field for the JSONL memory archive directory. However, `factory._build_jsonl_store()` currently constructs the JSONL store using a hardcoded path.

This prevents user configuration from controlling the JSONL archive location.

## Scope
Update:
- `agent/factory.py`
- Memory factory tests

## Required Changes
- Replace the hardcoded JSONL path with:
  - `Path(ctx.cfg.memory.memory_jsonl_dir) / "memories.jsonl"`
- Ensure the result is passed to `JsonlMemoryStore`.
- Remove hardcoded `/memories.jsonl`.
- Add debug logging or status output if useful.

## Tests to Add
- Add a factory-level test verifying:
  - `memory_jsonl_dir="/opt/llm/memory"` produces `/opt/llm/memory/memories.jsonl`.
  - No hardcoded root-level `/memories.jsonl` path is used.

## Acceptance Criteria
- JSONL memory archive path is derived from `memory_jsonl_dir`.
- Existing memory ingestion and JSONL write behavior still works.
- Configuration and implementation are consistent.
