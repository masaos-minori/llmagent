# Preserve scoped DB command regression tests

## Priority
Medium

## Labels
tests, db, rag, session

## Dependencies
- Add tests that flat DB aliases are invalid

## Background
Removing flat aliases must not break canonical scoped DB commands.

## Task
Ensure scoped DB command tests remain valid and complete.

## Implementation Requirements
Keep or strengthen tests for:
- `/db rag urls`
- `/db rag clean <url>`
- `/db rag rebuild-fts`
- `/db rag recover`
- `/db rag consistency`
- `/db session recover`
- `/db session stats`
- `/db session health`
- `/db session checkpoint`
- `/db session vacuum`
- `/db session purge`

## Acceptance Criteria
- `/db rag urls` still calls `rag_list_documents`.
- `/db rag clean <url>` still calls `rag_delete_document`.
- `/db rag rebuild-fts` still calls `RagMaintenanceService.rebuild_fts()`.
- `/db rag recover` still calls `RagMaintenanceService.recover()`.
- `/db session recover` still calls `DbMaintenanceService.recover_session()`.
