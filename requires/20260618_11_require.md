# [mcp][response] Improve truncation metadata to show actual visible bytes

## Summary
Current truncation metadata reports the configured limit, not the actual shown byte count.

## Problem
The truncation message can be misleading during troubleshooting.

## Scope
- `mcp/server.py`
- truncation metadata
- response display behavior

## Task
- include the actual visible byte count in truncation metadata
- or update the truncation message so it is less confusing
- keep compatibility where possible

## Expected Result
Truncated responses are easier to understand during debugging and operations.

## Acceptance Criteria
- [ ] truncation metadata reflects actual visible output more clearly
- [ ] operator confusion is reduced
- [ ] response compatibility is preserved as much as possible
