# Make `ToolResultStore.store()` return a non-optional integer ID

## Priority
Medium

## Labels
- db
- tool-results
- type-safety
- medium-priority

## Dependencies
None

## Background
`ToolResultStore.store()` is used by tool execution flow to persist full tool results and return a result ID. The caller uses this ID when replacing oversized tool output with a hint.

Successful inserts should always return an integer row ID. A missing `lastrowid` should be treated as an error.

## Scope
Modify:
- `db/tool_results.py`
- `tests/test_tool_result_store.py`

Review:
- `agent/tool_runner.py`

## Required Changes
- Change `ToolResultStore.store()` return type to `int`.
- After insert, check `cur.lastrowid`.
- If `lastrowid is None`, raise `RuntimeError`.
- Update docstrings to remove `int | None`.
- Keep DB errors fail-fast and re-raised.

## Tests
Update `tests/test_tool_result_store.py`.

Add tests for:
- Successful store returns `int`.
- Stored result can be fetched by the returned ID.
- Simulated `lastrowid is None` raises `RuntimeError`.

## Acceptance Criteria
- `ToolResultStore.store()` always returns `int` on success.
- Callers no longer need to handle `None` for successful inserts.
- Existing tool result tests pass.
