# L-1: Update lifecycle-related comments and docstrings

## Priority

Low

## Background

Several comments and docstrings no longer match the current lifecycle design.

Examples:
- `LifecycleManagerProtocol` says `HttpServerLifecycleManager` satisfies the protocol structurally, but `_ServerLifecycleRouter` is the actual facade.
- `startup.py` mentions `Ondemand servers`, but startup mode currently supports only persistent and subprocess.
- `verify_running()` mentions optional restart, but it only checks and returns a boolean.

## Target files

- `agent/lifecycle_protocol.py`
- `agent/startup.py`
- `agent/http_lifecycle.py`
- `agent/factory.py`

## Implementation tasks

- Update lifecycle protocol docstring to identify `_ServerLifecycleRouter` as the protocol facade.
- Replace or clarify the `Ondemand servers` comment in `startup.py`.
- Update `verify_running()` docstring to state it only checks process state.
- Document stderr log file redirect strategy in `http_lifecycle.py`.
- Document process group shutdown behavior.
- Document shutdown idempotency and cleanup expectations.

## Acceptance criteria

- Comments accurately describe runtime behavior.
- No docstring claims that `verify_running()` restarts anything.
- No docstring refers to unsupported ondemand startup mode as current behavior.
- Lifecycle facade vs low-level manager responsibilities are clear.

## Test plan

- Documentation-only change.
- Run linters or docstring checks if available.
