# [Medium] Document `__list_tools__` as stdio-only control RPC

## Priority
Medium

## Background
`__list_tools__` is used by stdio health checks. Since stdio transport is retained, this reserved RPC should remain, but it must be clearly scoped to stdio control-plane usage.

## Goal
Keep `__list_tools__` as a stdio-only control RPC and prevent confusion with HTTP `/v1/tools`.

## Scope
Affected files:
- `mcp/server.py`
- `shared/tool_executor.py`
- `docs/04_mcp_02_protocol_and_transport.md`
- `docs/04_mcp_03_routing_lifecycle_and_execution.md`

## Implementation Tasks
- Keep `__list_tools__`.
- Document it as `healthcheck_mode="ping_tool"` control RPC for stdio only.
- State that HTTP tool listing uses `/v1/tools`.
- State that `__list_tools__` is not part of routing ownership.
- Keep the reserved `__` prefix rule for user-defined tools.

## Acceptance Criteria
- `__list_tools__` remains available for stdio health checks.
- `/v1/tools` is documented as the HTTP tool-list source.
- Docs clearly separate stdio control RPC from routing.
- No routing logic depends on `__list_tools__`.

## Validation
- Run stdio healthcheck tests if available.
- Run `rg -n "__list_tools__" docs mcp shared` and verify all references describe stdio-only control usage.
