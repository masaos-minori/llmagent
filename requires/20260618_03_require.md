# [mcp][routing] Simplify and standardize tool routing rules

## Summary
Tool routing currently mixes explicit config mapping and static fallback rules.

## Problem
Some tools are resolved by config, others by static fallback sets.
Some tools, such as `query_sqlite`, are special cases and require explicit config only.
This makes routing harder to understand and maintain.

## Scope
- `shared/route_resolver.py`
- tool routing config
- tool registration flow for new servers

## Task
- define one clear primary routing method
- reduce special-case tools where possible
- document how new tools are added to routing
- improve startup validation for routing gaps

## Expected Result
Tool routing becomes predictable and easier to maintain.

## Acceptance Criteria
- [ ] primary routing method is clearly defined
- [ ] special-case routing exceptions are reduced
- [ ] adding a new server/tool has one clear routing procedure
