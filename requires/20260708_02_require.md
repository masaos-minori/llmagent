# H-2: Remove ToolResultStore writes from tool result collection

## Priority

High

## Background

`_collect_tool_result_msgs()` currently persists full tool outputs into `ctx.tool_result_store` and receives a `result_id` used by the `/tool show` retrieval hint.

The full tool output archive is being removed. Tool outputs should remain only in the normal conversation message flow after truncation.

## Target files

- `agent/tool_runner.py`

## Implementation tasks

- Remove `ctx.tool_result_store.store(...)` from `_collect_tool_result_msgs()`.
- Remove `result_id` creation and usage.
- Remove persistence of:
  - `full_text`
  - `summary`
  - `args_masked`
  - `is_error`
  - `tool_name`
  - `turn`
- Keep `ctx.conv.history.append({"role": "tool", ...})`.
- Keep `ctx.session.save_many(tool_msgs)`.
- Ensure tool result persistence to the `messages` table still works.
- Ensure tool statistics still update via `_update_stats_for_result()`.

## Acceptance criteria

- Tool execution no longer writes to `ToolResultStore`.
- Tool result full text is not persisted to the `tool_results` table.
- Tool role messages are still added to `ctx.conv.history`.
- Tool role messages are still saved through `ctx.session.save_many(...)`.
- Tool call and tool error counters still work.

## Test plan

- Execute a successful tool call and verify a tool message is added to history.
- Execute a failing tool call and verify stats are updated.
- Verify no `ctx.tool_result_store.store(...)` call occurs.
- Verify session message persistence still contains the LLM-visible tool message.
