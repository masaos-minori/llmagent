# L-3: Update turn-flow and LLM streaming documentation for diagnostics-only partial completions

## Priority

Low

## Background

Turn-flow and streaming documentation currently says partial completions are stored in both ToolResultStore and session diagnostics, and that users can inspect them with `/tool show llm_partial_completion`.

After removal, partial completions should be documented as diagnostics-only.

## Target files

- `05_agent_03_turn-processing-flow.md`
- `05_agent_05_llm-and-streaming.md`
- `05_agent_06_tool-execution-and-approval.md`

## Implementation tasks

- Remove references to `tool_result_store` from partial completion sections.
- Remove references to `/tool show llm_partial_completion`.
- State that partial completions are stored in `session_diagnostics`.
- Preserve the invariant that partial content is not added to `ctx.conv.history`.
- Remove Tool Result Summarization documentation.
- Remove `is_summarized()` documentation.

## Acceptance criteria

- Partial completion docs describe diagnostics-only persistence.
- No docs instruct users to inspect partial completions via `/tool show`.
- Tool Result Summarization is no longer documented as active behavior.
- LLM context pollution prevention remains documented.

## Test plan

- Search docs for `llm_partial_completion`.
- Search docs for `tool_result_store`.
- Search docs for `Tool Result Summarization`.
- Verify remaining partial completion documentation is consistent.
