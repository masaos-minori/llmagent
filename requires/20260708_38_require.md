# M-6: Update tests for MCP restart-required reload behavior

## Priority

Medium

## Background

Existing tests may expect MCP URL changes to be applied and auth/startup mode changes to be deferred.

Those expectations must be replaced with restart-required behavior and no mutation.

## Target files

- `tests/test_config_reload.py`
- `tests/test_mcp_reload.py`
- `tests/test_tool_executor.py`
- `tests/test_http_transport.py`
- `tests/test_lifecycle.py`
- `tests/test_repl_health.py`
- `tests/test_cmd_reload.py`

## Implementation tasks

- Update URL reload tests:
  - expect `needs_restart`
  - no MCP entry in `applied`
  - running config URL unchanged
- Update auth token reload tests:
  - expect `needs_restart`
  - no MCP entry in `deferred`
  - running config token unchanged
- Update startup mode reload tests:
  - expect `needs_restart`
  - no MCP entry in `deferred`
  - running config startup mode unchanged
- Add tests for:
  - `cmd`
  - `env`
  - `call_timeout_sec`
  - `startup_timeout_sec`
  - `tool_names`
  - server add
  - server remove
  - `github_server_url` removal
- Update `/reload` output tests:
  - `[RESTART]` appears
  - `[DEFER]` does not appear for MCP changes
- Update ToolExecutor tests:
  - `apply_config()` only changes cache TTL
  - transports are not recreated

## Acceptance criteria

- Tests reflect restart-required MCP reload behavior.
- Tests prove no running MCP config mutation occurs.
- Tests prove no MCP deferred entries are emitted.
- Tests prove `github_server_url` is removed or rejected.

## Test plan

- Run targeted config reload tests.
- Run ToolExecutor tests.
- Run HTTP transport tests.
- Run lifecycle/watchdog tests if present.
- Run command output tests for `/reload`.
