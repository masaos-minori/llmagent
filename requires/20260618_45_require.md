# [rag][query][refiner] Document _augment_refiner() empty-result fallback behavior

## Summary
The refiner fallback path is not documented clearly enough.

## Problem
When `_augment_refiner()` returns an empty string, the caller falls back to raw chunks.
This behavior is not explained clearly enough.

## Scope
- `rag/pipeline.py`
- refiner behavior
- refiner documentation
- query diagnostics

## Task
- document the refiner behavior for:
  - success
  - exception
  - empty output
  - raw-chunk fallback
- make fallback reasons easier to understand

## Expected Result
Refiner behavior is easier to debug and reason about.

## Acceptance Criteria
- [ ] empty-output fallback is documented
- [ ] exception fallback is documented
- [ ] raw-chunk fallback behavior is documented clearly
