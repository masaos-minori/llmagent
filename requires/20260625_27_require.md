## [High][eventbus][tests] Add end-to-end tests for consumer failure, timeout, retry, and inline DLQ promotion

### Summary
The retry and DLQ redesign affects delivery semantics, not just individual utility functions.

### Problem
Without end-to-end tests, the system can regress in the most important failure paths:
consumer failure, timeout, retry counting, and DLQ promotion timing.

### Scope
- tests/test_eventbus*.py
- consumer failure scenarios
- timeout scenarios
- retry accounting
- DLQ timing behavior

### Task
- add integration tests for normal consumer failure
- add integration tests for consumer timeout
- verify retry counters change on real delivery failure
- verify inline DLQ promotion happens immediately at threshold
- verify requeue and subsequent redelivery behavior

### Expected Result
The Event Bus failure lifecycle is protected by regression tests.

### Acceptance Criteria
- tests cover consumer failure
- tests cover timeout handling
- tests verify retry counter semantics
- tests verify immediate DLQ promotion
- tests verify requeue behavior after DLQ promotion
