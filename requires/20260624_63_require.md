# [High] Introduce a centralized repository gateway for all write operations

## Summary
Create a single repository write boundary and route all local/GitHub write operations through it.

## Problem
Repository write behavior is currently distributed across tool execution paths.
This creates risk of:
- approval bypass
- policy bypass
- inconsistent audit coverage
- plugin-based write shortcuts

## Goal
Establish one centralized gateway for all repository write operations.

## Scope
Target files may include:
- new `agent/repository_gateway.py`
- `shared/tool_executor.py`
- workflow publish-related code
- GitHub/file write integration paths

## Required Changes
1. Create `repository_gateway.py` as the sole write entry point.
2. Route destructive/local/GitHub write operations through the gateway.
3. Make the gateway enforce:
   - policy checks
   - approval checks
   - audit logging
4. Prevent direct write execution from plugin tools and ad hoc MCP paths where possible.
5. Define clear read vs write boundaries.

## Acceptance Criteria
- All repository write operations flow through the gateway.
- The gateway emits consistent audit records.
- Approval and policy are enforced before write execution.
- Direct write paths outside the gateway are removed or blocked by tests.

## Out of Scope
- Full Publisher role introduction
- Complete workflow phase redesign
