# Title
Reorganize Event Bus tests by feature and rename test files to function-based names

## Summary
Refactor the current Event Bus test layout so tests are grouped by feature instead of implementation phase, and rename the test files to clearer, function-based names.

## Background
The current test layout is phase-based:
- `tests/test_eventbus_phase1.py`
- `tests/test_eventbus_phase2.py`
- `tests/test_eventbus_phase3.py`

According to the completed implementation plan:
- Phase 1 covers health, publish, idempotency, and schema validation.
- Phase 2 covers replay, replay filtering, subscribe behavior, and offset read/write behavior.
- Phase 3 covers DLQ promotion, DLQ listing, and DLQ requeue behavior.

This structure was useful during phased implementation, but it is less clear now that the feature is complete.

## Problem
The current phase-based test naming does not match the long-term maintenance view of the Event Bus.
Future maintainers and AI agents need feature-oriented test discovery, not historical phase-oriented grouping.
Tests should be grouped by runtime behavior and subsystem responsibility, such as:
- publish / validation
- replay / subscribe
- offsets
- DLQ

## Required Changes
Reorganize the existing tests by feature and rename the files accordingly.

Suggested target structure:
- `tests/test_eventbus_publish.py`
- `tests/test_eventbus_replay_subscribe.py`
- `tests/test_eventbus_offsets.py`
- `tests/test_eventbus_dlq.py`

Move the existing tests into the most appropriate file based on behavior:
- health / publish / idempotency / schema validation → publish-oriented test module.
- replay / replay filtering / subscribe query behavior → replay/subscribe test module.
- offset read/write and consumer resume behavior → offsets test module.
- DLQ promotion / DLQ list / DLQ requeue → DLQ test module.

If needed, update imports, fixtures, and test names so the new structure is consistent and easy to navigate.

## Acceptance Criteria
- The old phase-based test files no longer exist.
- Tests are grouped by feature, not by historical implementation phase.
- File names clearly communicate the feature under test.
- The full Event Bus test suite still passes after the reorganization.
- Test discovery is easier for both humans and AI implementers.

## Out of Scope
- Adding new Event Bus runtime behavior.
- Refactoring production code unless required to preserve test execution.
- Changing delivery semantics, DLQ logic, or subscribe behavior.

## Components
- `tests/test_eventbus_phase1.py`
- `tests/test_eventbus_phase2.py`
- `tests/test_eventbus_phase3.py`
- Any shared Event Bus test fixtures
- Event Bus test execution commands referenced in the completed plan
