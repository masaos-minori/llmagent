## Title
[Medium] Strengthen workflow definition validation

## Priority
Medium

## Labels
workflow, validation, config, robustness

## Dependencies
None

## Background
The default workflow definition includes `plan`, `execute`, `verify`, and `retry` stages. Current loader tests validate basic schema errors, but do not appear to enforce duplicate stage ID rejection or required stage presence.

Invalid workflow definitions should fail early.

## Scope
Update:
- `agent/workflow/workflow_loader.py`
- `tests/test_workflow_loader.py`
- Workflow documentation

## Required Changes
- Reject duplicate stage IDs.
- Require at least:
  - `plan`
  - `execute`
  - `verify`
- Validate retry policy:
  - `max_attempts >= 1`
  - `backoff` is one of supported values
  - `backoff_sec >= 0`
- Document the role of the optional `retry` stage if it remains in `default.json`.

## Tests to Add
- `test_duplicate_stage_ids_rejected`
- `test_missing_plan_stage_rejected`
- `test_missing_execute_stage_rejected`
- `test_missing_verify_stage_rejected`
- `test_invalid_retry_policy_rejected`

## Acceptance Criteria
- Invalid workflow definitions fail during load.
- Error messages identify the invalid field.
- Default workflow remains valid.
