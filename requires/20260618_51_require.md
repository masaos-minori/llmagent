# [rag][observability] Improve ingestion and query diagnostics for fallback and quality analysis

## Summary
The pipeline already records some runtime data, but fallback and quality problems are still hard to trace.

## Problem
It is still hard to inspect:
- refiner fallback
- remote RAG fallback
- skipped chunks
- consistency warnings
- retrieval-quality drop causes

## Scope
- query diagnostics
- ingestion diagnostics
- timing and fallback metadata
- quality analysis support

## Task
- add more structured diagnostics for:
  - refiner usage
  - refiner fallback reason
  - remote RAG fallback reason
  - skipped chunk counts
  - consistency warning counts
- make quality problems easier to investigate

## Expected Result
Fallback behavior and retrieval-quality problems are easier to debug.

## Acceptance Criteria
- [ ] fallback-related diagnostics are added
- [ ] skipped-chunk visibility is improved
- [ ] quality-analysis signals are easier to inspect
