# [RAG][Observability] Add stage-level observability for pipeline fallback behavior

## Summary
The pipeline has many fallback paths, but stage-level results are not visible enough.

## Problem
It is hard to tell:
- which stage failed
- which stage used fallback
- whether a result is degraded or fully normal

## Scope
- `rag/pipeline.py`
- stage execution flow
- runtime logging and metrics

## Task
- expose stage-level success/failure/fallback information
- record elapsed time per stage
- make degraded behavior visible in logs or debug output

## Expected Result
Pipeline execution is easier to debug and operate.

## Acceptance Criteria
- [ ] stage-level results are collected
- [ ] fallback behavior is visible
- [ ] debug or logs can show stage outcomes clearly
