# Redefine static tool_definitions and tool_constants as compatibility-only

## Priority

Medium

## Target files

- agent/config_dataclasses.py
- agent/config_builders.py
- agent/services/config_reload.py
- shared/tool_constants.py
- shared/tool_registry.py
- shared/tool_executor_helpers.py
- agent/tool_policy.py
- docs related to tool definitions
- tests using static tool_definitions or tool_constants

## Reason

Static `tool_definitions` and `tool_constants.py` are currently used as runtime sources for tool schema, routing seed, side-effect classification, and policy classification.

The runtime source should become RuntimeToolRegistry built from MCP `/v1/tools`.

## Intent

Keep static definitions only as migration compatibility, fallback, or test fixture support.

## Scope

Redefine:

- `tool_definitions` as fallback or test fixture only
- `tool_constants.py` as compatibility-only
- static ToolRegistry as non-authoritative for runtime execution

## Acceptance criteria

- Main runtime execution path does not depend on static tool_definitions.
- Main runtime execution path does not depend on tool_constants as source of truth.
- Remaining usage is clearly marked compatibility-only.
- Documentation states MCP `/v1/tools` is the normal source.
