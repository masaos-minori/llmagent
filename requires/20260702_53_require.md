# Remove `McpServerConfig._cast_enums()` compatibility shim

## Priority

High

## Summary

Remove the compatibility shim `McpServerConfig._cast_enums()` and replace it with strict enum type validation.

## Background

`McpServerConfig._cast_enums()` currently converts string inputs into enum values. This is documented as a compatibility shim for direct runtime callers.

The canonical config loading path already performs enum conversion in `_build_single_server()` before constructing `McpServerConfig`.

Therefore, enum casting inside `McpServerConfig` is compatibility-only and should be removed.

## Goal

Make `McpServerConfig` accept normalized enum values only, while keeping builder-based TOML config loading behavior unchanged.

## Target files

- `shared/mcp_config.py`
- `tests/test_mcp_config.py`
- MCP configuration documentation

## Scope

### In scope

- Remove `_cast_enums()`.
- Remove calls to `_cast_enums()`.
- Add strict enum type validation.
- Keep `_build_single_server()` enum conversion.
- Update tests and documentation.

### Out of scope

- Removing enum conversion from `_build_single_server()`.
- Changing TOML config format.
- Supporting string enum values in direct runtime construction.

## Implementation tasks

- Delete `McpServerConfig._cast_enums()`.
- Update `McpServerConfig.__post_init__()` so it calls:
  - `_validate_enum_types()`
  - `_validate_cross_fields()`
- Add `_validate_enum_types()`.

- `_validate_enum_types()` must validate:
  - `transport` is an instance of `TransportType`
  - `startup_mode` is an instance of `StartupMode`
  - `healthcheck_mode` is an instance of `HealthcheckMode`

- If any of the fields is not the expected enum instance, raise `ValueError`.
- Keep `_build_single_server()` converting TOML strings to enum values before constructing `McpServerConfig`.
- Keep existing tests that verify invalid direct string values fail.
- Add a test that verifies valid direct string values are rejected.

## Expected behavior

- Valid direct construction:

    McpServerConfig(TransportType.HTTP, "http://127.0.0.1:8000")

  should still work.

- Invalid direct construction:

    McpServerConfig("http", "http://127.0.0.1:8000")

  should fail with `ValueError`.

- Builder-based construction from TOML-style strings should still work through `_build_mcp_servers()`.

## Acceptance criteria

- `_cast_enums()` does not exist.
- No call to `_cast_enums()` remains.
- Direct runtime construction requires enum values.
- `_build_mcp_servers()` still accepts TOML string values and returns normalized `McpServerConfig` objects.
- `tests/test_mcp_config.py` passes.
