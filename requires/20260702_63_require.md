# H-3: Consolidate built-in slash command definitions into `command_defs.py`

## Priority

High

## Labels

refactor, agent, cli, command-registry, maintainability

## Background

Built-in slash command definitions are currently duplicated.

`command_defs.py` documents `_COMMANDS` as the single source of truth, but `registry.py` also defines its own `_COMMANDS` list.

This creates drift risk between:

- command dispatch
- `/help` output
- documentation
- handler validation
- test coverage

Known drift examples:

- `/approve` and `/reject` are present in `registry.py`
- `/db` must be async because `_cmd_db()` is an async handler

## Goal

Make `scripts/agent/commands/command_defs.py` the only source of truth for built-in slash command definitions.

## Scope

### In scope

- Move all current built-in command definitions into `command_defs.py`
- Remove `_COMMANDS` definition from `registry.py`
- Import `_COMMANDS` from `command_defs.py`
- Preserve current command behavior
- Preserve command order in `/help`

### Out of scope

- `/exit` handling
- plugin command registry redesign
- sqlite-vec handling

## Required changes

In `registry.py`, replace local `_COMMANDS` definition with:

- `from agent.commands.command_defs import _COMMANDS, CommandDef`

Ensure `command_defs.py` includes:

- `/help`
- `/config`
- `/stats`
- `/context`
- `/plan`
- `/undo`
- `/reload`
- `/compact`
- `/mcp`
- `/session`
- `/clear`
- `/rag`
- `/export`
- `/history`
- `/system`
- `/db`
- `/tool`
- `/set`
- `/memory`
- `/debug`
- `/audit`
- `/approve`
- `/reject`
- `/plugin`
- `/mdq`

Ensure `/db` has `is_async=True`.

## Acceptance criteria

- `_COMMANDS` is defined only in `command_defs.py`
- `registry.py` imports `_COMMANDS`
- `/help` lists the same command set used by dispatch
- `/approve` and `/reject` remain available
- `/db` dispatch still awaits `_cmd_db()`
- Existing command behavior is preserved

## Validation

Run:

- `grep -R "^_COMMANDS" -n scripts/agent/commands`

Expected result:

- Only `scripts/agent/commands/command_defs.py` defines `_COMMANDS`

Manual validation:

- Start agent
- Run `/help`
- Confirm `/approve`, `/reject`, `/db`, `/mdq`, and `/plugin` are listed
- Run `/db help`
