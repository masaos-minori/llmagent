# Add automated synchronization checks between CommandDef and command handlers

## Priority

Medium

## Type

Implementation / Test / Maintainability

## Background

Built-in slash commands are defined in command definitions and dispatched by `CommandRegistry`.

Plugin commands are lower priority than built-in commands. Plugin commands that shadow built-in commands should be rejected at load time.

## Problem

If command definitions and command handlers drift apart, the following behaviors can become inconsistent:

- Dispatch behavior
- `/help` output
- Plugin shadow rejection
- Tests
- Operator expectations

## Goal

Ensure command definitions and command handlers remain synchronized.

## Scope

Affected areas:

- `agent/commands/command_defs_list.py`
- `agent/commands/registry.py`
- Command mixin modules
- `/help` output generation
- Plugin command shadow detection tests

## Implementation Tasks

- Add a test that every `CommandDef` has a corresponding handler.
- Add a test that every built-in handler is represented by a `CommandDef`.
- Verify `/help` output is generated from the command definition source.
- Add tests for exact command matching.
- Add tests for prefix command matching.
- Add tests for prefix/exact conflicts.
- Add tests that plugin command shadowing is rejected for every built-in command.

## Acceptance Criteria

- CI fails when a built-in command handler lacks a `CommandDef`.
- CI fails when a `CommandDef` has no dispatch handler.
- `/help` output remains in sync with command definitions.
- Plugin shadow rejection covers all built-in commands.
- Prefix commands do not accidentally shadow exact commands.

## Validation Plan

Add tests for:

- `/mcp`
- `/mcp status`
- `/mdq`
- `/plugin status`
- `/approve`
- `/reject`
- `/memory`
- Prefix command dispatch.
- Exact command dispatch.
- Plugin shadow conflict.

## Risks

The test may expose existing command drift. Fix the drift in implementation rather than weakening the test.

## Out of Scope

- Redesigning the command system.
- Changing user-facing command names unless required by a conflict.
- Documentation-only updates.
