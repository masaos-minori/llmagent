# [High] Add Event Bus schema creation to db/create_schema.py

## Goal

Create `eventbus.sqlite` through the unified DB schema creation entrypoint.

## Background

`db/create_schema.py` already initializes RAG, session, and workflow schemas through `create_rag_schema()`, `create_session_schema()`, `create_workflow_schema()`, and `create_schema()`. Event Bus schema creation should be added to the same entrypoint. 【2-418bb2】

## Scope

- Schema creation only
- No Event Bus Python runtime integration
- No Event Bus repository or dispatcher

## Implementation Tasks

- Import `build_eventbus_schema_sql()` from `db.schema_sql`.
- Add `create_eventbus_schema()`.
- Implement `create_eventbus_schema()` using `SQLiteHelper("eventbus")`.
- Execute `build_eventbus_schema_sql()` with `db.executescript(...)`.
- Update `create_schema()` to call:
  - `create_rag_schema()`
  - `create_session_schema()`
  - `create_workflow_schema()`
  - `create_eventbus_schema()`
- Update logging to mention Event Bus schema creation.
- Update tests for `create_schema()`.

## Acceptance Criteria

- Running `python -m db.create_schema` creates:
  - `rag.sqlite`
  - `session.sqlite`
  - `workflow.sqlite`
  - `eventbus.sqlite`
- `eventbus.sqlite` contains the `events` table.
- `eventbus.sqlite` contains all expected indexes.
- Event Bus runtime integration is not added.

## Validation

- Run:

  PYTHONPATH=scripts python -m db.create_schema

- Verify:

  sqlite3 /opt/llm/db/eventbus.sqlite ".tables"
  sqlite3 /opt/llm/db/eventbus.sqlite ".indexes events"

- Run schema creation tests.
