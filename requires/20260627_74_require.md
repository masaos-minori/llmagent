# Roll back SQLite transactions on all normal exceptions

## Priority
High

## Labels
- db
- sqlite
- transactions
- reliability
- high-priority

## Dependencies
None

## Background
`SQLiteHelper.begin_immediate()` and `SQLiteHelper.begin_exclusive()` currently roll back only when `sqlite3.Error` is raised.

If a normal Python exception such as `ValueError`, `RuntimeError`, or `TypeError` occurs inside the transaction block, the transaction may remain open or partially applied.

## Scope
Modify:
- `db/helper.py`

Add or update tests:
- `tests/test_db_store_impl.py`
- or create `tests/test_sqlite_helper.py`

## Required Changes
- Change transaction context managers to catch `Exception`, not only `sqlite3.Error`.
- Do not catch `BaseException`.
- Attempt rollback for all normal exceptions.
- If rollback itself fails, suppress only SQLite-related rollback errors.
- Re-raise the original exception.

## Tests
Add tests for:
- `begin_immediate()` rolls back when `ValueError` is raised inside the block.
- `begin_exclusive()` rolls back when `RuntimeError` is raised inside the block.
- A subsequent insert can run after rollback.
- Existing sqlite3 error behavior remains unchanged.

## Acceptance Criteria
- Normal exceptions inside a transaction trigger rollback.
- The original exception is re-raised.
- No dangling transaction remains.
- Existing DB tests continue to pass.
