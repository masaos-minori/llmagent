# Title
Synchronize compressed conversation history with session persistence

## Priority
High

## Labels
- bug
- agent
- history
- persistence
- session
- data-consistency

## Dependencies
- None

## Summary
Fix the mismatch between in-memory compressed conversation history and the history reconstructed from `session.sqlite`.

## Background
The current turn flow compresses `ctx.conv.history` when limits are exceeded.
`/session load <id>` reconstructs history from the database.
The design docs do not clearly show how compressed history is synchronized back to persistent storage.

## Problem
A long-running session may continue with compressed in-memory history, but a later `/session load` may restore an uncompressed or semantically different history.
This breaks session reproducibility and may change the context seen by the LLM after reload.

## Required Changes
- Define the canonical persistence model for compressed history.
- Either:
  - persist compressed history back into `messages`, or
  - persist enough compression metadata to reconstruct the same effective history on reload.
- Ensure `/session load`, `/history`, and `/export` behave consistently with the compressed session state.
- Add regression tests for “compress → reload session → equivalent restored history”.

## Acceptance Criteria
- Reloaded sessions are semantically consistent with the compressed in-memory state.
- Compressed and restored histories no longer diverge silently.
- The persistence model is documented clearly.

## Out of Scope
- Full redesign of HistoryManager
- Changing compression policy itself

## Components
- `agent/history.py`
- `agent/session.py`

- `/history`
- `/export`
- related docs and tests

## AI Implementation Instruction
Implement this as a history/persistence consistency fix.

Strict rules:
- Do not leave compressed history as an in-memory-only behavior if `/session load` must reconstruct session state.
- Prefer explicit persistence or explicit reconstruction metadata over hidden heuristics.
- Preserve current compression semantics unless a contradiction requires change.
- If the intended canonical source of compressed history is still unclear, mark it as: `Needs confirmation`
