# [Medium] Use `rag.types.RagHit` as the only canonical hit type alias

## Priority
Medium

## Difficulty
Medium

## Background
`rag/types.py` defines `RagHit`, but several modules locally redefine `RagHit = RawHit | MergedHit | RankedHit`.

This creates duplicate type aliases and makes future hit model changes error-prone.

## Goal
Define `RagHit` in exactly one place: `rag.types`.

## Scope
Affected files:

- `scripts/rag/types.py`
- `scripts/rag/stage.py`
- `scripts/rag/pipeline.py`
- `scripts/rag/repository.py`
- `scripts/rag/llm_client.py`
- `scripts/rag/llm_prompts.py`
- `scripts/rag/pipeline_refiner.py`
- `scripts/mcp/rag_pipeline/service.py`

## Implementation Tasks
- Keep `RagHit` definition in `rag.types`.
- Replace local aliases with `from rag.types import RagHit`.
- Remove duplicate `RawHit | MergedHit | RankedHit` aliases from other modules.
- Update `__all__` if necessary.

## Acceptance Criteria
- Only `scripts/rag/types.py` defines `RagHit`.
- All other modules import `RagHit` from `rag.types`.
- Tests pass without type alias regressions.

## Validation
- Run `rg -n "RagHit\\s*=" scripts/rag scripts/mcp`.
- Expected result: only `scripts/rag/types.py` contains the canonical alias definition.
