# [High] Move startup mode validation out of tests/test_route_resolver.py

## Priority

High

## Goal

Keep tests/test_route_resolver.py focused only on routing behavior.

## Background

tests/test_route_resolver.py currently mixes routing tests, config validation tests, and removed stdio startup mode tests. Startup mode validation should belong to tests/test_mcp_config.py.

## Scope

Affected files:

- tests/test_route_resolver.py
- tests/test_mcp_config.py

## Tasks

- Remove startup mode validation tests from tests/test_route_resolver.py.
- Move relevant HTTP-only startup mode validation to tests/test_mcp_config.py.
- Delete tests that validate stdio startup mode behavior.
- Keep only route resolution, unknown tool, registry routing, and config non-routing tests in tests/test_route_resolver.py.

## Unknowns / Open Questions

- Confirm whether tests/test_mcp_config.py already covers all HTTP-only startup mode cases.
- Confirm whether strict-mode route resolver error message tests depend on config validation.

## Acceptance Criteria

- tests/test_route_resolver.py does not validate StartupMode.
- tests/test_mcp_config.py owns McpServerConfig validation.
- No stdio startup mode test remains.

## Validation

Run:
```bash
uv run pytest tests/test_route_resolver.py tests/test_mcp_config.py -v
```
