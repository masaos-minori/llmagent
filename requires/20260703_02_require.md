# Document `command_defs_list.py` as the canonical source for built-in commands

## Priority

High

## Labels

docs, agent, cli, command-registry, maintainability

## Background

The command registry currently separates command metadata responsibilities across modules:

- `command_defs.py` defines `CommandDef` and `SubcommandSpec`
- `command_defs_list.py` defines `_COMMANDS`
- `registry.py` imports `_COMMANDS` and performs dispatch

This separation is valid, but it must be explicitly documented to prevent future duplication of `_COMMANDS` in `registry.py` or `command_defs.py`.

## Goal

Make the canonical command definition ownership explicit.

## Scope

### In scope

- Update module docstrings and developer documentation
- Clarify that `_COMMANDS` must be defined only in `command_defs_list.py`
- Clarify that `registry.py` must import `_COMMANDS`
- Clarify that `command_defs.py` contains only dataclass definitions

### Out of scope

- Changing command behavior
- Adding or removing commands
- `/exit` handling
- Plugin command behavior

## Required documentation

Document the following ownership model:

- `scripts/agent/commands/command_defs.py`
  - Owns `CommandDef`
  - Owns `SubcommandSpec`
  - Does not own `_COMMANDS`

- `scripts/agent/commands/command_defs_list.py`
  - Owns `_COMMANDS`
  - Is the single source of truth for built-in slash commands

- `scripts/agent/commands/registry.py`
  - Imports `_COMMANDS`
  - Owns dispatch behavior
  - Does not define `_COMMANDS`

## Acceptance criteria

- Documentation clearly states that `_COMMANDS` lives only in `command_defs_list.py`
- `registry.py` does not define `_COMMANDS`
- `command_defs.py` does not define `_COMMANDS`
- Future command additions are directed to `command_defs_list.py`

## Validation

Run:

- `grep -R "^_COMMANDS" -n scripts/agent/commands`

Expected result:

- Only `scripts/agent/commands/command_defs_list.py` defines `_COMMANDS`

## Affected files

- `scripts/agent/commands/command_defs.py`
- `scripts/agent/commands/command_defs_list.py`
- `scripts/agent/commands/registry.py`
- Developer documentation

## Dependencies

None
