# H-3: Remove /tool show retrieval hint from per-turn tool result limit handling

## Priority

High

## Background

`TURN_LIMIT_HINT` currently instructs users to retrieve omitted full output via `/tool show <id>`. Since `/tool show` and `ToolResultStore` are being removed, this hint must no longer reference retrieval by result ID.

## Target files

- `agent/tool_result_formatter.py`
- `agent/tool_runner.py`

## Implementation tasks

- Replace `TURN_LIMIT_HINT` with a retrieval-free message:
  - `[Result omitted: per-turn tool result limit reached.]`
- Remove the `result_id` argument from `_apply_turn_char_limit()`.
- Remove `id_hint` generation.
- Remove `TURN_LIMIT_HINT.replace(...)` logic.
- Update all call sites of `_apply_turn_char_limit()`.
- Ensure the per-turn character limit still prevents oversized tool output from being injected into the LLM context.

## Acceptance criteria

- No tool result limit message mentions `/tool show`.
- No tool result limit message includes a result ID.
- Per-turn tool result limiting still works.
- Tool messages remain valid OpenAI-compatible tool role messages.

## Test plan

- Configure a small `tool_results_turn_max_chars`.
- Execute multiple tool calls whose combined output exceeds the limit.
- Verify later tool messages are replaced with the new omission hint.
- Verify the hint contains no `/tool show` reference and no result ID.
