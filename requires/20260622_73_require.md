# Title
Add deterministic fallback and visible failure handling for session title generation

## Summary
Ensure session title generation failures never leave a session without a usable title, and make those failures visible to operators.

## Background
Session title generation runs asynchronously on the first turn and may fail silently except for logs.
There is no guaranteed fallback title today.

## Problem
A background failure can leave the session title empty or low quality, which weakens session management and makes the failure hard to notice.

## Required Changes
- Add a deterministic fallback title when LLM title generation fails.
- Use a stable fallback such as the first user input truncated to a safe length.
- Surface title-generation failure through audit/debug/warning output.
- Ensure session listings never show an empty title.

## Acceptance Criteria
- Every session ends up with a non-empty title.
- Title failure is visible to operators.
- Existing successful title generation behavior still works.

## Out of Scope
- Advanced title re-ranking or title editing workflows
- Redesigning session lifecycle logic

## AI Implementation Instruction
Implement this as a reliability and operator-visibility task.

Strict rules:
- Do not remove async title generation unless strictly necessary.
- Add a deterministic fallback, not another best-effort retry chain.
- Preserve current successful title behavior.
- Keep failure visible in at least one operator-facing surface.
- If fallback formatting rules are unclear, mark them as: Needs confirmation

Deliverables:
1. Fallback title behavior
2. Failure visibility
3. Validation that session list output never shows empty titles
