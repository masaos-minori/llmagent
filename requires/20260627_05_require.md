# Title
Synchronize RAG consistency report fields, CLI output, and documentation

## Priority
Medium

## Labels
- enhancement
- rag
- consistency-check
- operations
- diagnostics
- cli

## Dependencies
- None

## Summary
Ensure RAG consistency checks expose the fields and affected identifiers promised by the operations documentation.

## Background
The operations documentation describes both count-based consistency fields and a repair flow that includes affected `chunk_id`, `doc_id`, and URL identifiers. The implementation and report type must be verified against this documentation.

## Problem
If `/db consistency` only reports counts, operators still need manual database investigation to repair FTS or vector index issues.

## Required Changes
- Inspect `RagConsistencyReport` and `check_rag_consistency()`.
- Ensure the report includes or can derive:
  - `chunks`
  - `fts`
  - `vec`
  - `fts_gap`
  - `fts_orphan_count`
  - `orphan_vec_count`
  - affected `chunk_id`
  - affected `doc_id`
  - affected URL
- Limit affected identifiers to a safe maximum, such as 10 per category.
- Update `summarize_issues()` to include repair hints.
- Ensure `/db consistency` prints both counts and actionable repair guidance.
- Keep the consistency check read-only.
- Add tests for all mismatch categories.

## Acceptance Criteria
- `/db consistency` output matches the documentation.
- Affected identifiers are shown where feasible.
- Repair hints distinguish:
  - FTS rebuild
  - forced re-ingestion
  - manual investigation
- The check does not mutate data.
- Tests cover FTS gap, FTS orphan, vector orphan, and vector/chunk count mismatch.

## Out of Scope
- Automatic destructive repair.
- Background self-healing jobs.
- Changing ingestion behavior unless needed to expose identifiers.

## Components
- `db/maintenance.py`
- `/db consistency` command implementation
- `03_rag_05_configuration_and_operations.md`
- `tests/test_rag_consistency.py`

## AI Implementation Instruction
Implement this as an operational diagnostics alignment task.

Strict rules:
- Do not make `check_rag_consistency()` perform repairs.
- Do not print vague messages without repair guidance.
- Keep large-database safety by limiting affected identifier output.
- If URL resolution is expensive or ambiguous, mark it as: `Needs confirmation`.
