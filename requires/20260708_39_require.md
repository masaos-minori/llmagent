# L-1: Update Agent Configuration documentation for MCP restart-required behavior

## Priority

Low

## Background

Agent configuration documentation currently describes MCP HTTP URL as hot-reloadable and `auth_token` / `startup_mode` as deferred.

That behavior is being removed.

## Target files

- `05_agent_08_configuration.md`
- related configuration docs
- config samples

## Implementation tasks

- Remove wording that says MCP HTTP URL is hot-reloadable.
- Remove wording that says MCP `auth_token` is deferred.
- Remove wording that says MCP `startup_mode` is deferred.
- State that all MCP server definition changes are restart-required.
- Add restart-required examples:
  - `mcp/<server>.url`
  - `mcp/<server>.auth_token`
  - `mcp/<server>.startup_mode`
  - `mcp/<server>.cmd`
  - `mcp/<server>.env`
- Remove `github_server_url` from documentation.
- State that GitHub MCP endpoint is configured only through `mcp_servers.github.url`.
- Align Agent docs with MCP docs on config ownership.

## Acceptance criteria

- Configuration docs no longer mention MCP URL hot reload.
- Configuration docs no longer mention MCP deferred auth or startup mode.
- `github_server_url` is not documented as active config.
- Restart-required MCP behavior is clearly stated.

## Test plan

- Search docs for `HTTP URL: hot-reloadable`.
- Search docs for `auth_token, startup_mode: deferred`.
- Search docs for `github_server_url`.
- Verify remaining text points to `mcp_servers.github.url`.
