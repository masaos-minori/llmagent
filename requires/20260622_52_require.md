# Title
Add `dry_run` support to `create_directory`

## Summary
Extend `create_directory` so that it supports `dry_run` consistently with other file mutation tools.

## Background
Most file-write tools already support preview-style execution, but `create_directory` does not.

## Problem
This makes the safety model inconsistent across write-oriented tools and prevents a uniform “preview first” workflow.

## Required Changes
- Add `dry_run` to the request model for `create_directory`.
- Add `dry_run` to the MCP tool schema.
- Implement preview-only behavior in the service layer.
- Document the behavior consistently with other write tools.

## Acceptance Criteria
- `create_directory` accepts `dry_run=True`.
- Preview mode validates and reports the intended action without creating directories.
- Model, schema, and service behavior are consistent.
- Documentation no longer treats `create_directory` as an unexplained exception.

## Out of Scope
- Broader redesign of file-write approval logic
- Permission model changes

## AI Implementation Instruction
Implement this as a consistency and safety-alignment task.

Strict rules:
- Match the existing `dry_run` pattern used by other file mutation tools.
- Do not introduce hidden side effects in preview mode.
- Keep schema/model/service alignment exact.
- If the current service layer cannot support a safe preview path cleanly, mark it as: Needs confirmation

Deliverables:
1. `dry_run` support in request model
2. `dry_run` support in schema
3. Preview behavior in service layer
4. Updated docs
