# [High] Remove stdio transport from MCP runtime implementation

## Goal

Remove stdio transport support from the MCP runtime and make MCP transport HTTP-only.

## Background

The current implementation still contains stdio support in multiple runtime layers, including `TransportType.STDIO`, stdio startup modes, stdio healthcheck modes, `StdioServerLifecycleManager`, `StdioLifecycleProtocol`, and stdio watchdog/tool-list paths.

## Scope

Runtime implementation only.

Do not handle the following in this issue:

- `agent/config.py` re-export stub cleanup
- `McpServerConfig._cast_enums()` compatibility shim removal
- MCP documentation cleanup
- `_MCP_TOOLS` to `TOOL_LIST` migration
- Audit format migration

## Implementation Tasks

- Remove `TransportType.STDIO` from `shared/mcp_config.py`.
- Remove stdio-only startup and healthcheck modes:
  - `StartupMode.ONDEMAND`
  - `HealthcheckMode.PROCESS`
  - `HealthcheckMode.PING_TOOL`
- Remove stdio-only config semantics:
  - `cmd` for stdio
  - `working_dir`
  - `env`
  - `idle_timeout_sec`
- Remove `StdioTransport` from `shared/tool_executor.py`.
- Remove `StdioServerLifecycleManager`.
- Delete or fully detach `agent/stdio_lifecycle.py`.
- Remove `StdioLifecycleProtocol` and `restart_stdio()` from `agent/lifecycle_protocol.py`.
- Remove stdio branches from `agent/factory.py`.
- Remove stdio watchdog and tool-list probing from `agent/repl_health.py`.
- Remove `__list_tools__` support.
- Remove stdio JSON-RPC handling from `mcp/server.py`.
- Remove `run_stdio()` and `--stdio` entrypoint handling.

## Acceptance Criteria

- MCP runtime supports HTTP transport only.
- No active runtime implementation references:
  - `StdioTransport`
  - `TransportType.STDIO`
  - `HealthcheckMode.PING_TOOL`
  - `__list_tools__`
  - `run_stdio`
  - `--stdio`
- `agent/stdio_lifecycle.py` is removed or no longer imported.
- Lifecycle management has no stdio-specific methods.
- Watchdog checks only HTTP `/health`.
- `/v1/tools` is the only live tool discovery path.

## Validation

- Run MCP-related unit tests.
- Run route resolver tests.
- Run tool executor tests.
- Start the agent with HTTP MCP servers.
- Verify MCP tool calls route through HTTP only.
- Verify no stdio config is accepted.
