# Canonicalize the MDQ vs RAG boundary into a single source of truth

## Summary
Make the MDQ vs RAG boundary definition canonical in one document and convert duplicate sections in other MCP documents into references.

## Background
The MDQ vs RAG boundary is currently described both in `04_mcp_07_mdq_rag_boundary.md` and inside `04_mcp_05_security_and_safety_model.md`. This duplication increases the risk of documentation drift and inconsistent operational guidance.

## Objective
Define exactly one canonical document for:
- system ownership
- routing policy
- fallback behavior
- allowed and forbidden DB access paths
- migration criteria

## In Scope
- `04_mcp_07_mdq_rag_boundary.md`
- `04_mcp_05_security_and_safety_model.md`
- `04_mcp_00_document-guide.md`

## Required Changes
- Select one canonical document for MDQ/RAG boundary rules
- Remove duplicated normative content from the non-canonical document
- Replace duplicated sections with concise references to the canonical source
- Ensure the document guide points readers to the canonical location

## Acceptance Criteria
- Only one MCP document contains the normative MDQ/RAG boundary rules
- Other MCP documents reference the canonical boundary document instead of duplicating it
- No contradiction remains between security guidance and boundary guidance

## Out of Scope
- Changes to actual MDQ or RAG implementation
- Changes to agent routing code

## Notes
If any boundary rule is still not fully confirmed, explicitly mark it as `Unknown` or `Needs confirmation` instead of duplicating ambiguous text.
