## [Low][ops][eventbus] Add operational guidance for retry policy, DLQ promotion policy, and consumer timeout handling

### Summary
The implementation change affects runtime operations, not just code and schema.

### Problem
Operators need explicit guidance for how retries, DLQ promotion, and consumer timeout are supposed to work after the redesign.

### Scope
- 06_eventbus_05_configuration_deploy_and_operations.md
- operational runbooks
- runtime troubleshooting guidance

### Task
- document retry policy and retry limits
- document DLQ promotion policy and requeue policy
- document consumer timeout handling and expected operator actions
- document how to inspect retry backlog and DLQ lag in operations

### Expected Result
Operations can understand and support the new delivery semantics without reading source code.

### Acceptance Criteria
- ops docs explain retry policy
- ops docs explain DLQ promotion and requeue policy
- ops docs explain timeout handling
- troubleshooting guidance references the new delivery metrics
