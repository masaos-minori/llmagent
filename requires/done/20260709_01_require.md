# [High] Disallow GitHub MCP allowed_repos_mode="fail_open" in production

## Labels
priority/high, area/security, area/github-mcp, backward-compat-removal

## Summary
Disallow `allowed_repos_mode="fail_open"` when `security_profile="production"`.

`fail_open` must remain available only as a local/development backward compatibility mode, but production startup must fail fast if it is configured.

## Background
GitHub MCP supports two modes for empty `allowed_repos`.

- `fail_closed`: empty list denies all GitHub write operations.
- `fail_open`: empty list allows all repositories.

The current default is `fail_closed`, but `fail_open` remains available for backward compatibility. This is unsafe for production deployments.

## Required Changes

### Implementation
- Add `allowed_repos_mode: str` to `GitHubAuditConfig`.
- Update `load_github_audit_config()` to return `cfg.allowed_repos_mode`.
- In startup security audit, fail with `RuntimeError` when:
  - `security_profile == "production"`
  - GitHub MCP `allowed_repos_mode == "fail_open"`
- In local/development mode, keep `fail_open` allowed but emit a warning.
- Keep existing `fail_open` runtime behavior for backward compatibility.

### Documentation
- Update `config/github_mcp_server.toml` comment.

Use this wording:

- `"fail_open" = allow all when allowed_repos is empty; retained for backward compatibility.`
- `"fail_closed" = deny all when allowed_repos is empty; default and recommended.`

- Update `04_mcp_05_security_and_safety_model.md` to state that `fail_open` is prohibited in production.
- Update `04_mcp_06_configuration_and_operations.md` Pre-Production checklist to require `allowed_repos_mode="fail_closed"`.

### Tests
- Keep existing fail-open behavior tests as backward compatibility tests.
- Add a production startup/audit test for:
  - production + fail_open => RuntimeError
  - local + fail_open => warning only
  - production + fail_closed => no fail-open error

## Acceptance Criteria
- Production startup fails when GitHub MCP is configured with `allowed_repos_mode="fail_open"`.
- Local/development startup does not fail but emits a warning.
- GitHub MCP default remains `fail_closed`.
- The misleading phrase `backward compat default` is removed from config comments.
- Tests cover production and local behavior.

## Target Files
- `agent/security_audit_config.py`
- `agent/repl_health.py`
- `mcp/github/models_config.py`
- `mcp/github/service_security.py`
- `config/github_mcp_server.toml`
- `04_mcp_05_security_and_safety_model.md`
- `04_mcp_06_configuration_and_operations.md`
- `tests/test_github_mcp_service.py`

## Non-Goals
- Do not remove `allowed_repos_mode="fail_open"` entirely.
- Do not change the default from `fail_closed`.
- Do not change read-only GitHub operations.
