# [Low] Document production Markdown compatibility scope

## Priority
Low

## Labels
`mcp`, `mdq`, `documentation`, `markdown`, `specification`

## Dependencies
- Replace line-based Markdown parser with hierarchy-aware parser

## Background
Production MDQ must clearly define which Markdown features are supported. The current behavior is implicit and parser-dependent.

## Task
Document the supported Markdown compatibility scope.

## Implementation Requirements
Document support status for:
- CommonMark headings
- ATX headings
- Setext headings
- fenced code blocks
- frontmatter
- GitHub Flavored Markdown tables
- inline tags
- HTML blocks
- MDX
- duplicate headings
- content before first heading

## Acceptance Criteria
- Documentation clearly states supported and unsupported Markdown features.
- Parser tests align with the documented scope.
- Unsupported syntax has predictable fallback behavior.
