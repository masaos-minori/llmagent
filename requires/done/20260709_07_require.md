# [Low] Review and deprecate healthcheck_mode="" auto-inference

## Labels
priority/low, area/mcp, area/config, cleanup

## Summary
Review the current `healthcheck_mode=""` auto-inference behavior and prepare a deprecation path.

## Background
`healthcheck_mode` currently defaults to an empty string in documentation and is auto-inferred as `"http"`.

Since the only current transport type is HTTP, the independent `healthcheck_mode` setting has limited value and mostly preserves compatibility with older configs.

## Required Changes

### Review
- Confirm whether any non-HTTP healthcheck modes are planned.
- Confirm all current MCP servers use HTTP health checks.

### Implementation Options
Choose one:

Option A:

- Keep `healthcheck_mode` for now.
- Emit a warning when `healthcheck_mode=""` is provided explicitly.
- Document that the field is deprecated.

Option B:

- Derive healthcheck mode from transport.
- Remove the empty-string auto-inference branch.
- Keep explicit `"http"` only.

Option C:

- Remove `healthcheck_mode` from config entirely in a future breaking-change task.

### Documentation
- Update `04_mcp_06_configuration_and_operations.md`:
  - state that HTTP is the only current healthcheck mode
  - mark empty-string auto-inference as compatibility behavior
  - describe the selected deprecation path

### Tests
Add or update tests for:

- omitted `healthcheck_mode`
- explicit empty `healthcheck_mode`
- explicit `"http"`
- invalid healthcheck mode

## Acceptance Criteria
- The intended future of `healthcheck_mode` is documented.
- Empty-string auto-inference is explicitly described as compatibility behavior or deprecated.
- Tests cover current and selected future behavior.

## Target Files
- `shared/mcp_config.py`
- `04_mcp_06_configuration_and_operations.md`
- `config/agent.toml`
- `tests/test_mcp_config*.py`

## Non-Goals
- Do not change `startup_mode`.
- Do not change health endpoint semantics.
- Do not introduce non-HTTP transports.
