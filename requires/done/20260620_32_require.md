# Title
Prevent stale ETag / Last-Modified overwrite during ingester skip paths

## Summary
Protect document metadata from being overwritten by stale chunk-file metadata when ingestion skips an already-registered URL.

## Background
The design intentionally updates `documents.etag` and `documents.last_modified` during skip paths so the crawler can benefit from Conditional GET on later crawl cycles.
However, the docs explicitly raise the risk that stale values from older chunk files could overwrite newer metadata.

## Problem
If stale metadata is written back during skip behavior, Conditional GET optimization may become less reliable or incorrect over time.

## Required Changes
- Add freshness validation before updating `documents.etag` and `documents.last_modified` on skip paths.
- Ensure newer metadata is not overwritten by older metadata from stale chunk files.
- Define a deterministic precedence rule (for example, based on source recency or fetch timestamp).
- Update documentation to explain the guarded skip-path behavior clearly.

## Acceptance Criteria
- Skip-path metadata updates cannot blindly overwrite newer document metadata.
- Conditional GET support remains intact.
- The protection rule is documented and testable.

## Out of Scope
- Removing ETag/Last-Modified propagation entirely
- Replacing Conditional GET design

## AI Implementation Instruction
Implement this as a metadata-correctness and ingestion-safety task.

Strict rules:
- Do not remove skip-path metadata updates unless the design is explicitly changed elsewhere.
- Preserve the intended Conditional GET optimization.
- Add a guard so stale metadata cannot overwrite newer values.
- Make the precedence rule explicit and documented.
- If the correct freshness comparison signal is unclear, mark it as: Needs confirmation

Deliverables:
1. Guarded ETag/Last-Modified update logic
2. Tests or validation for stale-overwrite prevention
3. Updated documentation of skip-path semantics
