# [RAG][Ingestion] Fix _read_chunk_json() so chunk metadata is not lost

## Summary
`_read_chunk_json()` drops important fields from chunk JSON files.

## Problem
The current code reads chunk JSON through a dataclass and then converts it back to a dict.
Fields that are not declared in the dataclass are lost.

This causes these problems:
- `chunking_strategy` is lost
- `normalized_content` is lost
- `chunk_index` is lost

## Scope
- `scripts/rag/ingestion/ingester.py`

## Task
- replace dataclass-based chunk JSON reading with raw JSON dict loading
- keep all fields from the chunk JSON file
- make sure the ingester uses the real values from the file

## Expected Result
- `chunking_strategy` is preserved
- `normalized_content` is preserved
- `chunk_index` is preserved

## Acceptance Criteria
- [ ] `_read_chunk_json()` reads raw JSON without dropping fields
- [ ] `chunking_strategy` is written correctly
- [ ] `normalized_content` is written correctly
- [ ] `chunk_index` is written correctly
