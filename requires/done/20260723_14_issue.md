# Agent Startup Processing — RAG Consistency Check Swallowed by Exception Handler

## Summary

When `_validate_rag_consistency()` encounters an exception during dimension validation, the error is logged but the entire startup sequence continues without failing. This means RAG consistency issues go undetected silently, potentially causing incorrect similarity search results across all sessions.

## Severity

High

## Files

- `scripts/agent/startup.py:276-287`

## Details

In `StartupOrchestrator._validate_rag_consistency()`:

```python
async def _validate_rag_consistency(self) -> None:
    ctx = self._ctx
    if ctx.services_required.rag_maintenance is None:
        return
    db_path = ctx.services_required.rag_maintenance.db_path
    if db_path is None:
        return
    try:
        actual_dim = ctx.services_required.rag_maintenance.get_embedding_dimension(db_path)
        model_dim = ctx.services_required.rag_maintenance.model_embedding_dim
        if actual_dim != model_dim:
            msg = f"RAG DB vector dim ({actual_dim}) != model dim ({model_dim})"
            logger.error(msg)
            self._view.write_fatal(f"{OutputTag.FATAL} {msg}")
    except Exception as e:
        logger.error("Failed to validate RAG consistency: %s", e)
```

The broad `except Exception` clause swallows ALL exceptions, including:
1. `sqlite3.DatabaseError` — corrupted RAG database
2. `ConnectionRefusedError` — embedding service unreachable
3. `ValueError` — invalid dimension value returned from the database
4. `RuntimeError` — schema missing in RAG database

When any of these occur, the operator sees only "Failed to validate RAG consistency: <error>" in logs, with no indication of what went wrong or how to fix it. The startup proceeds normally, and subsequent RAG queries return meaningless results because the embeddings are incompatible.

This is particularly dangerous because:
1. The error is logged at ERROR level but doesn't halt startup
2. No warning is shown to the operator about degraded functionality
3. The root cause is opaque without reading source code
4. All RAG-dependent features silently degrade

## Reproduction Steps

1. Corrupt the RAG database (e.g., delete the vectors table)
2. Start the agent REPL
3. Observe that startup completes successfully despite RAG being broken
4. Perform a query that uses RAG — observe incorrect/noisy results

## Expected Behavior

RAG consistency failures should halt startup or at minimum emit a clear warning that RAG is unavailable, so operators know to take corrective action.

## Proposed Fix

Narrow the exception handling and distinguish between recoverable and non-recoverable errors:

```python
async def _validate_rag_consistency(self) -> None:
    ctx = self._ctx
    if ctx.services_required.rag_maintenance is None:
        return
    db_path = ctx.services_required.rag_maintenance.db_path
    if db_path is None:
        return
    try:
        actual_dim = ctx.services_required.rag_maintenance.get_embedding_dimension(db_path)
        model_dim = ctx.services_required.rag_maintenance.model_embedding_dim
        if actual_dim != model_dim:
            msg = f"RAG DB vector dim ({actual_dim}) != model dim ({model_dim})"
            logger.error(msg)
            self._view.write_fatal(f"{OutputTag.FATAL} {msg}")
            raise RuntimeError(msg)  # Halt startup on dimension mismatch
    except (sqlite3.DatabaseError, ConnectionRefusedError) as e:
        logger.critical(
            "RAG database inaccessible: %s — RAG features disabled until fixed",
            e,
        )
        self._view.write_warning(
            f"{OutputTag.NON_FATAL} RAG database inaccessible: {e}. "
            f"RAG features disabled. Run: bash deploy/init_db.sh to initialize."
        )
        # Disable RAG for this session
        ctx.services_required.rag_maintenance = None
    except ValueError as e:
        logger.error("Invalid RAG dimension value: %s", e)
        self._view.write_warning(f"{OutputTag.NON_FATAL} Invalid RAG dimension: {e}")
    except Exception as e:
        logger.warning("Non-critical RAG validation issue: %s", e)
        # Don't halt startup for minor issues
```
