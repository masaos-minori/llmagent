# GitHub MCPサーバーの起動失敗 — 原因と修正方針

## 概要

GitHub MCPサーバーがエージェント起動時に常に失敗する。stderrログから、ループインポートによる `ImportError` が原因であることが特定された。

## エラー内容

```
Traceback (most recent call last):
  File "/opt/llm/scripts/mcp_servers/github/server.py", line 50, in <module>
    from mcp_servers.github.server_file import router as file_router
  File "/opt/llm/scripts/mcp_servers/github/server_file.py", line 22, in <module>
    from mcp_servers.github.server_common import _get_service, _info
  File "/opt/llm/scripts/mcp_servers/github/server_common.py", line 12, in <module>
    from mcp_servers.github.github_server import _service, logger  # noqa: PLC0415
    ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "/opt/llm/scripts/mcp_servers/github/server.py", line 50, in <module>
    from mcp_servers.github.server_file import router as file_router
ImportError: cannot import name 'router' from 'mcp_servers.github.server_file'
```

## 原因

3つのファイルが循環参照している：

```
server.py ──→ server_file.py ──→ server_common.py ──→ server.py (ループ)
```

1. `server.py` はモジュールレベルで `server_file.router` をインポート
2. `server_file.py` はモジュールレベルで `server_common._get_service` をインポート
3. `server_common.py` はモジュールレベルで `server._service, logger` をインポート
4. `server.py` はまだ初期化中で `router` が存在しない → ImportError

## 影響範囲

- GitHub MCPサーバーの起動失敗
- GitHub関連ツール（create_branch, create_or_update_file, push_files, delete_file, create_issue, add_issue_comment, create_pull_request, update_pull_request, merge_pull_request など）のすべてが使えない
- CICD MCPサーバーも同様の構造で `GITHUB_TOKEN` エラーが発生（ただしこれは別の原因）

## 修正方針

### アプローチ: 遅延インポート（import-on-demand）

モジュールレベルでの循環インポートを解消するため、必要な箇所で遅延インポートを行う。

#### 変更対象ファイル

1. **`/opt/llm/scripts/mcp_servers/github/server.py`**
   - モジュールレベルの `from mcp_servers.github.server_file import router as file_router` を削除
   - FastAPI app にルーティングを追加する箇所を関数内へ移動

2. **`/opt/llm/scripts/mcp_servers/github/server_file.py`**
   - モジュールレベルの `from mcp_servers.github.server_common import _get_service, _info` を削除
   - `_get_service`, `_info` を使う各関数の先頭で遅延インポート

3. **`/opt/llm/scripts/mcp_servers/github/server_common.py`**
   - モジュールレベルの `from mcp_servers.github.github_server import _service, logger` を削除
   - `_service`, `logger` を使う各関数の先頭で遅延インポート

#### 具体的な修正例

**server.py:**
```python
# Before (line ~50)
from mcp_servers.github.server_file import router as file_router

app = FastAPI(title="GitHub MCP Server")
app.include_router(file_router)

# After
app = FastAPI(title="GitHub MCP Server")

def _include_routers():
    from mcp_servers.github.server_file import router as file_router
    app.include_router(file_router)

_include_routers()
```

**server_file.py:**
```python
# Before (line ~22)
from mcp_servers.github.server_common import _get_service, _info

async def create_file(...) -> str:
    service = _get_service()
    info = _info()
    ...

# After
async def create_file(...) -> str:
    from mcp_servers.github.server_common import _get_service, _info
    service = _get_service()
    info = _info()
    ...
```

**server_common.py:**
```python
# Before (line ~12)
from mcp_servers.github.github_server import _service, logger  # noqa: PLC0415

def _get_service() -> GitHubService:
    return _service()

# After
def _get_service() -> GitHubService:
    from mcp_servers.github.github_server import _service
    return _service()

def _info() -> dict:
    from mcp_servers.github.github_server import logger
    logger.info("...")
    ...
```

### 代替アプローチ

もし遅延インポートが複雑すぎる場合は、`_service` と `logger` を `server_common.py` で定義し、`server.py` から依存を逆転させる。ただし、この場合でも `server_file` のインポート問題は残るため、遅延インポートとの併用が必要。

## 検証方法

```bash
# GitHub MCPサーバーの起動テスト
uv run --directory /opt/llm python /opt/llm/scripts/mcp_servers/github/server.py &
sleep 2
curl http://localhost:8006/health
kill %1

# エージェント経由での起動テスト
# agent.toml の github セクションを確認後、エージェントを再起動
```

## 備考

- 最初に stderr ログに `/opt/llm/scripts/mcp/github/server.py` が表示されていたのは、空のディレクトリであり存在しないファイル。これはエージェントの構成ミスまたは古い構成が残っていた可能性。
- CICD MCPサーバーのエラーは別の原因（`GITHUB_TOKEN` 未設定）であり、本件とは無関係。

## 計画（20260712_01_plan.md）の実行結果

### Goal

GitHub MCP サーバーの起動失敗を修正する。`server.py` のモジュールレベルでの循環インポート（`server.py → server_file.py → server_common.py → server.py`）により `ImportError: cannot import name 'router' from 'mcp_servers.github.server_file'` が発生し、GitHub 関連ツールがすべて使えなくなる。

### Scope

- `/opt/llm/scripts/mcp_servers/github/server.py` — モジュールレベルのルーターインポートを遅延インポートへ変更
- `/opt/llm/scripts/mcp_servers/github/server_file.py` — モジュールレベルの `_get_service`, `_info` インポートを削除（既に遅延インポート済み）
- `/opt/llm/scripts/mcp_servers/github/server_common.py` — モジュールレベルの `_service`, `logger` インポートを削除（既に遅延インポート済み）

### Out-of-Scope

- CICD MCP サーバーのエラー（`GITHUB_TOKEN` 未設定）— 別の原因であり本件と無関係
- `/opt/llm/scripts/mcp/github/` ディレクトリ — 空のディレクトリであり古い構成の残骸。実装時に削除する可能性があるが、本計画では対象外
- 他の MCP サーバーの循環参照チェック — GitHub に限定して対応。他は別途調査・対応

### Assumptions

1. **`server_common.py` は既に遅延インポートになっている。** 確認済み：`_get_service()`, `_info()` 関数内で `from mcp_servers.github.github_server import _service` / `logger` を遅延インポートしている。修正不要。
2. **`server_file.py` も既に遅延インポートになっている。** 確認済み：各エンドポイント関数内で `_get_service`, `_info` を遅延インポートしている。修正不要。
3. **問題の原因は `server.py` のみ。** モジュールレベルで `from mcp_servers.github.server_file import router as file_router` などルーターをインポートしており、これが `server_file.py` → `server_common.py` → `server.py` のループを生成する。
4. **直接実行とサブプロセス実行で挙動が異なる可能性がある。** 直接実行は成功したが、エージェントのサブプロセス起動時は失敗していた。これは uv の環境変数やパスの扱いの違いによる可能性があり、修正後は両方で動作することを確認する必要がある。

### Design

#### 1. `server.py` の修正

**Before:**
```python
from mcp_servers.github.server_file import router as file_router
from mcp_servers.github.server_issues import router as issues_router
from mcp_servers.github.server_pull_requests import router as pr_router
from mcp_servers.github.server_repository import router as repo_router

app = FastAPI(...)

setup_exception_handlers(app)
app.include_router(repo_router)
app.include_router(file_router)
app.include_router(issues_router)
app.include_router(pr_router)
```

**After:**
```python
# ルーターインポートを削除

app = FastAPI(...)

setup_exception_handlers(app)

def _include_routers():
    from mcp_servers.github.server_file import router as file_router
    from mcp_servers.github.server_issues import router as issues_router
    from mcp_servers.github.server_pull_requests import router as pr_router
    from mcp_servers.github.server_repository import router as repo_router
    app.include_router(repo_router)
    app.include_router(file_router)
    app.include_router(issues_router)
    app.include_router(pr_router)

_include_routers()
```

#### 2. テスト方針

- `server.py` をインポートして `app.title` が取得できることを確認
- GitHub MCP サーバーを起動し、`/health` エンドポイントが HTTP 200 を返すことを確認
- エージェント経由で GitHub MCP サーバーが正常に起動することを確認

### Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| 遅延インポートにより初回リクエスト時のオーバーヘッドが発生する | Low | ルーターの登録は一度だけ（アプリ起動時）、Depends 経由での `_get_service` の呼び出しは毎リクエストだが、遅延インポートのオーバーヘッドは無視できる程度 |
| 他の MCP サーバーにも同様の循環参照がある | Medium | 全 MCP サーバー（git, cicd, shell, web_search, rag_pipeline）を同様に確認する必要がある |
| uv の環境変数の扱いの違いにより、直接実行とサブプロセス実行で挙動が変わる | Low | 修正後に両方のパターンでテストすることで確認 |
