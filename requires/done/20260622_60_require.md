# Title
Standardize shell MCP `/health` response and documentation

## Summary
Define one canonical `/health` response shape for `shell-mcp` and align all active documentation to that shape.

## Background
Current docs are inconsistent about whether shell `/health` exposes sandbox backend details or only the base health format.

## Problem
Operators need a reliable way to verify runtime sandbox state.
Documentation ambiguity makes verification harder and less trustworthy.

## Required Changes
- Decide whether shell `/health` will expose sandbox backend details.
- If yes, define a stable field such as `details.sandbox_backend`.
- If no, remove all docs that imply otherwise.
- Update health-check guidance and examples accordingly.

## Acceptance Criteria
- One canonical shell `/health` response is documented.
- Runtime behavior matches the documented model.
- Operators can reliably verify sandbox state if it is meant to be visible.

## Out of Scope
- Redesigning all MCP health endpoints
- Adding unrelated shell diagnostics

## AI Implementation Instruction
Implement this as a documentation-and-interface consistency task.

Strict rules:
- Do not leave health-output ambiguity in place.
- Keep operator verification simple and explicit.
- If you add a field, document it everywhere relevant.
- If actual implementation behavior is unclear, mark it as: Needs confirmation

Deliverables:
1. Standardized `/health` behavior
2. Aligned documentation
3. Reliable operator verification guidance
