# M-4: Extend McpProbeResult with lifecycle and health detail fields

## Priority

Medium

## Background

`McpProbeResult` currently contains basic status fields only. It cannot represent lifecycle process details or health endpoint flags such as restart recommendation.

## Target files

- `agent/services/models.py`
- `agent/services/mcp_status.py`
- status renderer command file

## Implementation tasks

- Extend `McpProbeResult` with optional fields:
  - `managed`
  - `pid`
  - `pgid`
  - `running`
  - `lifecycle_state`
  - `last_exit_code`
  - `last_shutdown_result`
  - `restart_recommended`
  - `operator_action_required`
  - `health_reason`
  - `stderr_log`
- Keep defaults backward compatible.
- Populate new fields in `McpStatusService`.
- Update renderer to display the most important new fields.

## Acceptance criteria

- Existing construction of `McpProbeResult` remains backward compatible.
- New lifecycle fields are populated when lifecycle snapshot is available.
- Health endpoint flags are visible in status output.
- Status output remains readable.

## Test plan

- Unit-test DTO construction with old and new fields.
- Unit-test `McpStatusService` populates new fields.
- Snapshot-test `/mcp status` rendering if renderer tests exist.
