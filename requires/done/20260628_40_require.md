# Preserve `/audit` regression tests

## Priority
Medium

## Labels
tests, audit, observability

## Dependencies
- Keep `/audit` as the only audit log browsing command

## Background
Removing `/debug audit` must not break the dedicated `/audit` command. Existing `/audit` tests cover default tail, turn filtering, tool filtering, and unknown subcommands.

## Task
Keep and, if necessary, strengthen `/audit` regression tests.

## Implementation Requirements
- Keep existing tests for:
  - default `/audit` tail behavior
  - `/audit turn <task_id>`
  - `/audit tool <name>`
  - unknown `/audit` subcommand usage
- Add additional tests only if needed to confirm `/audit` is the audit browsing path after `/debug audit` removal.

## Acceptance Criteria
- `/audit` tests continue to pass.
- `/audit` remains the supported audit log browsing interface.
- No `/audit` functionality is removed while deleting `/debug audit`.

## Source Context
`test_cmd_audit.py` already tests `_AuditMixin._cmd_audit()` for tail, turn, tool, and unknown subcommand behavior.
