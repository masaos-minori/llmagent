# Title
Resolve SemanticCache eviction-policy inconsistency and align implementation

## Summary
Determine the real SemanticCache eviction policy, align implementation and documentation to that policy, and expose the policy clearly in runtime/operator documentation.

## Background
The docs currently describe SemanticCache inconsistently.
One place refers to it as LRU, while another describes pruning by oldest entries (FIFO-style behavior).

## Problem
Cache policy affects hit behavior, performance expectations, and tuning decisions.
An inconsistent description creates operational and implementation ambiguity.

## Required Changes
- Verify the actual current eviction behavior in code.
- Decide whether the intended policy is FIFO or LRU.
- Align implementation and documentation to one policy.
- Expose the policy clearly in docs and, if appropriate, runtime diagnostics.

## Acceptance Criteria
- The eviction policy is unambiguous across code and docs.
- Runtime behavior matches the documented policy.
- Operators can reason correctly about cache behavior and tuning.

## Out of Scope
- Replacing the semantic cache architecture entirely
- Changing cache hit semantics unrelated to eviction behavior

## AI Implementation Instruction
Implement this as a behavior-clarification and consistency task.

Strict rules:
- Verify actual implementation before rewriting docs.
- Do not leave mixed terminology in place.
- If a policy change is required, make it explicit and update accompanying docs/tests.
- If the intended policy in the codebase is not obvious, mark it as: Needs confirmation

Deliverables:
1. Confirmed and aligned eviction policy
2. Updated implementation or docs, as required
3. Clear operator-facing explanation of cache behavior
