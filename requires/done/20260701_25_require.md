## Issue 5 (Medium): Fix system message duplication in `restore_session()`

## Title
Fix system message duplication and ephemeral message leakage in `restore_session()`

## Summary
`restore_session()` prepends current in-memory system messages to DB-fetched messages without deduplication. This causes system prompt duplication and allows `_memory_injected` messages to leak into the restored session context.

## Problem
`session_restore.py:33–34`:
```python
system_msgs = [m for m in ctx.conv.history if m["role"] == "system"]
ctx.conv.history = system_msgs + messages
```
- `messages` from DB may already contain system messages (the initial system prompt persisted at session start).
- `system_msgs` from current in-memory history may include `_memory_injected=True` markers from the previous session's turns.

## Required Changes
- `scripts/agent/services/session_restore.py` — `restore_session()`:
  1. Limit `system_msgs` to only the current canonical system prompt (`ctx.conv.system_prompt_content`), not all system messages from in-memory history.
  2. Filter out `_memory_injected=True` messages from the prepended system messages.
  3. Deduplicate against system messages already present in `messages` from DB.

## Acceptance Criteria
- After `/session load <id>`, system prompt appears exactly once in `ctx.conv.history`.
- Memory injection markers from the previous in-memory state are not carried into the restored session.
- `/session load` followed by LLM query works correctly.

## Out of Scope
- Changes to `fetch_messages()` or `SessionMessageRepository`.
- Changes to how system prompt is initially set in `_setup_prompt()`.

## AI Implementation Instruction
**What to change:**
- File: `scripts/agent/services/session_restore.py` — `restore_session()`
- Replace `system_msgs = [m for m in ctx.conv.history if m["role"] == "system"]` with a single canonical system prompt message: `system_msgs = [{"role": "system", "content": ctx.conv.system_prompt_content}] if ctx.conv.system_prompt_content else []`
- Then set: `ctx.conv.history = system_msgs + [m for m in messages if m["role"] != "system" or not system_msgs]` — or more precisely, prepend the canonical system prompt and exclude DB-sourced system messages if a canonical one was prepended.

**What NOT to change:**
- Do not change `ctx.session.session_id = session_id` assignment.
- Do not change `reset_session_stats(ctx)` call.
- Do not modify `fetch_messages()`.

**Boundary awareness:**
- `ctx.conv.system_prompt_content` is the canonical source for the system prompt (set by `_setup_prompt()` and updated by `/system` command and `_sync_system_prompt()`). This should be the single source of truth when rebuilding history on reload.
- DB-persisted system messages (from `messages` table) are redundant if the canonical content is used. Decide whether to include or exclude them — excluding is safer to avoid duplication.

**Needs confirmation:** Whether DB-persisted system messages should be completely excluded (replaced by current canonical content) or merged. The current behavior of including both leads to duplication; excluding DB system messages in favor of the current canonical prompt is the recommended approach.
