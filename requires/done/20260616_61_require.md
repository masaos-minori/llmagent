# [shared][config] Include common.toml in the main config loading flow

## Summary
`ConfigLoader.load_all()` does not load `common.toml`.

## Problem
`common.toml` contains important values such as:
- DB paths
- sqlite-vec path
- embedding settings

But `load_all()` skips it.
Some modules load `common.toml` separately as a workaround.

## Scope
- `shared/config_loader.py`
- `db/config.py`
- `db/helper.py`

## Task
- update the config loading design so `common.toml` is part of the main config flow
- remove or reduce workaround loading where possible
- make the behavior clear in docs

## Expected Result
- config loading is consistent
- DB and embedding settings come from one predictable path

## Acceptance Criteria
- [ ] `common.toml` is included in the main config flow OR the API name clearly says it is excluded
- [ ] workaround loading is removed or clearly justified
- [ ] docs explain the final behavior clearly
