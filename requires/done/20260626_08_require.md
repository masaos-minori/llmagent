## Summary
Add cross-cutting regression tests that lock down the main inconsistencies identified in the current design.

## Background
Several design risks in the current docs affect runtime correctness and operator expectations:
- compressed history may not reload consistently
- diagnostic persistence is ambiguous
- `/undo` may not align with stored artifacts
- memory retrieval boundaries may ignore `branch`
- JSONL configuration naming may be inconsistent

## Problem
Without explicit regression tests, these issues can silently regress even if docs are updated.

## Required Changes
- Add a compression → session reload consistency test.
- Add diagnostic persistence and restore-path tests.
- Add `/undo` + `/tool show` consistency tests.
- Add branch-aware memory retrieval tests.
- Add JSONL config resolution tests.

## Acceptance Criteria
- The issues identified in the design review are reproducible and detectable by tests.
- Future changes are less likely to reintroduce the same inconsistencies.
- Tests map clearly to each corrected behavior.
