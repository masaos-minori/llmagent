# H-5: Remove ToolResultStore persistence from ErrorInjectionService

## Priority

High

## Background

`ErrorInjectionService.inject_mid_turn_error()` stores mid-turn LLM transport errors in both diagnostics and `ToolResultStore` under the pseudo tool name `llm_transport_error`.

The ToolResultStore path is being removed. Mid-turn LLM errors should be persisted only through diagnostics.

## Target files

- `agent/error_injection_service.py`

## Implementation tasks

- Remove the `ctx.tool_result_store.store(...)` call from `inject_mid_turn_error()`.
- Keep the `ctx.diagnostics.save(...)` call.
- Keep the returned summary string.
- Remove `tool_name="llm_transport_error"` archive behavior.
- Update the module docstring to remove mention of the tool-result store.

## Acceptance criteria

- `ErrorInjectionService` no longer references `ctx.tool_result_store`.
- Mid-turn LLM transport errors are still stored in `session_diagnostics`.
- Conversation history remains unchanged by this service.
- Existing callers still receive the same summary string.

## Test plan

- Simulate a mid-turn `LLMTransportError`.
- Verify diagnostic persistence occurs.
- Verify no ToolResultStore write occurs.
- Verify returned summary remains stable.
