# [High] Add `--targets-file` support to RAG crawler and remove crawler `--force` documentation

## Priority
High

## Background
The RAG documentation describes local file ingestion using `crawler.py --targets-file`, but the current crawler CLI only defines `--url` and `--lang`. The documentation also shows crawler `--force` usage, while another section states crawler `--force` is not applicable. `ingester.py` already owns `--force` re-ingestion behavior.

Evidence:
- `crawler.py` currently defines only `--url` and `--lang`.
- `03_rag_05_configuration_and_operations.md` mentions `--targets-file` and crawler `--force`, but also states crawler `--force` is not applicable.
- `ingester.py` implements `--force` for delete and re-ingest.

## Goal
Add `--targets-file` to `crawler.py` and remove crawler `--force` references from active docs.

## Scope
Affected files:
- `scripts/rag/ingestion/crawler.py`
- `docs/03_rag_02_ingestion_pipeline.md`
- `docs/03_rag_05_configuration_and_operations.md`
- New or existing crawler tests

## Implementation Tasks
- Add `--targets-file PATH` to `crawler.py`.
- Use the same target format as `config/rag_pipeline.toml` `target_urls`.
- Reject simultaneous use of `--url` and `--targets-file` with a clear error.
- Support both `http(s)://` and `file://` targets.
- Keep crawler `--force` unsupported.
- Remove crawler `--force` examples from active docs.
- Keep `--force` documented only for `ingester.py`.
- Add tests for:
  - `--targets-file` parsing
  - conflict between `--url` and `--targets-file`
  - `file://` target handling
  - crawler `--force` rejected or absent from help

## Open Questions / Unknowns
- Exact `--targets-file` format must be confirmed:
  - TOML with `target_urls`
  - JSON list
  - same structure as `rag_pipeline.toml`
- Should `--targets-file` paths be resolved relative to CWD or project root?
- Should invalid targets fail the whole run or be skipped with warnings?

## Acceptance Criteria
- `crawler.py --targets-file path/to/targets.toml` works.
- `crawler.py --force` is not documented as supported.
- `crawler.py --help` and docs agree.
- Local file ingestion docs are executable as written.

## Validation
- Run `uv run python scripts/rag/ingestion/crawler.py --help`.
- Run crawler with a small targets file containing one `file://` and one `https://` target.
- Run relevant crawler tests.
