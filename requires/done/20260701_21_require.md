## Issue 1: Fix unclosed StateStore in `_init_workflow_task()`

## Title
Fix unclosed StateStore instance leaking DB connection in `_init_workflow_task()`

## Summary
- An anonymous `StateStore()` instance in `Orchestrator._init_workflow_task()` is created but never closed, leaking an open `workflow.sqlite` connection.
- The sibling code path (when `existing_task_id is not None`) already correctly calls `store.close()`, but the primary path does not.
- In long-running sessions with many turns, this may exhaust SQLite file descriptors and block WAL checkpoints.

## Background
`StateStore.__init__()` opens a `workflow.sqlite` connection via `SQLiteHelper("workflow").open(write_mode=True)`. The class has an explicit `close()` method that must be called. `_handle_workflow_engine()` properly closes its `store` instance in a `finally` block. The `_init_workflow_task()` path for new tasks (when `existing_task_id is None`) creates a `StateStore()` without binding it to a variable, making `close()` impossible.

## Problem
In `orchestrator.py:275`:
```python
task = StateStore().create_task(
    session_id=session_id,
    ...
)
```
The `StateStore()` instance is created anonymously. `close()` is never called. The connection is released only when GC runs, which is non-deterministic and may be delayed in long-running asyncio event loops.

## Required Changes
- `scripts/agent/orchestrator.py` — `Orchestrator._init_workflow_task()`: bind `StateStore()` to a local variable and wrap with `try/finally: store.close()`.
- Do NOT change the `existing_task_id is not None` branch — it already handles `close()` correctly.
- Do NOT change the `StateStore()` instance created in `_handle_workflow_engine()` — it is handled separately.
- Consider whether to add `__enter__`/`__exit__` to `StateStore` for context manager usage (optional — confirm with maintainer before adding).

## Acceptance Criteria
- All code paths in `_init_workflow_task()` call `store.close()` before returning, even if an exception is raised.
- No new `StateStore()` instances are created without a matching `close()` call.
- Existing tests pass. A test verifying `StateStore.close()` is called on exception should be added.

## Out of Scope
- Changes to `StateStore` API beyond adding `__enter__`/`__exit__`.
- Changes to other `StateStore` usage sites outside `_init_workflow_task()`.
