# Title
Improve tool-call scheduling to avoid unnecessary round-wide serialization

## Summary
Refine scheduling so that read-only tool calls are not unnecessarily serialized just because a side-effect tool appears in the same round.

## Background
Current behavior serializes the entire tool-call round when any side-effect tool is present.

## Problem
This is safe but overly coarse and increases latency even when some calls are logically independent.

## Required Changes
- Separate independent read-only tool calls from side-effect tool calls where safe.
- Preserve ordering guarantees for side-effect tools.
- Make execution order explainable via debug or trace output.

## Acceptance Criteria
- Independent read-only tools are not always forced into sequential execution.
- Safety guarantees for side-effect tools are preserved.
- Execution order can be explained for debugging and review.

## Out of Scope
- Full workflow orchestration redesign
- Cross-turn planning logic

## AI Implementation Instruction
Implement this as a scheduling-granularity improvement.

Strict rules:
- Do not weaken side-effect safety.
- Do not introduce non-deterministic behavior without visibility.
- Prefer explicit dependency-aware grouping over broad serialization.
- If dependency rules are unclear, mark them as: Needs confirmation

Deliverables:
1. Improved scheduling strategy
2. Debug/trace visibility for execution order
3. Preserved safety guarantees
