# [MCP][Routing][SQLite] Add query_sqlite to the static routing table

## Summary
`query_sqlite` is a documented SQLite tool, but it is not present in the static routing table.

## Problem
Routing works only if `tool_names = ["query_sqlite"]` is configured manually.
This is fragile and easy to break.

## Scope
- `shared/tool_constants.py`
- `shared/route_resolver.py`

## Task
- add a sqlite tool set containing `query_sqlite`
- update fallback routing to map `query_sqlite` to the sqlite server key
- keep config-based routing as an optional override, not a requirement

## Expected Behavior
- `query_sqlite` resolves correctly without relying on manual `tool_names`
- sqlite routing is consistent with other built-in MCP tools

## Acceptance Criteria
- [ ] `query_sqlite` is included in static routing
- [ ] fallback routing resolves `query_sqlite` correctly
- [ ] startup does not fail because sqlite tool names were omitted from config
