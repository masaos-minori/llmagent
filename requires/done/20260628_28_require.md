# Add registry-level tests for removed `/ingest` command

## Priority
Medium

## Labels
tests, agent, commands, ingest, breaking-change

## Dependencies
- Remove `/ingest` command registration from CommandRegistry

## Background
After `/ingest` removal, the command should no longer be recognized as a built-in slash command.

## Task
Add registry-level tests ensuring `/ingest` is no longer dispatched.

## Implementation Requirements
Add tests for:
- `/ingest https://example.com`
- `/ingest /tmp/file.md`
- `/ingest https://example.com --snippets-only`

Expected behavior should match the project’s unknown-command behavior.

Also verify the following commands remain registered:
- `/rag search query`
- `/export`
- `/compact`

## Acceptance Criteria
- `/ingest ...` is treated as unknown or unhandled.
- `/rag`, `/export`, and `/compact` remain available.
- Tests do not depend on `_cmd_ingest()`.

## Source Context
`/ingest` is currently registered in `registry.py`; `/rag`, `/export`, and `/compact` must remain.
