# Clarify and enforce the boundary between security audit and config isolation

## Priority

Medium

## Type

Architecture / Implementation / Static Analysis

## Background

MCP servers are intended to load only their own `*_mcp_server.toml` files.

The agent process may inspect server configs for startup security audit purposes, but this must remain a narrow and explicit exception.

## Problem

If the agent freely imports and uses server config models outside security audit, process isolation becomes unclear.

This can create hidden runtime dependencies between the agent and MCP server application configuration.

## Goal

Keep MCP server config isolation intact while allowing explicit security audit access.

## Scope

Affected areas:

- `agent/repl_health.py`
- MCP server config model imports
- Config loader isolation
- Static analysis checks
- Startup validation tests

## Implementation Tasks

- Create a narrow security-audit-only API for reading MCP server security config.
- Separate audit config loading from server runtime config loading.
- Add a static check that prevents non-audit agent code from importing server config models.
- Verify MCP server processes cannot read config files other than their own.
- Treat config isolation violations as startup errors.
- Ensure security audit exceptions are documented in code comments and tests.

## Acceptance Criteria

- Agent runtime logic does not depend on MCP server app config except through the audit API.
- Security audit can still inspect necessary server security settings.
- MCP server process config isolation remains enforced.
- Static check fails on unauthorized imports.
- Tests cover allowed and forbidden access paths.

## Validation Plan

Add tests for:

- Allowed audit access to shell config.
- Allowed audit access to git config.
- Allowed audit access to GitHub config.
- Allowed audit access to CI/CD config.
- Forbidden non-audit import from agent code.
- Server process attempting to read another server's config.
- Server process attempting to read `agent.toml`.

## Risks

Some existing agent code may import server config models directly. Move such usage behind the audit API.

## Out of Scope

- Removing the security audit.
- Merging agent and server configs.
- Documentation-only updates.
