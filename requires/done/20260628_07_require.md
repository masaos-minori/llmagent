# Update MCP command tests after `/mcp install` removal

## Priority
Medium

## Labels
tests, agent, commands, mcp

## Dependencies
- Remove `/mcp install` dispatch from MCP command handler

## Background
`test_cmd_mcp.py` currently contains tests for `_cmd_mcp_install()` and the installer service integration. These tests must be removed or replaced with rejection tests.

## Task
Update MCP command tests to reflect that `/mcp install` is no longer supported.

## Implementation Requirements
- Remove the `TestCmdMcpInstall` test class.
- Remove tests that directly call `_cmd_mcp_install()`.
- Remove patches for:
  - `mcp.installer_validation.validate_server_name`
  - `agent.commands.cmd_mcp.McpInstallService`
  - `agent.commands.cmd_mcp.CliInstallQA`
- Add a test verifying that `/mcp install test` is rejected.
- Add/keep tests verifying:
  - `/mcp` still works
  - `/mcp status` still works
  - unknown subcommands are rejected

## Acceptance Criteria
- No test calls `_cmd_mcp_install()`.
- `/mcp install test` is expected to fail.
- Existing MCP status tests continue to pass.
- Test suite has no dependency on MCP installer services.

## Source Context
`test_cmd_mcp.py` currently includes `TestCmdMcpInstall` with tests for invalid install names and validated server-name execution.
