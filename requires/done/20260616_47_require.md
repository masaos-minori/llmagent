# [RAG][Cache] Make SemanticCache safe for concurrent use

## Summary
The current SemanticCache behavior under concurrent access is not defined.

## Problem
If multiple async tasks share one pipeline instance, cache writes and pruning may race.

## Scope
- `rag/cache.py`

## Task
- add concurrency protection for cache operations
- define safe behavior for concurrent `put()`, `lookup()`, and `prune()`
- add tests for concurrent access

## Expected Result
The cache works correctly when shared across concurrent requests.

## Acceptance Criteria
- [ ] cache operations are protected from races
- [ ] concurrent tests exist
- [ ] no inconsistent state appears under concurrent use
