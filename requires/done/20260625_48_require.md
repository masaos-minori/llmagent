## [High][eventbus][db] Add indexes that support DLQ promotion and DLQ listing queries

### Summary
The current schema provides indexes for `topic` and `seq`, but not for DLQ-related filtering.

### Problem
DLQ promotion and DLQ listing queries can degrade into full scans as the event table grows.

### Scope
- schema.sql
- DLQ queries in dlq.py and app.py
- DB schema documentation

### Task
- analyze the current DLQ-related query patterns
- add indexes that support:
  - retry-threshold filtering
  - `dlq_at IS NULL` / `dlq_at IS NOT NULL` filtering
  - DLQ listing by sequence
- validate the execution plan improvement
- update schema docs and migration logic if needed

### Expected Result
DLQ promotion and DLQ listing remain efficient as event volume grows.

### Acceptance Criteria
- DLQ-related queries are backed by appropriate indexes
- schema docs reflect the new indexes
- migration/init path creates the new indexes
- tests or query-plan checks verify the improvement
