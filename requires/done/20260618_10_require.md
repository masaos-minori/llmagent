# [mcp][execution] Record and analyze side-effect round serialization

## Summary
If one side-effect tool appears in a round, the whole round becomes serialized.

## Problem
This is safe, but it may reduce useful parallelism more than necessary.

## Scope
- tool execution flow
- execution observability
- side-effect scheduling behavior

## Task
- record when round-wide serialization happens
- record why it happened
- add observability before attempting optimization

## Expected Result
The team can understand the real cost of side-effect serialization before changing behavior.

## Acceptance Criteria
- [ ] serialization events are recorded
- [ ] serialization reasons are visible
- [ ] real execution impact can be analyzed
