# [Medium] Document the AugmentStage content-only LLM context invariant

## Goal

Document that AugmentStage must output `content`, never `normalized_content`, into LLM context.

## Background

The query pipeline docs state that AugmentStage formats chunks using `sanitize_document(c.content)`. DESIGN-2 requires `normalized_content` to remain search-only and never be exposed as LLM context. 【3-6a9d52】【1-54c80f】

## Scope

Documentation first.

Implementation changes are required only if AugmentStage currently violates this invariant.

## Implementation Tasks

- Update `03_rag_03_query_pipeline.md`.
- Update `03_rag_90_inconsistencies_and_known_issues.md`.
- State that AugmentStage formats `content` only.
- State that `normalized_content` is search-only.
- State that RAG context blocks must contain original readable chunk text.
- Add a note that replacing `content` with `normalized_content` would degrade LLM context quality.
- If implementation is checked, verify that AugmentStage does not read `normalized_content`.

## Acceptance Criteria

- `03_rag_03` explicitly documents the content-only invariant.
- DESIGN-2 references this invariant.
- Docs do not imply that normalized text can be sent to the LLM.

## Validation

- Review AugmentStage documentation.
- If source is available, verify AugmentStage uses `content`.
- Run RAG query pipeline tests if available.
