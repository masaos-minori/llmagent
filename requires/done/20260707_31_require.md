# M-2: Replace string comparisons with enum comparisons for MCP config

## Priority

Medium

## Background

`McpServerConfig` normalizes transport and startup mode to enum values. Several call sites still compare against string literals such as `"http"` and `"subprocess"`.

## Target files

- `agent/factory.py`
- `agent/startup.py`
- `agent/repl_health.py`
- `agent/services/mcp_status.py`

## Implementation tasks

- Import `TransportType` and `StartupMode` where needed.
- Replace `cfg.transport == "http"` with `cfg.transport == TransportType.HTTP`.
- Replace `cfg.startup_mode == "subprocess"` with `cfg.startup_mode == StartupMode.SUBPROCESS`.
- Replace negative string comparisons accordingly.
- Keep behavior unchanged.

## Acceptance criteria

- No lifecycle, startup, watchdog, or status code uses raw `"http"` or `"subprocess"` comparisons.
- Existing tests continue to pass.
- Behavior remains unchanged.

## Test plan

- Run unit tests for startup, watchdog, lifecycle router, and MCP status.
- Add focused tests for subprocess and persistent server branch selection.
