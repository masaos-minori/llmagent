# [RAG][Docs] Normalize old and new RAG terminology in docs

## Summary
The docs still contain old names and counting styles that can confuse readers and AI tools.

## Problem
Examples:
- `web_crawler.py` vs `crawler.py`
- `rag_ingester.py` vs `ingester.py`
- `3 steps` vs `4 phases`

## Scope
- RAG docs
- file path references
- terminology guidelines

## Task
- use one canonical file name for each component
- use one canonical pipeline description style
- keep old names only as legacy notes when necessary

## Expected Result
Docs are easier to read and easier for AI tools to follow.

## Acceptance Criteria
- [ ] file names are consistent
- [ ] stage/phase wording is consistent
- [ ] legacy names are clearly marked as historical only
