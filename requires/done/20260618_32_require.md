# [agent][config][reload] Improve /reload feedback for non-reloadable settings

## Summary
Some settings require restart, but /reload feedback may still be too easy to misunderstand.

## Problem
Users may think a setting changed successfully even when restart is still required.

## Scope
- `/reload` output
- config reload UX
- restart-required reporting

## Task
- improve `/reload` output
- clearly separate:
  - applied now
  - restart required
- make important non-reloadable settings harder to miss

## Expected Result
Users can understand which config changes took effect and which did not.

## Acceptance Criteria
- [ ] `/reload` output clearly shows applied vs restart-required items
- [ ] users are less likely to misunderstand non-reloadable changes
- [ ] config reload UX is clearer
