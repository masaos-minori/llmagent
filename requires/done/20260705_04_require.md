# Enforce plugin tool contract at registration time and runtime

## Priority

High

## Type

Implementation / Test / Safety

## Background

Plugin tools are local Python handlers registered with `@register_tool`. They bypass MCP routing and are checked before MCP tool dispatch.

The expected plugin tool contract is:

- Handler must be async.
- Handler must declare return annotation `tuple[str, bool]`.
- Runtime return value must be a tuple of exactly two elements.
- First element must be `str`.
- Second element must be `bool`.

## Problem

If plugin tool contract violations are detected only at call time, invalid plugins may load successfully and fail during production tool execution.

If only annotation validation exists, a plugin may still return invalid runtime values.

## Goal

Use both registration-time validation and runtime validation to fail early and fail safely.

## Scope

Affected areas:

- `shared/plugin_registry.py`
- `shared/plugin_tool_invoker.py`
- `shared/tool_executor.py`
- Plugin loading logic
- Plugin strict mode behavior
- Plugin tests

## Implementation Tasks

- Validate plugin tool return annotation at registration time.
- Reject plugin tools without `tuple[str, bool]` return annotation.
- Reject non-async plugin tool handlers.
- Validate runtime return values when the plugin tool is invoked.
- Ensure tuple length is exactly 2.
- Ensure result element 0 is `str`.
- Ensure result element 1 is `bool`.
- In `plugin_strict=true`, surface validation errors as startup failures.
- In `plugin_strict=false`, skip invalid plugin tools and continue loading other plugins.

## Acceptance Criteria

- Invalid plugin tools are not registered.
- Invalid runtime return values produce clear tool execution errors.
- One invalid plugin does not break unrelated valid plugins in non-strict mode.
- Strict mode fails startup when invalid plugin tools are found.
- Plugin load result clearly reports failed plugin tools.

## Validation Plan

Add tests for:

- Missing return annotation.
- Wrong return annotation.
- Non-async handler.
- Runtime return value is not a tuple.
- Runtime tuple length is not 2.
- Runtime first element is not `str`.
- Runtime second element is not `bool`.
- Strict mode behavior.
- Non-strict mode behavior.

## Risks

Some existing plugins may fail to load after validation is tightened. This is intentional and should be treated as contract enforcement.

## Out of Scope

- Changing MCP tool return contracts.
- Adding plugin sandboxing.
- Documentation-only updates.
