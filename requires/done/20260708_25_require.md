# H-3: Remove HTTP URL hot reload for MCP servers

## Priority

High

## Background

HTTP URL hot reload for MCP servers is not reliable because existing `HttpTransport` instances keep their startup `base_url` snapshot.

Changing `ctx.cfg.mcp.mcp_servers[key].url` during `/reload` does not update already created transports.

## Target files

- `agent/services/config_reload.py`
- `shared/http_transport.py` tests if applicable
- reload command tests

## Implementation tasks

- Remove runtime URL assignment from MCP reload handling.
- Do not execute behavior equivalent to:
  - `old_srv.url = new_srv.url`
- Detect URL differences between old and new MCP server config.
- Add `mcp/<server_key>.url` to `ConfigReloadOutcome.needs_restart`.
- Ensure URL changes do not appear in `ConfigReloadOutcome.applied`.
- Ensure URL changes do not appear in `ConfigReloadOutcome.deferred`.

## Acceptance criteria

- Changing an MCP server URL via config reload never changes running config.
- Changing an MCP server URL reports `[RESTART] - mcp/<server_key>.url`.
- Existing `HttpTransport` instances keep their startup `_base_url`.
- No `/reload` output claims MCP URL changes were applied at runtime.

## Test plan

- Start with an MCP server URL such as `http://127.0.0.1:8005`.
- Reload config with URL changed to another value.
- Verify `needs_restart` contains `mcp/<server_key>.url`.
- Verify running config URL is unchanged.
- Verify no MCP URL item appears under applied or deferred output.
