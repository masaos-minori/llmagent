# [High] Validate workflow definition during deployment

## Background

The workflow definition must be valid before deployment completes. Runtime startup validation is not sufficient because invalid artifacts should be rejected during deployment.

## Goal

Add deploy-time workflow definition validation.

## Scope

- `deploy/deploy.sh`
- `agent/workflow/validate.py`, or equivalent validation module

## Tasks

- [ ] Add a CLI command to validate workflow definitions.
- [ ] Recommended command: `python -m agent.workflow.validate config/workflows/default.json`
- [ ] Validate that the workflow JSON is parseable.
- [ ] Validate required top-level fields: `name`, `version`, `stages`, `retry_policy`.
- [ ] Validate required stages: `plan`, `execute`, `verify`.
- [ ] Validate duplicate stage IDs.
- [ ] Validate required stage fields: `id`, `description`, `timeout_sec`, `retryable`.
- [ ] Validate retry policy: `max_attempts >= 1`.
- [ ] Validate retry policy: `backoff` is either `fixed` or `exponential`.
- [ ] Validate retry policy: `backoff_sec >= 0`.
- [ ] Call the validation command from `deploy.sh`.
- [ ] Abort deployment on validation failure.

## Acceptance Criteria

- [ ] Invalid JSON causes deployment failure.
- [ ] Missing required stages cause deployment failure.
- [ ] Duplicate stage IDs cause deployment failure.
- [ ] Invalid retry policy causes deployment failure.
- [ ] Valid `default.json` passes validation.
- [ ] Validation failure exits with code `1`.

## Notes

The validation command must not start the agent or any MCP/LLM service.
