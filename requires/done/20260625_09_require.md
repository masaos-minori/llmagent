## [High][security][shell] Enforce firejail sandbox as the production-safe default path for shell-mcp

### Summary
`shell_sandbox_backend="none"` is documented as dangerous, and production startup is supposed to reject it.

### Problem
The current design still exposes `none` as the baseline default in several places.
This makes the effective production safety posture too dependent on startup-time auditing.

### Scope
- mcp/shell/*
- config/shell_mcp_server.toml
- startup security validation
- shell security documentation

### Task
- make production mode require `shell_sandbox_backend="firejail"`
- treat `sandbox_backend="none"` as a hard startup error in production
- make safe production configuration the default in production-facing templates
- remove ambiguous wording about default vs effective production value
- add regression tests for startup rejection

### Expected Result
Production deployments cannot run shell-mcp without an OS-level sandbox.

### Acceptance Criteria
- production startup fails when `sandbox_backend="none"`
- production templates default to `firejail`
- docs clearly distinguish local default vs production effective requirement
- tests verify startup rejection and safe startup success
