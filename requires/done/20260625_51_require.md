## [Medium][eventbus][config] Add explicit configuration validation for value ranges and required path semantics

### Summary
The current Event Bus config loader performs minimal structural loading but almost no semantic validation.

### Problem
Invalid values can survive startup and fail later at runtime in less obvious ways.

### Scope
- config.py
- EventBusConfig validation
- startup configuration checks
- config documentation

### Task
- add validation rules for numeric fields such as:
  - `poll_interval_ms`
  - `offset_checkpoint_interval`
  - `max_retry`
- validate required path-like fields for non-empty semantics
- fail fast on invalid configuration
- document validation rules clearly
- add tests for invalid config inputs

### Expected Result
Invalid Event Bus configuration is rejected at startup with clear errors.

### Acceptance Criteria
- invalid numeric ranges are rejected
- invalid/missing critical path values are rejected
- config validation behavior is documented
- tests cover invalid configuration cases
