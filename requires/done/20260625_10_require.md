## [High][security][startup] Convert dangerous production security warnings into hard startup failures

### Summary
Some dangerous security states are audited at startup, but warning-only handling is still too permissive for production.

### Problem
Warning-based security posture can be ignored in automated or lightly monitored environments.
Production should fail fast on known dangerous security states.

### Scope
- agent/repl_health.py
- startup security checks
- production security policy
- MCP startup validation documentation

### Task
- identify all dangerous production states that must become hard errors
- enforce hard failure for:
  - empty workflow allowlists where policy should be fail-closed
  - `shell_sandbox_backend="none"` in production
  - other documented fail-open workflow controls in production
- update log messages and operator guidance
- add tests for production startup failure cases

### Expected Result
Production startup fails instead of warning when a known dangerous security state is present.

### Acceptance Criteria
- dangerous production states are rejected at startup
- warning-only handling is removed for those cases
- startup diagnostics remain explicit and actionable
- tests cover all fail-fast paths
