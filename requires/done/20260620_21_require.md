# Title
Expose workflow execution mode in normal runtime UI

## Summary
Make it easy for operators to tell whether the current session is using workflow-engine execution or direct fallback execution.

## Background
Workflow mode is an important runtime distinction, but it is not obvious from normal session interaction.

## Problem
Operators may not realize which execution model is active, making debugging and operational interpretation harder.

## Required Changes
- Expose workflow mode in one or more normal runtime views such as:
  - startup banner
  - `/stats`
  - `/context`
- Make approval-pending workflow state easier to recognize.

## Acceptance Criteria
- Users can tell which execution mode is active without digging into deep diagnostics.
- Approval-pending state is easy to identify.

## Out of Scope
- Redesigning the workflow engine itself

## AI Implementation Instruction
Implement this as a runtime-visibility task.

Strict rules:
- Do not redesign workflow behavior.
- Focus on lightweight, high-signal status exposure.
- Avoid cluttering the UI with too much internal detail.
- If mode naming is unclear, mark it as: Needs confirmation

Deliverables:
1. Workflow mode visibility in normal runtime UI
2. Easier visibility for approval-pending state
