# [Medium] Remove GitHub `_MCP_TOOLS` backward compatibility alias

## Priority
Medium

## Background
GitHub MCP tools expose `TOOL_LIST` as the canonical export, while `_MCP_TOOLS` is kept only for backward compatibility. The GitHub MCP server already imports and uses `TOOL_LIST`. 【3-018a33】【6-31c99f】

## Goal
Remove the GitHub `_MCP_TOOLS` alias if no active code depends on it.

## Scope
Affected files:
- `mcp/github/tools.py`
- `mcp/github/server.py`
- tests
- docs

## Implementation Tasks
- Search the repository for `_MCP_TOOLS`.
- Replace GitHub-related `_MCP_TOOLS` imports with `TOOL_LIST`.
- Remove `_MCP_TOOLS` from `mcp/github/tools.py` if unused.
- Keep this issue scoped to GitHub MCP only.
- If other MCP servers still expose `_MCP_TOOLS`, create a separate follow-up issue.

## Acceptance Criteria
- GitHub MCP uses only `TOOL_LIST`.
- No GitHub code or test imports `mcp.github.tools._MCP_TOOLS`.
- GitHub tools remain exposed correctly through `/v1/tools`.
- Tests pass.

## Validation
- Run: `rg -n "_MCP_TOOLS" .`
- Run: `pytest tests/test_github_mcp_service.py -v`
