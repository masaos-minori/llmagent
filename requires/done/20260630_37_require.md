# [Low] Add documentation sync check for slash command references

## Priority
Low

## Background
The short command summary and full command reference can drift. This previously allowed stale command references such as `/mcp install` even though implementation only supports `/mcp` and `/mcp status`. 【7-156c6d】【9-3d3b2d】【6-dc29e4】

## Goal
Prevent active command documentation from diverging from the actual command registry.

## Scope
Affected files:
- `agent/commands/registry.py`
- `docs/05_agent_01_system-overview.md`
- `docs/05_agent_07_cli-and-commands.md`
- `tests/docs/test_command_docs_sync.py`

## Implementation Tasks
- Add a docs test that extracts built-in command names from `agent/commands/registry.py`.
- Compare active command docs against the registered command list.
- Allow migration/archive sections to mention removed commands.
- Fail if active command references include unregistered commands.
- Check at least:
  - `/mcp`
  - `/db`
  - `/debug`
  - `/audit`
  - `/memory`
  - `/mdq`
  - `/rag`
  - `/plugin`

## Acceptance Criteria
- Active docs match the registered command surface.
- Removed commands appear only in migration/archive notes.
- CI detects stale command references.

## Validation
- Run: `pytest tests/docs/test_command_docs_sync.py -v`
