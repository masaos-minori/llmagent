# [Medium] Replace `Any` pipeline field in `RagPipelineMCPService` with a Protocol

## Priority
Medium

## Difficulty
Low

## Background
`RagPipelineMCPService` stores `_pipeline` as `Any | None` to avoid circular imports and module-load side effects.

This avoids runtime import issues but weakens type safety for:

- `augment()`
- `last_fetch_result`
- `last_timings`

## Goal
Keep lazy import behavior while replacing `Any` with a typed Protocol.

## Scope
Affected files:

- `scripts/mcp/rag_pipeline/service.py`

## Implementation Tasks
- Define a local `RagPipelineLike` Protocol in `service.py`.
- Include required members:
  - `augment(...)`
  - `last_fetch_result`
  - `last_timings`
- Change `_pipeline: Any | None` to `_pipeline: RagPipelineLike | None`.
- Keep lazy import inside `start()` to avoid circular import side effects.

## Acceptance Criteria
- `service.py` no longer stores `_pipeline` as `Any`.
- Type checkers can validate calls to:
  - `augment()`
  - `last_fetch_result`
  - `last_timings`
- Runtime behavior remains unchanged.

## Validation
- Run type checking if available.
- Run RAG MCP service startup tests.
- Run RAG pipeline service tests.
