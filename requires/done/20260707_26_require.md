# H-6: Add startup rollback shutdown for already-started MCP subprocesses

## Priority

High

## Background

`StartupOrchestrator.run()` starts HTTP subprocess MCP servers and then runs service validation. If validation fails after some subprocesses have already started, those subprocesses may remain running because rollback shutdown is not implemented.

## Target files

- `agent/startup.py`

## Implementation tasks

- Wrap the startup sequence after `_initialize()` in `try/except`.
- Track whether `_start_servers()` has been reached.
- If later startup steps fail, call `ctx.services_required.lifecycle.shutdown_all()`.
- Do not hide the original startup exception.
- Log rollback shutdown failures as warnings.
- Keep `_initialize()` failures separate because services may not be initialized yet.

## Acceptance criteria

- If `_check_services()` raises after servers are started, `shutdown_all()` is called.
- If `_recover_pending_approvals()` or `_setup_prompt()` raises after servers are started, `shutdown_all()` is called.
- The original startup exception is re-raised.
- Rollback shutdown failure is logged without replacing the original failure.

## Test plan

- Mock `_start_servers()` success and `_check_services()` failure; assert `shutdown_all()` is called.
- Mock rollback `shutdown_all()` failure; assert original exception is preserved.
- Mock `_initialize()` failure; assert shutdown is not attempted before lifecycle exists.
