# Validate `n` in `ToolResultStore.list_recent()`

## Priority
Medium

## Labels
- db
- tool-results
- validation
- medium-priority

## Dependencies
None

## Background
`ToolResultStore.list_recent(session_id, n)` passes `n` to SQL `LIMIT`. The behavior for `n <= 0` should be explicit and safe.

## Scope
Modify:
- `db/tool_results.py`
- `tests/test_tool_result_store.py`

## Required Changes
- If `session_id is None`, continue returning `[]`.
- If `n < 1`, return `[]` without querying the database.
- Keep existing oldest-first return ordering.
- Do not change the partial-row behavior of `list_recent()`.

## Tests
Update `tests/test_tool_result_store.py`.

Add tests for:
- `list_recent(session_id, n=0)` returns `[]`.
- `list_recent(session_id, n=-1)` returns `[]`.
- `list_recent(session_id, n=1)` returns one result.
- Existing `session_id=None` behavior remains unchanged.
- Existing oldest-first ordering remains unchanged.

## Acceptance Criteria
- Invalid limits are not passed to SQL.
- `list_recent()` behavior is explicit for edge cases.
- Existing tests continue to pass.
