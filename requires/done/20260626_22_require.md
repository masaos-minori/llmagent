# [Medium] Align search_docs schema with actual implementation

## Priority
Medium

## Labels
`mcp`, `mdq`, `search`, `tool-schema`, `api-consistency`

## Dependencies
- Redesign MDQ SQLite schema for production indexing
- Introduce stable chunk IDs for MDQ chunks

## Background
`SearchDocsRequest` exposes fields such as `mode`, `tag_filter`, and `heading_prefix`, but the current implementation only partially supports filtering. Tool schemas must not advertise unsupported behavior.

## Task
Make `search_docs` schema and implementation consistent.

## Implementation Requirements
- Implement or remove `mode`.
- Implement or remove `tag_filter`.
- Implement `heading_prefix` filtering.
- Preserve `path_prefix` filtering.
- Return enough metadata to support follow-up `get_chunk` calls:
  - `chunk_id`
  - `source_path`
  - `heading_path`
  - `score`
  - `start_line`
  - `end_line`
  - `token_count`
  - snippet

## Acceptance Criteria
- Every exposed request field affects behavior or is removed.
- `search_docs` results are directly actionable.
- Search results include stable chunk IDs.
- Tests cover filters and empty results.
