## [High][eventbus][security] Change the default deployment example to loopback/private bind for the unauthenticated internal-only Event Bus

### Summary
The Event Bus is explicitly documented as an internal-only service with no authentication or ACLs.

### Problem
The deployment example currently binds the service to `0.0.0.0`, which increases the risk of accidental exposure even though the design assumes trusted internal-only access.

### Scope
- 06_eventbus_05_configuration_deploy_and_operations.md
- deployment examples
- operational guidance
- security posture documentation

### Task
- replace the default deployment example from `0.0.0.0` to loopback or a clearly private-only bind
- document that public exposure is forbidden unless an explicit security layer is added
- add a startup or deployment warning if the service is configured for wide exposure without authentication
- align deployment examples with the documented internal-only security model

### Expected Result
The default deployment guidance no longer encourages risky exposure of an unauthenticated Event Bus.

### Acceptance Criteria
- the default deployment example is loopback/private-only
- docs explicitly warn against public exposure
- deployment guidance is consistent with the internal-only threat model
- risky wide-bind usage is called out clearly
