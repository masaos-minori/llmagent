## Background

The Agent documentation states that `command_defs_list.py` is the canonical source for slash command definitions. It also notes that `AgentREPL.SLASH_COMMANDS` is used for readline tab completion and may omit valid commands.

The documentation also records differences between the documented prompt style and the current prompt implementation. In addition, it includes structural notes about CommandRegistry mixins and notes that previous mixin counts became stale.

All of these issues are centered on the CLI command surface and its documentation.

## Problem

The CLI command surface has several related drift risks:

- A command may be valid but absent from tab completion.
- Absence from completion may be mistaken for absence from the command registry.
- REPL prompt documentation may not match actual runtime behavior.
- CommandRegistry mixin counts or structures may become stale in documentation.
- New command additions may update one place but not the others.

These issues are individually small, but together they reduce confidence in the CLI reference and make AI implementation agents more likely to follow stale documentation.

## Reason for change

Slash commands are the primary local control surface for the agent. The command registry, completion list, prompt behavior, and documentation must describe the same user-facing interface.

The command system should have one canonical definition source, and any derived or intentionally incomplete views such as tab completion should be validated or explicitly annotated.

## Design intent

Establish a clear CLI command documentation model:

- `_COMMANDS` remains the canonical command source.
- Completion behavior is generated from or validated against the canonical source.
- Intentional completion exclusions are represented as metadata.
- REPL prompt behavior is either updated to match documentation or documentation is updated to match implementation.
- CommandRegistry mixin documentation avoids fragile unvalidated counts.

## Implementation instructions

### Slash command registry and completion

- Review `_COMMANDS` in `command_defs_list.py`.
- Review `AgentREPL.SLASH_COMMANDS`.
- Decide whether completion candidates should be generated from `_COMMANDS`.
- If not all commands should be completed, add explicit metadata such as `completion_excluded`.
- Add tests that detect unexpected command/completion drift.
- Document intentional exclusions and their reasons.

### REPL prompt

- Review current prompt implementation.
- Decide whether the prompt should remain a simple fixed prompt or use an `agent>` style.
- Decide whether session id should be displayed.
- Align documentation with the chosen behavior.
- Update CLI tests or snapshots if present.
- Ensure multiline prompt behavior is documented consistently.

### CommandRegistry structure docs

- Review the current CommandRegistry mixin list.
- Avoid hard-coded mixin counts unless validated.
- Consider adding a script or test that extracts mixin and command information.
- Align command category tables with actual command definitions.
- Update documentation consistency checklist for command changes.

## Acceptance criteria

- `_COMMANDS` is clearly documented as the canonical command source.
- Completion behavior is generated from or validated against `_COMMANDS`.
- Intentional completion exclusions are explicit and documented.
- REPL prompt documentation matches runtime behavior.
- CommandRegistry mixin documentation is either generated, validated, or avoids fragile exact counts.
- Tests detect command/completion drift.
- No valid command is treated as invalid merely because it is not in completion.

## Non-goals

- Do not redesign the slash command framework.
- Do not remove valid slash commands.
- Do not change command business behavior unless required for consistency.
- Do not redesign the full CLI.
