# [High] Align mdq-mcp with MCPServer base class behavior

## Priority
High

## Labels
`mcp`, `mdq`, `protocol`, `server`, `architecture`

## Dependencies
- Fix mdq-mcp syntax errors and incomplete server entry point

## Background
All MCP servers should follow the common MCP server contract, including shared endpoint behavior, request handling, tool listing, authentication, truncation, and stdio support where applicable.

## Task
Refactor `mdq-mcp` to conform to the shared `MCPServer` base class conventions.

## Implementation Requirements
- Define server metadata:
  - `server_name`
  - `server_version`
  - `http_host`
  - `http_port`
  - `app_module`
  - `mcp_tools`
- Implement `dispatch(name, args)`.
- Route all 7 MDQ tools through the dispatch method.
- Ensure `/v1/tools` includes `server_key: "mdq"`.
- Ensure common MCP behavior is preserved:
  - request ID handling
  - auth middleware compatibility
  - response truncation
  - stdio mode compatibility, if supported by the base class

## Acceptance Criteria
- `mdq-mcp` follows the same lifecycle and endpoint conventions as other MCP servers.
- `/v1/tools` returns all 7 MDQ tools.
- `/v1/call_tool` can invoke each MDQ tool.
- Tool errors are returned as MCP tool errors instead of breaking the transport contract.
