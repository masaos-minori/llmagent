## [High][approval] Consolidate tool-level approval and workflow-level approval into one canonical model

### Summary
The current agent design has two approval paths:
1. tool-level approval before tool execution
2. workflow-level approval suspension handled by `/approve` and `/reject`

This creates duplicated approval responsibility and ambiguous behavior.

### Problem
Approval logic is split across multiple layers.
This can produce inconsistent approval prompts, inconsistent state transitions, and confusing operator behavior.

### Scope
- shared/tool_executor.py
- agent/workflow/workflow_engine.py
- agent/orchestrator.py
- agent/commands/cmd_workflow.py
- approval-related documentation

### Task
- audit all approval entry points
- define one canonical approval model
- decide which layer owns approval state and approval transitions
- remove duplicated approval logic
- make `/approve` and `/reject` resolve only the canonical approval object
- update all related docs to one consistent approval flow

### Expected Result
The system has exactly one approval model and one source of truth for approval state.

### Acceptance Criteria
- tool approval and workflow approval no longer conflict
- only one approval path is used for risky write operations
- `/approve` and `/reject` resolve the canonical approval record only
- approval behavior is documented consistently
