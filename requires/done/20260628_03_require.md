# Update `/mcp` command definition and help text

## Priority
Medium

## Labels
agent, commands, help, docs, mcp

## Dependencies
- Remove `/mcp install` dispatch from MCP command handler

## Background
The command registry currently describes `/mcp` as supporting an `install` subcommand. After removing `/mcp install`, the command description shown by `/help` must be updated.

## Task
Update the `/mcp` command definition so it describes status/health inspection only.

## Implementation Requirements
- Update the `/mcp` `CommandDef` in `registry.py`.
- Remove the phrase indicating that `install` scaffolds a new server.
- Use a concise description such as:
  - `MCP server status, tool list, connectivity check`
- Ensure `/help` no longer suggests `/mcp install`.

## Acceptance Criteria
- `/help` does not mention `/mcp install`.
- `registry.py` no longer describes `/mcp install`.
- `/mcp` remains registered as a prefix async command.

## Source Context
`registry.py` currently defines `/mcp` with a description that explicitly mentions `install` scaffolding a new server.
