# Remove workflow_mode="auto" startup fallback and degradation

## Priority

High

## Type

Breaking Change / Startup Validation / Workflow

## Background

The legacy `auto` mode allows workflow startup degradation. It permits the agent to continue startup even when workflow loading fails.

The target architecture requires workflow readiness before the agent can process user turns.

## Problem

Startup degradation hides workflow configuration and schema problems.

Examples:

- Missing workflow definition
- Invalid workflow JSON
- Missing workflow database
- Missing workflow tables
- Invalid workflow schema

These must fail at startup, not later during turn processing.

## Goal

Remove startup fallback behavior and make workflow readiness a mandatory startup gate.

## Scope

Affected areas:

- Startup validation
- Workflow loader usage
- Orchestrator construction
- Workflow execution policy
- Error messages
- Tests relying on degraded startup

## Implementation Tasks

- Remove `allow_startup_fallback()`.
- Remove the `auto` mode from all validation paths.
- Fail startup on workflow loader failure.
- Fail startup on missing workflow definition.
- Fail startup on workflow schema failure.
- Remove warning-only startup degradation behavior.
- Remove remediation text suggesting disabled or auto mode.
- Ensure Orchestrator is not constructed unless workflow validation passes.

## Acceptance Criteria

- Workflow loader failure aborts startup.
- Missing workflow definition aborts startup.
- Invalid workflow definition aborts startup.
- Workflow schema failure aborts startup.
- No user turn can begin after workflow startup validation failure.
- No error message suggests `workflow_mode=auto` or `workflow_mode=disabled`.

## Validation Plan

Add tests for:

- Missing `config/workflows/default.json`.
- Invalid workflow JSON.
- Workflow missing required stage.
- Workflow retry policy invalid.
- Workflow schema missing.
- Startup validation aggregates workflow fatal errors.
- No degraded startup path remains.

## Risks

Local development environments must initialize workflow config and schema before startup.

## Out of Scope

- Degraded operation.
- Auto fallback.
- Migration support.
