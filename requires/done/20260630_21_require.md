# [Low] Update MCP routing documentation to a two-layer model

## Priority
Low

## Background
MCP docs currently describe a four-layer routing cascade that includes config fallback and static fallback. After removing compatibility fallback paths, docs must describe the simplified routing model.

## Goal
Update routing documentation to match the new source-of-truth model.

## Scope
Affected files:
- `docs/04_mcp_01_system_overview.md`
- `docs/04_mcp_03_routing_lifecycle_and_execution.md`
- `docs/04_mcp_06_configuration_and_operations.md`

## Implementation Tasks
- Define routing priority as:
  1. live `/v1/tools` discovery
  2. `ToolRegistry`
- Move `tool_names` to a validation metadata section.
- Describe `tool_constants.py` as registry seed and classification data.
- Remove static fallback references.
- Remove config fallback references.
- Remove GitHub prefix exception references.
- Keep stdio transport documentation, but separate it from routing.

## Acceptance Criteria
- Docs no longer describe static fallback routing.
- Docs no longer describe config `tool_names` as routing input.
- Docs no longer describe GitHub prefix exception.
- stdio remains documented as an explicit transport option.

## Validation
- Run `rg -n "Static fallback|last-resort fallback|github_ prefix|prefix exception|tool_names.*routing" docs/04_mcp_*.md`.
