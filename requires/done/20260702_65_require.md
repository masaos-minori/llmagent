# H-5: Document session and workflow schema initialization for minimal agent startup

## Priority

High

## Labels

docs, db, startup, agent, operations

## Background

The agent requires `session.sqlite` to contain the `sessions` table before the REPL loop can start a session.

If the session schema is missing, startup reaches the REPL banner but then fails when `ctx.session.start()` runs:

`sqlite3.OperationalError: no such table: sessions`

`create_schema()` creates all schemas, but it starts with the RAG schema. Minimal agent startup should have a documented path for initializing session and workflow schemas directly.

## Goal

Document a minimal database initialization path for local development and first-time startup.

## Scope

### In scope

- Document how to initialize `session.sqlite`
- Document how to initialize `workflow.sqlite` when workflow tracking is enabled
- Add verification commands for required tables

### Out of scope

- sqlite-vec path handling
- RAG schema initialization changes
- `/exit` handling

## Required documentation

Add a section such as "Minimal Agent DB initialization" with:

1. Session and workflow schema initialization command
2. Table verification commands
3. Re-run safety note

Recommended command:

`PYTHONPATH=scripts uv run python - <<'PY'`
`from db.create_schema import create_session_schema, create_workflow_schema`
`create_session_schema()`
`create_workflow_schema()`
`PY`

Verification examples:

- Check `session.sqlite` has `sessions`
- Check `session.sqlite` has `messages`
- Check `workflow.sqlite` has `tasks`

## Acceptance criteria

- A new developer can initialize the minimum DB state before `python -m agent`
- Missing `sessions` table is clearly documented as a schema initialization issue
- The documented commands are idempotent
- The instructions do not require fixing sqlite-vec

## Validation

Run the documented commands in a fresh local environment.

Then run:

- `cd scripts`
- `uv run python -m agent`

Expected result:

- Agent starts without `no such table: sessions`

## Affected files

- README or developer setup guide
- Agent operations documentation
- Optionally `scripts/db/create_schema.py` module docstring

## Dependencies

None
