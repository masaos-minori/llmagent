# [Medium] Simplify lifecycle interfaces after stdio removal

## Goal

Simplify lifecycle protocols and factory wiring after removing stdio transport.

## Background

The lifecycle protocol currently includes stdio-specific methods such as `StdioLifecycleProtocol.restart_stdio()`. There is no `agent/http_lifecycle.py`, so lifecycle cleanup must be based on the existing factory/protocol/lifecycle files.

## Scope

Lifecycle structure cleanup only.

Do not assume `agent/http_lifecycle.py` exists.

## Implementation Tasks

- Remove `StdioLifecycleProtocol`.
- Remove `restart_stdio()` from lifecycle interfaces.
- Remove or simplify `shutdown_idle()` if it only existed for stdio ondemand idle shutdown.
- Update `agent/factory.py` to avoid stdio lifecycle imports.
- Remove lifecycle router branches for stdio.
- Ensure lifecycle manager behavior is HTTP-only.
- Update typing/protocol tests if any exist.
- Update docs that describe lifecycle routing across HTTP and stdio.

## Acceptance Criteria

- Lifecycle protocol has no stdio-specific method.
- Factory wiring does not import stdio lifecycle.
- `agent/stdio_lifecycle.py` is removed or no longer referenced.
- Watchdog and lifecycle operations are HTTP-only.
- No reference to non-existent `agent/http_lifecycle.py` is introduced.

## Validation

- Run type checks if available.
- Run lifecycle/factory tests if available.
- Grep for:
  - `StdioLifecycleProtocol`
  - `restart_stdio`
  - `StdioServerLifecycleManager`
  - `stdio_lifecycle`
