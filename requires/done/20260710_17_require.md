# [High] Align DESIGN-3 deletion contract, implementation comments, and integrity tests

## Background

RAG DESIGN-3 deletion documentation currently contains stale wording around the deletion order.

The canonical deletion contract should be updated to:

1. Delete `chunks_vec` rows first.
2. Delete the `documents` row.
3. Let `ON DELETE CASCADE` delete `chunks`.
4. Treat `chunks_vec_ad` trigger as an additional defensive cleanup path.

This matches the RAG schema, where `chunks.doc_id` references `documents.doc_id` with `ON DELETE CASCADE`, and write-mode SQLite connections enable `PRAGMA foreign_keys=ON`.

The MCP delete path already follows this model by deleting `chunks_vec` first and then deleting `documents`. The ingestion delete helper currently performs an explicit `chunks` delete as well, which is safe but no longer matches the canonical wording.

## Goal

Make DESIGN-3 deletion behavior consistent across documentation, ingestion implementation comments, MCP deletion behavior, schema notes, and index integrity tests.
