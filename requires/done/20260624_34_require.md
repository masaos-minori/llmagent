# Make `use_tool_dag` default semantics consistent across Agent docs

## Summary
Unify all documentation around the default value and default behavior of `use_tool_dag`.

## Background
The main Agent docs describe DAG scheduling as the default behavior, while the dedicated design note states that the deployed builder default may still override it to `False`. This is a meaningful behavioral conflict.

## Objective
Ensure the documentation set describes one final default for:
- `use_tool_dag`
- mixed read/write scheduling
- legacy full-round serialization fallback

## In Scope
- `05_agent_06_tool-execution-and-approval.md`
- `05_agent_08_configuration.md`
- `05_agent_16_dependency-aware-tool-scheduling.md`

## Required Changes
- Compare the design note and the main docs
- Decide whether the intended documented default is now final
- Update all descriptions of:
  - default value
  - scheduling semantics
  - legacy compatibility behavior
- If a rollout note is still needed, mark it clearly as implementation status, not as normative behavior

## Acceptance Criteria
- `use_tool_dag` has one documented default value across all Agent docs
- Mixed read/write scheduling behavior is documented consistently
- Legacy “serialize whole round” behavior is described as opt-in or legacy as appropriate

## Out of Scope
- Rewriting the scheduler implementation
- Changing resource-scope semantics

## Notes
If the implementation rollout status is still transitional, separate “current code status” from “intended default design” explicitly.
