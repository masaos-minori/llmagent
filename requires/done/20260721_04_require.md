# [Low] Remove dead `mqe_prompt_template` / `rerank_prompt_template` keys from `config/agent.toml`

## Priority
Low

## Labels
priority/low, area/config, area/rag, cleanup

## Summary
`config/agent.toml` declares two top-level keys, `mqe_prompt_template` (line 169) and
`rerank_prompt_template` (line 170), under the "System prompts" section. Neither key is read by
`AgentConfig` (`scripts/agent/config_dataclasses.py`) or its builder
(`scripts/agent/config_builders.py`) — a full-repo grep for both key names outside
`scripts/mcp_servers/rag_pipeline/` and `scripts/rag/` returns zero hits in any agent-side config
code. They must be removed from `config/agent.toml` as dead configuration.

## Background
The agent's MQE (multi-query expansion) and rerank LLM prompts are actually driven by
`config/rag_pipeline_mcp_server.toml`'s own `mqe_prompt_template` / `rerank_prompt_template` keys,
consumed via:
- `scripts/mcp_servers/rag_pipeline/rag_pipeline_models.py:46-47,77-78` (`RagPipelineConfig` dataclass
  fields and `from_dict()`)
- `scripts/mcp_servers/rag_pipeline/rag_pipeline_service.py:93-94` (passed into the pipeline adapter)
- `scripts/rag/llm_prompts.py:126,172` (actual template substitution for MQE and rerank calls)

`config/agent.toml`'s copies of these two keys (lines 169-170) are loaded into the merged config dict
by `ConfigLoader.load_all()` like every other key, but `_build_tool_config()` / `_build_llm_config()` /
`_build_rag_config()` (`scripts/agent/config_builders.py`) never reference either key name, and no
field named `mqe_prompt_template` or `rerank_prompt_template` exists anywhere in `AgentConfig`'s eight
sub-configs (`scripts/agent/config_dataclasses.py`). Per the documented Pydantic-dataclass behavior
("unknown fields are silently ignored", `docs/05_agent_08_01_configuration-loading-agent-config-part1.md`),
these two keys are parsed and then discarded — editing them in `config/agent.toml` has zero effect on
runtime behavior. This is a maintenance hazard: an operator who edits `config/agent.toml`'s
`mqe_prompt_template` expecting to change MQE phrasing will silently fail to do so; the only file that
actually controls that behavior is `config/rag_pipeline_mcp_server.toml`.

This is a narrower, more precisely-scoped follow-on to the RAG-config cleanup already completed in
`requires/done/20260714_13_require.md` (which removed `agent.toml`'s former RAG-pipeline-section keys
such as `use_semantic_cache`, `top_k_search`, `rrf_k`, etc. — confirmed absent from current
`config/agent.toml` as of 2026-07-21). `mqe_prompt_template` / `rerank_prompt_template` sit in a
different section (System Prompts, not RAG pipeline) and were not listed in that requirement's Bucket
1-4 enumeration, so they were missed by that cleanup pass.

## Required Changes
- Remove `mqe_prompt_template` (line 169) and `rerank_prompt_template` (line 170) from
  `config/agent.toml`.
- Grep `config/agent.toml` and confirm no other reference to either key remains (e.g. in comments
  documenting them).
- Do NOT touch `config/rag_pipeline_mcp_server.toml`'s identically-named keys — those are the real,
  effective settings and are out of scope.

  fields (e.g. alongside `system_prompt_tool` in the System Prompts section of any `05_agent_08_*` doc)
  and correct or remove those entries.
