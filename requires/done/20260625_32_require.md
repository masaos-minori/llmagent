## [Low][docs][eventbus] Document the new delivery state machine for retry, ack, offset, and DLQ behavior

### Summary
The current docs describe the existing unusual retry/DLQ behavior.
Once the implementation is fixed, the delivery state machine must be documented explicitly.

### Problem
If the improved lifecycle is not documented clearly, future maintainers will reintroduce ambiguity around retry, ack, and DLQ semantics.

### Scope
- 06_eventbus_03_persistence_schema_and_replay.md
- 06_eventbus_04_dlq_offsets_and_delivery_semantics.md
- 06_eventbus_06_reference_api.md

### Task
- define the delivery state machine explicitly
- document how ack, nack, timeout, retry, requeue, and DLQ promotion interact
- document the revised meaning of counters and offsets
- update examples and field semantics accordingly

### Expected Result
The Event Bus docs describe one clear delivery lifecycle with no ambiguous retry or DLQ behavior.

### Acceptance Criteria
- docs include an explicit delivery state machine
- retry and requeue semantics are clearly separated
- ack/offset semantics are documented
- DLQ promotion timing is documented correctly
