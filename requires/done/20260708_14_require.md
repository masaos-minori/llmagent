# M-4: Rename /context budget category from tool_results to tool_messages or tool_context

## Priority

Medium

## Background

`context_view.py` uses the name `tool_results` for LLM-context budget related to tool role messages and assistant `tool_calls` JSON. This is not the same as the removed `tool_results` database table.

To avoid confusion, the budget category should be renamed.

## Target files

- `agent/services/context_view.py`
- `agent/services/models.py`
- `agent/commands/cmd_context.py`

## Implementation tasks

- Rename the character budget field:
  - from `tool_results`
  - to `tool_messages` or `tool_context`
- Rename token budget field if applicable:
  - from `token_tool_results`
  - to `token_tool_messages` or `token_tool_context`
- Update `budget_breakdown()`.
- Update `_build_budget()`.
- Update `_token_breakdown()` output mapping.
- Update `/context` display labels.
- Keep the underlying calculation semantics:
  - tool role content counts toward tool-related context
  - assistant `tool_calls` JSON counts toward tool-related context

## Acceptance criteria

- `/context` no longer displays `tool_results` as a budget category.
- The new label clearly refers to LLM-visible tool messages or tool context.
- Budget totals remain unchanged.
- Token estimate totals remain unchanged.

## Test plan

- Build a sample history with system, user, assistant, assistant tool_calls, and tool messages.
- Verify character totals are unchanged after rename.
- Verify token estimate totals are unchanged after rename.
- Verify `/context` displays the new category name.
