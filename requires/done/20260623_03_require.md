## Title: [HIGH] Implement Graceful Shutdown on SIGTERM with Timeout Control

### Context
`agent.py` converts `SIGTERM` into `SystemExit(0)` and sets a `_shutdown_requested` flag, exiting the REPL loop after the current input wait.

### Problem Statement
If a `SIGTERM` arrives while a tool or an MCP server call is mid-execution, the agent might abort abruptly, leading to orphaned subprocesses or uncommitted database states.

### Requirements
- **Signal Handler Enhancement**: Enhance the `SIGTERM` handler in `agent.py`. If a tool execution loop is currently active (e.g., inside `execute_all_tool_calls()`), wait for the active turn to complete or safely rollback.
- **Timeout Guard**: Implement a configurable timeout (default: 10.0 seconds). If the active execution does not gracefully terminate within this window, force a safe database checkpoint/commit and exit.

### Constraints
- Do NOT block `SIGTERM` indefinitely. Ensure a hard timeout exists to prevent deadlocks.
- Do NOT use low-level `os._exit()` unless the graceful timeout expires.

### Acceptance Criteria (Definition of Done)
- [ ] Sending `SIGTERM` while a tool is running allows the current tool call to finish and log its result before the system exits.
- [ ] Database WAL checkpoints are safely triggered before the final exit.
