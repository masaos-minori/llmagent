# [agent][tools][execution] Record and analyze round-wide serialization caused by side-effect tools

## Summary
When one side-effect tool appears in a round, all tools in that round become sequential.

## Problem
This is safe, but it may reduce useful parallelism more than necessary.

## Scope
- tool execution flow
- tool execution stats
- diagnostics and observability

## Task
- record when round-wide serialization happens
- record which tool caused it
- record timing impact
- prepare data for future optimization

## Expected Result
The team can measure the real cost of safe serialization before changing execution rules.

## Acceptance Criteria
- [ ] serialization events are recorded
- [ ] the tool that triggered serialization is recorded
- [ ] timing impact can be analyzed
