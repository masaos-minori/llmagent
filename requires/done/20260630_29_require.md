# [High] Add regression tests for removed slash commands

## Priority
High

## Background
Several legacy slash commands are already removed from implementation and should be protected by regression tests. `/mcp install` is rejected because `/mcp` only accepts `status` or empty subcommand. `/db` only dispatches `rag`, `session`, or `help`. `/debug audit` is not supported by `cmd_debug.py`, and `/note` is not registered in the command registry. 【6-dc29e4】【7-c5a22c】【4-43114d】【5-25552b】

## Goal
Ensure removed slash commands stay removed.

## Scope
Affected files:
- `agent/commands/cmd_mcp.py`
- `agent/commands/cmd_db.py`
- `agent/commands/cmd_debug.py`
- `agent/commands/registry.py`
- `tests/test_cmd_mcp.py`
- `tests/test_cmd_db.py`
- `tests/test_cmd_debug.py`
- `tests/test_command_registry.py`
- Agent command documentation

## Removed or Unsupported Commands
- `/mcp install`
- `/db urls`
- `/db clean`
- `/db rebuild-fts`
- `/db recover`
- `/debug audit`
- `/note ...`

## Implementation Tasks
- Add a test that `/mcp install` is rejected.
- Add tests that flat `/db` aliases are rejected.
- Add a test that `/debug audit` is rejected.
- Add a test that `/note` is not registered as a built-in command.
- Remove deleted commands from active docs.
- Keep deleted commands only in migration/archive notes if needed.

## Acceptance Criteria
- Removed command aliases are rejected.
- Active command references only list canonical commands.
- Tests fail if a removed command is reintroduced.
- Tests pass.

## Validation
- Run: `pytest tests/test_cmd_mcp.py tests/test_cmd_db.py tests/test_cmd_debug.py tests/test_command_registry.py -v`
- Run: `rg -n "/mcp.*install|/debug audit|/note|/db urls|/db clean|/db rebuild-fts|/db recover" docs agent tests`
