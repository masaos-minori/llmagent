# Known Issues Cleanup Across All Documentation Sets

## Priority

Medium

## Target Files

- `docs/03_rag_90_inconsistencies_and_known_issues.md`
- `docs/04_mcp_90_inconsistencies_and_known_issues.md`
- `docs/05_agent_90_inconsistencies_and_known_issues.md`
- `docs/06_eventbus_90_inconsistencies_and_known_issues.md`
- `docs/90_shared_90_inconsistencies_and_known_issues.md`

## Background

Known Issues / Inconsistencies files across all documentation sets need cleanup to ensure they contain only unresolved issues that affect current specifications.

## Problem

Known Issues files may contain resolved issues, historical notes, future plans, and work memos that should not be kept as active issues. This creates noise and confusion about what actually needs attention.

## Reason for Change

Known Issues should only contain unresolved items. Resolved issues should not be kept as a history log. Current spec explanations should be moved to the main body. History items not affecting current specs should be removed.

## Implementation Intent

Classify each item in Known Issues files into "keep" or "delete" categories and clean up accordingly.

## Implementation Instructions

### Keep (move to Known Issues):

- Unresolved inconsistencies affecting current specs
- Undetermined items requiring implementation verification
- Spec differences requiring design decisions
- Behaviors that cannot be guaranteed due to insufficient tests
- Items where it's unclear whether to fix implementation or documentation

### Delete:

- Already-resolved issues
- History of deleted features
- Past histories not affecting current specs
- Simple work memos
- Future plans that are not yet implemented
- Items no longer needing to be Known Issues because the main text has been fixed

### Process for each file:

1. Review each item in the Known Issues file
2. Classify as keep or delete
3. Move items that belong in the main body to the appropriate section
4. Remove items that are resolved or irrelevant
5. Ensure only unresolved issues affecting current specs remain

## Acceptance Criteria

1. Known Issues contain only unresolved items
2. Resolved issues are not kept as a history log
3. Current spec explanations are moved to the main body
4. History items not affecting current specs are removed
