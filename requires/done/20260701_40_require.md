# [Low] Strengthen compatibility-remnant checks

## Goal

Update static compatibility checks so removed MCP compatibility paths do not reappear.

## Background

The cleanup removes stdio transport, `_MCP_TOOLS`, key=value MCP audit logs, deleted-doc references, and deleted feature history. These should be protected by static checks.

## Scope

Static checks only.

## Implementation Tasks

- Update `scripts/checks/check_no_compat.py`.
- Add checks for disallowed strings:
  - `_MCP_TOOLS`
  - `StdioTransport`
  - `TransportType.STDIO`
  - `HealthcheckMode.PING_TOOL`
  - `__list_tools__`
  - `run_stdio`
  - `--stdio`
  - `AUDIT session=`
  - `04_mcp_07_mdq_rag_boundary.md`
  - `POST /v1/search`
  - `/mcp install`
  - `mdq.db`
- Ensure allowed false positives, if any, are explicitly documented.
- Wire the check into CI if not already wired.

## Acceptance Criteria

- Compatibility remnants are detected automatically.
- `_MCP_TOOLS` cannot be reintroduced silently.
- stdio transport cannot be reintroduced silently.
- key=value MCP audit format cannot be reintroduced silently.
- deleted doc references cannot be reintroduced silently.

## Validation

- Run `python scripts/checks/check_no_compat.py`.
- Temporarily insert one forbidden string in a scratch file and confirm the check fails.
- Remove scratch change and confirm the check passes.
