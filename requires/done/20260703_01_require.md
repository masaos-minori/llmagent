# Add CommandDef and handler signature consistency tests

## Priority

High

## Labels

test, agent, cli, command-registry, regression

## Background

Built-in slash commands are defined declaratively by `CommandDef` entries.

`CommandRegistry.__init__()` currently validates only that each `CommandDef.handler` name exists on `CommandRegistry`. This does not verify that the handler signature matches the command definition.

Past issues showed that a handler can exist but still fail at runtime because the dispatch call shape does not match the handler signature.

## Goal

Add regression tests that validate the consistency between `CommandDef` metadata and actual handler methods.

## Scope

### In scope

- Add tests for all built-in command definitions in `_COMMANDS`
- Validate handler existence
- Validate exact command handler signatures
- Validate prefix command handler signatures
- Validate async/sync consistency

### Out of scope

- Testing external LLM, MCP, DB, or embedding services
- `/exit` handling
- sqlite-vec path handling
- Plugin command execution behavior

## Requirements

Tests must verify:

- Every `CommandDef.handler` exists on `CommandRegistry`
- `prefix=False` handlers can be called without an argument
- `prefix=True` handlers accept an `args: str` argument
- `is_async=True` handlers are coroutine functions
- `is_async=False` handlers are not coroutine functions

## Suggested implementation

Use:

- `inspect.signature()`
- `inspect.iscoroutinefunction()`

For bound methods:

- Exact commands should have zero required positional parameters
- Prefix commands should accept one positional parameter for `args`

## Acceptance criteria

- A missing command handler fails the test suite
- An exact command handler that requires `args` fails the test suite
- A prefix command handler that cannot accept `args` fails the test suite
- A wrong `is_async` flag fails the test suite
- Tests do not require running external services

## Validation

Run:

- `uv run pytest tests/ -k command`

Expected result:

- All command definition consistency tests pass
- No LLM, MCP, embedding, or database service is required

## Affected files

- `tests/`
- `scripts/agent/commands/registry.py`
- `scripts/agent/commands/command_defs.py`
- `scripts/agent/commands/command_defs_list.py`

## Dependencies

None
