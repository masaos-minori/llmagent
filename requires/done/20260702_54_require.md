# Remove backward-compatible re-export stub from `mcp/github/service.py`

## Priority

High

## Summary

Remove the backward-compatible re-export stub from `mcp/github/service.py` and update all imports to direct implementation modules.

## Background

`mcp/github/service.py` is a backward-compatible re-export stub. It re-exports:

- `GitHubService` from `mcp.github.service_dispatch`
- `build_service` from `mcp.github.service_init`
- `_GITHUB_TOKEN` from `mcp.github.service_init`

This old import path exists only for compatibility and should be removed.

## Goal

Remove `mcp.github.service` as an import path without changing GitHub MCP behavior.

## Target files

- `mcp/github/service.py`
- `mcp/github/server.py`
- `mcp/github/service_dispatch.py`
- `mcp/github/service_init.py`
- GitHub MCP tests
- Documentation files

## Scope

### In scope

- Replace imports from `mcp.github.service`.
- Delete `mcp/github/service.py`.
- Update documentation.
- Preserve GitHub MCP server behavior.

### Out of scope

- Changing `GitHubService` behavior.
- Changing `build_service()` behavior.
- Changing GitHub MCP tools or endpoints.
- Keeping deprecated import aliases.

## Implementation tasks

- Update `mcp/github/server.py`.

- Replace this import:

    from mcp.github.service import _GITHUB_TOKEN, GitHubService, build_service

- With direct imports:

    from mcp.github.service_dispatch import GitHubService
    from mcp.github.service_init import _GITHUB_TOKEN, build_service

- Search for old imports:

    grep -R "from mcp.github.service import\|import mcp.github.service" -n scripts tests docs

- Replace all old imports with direct imports.
- Delete `mcp/github/service.py`.
- Remove documentation that recommends or allows `mcp.github.service`.

## Acceptance criteria

- `mcp/github/service.py` no longer exists.
- No code imports from `mcp.github.service`.
- `mcp/github/server.py` imports directly from:
  - `mcp.github.service_dispatch`
  - `mcp.github.service_init`
- GitHub MCP server imports successfully.
- GitHub MCP endpoint and tool behavior are unchanged.
