# Require explicit approval IDs for /approve and /reject

## Priority

Medium

## Type

Workflow Approval / CLI / Safety

## Background

The workflow approval commands currently support explicit approval IDs.

They may also allow ID-less resolution when exactly one pending approval exists.

Migration compatibility is not required.

## Problem

ID-less approval resolution is ambiguous by design and depends on global pending approval state.

Even if only one pending approval exists at the time, keeping ID-less behavior makes the command contract less safe and less explicit.

## Goal

Require explicit approval IDs for all workflow approval decisions.

## Scope

Affected areas:

- `/approve` command handler
- `/reject` command handler
- Command parser
- Help text
- Startup recovery notice
- Approval tests

## Implementation Tasks

- Change syntax to:
  - `/approve <approval_id> [reason]`
  - `/reject <approval_id> [reason]`
- Reject ID-less `/approve`.
- Reject ID-less `/reject`.
- Remove latest-pending fallback from command handlers.
- Do not call `find_latest_pending_approval()` from approve/reject commands.
- Update validation messages.
- Update command help.
- Ensure startup notices include approval IDs.

## Acceptance Criteria

- `/approve` without ID returns validation error.
- `/reject` without ID returns validation error.
- `/approve <approval_id>` resolves only that approval.
- `/reject <approval_id>` resolves only that approval.
- Multiple pending approvals are safe by construction.
- Latest-pending lookup is not used for approval decisions.

## Validation Plan

Add tests for:

- `/approve` without ID.
- `/reject` without ID.
- `/approve <valid_id>`.
- `/reject <valid_id>`.
- `/approve <missing_id>`.
- `/reject <resolved_id>`.
- Multiple pending approvals.
- Startup notice includes approval ID.

## Risks

Operators must copy approval IDs explicitly. This is intentional.

## Out of Scope

- Legacy ID-less approval syntax.
- Latest-only approval recovery.
- Migration support.
