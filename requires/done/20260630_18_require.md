# [Medium] Add registry/discovery consistency tests for RAG MCP tools

## Priority
Medium

## Background
RAG MCP defines four production tools in `_MCP_TOOLS`. After removing fallback routing, these tools must be resolvable through `ToolRegistry` and live discovery only.

## Goal
Ensure RAG MCP tools are consistently represented in tool schema, discovery output, and registry ownership.

## Scope
Affected files:
- `scripts/mcp/rag_pipeline/tools.py`
- `scripts/mcp/rag_pipeline/server.py`
- `shared/tool_registry.py`
- `tests/`

## Implementation Tasks
- Verify the following RAG tools are registered:
  - `rag_run_pipeline`
  - `rag_debug_pipeline`
  - `rag_list_documents`
  - `rag_delete_document`
- Add tests comparing `_MCP_TOOLS` names with RAG tools in `ToolRegistry`.
- Add tests verifying `/v1/tools` returns `server_key="rag_pipeline"` for RAG tools.
- Add tests proving these tools resolve without static fallback.
- Ensure all four RAG tools keep production status.

## Acceptance Criteria
- RAG tools are registered in `ToolRegistry`.
- `/v1/tools` discovery and registry ownership agree.
- RAG tools route without static fallback.
- Tests fail if a RAG tool is added to `_MCP_TOOLS` but not registered.

## Validation
- Run RAG MCP tests.
- Run `pytest tests/test_route_resolver.py -v`.
- Add or run a discovery-map test for `rag_pipeline`.
