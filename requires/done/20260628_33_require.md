# Remove `/debug audit` branch from debug command handler

## Priority
High

## Labels
agent, commands, debug, audit, cleanup, breaking-change

## Dependencies
None

## Background
`/debug audit` is currently implemented inside `cmd_debug.py` as a shortcut that reads the audit log file and prints the last 20 lines. This overlaps with the dedicated `/audit` command, which already supports audit log browsing through `tail`, `turn`, and `tool` subcommands.

## Task
Remove the `/debug audit` subcommand from the debug command handler.

## Implementation Requirements
- Update `cmd_debug.py`.
- Remove the `if sub == "audit":` branch from `_DebugMixin._cmd_debug()`.
- Remove direct audit log tail reading from the debug command.
- Ensure `/debug audit` no longer displays audit log contents.
- Keep the following debug commands:
  - `/debug`
  - `/debug verbose`
  - `/debug normal`

## Acceptance Criteria
- `/debug audit` does not print audit log lines.
- `/debug` still toggles debug mode.
- `/debug verbose` still sets debug log level.
- `/debug normal` still restores normal log level.
- Audit log browsing remains available via `/audit`.

## Source Context
`cmd_debug.py` currently handles `/debug audit` by reading `ctx.cfg.obs.audit_log_file` and printing the last 20 lines. `cmd_audit.py` already provides dedicated audit browsing functionality.
