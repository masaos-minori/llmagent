# Remove or isolate `agent/services/mcp_install.py`

## Priority
High

## Labels
agent, mcp, cleanup, dead-code, breaking-change

## Dependencies
- Remove `/mcp install` dispatch from MCP command handler
- Remove MCP install-specific imports from `cmd_mcp.py`

## Background
`agent/services/mcp_install.py` implements the scaffold generation workflow used by `/mcp install`. It defines `McpInstallParams`, `ScaffoldResult`, `InstallQA`, `CliInstallQA`, and `McpInstallService`.

## Task
Determine whether `agent/services/mcp_install.py` is still used outside `/mcp install`. If it is not, delete it. If it is used by an external developer tool, move it out of the Agent REPL runtime path.

## Implementation Requirements
- Search the repository for:
  - `McpInstallService`
  - `CliInstallQA`
  - `McpInstallParams`
  - `ScaffoldResult`
  - `InstallQA`
- If references exist only in `/mcp install` code and tests, delete `agent/services/mcp_install.py`.
- If another developer workflow uses this service, move ownership outside the Agent REPL command layer.
- Remove related imports and exports if any.

## Acceptance Criteria
- Agent runtime does not import `agent.services.mcp_install`.
- `/mcp` command has no dependency on installer services.
- If deleted, no tests or modules reference `McpInstallService`.
- If retained, it is clearly outside the Agent REPL runtime path.

## Source Context
`mcp_install.py` is specifically described as `McpInstallService — scaffold new MCP server template files` and is extracted from `cmd_mcp._McpMixin` for the `/mcp install` wizard.
