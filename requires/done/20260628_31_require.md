# Update help text and documentation for `/ingest` removal

## Priority
Medium

## Labels
docs, agent, commands, rag, ingest, breaking-change

## Dependencies
- Remove `/ingest` command registration from CommandRegistry
- Remove `_cmd_ingest()` from the Agent command mixin

## Background
The Agent documentation and help text currently describe `/ingest` as an Agent REPL command for crawling and ingesting documents into the RAG DB.

## Task
Remove `/ingest` from user-facing help and documentation.

## Implementation Requirements
- Ensure `/help` no longer lists `/ingest`.
- Update Agent command documentation:
  - remove `/ingest`
  - remove “Ingest / RAG” category or rename it to “RAG”
  - keep `/rag search <query> [--debug]`
- Update operations documentation so ingestion diagnostics are not described as an Agent REPL command.
- Document that RAG ingestion should be run outside the Agent REPL.

## Acceptance Criteria
- Docs no longer present `/ingest` as an available command.
- `/rag search` remains documented.
- The replacement operational workflow is clearly stated.

## Source Context
Agent docs currently list `/ingest` as a slash command and describe ingestion diagnostic output.
