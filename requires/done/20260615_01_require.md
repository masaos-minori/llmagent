## BUG-1 — `create_schema.py` モジュールレベル ConfigLoader の遅延化

**修正ファイル**: `scripts/db/create_schema.py`

**根本原因**: L34 の `logger = Logger(__name__, _get_schema_log_path())` が
import 時に `ConfigLoader().load("common.toml")` を呼ぶため、
`common.toml` のない環境でテスト collection が失敗する。

**修正内容**

```python
# 変更前 (L34)
logger = Logger(__name__, _get_schema_log_path())

# 変更後
_logger: Logger | None = None

def _get_logger() -> Logger:
    global _logger
    if _logger is None:
        _logger = Logger(__name__, _get_schema_log_path())
    return _logger
```

`logger` を直接参照している全箇所を `_get_logger()` 呼び出しに置き換える。

**影響範囲**: `scripts/db/create_schema.py` のみ
**テスト回復**: `tests/test_create_schema.py` 全 11 件

**ToolCallResult フィールド** (`scripts/shared/tool_executor.py:47-56`):

`transport.call()` / `_raw_execute()` / `execute()` の戻り値を
tuple として unpack している全箇所を属性アクセスに変更する。
