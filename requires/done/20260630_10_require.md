# [Low] Add CI check for backward compatibility leftovers

## Priority
Low

## Difficulty
Medium

## Background
Backward compatibility leftovers were found in active source code and comments, including:

- `rag/models.py` as a re-export stub
- module-level config cache references in RAG MCP service comments

## Goal
Prevent backward compatibility layers and stale migration wording from re-entering active source code.

## Scope
Affected files:

- `scripts/checks/check_no_compat.py`
- `scripts/checks/check_docs_consistency.py`
- CI workflow files
- source and docs under:
  - `scripts/`
  - `docs/`
  - `tests/`

## Implementation Tasks
- Add a check that scans active source and docs for stale compatibility patterns.
- Suggested patterns:
  - `backward compatibility`
  - `re-export stub`
  - `from rag.models import`
  - `import rag.models`
  - `rag.llm`
  - `module-level _cfg`
  - `_cfg cache`
- Add allowlist support for archive or migration note sections only.
- Wire the check into CI.
- Document how to update the allowlist when adding archive-only migration notes.

## Acceptance Criteria
- CI fails if new active source code introduces a compatibility stub.
- CI fails if active docs recommend old import paths.
- Archive or migration notes can be explicitly allowlisted.
- The check reports file path and line number.

## Validation
- Run `python scripts/checks/check_no_compat.py`.
- Intentionally add `from rag.models import X` in a test fixture and confirm the check fails.
