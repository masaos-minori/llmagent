# [Low] Define a unified MCP tool schema export naming policy

## Priority
Low

## Background
GitHub MCP uses `TOOL_LIST` as canonical export and keeps `_MCP_TOOLS` only for backward compatibility. RAG MCP tests still reference `_MCP_TOOLS`. A consistent naming policy should be defined across MCP servers. 【3-018a33】【3-828f7a】

## Goal
Define whether all MCP servers should use `TOOL_LIST` as the canonical tool schema export.

## Scope
Affected files:
- `mcp/github/tools.py`
- `mcp/rag_pipeline/tools.py`
- other `mcp/*/tools.py` files
- tests
- docs

## Implementation Tasks
- Keep GitHub `TOOL_LIST` as canonical.
- Remove GitHub `_MCP_TOOLS` after confirming no active references remain.
- Decide whether RAG MCP and other MCP servers should migrate to `TOOL_LIST`.
- If migration is approved, create follow-up issues per MCP server.
- Update tests to reference canonical exports.

## Acceptance Criteria
- GitHub compatibility alias is addressed.
- A clear export naming policy exists for MCP tool schema modules.
- RAG and other MCP server migrations are tracked separately if needed.

## Validation
- Run: `rg -n "_MCP_TOOLS|TOOL_LIST" mcp tests docs`
