# [Medium] Print workflow metadata in deploy logs

## Background

Workflow definitions are mandatory deployment artifacts. Operators need to know which workflow definition was deployed.

## Goal

Print workflow metadata during deployment.

## Scope

- `deploy/deploy.sh`
- Workflow validation CLI, if implemented

## Tasks

- [ ] Parse workflow `name`.
- [ ] Parse workflow `version`.
- [ ] Parse workflow stage IDs.
- [ ] Compute SHA256 checksum of `config/workflows/default.json`.
- [ ] Print source workflow metadata before copy.
- [ ] Print deployed workflow metadata after copy.
- [ ] Print checksum of both source and deployed files.
- [ ] Fail deployment if source and deployed checksum differ.

## Suggested Output

`Workflow definition:`

`Source   : config/workflows/default.json`

`Deployed : /opt/llm/config/workflows/default.json`

`Name     : default`

`Version  : 1.0.0`

`Stages   : plan, execute, verify, retry`

`SHA256   : <checksum>`

## Acceptance Criteria

- [ ] Deploy logs include workflow name.
- [ ] Deploy logs include workflow version.
- [ ] Deploy logs include stage list.
- [ ] Deploy logs include SHA256 checksum.
- [ ] Source and deployed checksums are compared.

## Notes

This improves traceability and rollback diagnostics.
