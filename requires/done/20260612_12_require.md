# Goal

Create a work plan for changing note auto-injection from a full-injection model to a retrieval-based injection model.

The plan must cover the following goals:
- Replace full note injection with search-based note injection.
- Inject only pinned notes at session start.
- During normal turns, search for and inject only notes related to the current query.
- Add support for `/note pin`, `/note unpin`, and `/note search`.
- Create the plan only.
- Do not implement anything.

# Design

## Injection strategy split
Design the new note injection behavior as follows:
- At session start:
  - stop injecting all notes
  - inject only notes where `pinned = true`
- During normal turns:
  - search notes using the current query
  - inject only relevant notes

Purpose:
- prevent prompt growth as notes increase
- avoid constantly injecting unrelated notes

## Notes schema extension
Add the following candidate fields:
- `pinned`
- `priority`
- `tags`

Because the current schema is unknown:
- it is still unknown whether this should be done with `ALTER TABLE` or table recreation

## Command design
Extend the dispatcher in `cmd/notes.py` with:
- `/note pin <id>`
- `/note unpin <id>`
- `/note search <query>`

Current dispatcher only supports:
- `add`
- `list`
- `delete`

## Injection return format
Prefer a structured response instead of plain `list[str]`.

The structured response should be able to carry information such as:
- note source
- pinned state
- priority
- matched_by

## Search quality design
To reduce the risk of missed retrieval:
- keep a two-layer model
  - pinned-note injection
  - query-based retrieval injection

Role split:
- pinned notes = guaranteed baseline context at startup
- retrieval injection = relevant note extraction during normal turns
