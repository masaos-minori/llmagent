# [High] Add regression tests for DESIGN-2 content-only invariant

## Priority
High

## Background
DESIGN-2 states that `chunks.content` is the only text allowed in LLM context, while `chunks.normalized_content` is only for FTS5 indexing. The current `AugmentStage` implementation uses `c.content`, which matches the design, but regression tests for this invariant are missing.

Evidence:
- DESIGN-2 defines `content` for LLM context and `normalized_content` for FTS5 only.
- `augment.py` formats only `c.content`.
- Existing `test_fts_fallback.py` covers FTS fallback behavior but not AugmentStage content-only output.

## Goal
Prevent `normalized_content` from ever leaking into RAG context blocks.

## Scope
Affected files:
- `tests/test_rag_pipeline.py`
- `tests/test_rag_pipeline_stage.py`
- `docs/03_rag_90_inconsistencies_and_known_issues.md`

## Implementation Tasks
- Add a `_format_chunks()` test in `tests/test_rag_pipeline.py`.
- Add an `AugmentStage.run()` test in `tests/test_rag_pipeline_stage.py`.
- Test with:
  - original content text
  - distinct normalized text
  - assert only original content appears in output.
- Update `03_rag_90` to mark DESIGN-2 content-only tests as implemented.

## Open Questions / Unknowns
- Need to confirm whether `RagHit` dataclasses can carry `normalized_content`. If not, the test should simulate normalized text as a forbidden string.
- Need to confirm whether `_format_chunks()` should remain duplicated in both `rag.pipeline` and `rag.stages.augment`, or one should be authoritative.

## Acceptance Criteria
- RAG context includes original readable content.
- RAG context does not include normalized FTS text.
- Combining FTS search and AugmentStage still returns original content.

## Validation
- Run `uv run pytest tests/test_rag_pipeline.py tests/test_rag_pipeline_stage.py -v`.
