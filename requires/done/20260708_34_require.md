# M-2: Clarify ConfigReloadOutcome.skipped semantics

## Priority

Medium

## Background

`ConfigReloadOutcome.skipped` can be confused with `needs_restart` when the same MCP server change is reported in both sections.

MCP server definition changes should be represented as restart-required, not skipped.

## Target files

- `agent/services/config_reload.py`
- `agent/commands/cmd_config.py`
- tests for reload output

## Implementation tasks

- Stop adding MCP server definition changes to `skipped`.
- Keep new, removed, renamed, or changed MCP servers in `needs_restart`.
- Define `skipped` as a separate category for future non-MCP ignored items, if still needed.
- Ensure `/reload` output does not list the same MCP item under both `[SKIP]` and `[RESTART]`.
- Update tests and docs to reflect the new semantics.

## Acceptance criteria

- MCP server changes are not reported as skipped.
- New MCP servers appear only under restart-required.
- Removed MCP servers appear only under restart-required.
- `/reload` output contains no duplicate classification for the same MCP item.

## Test plan

- Reload with a new server.
- Reload with a removed server.
- Verify `skipped` is empty for those cases.
- Verify `needs_restart` contains the expected entries.
