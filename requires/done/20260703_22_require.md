# Add PluginToolInvoker and remove plugin execution details from ToolExecutor.execute

## Priority

High

## Goal

Move plugin tool lookup, execution, error handling, and defensive validation out of `ToolExecutor.execute()`.

`ToolExecutor.execute()` should become a thin orchestration entry point:

1. Try plugin tool execution.
2. If plugin tool is not found, continue to the MCP cache-aware execution path.

## Background

Plugin tools bypass MCP routing and cache. They also take precedence over MCP tools.

Currently, plugin execution details are embedded directly in `ToolExecutor.execute()`. This mixes local plugin execution with MCP routing/cache/transport concerns.

## Scope

Affected files:

- `shared/tool_executor.py`
- `shared/plugin_tool_invoker.py` new file
- `shared/plugin_registry.py`
- `shared/transport_dto.py`
- tests for plugin invocation

## Implementation requirements

Create:

- `shared/plugin_tool_invoker.py`

Add:

- `PluginToolInvoker`

Expected API:

    class PluginToolInvoker:
        async def try_execute(
            self,
            tool_name: str,
            args: dict[str, Any],
        ) -> ToolCallResult | None:
            ...

Behavior:

- If no plugin tool exists, return `None`.
- If a plugin tool exists, execute it and return `ToolCallResult`.
- Plugin tools must bypass cache.
- Plugin tools must bypass MCP routing.
- Plugin tools must take precedence over MCP tools.
- Plugin execution exceptions must be converted to `ToolCallResult(is_error=True, error_type="tool")`.
- Keep defensive runtime validation of plugin return values even though registration-time validation is canonical.

Update `ToolExecutor.execute()` to approximately:

    async def execute(self, tool_name: str, args: dict[str, Any]) -> ToolCallResult:
        plugin_result = await self._plugin_invoker.try_execute(tool_name, args)
        if plugin_result is not None:
            return plugin_result

        return await self._execute_with_cache(tool_name, args)

## Acceptance criteria

- `ToolExecutor.execute()` no longer directly calls `plugin_registry.get_tool()`.
- `ToolExecutor.execute()` no longer contains detailed plugin return validation.
- Plugin execution errors do not propagate outside the invoker.
- Plugin tool results still use `request_id=""` and `server_key=""`.
- Plugin tools still bypass cache.
- Plugin tools still have higher precedence than MCP tools.

## Validation plan

- Add `tests/shared/test_plugin_tool_invoker.py`.
- Test plugin not found returns `None`.
- Test valid plugin execution returns `ToolCallResult`.
- Test plugin exception returns error result.
- Test plugin tool bypasses cache by mocking the MCP path.
- Test plugin tool shadows MCP tool when conflict policy allows it.

## Out of scope

- Changing plugin conflict policy.
- Changing plugin load order.
- Changing plugin command behavior.
