# H-7: Remove AgentContext.tool_result_store

## Priority

High

## Background

`AgentContext` owns `tool_result_store`, which is used by tool execution, partial completion handling, error injection, undo, and `/tool` commands.

After removing ToolResultStore-based behavior, `AgentContext` should no longer create or expose `tool_result_store`.

## Target files

- `agent/context.py`
- `agent/tool_runner.py`
- `agent/llm_transport_errors.py`
- `agent/error_injection_service.py`
- `agent/services/undo_service.py`
- `agent/commands/cmd_tooling.py`

## Implementation tasks

- Remove the `ToolResultStore` import from `agent/context.py`.
- Remove `self.tool_result_store = ToolResultStore()` from `AgentContext.__init__()`.
- Remove all references to `ctx.tool_result_store`.
- Ensure no code path requires `tool_result_store` on `AgentContext`.
- Update type hints and tests accordingly.

## Acceptance criteria

- `AgentContext` no longer has a `tool_result_store` field.
- The application starts without importing `db.tool_results`.
- Tool execution, partial completion handling, undo, and commands work without `ctx.tool_result_store`.
- Static grep for `tool_result_store` returns no runtime references except possibly migration notes or deprecated tests.

## Test plan

- Instantiate `AgentContext`.
- Build agent context through the factory.
- Run a normal turn without tools.
- Run a turn with tools.
- Simulate partial completion.
- Run `/undo`.
- Verify no `AttributeError` occurs.
