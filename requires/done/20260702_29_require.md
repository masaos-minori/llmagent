# [Medium] Add live discovery priority tests for ToolRouteResolver

## Priority

Medium

## Goal

Test the full two-layer routing priority.

## Background

Routing priority is:

1. Live /v1/tools discovery
2. ToolRegistry

Existing tests cover registry routing and config non-routing, but live discovery override is not clearly covered.

## Scope

Affected files:

- tests/test_route_resolver.py
- shared/route_resolver.py

## Tasks

- Add tests for build_discovery_map().
- Add a test where live discovery routes a known registry tool to a different server key.
- Assert live discovery wins over ToolRegistry.
- Add a test for live metadata without server_key.
- Add a test for duplicate live tool names if behavior is defined.
- Keep unknown tool behavior unchanged.

## Unknowns / Open Questions

- Confirm whether duplicate live tool names should use first-wins, last-wins, or raise.
- Confirm whether missing server_key should be ignored or treated as drift.

## Acceptance Criteria

- Discovery map priority is covered by tests.
- Registry fallback still works when no discovery map entry exists.
- Unknown tools still raise ValueError.

## Validation

Run:
```bash
uv run pytest tests/test_route_resolver.py -v
```
