# Remove backward-compatible re-export stub from `agent/config.py`

## Priority

High

## Summary

Remove the backward-compatible re-export stub from `agent/config.py` and require all callers to import directly from the split configuration modules.

## Background

`agent/config.py` is currently a backward-compatible re-export stub. It re-exports public symbols from:

- `agent.config_builders`
- `agent.config_dataclasses`

This exists only to keep old imports working. The project policy is to remove compatibility-only stubs.

## Goal

Remove the `agent.config` compatibility import path without changing runtime configuration behavior.

## Target files

- `agent/config.py`
- `agent/config_builders.py`
- `agent/config_dataclasses.py`
- Python files under `scripts/`
- Python files under `tests/`
- Documentation files

## Scope

### In scope

- Replace all imports from `agent.config`.
- Delete `agent/config.py`.
- Remove comments that mention re-export through `agent.config`.
- Update documentation to use direct import paths.

### Out of scope

- Changing config dataclass behavior.
- Changing config loading behavior.
- Keeping deprecated import aliases.
- Adding warning-only deprecation shims.

## Implementation tasks

- Search for old imports:

    grep -R "from agent.config import\|import agent.config" -n scripts tests docs

- Replace builder and loader imports with direct imports from `agent.config_builders`.

  Examples:

  - `build_agent_config`
  - `load_config`
  - `ConfigLoadError`
  - `_CONFIG_DIR`
  - `_build_llm_config`
  - `_build_rag_config`
  - `_build_tool_config`
  - `_build_memory_config`
  - `_build_approval_config`
  - `_build_mcp_servers`

- Replace dataclass imports with direct imports from `agent.config_dataclasses`.

  Examples:

  - `AgentConfig`
  - `LLMConfig`
  - `RAGConfig`
  - `ToolConfig`
  - `MemoryConfig`
  - `MCPConfig`
  - `ApprovalConfig`
  - `ObservabilityConfig`

- Delete `agent/config.py`.
- Remove any comment in `agent/config_builders.py` that says symbols are re-exported via `agent.config`.
- Remove documentation that says old `agent.config` imports continue to work.

## Acceptance criteria

- `agent/config.py` no longer exists.
- No code imports from `agent.config`.
- No documentation recommends `agent.config`.
- `build_agent_config()` behavior is unchanged.
- Config dataclass behavior is unchanged.
- Tests pass.
