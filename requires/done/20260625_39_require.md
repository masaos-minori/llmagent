## [Medium][eventbus][routing] Implement topic-aware fan-out in the in-memory broker

### Summary
Current topic filtering is performed through repeated SQL queries in the subscribe loop.

### Problem
Database-driven topic filtering is inefficient when many subscribers watch overlapping topic sets.

### Scope
- in-memory Pub/Sub layer
- subscribe topic filtering
- subscriber registration model

### Task
- add topic-aware routing to the in-memory broker
- fan out only matching events to each subscriber
- avoid re-querying SQLite for live topic filtering
- support both “all topics” subscribers and filtered subscribers
- add tests for topic fan-out correctness

### Expected Result
Topic filtering for live delivery happens in memory instead of repeated SQL polling.

### Acceptance Criteria
- live topic filtering is handled by the broker
- SQL is no longer the primary filter path for live subscribers
- filtered and unfiltered subscribers both work correctly
- tests cover topic routing behavior
