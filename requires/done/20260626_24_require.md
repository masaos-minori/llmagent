# [Medium] Make outline use the MDQ index and return hierarchy metadata

## Priority
Medium

## Labels
`mcp`, `mdq`, `outline`, `markdown`, `api`

## Dependencies
- Redesign MDQ SQLite schema for production indexing
- Replace line-based Markdown parser with hierarchy-aware parser

## Background
Current `outline` reparses a file and returns a flat heading list. Production MDQ should use indexed metadata and return a heading hierarchy that can guide chunk retrieval.

## Task
Implement index-backed outline retrieval.

## Implementation Requirements
- Read outline data from indexed chunks.
- Return hierarchy information:
  - heading
  - level
  - heading_path
  - chunk_id
  - start_line
  - end_line
- Add `max_depth`.
- Add `max_items`.
- Return stale index warning if the source file changed after indexing.

## Acceptance Criteria
- `outline` no longer reparses arbitrary files directly.
- Output includes chunk IDs.
- Output preserves heading hierarchy.
- Large outlines are capped.
