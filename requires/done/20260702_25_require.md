# [High] Add explicit HTTP-only rejection tests for McpServerConfig

## Priority

High

## Goal

Lock down the HTTP-only MCP configuration policy.

## Background

McpServerConfig currently supports HTTP-only transport. Removed stdio-related values should be explicitly rejected by tests.

## Scope

Affected files:

- tests/test_mcp_config.py
- shared/mcp_config.py

## Tasks

- Add a test that transport="stdio" raises ValueError.
- Add a test that startup_mode="ondemand" raises ValueError.
- Add a test that healthcheck_mode="ping_tool" raises ValueError.
- Keep valid tests for transport="http".
- Keep valid tests for startup_mode="persistent".
- Keep valid tests for startup_mode="subprocess".
- Keep valid tests for healthcheck_mode="http".

## Unknowns / Open Questions

- Confirm the exact exception messages from enum conversion.
- Confirm whether config loader normalizes these fields before constructing McpServerConfig.

## Acceptance Criteria

- Removed stdio-related values are explicitly rejected.
- HTTP-only values remain valid.
- Future reintroduction of stdio config values fails tests.

## Validation

Run:
```bash
uv run pytest tests/test_mcp_config.py -v
```
