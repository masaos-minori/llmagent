# H-7: Clean up _http_procs entries during shutdown_all

## Priority

High

## Background

`HttpServerLifecycleManager.restart()` removes the process entry using `pop()` before terminating it. However, `shutdown_all()` iterates over `_http_procs` but does not remove entries. This leaves stale process records after shutdown.

## Target files

- `agent/http_lifecycle.py`

## Implementation tasks

- Change `shutdown_all()` to pop entries from `_http_procs`.
- Clean up both running and already-exited processes.
- Close associated stderr log file handles if using per-server log file redirect.
- Make repeated `shutdown_all()` calls safe no-ops.
- Ensure one failed shutdown does not prevent cleanup of other process entries.

## Acceptance criteria

- After `shutdown_all()`, `_http_procs` is empty.
- Already-exited processes are removed.
- Repeated `shutdown_all()` does not raise.
- Shutdown errors are logged and do not stop cleanup for other servers.

## Test plan

- Add two fake processes and call `shutdown_all()`; assert `_http_procs` is empty.
- Include one already-exited process and assert it is also removed.
- Simulate termination failure and verify cleanup continues.
- Call `shutdown_all()` twice and assert no error.
