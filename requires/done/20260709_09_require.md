# [High] Fix stale documentation for tool_results_turn_max_chars

## Labels
priority/high, area/docs, area/tool-execution, safety

## Summary
Fix stale documentation for `tool_results_turn_max_chars`.

This field must not be removed. It is an active per-turn safety limit.

## Background
`tool_results_turn_max_chars` was previously described as deprecated and no longer enforced because the old `tool_results` persistence table was removed.

That premise is incorrect.

The field is actively used as a cumulative per-turn tool-result text limit. It is distinct from `tool_result_max_llm_chars`. 【1-b8f45a】

`TURN_LIMIT_HINT` is documented as the hint appended when a tool result is dropped due to the per-turn limit. 【1-5c05eb】

## Required Changes

### Documentation
Update `05_agent_08_configuration.md`.

Replace stale wording such as:

- deprecated
- tool_results table removed
- no longer enforced

with an accurate description:

- maximum cumulative tool result characters added to LLM context in a single turn
- protects against excessive per-turn context growth from multiple tool outputs
- when exceeded, omitted results are replaced with a turn-limit hint

### Clarify related limits
Document the difference:

- `tool_result_max_llm_chars`
  - per single tool result
- `tool_results_turn_max_chars`
  - cumulative per turn across all tool results

### Cross-reference
Add or improve reference to `TURN_LIMIT_HINT` behavior in `05_agent_06_tool-execution-and-approval.md`.

## Acceptance Criteria
- Documentation no longer marks `tool_results_turn_max_chars` as deprecated.
- Documentation accurately describes the active per-turn safety limit.
- Difference from `tool_result_max_llm_chars` is clear.
- No code removes or changes the field.
