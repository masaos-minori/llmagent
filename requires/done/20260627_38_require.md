## Title
[Medium] Correct `/memory rebuild` help text and separate JSONL import from index repair

## Priority
Medium

## Labels
memory, cli, documentation, ux

## Dependencies
H-02 should be completed first.

## Background
The current `/memory rebuild` command imports records from the JSONL archive. It explicitly warns that deleted entries and pin state are not replayed.

However, the command help says it rebuilds SQLite from the JSONL source of truth, which is incorrect. JSONL is an append-only archive, not the authoritative state.

## Scope
Update:
- `agent/commands/cmd_memory.py`
- Memory documentation
- Memory command tests

## Required Changes
- Change help text from “JSONL source of truth” to “JSONL archive”.
- Consider adding `/memory import-jsonl` as an alias.
- Clarify that JSONL import does not replay:
  - deletes
  - pin state
  - unpin state
- Separate JSONL import from FTS/vector index repair.

## Tests to Add
- `test_memory_help_describes_jsonl_as_archive`
- `test_memory_rebuild_dry_run_mentions_archive`
- `test_memory_import_jsonl_alias_if_added`

## Acceptance Criteria
- CLI help no longer describes JSONL as source of truth.
- User-facing messages match actual behavior.
- JSONL import and SQLite index repair are clearly distinct.
