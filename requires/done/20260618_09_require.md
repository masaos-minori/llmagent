# [mcp][config][startup] Add startup checks for high-impact deny-all configurations

## Summary
Some empty allowlists or flags effectively disable whole MCP servers, but this may be accidental.

## Problem
It is hard to tell whether a deny-all state is intentional or caused by missing configuration.

## Scope
- startup validation
- MCP config checks
- operator-facing warnings

## Task
- detect high-impact deny-all configurations at startup
- distinguish intentional lockdown from likely misconfiguration
- improve warning messages for these cases

## Expected Result
Operators can detect dangerous or accidental deny-all configurations early.

## Acceptance Criteria
- [ ] high-impact deny-all settings are checked at startup
- [ ] startup warnings explain likely misconfiguration clearly
- [ ] deny-all states are easier to understand during operations
