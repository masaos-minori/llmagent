## Goal

Define metadata conventions for AI agents to select relevant documents when reading the design documentation set.

## Scope

Create a governance document (`docs/00_governance_06_ai-reading-metadata.md`) that defines:
- Recommended additional metadata fields beyond existing Front Matter fields
- Field definitions with allowed values for each recommended field
- Usage examples showing how to apply the metadata
- Migration policy for gradual adoption across existing documents

## Assumptions

- Existing metadata fields (title, category, tags, related, keywords) should be preserved and extended, not replaced.
- Metadata additions should be optional and gradually adopted during normal document updates.
- The eight recommended fields listed in the issue cover the primary AI selection needs.

## Design

### Existing Metadata Fields

Preserve and extend these existing fields:
- title
- category
- tags
- related
- keywords

### Recommended Additional Fields

Define the following eight fields:

1. **doc_type**: Classification of document purpose
   - Allowed values: overview, specification, guide, governance, reference, operations, note
   - Example: `doc_type: governance`

2. **audience**: Intended reader profile
   - Allowed values: developer, operator, architect, reviewer, ai-agent
   - Example: `audience: ai-agent`

3. **ai_use**: How an AI agent should use this document
   - Allowed values: routing, context, decision, verification, none
   - Example: `ai_use: routing`

4. **canonical**: Whether this is the canonical source for its topic
   - Allowed values: true, false, partial
   - Example: `canonical: true`

5. **stability**: How stable the document's content is expected to be
   - Allowed values: stable, evolving, experimental
   - Example: `stability: stable`

6. **last_reviewed**: Date of last human review
   - Format: YYYY-MM-DD
   - Example: `last_reviewed: 2026-07-22`

7. **supersedes**: Documents this version replaces
   - Format: list of relative paths
   - Example: `supersedes: ["05_agent_01_system-overview.md"]`

8. **superseded_by**: Document that supersedes this one
   - Format: relative path or null if none
   - Example: `superseded_by: null`

Show complete Front Matter block with both existing and new fields applied.

### Migration Policy

- No bulk migration of existing documents.
- Add new fields during normal document update cycles.
- Prioritize high-traffic documents first.
- Track adoption progress separately.

### Non-Goals

- Not replacing existing metadata fields.
- Not requiring immediate adoption across all documents.
- Not changing document structure or placement.
