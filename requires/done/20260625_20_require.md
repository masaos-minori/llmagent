## [Medium][eventbus][ops] Add tuning guidance for poll interval, checkpoint interval, and threadpool sizing

### Summary
The current config documents polling and checkpoint settings, but not how they interact with concurrency and thread offload.

### Problem
After moving blocking I/O off the event loop, operators still need guidance for tuning the workload profile.

### Scope
- config.py
- 06_eventbus_05_configuration_deploy_and_operations.md
- deployment notes
- runtime tuning guidance

### Facts
- `poll_interval_ms` defaults to 500 ms. 【6-4e74b8】【6-b6fcb8】
- `offset_checkpoint_interval` defaults to 10. 【6-4e74b8】【6-b6fcb8】
- `/subscribe` polls the DB repeatedly based on `poll_interval_ms`. 【1-2f69b2】【1-6bdda3】

### Task
- document how `poll_interval_ms` affects DB pressure and SSE latency
- document how checkpoint frequency affects file-I/O pressure
- document threadpool sizing assumptions for DB/file offload
- provide recommended starting values for low, medium, and high fan-out cases

### Expected Result
Operations has a usable tuning guide for the new offload-based Event Bus runtime model.

### Acceptance Criteria
- tuning guidance exists for polling, checkpointing, and threadpool capacity
- the docs explain trade-offs clearly
- operators can tune for latency vs I/O pressure explicitly
