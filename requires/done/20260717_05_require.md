# Use RuntimeToolRegistry for LLM tool schema and help output

## Priority

High

## Target files

- agent/llm_turn_runner.py
- shared/llm_client.py
- agent/commands/registry.py
- shared/runtime_tool_registry.py
- tests/agent/test_llm_turn_runner.py
- tests/agent/commands/test_help.py

## Reason

LLM-visible tools are currently taken from static `ctx.cfg.tool.tool_definitions`. This should be replaced by RuntimeToolRegistry output derived from MCP `/v1/tools`.

The `/help` command also reports tool count using static tool definitions, which will become inaccurate after RuntimeToolRegistry is introduced.

## Intent

Make RuntimeToolRegistry the source for LLM-visible tools and user-facing tool counts.

## Scope

Replace:

- `ctx.cfg.tool.tool_definitions`

with:

- `ctx.services_required.runtime_tools.llm_tool_definitions()`

Update `/help` to show RuntimeToolRegistry-derived counts.

Suggested help counts:

- total discovered tools
- LLM-enabled tools

## Acceptance criteria

- LLMTurnRunner no longer passes static tool_definitions to LLMClient.
- LLM receives schemas generated from RuntimeToolRegistry.
- Policy-disabled tools are not exposed to the LLM.
- `/help` no longer uses `len(ctx.cfg.tool.tool_definitions)`.
