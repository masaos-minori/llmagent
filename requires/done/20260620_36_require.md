# Title
Add integration tests for FTS5 fallback when `normalized_content` is NULL

## Summary
Add integration coverage to verify that FTS5 search works correctly for English and code chunks when `normalized_content` is NULL.

## Background
The design relies on `COALESCE(normalized_content, content)` for FTS5 indexing.
For English and code chunks, `normalized_content` is NULL, so the index should fall back to `content`.

## Problem
This behavior is design-intent, but the docs explicitly call out the need to verify it across SQLite versions and environments.

## Required Changes
- Add integration tests that ingest English/code chunks with `normalized_content=NULL`.
- Verify that FTS5 indexing and querying still return expected matches.
- Cover the trigger-based indexing path, not just direct SQL assumptions.
- Document the test intent and why this behavior matters.

## Acceptance Criteria
- Integration tests prove that English/code FTS search works when `normalized_content` is NULL.
- The tests fail if fallback indexing breaks.
- Documentation references the validated behavior.

## Out of Scope
- Changing Japanese normalization logic
- Replacing FTS5

## AI Implementation Instruction
Implement this as an integration-test and correctness-validation task.

Strict rules:
- Test the real trigger-backed indexing path.
- Do not assume SQLite-version behavior without verification.
- Keep tests representative of actual ingestion output.
- If environment-specific SQLite limitations are discovered, mark them as: Needs confirmation

Deliverables:
