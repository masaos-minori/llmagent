# Title
Simplify `web-search-mcp` to DuckDuckGo only

## Summary
Remove Brave and Bing support from `web-search-mcp` and keep DuckDuckGo as the only search provider.

## Background
The current server supports three providers:
- Brave
- Bing
- DuckDuckGo

It tries providers in configured order and falls back when one fails or returns zero results.
The MCP tool surface only exposes one tool: `search_web`.

## Problem
Multiple providers increase complexity:
- extra API key handling
- fallback logic
- more failure paths
- more config and health-check noise

If DuckDuckGo is enough, the server should be simplified.

## Required Changes
- Remove Brave-specific code from `server.py`
- Remove Bing-specific code from `server.py`
- Keep DuckDuckGo as the only provider
- Remove provider fallback loop
- Remove Brave/Bing API key warnings
- Simplify `/health` so it no longer reports Brave/Bing dependency state
- Update config/model handling so it no longer assumes multiple providers
- Update related documentation to match DuckDuckGo-only behavior

## Acceptance Criteria
- Brave and Bing code paths are removed
- DuckDuckGo is the only active provider
- `/search` still works through `search_web`
- `/health` no longer mentions Brave/Bing API keys
- Config and docs match the simplified implementation

## Out of Scope
- Renaming `search_web`
- Replacing DuckDuckGo with another provider
- Refactoring unrelated MCP servers

## Components
- `server.py`
- `models.py`
- `tools.py`
- `04_mcp_04_server_catalog.md` (if provider descriptions must be updated)

## AI Implementation Instruction
Implement this as a provider-simplification task.

Strict rules:
- Do not keep Brave/Bing as commented-out code
- Do not keep API key warnings for removed providers
- Keep the external MCP tool contract (`search_web`) stable
- Update health/config/docs together with the code
- If a config field or doc dependency cannot be removed safely, mark it as: `Needs confirmation`
