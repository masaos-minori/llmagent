# M-1: Fix plugin example import path

## Priority

Medium

## Labels

bug, plugin, docs, developer-experience

## Background

The example plugin currently imports decorators from `plugin_registry`.

At startup, the plugin is skipped with:

`ModuleNotFoundError: No module named 'plugin_registry'`

The current plugin architecture exposes decorators through `shared.plugin_registry`.

## Goal

Make the bundled example plugin load correctly or remain a valid copy-and-edit template.

## Scope

### In scope

- Update `plugins/example.py`
- Use the current plugin registry import path
- Preserve existing example command, tool, and pipeline hook behavior

### Out of scope

- Plugin registry redesign
- Plugin strict mode changes
- Tool conflict policy changes

## Required change

Replace the old import:

`from plugin_registry import register_command, register_pipeline_stage, register_tool`

With:

`from shared.plugin_registry import register_command, register_pipeline_stage, register_tool`

## Acceptance criteria

- `plugins/example.py` no longer fails with `ModuleNotFoundError`
- Startup no longer logs `Plugin load failed (example.py): ModuleNotFoundError: No module named 'plugin_registry'`
- Example remains usable as a template for custom plugins

## Validation

Run:

- `cd scripts`
- `uv run python -m agent`

Expected result:

- Plugin startup does not report `example.py` import failure
- `/plugin status` shows expected plugin load status

## Affected files

- `plugins/example.py`

## Dependencies

None
