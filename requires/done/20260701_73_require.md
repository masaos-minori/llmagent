# [High] Clarify FTS5 rebuild and repair rules for normalized_content

## Goal

Clarify that FTS5 rebuild and repair operations must use the same indexing rule as normal FTS5 triggers: `COALESCE(normalized_content, content)`.

## Background

The docs state that FTS5 triggers index `COALESCE(normalized_content, content)`, so Japanese chunks use normalized Sudachi forms and English/code chunks fall back to `content`. The operations docs also describe `/db rag rebuild-fts`, so rebuild logic must preserve the same rule. 【4-cb374e】【2-58c0bb】【5-63b173】

## Scope

Documentation first.

Implementation changes are required only if the existing rebuild code does not follow this rule.

## Implementation Tasks

- Update `03_rag_05_configuration_and_operations.md`.
- Update `03_rag_90_inconsistencies_and_known_issues.md`.
- Document that `/db rag rebuild-fts` rebuilds `chunks_fts` from canonical `chunks`.
- Document that rebuild must index `COALESCE(normalized_content, content)`.
- Document that Japanese chunks should index `normalized_content` when present.
- Document that English/code chunks should fall back to `content`.
- Document that `chunks_fts` must not be manually edited as canonical data.
- If implementation does not already follow this rule, update the rebuild logic.

## Acceptance Criteria

- FTS5 trigger and FTS5 rebuild rules are documented as identical.
- Rebuild behavior is explained as derived-index repair from canonical `chunks`.
- Docs do not describe `chunks_fts` as an independently maintained source of truth.
- If implementation changes are needed, tests cover the rebuild rule.
