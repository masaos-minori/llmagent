# [mcp][observability] Add server-level monitoring for tool errors

## Summary
Tool-level errors use HTTP 200 with `is_error=true`, which makes monitoring harder.

## Problem
Transport-level failures and tool-level failures are different, but current monitoring can hide this difference.

## Scope
- MCP metrics
- audit aggregation
- per-server error reporting

## Task
- distinguish transport errors from tool errors in monitoring
- add server-level tool error counters
- improve operational visibility for repeated tool failures

## Expected Result
Operators can detect failing tools even when HTTP status remains 200.

## Acceptance Criteria
- [ ] tool errors are monitored separately from transport errors
- [ ] per-server tool error counts are available
- [ ] repeated tool failures are easier to detect
