# [High] Enforce result size limits for MDQ tools

## Priority
High

## Labels
`mcp`, `mdq`, `limits`, `context-compression`, `safety`

## Dependencies
- Redesign MDQ SQLite schema for production indexing

## Background
MCP responses have a global maximum response size. MDQ tools can easily return large Markdown chunks, large grep results, or huge outlines. Production MDQ must enforce server-side result limits before MCP-level truncation occurs.

## Task
Add explicit output limits to MDQ tools.

## Implementation Requirements
Add configurable limits:
- `max_results_limit`
- `max_chars_per_chunk`
- `max_total_result_chars`
- `max_outline_items`
- `max_grep_matches`

Apply limits to:
- `search_docs`
- `get_chunk`
- `outline`
- `grep_docs`

When output is truncated:
- Clearly mark it as truncated.
- Include original size if available.
- Suggest narrowing the query or using `get_chunk`.

## Acceptance Criteria
- No MDQ tool can return unbounded output.
- Large chunks are truncated predictably.
- Large grep result sets are capped.
- Large outlines are capped.
- Tool-level truncation happens before MCP global truncation.
