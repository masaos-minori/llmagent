# H-4: Add tests for CommandDef and handler consistency

## Priority

High

## Labels

test, agent, cli, command-registry, regression

## Background

Recent startup/runtime failures were caused by mismatches between built-in command definitions and handler implementations.

Examples:

- A `CommandDef` referenced `_cmd_mdq` while the parent dispatcher was missing earlier
- `/help` is an exact command, but dispatch passed an unused `str` argument to `_cmd_help()`

These issues should be caught by tests before runtime.

## Goal

Add regression tests that validate built-in command definitions against `CommandRegistry`.

## Scope

### In scope

- Add tests for all built-in `CommandDef` entries
- Validate handler existence
- Validate exact vs prefix handler call shape
- Validate async flag consistency

### Out of scope

- Testing actual MCP server behavior
- Testing `/exit`
- Testing sqlite-vec behavior

## Required test coverage

Add tests that verify:

1. Every `CommandDef.handler` exists on `CommandRegistry`
2. `prefix=False` handlers can be called with no argument
3. `prefix=True` handlers accept one `str` argument
4. `is_async=True` handlers are coroutine functions
5. `is_async=False` handlers are not coroutine functions

## Suggested tests

Use `inspect.signature()` and `inspect.iscoroutinefunction()`.

For bound methods:

- Exact command handlers should have zero required parameters
- Prefix command handlers should accept one required or optional parameter for args

## Acceptance criteria

- Missing handler names fail tests
- Wrong async flags fail tests
- Exact handlers requiring incorrect args fail tests
- Prefix handlers not accepting args fail tests
- Tests run without starting external MCP servers

## Validation

Run:

- `uv run pytest tests/ -k command`

Expected result:

- Command definition tests pass
- No external LLM, embed, or MCP server is required

- `scripts/agent/commands/registry.py`
