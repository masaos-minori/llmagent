# [Medium] Add workflow schema version tracking

## Background

The workflow database schema currently creates the required workflow tables and incremental migrations exist. Explicit workflow schema version tracking is needed for safer upgrades.

## Goal

Add schema version tracking for `workflow.sqlite`.

## Scope

- `db/schema_sql.py`
- `db/create_schema.py`
- Workflow schema validation logic

## Tasks

- [ ] Add a `workflow_schema_version` table.
- [ ] Store the current schema version during workflow schema creation.
- [ ] Update the version when migrations are applied.
- [ ] Add validation logic to compare expected schema version with actual schema version.
- [ ] Include schema version in startup validation output.
- [ ] Include schema version in setup validation output.
- [ ] Document recovery steps for schema version mismatch.

## Suggested Table Shape

- `version TEXT NOT NULL`
- `applied_at TEXT NOT NULL DEFAULT current UTC timestamp`

## Acceptance Criteria

- [ ] New workflow DBs record schema version.
- [ ] Existing DBs can be migrated idempotently.
- [ ] Schema version mismatch is detected.
- [ ] Error message includes remediation.

## Notes

Do not break existing workflow tables or current migrations.
