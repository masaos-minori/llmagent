# Add shared structured log keys across crawl, chunk, ingest, and query paths

## Summary
Improve operational traceability by adding shared structured log keys across the RAG lifecycle.

## Background
Current documentation describes log files and log levels, but not a unified correlation model across the end-to-end RAG flow.

## Objective
Make it easier to trace one URL or one logical document through crawl, chunking, ingestion, and retrieval.

## In Scope
- `scripts/rag/ingestion/crawler.py`
- `scripts/rag/ingestion/chunk_splitter.py`
- `scripts/rag/ingestion/ingester.py`
- `scripts/rag/pipeline.py`
- Related operational documentation

## Required Changes
- Add shared structured fields where applicable, for example:
  - `url`
  - `doc_id`
  - `chunk_id`
  - `source_type`
  - `stage_name`
- Standardize naming across components
- Update documentation with the new logging guidance

## Acceptance Criteria
- A single URL can be correlated across crawl, chunk, ingest, and query logs
- Structured field names are consistent
- Logging changes do not remove current human-readable context

## Out of Scope
- Full observability stack integration
- Log shipping infrastructure

## Implementation Notes
Keep log keys stable and implementation-oriented. Avoid introducing multiple synonymous field names.
