# [shared][typing] Change McpServerConfig.transport to a Literal type

## Summary
`McpServerConfig.transport` is typed as `str`, but only two values are valid.

## Problem
Runtime only accepts:
- `"http"`
- `"stdio"`

But static type checkers cannot enforce this today.

## Scope
- `shared/mcp_config.py`

## Task
- change `transport` type from `str` to `Literal["http", "stdio"]`

## Expected Result
- invalid values can be caught earlier by type checking

## Acceptance Criteria
- [ ] `transport` uses `Literal["http", "stdio"]`
- [ ] runtime validation still works
- [ ] docs match the new type
