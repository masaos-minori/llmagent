# Title
Prevent stale ETag / Last-Modified overwrite during ingester skip paths

## Summary
Protect `documents.etag` and `documents.last_modified` from being overwritten by stale metadata when ingestion skips an already-registered URL.

## Background
The current design intentionally updates ETag / Last-Modified on skip paths so future crawls can benefit from Conditional GET.
However, the docs explicitly call out the risk that stale chunk-file metadata could overwrite newer values.

## Problem
If stale metadata is written back during skip behavior, Conditional GET optimization may become incorrect or less reliable over time.

## Required Changes
- Add freshness validation before updating `documents.etag` and `documents.last_modified` on skip paths.
- Ensure newer metadata is not overwritten by older metadata from stale chunk files.
- Define a deterministic precedence rule, for example based on crawl freshness or fetch timestamp.
- Update documentation to explain the guarded skip-path behavior clearly.

## Acceptance Criteria
- Skip-path metadata updates cannot blindly overwrite newer values.
- Conditional GET support remains intact.
- The protection rule is documented and testable.

## Out of Scope
- Removing ETag/Last-Modified propagation entirely
- Replacing Conditional GET design

## AI Implementation Instruction
Implement this as a metadata-correctness and ingestion-safety task.

Strict rules:
- Do not remove skip-path metadata updates unless another issue explicitly changes the design.
- Preserve Conditional GET optimization.
- Add a guard so stale metadata cannot overwrite newer values.
- Make the precedence rule explicit and documented.
- If the correct freshness comparison signal is unclear, mark it as: Needs confirmation

Deliverables:
1. Guarded metadata update logic
2. Tests or validation for stale-overwrite prevention
3. Updated docs for skip-path semantics
