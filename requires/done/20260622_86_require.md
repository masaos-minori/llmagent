# Title
Resolve SemanticCache eviction-policy inconsistency and align implementation

## Summary
Determine the real SemanticCache eviction policy, align implementation and documentation to that policy, and make the policy explicit.

## Background
The docs describe SemanticCache inconsistently.
One place refers to it as LRU, while another describes pruning oldest entries, which is FIFO-like behavior.

## Problem
Cache policy affects hit behavior, performance expectations, and tuning decisions.
Inconsistent descriptions create implementation and operational ambiguity.

## Required Changes
- Verify the actual current eviction behavior in code.
- Decide whether the intended policy is FIFO or LRU.
- Align implementation and documentation to one policy.
- Expose the policy clearly in docs and, if useful, diagnostics.

## Acceptance Criteria
- The eviction policy is unambiguous across code and docs.
- Runtime behavior matches the documented policy.
- Operators can reason correctly about cache behavior and tuning.

## Out of Scope
- Replacing the semantic cache architecture entirely
- Changing cache-hit semantics unrelated to eviction policy
