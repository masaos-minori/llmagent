# Require: Add exception logging for background tasks in first turn

## Priority

Medium

## Target files

- `scripts/agent/orchestrator.py`

## Background

During the first user message processing, a background task is created for operations like session title generation (`_on_first_turn`). The task is added to `self._background_tasks` with a `done_callback` that only discards the completed task from the set.

## Problem

When the background task raises an exception (e.g., `sqlite3.Error`, `RuntimeError`), asyncio's default behavior logs the unhandled exception only in `Task.__del__()`, which occurs when the event loop ends. During normal operation, the exception is never logged, so operators cannot tell why the background operation failed. The task is simply removed from the set without any error reporting.

The scenario:
1. First user message sent
2. `_on_first_turn` starts a background task for session title generation
3. Task raises `sqlite3.Error` or `RuntimeError` during execution
4. Exception is only logged at event loop termination time
5. Operator notices the title was not generated but has no idea why

## Reason for change

Operators need immediate visibility into background task failures. Silent swallowing of exceptions makes debugging impossible and can hide data corruption issues.

## Implementation intent

Add a `done_callback` that catches exceptions from background tasks and logs them immediately, preventing silent failure.

## Implementation instructions

In `scripts/agent/orchestrator.py`, around line 543-546, replace:

```python
if ctx.stats.stat_turns == 1 and self._on_first_turn is not None:
    _task = asyncio.create_task(self._on_first_turn(line))
    self._background_tasks.add(_task)
    _task.add_done_callback(self._background_tasks.discard)
```

with:

```python
def _discard_and_log(task: asyncio.Task) -> None:
    """Discard completed task from set and log any exception."""
    self._background_tasks.discard(task)
    exc = task.exception()
    if exc is not None:
        logger.error("Background task failed: %s", exc)

if ctx.stats.stat_turns == 1 and self._on_first_turn is not None:
    _task = asyncio.create_task(self._on_first_turn(line))
    self._background_tasks.add(_task)
    _task.add_done_callback(_discard_and_log)
```

Key points:
1. Define `_discard_and_log` as a method on `Orchestrator` (not a standalone function) so it has access to `self._background_tasks` and `self.logger`
2. Use `task.exception()` to retrieve the exception — this must be called inside the callback, not before the task completes
3. Log at ERROR level since unexpected failures should be visible to operators

## Acceptance criteria

- When a background task succeeds, behavior is unchanged (no additional logging)
- When a background task fails, an ERROR-level log is emitted immediately with the exception details
- The task is still properly discarded from `self._background_tasks` after completion
- No regression in session title generation under normal operation

## Tests

- Unit test: Mock `_on_first_turn` to raise an exception, verify ERROR log is emitted
- Unit test: Mock `_on_first_turn` to succeed, verify no additional log beyond normal operation
- Integration test: Simulate a database error during first-turn background task, verify error is logged
