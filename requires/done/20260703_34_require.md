# Add tests for HTTP-only ToolTransportInvoker

## Priority

Medium

## Goal

Add tests for the new transport invocation layer.

## Scope

Test file:

- `tests/shared/test_tool_transport_invoker.py`

## Test cases

- Health unavailable server returns error result and does not call transport.
- HALF_OPEN server allows trial dispatch.
- `lifecycle.ensure_ready(server_key)` is called before transport invocation.
- Semaphore limits concurrent execution when configured.
- Successful `HttpTransport.call()` records health success.
- `result.is_error=True and error_type="tool"` increments `stat_tool_errors`.
- `TransportError` increments `stat_transport_errors`.
- `TransportError` records health failure.
- `TransportError` returns `ToolCallResult(error_type="transport")`.
- Missing transport returns tool error result.
- Unknown concurrency limit keys are handled consistently with current behavior.

## Acceptance criteria

- Tests do not require real HTTP servers.
- Tests use fake or mocked `HttpTransport`.
- Tests verify stats are owned by `ToolTransportInvoker`.
- Tests verify `ToolExecutor` can expose stats through properties.
- No stdio transport behavior is introduced.

## Out of scope

- Testing actual network calls.
- Testing stdio transport.
