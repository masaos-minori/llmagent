# Add a validation-focused checklist for new MCP server and tool registration

## Summary
Strengthen the manual registration procedure for new MCP servers and tools by adding an explicit validation checklist that detects missed update points.

## Background
Adding a new server or tool requires updating multiple files across routing, config, deployment, and documentation. This is structurally error-prone.

## Objective
Make manual MCP extension work safer and more repeatable.

## In Scope
- `04_mcp_03_routing_lifecycle_and_execution.md`
- `04_mcp_06_configuration_and_operations.md`
- Wizard / generator guidance where documented

## Required Changes
- Add a validation checklist for:
  - `shared/tool_constants.py`
  - `config/agent.toml`
  - `config/tools_definitions.toml`
  - server config file
  - deployment scripts
  - startup verification
- Distinguish required updates from optional or generated updates
- Ensure the manual path and the wizard path do not contradict each other
- Add a final verification step with expected outcomes

## Acceptance Criteria
- A manual server/tool addition can be validated without guessing
- The documentation clearly states which artifacts must always be updated
- The wizard path and manual path describe the same final system state

## Out of Scope
- Building a new code generator
- Redesigning MCP deployment

## Notes
Optimize for preventing omissions by AI agents and human engineers.
