## [Medium][eventbus][io] Offload JSONL archive writes, offset writes, and DLQ file writes from the event loop

### Summary
The current runtime blocks not only on SQLite, but also on synchronous file I/O for archive, offsets, and DLQ files.

### Problem
Even after DB offload, synchronous file I/O can still block the event loop and keep SSE latency high.

### Scope
- _append_jsonl() in app.py
- offsets.py
- dlq.py
- file-write paths used by publish/subscribe/DLQ

### Facts
- `_append_jsonl()` performs synchronous append, flush, and `os.fsync()`. 【1-6bdda3】
- `write_offset()` uses synchronous `Path.write_text()`. 【3-d430e3】
- `_atomic_write()` in DLQ logic uses synchronous temp-file write and `os.replace()`. 【1-835298】

### Task
- move archive writes, offset writes, and DLQ file writes behind offload helpers
- ensure publish path does not perform blocking file I/O on the event loop
- ensure subscribe checkpoints do not block streaming delivery
- ensure DLQ file writes do not block API handling
- add regression tests for file-I/O offload paths

### Expected Result
All heavy or durability-sensitive file I/O is offloaded from the event loop.

### Acceptance Criteria
- JSONL append is offloaded
- offset reads/writes are offloaded where needed
- DLQ file writes are offloaded
- file-I/O blocking no longer remains on the event loop path
