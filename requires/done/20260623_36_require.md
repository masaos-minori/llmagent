# Title
Remove `formatter.py` default singleton behavior and migrate callers to explicit output injection

## Summary
Delete the module-level default output instance pattern in `formatter.py` and update callers to use explicitly injected output ports.

## Background
The refactoring plan identifies `_default_out: CliOutputPort()` as hidden global state that creates testing coupling.
The wrapper functions in `formatter.py` are also described as duplicative.

## Problem
Implicit global output state makes unit tests harder and hides dependencies between command code and output behavior.

## Required Changes
- Remove the module-level default instance pattern.
- Delete `formatter.py` if the wrapper layer has no remaining value.
- Update callers to use explicit `OutputPort` injection or direct output methods.
- Keep behavior stable while removing implicit global state.

## Acceptance Criteria
- No module-level default `CliOutputPort` instance remains.
- Command code does not depend on hidden formatter globals.
- Tests no longer require monkey-patching a shared default output object.

## Out of Scope
- Redesigning the output port abstraction
- Changing output formatting semantics beyond dependency cleanup

## Components
- `scripts/agent/commands/formatter.py`
- All callers using `formatter.print_success`, `formatter.print_error`, or similar wrappers

## AI Implementation Instruction
Implement this as a hidden-global-state removal task.

Strict rules:
- Do not preserve implicit default output state.
- Prefer explicit dependency injection.
- Do not over-engineer the replacement path.
- If a caller lacks an obvious injected output object, mark it as: `Needs confirmation`
