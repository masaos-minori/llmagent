# Agent Startup Processing — Background Task Reference Lost After First Turn

## Summary

The background task started at startup (`_start_background_task`) is stored in `ctx.background_task`, but after the first turn completes, the task reference may be lost if the task is awaited and garbage collected without being replaced. The `_start_background_task` method creates a task but doesn't guarantee it persists beyond the first turn.

## Severity

Low

## Files

- `scripts/agent/orchestrator.py:543-557`

## Details

In `Orchestrator._start_background_task()`:

```python
async def _start_background_task(self, ctx: AgentContext) -> None:
    if ctx.background_task is not None:
        return
    loop = asyncio.get_event_loop()
    ctx.background_task = loop.create_task(
        self._run_background_tasks(ctx),
        name="agent-background-tasks",
    )
```

The task is created once and stored in `ctx.background_task`. However, `_run_background_tasks()` runs until the event loop stops:

```python
async def _run_background_tasks(self, ctx: AgentContext) -> None:
    while True:
        await asyncio.sleep(1)
        await self._check_health(ctx)
        await self._flush_memories(ctx)
```

The issue is not with the background task itself (it runs indefinitely), but with the fact that there's no mechanism to detect if the background task exits unexpectedly. If `_run_background_tasks()` raises an unhandled exception, the task terminates silently:

```python
except Exception as exc:
    logger.error("Background task failed: %s", exc)
```

But this exception is caught inside the task's coroutine, so no one observes the termination. The `ctx.background_task` reference becomes stale — it points to a completed task whose result is `None` (exception swallowed). Subsequent code that checks `if ctx.background_task is not None:` will assume the background task is running, but it isn't.

## Reproduction Steps

1. Make `_flush_memories()` raise an exception (e.g., break the memory backend connection)
2. Wait for the background task to fail
3. Check `ctx.background_task.done()` — it will be True
4. Observe that no alert is raised about the background task failure

## Expected Behavior

If the background task fails, the system should either:
1. Restart the background task automatically, or
2. Raise an alert that the background task has stopped

## Proposed Fix

After each turn, verify the background task is still alive:

```python
async def _check_background_task(self, ctx: AgentContext) -> None:
    if ctx.background_task is not None and ctx.background_task.done():
        exc = ctx.background_task.exception(timeout=0)
        if exc is not None:
            logger.critical("Background task terminated with exception: %s", exc)
            ctx.background_task = None  # Force restart on next turn
```

Call this method in `AgentREPL._close_resources()` or at the start of each turn.
