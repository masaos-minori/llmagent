# [High] Unify SQLite timestamp defaults to ISO-8601 UTC Z suffix

## Goal

Unify SQLite timestamp defaults across Shared/DB schemas.

## Background

`db/schema_sql.py` currently mixes `datetime('now')` and `strftime('%Y-%m-%dT%H:%M:%SZ', 'now')`. The cleanup policy is to stop preserving existing timestamp format differences and use ISO-8601 UTC Z suffix consistently. 【3-6761a2】

## Scope

- SQLite DDL defaults
- Documentation
- Schema tests

This issue does not implement Event Bus runtime timestamp writers.

## Implementation Tasks

- Update `db/schema_sql.py`.
- Replace `datetime('now')` defaults with:

  strftime('%Y-%m-%dT%H:%M:%SZ', 'now')

- Update at least these columns:
  - `documents.fetched_at`
  - `sessions.created_at`
  - `messages.created_at`
  - `memories.created_at`
  - `memories.updated_at`
- Ensure existing ISO-8601 UTC Z defaults remain unchanged:
  - `tool_results.created_at`
  - `session_diagnostics.created_at`
- Apply the same timestamp format policy to Event Bus DDL.
- If `events.published_at` receives a DEFAULT, use ISO-8601 UTC Z suffix.
- Document that nullable Event Bus timestamps such as `acked_at` and `dlq_at` must use ISO-8601 UTC Z suffix when written by future runtime code.

## Acceptance Criteria

- New schema DDL no longer uses `datetime('now')`.
- SQLite DEFAULT timestamps use ISO-8601 UTC Z suffix.
- Docs no longer say existing timestamp formats are preserved.
- Tests assert the expected timestamp default expressions.

## Validation

- Run:

  rg -n "datetime\\('now'\\)" db docs tests

- Confirm no active schema defaults use `datetime('now')`.
- Run schema creation tests.
- Inspect generated table definitions with:

  sqlite3 /opt/llm/db/session.sqlite ".schema messages"
