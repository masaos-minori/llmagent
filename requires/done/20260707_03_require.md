# Remove the workflow_mode="disabled" direct execution path

## Priority

High

## Type

Breaking Change / Safety / Workflow

## Background

The legacy disabled mode allows direct LLM/tool execution without the WorkflowEngine.

The target architecture requires every turn to be represented as a workflow task with stage tracking, idempotency, approval, artifacts, and audit correlation.

## Problem

The disabled direct path bypasses workflow persistence and safety controls.

It can execute LLM and tool logic without:

- `tasks`
- `attempts`
- `processed_events`
- `artifacts`
- `approvals`
- workflow audit context

## Goal

Remove all direct execution behavior associated with disabled workflow mode.

## Scope

Affected areas:

- Orchestrator turn handling
- Workflow execution policy
- LLM turn processing
- Tests for direct execution
- Logs and user-facing messages

## Implementation Tasks

- Remove any branch equivalent to `not is_workflow_enabled()`.
- Remove `is_workflow_enabled()`.
- Remove direct calls to `_process_turn()` outside a workflow stage.
- Remove logs such as `Workflow mode=disabled`.
- Remove direct execution tests.
- Ensure LLM/tool execution occurs only inside WorkflowEngine stage callbacks.
- Ensure workflow unavailable state fails fast instead of executing directly.

## Acceptance Criteria

- There is no workflow-disabled branch in production code.
- No user turn can execute without a workflow task.
- Direct `_process_turn()` entrypoints are removed or made workflow-context-only.
- Tests confirm tool execution is not called when workflow creation fails.
- Logs no longer mention disabled workflow mode.

## Validation Plan

Add tests for:

- Workflow unavailable: no LLM call.
- Workflow unavailable: no tool call.
- Missing workflow definition: startup failure.
- Missing workflow state store: startup failure.
- Static scan finds no disabled-mode branches.

## Risks

Direct execution tests and local workflows must be rewritten around workflow stages.

## Out of Scope

- Maintaining a hidden direct-execution flag.
- Compatibility fallback.
- Documentation-only updates.
