# [High] Improve watchdog health probe handling for malformed and degraded responses

## Background

The watchdog probes MCP server `/health` endpoints and updates shared HealthRegistry state. Malformed health responses and degraded health bodies should be distinguishable from unreachable servers and normal HTTP errors.

Current concerns:

- Malformed health JSON can be collapsed into an empty body.
- Parse failures lose diagnostic information.
- Logs do not clearly separate unreachable, non-200, malformed JSON, and degraded health body cases.
- Degraded responses with `restart_recommended=false` should preserve useful reason information.

## Goal

Make watchdog health probe handling explicit and diagnosable.

## Scope

- `agent/repl_health.py`
- Health probe result model
- Watchdog logs
- Watchdog tests

## Tasks

- [ ] Add an explicit parse failure indicator to the health probe result.
- [ ] Preserve malformed health response details in logs.
- [ ] Record `malformed_health_response` as degraded reason when applicable.
- [ ] Decide whether malformed health responses should be treated as operator-action-required.
- [ ] Log HTTP unreachable separately.
- [ ] Log HTTP non-200 separately.
- [ ] Log JSON parse failure separately.
- [ ] Log degraded body with `restart_recommended=false` separately.
- [ ] Log degraded body with `restart_recommended=true` separately.
- [ ] Include server key and URL in each watchdog health failure log message.
- [ ] Add tests for invalid JSON returned by `/health`.
- [ ] Add tests for degraded health body with and without restart recommendation.

## Acceptance Criteria

- [ ] Malformed `/health` responses are not silently converted to an empty body.
- [ ] Operators can identify the exact health failure class from logs.
- [ ] HealthRegistry receives a useful degraded reason.
- [ ] Existing restart behavior remains unchanged unless explicitly modified.
- [ ] Watchdog behavior is covered by tests.

## Notes

Do not restart solely because of malformed JSON unless the design explicitly chooses that behavior.
