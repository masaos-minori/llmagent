# Reclassify `use_rrf=false` as an experimental or diagnostic-only setting

## Summary
Reduce accidental misuse of `use_rrf=false` by treating it as a non-default experimental setting with explicit warnings.

## Background
The documentation states that disabling RRF removes meaningful fusion ranking benefits, especially for MQE-generated query variants.

## Objective
Prevent operators from enabling a documented quality-degrading mode without clear intent.

## In Scope
- `config/agent.toml`
- `scripts/rag/pipeline.py`
- `03_rag_03_query_pipeline.md`
- `03_rag_05_configuration_and_operations.md`

## Required Changes
- Mark `use_rrf=false` as experimental, diagnostic, or non-recommended
- Emit a clear warning when it is enabled
- Update documentation to explain the expected quality trade-off in operator-facing language

## Acceptance Criteria
- Enabling `use_rrf=false` produces an explicit warning
- Documentation treats it as a special-purpose mode, not a neutral alternative
- The warning message references retrieval-quality impact

## Out of Scope
- Removing the setting entirely
- Changing the underlying dedup-only behavior

## Implementation Notes
Prefer clarity over flexibility. Operators should understand that this mode sacrifices ranking quality.
