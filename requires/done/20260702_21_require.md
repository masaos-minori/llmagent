# [Low] Add docs consistency check for RAG CLI option drift

## Priority
Low

## Background
The RAG docs drifted from `crawler.py` argparse options. Docs mentioned `--targets-file` and crawler `--force`, while current `crawler.py` only supports `--url` and `--lang`.

Evidence:
- Current `crawler.py` argparse defines `--url` and `--lang`.
- `03_rag_05` contains stale or not-yet-implemented crawler option examples.
- `check_docs_consistency.py` already performs RAG docs checks.

## Goal
Detect CLI option drift between RAG docs and argparse definitions.

## Scope
Affected files:
- `scripts/checks/check_docs_consistency.py`
- `scripts/rag/ingestion/crawler.py`
- `docs/03_rag_*.md`

## Implementation Tasks
- Add a lightweight check for `crawler.py` CLI options referenced in docs.
- Compare referenced options against argparse-defined options.
- Allow documented planned options only if marked as `planned` or `not implemented`.
- Initially cover only `crawler.py`.
- Later extend to `chunk_splitter.py` and `ingester.py`.

## Open Questions / Unknowns
- Need to decide whether to parse argparse dynamically or use a static allowlist.
- Need to decide how to handle planned options like `--targets-file` before implementation lands.
- Need CI integration status for `check_docs_consistency.py`.

## Acceptance Criteria
- Docs cannot claim crawler `--force` is supported.
- Docs cannot reference new options without marking them as implemented or planned.
- The checker catches future CLI/docs drift.

## Validation
- Run `python scripts/checks/check_docs_consistency.py docs/03_rag_*.md`.
