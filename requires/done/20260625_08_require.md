## [High][security][github] Make github-mcp allowed_workflows fail-closed or remove it if it is not implemented

### Summary
`github-mcp.allowed_workflows` is documented as fail-open when empty.

### Problem
Workflow-related authorization should not default to unrestricted access.
If this setting is implemented, it should be fail-closed.
If it is not implemented, it should be removed from docs and config.

### Scope
- mcp/github/*
- config/github_mcp_server.toml
- security enforcement
- github security documentation

### Task
- verify whether `allowed_workflows` is implemented in github-mcp
- if implemented, change empty behavior to deny-all
- if not implemented, remove it from documentation and config references
- align server catalog, security model, and live config schema
- add tests for the final behavior

### Expected Result
Workflow-related access control in github-mcp is explicit, implemented, and fail-closed by default.

### Acceptance Criteria
- `allowed_workflows` has one clear ownership and one clear behavior
- empty workflow authorization no longer grants unrestricted access
- docs and implementation match
- tests cover the final behavior
