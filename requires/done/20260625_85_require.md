## [Medium][rag][repo] Remove retained legacy RAG source docs from normal search and reference pathways if they are kept for history only

### Summary
The current guide says old source files are retained for historical reference, but the new restructured docs are the primary reference. 【3-64059d】

### Problem
Even if old files remain in the repository, they should not continue to behave like active design references.
Otherwise, backward-compatible documentation drift remains even after code cleanup.

### Scope
- repository layout
- documentation indexing
- AI routing guidance
- developer navigation paths

### Task
- remove retained legacy source docs from active reference indexes and guidance
- mark them clearly as archive/historical material if they remain in the repository
- prevent normal developer and AI routing flows from preferring or surfacing them as current specs
- update documentation guidance to reflect the new reference boundary

### Expected Result
Retained historical docs no longer participate in normal “current spec” discovery flows. 【3-64059d】

### Acceptance Criteria
- legacy docs are no longer part of active reference routing
- archive/historical status is visually explicit
- search and guidance bias toward the restructured docs only
