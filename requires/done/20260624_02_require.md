# Standardize the RAG pipeline config file name across documentation

## Summary
Replace inconsistent config file naming and make `config/rag_pipeline.toml` the single documented name everywhere.

## Background
The documentation mostly refers to `config/rag_pipeline.toml`, but at least one section uses a different config file name. This creates operator confusion and weakens documentation reliability.

## Objective
Make `config/rag_pipeline.toml` the only documented config file name for RAG crawling and chunking behavior.

## In Scope
- `03_rag_05_configuration_and_operations.md`
- Any linked RAG document that refers to crawl / chunk / ingest configuration
- Cross-references in the RAG document set

## Required Changes
- Find all references to outdated or inconsistent config file names
- Replace them with `config/rag_pipeline.toml`
- Verify that related prose, examples, and command snippets remain correct
- Update any internal links or wording that imply another canonical config file

## Acceptance Criteria
- All RAG documentation uses `config/rag_pipeline.toml` consistently
- No alternate config file name remains in the RAG document set
- Examples and operational steps remain syntactically coherent after replacement

## Out of Scope
- Changes to actual config loader implementation
- Changes to non-RAG config files such as `config/agent.toml` or `config/common.toml`

## Implementation Notes
Keep the change minimal and documentation-focused. Do not rename real files in the repository unless a separate implementation task explicitly requires it.
