## [High][eventbus][publish] Add post-commit live notification from publish() to active subscribers

### Summary
The current publish path stores the event durably, but it does not notify live subscribers directly.

### Problem
Without a post-commit notification step, subscribers only see new events on their next poll cycle.

### Scope
- app.py
- publish()
- durable commit path
- notify-after-commit flow

### Task
- modify `publish()` so that it emits an in-memory notification after SQLite commit succeeds
- include enough metadata for subscribers to receive the event immediately
- ensure notification happens only after durable DB persistence is confirmed
- keep duplicate publish behavior consistent with the existing idempotent insert semantics
- add tests for post-commit notify behavior

### Expected Result
A successfully published event is immediately visible to active subscribers.

### Acceptance Criteria
- publish triggers an in-memory notify after durable commit
- subscribers can consume the event without waiting for a poll cycle
- duplicate publish behavior remains correct
- tests verify notify-after-commit ordering
