## Required Wording

Routing:

- `ToolRegistry` is the routing authority.
- live `/v1/tools` is validation input only.
- config `tool_names` is drift-validation metadata only.

Transport:

- HTTP is the active transport unless stdio is explicitly formalized.
- `HttpTransport` raises `TransportError` for transport failures.
- `ToolExecutor` maps transport failures into `ToolCallResult(error_type="transport")`.

Health:

- Health response separates liveness, readiness, restart recommendation, and operator action.
- Watchdog does not restart on dependency/config failures unless restart is recommended.

Audit:

- Shared audit log is JSON-lines.
- Use source/event fields to distinguish MCP server and agent events.

## Acceptance Criteria

- Docs use one consistent routing authority model.
- Docs do not describe live discovery as routing override.
- Docs do not describe stale stdio/ondemand behavior unless supported.
- Docs do not describe key=value audit as the active shared audit format.
- Known issues no longer contain stale resolved contradictions.
