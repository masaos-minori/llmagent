# [mcp][ops][watchdog] Show watchdog state in operational status output

## Summary
Watchdog behavior depends on profile and config, but it is easy to miss whether it is active.

## Problem
In local mode, watchdog is disabled by default.
Operators may not notice that failed MCP subprocesses will not be auto-restarted.

## Scope
- `/mcp`
- startup logs
- watchdog operational visibility

## Task
- show whether watchdog is enabled or disabled
- show watchdog interval and restart policy
- make local vs production behavior easier to notice

## Expected Result
Operators can quickly see whether MCP auto-recovery is active.

## Acceptance Criteria
- [ ] watchdog enabled/disabled state is visible
- [ ] watchdog interval is visible
- [ ] startup and status output match real watchdog behavior
