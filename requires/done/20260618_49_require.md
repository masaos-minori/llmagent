# [rag][query][fusion] Expose use_rrf and rrf_k more clearly at runtime

## Summary
`use_rrf` and `rrf_k` are meaningful, but their active runtime mode is still easy to miss.

## Problem
It is easy to misunderstand:
- whether RRF is active
- whether dedup fallback is active
- what `rrf_k` value is currently used

## Scope
- runtime status
- debug output
- fusion-stage visibility
- docs and logs

## Task
- expose the current `use_rrf` mode
- expose the current `rrf_k` value
- make fusion mode easier to see in logs or debug output

## Expected Result
The active fusion behavior is easier to inspect and tune.

## Acceptance Criteria
- [ ] current `use_rrf` mode is visible
- [ ] current `rrf_k` value is visible
- [ ] fusion behavior is easier to understand at runtime
