## [High][plugin][validation] Enforce plugin tool contract validation at registration time

### Summary
Plugin tools are weakly validated at registration time and fully validated only at first execution.

### Problem
Invalid plugin tools can load successfully and fail later at runtime, including tools that shadow MCP tools.

### Scope
- shared/plugin_registry.py
- shared/tool_executor.py
- plugin configuration
- plugin extension documentation

### Task
- validate plugin tool signature and return contract at registration time
- reject invalid plugin tools during startup
- keep conflict/shadow policy strict by default
- emit explicit startup diagnostics for rejected plugin tools
- update extension-point docs accordingly

### Expected Result
Invalid plugin tools fail fast at startup instead of failing only on first runtime call.

### Acceptance Criteria
- invalid plugin tool contracts are rejected during registration
- startup logs explain why a plugin tool was rejected
- runtime no longer discovers basic plugin interface errors for the first time
- plugin validation behavior is documented consistently
