# [Medium] Document RAG MCP service direct DB access as an internal owner operation

## Priority
Medium

## Difficulty
Low

## Background
`RagPipelineMCPService` directly accesses `rag.sqlite` through `SQLiteHelper("rag")` for `list_documents()` and `delete_document()`.

This is valid if treated as an internal operation of the RAG MCP service owner, not as Agent-layer direct DB access.

## Goal
Clarify that direct `rag.sqlite` operations are only allowed inside the RAG MCP service or explicit RAG maintenance owner boundary.

## Scope
Affected files:

- `scripts/mcp/rag_pipeline/service.py`
- `docs/05_agent_09_data-layer.md`
- `docs/03_rag_01_system_overview.md`
- `docs/03_rag_05_configuration_and_operations.md`

## Implementation Tasks
- Document `list_documents()` and `delete_document()` as internal RAG MCP service operations.
- Clarify that Agent application code must not access `rag.sqlite` directly.
- Explain that Agent commands call MCP tools or approved maintenance services.
- Keep deletion order safe:
  - delete `chunks_vec` rows first
  - delete `documents`
  - rely on cascade or triggers for derived records where applicable

## Acceptance Criteria
- Documentation clearly distinguishes:
  - Agent normal path: MCP tool call
  - RAG MCP internal path: allowed direct `rag.sqlite` access
  - Maintenance exception path: explicit maintenance service only
- No documentation suggests `AgentSession` owns RAG document access.
- RAG document deletion behavior remains unchanged.

## Validation
- Review docs for:
  - `direct DB access`
  - `rag.sqlite`
  - `AgentSession`
  - `rag-pipeline-mcp`
