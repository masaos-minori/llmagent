# [mcp][health] Implement real health state transitions in McpServerHealthRegistry

## Summary
The health registry exists, but it does not change state in real execution.

## Problem
`ToolExecutor` checks `is_unavailable()`, but it does not record success or failure correctly.
Because of this, servers never move from HEALTHY to DEGRADED or UNAVAILABLE.

## Scope
- `shared/tool_executor.py`
- `shared/mcp_config.py`

## Task
- call `record_failure(server_key)` after transport errors
- call `record_success(server_key)` after successful tool responses
- make sure degraded and unavailable states can happen in practice

## Expected Result
Failing MCP servers can be detected and isolated automatically.

## Acceptance Criteria
- [ ] transport failures update server health state
- [ ] successful calls reset or recover health state
- [ ] unavailable servers can be blocked by real state transitions
