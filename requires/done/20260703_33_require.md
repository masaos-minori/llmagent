# Add tests for ToolResultCache and ToolExecutor stampede protection

## Priority

Medium

## Goal

Add regression tests for cache behavior and stampede protection boundaries.

## Scope

Test files:

- `tests/shared/test_tool_result_cache.py`
- `tests/shared/test_tool_executor_stampede.py`

## ToolResultCache test cases

- Cache key is `tool_name` plus serialized sorted JSON args.
- Cache hit returns `ToolCallResult` with `request_id=""`.
- Cache hit returns `server_key=""`.
- Expired entry is removed.
- Cache hit moves entry to LRU tail.
- Max-size overflow evicts the oldest entry.
- `is_error=True` result is not stored.
- `clear()` removes all entries.
- `apply_config()` updates TTL and max size.

## Stampede protection test cases

- Same cache key concurrent miss calls `_raw_execute()` once.
- Successful execution result is shared by all callers.
- `_raw_execute()` exception is propagated to all callers.
- No caller hangs when `_raw_execute()` raises.
- `_inflight` entry is cleaned after success.
- `_inflight` entry is cleaned after exception.
- Failed `ToolCallResult` is not cached.

## Acceptance criteria

- Cache tests do not depend on real time without controlling time or using short deterministic time intervals.
- Stampede tests use async test utilities.
- Stampede protection remains owned by `ToolExecutor`.
- `ToolResultCache` does not own in-flight futures.

## Out of scope

- Persistent cache.
- Moving stampede protection into cache.
