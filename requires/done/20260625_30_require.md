## [Medium][eventbus][dlq] Reduce the periodic DLQ loop to recovery and sweep duties only

### Summary
The current DLQ loop is responsible for the primary promotion path, which creates delayed failure isolation.

### Problem
A periodic sweep is useful for recovery and consistency checks, but it is too slow to be the primary path for terminal-failure isolation.

### Scope
- _dlq_loop() in app.py
- dlq.py
- DLQ architecture
- recovery/sweep responsibilities

### Task
- remove threshold-based primary promotion from the periodic loop
- keep the loop only for:
  - orphan recovery
  - timeout sweep
  - consistency repair
- document the new responsibility split between inline promotion and background sweep
- add tests for the new sweep-only behavior

### Expected Result
The background DLQ loop is no longer responsible for first-line terminal-failure quarantine.

### Acceptance Criteria
- primary DLQ promotion happens inline, not in the periodic loop
- the periodic loop has a narrow recovery/sweep purpose
- documentation explains the new role clearly
- tests verify the revised loop responsibilities
