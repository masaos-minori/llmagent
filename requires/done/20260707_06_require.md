# Require workflow context in LLMTurnRunner.run

## Priority

High

## Type

Architecture / Runtime Safety / Workflow

## Background

`LLMTurnRunner.run()` currently accepts only an LLM URL and can execute the LLM streaming and tool-call loop without explicit workflow context.

The workflow-only architecture requires all LLM and tool execution to be associated with a workflow task and stage.

## Problem

If `LLMTurnRunner.run()` can be called without workflow context, direct-path execution can reappear through tests or future call sites.

This bypasses:

- workflow task tracking
- stage attempts
- approval gates
- idempotency
- artifact tracking
- audit correlation

## Goal

Make workflow context mandatory for all LLM/tool loop execution.

## Scope

Affected areas:

- `LLMTurnRunner`
- Orchestrator stage callbacks
- Tool runner integration
- Tests that directly call `LLMTurnRunner.run()`

## Implementation Tasks

- Change `LLMTurnRunner.run()` to require:
  - `workflow_id`
  - `task_id`
  - `stage_id`
  - `attempt_id`
- Remove the legacy `run(llm_url)` signature.
- Raise `RuntimeError` if workflow context is missing.
- Pass workflow context from the execute stage callback.
- Pass workflow context into `_stream_llm()`.
- Pass workflow context into `_handle_llm_error()`.
- Pass workflow context into tool execution.
- Rewrite direct runner tests to use a workflow stage harness.

## Acceptance Criteria

- `LLMTurnRunner.run()` cannot be called without workflow context.
- All production calls provide workflow context.
- Direct tests using old signature fail and are migrated.
- LLM errors include workflow context.
- Tool execution receives workflow context.
- No call site uses `run(llm_url)`.

## Validation Plan

Add tests for:

- Calling `run()` without workflow context raises.
- Calling `run()` with workflow context succeeds.
- Execute stage passes context to runner.
- Tool execution receives workflow context.
- Mid-turn LLM error records workflow context.
- Static scan catches legacy `LLMTurnRunner.run(llm_url)` usage.

## Risks

This requires broad test updates around LLM streaming and tool-loop tests.

## Out of Scope

- Changing LLM streaming protocol.
- Removing tool loop behavior.
- Compatibility wrapper for old signature.
