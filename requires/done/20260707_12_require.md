# Extend LLMTurnRunner._span_ctx for workflow stage context

## Priority

Medium

## Type

Observability / OpenTelemetry / Workflow

## Background

`LLMTurnRunner._span_ctx()` can attach task ID, session ID, and model URL to spans.

Workflow-only execution needs richer span context, including workflow, stage, and attempt identifiers.

## Problem

Task ID alone is insufficient to correlate LLM spans with workflow stages and attempts.

This makes it harder to diagnose which stage and retry attempt produced a given LLM call.

## Goal

Attach complete workflow context to LLM spans.

## Scope

Affected areas:

- `LLMTurnRunner._span_ctx()`
- LLM span creation
- Orchestrator execute stage callback
- Tests for OTel span attributes

## Implementation Tasks

- Add parameters to `_span_ctx()`:
  - `workflow_id`
  - `stage_id`
  - `attempt_id`
- Preserve current parameters:
  - `task_id`
  - `session_id`
  - `model_url`
- Add span attributes:
  - `workflow.workflow_id`
  - `workflow.task_id`
  - `workflow.stage_id`
  - `workflow.attempt_id`
  - `workflow.session_id`
  - `llm.model_url`
- Ensure no-op span path uses the same signature.
- Update LLM span call sites.

## Acceptance Criteria

- LLM spans include workflow ID.
- LLM spans include task ID.
- LLM spans include stage ID.
- LLM spans include attempt ID.
- LLM spans include session ID.
- Existing no-op tracer behavior remains safe.

## Validation Plan

Add tests for:

- Span attributes with real tracer mock.
- No-op tracer path.
- Missing workflow context rejected before span creation.
- Execute stage LLM span includes correct stage ID.
- Retry attempt span includes correct attempt ID.

## Risks

Span attribute names must stay consistent with existing workflow span naming conventions.

## Out of Scope

- Changing tracing backend.
- Changing workflow span names.
- Migration support.
