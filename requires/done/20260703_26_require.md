# Refactor ToolExecutor.__init__ responsibilities after extracting cache and transport components

## Priority

High

## Goal

Reduce `ToolExecutor.__init__()` responsibilities by moving cache and transport initialization into dedicated components.

## Background

`ToolExecutor.__init__()` currently owns too many concerns:

- HTTP client reference
- cache TTL and max-size config
- `OrderedDict` cache initialization
- statistics initialization
- lifecycle reference
- health registry reference
- in-flight future map
- concurrency limit validation
- `HttpTransport` initialization for every server
- route resolver initialization

After extracting `ToolResultCache` and `ToolTransportInvoker`, the constructor should become dependency wiring rather than implementation ownership.

## Scope

Affected files:

- `shared/tool_executor.py`
- `shared/tool_cache.py`
- `shared/tool_transport_invoker.py`
- tests that instantiate `ToolExecutor`

## Implementation requirements

Keep the existing `ToolExecutor.__init__()` public signature in Phase 1.

Internally initialize:

- `PluginToolInvoker`
- `ToolResultCache`
- `ToolRouteResolver` or route service
- `ToolTransportInvoker`
- `_inflight`

Move out of `ToolExecutor`:

- Direct `OrderedDict[str, CacheEntry]` ownership
- Direct `_transports` ownership
- Direct `_concurrency_limits` ownership, if transport invoker owns semaphores
- Direct `_semaphores` ownership, if transport invoker owns semaphores

## Acceptance criteria

- `ToolExecutor.__init__()` signature remains stable.
- `ToolExecutor` no longer directly creates `HttpTransport` instances.
- `ToolExecutor` no longer directly owns `OrderedDict[str, CacheEntry]`.
- `ToolExecutor` still owns `_inflight`.
- Existing callers can instantiate `ToolExecutor` without changes.
- Existing behavior remains unchanged.

## Validation plan

- Run existing ToolExecutor tests.
- Add constructor-level tests if needed.
- Verify unknown concurrency limit warning still occurs in the correct component.
- Verify cache config is applied to `ToolResultCache`.
- Verify transport config is applied to `ToolTransportInvoker`.

## Out of scope

- Public constructor signature redesign.
- Dependency injection framework changes.
