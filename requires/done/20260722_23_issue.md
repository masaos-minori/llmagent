# 正典ソースルール docs/00_governance_02_canonical-source-rule.md を追加する

## 概要

LLM エージェント設計書群における canonical source、文書間矛盾、コードと文書の不一致の扱いを定義する。

## 修正理由

現在の設計書群では、領域ごとに document-guide や Known Issues が存在するが、設計書群全体に共通する正典ソースの扱いが一元化されていない。

特に、同じ仕様が複数の文書に記載される場合、どの文書を優先すべきかが領域ごとに分散している。

また、コードと文書が矛盾した場合に、単純にコードを正とするのではなく、以下のような分類が必要である。

- 文書が古い
- コードが現行仕様である
- コードが設計から逸脱している
- 暫定実装である
- 実装バグである
- 設計意図が未確定である

## 修正意図

この文書の意図は、文書間またはコードと文書の矛盾を発見した際に、場当たり的に修正するのではなく、一定の手順で記録・判断・修正できるようにすることである。

この文書により、以下を実現する。

- 各領域の正典ソースを明確化する
- 文書間矛盾の解決手順を定義する
- コードと文書の不一致を分類する
- Known Issues への登録ルールを明確化する
- 修正後の解決記録を残す

## 対象ファイル

以下のセクションを含む文書を作成する。

## 必須要件

- フラット構成を維持する。
- 各領域の document-guide を正典定義の入口として扱う。
- 「コードが常に正」とは断定しない。
- コードと文書が矛盾する場合の分類を定義する。
- Known Issues への登録ルールを含める。

- 実装コードは変更しないこと。

- Known Issues の既存エントリを変更しない。
- document-guide のリンク追記はこの Issue では行わない。
- EventBus と Workflow の関係整理を追加しない。
