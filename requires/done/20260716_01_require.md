# Remove obsolete and non-functional MDQ configuration compatibility paths from `service.py`

## Priority

High

## Target files

- `mcp_servers/mdq/service.py`
- `config/mdq_mcp_server.toml`
- `docs/04_mcp_*.md`
- `tests/test_mdq_*.py`

## Type

Backward compatibility removal / configuration cleanup

## Background

The MDQ configuration still contains several keys that are either documented as removed, unused, or only partially implemented.

The most important example is `audit_log_path`.

The MDQ config explicitly states that `audit_log_path` was removed because `MdqService.audit_log_path` was parsed but never subsequently read. Audit events are now emitted through `_audit_log()` into the standard `mdq-mcp.log` application logger.

However, `MdqService` still reads `audit_log_path` from configuration and assigns it to `self.audit_log_path`.

This creates a misleading backward compatibility surface. Users may believe that setting `audit_log_path` changes audit behavior, while the runtime audit path is actually controlled elsewhere.

The same file also reads additional compatibility or placeholder settings, including embedding/hybrid search settings and summary cache settings. These should be reviewed and removed unless they are intentionally supported production features.

## Problem

`service.py` currently accepts configuration keys that do not represent reliable production behavior.

This causes several issues:

- Configuration appears to support behavior that is not actually implemented
- Removed keys are still silently accepted
- Operators may rely on settings that have no runtime effect
- Documentation and implementation drift over time
- Future maintainers may mistake placeholder fields for supported features

## Intent

Reduce the MDQ configuration surface to production-supported behavior only.

The intent is not to remove useful functionality. The intent is to remove compatibility paths, placeholders, and misleading settings that do not have complete runtime behavior.

## Tasks

- [ ] Remove `self.audit_log_path` from `mcp_servers/mdq/service.py`
- [ ] Remove any remaining runtime reads of `audit_log_path`
- [ ] Treat `audit_log_path` as a removed or invalid config key
- [ ] Remove `audit_log_path` from config examples
- [ ] Confirm audit events still go through `_audit_log()` and `mdq-mcp.log`
- [ ] Review `summary_cache_enabled`, `summary_threshold`, and `summary_model`
- [ ] Remove summary cache config reads if summary generation remains a stub
- [ ] Review `use_embedding`, `embedding_dims`, `vector_table`, and `embedding_model`
- [ ] Remove embedding config reads if hybrid search remains unsupported
- [ ] Review `max_search_results` and `max_results_limit`
- [ ] Keep only one canonical search result limit setting
- [ ] Review `enable_refresh` and confirm whether it is enforced
- [ ] Remove `enable_refresh` if it is read but not enforced
- [ ] Update MDQ configuration documentation
- [ ] Add tests for removed config key handling
- [ ] Add tests proving removed keys do not silently change runtime behavior

## Acceptance criteria

- `audit_log_path` is no longer read by `MdqService`
- Removed config keys are rejected or explicitly warned about
- MDQ audit behavior remains unchanged
- Unsupported embedding and summary settings are not exposed as working configuration
- Duplicate search limit settings are consolidated
- Runtime config fields match documented production behavior
- Tests cover removed-key handling

## Non-goals

- Do not implement a dedicated audit file writer
- Do not implement hybrid search in this cleanup issue
- Do not implement LLM-based summary generation in this cleanup issue
