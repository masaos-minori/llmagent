# [shared][events] Clarify that ArtifactEvent is data-only, or implement an event bus

## Summary
`ArtifactEvent` exists as a typed structure, but no event bus is implemented.

## Problem
Readers may assume that creating an `ArtifactEvent` means something is delivered somewhere.
Today, it does not.

## Scope
- `shared/events.py`
- docs for shared types and known issues

## Task
Choose one option:

### Option A
Clearly document that `ArtifactEvent` is data-only and has no delivery system.

### Option B
Implement a minimal event bus that can publish and consume `ArtifactEvent`.

## Expected Result
- event behavior is no longer ambiguous

## Acceptance Criteria
- [ ] data-only behavior is clearly documented OR event delivery exists
- [ ] docs and implementation match
