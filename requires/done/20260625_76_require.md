## [Low][repo][docs] Decide whether retained legacy MCP source documents should be archived or removed after compatibility cleanup

### Summary
The MCP document guide states that the restructured documentation is the primary reference, while old source files are retained unchanged. 【2-a4eb18】

### Problem
Even after runtime compatibility cleanup, keeping legacy doc assets unbounded can preserve ambiguity about old vs current interfaces.

### Scope
- legacy MCP source docs
- repository documentation layout
- document-guide policy

### Task
- decide whether legacy source documents should remain, be archived, or be removed
- ensure they are clearly non-canonical if retained
- reduce the chance that future readers mistake retained legacy text for supported runtime behavior
- update the document guide accordingly

### Expected Result
Legacy doc assets are clearly separated from active runtime specification after backward-compatibility cleanup. 【2-a4eb18】

### Acceptance Criteria
- a clear archive/remove policy exists for retained legacy docs
- legacy docs are no longer ambiguous as active runtime reference
- the document guide reflects the final policy
