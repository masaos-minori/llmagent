# [shared][token] Remove module-level global state from token_counter

## Summary
`token_counter` uses a module-level flag to suppress repeated warnings.

## Problem
This global state can leak between tests and between users of the module.

## Scope
- `shared/token_counter.py`

## Task
- remove module-level warning state
- move warning suppression to:
  - instance state
  - caller state
  - or another safer mechanism

## Expected Result
- token counting behavior is easier to test and isolate

## Acceptance Criteria
- [ ] no module-level warning flag remains
- [ ] warning behavior is still controlled
- [ ] tests do not depend on global state
