# 各 document-guide に Governance 参照を追記する

## 概要

各領域の document-guide に、文書統制ルールへの参照を追記する。

## 修正理由

governance 文書を追加しても、各領域の入口である document-guide から参照できなければ、実際の運用で利用されにくい。

RAG / MCP / Agent / EventBus / Shared/DB などの領域別 document-guide から governance 文書へリンクすることで、読者や AI エージェントが各領域の仕様を読む際に、正典ソース、証拠ラベル、Known Issues、廃止項目、AI 読み込みメタデータのルールへ到達できるようにする。

## 修正意図

この修正の意図は、各 document-guide の本文や構成を大きく変更することではなく、短い Governance セクションを追加して、横断統制ルールへの導線を整備することである。

既存の読み順、ファイル索引、関連文書一覧は維持する。

## 対象ファイル

存在する範囲で以下を対象とする。

- `docs/03_rag_00_document-guide.md`
- `docs/04_mcp_00_document-guide.md`
- `docs/05_agent_00_document-guide.md`
- `docs/06_eventbus_00_document-guide.md`
- `docs/90_shared_00_document-guide.md`

存在しないファイルがある場合は、作成せず、報告のみ行う。

## 実施内容

各 document-guide に `Governance` セクションを追加する。

## 必須リンク

以下へのリンクを含める。

- `00_governance_01_documentation-governance.md`
- `00_governance_02_canonical-source-rule.md`
- `00_governance_03_evidence-labels.md`
- `00_governance_04_known-issues-template.md`
- `00_governance_05_deprecated-items.md`
- `00_governance_06_ai-reading-metadata.md`

## AI 実装時の共通指示

- この Issue の対象範囲のみを実施すること。
- 実装コードは変更しないこと。
- 既存ドキュメントの移動、統合、削除、リネームは行わないこと。
- `docs/` 以下に新規サブディレクトリを作成しないこと。
- Markdown の構造を壊さないこと。
- コードブロック、リンク、見出し階層を正しく維持すること。
- 判断に迷う場合は推測で修正せず、未解決事項として報告すること。
- EventBus と Workflow の関係整理は本 Issue では行わないこと。
- ドキュメント分割サイズを変更しないこと。
- 作業後、変更ファイル、変更概要、未解決事項を報告すること。

## 作業禁止事項

- document-guide の構成を大きく変更しない。
- 既存の読み順やファイル索引を削除しない。
- 既存の Known Issues 内容を修正しない。
- EventBus と Workflow の関係整理を追加しない。
- 既存文書を移動しない。

## 検証観点

- 各 document-guide から governance 文書へリンクできること。
- 追記が短く、既存 guide の役割を妨げないこと。
- リンクパスが `docs/` フラット構成に適合していること。
- 存在しない対象ファイルを勝手に作成していないこと。

## 受け入れ条件

- [ ] 存在する各 document-guide に Governance セクションが追加されている。
- [ ] 6つの governance 文書へのリンクが含まれている。
- [ ] 既存の guide 内容を削除していない。
- [ ] 存在しない対象ファイルは報告されている。
- [ ] `docs/` 以下に新規サブディレクトリを作成していない。

## 作業後の報告内容

1. 変更したファイル
2. 見つからなかった対象ファイル
3. 追加した Governance セクションの概要
4. リンク検証結果
