# [RAG][DB][Ingestion] Isolate delete-order logic for chunks_vec → chunks → documents

## Summary
Correct delete order is required during force reinsert, but this rule is fragile and easy to break.

## Problem
`chunks_vec` has no foreign key constraint, so deleting rows in the wrong order can leave orphaned vector records.

## Scope
- `scripts/rag/ingestion/ingester.py`
- repository-level delete helpers

## Task
- move force-delete logic into one dedicated function
- make the delete order explicit and reusable
- add tests that verify correct delete order behavior

## Expected Result
Delete-order rules are no longer spread across the codebase.

## Acceptance Criteria
- [ ] one dedicated delete helper exists
- [ ] delete order is explicit in code
- [ ] tests verify correct order and no orphaned vectors
