# 既存 Known Issues のテンプレート移行計画を作成する

## 概要

既存の Known Issues / Inconsistencies 文書を、新しい共通テンプレートへ段階移行するための計画を作成する。

## 修正理由

各領域には Known Issues / Inconsistencies 系文書が存在するが、形式や記載粒度が必ずしも統一されていない。

ただし、既存エントリを一度に書き換えると、履歴、文脈、既存 ID、関連文書との対応を失うリスクがある。

## 修正意図

この修正の意図は、既存 Known Issues を直ちに書き換えることではなく、移行対象、優先度、順序、リスクを整理した移行計画を作成することである。

実際の移行作業は、後続の個別 Issue に分割する。

## 対象ファイル

- `docs/00_governance_08_known-issues-migration-plan.md`

新規作成する。

## 対象となる既存 Known Issues 文書

存在する範囲で以下を調査対象にする。

- RAG Known Issues / Inconsistencies
- MCP Known Issues / Inconsistencies
- Agent Known Issues / Inconsistencies
- EventBus Known Issues / Inconsistencies
- Shared/DB Known Issues / Inconsistencies

## 実施内容

以下を含む移行計画を作成する。

- Purpose
- Scope
- Target Files
- Current Format Summary
- Migration Policy
- Priority
- Suggested Migration Order
- Risks
- Non-Goals
- Acceptance Criteria for Future Migration

## 必須要件

- 既存 Known Issues 文書の内容を直接変更しない。
- 移行計画のみを作成する。
- 各ファイルの大まかなエントリ数または規模感を記録する。
- 移行優先度を定義する。
- 既存 ID がある場合は尊重する方針を記載する。
- 既存の履歴情報を失わない方針を記載する。

## 優先度基準

以下を定義する。

- security impact
- operator impact
- document-code mismatch
- implementation bug
- deprecated description
- low-risk formatting only

## AI 実装時の共通指示

- この Issue の対象範囲のみを実施すること。
- 実装コードは変更しないこと。
- 既存ドキュメントの移動、統合、削除、リネームは行わないこと。
- `docs/` 以下に新規サブディレクトリを作成しないこと。
- Markdown の構造を壊さないこと。
- コードブロック、リンク、見出し階層を正しく維持すること。
- 判断に迷う場合は推測で修正せず、未解決事項として報告すること。
- EventBus と Workflow の関係整理は本 Issue では行わないこと。
- ドキュメント分割サイズを変更しないこと。
- 作業後、変更ファイル、変更概要、未解決事項を報告すること。

## 作業禁止事項

- 既存 Known Issues エントリを新形式に書き換えない。
- 既存 ID を変更しない。
- 既存エントリを削除しない。
- 複数領域の Known Issues を単一ファイルに統合しない。
- EventBus と Workflow の関係整理を追加しない。

## 検証観点

- どの Known Issues 文書を移行対象にするか分かること。
- 移行順序が明確であること。
- 既存 ID と履歴を保持する方針があること。
- 実際の移行作業を別 Issue に分割できること。

## 受け入れ条件

- [ ] `docs/00_governance_08_known-issues-migration-plan.md` が作成されている。
- [ ] Target Files が列挙されている。
- [ ] Migration Policy が定義されている。
- [ ] Priority 基準が定義されている。
- [ ] Suggested Migration Order がある。
- [ ] 既存 Known Issues 文書は変更していない。

## 作業後の報告内容

1. 作成したファイル
2. 調査対象にした Known Issues 文書
3. 推奨移行順序
4. 次に作成すべき移行 Issue 案
