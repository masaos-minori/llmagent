# Decide and enforce workflow context strategy for approval records

## Priority

Medium

## Type

Workflow Approval / Data Integrity / Audit

## Background

Approval audit events can derive `workflow_id` by joining from approval task ID to the task record.

The approval record itself may not directly store `workflow_id`.

Migration compatibility is not required, so the schema can be simplified around the target model.

## Problem

If approval records do not directly carry workflow context, audit and recovery logic must always join through tasks.

This is acceptable only if enforced consistently.

## Goal

Choose one strategy and enforce it:

1. Store `workflow_id` directly on approval records.
2. Require task join for workflow context and fail if task context is missing.

## Scope

Affected areas:

- Workflow schema
- Approval creation
- Approval resolution
- Approval audit
- Startup recovery
- Tests

## Implementation Tasks

- Decide whether `approvals.workflow_id` should exist.
- If yes:
  - Add `workflow_id` to approval schema.
  - Require `workflow_id` in `request_approval()`.
  - Persist workflow ID on approval creation.
- If no:
  - Ensure all approval audit and recovery paths join through tasks.
  - Fail if task is missing.
  - Fail if task workflow ID is missing.
- Ensure audit events always include workflow ID.
- Ensure startup recovery can display workflow ID.

## Acceptance Criteria

- Approval audit events always include workflow ID.
- Approval recovery can identify workflow ID.
- Approval resolution fails if workflow context cannot be resolved.
- No audit event emits `workflow_id=null` or empty workflow ID.
- Tests cover the selected strategy.

## Validation Plan

Add tests for:

- Approval creation includes or resolves workflow ID.
- Approval audit includes workflow ID.
- Approval resolution includes workflow ID.
- Missing task fails.
- Task without workflow ID fails.
- Startup recovery displays workflow ID.

## Risks

Adding workflow ID directly to approvals duplicates task information. Not adding it requires strict task join checks.

## Out of Scope

- Supporting legacy approval rows.
- Backfill.
- Migration.
