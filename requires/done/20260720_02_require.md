# [High] Merge browser-mcp into web-search-mcp: unify web content retrieval under one server

## Priority

High

## Summary

Merge the `browser` MCP server (`browser_fetch` tool) into the `web_search` MCP server (`search_web` tool), consolidating both web-content-retrieval tools under a single FastAPI server (port 8004). This eliminates architectural asymmetry between the two servers and simplifies deployment, monitoring, and maintenance.

Current state:

| Server | Port | Tools | Architecture |
|---|---|---|---|
| browser-mcp | 8016 | `browser_fetch` | Service layer (`BrowserService`) + domain allowlist + HTML text extraction |
| web-search-mcp | 8004 | `search_web` | Flat formatters + DuckDuckGo only |

After merge:

| Server | Port | Tools | Architecture |
|---|---|---|---|
| web-search-mcp | 8004 | `search_web`, `browser_fetch` | Unified service layer + domain allowlist + multi-provider search |

## Reason for change

The two servers have fundamentally different architectures despite serving similar purposes (retrieving web content):

1. **Architectural asymmetry**: `browser` has a proper service layer with security controls (domain allowlist, IP rejection, size limits), while `web_search` has flat logic in `formatters.py`.
2. **Deployment overhead**: Two separate processes, ports, config files, and health endpoints for related functionality.
3. **Inconsistent patterns**: Different dispatch table construction approaches (dynamic vs static), different error handling patterns.
4. **Maintenance burden**: Security updates, logging, audit trails need to be maintained in two places.

- `web-search-mcp` on port 8004 hosts both `search_web` and `browser_fetch` tools.

### 2. Add browser fetch provider to search_provider.py

Add to `scripts/mcp_servers/web_search/search_provider.py`:

```python
import ipaddress
from urllib.parse import urlsplit

import httpx
from bs4 import BeautifulSoup

from mcp_servers.web_search.web_search_models import (
    BrowserAuthorizationError,
    BrowserConfig,
    BrowserFetchRequest,
    BrowserFetchResponse,
    BrowserValidationError,
)


async def fetch_browser(cfg: BrowserConfig, req: BrowserFetchRequest) -> BrowserFetchResponse:
    """Fetch a URL with domain allowlist enforcement and extract visible text."""

    def _check_domain(url: str) -> str:
        parts = urlsplit(url)
        if parts.scheme not in ("http", "https"):
            raise BrowserValidationError(f"Unsupported URL scheme: {parts.scheme!r}")
        hostname = (parts.hostname or "").lower()
        if not hostname:
            raise BrowserValidationError("URL is missing a hostname")
        try:
            ip = ipaddress.ip_address(hostname)
        except ValueError:
            pass
        else:
            if (
                ip.is_loopback
                or ip.is_link_local
                or ip.is_private
                or ip.is_reserved
                or ip.is_multicast
            ):
                raise BrowserAuthorizationError(f"IP-literal target not allowed: {hostname!r}")
        if hostname not in cfg.allowed_domains:
            raise BrowserAuthorizationError(f"Domain not in allowlist: {hostname!r}")
        return hostname

    _check_domain(req.url)
    max_response_kb = min(req.max_response_kb or cfg.max_response_kb, cfg.max_response_kb)

    async with httpx.AsyncClient(timeout=cfg.timeout_sec) as client:
        resp = await client.get(req.url)

    soup = BeautifulSoup(resp.text, "lxml")
    for tag in soup(["script", "style"]):
        tag.decompose()
    text = soup.get_text(separator="\n", strip=True)

    encoded = text.encode("utf-8")
    max_bytes = max_response_kb * 1024
    truncated = False
    if len(encoded) > max_bytes:
        text = encoded[:max_bytes].decode("utf-8", errors="replace")
        truncated = True

    return BrowserFetchResponse(
        text=text,
        truncated=truncated,
        url=req.url,
        status_code=resp.status_code,
        elapsed_sec=resp.elapsed.total_seconds(),
    )
```

### 3. Add browser_fetch formatter to formatters.py

```python
async def fdisp_browser_fetch(args: dict[str, Any], cfg: BrowserConfig) -> str:
    """Format browser_fetch MCP tool result as plain text for LLM."""
    req = BrowserFetchRequest(**args)
    result = await fetch_browser(cfg, req)
    parts: list[str] = [
        f"status_code={result.status_code} elapsed={result.elapsed_sec}s"
    ]
    if result.truncated:
        parts.append("[RESPONSE TRUNCATED]")
    parts.append(result.text)
    return "\n".join(parts)
```

## Notes for implementer

### Migration risk assessment

Medium risk because:
- Browser security controls (domain allowlist, IP rejection) must be preserved exactly
- Config field names change (`allowed_domains` → `browser_allowed_domains`)
- Port change from 8016 to 8004 requires updating all callers
- Docker Compose changes affect deployment

### What NOT to change

- Do not modify `search_web` tool behavior — it should work identically after merge
- Do not modify DuckDuckGo search provider logic
- Do not change audit log format or structure
- Do not add new tools beyond `browser_fetch`

### Config migration note

Existing deployments using `browser_mcp_server.toml` will need to migrate their config:
- Copy `allowed_domains` values to `browser_allowed_domains` in `web_search_mcp_server.toml`
- Copy `max_response_kb` value to `browser_max_response_kb`
- Copy `timeout_sec` value to `browser_timeout_sec`
- Copy `auth_token` value to `browser_auth_token`
- Delete `browser_mcp_server.toml` after migration
