# [High] Add DESIGN-3 index integrity tests for FTS and vector derived indexes

## Priority
High

## Background
DESIGN-3 defines `documents` and `chunks` as canonical stores, while `chunks_fts` and `chunks_vec` are derived indexes. `rebuild_fts()` already uses `COALESCE(normalized_content, content)`, and `ingester.py` declares a deletion order invariant for `chunks_vec → chunks → documents`. These invariants need regression tests.

Evidence:
- DESIGN-3 defines canonical vs derived index responsibilities and required deletion order.
- `RagMaintenanceService.rebuild_fts()` uses `COALESCE(normalized_content, content)`.
- `ingester.py` declares `delete_document_chain()` with deletion order `chunks_vec → chunks → documents`.

## Goal
Protect derived index consistency for FTS5 and sqlite-vec indexes.

## Scope
Affected files:
- `tests/test_fts_fallback.py`
- New `tests/test_rag_index_integrity.py`
- `agent/services/rag_maintenance_service.py`
- `scripts/rag/ingestion/ingester.py`
- `docs/03_rag_90_inconsistencies_and_known_issues.md`

## Implementation Tasks
- Add a test that `rebuild_fts()` indexes `COALESCE(normalized_content, content)`.
- Add a test that deleting a document leaves no orphan `chunks_vec` rows.
- Add a test that force re-ingestion or delete path follows the deletion order invariant.
- Verify `RagMaintenanceService.reconcile_url()` deletes FTS rows correctly.
- If needed, change FTS deletion to use `rowid` instead of `chunk_id`.
- Update `03_rag_90` DESIGN-3 missing test section with real test file names.

## Open Questions / Unknowns
- Need actual `chunks_fts` schema and whether `DELETE FROM chunks_fts WHERE chunk_id IN (...)` is valid.
- Need full `delete_document_chain()` body to confirm transaction and rollback behavior.
- Need to confirm whether `chunks.embedding` exists in the current schema, because `rebuild_vec()` reads it.

## Acceptance Criteria
- FTS rebuild uses the same text as the trigger path.
- Vector orphans are detected or prevented.
- Deletion order violations are caught by tests.
- `reconcile_url()` is verified against real schema semantics.

## Validation
- Run `uv run pytest tests/test_fts_fallback.py tests/test_rag_index_integrity.py -v`.
- Run `/db consistency` on a test DB after simulated delete/rebuild operations.
