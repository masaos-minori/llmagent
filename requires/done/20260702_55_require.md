# Remove package-level GitHub service re-exports from `mcp/github/__init__.py`

## Priority

High

## Summary

Remove package-level service re-exports from `mcp/github/__init__.py`.

## Background

`mcp/github/__init__.py` currently re-exports GitHub service symbols through `mcp.github.service`:

- `GitHubService`
- `build_service`
- `_GITHUB_TOKEN`

This creates another backward-compatible import path:

- `from mcp.github import GitHubService`
- `from mcp.github import build_service`
- `from mcp.github import _GITHUB_TOKEN`

Since `mcp/github/service.py` is being removed, this package-level re-export must also be removed or rewritten. The project policy is not to keep compatibility-only import paths.

## Goal

Remove package-level compatibility re-exports and require direct imports from implementation modules.

## Target files

- `mcp/github/__init__.py`
- Documentation files
- Tests if they import service symbols from `mcp.github`

## Scope

### In scope

- Remove service symbol re-exports from `mcp/github/__init__.py`.
- Remove `__all__` entries for service symbols.
- Update package docstring.
- Update any imports that depend on package-level service re-exports.

### Out of scope

- Removing the `mcp.github` package itself.
- Changing GitHub MCP server behavior.
- Changing service implementation modules.

## Implementation tasks

- Remove this import from `mcp/github/__init__.py`:

    from mcp.github.service import _GITHUB_TOKEN, GitHubService, build_service

- Remove these entries from `__all__`:
  - `GitHubService`
  - `build_service`
  - `_GITHUB_TOKEN`

- Remove or update docstring entries that mention:
  - `service — re-export stub for backward compatibility`
  - package-level re-export convenience

- Search for package-level service imports:

    grep -R "from mcp.github import GitHubService\|from mcp.github import build_service\|from mcp.github import _GITHUB_TOKEN" -n scripts tests docs

- Replace them with direct imports:
  - `from mcp.github.service_dispatch import GitHubService`
  - `from mcp.github.service_init import build_service, _GITHUB_TOKEN`

## Acceptance criteria

- `mcp/github/__init__.py` does not import from `mcp.github.service`.
- `mcp/github/__init__.py` does not re-export service symbols.
- No code imports service symbols from `mcp.github`.
- No documentation recommends package-level service imports.

## Validation

- Expected result:
