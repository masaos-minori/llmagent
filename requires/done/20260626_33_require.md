# [Low] Add Agent slash commands for MDQ operations

## Priority
Low

## Labels
`agent`, `mcp`, `mdq`, `cli`, `operations`

## Dependencies
- Implement production-grade /health endpoint for mdq-mcp
- Align search_docs schema with actual implementation
- Implement true incremental refresh_index behavior

## Background
MDQ tools are available to the LLM through MCP, but operators also need direct REPL commands to inspect, refresh, and troubleshoot the MDQ index.

## Task
Add Agent slash commands for MDQ operations.

## Implementation Requirements
Add commands:
- `/mdq status`
- `/mdq index <path> [--force]`
- `/mdq refresh <path>`
- `/mdq search <query>`
- `/mdq outline <path>`
- `/mdq get <chunk_id>`
- `/mdq grep <pattern>`

Commands must call MDQ through MCP tools, not by directly reading `mdq.sqlite`.

## Acceptance Criteria
- All `/mdq` commands use MCP tool calls.
- No Agent code directly imports or queries `mdq.sqlite`.
- `/mdq status` reports health and index statistics.
- `/mdq search` and `/mdq get` support manual operator workflows.
