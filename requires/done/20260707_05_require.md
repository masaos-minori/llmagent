# Make workflow-level approval always-on

## Priority

High

## Type

Breaking Change / Workflow Approval / Safety

## Background

Workflow-level approval currently depends on a `require_approval` flag and can be disabled.

The target architecture requires approval between the execute and verify stages for every workflow task.

Tool-level approval remains separate and continues to operate per tool call.

## Problem

Optional workflow-level approval weakens the workflow safety model.

If approval can be disabled, a workflow task can proceed from execute to verify without task-level human sign-off.

## Goal

Remove the approval-disabled path and make workflow-level approval mandatory.

## Scope

Affected areas:

- `AgentConfig`
- `WorkflowEngine`
- Orchestrator workflow engine construction
- Workflow approval tests
- CLI approval flow
- Audit events

## Implementation Tasks

- Remove `AgentConfig.workflow_require_approval`.
- Remove `require_approval` from `WorkflowEngine.__init__()`.
- Remove `_require_approval` state from `WorkflowEngine`.
- Always call `_gate_approval(task)` after execute and before verify.
- Reject config files that still contain `workflow_require_approval`.
- Remove tests that assume approval is disabled.
- Add tests that verify approval is always requested before verify.

## Acceptance Criteria

- WorkflowEngine always creates or checks approval after execute.
- Verify stage never runs until approval is approved.
- Rejected approval halts the task.
- Pending approval suspends the workflow.
- There is no config field that disables workflow approval.
- Existing approval-disabled tests are removed or rewritten.

## Validation Plan

Add tests for:

- Execute stage creates pending approval.
- Verify stage is skipped while approval is pending.
- `/approve <approval_id>` allows verify to continue.
- `/reject <approval_id>` halts task.
- Config with `workflow_require_approval=false` fails.
- Config with `workflow_require_approval=true` fails if the field remains present.

## Risks

All workflow tasks now require approval before verification. This is intentional.

## Out of Scope

- Removing tool-level approval.
- Combining tool-level and workflow-level approval.
- Backward compatibility.
