# [Agent][Watchdog] Enable watchdog by default in production profiles

## Summary
The MCP watchdog exists, but its default interval is `0`, which disables it.

## Problem
A disabled watchdog weakens automatic recovery for MCP server failures in real deployments.

## Scope
- `cfg.mcp.mcp_watchdog_interval`
- startup defaults
- production deployment configuration

## Task
- define a production profile with watchdog enabled by default
- document the recommended watchdog settings for production
- ensure startup logs show whether watchdog is active or disabled

## Expected Behavior
- production deployments use watchdog-based recovery by default
- operators can see watchdog status immediately at startup

## Acceptance Criteria
- [ ] production profile enables watchdog
- [ ] startup logs show watchdog enabled/disabled state
- [ ] docs include the production default and rationale
