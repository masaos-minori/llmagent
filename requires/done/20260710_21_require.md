# [High] Harden ToolExecutor caching, stampede protection, and cache metadata behavior

## Background

`ToolExecutor` implements its own internal `OrderedDict` cache and inflight Future map.

Normal cache stampede protection is tested, but exception behavior is not sufficiently covered. If `_raw_execute()` raises before setting the shared Future result, concurrent callers can be left waiting.

There is also a separate `ToolResultCache` class in `shared/tool_cache.py`, which overlaps conceptually with the internal ToolExecutor cache.

## Goal

Make ToolExecutor cache behavior safe, explicit, and consistently documented.

## Scope

- `shared/tool_executor.py`
- `shared/tool_cache.py`

- `90_99.md`

## Notes for AI Implementer

Do not silently swallow exceptions in stampede protection. Release all waiters and preserve observable failure behavior.
