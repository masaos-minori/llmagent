# Rename shared/tool_routing.py to shared/tool_routing_validation.py

## Priority

High

## Goal

Make the module name reflect its actual responsibility: routing drift validation, not runtime routing.

## Background

`shared/tool_routing.py` validates registry/config/live tool list consistency. It does not perform runtime routing.

Runtime routing belongs to `ToolRouteResolver` and `ToolRegistry`.

## Scope

Affected files:

- `shared/tool_routing.py`
- `shared/tool_routing_validation.py` new renamed file
- all imports of `shared.tool_routing`
- tests importing `shared.tool_routing`

## Implementation requirements

Rename:

- from `shared/tool_routing.py`
- to `shared/tool_routing_validation.py`

Move functions:

- `validate_routing_against_config()`
- `validate_routing_against_live()`
- `validate_all_routing()`

Update all imports to the new module name.

Do not leave a backward-compatible re-export stub.

## Acceptance criteria

- `shared/tool_routing.py` no longer exists.
- `shared/tool_routing_validation.py` exists.
- All imports use `shared.tool_routing_validation`.
- No compatibility stub is left behind.
- Validation functions remain behaviorally unchanged.
- Runtime execution path does not call validation-only functions.
- Startup/readiness validation still works.

## Validation plan

- Search for `shared.tool_routing`.
- Search for `tool_routing`.
- Run unit tests.
- Run startup validation tests if available.
- Run import checks.

## Out of scope

- Changing validation semantics.
- Changing `ToolRouteResolver`.
