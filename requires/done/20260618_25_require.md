# [agent][observability][workflow] Add workflow-level observability

## Summary
Current observability is still focused on turns and sessions.

## Problem
The system can trace turn-level execution, but larger units such as:
- workflow
- task
- retry
- approval
- artifact

are not tracked clearly enough.

## Scope
- audit log
- diagnostics.jsonl
- OTel spans
- workflow observability design

## Task
- add workflow and task identifiers to observability data
- track retry and approval events
- add artifact-related tracking fields
- make workflow execution easier to inspect

## Expected Result
Execution can be traced at workflow/task level, not only at turn level.

## Acceptance Criteria
- [ ] workflow identifiers are included in observability design
- [ ] task identifiers are included
- [ ] retry and approval events are traceable
- [ ] artifact tracking fields are defined
