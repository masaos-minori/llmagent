# [High] Align RAG rerank failure behavior with implementation

## Priority
High

## Background
`RerankStage` only falls back to RRF order when `use_rerank=False`. When `use_rerank=True`, `llm.cross_encoder_rerank()` exceptions are not caught in `rerank.py`, so failures propagate. However, the operations document says cross-encoder failure falls back to RRF order.

Evidence:
- `rerank.py` raises on LLM failure and only uses fallback when `use_rerank=False`.
- `03_rag_03_query_pipeline.md` says cross-encoder failure propagates.
- `03_rag_05_configuration_and_operations.md` says cross-encoder failure falls back to RRF order.

## Goal
Make docs and tests match the actual rerank behavior.

## Scope
Affected files:
- `scripts/rag/stages/rerank.py`
- `scripts/rag/pipeline.py`
- `tests/test_rag_pipeline_stage.py`
- `docs/03_rag_03_query_pipeline.md`
- `docs/03_rag_05_configuration_and_operations.md`

## Implementation Tasks
- Update `03_rag_05_configuration_and_operations.md`:
  - `use_rerank=False` uses RRF order + dedup.
  - `use_rerank=True` propagates `RagRerankError`.
- Add a test to `tests/test_rag_pipeline_stage.py`:
  - configure `use_rerank=True`
  - mock `cross_encoder_rerank()` to raise
  - assert the exception propagates.
- Inspect `RagPipeline.run()` to confirm whether it catches stage exceptions.
- If `RagPipeline.run()` catches rerank exceptions, update this issue with the actual behavior before changing docs.

## Open Questions / Unknowns
- Need full `scripts/rag/pipeline.py` implementation around stage execution.
- Need to confirm whether `RagPipeline.run()` records rerank failure as `StageResult.status="failure"` before propagating.
- Need to confirm expected MCP response when rerank failure propagates through `RagPipelineMCPService`.

## Acceptance Criteria
- Docs no longer claim cross-encoder failure falls back to RRF when `use_rerank=True`.
- Test coverage exists for rerank failure propagation.
- Rerank fallback wording is limited to `use_rerank=False`.

## Validation
- Run `uv run pytest tests/test_rag_pipeline_stage.py -v`.
- Search docs with `rg -n "Cross-encoder failure|RRF order|RagRerankError|rerank.*fallback" docs scripts tests`.
