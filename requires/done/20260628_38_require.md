# Make unknown `/debug` subcommands explicit validation errors

## Priority
Medium

## Labels
agent, commands, debug, validation, cleanup

## Dependencies
- Remove `/debug audit` branch from debug command handler

## Background
The current `_cmd_debug()` implementation falls through to debug mode toggle for unrecognized subcommands. After removing `/debug audit`, `/debug audit` should not accidentally toggle debug mode.

## Task
Make `_cmd_debug()` reject unknown subcommands explicitly.

## Implementation Requirements
- Update `_cmd_debug()` behavior:
  - `/debug` toggles debug mode.
  - `/debug verbose` sets DEBUG log level.
  - `/debug normal` restores INFO log level.
  - `/debug <anything-else>` returns a validation error.
- Ensure `/debug audit` is treated as invalid.
- Use the project’s standard validation output style.

## Acceptance Criteria
- `/debug audit` is rejected and does not toggle debug mode.
- `/debug foo` is rejected and does not toggle debug mode.
- `/debug` without args still toggles debug mode.
- `/debug verbose` and `/debug normal` still work.

## Source Context
`cmd_debug.py` currently checks `audit`, `verbose`, and `normal`, then falls through to toggling debug mode.
