# [High] Remove obsolete web_search_url from agent configuration

## Labels
priority/high, area/config, area/web-search, area/mcp, cleanup

## Summary
Remove `web_search_url` from agent configuration and use `mcp_servers.web_search.url` as the authoritative web search endpoint.

The confirmed specification is:

- `web_search_url` in `agent.toml` is unnecessary.
- Web search must use the `mcp_servers.web_search` MCP server entry.

## Background
The provided configuration still contains `web_search_url`, but web search is routed through the MCP server entry `mcp_servers.web_search`.

Keeping both fields creates duplicate sources of truth.

## Required Changes

- Remove `web_search_url` from `config/agent.toml`.
- Remove `web_search_url` from agent config dataclasses if present.
- Remove builder logic that reads or propagates `web_search_url`.
- Update docs to state that web search uses `mcp_servers.web_search.url`.
- Decide and implement handling for existing `web_search_url` keys:
  - Option A: reject as a forbidden key
  - Option B: emit migration warning and ignore
- Prefer Option A or Option B. Do not keep two active endpoint fields.

## Acceptance Criteria
- `web_search_url` is no longer documented as valid agent configuration.
- Web search endpoint source of truth is `mcp_servers.web_search.url`.
- Duplicate endpoint configuration is removed or ignored with explicit migration handling.
- Tests cover config loading with and without `web_search_url`.
- Documentation and example config are updated.
