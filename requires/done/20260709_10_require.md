# [Medium] Improve plugin tool audit and observability

## Labels
priority/medium, area/plugin, area/audit, area/observability

## Summary
Improve audit and observability for plugin tool execution.

## Background
Plugin tools bypass MCP routing and are executed before MCP dispatch. They are operationally different from MCP tools. 【2-a4e3df】

Current operations and audit documentation primarily focuses on MCP tool execution and cross-server tracing. 【4-0fe60e】

## Required Changes

### Audit Events
Ensure plugin tool execution emits audit events with an explicit source.

Add or standardize:

- `source="plugin"` for plugin tools
- a clear existing or new value for MCP tools, such as `source="mcp"` or the current compatible value

Ensure plugin tool result records are distinguishable from MCP tool results.

Document whether plugin execution has:

- empty `server_key`
- empty `request_id`
- no MCP request correlation

### Plugin Status
Consider improving `/plugin status` to include:

- last load result
- failed plugin details
- conflict counts
- rejected command shadow count
- allowed/rejected tool conflict details if available

### Documentation
Update:

- `05_agent_10_operations-and-observability.md`
- `05_agent_11_extension-points.md`

Document:

- how to identify plugin tool execution in audit logs
- how plugin execution differs from MCP execution
- known observability limitations

### Tests
Add or update tests for:

- plugin execution audit source
- plugin result distinguishability
- `/plugin status` output where applicable

## Acceptance Criteria
- Plugin executions are clearly identifiable in audit logs.
- MCP and plugin tool executions can be distinguished operationally.
- Documentation explains plugin observability fields and limitations.
- Tests cover plugin audit or status behavior.

## Target Files
- `shared/plugin_tool_invoker.py`
- `shared/tool_executor.py`
- `agent/tool_audit.py`
- `agent/commands/cmd_plugins.py`
- `05_agent_10_operations-and-observability.md`
- `05_agent_11_extension-points.md`
- `tests/test_plugin_registry.py`
- `tests/test_plugin_contract.py`

## Non-Goals
- Do not ban plugin tools.
- Do not remove `plugin_tool_override`.
- Do not route plugin tools through MCP.
