# H-9: Implement LifecycleState tracking in ServerLifecycleRouter

## Priority

High

## Background

`LifecycleState` exists, but `_ServerLifecycleRouter.get_transport_state()` currently returns `UNKNOWN` for every server. This makes lifecycle status unusable for status display and diagnostics.

## Target files

- `agent/factory.py`
- `agent/lifecycle.py` if helper adjustments are needed

## Implementation tasks

- Add `self._states: dict[str, LifecycleState] = {}` to `_ServerLifecycleRouter`.
- Update state during lifecycle operations:
  - start begins: `STARTING`
  - start succeeds: `RUNNING`
  - start fails: `FAILED`
  - restart begins: `STARTING`
  - shutdown succeeds: `STOPPED`
  - shutdown fails: `FAILED`
- Implement `get_transport_state(server_key)` using the state map.
- Return `UNKNOWN` only for unknown or never-seen servers.
- Optionally use `assert_valid_transition()`.

## Acceptance criteria

- `get_transport_state()` returns meaningful values after start, restart, failure, and shutdown.
- Unknown server keys return `UNKNOWN`.
- Startup failures are reflected as `FAILED`.
- Shutdown success is reflected as `STOPPED`.

## Test plan

- Test state transitions for successful start.
- Test state transitions for start failure.
- Test state transitions for restart.
- Test state transitions for shutdown.
- Test unknown server state.
