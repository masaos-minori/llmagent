# Require workflow context on LLM, tool, audit, and diagnostic events

## Priority

High

## Type

Observability / Audit / Workflow

## Background

Workflow-only execution requires every runtime event to be correlated with workflow identifiers.

Relevant identifiers are:

- `workflow_id`
- `task_id`
- `stage_id`
- `attempt_id`
- `session_id`

## Problem

Some diagnostic and LLM/tool-loop paths can currently persist events without workflow context.

For example, mid-turn LLM errors may be stored in diagnostics without workflow identifiers.

## Goal

Ensure every LLM/tool execution event and diagnostic record carries workflow context.

## Scope

Affected areas:

- `LLMTurnRunner`
- `DiagnosticStore`
- Tool execution audit
- Tool runner
- OTel spans
- Session diagnostics
- Tests for event schema

## Implementation Tasks

- Pass workflow context to `DiagnosticStore.save()`.
- Include workflow context in mid-turn error payloads.
- Include workflow context in partial completion diagnostics.
- Include workflow context in tool execution audit events.
- Include workflow context in LLM spans.
- Include workflow context in tool execution spans if present.
- Fail tests if any workflow execution event has empty workflow context.
- Remove direct-path placeholder IDs.

## Acceptance Criteria

- Mid-turn LLM errors include workflow context.
- Partial completions include workflow context.
- Tool execution audit events include workflow context.
- LLM spans include workflow context.
- Tool spans include workflow context where applicable.
- Tests fail when workflow context is empty during workflow execution.

## Validation Plan

Add tests for:

- LLM success event context.
- LLM transport error context.
- Partial completion context.
- Tool execution context.
- Tool error context.
- OTel span attributes.
- DiagnosticStore persisted fields.

## Risks

Existing log consumers may need to handle new required fields.

## Out of Scope

- Changing audit log storage backend.
- Changing diagnostic table storage backend.
- Migration support.
