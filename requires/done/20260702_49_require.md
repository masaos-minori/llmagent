# Decide whether `artifact_type` should be persisted

## Priority

Low

## Summary

Decide whether `artifact_type` from `ChunkJsonRaw` should be stored in the DB or remain ingestion-only metadata.

## Background

`ChunkJsonRaw` includes `artifact_type`, but no corresponding DB column has been confirmed.

## Goal

Remove ambiguity around `artifact_type`.

## Scope

### In scope

- Review usage of `artifact_type`.
- Decide persistence policy.
- Document the decision.

### Out of scope

- Broad artifact-management redesign.

## Implementation options

### Option A: Persist it

Add a column such as:

- `artifact_type TEXT NOT NULL DEFAULT ''`

### Option B: Do not persist it

Document it as ingestion pipeline metadata only.

## Acceptance criteria

- `artifact_type` is either mapped to a DB column or explicitly documented as non-persisted.
- `ChunkSplitter`, `pipeline_utils`, and DB docs are consistent.

## Validation

- Search usage:
```bash
grep -R "artifact_type" scripts tests docs -n
```

## Confirmation required

- Confirm actual business use of `artifact_type`.
- Confirm whether it is needed for search filtering, debugging, or artifact provenance.
