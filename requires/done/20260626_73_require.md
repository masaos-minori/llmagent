## Summary
Add operational regression tests that lock down Event Bus delivery semantics, health behavior, offset recovery, subscribe replay/push behavior, and DLQ lifecycle.

## Background
Event Bus behavior depends on runtime interaction between SQLite persistence, SSE subscribe, EventBroker live delivery, offset files, DLQ promotion, and health reporting.

## Problem
Without cross-cutting regression tests, delivery semantics and operational behavior can regress even if unit tests pass.

## Required Changes
- Add tests for:
  - `/health` 200 vs 503 status mapping
  - ack-only offset advancement
  - reconnect replay from last ack
  - crash-before-ack replay
  - replay-to-live transition race
  - slow consumer queue overflow and replay recovery
  - DLQ promotion
  - DLQ requeue
  - DLQ re-promotion when retry_count remains above threshold
  - unsafe public bind guard
  - concurrent publish/replay/ack stress behavior
- Keep tests deterministic and isolated.
- Use temporary directories for DB, JSONL, offsets, and deadletter files.

## Acceptance Criteria
- Core Event Bus reliability invariants have regression tests.
- Tests fail if offset or health semantics regress.
- Tests cover both normal and degraded operational paths.
- Tests do not depend on persistent local machine state.

## Out of Scope
- Large-scale load testing.
- Distributed multi-node tests.
- Benchmarking throughput.
