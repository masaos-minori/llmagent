## [context][type] ctx.services の Any 型を AppServices | None に変更する

### Summary
`AgentContext.services` が `Any` 型として宣言されており、型チェッカーがアクセス誤りを検出できない。`services_required` プロパティが存在するが、主要コンポーネントは直接 `ctx.services.xxx` にアクセスしており暫定措置が恒久化している。

### Background
`AgentContext` は DI ハブとして設計されており、`ctx.services` には `AppServices` インスタンスが注入される。`factory.build_agent_context()` が完了した後は常に `AppServices` が存在するが、初期化前は `None` である。この `None` ケースへの対応として `services_required` プロパティが用意されているが、`context.py:185` のコメントでは「cascading union-attr errors を避けるための暫定措置」と説明されており、型安全への移行が後回しになっている。

### Problem
`context.py:186` の `self.services: Any = None` により、`orchestrator.py`、`llm_turn_runner.py`、`repl.py` などでの `ctx.services.xxx` アクセスは型チェッカーによる静的検証の対象外になっている。フィールド名の誤りや None アクセスは実行時まで発見されない。また `repl.py:307` では `svc = self._ctx.services; if svc is not None:` という手動 None チェックが必要になっており、型情報があれば不要なパターンが散在している。

### Scope
- `context.py` の `services` フィールドの型変更
- `orchestrator.py` と `llm_turn_runner.py` での `ctx.services.xxx` アクセスを `ctx.services_required.xxx` 経由に変更
- `repl.py` の `svc = self._ctx.services; if svc is not None:` パターンはそのまま維持 (`_close_resources()` 等は明示的 None チェックが適切)
- `commands/` 配下の `ctx.services.xxx` アクセスは範囲外 (段階的移行)

### Task
1. `context.py:186` の `self.services: Any = None` を `self.services: AppServices | None = None` に変更する
2. `context.py:185` の「暫定措置」コメントを削除またはマイグレーション完了に書き換える
3. `orchestrator.py` の `ctx.services.hist_mgr`、`ctx.services.audit_logger`、`ctx.services.llm`、`ctx.services.memory` などへの直接アクセスを `ctx.services_required.xxx` 経由に変更する
4. `llm_turn_runner.py:177` の `ctx.services.llm` アクセスを `ctx.services_required.llm` に変更する
5. mypy を実行し型エラーがないことを確認する
6. `ruff check` を実行しリント違反がないことを確認する

### Acceptance Criteria
- `context.py` に `self.services: Any` の宣言が存在しない
- `orchestrator.py` と `llm_turn_runner.py` で `ctx.services.xxx` の直接アクセスが存在しない (必ず `ctx.services_required.xxx` または明示的な `svc = ctx.services; if svc is not None: svc.xxx` のパターン)
- mypy が型エラーを報告しない
- `ruff check` が新規エラーを報告しない
- 既存テストが通過する
