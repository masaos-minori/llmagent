# Document `tool_results.undone` as part of the official schema and API behavior

## Priority

Medium

## Summary

Update documentation to include `tool_results.undone` and its behavior.

## Background

`tool_results.undone` is part of the current DDL and is used by `ToolResultStore.store()`, `get()`, `list_recent()`, and `mark_turn_undone()`.

Tests verify turn-level marking behavior.

## Goal

Make `tool_results.undone` a documented part of the session DB schema and API behavior.

## Target files

- `90_shared_04_db_architecture_and_schema.md`
- `90_shared_05_db_api_and_operations.md`

## Scope

### In scope

- Schema documentation.
- DB API documentation.
- Tool result behavior documentation.

### Out of scope

- Removing `undone`.
- Physical deletion of tool results on undo.

## Implementation tasks

- Add `undone` to the `tool_results` schema documentation:
  - `undone INTEGER NOT NULL DEFAULT 0`
- Document behavior:
  - `undone = 0`: active tool result
  - `undone = 1`: result belongs to an undone turn
  - full text remains physically stored
- Document `ToolResultStore.mark_turn_undone(session_id, turn)`.

## Acceptance criteria

- `tool_results.undone` appears in schema docs.
- `ToolResultStore` docs describe `mark_turn_undone()`.
- Documentation states that undo is logical marking, not physical deletion.

## Validation

- Run:
```bash
    uv run pytest tests/test_tool_result_store.py
```
## Confirmation required

- Confirm how `/tool show <id>` should display or warn about `undone=True`.
