# [High] Unify workflow and approval source of truth in durable storage

## Summary
Move the source of truth for workflow, task, attempt, approval, and artifact state into durable workflow storage.
Reduce `AgentContext` workflow fields to UI-only cache or remove them entirely.

## Problem
Workflow state is currently split across:
- `workflow.sqlite` durable records
- `ctx.workflow`
- `ctx.turn.pending_approval_id`

This causes restart-safety, approval resume, and retry consistency issues.

## Goal
Make durable workflow storage the only authoritative source for:
- workflow state
- task state
- retry state
- approval state
- artifact state

## Scope
Target files may include:
- `agent/context.py`
- `agent/orchestrator.py`
- `agent/workflow/state_store.py`
- workflow command handlers related to `/approve` and `/reject`

## Required Changes
1. Remove authoritative workflow state from `AgentContext`.
2. Add or extend a query API in workflow state storage so UI commands can resolve:
   - pending approvals
   - current workflow status
   - latest task state
3. Make `/approve` and `/reject` resolve approval state from durable storage.
4. Keep only minimal display-oriented workflow cache in UI state if strictly necessary.

## Acceptance Criteria
- Restarting the REPL does not lose approval/resume capability.
- `/approve` and `/reject` work using durable workflow records only.
- `AgentContext` is no longer the source of truth for workflow lifecycle state.
- Tests cover restart + resume + pending approval scenarios.

## Out of Scope
- Full workflow phase redesign
- Repository write boundary refactor
