# Expose RuntimeToolRegistry status in MCP diagnostics

## Priority

Medium

## Target files

- agent/commands/cmd_mcp.py
- agent/commands/registry.py
- shared/runtime_tool_registry.py
- agent/context.py
- tests/agent/commands/test_cmd_mcp.py
- tests/agent/commands/test_help.py

## Reason

After RuntimeToolRegistry becomes the execution index, operators need visibility into discovered tools and policy annotations.

## Intent

Expose runtime tool discovery and policy state through CLI diagnostics.

## Scope

Update `/mcp status` or add related subcommands to show:

- server_key
- tool name
- enabled_for_llm
- agent_safety_tier
- requires_approval
- is_write
- requires_serial

Optionally add:

- `/mcp tools`
- `/mcp tools dump`
- `/mcp tools diff`

## Acceptance criteria

- Operators can inspect RuntimeToolRegistry contents.
- Disabled or policy-filtered tools are visible.
- Output is concise and safe for logs.
