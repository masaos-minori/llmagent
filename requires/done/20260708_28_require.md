# H-6: Treat all McpServerConfig field changes as restart-required

## Priority

High

## Background

McpServerConfig contains transport, authentication, lifecycle, timeout, command, environment, and validation-related settings.

These settings are used by ToolExecutor, HttpTransport, lifecycle routing, and watchdog. Partial hot reload creates inconsistent runtime behavior.

## Target files

- `agent/services/config_reload.py`
- `shared/mcp_config.py` if helper placement is preferred there
- tests for reload classification

## Implementation tasks

- Compare all restart-bound MCP server definition fields:
  - `transport`
  - `url`
  - `startup_mode`
  - `healthcheck_mode`
  - `call_timeout_sec`
  - `startup_timeout_sec`
  - `tool_names`
  - `auth_token`
  - `role`
  - `cmd`
  - `env`
- For every changed field, add `mcp/<server_key>.<field>` to `ConfigReloadOutcome.needs_restart`.
- Do not mutate the running config for any of these changes.
- Do not classify any of these changes as runtime-applied or deferred.

## Acceptance criteria

- Every changed McpServerConfig field is restart-required.
- No MCP server definition field is hot-reloadable.
- No MCP server definition field is deferred.
- Reload output clearly identifies changed fields.

## Test plan

- Parameterize reload tests across all listed fields.
- For each changed field, verify:
  - `needs_restart` contains the correct `mcp/<key>.<field>` entry
  - `applied` contains no MCP entry
  - `deferred` contains no MCP entry
  - running config is unchanged
