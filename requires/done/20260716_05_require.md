# Remove MDQ summary cache stub and `use_summary` request compatibility path

## Priority

Medium

## Target files

- `mcp_servers/mdq/service.py`
- `mcp_servers/mdq/models.py`
- `mcp_servers/mdq/db_schema.py`
- `config/mdq_mcp_server.toml`
- `docs/04_mcp_*.md`
- `tests/test_mdq_get_chunk*.py`

## Type

Backward compatibility removal / stub cleanup

## Background

MDQ includes a summary cache path for large chunks:

- Config keys: `summary_cache_enabled`, `summary_threshold`, `summary_model`
- Request field: `GetChunkRequest.use_summary`
- Model types: `ChunkSummary`, `GetChunkSummaryResponse`
- Runtime method: `_generate_and_cache_summary()`

However, `_generate_and_cache_summary()` returns `None` when `summary_model == "default"` and contains only future-work comments for actual LLM integration.

As a result, this is not a complete production feature. It is a compatibility or placeholder surface.

## Problem

The summary cache path increases complexity without providing reliable behavior.

Risks:

- Users may enable config expecting summaries to be generated
- `use_summary` appears to be a supported request option
- Future maintainers may assume LLM summarization is already integrated
- `chunk_summaries` schema may remain unused
- Test coverage may not reflect actual no-op behavior

## Intent

Remove the summary cache placeholder unless there is a decision to implement real summarization.

This cleanup should make `get_chunk()` behavior simpler and predictable.

## Tasks

- [ ] Decide whether summary generation is a supported near-term feature
- [ ] If not supported, remove `summary_cache_enabled` from config
- [ ] Remove `summary_threshold` from config
- [ ] Remove `summary_model` from config
- [ ] Remove summary config reads from `MdqService`
- [ ] Remove `_generate_and_cache_summary()`
- [ ] Remove `use_summary` from `GetChunkRequest`
- [ ] Remove summary branch from `get_chunk()`
- [ ] Remove `ChunkSummary` if unused
- [ ] Remove `GetChunkSummaryResponse` if unused
- [ ] Review `db_schema.py` for `chunk_summaries`
- [ ] Decide whether to drop, ignore, or migrate `chunk_summaries`
- [ ] Update docs and examples
- [ ] Add tests for normal `get_chunk()` behavior after cleanup
- [ ] Add tests that `use_summary` is no longer accepted if removed

## Acceptance criteria

- `get_chunk()` no longer contains summary stub behavior
- No config key enables non-functional summary behavior
- Summary-related models are removed unless actively used
- Database schema treatment is documented
- Tests cover the final behavior

## Non-goals

- Do not implement LLM summarization in this issue
- Do not add background summary generation
