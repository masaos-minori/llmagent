# M-1: Remove Tool Result Summarization config from dataclass, builder, reload, and display

## Priority

Medium

## Background

Tool Result Summarization is being removed, but the configuration still exists in multiple places:

- `ToolConfig.use_tool_summarize`
- `ToolConfig.tool_summarize_threshold`
- `_build_tool_config()`
- `/reload` application logic
- `/config` display output

These settings must no longer appear as active runtime features.

## Target files

- `agent/config_dataclasses.py`
- `agent/config_builders.py`
- `agent/services/config_reload.py`
- `agent/commands/cmd_config_display.py`

## Implementation tasks

- Remove or deprecate `ToolConfig.use_tool_summarize`.
- Remove or deprecate `ToolConfig.tool_summarize_threshold`.
- Remove both fields from `_build_tool_config()`.
- Remove `use_tool_summarize` handling from `ConfigReloadService._apply_tool_params()`.
- Remove `tool_summarize_threshold` handling from `ConfigReloadService._apply_tool_params()`.
- Remove `use_tool_summarize` and `tool_summarize_thr` from `/config` execution settings display.
- Decide how to handle existing config files that still contain these keys:
  - ignore with warning
  - fail validation
  - allow one compatibility release with deprecation warnings

## Acceptance criteria

- `/config` no longer displays summarization settings.
- `/reload` no longer applies summarization settings.
- ToolConfig no longer exposes active summarization settings.
- Old config keys do not make users believe the feature still works.
- Tool execution does not depend on removed config fields.

## Test plan

- Load config without summarization fields.
- Load config with old summarization fields and verify chosen behavior.
- Run `/config` and verify summarization settings are absent.
- Run `/reload` and verify summarization settings are not applied.
