# Add CI static scan for workflow compatibility and direct-path patterns

## Priority

Low

## Type

CI / Static Analysis / Regression Prevention

## Background

After removing workflow compatibility behavior, stale symbols and strings may remain in code, tests, comments, or error messages.

These can reintroduce direct-path behavior later.

## Problem

Compatibility patterns such as workflow modes, startup fallback, and direct execution may reappear without test failures unless explicitly scanned.

## Goal

Prevent reintroduction of workflow compatibility paths.

## Scope

Affected areas:

- CI checks
- Source tree scan
- Test tree scan
- Allowlist for intentional references

## Implementation Tasks

- Add a static scan for forbidden patterns.
- Use an explicit allowlist for false positives.
- Require a review comment for allowlist additions.
- Fail CI on unauthorized matches.

Forbidden patterns should include:

- `workflow_mode`
- `allow_startup_fallback`
- `is_workflow_enabled`
- `mode="disabled"`
- `mode="auto"`
- `workflow_mode="disabled"`
- `workflow_mode="auto"`
- `direct LLM path`
- `direct-execution fallback`
- `Workflow mode=disabled`
- `LLMTurnRunner.run(` without workflow context

## Acceptance Criteria

- CI fails on forbidden compatibility patterns.
- CI allows approved exceptions only through explicit allowlist.
- Static scan covers production code and tests.
- Direct-path patterns cannot be reintroduced silently.
- The allowlist is small and reviewable.

## Validation Plan

Add tests or fixtures for:

- Forbidden `workflow_mode` reference.
- Forbidden `allow_startup_fallback` reference.
- Forbidden direct execution reference.
- Allowed reference in static scan test fixture.
- Unauthorized direct `LLMTurnRunner.run()` usage.
- Authorized workflow-context runner usage.

## Risks

Static scans may produce false positives. Use a reviewed allowlist rather than weakening the check.

## Out of Scope

- Semantic code analysis.
- Automatic code rewriting.
- Migration support.
