# [Low] Update compatibility-remnant checks for Shared/DB schema cleanup

## Goal

Prevent removed Shared/DB schema compatibility remnants from being reintroduced.

## Scope

Static checks only.

## Implementation Tasks

- Update `scripts/checks/check_no_compat.py` or the equivalent compatibility check script.
- Add checks for stale references:
  - `workflow_schema.py`
  - `db.workflow_schema`
  - `python -m db.workflow_schema`
  - `07_ref-sqlite.md`
  - `07_spec_db.md`
  - `DOCMISS-01`
  - `UNDOC-03`
  - `TYPE-01`
  - `DESIGN-01`
  - `UNIMPL-01`
- Add checks for deprecated direct Event Bus SQL initialization:
  - `.read eventbus/schema.sql`
  - `eventbus/schema.sql` as schema initialization source
- Do not block the `retry_count` column, because it is intentionally retained as deprecated schema.
- Do not block `messages.tool_call_id`, because it is intentionally retained and used by `SessionMessageRepository`.

## Acceptance Criteria

- Stale workflow schema entrypoints are detected.
- Old source document references are detected.
- Missing issue ID references are detected.
- Direct Event Bus SQL file initialization is detected.
- Intentional retained fields are not falsely blocked.

## Validation

- Run the compatibility check script.
- Insert one temporary stale reference and confirm the check fails.
- Remove the temporary reference and confirm the check passes.
