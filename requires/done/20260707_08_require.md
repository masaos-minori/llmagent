# Add mandatory workflow SQLite schema preflight

## Priority

Medium

## Type

Startup Validation / Database / Workflow

## Background

Workflow execution depends on `workflow.sqlite`.

Required workflow tables include:

- `tasks`
- `attempts`
- `processed_events`
- `artifacts`
- `approvals`

Migration compatibility is not required. Missing schema should fail startup.

## Problem

If workflow schema issues are detected only during turn execution, operators receive late and unclear runtime errors.

Workflow-only execution requires schema readiness before the REPL starts.

## Goal

Validate workflow database schema at startup and fail fast if required structures are missing.

## Scope

Affected areas:

- Startup validation
- Workflow DB helper
- Workflow schema checks
- Startup error reporting
- Tests

## Implementation Tasks

- Check that `workflow.sqlite` exists or can be opened.
- Check required tables exist.
- Check required columns exist.
- Treat missing schema as startup fatal.
- Provide remediation: recreate DB or run `create_workflow_schema()`.
- Prevent Orchestrator construction when schema preflight fails.

## Acceptance Criteria

- Missing workflow DB fails startup.
- Missing workflow table fails startup.
- Missing required column fails startup.
- Error message identifies missing table or column.
- Error message includes remediation.
- Runtime turn processing does not encounter schema-related `OperationalError` for missing required schema.

## Validation Plan

Add tests for:

- Valid workflow schema.
- Missing workflow DB.
- Missing `tasks`.
- Missing `attempts`.
- Missing `processed_events`.
- Missing `artifacts`.
- Missing `approvals`.
- Missing required workflow column.

## Risks

Developers must initialize workflow schema before running the agent.

## Out of Scope

- Schema migration.
- Backfill.
- Partial compatibility with old workflow DBs.
