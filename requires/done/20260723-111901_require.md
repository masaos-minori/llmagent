# Requirement: Detect Silent Background Task Failures and Alert Operators

## Priority

Low

## Target files

- `scripts/agent/orchestrator.py`

## Background

In `Orchestrator._handle_turn_start()`, first-turn background tasks are tracked via `self._background_tasks` (a set of `asyncio.Task` objects). When a task completes, a done callback `_discard_and_log()` is invoked:

```python
def _discard_and_log(task: asyncio.Task[Any]) -> None:
    exc = task.exception()
    if exc is not None:
        if isinstance(exc, asyncio.CancelledError):
            pass
        else:
            logger.error("First-turn background task failed: %s", exc)
    self._background_tasks.discard(task)
```

This callback logs the error but takes no further action. The task is discarded from the set, and no alert is raised. There is no mechanism to detect when background tasks fail and notify operators.

Note: The original issue references `_start_background_task()` and `ctx.background_task` which do not currently exist in the codebase. The current implementation uses `self._background_tasks` as a set managed within the Orchestrator class. The underlying concern — silent background task failures — remains valid regardless of the specific implementation details.

## Problem

When a background task fails (e.g., health check failure, memory flush failure), the error is logged but no operator-facing alert is generated. The operator sees nothing in the REPL output, and the failure may go unnoticed until symptoms appear elsewhere. For long-running deployments, silent background task failures can accumulate without detection.

## Reason for change

Background task failures must produce visible alerts so operators know when critical maintenance operations (health checks, memory flushing) have failed. Logging alone is insufficient — the operator needs immediate visibility into failures that could affect system reliability.

## Implementation intent

Add a mechanism to detect background task failures and emit operator-visible warnings. On first failure per session, escalate to a warning; on repeated failures, escalate further.

## Implementation instructions

1. In `Orchestrator.__init__()`, add a tracking field for consecutive background task failures:
   ```python
   self._consecutive_bg_failures: int = 0
   ```
2. Modify `_discard_and_log()` to increment the counter on failure and emit a warning on the first failure:
   ```python
   def _discard_and_log(self, task: asyncio.Task[Any]) -> None:
       exc = task.exception()
       if exc is not None:
           if isinstance(exc, asyncio.CancelledError):
               self._consecutive_bg_failures = 0
           else:
               self._consecutive_bg_failures += 1
               if self._consecutive_bg_failures == 1:
                   logger.warning("First background task failure: %s", exc)
               else:
                   logger.error("Consecutive background task failures (%d): %s",
                               self._consecutive_bg_failures, exc)
       self._background_tasks.discard(task)
   ```
3. Add a threshold constant (e.g., `BG_FAILURE_THRESHOLD = 5`) and consider escalating to a FATAL message after N consecutive failures.

## Acceptance criteria

- [ ] First background task failure produces a WARNING-level log entry
- [ ] Consecutive failures produce increasingly severe log entries
- [ ] No regression in normal operation when background tasks succeed
- [ ] Cancelled tasks do not count toward consecutive failure count

## Tests

- Unit test: Mock a background task to raise an exception, verify the first failure triggers a WARNING log
- Unit test: Verify consecutive failures increment the counter and escalate severity
- Integration test: Break the memory backend, trigger multiple turns, observe escalating log messages
