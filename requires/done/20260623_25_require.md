# Title
Improve offset durability beyond disconnect-only persistence

## Summary
Reduce progress loss on abnormal termination by adding periodic or threshold-based offset checkpointing in addition to disconnect-time persistence.

## Background
Offsets are currently written when the SSE generator is cancelled during disconnect handling.
That means abrupt process termination can lose recent consumer progress.

## Problem
The current offset model is best-effort and may be too weak if consumers need stronger resume guarantees.

## Required Changes
- Define the intended durability level of offsets.
- Add periodic or threshold-based calls to `write_offset()` if stronger durability is desired.
- Document the tradeoff between persistence frequency and I/O cost.

## Acceptance Criteria
- Offset-loss risk on abnormal termination is reduced or explicitly documented as acceptable.
- Operators understand whether offsets are best-effort or durable checkpoints.

## Out of Scope
- Replacing file-based offsets with a new persistence backend

## Components
- `scripts/eventbus/offsets.py`
- `scripts/eventbus/app.py`
- Event Bus delivery-semantics documentation
- Offset-related tests
