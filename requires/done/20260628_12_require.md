# Remove note delegation APIs from `AgentSession`

## Priority
High

## Labels
agent, session, notes, cleanup

## Dependencies
- Delete `_NotesMixin` and `cmd_notes.py`

## Background
`AgentSession` currently creates a `NoteRepository` instance and exposes note-related delegation methods. These methods are used by the `/note` command and startup note injection.

## Task
Remove note-related APIs from `AgentSession`.

## Implementation Requirements
- Remove `from agent.note_repo import NoteRepository`.
- Remove `self._note_repo = NoteRepository()` from `AgentSession.__init__`.
- Remove the following methods from `AgentSession`:
  - `add_note`
  - `list_notes`
  - `delete_note`
  - `get_all_note_contents`
  - `pin_note`
  - `unpin_note`
  - `get_pinned_notes`
  - `search_notes`
- Keep session/message/diagnostic functionality unchanged.

## Acceptance Criteria
- `AgentSession` no longer depends on `NoteRepository`.
- `AgentSession` still supports session creation, message persistence, diagnostics, title updates, session listing, deletion, and undo.
- No note-related method remains on `AgentSession`.

## Source Context
`session.py` currently delegates note operations to `NoteRepository`, including CRUD, pin/unpin, search, and prompt-injection helpers.
