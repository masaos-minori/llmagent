# Align `tool_definitions_strict` startup semantics with MCP validation behavior

## Summary
Correct the Agent configuration documentation so that `tool_definitions_strict` matches the actual documented MCP startup-validation model.

## Background
The Agent config documentation currently describes stricter startup-abort semantics than the MCP operations docs. In particular, the “all servers unreachable” case is documented differently across layers.

## Objective
Make the Agent-layer docs consistent with the MCP-layer startup-validation rules.

## In Scope
- `05_agent_08_configuration.md`
- Cross-references to MCP validation behavior if present
- Any Agent startup documentation that summarizes strict validation

## Required Changes
- Update the `tool_definitions_strict` field description
- Ensure the following cases are described consistently:
  - tool mismatch
  - partial server unreachability
  - all servers unreachable
  - validation skipped
- Link to the canonical MCP behavior if needed

## Acceptance Criteria
- The Agent docs no longer claim startup abort on “all servers unreachable” if MCP docs say validation is skipped
- A reader can infer the final startup behavior without reading both document sets side by side
- Strict-mode behavior is described in a deterministic way

## Out of Scope
- Changing MCP validation logic
- Changing config key names

## Notes
Do not duplicate the full MCP behavior table unless necessary. Prefer one canonical behavior source plus a short Agent-facing summary.
