# [Medium] Resolve MDQ partial feature/config cleanup in service layer

## Priority

Medium

## Type

Implementation / Configuration Cleanup / Observability / Feature Gate Consistency

## Target files

- `mcp_servers/mdq/service.py`
- `mcp_servers/mdq/server.py`
- `mcp_servers/mdq/models.py`
- `mcp_servers/mdq/db_schema.py`
- `config/mdq_mcp_server.toml`
- MDQ service/config tests
- MDQ documentation

## Consolidated from

- Decide whether to remove or complete MDQ summary cache stub
- Remove unused MDQ `audit_log_path` setting or document active behavior
- Implement or remove `enable_refresh` feature gate
- Keep `enable_grep` as an active runtime feature gate

## Background

`service.py` reads summary-related configuration values such as `summary_cache_enabled`, `summary_threshold`, and `summary_model`. `get_chunk()` contains logic that reads from `chunk_summaries` and may start a background summary generation task. However, `_generate_and_cache_summary()` currently returns `None` when `summary_model` is `default`, and the actual LLM summary generation is only described as future work. 【6-51b4fa】

`service.py` also reads `audit_log_path`, while `server.py` performs audit logging through `_audit_log()` during tool calls and error handling. 【6-51b4fa】【1-bfd1ab】

`service.py` reads `enable_refresh`, but the provided `refresh_index()` path does not show a rejection check when `enable_refresh` is false. In contrast, `enable_grep` is actively checked by `grep_docs()`, which raises `MdqValidationError` when grep is disabled. 【6-51b4fa】

## Problem

Several MDQ service-layer settings appear partially implemented or potentially unused:

- summary cache can appear enabled even though generation is a stub,
- `audit_log_path` may not control actual MDQ audit output,
- `enable_refresh` may not gate refresh behavior,
- `enable_grep` is a valid active gate and should not be treated as dead config.

These concerns share the same target area: MDQ service configuration and runtime behavior. They should be handled together to avoid scattered cleanup and inconsistent config validation.

## Reason for change

Configuration must accurately describe runtime behavior. A setting that does nothing, only partially works, or silently starts a no-op background path increases operational risk.

The service layer should not expose feature flags unless they are active, tested, and documented. Conversely, active gates such as `enable_grep` should be preserved and documented to avoid accidental removal.

## Design intent

Clean up MDQ service configuration so each setting falls into one of three explicit categories:

- active and tested,
- deprecated with warning,
- removed and rejected by config validation.

No setting should silently imply behavior that does not exist.

## Implementation instructions

### Summary cache

Choose one path.

Option A: Remove the stub.

- Remove summary cache config reads.
- Remove `_generate_and_cache_summary()`.
- Remove summary cache branching from `get_chunk()`.
- Remove request fields that exist only for the stub, if applicable.
- Remove related docs and tests.

Option B: Complete the feature.

- Confirm `chunk_summaries` exists in MDQ schema.
- Implement actual summary generation.
- Implement cache writes.
- Handle cache miss deterministically.
- Avoid silent background failures.
- Add tests for cache hit, cache miss, disabled cache, and generation failure.

### Audit config

- Search all references to `audit_log_path`.
- If MDQ does not use `self.audit_log_path`, remove it.
- If retained for compatibility, emit a deprecation warning.
- Document that active MDQ audit logging is performed through `_audit_log()` in `server.py`.

### Refresh gate

Choose one path.

Option A: Make `enable_refresh` active.

- Add an early check in `refresh_index()`.
- Raise `MdqValidationError` when disabled.
- Add tests for enabled and disabled behavior.

Option B: Remove it.

- Remove the config read.
- Remove the config key from MDQ config.
- Add config validation for removed keys if applicable.

### Grep gate

- Keep `enable_grep` as an active setting.
- Add or preserve tests showing that disabled grep raises `MdqValidationError`.
- Document that `enable_grep` is not the same as unused config cleanup.

## Acceptance criteria

- No MDQ config key remains silently unused.
- No MDQ config key enables a partial no-op feature.
- Summary cache is either fully implemented or fully removed.
- MDQ audit behavior is documented accurately.
- `enable_refresh` either gates runtime behavior or is removed.
- `enable_grep` remains protected as an active gate.
- Tests cover the final behavior.

## Non-goals

- Do not change global audit behavior for non-MDQ servers.
- Do not implement a general background task system in this issue.
- Do not change search mode behavior here unless required by the separate search cleanup issue.
