# Align documentation with the no-backward-compatibility policy

## Priority

Medium

## Summary

Update documentation so it no longer describes or recommends compatibility-only import paths or shims.

## Background

Compatibility-only stubs and shims are being removed. Documentation must not continue to describe old import paths as valid or supported.

## Goal

Make documentation reflect the direct-import-only and no-compatibility-shim policy.

## Target files

- Agent configuration docs
- MCP configuration docs
- GitHub MCP docs
- Known-issues docs
- Import guide docs
- README files if applicable

## Scope

### In scope

- Remove `agent.config` import guidance.
- Remove `McpServerConfig._cast_enums()` compatibility description.
- Remove `mcp.github.service` import guidance.
- Remove package-level `mcp.github` service re-export guidance.
- Document official direct import paths.

### Out of scope

- Changing runtime behavior.
- Reintroducing deprecated import aliases.
- Keeping old paths as supported.

## Implementation tasks

- Replace `agent.config` examples with:
  - `agent.config_builders`
  - `agent.config_dataclasses`

- Replace `mcp.github.service` examples with:
  - `mcp.github.service_dispatch`
  - `mcp.github.service_init`

- Remove documentation that says existing imports continue to work.
- Remove documentation that suggests enum string casting happens inside `McpServerConfig`.
- State that TOML string enum conversion happens in config builder code, not inside the dataclass.

## Acceptance criteria

- Docs do not recommend `agent.config`.
- Docs do not recommend `mcp.github.service`.
- Docs do not recommend `from mcp.github import GitHubService`.
- Docs do not mention `_cast_enums()` as supported behavior.
- Docs describe only direct import paths.

## Validation

- Run:
```bash
grep -R "agent.config\|mcp.github.service\|from mcp.github import GitHubService\|_cast_enums\|compatibility shim\|re-export stub" -n docs scripts tests
```
- Review any remaining matches manually.

## Undecided items / confirmation required

- Confirm whether historical known-issues entries should be deleted or marked as resolved.
- Confirm whether docs should include a short migration note or avoid old paths entirely.
