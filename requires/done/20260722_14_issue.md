# 作業5: Shared / DB ドキュメント整理

## 対象

Shared / DB 関連ドキュメントを整理してください。

主な対象ファイル候補:

- `90_shared_00_document-guide.md`
- `90_shared_01_*`
- `90_shared_02_*`
- `90_shared_03_*`
- `90_shared_04_*`
- `90_shared_05_*`
- `90_shared_90_inconsistencies_and_known_issues.md`

## 重点確認項目

以下を重点的に確認してください。

- shared/ と db/ の責務説明が現行実装と一致しているか
- import direction / importlinter 制約の説明が実装・設定と一致しているか
- LLMMessage / RuntimeTool / RuntimeToolRegistry / ToolSpec / ToolCallResult の field 説明が現行コードと一致しているか
- ToolExecutor / ToolRegistry / RuntimeToolRegistry の責務説明が MCP 側ドキュメントと矛盾していないか
- ToolResultCache など実運用で使われていないコンポーネントの説明が過剰でないか
- 削除済みレガシーソースや旧設定ファイルの説明が本文に不要に残っていないか
- DB schema / helper / maintenance / recovery の説明が現行実装と一致しているか
- Shared 側と Agent / MCP 側で同一内容の説明が重複していないか

## 修正方針

- shared/ の型・DTO 説明は `90_shared_02_*` に集約する。
- ToolExecutor / ToolRegistry / RuntimeToolRegistry など shared runtime 説明は `90_shared_03_*` に集約する。
- DB schema は `90_shared_04_*` に集約する。
- DB operations は `90_shared_05_*` に集約する。
- MCP 固有の behavior は MCP ドキュメントへ参照リンクを張り、Shared 側で重複説明しすぎない。
- Agent 固有の behavior は Agent ドキュメントへ参照リンクを張り、Shared 側で重複説明しすぎない。
- 現行仕様に不要な履歴説明は削除する。

## 出力

- 修正対象ファイル
- 章・節
- 実装と異なる説明
- 他ドキュメントと重複する説明
- 集約先
- 削除する履歴説明
- 追加で必要な実装ファイル

基本方針は以下です。

1. 実装とドキュメントが矛盾する場合は、実装を正本としてドキュメントを修正する。
2. 現行仕様に影響しない過去経緯・移行履歴・旧仕様説明は削除する。
3. 実装で削除済みの機能、API、CLIコマンド、設定キー、DBカラム、型、関数、ファイルの説明は削除する。
4. 同一内容の説明が複数箇所にある場合は、正典となる1箇所に集約し、他は削除または参照リンクだけにする。
5. 判断できない場合は推測せず、必要な実装ファイル・設定ファイル・テストファイルを列挙する。
6. Known Issues に残すのは、現行仕様に影響する未解決事項だけにする。
7. 修正後は File Index、AI Query Routing Table、Related Documents、内部リンクも整合させる。
