# [High] Add path allowlist and index root jail to mdq-mcp

## Priority
High

## Labels
`mcp`, `mdq`, `security`, `path-jail`, `production-readiness`

## Dependencies
- Implement production-grade /health endpoint for mdq-mcp

## Background
`index_paths`, `refresh_index`, and `outline` currently accept file paths directly. Production MCP servers that access local files must enforce path allowlists and fail-closed behavior.

## Task
Implement MDQ path access control using configured allowed roots.

## Implementation Requirements
- Add MDQ path allowlist configuration to `config/mdq_mcp_server.toml`.
- Use one of the following names consistently:
  - `allowed_dirs`
  - `index_roots`
- Resolve all paths with `Path.resolve()` before authorization.
- Reject paths outside the configured allowed roots.
- Treat an empty allowlist as fail-closed.
- Apply path authorization to:
  - `index_paths`
  - `refresh_index`
  - `outline`
  - any future path-based MDQ tool
- Prevent `..` traversal and symlink escape.

## Acceptance Criteria
- Paths outside configured roots are denied.
- Symlink escape attempts are denied.
- Empty allowlist denies all file access.
- Tests cover allowed path, denied path, `..`, and symlink cases.
