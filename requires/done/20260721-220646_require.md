# Require: Fix SIGINT handler restoration when signal.signal() raises ValueError

## Priority

Low

## Target files

- `scripts/agent/http_lifecycle.py`

## Background

In `shutdown_all()`, the SIGINT handler is temporarily replaced with a custom handler (`_absorb_sigint_during_shutdown`) to prevent Ctrl-C from interrupting the shutdown process. After cleanup completes, the original handler is restored in a `finally` block.

## Problem

When `signal.getsignal(signal.SIGINT)` succeeds but `signal.signal(signal.SIGINT, handler)` raises `ValueError` afterward, the current code sets `old_sigint = None` in the `except ValueError:` block. This causes the `finally` block to skip restoring the original handler because `if old_sigint is not None:` evaluates to False. The original SIGINT handler is lost permanently.

The scenario:
1. `signal.getsignal(signal.SIGINT)` succeeds (not on main thread)
2. `signal.signal(signal.SIGINT, handler)` raises `ValueError` (rare case)
3. `old_sigint = None` is set in the except block
4. Original SIGINT handler is never restored

## Reason for change

Even though this is a rare edge case, losing the SIGINT handler can cause the application to become unresponsive to Ctrl-C indefinitely. The fix is simple and eliminates the risk.

## Implementation intent

Separate the `getsignal` call from the `signal` call so that the original handler is always saved before attempting to install the new one.

## Implementation instructions

In `scripts/agent/http_lifecycle.py`, around line 335-341, replace:

```python
old_sigint: object | None = None
try:
    old_sigint = signal.getsignal(signal.SIGINT)
    signal.signal(signal.SIGINT, self._absorb_sigint_during_shutdown)
except ValueError:
    # Not on the main thread â€” proceed without the guard rather than fail shutdown.
    old_sigint = None
```

with:

```python
old_sigint: object | None = None
try:
    old_sigint = signal.getsignal(signal.SIGINT)
except ValueError:
    # Not on the main thread â€” cannot get the current handler.
    old_sigint = None

try:
    signal.signal(signal.SIGINT, self._absorb_sigint_during_shutdown)
except ValueError:
    # Not on the main thread â€” proceed without the guard rather than fail shutdown.
    # old_sigint may still be valid from the previous try block above.
    pass
```

This ensures `old_sigint` is saved before any potential exception, and the restore logic in the `finally` block works correctly regardless of which step fails.

## Acceptance criteria

- When both `getsignal` and `signal` succeed, behavior is unchanged
- When `getsignal` fails, no handler is installed or restored (unchanged behavior)
- When `getsignal` succeeds but `signal` fails, the original handler is still restored via the `finally` block
- No regression in shutdown behavior under normal operation

## Tests

- Unit test: Mock `signal.signal` to raise ValueError after successful `getsignal`, verify original handler is restored
- Unit test: Mock `signal.getsignal` to raise ValueError, verify no handler is installed
- Integration test: Simulate SIGINT during shutdown, verify handler is properly absorbed and restored
