## Title

Restrict approval dry-run tools to dry-run-capable tools

## Priority

High

## Target files

- `agent/tool_approval.py`
- `agent/tool_result_formatter.py`
- `agent/config_builders.py`
- `agent/config_dataclasses.py`
- `agent/services/config_validators.py`
- `tests/test_tool_approval_preflight.py`
- `tests/test_tool_approval_repos.py`
- `tests/test_tool_approval_risk.py`

## Background

Approval preview can optionally execute a dry-run by injecting `dry_run=True` into tool arguments.

This is controlled by `approval_dry_run_tools`.

The default dry-run tools are file-oriented, but configuration may include GitHub, Git, CICD, or shell tools.

In the current approval flow, if dry-run returns `is_error=True`, HIGH-risk operations are denied before the interactive prompt.

## Problem

If a tool does not support `dry_run` but is listed in `approval_dry_run_tools`, the preview can fail.

For HIGH-risk tools, this can cause false denial before the user has a chance to approve.

## Reason for change

Dry-run preview is a safety and usability feature. It should not make unrelated tools unusable simply because they do not support the `dry_run` argument.

Unsupported dry-run and dangerous dry-run results must be treated differently.

## Implementation intent

Restrict dry-run execution to tools that are known to support it.

For other tools, use static preview only.

## Implementation instructions

- Define a canonical list or capability flag for dry-run-capable tools.
- Validate `approval_dry_run_tools` at startup.
- Decide behavior for unsupported entries:
  - reject startup, or
  - warn and ignore unsupported entries
- Prefer warning and ignoring unsupported entries unless strict config is enabled.
- Do not call `tools.execute(..., dry_run=True)` for unsupported tools.
- Use `build_preview()` for unsupported tools.
- Distinguish dry-run failure categories:
  - unsupported dry-run argument
  - transport or server failure
  - valid dry-run result that reports a blocking error
- Keep HIGH-risk blocking behavior only for meaningful dry-run blocking results.

## Acceptance criteria

- Only dry-run-capable tools receive `dry_run=True`.
- Unsupported tools use static preview.
- HIGH-risk unsupported tools are not denied solely due to missing dry-run support.
- Config validation reports unsupported dry-run tool names.
- Existing supported file dry-run behavior remains unchanged.

## Tests

Add or update tests for:

- Supported dry-run tool appends dry-run output to preview.
- Unsupported dry-run tool uses static preview.
- Unsupported dry-run tool does not call `tools.execute()` with `dry_run=True`.
- HIGH-risk unsupported tool is not denied only due to unsupported dry-run.
- HIGH-risk supported tool with real dry-run blocking error is denied.
