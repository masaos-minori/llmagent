# [High] Rewrite 04_mcp_06_configuration_and_operations.md for HTTP-only McpServerConfig

## Priority

High

## Goal

Make configuration documentation match the current McpServerConfig dataclass.

## Background

04_mcp_06_configuration_and_operations.md still documents stdio fields and modes, including TransportType.STDIO, cmd, ondemand, ping_tool, working_dir, env, and stdio subprocess operation. The current McpServerConfig is HTTP-only.

## Scope

Affected files:

- docs/04_mcp_06_configuration_and_operations.md
- shared/mcp_config.py
- tests/test_mcp_config.py

## Tasks

- Update the McpServerConfig Fields section.
- Document only current fields:
  - transport
  - url
  - startup_mode
  - healthcheck_mode
  - call_timeout_sec
  - startup_timeout_sec
  - tool_names
  - auth_token
  - role
- State that transport only supports http.
- State that startup_mode only supports persistent and subprocess.
- State that healthcheck_mode only supports http.
- Remove unsupported fields:
  - cmd
  - idle_timeout_sec
  - working_dir
  - env
- Remove stdio operation examples.
- Remove stdio timeout from default values.
- Add call_timeout_sec to major default values.

## Unknowns / Open Questions

- Confirm whether cmd exists elsewhere for HTTP subprocess lifecycle.
- Confirm whether config/mcp_servers.toml still contains ignored legacy fields.

## Acceptance Criteria

- 04_mcp_06 matches shared/mcp_config.py.
- No stdio-only config field appears in active configuration docs.
- Operators can configure only supported HTTP modes from the documentation.
