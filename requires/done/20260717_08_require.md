# Reapply Agent policy to RuntimeToolRegistry after /reload

## Priority

Medium

## Target files

- agent/services/config_reload.py
- shared/runtime_tool_registry.py
- agent/context.py
- tests/agent/services/test_config_reload.py

## Reason

`/reload` is limited to Agent configuration reload. It must not reload MCP server settings, rediscover MCP tools, or rebuild RuntimeToolRegistry.

However, Agent policy fields such as `allowed_tools`, `tool_safety_tiers`, and approval rules may change.

## Intent

Allow `/reload` to update policy annotations on the existing runtime tool set.

## Scope

After successful config reload, call:

- `runtime_tools.apply_policy(ctx.cfg)`

Update only:

- `enabled_for_llm`
- `requires_approval`
- `agent_safety_tier`

Do not update:

- discovered tool set
- server_key
- server_url
- raw schema
- discovery metadata

## Acceptance criteria

- `/reload` does not call `/v1/tools`.
- `/reload` does not rebuild RuntimeToolRegistry.
- Agent policy changes are reflected in LLM-visible tools.
- MCP server changes remain restart-required.
