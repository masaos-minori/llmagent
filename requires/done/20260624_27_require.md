# Refine the MCP document guide routing table to point to the most specific chapter

## Summary
Improve the MCP document guide so that common questions route to the most precise and least ambiguous chapter.

## Background
The guide already provides a routing table, but several topics span multiple chapters and can be routed more precisely.

## Objective
Reduce navigation ambiguity for both humans and AI systems.

## In Scope
- `04_mcp_00_document-guide.md`

## Required Changes
- Review routing-table entries for:
  - mdq-mcp implementation status
  - strict startup validation
  - watchdog behavior
  - fail-open / fail-closed controls
- Point each question to the most specific document first
- Use secondary references only where genuinely necessary

## Acceptance Criteria
- The routing table sends readers to the most authoritative chapter for each question
- Cross-references are reduced where a better primary target exists
- The guide no longer uses wording that can mislead readers about mdq-mcp status

## Out of Scope
- Rewriting the full document set
- Changing file names

## Notes
Optimize for precision. The guide should minimize unnecessary multi-hop navigation.
