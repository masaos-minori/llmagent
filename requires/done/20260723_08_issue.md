# Agent Startup Processing — Asymmetric Error Handling for Memory Injection During Startup

## Summary

If `on_session_start()` raises an exception during startup, the entire startup sequence fails because no try/except wraps this call. However, the memory layer is optional (`use_memory_layer=False` disables it), so this creates asymmetric behavior where production deployments with memory enabled fail hard, while those without memory continue silently.

## Severity

Medium

## Files

- `scripts/agent/startup.py:348-365`

## Details

In `StartupOrchestrator._setup_prompt()`:

```python
async def _setup_prompt(self) -> None:
    ctx = self._ctx
    initial_prompt = ctx.cfg.tool.system_prompts.get(
        ctx.conv.system_prompt_name,
        ctx.cfg.tool.system_prompt_tool,
    )
    if ctx.services_required.memory is not None:
        memory_snippets = ctx.services_required.memory.on_session_start(
            ctx.session.session_id,
        )
        if memory_snippets:
            max_snippets = ctx.cfg.agent_memory_max_startup_snippets
            if len(memory_snippets) > max_snippets:
                logger.warning(...)
                memory_snippets = memory_snippets[:max_snippets]
            memory_block = "\n\n[Relevant memories]\n" + "\n".join(...)
            initial_prompt = initial_prompt + memory_block
    ctx.conv.system_prompt_content = initial_prompt
    ctx.conv.history = [{"role": "system", "content": initial_prompt}]
```

If `on_session_start()` raises an exception (e.g., DB connection error, network timeout), the exception propagates up through `StartupOrchestrator.run()`:

```python
try:
    await self._start_servers()
    _servers_started = True
    await self._check_services()
    await self._recover_pending_approvals()
    await self._setup_prompt()  # <-- exception here
except Exception as setup_err:
    if _servers_started:
        try:
            await self._ctx.services_required.lifecycle.shutdown_all()
        except Exception as shutdown_err:
            logger.error("CRITICAL: Startup rollback FAILED...")
    raise setup_err
```

The rollback path shuts down MCP subprocesses, but the error message shown to the operator is the raw exception from `on_session_start()`, which may be cryptic (e.g., "connection refused"). This is especially problematic because:

1. Production deployments with memory enabled fail hard — no fallback
2. Deployments without memory continue silently — no warning about the missing feature
3. The operator has no indication that memory injection failed vs. never attempted

## Reproduction Steps

1. Configure `use_memory_layer=True` in config
2. Make the memory backend unreachable (e.g., stop the embedding service)
3. Start the agent REPL
4. Observe that startup fails with a cryptic error instead of continuing without memory

## Expected Behavior

Memory injection failures should be treated as non-fatal warnings, similar to other optional services. The agent should continue operating without memory snippets when the memory layer is unavailable.

## Proposed Fix

Wrap the memory injection in a try/except and emit a warning instead of failing:

```python
async def _setup_prompt(self) -> None:
    ctx = self._ctx
    initial_prompt = ctx.cfg.tool.system_prompts.get(
        ctx.conv.system_prompt_name,
        ctx.cfg.tool.system_prompt_tool,
    )
    if ctx.services_required.memory is not None:
        try:
            memory_snippets = ctx.services_required.memory.on_session_start(
                ctx.session.session_id,
            )
            if memory_snippets:
                max_snippets = ctx.cfg.agent_memory_max_startup_snippets
                if len(memory_snippets) > max_snippets:
                    logger.warning(...)
                    memory_snippets = memory_snippets[:max_snippets]
                memory_block = "\n\n[Relevant memories]\n" + "\n".join(...)
                initial_prompt = initial_prompt + memory_block
        except Exception as exc:
            logger.warning(
                "Memory injection failed during startup: %s; continuing without memory",
                exc,
            )
            self._view.write_warning(f"{OutputTag.NON_FATAL} Memory injection failed: {exc}")
    ctx.conv.system_prompt_content = initial_prompt
    ctx.conv.history = [{"role": "system", "content": initial_prompt}]
```
