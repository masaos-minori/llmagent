# [High] Migrate all MCP tool schema exports from _MCP_TOOLS to TOOL_LIST

## Goal

Make `TOOL_LIST` the only valid MCP tool schema export and remove `_MCP_TOOLS` completely.

## Background

The MCP tool schema export policy defines `TOOL_LIST` as the canonical export name. `_MCP_TOOLS` must be removed, including compatibility aliases.

## Scope

- MCP server tool schema modules
- MCP server imports
- Tests
- Compatibility checks

## Implementation Tasks

- Update all `mcp/*/tools.py` files:
  - Rename `_MCP_TOOLS` to `TOOL_LIST`.
  - Do not keep `_MCP_TOOLS = TOOL_LIST`.
  - Do not keep any `_MCP_TOOLS` compatibility alias.
- Update all `mcp/*/server.py` imports to import `TOOL_LIST`.
- Update MCP server classes that set `mcp_tools`.
- Update tests that reference `_MCP_TOOLS`.
- Update `scripts/checks/check_no_compat.py` to reject `_MCP_TOOLS`.
- Update `04_mcp_07_tool_schema_export_policy.md` from migration-plan wording to current-policy wording.

## Acceptance Criteria

- `rg -n "_MCP_TOOLS"` returns no active code, test, or active doc matches.
- Every MCP tool schema module exports `TOOL_LIST`.
- No compatibility alias remains.
- MCP `/v1/tools` still returns the expected tool list for every server.

## Validation

- Run MCP service tests.
- Run tool schema tests.
- Run `python scripts/checks/check_no_compat.py`.
- Run `rg -n "_MCP_TOOLS" .` and confirm no active matches.
