# Document `messages.tool_call_id` as an active column

## Priority

High

## Summary

Update documentation to state that `messages.tool_call_id` is actively used.

## Background

`messages.tool_call_id` is saved and restored by `SessionMessageRepository`.

`MessageRow` also includes `tool_call_id`.

Therefore, documentation that marks this column as unused is incorrect.

## Goal

Align documentation with current message persistence behavior.

## Target files

- `90_shared_04_db_architecture_and_schema.md`
- `90_shared_05_db_api_and_operations.md`
- `90_shared_90_inconsistencies_and_known_issues.md`

## Scope

### In scope

- Update schema documentation.
- Update DB API documentation.
- Update known-issues documentation.

### Out of scope

- Removing `messages.tool_call_id`.
- Changing message persistence behavior.

## Implementation tasks

- Remove any statement that says `messages.tool_call_id` is unused.
- Document `messages.tool_call_id` as:
  - used for `tool` role messages
  - used to correlate a tool response with an assistant tool call
  - persisted and restored by `SessionMessageRepository`
- Ensure `LLMMessage.tool_call_id` and `messages.tool_call_id` are described consistently.

## Acceptance criteria

- No documentation says `messages.tool_call_id` is unused.
- Documentation explains the relation between `tool_calls` and `tool_call_id`.
- Known issue entry, if present, is removed or marked resolved.

## Validation

- Search for stale unused-column statements:
```bash
grep -R "tool_call_id.*UNUSED\|UNUSED.*tool_call_id" docs scripts -n
```

## Confirmation required

- Confirm whether `tool_call_id` should be required for all `tool` role messages or only stored when available.
