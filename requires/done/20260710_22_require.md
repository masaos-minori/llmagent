# [High] Align ToolCallResult and plugin execution contracts with implementation and tests

## Background

`ToolCallResult` includes:

- `output`
- `is_error`
- `request_id`
- `server_key`
- `source`
- `error_type`

MCP transport results use `source="mcp"`. Plugin tools use `source="plugin"`. Plugin return contract violations use `error_type="plugin_contract"`.

`PluginLoadResult` uses `command_shadows_rejected`, not `command_shadows`.

Plugin strict mode attempts all plugins first, then raises one aggregated `PluginLoadError` if there are load failures or strict conflict rejections.

## Goal

Make DTO, plugin execution, plugin loading, and documentation contracts consistent.
