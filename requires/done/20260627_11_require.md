## Summary
Add cross-cutting regression tests that protect key RAG invariants across ingestion, schema ownership, consistency repair guidance, HTTP mode, RRF diagnostics, and Refiner diagnostics.

## Background
RAG behavior spans ingestion files, SQLite schema, query pipeline stages, HTTP mode, semantic cache, and operations commands. Unit tests alone may not catch integration regressions.

## Problem
Important documented invariants can regress across component boundaries, especially around `.json` artifact lifecycle, schema ownership, consistency checks, HTTP mode, and diagnostics.

## Required Changes
- Add tests for crawler → chunk splitter → ingester `.json` lifecycle.
- Add tests for local file SHA-256 re-ingestion behavior.
- Add tests that RAG schema does not require Agent session tables.
- Add tests for `/db rebuild-fts` repair flow.
- Add tests for `remote_empty` vs `in_process_fallback`.
- Add tests for `use_rrf=False` diagnostics and ranking degradation.
- Add tests for Refiner fallback diagnostics.
- Use temporary directories and isolated SQLite databases.
