# [MCP][Watchdog] Enable and extend watchdog-based self-healing

## Summary
The watchdog exists, but it is disabled by default and mainly described for subprocess-managed HTTP servers.

## Problem
Most servers are persistent HTTP processes, so default self-healing coverage is limited.

## Scope
- watchdog configuration
- lifecycle router
- operational defaults

## Task
- define a production profile that enables watchdog checks
- add health probing for persistent HTTP servers
- align watchdog restart behavior with health-registry state once health transitions are fixed

## Expected Behavior
- production deployments have watchdog coverage by default
- common server failures can be detected and recovered automatically

## Acceptance Criteria
- [ ] production config enables watchdog checks
- [ ] persistent HTTP servers are probed
- [ ] restart behavior works for common failure cases
