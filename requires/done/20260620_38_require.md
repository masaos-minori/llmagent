# Title
Improve explicitness of ingestion file formats in `rag-src` and chunk outputs

## Summary
Reduce ambiguity around ingestion artifacts that use `.txt` extensions while actually containing JSON payloads.

## Background
Crawler outputs and chunk files use `.txt` filenames even though their content is structured JSON.

## Problem
This can confuse operators, scripts, backup tooling, and any external process that assumes `.txt` implies plain text rather than JSON content.

## Required Changes
- Evaluate whether file naming or extension conventions should be clarified or improved.
- At minimum, make the “extension is `.txt`, content is JSON” rule extremely explicit in docs and operational guidance.
- Review whether metadata or naming changes can improve accidental misuse resistance without breaking compatibility.

## Acceptance Criteria
- Operators can easily tell that these files are JSON-backed artifacts, not plain text notes.
- Documentation makes the format unambiguous.
- Any chosen compatibility-preserving improvement is clearly documented.

## Out of Scope
- Full redesign of the ingestion artifact pipeline
- Breaking file-path compatibility without migration planning

## AI Implementation Instruction
Implement this as a format-clarity and operability task.

Strict rules:
- Do not break compatibility casually.
- If extensions remain unchanged, documentation must become explicit enough to prevent misuse.
- Prefer minimal-risk improvements over broad artifact redesign.
- If extension changes are considered, mark them as: Needs confirmation unless a migration plan is included

Deliverables:
1. Improved clarity around artifact format
2. Documentation and operator guidance updates
3. Optional compatibility-safe naming improvement proposal
