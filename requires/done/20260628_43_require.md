# Update `/db` command docstring to scoped-only forms

## Priority
High

## Labels
agent, commands, db, cleanup, breaking-change

## Dependencies
None

## Background
`cmd_db.py` already dispatches only scoped DB commands:

- `/db rag ...`
- `/db session ...`
- `/db help`

Flat forms such as `/db urls`, `/db clean`, `/db rebuild-fts`, and `/db recover` are not dispatched by the current implementation.

## Task
Update the `_cmd_db()` docstring so it no longer suggests flat `/db <subcmd>` forms.

## Implementation Requirements
- Update the `_cmd_db()` docstring in `cmd_db.py`.
- Replace wording like:
  - `Handle /db [rag|session] <subcmd> | /db <subcmd>.`
- Use wording like:
  - `Handle /db rag <subcmd>, /db session <subcmd>, or /db help.`
- Do not add or restore any flat DB aliases.

## Acceptance Criteria
- `/db urls` remains invalid.
- `/db clean <url>` remains invalid.
- `/db rebuild-fts` remains invalid.
- `/db recover` remains invalid.
- `/db rag ...`, `/db session ...`, and `/db help` continue to work.
