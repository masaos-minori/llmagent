# Title
Change `crawler.py` and `chunk_splitter.py` output file extensions from `.txt` to `.json`

## Summary
Update the RAG ingestion pipeline so crawler output files and chunk files use the `.json` extension instead of `.txt`.

## Background
Current crawler output files and chunk files contain JSON data, but their filenames use the `.txt` extension.
This makes the file format less explicit for operators, tools, and documentation.

## Problem
The current `.txt` extension is misleading because the file content is structured JSON.
This increases the risk of confusion in manual inspection, tooling, backup handling, and documentation.

## Required Changes
- Change crawler output filenames from `.txt` to `.json`
- Change chunk splitter output filenames from `.txt` to `.json`
- Update ingester logic to read `.json` files instead of `.txt`
- Keep `registered/` output filenames consistent with the new `.json` extension
- Update all related documentation and examples
- Define and document how existing `.txt` files should be handled during migration

## Acceptance Criteria
- Crawler outputs use `.json`
- Chunk outputs use `.json`
- Ingester works correctly with `.json` files
- Documentation no longer describes JSON content with `.txt` filenames
- Migration or compatibility behavior for old `.txt` files is documented
- Related tests are updated

## Out of Scope
- Changing the JSON payload schema
- Redesigning the RAG database schema
- Changing query pipeline behavior
- Switching to `.jsonl` or another file format

## Components
- `scripts/rag/ingestion/crawler.py`
- `scripts/rag/ingestion/chunk_splitter.py`
- `scripts/rag/ingestion/ingester.py`
- `03_rag_01_system_overview.md`
- `03_rag_02_ingestion_pipeline.md`
- `03_rag_04_data_model_and_interfaces.md`
- `03_rag_05_configuration_and_operations.md`
- related tests
