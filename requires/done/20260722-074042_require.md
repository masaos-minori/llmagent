# StateStore Internal Implementation Dependency

## Priority

Low

## Target Files

- `scripts/agent/startup.py` (line 310)
- `scripts/agent/state_store.py` (needs new public API method)

## Background

`StartupOrchestrator._recover_pending_approvals()` accesses `StateStore._db` directly to pass the database connection to `find_latest_pending_approval()`. The `_db` attribute is private (single underscore prefix), indicating internal implementation detail.

## Problem

In `_recover_pending_approvals()` at line 310:

```python
result = find_latest_pending_approval(store._db)
```

The issue is twofold:
1. **Internal dependency**: `StateStore._db` is a private attribute. Accessing it violates encapsulation — any change to `StateStore`'s internal representation will break this code silently.
2. **Incomplete recovery**: `find_latest_pending_approval()` returns only the latest pending approval. If multiple approvals are pending, earlier ones are lost.

## Reason for Change

- Private attribute access creates fragile coupling between modules
- Future refactoring of `StateStore` may break this without compile-time detection
- Only one pending approval is recovered while others are silently discarded

## Implementation Intent

Add a public method to `StateStore` that exposes the database connection for external queries. Consider whether all pending approvals should be recovered instead of just the latest one.

## Implementation Instructions

Refactor `StateStore` to add a public accessor method:

```python
class StateStore:
    def get_connection(self):
        """Return the underlying DB connection for external queries.
        
        This method exists to support cross-module queries that need direct
        access to the connection without duplicating initialization logic.
        """
        return self._db
```

Update `startup.py` line 310:

```python
result = find_latest_pending_approval(store.get_connection())
```

Additionally, consider fixing the incomplete recovery by using a method that returns all pending approvals:

```python
async def _recover_pending_approvals(self) -> None:
    """Restore workflow approval-pending state from a previous session."""
    ctx = self._ctx
    if ctx.workflow is None:
        return
    store = StateStore()
    try:
        # Recover ALL pending approvals, not just the latest
        results = find_all_pending_approvals(store.get_connection())
    finally:
        store.close()
    for result in results or []:
        # Process each pending approval
        ...
```

Key points:
- Replace all `_db` access with public methods
- Document the purpose of the accessor method clearly
- Consider adding similar accessor methods to other stores if needed

## Acceptance Criteria

1. No private attributes (`_db`) are accessed outside their defining class
2. `StateStore` has a documented public method for external query access
3. All pending approvals are recovered, not just the latest one
4. Existing tests still pass after the refactor

## Tests

- Unit test: verify `get_connection()` returns the same object as `_db`
- Integration test: create multiple pending approvals, verify all are recovered during startup
