# Add tests for plugin registration-time validation and PluginToolInvoker

## Priority

Medium

## Goal

Add regression tests for plugin registration validation and plugin invocation behavior.

## Scope

Test files:

- `tests/shared/test_plugin_tool_registration.py`
- `tests/shared/test_plugin_tool_invoker.py`

## Test cases

Registration validation:

- Plugin with return annotation `tuple[str, bool]` registers successfully.
- Plugin without return annotation fails registration.
- Plugin with return annotation `tuple[str, str]` fails registration.
- Invalid plugin is not added to the tool registry.
- Strict mode aggregates plugin load failures and raises `PluginLoadError`.
- Non-strict mode records failure and continues.

Plugin invoker:

- Missing plugin returns `None`.
- Valid plugin returns `ToolCallResult`.
- Plugin exception returns `ToolCallResult(is_error=True, error_type="tool")`.
- Plugin result uses `request_id=""`.
- Plugin result uses `server_key=""`.
- Plugin execution bypasses cache.
- Plugin execution bypasses MCP routing.

## Acceptance criteria

- Tests cover both registration-time and runtime defensive behavior.
- Tests pass without requiring real MCP servers.
- Tests do not depend on plugin load order unless explicitly needed.
- Plugin registry state is reset between tests.

## Out of scope

- Testing plugin command dispatch.
- Testing pipeline stage plugins.
