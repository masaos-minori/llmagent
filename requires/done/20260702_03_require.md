## [docs][tool-loop-guard] ToolLoopGuard の dedup/cycle/retry 動作説明を修正する

### Summary
`docs/05_agent_03_turn-processing-flow.md` には `ToolLoopGuard` の挙動として「→ inject hint to LLM」と記載されているが、実際のコードは hint を LLM コンテキストに注入せず、診断チャンネルに保存してループを終了する。ドキュメントと実装が乖離しており、実装者・AI エージェントが誤った動作を期待する可能性がある。

### Background
`ToolLoopGuard` (`tool_loop_guard.py`) は LLM ツールコールループの無限ループを防ぐガードである。`DEDUP_HINT`、`CYCLE_HINT`、`RETRY_HINT` という 3 つのヒント文字列が定義されているが、これらは `_save_guard_hint()` を経由して `session_diagnostics` テーブルに保存されるのみで、LLM の会話履歴 (`ctx.conv.history`) には追加されない。ループが終了するとユーザーには終了メッセージ (`TurnResult(action="fail", answer=msg)`) が返される。

### Problem
`docs/05_agent_03_turn-processing-flow.md:100–103` の記述:
```
- **Dedup:** same (name, args) seen > `tool_dedup_max_repeats` times → inject hint
- **Cycle detection:** same tool sequence in last `tool_cycle_detect_window` rounds → warn
- **Retry:** errored (name, args) called again > `tool_error_retry_max` → block
```
の「inject hint」という記述が実装と一致していない。`llm_turn_runner.py:83–89` では guard の返り値が非 `None` の場合に即 `TurnResult(action="fail")` で返却しており、LLM history への注入は行われない。

### Scope
- `docs/05_agent_03_turn-processing-flow.md` の `ToolLoopGuard` セクションの記述修正
- コードの変更は範囲外 (どちらを正とするかの設計判断を先に確認すること)

### Task
設計意図を確認したうえで、以下のいずれかを実施する:

**オプション A: ドキュメントを実装に合わせる (推奨)**
1. `docs/05_agent_03_turn-processing-flow.md:100–103` の ToolLoopGuard セクションを以下のように更新する:
   - 「inject hint」→「terminate loop and return message to user; hint stored in session_diagnostics」
   - 実際の終了メッセージ例 (`"Repeated tool call detected."`) を記載する
2. 診断チャンネルへの保存経路 (`session_diagnostics` テーブル、`kind="guard_hint"`) を注記する

**オプション B: 実装をドキュメントに合わせる (LLM への hint 注入を実装)**
1. `tool_loop_guard.py` の `check_dedup()`、`check_cycle()`、`check_retry()` を修正し、guard hit 時に `DEDUP_HINT`/`CYCLE_HINT`/`RETRY_HINT` を `{"role": "system", "content": hint, "_ephemeral": True}` として `ctx.conv.history` に追加する
2. 各 `check_*` メソッドが `None` を返してループを継続させるか、`TurnResult` の action を変更するかを決定する
3. `_ephemeral` マーカーは `_sync_system_prompt()` が次ターン開始時に除去するため、LLM context には 1 ターンのみ残る
