# [High] Integrate Event Bus DDL into db/schema_sql.py

## Goal

Move Event Bus schema DDL into `db/schema_sql.py` so that all DB DDL is managed by a common schema source.

## Background

The current Event Bus schema exists as a standalone `schema.sql` file. It defines the `events` table and indexes. `retry_count` is marked as deprecated but must remain as part of the existing schema. 【3-acbc81】

## Scope

- DDL consolidation only
- No Event Bus Python runtime integration
- No Event Bus repository, publisher, subscriber, or dispatcher

## Implementation Tasks

- Add `_EVENTBUS_SCHEMA` to `db/schema_sql.py`.
- Add `build_eventbus_schema_sql() -> str`.
- Move the `events` table DDL into `_EVENTBUS_SCHEMA`.
- Move these indexes into `_EVENTBUS_SCHEMA`:
  - `idx_events_topic`
  - `idx_events_seq`
  - `idx_events_dlq_at`
  - `idx_events_dlq_seq`
- Keep `retry_count` in the schema.
- Keep `retry_count` marked as deprecated.
- Document that new runtime code must use `delivery_failure_count`.
- Do not remove `delivery_failure_count`.
- Do not remove `dlq_requeue_count`.
- Do not remove `dlq_at`.

## Timestamp Policy

Unify Event Bus timestamp format with the Shared/DB policy.

- `published_at` should use ISO-8601 UTC Z suffix.
- If a DEFAULT is added, use:
  - `strftime('%Y-%m-%dT%H:%M:%SZ', 'now')`
- `acked_at` and `dlq_at` remain nullable.
- Future runtime writers must write timestamp values using ISO-8601 UTC Z suffix.

## Acceptance Criteria

- `db/schema_sql.py` contains `build_eventbus_schema_sql()`.
- Event Bus DDL is no longer maintained only in a standalone SQL file.
- `retry_count` remains in the schema as deprecated.
- Event Bus timestamp policy is documented.
- No Event Bus runtime code is introduced.

## Validation

- Run schema SQL tests.
- Verify `build_eventbus_schema_sql()` returns valid SQL.
- Apply the returned SQL to an empty SQLite database and confirm:
  - `events` table exists
  - all expected indexes exist
