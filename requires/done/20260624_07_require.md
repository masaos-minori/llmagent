# Add explicit artifact metadata to JSON payloads stored with `.txt` extensions

## Summary
Improve artifact self-description by adding explicit metadata fields to crawler, chunk, and registered JSON payloads.

## Background
RAG artifacts use `.txt` filenames while storing JSON payloads. This is documented, but the payloads themselves do not visibly expose schema identity in the document examples.

## Objective
Make each artifact self-describing and easier to validate programmatically.

## In Scope
- `scripts/rag/ingestion/crawler.py`
- `scripts/rag/ingestion/chunk_splitter.py`
- `scripts/rag/ingestion/ingester.py`
- `03_rag_02_ingestion_pipeline.md`
- `03_rag_04_data_model_and_interfaces.md`

## Required Changes
- Add explicit metadata fields such as:
  - `schema_version`
  - `artifact_type`
  - `created_by`
- Ensure all emitted JSON artifacts include these fields
- Add validation on artifact read paths
- Update the documented example payloads

## Acceptance Criteria
- Each artifact payload can be identified without relying only on its file path
- Read paths fail clearly when metadata is invalid or missing
- Documentation examples include the new metadata fields
- Existing artifact categories remain distinguishable

## Out of Scope
- Renaming `.txt` artifacts to `.json`
- Backfilling historical files unless needed for compatibility

## Implementation Notes
Design the metadata so that future schema migrations can be introduced cleanly.
