# [RAG][DB] Add consistency checks for chunks_fts and chunks_vec

## Summary
Derived tables can become inconsistent, but there is no clear automated consistency check.

## Problem
- `chunks_fts` depends on trigger-based sync
- `chunks_vec` has no foreign key constraint
- force reinsert depends on correct delete order

## Scope
- DB maintenance logic
- ingestion maintenance helpers
- RAG operational checks

## Task
Add checks for:
- `chunks` count vs `chunks_fts` count
- orphan records in `chunks_vec`
- consistency after force reinsert

## Expected Result
Derived table problems are detected early.

## Acceptance Criteria
- [ ] FTS consistency check exists
- [ ] vector-table orphan check exists
- [ ] maintenance docs mention these checks
