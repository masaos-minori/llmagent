## [Medium][docs][shell] Clarify shell sandbox default vs effective production value

### Summary
The documentation is inconsistent about whether `shell_sandbox_backend="none"` is the default, or whether production effectively defaults to `firejail`.

### Problem
Operators may misunderstand whether the system is safe by default or only safe after profile-based enforcement.

### Scope
- docs/04_mcp_04_server_catalog.md
- docs/04_mcp_05_security_and_safety_model.md
- docs/04_mcp_06_configuration_and_operations.md

### Task
- define one exact meaning for:
  - raw config default
  - local/default development posture
  - production effective requirement
- update all docs to use the same wording
- explain clearly whether production safety is config default, profile override, or startup enforcement
- remove conflicting wording

### Expected Result
The docs clearly distinguish raw default values from production-enforced behavior.

### Acceptance Criteria
- shell sandbox semantics are described consistently across all docs
- local/dev behavior and production behavior are clearly separated
- there is no conflicting wording about the default anymore
