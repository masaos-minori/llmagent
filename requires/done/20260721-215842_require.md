# Require: Add warning log when memory branch resolution fails

## Priority

Medium

## Target files

- `scripts/agent/factory.py`

## Background

The memory service resolves branch context at build time via `get_repo_info()`. When Git repository info retrieval fails, the branch falls back to empty string (global scope) silently without any log output.

## Problem

When `get_repo_info()` fails, `_git.success` becomes False and `_branch = ""` is applied silently. No WARNING or ERROR log is emitted. Operators cannot tell if branch-scoped memory injection has been silently disabled. Falling back to "global scope" risks injecting unintended memory.

## Reason for change

Operators need visibility into silent fallback behavior. Without logging, they cannot diagnose why branch-scoped memory injection is not working as expected.

## Implementation intent

Add a WARNING-level log message when `get_repo_info()` fails during memory branch resolution, making the silent fallback observable.

## Implementation instructions

In `scripts/agent/factory.py`, around line 297-301, add a WARNING log when `_git.success` is False:

```python
_git = get_repo_info()
_branch = ""
if _git.success and _git.data:
    _raw = _git.data.get("branch", "")
    _branch = "" if _raw == "HEAD (detached)" else _raw
else:
    _logger.warning(
        "Memory branch resolution failed: %s; falling back to global scope",
        _git.failure_reason if _git.failure_reason else "unknown",
    )
```

Use the same `logger` instance already available in the module.

## Acceptance criteria

- When `get_repo_info()` succeeds, no additional log is emitted (behavior unchanged)
- When `get_repo_info()` fails, a WARNING-level log message is emitted indicating the failure reason and that global scope fallback was used
- Branch-scoped memory injection continues to work correctly when Git repo info is available
- No errors or warnings are introduced when running in non-Git directories (expected behavior)

## Tests

- Unit test: Mock `get_repo_info()` to return failure, assert WARNING log is emitted
- Unit test: Mock `get_repo_info()` to return success, assert no additional log beyond normal operation
- Integration test: Run agent in a directory without a Git repository, verify WARNING appears in logs
