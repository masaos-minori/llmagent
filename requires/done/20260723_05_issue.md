# Agent Startup Processing — SIGINT Signal Handler Not Restored Before REPL Loop

## Summary

After `shutdown_all()` installs a SIGINT guard handler (`_absorb_sigint_during_shutdown`), the handler is restored in the `finally` block. However, `_sigterm_handler` installed at line 455 uses `loop.add_signal_handler(signal.SIGTERM, _sigterm_handler)` which sets up a signal handler that persists through the entire REPL loop. There's no corresponding SIGINT handler set up for the main REPL loop — only SIGTERM. If Ctrl-C arrives while the REPL is running normally (not during shutdown), it will raise `KeyboardInterrupt` caught by `_read_input`, but there's no graceful shutdown path for SIGINT during normal operation.

## Severity

Medium

## Files

- `scripts/agent/repl.py:341-364`

## Details

The startup sequence in `AgentREPL.run()` installs a SIGTERM handler via `loop.add_signal_handler()`. During shutdown, `HttpServerLifecycleManager.shutdown_all()` temporarily replaces the SIGINT handler with `_absorb_sigint_during_shutdown` to prevent orphaned subprocesses from a second Ctrl-C. After cleanup completes, the original handler is restored.

However, during normal REPL operation, there is no SIGINT handler installed. When the user presses Ctrl-C outside of shutdown context, the default behavior raises `KeyboardInterrupt`, which is caught by `_read_input()`:

```python
except KeyboardInterrupt:
    self._view.write_turn_end()
    return None
```

This silently returns `None` from `_read_input()`, causing the REPL loop to exit without any graceful shutdown of MCP subprocesses, HTTP clients, or other resources. The `_close_resources()` method in the `finally` block should handle this, but if the exception propagates before reaching the finally block, resources may be left in inconsistent state.

## Reproduction Steps

1. Start the agent REPL
2. Wait for full startup completion (all MCP servers healthy)
3. Press Ctrl-C during normal operation (not during shutdown)
4. Observe whether MCP subprocesses are properly terminated

## Expected Behavior

SIGINT should trigger the same graceful shutdown path as SIGTERM — setting `ctx.conv.shutdown_requested = True` and calling `_close_resources()` to clean up all managed resources.

## Proposed Fix

Install a SIGINT handler alongside the SIGTERM handler in `AgentREPL.run()`:

```python
try:
    loop.add_signal_handler(signal.SIGTERM, _sigterm_handler)
    loop.add_signal_handler(signal.SIGINT, _sigterm_handler)  # Add SIGINT handler
except NotImplementedError:
    import signal as _signal
    _signal.signal(_signal.SIGTERM, lambda *_: _sigterm_handler())
    _signal.signal(signal.SIGINT, lambda *_: _sigterm_handler())  # Add SIGINT handler
```

Alternatively, unify both signals under a single handler function.
