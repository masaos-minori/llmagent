# Extract HTTP-only ToolTransportInvoker from ToolExecutor

## Priority

High

## Goal

Move MCP transport invocation responsibilities out of `ToolExecutor` into a dedicated HTTP-only transport invoker.

## Background

`ToolExecutor` currently owns transport initialization, health checks, lifecycle readiness, semaphore application, transport execution, transport error conversion, and transport/tool error statistics.

These responsibilities should be moved to `ToolTransportInvoker`.

## Scope

Affected files:

- `shared/tool_executor.py`
- `shared/tool_transport_invoker.py` new file
- `shared/http_transport.py`
- `shared/mcp_config.py`
- `shared/transport_dto.py`
- tests for transport invocation

## Implementation requirements

Create:

- `shared/tool_transport_invoker.py`

Add:

- `ToolTransportInvoker`

Expected API:

    class ToolTransportInvoker:
        async def invoke(
            self,
            server_key: str,
            tool_name: str,
            args: dict[str, Any],
        ) -> ToolCallResult:
            ...

Responsibilities to move into `ToolTransportInvoker`:

- Initialize and hold `HttpTransport` instances.
- Check server health.
- Call `lifecycle.ensure_ready(server_key)`.
- Apply per-server semaphore.
- Call `HttpTransport.call(tool_name, args)`.
- Record successful transport calls.
- Record tool-level errors where `result.is_error and result.error_type == "tool"`.
- Record transport-level errors.
- Convert `TransportError` into `ToolCallResult(error_type="transport")`.

Do not add stdio support.

## Acceptance criteria

- `ToolExecutor` no longer calls `HttpTransport.call()` directly.
- `ToolExecutor` no longer catches `TransportError` directly.
- `ToolExecutor` no longer owns `_transports`.
- Health unavailable behavior is preserved.
- Lifecycle readiness behavior is preserved.
- Semaphore behavior is preserved.
- Transport error behavior is preserved.
- Tool-level error statistics are preserved.
- Transport-level error statistics are preserved.
- No stdio branch is introduced.

## Validation plan

- Add `tests/shared/test_tool_transport_invoker.py`.
- Test health unavailable path.
- Test lifecycle readiness call.
- Test semaphore-limited execution.
- Test successful transport call records health success.
- Test tool-level error increments tool error stats.
- Test `TransportError` increments transport error stats.
- Test `TransportError` records health failure.
- Test `TransportError` is converted to `ToolCallResult(error_type="transport")`.

## Out of scope

- Supporting stdio transport.
- Changing `HttpTransport` API.
- Changing `ToolCallResult` schema.
