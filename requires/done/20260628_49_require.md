# Add tests that flat DB aliases are invalid

## Priority
Medium

## Labels
tests, agent, commands, db, breaking-change

## Dependencies
- Update `/db` command docstring to scoped-only forms

## Background
`cmd_db.py` already rejects flat `/db <subcmd>` forms. This behavior should be locked with tests so flat aliases are not accidentally restored.

## Task
Add regression tests verifying removed flat DB aliases are invalid.

## Implementation Requirements
- Update `test_agent_cmd_db.py`.
- Add tests for the following inputs:
  - `/db urls`
  - `/db clean http://example.com`
  - `/db rebuild-fts`
  - `/db recover`
- Expected behavior should match current `_cmd_db()` validation behavior.
- Tests should confirm these inputs do not reach RAG or Session maintenance services.

## Acceptance Criteria
- Flat DB aliases produce validation errors.
- No flat alias calls `RagMaintenanceService`.
- No flat alias calls `DbMaintenanceService`.
- Existing scoped DB command tests continue to pass.
