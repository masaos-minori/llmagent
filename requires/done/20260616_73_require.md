# [shared][logging] Make logger file-write fallback visible

## Summary
The logger falls back silently when file logging fails.

## Problem
If file logging stops working, operators may not notice.
Logs may only go to the stream handler.

## Scope
- `shared/logger.py`

## Task
- emit a visible warning when file logging fails
- keep stream fallback if needed
- make this state easy to notice in operations

## Expected Result
- file logging failure is not silent anymore

## Acceptance Criteria
- [ ] file logging failure triggers a visible warning
- [ ] stream fallback still works
- [ ] docs explain the fallback behavior
