# Add `chunk_type` and `source_file` to the canonical `chunks` schema

## Priority

High

## Summary

Add `chunk_type` and `source_file` columns to the canonical `chunks` table DDL in `schema_sql.py`.

## Background

The RAG ingestion implementation inserts `chunk_type` and `source_file` into the `chunks` table.

`ChunkSplitter.py` emits these fields, and `pipeline_utils.py` reads them into the chunk document flow.

The current `chunks` DDL in `schema_sql.py` does not define these columns, which can break ingestion on a newly recreated DB.

## Goal

Make the latest RAG schema match the current ingestion implementation.

## Scope

### In scope

- Update `schema_sql.py`.
- Update schema documentation.
- Update tests that use patched RAG DDL.

### Out of scope

- Backward-compatible `ALTER TABLE`.
- Migration for existing DB files.

## Implementation tasks

- Update `_RAG_SCHEMA_TEMPLATE` in `schema_sql.py`.
- Add these columns to the `chunks` table:
  - `chunk_type TEXT NOT NULL DEFAULT 'text'`
  - `source_file TEXT NOT NULL DEFAULT ''`
- Do not add migration logic.
- Ensure the latest schema supports ingestion SQL that inserts:
  - `doc_id`
  - `chunk_index`
  - `content`
  - `normalized_content`
  - `chunk_type`
  - `source_file`

## Acceptance criteria

- New DB creation creates `chunks.chunk_type`.
- New DB creation creates `chunks.source_file`.
- RAG ingestion can insert into `chunks` with `chunk_type` and `source_file`.
- No migration helper is used to add these columns.

## Validation

- Run:
```bash
uv run pytest tests/test_create_schema.py
```
- Run RAG ingestion tests if available.
- Add or update a schema-level test that confirms both columns exist.

## Confirmation required

- Confirm whether `chunk_type` should be constrained to only `text` and `code`.
- Current recommendation: do not add a `CHECK` constraint unless all producers are confirmed.
