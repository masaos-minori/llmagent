# [db][docs][workflow] Make workflow.sqlite a first-class database in all docs

## Summary
`workflow.sqlite` exists and is supported by `SQLiteHelper`, but its visibility is weaker than `rag.sqlite` and `session.sqlite`.

## Problem
This makes the DB landscape look less consistent than it really is.

## Scope
- DB overview docs
- schema docs
- API docs
- guide docs

## Task
- document `workflow.sqlite` consistently as one of the three main DB files
- make its bootstrap and schema references easy to find

## Expected Result
- all three DB files are treated as first-class platform databases

## Acceptance Criteria
- [ ] workflow DB appears consistently in overview and schema docs
- [ ] SQLiteHelper support for `"workflow"` is easy to find
- [ ] docs no longer treat workflow DB as a secondary or hidden case
