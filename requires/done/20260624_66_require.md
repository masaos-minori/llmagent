# Canonicalize the MDQ vs RAG boundary into a single source of truth

## Summary
Make the MDQ vs RAG boundary definition canonical in one document and replace duplicated normative text elsewhere with references.

## Background
The current MCP docs describe the MDQ vs RAG boundary in both:
- `04_mcp_07_mdq_rag_boundary.md`
- `04_mcp_05_security_and_safety_model.md`

This duplication increases the risk of drift in:
- ownership rules
- routing policy
- fallback behavior
- allowed/forbidden DB access paths
- migration criteria

## Objective
Define exactly one canonical document for all MDQ/RAG boundary rules.

## In Scope
- `04_mcp_07_mdq_rag_boundary.md`
- `04_mcp_05_security_and_safety_model.md`
- `04_mcp_00_document-guide.md`

## Required Changes
- Select one canonical boundary document
- Remove duplicated normative text from the non-canonical location
- Replace duplicated text with short references to the canonical document
- Update the document guide so it points readers to the canonical source first

## Acceptance Criteria
- Only one document contains the normative MDQ/RAG boundary rules
- Other documents reference the canonical source instead of duplicating it
- No contradiction remains between security guidance and boundary guidance

## Out of Scope
- Changes to MDQ or RAG implementation
- Agent routing code changes

## Notes
If any rule remains uncertain, explicitly mark it as `Unknown` or `Needs confirmation` instead of leaving two inconsistent descriptions.
