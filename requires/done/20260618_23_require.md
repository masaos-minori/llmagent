# [agent][config] Fix config loading and /reload consistency

## Summary
The config loading flow and the `/reload` behavior are not explained consistently.

## Problem
The docs describe multiple config files and loading paths, but the reload behavior is not fully clear.
It is hard to tell:
- which files are loaded normally
- which files are reloaded
- which changes require restart

## Scope
- `agent/config.py`
- config loading docs
- `/reload` behavior
- runtime config sync

## Task
- define one clear config loading model
- define exactly what `/reload` updates
- define exactly what still requires restart
- align docs with real implementation

## Expected Result
Config loading and reload behavior become predictable.

## Acceptance Criteria
- [ ] config loading flow is documented clearly
- [ ] `/reload` scope is documented clearly
- [ ] docs and implementation match
