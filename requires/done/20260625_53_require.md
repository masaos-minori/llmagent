## [Low][eventbus][architecture] Replace module-global mutable state with app state or dependency-injected runtime state

### Summary
The Event Bus currently stores runtime state in module-global variables.

### Problem
Module-global mutable state makes testing, isolation, and future extension harder than necessary.

### Scope
- app.py
- lifespan initialization
- shared runtime state management
- testability and maintainability

### Task
- move runtime state from module globals into FastAPI app state or an explicit runtime container
- make handlers access runtime state through a structured dependency path
- reduce reliance on hidden shared mutable state
- add tests that validate isolated app instances where practical

### Expected Result
Event Bus runtime state is explicit, structured, and easier to test and evolve.

### Acceptance Criteria
- module-global mutable state is removed or minimized
- handlers use explicit runtime state access
- the code is easier to instantiate in tests
- runtime ownership is documented clearly
