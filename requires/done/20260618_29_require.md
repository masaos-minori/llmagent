# [agent][commands][db] Make /db command scope easier to understand

## Summary
`/db` looks like a general database command group, but it mainly targets RAG-related maintenance.

## Problem
Users may assume `/db` manages all platform databases equally, but the command set is mostly focused on RAG-side data.

## Scope
- `/db` command help
- `/db` command output
- command docs
- DB scope explanation

## Task
- clarify which database each `/db` command targets
- improve help and output wording
- consider namespacing database commands if needed

## Expected Result
Users can understand what `/db` actually operates on.

## Acceptance Criteria
- [ ] `/db` help clearly explains target DBs
- [ ] output formatting clarifies DB scope
- [ ] command naming is less misleading
