# [agent][plugins] Strengthen plugin strictness and startup diagnostics

## Summary
Plugin loading is flexible, but it is still too fail-open for safer operation.

## Problem
Plugin import failures can be skipped.
Plugin tool conflicts can be hard to notice.
Bad return values are detected only when the tool is called.

## Scope
- plugin loading
- plugin startup reporting
- plugin tool validation

## Task
- improve startup reporting for plugin load results
- recommend strict mode more clearly for CI and production
- add earlier validation where possible
- make plugin shadowing easier to detect

## Expected Result
Plugin failures and conflicts become easier to detect at startup.

## Acceptance Criteria
- [ ] plugin startup diagnostics are clearer
- [ ] plugin failures are easier to inspect
- [ ] tool shadowing is easier to see
- [ ] strict mode is easier to use in production and CI
