# [rag][config][embedding] Align embedding dimension defaults with production configuration

## Summary
The runtime embedding dimension and the dataclass default do not match.

## Problem
Production uses 384 dimensions, but a dataclass default still shows 768.
This can confuse developers and tests.

## Scope
- RAG config defaults
- embedding dimension validation
- startup checks
- diagnostics

## Task
- align the default embedding dimension with production config
- or clearly enforce runtime override rules
- add a startup check for dimension consistency
- make diagnostics reflect the authoritative dimension

## Expected Result
There is one clear authoritative embedding dimension for runtime behavior.

## Acceptance Criteria
- [ ] runtime and default embedding dimension are no longer misleading
- [ ] startup can detect dimension mismatch
- [ ] diagnostics reflect the real configured dimension
