# [High] Update tests for HTTP-only MCP transport

## Goal

Remove stdio-specific tests and rewrite MCP config/tool executor tests for HTTP-only transport.

## Background

The test suite currently includes explicit tests for `StdioTransport`, `TransportType.STDIO`, `StartupMode.ONDEMAND`, and `HealthcheckMode.PING_TOOL`. These must be removed after MCP transport becomes HTTP-only.

## Scope

Tests only.

Runtime removal is handled in a separate issue.

## Implementation Tasks

- Remove stdio-specific tests from `tests/test_tool_executor.py`:
  - `TestStdioTransportResponseId`
  - `TestStdioTransportStop`
- Remove stdio-specific tests from `tests/test_mcp_config.py`:
  - `test_valid_stdio_config`
  - `test_explicit_startup_mode_ondemand`
  - `test_explicit_healthcheck_ping_tool`
  - `test_idle_timeout_default_zero` if it only applies to stdio
  - `test_ondemand_http_raises`
  - `test_subprocess_stdio_raises`
  - stdio branch inside `test_toml_string_values_parsed_from_config`
- Replace removed config tests with HTTP-only parse/validation tests.
- Keep HTTP retry tests for `HttpTransport`.
- Ensure test names no longer mention stdio.

## Acceptance Criteria

- No tests import `StdioTransport`.
- No tests reference `TransportType.STDIO`.
- No tests reference `HealthcheckMode.PING_TOOL`.
- `tests/test_tool_executor.py` covers HTTP transport, cache, retries, and transport error behavior only.
- `tests/test_mcp_config.py` validates HTTP-only config behavior.

## Validation

- Run `pytest tests/test_tool_executor.py -v`.
- Run `pytest tests/test_mcp_config.py -v`.
- Run full MCP-related test suite if available.
