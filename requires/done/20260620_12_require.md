# Title
Fix the `use_memory_layer` default-value inconsistency and make runtime state visible

## Summary
Determine the real implementation default for `use_memory_layer`, align all documentation to that value, and make the enabled/disabled state visible in normal runtime views.

## Background
The documentation set does not agree on the default value of `use_memory_layer`.
Some docs imply the default is enabled, while configuration and API references imply it is disabled.

## Problem
If operators and AI systems cannot reliably tell whether the memory layer is enabled by default, they may misconfigure or misreason about runtime behavior.

## Required Changes
- Confirm the real implementation default for `use_memory_layer`.
- Update all relevant documents so they describe the same default value.
- Expose memory-layer enabled/disabled state through normal operator-facing views such as startup banner, `/config`, or `/stats`.

## Acceptance Criteria
- All docs agree on the default value of `use_memory_layer`.
- Runtime UI exposes whether the memory layer is currently enabled or disabled.
- No active documentation implies contradictory default behavior.

- Changing embedding API behavior
