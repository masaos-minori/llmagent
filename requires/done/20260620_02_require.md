## Summary
Standardize the MDQ database filename and configuration path to `mdq.sqlite`.

## Background
Current MDQ configuration is inconsistent:
- TOML configuration uses `mdq.sqlite`
- JSON configuration uses `mdq.db`

The documented and code-level default is `mdq.sqlite`.
For consistency with the RAG naming convention (`rag.sqlite`), MDQ should also use `mdq.sqlite`.

## Problem
A split naming convention creates avoidable confusion for:
- deployment
- operations
- migration
- debugging
- documentation

## Required Changes
- Audit all MDQ configuration files and references to the MDQ database path.
- Make `mdq.sqlite` the canonical database filename everywhere.
- Update any JSON config still using `mdq.db`.
- Add a migration note or compatibility warning if existing environments may still reference `mdq.db`.
- Update docs, examples, and operational guidance to use `mdq.sqlite` consistently.

## Acceptance Criteria
- All MDQ configuration and documentation references use `mdq.sqlite`.
- No active configuration path still points to `mdq.db` unless explicitly part of a documented temporary compatibility path.
- A migration note exists for environments that may still rely on `mdq.db`.

## Out of Scope
- Changing MDQ schema
- Changing MDQ indexing/search behavior
