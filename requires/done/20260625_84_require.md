## [Medium][rag][docs] Reclassify retained old source files as archive-only or remove them from active documentation flow

### Summary
The RAG document guide explicitly states that the restructured docs are the primary reference, while the original source files are retained as-is for historical reference. 【3-64059d】

### Problem
Keeping retained legacy source docs in the normal documentation flow creates a two-layer source-of-truth model and increases the chance that maintainers or AI agents read old material as if it were still active specification.

### Scope
- `03_rag_00_document-guide.md`
- legacy source-file placement and naming
- documentation navigation and search flow
- repository documentation policy

### Task
- decide whether old source files should be:
  - archived
  - removed
  - or explicitly isolated outside active documentation flow
- ensure legacy docs are clearly marked non-canonical if retained
- remove legacy docs from active navigation and AI routing tables if they are no longer intended for active reference
- update the document guide accordingly

### Expected Result
The restructured RAG docs become the only active specification surface, while retained historical docs are clearly isolated. 【3-64059d】

### Acceptance Criteria
- old source docs are either archived, removed, or clearly isolated
- active navigation no longer treats retained legacy docs as normal references
- the document guide reflects the final archive/remove policy
