# Clean up MDQ indexing control configuration and serialization behavior

## Priority

Medium

## Target files

- `mcp_servers/mdq/service.py`
- `mcp_servers/mdq/tools.py`
- `config/mdq_mcp_server.toml`
- `docs/04_mcp_*.md`
- `tests/test_mdq_index*.py`

## Type

Backward compatibility removal / indexing behavior cleanup

## Background

MDQ indexing behavior is controlled through several mechanisms:

- `index_paths`
- `refresh_index`
- `enable_refresh`
- `requires_serial=True`
- Internal `asyncio.Lock` in `MdqService`
- Removed config key `concurrency_limit`

The config explicitly states that `concurrency_limit` was removed because serialization is implemented by a fixed internal lock.

However, `enable_refresh` is read by `MdqService`, and its enforcement must be confirmed.

## Problem

Indexing control behavior may be split across runtime locks, schema metadata, and config flags.

Risks:

- Removed `concurrency_limit` may still be documented or assumed
- `enable_refresh` may be read but not enforced
- Tool metadata and runtime serialization may drift
- Operators may misunderstand how indexing concurrency is controlled

## Intent

Clarify and simplify MDQ indexing behavior.

The intended production model should be:

- `index_paths` and `refresh_index` are serialized
- Serialization is implemented by the internal lock
- Removed config keys have no runtime effect
- Retained feature gates are enforced

## Tasks

### Removed concurrency setting

- [ ] Ensure `concurrency_limit` is not read anywhere in MDQ runtime code
- [ ] Treat `concurrency_limit` as removed or invalid config
- [ ] Remove `concurrency_limit` from docs and examples
- [ ] Document internal lock as the serialization mechanism

### Serialization consistency

- [ ] Confirm `index_paths` has `requires_serial=True`
- [ ] Confirm `refresh_index` has `requires_serial=True`
- [ ] Confirm both operations use the internal `_index_lock`
- [ ] Add a test that indexing operations are serialized

### Refresh gate cleanup

- [ ] Search all references to `enable_refresh`
- [ ] Confirm whether `refresh_index()` checks `enable_refresh`
- [ ] If `enable_refresh` is unused, remove it from config and service
- [ ] If `enable_refresh` is retained, enforce it in `refresh_index()`
- [ ] Add tests for disabled refresh behavior if retained

### Grep gate classification

- [ ] Confirm `enable_grep` is intentionally supported
- [ ] Keep `enable_grep` only if it remains a supported production feature gate
- [ ] Add tests for disabled grep behavior

## Acceptance criteria

- Removed `concurrency_limit` has no runtime or documentation footprint
- Index and refresh serialization behavior is tested
- `enable_refresh` is either enforced or removed
- `enable_grep` is classified as supported or removed
- Documentation matches runtime behavior
