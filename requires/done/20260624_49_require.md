# Make SQLiteDocumentStore.doc_upsert() return doc_id through an explicit contract

## Summary
Replace the current `lastrowid`-dependent `doc_upsert()` return behavior with a clearer and more explicit doc-id retrieval contract.

## Background
`doc_upsert()` currently performs `INSERT ... ON CONFLICT DO UPDATE` and then returns `cur.lastrowid`. This is compact, but it makes the correctness of the returned `doc_id` depend on SQL-driver semantics that are not self-evident to readers.

## Objective
Make doc-id retrieval explicit and robust for both insert and update paths.

## In Scope
- `store_impl.py`

## Required Changes
- Refactor `doc_upsert()` so the returned `doc_id` is obtained explicitly
- Prefer one of:
  - `RETURNING doc_id` if supported by the deployment baseline, or
  - a follow-up `SELECT doc_id FROM documents WHERE url = ?`
- Keep upsert semantics unchanged from the caller perspective

## Acceptance Criteria
- `doc_upsert()` returns the correct `doc_id` for both insert and update paths
- The implementation no longer relies on an opaque `lastrowid` expectation
- The ID-return contract is easier to review and reason about

## Out of Scope
- Rewriting document schema
- Changing conflict target or update fields

## Implementation Notes
Choose clarity over cleverness. The returned identifier must be obviously correct from the code alone.
