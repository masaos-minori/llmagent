# H-4: Remove ToolResultStore persistence from partial completion handling

## Priority

High

## Background

`handle_partial_completion()` currently stores partial LLM completion information in both the diagnostic channel and `ctx.tool_result_store` using the pseudo tool name `llm_partial_completion`.

Partial completions should be stored only in `session_diagnostics`.

## Target files

- `agent/llm_transport_errors.py`

## Implementation tasks

- Remove the `ctx.tool_result_store.store(...)` call from `handle_partial_completion()`.
- Remove the `try/except` block that catches `ToolResultStore.store` failures.
- Remove the warning message for failed partial completion storage in `ToolResultStore`.
- Keep `diagnostic_store.save(...)`.
- Keep `diagnostic_store.save_partial_completion(...)`.
- Keep `ctx.services_required.llm.stat_partial_completions += 1`.
- Keep the existing log that a partial completion was saved, but ensure the log no longer implies ToolResultStore usage.

## Acceptance criteria

- Partial completions are not written to `tool_results`.
- Partial completions are still persisted through `session_diagnostics`.
- Partial completion counters still increment.
- Partial content is not inserted into `ctx.conv.history`.
- User-facing and log messages do not mention `/tool show` or `tool_result`.

## Test plan

- Simulate `LLMTransportError` with non-empty `partial_text`.
- Verify `DiagnosticStore.save(...)` is called.
- Verify `DiagnosticStore.save_partial_completion(...)` is called.
- Verify `ctx.tool_result_store.store(...)` is not called.
- Verify `stat_partial_completions` increments.
