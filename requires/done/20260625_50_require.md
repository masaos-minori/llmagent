## [Medium][eventbus][archive] Add reconcile and repair support for SQLite vs JSONL archive divergence

### Summary
The Event Bus treats SQLite as authoritative and the JSONL archive as supplementary.

### Problem
When JSONL append fails, the system logs a warning but still returns success.
This can leave SQLite and the JSONL archive permanently diverged with no formal repair path.

### Scope
- publish path
- _append_jsonl()
- archive integrity policy
- operational tooling and docs

### Task
- define how archive divergence is detected and surfaced
- add a reconcile or repair mechanism for JSONL archive gaps
- expose metrics or admin diagnostics for divergence
- document the intended operational role of the archive clearly
- add tests for archive-failure and reconciliation behavior

### Expected Result
Archive divergence is detectable and recoverable instead of remaining a silent warning-only condition.

### Acceptance Criteria
- archive divergence can be detected explicitly
- a documented repair or reconcile path exists
- operators can inspect archive integrity
- tests cover archive-failure behavior
