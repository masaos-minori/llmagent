# Add schema tests for MCP runtime availability metadata

## Priority

Medium

## Target files

- tests/mcp_servers/test_tool_schema.py
- tests/mcp_servers/test_tools_endpoint.py
- mcp_servers/file/read_tools.py
- mcp_servers/file/write_tools.py
- mcp_servers/file/delete_tools.py
- mcp_servers/git/tools.py
- mcp_servers/file/read_server.py
- mcp_servers/file/write_server.py
- mcp_servers/file/delete_server.py
- mcp_servers/git/server.py

## Reason

RuntimeToolRegistry depends on `/v1/tools` metadata being valid.

The new metadata fields must be consistently present and correctly typed. Without tests, malformed metadata can break Agent startup, LLM tool exposure, routing, or diagnostics.

## Intent

Prevent invalid runtime availability metadata from entering Agent discovery.

## Scope

Add schema validation for each tool returned by `/v1/tools`.

Required fields:

- `name`: non-empty string
- `description`: non-empty string
- `inputSchema`: object
- `status`: string
- `config_dependent`: boolean
- `enabled`: boolean
- `disabled_reason`: string

Validation rules:

- `requires_config` must not be present
- if `enabled` is true, `disabled_reason` must be empty
- if `enabled` is false, `disabled_reason` must be non-empty

Test both enabled and disabled server states.

## Acceptance criteria

- Schema tests fail on missing required fields.
- Schema tests fail on invalid field types.
- Schema tests fail if `requires_config` is present.
- Schema tests cover enabled file server state.
- Schema tests cover disabled file server state.
- Schema tests cover enabled git server state.
- Schema tests cover disabled git states for empty allowlist and read-only mode.
