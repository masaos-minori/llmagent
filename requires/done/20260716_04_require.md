# Simplify MDQ search configuration and remove unsupported hybrid search placeholders

## Priority

Medium

## Target files

- `mcp_servers/mdq/tools.py`
- `mcp_servers/mdq/models.py`
- `mcp_servers/mdq/service.py`
- `mcp_servers/mdq/search.py`
- `config/mdq_mcp_server.toml`
- `docs/04_mcp_*.md`
- `tests/test_mdq_search*.py`

## Type

Backward compatibility removal / search API cleanup

## Background

MDQ currently exposes search-related compatibility or placeholder surfaces.

Known examples:

- `tools.py` describes `search_docs.mode` as `bm25/grep` and mentions that `hybrid` is not yet supported
- `models.py` defines `SearchDocsRequest.mode` as a plain string
- MDQ config includes embedding and hybrid-search related fields
- `service.py` reads embedding-related configuration
- `models.py` contains `EmbeddingResult`
- Config includes both `max_search_results` and `max_results_limit`

This makes unsupported search modes and unclear limit precedence appear more stable than they are.

## Problem

Search configuration and request behavior are ambiguous.

Current risks:

- Users may try `hybrid` because it is mentioned in schema text
- Embedding configuration may appear supported despite no production hybrid search path
- Duplicate limit settings make precedence unclear
- Search mode validation is too weak if arbitrary strings are accepted
- Future implementers may treat placeholders as supported API

## Intent

Restrict MDQ search behavior to production-supported modes and simplify result-limit configuration.

Supported production search modes should be explicit and enforced.

## Tasks

### Search mode cleanup

- [ ] Remove `hybrid is not yet supported` from `tools.py`
- [ ] Define supported search modes explicitly
- [ ] Restrict `SearchDocsRequest.mode` to supported values
- [ ] Reject unsupported modes with a validation error
- [ ] Confirm `search.py` implements only supported modes
- [ ] Update docs to describe supported search modes only

### Embedding placeholder cleanup

- [ ] Remove `use_embedding` from MDQ config if hybrid is not supported
- [ ] Remove `embedding_dims` from MDQ config if hybrid is not supported
- [ ] Remove `vector_table` from MDQ config if hybrid is not supported
- [ ] Remove `embedding_model` from MDQ config if hybrid is not supported
- [ ] Remove embedding-related reads from `service.py`
- [ ] Remove `EmbeddingResult` if unused
- [ ] Remove vector table initialization if only used for unsupported hybrid behavior

### Search limit cleanup

- [ ] Locate all uses of `max_search_results`
- [ ] Locate all uses of `max_results_limit`
- [ ] Choose one canonical config key
- [ ] Remove the non-canonical key
- [ ] Define request override behavior for `SearchDocsRequest.max_results_limit`
- [ ] Ensure request overrides are capped by the canonical server-side limit
- [ ] Add tests for default limit behavior
- [ ] Add tests for override cap behavior
- [ ] Add tests for removed alias handling

## Acceptance criteria

- Unsupported `hybrid` mode is not documented as available
- Unsupported modes are rejected
- Embedding config is not exposed unless hybrid search is implemented
- Only one canonical server-side result limit exists
- Request-level result limits have clear bounded behavior
- Search docs and tests reflect the production behavior

## Suggested supported modes

- `bm25`
- `grep`
