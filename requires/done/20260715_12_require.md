## Background

MemoryServices are currently optional and depend on `use_memory_layer`. JSONL store construction and `EmbeddingClient(enabled=...)` are also wired into runtime construction.

## Problem

The target Memory design requires:

- MemoryServices to be constructed at startup
- SQLite to be the authoritative Memory store
- No runtime JSONL persistence
- No normal embedding-disabled mode
- No FTS-only fallback as a successful retrieval path

## Implementation Tasks

### Service construction

- [ ] Remove the `use_memory_layer` gate from Memory service construction.
- [ ] Ensure Memory construction always returns a `MemoryServices` instance.
- [ ] Remove the `MemoryServices | None` construction model.

### JSONL removal

- [ ] Remove JSONL store construction from `factory.py`.
- [ ] Remove the JSONL store argument from ingestion service construction.
- [ ] Remove `JsonlMemoryStore` from the runtime ingestion path.
- [ ] Remove runtime writes to `memories.jsonl`.
- [ ] Delete `jsonl_store.py` or isolate it as migration-only if it is still needed.

### Embedding-required model

- [ ] Remove the `enabled=` argument when constructing `EmbeddingClient`.
- [ ] Remove the normal `EmbeddingErrorKind.DISABLED` path if it only represents config-disabled embeddings.
- [ ] Keep circuit breaker behavior for runtime degraded/unavailable states.
- [ ] Keep `memory_local_only` validation behavior.

### Retriever behavior

- [ ] Remove FTS-only fallback from `HybridRetriever.search()`.
- [ ] Remove `fts_fallback_count`.
- [ ] Remove `last_retrieval_mode = "fts_only"`.
- [ ] Define behavior for:
  - missing embedding
  - vector index unavailable
  - vector query succeeds but returns zero rows

## Acceptance Criteria

- [ ] MemoryServices are always available after successful startup.
- [ ] Runtime Memory ingestion writes only to SQLite.
- [ ] No runtime code writes JSONL Memory archives.
- [ ] Embeddings cannot be disabled by a normal config toggle.
- [ ] Retriever does not silently return FTS-only results.
- [ ] Degraded and unavailable states are explicit and observable.
