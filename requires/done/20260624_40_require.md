# Add maintenance guidance for keeping command summaries and detailed command docs in sync

## Summary
Reduce command-documentation drift by documenting how the high-level command summary and the full command reference should be maintained together.

## Background
The system overview and CLI chapter both describe command groups. This is useful, but future command additions can easily update one file and not the other.

## Objective
Make the documentation maintenance boundary explicit.

## In Scope
- `05_agent_01_system-overview.md`
- `05_agent_07_cli-and-commands.md`

## Required Changes
- Add a short maintenance note describing:
  - what belongs in the overview summary
  - what belongs in the detailed CLI chapter
  - what must be updated when a new command is added
- Clarify whether the overview command list is exhaustive or representative

## Acceptance Criteria
- Maintainers can tell which command changes require updates in both files
- The overview remains concise without becoming misleading
- Drift risk is reduced through explicit maintenance guidance

## Out of Scope
- Reorganizing command groups
- Changing CLI behavior

## Notes
Keep the overview lightweight, but document the synchronization rule clearly.
