# Align `DocumentRow.lang` nullability with the database schema

## Priority
Low

## Labels
- db
- typing
- cleanup
- low-priority

## Dependencies
None

## Background
The `documents.lang` column is defined as `NOT NULL`, but `DocumentRow.lang` is typed as `str | None`.

The DTO should match the schema unless there is a known reason to preserve optionality.

## Scope
Modify:
- `db/models.py`
- `db/store_impl.py`
- `tests/test_db_store_impl.py`

## Required Changes
- Change `DocumentRow.lang` from `str | None` to `str`.
- Simplify `SQLiteDocumentStore.doc_get()` and `doc_list()` handling of `lang`.
- Preserve runtime behavior for valid rows.

## Tests
Update `tests/test_db_store_impl.py`.

Add tests for:
- `doc_get()` returns `DocumentRow.lang` as `str`.
- `doc_list()` returns `DocumentRow.lang` as `str`.

## Acceptance Criteria
- DTO nullability matches schema nullability.
- Existing document store tests pass.
