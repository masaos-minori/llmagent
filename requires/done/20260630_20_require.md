# [Low] Refactor route resolver tests for the new routing model

## Priority
Low

## Background
Current route resolver tests still assume config fallback, GitHub prefix fallback, and strict error messages pointing to `tool_constants.py`. These expectations must be updated after routing fallback removal.

## Goal
Align `tests/test_route_resolver.py` with the new routing model: live discovery plus `ToolRegistry`.

## Scope
Affected files:
- `tests/test_route_resolver.py`

## Implementation Tasks
- Keep registry-based routing tests.
- Remove config `tool_names` fallback tests.
- Replace GitHub prefix routing tests with explicit registry-based GitHub tool tests.
- Update unknown tool error-message expectations to reference `ToolRegistry`, not `tool_constants.py`.
- Remove or update `warn_on_fallback` tests.
- Add tests proving config-only tools do not route.

## Acceptance Criteria
- Tests no longer assume static fallback.
- Tests no longer assume GitHub prefix fallback.
- Tests no longer assume config `tool_names` fallback.
- Tests verify that registered tools route and unregistered tools fail.

## Validation
- Run `pytest tests/test_route_resolver.py -v`.
