## [Medium][eventbus][offsets] Redefine offset checkpoint behavior for the hybrid replay + push delivery model

### Summary
The Event Bus already persists offsets for replay/resume, but the subscribe architecture is changing.

### Problem
Offset semantics must remain correct when replay is database-based and live delivery is memory-based.

### Scope
- app.py
- offsets.py
- subscribe checkpoint logic
- replay/resume semantics

### Task
- define how offsets advance during replay and during live push delivery
- keep resume behavior compatible with `consumer_id`
- ensure reconnect starts from the correct durable sequence
- review whether checkpoint timing should change in the new model
- add tests for offset correctness across reconnects

### Expected Result
Replay and live push delivery share one consistent offset model.

### Acceptance Criteria
- offset behavior is explicitly defined for replay and live paths
- reconnect resumes correctly
- offsets remain durable and consistent
- tests verify offset correctness
