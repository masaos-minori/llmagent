# Markdown 向け ingest の標準化とデフォルト最適化

- 改善案:
  - md_index_enable=false を前提とした任意機能ではなく、Markdown は見出しベース chunking を標準とし、長大セクションのみ通常 chunking にフォールバックする方針へ変更すること。
- 効果:
  - RAG の局所性が上がり、同一文書の近接チャンクが過剰に混入しにくくなること。ドキュメント系質問の精度改善。
- リスク:
  - セクション分割が細かすぎると文脈不足になること。非 Markdown 文書との扱い差を吸収する必要があること。
- 実装方式:
  - .md は既定で見出しベース chunking、長大節は通常 text chunking へフォールバックし、documents テーブルへ chunking_strategy を持たせ、/db urls で表示する方式。
- 実装対象:
  - ChunkSplitter / RagIngester 相当
  - agent_commands.py
  - config/rag_pipeline.json
  - DB スキーマ
