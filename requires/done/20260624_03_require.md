# Clarify and standardize the MCP entrypoint for the RAG pipeline

## Summary
Clarify the role of the RAG MCP server entrypoint and remove ambiguous references between `server.py` and `service.py`.

## Background
The current documentation points to different MCP-side files for the RAG pipeline. This makes the execution path unclear for implementers.

## Objective
Define one clear documented execution path for the RAG MCP layer and standardize all cross-document references.

## In Scope
- `03_rag_01_system_overview.md`
- `03_rag_03_query_pipeline.md`
- `scripts/mcp/rag_pipeline/` documentation references
- Any cross-reference that explains how `RagPipeline.augment()` is exposed to MCP

## Required Changes
- Determine the intended responsibility split between:
  - `scripts/mcp/rag_pipeline/server.py`
  - `scripts/mcp/rag_pipeline/service.py`
- Rewrite documentation so that:
  - the entrypoint is explicit
  - the wrapper / adapter responsibilities are explicit
  - the call path from MCP to `scripts/rag/pipeline.py` is explicit
- Standardize all references across the document set

## Acceptance Criteria
- A reader can identify the MCP entrypoint and the call path without ambiguity
- `server.py` and `service.py` responsibilities are described consistently
- No document contradicts another about the MCP-side integration path

## Out of Scope
- Refactoring actual MCP server implementation
- Renaming files

## Implementation Notes
If the exact role split is still uncertain after document review, introduce an explicit "Unknown / Needs Confirmation" subsection instead of guessing.
