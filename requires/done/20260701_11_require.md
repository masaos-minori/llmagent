## Issue 11: [Medium] Resolve /reject command registration mismatch

### Goal

Resolve the mismatch where `/reject` is implemented but not registered in `CommandRegistry`.

### Background

`cmd_workflow.py` implements both `_cmd_approve()` and `_cmd_reject()`.
However, `registry.py` registers `/approve` but does not list `/reject` in `_COMMANDS`.

### Scope

- Decide whether `/reject` should be supported.
- Then update either implementation or documentation accordingly.

### Implementation Tasks

- If `/reject` is intended to be supported:
  - Add a `CommandDef` for `/reject` in `registry.py`.
  - Ensure help output includes `/reject`.
  - Ensure `05_agent_07_cli-and-commands.md` documents `/reject`.
- If `/reject` is not intended to be supported:
  - Remove or deprecate `_cmd_reject()` implementation.
  - Remove `/reject` from active docs.

### Suggested CommandDef

```python
CommandDef(
    "/reject",
    True,
    False,
    "_cmd_reject",
    "[reason]  Reject the pending workflow task",
)
```

### Acceptance Criteria

- Implementation and documentation agree on whether `/reject` exists.
- If supported, `/reject` is registered and dispatchable.
- If unsupported, it is not documented as an active command.

### Validation

- Run `/help` if available.
- Run `/reject test` in a controlled environment if supported.
