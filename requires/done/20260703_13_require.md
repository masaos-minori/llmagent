# [High] Redesign MCP health semantics and watchdog restart policy

## Goal

Separate MCP health concepts so the watchdog does not restart servers merely because dependencies or configuration are not ready.

## Background

The current watchdog uses only the `/health` HTTP status code. It does not inspect body fields such as `ready`, `dependencies`, or `details`.

This makes the following cases indistinguishable:

- process failure that may be fixed by restart
- dependency failure
- missing credential
- missing config
- database file missing
- external service unavailable

As a result, `HTTP 503` can trigger restart even when manual operator action is required.

## Scope

Update:

- `agent/repl_health.py`
- `mcp/server.py`
- relevant `mcp/*/server.py` health implementations
- `tests/test_mcp_health.py`
- new file: `tests/test_watchdog.py`
- MCP docs:
  - `04_mcp_02_protocol_and_transport.md`
  - `04_mcp_03_routing_lifecycle_and_execution.md`
  - `04_mcp_06_configuration_and_operations.md`
  - `04_mcp_90_inconsistencies_and_known_issues.md`

## Required Health Model

Health response body must distinguish:

- `liveness`
  - process is alive
- `ready`
  - dependencies are ready for normal tool execution
- `restart_recommended`
  - watchdog should restart this server process
- `operator_action_required`
  - manual config, credential, or dependency action is required

## Implementation Tasks

- Extend MCP `/health` response schema to include:
  - `restart_recommended`
  - `operator_action_required`
- Update server health implementations to classify dependency failures.
- Update watchdog probing logic to parse `/health` response body.
- Restart subprocess servers only when restart is recommended.
- Do not restart on configuration or credential failures.
- Keep externally managed servers as warning-only for restart actions.
- Fix `watchdog_loop()` so `mcp_watchdog_interval <= 0` returns immediately and does not enter a sleep loop.
- Add `tests/test_watchdog.py`.
- Update MCP docs to reflect liveness/readiness/restart separation.

## Acceptance Criteria

- Watchdog does not restart for non-restartable readiness failures.
- Watchdog restarts subprocess servers only when `restart_recommended=true`.
- `operator_action_required=true` is logged clearly.
- `mcp_watchdog_interval <= 0` disables the watchdog without entering the loop.
- Health docs no longer imply every dependency failure should trigger restart.

## Validation

Run:

    uv run pytest tests/test_mcp_health.py tests/test_watchdog.py -v

Test cases must include:

- `HTTP 503` + `restart_recommended=true` triggers restart.
- `HTTP 503` + `restart_recommended=false` does not restart.
- `HTTP 200` + `ready=false` does not restart.
- `mcp_watchdog_interval=0` exits watchdog loop immediately.
