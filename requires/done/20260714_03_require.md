# [High] Align configuration documentation with consolidated config/agent.toml layout

## Labels
priority/high, area/config, area/docs, consistency

## Summary
Update the documentation so it consistently states that the agent process configuration is consolidated in `config/agent.toml`.

The confirmed specification is:

- Agent configuration is consolidated in `config/agent.toml`.
- Historical split config files are no longer canonical.
- `/reload` must not be documented as loading the old 12 base config files.

## Background
Some documentation already states that prior split config files were consolidated into `agent.toml`.

However, other documentation still says `/reload` loads all 12 base config files. This is obsolete and confusing.

## Required Changes

- Remove or rewrite references to “12 base config files”.
- Describe `config/agent.toml` as the canonical agent process configuration file.
- Keep any old split config references only as historical notes.
- Update `/reload` documentation to describe current behavior.
- Ensure `ConfigLoader.load_all()` documentation matches implementation.
- Update the document guide and reference API if they still imply split-file ownership.

## Suggested Replacement Text
The agent process configuration is consolidated in `config/agent.toml`. Historical split configuration files are no longer canonical.

## Acceptance Criteria
- Documentation no longer says `/reload` loads the old 12 base config files.
- `config/agent.toml` is clearly documented as the canonical agent process configuration file.
- Historical split files are described only as legacy/historical background.
- Configuration loading, hot reload, and reference docs use the same model.
- The documentation matches `ConfigLoader.load_all()` behavior.
