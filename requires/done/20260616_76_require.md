# [db][maintenance] Add strict and best-effort modes for maintenance operations

## Summary
Some maintenance functions are strict and propagate exceptions directly.

## Problem
This is sometimes correct, but maintenance tasks often need more controlled runtime behavior.

## Scope
- `db/maintenance.py`

## Task
- define clear operation modes for maintenance tasks:
  - strict
  - best-effort
- make behavior explicit for:
  - memory prune
  - checkpoint
  - vacuum
  - recovery
- improve result reporting where needed

## Expected Result
- maintenance behavior is predictable for both automation and manual ops

## Acceptance Criteria
- [ ] strict vs best-effort behavior is defined
- [ ] maintenance result reporting is clearer
- [ ] docs explain the behavior for each mode
