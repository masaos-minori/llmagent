## [Low][agent][release] Add a one-time migration note for removed backward-compatibility aliases, without preserving runtime support

### Summary
Removing explicit backward-compatibility aliases is a breaking CLI change.

### Problem
Operators may still have existing habits or runbooks that use the deleted alias forms.
Runtime compatibility should still be removed, but migration guidance may be needed.

### Scope
- docs/05_agent_07_cli-and-commands.md
- docs/05_agent_00_document-guide.md
- operator-facing notes or release notes if maintained separately

### Facts
- The old flat `/db` commands are currently documented as compatibility aliases. 【1-819e5c】【2-3cddb8】
- `/db recover` is explicitly documented as a compatibility alias. 【3-4da049】

### Task
- add a short migration note mapping deleted old commands to the new scoped commands
- do not preserve alias behavior in the runtime
- keep the migration note concise and clearly marked as transitional documentation only

### Example Migration Mapping
- `/db stats` -> use `/db rag stats` and/or `/db session stats`
- `/db recover` -> use `/db rag recover` or `/db session recover`

### Expected Result
Users get a clear migration path, while runtime backward compatibility is fully removed.

### Acceptance Criteria
- migration guidance exists in docs or release notes
- runtime alias support is fully removed
- the migration note does not reintroduce compatibility behavior
