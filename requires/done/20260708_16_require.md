# M-6: Decide and implement UndoResult.n_artifacts_marked compatibility behavior

## Priority

Medium

## Background

`UndoResult` includes `n_artifacts_marked`, which previously represented how many `tool_results` rows were marked as undone.

After removing `tool_results.undone`, this field no longer has a real meaning.

## Target files

- `agent/services/models.py`
- `agent/services/undo_service.py`
- command output tests if any

## Implementation tasks

- Decide whether to:
  - remove `UndoResult.n_artifacts_marked`, or
  - keep it as a deprecated always-zero field for short-term compatibility
- If removing:
  - update `UndoResult`
  - update all constructors
  - update tests
- If keeping:
  - document that it is deprecated and always `0`
  - ensure no runtime logic depends on it
- Update `/undo` output only if needed.

## Acceptance criteria

- `UndoResult` no longer implies `tool_results` artifact retention.
- `/undo` behavior remains clear and stable.
- Tests reflect the new compatibility decision.
- No code uses `n_artifacts_marked` to infer persisted tool artifacts.

## Test plan

- Unit-test `undo_last_turn()`.
- Verify returned `UndoResult` matches the chosen behavior.
- Verify no ToolResultStore calls happen.
- Verify command output remains stable.
