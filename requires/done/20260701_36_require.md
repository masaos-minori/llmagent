# [High] Remove stdio transport documentation and make MCP HTTP-only in docs

## Goal

Update MCP documentation so that HTTP is the only supported MCP transport.

## Background

MCP docs currently describe stdio transport, `--stdio`, `__list_tools__`, `ping_tool`, stdio subprocess operation, and stdio scaling behavior. These must be removed after the runtime becomes HTTP-only.

## Scope

Documentation only.

Runtime removal is handled separately.

## Implementation Tasks

- Update `04_mcp_01_system_overview.md`.
- Update `04_mcp_02_protocol_and_transport.md`.
- Update `04_mcp_03_routing_lifecycle_and_execution.md`.
- Update `04_mcp_06_configuration_and_operations.md`.
- Remove all stdio sections:
  - stdio transport format
  - `--stdio`
  - `__list_tools__`
  - `healthcheck_mode="ping_tool"`
  - stdio persistent mode
  - stdio ondemand mode
  - stdio subprocess operation examples
  - stdio scalability limits
  - stdio lifecycle and idle timeout
- State that MCP transport is HTTP-only.
- State that live tool discovery uses `GET /v1/tools`.

## Acceptance Criteria

- Active MCP docs describe HTTP-only transport.
- No active docs describe stdio as supported.
- No active docs mention `__list_tools__`.
- Startup modes are HTTP-only.
- Health checks are HTTP `/health` only.

## Validation

- Grep MCP docs for:
  - `stdio`
  - `--stdio`
  - `__list_tools__`
  - `ping_tool`
  - `ondemand`
- Confirm any remaining matches are either removed or explicitly unrelated to MCP transport.
