## Title
[Medium] Document MCP tool routing authority and fallback priority

## Priority
Medium

## Labels
documentation, mcp, routing, tools

## Dependencies
None

## Background
Tool routing priority is implemented as:
1. Live-discovered tool metadata
2. ToolRegistry
3. Configured `mcp_servers.*.tool_names`
4. Static fallback constants

This should be documented clearly to avoid confusion about the source of truth.

## Scope
Update:
- Tool execution documentation
- MCP routing documentation
- Reference API documentation if present

## Required Changes
- Document routing priority exactly as implemented.
- Explain that ToolRegistry is canonical registry metadata, but runtime live discovery can override routing.
- Explain that config `tool_names` are fallback routing metadata.
- Mark static constants as compatibility / emergency fallback only.
- Document how to add a new tool safely.

## Acceptance Criteria
- Developers can identify where to register a new tool.
- Runtime routing behavior is documented.
- Config, registry, and live discovery roles are not ambiguous.
