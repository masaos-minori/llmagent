# [agent][history][diagnostics] Separate diagnostic data more clearly from normal conversation history

## Summary
Partial completions already use diagnostic storage, but other execution diagnostics still mix with normal history.

## Problem
Some execution failures are still turned into synthetic messages in conversation history.
This mixes user-facing context with internal execution diagnostics.

## Scope
- turn error handling
- diagnostic persistence
- conversation history behavior

## Task
- move more execution diagnostics into a diagnostic channel
- keep conversation history cleaner
- preserve replay and debugging usefulness

## Expected Result
Normal conversation history is cleaner, and diagnostic data is easier to analyze separately.

## Acceptance Criteria
- [ ] diagnostic data is separated more clearly from normal history
- [ ] execution-noise content is reduced in user-facing history
