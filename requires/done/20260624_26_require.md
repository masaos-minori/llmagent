# Align `/health` probe examples with documented server-specific health fields

## Summary
Update health-probe examples so they consistently reflect the documented base response shape plus server-specific override fields.

## Background
The protocol and server catalog documents describe server-specific `/health` behavior, while the operations document uses simplified probe guidance. The current wording can be made more consistent.

## Objective
Ensure `/health` examples are interpreted correctly for every MCP server.

## In Scope
- `04_mcp_06_configuration_and_operations.md`
- Cross-reference consistency with:
  - `04_mcp_02_protocol_and_transport.md`
  - `04_mcp_04_server_catalog.md`

## Required Changes
- Update probe examples to distinguish:
  - base response shape
  - server-specific extra fields
- Remove wording that implies a server has only the base response if documented overrides exist
- Keep probe examples concise but accurate

## Acceptance Criteria
- `/health` examples do not contradict the protocol and server catalog chapters
- Operators can tell which fields are common and which are server-specific
- Health-probe guidance remains concise and operationally useful

## Out of Scope
- Changing actual `/health` endpoints
- Adding new health fields

## Notes
Prefer consistency over brevity where the two conflict.
