# [MCP][Routing] Move routing toward live /v1/tools discovery

## Summary
Routing currently depends on both static tool constants and optional config-driven tool names.

## Problem
This creates maintenance overhead and makes new tool onboarding fragile.

## Scope
- `shared/route_resolver.py`
- tool discovery / startup validation flow

## Task
- design routing to prefer live-discovered tool metadata from each server
- keep static fallback only for compatibility or emergency use
- reduce the number of manual routing update points

## Expected Behavior
- new tools can be routed with less manual code change
- routing derives more truth from live server metadata

## Acceptance Criteria
- [ ] live-discovered tools can populate routing
- [ ] static fallback remains optional
- [ ] adding a new tool requires fewer manual updates
