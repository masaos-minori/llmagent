## [High][security][cicd] Change cicd-mcp workflow_allowlist from fail-open to fail-closed

### Summary
`cicd-mcp.workflow_allowlist` currently behaves as fail-open when empty.
This is inconsistent with the stricter fail-closed controls used elsewhere.

### Problem
An empty workflow allowlist should not silently grant access to all workflows.
The current behavior creates asymmetric security policy inside the same server.

### Scope
- mcp/cicd/*
- config/cicd_mcp_server.toml
- security validation at startup
- MCP security documentation

### Task
- change `workflow_allowlist` empty behavior from allow-all to deny-all
- keep behavior consistent with `repo_allowlist`
- update validation and error messages
- update docs to reflect the new fail-closed behavior
- add regression tests for empty allowlist cases

### Expected Result
Empty `workflow_allowlist` denies all workflow access by default.

### Acceptance Criteria
- empty `workflow_allowlist` denies all workflow execution
- production and local behavior are explicit and consistent
- docs no longer describe `workflow_allowlist` as fail-open
- tests cover empty and non-empty allowlist behavior
