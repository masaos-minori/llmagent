# Add generation-aware invalidation to SemanticCache

## Summary
Prevent stale semantic cache hits after ingestion or configuration changes by introducing generation-aware invalidation.

## Background
The documented semantic cache is FIFO and keyed by embedding similarity plus history context. Cache generation invalidation is not clearly documented.

## Objective
Ensure semantic cache entries can be invalidated when the indexed corpus or relevant retrieval configuration changes.

## In Scope
- `scripts/rag/cache.py`
- `scripts/rag/pipeline.py`
- Any ingest completion path that should trigger cache invalidation

## Required Changes
- Introduce cache generation metadata such as:
  - `corpus_version`
  - `config_version`
  - optional embedding model version
- Include generation-aware checks in lookup logic
- Add invalidation or generation-bump behavior after corpus changes
- Update documentation if cache semantics change

## Acceptance Criteria
- Cache entries can be invalidated after relevant corpus changes
- Cache behavior remains deterministic
- Existing history-context separation is preserved
- The final invalidation model is documented clearly

## Out of Scope
- Replacing FIFO with LRU unless explicitly justified
- Redesigning the entire cache implementation

## Implementation Notes
Keep the implementation simple and observable. Do not add complexity unless it directly prevents stale retrieval context.
