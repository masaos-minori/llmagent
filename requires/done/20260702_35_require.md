# [Low] Split responsibilities across MCP routing, config, and registry tests

## Priority

Low

## Goal

Make MCP tests easier for implementation agents to maintain.

## Background

Some tests currently mix route resolution, config model validation, and registry drift validation. tests/test_route_resolver.py should focus on routing behavior.

## Scope

Affected files:

- tests/test_route_resolver.py
- tests/test_mcp_config.py
- tests/test_tool_registry.py
- tests/test_tool_registry_counts.py

## Tasks

- Keep route resolution behavior in tests/test_route_resolver.py.
- Keep McpServerConfig enum and field validation in tests/test_mcp_config.py.
- Move registry drift validation tests to tests/test_tool_registry.py.
- Keep count regression in tests/test_tool_registry_counts.py.
- Rename classes and test methods to reflect their scope.

## Unknowns / Open Questions

## Acceptance Criteria

- Each test file has one clear responsibility.
- No routing test depends on removed stdio config behavior.
- Test names clearly describe the invariant being checked.

## Validation
