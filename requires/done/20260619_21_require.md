# Title
Fix `github-mcp` `app_module` to match the standard server module path pattern

## Summary
Correct the `app_module` value for `github-mcp` so that it matches the module path pattern used by the other MCP servers.

## Background
`github-mcp` currently uses a different `app_module` string than other servers.
This can break `run_http()` import behavior even though the current OpenRC startup path avoids the failure.

## Problem
This is an implementation inconsistency and a latent startup bug.

## Required Changes
- Change the `app_module` value to the standard dotted module path.
- Add or update a regression test that verifies all server `app_module` values are importable via the standard startup path.
- Document the fix if any environment-specific startup path still differs.

## Acceptance Criteria
- `github-mcp` uses the same `app_module` convention as the other servers.
- `run_http()` works consistently for all MCP servers.
- CI or tests detect future regressions.

## Out of Scope
- Replacing the OpenRC startup mechanism.
- Refactoring unrelated GitHub MCP features.

## Notes
This is a consistency and startup-safety fix.
