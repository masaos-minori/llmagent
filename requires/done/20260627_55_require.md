## Title
[High] Synchronize Event Bus schema documentation with `schema.sql` and `event_envelope.json`

## Priority
High

## Labels
documentation, eventbus, schema, api-contract

## Dependencies
None

## Background
The Event Bus database schema is defined in `scripts/eventbus/schema.sql`. The publish envelope contract is defined in `schemas/event_envelope.json`.

The documentation must reflect both schema contracts accurately.

## Scope
Update:
- `docs/06_eventbus_03_persistence_schema_and_replay.md`
- `docs/06_eventbus_02_http_api_and_runtime.md`
- `docs/06_eventbus_06_reference_api.md`
- `docs/06_eventbus_90_inconsistencies_and_known_issues.md`

## Required Changes
- Update Event Bus DB DDL documentation to match `schema.sql`.
- Document:
  - `acked_at`
  - `retry_count` as deprecated
  - `delivery_failure_count`
  - `dlq_requeue_count`
  - `dlq_at`
- Document indexes:
  - `idx_events_topic`
  - `idx_events_seq`
  - `idx_events_dlq_at`
  - `idx_events_dlq_seq`
- Update publish envelope documentation to match `event_envelope.json`.
- Document:
  - required fields: `event_id`, `topic`, `payload`, `producer`, `published_at`
  - `event_id` must be UUID v4
  - `payload` must be an object
  - `additionalProperties` is false
  - `schema_version` is optional
  - `published_at` must be date-time formatted

## Acceptance Criteria
- DB schema documentation matches `scripts/eventbus/schema.sql`.
- Publish request documentation matches `schemas/event_envelope.json`.
- Deprecated fields are clearly separated from active fields.
