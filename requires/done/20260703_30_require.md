# Remove backward-compatible re-exports from shared/tool_executor.py

## Priority

High

## Goal

Remove import compatibility paths from `shared/tool_executor.py` so the module focuses on `ToolExecutor`.

## Background

`shared/tool_executor.py` currently re-exports symbols for backward compatibility, including helper functions and transport DTOs.

These should be imported from their canonical modules.

## Scope

Affected files:

- `shared/tool_executor.py`
- all import sites using re-exported symbols
- tests importing re-exported symbols

## Re-exported symbols to remove

- `format_transport_error`
- `is_side_effect`
- `tool_hash_key`
- `ToolCallResult`
- `TransportErrorInfo`

## Required import replacements

Use canonical modules:

    from shared.transport_dto import ToolCallResult
    from shared.transport_dto import TransportErrorInfo
    from shared.tool_executor_helpers import format_transport_error
    from shared.tool_executor_helpers import is_side_effect
    from shared.tool_executor_helpers import tool_hash_key

## Acceptance criteria

- `shared.tool_executor` no longer imports symbols only for backward-compatible re-export.
- All call sites import from canonical modules.
- No backward-compatible stub remains.
- No import errors remain.
- `ToolExecutor` behavior is unchanged.

## Validation plan

- Search for `from shared.tool_executor import`.
- Replace imports with canonical modules.
- Run unit tests.
- Run static import checks if available.

## Out of scope

- Splitting `tool_executor_helpers.py`.
- Introducing new type aliases.
