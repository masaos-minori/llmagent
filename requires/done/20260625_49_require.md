## [Medium][eventbus][offsets] Make consumer_id to offset-filename mapping deterministic, collision-resistant, and consistent with documentation

### Summary
The current offset filename sanitization is inconsistent between docs and code and may allow collisions.

### Problem
A weak or inconsistent mapping from `consumer_id` to offset filename can cause overwrites, ambiguity, or implementation/documentation drift.

### Scope
- offsets.py
- offset file naming logic
- offset documentation

### Task
- define one canonical consumer-id sanitization rule
- make code and docs use the same rule
- reduce collision risk by using a stronger mapping strategy if needed
- preserve backward compatibility or provide a migration strategy if offset filenames change
- add tests for filename mapping edge cases

### Expected Result
Offset files are named deterministically, safely, and consistently across code and docs.

### Acceptance Criteria
- docs and code use the same sanitization rule
- collision risk is reduced explicitly
- unsafe or ambiguous filename mappings are tested
- backward-compatibility behavior is documented if relevant
