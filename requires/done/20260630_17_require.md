# [Medium] Move enum string conversion out of McpServerConfig runtime dataclass

## Priority
Medium

## Background
`McpServerConfig` accepts string literals and converts them to enums in `__post_init__`, while the config loader also performs enum conversion. This duplicates compatibility behavior for direct runtime callers.

## Goal
Move raw config parsing to the loader and make `McpServerConfig` expect normalized enum values.

## Scope
Affected files:
- `shared/mcp_config.py`
- `tests/test_mcp_config.py`
- `tests/test_route_resolver.py`
- `docs/04_mcp_06_configuration_and_operations.md`

## Implementation Tasks
- Remove or reduce `_cast_enums()` in `McpServerConfig`.
- Keep string-to-enum conversion in `_build_single_server()`.
- Ensure `_build_mcp_servers()` still accepts TOML string values.
- Update direct constructor tests to pass enum values.
- Move `healthcheck_mode=""` auto-inference to loader logic if needed.
- Remove documentation that says string literals are accepted at runtime by `__post_init__`.

## Acceptance Criteria
- Raw TOML values are parsed by the loader.
- Runtime `McpServerConfig` instances use normalized enum values.
- Direct construction with arbitrary strings is no longer supported unless explicitly intended.
- Existing config loading behavior remains compatible with TOML files.

## Validation
- Run `pytest tests/test_mcp_config.py tests/test_route_resolver.py -v`.
- Add tests showing `_build_mcp_servers()` still accepts TOML string values.
- Add tests showing direct invalid runtime construction fails clearly.
