# Title
Define `/undo` consistency with `tool_results` and partial-completion artifacts

## Priority
High

## Labels
- bug
- agent
- undo
- tool-results
- diagnostics
- session-consistency

## Summary
Clarify and implement how `/undo` should interact with full tool-result archives and partial-completion artifacts.

## Background
`/undo` is documented as removing the last user+assistant turn from history and DB.
At the same time, full tool outputs are stored outside `messages` in `tool_results`.
Partial LLM completions are also stored as diagnostic/tool-result artifacts.

## Problem
After `/undo`, obsolete tool outputs or partial-completion artifacts may still remain visible via `/tool list` or `/tool show`, even though the conversation turn was removed.
This creates a responsibility mismatch between conversation rollback and artifact persistence.

## Required Changes
- Decide whether `/undo` should remove, retain, or mark obsolete tool-result artifacts for the undone turn.
- Apply the same policy to partial-completion artifacts.
- Define the boundary between conversation history rollback and archive retention.
- Update `/tool list`, `/tool show`, and `/undo` docs accordingly.
- Add regression tests for “undo turn → inspect tool results”.

## Acceptance Criteria
- `/undo` no longer leaves artifact behavior undefined.
- Tool-result visibility after undo is intentional and documented.
- Conversation history and artifact storage no longer contradict each other semantically.

## Out of Scope
- Full redesign of `tool_results`
- Full redesign of `/tool` commands

## Components
- `/undo` command path
- `tool_results` handling
- partial completion artifact handling
- `/tool list`
- `/tool show`
- related docs and tests

## AI Implementation Instruction
Implement this as an undo/artifact consistency task.

Strict rules:
- Do not silently keep or delete artifacts without documenting the policy.
- Keep the distinction between conversation history and full-output archive explicit.
- If exact artifact ownership by turn is currently hard to derive, add traceable linkage and mark uncertain cases as: `Needs confirmation`
