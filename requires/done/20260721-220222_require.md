# Require: Make workflow schema check read-only mode explicit

## Priority

Low

## Target files

- `scripts/agent/repl_health.py`

## Background

The `check_workflow_schema()` function validates the workflow database schema by reading table/column metadata. It opens the database with `write_mode=False` to ensure no writes occur during validation. However, this parameter is a magic boolean that is easy to misread.

## Problem

- `write_mode=False` is a non-obvious way to express "read-only access". A future developer might accidentally change it to `True`, enabling unintended writes during schema validation.
- When `db_path=None`, the default workflow DB path is used, but if the DB file does not exist, an exception is raised without any handling.
- No return value — callers cannot programmatically determine whether the schema is valid.

## Reason for change

Improve code clarity and safety by making the read-only intent explicit, adding graceful handling for missing DB files, and returning a result object instead of raising exceptions.

## Implementation intent

Make the read-only mode explicit via a named constant, add graceful handling for missing DB files using os.path.exists(), and change the function to return a result object instead of raising exceptions.

## Implementation instructions

In `scripts/agent/repl_health.py`:

1. Add a module-level constant:
```python
_WORKFLOW_SCHEMA_READ_ONLY = True
```

2. Replace the magic boolean:
```python
db.open(write_mode=_WORKFLOW_SCHEMA_READ_ONLY, row_factory=False)
```

3. Change the function signature and return type:
```python
from dataclasses import dataclass

@dataclass(frozen=True)
class SchemaCheckResult:
    valid: bool
    error: str | None = None

def check_workflow_schema(db_path: str | None = None) -> SchemaCheckResult:
    """Return SchemaCheckResult indicating whether the workflow DB schema is valid."""
    from db.helper import SQLiteHelper  # noqa: PLC0415
    from db.schema_sql import WORKFLOW_SCHEMA_VERSION  # noqa: PLC0415

    db = SQLiteHelper(target="workflow", db_path=db_path)
    # Note: sqlite3.connect() creates files automatically; check existence first
    if db_path and not os.path.exists(db_path):
        return SchemaCheckResult(valid=False, error=f"Workflow DB not found: {db_path}")
    try:
        db.open(write_mode=_WORKFLOW_SCHEMA_READ_ONLY, row_factory=False)
    except Exception as e:
        return SchemaCheckResult(valid=False, error=f"Failed to open workflow DB: {e}")
```

4. Wrap the rest of the validation logic in a try/except block and return appropriate results.

## Acceptance criteria

- `write_mode=False` is replaced with a named constant `_WORKFLOW_SCHEMA_READ_ONLY`
- Missing DB file returns a `SchemaCheckResult(valid=False)` instead of raising an exception
- Other DB open failures also return `SchemaCheckResult(valid=False)` with an error message
- Valid schemas still pass without errors
- Callers can now programmatically check the result instead of catching exceptions

## Tests

- Unit test: Mock missing DB file, verify SchemaCheckResult(valid=False) is returned
- Unit test: Mock successful schema validation, verify SchemaCheckResult(valid=True) is returned
- Unit test: Verify the constant `_WORKFLOW_SCHEMA_READ_ONLY` exists and equals True
