# [Agent][Plugins] Add strict plugin loading mode

## Summary
Plugin loading is fail-open today. Broken plugin files are logged and skipped, and a missing plugin directory is treated as normal.

## Problem
This makes startup more tolerant, but it also hides plugin failures.
In production, a missing or broken plugin may silently remove expected behavior.

## Scope
- `shared/plugin_registry.py`
- plugin loading path during agent startup
- plugin-related startup logs

## Task
- add a strict plugin loading mode
- in strict mode, fail startup when a plugin import fails
- keep current fail-open behavior only for explicit development mode
- improve logging so skipped plugins are visible and actionable

## Expected Behavior
- production can require all expected plugins to load successfully
- development can still use fail-open plugin loading when desired
- plugin load failures are easy to diagnose

## Acceptance Criteria
- [ ] strict mode exists for plugin loading
- [ ] plugin import failures can fail startup in strict mode
- [ ] skipped plugins are clearly logged with file name and error summary
- [ ] missing plugin directory behavior is explicit and documented
