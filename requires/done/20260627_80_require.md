# Add workflow database rotation APIs

## Priority
Medium

## Labels
- db
- maintenance
- workflow
- backup
- medium-priority

## Dependencies
None

## Background
The system uses three SQLite databases: RAG, session, and workflow. Existing DB rotation support focuses on RAG and session databases. Workflow database rotation should be supported without breaking existing APIs.

## Scope
Modify:
- `db/maintenance.py`
- `db/__init__.py`
- `tests/test_db_maintenance.py`

## Required Changes
- Add `rotate_workflow_db(archive_dir: str | Path | None = None) -> Path`.
- Add `rotate_all_dbs(...)` for rag, session, and workflow.
- Preserve existing `rotate_db()` behavior and return type for backward compatibility.
- Export new APIs from `db/__init__.py`.

## Tests
Update `tests/test_db_maintenance.py`.

Add tests for:
- `rotate_workflow_db()` archives workflow.sqlite.
- `rotate_all_dbs()` archives all three DB files.
- Existing `rotate_db()` behavior remains unchanged.
- Missing DB file behavior is explicit and tested.

## Acceptance Criteria
- workflow.sqlite can be archived through DB maintenance APIs.
- Existing callers of `rotate_db()` are not broken.
- New APIs are exported from `db`.
