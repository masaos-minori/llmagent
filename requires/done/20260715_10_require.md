# Use command metadata as the single source of truth for dispatch, help, completion, and REPL-reserved commands

## Priority

High

## Type

CLI / REPL / Command Architecture

## Primary Files

- `agent/repl.py`
- `agent/commands/registry.py`
- `agent/commands/command_defs.py`
- `agent/commands/command_defs_list.py`

## Do Not Modify in This Issue

- `agent/commands/cmd_memory.py`
- `agent/commands/memory_*`
- `agent/config_*`
- `agent/tool_policy.py`
- `agent/tool_approval.py`
- Documentation files
- Test files, except command metadata tests if required

## Background

`_COMMANDS` is already used as the source of truth for command dispatch and `/help` output in `CommandRegistry`. However, `AgentREPL.SLASH_COMMANDS` is still maintained separately for tab completion and does not include all available commands.

`/exit` is also a special case. It is currently handled directly by the REPL before command dispatch and is not part of `_COMMANDS`.

## Problem

Command metadata is currently split across multiple places:

- `_COMMANDS` for dispatch and help
- `AgentREPL.SLASH_COMMANDS` for tab completion
- REPL-local handling for `/exit`

This causes drift between valid commands, help output, and tab completion candidates.

## Implementation Tasks

- [ ] Remove the manually maintained `AgentREPL.SLASH_COMMANDS` list.
- [ ] Generate tab completion candidates from `_COMMANDS`.
- [ ] Define `/exit` explicitly as a REPL-reserved command.
- [ ] Keep `/exit` outside `CommandRegistry.dispatch()` unless a deliberate design change is made.
- [ ] Include `/exit` in completion candidates.
- [ ] Include `/exit` in `/help` output as a REPL control command.
- [ ] Ensure `/memory`, `/audit`, `/plugin`, `/skill`, and `/mdq` are available as completion candidates.
- [ ] Add helper functions if useful, for example:
  - `builtin_command_names()`
  - `reserved_repl_command_names()`
  - `completion_command_names()`

## Acceptance Criteria

- [ ] Dispatch, `/help`, and tab completion are derived from `_COMMANDS` plus explicitly defined REPL-reserved commands.
- [ ] `/exit` behavior is clear in code comments and help output.
- [ ] No manually duplicated slash command list remains in `AgentREPL`.
- [ ] Every built-in command from `_COMMANDS` is available for tab completion.
- [ ] `CommandRegistry` still validates handler existence at startup.

## Dependencies

- Memory command surface cleanup is handled in a separate issue.
- Do not update `/memory` subcommand behavior in this issue.
