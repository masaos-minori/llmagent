# [Medium] Add ToolRegistry ownership and live drift validation tests

## Priority

Medium

## Goal

Lock down ToolRegistry ownership rules and live drift validation.

## Background

ToolRegistry is the source of truth for tool ownership. Live /v1/tools responses can override routing at runtime, so drift validation must be tested.

## Scope

Affected files:

- shared/tool_registry.py
- tests/test_tool_registry.py
- tests/test_tool_registry_counts.py

- Add a duplicate ownership rejection test.
- Add tests for validate_routing_against_live().
- Test live tool owner mismatch against registry owner.
- Test live tool missing from registry.
- Test registry tool missing from live list.
- Test that validate_all_routing() includes config and live validation results.
- Keep existing count tests:

  - GitHub 21 tools
  - RAG pipeline 4 tools

- Live-vs-registry drift is detectable.
- Routing drift behavior is covered by tests.
