# [db][docs][schema] Document DB triggers in the schema docs

## Summary
Important triggers exist in schema creation code, but they are not documented clearly enough in the schema docs.

## Problem
This makes it easy to miss that:
- FTS tables are auto-synced
- some vector cleanup is automatic

## Scope
- `06_shared_04_db_architecture_and_schema.md`
- `db/create_schema.py`

## Task
- add trigger DDL or trigger summaries to schema docs
- explain what each trigger does
- clearly say that manual sync is not needed

## Expected Result
- developers can understand automatic DB behavior from the docs alone

## Acceptance Criteria
- [ ] trigger behavior is documented
- [ ] sync rules for derived tables are documented
- [ ] docs warn against manual duplicate sync logic
