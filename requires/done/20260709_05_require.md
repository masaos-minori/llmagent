# [Medium] Clean up legacy tool execution and scheduling compatibility settings

## Labels
priority/medium, area/tool-execution, area/config, backward-compat-removal

## Summary
Remove obsolete tool execution settings and restrict legacy scheduling mode.

This task covers:

- removing `tool_results_turn_max_chars`
- treating `use_tool_dag=false` as legacy compatibility behavior

## Background
`tool_results_turn_max_chars` is deprecated and no longer enforced because the old `tool_results` table has been removed.

`use_tool_dag=false` enables legacy round-wide serialization behavior. The current standard is resource-scoped DAG scheduling based on `resource_scope`, `requires_serial`, and side-effect grouping.

## Required Changes

### Remove `tool_results_turn_max_chars`
- Remove the field from `ToolConfig`.
- Remove config builder parsing.
- Remove the key from `config/agent.toml`.
- Remove documentation for the field.
- Optionally add a migration warning if the key remains in config.

### Restrict `use_tool_dag=false`
Production behavior:

- production + `use_tool_dag=false` => RuntimeError or startup audit error

Local/development behavior:

- `use_tool_dag=false` remains allowed temporarily
- emit a warning that this is legacy compatibility mode

Future direction:

- consider removing `use_tool_dag` entirely and making DAG scheduling mandatory

### Documentation
- Update `05_agent_08_configuration.md`:
  - remove `tool_results_turn_max_chars`
  - describe `use_tool_dag=false` as legacy compatibility mode
  - document production requirement for DAG scheduling
- Update `04_mcp_06_configuration_and_operations.md`:
  - ensure resource-scoped scheduling is documented as the standard behavior
  - mark legacy round-wide serialization as non-production

### Tests
Update or add tests for:

- `tool_results_turn_max_chars` no longer part of `ToolConfig`
- old config key is ignored or warned about according to the chosen migration behavior
- production + `use_tool_dag=false` fails
- local/development + `use_tool_dag=false` warns only
- DAG scheduler behavior remains unchanged

Use existing `tests/test_tool_scheduler.py` for scheduler behavior tests.

Use `tests/test_production_config_validator.py` for production profile validation.

## Acceptance Criteria
- `tool_results_turn_max_chars` is removed from runtime config.
- Production rejects `use_tool_dag=false`.
- Local/development still allows `use_tool_dag=false` with a warning.
- Existing DAG scheduler tests continue to pass.
- Documentation no longer describes obsolete tool result table behavior.

## Target Files
- `agent/config_dataclasses.py`

## Non-Goals
- Do not rewrite the tool scheduler.
- Do not remove `use_tool_dag` immediately unless explicitly approved.
- Do not change `resource_scope` or `requires_serial` semantics.
