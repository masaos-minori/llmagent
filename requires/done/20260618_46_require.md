# [rag][architecture] Define the boundary between RAG, MDQ, and the Agent layer

## Summary
The role boundary between RAG, MDQ, and the Agent layer is still not defined clearly enough.

## Problem
Markdown content can overlap between:
- RAG
- MDQ
- agent-side document access

Without clear rules, ownership and migration become confusing.

## Scope
- RAG docs
- MDQ boundary docs
- agent integration boundary

## Task
- define when to use RAG
- define when to use MDQ
- define how the agent should access each system
- document migration criteria for overlapping use cases

## Expected Result
The system has clear ownership boundaries for document retrieval and indexing.

## Acceptance Criteria
- [ ] RAG vs MDQ usage rules are documented
- [ ] agent integration boundary is documented
- [ ] overlapping Markdown use cases are clarified
