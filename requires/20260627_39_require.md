## Title
[Medium] Fix memory consistency repair guidance for FTS and vector indexes

## Priority
Medium

## Labels
memory, consistency, cli, observability

## Dependencies
M-02

## Background
`/memory check-consistency` compares SQLite memory tables, FTS rows, vector rows, and JSONL archive count. When inconsistent, it currently suggests `/memory rebuild`.

However, `/memory rebuild` imports from JSONL and does not properly repair FTS/vector indexes.

## Scope
Update:
- `agent/commands/cmd_memory.py`
- Memory store layer if repair methods are needed
- Memory CLI tests

## Required Changes
- Replace misleading repair guidance.
- Add or define a dedicated FTS rebuild command.
- Clarify that vector index repair requires embedding regeneration.
- Do not recommend JSONL import as FTS/vector repair.

## Possible Commands
- `/memory rebuild-fts`
- `/memory rebuild-vec`
- `/memory repair-indexes`

## Tests to Add
- `test_memory_check_consistency_does_not_recommend_jsonl_import_for_index_repair`
- `test_memory_check_consistency_reports_jsonl_count_as_info_only`

## Acceptance Criteria
- Consistency output clearly distinguishes:
  - SQLite authoritative records
  - JSONL archive records
  - FTS index rows
  - vector index rows
- Repair guidance is accurate.
