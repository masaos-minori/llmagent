# [Low] Design optional embedding or hybrid search for MDQ Phase 2

## Priority
Low

## Labels
`mcp`, `mdq`, `embedding`, `hybrid-search`, `phase-2`

## Dependencies
- Align search_docs schema with actual implementation
- Redesign MDQ SQLite schema for production indexing

## Background
MDQ should first be production-ready for Markdown structural retrieval using FTS5 and grep. Optional embedding or hybrid search can be added later if Markdown-only semantic retrieval is needed.

## Task
Design an optional embedding/hybrid search extension for MDQ.

## Implementation Requirements
- Keep FTS5-only as production baseline.
- Add `use_embedding = false` by default.
- Define vector table only when embedding is enabled.
- Use RRF or another explicit merge strategy if hybrid search is enabled.
- Preserve RAG/MDQ responsibility boundary.
- Document when to use MDQ hybrid search versus RAG.

## Acceptance Criteria
- Hybrid search is optional.
- Existing MDQ FTS5 behavior remains unchanged.
- RAG remains the production default for general semantic retrieval.
- MDQ hybrid search does not access `rag.sqlite`.
