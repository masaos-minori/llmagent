# H-3: Add shutdown guard to ServerLifecycleRouter

## Priority

High

## Background

`_ServerLifecycleRouter` is the lifecycle facade injected as `LifecycleManagerProtocol`.

Watchdog can call `lifecycle.restart(server_key)`, and `ToolExecutor` can call `lifecycle.ensure_ready(server_key)` before each tool call. During shutdown, lifecycle APIs must not start or restart subprocesses.

## Target files

- `agent/factory.py`

## Implementation tasks

- Add `self._shutting_down = False` to `_ServerLifecycleRouter.__init__()`.
- Set `self._shutting_down = True` at the beginning of `shutdown_all()`.
- Make the following methods return without starting or restarting subprocesses when `_shutting_down` is true:
  - `ensure_ready()`
  - `start_http_subprocess()`
  - `restart()`
  - `shutdown_idle()`
- Log a debug or warning message when a start or restart request is ignored due to shutdown.
- Ensure the guard is idempotent.

## Acceptance criteria

- Once `shutdown_all()` starts, no later call to `ensure_ready()` can start a subprocess.
- Once `shutdown_all()` starts, no later call to `restart()` can restart a subprocess.
- `shutdown_idle()` remains safe during shutdown.
- Repeated `shutdown_all()` calls are safe.

## Test plan

- Call `shutdown_all()` and then `restart(server_key)`; assert no process is started.
- Call `shutdown_all()` and then `ensure_ready(server_key)`; assert no process is started.
- Verify ignored restart/start calls are logged.
- Verify repeated `shutdown_all()` calls do not raise.
