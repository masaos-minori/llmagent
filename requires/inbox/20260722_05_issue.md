# Verify and document web-search/browser_fetch runtime availability behavior

## Priority

High

## Scope

Documentation and implementation verification.

## Target documentation sections

- `04_mcp_04_01_web-search-file-read-github.md`
  - `web-search-mcp`
  - `browser_fetch`
  - `設定パラメータ`
  - `実装上の補足`
- `04_mcp_03_06_tool-runtime-availability-metadata.md`
  - `config_dependent`
  - `enabled / disabled_reason`

## Problem

`browser_fetch` is marked as `config_dependent=true`, but it is not confirmed from the reviewed files whether `/v1/tools` returns runtime availability fields for it.

The web-search server delegates `/v1/tools` response construction to `build_tools_response()`. The behavior of that helper must be verified before the docs can claim full availability metadata support.

## Required investigation

Inspect `mcp_servers/server.py::build_tools_response()` and verify:

- Does it add `server_key`?
- Does it add `schema_version`?
- Does it add `enabled`?
- Does it add `disabled_reason`?
- Does it use server-specific config state?
- Does `browser_allowed_domains=[]` disable `browser_fetch`?
- Is `search_web` independent of browser-specific allowlist configuration?

## Required documentation outcome

Document one of the following:

### If availability metadata is implemented

- `browser_fetch` is `config_dependent=true`.
- `browser_fetch` is disabled when the browser domain allowlist denies all access.
- `/v1/tools` returns the correct `enabled` and `disabled_reason`.
- `search_web` availability is documented separately from `browser_fetch`.

### If availability metadata is not implemented

- `browser_fetch` currently has `config_dependent=true`.
- Runtime `enabled/disabled_reason` support is not yet implemented for web-search.
- Add a follow-up implementation task.

## Reason for change

browser_fetch is controlled by fail-closed domain allowlisting. If availability metadata is inaccurate or undocumented, the LLM and operator may not understand why browser_fetch is unavailable.

## Acceptance criteria

- web-search catalog accurately describes current `browser_fetch` availability behavior.
- Runtime availability metadata docs do not overstate web-search support.
- Required follow-up implementation work is explicitly documented if missing.
