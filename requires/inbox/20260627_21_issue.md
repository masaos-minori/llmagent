# docs/00_index.md を最上位ドキュメント入口として整備する

## 概要

LLM エージェント設計書群の最上位入口として `docs/00_index.md` を整備する。

`docs/` 以下はフラット構成を維持し、既存のドキュメント分割方針は変更しない。

## 修正理由

現在の設計書群は、Overview / Deployment / RAG / MCP / Agent / EventBus / Shared/DB などに分割されており、各領域には document-guide や related documents が存在する。

一方で、設計書群全体を読む際の最初の入口、読者別の読み順、governance 文書への導線、正典ソースの参照先が分散している。

このため、AI エージェントや新規参加者が最初に参照すべき文書を判断しやすくするために、`docs/00_index.md` を最上位入口として明確化する。

## 修正意図

この修正の意図は、既存文書を統合・再編することではなく、既存の細かい分割構成を維持したまま、全体の入口とルーティングを明確にすることである。

`00_index.md` には詳細仕様を集約せず、以下に役割を限定する。

- 設計書群全体の入口
- 読者別の読み順
- 各領域の document-guide への導線
- governance 文書への導線
- AI 読み込み時の初期参照先

## 対象ファイル

- `docs/00_index.md`

存在しない場合は新規作成する。
既に存在する場合は、既存内容を尊重して追記・整理する。

## 実施内容

以下のセクションを追加または整理する。

- Purpose
- Reading Routes
- Document Sets
- Governance Documents
- Canonical Source Reference
- AI Reading Guidance
- Notes

## 必須要件

- `docs/` 以下はフラット構成のままにする。
- `00_governance_*.md` へのリンクを含める。
- 各領域の document-guide へのリンクを含める。
- EventBus と Workflow の関係整理は追加しない。
- 既存のドキュメント分割サイズを変更しない。
- 詳細仕様を `00_index.md` に移動しない。
- `00_index.md` は入口・ルーティング・参照案内に限定する。
