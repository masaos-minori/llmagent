## 重点確認項目

以下を重点的に確認してください。

- 現行 slash command 一覧とドキュメントのコマンド表が一致しているか
- 削除済みコマンドの説明が残っていないか
- WorkflowEngine が必須である現行実装と一致しているか
- 直接実行 fallback があるかのような旧説明が残っていないか
- DiagnosticStore / session_diagnostics の説明が現行実装と一致しているか
- 削除済み diagnostics.jsonl などの説明が残っていないか
- LLMTurnRunner / ToolLoopGuard / partial completion の説明が現行挙動と一致しているか
- CLIView / OutputTag / startup validation 表示の説明が実装と一致しているか
- AgentContext / AppServices の初期化フローが現行実装と一致しているか
- 同じ turn flow や workflow 説明が複数章に重複していないか

## 修正方針

- Agent 全体像は `05_agent_01_system-overview.md` に残す。
- 詳細な runtime architecture は `05_agent_02_*` に集約する。
- turn flow は `05_agent_03_*` に集約する。
- state / persistence は `05_agent_04_*` に集約する。
- CLI command reference は `05_agent_07_*` に集約する。
- startup / health / diagnostics は `05_agent_10_*` に集約する。
- 削除済みコマンドや旧実装説明は削除する。
- 未解決の不整合だけを `05_agent_90_inconsistencies_and_known_issues.md` に残す。

基本方針は以下です。

1. 実装とドキュメントが矛盾する場合は、実装を正本としてドキュメントを修正する。
2. 現行仕様に影響しない過去経緯・移行履歴・旧仕様説明は削除する。
3. 実装で削除済みの機能、API、CLIコマンド、設定キー、DBカラム、型、関数、ファイルの説明は削除する。
4. 同一内容の説明が複数箇所にある場合は、正典となる1箇所に集約し、他は削除または参照リンクだけにする。
5. 判断できない場合は推測せず、必要な実装ファイル・設定ファイル・テストファイルを列挙する。
6. Known Issues に残すのは、現行仕様に影響する未解決事項だけにする。
7. 修正後は File Index、AI Query Routing Table、Related Documents、内部リンクも整合させる。
