# 文書統制ルール docs/00_governance_01_documentation-governance.md を追加する

## 概要

LLM エージェント設計書群の維持管理ルールを定義する governance 文書を追加する。

## 修正理由

現在の設計書群には、領域別の document-guide、関連文書、Known Issues、設計ノートが存在する。

しかし、設計書群全体に共通する文書運用ルール、更新条件、レビュー条件、文書種別の定義が明確に一元化されていない。

そのため、設計変更、設定変更、MCP tool 仕様変更、DB schema 変更、安全境界変更などが発生した際に、どのドキュメントを更新すべきか判断しにくい。

## 修正意図

この文書は、詳細仕様を記載する文書ではなく、設計書群を維持管理するための横断ルールを定義する。

目的は以下である。

- 文書種別を明確化する
- 変更時に更新すべき文書を判断可能にする
- 安全境界や DB schema 変更時のレビュー漏れを防ぐ
- 各領域の document-guide や Known Issues の運用を支援する

## 対象ファイル

- `docs/00_governance_01_documentation-governance.md`

## 実施内容

以下のセクションを含む文書を作成する。

- Purpose
- Scope
- Document Classes
- Update Rule
- Review Rule
- Change Impact Rule
- Non-Goals
- Related Governance Documents

## 対象範囲

対象に含める領域:

- Overview
- Deployment
- RAG
- MCP
- Agent
- EventBus
- Shared/DB
- Governance

対象外:

- ソースコードそのもの
- 一時的な作業メモ
- 個人検証メモ
- EventBus と Workflow の関係再整理
- ドキュメント分割方針の変更

## 必須要件

- `docs/` 直下に作成する。
- サブディレクトリは作成しない。
- 既存ドキュメントの構成変更は行わない。
- この文書は文書運用ルールに限定する。
- 具体的な RAG / MCP / Agent の詳細仕様は記載しない。

## AI 実装時の共通指示

- この Issue の対象範囲のみを実施すること。
- 実装コードは変更しないこと。
- 既存ドキュメントの移動、統合、削除、リネームは行わないこと。
- `docs/` 以下に新規サブディレクトリを作成しないこと。
- Markdown の構造を壊さないこと。
- コードブロック、リンク、見出し階層を正しく維持すること。
- 判断に迷う場合は推測で修正せず、未解決事項として報告すること。
- EventBus と Workflow の関係整理は本 Issue では行わないこと。
- ドキュメント分割サイズを変更しないこと。
- 作業後、変更ファイル、変更概要、未解決事項を報告すること。

## 作業禁止事項

- 他の governance 文書を同時に作成しない。
- 既存 document-guide を修正しない。
- Known Issues の個別エントリを修正しない。
- Deprecated Items の具体一覧はこの文書に書かない。

## 検証観点

- 文書統制の目的が明確であること。
- どの変更でどの文書を更新すべきか判断できること。
- 他の governance 文書との責務分担が明確であること。
- Markdown が壊れていないこと。
