# 廃止項目一覧 docs/00_governance_05_deprecated-items.md を追加する

## 概要

旧設定ファイル名、旧概念、旧コマンドなどを集約管理する Deprecated Items 文書を追加する。

## 修正理由

既存の設計書群には、過去の設計や旧構成に由来する記述が一部残っている。

例として、旧設定ファイル名や廃止された概念が本文中に注記として現れる場合、それぞれの箇所で個別に説明すると、現行仕様との関係が分かりにくくなる。

## 修正意図

この文書の意図は、旧記述や廃止済み項目を各本文から即時削除することではなく、廃止項目の参照先を一元化することである。

この文書により、以下を実現する。

- 旧設定名と現行設定名の対応を確認できる
- 廃止済み概念を現行仕様と混同しない
- 本文中の旧記述を段階的に参照集約できる
- AI が古い仕様を現行仕様として誤認するリスクを減らす

## 対象ファイル

- `docs/00_governance_05_deprecated-items.md`

## 実施内容

以下のセクションを含む文書を作成する。

- Purpose
- Deprecated Configuration Files
- Deprecated Concepts
- Deprecated Commands
- Deprecated Document References
- How to Refer to Deprecated Items
- Maintenance Rule

## 初期登録候補

以下を初期登録候補として記載する。

- `config/rag_pipeline.toml`
- `common.toml`
- workflow optional mode
- shared common config

ただし、実装や既存文書で確認できる範囲に限定し、断定できないものは `Needs confirmation` とする。

## 必須要件

廃止項目ごとに以下を持たせる。

- Deprecated Name
- Current Replacement
- Status
- Notes
- Evidence

## AI 実装時の共通指示

- この Issue の対象範囲のみを実施すること。
- 実装コードは変更しないこと。
- 既存ドキュメントの移動、統合、削除、リネームは行わないこと。
- `docs/` 以下に新規サブディレクトリを作成しないこと。
- Markdown の構造を壊さないこと。
- コードブロック、リンク、見出し階層を正しく維持すること。
- 判断に迷う場合は推測で修正せず、未解決事項として報告すること。
- EventBus と Workflow の関係整理は本 Issue では行わないこと。
- ドキュメント分割サイズを変更しないこと。
- 作業後、変更ファイル、変更概要、未解決事項を報告すること。

## 作業禁止事項

- 既存文書中の旧記述を削除しない。
- 旧設定名の全検索・全置換は行わない。
- 既存仕様を変更しない。
- 未確認の廃止項目を断定しない。

## 検証観点

- 旧設定・旧概念の扱いが分かること。
- 現行置換先が分かること。
- 未確認項目を断定していないこと。
- 今後、本文側からこの文書へ参照集約できること。
