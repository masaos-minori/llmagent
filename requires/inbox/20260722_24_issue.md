# 証拠ラベル定義 docs/00_governance_03_evidence-labels.md を追加する

## 概要

設計書群で使用する証拠ラベルを定義する。

## 修正理由

既存の設計書群では、実装根拠や確認状態を示すために、以下のような表現が使用されている。

- Explicit in code
- Strongly implied by code
- Needs confirmation
- Deprecated

これらは有用だが、ラベルの意味や使用条件が統一されていないと、読み手によって解釈が変わる可能性がある。

特に `Needs confirmation` は、未確認状態を表す重要なラベルであり、放置されると仕様の信頼性が低下する。

## 修正意図

この文書の意図は、設計書内の根拠表現を標準化し、各記述の信頼度や確認状態を読み手が判断できるようにすることである。

この文書により、以下を実現する。

- 実装確認済みの記述を明確化する
- 実装から推測した記述を区別する
- 未確認事項を追跡可能にする
- 廃止済み記述を現行仕様と混同しないようにする
- AI が設計書を読む際に根拠の強さを判断しやすくする

## 対象ファイル

- `docs/00_governance_03_evidence-labels.md`

## 実施内容

以下の証拠ラベルを定義する。

- Explicit in code
- Strongly implied by code
- Documentation only
- Needs confirmation
- Deprecated
- Verified by test
- Operationally observed

## 必須要件

- 各ラベルについて、意味、使用条件、使用例、注意点を記載する。
- `Needs confirmation` は一時状態として定義する。
- `Needs confirmation` に必須項目を定義する。
- `Deprecated` は現行仕様ではない旧記述として定義する。
- 判断に迷う場合の扱いを記載する。

## Needs confirmation 必須項目

以下を定義する。

- Question
- Evidence
- Impact
- Required Action
- Target Document
- Review Timing
