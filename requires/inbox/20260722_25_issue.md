# Known Issues 共通テンプレート docs/00_governance_04_known-issues-template.md を追加する

## 概要

各領域の Known Issues / Inconsistencies 文書で利用する共通エントリ形式を定義する。

## 修正理由

既存の設計書群には、RAG / MCP / Agent / EventBus / Shared/DB などの領域ごとに Known Issues または Inconsistencies 系の文書が存在する。

しかし、各エントリの形式、ステータス、重要度、種別、修正方針の記載粒度が統一されていない場合、未解決課題の優先順位や修正状況を判断しにくい。

## 修正意図

この文書の意図は、Known Issues を単一ファイルに統合することではなく、各領域の Known Issues 文書を維持したまま、エントリ形式だけを統一することである。

この文書により、以下を実現する。

- 未解決・調査中・修正済み・将来対応を区別できる
- 文書と実装の不一致を追跡できる
- 影響度と修正優先度を判断できる
- AI が未解決課題を抽出しやすくなる
- 将来の段階的移行がしやすくなる

## AI 実装時の共通指示

- この Issue の対象範囲のみを実施すること。
- 実装コードは変更しないこと。
- 既存ドキュメントの移動、統合、削除、リネームは行わないこと。
- `docs/` 以下に新規サブディレクトリを作成しないこと。
- Markdown の構造を壊さないこと。
- コードブロック、リンク、見出し階層を正しく維持すること。
- 判断に迷う場合は推測で修正せず、未解決事項として報告すること。
- EventBus と Workflow の関係整理は本 Issue では行わないこと。
- ドキュメント分割サイズを変更しないこと。
- 作業後、変更ファイル、変更概要、未解決事項を報告すること。

## 作業禁止事項

- 既存 Known Issues ファイルを修正しない。
- 既存エントリを新テンプレートに移行しない。
- 個別の不整合内容を追加しない。
- Known Issues を単一ファイルに統合しない。

## 検証観点

- 各領域の Known Issues に適用可能な形式であること。
- Status と Type が明確であること。
- 既存 Known Issues の段階的移行に使えること。
- AI が未解決項目を抽出しやすい形式であること。
