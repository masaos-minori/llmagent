# Remove all schema migration logic from `create_schema.py`

## Priority

High

## Summary

Remove all backward-compatible schema migration logic from `create_schema.py` and make schema creation DDL-only.

## Background

The project policy is that existing DB-compatible schema migration is not supported. Schema changes must be applied by recreating DB files from the latest DDL.

`create_schema.py` still contains migration helpers such as:

- `_migrate_rag_schema()`
- `_migrate_session_schema()`
- `_migrate_workflow_schema()`
- `_migrate_add_undone_column()`

These helpers conflict with the DDL-only schema creation policy.

## Goal

Make `create_schema.py` execute only the canonical DDL from `schema_sql.py`.

## Scope

### In scope

- Remove all migration helper functions.
- Remove all migration calls.
- Remove direct `db.conn` access used only for migration.
- Keep DDL execution and DDL error handling.

### Out of scope

- Existing DB migration.
- Backward compatibility for old DB schemas.
- Data migration tooling.

## Implementation tasks

- Remove these functions from `create_schema.py`:
  - `_migrate_rag_schema()`
  - `_migrate_session_schema()`
  - `_migrate_workflow_schema()`
  - `_migrate_add_undone_column()`
- Remove all calls to those functions.
- Ensure these functions only execute DDL:
  - `create_rag_schema()`
  - `create_session_schema()`
  - `create_workflow_schema()`
- Remove migration-related comments.
- Remove stale `type: ignore` comments related to direct `db.conn` access.
- Keep existing logging and exception handling for DDL failures.

## Acceptance criteria

- `create_schema.py` contains no `_migrate_*` function.
- `create_schema.py` does not execute `ALTER TABLE`.
- `create_schema.py` does not access `db.conn` directly.
- `create_schema()` still creates all three DB schemas:
  - `rag.sqlite`
  - `session.sqlite`
  - `workflow.sqlite`
- Existing DB schema differences are not patched automatically.

## Validation

- Run:
```bash
uv run pytest tests/test_create_schema.py
```
- Run:
```bash
python -c "from db.create_schema import create_schema; create_schema()"
```

## Confirmation required

None. The policy is fixed: existing DB migration is not supported.
