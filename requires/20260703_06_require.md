# Define and enforce plugin command shadowing policy

## Priority

Medium

## Labels

plugin, security, cli, design, command-registry

## Background

Plugin command conflict validation detects when a plugin command has the same name as a built-in command.

Currently, built-in commands are dispatched before plugin commands, so a plugin command with the same name normally cannot override built-in behavior.

However, the shadowing plugin command can remain in the plugin registry, which may create confusion in status output, tests, or future dispatch changes.

## Goal

Define a clear policy for plugin commands that shadow built-in command names.

## Scope

### In scope

- Decide the canonical behavior for plugin command shadows
- Update implementation and documentation accordingly
- Ensure `/plugin status` reflects the chosen behavior

### Out of scope

- Plugin tool shadowing policy
- MCP tool conflict behavior
- Built-in command list changes

## Policy options

Choose one:

### Option A: Reject shadowing plugin commands

- Remove plugin commands from the registry when they shadow built-in commands
- Record them as rejected or shadowed in plugin load result
- Prefer this option for clarity and safety

### Option B: Keep shadowing plugin commands but make them inert

- Leave plugin commands in the registry
- Document that built-in command priority prevents execution
- Ensure status output clearly reports that they are shadowed

## Recommended option

Use Option A unless there is a strong compatibility reason to keep shadowed commands registered.

## Acceptance criteria

- The chosen policy is documented
- Runtime behavior matches the documented policy
- Plugin status output is not misleading
- Tests cover a plugin command that shadows a built-in command

## Validation

Create a test plugin registering `/help`.

Expected result depends on the selected policy:

- Option A: plugin `/help` is rejected or removed
- Option B: plugin `/help` is retained but never dispatched before built-in `/help`

## Affected files

- `scripts/shared/plugin_registry.py`
- `scripts/agent/commands/registry.py`
- Plugin tests
- Plugin documentation

## Dependencies

None
