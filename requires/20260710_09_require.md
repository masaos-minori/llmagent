# [Low] Add workflow mandatory deployment runbook

## Background

Because workflow is mandatory, common deployment and startup failures need an operator runbook.

## Goal

Create a runbook for workflow deployment, validation, and recovery.

## Scope

- Operations documentation
- Deployment documentation

## Tasks

- [ ] Add a runbook section for missing `config/workflows/default.json`.
- [ ] Add a runbook section for invalid workflow JSON.
- [ ] Add a runbook section for missing required stages.
- [ ] Add a runbook section for invalid retry policy.
- [ ] Add a runbook section for missing or incomplete `workflow.sqlite`.
- [ ] Add a runbook section for schema version mismatch.
- [ ] Add a runbook section for workflow definition update requiring restart.
- [ ] Add commands for validating workflow definition.
- [ ] Add commands for validating workflow DB schema.

## Acceptance Criteria

- [ ] Operators can recover from missing workflow definition.
- [ ] Operators can recover from invalid workflow definition.
- [ ] Operators can recover from workflow DB initialization failures.
- [ ] Runbook explicitly states that workflow cannot be disabled.
- [ ] Runbook includes copy-pasteable commands.

## Notes

The runbook should be operationally focused and avoid implementation internals unless necessary.
