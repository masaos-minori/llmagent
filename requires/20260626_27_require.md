# [Medium] Improve MDQ audit logging target and details

## Priority
Medium

## Labels
`mcp`, `mdq`, `audit`, `observability`

## Dependencies
- Align mdq-mcp with MCPServer base class behavior

## Background
Current audit target extraction only checks `query` and `path`. This loses important information for `get_chunk`, `grep_docs`, and indexing tools.

## Task
Improve MDQ audit target and detail generation per tool.

## Implementation Requirements
Set audit target as follows:
- `search_docs`: query plus optional path filter
- `get_chunk`: chunk ID
- `outline`: path
- `index_paths`: path count or first path
- `refresh_index`: path count or first path
- `grep_docs`: regex pattern
- `stats`: service name or DB path

Add useful detail fields where supported:
- result count
- elapsed time
- truncated flag
- indexed count
- skipped count
- deleted count
- error kind

## Acceptance Criteria
- Audit logs can identify what each MDQ call targeted.
- `X-Session-Id` and `X-Request-Id` correlation remains intact.
- Operational debugging can distinguish search, chunk retrieval, grep, and indexing calls.
