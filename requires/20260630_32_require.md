# [Medium] Validate GitHub tool definitions and MCP server config consistency

## Priority
Medium

## Background
GitHub MCP tool schemas are defined in code, but config-side definitions and MCP server configuration still need validation. ConfigLoader loads both `mcp_servers.toml` and `tools_definitions.toml` as base config files. 【8-1f7596】

## Goal
Ensure GitHub `ToolRegistry` ownership, GitHub MCP `TOOL_LIST`, LLM tool definitions, and MCP server config are consistent.

## Scope
Affected files:
- `config/tools_definitions.toml`
- `config/mcp_servers.toml`
- `shared/tool_registry.py`
- tests

## Implementation Tasks
- Verify all 21 GitHub tool names exist in `config/tools_definitions.toml`.
- Verify `[mcp_servers.github]` exists in `config/mcp_servers.toml`.
- Ensure `tool_names` is not required for routing.
- If `tool_names` remains in config, treat it as validation metadata only.
- Add tests comparing:
  - GitHub `TOOL_LIST`
  - `ToolRegistry` GitHub tools
  - LLM tool definitions
  - optional config `tool_names`

## Acceptance Criteria
- GitHub tool definitions are complete and consistent.
- Routing does not depend on config `tool_names`.
- Config drift is reported as validation output, not used for routing.
- Tests pass.

## Validation
- Run: `pytest tests/test_github_tool_registry.py tests/test_tool_registry_counts.py -v`
