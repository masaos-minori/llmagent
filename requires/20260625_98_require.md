## Summary
Extend `/memory` commands so humans can inspect and correct stored knowledge more easily using source-type and context-boundary views. The current CLI already includes list/search/show/pin/unpin/delete/prune/status/check-consistency/rebuild.

## Background
The current `/memory` command set is operationally useful, but it is weaker for human review workflows such as:
- “show only project rules”
- “show only failures”
- “show decisions for this repo/branch”

## Problem
If humans cannot inspect and correct persistent memory easily, long-term quality will drift.

## Required Changes
- Consider source-type filtering on `/memory list`.
- Consider project/repo/branch filtering on `/memory list` and `/memory search`.
- Consider quick views for pinned rules, recent failures, or decision-only entries.

## Acceptance Criteria
- Humans can inspect stored memory more efficiently.
- Context-boundary filters align with the retrieval model.
- CLI and docs remain consistent.

## Out of Scope
- Full UI redesign
- Memory data model redesign
