# [shared][plugins] Add strict plugin loading mode

## Summary
Plugin loading is fail-open today.

## Problem
In production, a broken plugin may be skipped silently and the process still starts.

## Scope
- `shared/plugin_registry.py`
- startup integration points

## Task
- add a strict mode for plugin loading
- in strict mode:
  - failed plugin import should fail startup
- keep current fail-open behavior as an explicit non-strict mode

## Expected Result
- production can require all expected plugins to load successfully

## Acceptance Criteria
- [ ] strict mode exists
- [ ] startup can fail on plugin load errors in strict mode
- [ ] non-strict fail-open mode still exists
