# [db][typing] Change SQLiteHelper.target to a safer typed value

## Summary
`SQLiteHelper(target)` uses plain strings to choose the DB.

## Problem
Valid values are:
- `"rag"`
- `"session"`
- `"workflow"`

But the API is string-based and easy to misuse.

## Scope
- `db/helper.py`

## Task
- change the target argument to:
  - `Literal[...]`
  - or an Enum
- keep clear validation for invalid targets

## Expected Result
- target selection is safer and easier for tools and type checkers

## Acceptance Criteria
- [ ] target uses a safer type than plain `str`
- [ ] invalid target handling still works
- [ ] docs show the final accepted values clearly
