# [Medium] Lock config tool_names as validation-only metadata

## Goal

Ensure `config/mcp_servers.toml tool_names` is consistently documented and treated as validation-only metadata, never as a routing source.

## Background

`tool_names` should remain available for drift validation, but it must not be used for routing. Routing must be resolved only by live `/v1/tools` discovery and `ToolRegistry`.

## Scope

- Documentation
- Comments
- Tests, if any test implies routing behavior from `tool_names`

## Implementation Tasks

- Keep `tool_names`.
- Do not use `tool_names` for routing.
- Update comments in:
  - `shared/mcp_config.py`
  - `shared/route_resolver.py`
  - `shared/tool_registry.py`
  - `shared/tool_constants.py`
- Update docs:
  - `04_mcp_03_routing_lifecycle_and_execution.md`
  - `04_mcp_06_configuration_and_operations.md`
- Remove wording such as:
  - routing hint
  - routing fallback
  - config routing source
  - config-owned routing
- Ensure docs use the required wording:
  - `tool_names is validation metadata only.`
  - `tool_names is not used for routing.`
  - `Routing is resolved by live /v1/tools discovery and ToolRegistry.`

## Acceptance Criteria

- `tool_names` remains available for drift validation.
- No implementation path uses `tool_names` for routing.
- Docs consistently describe `tool_names` as validation-only.
- Tests do not imply that `tool_names` controls routing.

## Validation

- Review route resolver tests.
- Review tool registry tests.
- Grep docs for `tool_names` and confirm wording is consistent.
