# [Medium] Add routing, tool definition, and safety tier consistency tests for MDQ

## Priority
Medium

## Labels
`mcp`, `mdq`, `tests`, `routing`, `tool-definitions`

## Dependencies
- Promote mdq-mcp from experimental/stub to production status
- Treat index_paths and refresh_index as side-effect tools

## Background
MDQ tool names must remain consistent across tool constants, tool registry, live `/v1/tools`, tool definitions, and safety tiers.

## Task
Add tests that prevent MDQ routing and metadata drift.

## Implementation Requirements
Add tests for:
- `MDQ_TOOLS` contains exactly the 7 expected tools.
- Tool registry maps all MDQ tools to server key `mdq`.
- `/v1/tools` returns the same MDQ tool names.
- Tool definitions expose all MDQ tools required by the LLM.
- Safety tiers classify:
  - read-only tools as read-only
  - indexing tools as write/admin
- No MDQ tool is accidentally unmapped.

## Acceptance Criteria
- Route resolver tests pass.
- Tool metadata tests pass.
- Safety tier tests pass.
- Startup validation produces no MDQ drift warnings.
