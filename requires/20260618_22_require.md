# [agent][workflow] Make workflow/run state a first-class execution model

## Summary
The current agent is still centered on session and turn state.

## Problem
Workflow execution exists, but it is not the main execution model.
Important runtime concepts such as:
- workflow
- task
- retry
- approval
- artifact

are not first-class state yet.

## Scope
- workflow state model
- `workflow.sqlite`
- state ownership design

## Task
- define workflow/run as a first-class execution model
- define state for:
  - workflow
  - task
  - artifact
  - retry
  - approval
- separate conversation history from execution state

## Expected Result
Execution state is managed as workflow/run state, not only as turn history.

## Acceptance Criteria
- [ ] workflow state model is defined
- [ ] task state model is defined
- [ ] retry state is defined
- [ ] approval state is defined
- [ ] artifact references are part of workflow state
