# [RAG][Fusion] Make use_rrf work or remove it

## Summary
The config flag `use_rrf` exists, but the current implementation always uses RRF.

## Problem
This creates a mismatch between configuration and real behavior.

## Scope
- `rag/stages/fusion.py`
- RAG config and docs

## Task
Choose one option:

### Option A
Implement `use_rrf=False` properly.

### Option B
Remove `use_rrf` from config and docs if RRF should always be used.

## Expected Result
The config and the implementation match.

## Acceptance Criteria
- [ ] `use_rrf=False` changes behavior OR the flag is removed
- [ ] docs match the final behavior
- [ ] dead code related to non-RRF fallback is removed or activated
