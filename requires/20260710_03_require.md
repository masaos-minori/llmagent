# [High] Ensure init_db.sh initializes workflow.sqlite schema

## Background

The workflow engine depends on `workflow.sqlite` tables such as `tasks`, `attempts`, `processed_events`, `artifacts`, and `approvals`. The schema creation function exists, but the deployment flow must ensure it is executed.

## Goal

Make workflow DB schema initialization mandatory in the DB initialization step.

## Scope

- `deploy/init_db.sh`
- `db/create_schema.py`
- `db/schema_sql.py`

## Tasks

- [ ] Verify whether `deploy/init_db.sh` calls `create_workflow_schema()`.
- [ ] If missing, update `init_db.sh` to call `create_workflow_schema()`.
- [ ] After initialization, verify that `tasks` exists.
- [ ] After initialization, verify that `attempts` exists.
- [ ] After initialization, verify that `processed_events` exists.
- [ ] After initialization, verify that `artifacts` exists.
- [ ] After initialization, verify that `approvals` exists.
- [ ] Fail with non-zero exit code if any required table is missing.
- [ ] Print remediation instructions when schema initialization fails.

## Suggested Remediation Message

`Workflow database schema is missing or incomplete.`

`Run the workflow schema initialization step before starting the agent.`

## Acceptance Criteria

- [ ] `init_db.sh` creates or verifies `workflow.sqlite`.
- [ ] Missing workflow tables are detected.
- [ ] DB initialization fails fast on schema errors.
- [ ] Re-running DB initialization remains idempotent.

## Notes

Do not silently ignore workflow schema initialization failures.
