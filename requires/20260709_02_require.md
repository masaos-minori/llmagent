# [High] Remove MCP reload deferred behavior and classify auth/lifecycle changes as restart-required

## Labels
priority/high, area/config-reload, area/mcp, backward-compat-removal

## Summary
Remove `/reload` deferred behavior for MCP server auth and lifecycle settings.

`auth_token` and `startup_mode` changes must be classified as restart-required instead of deferred.

## Background
Current reload behavior stores some MCP config changes in the running config while they only take effect on the next connection or subprocess start. This creates an ambiguous intermediate state between `ctx.cfg` and the actual active transport/process state.

The affected fields are:

- `mcp_servers.<key>.auth_token`
- `mcp_servers.<key>.startup_mode`

## Required Changes

### Implementation
- Change `auth_token` reload classification from `deferred` to `needs_restart`.
- Change `startup_mode` reload classification from `deferred` to `needs_restart`.
- When a field is classified as restart-required, do not mutate the running server config object immediately.
- Remove or phase out `ConfigReloadOutcome.deferred`.
- Remove `/reload` output handling for `Deferred (next connection)` and `[DEFER]`.

### CLI Behavior
Replace deferred output with restart-required output.

Old behavior:

- `Deferred (next connection):`
- `[DEFER] - mcp/<key>.auth_token`
- `[DEFER] - mcp/<key>.startup_mode`

New behavior:

- `Restart required:`
- `[RESTART] - mcp/<key>.auth_token`
- `[RESTART] - mcp/<key>.startup_mode`

### Documentation
- Remove `Deferred` category from `05_agent_07_cli-and-commands.md`.
- Remove `Deferred` category from `05_agent_08_configuration.md`.
- Update hot-reload matrix:
  - MCP auth/lifecycle settings are restart-required.
  - New MCP servers remain restart-required.
  - Transport changes remain restart-required.

### Tests
Update existing tests that assert deferred behavior.

Required test updates:

- Rename or replace `test_auth_token_change_populates_deferred`.
- Rename or replace `test_startup_mode_change_populates_deferred`.
- Remove or rewrite `test_deferred_values_stored_in_cfg`.
- Add assertions that restart-required changes do not mutate active runtime config immediately.

## Acceptance Criteria
- `/reload` no longer displays `Deferred` or `[DEFER]`.
- `auth_token` changes are reported as restart-required.
- `startup_mode` changes are reported as restart-required.
- Running MCP server config is not partially mutated for restart-required fields.
- Documentation no longer describes deferred reload behavior.
- Tests reflect restart-required behavior.

## Target Files
- `agent/services/config_reload.py`
- `agent/commands/cmd_config.py`
- `tests/test_config_reload.py`
- `tests/test_config_reload_classification.py`
- `05_agent_07_cli-and-commands.md`
- `05_agent_08_configuration.md`

## Non-Goals
- Do not remove `/reload`.
- Do not change hot reload for LLM, RAG, or tool cache settings.
- Do not change startup-only behavior for memory/plugin initialization.
