# [High] Register all GitHub MCP tools in ToolRegistry and remove `github_` prefix routing

## Priority
High

## Background
GitHub tools are currently treated as prefix-routed tools outside `ToolRegistry`. GitHub MCP defines 21 tools across repository, file, issue, and pull request domains, and the server already returns `server_key="github"` from `/v1/tools`. 【3-cec503】【4-e91cdf】【5-29f6b7】【1-c49f0c】【6-31c99f】

## Goal
Remove `github_` prefix routing and register all GitHub tools explicitly in `ToolRegistry`.

## Implementation Tasks
- Remove `_GITHUB_PREFIX = "github_"` from `shared/route_resolver.py`.
- Remove GitHub prefix-match routing logic.
- Register all 21 GitHub MCP tools with `server_key="github"` in `ToolRegistry`.
- Replace GitHub prefix routing tests with registry-based routing tests.
- Update registry total count tests from `45` to `66`, unless other tool changes alter the expected count.
- Add a test that `registry.get_tool_names("github")` returns 21 tools.
- Add a discovery/registry consistency test for GitHub `/v1/tools`.

## Acceptance Criteria
- No GitHub tool routes by prefix matching.
- Every GitHub MCP tool is explicitly owned by `server_key="github"`.
- GitHub tools resolve through `ToolRegistry` or live discovery only.
- Registry count includes GitHub tools.
- Tests pass.
