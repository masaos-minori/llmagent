# [shared][plugins] Return structured plugin load results instead of only an integer

## Summary
`load_plugins()` returns only the number of loaded plugins.

## Problem
Failed plugins are only visible in logs.
There is no machine-readable result that shows:
- which plugin failed
- why it failed

## Scope
- `shared/plugin_registry.py`

## Task
- change `load_plugins()` to return structured results
- include:
  - loaded count
  - failed plugin list
  - error message for each failed plugin

## Expected Result
- plugin loading can be checked by code, not only by reading logs

## Acceptance Criteria
- [ ] return type includes success and failure details
- [ ] failed plugin path is included
- [ ] failure reason is included
