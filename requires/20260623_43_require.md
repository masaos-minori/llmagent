# Title
Validate row width explicitly in `output_port.py` table rendering

## Summary
Add explicit validation for row/header width mismatches in `CliOutputTable.write_table()` to prevent malformed table rendering behavior.

## Background
The plan notes that width calculation can still be fragile if row cell counts do not match header count.

## Problem
Without explicit validation, malformed rows may lead to broken rendering or hidden formatting bugs.

## Required Changes
- Validate that each row length matches the header count before rendering.
- Raise a clear `ValueError` on mismatch.
- Keep normal table rendering behavior unchanged when input is valid.

## Acceptance Criteria
- Mismatched row widths are rejected explicitly.
- Valid tables still render the same way.
- The error message is clear enough for debugging.

## Out of Scope
- Redesigning table rendering behavior
- Adding new formatting features

## Components
- `scripts/agent/commands/output_port.py`

## AI Implementation Instruction
Implement this as an input-shape validation task.

Strict rules:
- Do not silently truncate or pad malformed rows.
- Fail clearly on invalid table shape.
- Preserve existing valid rendering behavior.
- If header/row optionality is unclear, mark it as: `Needs confirmation`
