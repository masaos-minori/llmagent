# Preserve standalone RAG ingestion pipeline modules

## Priority
High

## Labels
rag, ingestion, architecture, safety

## Dependencies
- Remove `_cmd_ingest()` from the Agent command mixin
- Remove or isolate `IngestWorkflowService`

## Background
Removing `/ingest` should remove only the Agent REPL command path. The standalone RAG ingestion pipeline remains a separate operational capability.

## Task
Ensure standalone RAG ingestion modules are not deleted as part of `/ingest` removal.

## Implementation Requirements
Keep the following modules unless proven unused by the standalone pipeline:
- `crawler.py`
- `chunk_splitter.py`
- `ingester.py`
- `chunk_utils.py`
- `chunk_english.py`
- `chunk_japanese.py`
- `crawler_utils.py`
- `pipeline_utils.py`

## Acceptance Criteria
- Standalone RAG ingestion modules remain importable.
- `Crawler.py → ChunkSplitter.py → RagIngester.py` workflow remains available outside the Agent REPL.
- Only the Agent `/ingest` command path is removed.

## Source Context
The ingestion modules are documented as a standalone RAG pipeline: crawler output, chunk splitting, and DB ingestion.
