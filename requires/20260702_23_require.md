# [High] Remove stdio fixtures from tests/test_route_resolver.py

## Priority

High

## Goal

Make tests/test_route_resolver.py HTTP-only.

## Background

The MCP configuration model is HTTP-only. However, tests/test_route_resolver.py still defines _stdio() and uses McpServerConfig("stdio", ...). This conflicts with the current McpServerConfig design.

## Scope

Affected files:

- tests/test_route_resolver.py

## Tasks

- Remove the _stdio() helper.
- Remove all McpServerConfig("stdio", ...) usages.
- Replace stdio-based fixtures with HTTP-only fixtures.
- Keep tests proving that config.tool_names is not a routing input.
- Use HTTP configs for config-only routing tests.
- Remove legacy cmd arguments if the current McpServerConfig constructor does not accept them.

## Unknowns / Open Questions

- Confirm the current McpServerConfig constructor signature.
- Confirm whether any route resolver test still depends on legacy cmd arguments.

## Acceptance Criteria

- tests/test_route_resolver.py has no active stdio reference.
- Routing tests use HTTP-only McpServerConfig.
- Config-only tools still do not route.
- Known registry tools still resolve correctly.

## Validation

Run:
```bash
uv run pytest tests/test_route_resolver.py -v
```
Search:
```bash
rg -n "stdio|_stdio|cmd=|both transports" tests/test_route_resolver.py
```
