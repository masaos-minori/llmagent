## [Medium][eventbus][config] Reclassify poll_interval_ms as a fallback or resync control instead of a primary live-delivery knob

### Summary
`poll_interval_ms` currently controls the main live-delivery path.

### Problem
A polling interval should not define normal live-delivery latency once push notification exists.

### Scope
- config.py
- subscribe runtime behavior
- operational semantics of poll_interval_ms
- docs

### Task
- change the meaning of `poll_interval_ms` so it is no longer the main live-delivery control
- use it only for fallback polling, resync, or broker-recovery behavior if needed
- document the new role of the setting
- add tests for fallback/resync mode if retained

### Expected Result
Polling becomes a safety net, not the main live-delivery mechanism.

### Acceptance Criteria
- normal live delivery does not depend on `poll_interval_ms`
- the config field has a clearly documented fallback/resync meaning
- no hidden primary-latency dependency remains
