# [shared][json] Add a safer wrapper for orjson.dumps(...).decode()

## Summary
The project rule says `orjson.dumps()` returns bytes and must be decoded when a string is needed.

## Problem
This is easy to forget and can create type bugs.

## Scope
- shared utility layer
- common serialization call sites

## Task
- add one helper for string-producing JSON serialization
- use it where string output is needed
- document the helper as the preferred API

## Expected Result
- fewer bytes/string mistakes
- clearer JSON API usage across layers

## Acceptance Criteria
- [ ] a helper exists for string JSON serialization
- [ ] direct `.decode()` repetition is reduced
- [ ] docs recommend the helper clearly
