# Title
Apply strict type validation and typed document rows in `cmd_db.py`

## Summary
Refactor `cmd_db.py` to remove `.isdigit()` checks and unconditional `str()` coercion, and introduce typed row handling for document-list output.

## Background
The plan explicitly flags:
- unnecessary `str()` coercion,
- weak `.isdigit()` parsing,
- and raw row handling in `cmd_db.py`.

## Problem
Loose parsing and unconditional string conversion blur the distinction between invalid data and valid values, and they weaken the command contract.

## Required Changes
- Remove unnecessary `str()` wrappers around already-validated values.
- Introduce a strict positive-integer parser helper.
- Replace all `.isdigit()`-based parsing in this file.
- Introduce a typed `DocumentRow` for document iteration and rendering.
- Validate types before conversion.

## Acceptance Criteria
- `.isdigit()` is no longer used in `cmd_db.py`.
- Unconditional `str(...)` coercion is removed from the flagged paths.
- Document rows are processed through a typed structure.
- Invalid numeric input causes explicit validation failure.

## Out of Scope
- Redesigning `/db` command behavior
- Changing the data source for document listing

## Components
- `scripts/agent/commands/cmd_db.py`
- Shared validation helper if extracted
- `scripts/agent/commands/models.py` or command DTO module

## AI Implementation Instruction
Implement this as a strict-input-validation and typed-row task.

Strict rules:
- Do not use `.isdigit()` in new logic.
- Do not collapse `None`, empty string, and invalid string into the same behavior.
- Validate before converting.
- If row shape is not fully stable, mark it as: `Needs confirmation`
