# Update documentation to remove `/note` command and notes feature

## Priority
Medium

## Labels
docs, agent, notes, memory, breaking-change

## Dependencies
- Remove `/note` command registration from CommandRegistry
- Remove `auto_inject_notes` from configuration

## Background
Agent documentation currently describes `/note` commands and the `notes` table as part of the Agent feature set. After removing notes, documentation must point users to the memory layer instead.

## Task
Remove `/note` documentation and document `/memory` as the replacement.

## Implementation Requirements
Update relevant Agent docs:
- Remove the Note command category.
- Remove `/note add|list|delete` descriptions.
- Remove or mark the `notes` table as legacy/unused, depending on the final DB schema decision.
- Remove references to automatic note injection.
- Add replacement guidance using:
  - `/memory list`
  - `/memory search`
  - `/memory show`
  - `/memory pin`
  - `/memory unpin`
  - `/memory delete`
  - `/memory prune`
  - `/memory status`

## Acceptance Criteria
- Docs no longer present `/note` as an available command.
- Docs do not describe note injection as active behavior.
- Users are directed to `/memory` for long-term searchable context.

## Source Context
The current documentation presents `/note add|list|delete` as slash commands and describes notes as session-independent persistent data.
