# Add strict mode support to `ConfigLoader.load_all()`

## Priority
Medium

## Labels
- config
- operations
- production-readiness
- medium-priority

## Dependencies
None

## Background
`ConfigLoader.load_all()` currently skips missing config files with a debug log. This is convenient for local development, but production deployments should fail fast when required config files are missing.

## Scope
Modify:
- `shared/config_loader.py`

Add tests:
- `tests/test_config_loader.py`

## Required Changes
- Change the method signature to:
  - `load_all(strict: bool = False) -> dict[str, Any]`
- Preserve current behavior when `strict=False`.
- Define a required config file set.
- When `strict=True`, raise `ConfigMissingError` if any required file is missing.
- Decide whether `mdq_mcp_server.toml` is required or optional and encode that decision explicitly.
- Keep optional config files skippable.

## Tests
Add `tests/test_config_loader.py`.

Test:
- `strict=False` skips missing files.
- `strict=True` raises on missing required files.
- `strict=True` allows missing optional files.
- Existing `load()` behavior is unchanged.
- Meta keys starting with `_` are still filtered.

## Acceptance Criteria
- Local compatibility is preserved.
- Production-style strict loading can detect missing required config.
- Error messages identify the missing config file clearly.
