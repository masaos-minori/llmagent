# [rag][ingestion][local-files] Define a better incremental update strategy for file:// ingestion

## Summary
`file://` ingestion works, but its update strategy is weaker than web ingestion.

## Problem
Local file ingestion does not use Conditional GET or ETag-based refresh.
This makes repeated ingestion less efficient and less predictable.

## Scope
- local file ingestion
- `file://` URL handling
- incremental update strategy
- ingestion documentation

## Task
- define how local files should be checked for updates
- consider a file mtime or hash-based no-op strategy
- document how local-file ingestion differs from web ingestion

## Expected Result
Local-file ingestion becomes easier to reason about and cheaper to repeat.

## Acceptance Criteria
- [ ] local-file update behavior is documented clearly
- [ ] a no-op or incremental strategy is defined
- [ ] differences from web ingestion are documented
