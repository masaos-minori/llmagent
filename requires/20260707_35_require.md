# M-6: Add lifecycle snapshot API for MCP subprocess status

## Priority

Medium

## Background

`/mcp status` should show whether a server is agent-managed, its PID/PGID, running state, lifecycle state, and stderr log path. The lifecycle manager currently does not expose a snapshot API.

## Target files

- `agent/http_lifecycle.py`
- `agent/factory.py`
- optionally `agent/services/models.py`

## Implementation tasks

- Add a process info DTO or dictionary structure.
- Implement `get_process_info(server_key)`.
- Implement `list_processes()`.
- Include fields:
  - `server_key`
  - `managed`
  - `pid`
  - `pgid`
  - `running`
  - `last_exit_code`
  - `stderr_log`
- Expose snapshot through `_ServerLifecycleRouter`.
- Use lifecycle state from `get_transport_state()`.

## Acceptance criteria

- Status code can query lifecycle process details without touching private maps.
- Persistent externally managed servers are marked `managed=false`.
- Subprocess servers started by the agent are marked `managed=true`.
- Snapshot remains safe after shutdown.

## Test plan

- Test snapshot for running subprocess.
- Test snapshot for stopped subprocess.
- Test snapshot after shutdown cleanup.
- Test persistent server snapshot behavior.
