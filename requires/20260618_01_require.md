# [mcp][mdq] Mark mdq-mcp as stub or remove it from production use

## Summary
`mdq-mcp` looks like a complete MCP server, but most tools are still stub implementations.

## Problem
The server exposes 7 tools, but most of them return placeholder strings instead of real search or indexing results.
This can mislead users and AI systems.

## Scope
- `mcp/mdq/service.py`
- `mcp/mdq/indexer.py`
- `mcp/mdq/search.py`
- mdq server exposure in `/v1/tools`
- mdq health/status reporting

## Task
Choose one of the following:
- clearly mark `mdq-mcp` as a stub / non-production server
- remove mdq tools from production routing until they are real
- expose stub status in `/health`

## Expected Result
Users and AI systems do not mistake `mdq-mcp` for a real search/indexing server.

## Acceptance Criteria
- [ ] mdq server status clearly shows stub or non-production state
- [ ] mdq tools are not treated as production-ready by mistake
- [ ] documentation and runtime behavior are consistent
