# [shared][git] Return failure reasons from git_helper.get_repo_info()

## Summary
`get_repo_info()` returns `None` on any error.

## Problem
The caller cannot tell why it failed.
Examples:
- not a git repo
- GitPython not installed
- permission problem
- other runtime error

## Scope
- `shared/git_helper.py`

## Task
- change the API to return a structured result with:
  - success flag
  - data
  - failure reason
- or raise more specific exceptions

## Expected Result
- callers can handle failure correctly
- debugging is easier

## Acceptance Criteria
- [ ] failure reason is available to the caller
- [ ] broad `None`-only failure is removed
- [ ] docs explain the new behavior
