# [MCP][Security] Normalize fail-open / fail-closed defaults and warn on risky settings

## Summary
Security defaults are inconsistent across servers, which makes behavior harder to reason about.

## Problem
Some controls are fail-closed, some are fail-open, and some are disabled when empty.
This increases the chance of unsafe configuration assumptions.

## Scope
- server config loading
- startup logging
- security docs and defaults

## Task
- review current defaults and align dangerous operations toward fail-closed where possible
- emit warnings for risky startup conditions such as:
  - auth disabled
  - workflow allowlist fail-open behavior
  - shell sandbox disabled
- document the final policy clearly

## Acceptance Criteria
- [ ] risky defaults are visible at startup
- [ ] dangerous operations use safer defaults where practical
- [ ] security behavior is more predictable across servers
