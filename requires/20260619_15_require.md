# Title
Make GitHub write-operation defaults safer for production

## Summary
Change GitHub MCP defaults so that force-push is not allowed by default and PR review requirements are safer by default.

## Background
The current GitHub MCP server documents `allow_force_push = true` and `require_pr_review = false` as defaults.
These defaults are too permissive for production-oriented automation.

## Problem
Repository write actions are high-impact operations.
Safe default policy should prioritize review and prevent forceful mutation unless explicitly enabled.

## Required Changes
- Change the default for `allow_force_push` to `false`.
- Change the default for `require_pr_review` to `true`, or document a clearly safer equivalent policy.
- Update any corresponding startup audit or configuration documentation.
- Preserve explicit opt-in for teams that need more permissive behavior.

## Acceptance Criteria
- Default GitHub write behavior is safer than the current baseline.
- Production docs reflect the new safer defaults.
- Existing override mechanisms remain available but explicit.

## Out of Scope
- Redesigning all GitHub write workflows.
- Extending review policy to external systems.

## Notes
This is a defaults-hardening task, not a full policy engine redesign.
