# Title
Standardize shell MCP `/health` response and documentation

## Summary
Define one authoritative `/health` response shape for `shell-mcp` and make all documentation match it.

## Background
Current documentation is inconsistent about whether shell `/health` exposes sandbox backend details or only returns the base health structure.

## Problem
Operators need a reliable way to verify which sandbox backend is active.
Documentation ambiguity makes operational verification harder.

## Required Changes
- Decide whether shell `/health` will expose sandbox backend details.
- If yes, define a stable field such as `details.sandbox_backend`.
- If no, remove all conflicting references that imply otherwise.
- Update health-check documentation and operational probes.

## Acceptance Criteria
- One canonical shell `/health` response model is documented.
- Runtime behavior matches the documented model.
- Operators can reliably verify sandbox state if that is part of the design.

## Out of Scope
- Redesigning all health endpoints.
- Adding unrelated shell diagnostics.

## Notes
Operational verification should not depend on ambiguous doc interpretation.
