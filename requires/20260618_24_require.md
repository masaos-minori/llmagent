# [agent][session][rag] Remove RAG-specific DB operations from AgentSession

## Summary
`AgentSession` still owns some RAG-layer database operations.

## Problem
The session layer should focus on:
- sessions
- messages
- notes

But some RAG-related document operations still depend on the agent session layer.

## Scope
- `agent/session.py`
- `/db` command integration
- RAG maintenance path

## Task
- move RAG-specific document operations out of `AgentSession`
- use a RAG-facing service or MCP path for RAG document maintenance
- keep `AgentSession` focused on session-layer persistence

## Expected Result
The session layer no longer depends directly on RAG schema details.

## Acceptance Criteria
- [ ] `AgentSession` no longer owns RAG-specific delete/list logic
- [ ] `/db` RAG actions use a clearer RAG-facing path
- [ ] responsibility boundaries are documented clearly
