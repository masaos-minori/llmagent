# MCP Server Shutdown: SIGINT Signal Handler Lost on ValueError

## Priority

Medium

## Target Files

- `scripts/agent/http_lifecycle.py` (lines 342-347)

## Background

When `McpServerLifecycle.shutdown_all()` runs, it installs a temporary SIGINT handler (`_absorb_sigint_during_shutdown`) to absorb a second SIGINT that may arrive during cleanup. If `signal.signal()` raises `ValueError` (not on main thread), the original handler reference is set to `None`.

## Problem

In `shutdown_all()` at lines 342-347:

```python
if old_sigint is not None:
    try:
        signal.signal(signal.SIGINT, self._absorb_sigint_during_shutdown)
    except ValueError:
        # signal.signal() failed — do not install the guard handler.
        old_sigint = None
```

The sequence is:
1. `getsignal(SIGINT)` returns the current handler (e.g., default handler or custom handler)
2. `signal.signal(SIGINT, new_handler)` raises `ValueError` because we are not on the main thread
3. `old_sigint = None` is executed — the original handler reference is lost
4. Later in the `finally` block, `signal.signal(signal.SIGINT, old_sigint)` is called with `None`, which means "reset to default handler" instead of "restore previous handler"

This causes two issues:
- The original SIGINT handler is lost — any custom handler installed by other code is replaced with the default
- A SIGINT received after this point will terminate the process immediately instead of being absorbed by `_absorb_sigint_during_shutdown`

## Reason for Change

- Custom SIGINT handlers installed elsewhere in the application are silently overwritten
- Graceful shutdown behavior is broken when shutdown_all() is called from a non-main thread
- The intent was clearly to preserve the original handler, but the implementation loses it

## Implementation Intent

When `signal.signal()` fails due to threading constraints, keep the original handler intact instead of setting it to `None`. Do not attempt to restore it later since we cannot modify signal handlers from a non-main thread anyway.

## Implementation Instructions

Refactor lines 342-347 to handle the `ValueError` case correctly:

```python
if old_sigint is not None:
    try:
        signal.signal(signal.SIGINT, self._absorb_sigint_during_shutdown)
    except ValueError:
        # Not on main thread — cannot install the guard handler.
        # Keep original handler intact; do not reset it.
        pass
```

Key points:
- Remove the `old_sigint = None` assignment
- Add logging to surface the issue during runtime (optional but recommended):
  ```python
  except ValueError:
      logger.debug("Cannot install SIGINT guard handler (not on main thread)")
      pass
  ```
- The `finally` block's `signal.signal(signal.SIGINT, old_sigint)` call will now work correctly because `old_sigint` still holds the original handler reference

## Acceptance Criteria

1. When `signal.signal()` raises `ValueError`, the original SIGINT handler is preserved
2. No silent overwriting of custom SIGINT handlers occurs
3. The `finally` block's handler restoration works correctly in both success and failure paths
4. A debug-level log message is emitted when the guard handler cannot be installed

## Tests

- Unit test: mock `signal.signal()` to raise `ValueError` during shutdown, verify original handler is preserved via `getsignal()`
- Integration test: trigger shutdown from a non-main thread, verify SIGINT is handled gracefully
