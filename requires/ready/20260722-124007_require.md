# MCP web-search: Verify and Document browser_fetch Runtime Availability Behavior

## Priority

High

## Target Files

- `docs/04_mcp_04_01_web-search-file-read-github.md` — update web-search section
- `docs/04_mcp_03_06_tool-runtime-availability-metadata.md` — clarify config_dependent mapping

## Background

browser_fetch is marked as config_dependent=true, but it is not confirmed from the reviewed files whether /v1/tools returns runtime availability fields for it. The web-search server delegates /v1/tools response construction to build_tools_response().

## Problem

browser_fetch is controlled by fail-closed domain allowlisting. If availability metadata is inaccurate or undocumented, the LLM and operator may not understand why browser_fetch is unavailable.

## Reason for Change

browser_fetch is controlled by fail-closed domain allowlisting. If availability metadata is inaccurate or undocumented, the LLM and operator may not understand why browser_fetch is unavailable.

## Implementation Intent

Investigate web-search server's build_tools_response() to determine whether runtime availability metadata is implemented, then document accordingly.

## Implementation Instructions

### Step 1: Investigate build_tools_response()

Inspect `mcp_servers/server.py::build_tools_response()` and verify:
- Does it add `server_key`?
- Does it add `schema_version`?
- Does it add `enabled`?
- Does it add `disabled_reason`?
- Does it use server-specific config state?
- Does `browser_allowed_domains=[]` disable `browser_fetch`?
- Is `search_web` independent of browser-specific allowlist configuration?

### Step 2: Update documentation based on findings

#### If availability metadata IS implemented

In the web-search section, add:

```markdown
### Availability metadata

- `browser_fetch` is `config_dependent=true`
- `browser_fetch` is disabled when the browser domain allowlist denies all access
- `/v1/tools` returns the correct `enabled` and `disabled_reason`
- `search_web` availability is separate from `browser_fetch`
```

#### If availability metadata is NOT implemented

In the web-search section, add:

```markdown
### Availability metadata (not yet implemented)

- `browser_fetch` currently has `config_dependent=true`
- Runtime `enabled/disabled_reason` support is not yet implemented for web-search
- Follow-up implementation task required
```

### Step 3: Create follow-up implementation task if needed

If availability metadata is not implemented, create a new requirement for implementing it in the web-search server.

## Acceptance Criteria

1. web-search catalog accurately describes current browser_fetch availability behavior
2. Runtime availability metadata docs do not overstate web-search support
3. Required follow-up implementation work is explicitly documented if missing
