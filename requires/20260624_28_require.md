# Add update-maintenance guidance to keep MCP overview and server catalog in sync

## Summary
Document how to maintain consistency between the high-level server overview and the detailed server catalog.

## Background
The MCP system overview and server catalog both describe server inventory, tool counts, and roles. This is useful for readers but creates drift risk when server definitions change.

## Objective
Reduce documentation drift between overview-level and detail-level server descriptions.

## In Scope
- `04_mcp_01_system_overview.md`
- `04_mcp_04_server_catalog.md`

## Required Changes
- Add a short maintenance note that defines:
  - what belongs in the overview
  - what belongs only in the detailed catalog
  - what must be updated whenever a server changes
- Clarify whether tool counts in the overview are normative or summary-only
- Add a simple synchronization checklist for future updates

## Acceptance Criteria
- Maintainers can tell which changes require updates to both files
- Overview and catalog responsibilities are clearly separated
- Future drift risk is reduced through explicit maintenance guidance

## Out of Scope
- Reorganizing all MCP docs
- Changing server inventory

## Notes
Keep the overview lightweight, but make explicit what must stay synchronized.
