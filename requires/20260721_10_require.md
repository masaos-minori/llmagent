## Title

Make memory retrieval degrade to FTS-only when vector KNN fails

## Priority

High

## Target files

- `agent/memory/retriever.py`
- `agent/commands/memory_status.py`
- `tests/test_memory_retriever.py`
- `tests/test_memory_status.py`

## Background

The memory retriever supports hybrid retrieval using FTS5 and optional vector KNN search.

`HybridRetriever.search()` already falls back to FTS-only when no embedding is provided or when vector search returns an empty result. However, it does not catch exceptions raised by `VectorRetriever.knn_search()`.

Direct KNN search is expected to raise when the vector table is unavailable. This is acceptable for direct KNN calls, but hybrid user-facing retrieval should degrade gracefully.

## Problem

When `memories_vec` is missing, sqlite-vec is unavailable, or the vector schema is inconsistent, hybrid memory search can fail entirely even though FTS results are available.

This can cause memory injection to fail because retriever errors propagate through the memory injection path.

## Reason for change

Memory retrieval is an auxiliary feature. Vector search failure should not prevent the agent from continuing when FTS results are available.

The existing fields `last_retrieval_mode` and `fts_fallback_count` indicate that degraded operation is already part of the design, but exception-based fallback is missing.

## Implementation intent

Implement a safe degraded mode for hybrid retrieval:

- keep FTS search fail-fast
- treat vector KNN failure as non-fatal
- return FTS results when vector search fails
- make the fallback visible through status and logs

## Implementation instructions

- In `HybridRetriever.search()`, wrap the vector KNN call with exception handling.
- Catch at least `sqlite3.OperationalError`.
- Consider catching only narrow sqlite/vector-related exceptions.
- On vector failure:
  - log a warning with exception class and message
  - set `self.last_retrieval_mode = "fts_only"`
  - increment `self.fts_fallback_count`
  - return `fts_hits`
- Do not change direct `HybridRetriever.knn_search()` behavior unless separately required.
- Ensure `/memory status` can expose the latest retrieval mode and fallback count if it does not already.

## Acceptance criteria

- Hybrid search with an embedding returns FTS hits when `memories_vec` is missing.
- Vector KNN exceptions do not cause hybrid search to fail.
- Direct `knn_search()` can still raise when called explicitly.
- `last_retrieval_mode` is set to `"fts_only"` after fallback.
- `fts_fallback_count` increments after fallback.
- A warning log is emitted for vector fallback.
- Existing FTS-only behavior remains unchanged.

## Tests

Add or update tests for:

- Hybrid search with `embedding` provided and no `memories_vec` table returns FTS results.
- `last_retrieval_mode` becomes `"fts_only"` after vector failure.
- `fts_fallback_count` increments after vector failure.
- Direct `knn_search()` missing-table behavior remains unchanged if direct KNN is intentionally fail-fast.
