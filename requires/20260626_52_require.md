# Title
Standardize RAG ingestion terminology for scripts and processing phases

## Priority
Low

## Labels

## Dependencies

## Summary
Standardize terminology across RAG docs so “3 scripts” and “4 processing phases” are consistently distinguished.

## Background
The RAG overview explains that both “3 scripts” and “4 processing phases” are correct from different perspectives:
- scripts: crawler, chunk_splitter, ingester
- phases: crawl, chunk, embed, store

## Problem
If the terminology is not consistent, readers may confuse script boundaries with processing boundaries or query pipeline stages.

## Required Changes
- Use “script” for executable files:
  - `crawler.py`
  - `chunk_splitter.py`
  - `ingester.py`
- Use “processing phase” for:
  - crawl
  - chunk
  - embed
  - store
- Avoid using “stage” for ingestion unless explicitly contrasted with query pipeline stages.
- Update ingestion, overview, and operations docs.

## Acceptance Criteria
- RAG ingestion terminology is consistent across docs.
- Query pipeline stages are not confused with ingestion phases.
- Human and AI readers can distinguish executable scripts from logical phases.

## Out of Scope
- Runtime behavior changes.
- Query pipeline stage redesign.
