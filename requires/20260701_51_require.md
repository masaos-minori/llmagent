# [High] Add eventbus_db_path to DbConfig, common.toml, and SQLiteHelper

## Goal

Add `eventbus_db_path` as a first-class DB path managed by `DbConfig` and support `SQLiteHelper("eventbus")`.

## Background

Shared/DB schema integration now includes `eventbus.sqlite` alongside `rag.sqlite`, `session.sqlite`, and `workflow.sqlite`. Current docs and implementation describe only `rag_db_path`, `session_db_path`, and `workflow_db_path`, and `SQLiteHelper` currently supports only `rag`, `session`, and `workflow` targets. 【1-329c46】【2-bcb9b9】

## Scope

- DB config
- SQLite helper target support
- Default config
- Tests
- Documentation

Do not implement Event Bus runtime integration in this issue.

## Implementation Tasks

- Add `eventbus_db_path` to `DbConfig`.
- Add `eventbus_db_path` to `common.toml`.
- Set the default path to `/opt/llm/db/eventbus.sqlite`.
- Add `DbTarget.EVENTBUS = "eventbus"`.
- Update `SQLiteHelper` to accept:
  - `DbTarget.EVENTBUS`
  - `"eventbus"`
- Ensure `SQLiteHelper("eventbus")` resolves `DbConfig.eventbus_db_path`.
- Ensure sqlite-vec is not loaded for the `eventbus` target.
- Update DB config tests.
- Update SQLiteHelper tests.
- Update Shared/DB docs to include `eventbus.sqlite`.

## Acceptance Criteria

- `DbConfig` exposes `eventbus_db_path`.
- `common.toml` defines `eventbus_db_path`.
- `SQLiteHelper("eventbus")` opens `eventbus.sqlite`.
- Invalid DB targets still raise `ValueError`.
- `eventbus` does not load sqlite-vec.
- Docs list four DB files:
  - `rag.sqlite`
  - `session.sqlite`
  - `workflow.sqlite`
  - `eventbus.sqlite`

## Validation

- Run DB config tests.
- Run SQLiteHelper tests.
- Manually verify:

  python -c "from db.helper import SQLiteHelper; print(SQLiteHelper('eventbus').DB_PATH)"

- Confirm the output points to `/opt/llm/db/eventbus.sqlite` or the configured `common.toml` value.
