# Add migration note for `/note` removal

## Priority
Low

## Labels
docs, migration, notes, breaking-change

## Dependencies
- Update documentation to remove `/note` command and notes feature

## Background
The `/note` command group is removed without backward-compatible aliases. Users need a concise migration note explaining the replacement.

## Task
Add a migration note for the removal of `/note`.

## Implementation Requirements
Add a short migration note containing:

Removed commands:
- `/note add`
- `/note list`
- `/note delete`
- `/note pin`
- `/note unpin`
- `/note search`

Reason:
- Persistent notes are removed from the Agent command layer.
- Long-term searchable context should use the memory layer.

Replacement:
- `/memory list`
- `/memory search`
- `/memory show`
- `/memory pin`
- `/memory unpin`
- `/memory delete`
- `/memory prune`
- `/memory status`

## Acceptance Criteria
- Migration note clearly states that `/note` is removed.
- No backward-compatible alias is documented.
- `/memory` is documented as the replacement path.

## Source Context
Although the requested removal mentions `/note add|list|delete`, the implementation also includes `/note pin|unpin|search`, so the entire `/note` command group should be documented as removed.
