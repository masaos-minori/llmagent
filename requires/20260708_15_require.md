# M-5: Remove tool_result_summary from session-end diagnostics

## Priority

Medium

## Background

Session-end diagnostics currently include `tool_result_summary`, derived from the `tool_results` table.

Since `tool_results` is being removed, session summaries should no longer query or include this field.

## Target files

- `agent/repl.py`
- documentation files for operations and diagnostics

## Implementation tasks

- Remove `tool_results` table queries from `_persist_session_diagnostics()`.
- Remove the `tool_result_summary` field from the session summary JSON.
- Continue reporting:
  - total turns
  - tool call count
  - tool error count
  - partial completion count
  - latency summary
  - workflow counts
  - diagnostic counts
- Use `ctx.stats.stat_tool_calls` and `ctx.stats.stat_tool_errors` as the source for tool execution counts.

## Acceptance criteria

- Session-end summary no longer queries `tool_results`.
- Session-end summary no longer contains `tool_result_summary`.
- Shutdown diagnostics still persist successfully.
- Tool call and tool error counts remain available.

## Test plan

- End a session with tool calls.
- Verify `session_summary` is saved.
- Verify JSON has no `tool_result_summary`.
- Verify `tool_calls` and `tool_errors` remain correct.
- Verify no SQL query references `tool_results`.
