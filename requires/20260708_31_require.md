# H-9: Remove MCP deferred output from /reload

## Priority

High

## Background

`/reload` currently displays deferred changes as “next connection” changes. This is invalid for MCP server definitions after removing deferred auth token and startup mode behavior.

MCP server definition changes require full agent restart.

## Target files

- `agent/services/config_reload.py`
- `agent/commands/cmd_config.py`
- reload command tests

## Implementation tasks

- Ensure MCP server definition changes never populate `ConfigReloadOutcome.deferred`.
- Remove MCP-specific deferred entries:
  - `mcp/<server_key>.auth_token`
  - `mcp/<server_key>.startup_mode`
- Keep generic deferred output support only if other subsystems still need it.
- If `deferred` has no remaining uses, consider marking it deprecated in a separate cleanup.
- Ensure `/reload` prints MCP changes only under `Restart required`.

## Acceptance criteria

- `/reload` never prints `[DEFER]` for MCP server config changes.
- MCP auth token and startup mode changes print as `[RESTART]`.
- If there are no non-MCP deferred items, no deferred section is printed.
- Reload command summary does not say MCP changes are deferred to the next connection.

## Test plan

- Reload with auth token changed.
- Reload with startup mode changed.
- Verify `/reload` output contains `[RESTART]`.
- Verify `/reload` output does not contain `[DEFER]`.
