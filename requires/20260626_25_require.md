# [Medium] Harden SQLite connection and transaction management

## Priority
Medium

## Labels
`mcp`, `mdq`, `sqlite`, `reliability`, `concurrency`

## Dependencies
- Redesign MDQ SQLite schema for production indexing

## Background
Current MDQ DB access uses direct `sqlite3.connect()` without explicit WAL mode, busy timeout, transaction strategy, or consistency checks.

## Task
Improve MDQ SQLite reliability for production use.

## Implementation Requirements
- Enable WAL mode.
- Set `busy_timeout`.
- Use explicit write transactions.
- Prefer atomic replace per document during indexing.
- Add FTS consistency check.
- Add FTS rebuild operation if needed.
- Ensure connections are closed reliably.

## Acceptance Criteria
- Concurrent reads during indexing are safe.
- DB locked errors are reduced or handled cleanly.
- FTS row count can be checked against chunk row count.
- A rebuild path exists for corrupted or stale FTS index.
