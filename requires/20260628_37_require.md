# Keep `/audit` as the only audit log browsing command

## Priority
High

## Labels
agent, commands, audit, observability

## Dependencies
- Remove `/debug audit` branch from debug command handler

## Background
Audit log browsing should be handled by the dedicated `/audit` command. `cmd_audit.py` already implements default tail, `turn`, and `tool` subcommands.

## Task
Ensure audit log browsing remains available only through `/audit`.

## Implementation Requirements
- Do not remove `_AuditMixin`.
- Keep the following methods in `cmd_audit.py`:
  - `_audit_tail`
  - `_audit_turn`
  - `_audit_tool`
  - `_cmd_audit`
- Do not move `/debug audit` logic into `/audit`; equivalent functionality already exists.
- Confirm the following commands remain supported:
  - `/audit`
  - `/audit tail N`
  - `/audit turn <task_id>`
  - `/audit tool <name>`

## Acceptance Criteria
- `/audit` still supports default tail behavior.
- `/audit tail N` still works.
- `/audit turn <task_id>` still works.
- `/audit tool <name>` still works.
- `cmd_debug.py` no longer browses audit logs.

## Source Context
`cmd_audit.py` already implements audit log tailing, turn filtering, and tool filtering.
