# Add tests to lock down ToolExecutor execution order

## Priority

Medium

## Type

Implementation / Test / Runtime Safety

## Background

`ToolExecutor` is responsible for executing tool calls through plugin tools, cache, routing, health checks, lifecycle readiness, concurrency limits, and HTTP transport.

The intended execution order is critical for correctness.

## Problem

If execution order changes unintentionally, it can affect safety and behavior.

Examples:

- Plugin tools might be incorrectly cached as MCP results.
- Cache hits might update health state even though no live request occurred.
- Unknown tools might fail too late.
- Transport errors and tool-level errors might be mixed.
- Lifecycle readiness might be skipped.

## Goal

Codify ToolExecutor execution order in tests.

## Scope

Affected areas:

- `shared/tool_executor.py`
- `shared/plugin_tool_invoker.py`
- `shared/route_resolver.py`
- `shared/tool_cache.py`
- `shared/mcp_health.py`
- ToolExecutor tests

## Implementation Tasks

- Add tests verifying the expected execution order:
  - plugin tool check
  - cache check
  - route resolve
  - health pre-flight
  - lifecycle ensure-ready
  - concurrency semaphore
  - transport call
  - health registry update
- Verify plugin tools bypass MCP routing.
- Verify plugin tools do not use the MCP TTL cache.
- Verify cache hit does not update HealthRegistry.
- Verify unknown tools fail with clear `ValueError`.
- Verify transport failures produce `error_type="transport"`.
- Verify tool-level errors produce `error_type="tool"` or equivalent tool-error classification.

## Acceptance Criteria

- Execution order is covered by tests.
- Plugin tool execution bypasses routing and MCP cache.
- Cache hit returns without live server call.
- Unknown tool fails before transport.
- HealthRegistry updates only when appropriate.
- Transport and tool-level errors remain distinguishable.

## Validation Plan

Add tests using mocked components for:

- Plugin tool hit.
- Cache hit.
- Unknown tool.
- Unavailable server.
- Lifecycle ensure-ready failure.
- Transport success.
- Transport failure.
- Tool-level error.
- Concurrency limit acquisition.

## Risks

Tests may expose existing implicit behavior. Prefer making behavior explicit and stable.

## Out of Scope

- Rewriting ToolExecutor.
- Changing plugin public API.
- Documentation-only updates.
