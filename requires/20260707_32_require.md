# M-3: Make /mcp status avoid false HEALTHY fallback when health registry is missing

## Priority

Medium

## Background

`McpStatusService` currently falls back to `HEALTHY` when `ctx.services_required.health_registry` is missing. This can hide tracking failures and make an untracked server appear healthy.

## Target files

- `agent/services/mcp_status.py`
- `agent/services/models.py`
- status renderer command file

## Implementation tasks

- Prefer ensuring health registry is always injected during normal startup.
- Add defensive fallback when registry is still missing:
  - use `health="not_tracked"`, or
  - add `UNKNOWN` to `McpServerHealthState`, or
  - display a clear untracked state outside the enum
- Avoid returning `HEALTHY` when health is not actually tracked.
- Update status renderer if needed.

## Acceptance criteria

- Missing health registry no longer appears as healthy status.
- Normal startup path uses a non-null health registry.
- `/mcp status` clearly distinguishes live availability from health tracking.

## Test plan

- Test status output with registry present and healthy.
- Test status output with registry present and degraded.
- Test status output with registry missing.
