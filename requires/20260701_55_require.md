# [High] Fix deploy/init_db.sh to use the unified schema initialization entrypoint

## Goal

Fix `deploy/init_db.sh` and make it call the unified schema initialization entrypoint.

## Background

The current `init_db.sh` intends to initialize `rag.sqlite`, `session.sqlite`, `workflow.sqlite`, and Event Bus DBs, but the attached script contains broken variable references such as `$/...` and does not correctly execute `db.create_schema`. 【4-f3703c】

## Scope

- Deployment initialization script only
- No Event Bus runtime integration

## Implementation Tasks

- Fix broken variable references.
- Use `DEPLOY_SCRIPTS="/opt/llm/scripts"`.
- Use `DEPLOY_DB="/opt/llm/db"`.
- Ensure `mkdir -p "$DEPLOY_DB"` is called.
- Replace direct SQL file loading for Event Bus with:

  (cd /opt/llm && PYTHONPATH="$DEPLOY_SCRIPTS" python -m db.create_schema)

- Remove `.read eventbus/schema.sql`.
- Do not call `python -m db.workflow_schema`.
- Verify tables for:
  - `rag.sqlite`
  - `session.sqlite`
  - `workflow.sqlite`
  - `eventbus.sqlite`
- Update expected table comments.

## Acceptance Criteria

- `init_db.sh` initializes all four DBs through `python -m db.create_schema`.
- `init_db.sh` no longer reads `eventbus/schema.sql` directly.
- `init_db.sh` no longer references `workflow_schema.py`.
- The script uses valid shell variable expansion.
- The script exits non-zero on schema initialization failure.

## Validation

- Run shellcheck if available.
- Run:

  bash deploy/init_db.sh

- Verify:
  - `/opt/llm/db/rag.sqlite`
  - `/opt/llm/db/session.sqlite`
  - `/opt/llm/db/workflow.sqlite`
  - `/opt/llm/db/eventbus.sqlite`
- Verify expected tables with `sqlite3`.
