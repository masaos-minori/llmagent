# Title
Add deterministic fallback and visible failure handling for session title generation

## Summary
Improve session title generation so that title creation failure does not leave the session with an empty or default title.

## Background
Session title generation currently runs asynchronously in a fire-and-forget task.
When it fails, the error is logged, but no fallback title is created and no user-visible signal is produced.

## Problem
Session titles are useful for session management, browsing, and later recall.
Empty or missing titles reduce usability and make failures hard to detect.

## Required Changes
- Add a deterministic fallback title when LLM-based title generation fails.
- Use a simple fallback such as the first user input truncated to a safe length.
- Make title-generation failure observable through logs, audit events, or debug output.
- Ensure session list views never show an empty title.

## Acceptance Criteria
- A session always ends up with a non-empty title.
- LLM title generation failure does not silently leave the title unset.
- Failure is observable in at least one operator-facing channel.
- Existing successful title generation behavior remains intact.

## Out of Scope
- Rewriting the full session management system.
- Adding advanced title ranking or title revision workflows.

## Notes
Use a deterministic fallback, not another best-effort async call.
