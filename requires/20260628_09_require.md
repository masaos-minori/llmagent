# Add migration note for `/mcp install` removal

## Priority
Low

## Labels
docs, migration, mcp, breaking-change

## Dependencies
- Update MCP documentation to remove `/mcp install` wizard

## Background
Because backward compatibility is intentionally ignored, users and developers need a concise migration note explaining why `/mcp install` was removed and what to use instead.

## Task
Add a migration note documenting the removal of `/mcp install`.

## Implementation Requirements
Add a short migration note containing:

Removed command:
- `/mcp install <server-name>`

Reason:
- MCP server scaffold generation is a developer workflow.
- Runtime Agent REPL should only inspect MCP server status and health.

Replacement:
- Add MCP servers manually using the new-server checklist.
- Verify the added server using `/mcp`.

## Acceptance Criteria
- Migration note clearly states that `/mcp install` is removed.
- No backward-compatible alias is documented.
- The replacement workflow is clear.

## Source Context
The previous behavior allowed scaffold generation through the Agent REPL. The new behavior keeps `/mcp` as a status/health inspection command only.
