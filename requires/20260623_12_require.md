# Task: Expand Scope of /reload Command and Fix Configuration Shadowing

## Overview and Context
The `/reload` command currently only reloads `common.toml` and `agent.toml` into the active runtime. However, configuration settings are distributed across roughly 10 granular files, such as `tools.toml`, `security.toml`, and `rag.toml`. Changes to those sub-configurations are ignored during `/reload`. Furthermore, field properties like `http` timeouts and `llm` definitions are duplicated across `common.toml`, `llm.toml`, and `http.toml`, creating an ambiguous shadowing behavior.

## Acceptance Criteria (AC)
- [ ] Modify the `/reload` handler to invoke a unified `ConfigLoader.load_all()` function that comprehensively parses all valid `.toml` configuration structures in the target directory.
- [ ] Enforce a strict configuration overriding priority order within the consolidation logic: specific files shadow generic ones (`http.toml` > `llm.toml` > `common.toml`).
- [ ] Validate configurations immediately after reloading and throw an explicit configuration error if schemas are violated.

## Technical Specifications and Implementation Steps
1. Target Files: `agent/cli_view.py`, `shared/config_loader.py`
2. Step 1 (Refactor Loader): Update `ConfigLoader` to scan the config directory for all designated sub-configuration files rather than a hardcoded list of two files.
3. Step 2 (Implement Precedence): Implement dictionary merging that guarantees specific override priority hierarchies.
4. Step 3 (Command Integration): Bind this complete loader to the execution block of the `/reload` CLI router command.

## Validation and Testing Plan
- Command: `pytest tests/shared/test_config_hot_reload.py`
- Required Test Cases:
  - `test_reload_includes_security_and_tools`: Modifies values in `security.toml` and validates that executing `/reload` makes the new security parameters accessible in runtime.
  - `test_configuration_shadowing_precedence`: Verifies that values in `http.toml` correctly overwrite conflicting fields present inside `common.toml`.

## Constraints and Core Rules
- Fail-fast immediately if any configuration file contains syntax errors or schema violations during the live reload phase.
