# [High] Remove GitHub `github_` prefix fallback routing

## Priority
High

## Background
GitHub tools are currently routable through a `github_` prefix fallback. This creates a special routing exception that bypasses explicit ownership registration in `ToolRegistry`.

## Goal
Remove GitHub prefix-based fallback routing and require GitHub tools to be registered explicitly.

## Scope
Affected files:
- `shared/route_resolver.py`
- `shared/tool_registry.py`
- `shared/tool_constants.py`
- `tests/test_route_resolver.py`
- `docs/04_mcp_03_routing_lifecycle_and_execution.md`
- `docs/04_mcp_06_configuration_and_operations.md`

## Implementation Tasks
- Remove `_GITHUB_PREFIX = "github_"` from `shared/route_resolver.py`.
- Remove all prefix-match routing logic for GitHub tools.
- Explicitly register all GitHub MCP tools in `ToolRegistry`.
- Ensure GitHub tools resolve only through:
  - live `/v1/tools` discovery
  - `ToolRegistry`
- Replace prefix-routing tests with registry-based GitHub routing tests.
- Remove GitHub prefix exception documentation.

## Acceptance Criteria
- `github_*` tool names are not routed by prefix matching.
- Every GitHub tool that should be callable is explicitly registered.
- A GitHub tool missing from both discovery and registry fails with a routing error.
- Active docs no longer mention GitHub prefix routing as an exception.

## Validation
- Run `pytest tests/test_route_resolver.py -v`.
- Run `rg -n "_GITHUB_PREFIX|github_ prefix|prefix exception|github_\\*" shared docs tests`.
- Active routing references to prefix fallback must be gone.
