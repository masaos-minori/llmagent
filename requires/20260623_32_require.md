# Title
Replace `dict[str, Any]` in `models.py` with typed sub-structs

## Summary
Refactor `scripts/agent/commands/models.py` to replace generic `dict[str, Any]` fields with dedicated typed DTOs.

## Background
The current plan identifies `StatsViewModel.latency` and `ToolResultView.args_masked` as weakly typed.
The repository-wide command-layer policy explicitly requires removing `dict[str, Any]` at boundary crossings and replacing them with typed DTOs.

## Problem
Generic dictionaries weaken type safety, reduce readability, and make downstream validation harder.

## Required Changes
- Add `LatencySnapshot` as an immutable typed structure.
- Add `MaskedArgs` as an immutable typed wrapper for masked arguments.
- Replace:
  - `latency: dict[str, Any] | None` → `latency: LatencySnapshot | None`
  - `args_masked: dict[str, Any]` → `args_masked: MaskedArgs`
- Remove unused `Any` imports if they are no longer needed.

## Acceptance Criteria
- No `dict[str, Any]` remains in the targeted model fields.
- The new DTOs are explicit, typed, and immutable or read-only as intended.
- The affected view models compile cleanly under strict typing.

## Out of Scope
- Refactoring unrelated command modules
- Changing presentation behavior beyond typing cleanup

## Components
- `scripts/agent/commands/models.py`
