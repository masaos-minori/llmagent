# Review and remove MCP installer module chain

## Priority
Medium

## Labels
mcp, cleanup, dead-code, installer

## Dependencies
- Remove or isolate `agent/services/mcp_install.py`
- Remove unused MCP installer validation helpers if no longer referenced

## Background
`McpInstallService.run()` lazily imports several MCP installer modules to generate scaffold files and snippets. After `/mcp install` removal, these modules may become dead code.

## Task
Review the MCP installer module chain and remove modules that are only used by the removed installer workflow.

## Implementation Requirements
- Search for references to:
  - `mcp.installer_port`
  - `mcp.installer_templates`
  - `mcp.installer_writer`
  - `mcp.installer_validation`
- If these modules are only used by `McpInstallService`, delete them.
- Remove associated templates or generated-snippet helpers if they are no longer used.
- Remove related tests if they only validate the removed installer workflow.

## Acceptance Criteria
- No dead installer modules remain in the Agent/MCP runtime path.
- Repository-wide search confirms installer modules are either removed or owned by a non-REPL developer workflow.
- Lint/type checks pass.

## Source Context
`McpInstallService.run()` imports `installer_templates`, `installer_validation`, and `installer_writer` to generate MCP scaffold files and configuration snippets.
