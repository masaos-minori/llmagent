# Define a timestamp format policy for SQLite schemas

## Priority
Low

## Labels
- db
- schema
- cleanup
- documentation
- low-priority

## Dependencies
None

## Background
SQLite schema definitions currently use a mix of `datetime('now')` and ISO-8601 UTC-style `strftime('%Y-%m-%dT%H:%M:%SZ', 'now')`.

A consistent policy should be documented before changing existing schemas.

## Scope
Review:
- `db/schema_sql.py`
- DB documentation

Optional code changes should be deferred unless the policy is approved.

## Required Changes
- Decide whether new timestamps should use ISO-8601 UTC.
- Document the chosen timestamp format policy.
- Identify tables currently using each format.
- Do not perform broad migration in this issue unless explicitly approved.

## Acceptance Criteria
- Timestamp format policy is documented.
- Existing mixed formats are listed.
- Future schema additions have a clear convention.
