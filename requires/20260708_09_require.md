# H-9: Remove or fully orphan ToolResultStore and tool_results DB access

## Priority

High

## Background

`ToolResultStore` is the SQLite-backed full tool output archive. It provides `store()`, `get()`, `list_recent()`, and `mark_turn_undone()` over the `tool_results` table.

This archive is being removed.

## Target files

- `db/tool_results.py`
- `db/models.py`
- `agent/commands/models.py`
- any imports of `ToolResultStore`, `ToolResultRow`, or `ToolResultView`

## Implementation tasks

- Remove or fully orphan `db/tool_results.py`.
- Remove all imports of `ToolResultStore`.
- Remove all imports of `ToolResultRow`.
- Remove `ToolResultRow` from `db.models` if no longer used.
- Remove `ToolResultView` from `agent.commands.models`.
- Remove `/tool`-specific DTO usage.
- Verify no runtime code performs INSERT, SELECT, or UPDATE against `tool_results`.

## Acceptance criteria

- Runtime code no longer imports `db.tool_results`.
- Runtime code no longer imports `ToolResultRow`.
- Runtime code no longer imports `ToolResultView`.
- No runtime SQL references `tool_results`.
- Existing non-tool-result DTOs remain intact.

## Test plan

- Static grep for `ToolResultStore`.
- Static grep for `ToolResultRow`.
- Static grep for `ToolResultView`.
- Static grep for SQL references to `tool_results`.
- Run command registry initialization.
- Run normal tool execution.
