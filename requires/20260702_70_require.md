# L-3: Generate `/help` from canonical command definitions

## Priority

Low

## Labels

cli, help, docs, maintainability

## Background

`_cmd_help()` prints slash command help by iterating over `_COMMANDS`.

After command definitions are consolidated into `command_defs.py`, `/help` should use only that canonical list.

This prevents drift between dispatch behavior and help output.

## Goal

Ensure `/help` is generated from the canonical built-in command definition list.

## Scope

### In scope

- Make `_cmd_help()` use `_COMMANDS` imported from `command_defs.py`
- Preserve existing help output format
- Optionally include subcommand metadata when available

### Out of scope

- Full redesign of the help UI
- `/exit` handling
- plugin command help display redesign

## Acceptance criteria

- `/help` displays commands from the canonical `_COMMANDS`
- Commands added to `command_defs.py` appear in `/help`
- Commands removed from `command_defs.py` disappear from `/help`
- No separate hard-coded command list is used by `/help`

## Validation

Run:

- `uv run python -m agent`
- `/help`

Expected result:

- Help output reflects `command_defs.py`
- `/approve`, `/reject`, `/db`, `/mdq`, and `/plugin` are listed if present in the canonical command definitions

## Affected files

- `scripts/agent/commands/registry.py`
- `scripts/agent/commands/command_defs.py`

## Dependencies

Depends on:

- H-3
